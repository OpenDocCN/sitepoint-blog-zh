# 辩论——所见即所得:网页设计师的梦想

> 原文:[https://www.sitepoint.com/wysiwyg-web-designers-dream/](https://www.sitepoint.com/wysiwyg-web-designers-dream/)

截止日期。

这些是大多数 Web 开发人员生活中的祸根，通常，随着项目变得越来越复杂，任何提高工作速度和生产率的方法都会大受欢迎！

简单来说:手工编码很挑剔。当你手工编码时，很容易忘记按“tab”——当你到达代码末尾时，你会发现每一行都不正常！在一个好的所见即所得编辑器的帮助下，你可以很容易地避免这种手工编码的灾难。

##### 基础知识

所见即所得编辑器，如 Dreamweaver，允许用户创建他们的网站作为一个全过程。例如，您单击“插入图像”并选择一个图像，Dreamweaver 只需在后台插入代码。手工编写 HTML 代码来实现同样的结果将花费更长的时间(在大多数情况下),并且您很容易打印错一个单词，或者漏掉一个引号，这将需要稍后进行冗长的代码调试，并且花费您更多的时间。

同样，插入表格、图像或一些复杂的 JavaScript 也非常容易。这些任务也是通过点击一个按钮来完成的，而不是费力地手工输入简单的代码。你的 WYSIWYG 编辑器整洁地跟踪所有这些标签，并防止语法错误的可能性。许多编辑器甚至可以通知您预先存在的代码中可能存在的任何错误——它们自动清理 HTML，找到不需要的标签，甚至包括拼写检查器(这是许多人，包括我自己，经常需要的)!).

最终产品是标签式的、一致的代码——当您需要添加到现有代码中，但手头没有 WYSIWYG 编辑器时，或者当服务器端代码(实际上无法通过这种编辑器实现)需要合并时，这是天赐之物。

##### 设计师的梦想

许多设计师就是这样:例如，他们绝对是 Photoshop 方面的奇才。但是当涉及到编码他们的设计时，他们就迷失了！通常更容易拖延和浪费时间，而不是面对解决在哪里放置表格以及如何用 HTML 代码将它们有吸引力的布局放在一起的任务。

简单地在编辑器中打开你的站点，并在浏览器中查看布局，这真的很方便——特别是对于那些很难想象所有这些`<tr>`和`<td>`标签实际上是如何创建布局的人来说！这就是为什么如此多的设计师发现剪切他们的图像，启动所见即所得编辑器，并简单地将它们插入到一个新的 HTML 文档中更容易。他们的模板基本上在 5 分钟左右就可以使用了。它们还提供了一系列其他优势。

***增强网站***

WYSIWYG 编辑器还可以非常容易地将动态和模糊的元素添加到您的站点中！想要一个 JavaScript 滚动菜单吗？这就像点击一个按钮一样简单——一点也不麻烦！您可以使用相同的方法添加表单和 CSS，更重要的是，一些编辑器甚至允许您添加功能全面的服务器端代码。许多人不愿意花无数的时间去试图弄清楚他们的代码实际上是如何工作的，这也不足为奇。既然已经为您完成并打包成一个简单的程序，为什么还要这样做呢？

***维护站点结构***

使用 WYSIWYG 编辑器的真正优势是，所有的页面和元素都可以在一个“树”视图中实时查看，因此您可以真正感受到您的网站的布局方式——更不用说轻松地仔细检查哪些页面链接到了哪些页面了！只需按下一个按钮，你就可以更新整个网站的链接，或者在整个网站上进行搜索和替换。

***团队互动***

许多 WYSIWYG 编辑器也提供了一个很好的与他人互动的工具。如果一个团队中从事某个项目的所有设计人员都使用同一个编辑器，您可以很容易地将网站转给其他人，让他们输入内容并进行修改。

##### 用户友好的选择

这些编辑器允许设计者使用模板，对于不太了解的用户来说，它们是在家用电脑上编辑文件的一种快速简单的方法。许多客户现在也想以某种方式编辑他们的网站，除非他们使用复杂的内容管理系统，否则这是非常困难的。通常 WYSIWYG 编辑器将是最直接和最省事的方法，通过它他们可以编辑内容，而不必费力地通过数百行代码。

总而言之，WYSIWYG 编辑器可以让你快速修改其他人的代码(有时是混乱的)，并且用最少的麻烦和技术知识快速创建和管理你自己的站点。这是许多设计师信赖的工具——原因显而易见。手工编码可能更有声望，但是为了简单性、灵活性和易用性，有时你就是不能越过 WYSIWYG 编辑器。
