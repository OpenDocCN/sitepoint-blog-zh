# @media 2008:探索另类网络

> 原文：<https://www.sitepoint.com/media2008-day-one/>

@media 2008 会议在南岸中心举行，对于任何来伦敦的游客来说，这都是一个绝佳的地点。我们喜欢早上沿着泰晤士河漫步，经过伦敦眼，从我们的酒店到比赛场地。

开幕主题演讲来自 Jeffrey Veen，我很高兴今年能听到他的演讲，因为两年前我错过了，因为我在为小组做最后的准备。Veen 是一个很有成就的演讲者，听起来总是很愉快，我觉得我从他关于“通过数据设计我们的方式”的演讲(即使作为一个非设计师)中学到了很多。Veen 谈到了设计师如何理解裸数据，添加元数据来帮助描述数据，然后使用设计来使数据可访问和可读。他警告不要试图添加装饰，他称之为“图表垃圾”，要求我们“讲述”数据中的故事，而不仅仅是让它看起来漂亮。

在这之后，会议分成了两条轨道，我选择了“例如:BBC 主页&清除左侧”。这是由汤姆·卡特赖特和克莱尔·罗伯茨开始的两个小型演示，他们向我们介绍了 BBC 主页的重新设计及其背后的过程。我们了解到 BBC 的开发人员有一项非常艰巨的工作，要用唯一可能的服务器端处理来构建一个现代化的主页，那就是 Apache mod_include。新的主页建立在这个框架之上。他们为主页变宽的决定辩护(它是固定宽度的，需要 1024×768 的分辨率),说他们对主页变窄的抱怨和对主页变宽的抱怨一样多。BBC 网站的新版本将是使用 Zend 框架的 PHP 有一个关于为什么选择 Zend 的问题，但不幸的是他们无法回答，我也很想知道。

BBC 的后续报道是来自 [Clear Left](http://www.clearleft.com) 的 James Box 谈论他们新推出的网络应用 [Eden Bee](http://www.edenbee.com) 。James 是一名信息架构师，他从信息的角度讲述了设计过程。发现其他人工作背后的思维过程总是很有趣，所以我真的很喜欢这个环节——如果你有兴趣发现自己的“碳足迹”并致力于减少它的小步骤，那么 Eden Bee 是值得一试的。

午饭前，我呆在小房间里，听詹姆斯·格拉汉姆和拉克伦·亨特给我们讲述 HTML5。他们做了一个演示，试图解释 HTML 5 背后的理性，并展示了一些具体的例子。对于任何一直关注 HTML 5 发展的人来说，这并不是突破性的东西，但是对于不知道正在提出什么的人来说，这是一个坚实的介绍。最后，他们鼓励观众加入社区，加入邮件列表或论坛，或者阅读博客和维基。

午饭后，我去支持德鲁·麦克勒朗，他的演讲主题是“没有杀戮的内容管理”。德鲁是 edgeofmyseat.com[的另一半，他谈论了一般的内容管理系统，选择或指定内容管理系统时需要寻找的功能，还谈论了我们在 edgeofmyseat.com 内部开发的定制内容管理系统以及我们在构建它时做出的一些决定。](http://www.edgeofmyseat.com)

Drew 之后是 SitePoint 的作者 Stuart Langridge，他主持了一个关于 HTTP 响应代码的幽默会议。对于@media 受众来说，这是一个相当技术性的话题，显然是偏向于设计者而不是开发者。对于那些选择了更多技术路线而不是丹·鲁宾在主厅演讲的人来说，这里有一些有趣的东西。

当天的最后一场会议是“热门话题”小组讨论会，杰弗里·维恩、安迪·克拉克、印第·扬、布朗温·琼斯和丹·鲁宾回答了与会者提出的问题。小组讨论的主题包括如何从客户那里获得副本，以及小组成员对“Web 2.0”的看法。

总结第一天，这真是愉快的一天。我发现我花了一天时间在演示的“另类轨道”上，这比设计师关注的主轨道更具技术性。这确实意味着我错过了 Indi Young、Bronwyn Jones、安迪·克拉克和丹·鲁宾的演讲，从与参加过这些会议的人的交谈中，听起来好像他们都对人们真正有益。一旦有了更可靠的网络连接，我会添加今天演讲者发布的幻灯片的链接，以便稍后查看。我也把今天会议的一些照片放在了 Flickr 上。
