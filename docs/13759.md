# 更多 PHP 框架

> 原文:[https://www.sitepoint.com/more-php-frameworks/](https://www.sitepoint.com/more-php-frameworks/)

是的，我知道——随着[框架](http://framework.zend.com)的进展，我们不再需要那些讨厌的框架了。除了我最近看到的几个值得注意的例子。我没有深入研究过源代码——探索框架代码基础已经变得像看着油漆变干一样有趣，但是如果你想从新的角度来看这个问题，这里有*和*一些有趣的想法。

首先是[保罗·詹姆斯](http://www.peej.co.uk/)的 [Tonic](http://tonic.sourceforge.net/) ，它将自己标榜为“一个 RESTful Web 应用开发框架”。这很有趣，因为你写的控制器的 API 很像我在这里的[之前介绍过的](https://www.sitepoint.com/a-simple-wiki-with-webpy/) [web.py](http://webpy.org/) (实际上 tonic 是第一个)。一些有趣的想法——这是极少数关注 HTTP 的想法之一。你需要把它从 CVS BTW 中取出来(目前没有下载)。

在今天的框架嘉年华中，第二个是[代码点火器](http://www.codeigniter.com/)(感谢马克·巴顿的提示)——这是由组装 [pMachine](http://www.pmachine.com) 的同一批人提供的，因此认为它值得注意。从表面上看，这很像 Rails，但从 KISS 的角度来看，这很有吸引力，而且似乎有一种巧妙的 URL 映射方法。

任何——就是这样——对我来说再也不会有另一个框架了。如果你很想浪费时间，试试这里的[或者这里的](http://www.phpwact.org/php/mvc_frameworks)。否则，一个 [PHP 框架](http://framework.zend.com)来统治它们。
