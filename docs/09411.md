# 为云爆发播种

> 原文：<https://www.sitepoint.com/seeding-for-cloud-bursts-2/>

任何企业的圣杯是与顾客建立持久的关系。这种关系被设计得如此牢固，以确保顾客会因为每一个可能的需求而不断回来。零售商，被定义为向顾客提供成品的任何企业，尤其被驱使建立这些粘性关系，因为顾客通常可以相对容易地将其业务转移给竞争对手。

在我之前的文章中，我介绍了*云爆发*的概念，将社交媒体网络与现实世界的商业运作联系起来。我在这里对这个定义进行了提炼，阐明了云爆发是一种营销活动，它将社交媒体营销与传统的实体零售业务相结合，以推动销售或加强客户关系。简而言之，*云激发社交网络在零售商的实体门店开展业务，并加强关系*。

使用当前在云和移动环境中可用的工具，有许多方式可以实施云爆发营销活动。可以使用这些工具设计流程，将数据汇集在一起以管理云爆发。但是，对于云爆发解决方案提供商来说，存在着真正的有利可图的机会来简化和打包这些功能，为零售商提供一站式解决方案，将他们的客户与物理商店联系起来。

为了进一步描述零售商和解决方案提供商的机会，我在这篇文章中展示了一个示例用户故事，该用户故事定义了希望利用云爆发活动的快餐车的需求。我将保持用户故事解决方案不可知，除了识别脸书或 Twitter。我将它们作为社交媒体和营销管理员挖掘客户和建立名单的来源。避免使用现有产品有望防止我限制您关于如何支持云爆发活动的想法。

作为对我的帮助，我请求您提供反馈，并确定可用于支持所呈现的云爆发场景的具体产品或流程。如果您是一个工具的开发人员或产品经理，该工具可以支持场景的任何部分，请随时回复。我会在以后的帖子中回复您进一步的信息和讨论。

# 云爆发活动组件

在介绍用户故事之前，让我先回顾并定义云爆发营销活动成功所需的组件。尽管所有营销活动都必须具备许多关键要素，但以下这些要素必须明确定义，以便云爆发能够触及适当的客户并推动他们在商店的行为:

*   活动目标
*   目标市场
*   客户利益
*   通信电路

## 活动目标

每个云爆发营销活动都必须有一个特定的目标或一组目标。无论营销活动是为了推动销售点的销售，还是为了与客户建立联系并向他们介绍新的销售点，都必须定义您的目标客户数量以及您希望成功联系的客户数量。这是分析持续改进活动结果的基础。

## 目标市场

必须定义活动中包含的特定客户类型。这决定了用于“播云爆发”的数据源。你想从脸书这样的现有社交网络中吸引顾客吗？或者，您是否希望从现有的零售忠诚度计划名单中获得该名单？

## 客户利益

具体定义客户参与云爆发所获得的好处。无论这种利益是购买折扣、获得游戏武器或奖励的特殊途径，还是其他什么；突出优势，确保客户将优势与社交网络和零售商联系起来，以强化这种关系的价值。

## 通信电路

必须定义接触客户的沟通渠道，因为它将决定将客户与零售商联系起来的社交网络工具和相关技术。如果你试图通过专有渠道，如脸书消息或 Twitter 帖子来接触你的客户目标，那么不用说，你的所有目标市场都必须是该渠道的参与者。

# 现在是一个用户故事

一个流动的快餐车为那些喜欢他们的粉丝们维护了一个脸书页面和一个 Twitter 账户来交流每天的菜单和特色菜。假设大量人群吸引了更多的顾客，并希望在新地点开业一天后增加销售额，那么 cloud burst 活动的目标是在开业的最初几个小时增加销售额。

他们希望吸引当地的顾客，并奖励他们提前购买午餐，将原本缓慢的时间变成繁忙而有利可图的一小时。同样的逻辑可以用于在繁忙的午餐时间后推动卡车销售，从而延长午餐销售期。

为了播种云爆发列表，卡车的社交媒体管理员和营销大师从以下来源创建了潜在客户和细分市场列表:

1.  喜欢他们页面的脸书用户。
2.  推特粉丝。
3.  当天在销售点注册并提供电子邮件和联系信息以换取折扣的客户。
4.  在邻近地区旅行的客户使用已在云解决方案上注册的移动设备。

管理员为每个客户群定义一个在特定时间段内有效的唯一折扣代码。使用基于云的工具，管理员在前一天晚上加载前三个客户群的联系信息。他们为每个细分市场定制营销信息，并针对所使用的特定沟通渠道(如 Twitter、短信)进行格式化。该消息计划通过云解决方案发送给每个客户群。

云解决方案不断扫描在规定范围内的移动设备，这些设备在地理上足够近，可以在卡车上购买午餐。当设备进入目标范围时，特殊折扣代码被发送到那些客户设备。这些客户属于第四类客户。

当顾客来到卡车旁时，他们可以选择通过移动设备点餐，以绕过长长的队伍。考虑到他们作为忠实顾客的精英身份，他们还可以通过移动设备使用信用卡支付食物，并在单独的窗口取食物。

客户可以选择通过云解决方案在用户定义的组下注册，例如“Acme Corporation，Accounting”这使得用户可以将自己与更大的常客社区(更大的细分市场)联系起来，并有资格获得其他特殊折扣。通过增加与客户的接触点，你与客户的关系越紧密，他们就越忠诚。管理员可以保存这些大型社区的位置，并使用该信息来安排未来的卡车位置，因为他们知道他们的许多客户每天都在哪里。

云解决方案跟踪云爆发活动期间客户的所有活动和购买的产品。生成分析报告，显示每个客户群对产品销售的影响以及通过每个渠道成功接触客户的情况。

尽管网络上或应用程序中有许多优秀的工具和解决方案，但是快餐车能选择一个完整的服务来满足用户故事中定义的所有需求吗？我对此表示怀疑，但这正是那些准备搬家的人的机会所在。
