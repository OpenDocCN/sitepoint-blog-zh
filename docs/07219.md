# 响应性网页设计:响应性地使用字体

> 原文:[https://www . site point . com/understanding-responsive-web-design-how-to-manage-fonts/](https://www.sitepoint.com/understanding-responsive-web-design-how-to-manage-fonts/)

字体设计是响应式网页设计最重要的方面之一，如果你想让你的内容在各种屏幕尺寸下都能被接受，为移动设备优化字体是绝对必要的。幸运的是，构建灵活字体的过程并不十分困难。当我们谈到灵活性(这是本案例的指导原则)时，我们不能忽视我们为响应性网站选择的特定字体大小。为此，我们可能会使用不同的指标，包括像素、ems、rem 或百分比。选择正确的指标对于设计可延展的、响应性强的界面至关重要。

在这个关于响应式网页设计的延续中，我将解释如何响应性地管理动态字体，并且我将分析前面提到的所有指标，比较每个选择的优缺点。

### 像素

长期以来，像素因其固有的精确性和准确性而成为网页设计者的首选单位。一旦为字体选择了像素值，字体大小将在所有设备和浏览器中保持不变。虽然这种方法提供了非常精确的控制，但它违背了我们对灵活性和响应能力的需求。

正如大多数读者已经知道的那样，当您为“父”项设置维度时，它们将通过继承自动应用于任何“子”元素。(这就是为什么他们把 CSS 命名为"*层叠*样式表。)例如，将主体的字体大小设置为固定的像素值将渗透到设计中的几乎所有其他元素。当开发人员想要为页面的每个元素分配不同的样式声明时，他或她必须手动设置不同的参数来覆盖现有的样式。

因此，基本的编辑和调整需要对所有先前的设置进行细致的修改。这远不止是不方便；它会在各种浏览器和设备中导致严重的可访问性问题，其中许多会干扰字体大小的选择。一些访问者可能在阅读低于一定尺寸的文本时有困难，这是绝对应该避免的，特别是考虑到我们的主要目标是做出一个响应性的设计，适用于所有的屏幕尺寸和眼球。

如果你对你的设计采取一种前瞻性的方法，将固定的像素值指定为你的字体大小似乎不是最好的选择。今天，设备有不同的屏幕尺寸，以及不同的像素密度。为这一庞大的产品系列提供一种通用的字体大小与我们的响应方法背道而驰。

### 邮政快递

让我们继续考虑 ems。一个 *em* 相当于 CSS 规则 *font-size* 中定义的大小(以像素为单位)。例如，如果我们构建一个`<div>`并包含字体大小为 16 像素的文本，1 em 将对应于 16 像素，2 em 对应于 32 像素，依此类推。em 在所有浏览器中都是可调整大小的，没有必要为每个元素设置值，因为在这种情况下，规则可以应用于“父”元素，并通过级联过程自动应用于其各自的“子”元素。

显然，使用 ems 有利弊要考虑。虽然它们使站点类型的维护变得更容易(因为，如前所述，所有的更改将自动应用于整个内容)，但这可能会阻碍开发人员对字体的控制、精确性和可预测性的追求。幸运的是，通过对大部分内容使用相同的度量单位，只缩放一些文本元素，如`header`或`footer`，这个问题可以很容易地解决。

### 百分率

就像 ems 一样，百分比是字体大小的可调整单位，它们的 CSS 声明可以被继承。理论上，百分比和 ems 之间没有太大的区别；它们都是可延展的测量单位。理解 ems 和百分比之间的差异(无论它们有多小)同样重要的是决定你在整个设计中使用哪种测量单位。你当然不希望对你设计的某些元素使用 ems，而对其他元素使用百分比。排除百分比的一种方法是简单地进行以下站点范围的 CSS 声明:

```
{ font-size:100%; }
```

这有效地覆盖了任何浏览器默认值或其他不需要的百分比声明，并将字体大小留给其他方法和度量。在这种情况下，可以使用 ems 来调整内容的大小。

### Rems

有些人可能不知道，还有另一个选项可以提供字体大小的灵活性:rem 单位(“root Em”)。它的行为类似于前面提到的 em 单元，但有一个重要的区别:它的值是相对于文档的根元素(在 HTML 中，就是`<html>`元素)的，而不是相对于与之关联的任何其他元素的。这一重要区别使得解决嵌套元素出现的许多问题成为可能。而且，正如大多数设计师所知，他们的绝大多数设计本质上都包含嵌套元素；像人类一样，几乎每个人都有一个“父母”，并从他们那里继承了某些品质。

然而，需要注意的是，rem 是一个非常新的单位，因此并不是所有的桌面浏览器都支持、解释或正确显示它们。至少目前是这样。

### 结论

探索了字体的各种度量选项后，在响应式网页设计中，接近字体的最佳方式是什么？

当然，ems 使字体缩放更容易，网站维护也更容易。例如，如果您决定使用百分比来增加字体大小，您只需更改应用于正文的百分比，其他什么都不用做。维护实际上是一个大问题，值得在设计中认真考虑。忽略日常事务，比如易维护性和样式表的简单性，把注意力完全集中在美观性和功能性上是很好的。但是，在蓝天计划过程之外，一个好的设计师会给予实用性和实用性重要的考虑和优先权。

首先，你必须将所有固定大小的元素转换成能够保证更高流动性的东西。第二，你应该始终记住页面元素的比例，并仔细修改以保持这些比例。在这个系列的后面，我们会看到一些例子，我们在这些文章中描述的响应式网页设计的所有原则的实际应用。首先理解这些技术的基本方法是很重要的。在下一篇文章中，我们将继续讨论流体布局，以及如何在同一页面中结合固定宽度和流体宽度。
