# Rackspace 加大了攻击亚马逊的赌注

> 原文：<https://www.sitepoint.com/rackspace-ups-the-ante-in-assault-on-amazon/>

虚拟主机提供商 [Rackspace](http://www.rackspace.com/) 今天宣布，它已经收购了两家初创公司，以加强其云主机服务，从而应对亚马逊[的网络服务](http://aws.amazon.com/)栈。总部位于德克萨斯州圣安东尼奥的 Rackspace 支付了大约 1150 万美元的现金和股票(还有可能支付高达 1650 万美元的额外现金和股票)，以收购两家公司:VPS 提供商 [Slicehost](http://www.slicehost.com/) 和 [Jungle Disk](http://www.jungledisk.com/) ，这是一款使用亚马逊 S3 的在线存储应用程序。

Rackspace 的云托管部门 [Mosso](http://www.mosso.com/) 也宣布对其旗舰产品进行品牌重塑。托管云是一个可扩展的 web 托管环境，现在将被称为云网站，与亚马逊的 S3 竞争的文件存储产品 CloudFS 将被重命名为云文件。Rackspace 计划在今年晚些时候通过其云文件服务提供 CDN 服务，作为与 limited Networks 合作的一部分。

此外，Rackspace 宣布推出云服务器，这是一种按需服务器服务，将使用 Slicehost 的配置技术，该技术使用 Xen 虚拟化软件。

两家被收购的公司将继续独立运营，Jungle Disk 将继续向客户提供亚马逊 S3 作为存储选项。收购丛林磁盘似乎有点奇怪，因为这是一个消费者游戏，似乎不符合 Rackspace 的核心业务，除了作为他们的 Mosso 服务的展示品。另一方面，Slicehost 很有道理。

Slicehost 已经发展成为按需虚拟化服务器的主要参与者之一，并声称今天拥有超过 15，000 个在线“切片”。它们非常受 web 应用程序开发人员的欢迎，尤其是那些使用 Ruby on Rails 框架的开发人员(例如，我的站点 Rails 论坛的四分之三的创始人都有自己的部分)。这正是 Rackspace 希望从亚马逊吸引过来的早期采用者、网络应用开发者群体。

Slicehost 和 T2 丛林磁盘都有关于收购的常见问题。

*披露:Rackspace 和亚马逊都向 SitePoint 提供托管服务。*
