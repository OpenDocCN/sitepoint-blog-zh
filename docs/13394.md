# Silverlight 初具规模，MS 在 MIX’07 上获得活力

> 原文:[https://www . site point . com/Silverlight-takes-shape-ms-gets-dynamic-at-mix-07/](https://www.sitepoint.com/silverlight-takes-shape-ms-gets-dynamic-at-mix-07/)

如果你错过了这场喧嚣，微软的营销机器已经进入富媒体超速运转。首先，在 NAB，他们给 WPF/E 取了一个合理的非极客名字:Silverlight。Mix 07 的主题演讲是一系列令人印象深刻的演示——都可以在 [Mix 网站](http://www.visitmix.com)上在线获得——展示了 Silverlight 的 alpha 位可以做什么。真正令人兴奋的部分是完全的 CLR 集成，允许在客户端进行一些非常繁重的工作。NET 运行时。集成碰巧运行良好，并且可以从 Mac 上调试。如果我负责 Flex 团队，我会开始担心，非常担心。

另一个重大消息是，微软已经创建了动态语言运行时(Dynamic Language Runtime)层(位于 CLR 之上)，以启动动态语言。NET 语言开发。他们甚至开发了一些工具——比如 Jasper，一个 RAD 环境——需要使用支持后期绑定的语言。

我想。随着这些测试版的上线，NET 开发在未来的一年将会变得更加有趣。动态语言应该有助于使低端的东西更容易接近，像实体框架和 LINQ 这样的东西将在低端创造一些非常酷的可能性。

不幸的是，我明天要去拉斯维加斯，但不是去“正确的”会议。我会离开几个星期，但是这个月晚些时候应该会回来。

PS:我是用 Expression web 写的这个帖子。我要说的是，就目前而言，Dreamweaver 作为一个手工编码工具没有什么可担心的。而 Flash 就不那么安全了。人们可能会认为 Adobe 在其中加入了智能感知。
