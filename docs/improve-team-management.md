# 电子项目——改善团队管理

> 原文：<https://www.sitepoint.com/improve-team-management/>

有一段时间，我一直在寻找一个好的项目管理和协作系统，但毫无结果。Macromedia 最近推出的 [SiteSpring](http://macromedia.com/software/sitespring/) 引起了轩然大波，我想我应该亲自尝试一下。但是这并没有持续多久，因为我没有合适的操作系统来满足 SiteSpring 的要求。幸运的是，这只是一个小挫折，因为我的搜索很快就把我带到了[电子项目](http://www.eproject.com/)。

我很快发现，华盛顿州西雅图的 eProject Inc .有两个很棒的管理和合作软件版本: [eProject Express](http://www.eproject.com/ProductsExpress.asp) 供初学者和想测试服务的人使用，以及 [eProject Enterprise](http://www.eproject.com/ProductsEnterprise.asp) 供有成就的专业人士使用。

##### 电子项目快递

我最近注册了免费速成版，到目前为止我对它只有好话可说。这是 eProject 的介绍版，对于小型团队来说是一个很好的项目管理工具。除其他功能外，它还允许您:

*   互相发送信息，
*   张贴文件和文档(高达 50MB)，
*   显示任务及其状态，
*   在日历上张贴重要的日期，以及
*   按组组织团队成员。

简而言之，小团队需要的一切。另外，它还提供 vCalendar 和 vCard 支持。

这个版本的 eProject 最好的一点是它的工作方式和外观非常像微软的 Outlook(这并不奇怪，微软是它的主要合作伙伴)，唯一的区别是它面向团队管理。这种相似性大大降低了学习曲线。我对免费版的唯一抱怨是，它的强大功能有点匮乏，但这正是企业版的用途！

##### 电子工程企业

除了在视觉上更具吸引力，企业版还拥有比免费版更强大的功能——它将 eProject Express 带到了一个更高的水平。它还包含附加功能，允许用户:

*   举行讨论，
*   有民调，
*   存储您的团队和个人书签，
*   查看任务的时间表视图(非常类似于 MS Project)，
*   管理团队问题，
*   完成临时报告，
*   通过您的 Palm 或 WAP 手机连接到 eProject，

…以及更多！

eProject 在其收费模型中还包括了其他三个漂亮的特性，第一个是“执行仪表板”。在这里，公司领导可以快速获得所有项目的快照，按状态、预算、小组和项目经理进行筛选。

第二个特点源于 eProject 与微软的合作关系。最新版本的企业版(目前为 3.6)提供了与 Microsoft Project 的扩展集成。项目经理现在可以在 Microsoft Project 中维护他们的日程安排，同时使用 eProject Enterprise 向团队成员请求状态更新。

第三个特性是我最喜欢的一个，因为它解决了我的团队(和大多数团队)经常遇到的一个问题:eProject Enterprise 提供了改进的文档管理，包括版本控制、检入/检出状态、版本历史和注释。

与 Express 相比，Enterprise 还允许更大程度的定制。您的能力几乎是无限的:添加您的公司徽标来修改用户界面，将企业集成到您的公司门户，您甚至可以添加自定义功能！

##### 定价和底线

就像 Express 一样，Enterprise 的外观和功能与 MS Outlook 和 MS Project 非常相似，但同样，它面向团队环境。这种型号的许可证面向 25、50、75、100、250、500 和 1000 多名用户，根据您选择的产品包，一个完整的现场系统(包括运行软件的计算机)的价格最高可达 17，023 美元。虽然这听起来很贵，但与 SiteSpring 的价格相比还是可以接受的——从 3 个用户的 1999 美元到只有 10 个用户的 7499 美元。

与 eProject Express 相比，eProject Enterprise 更像是 Macromedia SiteSpring 的竞争对手，但尽管如此，对于预算有限的小团队和小组来说，精简版可能是正确的解决方案。

与必须安装在您的服务器上的 SiteSpring 不同，Express 和 Enterprise 都驻留在 eProject 的服务器上，如果您自己的服务器空间有限，这是非常好的。它还极大地减少了设置时间——我只花了五分钟就注册、设置了我的项目，并邀请我的团队成员加入。

eProject 还提供了更好的平台兼容性，因为它是独立于平台的，不像 SiteSpring(需要 Microsoft Windows NT 或 2000 才能运行)。另一方面，如果您想在自己的服务器上托管软件包，您可以选择使用 eProject 的现场版本；但是，为此您需要 Microsoft Windows 2000 Server 和 Microsoft SQL 7.0。

Express 和 Enterprise 的两个托管版本的唯一不足之处是缺乏适当的备份设施。目前，备份项目的唯一方法是使用他们的“光盘存档”订购表。这是一个方便的工具，但是每张 CD 一个项目——每张 CD 的成本为 29.99 美元——这是一个昂贵的工具，尤其是当你通常有多个项目时。

底线:无论你是需要企业的“大炮”，还是 Express 的低成本，如果你需要更好的团队管理和合作，eProject 肯定是值得考虑的。
