# 微软让我们重新爱上 IE 的 10 种方式

> 原文：<https://www.sitepoint.com/10-ways-microsoft-could-make-us-love-ie/>

我要坦白一件事。我喜欢网络浏览器。版本 1 几乎立即消失，版本 2 是基本的，但版本 3 是一个巨大的改进。第 4 版令人震惊，第 5 版、第 5.5 版和第 6.0 版都建立在它的成功之上。网景公司令人沮丧地失败了，Mozilla 的第一套软件也是臃肿不堪。Opera 出现了，但没有提供足够多的功能来证明它的价格。在世纪之交，IE 是这个星球上最好的浏览器。

Mozilla Phoenix 出现在 2002 年末。尽管努力保持对 IE 的忠诚，我还是很快改变了。凤凰最终成为火狐，浏览器是一个启示。与此同时，网络开发者开始质疑微软对网络的承诺。IE6 已经过时了，当我们向网络标准迈进时，恼人的错误出现了。

微软又花了 3 年时间才发布 IE7，坦率地说，这太糟糕了。企业讨厌它，因为它破坏了为 IE6 设计的系统，开发者讨厌它，因为它远远落后于竞争对手的浏览器。IE8 是一个进步，但是随着 IE9 的出现，微软缩小了差距，推出了一款现代浏览器。

不幸的是，IE9 在许多方面仍落后于竞争对手。但是微软有机会帮助我们重新爱上 ie 浏览器。以下是我的建议…

## 1.引入快速发布时间表

我不知道为什么微软采用了两年的浏览器发布时间表，但这太慢了。今天，IE9 支持的 HTML5 技术比 Firefox、Chrome、Safari 和 Opera 更少——2013 年它会有多糟糕？就我个人而言，我不介意 IE9 现在缺少一些 HTML5 功能，如果我知道它们将在几个月后到来的话。

这么说吧:微软可以每 3 年生产一个新的操作系统和办公套件，为什么他们的浏览器要花几乎同样长的时间呢？谷歌可以每 6 分钟(或大约 6 分钟)发布一个新版本的 Chrome 微软当然有资源让 IE 保持最新。

## 2.实施自动更新

从现在开始，微软应该自动更新浏览器。IE6、ie7 和 ie8 在未来几年仍将令人头疼，但没有什么理由认为这种命运会降临到 IE9 身上。

我知道政府和大型企业喜欢在将软件分发给所有用户之前对其进行评估。然而，我确信他们更喜欢每隔几个月就进行一次渐进的改进，而不是每隔几年就进行一次会破坏他们应用程序的大修。

## 3.给 IE 一个漂亮的界面

从历史上看，微软一直努力美化其应用程序。他们可能不是最好的，但他们是最有吸引力的。那么，为什么 IE9 比它的任何竞争对手的浏览器都丑呢？开发者工具尤其令人震惊…

## 4.改进开发人员工具

很多年前，微软意识到，如果开发者不为这个羽翼未丰的操作系统开发应用程序，Windows 就会失败。高质量的编程工具是该公司所做一切的核心。那么为什么 IE 的开发者工具这么差呢？

IE9 开发者工具是从大约 5 年前为 IE6/7 引入的开发者工具栏发展而来的。这是对 Firebug 持续成功的回应。这显然是一个由程序员设计 UI 的仓促项目，然而 IE9 的开发工具几乎没有改变。与 Firebug、webkit Inspector 和 Opera 蜻蜓相比，它又丑又笨重。

## 5.实施更好的附加系统

Web 开发人员了解 web 技术，但 IE 插件的创建者需要是 Windows 桌面程序员，并拥有 Visual Studio。唯一创建 IE 扩展的人是大公司和恶意软件经销商。

没必要这样。Chrome 提供了一个简单的类似 bookmarklet 的附加系统，Opera 呼吁建立一个浏览器扩展标准。微软可能有助于创建一个全行业的系统，吸引第三方开发者进入 IE 平台。

## 6.为 XP 用户做点什么

微软选择忽略我对在 XP 上安装 IE9 版本的请求。我并不感到惊讶，但他们可以对 50%无法安装最新浏览器的 Windows 用户采取一些措施。发布 IE8.1 .分发 Opera。分发免费的 Vista 光盘。

我们将等待 XP 消亡很长一段时间。

(这是一个更长远的尝试，但是如果 IE 也可以在 Mac OS 和 Linux 上使用，开发者的生活会容易得多。)

## 7.别再说什么遗产了

IE9 试图隐藏它的根源，但是 IE7 和 ie8 隐藏在表面之下。是时候抛弃他们的遗产向前迈进了。具体来说，让我们看到旧工具栏和令人困惑的互联网选项对话框的终结。

我也希望看到兼容性观点的终结。以 IE7 为目标的开发者应该让他们的网站在现代浏览器中崩溃！

## 8.记住速度不是一切

IE9 非常快。但这足以阻止用户流向竞争对手的浏览器吗？我表示怀疑。

在写这篇文章的时候，我使用的是 Firefox 3.6——被广泛认为是最慢的主流浏览器。启动速度可能令人恼火，但我很高兴用这些损失的时间换取 Firefox 提供的更多功能。诸如会话恢复和更好的选项卡处理等简单功能带来了很大的不同。

速度是有帮助的，但它不能代替功能。

## 9.停止浏览器 BS

我喜欢 IE9。我非常尊重微软。但是为什么他们会创建像 [*这样的误导性页面，把浏览器比作 Internet Explorer 9*](http://windows.microsoft.com/en-US/internet-explorer/products/ie-9/compare-browsers) ？

任何供应商都可以挑选一些功能，给他们的浏览器打上漂亮的大勾。不幸的是，这个页面给人的印象是微软很难拿出一份像样的清单。在底部，我有点期待看到微软写的**【有一个蓝色的‘e’图标】**【名称中有‘互联网’二字】*。*

 *## 10.带回创新

IE 团队一直在追赶其他供应商。10 年前并非如此:微软推出了浏览器插件、DOM、XMLHttpRequest (Ajax)、XML 工具、字体嵌入和许多 CSS 功能，这些功能刚刚成为 W3C 标准。当然，有一些可疑的技术，但 IE 引领了潮流。

今天，想出新的浏览器创意显然更加困难，但微软拥有比大多数公司更多的 IT R&D 资源。该行业将受益于一点创新和兴奋。

对微软和 IE 团队有什么建议吗？请提出建设性意见！…
