# 震撼搜索

> 原文:[https://www.sitepoint.com/shaking-up-search/](https://www.sitepoint.com/shaking-up-search/)

**人类索引看起来更好**

如果你阅读任何关于搜索引擎的书籍，那么“算法”这个词会很快跃入你的眼帘，也许还会加上一撮 MathML，以确保外行人确实感到眼花缭乱。各种各样的传说和谜团围绕着谷歌的搜索算法。同时，IBM 有这个[UIMA](http://www.research.ibm.com/UIMA/)——真正智能搜索的东西。

但是比较一下这些“PHP SOAP”的搜索结果——[http://del.icio.us/search/?all=php+soap](http://del.icio.us/search/?all=php+soap)和[http://www.google.com/search?q=php+soap](http://www.google.com/search?q=php+soap)。现在，谷歌返回的第一个结果是早已消亡的 [PHP SOAP 工具包](http://phpsoaptoolkit.sourceforge.net/phpsoap/)。与此同时，delicious 首先给了我这个教程: [PHP Web 服务**没有**SOAP](http://www.onlamp.com/pub/a/php/2003/10/30/amazon_rest.html)——如何计算结果的相关性？

换句话说，你的搜索算法需要有多智能才能返回像“胖子瘦”这样的结果？！？你在开玩笑，对吧？破拍都是死婴！”

正如我(嘲讽地)在这里指出的，人类非常擅长建立搜索引擎索引。它们自我维护，丰富，聪明，分布广泛。你的算法有多聪明并不重要——即使你能匹配人类的分类能力，这样做的经济学也会杀死你。

与此同时，[上一个有趣的点](http://unbeknownst.net/)原因[在这里](http://unbeknownst.net/?p=99)；

> 无论如何，博客们开始从 Del.icio.us 获得更多的流量

这并不是说 delicious 即将取代 Google——在我看来，delicious 是书呆子为书呆子开发的工具——在它目前的形式下，我看不出这个概念能接触到非书呆子，直到对一个每周(积极地)只花一个小时上网的人来说,“索引”变得毫不费力。但这不是重点——对我来说，有趣的是，人类构建的指数被证明至少与机器构建的指数一样好——delicious 在这里取得了突破，表明这是可以做到的。

**分发搜索**

另一方面，如果你去搜索“分布式搜索”，你会看到与搜索类似的现象——书呆子们追求算法，以允许巨大的搜索索引被分发。他们被困在如何应对对等端资源缺乏的问题上。

但是我认为类似的“人类黑客”也适用于这里——选择一个数字，我猜想任何给定的人搜索的 90%以上都是关于相同的基本主题——他们感兴趣的东西。大多数时候，个人只需要他们感兴趣的事物的索引。这本身并不能神奇地解决所有问题，但加入了 P2P 类型的“自组织社区”,看起来并不遥远。

总之。只是复述 Web 2.0 已经在说的话。
