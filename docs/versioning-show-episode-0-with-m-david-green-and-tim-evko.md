嗯，术语<q>版本化</q>是指从一个阶段转移到另一个阶段，以及你如何定义处于版本 0、版本 1、版本 1.1 意味着什么。将版本号应用于某个东西的原因是为了让使用它的人了解一点你在你自己的开发过程中的位置，以及你沿着你的轨迹的位置。

我喜欢这样。那真的很好。我也会选择这个作为我的答案。

*【笑声】*

我开玩笑的。出于这些原因，我喜欢标题<q>版本化</q>。我喜欢它也是因为，对我来说，它感觉不仅有增量版本，而且在网络连续体上每个人都有*不同的*版本——或者有些人选择称之为网络平台。

所以对我来说，我使用的网络版本是这个开发空间，在这里我构建软件，并与人们一起工作以实现商业目标——而对其他人来说，他们的网络版本可能是做出漂亮的设计，或者做出新的和直观的商业决策，或者创造不同的东西。

所以我喜欢标题版本化的另一个原因，只是因为它是每个人的创作空间的不同版本。

David [2:10]:

我真的很喜欢你带来的东西，因为当我在谈论版本控制时，我几乎是在考虑序列——几乎是旋律线——而你在谈论和声，这可以是多个人从他们自己的角度同时并行使用同一事物的不同版本。

好吧，我知道会这样，但我还是没有准备好！让我想想:我认为自己是 1.0 版本。当我构建软件时，我不喜欢发布 1.0。我喜欢发布 0.1，或者 0.01。我喜欢这样做，因为这是在发展中，它还没有准备好；这只是一个在这个空间公开的事情，但它没有完成。Done 是 1.0，然后之后的一切都在迭代，让它变得更好——至少对我来说是这样。

所以，我会说我已经通过了发展的一面。我有很好的基础，很好的了解这个空间，现在正在迭代。在我的职业生涯中，每天我都学到一些新东西——比如, <q>O，我不知道这是怎么回事,</q>或<q>我对语言的这一部分或网络的这一部分一无所知。</q>所以我会说我现在就在那里——1.0。

所以，这就是我在那里做的。我也喜欢写很多。我时不时会写文章，我还会发微博…如果那很重要的话！

David:

酷毙了。你能告诉我们一个你特别满意的副业项目吗？

Tim:

是的。事实上，我发起的最后一个副业项目是一个叫做 restaurantmenubuilder.com 的项目。

告诉我，大卫，如果这种情况以前发生在你身上——你正在寻找一家新餐馆吃饭，你在手机上调出一些东西，你正在浏览，你看到，啊，一个菜单链接。所以我要点击它，对吗？会发生什么？你必须下载一个 PDF 文件。

现在，对于大多数理性的人来说，这可能不是什么大问题。但我就是不想下载一个 PDF，损失手机的存储空间。所以我做了这个东西，让你只需要填写一堆表单选项，它就可以用 HTML 和 CSS 创建一个动态菜单。这是一个无服务器的应用程序，它只是在你的浏览器中运行，它创建了这个完整的菜单。它会将其保存到本地存储，最后，当您完成后，您可以下载并保存到您的设备上。

David [8:28]:

有意思！人们是如何使用它的？这是如何分配的？

Tim:

现在，只有 restaurantmenubuilder.com。我稍微推广了一下；我发了一些推文，我在网上发了一些帖子给不同的社区，告诉他们:<q>嘿，这是一个工具。</q>

我有谷歌分析，但我不怎么看。这是一个很难让我理解其中逻辑的项目，在我建立并购买域名并启动它之后，我休息了一段时间。我当时想，现在已经够了，以后我会推广剩下的。

但是我还没有看到任何人使用它。到目前为止，这只是一件我感到非常高兴的事情。

David:

我知道当你有这样的渴望时是什么样的，你只是[想]，<q>我知道如何建造它，我想把它放在那里，我只想把它给全世界。</q>

Tim:

是啊，肯定的。

David:

酷毙了。顺便问一下，你用什么建造的？

我最初是在 [CodePen](http://codepen.io/) 中开发的——这是一个由 Chris Coyier 和他的团队发布的在线 IDE。很直观。它被用于从会议到教学的许多场合。我的很多副业都是这样开始的。如果它们只是 HTML、CSS 和 JavaScript，我会直接跳到 CodePen 上演示一些东西。一旦我有了一个工作产品，我就会在 GitHub 中打开一个库，然后把它编码出来。

所以，这都是普通的——普通的 HTML。我正在使用 Sass——或 SCSS——和 JavaScript，以及本地存储和 web workers。

David:

非常酷。所以这是非常通用的。它不依赖于自身以外的任何东西。

Tim:

是啊。我喜欢尽可能以那种方式建造。我在这方面非常固执——我相信我们总有一天会进入这个领域。

David [10:13]:

嗯，我们可能会发现自己站在了这场争论的同一边。

Tim [10:22]:

不错！

那么，你呢，大卫？你是怎么进入这个行业的？

David:

嗯，这取决于你说的是什么行业。作为开发者，还是作为播客？

Tim:

哦，这是一个很好的分裂。所以先说发展。

David:

好的。嗯，我不是开发者*【咯咯笑】* …但我曾经是。我做了很多年的前端 web 工程师，也做过一些角色的设计师——因为公司有时并不真正知道前端工程师和设计师之间的区别。所以我得到了这个称号，尽管我不认为我应该得到它。

但作为一名前端开发人员，我进入其中只是因为对代码着迷。我开始是一名作家，网络就发生在我身边。我看着它，我说，<q>这是我想玩的东西。我自学了我需要知道的东西。</q>

Tim:

那真的很酷。你说你不再是开发商了？就像，你不全职开发，或者你真的不再编码了？

David:

好吧，我真的不再专业编码了。我编码是为了好玩；我为自己做了一些小项目。但我以前是全职的。我曾经在创业公司工作，是团队的一员，我为 StumbleUpon 编写了很多前端代码。我为保险公司做过类似的事情。事实上，很久以前，我是 www.apple.com 的网站管理员。

但是，最近几年，我发现自己更倾向于项目管理。我变得非常迷恋 scrum 和敏捷流程。我看到我的团队成员如何能够改善他们的合作方式，提高他们的生产力，提高他们评估事物的能力，我想更多地参与其中，并帮助更多的人发现这一点。

所以，我已经不再是一名亲力亲为的开发人员，现在，我经营着一家名为 [Agile 的小公司，在](http://www.agilethatworks.com/)工作。我是一名敏捷教练，我帮助公司进行敏捷过程，帮助他们改进 scrum 的工作方式。

Tim [12:38]:

那真的很酷。哇，首先，apple.com 的站长！那是一件非常令人兴奋的事情！

David:

公平地说，那是在 apple.com——嗯，[www.apple.com](http://www.apple.com)——还是个新事物的时候。Apple.com 存在的时间要比这长得多，因为苹果公司已经成为中坚力量很长时间了。但是 www.apple.com 的 T2 在 1994 年复苏了。它是由苹果图书馆的某个人托管的——事实上，是戴尔·米德——他把它保存在他桌子下面的一台 Power Macintosh 850 上……因为那时网络对很多人来说并不存在，也没有人知道它的存在。

我向企业沟通副总裁提出了一个项目，将它引入苹果的企业沟通团队，让它正式存在，创建产品库，以及人们需要的许多其他信息，然后将出现在公司不同部门的许多不同的 www.apple.com 小部门集中起来。

所以这很有趣，在大约一年半的时间里，我是那个网站的网站管理员，和一个优秀的团队一起工作，他们教会了我很多关于 HTML、网络分析和冗余的知识…

从企业沟通的背景进入 it 行业，我学到了很多东西，这让我着迷。当我离开苹果的时候，我已经是一名成熟的工程师了。*【咯咯笑】*

Tim:

这太不可思议了——真的很酷。你说过你用敏捷做了很多事情。还有什么？似乎你现在参与了很多事情。那么，还有什么其他的东西呢？

但是我也为 SitePoint 编写课件，我刚刚发布了关于 JavaScript 的入门课程。我正在[学习 JavaScript](https://www.sitepoint.com/premium/courses/javascript-next-steps-2921) 的中级课程。我还主持了另一个名为 [Hack the Process](http://www.hacktheprocess.com/) 的播客，在那里我帮助人们弄清楚如何从有意识地计划某事转变为实际做某事。

Tim [15:09]:

哇，那么，播客、创作、<q>教程</q> …

David:

对我来说，最难的部分似乎是决定我想专注于哪里。因为，直到几年前，我还是一名全职员工。我的职位是各公司的高级前端工程师，或者高级平台开发人员——或者其他什么。那是我感到舒服的地方。

但是我意识到，当我开始从事更敏捷的工作时，这是我可以更多地造福世界的事情。我觉得独立工作更舒服，从一家公司转到另一家公司，有点像在周围撒敏捷的仙尘。

Tim:

是啊，肯定的。那非常令人兴奋。

好了，现在是哲学问题的时候了。在你目前的职业生涯中，你是什么版本，为什么？

David:

哇哦。我现在的职业生涯，我不得不说我是 0.1 版本——因为，正如我所说的，我已经做了这么多我正在尝试做的不同的事情。而且，虽然我被开发工作吸引，我被课件创作吸引，我被播客吸引，我被敏捷教练吸引，但这些事情都没有成为我做的一件事。我不想放弃他们中的任何一个，在我心中，我觉得他们之间有一种相关性，如果我继续追求他们中的每一个，直到我不再觉得我想做些什么。

因此，正因为如此，在这一点上，我将称自己为 0.1，因为我仍在努力寻找它们如何融合在一起——而不一定要放弃任何仍然吸引我的东西。

Tim:

那真的很酷。是啊，肯定的。我喜欢网络的一点就是你正在做的事情:这个行业有这么多方面，有这么多不同的部分，无论是开发，还是播客，或者只是告诉人们如何更好地使用它。那真的很酷。

David [17:13]:

这很有趣。我有一个很好的理由来主持一个关于从计划到行动的播客，因为这是我自己职业发展的一个非常重要的部分——我意识到我所做的需要被分享，需要有一个观众。就好像这幅画不是艺术，直到有人对它做出反应。直到有了观众，直到有了反应，才真正有了意义。

Tim:

绝对的。

David:

蒂姆，我们正在做的这个播客怎么样？我们希望*这个*是什么？

Tim:

我们希望这是什么？这是一个非常好的问题。所以，我们首先通过讨论版本化对我们的意义来打开这个话题。有鉴于此，我真正感兴趣的网络版本之一就是构建它。我喜欢 web 开发，喜欢参与不同的项目，制作有趣或令人兴奋的东西。

所以，我真正想完成的事情之一是谈论为网络而建设。它是如何完成的，接下来会有什么，有什么新的，在版本方面会有什么新的东西出现，作为增量？十年前有表格，现在是 div、flexbox、floats 等等。

所以我肯定想花点时间谈谈它的机制。我们使用的是什么东西，即将出现的是什么新的令人兴奋的东西，我们最近看到的是什么东西？诸如此类的事情。

David:

那很有趣。这是很多人有兴趣定期关注和保持的事情。

当我想到版本控制时，我变得很有哲理，实际上，我开始思考这对我们的生活有什么影响，思考这种改变的方式。我与人们进行了令人惊讶的对话，例如，我们现在的许多社交交流正在从记录和永久存档的系统转移到更短暂的新交流方式，就像我们在街上的谈话一样。这对于我们交流的方式以及我们如何使用这些技术有什么影响？

Tim [19:43]:

是啊，肯定的。这是一个大问题，尤其是在沟通方面。我们可以有一整集都是关于这个的！*【咯咯笑】*

David:
