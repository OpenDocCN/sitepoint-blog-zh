# Flash 抄袭和当前的气候

> 原文：<https://www.sitepoint.com/flash-plagiarism-and-the-current-climate/>

如果你一直在一个热键上苦干，创建最新最棒的 Flash 应用程序，只看到它或它的衍生产品在网上到处出现，那么你已经被某个使用 SWF 反编译程序的人击中了，他们反编译了你的 SWF 文件，并根据他们自己的需要改变了内容的用途；无论是动作脚本、声音、图像还是结构。

当它是个人的 SWF 文件时，说它令人讨厌是一种轻描淡写的说法，但是当内容是商业企业的时候呢？如果要通过互联网向公众展示 swf，这方面的影响可能非常深远，需要将内容和业务逻辑从设计元素中分离出来。那么你能做些什么来对抗这种类型的剽窃来保护你的文件呢？

有以下几种选择(不是确定的列表):

–使用服务器端动作脚本(SSAS)

与 Flash MX / MX 2004 ActionScript 的概念完全相同，只是 ActionScript 驻留在服务器上，并通过几种不同的方法引用；这可能会变得昂贵，因为您将需要访问 Flash Communication Server 的安装。

–在 Flash MX 2004 中使用 SWC 文件

以。允许进一步保护 SWF 源信息的 SWC 文件。

–使用代码混淆

除了 Flash MX 之外，您还可以使用几个工具来混淆 SWF 文件，这将使 SWF 反编译器无法轻松地对它们进行反向工程，但这有内在的问题。一旦发布了新版本的混淆应用程序，这通常会被反编译应用程序的更新版本所抵消，因此您最终会不断地追赶自己的尾巴。

我一直找不到任何像样的混淆软件来停止反编译基本的 Flash MX 2004 SWF 文件，但这里有一个命令行混淆器，可能是一个值得关注的未来，当它最终发布的最终格式，希望为' 2004 年。

[action script Obfuscator Pro](http://www.genable.com/aso/preview.html)

当你处于它的接收端时，Flash 抄袭是严重的和令人讨厌的，但是有许多开源 Flash 格式规范的倡导者会有不同的看法。

当然，当你希望查看网页中的客户端代码时，每个人都曾在某个时候使用过“查看>源代码”来查看事情是如何完成的，但我不同意在任何层面上抄袭他人的作品。

当我收到更多信息或者有更多关于 SWF 保护的 Flash 社区更新时，我会更新这个帖子。
