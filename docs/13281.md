# 戴夫·格雷纳谈电子邮件的未来

> 原文：<https://www.sitepoint.com/dave-greiner-future-email/>

上周在举行的[网络方向南方会议上，我设法从](http://www.webdirections.org/) [Freshview](http://www.freshview.com/) (负责[竞选监控](http://www.campaignmonitor.com/)和[邮件构建](http://www.mailbuild.com/)的公司)挖来了戴夫·格雷纳。我们讨论了 HTML 电子邮件，推动电子邮件标准的支持，以及如何构建一个成功的 web 应用程序。

SitePoint:最近我退订了很多电子邮件简讯。你认为电子邮件作为一种大众传播媒介正在被淘汰吗？

啊，你从一开始就坚持我，不是吗？对！

**SP:哈哈！我能说什么呢？SitePoint 办公室里最常挂在嘴边的一句话是“电子邮件死了！”**

是啊，没错，我听够了。但我认为，在早期采用者的利基市场——技术社区——很容易草率行事，而不考虑世界其他地方和他们使用什么。

从营销的角度来看，电子邮件仍然是迄今为止任何在线营销努力中投资回报率最高的——它扼杀了搜索，扼杀了横幅广告，扼杀了几乎所有其他东西。所以我认为，从营销的角度来看，它非常活跃。除了一些早期采用者做的“退订”之外，我还没有看到任何东西会向我暗示电子邮件不能与 RSS 共存。

很明显，确保它在我们身边，并且我们密切关注它，对我们最有利。我们读了所有的“电子邮件死了！”帖子，不要着急！我们很清楚一些人的想法。但我个人并没有看到主流对电子邮件失去信心。垃圾邮件问题肯定是个问题。但我认为我们肯定开始在这方面取得进展，在未来五年左右，我们将在这方面看到一些变化。

你简单地提到了 RSS——你认为 RSS 有可能被主流采用吗？

可能不是目前的形式。但我认为这取决于操作系统如何让它尽可能简单易用，所以人们不需要学习仍然是一个新术语的新技术，需要你启动一个单独的应用程序来使用它。我想它被集成到 Outlook 和苹果邮件中的事实肯定会有所帮助。但我认为 RSS 和电子邮件仍然是两回事，尤其是在与电子邮件的一对一和一对多关系方面。您无法回复 RSS 源；你不能开始对话。创建一个定制的 RSS 源要困难得多。

因此，如果你收到一条特定的信息，你知道有人感兴趣——因为他们说他们喜欢冲浪，你知道他们是这个年龄组的人——那么你知道你可以向他们发送关于冲浪产品的有针对性的报价，他们会对此感兴趣的。然后你就可以得到回复，开始和那个人对话。这可能是最大的区别。

此外，仍然有大量的人不知道什么是 RSS。这些人只是想在他们的电子邮件中得到一些东西，因为他们知道电子邮件是如何工作的，这是他们习惯的。他们不想使用网络浏览器或其他应用程序。

不过，有些东西 RSS 更适合，把它们从你的电子邮件中删除也不错，因为你不希望它们占据空间——你只想阅读的东西。有了 RSS，你可以跟踪它，让它远离你的脸。

SP:我们来谈谈垃圾邮件。你暗示说，你认为我们正在取得进展——在这一点上，我父亲可能不同意你的观点。事实上，他可能会说情况越来越糟，而不是越来越好。

是的，我可能不会走得太远，说我们在上面。肯定不是。但我认为现在种子已经开始被缝合，这将开始给电子邮件增加更多的责任，这是非常重要的一层。

史迪芬:有什么例子吗？

一件大事是电子邮件认证，像微软的发件人 ID 和雅虎的域名密钥。基本上，这些只是增加了一层责任，要求人们在发送电子邮件时证明他们是谁。我认为这是重要的第一步。

当然，这不会阻止人们——垃圾邮件发送者也可以设置认证记录。但是，一旦我们开始将身份与声誉联系起来——例如，如果一个特定的 IP 地址发送大量电子邮件，并且为其设置了一个域名密钥——一旦该 IP 地址开始发送可疑的电子邮件，那么身份就已经被破坏了。

所以这是两件大事，信誉和认证。它们正在开始起飞——同样，它们可能在精明的电子邮件人群中更受欢迎——但越来越多的 ISP 开始支持这一点，这肯定是一件有前途的事情。这些事情当然需要时间。

很多人都喜欢电子邮件。但是你们已经采取了一些措施来消除 HTML 电子邮件不好的神话。告诉我们你做了什么。

我再次认为这是另一个早期采用者的神话——在我们的社区中有更多的纯粹主义者，他们为纯文本而活，在这次会议上，我们有几个人找到我们，告诉我们“HTML 电子邮件是邪恶的”和“它是魔鬼！”

我想人们必须意识到它不会消失——它是目前每一个流行的电子邮件应用程序的默认格式。我认为，就像网络一样，它比文本更容易被滥用:发送的不良电子邮件和设计的不良网页一样多。我认为对制造商来说，重要的事情是支持一些通用标准，这样人们就不必构建又大又丑又令人震惊的 HTML 电子邮件。

让我们来谈谈这个。最新版本的 Outlook 在相当大的程度上倒退了 HTML 电子邮件，而你们似乎已经把它作为自己的使命来弥补。

它确实把它倒过来了！自从那条新闻出来后，我们大概花了一年半的时间，观察其他电子邮件客户的发展方向，并对此进行抱怨。我们现在意识到那不会让任何人去任何地方。是时候有人试着做点什么了。

如果可以的话，我们正在建立一个组织，一个运动，试图扭转这种趋势。每个人都对电子邮件客户端制造商很苛刻，但有时支持标准——尤其是基于 web 的电子邮件客户端，当它是页面中的页面时——可能非常困难。这可能是一个挑战。第一步是意识到这一点，然后主动与他们合作，帮助他们改进，告诉他们需要改进的地方。

我们所做的是我们建立了一个标准的基线，我们认为这至少应该是一个好的开始，让每个人都能达到。我们目前正在研究所有主要的电子邮件客户端，整理出一个愿望清单——前 10 个。信息是，“如果你能做任何事，做这 10 件事，它会让你非常接近我们需要的地方。”

我们认为这是非常重要的第一步，每个人的反馈都很棒。我们已经有很多人在展位前停下来说，“祝贺你，太棒了，非常感谢！”甚至一些 SitePoint 的人也来过，给了我们一个，这很酷！

同样，就像垃圾邮件问题一样，这不是一夜之间就能解决的。但是我认为，根据我们目前收到的反馈，大部分大型客户端开发者很有可能会接受我们的要求。

那么这会导致一本封面上有你戴着帽子的书《用电子邮件标准进行设计》吗？或许是由 SitePoint 发布的？

哈哈！用泽尔德曼的姿势！我喜欢，这是一个选择！我能看见它！

史迪芬:嗯，这基本上和网络上的斗争一模一样，不是吗？

确实是。说实话，这是一个死气沉沉的镜像。有几个细微的区别。但是每个人的反应都是一样的…总是有很多人支持，也有很多人反对。

你注册 emailstandards.org 了吗？

我们首先抓住了 email-standards.org。但是我们实际上已经有人和我们取得了联系，并且说“我得到了 emailstandards.org，我很高兴把它捐献出来。”这是人们支持它的一个例子。所以是的:[email-standards.org](http://email-standards.org/)应该在一个月左右，我们希望。

像 SitePoint 一样，你们的总部在澳大利亚，但是你们的客户群主要在美国和欧洲。你如何解决地理障碍，以确保你仍然可以很好地服务你的客户？

从客户支持的角度来看，我们尝试并尽一切努力确保人们可以在无需联系我们的情况下得到问题的答案。因此，我们在应用程序中内置了一个非常全面的帮助系统:我们有客户论坛，我们在网站上有大量的信息和资源，并且在应用程序中有很多内嵌的帮助。因此，我们非常注重用户体验，尽量减少人们需要的帮助，这是不可避免的。

我们还将期望值设定在适当的水平。我们不提供全球电话支持，或类似的服务。我们只提供电子邮件支持。我们发现，如果我们在 12 小时内回复人们，他们总是非常高兴。我们只是确保我们这边回答支持问题的人了解所有关于产品和 HTML 电子邮件的知识，这样他们就可以直接给他们一个好的答复，而不是来回地回答。

我们做的另一件事是直接告诉人们，在申请中，在支持表格上，“嘿，现在是悉尼的凌晨 2 点，”所以他们不期望立即得到回复。当他们六个小时后收到一封信时，就像“哦，你起来了，太好了。谢谢你帮助我。”

史迪芬:所以你没有发现让别人知道你在澳大利亚有损你的名誉吗？

一点也不，不。当然我们也没有收到任何反馈。我的意思是，如果产品有效，做了他们需要做的事情，如果他们与我们有一点关系，那么他们就很高兴。我们对一切都持开放态度，这是信任的要素——我认为这也很有帮助。我认为，如果我们将自己展示为一家大企业，然后突然有客户发现我们是澳大利亚的一家小公司，那么这可能是一个问题。但我认为透明度也有助于这一点。

SP:你们确实因为自己编写的应用程序非常有用和直观而获得了很多好评。那么告诉我，你如何制作一个有用的、漂亮的网络应用程序？

哈哈，谢谢。我想你必须对它充满热情。创建一个界面并认为“这已经足够好了”是非常容易的但是我们的团队中有很多挑剔的人，他们说“嗯，这个怎么样，那个怎么样？”我们就页面上的某个元素展开了激烈的争论——关于哪种方式是最好的。我想你首先要有那种激情。

我们没有一个人有正式的设计背景，或培训背景或类似的东西。我想这只是一个观察一些行业领导者如何做事的问题，取出你认为对你有用的部分，添加你认为对你的用户有用的部分，并把它们融合在一起。

那么，我们对 Freshview 的未来有什么期待呢？

现在，我们有很多新的东西想添加到这两个应用程序中。我们总是只添加每个人都可以使用的东西，我们当然不会膨胀它们。但是我们有一些我们认为会很方便的杀手锏。

下周，我们将推出一项新功能，让人们在所有主要的电子邮件客户端生成电子邮件截图，并在发送给订户之前，通过所有的垃圾邮件过滤器。所以会有一点点费用。但就节省人们测试电子邮件的时间而言，这应该是非常值得的。这个 HTML 邮件一致性问题基本上是发送邮件的噩梦。所以这应该有望消除它的痛苦。

从长远来看，我们有一些想法。但我认为，现在我们只是要继续专注于我们的两个电子邮件应用程序，并使它们成为设计师的最佳选择。

戴夫，谢谢你的时间。

别担心。谢谢马特！
