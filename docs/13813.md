# 不仅仅是你做了什么…

> 原文:[https://www.sitepoint.com/its-not-just-what-you-do/](https://www.sitepoint.com/its-not-just-what-you-do/)

我周五开始了一项新任务，为一家医疗保健公司提供营销支持，包括网站开发。

为了给人留下好印象，我发出了一封营销信的初稿。

客户不喜欢它，我可以接受(这可能需要时间和一些迭代才能达成一致)。但是后来客户资助人把这封信发给了他的其他 5 个合伙人，征求他们的意见。当然，这伤害了我，因为每个人都看到了不可接受的第一个可交付成果。

所以我不仅失去了对“做什么”的控制，也失去了对“如何做”的控制。菜鸟错误。

我应该预先设定好最有效的合作方式。所以今天早上我做的第一件事就是发送了一份新的改进的草稿，同时请求客户发起人和我先对这个可交付成果感到满意，然后将它发送给其他人。这样，我们可以避免在早期草案上浪费其他合作伙伴的时间，并保持有效的沟通。

因此，在您管理项目时，请确保您将重点放在“是什么”和“如何”上。“如何”包括你如何沟通，如何提交可交付成果，可交付成果的格式，设定对草稿和已完成工作的期望等。

还有第三个关注领域，这个领域有点抽象。这就是“背景”，即当你工作时，你是谁，或者你来自哪里。一些设计师和开发人员表现得像个大明星，或者有头衔的小屁孩。对他们来说，这个项目是关于他们自己的，而不是客户。更有效的方法是让你的“背景”是关于为客户服务、付出额外的努力以及成为完美的职业。

什么，如何，谁:专业服务的三个角落。
