# Web 方向反思，第 4 部分:硬汉 Web 设计和 SVG 致敬

> 原文：<https://www.sitepoint.com/web-directions-4-hardboiled-web-design-svg/>

在我的最后一篇 [Web Directions @media](http://atmedia.webdirections.org/) 帖子中，我思考了两个会让网页设计者和开发者感兴趣的演示文稿…

## 安迪·克拉克:硬汉网页设计

安迪·克拉克是会议第二天的主题发言人，对我来说，他的发言是最有争议的。他的新书《硬汉网页设计》讨论了为什么网页开发过程需要改变以充分利用现代浏览器。他的主要观点是: **1。渐进增强还适用吗？**2003 年，人们首次提出了这样一个概念:为功能最差的设备构建，然后为现代浏览器进行增强。安迪认为从视觉设计的最小公分母开始是一个错误。 **2。自上而下的设计**安迪建议你先从最好的体验开始。然后，您可以删除较低版本浏览器的功能。 **3。网站不需要在每个浏览器中都看起来一样，一个网站在每个浏览器中都看起来一样是不可能的。安迪展示了一个令人惊叹的网站演示，使用了在 Chrome 和 Safari 中工作的 CSS3 转换和动画。在其他浏览器中，该网站很吸引人，但使用了完全不同的布局，效果也差得多。为了实现这一点，Andy 使用了 [Modernizr](http://www.modernizr.com/) :一个检测浏览器特性的 JavaScript 库。例如，如果浏览器支持边框半径，它会在 HTML 元素中添加一个“border radius”类，这样您就可以相应地修改您的 CSS:**

```
#myelement{	background: url(rounded-background-image.png) 0 0 no-repeat;}.borderradius #myelement{	background-image: none;	-webkit-border-radius: 5px;	-moz-border-radius: 5px;	border-radius: 5px;}
```

他声明 IE6 不是问题。IE6 用户可能会获得非常基本的体验，但他们对其他浏览器中的设计一无所知，很少抱怨！ **4。忘记当前的市场份额**如果你分析你当前的浏览器市场份额，你看到的是现在的情况。未来呢？6 个月后当你的新网站发布时，用户会使用什么？这是一个发人深省和有趣的演示，我同意安迪所说的大部分内容——尽管他承认他的一些想法处于不断变化的状态。我确实有一些保留意见:

*   渐进式改进有它的问题，但安迪似乎在倡导一种优雅退化的形式。从纯视觉设计的角度来看，这可能是一个好主意:你首先设计最好的体验。然而，代码开发是不同的——自下而上的工作通常更符合逻辑。
*   虽然我承认不同浏览器的网站看起来不同，但大多数网站都提供相似的体验。提供 2 或 3 种完全不同的布局在商业上可行吗？设计一个让你的客户满意的概念已经够难了！
*   Andy 在他的例子中使用了一些令人惊叹的 Webkit 效果，但是这些浏览器目前占据了大约 15%的市场份额。这个数字将会增长，其他浏览器可能会赶上来，但是你仍然花费大部分时间和预算为少数用户开发。有许多未知因素，在接下来的几个月里，你肯定需要更新你的代码。这可能会让一些客户有点紧张。
*   最后，IE8、Firefox 和 Opera 仍然是有能力的浏览器。它们可能不会提供同样的 CSS3 魔力，但是不借助于 Webkit-only 代码也有可能获得好的效果。

我期待着阅读安迪的书，看看他如何解决这些问题。

## Doug Schepers: SVG 的今天和明天

道格知道他的 SVG。说真的。忘记 PowerPoint 吧——甚至他的演示幻灯片都是在 Firefox 中显示的 SVG(他认为这是目前最强大的 SVG 浏览器)。像许多开发人员一样，当可缩放矢量图形在本世纪初宣布时，我对它们的前景感到兴奋。当微软忽略了这个标准时，这种幻想破灭了。但是，10 年过去了，SVG 又变得流行起来，它被承诺用于 IE9。Doug 的演示很有趣，他展示了一些强大的 SVG 例子。如果像我一样，您认为 SVG 是基本的线条和形状，那么您会对现在所能实现的东西感到惊讶。变换、动画和类似 Photoshop 的滤镜效果都是可能的。我推荐你看一下道格的在线演示，还有一段 T2 在 MIX10 上演讲的视频。最后，W3C 出版了一本免费的在线书籍— [《面向当今浏览器的 SVG 初级读本》](https://www.w3.org/Graphics/SVG/IG/resources/svgprimer.html) —如果你想了解关于这种格式的任何事情。
