# IE7 团队聊天记录

> 原文:[https://www.sitepoint.com/ie7-team-chat-transcript/](https://www.sitepoint.com/ie7-team-chat-transcript/)

微软发布了一份 2 月 9 日的聊天记录，其中 IE7 团队回答了开发者和其他公众关于浏览器的问题。

虽然聊天主要是由其他地方已经回答的问题主导的，但有一些信息值得注意。

首先，看起来微软仍然在做一些渲染上的改变，支持的 CSS 特性的最终列表仍然未知:

> **克里斯·威尔逊【MS】(专家):**
> **问:**IE7 最终版本会实现最大/最小高度/宽度吗？
> **答:**我们知道随着溢出的变化，最小/最大宽度/高度将变得更加重要。我们正在努力让它进入 IE7，但我们还不能承诺支持它。

听起来微软正在计划实现一个公共的错误跟踪数据库，这样开发人员就有一个集中的地方来报告和记录浏览器中呈现问题的解决方法:

> **阿努拉格【MSFT】(专家):**
> **问:**我希望看到一个类似 bugzilla 的公共 bug 分类系统……这难道不会帮助你剔除重复的反馈吗？我们正在努力实现一个类似的系统。感谢您的反馈。

我们已经确认 IE7 中不会有 JavaScript 补丁。尤其令人沮丧的是，我们仍然不得不忍受不标准和痛苦的事件处理模型:

> **戴夫·马西【MSFT】(主持人):**
> **问:**在你的博客文章中，你没有过多地讨论 Javascript 的变化(XMLHTTPRequest 是个例外)，你会计划改进对 JS 的支持吗？会添加对 obj.addEventListener()、obj.watch()之类的标准方法的支持吗？答:我们不希望 IE7 中的 JScript 有任何增加。我们已经听到了要求，并希望在未来专注于脚本引擎的改进。我们知道这里有很多工作要做:)

我们也有一个关于 IE8 发布窗口的承诺…尽管我不能屏住呼吸:

> **克里斯·威尔逊【MS】(专家):**
> **问:**[……]I E8 会在 2010 年之前发布吗？
> **答:**是的，IE8 会在 2010 年之前发布。

最后，我特别喜欢这句话:

> **克里斯·威尔逊【MS】(专家):**
> **问:**IE 团队是否担心 firefox 份额呈指数级增长，而 IE 却因为陷入操作系统依赖而无法修改和发布新版本？
> **答:**否

很公平，我想！
