# 纽约 SES:巴里迪勒得到搜索吗？

> 原文:[https://www . site point . com/ses-NYC-does-Barry-diller-get-search/](https://www.sitepoint.com/ses-nyc-does-barry-diller-get-search/)

搜索引擎战略会议于今天早上开始，丹尼·沙利文采访了 IAC/Interactive 的[巴里·迪勒，讨论了 Ask.com](http://www.iac.com/index/management/board_of_directors/bio_bod_barrydiller.htm)搜索门户[的品牌重塑和未来方向。](http://www.ask.com)

首先，让我说点巴里·迪勒的好话——**他懂定位**。他得到了品牌。他明白，用户将决定他负责的搜索引擎是生存、繁荣，还是变得无关紧要。他很有说服力。他让我开始尝试新的 Ask.com 搜索——我可以向你报告，这是我迄今为止在搜索门户网站上看到的最干净的界面和最好的工具集。

关于 Barry Diller，以及 ask 和 IAC 的整个组织，我要问的是 ***他们了解搜索吗？*** 我选择谷歌作为我的搜索引擎的原因并不是一个很酷的用户界面，而是谷歌提供的有机搜索结果的**质量。如果谷歌让我失望，我会选择雅虎和 MSN 的原因是，它们可能会提供好的搜索结果。**

Ask 的有机搜索结果由 Teoma 提供支持。Teoma/Ask 代表多年来一直来到 SES，谈论他们的前沿技术、先进的算法，并告诉我们(正如 Barry Diller 今天所做的那样)尝试一下 Ask。

不幸的是，每次我给他们一个尝试，我都有同样的感觉，无论界面多么酷，搜索结果都达不到谷歌、雅虎和 MSN 设定的标准。在我看来，问题不在于他们的算法，而在于他们对网络的搜索不够频繁。我在谷歌上找到的东西，尤其是涉及到更长的搜索查询时，比我在 Ask 上找到的要好。

一般的搜索(酒店、鲜花……)所有的“主要”搜索引擎通常会给你不错的结果。当然，有时谷歌的搜索结果也很糟糕…如果我搜索“航班”,我会在谷歌上找到相同的英国网站集合(我在美国)。MSN 和 Yahoo 做得更好，虽然我怀疑他们可能会手动调整这些引擎上非常流行的搜索结果，但我真的不在乎，因为他们向我展示了我想要的东西。

无论如何，迪勒先生…我喜欢这个界面。我喜欢这些工具。我讨厌搜索结果。如果你想让你的搜索引擎成为一个竞争者，他们必须做得更好。当谈到他们可以访问的信息量和他们的索引的新鲜度时，谷歌轻而易举地击败了你。

对于在纽约观看节目的读者来说，今天下午 1 点我将在搜索词研究上发表演讲，我将在节目中四处走动，直到周二下午 3 点左右。如果你见到我，我希望你能打个招呼。
