# 预览:JBuilder 2006 与 IntelliJ IDEA 5.0

> 原文：<https://www.sitepoint.com/preview-jbuilder-2006-vs-intellij-idea-50/>

几个月前我在 T1 上报道的所有 Java IDE 发布都有了结果，最近发布的有 T2 Eclipse 3.1 T3、T4 NetBeans 5.0 Beta T5、T6 IntelliJ IDEA 5.0 T7 和 T8 Borland JBuilder 2006 T9。

作为 JBuilder 的长期用户，我一直处于一个特殊的位置。在最近一次统计中，只有 8%的 Java 开发者在使用 JBuilder，最受欢迎的选择是免费的 Eclipse (76%)和 NetBeans (21%)，IDEA 是最受欢迎的商业选择，占 13%。随着这一波新的发行，我觉得是时候重新评估我自己的选择了。

在过去的几周里，我在 IDEA 5.0 和 JBuilder 2006 之间来回跳跃，从事各种项目，包括复杂的[网络应用](https://www.sitepoint.com/jetty-really-is-lightweight/)，繁重的[桌面应用](http://www.blogbridge.com/)，敏捷的[小程序](http://www.editize.com/)和轻量级的[手机游戏](http://j2mebomber.sourceforge.net/)。

JBuilder 一直是我的选择，因为它的特性非常丰富。以我的经验，如果 JBuilder 没有，那就不值得拥有。代价总是可用性——JBuilder 从来都不是一个特别吸引人的工作环境，一个糟糕的配置选择(例如，一个缓慢的 CVS 服务器)可能会立即让它崩溃。确实是一个值得尊敬的产品，它仍然受到一些遗留问题的困扰，比如不能处理安装路径中的空格。

然而，这是我能找到的功能最丰富的选项，尤其是现在 IDE 的不同版本中的功能似乎都很有意义(500 美元的开发人员版非常好，3500 美元的企业版令人震惊)。我总是对 JBuilder 出色的特性集无与伦比的灵活性感到满意。

相比之下，IDEA 是一个年轻的新贵，通过提供顶级的可用性、功能丰富但令人愉快的不可信的编码环境以及对开发人员生产力的关注而出名。在使用了几个小时并发现它是如何工作的之后，我已经发现了一些我从 JBuilder 中错过的东西，但我也发现自己更喜欢它做许多它可以做的事情的方式。

IDEA 背后的 IntelliJ 公司也与其开发者社区建立了更加开放的关系。它提供下一个版本的免费早期版本，一个开放的错误跟踪数据库(带投票)，以及一个活跃的论坛社区。

我将在接下来的两周内对这两个产品进行详细的评论，但是从我目前为止的修补来看，我可以说这将是一场势均力敌的比赛，最佳选择很可能取决于您自己的个人需求。当然，这两种产品都没有把对方甩在身后……竞争难道不是一件好事吗？
