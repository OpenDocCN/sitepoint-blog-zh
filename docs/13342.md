以下内容转载自《科技时报》第 169 期。
**截止 7 月 10 日，活跃会员三千万，我就是其中之一。在过去的两周里，我的活跃联系人中似乎有一半都加入了这个社交网站，包括那些不会对 MySpace 着迷的专业人士。那么是什么让脸书与众不同呢？**
![](../Images/b89e6472b5320266f2af6c5dafde6976.png)

嗯，首先，它有一两个设计人员，这意味着你实际上可以使用该网站，而无需开发面部虱子。

不幸的是，这些设计者仍然需要学习一些关于 web 标准的东西——这个网站充满了容易纠正的验证错误。

但更重要的是，脸书被设计成一个社交网络应用的开放平台，具有公共程序员功能(API ),使第三方开发者能够扩展和增强网站。

尽管开放，脸书仍然是一个有围墙的花园。如果你想和脸书的人联系，你需要使用脸书。在某些方面，脸书仍然比其他社交网站封闭得多。例如，脸书只允许你向其他脸书用户显示你的脸书个人资料。

我坚信成功的围墙花园时代即将结束。事实上，我预测脸书将是最后一个主要的围墙花园成功故事，最终将不得不适应成为一个基于标准 API 和数据格式的网站，或者被取代，这些标准 API 和数据格式将整个网络变成一个大的社交网络，在那里你可以与人们联系，不管他们使用哪个网站来存放他们的个人资料。

## 打破围墙

如果脸书的应用程序可以自由地做上面列出的所有事情——就像脸书的内置应用程序一样——那么脸书怎么会是一个有围墙的花园呢？

所有脸书应用程序都建立在该网站提供的社交网络基础之上:

*   脸书用户属于网络。
*   脸书用户有朋友，他们也是脸书用户。
*   脸书用户有一个个人资料页面，他们的朋友可以看到。
*   脸书用户可以查看一个可配置的新闻条目，显示他们朋友的个人资料页面的变化。

尽管事实是一个开放的平台建立在它们之上，但这些都是封闭的系统，它们在脸书社区和竞争的社交网络如 MySpace 和 Virb 之间设置了障碍。

但是说真的，为什么你必须是脸书的一员才能知道你的朋友在做什么呢？作为一个脸书用户，你为什么不能列出你没有脸书账户的朋友呢？

几个日益成熟的标准有可能使这些限制成为过去——也就是说，如果脸书愿意接受它们的话。

#### 饲料

如果有人花时间构建脸书的应用程序，那么提要(RSS T1 和 T2 Atom T3 格式)可以很容易地集成到脸书的新闻提要和迷你提要功能中。例如，当一个非脸书人的朋友在她的博客上发帖时，你可以在脸书收到通知。

但是脸书不允许你的非脸书朋友订阅，比如说，你的脸书迷你订阅。饲料技术将使这一点今天很容易实现，但脸书的围墙花园方法将阻止他们在短期内做到这一点。

#### OpenID

OpenID 是一种标准，允许个人通过一个 URI 来识别自己，并使用这个 URI 和一个密码登录多个网站和服务。没有多少网站支持 OpenID 作为一种登录方式，但随着像 [Plaxo](http://blog.plaxo.com/archives/2007/07/plaxo_is_now_op.html) 这样有远见的早期采用者加入进来，它的势头越来越快。

脸书不仅可以支持 OpenID 作为一种登录方式，还可以通过成为 OpenID 提供商来完全接受该标准。然后，您可以使用脸书个人资料的 URL 作为您的 OpenID URI，并使用您的脸书密码登录其他网站。接下来，脸书自然会让你和任何 OpenID 身份的人成为朋友，分享你的个人资料——不仅仅是脸书用户。

#### 微格式

微格式是简单的标准，允许你标记 HTML 文档中的元素，赋予它们额外的意义，并向第三方软件和服务公开信息。

脸书提供了列出你认识的人、发布你的联系信息以及为你计划参加的活动做广告的功能。在脸书之外，微格式可以在普通网页上标记所有这些类型的信息:

*   [XFN](http://gmpg.org/xfn) 让你记录你与他人的
    关系
*   [hCard](http://microformats.org/wiki/hcard) 让您提供结构化的联系信息
*   hCalendar 允许你标记你在网站上提到的事件

如果它允许你将非脸书人列为好友，脸书就可以从他们的个人主页或其他社交网络上的个人资料中获取这些类型的信息(假设这些网络使用微格式发布数据)。

此外，脸书可以从专门的服务中获取你自己的信息，如 T2 即将推出的 T3(支持 hCalendar)，而不是强迫你将所有个人信息直接输入脸书。

## 下一步

正如你所看到的，我在本期中提到的开放技术几乎已经准备好将整个网络变成下一个热门的社交网络，但是我们真的能指望外行人掌握像 OpenID 和微格式这样的标准吗？

未来的 Facebooks 将是让这些连接技术为大众所用的服务，让你可以通过多种社交网络服务(包括普通的 HTML 网站)与网络上的所有人联系，兼容性和开放性将成为杀手级功能。
