# 管理团队的基础知识，第二部分

> 原文：<https://www.sitepoint.com/team-management/>

![huddle](img/f725959a10205f77fb5be193b32d177d.png "huddle")昨天，我发布了[的前半部分成功团队管理的基础知识](https://www.sitepoint.com/managing-a-team/)。这些建议适用于任何领导团队的人，无论你是分包商、员工还是团队领导。这是帮助你成为一个更好、更有效的管理者的第二部分。

**千万不要错过庆典**

应该有意识地庆祝团队的成功，无论是通过祝贺的电子邮件、汇报电话会议还是其他方式来认可每个团队成员的努力并结束项目。

**打开你的门**

作为经理或团队领导，没有什么比被团队认为不可接近更糟糕的了。提倡门户开放政策，这样每个人都可以放心地向你提出问题、疑问和要求。

**按时支付**

一个非常简单的方法来表明你有多重视你的团队，那就是按时给他们发工资。如果每个人都知道他们会按时得到报酬，这会促进信任关系，你的团队成员可能会更愿意为你多做一点。

**放弃假设**

你可能认为你足够了解你的团队成员，可以根据他们的经验和可用性做出决定。但是与其假设这些信息，不如提前问他们以节省时间和不必要的压力。

**立即解决冲突**

在团队环境中工作时，总会有误解和分歧的机会。不要回避这种情况，而是立即解决问题，给每个团队成员一个机会表达他们的担忧，达成一致的解决方案，然后继续前进。

**说出期望**

每次向团队成员分配项目或任务时，都要非常具体。当你提供尽可能多的信息，并明确你希望他们在项目结束时做什么和提供什么时，你将获得最好的结果，以及最接近你所寻求的结果。

**感谢他们**

一句简单的“谢谢”对促进团队团结大有帮助。感激你的团队成员提供的所有支持，不要羞于感谢他们一路上所做的工作。

**利用个人技能**

你可能会为某个特定的需求带来一个团队成员，但是发现他们的技能超出了一个项目的范围。如果形势需要，愿意改变他们的角色，增加他们的责任。

**志愿者**

对于一个记得在战壕里是什么感觉的领导人，有一些话要说。让你的团队大吃一惊，加入到一个项目中，向他们展示你没有忘记你从哪里来。

**改进工作**

即使事情进展顺利，也总有改进的空间和机会。在庆祝成功的同时，制定绩效目标，这样就没有人会在工作中自满。

**X 标记点**

规划出一条达到特定回报的途径。奖励可以在达到几个里程碑、完成多个目标或一个成功项目结束后给予。

**私下大喊**

如果你需要谴责某个团队成员的失误，千万不要公开或在团队其他成员面前这样做。所有的人事问题都应该关起门来机智地处理。

拉上拉链

有些事最好不说。在盛怒之下说些什么之前走开。成功的领导者知道什么时候最好离开，直到头脑冷静下来。
