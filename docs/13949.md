# Firefox 的屏幕截图和 Devboi

> 原文:[https://www.sitepoint.com/screengrab-devboi-for-firefox/](https://www.sitepoint.com/screengrab-devboi-for-firefox/)

我已经有几个月没有好好看看 Firefox 的一些新扩展了，但是最近有两个扩展真正吸引了我。

**屏幕截图**

![ScreenGrab screenshot](../Images/995d96a625a2f1fe4598a07920c4e39b.png)首先是[截屏](https://addons.mozilla.org/extensions/moreinfo.php?id=1146&vid=5435)。像许多人一样，我看着这一个，立即想到'这是什么？“它能做什么，而我现在不能，”。

这个问题非常有价值的答案很简单——它有能力捕捉一个长长的滚动页面的所有部分，然后自动将它们粘合到一个又长又亮的新 PNG 中。如果你曾经浪费过几分钟时间手工做这个，你会发现这是一个非常巧妙的技巧。

事实上，我想不出其他基于客户端或其他方式的应用程序能够执行这个看似简单的任务——这只小马只有一个技巧，但却是一个好技巧。

**颠复**

Firefox bow 的第二个漂亮的新字符串是 Zhluk.com 的 DevBoi。

Devboi 是一个针对 Firefox 的 Web 开发者参考侧栏，现阶段涵盖 HTML 4.01、CSSS2.1、DOM 和 XUL。我希望随着时间的推移会增加更多。虽然没有一个方面是显著的，但整个包都是经过深思熟虑的，并且立即有用。

CSS 参考使用'♥'、“”和“x”让您一眼就能检查浏览器对任何属性的支持，并将每个链接到 [PPK 的 Quirksmode](http://www.quirksmode.org/) 参考，该参考详细说明了该属性的已知不一致、错误和解决方法。

HTML 参考显示了一个完整的标记列表，其中标记了不推荐使用的标记和其他有用的信息，并链接到每个标记的 W3C 说明页。XHTML 可能是下一版本的有用更新。

Devboi 还包括一个全面的 DOM 参考，一个 XUL 参考，甚至还有一个正在开发中的 Ruby 和 Rails 参考。

总之，太多的屏幕盖在这里，所以我扔在一起一个小演示，带你通过它。它展示了 Devboi 的一些优点，以及我对 Macromedia 令人印象深刻的 [Captivate](http://www.macromedia.com/software/captivate/) 的笨拙控制。
