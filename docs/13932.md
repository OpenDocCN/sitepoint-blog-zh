# Zend 和 IBM 合作开发新的 PHP IDE 和框架

> 原文:[https://www . site point . com/Zend-and-IBM-to-co-develop-new-PHP-ide-and-framework/](https://www.sitepoint.com/zend-and-ibm-to-co-develop-new-php-ide-and-framework/)

有传言称 Zend 和 IBM 正在酝酿一项名为 PHP 协作项目的大型开源计划。该项目将基于 PHP 和 [Eclipse](http://www.eclipse.org/) ，旨在与微软竞争。NET 平台在中小型企业(SMEs)中，他们的需求还不足以保证采用 Java 平台进行 Web 开发。

我确信这些谣言的时间，随着 2005 年 Zend/PHP 大会在旧金山举行，不仅仅是巧合。毫无疑问，官方声明指日可待。

据我所知，我们正在关注两个相关的项目:

*   一个新的 PHP IDE 建立在 Eclipse 上，这是 Java 开发人员最流行的开发环境
*   一个标准化企业级 PHP Web 应用程序架构的新框架

据推测，IDE 将在构建时考虑框架，提供专门的工具在规定的架构内工作(就像微软的 Visual Studio 一样)。网)。

这是 PHP 需要进一步深入企业空间的强心剂吗？或者 PHP 只是简单地“模仿”Java，而实际上它应该*利用自己的优势，朝着一个独特的方向发展？*

我所知道的是，如果我们最终得到一个丰富的框架，它盲目地实施 MVC 架构，并跳过各种各样的障碍来提供便利，如持久的应用程序状态和后台进程，那么开发人员还不如现在就切换到 Java，避免等待。另一方面，如果 Zend 和 IBM 能够产生一些独特的东西——一个标准的 Web 应用程序结构，避免重新发明轮子，同时充分利用 PHP 的敏捷、无状态和动态的特性——那么这可能比 PHP 4.0 中引入面向对象更重要。

不要误解我的意思，我认为面向对象编程和像 MVC 这样经过验证的设计模式是很棒的，PHP 肯定可以从中吸取教训，使大规模开发更加实用。但是至关重要的是，PHP 不能盲目地跟随它之前的平台。对于许多项目来说，今天的 PHP 比 Java 更合适，这是有原因的，冒着失去这种独特性的风险就是冒着 PHP 本身的未来的风险。
