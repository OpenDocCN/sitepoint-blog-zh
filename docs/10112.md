# 网络排版的十大禁忌

> 原文:[https://www . site point . com/top-10-dos-and-donts-of-web-typography/](https://www.sitepoint.com/top-10-dos-and-donts-of-web-typography/)

作为设计人员，设置好字体是一项常见而重要的任务。当[尊重我们设置的文本](https://www.sitepoint.com/setting-type-on-the-web-an-introduction/ "DesignFestival.com: Setting Type On the Web: An Introduction")时，我们需要确保它的[易读性和可读性](https://www.sitepoint.com/achieving-good-legibility-and-readability-on-the-web/ "DesignFestival.com: Achieving Good Legibility and Readability on the Web")。此外，我们应该提供视觉线索，突出文本之间的关系。这样做，我们为文本提供了一系列的品质，鼓励和授权读者快速浏览特定的信息片段或轻松消化文本的较大部分。

对于一个好的设计师来说，设置文本是第二天性，但是有一个快速的清单来确保基本知识被覆盖是很有用的，特别是对有抱负的新手来说。这是我的清单，作为一套通用的指导原则:

## 1.每个设计(或页面)使用的字体不要超过三种

每个设计不要超过三种字体，这是一个很好的基本规则。用各种各样的字体来应用和设置文本(即使它们是相似的)会产生一种不统一的感觉；读者得到的概念是，相关的(如果不是相同类型的)文本(例如，主体段落)是不相关的。使用更多种类的相似字体可能会避免这种不统一，但读者仍然会注意到——即使是下意识地——一种奇怪的现象。即使他们不能描述和来源的干扰，影响可以保留。

这个规则中的“或页面”是为了提供艺术自由。Jason Santa Maria 的网站恰当地说明了这一点，他的博客帖子表达了他创造性的多样化设计和风格，而他的网站总体上保持了完美的统一性。

## 2.在页面顶部设置大而吸引人的标题

过去，人们对在网上设置大型文本有一种惊人的恐惧。正文设置为 10px(甚至更低！)似乎泛滥成灾，标题很少被视为比字体稍大的东西。标题是导航到新页面时眼睛阅读的第一串文本，对于标记段落中的部分至关重要。因此，优雅的标题以其应有的大小和风格吸引读者，并为低层标题提供大量但不显眼的标志来引导读者。

## 3.正文尺寸 14px+

这是一个相关的，也许是最重要的经验法则:给正文提供在屏幕上舒适阅读所需的尺寸。当然，许多其他因素也会影响文本的易读性和可读性，尤其是更紧密设置的正文，但大小仍然是关键。

记住，我们中没有多少人有 20-20 的视力；一个好的默认值是 14 像素。

## 4.确保文本与背景形成良好的对比

加大字体大小是让你的文本更易读的一个可靠方法，但是易读性(结果是可读性)会因为文本和背景之间的对比度差而降低。

公认的标准是将深色文本放在浅色背景上，反之亦然。绝对避免冲突的颜色或在白色背景上几乎看不见的灰色。

需要特别注意深色背景上的浅色文本，因为它不像它的表亲那样具有相同的对比度(即使最初看起来是这样):相应地增加跟踪、度量和行距。

## 5.谨慎地强调和强调

这部分是编辑原则，部分是印刷原则:不要在文本中添加多余的下划线、粗体和斜体；为添加的和有意强调的*精确地应用强调和重音。*

另外，避免过长的带下划线的超链接，尤其是在衬线字体上，尤其是在较小的字体上。默认的`[text-decoration](https://reference.sitepoint.com/css/text-decoration "SitePoint.com CSS Reference Guide: text-decoration property"): underline;`将切断带有下行符号(主要是 g、j、p、q 和 y)的字形，使它们更难识别。一个方便的解决方案是不设置文本装饰，而是使用一个`[border-bottom](https://reference.sitepoint.com/css/border-bottom "SitePoint.com CSS Reference Guide: border-bottom property"): 1px;`或者只在鼠标悬停时使用`a[:hover](https://reference.sitepoint.com/css/pseudoclass-hover "SitePoint.com CSS Reference Guide: :hover pseudo-selector")`。

## 7.给铅字呼吸的空间；制定充分的措施和领导

从最后一条规则的结尾开始直接引导(没有双关的意思)，确保你的文本有足够的空间呼吸；给它所需要的垂直和水平的空间，以确保清晰易读的对比度。

检查每行不要太短，不要急不可耐地快速扫视下一行，但也不要太长，以免在到达一行的末尾时丢失，然后继续下一行。与行的长度齐头并进是最重要的:在行与行之间提供足够的空间，但不要太多以至于撕裂文本。

## 8.警惕不是为屏幕使用而设计的字体

扩展核心的网页安全字体是一个值得称赞的优点，但是只选择与你正在使用的媒体相匹配的高质量字体。世界上有大量的字体，其中只有一个子集适合设置正文，还有一个更小的子集已经过优化，或者专门为 web 使用而设计。

寻找好的[提示表](https://secure.wikimedia.org/wikipedia/en/wiki/Font_hinting "Wikipedia (EN): Font hinting")，在各种浏览器和操作系统上以较小的尺寸(10px、12px、14px)测试你的字体，以衡量各种字体渲染引擎显示字体的效果。

## 9.确保 webfont 资产已子集化并缓存

当使用 [`@font-face`](https://reference.sitepoint.com/css/at-fontface "SitePoint.com CSS Reference Guide: @font-face property") webfont 资产时，页面大小将会增加——用户代理(主要是浏览器)将需要下载字体，以便用它来呈现文本。减少这种加载大小的一种方法是对字体进行子集化:删除不需要的字符字形(例如，一个英文网站几乎不需要西里尔字母字形，反之亦然)。第二是建议用户代理将资源缓存一段时间，这样以后的访问就不会导致重新下载字体。

## 10.不要用漫画

更多的是个人情感和挫败感。但说真的，当有这么多其他字体可供选择时，包括像 [Comic Jens](http://klepas.org/comic-jens/#notebook "klepas.org: Comic Jens — no more excuses") 这样的免费许可字体，你为什么要使用这种过度使用且相当不吸引人的漫画字体呢？
