# 为你的网页设计客户服务

> 原文:[https://www.sitepoint.com/service-web-design-customers/](https://www.sitepoint.com/service-web-design-customers/)

**客服:如果服务员能做到，为什么 web 开发人员不能？**

构建一个跨浏览器兼容的 web 界面，与客户现有的会计和库存包集成，以便在线销售他们的产品？没问题。

在项目过程中与你的客户互动，确保他们对所得到的满意？问题！

众所周知，web 开发和客户服务很少能齐头并进。软件开发的技术细节和客户管理的社交细节之间的鸿沟很大。

但其实不必如此。

当你谈论客户服务时，建立一个网站就像在餐馆里上菜一样简单。

好吧，对于一些 web 开发人员来说，在餐馆里提供一顿饭也不是那么容易，但是我们可以假设我们大多数人至少在餐馆里被提供过一顿饭，所以我们已经对所涉及的概念有了一些理解。

现在让我们暂时放下怀疑，看看这个复杂的类比是如何展开的…

我们假设项目已经确定了范围，客户已经同意开始工作，是时候编写功能规范并开始构建了。

有点像顾客决定去餐馆吃饭，并且已经预订了。

为了给顾客上菜，服务员需要遵循几个步骤。如果我们在为客户建立网站时遵循这些步骤，客户服务就会变得非常简单。

##### 第一步:见面打招呼

当她的顾客到来时，服务员做的第一件事(我用一个女服务员来做类比，但在任何情况下我都没有刻板印象的意思)是介绍自己。这通常发生在顾客坐下、感到舒适并拿到菜单的同时。在这一点上，服务员把自己树立为一个在用餐期间照顾顾客的人。应该让顾客觉得，如果他们想要什么，她就是那个要问的人。

同样的道理也适用于你第一次接触需要网站的客户。当你开始一个新项目的时候，你应该做的第一件事就是和你的客户安排一次会面，唯一的目的就是介绍你自己，让他们知道你将负责他们网站的开发。花时间向客户解释项目将如何进行，你将成为他们的联系人，如果在项目期间他们需要知道或完成任何事情，你将是帮助他们的人。

##### 第二步:提供一份菜单

通常在服务员让顾客入座的同时，她会给他们一份菜单。这让他们对所提供的选择有了一个概念，并对将要供应的食物的味道有了一个大致的了解。

当你会见和问候你的客户时，你也应该有一些基础文件。有时，这可能是您的销售人员已经准备好的提案。有时候可能是你自己准备的东西。无论如何，这份文件应该给出一个非常广泛的概述类型的网站，你可以提供给你的客户。

##### 第三步:解释特色菜

一个好餐馆里的好服务员会向她的顾客解释当天的特色菜。理想情况下，她会熟记于心，并了解他们是如何准备的。此时，她的客户应该能够就他们不理解的事情提出任何问题。现在服务员有机会证明她知道自己在说什么。她可以向顾客解释说，coulis 只是一种水果酱的别称。

您应该能够向您的客户解释您可以提供哪些具体功能来帮助他们开展业务。您应该很清楚什么技术支持这种功能，以及如何提供这种功能。你还应该能够将专业术语翻译成日常语言。你可以向你的客户解释，html 只是用来写网页的语言的一个别出心裁的名字。

##### 第四步:接受订单

为了给顾客提供他们喜欢的饭菜，服务员首先要弄清楚他们想要什么。她不能匆忙完成这一部分，因为她做对了很重要。没有比端错饭更糟糕的了。她写下顾客点的菜是很重要的，这种方式可以向厨师解释顾客到底想要什么。

需求文档是成功的网站开发最重要的部分之一。重要的是，有人花时间与客户建立到底是什么，他们在追求。同样重要的是，这些请求要用简单明了的术语写下来，不要让开发人员误解。

##### 第五步:更换餐具

服务员的部分工作是知道什么餐用什么餐具。如果顾客点了汤，她需要一个汤匙。如果他点鱼，他需要一把鱼刀。服务员需要确保顾客拥有用餐所需的设备。

一个好的网站开发公司应该能够建议他们的客户有效运行和使用他们的网站所需的硬件和软件。建立一个客户看不到的杀手网站是没有意义的，因为他们没有支持它的浏览器。或者是因为没有人告诉客户他们需要一个可扩展的系统架构而无法处理它正在接收的流量。告诉客户他们的网站需要什么，如果可能的话，为他们争取。

##### 第六步:交给厨师

在大多数餐馆里，上菜的人和做饭的人不是同一个人。服务员有一套特殊的对人友好的技能，厨师有特殊的食物知识，知道如何准备食物以增加味道。一旦服务员接受了订单，她就把订单交给厨师为顾客做他们要求的饭菜。

网站也是如此。将开发留给技术人员和人们是一个好主意——取悦生产者、项目经理、销售经理或业务分析师。如果你是一个人，这并不总是可能的，但是如果你是，你应该能够把客户互动和技术开发分开。就像厨师喜欢在厨房做饭一样，开发人员也喜欢远离客户来完成他们的工作。

##### 第七步:让客户了解情况

如果由于某种原因，厨师没有及时准备好顾客的饭菜，服务员应该尽快通知顾客，并确保顾客满意。如果厨师被订单压得喘不过气来，饭菜从厨房出来的时间比平时长，服务员应该让她的桌子知道有短暂的延迟。她应该道歉，并提供给顾客一些别的东西让他们开心——更多的面包、更多的水、再来一杯酒？

网络制作公司也是如此。让客户知道任何延误是很重要的。理想的情况是，他们应该尽快知道任何问题，这样他们就可以决定采取什么行动。

##### 第八步:上菜

饭菜做好后，服务员的工作就是把它呈现给顾客。在高级餐厅，主餐的肉类部分应该放在离顾客最近的地方。切蛋糕时应该让尖的一端向内。展示非常重要。再好吃的饭，如果看起来很难吃也没人会想吃。

当你把你的网站交付给你的客户时，要确保展示符合要求。检查拼写、导航和设计的一致性。将网站展示给你的客户，向他们展示它是如何工作的。不要让他们自己去想。

##### 第九步:检查用餐情况

用餐后大约 5 分钟，服务员应该询问顾客是否喜欢，是否满足他们的要求。同样的事情应该发生在每一道菜上。在顾客吃过之后发现他们不喜欢这顿饭是没有好处的。纠正事情的机会将会过去，他们不会再回到餐馆吃饭，他们的朋友也不会。

确保客户对你的产品满意。没有人想发布一个没人喜欢的网站，所以在你还有时间改变它的时候，确保你在正确的轨道上。在发布前一天询问客户是否喜欢你所做的事情是没有用的，因为如果反应不好，你在这一点上也无能为力。尽你所能向客户展示你所能做的，并观察他们的反应。如果他们没有被打动，而且他们有理由不被打动，考虑改善情况的选择。

##### 第十步:提供更多课程

主餐结束后，如果顾客还没有太饱，服务员就该给他们上甜点了。

如果你已经交付了一个很棒的网站，并且没有用尽客户的软件开发预算，你可以开始谈论第二阶段了。如果你的服务到目前为止是好的，他们应该很乐意承诺。

##### 第十一步:交付账单

用餐结束后，服务员已经为顾客提供了他们可能想要的所有东西，她必须等着他们结账。账单应该清楚地说明顾客收到了什么以及每样东西的价格。一个好的服务员如果工作出色，可以得到丰厚的小费。

你给客户的发票也应该是你为他们做了什么的简明陈述。然而，在 web 开发中，等待客户来索要账单可能是不明智的。我不得不承认，在这个阶段，整个餐馆/网站的类比并不完全成立——在餐馆里，你不能让你的顾客离开，直到他们付款，在互联网行业，你对你的客户的控制有点不太明显。

财务慷慨在网站开发中也不常见。但也不是没听说过！一些企业提供有利可图的股权交易和激励措施。在所有情况下，如果你为顾客服务得很好，你可以期待推荐和回头客的回报，这比一次性的小费持续得更久。

##### 第十二步:说再见

一个优秀的服务员会用一句谢谢、一句再见和最后一句她希望顾客满意的话来结束她的服务。

web 开发中优秀的客户服务应该以同样的方式结束。项目应该正式结束，最好是对其成功进行回顾，应该感谢客户的业务关系，应该鼓励大家尽快再次合作。

这就是你要的——12 个简单步骤的客户服务。如果你的客户从现在开始对你的服务不完全满意，至少你将拥有转行的必要技能！
