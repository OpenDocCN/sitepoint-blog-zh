# 冷聚变一周:2 月 13 日至 19 日:隐藏的宝石

> 原文：<https://www.sitepoint.com/the-week-in-coldfusion-13-19-feb-hidden-gems/>

软件和信息行业协会[宣布了 2008 年 Codie 奖决赛选手](http://www.siia.net/codies/2008/), cold fusion 8 被列入最佳 Web 服务解决方案类别。获胜者将于 5 月揭晓。

一个针对 Mac 用户:ColdFusion 8 中内置的富文本编辑器使用 FCKEditor——但在 CF8 发布时，FCKEditor 不支持 Safari。[根据 Ben Forta](http://www.forta.com/blog/index.cfm/2008/2/15/ColdFusion-Rich-Text-Control-And-Safari) 的说法，下一个 CF 更新程序将“很快”推出，包括对 Leopard 和 64 位 Windows 的支持，将包括一个更新的库，增加对 WebKit 渲染引擎的支持。

Charlie Arehart 已经对所有 ColdFusion 播客进行了综述，包括那些已经开始和已经消失的播客。事实上，已经半途而废的比目前正在制作的新剧集还多！查理还将他在 [CFWACK](http://carehart.org/blog/client/index.cfm/2008/2/18/CF8_debugger_chapter_online) 中的章节以 PDF 格式发布在网上，在[上使用 CF8 调试器](http://carehart.org/blog/client/index.cfm/2008/2/18/CF8_debugger_chapter_online)。

一些技术上的东西——来自 ColdFusion 工程团队的 Rupesh Kumar 发布了代码，展示了如何使用 ColdFusion (通过 Java)从 swf 文件中读取元数据。对于动态嵌入 Flash 文件很方便——特别是为了获得正确的尺寸，正如 Rupesh 指出的。Anuj Gakhar 发现了另一个使用 Java 的隐藏瑰宝——使用一个内置库中的方法显示彩色编码源代码的[能力。](http://www.anujgakhar.com/2008/02/14/colored-code-with-coldfusion/)

Ben Nadel 创建了一个用户定义的函数，用于在 ColdFusion 中清除高 Ascii 值以实现网络安全，其灵感来自 Ray Camden 谈到的当 CF 遇到它不识别的字符时 RSS 解析中断。对清理微软 Word 输入也很有用。

对于那些对 ExtJS 框架感兴趣的人，Dan Vega 发布了一篇关于 ExtJS 表单入门的教程，内容包括创建和定制表单、使用 Ajax 提交表单以及使用 Ajax 加载表单数据。不满意 CF8 内置 Ajax 控件的默认外观？[托德·夏普分享了一个改变主题的小技巧](http://cfsilence.com/blog/client/index.cfm/2008/2/13/Sexy-ColdFusion-8-Ajax)。

如果您正在使用 CF8 中新的 CFPresentation 特性，您可能会对 Ben Forta 的这个[提示感兴趣:您可以通过编辑 XML 文件来更改默认颜色和字体。本提醒你先备份原件！CF8 的另一个新特性是对开源 Derby 数据库的支持。Sam Farmer 发布了](http://www.forta.com/blog/index.cfm/2008/2/13/Tweaking-ColdFusion-8-Presentations)[CFC 来简化创建新 Derby 数据库的过程](http://samfarmer.instantspot.com/blog/2008/02/13/How-to-Add-a-Derby-Database-with-DerbyCFC)。

最后，在框架舞台上——Bruce Phillips 发布了[Mach-II 1.5 版小学生指南——第一部分裸 Mach-II](http://www.brucephillips.name/blog/index.cfm/2008/2/9/The-Grade-Schoolers-Guide-to-MachII-Version-15--Part-1-Naked-MachII) 。

我上周提到，我已经收到了最新的 [Fusion Authority 季度更新](http://www.fusionauthority.com/quarterly/)的副本，但还没有时间做一个完整的审查。我现在已经读完了(并且很喜欢)，但是我认为[本·纳德尔已经用他的评论抢先了我一步](http://www.bennadel.com/index.cfm?dax=blog:1153.view)——对于他出色的概述，我没有太多可以补充的。
