# 使用 Java Studio Creator 快速开发 Web

> 原文:[https://www . site point . com/quick-and-dirty-web-development-with-Java-studio-creator/](https://www.sitepoint.com/quick-and-dirty-web-development-with-java-studio-creator/)

Sun 公司的最新努力是使 Java 成为一个对小型网络开发者更有吸引力的平台，这是以今年年中首次发布的 Java Studio Creator 的形式出现的。上周，宣布了一个免费的产品更新，现在可以支持 Mac 了，尽管据我所知，这个更新实际上从八月份就已经出来了。

抛开虚假的新闻稿不谈，Studio Creator 受到了批评，说没有简单易用的工具来快速组装 Java Web 应用程序而不接触代码，就像你可以在 Visual Studio .NET 中组装 ASP.NET 应用程序一样

Java Studio Creator 充分利用了设计 Java Web 应用程序接口的最新标准: [JavaServer Faces](http://java.sun.com/j2ee/javaserverfaces/) 。有了 JavaServer Faces，您不再需要用 HTML 表单标签来构建 Web 应用程序界面，而是可以使用一组更高级的**组件来为您生成 HTML 代码。**这些组件为跨请求的特定用户提供了验证用户输入和维护表单中的内容和选择的工具。

在 Studio Creator 中开发一个 Java Web 应用程序变成了一个非常快速而麻烦的过程，这通常正是你所需要的。建立一个数据库连接和几个查询，将几个界面组件拖到一个空白页上，调整它们的属性，然后双击编写 Java 代码，处理按钮按下和菜单选择等事件。

Java Studio Creator 实际上是建立在开源开发环境 [NetBeans](http://www.netbeans.org/) 之上的**，尽管像 [IntelliJ IDEA](http://www.jetbrains.com/) 这样的新竞争者越来越受欢迎，许多人仍然选择它来进行 Java 开发。**

我第一次看到 Java Studio Creator 是在去年的 JavaOne 大会上以“Project Rave”的名义演示的，当时它被用来快速组装和部署一个在手机上使用的应用程序。看起来这个功能，连同对开发桌面 Java 应用程序的支持，被搁置了，以利于使**成为一个可靠的、专注的快速 Web 应用程序开发工具。**

PC 杂志对最新发布的产品有一个快速但相当不错的评论。令人惊讶的是，更新**并没有在主要的 Java 网站上获得太多的报道。**我认为已经使用 Java 进行 Web 开发的人对这样的工具有点力不从心。然而，在[软件现实](http://www.softwarereality.com/reviews/creator_ea.jsp)可以找到对最初版本的可靠回顾。

Java Studio Creator 免费使用 30 天(T1)，之后你必须支付 99 美元购买该软件，这也可以让你获得一年的 Sun Developer Network 会员资格。
