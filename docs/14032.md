# 微软推出虚拟地球

> 原文：<https://www.sitepoint.com/microsoft-launches-virtual-earth/>

微软新的[虚拟地球](http://virtualearth.msn.com/)今天以测试版形式发布。虽然比预期的要早，但这一推出显然是对谷歌在其竞争对手[谷歌地图](http://maps.google.com)服务中的新混合视图的回应，它胜过了之前被吹捧为虚拟地球独有的标签航空照片功能。

虚拟地球是谷歌地图的直接竞争对手，它提供了一个免费的可平移、可缩放、DHTML 驱动的世界视图，可以在平面地图和卫星图像之间切换。

虚拟地球提供的用户界面比谷歌地图更具创新性，地图视图占据了浏览器窗口的大部分，浮动工具窗口出现在地图上。该界面允许你通过直接拖动地图，或者通过点击并拖动远离角落里的浮动指南针(微软称之为“游戏模式”)来四处平移。你也可以利用你的鼠标滚轮来放大和缩小。缩放是通过漂亮的动画效果完成的，在加载新影像之前，将当前显示的影像拉伸到新的比例。还包括一个“定位我”的功能，它尽最大努力查明您的当前位置使用您的 IP 地址或通过 ActiveX 控件。

在 Firefox 中玩 Virtual Earth 时，我遇到了几个与鼠标滚轮有关的 UI 错误(除了放大或缩小之外，地图视图还会向上或向下滚动)，但对于一个仓促发布的版本来说，它在跨浏览器支持方面相当稳定。为地图提供图像的服务器不太可靠——即使在北美的高速 ADSL 连接上，我也遇到了许多图像碎片，而这些碎片本应该是图像碎片。在大多数情况下，缩小然后再放大可以让这些图像在第二次尝试时正确加载。

与谷歌地图相比，虚拟地球相形见绌的地方在于其卫星图像的质量和国际覆盖面。尽管美国的主要中心都有不错的图像，但世界其他地方的图像分辨率似乎只有每像素 1 公里左右——这对导航毫无用处。此外，没有标签的高分辨率卫星图像莫名其妙地以黑白显示，我希望微软在服务更新中很快解决这个问题。

北美的道路地图非常好，但世界其他地区只能获得主要城市的标签。毫无疑问，随着时间的推移，这也会得到改善。

就 DHTML 而言，虚拟地球肯定比谷歌地图更令人印象深刻。但是如果你问我，我会说微软在可用性方面做了一些牺牲来实现其界面的花哨功能。对于像我这样精通 DHTML 的超级用户来说，这个界面当然很有吸引力，但是我不确定我的父母是否能理解。

**更新:**虚拟地球团队推出了一个[博客](http://spaces.msn.com/members/virtualEarth/)，让人们了解虚拟地球的最新发展。同时， [Via Virtual Earth](http://www.viavirtualearth.com/) 似乎是第一个致力于这项服务的第三方网站。
