Brad: 是的，没错，Allan，我认为你提到了主题框架，这是进入下一个主题的完美过渡，我知道如果你向不熟悉主题框架的人提到主题框架，听起来有点吓人，所以我们为什么不解释一下。什么是主题框架，它和标准的 WordPress 主题有什么不同？

一个主题框架本质上是一个增强的父主题。就 WordPress 而言，这意味着在你的主题文件夹中有你的父主题或框架，其中有你所有的主要代码，有你所有的模板文件，你的主要功能，这基本上就是你的 WordPress 主题的核心。当你把它作为一个框架使用时，你可以把它和一个子主题联系起来，基本上你可以用它来构建父框架主题。它允许你对框架主题进行非常动态的，甚至是复杂的修改，而不需要编辑主题文件本身。这样做的好处是，当 WordPress 更新时，甚至主题更新时，你也不会丢失那些改变。一般来说，如果你选择一个主题，然后直接对它进行调整，假设你在这个主题上工作了两个月，然后这个主题进行了升级，增加了许多新功能，当你进行升级时，你基本上丢失了所有的更改，这很糟糕(笑)，当你花了很长时间在某个东西上，然后你进行升级，所有的更改都消失了，这绝不是一件好事。因此，使用这个框架模型，你基本上可以使用子主题来完成所有的更改，所以你的更改或多或少会被编译到子主题中，然后父主题或多或少会保持不变，这样它就可以在你设计风格时经受住 WordPress 的升级、更改和改进，你知道，你的自定义功能都单独存在于子主题中，这样你就不会丢失更改。它有许多不同的实际实现，我提到了它们两者，只是为了澄清一下，任何主题都可以是父主题，这意味着你可以从任何主题构建子主题，所以任何主题都可能有更新或任何东西，当你准备好调整它时，你应该立即将其作为子主题启动。说到框架，这是一种独立的交易，框架或多或少是一种，就像我前面说的，一个加强版的父主题，它或多或少是用自定义编码构建的，允许你用你的子主题做更多的事情，而不是用一般的父主题。但是，是的，我认为它们真的很强大，我在所有基于客户的工作中使用它们，甚至我销售的一些主题都是基于——都是建立在框架之上的子主题。是的，我认为这是一种非常好的开发方式，速度非常快，它使整个开发过程非常快，因为你不必像平常一样重复那么多代码，如果你必须返回，基本上是在不同的模板文件中复制和粘贴代码，你不必担心子主题框架关系，因为主题太多了， 或者是因为大部分的编码已经为你处理好了，所以你想做的任何改变都只需要在子主题中完成，它只是改变了框架主题，使其具有很强的可扩展性。 所以，是的，我喜欢它们，我一直在用它们。真是啰嗦啊。
