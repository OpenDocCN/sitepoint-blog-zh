# 代码旅行:路的尽头

> 原文：<https://www.sitepoint.com/code-safari-end-of-the-road/>

这是我在 Ruby Source 上的代码 Safari 系列的最后一篇文章。我真的很喜欢写它，并尝试了一种新的格式，我希望你会觉得有用。在我的最后一篇文章中，我想留给你一些规划你自己的 safari 的技巧，因为这确实是整个系列的内容:不是你典型的如何做，而是一瞥我和许多其他开发人员每天用来弄清楚我们的工具如何工作的工作流程。

这里有五种做法，你应该融入到你的日常工作中。

### 读取代码

我从来不在没有仔细阅读源代码的情况下使用 gem 或第三方库。GitHub 和`gem unpack`很容易获得，没有理由不这样做。通过阅读代码，您不仅学到了很多东西，而且还验证了代码以适合您的项目的方式做了它所声称的事情。

### 从不崇拜货物

囤货是指在不理解的情况下复制你见过的技术或习语的行为。Ruby 充满了好的和坏的技术，当你第一次遇到它们的时候可能会感到困惑。

```
!!my_variable
<Hash[%w$he ll? [iuo]$.map {|x| [x, "hello"[/#{x}/, 0]] }]
class &lt;&lt; "abc"; self; end
```

总是敞开心扉去验证你对正在发生的事情的假设。

### 教导他人

我坚信，如果你不能向别人解释一个话题，那么你就没有完全理解它。教学有多种形式:写作、结对、白板，但所有这些都涉及概括和抽象概念，这是真正理解正在发生的事情的重要一步。

如果你没有博客，这是最简单的开始方式。很多人担心自己没有什么有趣的东西可说，但这无关紧要。花时间写下你学到的一项新技术或者你发现的一个 bug，这迫使你确保你已经完全探索了它，并且巩固了你的理解。

### 做笔记

我在桌子上放了一支笔和一个笔记本，在我编程的时候，我会写下任何让我感兴趣、困惑或愤怒的事情。我可以稍后重新查看这个列表，而不会被当前的任务分心。这对于保持注意力非常重要，同时也给你一个丰富的选择空间，让你下次有时间的时候可以研究一下(没有理由不休息！)

### 没有什么是太大的

代码可能会令人生畏，尤其是在你学习的时候，但是不要害怕看起来难以理解的代码库。总有下一步在你的掌握之中，即使只是重新格式化一些缩进，或者用`puts`语句验证一个假设。Ruby 对此特别友好，因为你可以只用一个文本编辑器就可以深入到你的系统中，并临时修改它们以弄清楚正在发生什么。一个方便的工具是`$LOAD_PATH`和`$LOADED_FEATURES`全局变量，它会告诉你已经加载了什么以及从哪里加载的。
