# 像素字体是 WDS08 的热门话题

> 原文：<https://www.sitepoint.com/pixel-fonts-a-hot-button-topic-at-wds08/>

[![Derek Featherstone](img/05db30e2eca91bb0600467b2e9960682.png)](http://www.flickr.com/photos/jcroft/2886749135/) 在上周于悉尼举行的 [Web Directions South 2008](http://south08.webdirections.org/) 会议上，Jeff Croft 在他关于[优雅的 Web 排版](http://www.webdirections.org/resources/jeff-croft-elegant-web-typography/)的演讲中提到，对于他的许多项目，他已经过渡到以像素指定字体大小，而不是像 ems 那样的[相对度量单位](https://reference.sitepoint.com/css/lengthunits)时，引起了一些人的惊讶。

传统上，网页设计者避免用像素指定字体大小，因为需要更大字体才能阅读的用户不能调整文本的大小。今天，每个主流浏览器都提供了页面缩放功能，甚至可以使用像素字体，但一个仍在普遍使用的旧版本浏览器不提供这种功能:Internet Explorer 6。

像素字体大小使设计者更容易将文本行的高度与页面上的其他设计元素相匹配，并实现统一的[“垂直节奏”](https://www.sitepoint.com/typography-baseline-rhythm-deciphered/)。使用相对字体大小也可以达到同样的效果，但是对于设计者来说，其中涉及的数学问题要复杂得多。

Jeff 认为，需要调整所有文本大小的用户可以简单地升级到当前的浏览器版本，因此对于大多数项目来说，像素字体是公平的。然而，他承认，在他有一点额外时间的项目中，或者在可访问性是一个特别关注的地方，他现在仍然会使用相对字体大小。

这种解决问题的方法并没有被会议上的许多观众接受。

第二天，在 T2 的一次采访中，我问网页可访问性专家德里克·费瑟斯通对这场争议有什么看法。他是这么说的:

> 基本上我还是要用相对单位，直到 IE6 消失。我不知道那会是什么时候。我知道我用来衡量我是否支持某个特定浏览器的一个标准是，当微软基本上说他们不再支持该浏览器或任何技术时，无论是浏览器、操作系统还是其他任何东西。当他们停止对它的支持时，那就是我也不再支持它的时候。
> 
> [……]Jeff 的假设是，需要调整文本大小的人会选择更好的工具，对我来说，这仍然是一个跳跃。我愿意相信，真的。不过，我认为，就最终用户教育而言，我们还有很长的路要走，因为人们不…可悲的是，有很多人使用电脑，他们不一定知道他们可以更换浏览器。他们使用他们电脑上的东西，这就是故事的结尾。
> 
> 现在，话虽如此，我们现在的处境，我的意思是，我有三个孩子；他们在四岁、七岁和九岁时对电脑的了解可能比我十九岁时还多，只是因为我年轻时我们不用电脑。所以，时代确实在变。我们仍然有一群人不是在电脑的环境中长大的，他们不一定会感到舒适。这是我们的问题吗？这个问题真的很好，我也不确定。
