# 十年来的五大网络技术

> 原文：<https://www.sitepoint.com/top-5-web-technologies-decade/>

过去的 10 年去了哪里？1999 年底，我们生活在对千年虫的恐惧中，互联网泡沫即将破裂。这十年见证了网络从一个笨拙早熟的孩子成长为一个有能力但不可预测的青少年。原创和创新技术不断涌现，我们现在有各种各样的浏览器、移动设备、HTML5 等。NET、PHP 5、Ruby on Rails、Adobe AIR、jQuery、web 服务、RSS、GMail、谷歌地图、维基百科、YouTube、脸书、Twitter、iPhones，以及基于网络的操作系统的兴起。

选择前五名是一项艰巨的任务，但在我看来，这些是在过去十年中产生重大影响的网络技术。事不宜迟，以下是我的获奖名单。

## 5.宽带和 Wi-fi

1999 年，只有少数幸运的人体验到了宽带的奢华——即使是这样，与今天的速度相比也是相对较慢的。我们大多数人都在与 56K 调制解调器、不可靠的速度和掉线做斗争。

宽带现在被认为是必不可少的公用事业。速度和带宽是显而易见的优势，但也许最重要的因素是网络总是可用的。宽带始终开通，wi-fi 让您轻松连接。从各种设备访问网络时几乎没有延迟；它变得易于使用，成为生活中必不可少的一部分。

## 4.萤火虫

在这个列表中包括一个 Firefox 扩展似乎有点奇怪，但不要低估 Firebug 的影响。在 1999 年，使用 IE 提供的工具进行 JavaScript 和 CSS 调试几乎是不可能的。如果没有 Firebug 和许多模仿其功能的工具，Web2.0 应用程序的发展肯定会更慢。

## 3.埃阿斯

异步 JavaScript 和 XML 不是一种技术，而是一个概念，它描述了如何在不使用页面刷新的情况下开发交互式 web 应用程序。严格地说，你不需要异步代码、JavaScript 或 XML，但是杰西·詹姆斯·加勒特在 2005 年的文章引起了全世界 web 开发者的注意。

我们还应该感谢微软在 1999 年末开发了 XMLHttpRequest 对象；这是大多数 Ajax 代码背后的核心技术。具有讽刺意味的是，XMLHttpRequest 为现在与微软视窗软件直接竞争的[网络应用](#webapps)提供了动力。

## 2.网络应用

Web 应用程序，或软件即服务，是一个已经存在了 10 多年的想法，但是名称已经随着时间的推移而改变。在 20 世纪 90 年代，瘦客户机计算是甲骨文推翻微软的宏伟计划，但更便宜、更快的 PC 硬件阻止了这一想法成为可行的成功。

在这十年中，使用 Ajax 的 Web2.0 应用程序已经发展到你需要的唯一桌面应用程序就是一个 Web 浏览器的地步。大多数一般的计算任务都可以在云中处理。2010 年，谷歌将发布[Chrome OS](https://www.sitepoint.com/google-launch-chrome-os-next-year/)——一个纯粹基于浏览器的操作系统，有可能颠覆 Windows 的市场主导地位。

## 1.Mozilla Firefox

今天，火狐被认为只是另一个浏览器。然而，它是十年来最具颠覆性的技术，为网络产业提供了一个转折点。2002 年，Internet Explorer 6.0 占据了 95%以上的市场份额，而网景被打败了(除了反垄断问题，网景 4 是一个糟糕的浏览器，应该被 IE 打败)。微软放弃了浏览器开发，认为“智能客户端”才是未来。当时，web 应用程序笨重且难以开发——智能客户端是具有丰富界面的 Windows 应用程序，利用了 web 的连接性。

尽管 Firefox 开始只是一个实验，但它为开发者提供了工具 T1 来创建快速的 T2 Ajax T4 网络应用 T5，并给了其他软件制造商生产他们自己的浏览器的信心。如果没有 Firefox，Web2.0 可能永远不会出现。它改变了业界对浏览器是什么以及它能实现什么的看法。它配得上第一的位置！
