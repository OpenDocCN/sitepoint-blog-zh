# 用户体验大师杰夫·维恩访谈

> 原文:[https://www.sitepoint.com/interview-jeff-veen/](https://www.sitepoint.com/interview-jeff-veen/)

杰弗里·维恩(Jeffrey Veen)是一位备受瞩目的作家、演说家和用户体验顾问，他曾参与设计过 [Flickr](http://flickr.com/) 、 [Blogger](http://blogger.com/) 、 [Typepad](http://typepad.com/) 和 [Google Analytics](http://analytics.google.com/) 等网站。Jeff 花了一些时间与 SitePoint 的 Josh Catone 一起回答了 SitePoint 论坛社区提交的一些问题。

澳大利亚读者可以看到 Jeff 在 9 月悉尼举行的 Web Directions South conference 上的演讲。SitePoint 的读者有资格享受票价折扣——在[这篇博文](https://www.sitepoint.com/learn-javascript-from-the-master-at-web-directions-south/)中阅读更多细节。

SP: Jeff，感谢你同意回答 SitePoint 社区的几个问题。

没问题，乔希。

在 Adaptive Path 做客户工作，你在那里创造了自己的产品( [Measure Map](http://www.measuremap.com/) )，然后转移到谷歌的企业文化，这两者之间的差异一定很大。你能谈谈你在这些对比鲜明的工作环境中的转变吗？是什么样的？

在某些方面，这是一个非常简单的转变。当我们创建 Adaptive Path 时，我们的目标是为我们能找到的最有趣的客户群做尽可能高质量的工作。我们非常幸运地做到了这一点，选择了满足这些目标的项目。我很快开始关注初创公司，因为我喜欢这种势头。

在与 Blogger、SixApart、Technorati 和 Flickr 等公司合作时，我意识到他们所做的那种快速、专注于网络的产品开发让我非常满意。最终，自适应路径启动了测量地图项目，试图实现这一目标。

我们有一个很棒的团队来开发这个产品，所以向谷歌的过渡很顺利。我们被收购的明确任务是重新设计分析，谷歌的人专注于帮助我们实现这一目标。他们让我们独自融入分析团队，并迅速开展工作。从本质上讲，我们能够保持创业时的工作方式。

你能分享一些有效的团队沟通策略，让大家保持一致吗？一个设计师需要对后端流程有多少了解才能与程序员交流，编程部门需要掌握多少设计原则？

我越来越不关心设计师和程序员之间的区别。相反，我喜欢小型的、高度积极的团队，他们可以从头到尾在整个过程中一起工作。一旦你让不同的人在一个项目的不同阶段工作，你就陷入了过程和人工交流的泥沼，比如规范、产品需求文档和风格指南。

相反，我试图与专注于积极用户体验的程序员和拥有强大技术技能的设计师合作。如果两者之间有足够的重叠，那么这一切就模糊成了一个团队，他们可以很好地合作，制造出伟大的产品。在 Measure Map 团队中，一些最好的设计见解来自我们的 Ruby 开发人员，我们的界面设计师编写了大部分可视化代码。所以真的，我不太在乎角色和责任。

我意识到这个理论不太容易适用于大型组织。但是，简而言之，这是谷歌的关键优势之一。他们有效地授权小团队完成创新工作，为他们提供资源以确保他们成功。这并不意味着创建某种形式的施乐 PARC 或雅虎砖房实验室，在那里你通过将有才华的员工从组织的其他部门中剔除来奖励他们。相反，这意味着将产品决策权委托给最接近用户的人。随着他们继续成长和成熟，保持这一优势将是谷歌面临的最大挑战。

最近 Gmail 和亚马逊网络服务的中断让一些人质疑转向云的明智性。当你所有的数据都在某个服务器上时，用户体验会因为云不可避免的关闭而降低吗？

云服务中断既痛苦又不足为奇。这是从本地存储过渡的早期阶段，因此服务中的每个小故障都会对我们试图建立的脆弱信任产生不适当的影响。但我也把它想象成进入我家的电。在极少数情况下，它会失败。我们坐在没有电视的黑暗中，担心着冰箱里的食物。然后它又回来了。但是依赖 it 作为重要服务的人有多种备份和应急措施，例如:医院和工厂。

云服务将会成熟并达到 99.9%的正常运行时间。但这仍然意味着每年有 9 个小时的停机时间。因此，依赖完美的应用程序将需要自己构建，或者开发自动切换提供商的方法，以实现不间断的服务。我不确定云计算的柴油发电机是什么，但我知道人们正在努力。

Mozilla 基金会最近邀请设计师们提出他们对网络未来的看法。Adaptive Path 是该项目的最初参与者之一，它展示了一种未来的 web 浏览器体验，这种体验非常注重社交互动。你认为浏览器及其界面的未来会怎样？

我没有参与极光项目，但是我的同事们取得的成果给我留下了深刻的印象。他们对浏览器未来的展望提出了一些非常关键的见解:个人数据的开放性和便携性，日益无缝的在线协作，以及适应用户行为的工具。

至少对我来说，他们提出的愿景实际上与浏览器中的 UI 组件关系不大。相反，我认为它指出了出版商需要发生的所有变化。用户越来越认为他们在网上创造的东西是他们的，他们应该能够随心所欲地处理这些东西。如果我想把我在脸书之外输入的朋友的联系信息带到其他地方，那是我的决定。我的购买历史、书评、照片、书签，以及其他我在网上产生的东西也是如此，不管是明示的还是暗示的。在所有创新的用户界面下，Aurora 毫不费力地展示了这一切的例子。

杰夫，非常感谢你抽出时间和我们交谈。祝你的澳大利亚之旅一切顺利——我最近参观了 SitePoint 办公室，40 个小时的运输时间令人难以忍受！

我的荣幸，乔希。是的，长途飞行是艰难的，但会议总是很棒，所以绝对值得。
