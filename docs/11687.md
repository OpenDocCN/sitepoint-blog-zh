## 采访记录

凯文:2009 年 9 月 11 日。在今天的节目中，SitePoint 最新著作的作者[《CSS 选集》第三版](https://www.sitepoint.com/books/cssant3/)。这是 SitePoint 播客#27:瑞秋·安德鲁的 CSS。

凯文:欢迎瑞秋·安德鲁来到 SitePoint 播客。

瑞秋:你好。

凯文:你为什么不向我们的听众介绍一下你自己。

Rachel: 我是一名英国网络开发人员，也是几本 SitePoint 书籍的作者，包括 CSS 选集。

凯文:公平地说，你是 SitePoint 最多产的作者之一。你已经写了三个版本的 CSS 选集了。你和我在《T2》中合作过，你所知道的关于 CSS 的一切都是错的。我错过了什么吗？

有一本关于 Dreamweaver 的书。

凯文:是的， [Dreamweaver 8](https://www.sitepoint.com/books/dreamweaver1/) 的书。

是的，我确实像 SitePoint 一样重写了原 CSS 书。

凯文:没错，你做了第二版的 [HTML 乌托邦](https://www.sitepoint.com/books/css2/)。没错。所以你在 SitePoint 写的书比任何人都多，然而，写书并不是你的日常工作；你是专业开发人员。事实上，你经营着一家网络开发公司。所以我想我的问题是，为什么要写书？

瑞秋:上周有人问我这个问题。我真正开始是因为我写了几篇人们喜欢的文章，当时 Wrox 的一部分 Glasshouse 的一个人说，你能写几章吗？

实际上，我完全是艺术出身，所以写作是我非常喜欢做的事情，对一个程序员来说可能不太寻常，真的。所以从那以后，我开始写了几个章节，最终完成了 CSS 选集，这是我写的第一本完整的书，只写了我自己。

但是我很容易就入门了，而且我很喜欢这样做。这当然不是我的主要工作，然而，如果我不做另一份工作，我就不能做它，因为我写的所有东西实际上都是我在做我所做的工作时发现的事情或发现的小窍门。

凯文:你有没有发现，因为你一直在写书，这给了你一个借口来花时间更新你的技能，并掌握最新的变化？

我想是的，尤其是在我的实际工作中，我做的后端开发比前端多得多。所以我显然也做了大量的 CSS 工作。

凯文:我们真的应该让你写一本关于 PHP 的书。

瑞秋:这实际上是我每天都在做的事情，这是 PHP，也是我做的事情。随着时间的推移，我对后端越来越感兴趣，这也是我大部分时间都在做的事情。

事实上，我想如果不是因为写作，我可能不会像现在这样关注 CSS，因为很明显，就即将出现的新事物而言，我们不能总是使用它们，我们倾向于在工作中构建的那种东西，你必须在各种不同的浏览器和事物中非常健壮。所以这给了我一个借口去看看接下来会发生什么。你知道，我真的没有太多的理由去看 CSS3，例如，我们在工作中做的大部分事情。所以这是一个很好的方面，我可以做一些研究，玩一些我不会看的东西。

你知道，我会找任何借口和你聊天，瑞秋，但我们今天在这里聊天的原因很明显是 CSS 选集的第三版已经出版了。你提到，当那是你第一次独自完成的书时，你对写作有了一个相当温和的入门，你是这样说的吗？

是的。嗯，我认为，CSS 选集的好处，我认为，作为一个作者，也作为一个读者，我得到的反馈是，它都是那些小部分。你知道，这是我如何做这件事，然后有一个解决方案，实际上作为一个作家，我认为，这使它很容易写，因为你可以坐下来说，“好吧，我现在要写这个解决方案。” :是的。
  **瑞秋:**从某种程度上来说，他们真的是自己写的，因为这是我在工作中做过的事情，我想，“是的，这将是那本书的一大亮点”，因为它很有用，几乎就像写教程一样，这本书的结构也很像那样。这是个别的解决方案，你不必阅读整本书。
 And 那么用那种方式，写得真好。我认为，如果你试图从头到尾编织一条共同的线索或者构建一些东西，那么写一本书就更难了。这是一本以这种方式写作、阅读和消费都相对较快的书。你不需要觉得你必须坐下来，投入大量的时间来完成整个事情。

Kevin: SitePoint 已经完成了一些这样的选集，这些问题和解决方案的书籍。你对那些打算学习 CSS 的人有什么想法？这是一个好办法吗？

你知道，我想经常是这样。我认为有一些非常基本的概念你需要记下来，我认为 [HTML 乌托邦:无表设计](https://www.sitepoint.com/books/css2/)这本书非常好，因为它确实带你了解了 CSS 非常非常基本的概念。我们在选集里没有做太多，虽然在开头有一章，概述了使用的术语和事物。但是一旦你有了基本的理解，我认为动手是最好的学习方式。你知道，人们经常问我他们如何学习，他们如何开始这样做，我会说你必须这样做。把基本思路记下来，然后开始玩。我认为选集非常适合这一点，因为你可以说，我如何设计列表的样式，如何设计数据表的样式……或者其他任何东西，你可以在样式表中进行调整和修改。这是掌握事物如何运作的一个很好的方法。

凯文:关于学习，尤其是客户端网络技术。我不知道你是否可以在任何网站上查看源代码，但我看到很多人学习这种东西的过程是他们将学习非常基础的东西，你知道。CSS 包含在一个名为。你用一个 [<链接>](https://reference.sitepoint.com/html/link) 标签和一个规则链接的 css 大概是这样的。然后他们会去玩其他人的代码，复制粘贴，尝试一些东西。这样他们会在旅程中走一定的距离，突然他们会想后退一步，好好学习。我似乎可以看到有人拿起一本像 HTML Utopia 这样的 CSS 入门书籍，阅读第一章，然后开始阅读像 CSS 选集这样的书籍。如果他们想深入了解级联工作原理之类的东西，你可以回到教程风格的书。但是对我来说，CSS 选集似乎特别适合通过例子来学习，一次解决一个问题，很多人似乎都去了。

是的，我认为——当然也包括我这一代的任何人，我现在开始觉得自己像一个网络老太太了。你知道，这就是我们学习的方式，因为没有别的东西。

你知道，当我开始学这个的时候，除了非常非常基本的 HTML 之外，很少有真正的老师教我怎么做。当然，当 CSS-P 出现的时候——我的意思是，我很早就在用它构建东西，实际上什么都没有。你知道，有很少的教程和东西，但它真的只是人们在做它。你会去看看他们做了什么，试一试，看它坏了，然后摆弄它。我认为，我们这些对这种东西有非常非常深刻理解的人，来自那种只能通过观察和摆弄来弄清楚的一代人。我认为现在有很多东西存在，其中一些可能相当令人困惑；你知道，如果你想学，从哪里开始。

你知道，我认为选集也过滤了很多。我的意思是，这一直是我的想法，我会看看人们提出的所有不同的技术和不同的做事方法，并尝试选择那些在所有浏览器上都能正常工作而不会有太多麻烦的东西。因此，你知道，其中一些不是最漂亮、最优雅、技术上最完美的解决方案，但对于那些没有时间花在调整每件小事上的人来说，它们是跨所有浏览器工作得最好的解决方案。

Kevin: 你提到了 CSS-P——CSS 定位——我认为这是这本书特别擅长的，因为有两三种方法可以实现任何给定的 CSS 布局，实际上，哪一种最适合这项工作取决于几个约束条件。这本书有一章是关于 CSS 布局的，如果你有这些限制，这是最好的解决方案。如果你有这些约束，那就是最好的解决方案。

蕾切尔:是的，我的意思是……很多事情都来自于工作中的实践，你知道，找出哪些事情是最可靠的。你知道，如果你需要做这种布局，这是一个很好的起点，显然会有例外，但你需要一个起点，特别是当你刚刚开始尝试并试图找出你见过的所有技术中哪一种最适合使用时。

凯文:这是这本书的第三版。这是 SitePoint 最受欢迎的书籍之一，这也是我们想要第三版的原因，但从作者的角度来看，你更新这本书，写第三版有什么道理呢？

瑞秋:我认为这是一个非常有趣的观点。当我回到第一版的时候，Netscape 4 还在现场，不是作为一个主流浏览器，但仍然是一个有足够多的人，特别是进入大学和事物的人在使用的浏览器。

凯文:是啊，我真不敢相信 SitePoint 出版了涵盖网景 4 的书籍。

我知道！我知道。

凯文:现在看来，这似乎是很久以前的事了。

是的，这让我觉得自己很老。第三版就像你的孩子在成长。

然后我们做最后一版的时候，我们在讨论 IE7 之类的东西。我们并没有真正放弃对 IE6 的任何形式的支持，它仍然是一个非常普通的浏览器。我认为在这一点上，考虑到一本书的寿命可能是 18 个月左右，我们开始看到 IE6 正在下降。当然，在工作中，我们能够开始对客户说，“嗯，是的，我们可以做所有的华而不实的事情。我们将不得不为 IE6 放弃其中一些。”因此，这可能是使用 gif，而不是透明的 png 或各种技术。但我们只是稍微放弃了这个。我们不会失去对 IE6 的支持。我们不是说，“如果你有 IE6，你不能浏览我们的网站。”我们要说的是，这种顶级视觉设计不会很好，因为它不支持这些东西。

如果你只有 10%的用户，比如说，在 IE6 中，那么你可能不想投入大量的开发预算来支持这 10%，只要这 10%的用户对网站有良好的体验，并且可以查看所有内容。

所以我想这就是我在 CSS 选集里某种程度上走的路。我仍然非常希望所有的解决方案都能在 IE6 中运行。所以我使用了不被支持的东西——一个很好的例子是 alpha transparency PNG 我已经解释了为什么它不工作和解决方法。无论你是使用一些黑客手段来强制透明，还是像我们现在在工作中经常做的那样，退回到使用 gif，因为有时只是改变背景图像并使用 gif 来消除透明的需要。

凯文:我很好奇；最近，我们在播客中多次谈到了 Internet Explorer 6 的状态。我很好奇现在顾客对这种情况的反应。当你和一个客户谈话时，你认为 IE6 是一个有效的下层浏览器，会有什么样的反应？他们现在接受了吗？

雷切尔:实际上是这样，老实说，我认为这纯粹是出于经济原因，因为通常情况下，你可以争取获得 IE6。即使是像 alpha transparency 这样的东西，这确实是一件需要很长时间的事情，如果你决定用 PNG 修复程序来破解它，这将导致各种其他问题…如果你仍然说，“嗯，我们可以尝试这样做，或者你可以缩小你的设计。”他们看着它，想，“好吧，我们可以设计出我们想要的设计，让它在 90%的浏览器上完全按照我们想要的那样工作，然后也许再缩小一点。”当他们从纯粹的实用和商业角度来看时，花费大量时间是没有意义的，因为这可能是一个非常复杂的设计。在一个即将消失的浏览器上花费的时间可能会非常多…

Kevin: 你能为他们量化这些吗？你是不是不得不说，“看，如果你想让它在 IE6 中看起来这么好，那就要多花这么多钱。”

雷切尔:我已经开始实际说明它的成本，而不仅仅是把它包装在设计中。因为对于一些客户来说，他们可能会看着他们的统计数据说，“嗯，实际上我们现在只得到 5%的人。”只是看地点。

一些进入企业市场的东西可能会有更多。但是，这种设计通常更容易实现。它往往是更前沿的东西，尤其是针对年轻人市场的东西，或者是他们真正想去的地方，并真正推动设计的发展。通常，这些市场没有太多的 IE6，因为他们是可以升级的个人，他们得到了更新的计算机，因此得到了更新版本的操作系统和更新的浏览器。所以我认为有必要向客户详细说明这一点，告诉他们“我们可以做到。我不是说这在技术上是不可能的。”我们对各种各样的事情都这样做，尤其是当我们谈到后端时。人们说，“你能做这个吗，你能做那个吗？”你会说，“嗯，是的，从技术上来说，我们可以做到，但这需要多长时间，值得吗？”

我觉得和前端设计一模一样。所以，当我整理提案时，我会试着说明资金的去向，这样就不会被卷入一些大的前端设计中。我们确实与设计机构合作。我在一家公司工作，这家公司专门为设计机构做网页开发。因此，我们的处境可能比那些必须向真正的最终客户推销的人稍微好一点，因为我们正在与那些确实了解这一切是关于什么以及这意味着什么的设计师交谈。

凯文:这本书的一个特点是 IE6 逐渐退居幕后，我想，新的浏览器将会是 IE8。目前为止你对 IE8 的印象如何？

老实说，日复一日，我对 IE8 没有任何问题。我知道人们发现的各种事情，每个浏览器都有漏洞。但实际上，作为一名 web 开发人员，在我的工作中，我通常可以构建一些东西——我在 Firefox 中工作，这是我的主要浏览器。我可以在火狐测试一下。我可以在 Safari 里看。我可以看歌剧。我在 IE8 中查看，它们看起来都一样。我很少遇到 IE8 特有的问题，这很好，因为这是我们都要求的。当然，你会遇到一些错误，有时你会在 Safari 或 Firefox 中遇到一些东西，你会想这到底是怎么回事？

凯文:我们有很多相同的经历。几乎每次我们在 IE8 中遇到问题，都是因为我们为 IE7 所做的事情…

瑞秋:是的。

凯文:…现在在 IE8 中这是不必要的。我想 IE7 也是如此。它正在修复令人痛苦的大 bug。IE8 也做了相当多的工作，但是 IE8 中有什么新的东西是你发现自己正在利用或想要利用的，因为 Safari 或 Firefox 的新版本出来了，我们正在谈论 web 字体，我们正在谈论能够给事物添加阴影，但是新版本的 Internet Explorer 出来了，看起来我们在谈论，“哦，我不用再处理那个 bug 了。”

雷切尔:我认为是这样。我认为真正的原因是人们玩 Safari 和 Firefox。他们经常有这些——特别是 Safari 很早就引入了一些对我们网络开发者来说很有趣的东西，但实际上，我们可能不会用在真正的网站上。我们没有在工作中使用，因为我们知道没有足够多的人真正拥有这个浏览器。

我认为 IE8 中的东西也是如此——当然，CSS 表格的东西已经存在于其他浏览器中了。我们目前还不能在生产工作中大量使用它，因为有太多的人使用 IE7，甚至 IE6。我们倾向于做的工作——我对这种事情非常务实，它必须有效，我会使用有效且无损害的方法，我会这样做。这往往是我带着这些东西去的地方，因为这些网站将存在很长一段时间。我还在维护我六年前建立的网站。我知道这些东西会存在很长一段时间，因此，我们正试图以一种非常稳健的方式来构建。

我认为，对浏览器中的新事物感到兴奋的问题是，你会感到非常沮丧，好像事情进展得不够快。我希望能够使用这些东西。所以我认为这是平衡的说法，是的，看看这些新的很酷的东西，但我也必须做我的工作。

凯文:你是否发现自己处于这样一个位置，比如说，当你谈到我们要淘汰 IE6 时，我想那时你会想，哇，现在我们可以开始使用除了 IE6 之外的所有东西都支持的东西了。当你淘汰旧的浏览器时，你的酷新功能就出现了。

是的，我认为是这样的，对我来说，通常只是因为 IE6 我们不能使用选择器之类的东西，或者如果我们使用它们的话，我们现在不得不用 JavaScript 替换它们。我认为，实际上正是这样的小事造成了最大的不同。不需要在你的第一段上贴一个类来让它看起来与众不同。当你开始说“是的，如果 IE6 的第一段没有那么大也没关系”时，你会注意到这一点。

所以我认为有两件事。我是那种喜欢看所有新出的很酷的东西并玩它的极客，还有一种人只需要经营一家企业和做一份工作。从这个意义上来说，往往是小事情造成了最大的不同，比如当我们不必担心某样东西是否有布局时，它会很可爱。

凯文:当然。说到 has 布局和 hacks 之类的，在我看来，CSS 总的来说……浏览器之间的竞争似乎有点公平了。当然，对黑客的需求比以往任何时候都少。就你自己来说，你是 [CSS hack](https://reference.sitepoint.com/css/hacks) 的人还是[条件评论](https://reference.sitepoint.com/css/conditionalcomments)的人？

瑞秋:这取决于工作。我仍然坚持认为，如果 IE6 中有两件事需要修正，那么使用 [`* html`](https://reference.sitepoint.com/css/workaroundsfilters) 是完全没问题的，因为这两件小事你放在你的样式表中，你不必给文档添加任何额外的东西。在你有很多东西的地方，例如，如果你正在破解透明性或其他东西，那么我会使用条件注释，因为这意味着我可以得到 CSS 和 JavaScript 以及其他任何东西；都在偏僻的地方。

所以我认为答案是视情况而定。我认为我们做的大多数事情…我们最近做的，有大量的 IE6 修复，所以，是的，我们倾向于把它们放在附加条件注释的东西里。我对这种让评论变得可分析的事情有一点意见，因为它已经发生了。我不确定这是不是一个很好的方法，但是我认为对于很多事情来说，你应该使用你能得到的最好的方法。

凯文:这真的是一件奇怪的事情，我实际上发现只要这只是一个互联网浏览器的事情，我就觉得没问题。

瑞秋:是的。

一旦它成为其他浏览器想要做的事情，那么你就开始对评论的内容有一个标准，只是觉得很讨厌。

雷切尔:那感觉不对。我还认为，它正在远离你想去的地方，那就是所有的浏览器都支持同样的东西。如果浏览器开始说，“哦，我们以一种奇怪的方式这样做没关系，因为你可以用条件注释来绕过它，”所以在我们知道之前，我们已经有了 20 个样式表。这不是我们想要去的地方，我认为这种东西第一次出现时，我非常担心它会被浏览器用作借口，说:“嗯，我们可以用不同的方式或其他方式来做这件事。”显然，这是我们试图远离的东西。但这似乎并没有发生，现在人们只是倾向于使用它来解决 IE 浏览器的错误，这没什么，因为如果你在处理一个错误，我认为无论你用什么方法来解决它，如果你能以尽可能干净的方式来做，这可能是没问题的。所以，这不仅仅是不断地绊倒你，嗯，这很有帮助。

凯文:我对讨论你在某本书上看到的 CSS 代码与你在日常工作中实际编写的代码有何不同很感兴趣。有什么区别吗？

瑞秋:不是很大。我认为肯定不是 CSS 选集，因为我试图尽可能多地写些实用的东西，或者只是举着用。

凯文:嗯，这本身就是一种约束，我在现实世界中编写自己的代码时可能没有这种约束。CSS 选集里的每一段代码都要尽可能的独立。你工作的时候有什么依赖吗？你是否有一个框架，或者你是否为每个项目从头开始？

Kevin: 所以你会发现这有一点限制，一旦你想做一些框架不能开箱即用的事情，你就必须绕过它？

是的，你必须解决这个问题。我不反对框架。如果人们发现这是一种有用的工作方式，那很酷，但我认为，是的，我更愿意实际上把每个项目作为一个单独的事情，并以尽可能干净的方式构建它。但这只是我；我对这些东西并不特别虔诚。

凯文:说到宗教，你会使用什么样的工具？因为开发人员似乎对他们的文本编辑器特别虔诚。

我只使用我在 PHP 中使用的 Zend。事实上，我仍然是一个老式的 Zend Studio 用户，因为我从来没有成功地将 Eclipse 版本安装到我的 Ubuntu box 上。所以，有一个新的版本出来了，我真的需要尝试，让这个工作。

所以，是的，我用的是老学校 Zend Studio，我用它做任何事情。这是一个垃圾的 CSS 编辑器，但我只是打字。我一直都是那种只需要一个漂亮记事本的人。我只是打字。我打字非常快，非常准确，并且我不使用编辑器中的很多工具。我喜欢 Zend for PHP，因为它足够智能，可以很好地处理旧 PHP，并为我自己的函数和事情提供提示，这很好。这对大项目很有用，但对于 CSS，我只是坐下来打字。

Kevin: 你在开发时通常会打开什么浏览器？你工作时测试的第一个浏览器是什么？

火狐浏览器，这主要是因为我在工作中使用 Ubuntu，而且我家里有一台 Mac 电脑。所以我可以使用 Firefox 和 [Web Developer toolbar](https://addons.mozilla.org/firefox/addon/60) 和其他东西，而 [Firebug](http://getfirebug.com/) 我可以在任何地方使用。Zend 也是一样，我可以把它安装在 Mac 和 Ubuntu 上。因此，拥有通用工具而不用担心我面对的是哪个操作系统是件好事。

Kevin: 让我们花点时间谈谈 CSS3，你提到因为你在写作，你有理由继续关注它。你对 CSS3 中即将出现的新东西有什么想法？相对于浏览器中引入的上一波新东西，它是如何引入的？他们做得更好吗？

雷切尔:我不知道，我认为对于所有这些东西，你只能等着看人们如何使用它，以及人们会学到什么。我认为外面有两种人:有些人只是……他们只是想知道什么有用，在看到其他人使用它并说“看，它有用”之前，他们不会去碰任何东西。它可以在这些浏览器中运行。”我认为这是新事物被接受的点，真正是当喜欢玩新东西的人得到它，玩它，写它，并说，“看，它在这些浏览器中工作，你可以使用它。”

凯文:然而，我是那种很容易被性感诱惑的人。

是的，当我有空闲时间的时候，我会说我发现那几乎是学术上的有趣，你知道，“这能做什么？如果实施的话，这部分规范会带来什么？”我想大多数人可能真的对东西不感兴趣，直到他们看到它的真正用途，并且看到它不会坏掉。

Kevin: 那么 CSS3 中有什么东西通过了考验吗？

Rachel: 有一些是没有的，但是我认为——我的意思是 IE8 很明显在帮助 CSS 表格之类的东西。我们开始看到更多的东西成为主流，但这可能真的有点早。我想我们会到达那里…

Kevin: 将 CSS3 作为一个东西来讨论几乎是人为的，因为它实际上是作为一堆小东西来对待的。

是的，我认为是一点一点进来的。我想人们可能甚至不会想，“这是 CSS3。它只是一个在浏览器中工作的新事物。”

凯文:以他们现在的方式，CSS3 永远也不会成功。他们只会不断添加更多的比特。

瑞秋:我想就是这样了。我认为更重要的是我们可以使用它的哪些部分，这才是我真正感兴趣的，因为它的内容可以实际使用，即使它只能在 Firefox 中使用，有时你可以添加一些小的调整，特别是针对个人内容或针对特定市场的内容，即使只有少数人真正看到它。我认为有相当多的人用他们自己的东西做那种事情。

我认为很容易认为所有这些东西的时间框架是如此之长，但实际上当你回头看，当你认为它实际上不是很久以前，我们正在与网景 4 战斗，并认为它永远不会消失，我们永远无法以合理的方式做 CSS 布局。

凯文:这种事情一波一波地来，而我们现在正处在一个席卷我们的浪潮之中。浏览器的新功能越来越多，越来越快。

雷切尔:是的，我认为这只是……是人，我想像我们这样写这种东西的人，看起来有点像大多数人的过滤器，他们只会在看到一些专家说“在这里，你这样做，它在这些浏览器中工作。”我认为这正是我作为一名作家所做的。

我喜欢边看边说，“好吧，你可以这样做，这里有一种方法可以让不支持它的浏览器绕过它……”我认为这本书在很多事情上是这样做的，特别是在 IE6 中不适用的事情。

凯文:你提到你在 Zend Studio 中编码，因为你花了很多时间在 PHP 上。在我们结束之前，我想花几分钟时间谈谈 T2 鲈鱼。鲈鱼是你卖的 CMS 产品，对吗？

是的，这是一个非常小的 CMS。作为一家公司，我们有一个我们开发的大型 CMS 框架，我们的许多项目都基于它，它非常灵活，但它的一个关键点是它是基于模板的，它不会强制依赖于某种特定的标记语言，它允许人们使用微格式等东西。

许多大大小小的内容管理器的大问题是，你会得到一个很大的文本区域，你刚刚把你的文本放进去了。因此，您失去了拥有结构化内容的能力，当然，对于微格式之类的东西，结构化数据非常重要。

此外，我是一个数据库极客，我喜欢正确地存储东西。如果某个东西是一个事件，我希望它存储为一个事件，因为这在将来可能会有用，这意味着我们可以在其他地方使用它。

无论如何，我的意思是，这是我们的工作和我们的主要产品的某个地方，我们在这上面建立了很多大的东西。但是我们也得到，你知道，因为我们与设计机构合作，他们有非常非常小的产品-他们需要在一个非常小的宣传册网站上发布的东西，但他们希望它是可编辑的，显然，对我们来说，在这些东西上实现我们的大 CMS 是完全多余的。

凯文:嗯，是的，当我们之前讨论 CSS 框架时，你提到学习框架和绕过框架的方法通常比自己动手更麻烦。我发现很多 CSS 都是这样。它们被称为 CMS，但在大多数情况下，它们是完整的框架，从 CMS 开始，然后按照 CMS 规定的风格编码。我发现 Perch 真正吸引人和有趣的地方是你从自己的代码开始。

雷切尔:我的意思是，这就是我们想要的东西，让人们用他们已经在建造的东西来建造东西。我敢肯定，如果他们想使用 Dreamweaver，他们可以像平时一样构建自己的页面，而不必考虑这样做是否可行，我是否可以将它拆分？因为即使使用像 WordPress 这样的东西，很多人会说，哦，我们可以免费使用 WordPress；要让你的模板发挥作用，需要做大量的工作。如果你是一名设计师，而不是一名程序员，也不想成为一名程序员，这似乎是一个很好的学习曲线。

这就是鲈鱼的情况。我们想要一些非常非常简单的东西，让人们可以建立和运行，但也允许这种结构化的内容，允许人们让他们的客户添加联系信息作为联系人，并能够不只是为人们提供一个大的文本块来输入，虽然他们可以这样做，如果他们想要的话，但我认为很大的力量是这个模板的想法，你可以用普通的 HTML 创建这些模板，然后用它们来结构化数据。

我认为它非常成功，我认为它是有需求的，人们正在用它做各种有趣的事情。看到人们用一些东西建立的网站非常有趣，这对我们来说是一个很大的突破，因为我们刚刚提供了服务。

**Rachel:** 一些概念显然来自我们的大型 CMS，这是一些想法和它的工作方式，但实际上作为它自己的产品，它开发得非常快，因为到那时，我们确切地知道它应该是什么。但是我的意思是，是的，有一些概念可以追溯到很久以前。

已经 35 岁了。
