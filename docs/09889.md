## 采访记录

路易斯:大家好，欢迎来到 SitePoint 播客的另一集。今天和我一起参加节目的是 Cody Fauser，Cody 是 Shopify 的首席技术官，这是一个非常受欢迎的基于 Ruby on Rails 的软件即服务电子商务平台，大家好，Cody。

科迪:嗨，你好吗？

路易:我很好，你呢？

科迪:太好了。

**路易:**牛逼。所以，在我们深入讨论 Shopify 和电子商务，可能还会讨论一下 Rails 之前，你能不能先谈谈你作为一名 web 开发人员的背景？

是的，当我看到 Slashdot 抨击 Ruby on Rails 开发的帖子时，我想我应该仔细看看，结果发现它解决了我当时在 web 开发框架中遇到的所有问题，这很好。从那以后，我就一头扎进去，开始把越来越多的时间投入到 Ruby on Rails 编程上，最终我和 Tobias Lutke 联系上了，他创建了 Shopify，是 Rails 核心团队的一员，最后加入了 Shopify 团队。

路易:好的。为了不熟悉 Shopify 的听众的利益，你想谈谈什么是 Shopify 吗？

**Cody:** 你的介绍可谓一针见血，但 Shopify 是一个托管电子商务平台，我们现在在该平台上有超过 13，000 家商家，基本上 Shopify 包括商家快速启动和运行在线业务所需的一切，并且风险最小，他们的目标是他们不需要自己的 it 部门来管理他们的在线商店，他们可以获得他们的设计，并且他们可以自己管理他们的日常业务运营。

**Louis:** 好的，对你们来说，Shopify 和许多其他电子商务平台的最大区别是不是需要更多的内部技术能力？

是的，Shopify 的宗旨是成为简单优雅的解决方案，所以还有其他电子商务解决方案，它们拥有电子商务所需的所有功能，这根本不是我们的宗旨，Shopify 的宗旨是将复杂性隐藏在直观易用的漂亮用户界面下，并且 Shopify 非常适合那些希望尝试新想法或开始尝试新产品或以最小风险进行实验的人。他们不必花很多时间来建立基础设施或托管，也没有大的安装费或类似的东西，所以你真的可以在几分钟内从零到 100。

**路易:**对。对你来说，这是你看到的东西，我想这是我感兴趣的事情之一，因为以前有这么多的电子商务平台，后来又出现了，我想起了以前的 ZenCart 和 OSCommerce，我想这是所谓的，最近的 Magento，所以有所有这些电子商务平台，人们建立和开发自己的主题和内部解决方案。你是否觉得随着时间的推移，这种东西的作用越来越小，或者你认为这种东西在某些拥有资源的组织中仍然有一席之地，而你们正在瞄准一个完全不同的市场？

是的，我认为随着时间的推移，自行安装类型的解决方案正在进入更大的业务领域，人们拥有专门的 it 资源来管理软件。对于手头没有这些资源的人来说，维护服务器和进行升级非常耗时和/或昂贵，没有人愿意在半夜被短信吵醒，说他们的电子商务网站正在支持他们的业务离线，所以拥有一个软件即服务产品确实可以解决所有这些问题。

**路易:**对。当你开始与 Shopify 合作时，我的理解是，Tobias 已经在开发它了，当你加入时，它已经是一个运行的平台了吗？或者当它向公众推出时，你在哪里？

**科迪:**当时托比实际上已经建立了一个滑雪板商店，他希望在网上开展这项滑雪板业务，他调查了所有可用的产品，但他没有找到任何真正符合他要求的产品，他对自己喜欢使用的软件类型有很高的期望，但他没有找到任何真正符合他要求的产品，所以他一直在工作。 基本上是从 Shopify 开始工作，或者像它被称为 Snow Devil，滑雪板商店，Rails 发布的那一天，非常非常第一次公开发布，所以他在他的家里做了大约一年左右，然后我在 2006 年发布前几个月加入进来，帮助完成了所有最后的功能，从那以后一直在这里。

**路易:**对。这真的很有趣。我记得在 Rails 早期，有很多人批评 Ruby 和 Rails 是不可扩展的平台；当时没有很多应用程序，我的意思是现在我们有 Twitters 和 GitHubs 以及很多基于 Ruby on Rails 的应用程序，但我记得早期 Shopify 是你在 Ruby on Rails 主页上看到的应用程序之一，他们说，看，这是一个基于 Rails 的应用程序的例子，可以扩展，所以你们从一开始就在那里。

科迪:是的，当然。你知道，我们非常感谢 Ruby on Rails 主页上的位置，特别是当我们在早期启动公司时，我们刚刚有了一个令人难以置信的 Rails 社区，这些年来他们真的帮助了我们。

**路易:**对。所以回到——我知道问任何开发 Rails 应用程序的人永恒的伸缩性问题有点老套，但是，你发现 Rails 有什么问题吗？几周前我采访了 GitHub 的 Tom Preston-Werner，他说，不，Rails 从来不是，或者说 Ruby 从来不是扩展应用程序的瓶颈；从扩展的角度来看，你们面临的挑战是什么？

科迪:你知道，对我们来说，一切都是一种交换，所以如果你想超快地开发软件，并把你的产品推向市场，显然你不是用汇编或 C 代码来写的，但事实是，你有你的产品推向市场，你有客户，这意味着你必须担心扩展问题。 所以你知道，像 Twitters 和这些类型的事情，他们确实变得非常成功，但也有问题，这是一个很好的过程，他们进入市场，然后他们优化代码路径，这些代码路径非常热门，并导致问题，这是我们使用新的遗迹监控工具时使用的相同技术，它会告诉我们应用程序中所有可怕的部分在哪里， 但它们通常与 Ruby 本身无关，就像我们做非最优的数据库查询或做糟糕的代码一样，所以我们真的不觉得 Ruby 在这一点上拖了我们的后腿。

**路易:**对。关于 Shopify 的另一件有趣的事情是，他们是少数几个，或者我猜他们是现在越来越多的内置应用程序商店的网络服务，所以我想我记得第一次看到它是在 37signals 应用程序中，如果我错了，你可以纠正我， 我想我也看到 FreshBooks 有类似的东西，他们有一套插件，我猜类似于你看到的 WordPress 或 Drupal 等自托管解决方案的插件，只是这些是你只需在服务器端打开的应用程序，其中一些是付费应用程序。 你想谈一谈开发它的思维过程吗，以及它是如何为你工作的？

**科迪:**你知道苹果应用商店对我们来说是一个巨大的灵感，实际上拥有一个应用商店是保持你的核心产品非常简单、精简和优雅的一个好方法，因为人们总是有一个软件不能满足的要求，所以如果你总是实现所有人都想要的所有这些功能，那么你的软件很快就会变得不直观、臃肿和混乱， 因此，我们不仅可以让人们开发他们需要的定制功能，我们还可以利用我们自己的计费平台，让他们为他们的应用程序收费，并且它可以无缝集成到商店的 Shopify 中，因此这对所有参与者来说都是一种双赢。 我们有相当多的，一些 Y Combinator 创业公司和这种事情，使用 Shopify 作为新想法，新商业概念的试验场，然后一旦他们证明这些想法在 Shopify 平台上可行，他们就可以将其扩展到一般市场。

**Louis:** 那么，当你说把它扩展到一般市场时，你是指它是一种销售产品，他们进入零售市场，还是指他们从技术角度扩展他们的平台，并在你的 API 基础上扩展他们的网站和平台？

**Cody:** 因为我们有如此多的商家，这是测试一个概念是否有效的好方法，所以我们有一个非常受欢迎的聊天应用程序，你可以将聊天嵌入到你的商店中，这样你就知道这不仅限于电子商务网站， 只是我们有一个很好的用户基础，一个简单的 API 和一个计费平台，允许他们制作产品原型，并很快将其推向市场，然后一旦他们知道这是一个可靠的想法，他们就可以提供它，并开始进入其他利基市场，扩大他们的视野。

哦，对了，对不起，我想我误解你了；我以为你说的是商家，其实你说的是应用开发者。

科迪:哦，对了，抱歉，我关注的是那里的应用开发者。

**Louis:** 是的，这真的很有趣，你实际上为开发者提供了一个平台。你有没有发现很多你的应用程序开发者也在使用 Ruby on Rails，或者他们有很多不同的人，因为 API 只是 REST 和 JSON？

是的，所以有很多 Rails 开发者，但是就像你说的，因为我们提供 XML 和 JSON，你知道，PHP 或者任何人想用来构建应用程序的技术都很好。我们为人们开始使用 Ruby 提供了非常棒的工具，所以这可能有助于我们的 API 采用 Ruby on Rails，但是，是的，它肯定对任何有兴趣集成它的人开放。

**路易:**对。我想再回来，稍微绕一圈，谈谈更一般的电子商务。正如我之前所说的，当你们开始时，有许多解决方案，所以其中一些是免费的解决方案，一些是托管的解决方案，还有付费安装的软件， 是什么让你们认为你们有一个产品能够在一个稳定的价格点上竞争，并且是一种重复计费的东西，人们可能不习惯为电子商务付费，他们喜欢哦，我只是下载一些开源软件，并把它放在每月 5 美元的主机上，我就准备好了。 是什么给了你们信心，让你们相信外面的人会愿意为更专业的解决方案付费？

**Cody:** 是的，正如我之前提到的，Toby 真的经历了市场上的每一种产品，当他早期开始在 Shopify 上工作时，他感觉到的所有痛苦都历历在目，所以基本上在推出之前的目标只是解决他在滑雪板商店开始时的每个痛点，所以一旦我们让 Shopify 工作，我们实际上从 Snow Devil 那里获得了设计，并在大约 20 分钟内实现了整个商店。因此，它实际上只是解决所有这些棘手问题，而不是其他产品，也不是我们认为我们可以比其他产品更好，只是如果我们解决了这些问题，那么像我们一样的其他人也会使用该产品，所以它完全是出于或多或少的需要。

路易:对。您之前也谈到了对于一些刚刚起步并希望测试产品或类似产品的小型组织或个人来说，Shopify 是一种低风险的好方法，而其他类型的自行安装解决方案可能仍然是拥有自己内部 it 部门的大型组织的可行选择。 但是我想您可能也有一些客户是较大的组织，他们可以维护自己的解决方案，但是已经决定采用软件即服务解决方案，这是正确的吗？

**科迪:**对，肯定是这样。例如，Rovio 销售大家都知道的愤怒的小鸟游戏，他们通过 Shopify 销售他们的毛绒玩具，所以这肯定会扩大到更大的组织，他们的核心能力是开发游戏，我很确定他们不想让他们的电子商务团队和运营团队管理电子商务服务器，因为他们专注于制作有利可图的游戏，所以我们在那里有很好的关系。

**Louis:** 是的，我想很多组织都是这样，如果你只看招聘、培训和维护团队来开发你自己的解决方案的成本，那么管理费用就比付给你们一点点利润要多。

科迪:完全正确。你知道，一个免费的开源解决方案看起来很便宜，直到你考虑到所有的时间，有一次它可能会下降，我们有专门的人 24/7 呼叫，以确保我们的服务是正常的，所以你真的不能用金钱来衡量。

路易:对。也许我会在这里稍微改变一下，开始进入更多的极客的东西，我们的许多观众都是 web 开发人员类型的人，所以我想给他们一点脂肪来咀嚼。关于扩大规模，甚至可能只是建立你的平台，因为电子商务是一种棘手的解决方案，不仅仅是建立一个 CMS，它是一个更难的解决方案，涉及许多移动部件，你们在开发和扩大平台时面临的最大挑战是什么？

**Cody:** 为了真正概括，我将进入更多细节，但我们认为电子商务是这样的东西，我们将解决问题，我们都将进入下一个业务，我们四年前就这样认为，我们仍然试图将一切联系在一起， 因此，在一个极其复杂的环境中，你要处理金融交易，还要试图在履行和库存方面端到端地简化事情，并拥有直观的结账和所有这些事情，所以对我们来说，试图掩盖直观界面背后的电子商务固有的复杂性一直是一个挑战，但这可能是足够的极客谈话。

**Louis:** 我想从更高的角度来考虑这个问题是很有趣的，我的意思是，看起来像是核心业务的东西，我们只是要建立一个电子商务平台，我们现在应该已经完成了，但所有这些不断的改进和不断的工作都在进行中，你又在谈论处理金融交易的事情；我想，建立一个 CMS 有很大的区别，如果有人发表评论，它会消失在空白中，这不是什么大不了的事情，但电子商务对错误的容忍度要低得多。

科迪:完全正确。这是我要谈的事情之一，就是你要试图正确地对一切进行建模，以便每次尝试进行金融交易都有记录的历史，这样你就永远不会—永远不会有任何数据丢失，这真的很重要，但除此之外，因为我们确实有新的夫妻店上线以及大型 Rovio 类型， 我们的商家是什么样的，他们在做什么，网站上有什么特色，这种令人难以置信的多样性很难准确地描述其中一个，所以有一天有人可能会上奥普拉，突然他们的网站被抨击，所以我们必须能够处理这一点，以及他们可以使用自己的液体和 HTML 开发他们商店的整个前端，这是不可预测的，因为我们无法控制这一点。 所以有很多事情要做，很多缓存，并确保我们可以处理所有那些奇怪的场景，你可能不必担心其他类型的产品，在登录屏幕后一切都受到保护。

路易:对。这是有趣的事情之一，因为你们除了向人们提供软件之外，还必须托管它，就像你说的，如果你的一个客户突然流量激增，你必须确保其他人的网站不受影响。

科迪:是的，没错。我们有一个圣诞节期间的人在周四注册，他们在周一或周二出现在美国的一个电视节目中，他们当天完成了 100 万美元的交易，50 美元的产品，所以在某些时候他们每分钟完成 300 多个订单，所以你真的会在你意想不到的地方看到这些疯狂的峰值。

**Louis:** 如果我没弄错的话，你们甚至把自己的托管业务都外包出去了，所以你们不在内部做，我想你们已经和 Rackspace 达成了协议，对吗？

**Cody:** 是的，我们现在在 Rackspace，所以我们有一个很大的服务器集群。

**Louis:** 你们是动态扩展集群，还是仍然手动处理？

**Cody:** 我们正在运行裸机服务器，所以实际上我们并没有——是的，所以我们使用云来提供某些服务，但是核心应用程序是在裸机服务器上，所以如果我们想要扩展，我们必须获得更多的服务器，所以我们只是确保我们有足够的资源来处理我们可能看到的任何峰值。

**路易:**对。好吧，那么也许从你面临的挑战继续，有没有什么事情——也许让你花一分钟时间，有没有什么特别酷的问题，你有机会解决，自从你在 Shopify 工作以来，或者甚至是你现在正在解决的事情，你认为很有趣，或者可能是一个挑战，它不是一个挑战或障碍，而是一个你有兴趣解决的事情？

科迪:是的，我想一般来说，你做的任何事情都很令人满意，因为我们部署软件的时候，我们通常一天部署一次或更多次，这取决于是否有任何代码要推出，但代码推出的时候，它出现在所有商店、所有用户和所有流量面前，所以你写的任何东西肯定都很令人满意，但我只写了一些东西， 整个计费系统对于客户来说是非常令人满意的，并且看到它在过去几年中是如何生存的，Shopify 的支付和处理订单管理基础设施也是如此； 支付和订单的数量之多令人难以置信，这也很有趣。Toby 和我总是开玩笑说，组织中最好的工作是运营开发，所以我们都喜欢在配置管理方面以某种编程级别接触服务器，所以我有机会提供很多帮助，或者在我们转移到机架空间并使用 Chef 配置管理系统实现一切自动化之前，帮助运营团队很多，这也真的很令人满意。

路易斯:你们现在有一个相当大的团队吗？目前你们总共有多少员工？
