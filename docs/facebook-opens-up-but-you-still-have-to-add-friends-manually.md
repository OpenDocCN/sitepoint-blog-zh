# 脸书打开了，但你仍然需要手动添加好友

> 原文：<https://www.sitepoint.com/facebook-opens-up-but-you-still-have-to-add-friends-manually/>

一年多前，两家相互竞争的照片共享服务偶然发现了一些围绕其数据开放的棘手问题: [Flickr 最初拒绝 Zooomr 访问其 API](https://www.sitepoint.com/flickr-zooomr-and-api-parity/) ,理由是它与竞争对手过于相似，但最终同意开放，前提是流媒体是双向渠道。

社交网络空间目前正在经历类似的发展阶段，并解决类似的问题——首先谷歌宣布了 [OpenSocial](http://code.google.com/apis/opensocial/) ，这是一个开放的标准，社交网络应用程序的开发者可以遵守它，以便他们可以在任何网络上运行，脸书最近采取行动，以摆脱他们作为[围墙花园的看法，由](https://www.facebook.com/sitepoint/)[发布了类似的(并不意外)公告](http://blogs.guardian.co.uk/technology/2007/12/13/facebook_opens_up_another_platform_war_beckons.html )。开发者面临着一个选择:一个选项存在，但只有两个玩家，另一个选项有很多合作伙伴，但尚未发布。

这一切都很好。我讨厌重复代码和浪费精力，所以事实上 [Bebo](http://bebo.com/) 用户现在可以和他们的朋友一起玩 [Scrabulous](http://scrabulous.com/) ，而 Scrabulous 团队不必编写 Bebo 版本的脸书应用程序。

但是那些用户和他们的*联系*呢？

所有这些开放性和关于 API 对等性和开放标准的讨论都无助于一个事实，即使用一个以上社交网络的任何人最大的抱怨是*他们不得不在每次注册时重新输入他们的数据*。当然，有黑客[的方法来获取你的大部分数据](http://zsnewsbits.blogspot.com/2007/10/new-facebook-app-lets-users-export.html)，但产生的数据并不存在任何类型的标准格式，你可以在不同的网络中重用，而且你可以检索的联系信息也受到限制。

此外，它会采用什么格式？XFN？哈！Brian Suda 最近写了一篇关于微格式如何成为这个问题的答案的文章，但是还没有足够的社交网络支持它们，所以这还不实用。

我是否天真地认为，如果我们快进几年，当每个人都在使用 [OpenID 和 OAuth](http://blog.wired.com/monkeybites/2007/12/thanks-to-openi.html) 时，我们将再次回到这样一个概念:除了 Web 之外，没有其他社交网络[？](http://themaninblue.com/writing/perspective/2007/09/03/)
