## 2005 年网络状况

作为主题发言人，莫莉热情洋溢地谈论了会议上的主要发言人，基本上做了任何优秀的主题发言人都应该做的事情:让与会者对他们在这里花费的时间和金钱感到满意。

她确实提到，在过去的一年中，Web 标准已经突破了许多障碍，在即使是最务实的 Web 设计者的意识中也获得了立足点；然而，据她估计，标准还有很长的路要走。

莫莉接着谈到了“思想孤岛”，她用这个比喻来描述不同的知识体，这些知识体必须集合在一起才能构建现代网站和应用程序。<ins>她接着指出，我们仍然缺乏一个有意义的标准术语来描述我们在这个领域的工作(你是设计师吗？网络作者？内容制作人？技术专家？程序员？应用程序开发人员？…还是别的什么？).当我们的专业领域如此重要时，为什么我们没有具体而有用的语言来描述它呢？对我来说，这与 Maxine 关于缺乏有意义的认证的观点是一致的。</ins>

如果说过去的一年向我们展示了什么，那就是今天每个人都在谈论的[杀手](http://www.flickr.com/) [应用](http://maps.google.com/)源于那些以意想不到的方式聚集在一起的思想孤岛。正是这种趋势推动了*语义网*(区别于[语义网](https://www.w3.org/2001/sw/))——对现有标准的巧妙利用使事情发生*现在*，而不是在 W3C 着手敲定推荐标准的时候。

## <cite>超越可用性:设计完整的用户体验</cite>，杰弗里·维恩

杰弗里做了一个非常有趣和鼓舞人心的演讲，但归根结底，他真正做的是快速总结经典的以用户为中心的设计原则，如自上而下与自下而上的设计。这些原则在今天仍然适用，但是如果你在过去接受过任何形式的人机界面培训，这只是一个回顾。

## 从 Comp 到 Code:把一份清单拉在一起

虽然 Eric 没有给出一行代码，但我的预期是正确的。他*所做的*是带我们经历他在将一个设计师的 Photoshop 排版翻译成 CSS 页面布局时所做的每一个决定。

在每一个阶段，他都会观察目标设计的特征，并排除不能产生预期结果的 CSS 布局选项。例如，全宽页脚和任意列高意味着 CSS 定位不起作用，这就需要考虑人造列和液体漂白剂的浮动框技术。

我带着一个“CSS 架构师”的设计过程的愿景离开了这个会议，他根据设计的需求决定使用什么技术，然后将编码和特定于浏览器的修复/破解交给代码猴子。Eric 甚至建议这个架构师角色有一天可以由自动化的专家系统来完成。

## <cite>提高表单的可访问性</cite>，Roger Hudson

这个会议实际上比我预期的要简单得多，但是看着 Roger 努力让一个现代的屏幕阅读器为演示而工作的经历本身就很有启发性。Lucas Chan，我们的一个 PHP 枪手，走出来形容今天用屏幕阅读器浏览网页的体验是“令人抓狂的”——这只是在这个演讲中提出的相对简单的例子。

一些快速外卖:

*   许多屏幕阅读器会忽略表单域及其标签文本周围的<label>标签</label>
*   带有指向表单字段 ID 的 for 属性的<label>标签工作得更频繁</label>
*   嵌套的<fieldsets>，尽管得到了[黄蜂](http://www.webstandards.org/)的认可，却被大多数屏幕阅读器所忽略</fieldsets>
*   如果不进行选择，就无法使用屏幕阅读器浏览单选按钮

很遗憾这次会议时间太紧，但我认为这是有意义的。

## 道格·鲍曼从战壕中冲出

在 75 分钟的会议中，道格花了 30 多分钟告诉我们，他即将对未来做出一些预测，同时一张又一张华丽的照片带着模糊的俏皮话闪过，比如“自然是一个系统”。然而，当他最终开始做这些预测时，其中一些非常有趣。

对即将到来的革命的想法——如投影、互动媒体、无线连接和小型设备的无限电源、位置等私人信息的共享，以及世界上穷人和弱势群体的可访问性——给了我们很多东西去思考。

Doug 对 CSS 的未来的看法是，网站拥有数百个适用于多种设备类型的样式表(例如，使用 [CSS3 媒体查询](https://www.w3.org/TR/2002/CR-css3-mediaqueries-20020708/))和用户可选择的样式表，这些样式表可以应用于支持标准“样式 API”的网站，这是令人大开眼界的，因为他警告说，有人可能会有更好的想法，并在一夜之间扫除所有这些 XHTML/CSS/JavaScript 的东西。

总之，我在这次会议上听到了许多鼓舞人心的事情，也见到了许多非常酷的人，他们中的许多人可能对我们在 SitePoint 所做的事情有非常深刻的见解。我想我已经可以很有信心地说 SitePoint 将会在明年的 WE06 上回归。
