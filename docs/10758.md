# UX 设计:它是什么，它需要什么，它将走向何方

> 原文：<https://www.sitepoint.com/ux-design-what-it-is-what-it-takes-and-where-its-going/>

SitePoint 是墨尔本 UX 澳洲会议的媒体合作伙伴，在会议开始前，James Mansfield 询问了三位与会者对 UX 设计新兴领域的看法:什么是 UX 设计师，他们是如何进入这个领域的，以及他们对未来的展望。接受采访的三位 UX 设计师是 Shane Morris ( [自动工作室](http://www.automaticstudio.com.au/About.html))、Daniel Szuc ( [Apogee HK](http://apogeehk.com/dano/) 和 SitePoint 的[可用性工具包](https://www.sitepoint.com/kits/usability1)的合著者)和 Iain Barker ( [Meld 工作室](http://meldstudios.com.au/))。他们的回答为有抱负的 UX 设计师提供了洞察力，并为有经验的从业者描绘了一个有趣的视角。

先生们好，感谢你们花时间回答一些问题。 **首先，用户体验设计师是一个相对较新的职位，定义起来有点棘手。相对于可用性顾问或交互设计师，你如何描述 UX 设计师的角色？你给自己什么头衔，为什么？我认为我们有时会被自己的流行语冲昏头脑。我们都想让自己的工作与众不同，但每一个新名词都伴随着教育市场的责任。这并不是贬低他们给我们的方向，而是我更喜欢使用一个市场理解的术语。目前，它似乎是 UX。每个市场对条款的接受程度都略有不同；比如可用性，UX，CX，设计，XD 等等。![](img/c676fd50b9fcc80b7a28eee4f752b9c4.png "logo")目前，我以“可用性顾问”为首，但根据工作的不同，我还经常身兼数职，包括协调员、设计师、战略家。对我来说，我们所做的核心是:我们如何带领团队踏上旅程，帮助他们制造或创造更好的东西，产生一些积极的影响？**

> 我们如何带领团队踏上旅程，帮助他们制造或创造更好的东西，产生一些积极的影响？

**【伊恩·巴克】:**我在不同的阶段都用过这三个术语。当我称自己为可用性顾问时，我是在一个高度专业化的角色中进行可用性测试，然后向客户团队投掷可用性的手榴弹。我会识别可用性问题并提出设计建议，但缺乏做出或实施决策的责任。当我称自己为交互/UX 设计师时，我对交互产品或服务的研究和设计负有更广泛的责任，并实际做出设计决策。我同意丹尼尔的观点，对于大多数客户来说，UXD 比交互设计更好理解。在我的 LinkedIn 个人资料中，我把自己描述为“以用户为中心的设计专家”，但这可能于事无补！**【Shane Morris】:**我认为这三个术语反映了市场的成熟，以及我们作为以用户为中心的设计专家能够为客户和雇主提供的东西。最初，我们都是关于*可用性*；然后是关于*互动*。如今，我们关心的是用户或客户与他们打交道的组织之间更广泛的关系，所以我们谈论*用户* *体验*。所以重点有所转移。我一直对华而不实的头衔保持警惕，但我很乐意称自己为用户体验架构师，因为这最能描述我现在为客户所做的事情。**谢谢各位，非常有趣的观点。你认为成为一名优秀的用户体验设计师需要什么？我认为这需要一些人际交往的技能，比如倾听各种观点的能力，寻找可以改进设计的元素，以及不断学习的能力。此外，还有一些更专业的技能，如清晰沟通、展现领导力，以及推销和推销自己和自己的想法。我还要加上讲故事——讲述我们所做的每件事情中引人入胜的故事的才能。**【IB】**:这是个大问题。一些方面包括对最终用户的同情，能够与各种利益相关者合作和交流，以及认识到何时妥协以及何时为一个想法而战。重要的是，你应该将失败视为通向更好解决方案的学习过程的一部分；准备好分享未完成的想法或问“愚蠢的问题”**【SM】**:我同意丹尼尔和伊恩的观点，但有趣的是，我们都没有关注“硬”技能:运行可用性测试、促进协作设计或构建原型。最重要的技能是“软”，移情最重要；也许这是因为硬技能通常是在工作中学习的，这当然是我的经验。**

> 我进行了我的第一次可用性测试，一个灯泡亮了

非常有趣的是，你们都把注意力放在了你们所谓的软性技能上。我想他们是最难教的年轻学生，正如谢恩指出的，他们通常是在工作中学习的。所以，对于那些有志于成为 UX 设计师的人，你对进入这个领域有什么建议？**【IB】:**我给刚开始做用户体验设计师的人的建议是:

*   选择一个真正让用户参与设计过程的组织
*   在提供经验和指导的良好环境中工作
*   不要被失败的恐惧所束缚
*   广泛阅读关于 UX 过程和技术的不同观点。

我在大学里学习了一门职业信息设计课程，在得到设计界面的工作之前，我花了几年时间做技术作者。在最初的几年里，我依靠基于观点的设计，这有时让我感到沮丧。然后，当我进行我的第一次可用性测试时，一个灯泡亮了。可用性测试是从用户体验开始的一个很好的方法。这是我们使用的更程序化的技术之一，从非正式到正式都适用；最棒的是，这是一个学习人类行为的绝佳环境。对我来说，在 UX 工作是一个幸运的意外。我在大学里学习了计算机科学和认知科学，但是后来发现了这个领域，并且我是合格的(按照当时的标准)。我很幸运，这个领域一直在发展，对我来说保持新鲜和挑战性。

> 可用性测试是一个人开始体验用户体验的好方法

**【DS】**:我从开发开始，然后在 90 年代初在一家电信公司工作时了解了“人的因素”，这导致了我对可用性的兴趣。从那以后，我转到了客户支持系统，在那里，我在开发 UI 标准和在产品中嵌入可用性的培训项目方面打下了良好的基础。**谢谢伙计们，这真的很有趣。你在这个行业已经有一段时间了，并且看到了它的成熟。你认为接下来会怎样？****【DS】**:我希望看到的一些进步包括共同的语言、标准和灵活的工具、对 UX 在产品战略中所扮演的领导角色的更好理解，以及设计被用来帮助企业阐明战略和计划。基本上，所有我们擅长的东西，甚至更多。**【IB】**:我看到人们对我们提供的价值有了更广泛的理解和接受，有了更多更多样的机会，需求大于供给，并且认识到我们应该影响业务战略。我是这样想的:我们曾经告诉人们他们的孩子很丑。然后我们进一步建议人们如何让他们的宝宝更有魅力。如今，我帮助公司确定他们想要什么样的孩子，或者他们是否真的更喜欢 Xbox。**【DS】**:2010 年的好消息是，商界对 UX 有着健康的兴趣；现在是我们决定在 UX 商业讨论中扮演什么角色的时候了。**【IB】**:很高兴看到组织建立了强大的内部 UX 能力，这些团队得到了更多的关注和认可。愿它长久持续下去。白天，詹姆斯·曼斯菲尔德为 99designs 设计用户体验，晚上，他扮演爸爸，在自己的博客上滔滔不绝地谈论 UX 设计。他喜欢骑自行车，食物，啤酒，AFL 和他的荷包蛋。
