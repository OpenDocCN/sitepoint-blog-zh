我们最近讨论了两种类型的托管服务:

*   [**专用服务器**](https://www.sitepoint.com/dedicated-server-hosting/)
    您在自己的主机数据中心拥有/租用一台或多台服务器，仅供自己使用。
*   [**共享服务器**](https://www.siteground.com/go/article-sp)
    你在一台服务器上租用空间和处理能力，这台服务器也可能被许多其他公司和网站使用。

一般来说，专用服务器通常更灵活，您可以根据需要进行配置。共享服务器提供的选项可能更少，但更易于管理，通常成本也更低。

云托管可以是两者之间的一个很好的折衷，并带来其他好处。在进一步讨论之前，让我们先讨论一下术语和概念。

## 什么是云？

“云”和“云计算”是网络营销术语。就是这样。你不用购买软件并把数据存储在你的电脑硬盘上，而是使用一个在线应用程序。它通常在 web 服务器上运行和存储数据，同时在 web 浏览器中显示界面。换句话说，您可以按需访问软件，并与其他人共享计算资源。

web 应用程序可以托管在共享或专用服务器上，但扩展可能会变得困难。也许应用程序可以很好地处理 50 个并发用户，但是它如何处理 1000 个呢？您能多快迁移到更好的服务或服务器？如果用户需求在几个小时后恢复到正常水平会怎样？

## 什么是云托管？

云托管允许按需快速调配资源。您可以快速添加更多处理器、RAM 或磁盘容量，并且主机(如 [SiteGround](https://www.siteground.com/go/article-sp) )可以根据流量峰值自动扩展。

当你知道在某个特定事件发生时你会有巨大的流量时，自动缩放是短期活动的一个很好的选择。在这些情况下，当流量增加时，会自动添加额外的资源，所以您不必担心。

技术各不相同，但大多数云主机都依赖于虚拟机。

## 什么是虚拟机？

您可能遇到过在另一个计算设备上重新创建一个计算设备的模拟器。例如，[互联网档案馆](https://archive.org/details/software)允许你从你的浏览器[玩旧街机游戏](https://archive.org/details/internetarcade)，运行 [Windows 3.1](https://archive.org/details/softwarelibrary_win3) 或[苹果麦金塔电脑](https://archive.org/details/softwarelibrary_mac)。

重要的是要明白这些是从旧磁盘和只读存储器芯片中获得的原始系统——它们是*而不是*忠实的再现。正在模拟硬件。遗留软件可能在说“把这个图像放到屏幕上”——在上面的例子中，这个指令被 JavaScript 截获，翻译成 HTML5 `canvas`代码，然后返回一个适当的响应。该软件认为它运行在兼容的设备上。

虚拟机(VM)是一个类似的概念。像 [VWware](https://www.vmware.com/) 、 [VirtualBox](https://www.virtualbox.org/) 和 [Hyper-V](https://www.microsoft.com/cloud-platform/server-virtualization) 这样的软件使用软件驱动的标准硬件再造在 PC 上模拟 PC。因此，您可以在任何其他操作系统“内部”运行任何客户操作系统。

因此，云主机可以提供与专用服务器功能相同的虚拟机。但是，它们只是模拟真实服务器的数据文件，因此有几个实际含义:

1.  虚拟服务器可以在几秒钟内创建、安装和配置。
2.  虚拟服务器只是数据。它可以很容易地被克隆、备份、回滚或移动到其他地方。
3.  简化了开发和部署。开发人员、暂存和生产环境可以使用相同的映像。测试和撤销替代配置很容易。
4.  web 应用程序可以在其自己的虚拟操作系统映像中运行。第二个应用程序可以使用另一个虚拟机；不同的技术组合没有冲突或困难的风险。
5.  处理、RAM 和存储不再依赖于真实的设备。虚拟机可以部署到一个服务器场，该服务器场可以根据需要自动分配资源。
6.  因为操作系统不是在真实的硬件上运行，所以故障风险大大降低。

## 所有云主机的工作方式都一样吗？

不会。大多数云主机使用某种虚拟机架构，但服务可能会有所不同。例如:

*   **基础设施即服务(IaaS)**
    IaaS 可能是最接近专用托管的方式，它提供虚拟化设施，但您需要选择操作系统、安装软件堆栈并保持更新。
*   **平台即服务(PaaS)**
    PaaS 可能是最接近共享托管的平台，它为运行特定类型的应用程序提供了一个特定的平台。
*   **软件即服务(SaaS)**
    通常是按需软件。例如，您可以使用第三方服务来存储数据，而不是维护您自己的 MySQL 安装。
*   **无服务器或充当服务(FaaS)**
    管理逻辑和状态的抽象应用环境。您可以编写使用该架构的功能或微服务，而不必担心底层软件。

并非所有的服务都完全符合这些类别之一！

## 云托管的利与弊

我之前的文章在比较[专用服务器](https://www.sitepoint.com/dedicated-server-hosting/)和[共享服务器](https://www.siteground.com/go/article-sp)时，考察了易用性、灵活性、功能、安全性和安全性。这些与云托管关系不大，因为服务将决定这些因素。例如，云托管可能像不受管理的专用服务器一样困难，也可能像最简单的共享服务器解决方案一样简单。这取决于你。

这给我们留下了三个其他指标…

### 表演

云主机可以随着流量的增加自动分配资源，尽管这取决于您的主机和服务计划。尽管通常可以升级 CPU、RAM 和磁盘空间的数量，但有些可能会有特定的限制。像 [SiteGround](https://www.siteground.com/go/article-sp) 这样的主机无需重启就可以应用新的限制。

### 可靠性

专用和共享服务器出现故障。主机采取预防措施将风险降至最低，但过热的 CPU 或磁盘损坏可能会导致系统停机。云服务对真实硬件的依赖性较小。处理可以跨多个服务器共享；失败仍然发生，但你的网站或应用程序仍然活跃。

### 费用

云托管经常被比作公用事业，比如煤气、电或水:你为你使用的东西付费。实际情况通常更复杂，定价会产生固定和可变费率的混合费用。你每月的付款可能会有很大的不同。

## 谁应该选择云托管？

从长远来看，我怀疑云托管将取代共享和专用服务器计划。无论是否意识到，每个人都会使用云服务。

也就是说，云托管行业还处于起步阶段。服务种类呈指数增长，市场每天都在变化。当服务没有直接可比性，并且在开始托管之前成本未知时，为您的业务选择正确的计划会很困难。

一些主机将云服务定位在共享和专用主机计划的中间。也许这是目前最好的选择。当你超越共享主机的限制时，考虑云服务。
