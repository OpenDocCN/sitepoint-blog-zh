# 使用 AMASS 在客户端存储大数据

> 原文:[https://www . site point . com/store-large-data-client-side-with-amass/](https://www.sitepoint.com/store-large-data-client-side-with-amass/)

AJAX 海量存储系统(AMASS)是 JavaScript 和 Flash 的巧妙结合，实际上与 AJAX 没有任何关系——除了它将有助于减少 AJAX 的使用量，以生成具有丰富用户界面的 Web 应用程序，就像桌面应用程序一样。

AMASS 在页面中插入一个不可见的 Flash 电影，以便从 Flash 插件中借用一个功能，从而将 JavaScript 的本地数据存储容量(存储在最终用户的机器上，而不是 Web 服务器上的数据)从 cookies 提供的 4KB 扩展到 100KB，或者甚至扩展到任意大小(如果用户在得到提示时授予权限)。

这有什么用呢？如今，随着用户在网页上输入大量数据，撤销和自动保存等功能开始变得至关重要，而 AMASS 使它们更容易实现(成本也更低)。

Gmail 最近增加了自动保存功能，在你处理邮件时，每隔几分钟就会保存一份草稿，这样当你两岁的孩子跑进来，再次按下你电脑上的重置按钮时，你就不会丢失你的工作。Gmail 通过定期向服务器发送你正在进行的工作的副本来实现这一点，但对于这样一个简单的功能来说，这是很大的带宽。将它发送到服务器的原因是，传统上，JavaScript 仅限于 4KB 的本地数据存储。根据你使用电子邮件的方式，超过 4KB 的邮件可能是家常便饭。随着 AMASS 提供的扩展容量，这一限制不再适用。

在最初的测试版本中，AMASS 只支持 Windows 上的 Internet Explorer 6、Firefox 和其他基于 Mozilla 的浏览器。Linux 未经测试，Mac 上也有已知问题。不过，这是一个创新的想法，如果作者能够解决兼容性问题，它可能会成为一个有用的工具。
