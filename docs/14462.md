# 最新的搜索引擎垃圾邮件技术

> 原文：<https://www.sitepoint.com/search-engine-spam-techniques/>

搜索引擎排名竞争非常激烈，网站所有者承受着压力，要尽一切努力在搜索结果中获得可见性。

这些压力来自很多方面:品牌限制、风格指南、法律问题、导航需求、销售转化需求、网站互动需求等等。

然而，事实仍然是，搜索引擎是为信息目的而设计的。这给试图利用搜索引擎来吸引搜索信息的用户，然后试图向他们出售商品的企业带来了障碍。为了克服这些障碍，许多企业使用越来越无情的策略——将他们引入不诚实领域的策略——来获得更高的搜索排名。

##### 利用引擎

今天，搜索引擎的利用是一个严重的问题，但是，不管喜欢与否，大多数企业都认为这是必须要做的事情，是在线业务的当务之急。然而，为了利用搜索引擎，大多数组织必须利用搜索引擎优化公司。在这些安排中，剥削，或不劳而获，成为客户和 SEO 提供商之间互动的中心主题。

数以千计的 SEO 提供商现在都在做生意，每个排名承诺者都比下一个更有名。对于许多这样的服务提供商来说，质量不是问题。重要的是做出承诺，击败竞争对手，为他们赢得客户。面对这些巨大且通常不合理的压力，有道德的 SEO 会退出优化项目。然而，不道德的 SEO 会接手这个项目，说“没问题。我会处理的。”

“处理”一个不可能的情况意味着发送垃圾邮件。客户对不可能的需求和对不劳而获的期望将 SEO 或网站管理员推向了搜索引擎垃圾邮件的不归路。这种方法包括研究和培养越来越多的狡猾的垃圾邮件技术。

解决这个问题的最好方法是将这些方法公之于众。如果每个人都知道垃圾邮件技术，它将停止工作。这是打败搜索引擎垃圾邮件的方法，也是本文的目的。

##### 谁负责？

搜索引擎看重流行的、内容丰富的网站；然而，许多网站所有者要么不能，要么不想花必要的钱来创造这种类型的内容和知名度。所需的资源，如研究人员、网站内容开发人员、文案撰写人员和熟练的 SEO，要么没有，要么超出了公司的财务资源。

这是不劳而获的情况下，启动所有的垃圾邮件项目。

TrafficPower 是一家 SEO 提供商，因谷歌采取行动禁止该公司及其客户进入其索引而声名狼藉。一位谷歌代表称“我相信一个搜索引擎优化说服了客户要么把垃圾 JavaScript 鼠标放在重定向上，链接到其他网站的入口页面上，要么两者都放在他们客户的网站上。除了搜索引擎优化设置的门户域名之外，这还会导致客户的网站被标记为垃圾网站。”

现在，交通动力公司的客户似乎正在起诉该公司，但损害已经造成。我们仍然想知道谁是罪魁祸首。

事实上，当一个网站利用垃圾邮件策略时，最终负责的是客户，而不是 SEO 提供商。客户端可以控制网站及其部署。当垃圾邮件发生时，网站所有者要负全部责任。

##### 顶级上市的诱惑

被一些黑幕运营商宣传的很好，排名便宜好拿。这个谎言——以及由此产生的期望——迫使一些 SEO 保证排名前五。这反过来给所有 SEO 提供商施加压力，要求他们提供类似的保证。

除了激怒搜索引擎公司，这样的保证是误导。顶级排名不能像广告购买一样被列入日程表。搜索引擎的有机结果是不出售的，正是这种诚实的元素确保了它们的持续流行:买不到的是值得信赖的。

当 SEO 不能如期实现排名时，他们被迫退还或许数千美元。由于许多人几乎无力支付他们的账单，他们没有能力归还这些钱。这为 SEO 垃圾邮件创造了条件。

有些垃圾邮件发送者不在乎这种或那种方式——他们不介意欺骗，因为他们没有道德意识。也有大型搜索引擎优化公司的任务是为客户创造排名，只是不应该尝试。他们想自动化搜索引擎优化过程，以增加收入。相比之下，搜索引擎希望从它们的索引中清除任何种类的自动化材料。

网站所有者的贪婪，加上搜索引擎垃圾邮件发送者的机会主义，为一个邪恶的联盟创造了条件。这只是我见过的一个垃圾网站的例子。

Spammingsite1.com 使用了几种类型的垃圾邮件来达到很好的效果:

*   鼠标激活的重定向
*   在`<h1>`标签中填充关键字的隐藏表格单元格
*   来自人造网站的链接

最终用户看到的页面不同于搜索引擎索引的页面。搜索引擎被这些策略欺骗了，而且，就像所有垃圾邮件的情况一样，失去了对它提供给搜索用户的产品的控制。

垃圾邮件网站 1 在搜索结果中处于领先地位，但这仅仅是因为垃圾邮件。对链接到垃圾邮件站点 1 的站点的检查揭示了一个质量可疑的站点列表，合法的站点所有者不会希望与这些站点相关联。其中一个网站是一个不断增长的开放目录副本列表——这些网站从[开放目录项目](http://www.dmoz.com)中获取所有内容。开放目录列表的副本对谷歌来说是一个巨大的问题。

##### 新内容类型的危险

像谷歌和雅虎！冒险进入新型网络内容，他们冒着被代码本身的复杂性所欺骗的风险。垃圾邮件制造者通过领先于搜索引擎的技术过滤能力而获得成功。

搜索引擎在抓取网站时会应用内容过滤器，之后，在所谓的后处理中。这种复杂的过滤是美妙的，但是它也受到工程师的想象力、远见和编程的限制。垃圾邮件发送者可以利用过滤器中的漏洞来欺骗系统。

有时，无害的网站会受到惩罚，因为它们看起来有一些垃圾邮件的特征。你的网站是其中之一吗？为什么到您站点的合法链接可能无法识别？它可能看起来像是搜索引擎的付费链接。这是搜索引擎的另一个大问题:他们的过滤器如此复杂，以至于变得几乎无法控制，无辜的网站被错误地惩罚。

搜索引擎对任何给定的网站及其所有者只能看到和知道这么多。一个 SEOs 的内容和链接是另一个的垃圾邮件，所以很难判断谁是垃圾邮件制造者。搜索引擎有不同的列表和内容评估准则，这一事实使问题变得更加复杂。

当然，有许多策略被认为是垃圾邮件。以下是一些目前最常见的垃圾邮件发送技术——应该避免的策略。

*   出版帝国
*   维基百科
*   网络博客
*   论坛
*   域名垃圾邮件
*   重复域
*   没有脚本标记的链接
*   Javascript 重定向
*   动态实时页面生成
*   HTML 不可见的表格单元格
*   DHTML 铺设和隐藏层下的文本
*   庞大的机器生成的网站
*   链接填充
*   隐形短信
*   链接农场

让我们更详细地讨论每一个问题。

***出版帝国***

当一个出版商建立了大量的互联网站，它可以产生很高的 PageRank 和随后的排名。搜索引擎很难惩罚这种形式的垃圾邮件，因为链接是合法的。任何一个商业实体都有权链接自己的网站。该公司可以在网站的内容主题之间建立进一步的重叠，这样搜索引擎才会真正重视这些链接。

互联网最大的出版商之一就是这种活动的例证。该企业拥有 120 多个网站，所有这些网站都与其他网站紧密相连。在这些网站上进行搜索，你几乎可以保证在搜索结果中看到该公司的其他网站。

许多排名最成功的网站使用这个系统——这种形式的垃圾邮件非常普遍。犯罪者基本上收集 PageRank 并在其网络内链接声誉，然后创造性地使用它来支配最佳关键词短语。搜索引擎还没有找到阻止这种技术的方法，但他们将不得不这样做。这种形式的垃圾邮件是搜索结果质量的主要威胁。

***维基***

维基是任何人都可以向其发布内容的网络存储库。它们可以是展示和编辑想法的一种很好的方式，而无需严密的审查，并且已经被证明对于需要全球用户输入的项目的创建、管理和维护非常成功。

然而，尽管它们有相当大的优势，维基经常未经审查的性质使它们很容易被滥用。像链接农场一样，维基的链接对所有人都是免费的。讽刺的是，维基的价值与基于流行度的搜索引擎是一致的。其中一些 wiki 拥有很高的 pagerank，这可以使 wiki 成为一个吸引人的地方，从那里可以获得到您站点的链接。但是，如果没有人的密切控制，用户可能会简单地添加他们的链接，作为利用维基公关的一种手段。在维基的另一个用户删除链接之前，链接的网站享受这种不道德活动的好处。搜索引擎垃圾邮件发送者有控制权。

***联网博客***

博客可以是由专业人士和专家提供的精确、最新和技术细节信息的来源。因此，博客对于渴求信息的搜索者来说是非常有价值的，并且非常受欢迎。

然而，一些垃圾邮件发送者创建了一个博客，塞满了垃圾内容，如对他们在 5:15 时的想法的评论，以及一两个链接和一些关键字丰富的文本。关键词丰富的思考不会给受骗的搜索者带来真正的价值。更糟糕的是，博客通常在一个免费的链接结构中运行，这进一步验证了搜索引擎索引中的链接站点。

***论坛***

像博客一样，论坛也是相关信息的丰富来源。

不幸的是，一些论坛参与者在论坛上发表评论只是为了发布他们自己网站的链接。如果用户向另一个论坛成员提供帮助或协助，这是可以接受的。事实上，他们应该因为这些信息而获得荣誉，这些信息可能是他们努力发现的。

然而，当帖子变得过多，并且完全由油嘴滑舌或不相关的评论组成时，那么链接的价值，或者实际上，整个论坛，就会受到质疑。一些论坛所有者只是希望他们能提高搜索引擎的排名。

***域名垃圾***

也许当今最流行的垃圾邮件技术包括创建和托管大量网站。这些网站除了为主网站提供排名支持之外，很少有任何内在价值。

我以前有几个客户使用过这种技术，并因此受到了惩罚。在我让他们完全去掉重复内容后，他们的排名得到了修复。

***重复域***

为什么谷歌检测不出两个只是域名不同的一模一样的重复网站？为什么谷歌会给同样的网站同样的短语第一和第二的排名？这种情况发生得太频繁了，这是因为谷歌专注于链接与主题相关的网站。

域名垃圾邮件通常是公司试图为其每个部门或子公司建立网站的结果。那些拥有许多子公司的公司从这些领域获得了巨大的推动。意识到这一点，垃圾邮件制造者越来越鼓励客户在不同的 IP 地址甚至不同的地理位置上托管站点。

谷歌使用的链接模式检测很难处理这种做法，目前无法应对。谷歌对权威网站的新强调实际上使事情变得更糟，因为权威可以获得它实际上并不值得的信任。

***链接里面没有脚本标签***

我最近发现的一个顶级发布网站秘密地使用了 no script 标签来链接它的站点。虽然我不能说出这个网站的名字，但我可以向你展示这项技术是如何运作的。

合法使用的话，当用户的浏览器(或搜索引擎机器人)关闭了 JavaScript 时，no script 标签提供了可爬行的链接。出现在 no script 标签中的任何内容在网页上是不可见的。

要真正使用，no script 标记必须包含链接，这些链接复制了实际页面中 JavaScript 代码中使用的链接。

但是在这种情况下，这些链接指向了策略性收集 PageRank 的网站。他们基本上是隐藏的，充当地下链接网络来支持出版商的排名。这个代码几乎出现在该网站的所有域名中——并且可能存在于其他人的网站中，这些人甚至可能不知道它的存在！有些页面只使用了一个结束标签`</NO SCRIPT)`,这也会让搜索引擎感到困惑。

***动态实时页面生成***

Web 服务器有可能根据任何页面请求的引用者来生成和提供不同的优化页面。

理论上，提供一个根据请求环境定制的页面没有任何问题——事实上，许多广告活动根据被点击的横幅类型提供不同的广告。定制广告被认为对用户来说更加有效和有用。

然而，有了动态页面垃圾邮件，该站点就装载了数百个这样的幻影页面(动态 URL ),它们充当到其他一些站点的附属链接。搜索引擎不想要附属链接。在我发现的案例中，所有的链接都计入了网站的反向链接数。

我不认为这是搜索引擎开始抓取动态 URL 时的想法——他们肯定不希望允许垃圾附属链接。

***DHTML 分层和隐藏文本***

使用 DHTML 分层，垃圾邮件制造者可以在图形下隐藏关键字层。一层在视觉上覆盖了另一层，然而隐藏在下层的文本可以被搜索引擎机器人读取——这是另一种高度非法的技术。

***HTML 隐藏表格单元格***

CSS 和 html 的联合力量以及松散的`dtd <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "https://www.w3.org/TR/html4/loose.dtd">`允许不道德的网站所有者隐藏加载了关键字和标题标签的表格单元格的内容。

CSS 允许网页元素的灵活定位；这是一种灵活的编码语言，目前搜索引擎还不能完全理解。简而言之，搜索引擎并不真正知道显示的是什么。这种欺骗可以在一个单独的 CSS 表中指定。css 文件)，搜索引擎可以对其进行索引，也可以不对其进行索引。然而，这个 CSS 样式表文件确实会影响页面上内容的显示。

在本例中，CSS 影响网页正文的显示，设置为 97%:

```
{font-family: Arial, Helvetica, sans-serif; width:97%; font-size: 10pt; overflow: hidden; color: #000000; margin: 0px;}.
```

在常规代码中。gif 文件可以以 150%的宽度放在页面中，保证页面的一部分不被看到。这额外的 50%为填充到`<h1>`标签中的关键词提供了足够的空间。

***庞大的机器生成的网站***

那些不熟悉 html、dhtml 或 css 技巧的网站管理员可以尝试一些更简单的东西。当没有足够的内容时，他们通常会尝试将最少量的内容扩展到数千页。这些页面是用模板构建的，里面的句子基本上是从一页到下一页随机排列的。唯一的标题标签被插入到生成的每个页面中。

这种技术基本上看到相同的页面重复数百到数千次。它甚至可以用一个计算机程序来完成，这个程序可以系统地将文本句子、段落和标题，包括关键词，填充到页面中。

这种技术最常用于销售产品范围有限的电子商务网站。通常，产品只是简单地重新组织或洗牌，以创建另一个看起来很独特的页面。它实际上是以无数种不同方式呈现的同一系列产品。

***垃圾链接***

为了最大化整个网站的 Pagerank 分布，一些垃圾邮件发送者会用链接填充页面，直到它只是一个链接页面，并且每个页面都链接到其他页面。

为什么要这么做？嗯，通过最大化链接的数量，垃圾邮件发送者更平均地在他或她的站点中传播 PageRank。当所有这些页面的链接都指向某个关键词主题的某个页面时，该网站就可以获得该关键词的更高排名。

链接交换也被认为是垃圾链接。这些联系是捏造的——不是个人选择的真实反映。大多数链接交换现在被过滤出搜索结果；但是，链接交换中的一些链接仍然被识别。

该系统允许服务器向机器人提供与人类访客不同的内容。这意味着搜索引擎可能会被欺骗。

***隐形文字***

不可见文本是不可见的，因为字体颜色与背景或背景图像的颜色相同。

在我看到的一个例子中，一个网站使用字体颜色“雪”来使文本在白色背景上变成白色。作者还以某种方式使用了这个字体标签，导致它与另一个标签重叠，从而进一步混淆了搜索引擎机器人。

下面的例子使用了黑色。gif 作为背景隐藏黑色文字。它上面还有一个 dhtml 层，用来进一步隐藏文本。

```
<body bgcolor="#000000">  

<table width="14%" border="0" cellpadding="6" cellspacing="0" bgcolor="#FFFFFF">  

  <tr>  

    <td background="black.gif"><font color="#000000">invisible text</font></td>  

  </tr>  

</table>  

<div id="Layer1" style="position:absolute; width:200px; height:115px; z-index:1; left: 5px; top: 8px; background-image: url(black.gif); layer-background-image: url(black.gif); border: 1px none #000000;"></div>  

</body>
```

机器人不能检测 dhtml 层中的文本是否与它下面的层中使用的背景相同。该层甚至可以设置在屏幕外，这样人们就永远看不到它。

***链接农场***

链接农场在网络上仍然很普遍，即使搜索引擎可以通过链接模式识别来检测它们的存在。由于垃圾链接是在宏观层面上进行的，所以搜索引擎必须能够查看大型复杂的链接网络，并删除那些机器生成的而不是真正的人选链接。

山顶算法是一个过滤器，它将数百个无用链接所获得的优势最小化。

##### 垃圾邮件处罚

每个搜索引擎都有自己独特的禁令和相关处罚。每个惩罚是对给定垃圾邮件技术所代表的搜索引擎的威胁程度的响应。

垃圾邮件发送者可能会被扣分，通过扣分，他们的网站在特定短语上的排名可能会显著下降。或者，如果搜索引擎愿意，可以对特定页面给予零页面排名惩罚，或者可以禁止整个网站。
