# 采访 Dreamweaver 任务组的 Rachel Andrew

> 原文：<https://www.sitepoint.com/dreamweaver-task-force/>

雷切尔·安德鲁经营着自己的网络解决方案公司。公司网站 edgeofmyseat.com 也是各种网络标准文章和 Dreamweaver 扩展的大本营。

雷切尔向那些通常不会停下来考虑相关问题的人宣扬实用标准。她是网络标准项目[www.webstandards.org](http://www.webstandards.org)的 Dreamweaver 任务组的成员，也是两本书的合著者。

最近，我和她谈论了关于标准、与 Macromedia、Dreamweaver 合作以及其他许多方面的问题…

Rachel，你认为标准在更广泛的网络社区中越来越被接受了吗？

绝对的。一年前，我只是说验证一个麻烦的页面可能是个好主意，就在邮件列表上引发了激烈的争论。那不再发生了。人们对什么是 Web 标准及其重要性有了更多的了解，对使用 CSS 进行布局、使用 XHTML 而不是 HTML、确保页面可访问性等等的兴趣也越来越大。

如果人们不愿意与时俱进，他们将会被甩在后面，而行业的新来者不会像四五年前那样工作，他们将会发现自己比这些人有真正的优势。

为什么你认为现在是为完全初学者写一本涵盖标准的书的时候？

对这个行业来说，一个完全的新人学习其他任何东西都是疯狂的。Web 标准真正成为焦点，不仅仅是因为它们本身很重要，还因为使用验证标记对许多其他问题都有影响。首先是可访问性:符合标准的文档被 Web 浏览器之外的设备正确解析的几率要高得多。如果你是从一开始就开始，你可能会对网络上使用的语言有一个坚实的理解，以及如何以正确的方式使用它们。

这是一个新来者可以真正获得优势的领域，因为他们不必“忘记”我们曾经用来处理两种主要浏览器及其相关专有标签和实现的旧技术。

**作为 Dreamweaver 任务组的一员，与 Macromedia 合作感觉如何？为什么 MX 仍然默认`<b>`和`<i>`而不是`<strong>`和`<em>`？**

我认为 Dreamweaver MX 发布前最棒的事情是改变了多少。我对 Macromedia 准备投入的大量工作印象深刻，以确保 Dreamweaver MX 支持 Web 标准——包括对 XHTML、验证器和新的 CSS 属性检查器的新支持。

最终，Dreamweaver MX 为我们提供了一个可视化开发环境，它可以生成验证 HTML 或 XHTML 文档类型的页面。与 Dreamweaver 4 相比，这是一个巨大的进步，这意味着即使是那些对标准知之甚少或漠不关心的设计人员也可以制作出比以前更符合标准的网站。事实上，包含了像验证器这样的东西也提高了人们的意识。如果某个不了解标准的人通过验证器运行一个页面，并返回一些错误，他们可能会研究他们需要做什么来修复这些错误——并在这个过程中变得更有见识。

对于那些确实了解这些问题并对早期版本的 Dreamweaver 感到非常沮丧的开发人员来说，Dreamweaver MX 是一个巨大的飞跃。当然，知识和手写标记的能力是不可替代的。但是，如果您喜欢以可视化的方式工作，Dreamweaver MX 是一个工具，它可以让您以可视化的方式工作，并创建干净、兼容的标记。

有些事情会让它变得更好，正如你所指出的，有些事情仍然可以追溯到一种更古老的做事方式(使用`<strong>`和`<em>`标记，以及属性检查器默认使用字体标记，这是其中的两个)。然而，很有可能许多 Dreamweaver 的用户不知道`<strong>`是什么意思，仍然在使用字体标签而不是 CSS 的网站上工作。如果 Dreamweaver 似乎已经放弃了对这些工作方式的支持，那么 Dreamweaver 的那部分观众将会对 Macromedia 感到相当不满！

我将 Dreamweaver MX 视为产品的“垫脚石”,该产品默认采用真正符合标准的工作方式。也许下一个版本将会是那个产品，默认将会是被认为是最佳实践的。到那时，用户将有希望找到这些更新的工作方式——或者是通过 Dreamweaver 界面，或者只是因为意识在不断增长——他们不会感到如此震惊！

Web 创作工具的开发人员和浏览器制造商现在将标准遵从性作为其产品的首要特性，这一事实确实证明了这个问题变得多么重要。

你认为网络的“下一件大事”是什么？您认为富互联网应用程序(例如 Flash remoting 和 ColdFusion MX)会变得普遍甚至取代 html 范式吗？

我要拿出我的水晶球！我不知道，我认为(x)html 将在很长一段时间内成为客户端 Web 开发的主要焦点。我认为富互联网应用肯定有市场，我想随着时间的推移，我们会看到更多大规模的 Flash 产品。

然而，这么多网站的目的是发布内容，并允许访问和索引这些内容，我们将在一段时间内标记内容！我认为，在不久的将来，对普通开发人员来说最有意义的发展将是越来越多的人通过移动设备——网络电话和其他手持设备——访问网络。

你是如何开始这个职业的？

完全是偶然！我是一名训练有素的舞蹈家和歌手，如果不是因为我成为了一名母亲，我可能还在做一些与戏剧相关的事情。我怀孕的时候得到了我的第一台电脑，我真的没有打算用它做很多事情。

然而，我的 ISP 刚刚开始为他们的帐户提供网络空间，他们有一些关于“如何创建网页”的优秀教程，就像在可视化编辑器出现之前一样，这些教程是关于如何编写 HTML 和基本 JavaScript 的教程。我并不打算学习如何做这件事，因为我可能会以此为职业，我只是享受乐趣，并学习一些新的东西。当人们开始付钱给我为他们设计网站时，我感到非常惊讶！

此时，我开始意识到我在开发方面的技能远远强于设计，所以我开始更多地关注后端:服务器端脚本和数据库开发。在我女儿三岁的时候，我需要重新开始全职工作，我很好地找到了一份工作，在一家机构做网站管理员。

我现在经营着自己的公司，但我仍然觉得我现在学到的东西和早年一样多。我必须投入大量时间学习新技能，完善现有技能，但这对我来说就是乐趣所在。总有一种成就感。几乎每天我都可以精确定位某个点，或者我已经写了一段令我自豪的代码，或者我已经在纸上设计了一个应用程序，我真的很期待投入其中并构建它。是的，我是一个极客…但是一个快乐的极客！

从那以后，你见过的最大的变化是什么？

我想我很幸运能在那时开始。当时有 HTML，一些非常基础的 JavaScript 和相对基础的使用 cgi 的服务器端脚本。然而，当时制作的大多数网站都是静态的 html 网站，你不用花太长时间就能达到有足够技能与之竞争的地步。对于现在进入这个行业的人来说，前景将会非常不同。有这么多的方向可以让你进入，有这么多的技术可以让你学习，当然，每个和你交谈的人都会对什么是最好的学习有不同的看法。

最大的变化可能是网络已经成为大生意。拥有一个网站不再是一个额外的选择。许多人想了解一家公司的第一件事就是上网。如果一家公司没有网站，或者他们的网站看起来很糟糕，人们在决定是否使用该公司的服务或购买他们的产品时会考虑这一点——就像你可能不会从又脏又乱的商店买东西一样。

此外，越来越多的人从静态网站转向“网络应用”网站。即使是小规模的客户也希望能够自己更新他们的内容，他们想要交互式功能，提供会员专用的内容，等等。这意味着个人网页设计师要么需要学习构建这些功能所必需的技术，要么与其他有能力的人一起工作。很多人现在都感到压力，希望能够“全部完成”，在学习所有这些技术时，有分散精力的危险。通常，最好的办法是决定你将在哪里成为“专家”,然后，如果你独自工作，和那些能提供你所没有的技能的人在一起。

当你在招聘的时候，你想要什么？

“懂”的人。喜欢学习的人，可以充满激情地谈论他们所做的工作和他们参与的项目。那些掌握最新技能的人，那些能够谈论当前网页设计或开发问题的人，以及那些能够看到自己特定技能之外的全局的人。

当你与那些对自己的工作充满热情的人一起工作时，他们热衷于尝试新事物和提出想法，而不是仅仅“做我们一直在做的事情”，这会创造一个真正富有成效和令人愉快的环境。我想和这种人一起工作。
