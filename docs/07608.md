## 采访记录

路易斯:大家好，欢迎来到 SitePoint 播客的另一集。今天和我一起参加节目的是 SitePoint 最新内容网站的执行主编，他是 JSPro.com，科林·伊格里克。你好，科林。

科林:嗨，路易斯。

路易斯:嗨，欢迎来到我们的节目。

科林:谢谢你邀请我。

路易斯:是的，我们刚刚推出了一个新网站，实际上，是关于 JavaScript 的。你想谈谈 JSPro 是什么以及你是如何参与其中的吗？

科林:好的，那么 JSPro 基本上是 SitePoint network 的一个新的与 JavaScript 相关的地方。我们将涉及 jQuery、Node.js、原生 JavaScript——普通的老式 JavaScript——以及这个主题的所有内容。我参与其中的方式是，我最初只是 SitePoint 的一名作家，我问编辑汤姆，“我能做些什么来更好地参与进来？”我本来只是打算写更多的文章，但他说，“哦，好吧，我们有这个新网站推出。您希望如何使用它？”我说，“当然。”

**路易:**对。所以我认为你从事 JavaScript 已经有一段时间了？

科林:我在高中时就自学了，基本上，从那以后，我的每份工作都是用它来完成的。

**路易:**对。当我发现这个新网站推出时，让我感到惊讶的事情之一是，我想，“哦，是的，太棒了——一个由 JavaScript 开发的好的新网站——这很有意义。”然后我对自己说，“嗯，真有趣。我不知道还有哪个网站能让我想到通过 JavaScript 找到教程和信息。”所以我的头顶上什么也没有，但是即使我往里面看，我也找不到任何东西。所以我想知道，这是否只是 JavaScript 发展新阶段的一种标志，即对于一个专注于 JavaScript 的网站来说，现在是时候了？我在想 CoffeeScript 和 Backbone.js 和 Node 之类的东西才出现了几年，所以你觉得现在是时候吗？

科林:嗯，我认为这是几种情况的结合。首先，很长一段时间以来，JavaScript 名声不佳，它只是青少年放在他们的网站上让人眼前一亮的东西。第二，有很多博客和随机的网站都有一些内容，但是 JavaScript 已经发展到有这么多不同的库和所有东西的地步——客户端，服务器端
——要把所有东西都放在一个地方真的很难，我们需要一个由真实的人组成的网站，或者像 JSPro 这样的工作列表。

路易:是的，我很期待看到你将要报道的内容。例如，你使用过 Node.js 吗？

科林:是的，我个人和专业上都和它合作过。

**路易:**对。对于那些不熟悉 Node 和 JavaScript 在服务器上的作用的听众来说，你想考虑在什么情况下使用 Node？

科林:node . js 确实很棒，因为很多人只把它和服务器联系在一起，但你可以用它做很多其他的事情，比如运行脚本，比如 shell 脚本，而不是必须使用 DOS 或 Perl 之类的东西。但是就服务器开发而言，它非常适合需要大量吞吐量的网站，因此每秒需要处理数万个请求，因为 Node.js 的引擎与 Apache 和微软的 web 服务器有着根本的不同。

**Louis:** 而对于一个从客户端 JavaScript 背景过来的人来说，在服务器端开始玩 JavaScript 是不是一个大狗屎——
端。有很多非常不同的 API 吗？

Colin:node . js 有一个相当小的核心，但是有很多库，所以你可以去 GitHub 上找一个 API 来做任何事情。它仍然只是普通的旧 JavaScript，所以你不必学习新的语法或类似的东西。编程模型都是异步的，这让我想起了客户端的 GUI 编程。所以他按了一个按钮，浏览器窗口中发生了一些事情。使用 Node.js，有人连接到您的服务器，然后发生了一些事情，或者您连接到数据库，然后等待其他事情发生，并使您的结果能够从查询中返回。所以这和管理 VHD 有很大不同。但是如果你熟悉客户端的 JavaScript，而不是输出。

**路易:**对。你提到它是普通的 JavaScript。这让我开始谈论 CoffeeScript。CoffeeScript 是你合作过的作品吗？

科林:我没和它合作过。我读了一些我们在 JSPro 上发布的文章。就我个人而言，我真的不喜欢复合 JavaScript 语言，但那是个人的事情。

路易:对，我也差不多是这样。我总是看着它并想，嗯，它有点漂亮，但是 JavaScript 并没有那么难。我没有发现打那些花括号真的会占用我那么多时间。

科林:对我来说，这也是一个问题，你写你的 CoffeeScript，然后你很可能把它编译成 JavaScript。如果有一个 bug，你需要调试 JavaScript，无论如何你都需要这个节点，所以。

路易:对。

科林:就像你说的，这不是最难学的语言，至少不是最基础的。

**路易:**对。所以你提到 JSPro 上已经有相当多的内容了。在你担任总编的第一段时间里，你最喜欢或最喜欢的作品是什么？

科林:对。我想，我们已经开始了大约两个星期，我一直在和几个作者合作:一个人，詹姆斯·爱德华兹，他被称为“兄弟蛋糕”。他在模块化设计等方面做了一些非常好的工作，在接下来的几周内，我们将有另一位作者杰夫·佩里森(Jeff Pereason)的五部
系列……抱歉，我把他的名字读错了，但他基本上将开发一款带有潜艇、导弹和鱼雷的 HTML5 游戏——这真的很酷。所以，你应该注意这一点。

**路易:**对。我知道有相当多的 HTML5 画布游戏库。这是他直接用 Canvas API 从头开始做的，还是他在使用某种库？

科林:我相信他正在和 jQuery 合作。

路易:没错，但没什么特别的游戏？

科林:我不这么认为，不。

**路易:**对。当我谈到框架或库的话题时，有没有什么关于 JavaScript 框架或库的东西是你最近遇到的，并且你认为非常酷的？

科林:我最近接触的很多东西都是 Cheerio，我不认为它很新。

路易:这对我来说是新的。对我来说是新的。

**科林:**是的，如果你不熟悉 Cheerio，它基本上是 Node.js 的 jQuery。所以，假设你正在进行网页抓取，你将所有这些网页拉入你的服务器来解析它们，而不是试图使用正则表达式或任何其他你会尝试的东西，你可以建立宿舍，然后使用与 jQuery 客户端相同的选择器进行查询。

路易:好吧。这太酷了。我刚从 GitHub 页面上找到的。我认为它还可以对 HTML 进行就地编辑？

科林:是的。

Louis: 所以它的操作选项和 jQuery 一样？

科林:是的。所以我发现它不如 jQuery 健壮，但它相当不错，也不老，所以我认为它只会越来越好。

**路易:**对。它是只支持 HTML 还是也支持 XML？或者这可能是他们的计划？

科林:我还没有用 XML 试过。我知道它不能为你解析 JavaScript，所以我最近做了大量的网络搜集工作，当真正解析 JavaScript 时，你可能需要使用另一个叫做 VM 的模块，你可以在其中解析并执行 JavaScript。这有点像 eval 函数，但是更好。与此完全相反的是，我一直在使用无头浏览器。它被称为 PhantomJS，基本上是一个 webkit，除了没有 GUI，所以你可以用它做任何事情。

是的，我对幻想曲很熟悉。我们用它来做一些事情，其中之一就是以无头模式运行 JavaScript 测试。

科林:到目前为止我都很喜欢。

路易:对。我在 Cheerio GitHub 页面上，它确实提到它也可以解析 XML 文档。所以这肯定很有趣。我会在展示笔记里放一个 GitHub 页面的链接。酷毙了。我想和你谈的另一件事是你在 JSPro 上写的一篇文章，我想，它指的是一种新的 JavaScript API，它被添加到一些浏览器中，这就是导航定时 API。

科林:对，对。最初，我为 SitePoint 写了那篇文章。那是在我还是个作家的时候。我们为全国发布的 JSPro 做了内容，所以我们把它发表在 JSPro 上。API 本身对客户端的网页体验并不太有用，但对开发人员来说，它有助于了解页面加载的情况。在这些 API 出现之前，许多人使用日期比较，这有很多缺点，例如:众所周知，JavaScript 计时不准确，但是我们只能计时这么多，因为只有从 JavaScript 加载到页面完成加载才能真正测试您的加载时间。你根本不知道服务器请求和 DNS 查询之类的事情是怎么回事。因此，导航定时 API 直接内置在浏览器中，它可以让您对定时数字进行更精细的控制。

**路易:**对。那么，这是否能让你获得很多通常通过查看网络选项卡获得的信息，比如 Chrome 中的开发工具？

科林:是的，差不多，除了这个可以让你以编程的方式使用它，而且它不是 Chrome 特有的。

**路易:**对。我只是在浏览一下可以参加的活动清单——内容相当广泛。例如，它的粒度有多细？
我可以查看作为页面加载一部分的每张图片的加载时间吗，或者是整个页面的加载时间？

科林:更多的是为了页面整体。

**路易:**对。所以在实践中，我想，你可能会使用的是…而不是在本地进行所有的调试，然后说，“嗯，从我的角度来看，它看起来很快”，例如，你可以为你的一些用户包含一些使用此 API 的代码，然后向服务器报告，看看你的站点在野外的性能如何？

科林:是的，没错，因为当你测试你自己的网页时，你通常在服务器附近或者在公司网络后面。当事情到了野外，任何事情都可能发生。因此，与你在 Chrome 上运行网站相比，这可以让你从世界另一端的某个人那里获得更多的报道。

**路易:**对。目前浏览器对这个 API 的支持怎么样？

科林:哦，那篇文章是我几个月前写的，所以我现在得去 caniuse.com 查一下。

**路易:**对。我们到了。刚拉上来。貌似 9 的 IE，所有现代的火狐和 Chrome，4 的安卓，黑莓 10，安卓的 Chrome 和火狐。所以它目前在 Opera 和 Safari 以及 iOS Safari 中缺失。我猜 IE 8 也是，但是 IE 8 不算。

科林:我认为这是很好的支持。我的意思是，任何时候你可以让网络浏览器工作，这总是一个加号。

**Louis:** 我看到 JSPro 上发布了更多的内容，几乎每天都有？目前的日程表是什么样的？

科林:目前的日程是周一、周三、周五和周六。

**路易:**对。我还知道你也在寻找作者？

科林:是啊，当然。我们一直在寻找更多的内容和更多的作者。只要去 JSPro.com，向下滚动到页面底部，点击“成为作者”链接，它会有你需要的所有说明，基本上，与我联系，我们将从那里开始。如果我们认为你是一个足够好的作家，那么我们发送一篇文章，如果你被接受，那么欢迎加入这个团队。

**路易:**对。你有什么特别想要的吗？这和 JavaScript as go 有什么关系吗？

科林:是的。显然，它必须与 JavaScript 相关，但我们的页面上有一个侧菜单:jQuery、API、工具和库、引擎、node . js——我们正在寻找所有这些东西以及更多。特别是，就我个人而言，现在我想要一篇 jQuery 介绍文章。我有几个人给我发了一些文章，内容包括如何写游戏和如何写插件，但是没有关于如何开始的，所以那会很棒。有几个人给我发来了 Node.js 的请求，但是，正如我前面所说的，有很多不同的库，所以肯定会在 Node.js 上寻找更多的东西。

路易:好吧。是的，所以再一次，任何人想检查一下，地址是 JSPro.com。就像你说的，成为作者的链接在页面底部。我可能会在节目坚果里放一个链接。

科林:还有最后一件事。如果你感兴趣，如果你是一个读者，在页面的右边有一个框，让你订阅我们的时事通讯，这将在未来几周内开始，所以注册吧。

路易:好吧。非常感谢你，科林，这周抽出时间和我谈话。如果有听众想了解你，你有自己的博客或推特吗，或者你现在只在 JSPro.com 上吗？

cjihrig。如果您有兴趣关注我，我在 JSPro 上的作者链接应该有所有这些信息。
