# 我喜欢辛纳屈的 7 件事

> 原文:[https://www.sitepoint.com/7-things-i-love-about-sinatra/](https://www.sitepoint.com/7-things-i-love-about-sinatra/)

几年前，我第一次听说了辛纳特拉，从此就爱上了它。对于那些没有听说过的人来说，Sinatra 是一种用于创建 web 应用程序的 DSL。它是用 Ruby 编写的，最初是由 Blake Mizerany 设计的，但是现在有一个核心团队负责维护和开发代码。如果你想知道，它是以弗兰克·辛纳屈命名的，显然是因为

> 他值得拥有一个以他的名字命名的网络框架。

以下是我如此喜爱辛纳屈的七个原因:

## 简单

您可以使用 Sinatra 在一个文件中创建一个全功能的 web 应用程序。不需要担心复杂的设置过程、配置或生成器:您只需打开一个文本编辑器就可以开始工作了。路由实现起来很简单，并且有很好的语法，使用了所有的 Http 动词 GET、POST、PUT 和 DELETE 使创建 RESTful 应用程序变得轻而易举。Sinatra 简化版让您编写简单而优雅的代码，产生惊人的效果。

## 灵活性

Sinatra 可以用来构建任何东西，从最小的微型网站到全面的 web 应用程序。它是 API 实现、中间件、小部件、脸书应用以及许多其他应用的完美选择。我经常用它来尝试新想法或快速制作网站原型。

Sinatra 构建在 Rack 之上，使得使用 Ruby Gems、Rack-Middleware 和它自己的扩展 API 进行扩展变得容易。这为扩展您的应用程序打开了一个可能性的世界，并避免您必须重新发明轮子。

## 它很轻

没有太多浪费的代码——基本上足够让你运行了，没有更多的。这意味着它没有 Rails 和其他框架在助手函数和生成器脚本方面的所有功能，但它也不会将您束缚在以某种方式做事，使其变得非常灵活。当你在 Sinatra 中编写应用程序时，不会有浪费的功能或未使用的功能，这意味着它使用更少的内存和运行更快。

## 公正的

你不需要使用 MVC 模式，也不需要依赖任何特定的 ORM (Active Record、Sequel、Mongoid 等)、JavaScript 框架(JQuery、MooTools、YUI 等)或模板系统(Haml、ERB、Slim 等)。这使您，即开发人员，回到了驾驶位置，让您完全控制如何最好地实现您的应用程序。这有时会使 Sinatra 开发看起来像是蛮荒的西部，当然会导致一些糟糕的代码…但是，嘿，至少这是你的糟糕代码！

## 它有助于提高您的 Ruby 熟练度

Sinatra 中的所有东西都是 Ruby 代码，所以你可以在编写应用程序时进行大量的练习。因为后台运行的“魔法”较少，所以您通常必须自己想办法实现某些类型的功能。这不一定是件坏事——总会有人愿意帮忙(见第 6 点)。Sinatra 不像 Rails 那样有自己的“编码风格”,更多的是传统的 Ruby 风格。Sinatra 的源代码只有 1500 多行有据可查的代码，这意味着你可以自己轻松阅读。它清晰、简洁，包含了一些优秀的 Ruby 编程实践的范例。

## 这个社区太棒了

辛纳特拉有一个友好和充满活力的社区，他们总是乐于助人。官网上的文档也很有价值。如果你被困住了，你可以去谷歌群组或者在 irc 频道(# sinatrarb)上找人。

## 很好玩！

这是最重要的一点——Sinatra 让我很享受摆弄代码的乐趣。我可以很快尝试新的想法或用不同的方法进行实验，而不会有任何大的开销。我不需要遵循任何指导方针或规则，我可以走我自己的路，让事情立刻运转起来，然后享受一整天的时间。

# 你和辛纳屈的第一次约会

你需要做的就是安装辛纳特拉宝石:

```
 $> gem install sinatra 
```

然后打开文本编辑器，输入以下两行:

```
 require "sinatra"
get("/") { "Hello World!" } 
```

这就是你的应用——没错，只有两行！你现在需要做的就是启动服务器。打开终端，在命令行中键入以下内容:

```
 $> ruby hello_world.rb 
```

你的第一个 Sinatra 应用可以在 [http://localhost:4567](http://localhost:4567) 找到。去吧，到处玩一玩……你会很快坠入爱河的！
