# 谷歌浪潮的再见

> 原文：<https://www.sitepoint.com/google-scraps-wave/>

谷歌以用户缺乏兴趣为由，取消了其 Wave 协作信息服务。谷歌高级运营副总裁 Urs hlz le 评论道:

> Wave 还没有看到我们所希望的用户接受度。我们不打算继续把 Wave 作为一个独立的产品来开发，但我们至少会在今年年底之前维护这个网站，并将这项技术扩展到谷歌的其他项目中。

Wave 的不被接受是出乎意料的。谷歌希望 Wave 成为在线通信发展的下一步:它在一个中心位置结合了电子邮件、聊天、头脑风暴、日程安排、文档、推特和订阅。开发者也可以使用 API 来创建他们自己的协作工具和小部件。

Google 最大的问题:*没人理解 Wave* 。虽然它在去年的 I/O 大会上获得了热烈的掌声，但该公司很难解释这个概念。尽管有大量的视频和 5 月份的[产品重新发布，Wave 仍然令人困惑，很少有人认识到它的好处。在我们最近的调查中，39%的受访者承认他们不了解该产品——考虑到 SitePoint 拥有高度技术化的受众，这是一个令人担忧的统计数据。](https://www.sitepoint.com/will-you-surf-on-googles-wave/)

像许多人一样，我在 Wave 发布后不久就尝试过，但很快就放弃了。早期版本漏洞百出，我无法测试协作功能，因为很少有人使用它。其他问题包括:

1.  尽管它被宣传为电子邮件、聊天和 Twitter 的替代品，但它与这些系统都不兼容。Wave 是一个独立的信息工具——你只能用它来联系其他 Wave 用户。
2.  系统不支持 Internet Explorer。忽视 50%以上的潜在市场不会提高采用率，尤其是当公司是主要目标时。
3.  Google Buzz 盖过了 Wave，引发了几个安全问题，造成了更多的混乱。
4.  早期的大肆宣传只能导致失望。

值得称赞的是，谷歌进行了许多冒险的努力，但当一个产品失败时，它会迅速果断地做出反应。它不怕承认失败——许多其他公司可以从这个例子中学习。
