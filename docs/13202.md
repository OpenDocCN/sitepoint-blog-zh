# 工业工程和标准:克里斯·威尔逊会谈

> 原文:[https://www.sitepoint.com/ie-standards-chris-wilson/](https://www.sitepoint.com/ie-standards-chris-wilson/)

在今年的 Web Directions South 上，我有机会与克里斯·威尔逊进行了一次无需介绍的交谈。他是微软 Internet Explorer 的平台架构师。如果你愿意，你可以听这个采访。

你好，克里斯。

克里斯·威尔逊:你好。

在过去的一年里，我听了你的几次演讲，我听到的共同主题是，你试图让开发人员站在微软的立场上，以及你在开发 Internet Explorer 时必须面对的现实。

克里斯·威尔逊:嗯嗯。

SP:你认为为什么现在微软的名声这么差，尤其是在开发者和设计者当中？

**克里斯·威尔逊:**嗯，我认为很多设计师和开发人员看到了我们今天拥有的平台和一些多年来没有得到解决的问题，他们非常沮丧，因为他们不得不解决这些问题，在某些情况下，他们花了很多时间来解决这些问题。我认为，我现在相信的一件事是，实际上，在过去的六个月左右的时间里，我们从 IE7 的发布中得到了非常积极的回应，正是来自这种群体。你知道，我一直在和一些人交谈，他们说“我对使用 IE6 感到非常不安，我喜欢 IE7”，我真的很惊喜你知道，“这不是我能要求的一切，但这确实是我需要的一切，是让我更容易完成工作的一切。”

哦，当然。

克里斯·威尔逊:这正是我们追求的目标。

没有一个浏览器版本——特别是没有一个单一的浏览器版本——会给你你绝对想要的一切，但是当然——是的，我想特别感谢你的关注，特别是在采用和支持 IE7 标准方面。这是绝对需要做的。

你在演讲的最后提到，Web 2.0 正在改变一些必须做出的假设。你能分享一下这对于下一个 IE 版本到底意味着什么的细节吗？

**克里斯·威尔逊:**这并不是说我们正在改变具体的功能。更多的是我们需要优化的场景，包括我们提供的 API 和功能，以及这些功能的性能调整等等。这是人们为网络编程的方式，他们建立网络存在的方式，与五年前有所不同——十年前，当然，甚至是五年前。就是不一样。你知道，那时你没有非常普遍的高度互动的 JavaScript 应用程序。当然有一些在那里，但是他们不是，你知道，几乎每个你去的网站的首页，我认为几年后会是。

是的。所以，你花了一点时间说，“是的，我们正在开发下一个版本的 Internet Explorer。”就在微软宣布他们正在开发 IE 之后，我们开始看到——你知道，我们推出了 IE 博客。我们开始看到的不是详细的路线图，而是一些初步的东西，“这些是我们要解决的首要问题。即使是我们不能告诉你的事情，这也是我们对我们所承担的事情的推理。”我们还没有真正看到 IE8。是不是太早了？

**克里斯·威尔逊:**嗯，我们在微软要努力解决的一件事就是预先宣布——比如，在我们真的真的非常有信心我们能推出高质量的产品之前就宣布。因为我们是一家如此大的公司，因为我们向如此多的人发货，但事实并非如此，我说的不是“因为我们向如此多的最终用户发货”，而是我们向如此多的公司和其他企业发货。真的，你知道，我们为许多其他行业树立了榜样。当我们发布新版本的 IE 浏览器时，很多公司都必须去测试他们的软件，你知道，他们必须弄清楚他们的工具应该产生什么样的内容，等等。因为我们这样做，因为有那么多人依赖我们，所以当我们宣布“这正是我们正在做的”或“这是我们的目标日期”时，我们必须非常非常小心，以 100%的信心当我们不这样做时，我们会让很多人对我们不满，当然，不仅仅是他们对我们不满；实际上，如果他们打赌我们会在给定的时间内发布某些东西，或者打赌我们会发布某个给定的功能，而我们却不发布，这可能会损害他们的商业模式。

SP:你确实提到过你——或者说你的团队正在为下一版本的 IE 开发一个全新的渲染引擎，我相信很多人会想，你知道，有 Gecko，有 WebKit，它们都是开源的，非常棒！微软现在真的有必要从头开始构建一个全新的渲染引擎吗？仅仅是许可问题吗？

克里斯·威尔逊:绝对不会，有几个原因。第一个是，我们实际上并没有建造一个全新的引擎，就像整个引擎一样——在 IE 中，我们的组件很久以前就被命名为“三叉戟”它有点像渲染引擎、对象模型和解析器——所有这些部分都被推到了一起。我们现在真正关注的是其中的布局引擎部分，这个部分决定了如何布局文本、内容、边框等。并且能够管理一般的文本流。这是一个在 CSS 2.1 规范中有很多细节的领域，你知道，我们很久以前就开始了这个领域，我们意识到我们可以继续修复它，我们可以继续对它做一些事情并添加更多的功能，因为我们实际上有许多功能我们从未完全利用它们。例如，在 IE5.5 中，我们引入了这种模板技术来为打印进行分页，并且我们有在屏幕场景中使用这种技术的想法，但是我们从来没有着手去做。所以我们本来可以做到这一点，但我们真正环顾公司，我们意识到，“哇，我们是微软。这里有很多有设计专长的人。有很多人真的知道如何正确布局。”因为我们已经很多年没有真正地对这些代码进行重大的修改了，哇，我们需要解放那些人，你知道，真正地允许他们做一些真正不可思议的事情。我想你可能会看到当我们发布为什么我们这样做。有很多好处，我们真的可以让自己在未来做一些令人惊叹的事情。当然，这并不意味着第一版就能做到所有这些，就像其他任何东西一样。但是它让我们在未来能够做一些真正伟大的事情。

至于在 WebKit 或 Gecko 或任何其他引擎上进行构建，我确信其中一部分可能是我们必须跨越一些许可挑战。但是，对我来说，最大的原因是，当我们发布代码时，特别是向 5 亿人发布代码时，我们有一种真正的责任，我们将为一组不属于我们的代码承担这种责任，我们没有想出它，我们不是这方面的专家，那里有很多代码。但是如果出了问题，我们会受到责备；除此之外，如果出了问题，我们也要对此负责。所以，你知道，如果有一个安全漏洞，我们必须去修复它，我们必须立即提供修复，或者它–我们正在为此而努力。这让它有点挑战性。当我看着其他工具包时，我认为他们在完成一些标准工作方面投入了大量的工作，或者，你知道，采用一堆我们还没有达到的不同标准，我认为他们已经做了一些伟大的事情；我不认为这很好，唯一的答案真的应该是，“让我们去拿那个引擎。”我知道，我非常希望从我的工作的标准化方面，你知道，从我的 HTML 工作组联合主席，以及协调我们的其他标准参与方面，看到一件事情，我也想指出的一件事情是，真的，你有越多不同的实现，你就越能更好地制定你的标准。因为，否则，你最终只能依靠一个实现来定义标准是什么，在某种程度上，IE 曾经也是这样。在此之前，网景公司定义了标准。我的意思是，我们必须与网景兼容，因为那是人们写内容的地方。所以我认为拥有多个实现确实是一件好事；这就是 W3C 要求它的原因。

你提到了你的 5 亿用户，你最近也说了很多，这是微软痴迷于保持向后兼容性的驱动力。保持向后兼容性的需要，特别是在 Internet Explorer 中，有多少是因为一个系统上一次只能安装一个版本的 Internet Explorer 这个古老的问题？

克里斯·威尔逊:令人惊讶的是，非常少，因为对我们来说，真正的挑战不是我们只能允许一种做事方式。虽然实际上完全并行不同版本的 IE 非常困难，但对我们来说，为旧的做事方式引入单独的代码路径并保持兼容是非常容易的。问题是知道什么时候打开它，知道对任何给定的内容使用哪条路径。这确实是我们兼容性挑战的关键，你知道，当我说在未来我们将不得不要求作者选择符合标准的行为时，基本上是这样的-当我们从网上获得一个网页，我们开始解析它并试图呈现它时，我们不知道该代码符合标准的程度如何。我们真的不知道。没有办法判断代码是否符合标准。这意味着我们真的不知道我们应该遵循哪一套行为。所以我认为，如果你希望用户这么做，用户并不知道，对吗？当我访问一个网页时，我不想说，“哦，看起来不太对，也许我应该试试旧的渲染引擎。”我也不想强迫人们在不同的浏览器之间来回切换。当然，如果我们允许多个版本的 IE 在同一个系统上并行运行，你仍然会遇到这样的问题，你点击一个链接，它需要在一个引擎中打开，你点击另一个链接，它需要在另一个引擎中打开。以这种方式浏览将是一个真正的挑战。用户模型会非常混乱。

是的。然而，至少到目前为止，像 Firefox 和 Opera 这样的竞争浏览器确实已经开始“用户决定他们想要使用的渲染引擎”，即使只是通过选择何时更新到最新版本。

克里斯·威尔逊:你知道，用户可以选择使用哪个浏览器。

是的。

克里斯·威尔逊:你可以同时使用其他一些浏览器，尽管有时这也会导致一些问题。但是我想你会发现这对网络开发者来说是一件好事。Web 开发人员需要那种在一台机器上运行多个版本的能力。这就是我们走上这条路的真正原因，嗯，“对我们来说，最好的方法是什么？”最好的方法——就像，即使我们今天解决了 IE 中的问题，也不能解决 IE 过去版本的问题。我们不能真的回去修改代码让它并行工作，因为如果我们这么做了，它就不再是 IE6 了，而是 IE6 的一个残缺版本。但这意味着我们只能在未来解决它。我们真的不想只是为了未来而解决它。我们能做的最好的事情是提供虚拟化技术来实现这一点，这就是为什么我们走上了这条道路，“嘿，让我们弄清楚如何提供一个免费的 Windows XP 虚拟机，以及一个免费的虚拟机，你知道，有 IE6，并且你可以升级到 IE7。”

SP:这是一个巨大的问题，我想这也是为什么人们普遍批评“为什么我不能安装多个版本的 IE 浏览器？”真的消失了，因为这是来自开发者社区的批评。

克里斯·威尔逊:很大程度上，是的。但这里的挑战是，你知道，当你想到普通的最终用户时，他们甚至不愿意考虑运行多个不同的浏览器，更不用说多个不同的浏览器版本了。他们只是想让网络工作，对吗？他们希望所有这些浏览器都可以互操作，并且都能正常工作。

然而，这种开发者选择加入的模式阻止了一件诱人的事情，那就是，如果我们能让所有旧的网络内容在一个新的渲染引擎上显示，会怎么样？这会带来什么样的可能性？仅仅因为一个站点不再被积极开发并不意味着一个使用新的渲染引擎的浏览器扩展不能让旧的站点以新的方式歌唱。那就太好了！

克里斯·威尔逊:但你是说浏览器扩展可以获取旧内容，并通过新的引擎呈现出来吗？

SP:如果旧内容恰好符合标准，或者足够简单，或者写得足够好，可以在新的渲染引擎中显示，那么能够利用这一点就太好了，而开发者选择加入模式在某种程度上阻止了这一点。

克里斯·威尔逊:嗯，但是如果你想的话，你已经可以这么做了。你当然可以编写一个插件，强制所有东西都处于“标准模式”——无论当时标准模式的等级如何。我认为真正的挑战是，这不是 web 开发人员编写内容的方式。比如，他们所做的就是写内容——让我们假设他们确实按照标准写了内容。他们开始的时候，你知道，所有相关的标准都打印在他们面前的桌子上，他们以这种方式编写他们的内容，然后他们需要让它发挥作用。他们为了让它工作而做的事情往往是对我们最有害的事情。你知道，他们在 IE7 中应用了导致问题的黑客技术，但这是使 IE6 正确渲染所必需的。我想你可能会发现其他浏览器也是如此。有时是因为它们有缺陷，或者缺少功能或其他什么；只是规范中有不明确的地方，他们用一种方式做，而其他浏览器用另一种方式做，你知道，web 开发人员必须处理这个问题。他们必须让它在所有浏览器中以相同的方式工作，这就是重点——就像，如果你可以获得完全符合标准的内容，然后试图从那里修复它，那将是非常棒的。问题是没有人以这种方式编写内容，部分原因是 a)对他们来说没有意义——他们不是在编写标准测试套件，而是在编写网页——b)他们没有任何东西来测试它，以验证他们确实按照标准完全正确地做了，这就是它看起来的样子。

在你的演讲中，有一点让我印象深刻，那就是，在理想世界中，随着时间的推移，使一个符合标准的网站适应新浏览器的难度应该会趋于零。

克里斯·威尔逊:是的。

SP:现在，有了这个开发者选择加入的模型，如果它进入 HTML 规范，所有浏览器最终都会采用，随着时间的推移，浏览器会有，你知道，五、六、七、十、十八个不同的呈现模式，或者随着时间的推移，这种差异趋向于零的趋势是否意味着有一天这种情况会消失？

**克里斯·威尔逊:**我认为，我个人的希望当然是有一天这种情况会消失，浏览器版本之间的差异不是我们内部所说的“突破性变化”它们不会真的有可能击垮写信给它们的人，你只是在那之后添加了一些东西。我不确定我们是否真的会达到这一步，因为我查看了所有现有的标准，包括 CSS 2.1 和 HTML 4.01，以及我们对 HTML 5 和所有其他相关 web 标准所做的工作，这些标准中的每一个都在纠正以前版本中的问题。它在消除歧义，纠正问题。我希望我们今天所写的规范——我知道我们今天所写的规范更能说明应该发生什么。它们更清晰，不会留给开发者太多的想象空间。与此同时，我知道还有一些东西没有声明，希望我们能够在发布标准之前修补其中的大部分漏洞，但是在某个时候，你知道，我们会提出一个我们认为符合标准的实现，然后其他人会提出一个我们认为符合标准的实现，他们确实符合标准，因为标准并没有做出决定。这确实是我们在 IE 中遇到的大量问题，因为我们在 CSS 1 推荐标准出现之前编写了我们的第一个 CSS 实现。我们写了第二个，你知道，在那之后不久，我想，或者实际上，可能还是在那之前——CSS 1 花了一段时间才盖章。但是，你知道，我们没有 CSS 2 的所有功能。我们没有指定这些特性，所以我们在 CSS 工作组中的那些人在我们的脑海中有不同的图片，这些图片是如何将大图结合在一起的，以及那些 CSS 1 特性应该如何交互的。所以，你知道，我们把它内化成代码的方式有点不同。

你是 W3 HTML 5 工作组的联合主席。

克里斯·威尔逊:是的。

你从那份名单中收到了多少邮件？

克里斯·威尔逊:嗯，我的意思是，任何人都可以加入是一件好事，所以你可以自己去发现！

我已经加入了那个名单…

克里斯·威尔逊:老实说，很多，当然我也有一些——我自己的工作邮件可能比这更多。但我认为，这个工作组平均每天大概会发出 50 条左右的信息。

你说过你——至少，你愿意承担这个角色的一个重要原因是你相信开放标准。

克里斯·威尔逊:嗯嗯。

到目前为止，你担任联席主席的经历是检验还是强化了这种信念？

**克里斯·威尔逊:**这当然没有动摇我的信念，那就是这需要发生，我们需要公开开发可以自由实施的标准。我仍然认为这是绝对重要的，这就是为什么我还在那里。当然存在一些挑战。对于 W3C 来说，这个组织实际上是一种开放的实验，我们在这方面面临一些挑战。你知道，有很多不同的意见，很难知道最终什么应该被认为是共识。到底什么才算公平？作为工作组主席之一，这当然对我来说是一个巨大的挑战。我们必须解决这个问题。

我相信其他工作组，比如 CSS 工作组，在他们仅有的一点空闲时间里，都在饶有兴趣地关注着。

克里斯·威尔逊:我绝对这么认为，是的。

太好了。非常感谢克里斯和我们聊天…

克里斯·威尔逊:谢谢！

还有，回头见。

克里斯·威尔逊:西亚！
