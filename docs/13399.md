# ASP.NET 和 CSS 与 Expression Web

> 原文:[https://www . site point . com/aspnet-and-CSS-with-expression-web/](https://www.sitepoint.com/aspnet-and-css-with-expression-web/)

最近，微软推出了一款新的 HTML 和 CSS 工具，名为 Expression Web。微软在网页设计方面并没有良好的记录；事实上，我有几个项目专门致力于修复用 FrontPage 制作的网站。因为我和 aside 一起做了很多工作，而且大部分是我自己的设计工作，所以我把恐惧放在一边，尝试了一下。

我用它做了几个基础项目，最初印象不太好。在很大程度上，它感觉像是 Dreamweaver 的一个不太精致的版本。然而，当我开始四处打探的时候，我发现它相当不错。

对于我以前的大多数项目，我会使用 Dreamweaver 编写标记和模板，然后使用 Visual Studio 使其工作。Dreamweaver 的代码着色和智能对 CSS 来说更好，当测试不同的字体大小或调整颜色时，拆分视图非常有用。最大的缺点是 Dreamweaver 不支持许多简洁的 ASP.NET 2.0 功能，如母版页，并且使用服务器控件总是有点棘手。

这就是 Expression Web 的用武之地；对于 HTML 和 CSS 工作，它几乎和 Dreamweaver 一样好，并且它支持绝大多数。NET 2.0 的特性和控件非常好。它也很好地为 CSS 新手提供了一些帮助。CSS 对话框不仅在应用样式方面提供了一些帮助，而且在构造选择器方面也提供了一些帮助。另一个不错的 CSS 特性是属性面板，它列出了当前应用于元素的所有样式，甚至包括那些被覆盖的样式。这已经成为解决 CSS 特异性问题的一个很好的工具。

总的来说，Expression Web 是一个可靠的工具。它确实有一些奇怪的地方，但这是第一次发布所期望的。很高兴看到 Micosoft 不仅提高了浏览器的标准，他们还开发了支持它的工具。我仍然会为我的 PHP 工作保留一份 Dreamweaver，但是 Expression Web 已经在我的硬盘上赢得了一席之地。
