# AJAX 是跨浏览器吗？

> 原文:[https://www.sitepoint.com/is-ajax-cross-browser/](https://www.sitepoint.com/is-ajax-cross-browser/)

在道场开发商 T2·亚历克斯·罗素的博客 T3 上，一场精彩的辩论正在进行。由[的 ajaxWrite](http://www.linspire.com/ajaxwrite.php) 的发布引发的争论引发了对 AJAX 对于跨浏览器兼容性到底意味着什么(如果有的话)的质疑。

ajaxWrite 不像 Writely T1 那样使用跨浏览器的 DHTML，而是使用 T2 可扩展用户界面语言 T3，这意味着它只能在火狐这样的 Mozilla 浏览器上工作。亚历克斯·罗素认为，用 AJAX 的名字来宣传这样一个应用程序是既误导又有害的。

一方面，AJAX 代表异步 JavaScript 和 XML。如果这被认为是该技术的完整定义，那么 AJAX 并没有强制要求向用户呈现界面的技术。尽管通常使用 DHTML(又名 DOM 脚本)编写，但是用 Mozilla 的 XUL 编写的用户界面是完全允许的。通过使用 JSON (AJAJ？).

另一方面，大多数 AJAX 应用程序相对于 ActiveX、Flash 和 Java 小程序的最大好处是，它可以跨浏览器工作，而不管可用的插件或专有技术。亚历克斯·罗素认为，通过使用 AJAX 的名字来推广使用特定浏览器用户界面技术的产品，从而破坏了 AJAX 的这一隐含优势，是“无耻营销”的一种“可憎行为”。

与此同时，ajaxWrite 开发者迈克尔·罗伯逊(Michael Robertson)已经在 T2 宣布“ajaxWrite 仅仅是一个开始”——他的公司承诺每周三发布一款新的应用，可以通过 ajaxlaunch.com 的 T4 进入。有理由期待这些都具有 XUL 界面，名字以“ajax”开头。
