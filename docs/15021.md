# 辩论——手工编码:终极自由

> 原文:[https://www.sitepoint.com/hand-coding-ultimate-freedom/](https://www.sitepoint.com/hand-coding-ultimate-freedom/)

***又是一场 SitePoint 辩论！今天，两位经验丰富的开发人员争论是手工编写 HTML 代码好，还是在所见即所得编辑器中创建网页好。不要错过[反对派的论点](http://www.webmasterbase.com/article/969)——他们可能会改变你对这场争论的看法……***

当需要坐下来为新网站的布局编写 HTML 代码时，您有两种选择:使用 WYSIWYG(所见即所得)编辑器，或者在记事本之类的文本编辑器中手工编写代码。

这两种方法都有一个学习曲线。这就是所有开发人员必须做出的决定:是使用所见即所得编辑器来生成 html 代码，还是投入更多的时间来学习“手工编码”HTML。

对于那些还不知道的人来说，所见即所得编辑器的基本卖点是易于使用。你不用写任何代码就可以创建你的整个网站。不用写出`<img src="images/main.jpg">`，您只需选择 Insert > Image，代码就会自动生成。这在使用表格时要有利得多，而不仅仅是插入图像。您只需告诉编辑器您想要多少行和多少列，而不是许多行看似复杂的 HTML，它就会为您完成工作。

最受欢迎的编辑器 Dreamweaver 在 Macromedia.com 上售价 399 美元。[深红编辑器](http://www.crimsoneditor.com/)(一个伟大的记事本替代品)是 0 美元。

##### HTML 并不适合所有人

虽然使用 WYSIWYG 编辑器为您生成代码可能是一个有吸引力的选择，但从长远来看，它不会给你最好的最终结果。

编码布局会变得非常混乱。html 最难的部分是使用表格，大多数布局使用十几个表格(有时更多)来获得正确的外观。你需要花很多时间使用 html 来熟悉这样的布局编码。手动学习代码要比使用编辑器花费更多的时间。

让我把这一点说清楚:对于只打算做一两个网站的人来说，可能不值得他们花时间去学习 HTML(没有到可以编写复杂布局代码的程度)。对于这些人来说，使用编辑器可能是更好的选择。然而，对于那些从事创建网站的人来说，如果你手工编写代码，你会比那些使用所见即所得编辑器编写代码的人更好，更有优势。

说手动编码更好是因为使用编辑器是“欺骗”只是无知。

##### 学习曲线

直到你熟悉了复杂的布局，并且花了相当多的时间处理代码，你才真正感受到学习 HTML 的全部好处。这意味着它需要更多的时间来开始制作网站。

这并不意味着你会浪费时间。如果你以前没有 HTML 的经验，学习使用编辑器可能是一项相当艰巨的任务，甚至可以和学习手工编码相提并论。如今，几乎在任何书店，你都可以找到 500 多页关于 Dreamweaver 和 Frontpage 的书籍，更不用说每本书附带的大量教程了。看完所有这些至少要花一两天的时间。

编码的标准方式是使用许多表格来定位站点上的元素。Yahoo.com 在他们的主页上使用了大约 35 张表格，而 SitePoint.com 的 T2 只用了大约 15 张。编写这样的布局代码可能会非常复杂。

然而，HTML 是标记——它不像大多数编程语言那样复杂。学习编码表对任何人来说都是可以实现的，不管他们是否有计算机或其他语言的经验。只要有愿望，任何人都可以在几周内——或者更少——通过一些常规练习和一点耐心，学会手工编写表格。

然而，现在这种经过时间考验的使用表的方法正在被 CSS Positioning 所取代。这是一种在页面上定位元素的新方法，完全不使用表。您可以告诉浏览器将元素放在哪里(例如，上边距之下 100 个像素，左边距之外 250 个像素)，而不是对表格进行编码以强制将元素放入所需的位置。这将继续让 Web 开发人员变得更加容易，特别是对于那些手动编写代码的人来说。使用 CSS-P 还有很多其他的好处，比如页面的文件大小更小，搜索引擎更容易为你的内容建立索引(从而在搜索引擎中获得更高的评级)，以及除了计算机之外，屏幕阅读器和 PDA 氏症患者等设备也可以访问你的内容。为了进一步了解 CSS-P，我推荐以下两篇文章: [CSS 定位](http://www.brainjar.com/css/positioning)和 [HTML 乌托邦——无需表格设计](http://www.webmasterbase.com/article/379)。

人们经常喜欢吹嘘他们“在记事本上写代码”，好像这在某种程度上使他们成为更好的开发人员。但是对于手工编码者来说，还有其他的选择。我推荐使用免费的 Crimson 编辑器或其他一些不错的文本编辑器。它有语法高亮显示(也叫颜色编码)，可以一次打开许多文件，还有一些其他的功能。它旨在帮助开发人员编写代码(但不仅仅是 html)。但是，它不会为您填充标签或属性。

##### 学习手工编码的优势

一旦你投入时间学习熟练地编写代码，你将享受到一些很酷的选择和自由，而那些受限于编辑的人却没有。

***时间和控制***

一旦你能够自信而熟练地编写代码，创建一个新的布局将比使用编辑器花费更少的时间。控制极小的细节，让布局看起来恰到好处变得更加容易。

***使用服务器端代码***

使用任何服务器端语言变得更加容易。如果你对 html 的了解非常有限，使用 PHP 或 ASP 会变得非常困难——如果不是不可能的话。同样，你对 html 越熟悉，事情就变得越容易。当我使用 PHP 时，我真的很期待有机会写一些好的表格！另外值得一提的是，目前还没有针对 Linux 的编辑器，所以如果您需要使用 Linux 机器来编写一些代码，您必须手工完成。

***版面维护和更新***

当你需要改变网站的布局时，如果你熟悉它的代码，这是非常容易的，这可以节省时间，并为你提供更多的控制你所做的改变(不弄乱任何东西)。

编辑有他们的位置，他们擅长他们所做的。我用过很多 Dreamweaver，我甚至教过它，但最终，如果开发人员决定不使用它，他或她会好得多。懂了 HTML，很多原本很麻烦的事情变得轻而易举。我很高兴我花了几个小时学习手工编码，而不是使用编辑器。

你怎么看？不要错过[反对者的观点](http://www.webmasterbase.com/article/969)——他们可能会改变你对手写代码争议的看法……
