# 充分利用 Firefox

> 原文:[https://www.sitepoint.com/making-the-most-of-firefox/](https://www.sitepoint.com/making-the-most-of-firefox/)

web 开发社区目前正积极地关注着凤凰/火鸟/Mozilla 浏览器系列的最新版本 [Firefox 0.8](http://www.mozilla.org/products/firefox/) 的发布消息。该版本标志着 Mozilla 项目的独立浏览器的第三次也是最后一次名称更改，这是由于与 Firebird 开源数据库项目的命名冲突造成的。[更名常见问题解答](http://www.mozilla.org/projects/firefox/firefox-name-faq.html)解释了新名称背后的原因。

主要的下载网址已经饱和了一整天，所以如果你想下载一个浏览器，你最好的选择是尝试一个[镜像](http://www.mozilla.org/mirrors.html)。

我今天已经测试了最新的版本，我的总体印象非常好。这款浏览器似乎比火鸟 0.7 快得多，火鸟 0.7 本身当然也不逊色。新图标集([由乔恩·希克斯](http://www.hicksdesign.co.uk/journal/archives/000377.php)设计)非常棒，史蒂文·加里蒂的 [Mozilla 视觉识别团队](http://actsofvolition.com/archives/2004/february/brandingmozilla)投资的其余工作也是如此。

从 web 开发的角度来看，Firefox 中最令人兴奋的新增功能是 [DOM Inspector](http://www.mozilla.org/projects/inspector/faq.html) 。这个探索浏览器文档对象模型的无价工具长期以来一直是整个 Mozilla 套件的杀手级特性，它包含在 Firefox 中进一步加强了浏览器作为基本开发工具的地位。

要真正充分利用 Firefox，您需要探索一些适用于浏览器的优秀扩展。我一直最喜欢的是[鼠标手势](http://optimoz.mozdev.org/gestures/)，一旦你花时间学习如何使用它们，它可以极大地提高你浏览网页的速度。 [LiveHTTPHeaders](http://livehttpheaders.mozdev.org/) 是另一个必不可少的安装，尽管它主要是作为服务器端开发的调试工具。最后，没有 Jesse Ruderman 的 [Web 开发书签](http://www.squarefree.com/bookmarklets/webdevel.html)的健康剂量，任何 Gecko-engine 浏览器安装都是不完整的，这些书签节省了我几天的时间，这主要归功于“测试样式”、“编辑样式”和“祖先”的强大组合。如果你还没有尝试过，你真的不会相信你错过了什么。
