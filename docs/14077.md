# 诺基亚将在手机浏览器中使用 Safari 组件

> 原文:[https://www . site point . com/Nokia-to-use-safari-components-for-mobile-browser/](https://www.sitepoint.com/nokia-to-use-safari-components-for-mobile-browser/)

这[揭示了苹果最近开放 KHTML 项目中修改后的核心组件源代码的举动，这些组件构成了 Safari 网络浏览器。诺基亚打算在其 60 系列智能手机平台上使用它。](http://press.nokia.com/PR/200506/998214_5.html)

苹果的这一明智之举让他们能够利用 Safari source 在向诺基亚手机浏览器过渡的过程中核心元素将发生的变化，或许还有资产可以引入其他产品。我们也不要忘记——当 iPod 部门在苹果公司成立，拥有自己的 R & D 和产品管道时，人们认为从长远来看，这将生产出比 iPod 更多的产品。

尽管我并不建议在不久的将来使用苹果 iPhone 或 iPDA。移动浏览技术肯定会派上用场。在某个时候，苹果可能会选择将其部分数字版权管理授权给其他移动硬件制造商，然后在这些设备上安装音乐播放器和 Quick Time 以及移动 Safari。通过一些改进的屏幕——彩色 iPod 可以很容易地结合 WiFi 和移动浏览器。

知道我们许多人都在不同的层面上探索移动 web 开发，这当然是值得关注的事情。它还表明，并非所有事情都是出于纯粹的利他主义或基于用户社区的抱怨——正如我们许多人认为的那样，这是源代码发布背后的核心驱动力。很明显，像诺基亚这样规模的公司愿意给 Safari 贴上“同类最佳开源解决方案”的标签，这是苹果向整个开源社区发布代码的绝佳动机。

话虽如此——它也肯定会受益于许多默默无闻、才华横溢的开发人员的贡献，他们在移动和桌面平台的社区中默默工作。
