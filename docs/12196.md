# 发布 CodeBurner 1.1 版:Firebug 的参考工具

> 原文:[https://www . site point . com/announcing-version-11-of-code burner-the-reference-tool-for-firebug/](https://www.sitepoint.com/announcing-version-11-of-codeburner-the-reference-tool-for-firebug/)

CodeBurner 的新版本已经发布。这个版本不依赖于 Firebug——它是一个独立的扩展，并且在有限的时间内与一本免费的书捆绑在一起！[立即拿起你的。](https://www.sitepoint.com/browser-devtools-secrets-start-up-network-and-performance/)T3】

我非常高兴地宣布我们非常有用的 Firebug 扩展的更新——以前称为 FireScope，现在称为 **CodeBurner** 。

我们更改了名称，因为 FireScope 已经被该行业的另一个组织使用，为了避免任何混淆，我们认为我们应该重新开始。惭愧，我很喜欢这个名字，我们花了几次迭代才想出这个名字！这让我同情火狐的开发者，他们在找到目前的名字之前经历了几个名字(还记得 Phoenix 吗，有人记得吗？).

![CodeBurner](../Images/a197f4202b3dcaf001ce12b638560195.png)

现在我们有了一个可以坚持的名字，而且不用再改了！你会看到我们仍然有同样时髦的标志和字体，当然，扩展本身仍然像以前一样有用和灵活。事实上，我们刚刚发布了 1.1 版，这是一个小而方便的升级版本，具有如下强大功能:

*   增加了对 Firebug 1.4 的支持
*   代码示例中的在线参考资料的直接链接，可以更方便地访问详细信息和现场演示(如果适用)
*   用户界面改进
*   修复了试图查看不存在的代码示例时的错误处理
*   国际化的固定支持级别类

如您所见，名称可能有所不同，但 CodeBurner 对于对查找 HTML 和 CSS 的语法、浏览器兼容性信息和代码示例感兴趣的 web 开发人员来说仍然是一个非常有用的扩展。你可以从扩展的主页安装最新版本，在[https://getfirebug.com/](https://getfirebug.com/)。

尽情享受吧！
