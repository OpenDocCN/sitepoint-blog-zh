# 微软、诺基亚采用 jQuery

> 原文:[https://www.sitepoint.com/microsoft-nokia-adopt-jquery/](https://www.sitepoint.com/microsoft-nokia-adopt-jquery/)

在 ASP.net 博客上，微软刚刚宣布 [jQuery 将随 Visual Studio](http://weblogs.asp.net/scottgu/archive/2008/09/28/jquery-and-microsoft.aspx) 的未来版本一起发布。该库将支持智能感知代码完成，并将很好地与 ASP.NET AJAX 一起工作。微软还将为该项目贡献测试和补丁，并将“按原样”发布，这意味着开发人员将能够利用所有现有的 jQuery 库和示例。

目前，在 ASP.NET 上开发 Javascript 通常涉及到微软自己的 JS 产品[ASP.NET AJAX 框架](http://www.asp.net/ajax/)。虽然它肯定是一个强大的框架，但它是 ASP 世界的原型，缺乏 jQuery 的简洁和简单。在公告博客上，Scott Guthrie 列举了一些 jQuery 样本来展示这个库的强大，并解释了为什么它被选为平台的一部分。Scott Hanselman 在他的计算机 Zen 博客上有一个[有趣的演示](http://www.hanselman.com/blog/jQueryToShipWithASPNETMVCAndVisualStudio.aspx)，演示了集成的本质。

同样，诺基亚也将在他们的 Web 运行时平台上提供 jQuery。该库将分布在所有包含运行时的手机上，使开发人员能够在为该平台构建应用程序时使用 jQuery。

这显然是 ASP.NET 开发者的一大胜利，给了他们开箱即用的成熟 JS 库的力量；jQuery 很好地抽象了主流浏览器的特性，并完成了这项工作。但是这对灯的世界意味着什么呢？jQuery 在过去的几年里非常受欢迎，并不缺少知名用户。这两个公告代表了 jQuery 平台的另一个重大胜利。

早在 2005 年，萨达姆·阿扎德就构建了一个庞大的 Javascript 库和框架列表；回顾这份清单，大多数项目从未达到临界质量，一些项目被合并，而其他项目仍然被孤立，有利于主要参与者。随着 Javascript 库市场的巩固，Prototype、jQuery 和 Mootools 已经成为开发者的选择；[谷歌似乎同意](http://google.com/trends?q=prototype%2Cjquery%2Cspry%2Cyui%2Cmootools&ctab=0&geo=all&date=all&sort=0)(注意，“原型”的歧义。).

Drupal 不久前采用了 jQuery，随着诺基亚的发布，这个库在平台上越来越受欢迎。无论你是在开发 CMS 插件还是移动网络应用，jQuery 都是一个很好的机会——还有无数其他的例子。作为一个框架，jQuery 是一个成熟的 Visual Basic——它解决了为 web 开发交互性的问题，虽然它可能不是最强大的选择，但它无疑是最受欢迎的选择。如果你一直在考虑你的 JS 框架，也许是时候尝试一下 jQuery 了。
