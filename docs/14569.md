# W3C 网络应用研讨会

> 原文:[https://www.sitepoint.com/w3c-workshop-on-web-applications/](https://www.sitepoint.com/w3c-workshop-on-web-applications/)

W3C 关于 Web 应用程序和复合文档的研讨会于本周早些时候举行。我对其目的的理解是给有影响力的人(以及在职者)提供一个指导 W3 努力的机会。如果你有时间，有必要看一下发布演示幻灯片的[议程](https://www.w3.org/2004/04/webapps-cdf-ws/agenda)，感受一下发生了什么。

有些“重点笔记”似乎是；

–W3 应专注于统一和利用现有标准–使整体大于部分之和
–超越浏览器:网络应“适用于”无线/手持设备
–声明式 XML 用户界面语言(如 XUL 或 XAML)是可取的

戴夫·拉格特的[网络应用新方向](https://www.w3.org/2004/06/dsr-slides.pdf) (PDF)尤其值得一看。Dave 赢得了 IMO 的“保持真实”奖，这是 HTML 表单标签的“发明者”所期待的。同样值得注意的是 Opera / Mozilla *联合* [立场文件](https://www.w3.org/2004/04/webapps-cdf-ws/papers/opera.html)——更多关于那个@ [Mozillazine](http://www.mozillazine.org/talkback.html?article=4788) 。

与此同时, [XULPlanet](http://www.xulplanet.com) 的[尼尔·迪肯](http://www.xulplanet.com/ndeakin/)进行了一次[的现实核查](http://www.xulplanet.com/ndeakin/article/261)。
