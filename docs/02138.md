现在，基本上你可以在页面上贴一个 PHP 标签，重新加载页面，它就会出现在管理中，然后你就可以开始了。您可以为结构化内容选择一个模板，然后开始编辑。你真的可以在一个小时左右的时间内编辑一个站点，完成安装，然后开始运行。显然，我们现在做的不仅仅是这些，但是基本的用例仍然存在，并且有很多 Perch 用户真的只做这些。他们制作了一个静态网站，他们制作了一些可编辑的内容，然后发送给客户。

Tim:

说到 CMS 内部的设计，我注意到你也非常关注的一件事就是 CSS 和 CSS 布局。在版本控制节目中，我们通常不谈论趋势技术，但我想花点时间谈谈 CSS 布局，特别是因为我看到了它的发展方向——用 CSS 编写布局的未来。你想花点时间谈谈你一直关注并为之兴奋的新 CSS 布局特性吗？

Rachel:

是啊，当然。我一直在关注 [CSS 网格布局](https://www.w3.org/TR/css-grid-1/)，这是一个新兴的规范，它第一次真正为 web 创建了一个基于网格的布局系统。很明显，我们已经有了像`float`和定位等东西很长时间了。我想大多数听众可能都意识到了使用浮动创建类似网格布局的问题。我们现在有了 [Flexbox](https://www.w3.org/TR/css-flexbox-1/) 。Flexbox 太棒了。这真的很棒，只要你试图布局的东西本质上是一维的——所以你试图布局一行或一列。

你会看到很多人试图把 Flexbox 变成一个网格系统。这是一项艰巨的工作，因为它不是为此而设计的。事实上，我今天早上回复了一个人，并为他做了一个简单的例子，来展示 Flexbox 和网格的区别，因为网格是二维的。所以你可以同时对行和列进行布局。

我想这个规范已经存在了大约四年了。它实际上是从微软开始的，它的这种版本。虽然，有些想法在 specs 中存在的时间比那更长。微软在 IE10 中实现了它的早期版本，从那以后它一直作为一个规范在开发中，现在在 Chrome、Firefox 和 Safari(开发者预览版)中落后于 flags。

所以来了。规格差不多完成了。显然，在旗帜被移除并发布之前，将会有一个大的测试过程，以确保一切正常。但是现在你可以玩的东西太多了。

Tim [7:53]:

似乎现在，在[can use](http://caniuse.com/#feat=css-grid)上，你是对的。它背后有很多旗帜，微软似乎对它有最好的支持，这在传统上是闻所未闻的，但看到它非常令人兴奋。看起来你所说的是我们可以期待它，不是很快，而是很快，能在所有的 evergreen 平台上使用。

Rachel:

是啊，肯定的。可能在明年年初，就发布周期之类的事情来说，以及人们在哪里。如果你在浏览器中启用了标志，我有很多例子可以让人们看。

有很多有用的东西。我认为我们将要看到的与 Flexbox 的诞生方式截然不同。对于 Flexbox，它出现在供应商前缀之后，所以人们可以开始使用它。然后就变了，然后又变了。然后大家都说，<q>啊，这个真的 bug，</q>感觉用起来不是很有信心。

对于网格布局来说，这将是非常不同的，因为所有的改变和对规范和实现的干扰都发生在标志的背后。唯一真正受到影响的人是我和我的例子，显然还有正在实施的浏览器供应商，但我们没有很多网站依赖于本质上是实验性的东西。

所以我认为当它出来的时候，人们会非常惊讶，它将会跨浏览器出现，并且跨浏览器相当标准，所以人们将能够使用和依赖它。

Tim:

这是一个非常令人兴奋的前景。你居然写了一本书叫， *[准备 CSS 网格布局](https://abookapart.com/products/get-ready-for-css-grid-layout)* ，中间隔了一本书。在写这篇文章的时候，你对网格布局有什么有趣的、令人兴奋的或意想不到的了解吗？

Rachel:

在这一点上，我写道，我知道规格相当好。自从我在 IE10 中第一次发现它，我就一直在研究这个规范。我现在是 CSS 工作组的特邀专家，布局是我真正关注的事情。

所以这并不奇怪。我认为我发现这个规范令人兴奋的地方是当我向人们介绍它的时候。我在开始的时候提到过，我一直在旅行，很多时间都是在像[一个单独的事件](https://aneventapart.com/)之类的事情上，关于 CSS 网格布局，以及向人们展示示例。

人们看到它时非常兴奋，因为它解决了许多布局问题。你可能会想，为什么我们以前没有这样做呢？你知道吗？我认为这是很酷的事情，实际上是解释它，人们看到它实际上是如何简单地使用，并对它感到兴奋，并期待拥有它。那真的很酷。

David:

我认为开发者一看到 Flexbox 就爱上了它，因为它解决了现有布局和浮动等存在的许多问题。现在他们在看格子，我觉得这是一个重新恋爱的新机会。我很好奇你是否认为这是对 Flexbox 的补充，或者是取代 Flexbox 的东西，人们应该为此做些什么准备。

Rachel:

这个问题经常出现。这些规格是互补的。它们代表两种不同的东西。正如我提到的，最大的区别是 Flexbox 是一维的，而 Grid 是二维的。从另一个角度来看，如果你有一堆项目，比如一些导航或其他什么，你真正关心的是布局方法看它们有多大，并很好地将它们隔开，这是 Flexbox 的一个用例。如果你想创建一个网格，然后向它扔东西，然后说，<q>现在你需要遵循这个网格的线条，</q>这就是网格的作用。Flexbox 更像是一种内容输出的方法，而 Grid 更像是在说，<q>这是一种结构，这种东西必须进入它，必须适合它。</q>

我们将看到人们一起使用这两者，因为在你的设计中，你会有一些你想放入那个结构的东西，你会有其他的东西——你会希望内容间隔开，这取决于你在那里有什么。

有很多相似之处。有一个规范叫做 [Box Alignment](https://www.w3.org/TR/css-align-3/) ，它本质上是 Flexbox 中所有好的对齐方式，但是被放在一个单独的规范中。Grid Layout 使用该规范进行对齐，因此名称略有不同的对齐属性完全相同，因为显然我们在 Grid Layout 中没有 Flex 容器，但它们在两种布局方法中的工作方式是相同的。

所以我认为如果人们很了解 Flexbox，他们会发现他们所理解的可以很好地移植到网格上。你只是在二维空间工作。

Tim [12:14]:

网格布局和 CSS3 [多栏布局](https://www.w3.org/TR/css3-multicol/)有什么关系吗？

Rachel:

不。还是那句话，不同的布局方法。这是一种不同的内容布局方式。有一些相似之处，在网格布局中我们有`grid-gap`，它有`column-gap`和`row-gap`属性，其工作方式与多列布局中的`column-gap`非常相似。它们是完全不同的东西。多栏的好处是可以让内容流动，并根据有多少栏进来而重新流动，这是一种稍微不同的做事方式。我认为这些东西都将一起工作，这取决于你想要实现的布局类型。

这绝对令人兴奋，而且肯定会让我的日常工作轻松很多。

Rachel:

是的。非常酷。这是一个非常激动人心的时刻。我感到非常幸运，它仍然是令人兴奋的，因为它对我来说，近 20 年前，当我开始这样做。

David:

Rachel:

是啊。

是啊。我能做到。我其实是个前舞者。我的背景，我训练做舞蹈，我上大学做舞蹈。我最终停止了跳舞，去了后台，在技术剧院工作。然后就怀上了女儿。那个女儿现在 19 岁了。她刚刚回大学去了。

我真的与自己无关，也与挣钱无关，因为我曾受训成为一名舞蹈演员，并在剧院工作。我只知道这些。我所知道的是如何打字。我想，好吧，我会买一台文字处理机，因为有了这个宝贝，我可以边打字边工作，我就能做到这一点。那是我可以在家做的事情。我被《个人电脑世界》的一名员工卖给了一台电脑，而不是文字处理器。这台电脑可以连接到互联网上。

从那以后，我发现了网络，事实上你可以把东西发布到网上，事实上你需要建立一个网站来把你的东西放在那里，因为那时没有脸书或者任何地方可以放你的照片。我刚刚开始学习这些东西，因为我想能够交流。我想把我的东西放到网上。这真的是一切开始的地方——只是自学这些东西，因为它看起来很有趣。

我很幸运，因为那时候你很快就成为了了解网站的人。一个有慈善机构的朋友说，<q>哦，你能帮我们建立一个网站吗？</q>于是我照做了。这些事情导致了各种低收入的自由职业，但最终导致能够在这个行业找到一份工作，实际上是在 2000 年为一家网络公司工作。这真的是一切的开始。

David:

我喜欢这样。符合很多人的故事。每个人都是从跳舞开始的。

Rachel:

只是一个非常明显的举动，是的。

David:
