# 采访:卡梅隆·亚当斯和 DrawPad

> 原文:[https://www.sitepoint.com/interview-cameron-adams-drawpad/](https://www.sitepoint.com/interview-cameron-adams-drawpad/)

在今年早些时候的 Ignite Sydney 活动中，与会者被要求使用一款 iPad 应用程序来绘制“是什么激发了你？”。 [Cameron Adams](http://twitter.com/themaninblue) 又名蓝衣男子，是 [DrawPad](https://vimeo.com/23967407) iPad 应用程序的开发者，并友好地同意与 BuildMobile 谈论此事。

你好，卡梅隆，为了那些不知道你和你的工作的人，你能告诉我们你是谁，你是做什么的吗？T3】

Hi Paul,

我经营我的网站“蓝衣男子”已经不到 10 年了，其中大部分时间我是一名自由设计师&程序员。我通常很难决定我是更像一个设计师还是一个程序员，所以我为自己开辟了一个利基，通过从事将两者结合成独特体验的项目——这是你仅靠设计师或程序员无法实现的。

这也帮助我站在了 web 技术的最前沿，我很幸运地在 2005 年左右赶上了 JavaScript 的流行浪潮，最终写了两本关于 JavaScript 的书，也写了几本 CSS 书。

今天，我从事任何能帮助我最好地实现我的想象力的技术，无论是 JavaScript、Flash、Processing，还是像 WebGL 这样的新兴技术。

***听起来很酷的小众。那么，在 DrawPad 中，设计师和编码员哪一方扮演了更重要的角色呢？虽然我们可以观看视频或阅读您的帖子，但也许您可以通过告诉我们 DrawPad 到底是什么来回答这个问题？*T3】**

Drawpad 肯定更多的是来自编码方面，因为它借鉴了其他人创建的一些现有项目，所以这个想法已经相当成熟了。主要是对结果进行编码，并用我的审美判断来指引我前进。此外，这类项目的截止日期总是很紧，这意味着你必须为皮革编写代码，以便在晚上做好准备。

自从看到 Grafitti Analysis 团队的工作后，我想用更容易获得的技术创建一个版本，即你可以在网络浏览器中运行的东西，并看看人们在现场活动中会如何使用它。iPad 的外形也很吸引人，因为人们可以伸出手触摸“画布”并用手指画画，所以很容易让人们在夜总会/酒吧类型的场所与它互动。

我和 Ignite Sydney 的运营人员关系很好，他们使用我的 [Visible Tweets](http://visibletweets.com/) 应用程序通过投影仪整夜显示实时推文。在与组织者交谈后，我们决定让观众在晚上更加参与进来，这似乎是我部署 Drawpad 的完美场所，所以我在活动开始前两周开始着手这项工作。

Drawpad 实际上有两个部分——绘图界面和显示界面。对于绘图界面，我们基本上有两个非常友好的人拿着 iPads 在人群中漫步，与人们交谈，让他们画出激发他们灵感的东西(因为 Ignite 是一个非常鼓舞人心的夜晚)。iPads 上的数据被输入网络服务器，一台连接到投影仪的笔记本电脑接收 2D 的绘图数据，并将其转换成一个看起来很时髦的 3D 图像，Z 轴是时间。通过这种方式，你可以得到人们所画的，但是随着时间的推移而被挤压的图像。当你移动一个虚拟相机通过这个 3D 物体时，你会得到一些很好的抽象形状，这些形状可以分解成人们所画的东西。

简而言之，这是一个让人们参与进来并在大型活动中吸引眼球的好方法。

***哇，那里有很多励志的东西可以去看看，谢谢。再深入一点，大部分动作都在包含`touchStart`、`touchMove`和`touchEnd`方法的`draw.js`文件中。你能告诉我们这一切是如何联系在一起的吗？***

当然可以。页面上几乎唯一的 HTML 元素是一个填充整个屏幕的`<canvas>`。我跟踪人们手指在页面上的移动，然后用这些坐标画出一条路径。

当用户将手指放在屏幕上时，canvas 元素接收到一个`touchStart`事件。这是 DrawPad 开始新路径的信号。关于浏览器中的触摸事件，最有趣的是，通常(在浏览器中)我们习惯于接收一个代表一个交互点的事件——比如说，鼠标点击或按键——但对于 iPad 这样的多点触摸设备，用户实际上可能会与无数个点进行交互(如果算上手指和脚趾，有 20 个),而`touchStart/touchMove/touchEnd`中的事件对象包含所有这些点的信息。

这意味着你必须小心你正在处理的触摸事件的哪个*部分*。你不会想把它们弄混，然后开始认为某人的小指是他们的食指。我见过的几个用于 iPad 的画布绘图应用程序只是在第一次触摸后丢弃任何触摸，所以你不会获得真正的多点触摸绘图体验。我想确定那个抽屉给了你这个。

在传递给`touchStart`的事件对象中，有一个`event.touches`属性列出了所有当前跟踪的触摸点(作为一个数组)。如果你想处理多点触摸，你必须遍历数组并单独处理每个点。在`touchStart`处理程序中，我就是这样做的，为我看到的每个新的接触点创建一个新的路径对象。幸运的是，每个触摸点都有一个唯一的标识符，因此当该点/手指四处移动时，您可以跟踪它，不会将其与任何其他点混淆。

一旦创建了路径的起点，我就监听任何`touchMove`事件。当这些被触发时，我检查触摸的标识符和点的新位置，然后将其添加到适当的路径。

同样，对于`touchEnd`，我识别触摸，然后关闭它的路径。使用`touchEnd`你必须注意的一件事是`event.touches`将不包含任何数据(因为触摸已经结束)，所以你必须查看`event.changedTouches`。

***很爽，我也差不多能跟上。体验的第二部分，其中数据以 z 轴为时间输入到 3D 可视化中，令人难以置信。请告诉我们一些关于它的事情，以及它在这个问题的上下文中意味着什么:web 技术将朝哪个方向发展？*T3】**

从人们的 iPad 涂鸦中产生的 3D 视觉效果有多种用途:A)看起来很酷 B)它从每个人的画中产生一些独特的东西(添加一些他们自己不能做的东西)C)它为整个体验提供一种神秘感和好奇心——这些流动的 3D 形式直到最后几秒钟才被识别出来，当它们结合到某人的画中时。

在公共项目中，A 总是强制的。对我来说，B 也是必要的，因为这是我作为数字艺术家的重要角色之一。我既是人们创作的推动者，也是我自己创作的创造者。这有点像我和参与者之间的舞蹈——他们给我一些东西，我用自己的动作回应。

人们在 2D 的 iPads 上画画，所以为了把他们变成三维物体，我用画他们的画的时间作为第三维。这样，他们的每一笔都有深度，与画的时间相对应。通过给每幅画一个 3D 结构，我可以做一系列的相机效果，让它看起来像你在每幅画中飞翔。

为了做 3D，我使用了一个名为 [Three.js](https://github.com/mrdoob/three.js/) 的库，它使用 WebGL 在浏览器中绘制 3D 对象。我没有足够的时间来确定绘制 3D 笔画所需的精确几何形状，所以我用球体来伪造它——每个 3D 笔画都由一堆球体组成，根据 x，y &时间或特定的笔画来定位。有点像 3D 像素。

有了这样一个项目，你可以很快看到网络技术成为事实上的过程，通过这个过程任何数字体验都可以被传递。传统上，像这样的安装需要像 Processing、Cinder 或 OpenFrameworks 这样的本机框架来实现。现在我们开始在浏览器中看到同样的事情。这对于可移植性和可访问性来说是非常好的——任何人都可以在网上看到它，而不需要下载一个特殊的程序，我可以简单地选择任何一台有互联网连接的计算机，并用它来显示我的可视化。笔记本电脑、台式机、Windows、Mac 没关系。

也有一些人已经熟悉 JavaScript 和浏览器技术，所以当这种类型的交互和图形功能对他们可用时，它会创建一个巨大的完全开放的数字艺术家社区。这通常会带来令人兴奋的结果！

这太棒了，非常感谢卡梅隆。我可以不停地问问题，但最后一个简短的问题。毫无疑问，音乐是你生活中很重要的一部分，对于大多数写代码的人来说似乎也是如此。我们做这个采访后，大多数人都放了音轨？T3】

在过去的一个月里，我最常播放的歌曲有点失真，因为我正在为 2011 年的[网络方向制作片头，片头设定为磁性人的“我需要空气”。我想我在三周的时间里大概听了一百遍。谢天谢地，我还没有完全厌倦，因为这是一首好歌。](http://south11.webdirections.org/)

为了获得真正的听觉享受，我一直在听福斯特乐队的专辑《火炬》、尼禄的《欢迎现实》和倭黑猩猩的《黑沙》。
