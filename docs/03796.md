其他地方几乎没有什么新闻；IE 上升，Firefox 下降，Opera 拒绝从 1.62%的水平上让步。所以让我们把注意力转向 Safari …

## Safari 是新 IE 吗？

诺兰·劳森最近的文章[*“Safari 是新 IE”*](http://nolanlawson.com/2015/06/30/safari-is-the-new-ie/)有一个有争议的标题，但提出了几个有趣的观点。苹果没有放弃 Safari，但可能会被指责故意忽视:

*   Safari 开发落后于其他浏览器。看一眼 caniuse.com 就能看出缺失的 web APIs 或前缀背后的 API 的数量。
*   浏览器的发布进度缓慢，通常与操作系统更新同步。新功能和错误修复越来越少。
*   苹果对标准的采用有很大的影响力，但是他们对 W3C 的参与并不热情。该公司乐于实施自己的 iOS 专用“标准”，同时拒绝添加其他地方发明的标准，例如[指针事件](http://timkadlec.com/2015/02/apples-web/)。
*   苹果很少在网络会议或在线开发社区上露面。
*   公司没有受到这种行为的影响，部分原因是我们避免批评他们。

从商业角度来说，这是有道理的。应用程序销售的 30%佣金远远高于网络应用程序的 0%。这让人想起微软在 2001 年宣布浏览器已经到达了一个顶点，联网的本地智能客户端(应用程序)是未来。现在的主要区别？苹果从未发布公告。

如果苹果不那么强大，或者不允许在 iOS 上使用其他浏览器，Safari 的演变就无关紧要了。 *(Chrome 等都是 Safari 皮肤。只有 Opera 使用自己的引擎，但它在服务器端呈现页面，不实现客户端 JavaScript。不到一半的 OSX 用户选择 Safari，但那些使用 iPhone 或 iPad 的用户别无选择。*

将此转化为实际数字，**超过 10%的网络用户被迫使用 Safari** 。抱怨 Safari 的不足或者劝说人们放弃他们的 iOS 设备是行不通的。开发人员有两种选择:

1.  避免 Safari 不支持的 API，坚持使用 2010 年的开发技术。
2.  使用[渐进式增强](https://www.sitepoint.com/javascript-dependency-backlash-myth-busting-progressive-enhancement/)并在浏览器支持特定 API 时提供更好的体验。如果许多网络应用在 Chrome 上更胜一筹，苹果就会采取行动。

我希望你选择第二种选择。不幸的是，当你和你的客户都有 iPhones 的时候，第一种选择就赢了。

## 2015 年 5 月至 6 月全球移动浏览器统计

上个月，手机使用量又增长了 1.5%，占所有网络活动的 36.27%。不要认为这是北半球夏季的异常现象；手机使用率很少下降。

热门移动浏览应用:

| 移动浏览器 | 五月 | 六月 | 变化 | 亲戚 |
| 铬 | 33.17% | 32.97% | -0.20% | -0.60% |
| 苹果手机 | 18.87% | 18.84% | -0.03% | -0.20% |
| 机器人 | 17.17% | 15.71% | -1.46% | -8.50% |
| UC 浏览器 | 14.53% | 14.91% | +0.38% | +2.60% |
| Opera Mini/Mobile | 9.92% | 11.93% | +2.01% | +20.30% |
| 不变的 | 2.15% | 2.06% | -0.09% | -4.20% |
| 其他人 | 4.19% | 3.58% | -0.61% | -14.60% |

上个月，Chrome 和 iPhone 版的 Safari 出现了反常的失误。随着用户迁移到预装 Chrome 的新版安卓系统，安卓浏览器下跌 1.5%。

最大赢家:UC 浏览器和 Opera。UC 浏览器是亚洲第二受欢迎的浏览器，拥有 25%的市场份额。在非洲，Opera 占有令人印象深刻的 50%份额。网络是全球性的；不要以为每个人都在使用最新的 iPhone 或 Galaxy 设备，只是因为你有一台。

最后，它告别了诺基亚浏览器，该浏览器的市场份额降至 0.75%，低于黑莓。我不能说我会怀念它——这款浏览器发布时很笨重。它适用于低端硬件，但 Opera 永远是更好的选择。
