路易斯:大家好，欢迎来到 SitePoint 播客的另一集。今天我很高兴请到了杰里米·基思。我认为杰里米不需要向这个节目的大多数听众做太多介绍。Jeremy 是英国的一名网页设计师，他是几本网页设计书籍的作者，最近的一本书是《网页设计师的 HTML 5 》,你好 Jeremy！

路易:你好吗？

路易:哦，这绝对是我的荣幸。我想凯文提到过他会在。净奖励。

杰里米:没错，是的。

**Louis:** 对，你当时表达了对上节目的兴趣，最近几周前，我们与 Myles 和 Max 讨论了你的一篇博客文章，你插话发表了评论，说嘿，有几件事我希望能够澄清，并在那里谈论，所以，是的，你很慷慨地提出来讨论这些问题。

杰里米:是的，非常感谢你让我参加这个节目。

路易:哦，当然。所以，促使我们来到这里的讨论主题是你不久前写的一篇名为[巨变](http://adactio.com/journal/4367/)的博客文章，我想是在讨论你对[响应式网页设计](http://www.alistapart.com/articles/responsive-web-design/)在未来网络中的作用的看法。那么，你想让我们开始吗？给我们一个纲要，告诉我们你写那篇文章时的立场和你来自哪里，以及你的感受。我想我们表达的方式可能没有完全抓住你的观点，所以我给你一个机会来写下来。

杰里米:嗯，这篇博文是关于未来的，也是关于现在的，我已经感觉到我们建立网站的方式正在发生转变，实际上我们多年来一直在建立特定设备的网站，只是设备一直是台式电脑、笔记本电脑，直到最近才开始为更新的设备、手机、平板电脑等建立特定设备的网站。但实际上，我们希望走向的是设备无关的 Web 开发，它只适应用户碰巧使用的任何设备。这也是我一直在反复强调的，我一直在反复强调设备不可知开发的想法，这确实是 Web 标准一直以来的目标，只是现在它终于开始实现了。事实上，这种情况在某种程度上让我想起了十年前的情况，当时有一种从表格到布局和字体标签的网络标准的转变，并且倾向于使用 CSS 来表现，结构和表现的分离。那时候，你可以在空气中感觉到这是一个令人兴奋的时代，人们花了很多时间来说服这是正确的事情，这需要一些相当高知名度的网站，如《连线》杂志的重新设计，ESPN 的重新设计，来说服人们这是值得做的，你有像禅宗花园这样的东西来改变人们的想法。现在情况也差不多了，我认为人们的思想正在发生转变，响应式网页设计是这种做事方式的先锋。但是，我想我刚才说的可能是我在博文中表达得不够清楚，但是我想有些人误解了我的意思，因为没关系，我们不需要太担心手机或平板电脑之类的新设备，我们需要做的只是利用我们现有的以桌面为中心的网站，重新编排内容，使其适合更小的屏幕，这不是我的意思，这不是我要说的。我要说的是，大量不同设备、大量不同屏幕尺寸、大量不同功能的激增意味着，我们多年来构建的以桌面为中心的网站将不再适用，解决方案不是为这些新设备构建一个单独的子域，对于这些设备的不同用户，解决方案是以一种与设备无关的方式从头开始重建网站， 我承认这是一个相当痛苦的变化，通常有政治原因，为什么这并不总是一个合理的选择，但我确实觉得不管你喜欢与否，这都会发生。 就像要么你现在就开始做，现在就开始改变以桌面为中心的 Web 开发方法，要么你将在接下来的 6 个月、12 个月、两年内落后于潮流，就像 10 年前一样，转向 CSS 并分离结构和表示；你可以在早期落后，也可以迎头赶上，但这是迟早的事。

**路易:**对。我想响应式网站设计的一个有趣的地方是，我认为你所说的大部分都是完全合理的，我们都可以支持它， 响应式网页设计是一种全新的技术，仍然存在一些问题，正如你所说的，人们已经做了或试图对移动问题做出回应，要么在子域上隔离事物，要么试图提出一种完全不同的布局，一些人已经有了一些关于最佳方式的预先形成的意见，这可能是为什么我们看到的摩擦比我们预期的多一点。

Jeremy: 是的，不要误会我的意思，我并不是说响应式设计是一切的答案，是 Web 开发的全部和终结，我同意它还处于早期阶段，有很多实验正在进行，这是令人兴奋的一点。我会将它与当时的情况进行比较，再一次，回到 Web 开发的早期，我将它与 Ajax 的情况进行比较，因为事情开始发生，而且是非常模糊的事情；大约在 2005 年，我们开始看到一些非常酷的应用程序出现，如 Gmail 和谷歌地图等，然后杰西·詹姆斯·加勒特出现了，他给了它一个标签，他称之为 Ajax，这是一种类似的情况，我们有一些事情发生了，伊森·马科特出现了，他说，好的，我们会把它称为响应性 Web 设计，我们马上有了一些东西来称呼它，这有助于具体化，人们可以支持这个标签，但它仍然是非常多的实验，尝试东西。我不想给人一种所有答案都在那里的印象，我们只需要实施它们，不，这在很大程度上仍是解决这些问题的一个阶段，只是我认为我们有这种方法的标签是好的，这种方法与过去两三年的默认方法不同，这种想法是，哦，我们只是为这种用户和那种用户创建一个单独的子域，从长远来看，这不会扩展。

**路易:**对。那么，你是否认为这永远不会是正确的方法，没有任何组织或出版物可以从专门为移动设备设计的网站中受益？然后，我认为有必要说，你的传统网站，任何网站都可以从响应技术中受益，因为你将能够适应更广泛的屏幕尺寸和设备。但是你的意思是说，一个专注于移动设备的网站，只有有限的几个选项，然后链接回主网站，是不合适的吗？

Jeremy: 不，当然，就像任何视情况而定的事情一样，就 Web 开发而言,“视情况而定”几乎是所有问题的答案。但是，我想我得到的是，到目前为止，默认的选择是你为移动设备创建一个单独的网站，然后只有在一些边缘情况下，你才会考虑让一个网站为每个人服务，你为每个人调整布局或设计，我对这个默认设置提出质疑。你是绝对正确的，确实存在这样的情况，应该为移动用户提供不同的 URL，可能是不同的内容子集，尽管给移动用户提供的内容较少，这对我来说确实很讨厌，但在这些情况下，更多的是关于上下文而不是内容。伊森谈到了这种情况，他举了一个活动网站的例子，他正在为 Happy Cog'aoke 建立网站，这是去年在西南偏南举办的卡拉 ok 活动，该网站是活动筹备阶段的主要网站，主要内容是投票和选歌，你非常需要坐在电脑前使用它。但是如果你在活动当天用移动设备访问网站，那么你可能会想知道活动在哪里，我怎么去，诸如此类的事情，所以在这种情况下，你会得到不同的东西是有道理的。但我认为这些都是例外，而不是默认情况，肯定有这样的情况，那绝对是你想做的，但也有危险，试图对人们的意图做出假设，试图从设备中假设上下文是危险的，这是我经常看到的情况。这种想法是，有人在小屏幕设备上访问，例如手机，然后他们可能会假设这些人不想访问我们的所有内容，他们只是想找到一个地址，或者他们只是想得到我们的电话号码，这样他们就可以打电话给我们，我认为这是一个危险的假设。这种假设在过去可能是正确的，当时手机坦率地说在处理网络内容方面不是很好，当时人们确实在移动设备上使用网络作为一种快速获取信息的方式，当他们匆忙时，当他们低头时，当他们在移动时，但是现在的数据表明，人们在各种情况下使用手机和其他移动设备，你知道， 坐在家里只是坐在沙发上，而不是去拿笔记本电脑或电脑，你可能会拿出手机来读一些博客文章，或者你可能会躺在床上看 iPad，这实际上是平板设备和移动设备的一个非常常见的使用案例。 因此，我认为仅仅从人们的设备上对他们的意图做出假设是不正确的，如果曾经是这样的话，做出这种假设是非常危险的，人们会非常沮丧，最终会出现这样的情况，人们会被自动重定向到一个移动网站，一个特定于移动设备的网站，这个网站有一个内容子集， 他们知道那里有更多的内容，因为他们当天早些时候在桌面设备上访问过同一个网站，他们试图找出他们如何获得这些内容，希望有查看完整网站的链接，对，这不可避免地要被包括在内。 所以，在某些情况下，你确实想为移动用户提供一个不同的网站，但我认为这种情况是少数，而不是大多数，我还认为试图根据人们的设备对他们的意图做出太多假设是危险的。

路易斯:是的，所以我认为你说的是，最终你真的要考虑将会使用你的网站的用户，而不是试图做出任何类型的概括；如果网站所有者拥有任何数据，或者作为设计师的你可以访问任何数据，或者如果你可以出去与用户交谈，了解他们想要什么，以及他们在使用网站时的环境，那么没有什么可以与之竞争。

杰里米:我不得不说，这也是双向的，就像我们不应该试图对一个碰巧用移动设备访问你网站的访问者的意图做出太多的假设，同样，仅仅因为有人用桌面浏览器访问并不意味着他们有足够的时间坐在那里等待页面加载，或者他们不着急，他们想快速获得信息，或者他们有足够的带宽。屏幕尺寸和带宽之间有一个可怕的融合，我们倾向于认为小屏幕尺寸意味着窄带宽，大屏幕尺寸意味着大带宽。虽然假设小屏幕可能意味着我们不能依赖这个人的快速连接可能是一个安全的赌注，但相反的情况并不成立，你不能因为某人有一个宽屏幕就认为他们有很多带宽。

路易斯:我想，在这场关于响应式网站设计的争论中，我最感兴趣的另一件事是，它揭示了一个事实，我想我们看到的许多传统桌面目标网站上有许多内容可能并不重要。人们提出关于移动网站的争论，说哦，我的餐馆需要一个移动网站，因为当人们忙碌时，他们只是想要我的地址、电话号码和菜单，人们总是想要你的地址、电话号码和菜单，这也应该是你桌面网站的核心信息，所有这些垃圾和人们往脸上塞食物的大照片实际上对任何人都没有帮助。

杰里米:正是。实际上，我今天在 Twitter 上收藏了某人的一条推文，他总结得很好，他说，“移动用户希望看到我们的菜单、工作时间和送货数量，桌面用户肯定希望看到这个一兆字节的 PNG，上面有人对着沙拉微笑。”

路易:(笑)没错，没错。我没见过。我当时一定也有类似的想法。

杰里米:是的，总结一下，对，实际上最近设备、屏幕尺寸和使用案例的激增带来的是，我们做传统网页设计的方式，如果你可以把这样一个年轻的行业称为传统的话，总是与现实脱节，对，我们在过去十年里做了很多假设，这些假设并没有真正基于任何好的数据或对真实用户的任何好的测试。我们一直在对屏幕尺寸、带宽、环境等所有这些实际上并不成立的东西做很多假设，我喜欢移动使用的这种爆炸是因为它真的凸显了我们的技术到目前为止在桌面上是多么过时。所以我实际上认为，当人们建立这些独立的移动网站时，他们通常对待移动网站的方式是在以正确的方式进行网页设计，他们在考虑什么是主要内容，当他们来到我们的企业时，他们想要什么。当然，一个显而易见的问题是，就像你说的，为什么我们认为在桌面网站上访问的人不会想要同样的东西，主要内容。所以，是的，这是一种移动设备的激增，设备的激增确实显示出我们的传统方法是多么的落伍。

路易斯:嗯，所以我现在想继续，谈谈作为一种技术的响应式设计，以及它目前所处的位置。这是另一件我们在之前的节目中没有完全展示的事情，当我和麦克斯和迈尔斯谈话的时候。那么，你认为它更像是一种哲学或最佳实践，还是仅仅是一套工具，你可以去使用，使你的网站尽可能好？

**Jeremy:** 当 Ethan 定义这个术语时，他说得相当清楚，他没有含糊其辞，他指的是三种具体的技术，即使用流畅的网格，拥有流畅的布局，拥有可调整大小或流畅的图像和其他媒体视频图像等等，然后使用媒体查询调整布局。现在人们真正关注的是媒体查询部分，有很多人混淆了响应式网站设计和媒体查询，然后他们互换使用这两个术语。实际上，媒体查询部分可能是响应式网站设计中最不有趣的部分，弄清楚如何做一个流畅的布局，如何处理图像和视频以及这种背景，这是真正困难的部分，媒体查询是相当容易的。实际上，即使你不使用媒体查询，而是使用类似于几年前 Cam Adams 的 JavaScript 的基于 JavaScript 的解决方案，根据屏幕大小提供不同的布局，我也会说这也是同样响应迅速的 Web 设计，媒体查询方面实际上只是它的一个脚注。但是人们已经非常专注于媒体查询部分，他们刚刚开始互换使用这个术语。所以我认为这个术语是 Ethan 定义的一套相当具体的技术，但你如何处理它可能会有很大的不同，因为你可以处理响应式 Web 设计的一种方法是从字面上理解一个现有的为桌面制作的桌面网站，它是以一种桌面特定的方式制作的，然后看你如何将内容重排以适合较小的屏幕， 你可能会取得一些成功，这取决于它的编码方式，就像我说的，如果一开始就用液体布局，你可能会有一些运气。 但是那种方法并不是我真正认为的响应性网页设计的最佳实践。我认为正确的方法是从你的基本内容和基本风格开始，比如颜色、排版等，然后在媒体查询中添加布局风格，或者你用来检测屏幕大小的任何其他技术。所以在某种程度上，这就像卢克·乌鲁布莱夫斯基的移动优先的想法，对，先考虑移动情况，然后再向上移动，但我认为专门考虑移动也是危险的，它更像是内容优先，你先考虑内容，然后再应用布局。现在，我认为这种方法比从桌面版本开始，并试图让它在较小的屏幕上很好地流动要好得多。如果你从内容开始，应用颜色样式和字体样式，然后根据屏幕宽度，在此基础上应用布局样式，这就是渐进式增强，对，我的意思是这就是我们所做的，这是最佳实践。正如我所说的，我认为过去十年我们在网络上做的很多事情显然是错误的，我们建立这些桌面特定网站的方式我认为我们已经用错误的方式做了很长时间，但渐进增强作为一种方法和技术是非常有弹性的，我一次又一次地看到这是一个非常好的解决方案，不是解决方案，而是应对许多新情况的好方法。再一次，把它与 Ajax 联系起来，当 Ajax 出现时，有很多实验，很多人把渐进增强扔出窗外，说哦，这是一种全新的做事方式，我们不再需要考虑那些东西了。但是事实证明，当您将它应用到 Ajax 时，渐进的增强会为功能更强大的浏览器带来最好的体验，但仍然确保所有东西都适用于旧浏览器。

Louis: 正如我们最近看到的，我想 Gawker Media 的重新设计是最大的例子，即使使用 Ajax，人们仍然没有完全理解这个想法，这已经六年了，所以我想响应式 Web 设计可能需要一段时间才能实现。
