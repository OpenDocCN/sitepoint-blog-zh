# 重置还是不重置——这是 CSS 的问题

> 原文:[https://www.sitepoint.com/css-resets-useful-or-useless/](https://www.sitepoint.com/css-resets-useful-or-useless/)

每个 web 浏览器都使用一个基本样式表。当你不提供定制的 CSS 时，它可以确保 HTML 被很好的渲染。你知道这种东西:蓝色代表未访问的链接，紫色代表已访问的链接，粗体代表`strong`标签，大字体代表`h1`标题等等。不幸的是，供应商喜欢让我们的生活变得复杂，每个浏览器使用不同的基础。那么，如何确保自定义 CSS 不受特定设备上实现的默认样式的影响呢？

输入 CSS 重置。这些方法去除了大部分样式，因此元素在不同的浏览器中呈现一致。通常，所有字体和行高都恢复为 100%，边距和填充设置为 0px，边框被移除，有序/无序列表变得无序。

Eric Meyer 的 CSS Reset 是第一个，也是最广为人知和使用最多的。来自 [HTML5 doctor](http://html5doctor.com/) 的 [HTML5 重置样式表](http://html5doctor.com/html-5-reset-stylesheet/)也可以帮助设计新的元素。还有其他几个—[CSSReset.com](http://www.cssreset.com/)提供了大量的选择和文档。

## 你使用 CSS 重置吗？你应该吗？

我随机查看了 30 个网站，大多数都使用了 CSS 重置。优势显而易见:

1.  他们提供了一个空白的画布；任何应用的风格(几乎)肯定是你自己的。
2.  开发可能更符合逻辑:您正在添加样式，而不是删除或修改它们。
3.  浏览器兼容性问题可以被最小化。

尽管有这些优势，我不使用 CSS 重置。实际上，这并不完全正确—我经常使用基本边距和填充重置，因为默认值很少有用:

```
 * { padding: 0; margin: 0; } 
```

这影响到每一个标签，一些开发者会告诉你这会导致混乱或 CSS 处理变慢。我还没有遇到过这样的情况。

我对 CSS 重置的主要问题是:

**附加页面重量**
大多数 CSS 重置会增加大约 2Kb 的代码(未压缩)。这听起来可能不多，但当我的大多数 CSS 文件很少超过 10Kb 时，这是一个很大的开销。

CSS 重置需要你重新设计所有元素的样式，这会增加文件的重量。不过有时候，我对浏览器的默认设置很满意。字体粗细、行高、链接轮廓、项目符号盘和其他样式通常都没问题。如果不是，我会相应地更改它们。

不同的浏览器呈现页面的方式不同
没有两种浏览器是完全相同的，然而有些人希望在所有设备上都能呈现像素级的完美效果。CSS 重置经常给人一种这个目标是可以实现的印象。不是的。

就我个人而言，我不介意 Firefox 中的标题比 IE 大 2 倍，或者 Opera 的“粗体”概念比 Chrome 略重。CSS 重置永远不会为您解决这些问题。

他们并没有否定浏览器测试的必要性
我不相信 CSS 重置有助于浏览器的一致性。您必须在尽可能多的设备上测试您的站点或应用程序，以便解决由代码或浏览器引起的错误和缺陷。

他们不适合我的工作流程
我不是 CSS 框架的粉丝。我很少遇到将通用 CSS 方法应用于正在构建的网站的情况。我可能会从其他地方复制一些代码片段，但一般来说，我更喜欢从一个空文件开始。它仍然是我的代码，当出现错误时，我知道该去哪里。

虽然它并不复杂，CSS 重置是一个框架，可以引入意想不到的结果。

**他们不节省时间**
CSS 重置曾经为你节省了几个小时的开发时间吗？

在我被一群愤怒的 CSS 重置粉丝攻击之前，我不是说你不应该使用它们。他们可能为你工作，这很好。我怀疑它们对那些刚接触 web 开发的人也很有用。然而，您添加 CSS 重置是因为它真正有用还是已经成为一种开发习惯？

作为一个测试，试着从你的网站上删除重置样式，看看它是否有显著的不同。

你使用 CSS 重置吗？你最近采用或放弃了这种做法吗？他们有帮助吗？
