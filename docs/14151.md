# 适应的时间？

> 原文:[https://www.sitepoint.com/time-to-adapt/](https://www.sitepoint.com/time-to-adapt/)

上周我在西雅图参加了吉尔·惠伦的“高排名研讨会”，那是一次奇妙的经历。90 年代，我在西雅图住了七年，从离华盛顿大学很近的一个有利位置体验了互联网的早期。

UW 做的一件很酷的事情是在他们自己的“UWTV”有线频道上播放讲座和演讲，这是我自 1997 年以来从未见过的，但却是我进入酒店房间后在电视上寻找的第一件事。是的，它还在继续，幸运的是，我能够听到 UW 的丹尼尔·威尔德关于[自适应用户界面](http://www.cs.washington.edu/ai/supple/)的[演讲的大部分内容。](http://www.cs.washington.edu/homes/weld/iui-2003.ppt)

好吧，我承认这有点怪，你可能很难理解这和搜索引擎营销的联系。

Weld 谈到的一件非常有趣的事情是，其他人使用关系马尔可夫模型来预测用户在网站上的行为。这并不是全新的(引用可以追溯到 1999 年，我相信)，但这是我第一次看到有人把这些点联系起来。

这项工作让我感兴趣的是，在过去的一年里，我花了很多时间来分析三种不同类型的 PPC 活动登陆页面的性能:
【list = 1】
[*]网站的主页或类别页面。
[*]特定搜索词的专用登陆页面。
[*]一个“提问”页面，为用户提供多种选择，试图更好地细分受众。如果你想要 *x* ，点击这里，如果你想要 *y* ，去那里等等。
[/list]

Weld 展示了一个非常有趣的图表，显示了一个类似的问题，即“智能”用户界面的设计者必须预测是否最好提供应用程序的默认行为，采取一些自动化的行动，或者提示用户做出选择。

每一个可能的决策的有效性，就像 PPC 活动登陆页面一样，取决于我们对用户意图的确定程度。当我们对用户想要什么有高度的信心时，应用程序可以自动采取行动，或者网站可以提供专门的登录页面。当不确定性很小时，最好的选择是交付默认行为，或者可能是站点的主页。

在这两者之间，我们对用户想要什么有一点了解，应用程序设计中最好的选择是提示用户做出决定。类似地，我发现，我们对许多搜索词的最佳搜索结果来自于使用一个“提问”页面，让访问者表明自己的身份。

我从 Weld 的演讲中得到的是一种可能的方法，可以帮助我们更好地预测特定页面的访问者想要什么，基于他们去过的地方。

有点跑题，但我希望你们中的一些人会觉得有趣。
