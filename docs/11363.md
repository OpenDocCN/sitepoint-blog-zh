Kevin: 这篇来自 Newism 的澳大利亚网页设计的[博客文章](http://newism.com.au/blog/post/102/a-real-world-case-study-on-html-5/)记录了他们第一次实际使用 HTML5 的经历。第一次。有很多人在谈论 HTML5，并就规范的小细节进行争论，但在实际使用 HTML5 时有一个很大的绊脚石，那就是 Internet Explorer 不像其他浏览器那样处理未知元素。我们期望浏览器的工作方式是，如果他们不理解 HTML 标签，他们会有效地忽略它。默认情况下，他们不会对它应用任何格式，但如果你想对它不识别的标签应用你自己的 CSS 样式，你应该有这个自由。

但不管出于什么原因，Internet Explorer 不会以这种方式处理未知标签。如果你有一个开始标签和一个结束标签，它会把开始标签当作未知元素的开始*和结束*，然后忽略你的结束标签。因此，为了让 HTML5 标签如`<header>`、`<footer>`和`<section>`等能够在 Internet Explorer 中工作，你需要在每个 HTML5 文档的顶部添加一些 JavaScript，这迫使 Internet Explorer 不得不这么做。它利用了一些古怪的行为，实际上可以欺骗 Internet Explorer 正确地识别和处理这些元素。因此，很多人一了解到，为了让网站在 Internet Explorer 中正常显示，他们必须在所有客户的浏览器上启用 JavaScript，就避免使用 HTML5。这似乎是一个足够大的理由不使用它，所以大多数人回避它。
