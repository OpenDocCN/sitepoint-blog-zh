# 8 权威网络字体堆栈文章

> 原文：<https://www.sitepoint.com/eight-definitive-font-stacks-3/>

##### 如何*相似*相似？

克莱恩注意到，根据字体的美感(类型和笔画的角度，比例，“看起来像*乔治亚*，“看起来像 *Arial* ”)对字体进行分组，在某些情况下可能是没有用的。即使是相似的字体也常常有明显的不同；例如，他注意到 [*Garamond* 与 *ITC Garamond* 家族是不同的字体家族，甚至是由不同的设计师](http://klepas.org/2008/10/25/whose-garamond-is-it-anyway/)创造的。

此外，还需要考虑重量、宽度和高度等基本方面，例如从“*巴斯克维尔*到*佐治亚*的秋千，前者的外观更轻、更小。”你可能想好好看看那些被列为彼此“相似”的字体，然后做出自己的选择。

在一个相关的项目中，来自 [Inspiration Bit](http://www.inspirationbit.com/) 的 Vivien 指出 *Garamond* 堆栈太大，设计师无法穿透 *Garamond* 字体，并建议将堆栈一分为二。我可以看到，也许一个 *Palatino* 驱动的栈和一个 *Garamond* 驱动的栈，有一些必要的重叠，可能是一个值得探索的可接受的替代方案。

##### 案例研究:Helvetica/Arial 无衬线字体堆栈

Helvetica/阿里亚堆栈是最大的堆栈之一，似乎也是最杂乱的堆栈之一。让我们把它拆开，看看它为什么是这样。

*   我们从*Frutiger/Frutiger Linotype*开始——我知道这是一个个人选择，虽然它为平面设计师所熟知，但它不会出现在许多普通用户的机器上。( *Frutiger Linotype* 是微软版本的标准 *Frutiger* 字体。)
*   由于大多数用户看不到显示的字体，我们转向 Univers，一种用于老式电脑的 Mac 字体。
*   大约一半的 Windows 用户使用 Vista 字体 Calibri T1，所以我接下来给出一个。
*   然后，我们有两种版本的 *Gill Sans* ，一种更古老且不太为人所知的 Mac 字体，以及两种版本的 *Myriad* ，一种被 Mac 用来取代旧的 warhorse 的 Adobe 字体 *Garamond* 。(你知道*万千亲*字体；它是用于[旧款 iPod 标志](http://forums.macrumors.com/showthread.php?t=266825)的。)
*   接下来，我们有许多类似的 Linux 字体，它们依赖于 Linux 用户的操作系统。
*   然后我们来看一个流行的 Windows 字体选择，Tahoma T1，也可以在大多数 Mac 电脑上使用，还有一个流行的 Mac 字体，T2 Geneva T3。我们以几乎通用的【Helveticas 纽伊/Helvetica 结束(主要是 Mac，有一些仿 Helvetica 的 Linux 版本)。
*   最后，我们将 Arial(Mac、Windows 和大量 Linux 用户)列为最后的退路。我们以通用的*无衬线*术语结束。

就我们网站的典型访问者而言，大多数 Mac 用户将在 Tahoma，Geneva 或 Helvetica/Helvetica Neue 看到显示，而大多数 Windows 用户将在 *Tahoma* 或 *Calibri* 看到显示。 *Arial* 是那些字体柜异常光秃秃的人的最后选择。

请注意，堆栈的优点是，在进入主要是通用的回退之前，几乎涵盖了所有选项: *Tahoma，Verdana，Helvetica，*和 *Arial* 。那些分布不太广泛的字体将使网站以这些字体显示。因此，在字体列表的明显混乱中,*有了一些有序的表象。*

克莱恩表示，这一系列字体结合了人文主义者的字体——T2 的 Frutiger、卡利布里的 Calibri、吉尔桑斯的 T3、T4 的 Tahoma 的 T5 和现实主义者的 T7，比如 Helvetica。他会把两者分开，但对你来说，决定什么最适合你的目的是很重要的；结合人文主义和现实主义/古典字体可能是合适的。你可以在维基百科[上开始钻研这个主题。](http://en.wikipedia.org/wiki/VOX-ATypI_classification)

##### 不要局限于网页安全字体！

许多设计师选择坚持使用通常被称为网页安全字体的字体，这种字体几乎可以在每台机器上找到。不可否认，这是一个完全合理的决定，有明显的好处，特别是简单:选择一两种网页安全字体，继续下一个设计决定。

不过，我想提出两个反驳。

1.  把自己限制在这种相对缺乏活力的字体上就是忽视了成千上万种可爱的替代字体；通过选择深思熟虑的组合，你可以确保 90%或更多的访问者至少有一种你在设计中使用的备选字体。
2.  “几乎每台机器”指的是 Windows 和 Mac，*不是* Linux。许多 Linux 用户有一些或所有这些字体，通常来自[微软 TrueType 核心字体](http://corefonts.sourceforge.net/)包，但是许多人没有它们。事实上，很多 Linux 用户更喜欢自己的字体；毕竟，使用 Linux 的部分目的是为了脱离微软。

克莱恩认为，随着你的堆栈和设计中的字体越来越多，作为设计基础的劣质字体被使用的可能性也越来越大:“有成千上万种字体……(但)只有少数字体是好的。”一个好的字体的元素包括强烈的美感、设计良好的提示表、良好的字距调整、对不同语言的支持、强大的 Unicode 支持和真正的斜体，而不是蹩脚的计算机生成的斜体。“随着可供选择的字体越来越多，选择劣质字体的几率会大大增加。

事实上，这几乎都归结于整体的排版。“他是正确的:更多的选择会产生更多的问题。还有那些网页安全字体？分别是: *Arial，Arial Black，Comic Sans MS，Courier New，Georgia，Impact，Times New Roman，Trebuchet MS，*和 *Verdana* 。

当你剔除那些你可能会拒绝的主流字体选择( *Arial Black、Comic Sans、Courier New、*和 *Impact* )时，你有五个选择——数一数——*五个*。我，我喜欢更大的选择。

##### 摘要

字体堆栈是实现更丰富、更独特的 web 排版的一种有用、实用的方式，而不是仅仅坚持 web 安全字体(并且不需要等待 CSS3 web 字体从浏览器供应商和字体制造商那里获得广泛支持)。然而，一个可靠的字体堆栈必须考虑许多因素，而不仅仅是安装了一种特定字体的用户数量。幸运的是，在本文中，我已经为您做了很多艰苦的工作。

有了这里列出的字体堆栈，您网站的内容可以被更多的用户以比您之前想象的更有趣的字体查看，而不会损害您设计的完整性。
