# 当心 CouchDB

> 原文:[https://www.sitepoint.com/watch-out-for-couchdb/](https://www.sitepoint.com/watch-out-for-couchdb/)

自从数据库项目开始以来，我一直在关注这个项目。这是程序员[达米恩·卡茨](http://damienkatz.net/)的得意项目。今天，我很兴奋地阅读了一个[更新](http://jan.prima.de/~jan/plok/archives/89-CouchDb-updates.html)，它宣布可以通过 RESTful [JSON](http://json.org/) API 访问，CouchDB 的查询语言将是 JavaScript。多酷啊。！

CouchDB 不是一个关系数据库，而是一个分布式文档数据库。不是将一行列值插入到表中，而是将一个带有任意数量的命名字段和值(现在表示为 JSON 对象)的文档保存到数据库中，它存在于一种可寻址的池中。可以创建、检索、更新和删除文档，而不必担心模式设计，因为没有模式设计。

如果你需要一个小的结构，你可以创建视图。视图是一种动态结构，其作用类似于搜索查询，提供与查询匹配的虚拟文档表。以前用专有语言表达的查询现在是一个 JavaScript 函数，用于确定视图中包含哪些文档。因为视图是完全虚拟的，所以您可以拥有任意多的视图，并且可以在任何时候添加或删除它们，而不会影响您的任何数据。视图被编入索引并定期更新，以跟踪数据库中文档的状态。

CouchDB 还有其他一些令人印象深刻的属性；它完全符合 ACID，具有内置的安全模型、双向增量复制和冲突解决。

但是它有什么用呢？文档很快指出这不是关系数据库的替代品，但它非常适合文档风格的应用程序，如博客、文档管理、错误跟踪、论坛等等。CouchDB wiki 的说:

> 只需很少的数据库工作，就可以构建一个具有粒度安全性和完整修订历史的分布式文档管理应用程序。

CouchDB 仍然只有 Alpha 版本，但是已经有可用的演示和用 PHP、Ruby 和其他语言编写的客户端库。您也可以下载 GPL 许可下的源代码。

我认为 CouchDB 将是值得关注的一个！
