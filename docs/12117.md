# 甲骨文和 MySQL:结盟还是死亡？

> 原文:[https://www.sitepoint.com/oracle-sun-mysql/](https://www.sitepoint.com/oracle-sun-mysql/)

令业界惊讶的是，在与 IBM 的谈判破裂后，甲骨文以 74 亿美元收购了太阳微系统公司。 [Sun 的未来似乎不确定](https://www.sitepoint.com/where-next-for-sun-microsystems/)，几乎没有人认为甲骨文会耐心等待(尽管这确实解释了为什么 Sun 准备放弃 IBM 的报价)。

甲骨文想要 Sun 有几个很好的理由…

*   Sun 的硬件和备份解决方案可能有利于甲骨文的软件
*   Java 编程语言
*   [VirtualBox 虚拟机软件](https://www.sitepoint.com/virtualbox-review/)
*   OpenOffice
*   MySQL——世界上最受欢迎的开源数据库。

虽然 MySQL 和 Oracle 不是直接的竞争对手，但那些更愤世嫉俗的人会认为 Oracle 会消灭竞争对手的数据库。这可能很困难，因为 MySQL 是开源的，任何人都可以为该应用做出贡献。然而，甲骨文可能会使该项目资源匮乏。

也有可能甲骨文不想要 MySQL，也许是因为反竞争法？他们可以将 MySQL“出售”给另一家公司 IBM 还会感兴趣吗？

不过，我怀疑甲骨文会更倾向于保留 MySQL 他们将控制更大的数据库市场份额，并能利用现有的用户群。它们可以提供进一步的投资，并创建增强两个系统之间关系的工具，例如，为那些需要更强大的数据库设施的用户提供一个简单的升级途径。

不幸的是，由于公司的结构和财务问题，许多 MySQL 的领导者和开发人员都离开了 Sun。该数据库的创始人和原始开发者 Michael Widenius 已经宣布，他将雇用所有核心 MySQL 人员，并分叉该项目，以创建一个独立和真正的开源实体。他准备与 Oracle 密切合作，确保 MySQL 在客户、开发人员和用户的信任和支持下以开放的方式积极开发。

无论发生什么，MySQL 的未来似乎是确定无疑的。然而，这个项目的方向是不确定的，谁知道拉里·埃里森在计划什么？
