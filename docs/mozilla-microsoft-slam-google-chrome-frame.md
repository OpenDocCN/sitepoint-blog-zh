# Mozilla 加入微软抨击谷歌 Chrome 框架

> 原文：<https://www.sitepoint.com/mozilla-microsoft-slam-google-chrome-frame/>

这是一个惊喜。Mozilla 猛烈抨击了谷歌插件 [Chrome Frame](https://www.sitepoint.com/google-chrome-frame-fixes-ie/) ，该插件*通过在 IE 界面中提供 Chrome 浏览器引擎来修复*IE 浏览器。谷歌认为创建一个 IE 插件比让他们的新 HTML5 应用向后兼容 IE6、IE7 和 IE8 更容易。

Mozilla 基金会主席、前 Mozilla 首席执行官米切尔·贝克质疑插件的决定:

> 镀铬边框整体效果不理想。
> 
> 我预测积极的结果不会持久，而且——就其被采用的程度而言——Chrome Frame 将导致我们大多数人(包括 Web 开发人员)的日益分裂和失控。
> 
> 但 Chrome Frame 最大的问题是，它把控制权交给了网站，而不是冲浪的人。这只会让用户感到困惑。一旦你的浏览器被分割成多个渲染引擎，跨网站管理信息就变得非常困难。有些信息可以从你使用的浏览器中管理，有些信息可以从 Chrome Frame 中管理。这破坏了浏览器帮助人们管理体验的最重要的方式之一。
> 
> 想象一下，一些网站有浏览器内置谷歌浏览器，脸书互联网站有浏览器内置脸书浏览器，iTunes 有苹果版本，你的移动网站有移动运营商版本。
> 
> 每个浏览器中的浏览器变体都有自己的特性集、自己的怪癖和自己的安全问题。结果是一种浏览器混乱，网络更不可知，更不可理解，当然也更不可管理。

Mozilla 的工程副总裁 Mike Shaver 补充道:

> 用户对 Web 安全模型和浏览器行为的理解，由于将软件的选择委托给他们访问的单个站点的开发者而受到严重阻碍。这是我们在 Flash、Silverlight 和 Java 等其他堆栈插件中反复看到的问题，我认为我们不需要在 HTML 5 的旗帜下再次看到这个问题。
> 
> 如果想使用 Chrome Frame snippet 的开发者简单地告诉用户他们的网站在 Chrome 中工作得更好，并指导他们如何安装它，这对网络来说会更好。

业界采用 web 标准已经花了足够长的时间，所以 Mozilla 对碎片化和网站采用自己的浏览器插件的担忧是有根据的。在一个理想的世界里，web 开发者永远不需要插件，HTML5 在一定程度上实现了这个梦想。

然而，Chrome Frame 的引入并不一定会为特定网站插件打开闸门。谷歌的目标是让 IE 用户在不愿意或不能使用替代浏览器的情况下访问现代网络应用。该插件与 IE 集成得很好:收藏夹、历史记录和 cookies 都是共享的，因此很少有用户会意识到他们已经切换到 Chrome 的视图。

我希望我们永远不会遇到网站可以指定应该使用哪个浏览器引擎的情况([微软调查过的一种可能性](https://www.sitepoint.com/ie8-standards-mode-opt-in/))。许多人会说这正是 Chrome Frame 所做的，但谷歌的被动实现更像是标准和怪癖模式之间的 DOCTYPE 切换。*(实际上，谷歌，这不是一个坏主意——如果用户访问 HTML5 页面，浏览器可以自动切换到 Chrome)。*

我们还应该注意到，Mozilla 是一个开源组织，可以占据道德制高点。他们通过生产比微软更好的浏览器来鼓励 Firefox 的采用。谷歌是一家商业公司，主要创建网络应用程序；他们的雄心和未来的利润可能会受到 IE 和它的稳步发展的阻碍。

Mozilla 的声明让该公司站在了微软的一边(微软也猛烈抨击了 Chrome Frame，但没有那么雄辩！)Mozilla 提出了一些有趣的观点，但他们的担忧是假设性的，有点反应过度。我们都希望企业和用户升级到最新的浏览器，但这并没有以我们希望的速度发生。Chrome Frame 是一个聪明的短期解决方案，可以帮助 HTML5 的采用。

你怎么想呢?Mozilla 的批评有根据吗？Chrome 框架会破坏网页吗？
