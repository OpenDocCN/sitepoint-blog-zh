# 你的网站需要数据库吗？

> 原文：<https://www.sitepoint.com/does-site-need-database/>

根据普遍的传说，数据库要么是地球的灾难，要么是人类的救星。他们要么把我们从日常生活的琐碎琐事中解放出来，要么侵犯我们的隐私，把我们个人生活的每一个细节暴露给任何有东西可卖的跨国公司。

然而，网站管理员需要对数据库有第三种看法——一种介于这一范围中间的看法。对于网站管理员来说，数据库是不可或缺的——它们可以极大地有利于网站的设计和维护，但是需要详细的技术知识和额外的资源。

这篇文章将让你清楚地了解一个实现良好的数据库能给你的站点带来的好处——所以一定要考虑与你自己的站点相关的每个问题。

##### 你的网站内容经常改变吗？

如果站点的内容每天都在发生巨大的变化，数据库可以简化站点的管理。一个很好的例子是[悉尼先驱晨报网站](http://www.smh.com.au)(《先驱报》是一家知名的澳大利亚报纸)。《悉尼先驱晨报》每天都刊登许多不同作者的文章。该网站本身有几个不同的索引页面，涉及体育、国家新闻、文化等等。

在这种情况下，数据库用于存储文章、图像、作者信息和页面布局。一旦一条信息存储在数据库中，网站管理员就可以使用 JSP 或 PHP 等服务器端脚本语言来自动生成用从数据库中检索的内容编译的网页。

像体育页面这样的页面可以每天自动生成，这为网站的站长节省了无数个小时的布局时间。此外，按作者、主题或日期搜索网站内容很简单——这也是由数据库软件处理的。通常这些任务必须手工完成，但是数据库驱动系统的实现节省了网站管理员每天的工作时间。

内容不断变化的小型企业的一个很好的例子是当地的旅行社。旅行社不断更新他们的价格信息、特价商品和可用目的地，对于一名员工来说，花大部分时间手动更新网页不太方便或高效。因此，在这种情况下，使用数据库来自动化某些过程可能会很方便。

##### 你的企业广泛使用电子商务吗？

像许多其他基于网络的公司一样，知名零售商亚马逊的生存依赖于一个数据库驱动的网站。他们出售的每个产品的详细信息，包括价格、描述、评论以及类似或补充产品，都存储在数据库中。

当客户在网站上搜索产品时，服务器端软件立即采取行动，参考数据库，并提取与该产品具体相关的信息。然后，这些信息被插入到预先设计的页面布局中，并显示给客户。

这种模式有明显的好处。它很快，确保用户请求的信息有一个好的、统一的表示，并最小化 Amazon 团队的数据维护。

然而，使用数据库进行电子商务的关键问题是时间。开发一个正常工作的电子商务系统可能需要几个月的时间，而且当您或您的员工定期更新产品线时，几乎每天都需要对内容进行维护。当然，一旦系统实现，从数据库中添加和删除产品应该相对容易，并且让您相应的网页自动更新。

另一个优化了电子商务好处的公司是 [Everything Linux](http://www.everythinglinux.com.au) ，它出售基于 Linux 操作系统的产品。

##### 你的网站使用用户驱动的内容吗？

在过去的几年里，互联网上由用户贡献内容驱动的网站数量有所增加。许多网站现在不同程度地利用用户内容。

例如，许多网站提供了一些机制，访问者可以通过这些机制参与到与网站目的相关的话题的讨论中。这种反馈可以采取多种形式——用户论坛或讨论板、反馈表以及“评论”或“评论”邀请。

例如，一个受欢迎的新闻网站 [ZDnet](http://www.zdnet.com) ，使用一个数据库来存储读者对文章的评论。这使得读者不仅可以看到文章，还可以看到其他读过文章的人的有见地的评论，他们觉得自己有东西可以贡献。

这种类型的互动可以改善访客体验，吸引用户参与网站，产生承诺，并允许网站所有者免费收集关于他们的产品、行业和用户偏好的广泛反馈。

要使用用户驱动的内容，您需要某种数据库来自动接收和发布您站点上的用户评论。您的数据库将从用户那里收集数据，您的 Web 服务器在生成网页时将引用这些数据。

你应该考虑用户反馈带来的许多好处。它:

*   让你的网站更有活力，
*   让你知道读者对你的内容的看法，
*   对你的网站产生忠诚度，并且
*   随着用户内容的增加，鼓励读者经常回来。

##### 你有资源来支持数据库驱动的网站吗？

数据库是一种专门的应用，需要一定的技能。如果你正在寻找一个数据库驱动的网站，你需要考虑你是否有必要的技能来实现和维护这样一个设置。甚至有必要雇用一名全职的数据库管理员。

此外，数据库驱动的网站通常需要自己的服务器，这可能最终会花费你相当多的钱来购买和托管在网络上，而不是简单的网站，往往会托管在其他人的服务器上。

因此，确保您在网站中构建的功能与您可用于管理和维护网站的时间和财务资源相匹配是非常重要的。你可能需要认真考虑你的网站的目的。问问你自己:

*   我的组织能否负担得起支持数据库所需的基础架构？
*   数据库开发需要多长时间？
*   我们内部有必要的技能吗？
*   谁将维护数据库？

……简而言之，你买得起吗？当你考虑实现一个数据库驱动的站点时，这是你必须考虑的最重要的事情。这些网站的流动性比一般的网站需要更多的工作、资金和技能。你能负担得起投入这些资源吗？

然而，如果你能对前三个问题做出肯定的回答，并且你有足够的资源来维护一个数据库驱动的网站，那么就去做吧！调查清楚你需要什么来建立网站，并确保让你的读者意识到你所做的额外努力。你最终肯定会得到一个动态的网站，一个有潜力吸引更多读者、客户和公众关注的网站！
