# 网页设计中的网格系统介绍

> 原文:[https://www . site point . com/an-introduction-to-grids-in-web-design/](https://www.sitepoint.com/an-introduction-to-grids-in-web-design/)

在网页设计中，网格系统是一种无形的结构，它将网页中的所有元素收集在一起。在本文中，我将介绍网格系统，解释它们是什么，它们的目的，以及它们背后的一些理论。

维基百科将网格系统解释为—

> 由一系列用于结构化内容的相交直线(垂直、水平和倾斜)或曲线组成的结构(通常是二维的)。网格作为一个骨架或框架，设计者可以在其上组织图形元素(图像、字形、段落等)。)以一种理性的、易于吸收的方式。

## 网格系统的本质

网格系统从来都是不可见的，但是通过网页中元素的位置可以看出网格“规则”的痕迹。网格系统还规定了设计元素的大小，如栏文本的宽度、元素的重复放置、图像周围的填充、单词间距、行高等。网格的主要目标是在一个设计中创建一个统一的连接，这反过来使网页内容更流畅，产生一个可读性更好、更有趣的网页设计。

## 网格系统中的统一

Alex White 的[平面设计元素](http://www.amazon.com/Elements-Graphic-Design-Space-Architecture/dp/1581152507/)通过网格系统解释了 unity 的使用:

> 设计的统一存在于所有元素都一致的地方。元素被做得看起来像是属于一起的，而不是看起来像是随机放置的……因此，没有统一性，设计就会变得混乱和不可读。但是没有变化，设计就会变得惰性、无生气、无趣。必须在两者之间找到平衡。

然而，只有在设计过程的初始阶段使用网格，网格系统对特定设计的好处才会生效。试图在现有设计中实现网格将不会创建相同的流畅布局或统一的内容。这个问题在约瑟夫·穆勒-布罗克曼的[网格系统是图形设计](http://www.amazon.com/Systems-Graphic-Systeme-Visuele-Gestaltung/dp/3721201450)中讨论过。他写道:

> 视觉设计中合适的网格使得:a)用视觉传达的手段客观地构建论点；b)系统地、有逻辑地构建文本和说明性材料；c)按照自己的节奏，以紧凑的方式组织文字和插图；d)将视觉材料放在一起，使其易于理解，并具有高度的张力。

## 超越网页设计的网格系统

网格不仅仅局限于网页和图形设计。几乎每一个实施任何形式设计的职业都有一个网格系统，专业人员用它来保证积极的元素定位。在所有现代设计实践中使用网格已经变得几乎在专业上至关重要。

## 有哪些网格系统可用？

供网页设计者使用的网格系统在互联网上随处可见。网格之王被认为是“960 网格”。960 栅格具有以下结构:

*   总宽度 960 像素。
*   最多 12 列，每列 60px 宽。
*   每列有 10px 左右边距。反过来，这使得边缘有 20px 的装订线空间。
*   总内容面积为 940 像素。

960 grid 受设计师欢迎的一个主要原因是它的灵活性。设计人员可以使用各种列，最多 12 列。最常见的例子有:9 x 3、3 x 3 x 3、4 x 4 x 4 x 4 和 10 x 2。该网格还被广泛用于各种网站，如 [Assistly](http://www.assistly.com/) 、 [Drupal](http://drupal.org/) 、 [51bits](http://51bits.com/) 和 [Sony Music](http://www.sonymusic.com/) 。要查看这些站点的网格系统，请访问官方的 [960 网格主页](http://960.gs/)。

960 网格一直与“蓝图网格”有着非常紧密的联系，但它们确实有很多不同之处。首先，Blueprint 有一个深入的排版设置，而 960 grid 系统设置只有最低限度的排版，作为指导，而不是出厂设置。我更喜欢这种方法，因为这两种网格系统都是几年前发布的，而 web 排版前沿已经发生了迅速的变化。因此，960 网格系统允许在设计中使用现代字体时有更大的灵活性。

960 网格框架的另一个显著特征是在设置中缺少引用。纳森·史密斯(960 网格的创造者)说:

> 这些是我很少使用的东西，我认为这更多的是面向设计和上下文的内容，而不是页面布局和原型。

我不完全确定拉/推引用的设计，坦率地说，我对它们出现在网站上有点反感，因为这让我觉得有人过度补偿了。此外，如果拉/推报价设计是在没有太多思考或指导的情况下产生的，它真的会拖垮一个网站的整体设计。然而，我知道这是现代网站中一个流行的设计元素，所以我很感激 Blueprint 为这个功能提供了一个模板。

这意味着，如果我为 960 vs Blueprint 创建一个评分系统，它目前是 1-1，并且下降到我的最后一个关键差异-垂直节奏。**垂直节奏**是网站内容中句子和段落之间的一致间距，或者说是行高。纳森·史密斯立即再次声明，虽然他觉得垂直节奏是网页设计中的一个重要因素，但他认为“它是脆弱的”，很容易被内容编辑在文本中放置图像所破坏。

我的结论是:我选择纳森·史密斯纯粹是因为，作为一名开发者，我知道有多少 CMS 允许用户将任意大小的图片放在文本中的任意位置。他们不考虑复杂的垂直节奏。有很多方法可以解决这个问题，但是我觉得要让它真正发挥作用还得视情况而定。如果你有一个项目，它会受到尊重，很好，但对于大多数情况下，我认为 960 有一个更现实的态度。

我的观点是 960 网格框架是更好的选择，但是老实说，把它们都试一试，看看哪一个最适合你。
