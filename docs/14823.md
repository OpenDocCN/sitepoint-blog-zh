# 易访问且有吸引力的网站

> 原文:[https://www.sitepoint.com/attractive-websites/](https://www.sitepoint.com/attractive-websites/)

网页可访问性并不是世界上最性感的话题——让我们面对现实吧！

因此，推销这个概念从来都不是那么容易的。当然，你可以喋喋不休地谈论所有的“商业利益”(潜在的受众增加、降低带宽成本、良好的公关)，但你真正需要做的是证明在不损害设计的情况下做到这一点是可能的。这通常是问题的开始。

当我们想到高度可访问的网站时，首先映入脑海的不一定是看起来很棒的例子。大多数挂着 Bobby AAA 或 WCAG 3 级认证徽章的网站也显示出界面乏味，导航缺乏想象力，性感程度不亚于伊比沙岛夜总会里半裸的 80 多岁老人。当你赞美一个可访问网站的优点时，这可不是你想给你的老板或潜在客户看的那种东西。

所以，给他们看看好东西！

##### 我们想要什么？更多！我们什么时候想要它？现在！

没有多少大公司的网站已经走上了可访问性的道路。当然，许多人已经做了一些象征性的手势，足以通过自动化(本质上是愚蠢的)博比 A 级测试。但是很少有，呃，“挑战极限”(好吧，你可以为此扇我一巴掌)。我这么说是什么意思？嗯，作为一个基本的列表，你可能期望看到这些策略在一个网站上实现，满足 Bobbly 级要求:

*   明智地使用“跳过导航”链接或仔细地对内容排序，这样导航就不会妨碍内容(第二种方法可能会进入高级领域)
*   导航不使用下拉列表(如在`<select>`标签中)(或者，如果使用，确保使用 Go 按钮激活，并提供一个`<noscript>`选项)
*   合适的 alt 属性，或者在不需要时为图像保留空白 alt 属性
*   明智的链接短语-没有“点击这里”类型的链接；当用户浏览时，每个链接都有其自身的意义

要求不高吧？如我所说，这是一个基本的列表，但我们应该要求更多:

*   无表格布局，在手持设备上运行良好，就像在配备 15 英寸全彩显示器的 PC 或 Mac 上一样
*   使用`<acronym>`和`<abbr>`元素阐明缩写短语
*   带有标题和 id 属性的数据表(使它们在 Jaws 和其他屏幕阅读器中可用)
*   主页上可访问性声明的链接
*   每个页面上都有指向站点地图的链接
*   根本没有框架内容(虽然 CSS 可以用来使它看起来像这样)

…这个清单还可以继续下去。事实上，我们需要确保至少再增加一项——它必须看起来像业务！但是除了个人博客之外，有多少网站做到了这一切呢？

##### 大男孩应该以身作则

在这个阶段，我可能应该提到 [Wired News](http://www.wired.com/) 和[ESPN](http://www.espn.com)——这两个网站被广泛报道，因为它们是大网站(我不知道我是否会将 Wired 描述为“公司”)，它们经历了大规模的底层结构重组。

然而，尽管非常明显地使用了 Web 标准进行布局，它们仍然不一定是完全可访问的(例如，如果你试图在 Netscape 4 上访问 ESPN，你不会得到无样式的文本；相反，你会被转到一个页面，解释为什么他们不愿意提供这种基本格式的内容。

除了这两个，你还能说出许多“超级环球公司”类型的人吗？

##### 已经够了！举个例子吧！

你想要一些好看的例子吗？好吧，但首先让我解释几件事:

这些网站并不是我真正想在这个列表中看到的大型企业，但希望它们对你来说是新的，并提供一些多样性。

我没有在各种各样的浏览器中亲自测试过每个网站——我在运行 Windows 2000 的 PC 上用 Firebird 和 IE6 测试过，我还在禁用样式表的情况下查看过它们，以感受它们在手持设备(或其他不支持 CSS 的浏览器)上的外观。

对，那是一条路！让我们看看一些漂亮的页面…

***一天电影***

![1214_oneday](../Images/411260c352b03a71dd6357dae8b3c37a.png)

由帕特里克·劳克(Patrick Lauke， [Accessify 论坛](http://www.accessifyforum.com)的定期海报，这里是 SitePoint)制作，[一天电影](http://www.onedayfilms.com/)大量使用 CSS 进行布局，并使用干净的灰度调色板来实现优雅的完成。

![1214_onedaynocss](../Images/62ce96cce962e8b59f8491c7c8085e99.png)

禁用 CSS 后，主要内容前几乎没有障碍——这里几乎不需要跳转导航链接。

 *![1214_einfach](../Images/2d3bfec12afc54a56b51b6bd2c4300e5.png)

这是我见过的最好的 CSS 布局之一。Einfach fr Alle，我可以翻译为“非常简单，适合所有人”(但这可能不太正确！)是 Tomas Caspers 的作品，与一个名为 Aktion Mensch 的项目有关。你必须向托马斯询问更多细节，但本质上这是一个德国项目，以多种不同的方式增强残疾用户的能力。

![1214_einfachnocss](../Images/dd2667932b72002eee8e59d276f2217a.png)

该网站是一个很好的有吸引力和无障碍设计的例子。请注意使用了`<link>`元素作为额外的导航工具(如果您使用的是 Mozilla ),还有旧的跳过导航链接——假设我对“Inhalt berspringen”的理解是正确的！

***Golf.uk.net***

 *![1214_golf](../Images/3701530d75806313f5fdf17977c1bdc4.png)

《Golf.uk.net》是汤姆·吉尔德的一件作品，他的名字会被一些人从他的[个人网站](http://tom.me.uk/)上知道。我讨厌高尔夫，但我真的很喜欢这个网站——它的外观和建造方式。

![1214_golfnocss](../Images/49938f334f53ce38009ff330beb3605d.png)

这在没有 CSS 支持的 PDA 上工作得相当愉快(尽管不可否认，脱离上下文的切片图形看起来有点奇怪)。跳过房子里的导航！

***托贝过去的字节***

![1214_torbays](../Images/7af053c73a350cef116709f1cd21aa74.png)

这个页面看起来离企业界越来越远([一个关于 Torbay](http://www.torbytes.co.uk/) 的历史网站)，如果我完全诚实的话，主导航菜单对我来说有点单调。

![1214_torbaysnocss](../Images/0cf149e48e23d948031deffd8478eb38.png)

然而，样式与内容的分离给我留下了深刻的印象(看看 CSS 禁用的截图)，尽管我个人不喜欢菜单项的简单性，但我喜欢它们使用正确的标记这一事实，即具有适当嵌套的列表项(`<ul><li>`)。感觉就像一个用爱精心制作的小网站。

***文学驼鹿*T3】**

![1214_litmoose](../Images/75508b08fe8a60c88e1015a2caa4a00c.png)

Wojtek 的网站[literal Moose](http://www.literarymoose.info/)在某些方面类似于 One Day Films(好吧，灰度效果)，并没有大胆宣称它支持可访问性，但它还是内置了。

![1214_litmoosenocss](../Images/ed43e1e5c2785a52620a4a8bc4e37e06.png)

访问键、跳过导航链接、导航列表项——它们都在这里。

***【网络编码】***

![1214_cybercodeur](../Images/cd63dac25c81a7ec4545d79b5cb83f34.png)

Denis Boudreau 可能是你熟悉的一个名字——他是少数几个法国(或魁北克)开发者中的一个，他们似乎很好地并且真正地解决了网页可访问性的概念，所以毫不奇怪，他的个人网络日志 [Cybercodeur](http://www.cybercodeur.net/) 展示了他对这个主题的了解。

![1214_cybercodeurnocss](../Images/14752bb5651279af984599ca8803d744.png)

“aller aux contenus”——这将是你的法语跳转导航链接。今天就去看看这个网站吧——你肯定会看到一些有趣的链接。

***OpenWeb***

![1214_openweb](../Images/6287549055f314c89aab6f0004863724.png)

如果我没记错的话，OpenWeb 是 Tristan Nitot 的作品，他曾经是 Netscape 的雇员，也是通用可访问性/标准的倡导者。该网站是一个很好的可访问性资源，根据你从初学者到专家的经验水平(假设你会说法语)提供内容，就像丹尼斯的网站一样，它身体力行。

![1214_openwebnocss](../Images/12335a68e4bb5a5ada9926b4a993c126.png)

另一个很棒的 CSS 布局，带有跳过导航链接(顺便说一句，没有隐藏在完全样式的版本中，就像许多其他网站一样)和良好的设备独立性。

##### 舍入

这是一个分类列表，我希望你喜欢看这些无障碍设计的例子。
