# 网站上线？检查您的副本

> 原文：<https://www.sitepoint.com/site-launched-review-copy/>

如果你正在开发一个应用程序或在线产品，你可能会经历非常清晰的开发、beta 测试、发布和修改过程。

如果你的产品或服务是在线的，每个过程都应该包括你在界面上使用的文本。

在过去的几个月里，我们看到:

1.  [如何对开发中的网页文本进行用户测试](https://www.sitepoint.com/test-your-web-copy-the-why-when-and-how/)以及
2.  如何使用 beta 测试反馈来增强你的文本,以便它在你的界面环境中更有效地交流。

今天，我们在这个等式上增加了第三步:发布后检查文本。这包括考虑错误消息、系统电子邮件、营销内容等等。

## 为什么要在发布后检查文本？

测试过程——从几个人的用户测试，到一群热情的拥护者的 beta 测试，再到发布到实际市场——每次都需要越来越大的样本量，最终的样本量是最大的。

对于开发人员来说，这提供了一个越来越清晰的画面，显示了对于大量以略微不同的方式使用系统的人来说，该系统是多么健壮和实用。

对于包括作家在内的 UX 专家和包括营销人员在内的沟通者来说，至少功能的健壮性的某些方面取决于它与用户沟通、预测他们的需求并对他们做出响应的有效性。

无论你的产品在技术上多么伟大，如果用户不能理解你的按钮文本或说明，他们就无法使用它。

发布后的文本审查可以让你就地、前后调整整个产品和整个营销信息。

基本上，它给你一个机会使信息一致，并使你的产品最接近用户的期望。

## 评论

好吧，所以你被评论说服了。但是你从哪里开始呢？

通常，您可以将文本审阅分为三个部分:

1.  产品界面，包括错误
2.  系统电子邮件和其他通知
3.  营销内容，包括销售页面、常见问题解答和帮助内容，可能还有一些营销活动电子邮件。

由于您的开发人员可能正在与错误作斗争并进行调整，所以从查看产品界面开始审查可能不是一个坏主意。毕竟，这是您的产品，上面列表中的其他项目需要围绕产品本身来构建。

先把产品做好，然后再调整你的通讯来匹配就更容易了。

### 查看产品界面

错误报告和客户支持请求是开始寻找界面文本问题的好地方，因为它们倾向于揭示误解或混乱的点。

你很可能会发现，在 beta 中工作良好的术语或措辞会迷惑更广泛的市场，所以在你真正考虑之前，不要把任何请求看得过于简单。

如此多的用户同时处理产品的不同流程，说明的表达方式是否存在逻辑问题就变得非常清楚了。很容易看出流程中各个步骤之间的联系是薄弱的还是失败的。从支持请求来看，修改这些问题非常简单。

一旦你觉得你的产品界面尽可能的紧密，并且你已经在必要的地方修改了你的[品牌词汇](https://www.sitepoint.com/tell-the-story-of-your-brand-service-or-product-with-a-brand-vocabulary/)，是时候转向你的系统邮件了。

### 查看系统电子邮件

要做到这一点，就像用户一样，从头到尾回顾一下你的界面。每次完成触发电子邮件或通知(或由电子邮件或通知触发)的操作时，请检查该通知是否反映了:

*   产品中触发器的上下文
*   在该上下文中使用的语言
*   收到通知时用户的上下文
*   你希望他们进入的环境。

这听起来有点复杂，但这意味着当用户收到通知时，他们需要知道它来自谁，为什么，以及他们需要做什么(如果有的话)。

例如，如果通知的触发是用户点击“注册”并输入用户名和密码以加入您的服务，您可能希望在系统电子邮件中引用他们的“注册”。

也许你会感谢他们的报名。也许你会在你的网站或帮助内容中给他们提供关键页面的链接，让他们更容易开始使用你的产品。

当你这样做的时候，你可能还想确保邮件之间的一致性，这样它们就有相同的品牌、语气、词汇等等。

### 审查营销内容

至少，如果在前两步的过程中，您已经更改了某些界面中的措辞，您可能需要重新拍摄营销宣传材料和常见问题解答的屏幕截图。

您可能还需要修改帮助内容中的屏幕参考，或者促销电子邮件中的说明。

如果你调整了你的品牌词汇，你可能需要改变营销材料中的某些参考。

## 好处

最近，我参与了一项发布后的文案审查，通过产品界面显著增强了品牌的声音。

这不是我所期待的结果，但发布后的审查允许我们将我们的界面调整为某种“最终版本”，它建立在我们通过用户和 beta 测试使用的界面文本的所有先前“草案”的基础上。

这一最终审查使我们能够结合我们对用户群的了解以及我们对(新)品牌和产品的定位和个性的日益增长的感觉。

发布后的审查将指导性文本从单纯的功能性转向清晰友好。我们的系统电子邮件从生硬但成功改进到轻松、高效和有效。我们的销售和帮助内容与产品相辅相成，使得使用体验看起来几乎毫不费力。

对于几个小时的工作来说，这是个不错的结果。

你完成了产品界面和支持界面的材料的发布后评审了吗？我很想在评论中听到它。
