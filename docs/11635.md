# 微软抨击谷歌 Chrome 框架

> 原文:[https://www . site point . com/Microsoft-Google-chrome-frame-security/](https://www.sitepoint.com/microsoft-google-chrome-frame-security/)

开球了！微软建议用户避免使用[谷歌 Chrome 框架](https://www.sitepoint.com/google-chrome-frame-fixes-ie/)，因为这是一个*“安全风险”*公司发言人发表了以下声明:

> 在 Internet Explorer 8 中，我们进行了重大改进和更新，使浏览器对我们的客户来说更加安全。
> 
> 鉴于插件的安全问题，特别是谷歌 Chrome，作为插件运行的谷歌 Chrome Frame 增加了恶意软件和恶意脚本的攻击范围。我们不建议我们的朋友和家人冒这个险。

谷歌迅速回击道:

> 使用 Google Chrome Frame 访问网站为 Internet Explorer 用户带来了 Google Chrome 的安全功能。
> 
> 它提供了 IE6 中没有的强大的网络钓鱼和恶意软件保护、强大的沙盒技术，以及在几天而不是几个月内就可以防御新兴的在线威胁。

微软的说法有什么实质内容吗？

引用安全性可能不是他们可以采取的最佳角度；微软住在一个大玻璃房子里，不应该扔巨石。IE 现在可能比其他任何浏览器都更安全，但情况并非总是如此。谷歌在 Chrome 上遇到了一些安全问题，但这些问题很快得到了解决。

还有，有多少病毒和恶意软件开发者是专门针对 Chrome 的？我怀疑与那些攻击 IE 的人相比，这只是一个小数字——它的市场份额要大得多。当你钓鱼(或钓鱼)时，顺理成章的是钓大鲨鱼而不是小鱼。

即使 Chrome 浏览器遭到破坏，也不能保证 Chrome Frame 会受到影响。正如我[最近报道的](https://www.sitepoint.com/google-chrome-frame-technical-details/)，这个插件运行在微软自己的沙箱 <acronym title="Browser Helper Object">BHO</acronym> 环境中。微软是说 Chrome Frame 可以中和 IE 的内部安全吗？如果是这样，这是一个阻止 IE 插件或完全停止使用 IE 的好理由。

最后，有趣的是微软只提到 IE8。如果每个人都升级到那个浏览器，Chrome 框架可能就不会存在了，但是许多用户仍然坚持使用 IE6 和 IE7。如果微软在新的浏览器上实现了 IE6 兼容模式，他们本可以解决这个问题，但这从未发生过，他们让谷歌来提供解决方案。

当然，微软不得不说点什么，他们不太可能对谷歌的产品表示赞赏。引用安全问题是一种廉价的策略；添加任何插件无疑会带来安全风险。然而，如果这些风险不是微不足道的，微软就应该移除他们的 BHO 系统。

来吧微软——不要浪费时间斥责谷歌，也不要无意中给 Chrome Frame 更多的免费宣传。提供你自己的创新解决方案来鼓励 IE 升级，而不是让竞争对手替你升级！
