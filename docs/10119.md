# 更改博客域名的三个选项

> 原文：<https://www.sitepoint.com/three-options-for-changing-the-domain-name-of-your-blog/>

改变你博客的域名可能会很痛苦。两年来，我在同一个博客上写了两次，我不想让我最大的敌人头疼。有一百万个活动部件，如果你忽略了一个，你可能会有大麻烦。

我学到的是，有选择…它不必是全部或什么都没有，你可以帮自己一个巨大的忙，如果你在潜水前探索选项。

当我决定我的博客需要一个新的域名时，我考虑了三个选择。

## 选项 1:整个东西，移动所有东西

此选项包括从您现有的域中导出您的所有内容，将其导入到您的新域中，然后设置 URL 重定向，将所有传入流量转移到您的新博客中。听起来很简单，对吧？不是的；这是对将要发生的事情过于简单的解释。比那激烈多了。

我第一次迁移走的是这条路，主要是因为我也是从 Typepad 迁移到 WordPress 的。保持在同一个博客平台上要容易一些，但是将整个博客从一个平台转移到另一个平台上需要你采取一些非常具体的步骤。

我的建议是首先备份你的数据库。然后，做你的研究，通读你能找到的所有教程。虽然有许多好的资源，但许多都有不同的步骤，在开始之前，您需要了解所有需要了解的内容。最后，不要开始迁移，除非你有足够的时间(至少两个小时)在迁移后进行故障排除和微调，除非你不在乎东西是否损坏。

这里有一些资源可以帮助您开始:

*   [移动博客](http://codex.wordpress.org/Moving_WordPress#Moving_WordPress_to_a_New_Server)，WordPress Codex
*   [如何将你的 WordPress 博客转移到一个新的域名](http://wphacks.com/how-to-move-wordpress-blog-new-domain-name/)，WPHacks.com
*   [用 10 个步骤将你的 WordPress 博客转移到一个新的域名](http://yoast.com/move-wordpress-blog-domain-10-steps/)，Yoast.com

*   从 Blogger 转移到 WordPress ，LifeHacker.com

## 选择 2:零零碎碎，只拿最重要的东西

这个选项只需要少量的文章和内容，并且需要手动的移动到你的新域。这是我为我的第二次博客搬迁所做的，就在上周。如果你没有成千上万的文章要转移，你在你的新博客上采取了某种新的方法，和/或你打算保留你的旧博客，至少一段时间，这是一个选择。

我上周的目标是带来最好的帖子，然后让旧博客安静地存在，并轻轻地将流量转移到新博客。我开始按照上述相同的过程从我的旧博客导出帖子和评论，然后将它们导入我的新博客。

在最初的内容移动之后，我做了一些小事情来平滑粗糙的边缘。以下是我为博客迁移所采取的一些步骤的概述:

*   我手动上传了所有的图片。
*   我在我的旧博客上使用了一个帖子特定的重定向插件，将流量重定向到新博客帖子([快速页面/帖子重定向插件](http://www.fischercreativemedia.com/wordpress-plugins/quick-pagepost-redirect-plugin/))。
*   我把我的 Feedburner 源改成了新的博客(并且更新了旧博客上对 feed 的引用以指向新博客)。
*   我在旧博客上添加了一个“我们搬家了”的通知，引导访问者到新博客。
*   我删除了旧博客的评论功能。

这不是一个完美的解决方案；你可能会失去一些读者。但对某些人来说这是个不错的选择。

## 选择 3:重新开始，离开

如果你写博客的时间不长，博客的访问量不大，改变了博客的主题，或者不值得将博客转移到一个新的领域，那就重新开始吧。

你可以采取上面的一些步骤来获取现有的流量，但是重新开始可能更有意义。有时候，最简单的选择真的是最好的选择。

你曾经改变过你博客的域名吗？你走了什么流程？你希望你做得不同吗？
