# 有状态 Web 服务

> 原文:[https://www.sitepoint.com/stateful-web-services/](https://www.sitepoint.com/stateful-web-services/)

我刚刚在爱丁堡的国家电子科学研究所参加了一个为期两天的课程，学习 Microsoft.NET 和 OGSI。期待一个更完整的帖子，然而，OGSI 的一个很好的特性是它允许你自动部署持久状态 web 服务(在一些基本的继承之后！).

你为什么需要国家？在传统的 web 服务中，对象实例被创建、调用，然后在请求完成后被销毁。下次调用您的服务时，会创建一个新的实例，依此类推。通过这种方式，web 服务是无状态的:一旦被销毁，它们就不会保留任何关于其变量和属性的信息。

你可以一起破解状态管理。您可以序列化 web 服务，并在将来出现请求时调用它。这不仅麻烦，而且难以以标准化、模块化的方式实现产生具有生命周期管理(即，可以存在一段设定的时间)的短暂服务的过程，以及分配标识符来管理不同的实例，使得每个用户可以具有他们自己的指定服务实例。

MS.NET·OGSI 给了你这种能力，并为。NET，兼容 [Globus Toolkit 3.0](http://www.globus.org/) 。
