# 开发人员的可用性

> 原文：<https://www.sitepoint.com/usability-for-developers/>

如果说在过去的六个月里有什么事情引起了我的注意，那就是包含可用性的网络角色种类的增加。我在很多领域都注意到了这一点:职位发布、行业事件、在线讨论和社交网站上的个人标签，如 [Web Connections](http://connections.webdirections.org/tags/usability) 。SitePoint 甚至发布了一个[可用性工具包](https://www.sitepoint.com/books/usability1/)并推出了这个博客。我觉得这很有趣，因为它标志着网络社区在开发过程中对可用性的重视程度发生了转变。

快速找工作(继续，我在给你找借口！)你会发现前端开发人员、信息架构师(IA)、制作人和视觉设计师都需要某种程度的可用性知识。我在这里一般使用“可用性”这个术语，来结合用户测试、用户体验和交互设计，但本质上，我所说的是以用户为中心的网站开发方法。

但是我呢？
我是丽莎·赫罗德，[访问测试](http://www.accesstesting.com)的高级可用性顾问。在过去的 8 年里，我一直从事网络方面的工作，担任过各种角色，包括:设计、开发、可访问性、可用性，甚至(在一个远离这里的世界里)一名专注于“网络内容”的标准教师。虽然我肯定是一个可用性的人，但我真的很有兴趣看到一个整体的 web 开发方法，将可用性和 web 标准结合在一起。因此，我观察到的这种将可用性融入各种网络角色的转变真的非常有趣。

在我看来，看待网站可用性有两种主要方式:老派和新派。

传统的人机交互(HCI)是我认为的“老派”方法。它起源于前网络时代，可用性专家评估桌面应用程序，技术考虑(如跨浏览器兼容性和 Javascript 支持)不太可能影响用户。如果你想对一个网站进行高水平的评估，这仍然是一个非常有效的方法，但是现在它不能提供很大的分析深度。

我称之为“新学派”的方法，要求可用性专家对网络有更多的技术知识。结构、内容、表现(CSS)和行为层都会影响网站的可访问性和用户体验。关于网络如何工作的知识允许可用性从业者与设计者和开发者更紧密地合作，以生产更好、更有用的产品。

我在这里所说的是，从项目开始，所有角色都需要关注网站团队的可用性。很明显，雇主在寻找具有可用性技能的设计师和开发人员。我们也看到越来越多有经验的架构师涌现出来——也就是具有技术技能的可用性从业者。所以，它正在那里发生。Fairfax、News 和 Westpac 的开发团队正以这种方式做着出色的工作，毫无疑问，还有许多其他团队也是如此…

你呢…？
