# FxCop

> 原文：<https://www.sitepoint.com/fxcop/>

对我来说，这是最好的工具。网到此为止。而且，我这里包括 nDoc:)

FxCop 是一个代码检查器，它检查程序集是否符合。NET 框架设计指南。而且，真的管用。算是吧。

该程序使用自省和灵活的规则引擎来反编译和分析托管程序集，并向您显示您的代码在哪里违反了准则。

作为测试，我用 FxCop 检查了我自己的博客代码。这一过程只需几秒钟，一系列的不良做法就出现了。我实际上从分析中学到了。例如，我用了一个马虎的:

 `if (bodyText == "")`

FxCop 告诉我使用以下命令检查空字符串:

 `if (bodyText.Length() == 0)`

因为这样效率更高。现在，这很酷。

它强调的许多问题实际上是由 Visual Studio 产生的。NET，如:

"使“CheckBox1”成为私有或内部的，并提供一个公共或受保护的属性来访问它."

然而，把这些错误放在一边，你真的可以得到一个如何改进你的编码的好主意。多棒的工具啊。

FxCop 是微软出品的，[免费下载](http://www.gotdotnet.com/team/fxcop/)。
