# Adobe 回击了苹果对 Flash 的批评

> 原文:[https://www.sitepoint.com/adobe-apple-flash-criticism/](https://www.sitepoint.com/adobe-apple-flash-criticism/)

手套已经脱了！史蒂夫·乔布斯指责 Flash 漏洞百出，Adobe 开发人员懒惰，Adobe 首席技术官凯文·林奇对此进行了反击。他在 Adobe 官方博客上发表了一些有趣的评论…

Flash 最初是为笔式计算平板电脑设计的，比这个市场准备起飞早了大约 15 年。

我没有意识到。Flash 由 SmartSketch 演变而来，smart sketch 是 PenPoint OS 的绘图应用程序，PenPoint OS 是早期基于平板电脑的操作系统。

通过增强 HTML 的功能，Flash 在采用方面取得了令人难以置信的成功，超过 85%的顶级网站包含 Flash 内容，并且 Flash 在网上超过 98%的计算机上运行。

98%的采用率听起来很合理，但是 85%的顶级网站呢？我想这可能是正确的，因为前 20 名将包括 YouTube、脸书和其他几个以视频和恼人的广告为特色的网站。然而，当你考虑全球网站技术统计时，Flash 的渗透率要小得多。

除了一家顶级制造商之外，我们即将为智能手机推出 Flash Player 10.1。

我想知道他指的是谁？

*那么，运行在苹果设备上的 Flash 呢？*

啊，他们！

*我们已经展示了 Flash 技术今天开始在这些设备上工作，它支持在 Flash 上构建 iPhone 的独立应用程序…其中一些应用程序已经在苹果应用商店中提供…同样的解决方案也将在 iPad 上工作。*

我不怀疑。当 Flash 在功能较弱的设备上运行时，苹果很难声称它对他们的平台来说太慢了。

从长远来看，一些人认为 HTML 最终会取代对 Flash 的需求，特别是 HTML 第 5 版的最新发展。我不认为这是一个替代另一个，当然不是今天，甚至在可预见的未来。

如果 HTML 能够可靠地做 Flash 所做的一切，那肯定会节省我们很多努力，但这似乎不会成为现实。即使在视频的情况下…即将到来的 HTML 视频实现也不能在一个共同的格式上达成一致。由于能够在不到一年的时间内更新大多数 Web 客户端，Flash 可以比 HTML 更快地跨各种浏览器向我们的客户提供这一创新。

嗯，他会这么说——尤其是有 Adobe 股东在听——但凯文说得有道理。HTML5、CSS3、JavaScript 和其他基于标准的技术有可能取代 Flash。然而，规范处于不断变化的状态，跨浏览器开发目前是不可能的。此外，成熟的 Flash 工具可以让设计人员在没有编程知识的情况下创建效果。HTML5 就不一样了。

从长远来看，开放获取已被证明比封闭方法更有效，在封闭方法中，制造商试图确定哪些用户能够看到或批准或不批准个别内容和应用程序。我们坚信，网络应该保持开放的环境，无论您使用何种查看设备，都可以持续访问内容和应用程序。

这是一个很好的观点。苹果希望为他们的客户提供出色的用户体验，但任何操作系统制造商都应该能够决定在他们的平台上开发和使用哪些应用程序吗？

也就是说，苹果提出了一些合理的批评。Flash 在一些系统上臃肿且漏洞百出，Mac OS X 版本收到了大量投诉。虽然 Flash 可能无处不在，但它不是一个开放的标准——Adobe 有自己的“围墙方法”。

但是公开的诽谤对苹果和 Adobe 都没有好处。赞同苹果的用户现在对 Flash 技术持负面看法。那些认为 Flash 是现代网络浏览必不可少的一部分的人不会买 iPad。两家公司都会输。

在我看来，苹果应该让他们的用户来决定。允许 Adobe 生产 iPhone/iPad Flash 播放器，并允许任何人随意安装。如果软件臃肿、有问题或运行缓慢，他们很快就会卸载。苹果是否急于放弃他们的应用商店垄断地位是另一回事。
