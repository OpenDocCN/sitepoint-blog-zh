# 我的(最好的)客户想要的一样东西

> 原文：<https://www.sitepoint.com/the-one-thing-my-best-clients-want/>

客户想要什么？这是一个古老的问题，显然没有唯一的答案。但是在我最好的客户关系中，我的客户最终希望我提供一件东西，而我能够给他们。

每当我感觉到客户想要这个东西，我就知道我处于一个很好的位置，可以享受一些乐趣，做一些好的工作，赚一些钱。那是什么东西？

**我最好的客户想要的东西**

我最好的客户不希望我写作或编辑。他们想让我解决他们的问题。

我最好的客户关系超越了我提供的基本技能。他们利用我能提供的其他经验和能力来解决额外的问题——甚至可能是我的客户还没有预见到的问题。

最终，当客户不把我视为一件商品——一个有能力做 X、Y 和 Z 的人——而是一个对市场、学科或行业有更广泛理解的专业人士，我可以用它来预测和解决他们的项目面临的问题，为整体结果增加价值时，我最好的客户关系就产生了。

当客户开始将我视为一个问题解决者时，他们的问题开始从他们与我会面的最初原因(例如，编辑或撰写一篇内容)扩展到更大的问题:沟通策略、目标受众访问、内容可用性等等。因为他们在寻找答案，所以他们往往很坦率，所以在这种情况下，我通常可以找到很多关于项目走向的信息，它如何适应更广泛的战略，他们已经考虑了什么，以及我可以在哪里提供帮助。

这些讨论为我提供了提议额外服务的机会，或引起对我可以增加价值的领域的注意，并解释这种价值是什么以及如何实现。他们让我很快建立起坚如磐石的关系。他们允许我展示我分享我的客户对他们项目的热情，并希望与他们一起做好工作。

**从商品走向有价值的顾问**

将你在工作关系中的地位从商品转变为有价值的顾问是一项挑战。在某些情况下，你的客户会拒绝将你视为除了设计师、开发商或作家之外的任何人——他们会在自己的头脑中给你定型。打破这种看法可能很难，因为根据我的经验，这样的客户很少愿意花时间谈论他们认为超出你的范围的任何业务方面。

但是有一些策略你可以用来让客户参与到更详细的讨论中，这将帮助你们双方理解与这个项目或其他项目相关的问题，你可以帮助他们解决这些问题。

**1。提问**

我通常发现，我与客户的初次会面和简短会面为更详细地探讨他们的需求提供了很好的机会。由于我已经提出了问题，客户通常有心情公开谈论他们在项目的其他领域面临的问题，可能还有他们的业务。

上周，当我接受一个新客户的书籍编辑的初步简报时，我们开始讨论她的目标受众。在那里，我问她打算如何推广这本书，随后的讨论让我有机会就她如何以不同的方式利用现有内容在特定的读者群中更有效地推广这本书提出一些即兴建议。这些想法中有许多对她来说都是全新的。

**2。提建议**

在那次简报会上，我向我的新客户提出的想法并不是特别具体，但当我解释这些想法时，我可以看出她没有想到这些可能性，并感谢我的投入。

通过提出有价值的建议来帮助解决客户面临的问题，你展示了你更广泛的专业技能和知识，并且你有机会用过去的客户或项目的例子来支持这些建议，在这些例子中你使用了类似的策略并取得了很大的效果。这有助于证明你的推荐是合格的，并鼓励你的客户不再将你视为一个技能组合，而开始将你视为一个有技能的解决方案提供商。

**3。支持这些建议**

如果你不向你的客户提供更多的信息、进一步的研究和/或关于你能提供的解决方案的书面建议来支持你的建议，你可能很难从你更广泛的技能组合中实现任何价值。

我和我的客户的谈话提醒了她一些她以前没有意识到的问题。她可能不确定这些促销问题有多重要，或者她应该如何解决这些问题。这种不确定性为我提供了一个绝佳的机会，让我站出来扮演我们在那次会议上为我勾画的“专家”角色。

因此，当我将项目估算发送给她时，我可能会包括一系列针对项目的更广泛的问题解决建议，以及他们自己的成本估算。或者，我会详细说明我们谈到的问题，并向她提供一些额外的背景信息，说明她小心处理这些问题的重要性。也许我会把我完成的其他项目中出现类似问题并成功解决的信息发给她，并提出和她单独讨论这些问题。

无论如何，如果我想帮她解决问题，我需要用一些具体的行动来支持我们的讨论。

**4。邀请参与**

让所有这些辛苦的工作白白浪费是没有意义的——如果我真的想要这份工作，我需要邀请我的客户同意带我去解决这些其他问题。

通过邀请他们让你做额外的工作来展示你对项目高质量成果的承诺，以及你对工作和客户的热情。

我认为客户真的希望他们的问题得到解决，这就是我满足这种需求的方式。你认为客户最想要什么？你如何确保你交付它？
