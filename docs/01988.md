近年来，JavaScript 见证了数量惊人的库和框架的出现和消失。很难做出在项目中使用哪个软件的重要决定，因为总是存在依赖维护者无法支持的库的风险，甚至可能最终放弃。

为了尝试和解决围绕 JavaScript 生态系统的支持和开发的一些问题，著名的 [jQuery Foundation](https://jquery.org/) 更名为 [JS Foundation](https://js.foundation/) 。让我们记住，在一年多以前，jQuery Foundation 本身与 Dojo Foundation 合并了(如果每次有人说“Foundation”时我都有一分钱就好了！).

## 前进的基础

JS 基金会执行董事 Kris Borchers 发表了一份关于其成立的声明:

> “Linux 基金会的主要使命是创造世界上最大的共享技术投资。JavaScript 是一种极其重要的编程语言，围绕它出现了许多开源项目。
> 
> 这些项目中的许多对于互联网的基础设施是必不可少的，因此 Linux 基金会认为确保他们拥有结构化的支持和中立的治理以确保他们的稳定性是很重要的，这就是 JS 基金会成立的原因。"

将两个基金会合并成一个需要极大的无私，所以让我们花一点时间来欣赏它。这可能意味着两件事情中的一件(或两件):

1.  我们已经厌倦了 JavaScript。很难跟上最新的技术，因为在您享受开发人员假期期间，工作流可能会发生变化。需要一个披着斗篷的十字军来给混乱的 JavaScript 之城带来秩序。
2.  jQuery 和 Dojo Foundation 有着相似的目标和价值观，所以他们认为联合起来是合适的。

鉴于第一种情况是技术社区中各种迷因和笑话背后的原因，清理并将所有人团结在一个旗帜下可能是一个健康的步骤。开源社区高度赞扬了去中心化，但是人们也普遍认为一个仁慈的独裁者实际上可以改善一个项目(或者在这种情况下，生态系统)的状态。

最近的一篇文章取笑了 JavaScript 的现状，批评了这种语言不断变化的前景。虽然有些夸张，但它传达的信息相当准确。如果你已经有一段时间没有使用 JavaScript 了，最近又回来了，你可能会有一个陡峭的学习曲线。也许这就是 JS 基金会可以解决一些问题的地方？

## 管理

基金会通过两个分支机构将业务和技术决策分开:董事会(任命执行董事)和技术咨询委员会，后者独立于董事会，处理基金会的所有技术问题。我必须说，这是明智之举。

## 贡献者许可协议(CLA)

像许多其他开源项目一样，JS 基金会也有一个 CLA。这些条款适用于您对 JS 基金会或其项目的所有捐赠。对一些人来说，这可能听起来很可怕，但这在当今却是司空见惯的事情。如果许可令你困惑，看看我去年写的关于[开源许可](https://www.sitepoint.com/introduction-to-open-source-licenses/)的文章。

## 项目

JS 基金会是各种项目的大本营，这个名单还在不断增加。其中最受欢迎的有 jQuery(惊喜惊喜！)、咕哝、要求。相当多的项目在重新发布之前也是 jQuery Foundation 的一部分，比如 Lodash 和 ESLint。新到的有摩卡、Moment 和 Webpack。

你可以在 JS 基金会网站上找到项目的[完整列表。](https://js.foundation/projects/)

## 联合王国

此外，由于共同的目标，JS 基金会和 Node.js 基金会将密切合作。最有趣的是，JS 基金会可能会促进对第三方包的支持，这是 Node 基金会尚未解决的问题。

真是有趣的时光。JS 基金会仍处于早期阶段，但它的计划看起来已经很有希望了。你怎么想呢?它会改变 JavaScript 的前景并有助于在一面旗帜下团结努力吗？请在评论中告诉我！
