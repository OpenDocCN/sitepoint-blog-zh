# 定义的信息架构

> 原文：<https://www.sitepoint.com/architecture-defined/>

你知道当你在一个网站上看到一堆导航选项可以点击吗？当你点击时，我是决定选项是什么、它们叫什么以及它们把你带到哪里的人

是的，我说过了。我必须解释信息架构(IA)很多，相当多。这些天，每个人似乎都同意这是一个热门话题。没有多少人真正知道内务部是干什么的。在 dice.com 或 hotjobs.com 搜索，你会看到各种各样奇怪的工作要求，从 perl 编码到图形设计。我一度认为信息架构师这个头衔会和网站管理员一样:这个头衔对于需要一个什么都会做的人的小公司来说很好，但是对于需要专家的公司来说就不好了。幸运的是，一些知名度很高的会计师事务所的不懈努力正在改变这种状况，现在我们至少缩小了自己的业务范围。

##### 那么 IA 是做什么的呢？

就像我们现实世界中的同名者一样，我们为人类设计生活、工作和玩耍的空间。最大的不同是我们使用的材料:水泥被同义词取代，木材有层次，钢铁有交互流。迷茫？我来讲个故事吧。哦，为了做到这一点，我将不得不透露我是一个大傻瓜。希望这不会拖累你。

当我在 geocities 上建立第一个网页时，它只有一页。里面有我朋友的照片，一点关于我的历史，还有一个给我发邮件的链接。相当典型。当时还没有信息架构，但肯定有一些与信息架构密切相关的信息设计。我知道来我网站的人需要知道他们在看什么，所以我在顶部写了一个大大的 H3 标题，“欢迎来到克里斯蒂娜·沃德克的个人网站”。接下来我知道我想要一些关于我的东西，所以我添加了历史和图片。最后，我想如果人们读完了这些，他们可能会想和我谈谈，于是我添加了我的电子邮件地址。奇怪的是，我做的是非常可靠的以用户为中心的设计。我问自己，访问我网站的人会如何使用它。

我的下一个网站更加努力。当我在评论 Snap.com 的美食网站时，我在网上收集了大量的食谱。我也有一些家庭食谱和一些从我丈夫的一个法国朋友那里学来的食谱。我想把它们都放在网上，这样我就可以从任何地方访问它们——我父母的房子，我的男朋友(现在是丈夫)或者家里。因为我还没有成为一名优秀的信息架构师，所以我从研究开始——我翻阅了所有的食谱，看看它们是如何组织的。我有太多的组织方案可供选择:地区，课程，关键成分，菜肴类型…老实说，我不知道该走哪条路。那时我使用了另一个很好的 IA 工具:卡片分类。这些是食谱，已经在索引卡上了。我只是把它们都放在地板上，然后开始把它们堆在一起。很快我就知道我需要参加课程组织。我的烹饪网站。我在 1996 年建立的网站今天仍然保持着同样的形式，我仍然在它上面做饭，感叹着奇怪的 gif 和幼稚的排版。但是我总能找到我要找的食谱。

##### 设计网站

所以上周末，我去了一个建筑师同事的办公室，我们花了一整天的时间重新设计一家 IT 解决方案公司的网站，该网站有 2000 多条内容。我们以我制作第一个网页的方式开始:我们仔细阅读文档，发现受众是谁，他们的主要需求是什么。我们写下了我们认为会使用该网站的三个人的名字，并在每个名字下面写下了他们的需求。总裁戴夫想知道他是否应该购买该产品。吉尔，这位潜在的雇员想知道这是不是一个很酷的工作场所。卡拉，想知道这是否是一个热门购买的投资者。

接下来，我们做了我为我的第二个网站做过的事情:打印出内容清单，并把它剪成小纸片。然后我们做了我做过的最大的卡片分类。我们将所有相似的内容分组，然后为这些内容创建一个层次结构。然后我们角色扮演。我们假装成不同类型的观众，因为他们试图使用层级“我是首席执行官戴夫，我需要知道如果我买了这个产品，它能解决我的问题吗？关于升级的部分在哪里？有产品评论吗？这些家伙做生意多久了？”诸如此类。我们对分组做了一些调整，然后就完成了。

当然，这并不是 IA 为网站所做的一切。这就是 IA 为内容网站所做的事情。但是你可以看到关键的原则。了解用户。了解商业产品。将两者结合在一起。

现在让你的网站更容易被你的客户使用…
