# 虚拟主机选项

> 原文：<https://www.sitepoint.com/web-hosting-options/>

关于互联网，我有两个理论。

**理论#1:** 聪明人建立互联网，更聪明的人管理互联网，但技术上有挑战的人使用互联网。

理论 2 网站有一个自然的发展过程。他们从一个免费的服务器开始，然后转移到一个付费的虚拟主机，在某个时候会有一个域名。从这里，他们向上移动到更好的虚拟主机，然后迁移到虚拟服务器。一旦他们达到这一点，剩下的唯一步骤就是移动到一个专用服务器，甚至多个服务器，这取决于配置。专用服务器可以租赁、管理，或者简单地放在一个数据中心。

尽管我肯定你很想让我扩展理论 1，但我还是会花时间讨论理论 2 的复杂性。更重要的是，我将跳过免费的虚拟主机和虚拟服务器，直接进入专用主机领域。

首先，让我告诉你这篇文章将如何发展。你看，就像互联网一样，这篇文章在你阅读的时候也在变化。见鬼，我写的时候它在变。我不能给你一个接下来会发生什么的确切想法，但是我会尽我所能让你为即将到来的事情做好准备。记住这一点，让我们开始我的旅程，进入专用主机的世界…

大多数人在选择专用服务器时首先会考虑的是价格(请注意，出于文章目的，“专用”一词包括专用服务器、托管服务器和并置服务器)。虽然这很重要，你的预算也是一个大问题，但实际上你还有很多计划要做。那些只看价格购物，试图找到最便宜的、拥有所有“酷”东西的人将会上一堂艰难的课。

##### 操作系统

那么，我必须做出的最大决定是什么？操作系统。是的，我知道你们中的许多人甚至因为我考虑 Win 2K 与*nix 而对我大喊大叫，但我必须考虑我的所有选择。那么我选择了什么？嗯，就像大多数的网络服务器一样，我选择了 nix 风格。准确的说是红帽 6.2。为什么？预算对我来说是一个很大的考虑因素，对你们很多人来说无疑也是如此。因为，我想在硬件和玩具上花更多的钱，我想我可以通过使用免费的操作系统来节省一大笔钱。

我选择红帽是因为它在行业中庞大的安装基数和广泛的支持渠道。事实上，大多数人都知道 Red Hat，这是有据可查的，RPM 的想法(是的，我知道其他口味有包装系统)从一开始就吸引了我。

***快速注释* *
RPM–Red Hat Package Manager–这是一种在 Red Hat 系统(或启用 RPM 的系统)上安装软件和程序的简单快捷的方法。RPM 将允许一个命令安装和卸载，而不需要“制作”文件、冗长的预配置和编译。换句话说，对于 Linux 新手来说，这是一个很好的工具。

##### 该系统

现在我已经整理好了操作系统，我开始着手服务器本身的机械部分。这引出了下一个大问题:我想要什么样的系统？专用服务器、托管服务器还是并置服务器？对一些人来说，这是一个基于你的预算的简单选择。对于其他人来说，这是一个知识问题，还有一些人认为这是一个控制问题。

##### 定义:

**并置服务器**–这是指您构建或购买一台服务器，然后将其放置(也称为并置)在数据中心。除了为服务器提供电源和互联网连接，托管公司不负责服务器的任何方面。

 <q>**优点:**你拥有硬件和软件，对一切拥有 100%的控制权。</q>

反对意见:你将不会得到托管公司的免费支持。

**专用服务器**–这是指您从一家公司租赁服务器，这家公司负责服务器的硬件，而您负责软件。

 <q>**优点:**你不必担心硬件故障或硬件升级，因为托管公司会对此负责。</q>

**缺点:**通常每月费用较高。你也被迫使用托管公司提供的任何硬件。(即 AMD 优于英特尔处理器)。硬件升级通常以每月重复费用的形式向您收费。

**托管服务器**–托管服务器是托管公司保留完全控制权的服务器。您只需为服务器提供数据，并指定如何维护该服务器。通常，客户端没有根访问权限，没有托管公司的授权，无法添加或删除程序。

 <q>**优点:**一般来说，你几乎不用担心，因为托管公司会承担所有的硬件、软件和安全问题。这让你有更多的时间专注于网站开发，而不是服务器管理</q>

缺点:通常是最昂贵的选择。与其他两个选项相比，您对服务器日常操作的控制能力确实更弱。然而，这是托管公司兑现其正常运行时间、安全性等承诺所必需的。

##### 我的解决方案

那么我选择了什么，为什么？嗯，虽然我很想使用专用服务器，但我还是选择了并置服务器。这篇文章的托管服务器有点傻，但这是一个 3 秒钟的好梦(改写之前的句子)。我决定的另一个重要因素是价格。对我来说，这也是环境使然，因为我的数据中心就在附近，所以我不能错过这个机会。

##### 硬件

因此，我有了自己的操作系统，并选择了我希望服务器托管的方式。这个过程的最后一步是挑选硬件和服务器类型。我们再次需要一些定义:

**白盒服务器(又名架装服务器)**–为此，我希望你低头看看你的电脑。那是一台白盒服务器。这些通常是自制的服务器，制造成本最低。

 <q>**优点:**容易制造、升级和维护，因为我们都在摆弄我们的家用电脑。</q>

**缺点:**托管公司通常会根据服务器的大小向你收取更高的费用。此外，由于该服务器必须放置在货架系统上，因此更容易被路人“移动”。

**机架式服务器**–这就是你在任何时候看到的托管公司服务器机房。这些服务器的列将在它们的机架中并排排列。这也是大多数托管公司将出租的服务器类型。

 <q>**优点:**一般来说，它们更容易安装，许多托管公司都会欣然接受。此外，这些类型的服务器的价格是基于每 U 的基础上。1U 大约有 1 3/4 英寸高。</q>

**缺点:**通常这些服务器很难升级，除非你预先花了一些钱购买热插拔设备(如硬盘等)。

我选择了机架式服务器。我将使用英特尔的 ISP 1100。它将配备一个英特尔 PIII 800 兆赫处理器，128 兆内存，和一个 Gig IDE 硬盘。

现在解决了这个问题，我们可以开始安装软件了。下次见面时，我们将讨论挑选额外的软件，当然，还有安装这些软件和操作系统。
