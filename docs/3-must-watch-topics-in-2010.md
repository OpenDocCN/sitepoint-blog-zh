# 2010 年 3 个必看话题

> 原文：<https://www.sitepoint.com/3-must-watch-topics-in-2010/>

![Danger!](img/f5792b4c2b1cd5d35276eaecca9d5f4f.png "979704_danger")**2010 年，以及未来，你最期待什么？有几个领域我很想关注——我怀疑在未来几年，它们将成为我们作为开发人员生活中的重要部分，如果你错过了这条船，你将会错过一些大机会……**

## 了解你未来的浏览器

超越你的开发伙伴的最简单的方法之一是密切关注你最喜欢的浏览器的即将到来的版本。以下是三个值得关注的重要浏览器:

*   11 月，我们得知 Internet Explorer 团队已经开始开发 IE9，这是一个好消息。[在一篇博客文章中，](http://blogs.msdn.com/ie/archive/2009/11/18/an-early-look-at-ie9-for-developers.aspx)该团队透露了他们浏览器最新版本的一些内幕:JavaScript 性能的巨大改进，图形硬件加速([查看 9 频道](http://channel9.msdn.com/posts/Charles/IE-9-Surfing-on-the-GPU-with-D2D/)的这个精彩视频)，以及 CSS3 支持方面的一些有希望的迹象。IE9 在 2011 年之前出现在桌面上的可能性很小，但与此同时，我希望我们都能期待 IE 团队的大量更新，因为他们正在开发最新版本。
*   火狐 3.6 测试版现在可以测试了，它充满了可爱的新功能。[根据发布说明，](https://developer.mozilla.org/En/Firefox_3.6_for_developers)我们可以期待看到一些处理背景图像和渐变的伟大的新方法，[对 HTML 5 的本地文件访问 API 的支持，](https://developer.mozilla.org/en/Using_files_from_web_applications)以及对 Firefox 插件开发者的改进。Firefox 3.6 最初定于 2009 年末发布，但尚未达到发布候选状态——所以请密切关注。
*   谷歌 Chrome 扩展终于来了。这将有助于 Chrome 在 2010 年蚕食 Firefox 的市场份额吗，尤其是在我们这些热爱 Web 开发者工具栏和 Firebug 的人当中？我们把这个问题告诉了我们的 Twitter 朋友，他们的反应确实不一:一些人绝对等不及了，而另一些朋友则相当忠实于 Firefox。有一点是肯定的——2010 年，随着更多扩展的出现，将会有更多尝试 Chrome 的动机。(我们在上周的*社区报*、转载了[精选的推文，如果你想要完整的独家新闻。)](https://www.sitepoint.com/the-second-coming-of-the-productivity-tool/)

## 移动设备无处不在——现在就加入吧！

在 2010 年，公平地说，任何想要跟上时代的开发者都需要专注于构建他们的移动开发技能。移动网络访问绝对越来越受欢迎，移动 Safari(iPhone 和 iPod touch 上使用的)占据了开发人员大量的时间和精力。专家们疯狂猜测下一版 iPhone 会出现什么……有机发光二极管屏幕？嵌入式 RFID 芯片？前置摄像头？与谷歌手机偷偷摸摸的预览相比，这算不了什么，Windows Mobile 7 应该也会在 2010 年末出现。

然而，我们网络开发者感兴趣的是移动网络浏览，这绝对是蓬勃发展的。如果你还没有提高你的移动网络开发技能，现在是一个很好的时机，在你错过一些惊人的机会之前加入进来。当然，Mobile Safari 在博客和媒体中获得了大量的报道，但如果这就是你关注的全部，你可能会想扩大你的视野 Opera Mobile 版本 10 的最新测试版中有很多值得喜欢的东西。

Opera Mobile 10 有趣的地方在于，它是第一个安装了 Presto 2.4 最新版本的浏览器。在 [Dev 上有一个关于 Opera Mobile 10 内部的很好的介绍。Opera](http://dev.opera.com/articles/view/opera-mobile-10-beta-developers-introduction/) ，它揭示了我们可以期待一些新的 CSS3 处理，使用 Opera 的蜻蜓工具进行远程调试，以及很好地使用`@media`规则进行移动专用开发。

Opera 拥有超过 4000 万移动网络用户，这是一个巨大的市场份额。现在很值得研究一下。

## 社交媒体将统治世界

好吧，这个标题可能有点夸张。但是，如果你在市场上，你最好很好地了解如何最好地利用社交媒体，并将其整合到你的项目中——因为社交媒体是任何在线做生意的人必备的元素。

好消息是，主要的社交网络使得与他们的成员互动变得极其容易，提取出美味的数据，并创建可爱的小部件。这里有一些有趣的地方可供选择:

*   无论你是在桌面客户端上工作，还是集成网络应用，或者只是想在你的博客上使用脸书连接，脸书的开发者 wiki 都充满了关于脸书应用开发的有用信息。
*   谷歌的[社交图 API](http://code.google.com/apis/socialgraph/) 使得获取一个人整个在线社交生活的信息变得轻而易举——鉴于我们每天使用的社交网络和应用数量惊人，获取这种信息是绝对必要的。在这里，你的第一步应该是[了解所有关于 API 的信息](http://code.google.com/apis/socialgraph/docs/api.html)，并[试用一些示例应用程序。](http://code.google.com/apis/socialgraph/docs/examples.html)顺便说一句，如果你对 Rails 感兴趣，Louis [在 SitePoint 的一篇教程中提到了这一点。](https://www.sitepoint.com/article/google-social-graph-api-ruby-rails)
*   很自然，Twitter 现在是一个非常重要的东西，而且很容易赶上这股潮流。一定要研究它的 API ,了解你可以与 Twitter 互动的所有不同方式；无论你喜欢什么平台或环境，总有一个[库可以帮助你。](http://apiwiki.twitter.com/Libraries) [我们最近也报道了 Twitter 的 API](https://www.sitepoint.com/article/share-media-flex-twitter)虽然这是一个 Flex 教程，但仍然有一些关于 Twitter 应用开发的有趣信息。
