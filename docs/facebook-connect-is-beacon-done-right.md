# 脸书连接信标做得对吗

> 原文：<https://www.sitepoint.com/facebook-connect-is-beacon-done-right/>

还记得脸书灯塔吗？去年 11 月，脸书首席执行官马克·扎克伯格在纽约市的独家、超级宣传、仅限受邀者的发布会上称其为“全新的在线广告方式”。但仅仅一个月后，他就发布了一份道歉声明。“我们在构建该功能时犯了很多错误，但我们在处理这些错误时犯了更多错误。我们只是在这个版本上做得不好，我为此道歉，”他在公司的博客上写道。问题？糟糕的隐私控制让用户觉得他们被欺骗了，向朋友和网络暴露了他们的购买习惯，并在没有同意的情况下成为了营销人员。

Beacon 背后的想法是合理的:如果你能让人们向朋友推销他们已经购买的产品，这些代言将比常规广告更有效，因为它们来自一个可信的来源。不幸的是，对于扎克伯格和脸书来说，这个实现还有很多不足之处，它被读写网评为 2007 年最大的技术失误之一。

今天，脸书宣布发布脸书连接，这是一种与第三方网站安全共享身份和社交图信息并将第三方数据带回脸书的方式。在很多方面，脸书联通是一个更好的灯塔。

## 贝肯怎么了

按照最初的设想，脸书的信标有点像单向数据可移植性。它从第三方网站吸收数据，但不让任何东西流回。它还只允许来自参与 Beacon 计划的合作伙伴站点的数据流入。由于不向其他网站开放，脸书限制了 Beacon 对用户的用处，因此它看起来更像垃圾邮件，而不是一个有价值的功能。

然而，Beacon 最大的问题是糟糕的隐私控制。Beacon 采取了先拍摄，后提问的方法。最初，不是让用户选择加入 Beacon，而是任何人在浏览一个启用 Beacon 的网站时登录脸书，都会发现他们的信息被自动共享回脸书。用户被要求在下次登录脸书时允许分享信息，但一些人抱怨说警告很容易被忽略。无论如何，要求用户完成额外的步骤来防止私人信息被共享不是一个好的做法。

## 关于连接，什么是正确的

今天在三藩市 F8 开发者大会上发布的脸书连接解决了 Beacon 的两个主要问题。

Connect basiscally 将脸书平台扩展到了社交网络之外，允许第三方网站访问脸书用户和社交图谱数据。让 Connect 联想到 Beacon 的是，这些网站上的用户活动也可以通过新闻反馈反馈到脸书。

> **脸书通官方新闻稿发布特色:**
> 
> *   可信身份验证–使用合作伙伴的脸书帐户轻松地在合作伙伴网站上进行身份验证
> *   真实身份–在可信的环境中通过网络利用他们的真实身份
> *   朋友链接——无论他们去哪里，都可以带着他们的朋友，在网络上的任何地方都可以创建可信的社交环境
> *   动态隐私——保证用户在脸书上设置的隐私设置将跟随他们，无论他们决定在哪里登录网站
> *   社交分发——通过 feeds 与脸书的朋友分享合作伙伴网站上的活动

通过从一开始就引入用户控制的隐私设置，并允许任何网站通过 Connect 接入脸书的用户群，脸书创造了他们应该在去年秋天推出的 Beacon 版本。对第三方来说，结果还是一样的，但用户更倾向于参与进来，因为他们将控制如何以及是否将他们的信息分享回脸书和其他网站。此外，开发者可以选择将脸书的社交体验扩展到脸书以外的网站。

对脸书来说，Connect 与 Beacon 有很大不同，因为它不是一个广告项目——当用户在第三方网站上的活动显示在新闻提要中时，脸书不会赚钱。然而，脸书意识到，越多的人和网站开始依赖他们的 API，他们就越会在我们的网络习惯和体验中根深蒂固，这将使他们更容易赚钱。

脸书连接将于今年秋天向所有用户开放。
