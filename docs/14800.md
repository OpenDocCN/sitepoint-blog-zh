# 网络开发的麦克·OS X

> 原文：<https://www.sitepoint.com/mac-os-x-web-development/>

Macintosh 长期以来一直被典型的网络开发者忽略，因为它被认为是一个利基操作系统和开发平台。在 OS X 之前，麦金塔几乎完全是设计师和艺术家的领地，只是网络开发拼图的另一块。现在情况不再是这样了。

##### 有什么大不了的？

对于任何 Web 开发计划，必须完成几层工作才能构成整个项目。这包括艺术作品、HTML、编程和/或脚本语言、数据库开发，有时还包括多媒体和 XML。那里没有消息。然而，新的是 Mac OS X 平台提供了一种一体化的方法。

苹果公司将设计、开发、编程和测试工具结合在一个系统上。在您的机器上本地运行服务器端应用程序和引擎的能力，以及您的标准 Web 开发工具，允许更快的生产和极其高效的测试。在本文中，我们将探索 OS X 作为 Web 开发平台的灵活性和功能性。在许多情况下，使用 OS X 的决定可以在构思、设计、开发和测试过程中将您的机器需求减少到一个。

##### 核心工具

让我们从 OS X 提供的核心工具开始。

***Web 服务器桌面上的***

全球大多数 Web 服务器运行 Apache 事实上，最新的 Netcraft Web 服务器调查显示这一数字为 64%。许多 Web 开发项目都基于 LAMP 组合——Linux、Apache、MySQL 以及 Perl、PHP 或 Python。其他数据库也可以在这里运行，包括流行的 PostgreSQL。此外，Sun Microsystems 的 Java 2 平台(用于 Java 服务器页面、java servelets 和 java beans)可以在 OS X 上本地运行。这为 Web 开发人员拓宽了范围，使 J2EE Web 应用成为可能。

这些服务器端引擎现在已经存在，或者可以安装在桌面机罩下。由于操作系统的核心运行稍加修改的 FreeBSD 内核，这些传统但先进的 Web 开发工具可以在台式机或笔记本电脑上本地运行。所有对 Web 开发人员来说最流行的管理工具也在这里，包括命令 shell、shell 脚本、CVS、SSH、FTP 和 WebDAV，以及最终的开源邮件服务器 Sendmail。

本质上，使用 OS X，可以从本地主机配置和管理完整的 Web 服务器环境，包括在本地网络上共享 Web 项目，以便在开发过程中进行协作。这消除了对用于开发的单独服务器的需要；开发人员不再需要始终在线的连接来进行审查和测试。一个额外的好处是可以通过命令行或文本编辑器离线访问所有支持这些服务器端引擎的配置文件。

***开发 GUI***

同样重要的是创建 Web 解决方案时必须使用的工具。这些仍然是图形编辑器、HTML 编辑器和专业应用程序(Flash 和多媒体创作等)。).

对于那些在 Adobe 或 Macromedia 等集成开发套件中工作的人来说，OS X 看起来与你熟悉的界面没有什么不同，因为这些应用程序都以相似的方式呈现。对于代码纯粹主义者来说，来自 Bare Bones Software 的 BBEdit 多年来已经证明了自己是一名高级文本和编程编辑。BBEdit 支持复杂的文本操作、HTML 中的语法检查以及用于编程和脚本语言的工具。BBEdit 还提供了对 CVS 的本地支持。

对于铁杆 Web app 程序员来说，Apple 提供了一个全面的面向对象编程应用程序 Project Builder，它可以非常高效地管理高级 Java 和 PHP 项目。

当然，在 Mac 上还有很多其他的网络开发工具；然而，Adobe、BBEdit 和 Macromedia 仍然是设计、布局和开发的标准软件包。与 Macintosh 平台缺乏软件种类的古老神话相反，Mac 产品目录包括 18，000 多种软件。

此外，作为 Unix 的变种，OS X 现在允许用户直接从 OS X 桌面运行流行的开源和专有工具(通常在独立的 Linux 工作站上运行)——包括 X11 应用程序。

***协作、评审和测试***

正如我前面提到的，在本地网络上共享您的本地 Web 服务器就像点击鼠标一样简单。如果您使用宽带专用 IP 地址，或者可以访问动态 DNS 服务，您也可以远程协作，通过互联网共享同一本地 Web 服务器。

在办公环境中，您会发现过去将您的 Mac 加入基于 Windows 或 Unix 的网络环境的问题不再存在。当与非 Mac 平台上的用户协作时，用于多平台网络的内置 OS X 解决方案允许即插即用式的简单性。当登录到您的本地网络时，您甚至可以与目录服务进行交互，例如微软的活动目录服务。

虽然这超出了本文的范围，但值得一提的是，OS X 上的 Microsoft Office 用户可以利用并连接到 Microsoft Exchange 服务器，这通过电子邮件通信、共享帖子和共享日历增加了协作的价值。

在评审和测试过程中，可以体验到将开发工作站和服务器结合在一起的一个主要好处。在修改和故障排除过程中，将文件拖放到 CVS 或直接拖放到 Web 服务器的能力可以节省宝贵的时间。本地“即时”测试动态数据库驱动的页面和服务器端脚本的能力是强大而高效的。

使用以标准为中心的浏览器，如 Safari(构建于基于 Linux 的 KHTML 项目之上)、Opera 和 OS X 上的 Netscape，测试机会显著增加。这些浏览器和传统的 Internet Explorer 在遵守万维网联盟 Internet 标准和测试 DHTML 和 JavaScript 方面取得了很大进展。(微软已经宣布，他们将不再更新超过 5.2.3 的浏览器，即当前版本)。

最后，在通过虚拟 PC 测试 Web 应用程序和项目的跨平台合规性时，您可以在 OS X 桌面上运行 Windows，虚拟 PC 可以运行所有主流 Windows 桌面操作系统。

##### 完整的解决方案

最后，我们发现 OS X 可以作为一个全面的 Web 开发环境，支持从初级网站项目到有许多参与者的复杂 Web 应用程序。上述工具都经过了多年的验证——关键是，在 OS X，它们被整合到一个平台上，以实现易用性和功能性。
