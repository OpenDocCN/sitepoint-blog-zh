# JumpBox 推出虚拟设备即服务

> 原文:[https://www . site point . com/jump box-launchs-virtual-appliances-as-a-service/](https://www.sitepoint.com/jumpbox-launches-virtual-appliances-as-a-service/)

大约 11 个月前，当我第一次听说亚利桑那州虚拟设备提供商 Tempe[jump box](http://www.jumpbox.com/)时，他们大约有 200 名付费客户，提供 10 种设备。从那以后，很多事情都变了。该公司现在拥有超过 1，000 名付费客户，并通过其 JumpBox Open 服务为超过 33 种不同的应用程序提供虚拟设备，该服务在订阅的基础上提供对流行的开源应用程序的设备的访问。据该公司称，已经部署了超过 20，000 个跳箱。

虚拟设备基本上是 web 应用程序的打包版本，可以通过本地网络(或单台机器)下载和轻松部署。通过 JumpBox 虚拟设备部署软件通常有两个好处。1.用户能够在本地网络上使用流行的开源 web 应用程序，如内容管理系统或错误跟踪器，而不必麻烦 IT 部门进行安装；2 .因为它们很容易安装/部署，所以非常适合在投资之前试用软件。

前一个用例使 JumpBox 虚拟设备成为 Trac 或 MediaWiki 等软件的理想选择，即使在本地使用，这些软件也很有用。然而，正是后者促使 JumpBox 推出了“由 JumpBox 提供动力”，该公司今天宣布了这一点。

Powered By JumpBox 是一项面向独立软件供应商(ISV)的服务，让他们能够为自己的产品创建 JumpBox 虚拟设备。这些 ISV 的主要用例是为客户提供他们软件的易于部署的试用版本。JumpBox 的第一个客户是 Six Apart，他们将发布他们流行的 MovableType Pro 博客平台的 JumpBox 虚拟设备。MovableType 的目标是减少由高级安装过程造成的障碍，并允许潜在客户更快地试用该软件。

除了可以部署在许多虚拟化平台上之外，JumpBox 虚拟设备还可以被推送到 Amazon EC2 等云计算环境中，这意味着客户从 web 应用程序的虚拟试用版本到公开部署版本相对来说没有什么痛苦。

除了创建可下载 web 应用程序的虚拟版本，JumpBox 首席运营官公司的肖恩·蒂尔尼告诉我，jump box 还可以用来创建基于云的 web 应用程序的可下载、可本地部署版本。这对 SaaS 提供商来说很有吸引力，他们可以很容易地为关注安全性和可靠性的潜在客户提供在线应用程序的下载版本。
