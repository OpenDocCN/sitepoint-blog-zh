# JavaScript 选集上一个迟来的词

> 原文：<https://www.sitepoint.com/a-belated-word-on-the-javascript-anthology/>

SitePoint 的最新作品[<cite>JavaScript 选集</cite>](https://www.sitepoint.com/blog/) 已经发布了两个多星期，我才意识到我们从未在这个博客上提到过它。

如果你对 JavaScript 是认真的(或者即使你对*变得*认真也是认真的)，这真的是我们迄今为止最甜蜜的头衔。诚然，我编辑了这个东西，所以我有点偏见，但事实上，我喜欢它甚至超过了[我自己的书](https://www.sitepoint.com/books/phpmysql1/)应该说些什么。首先，它是由 DOM 脚本界(née DHTML)的两位重量级人物撰写的:

**詹姆斯·爱德华兹**，更广为人知的名字是[兄弟蛋糕](http://www.brothercake.com/)，是 [UDM 的幕后推手——终极下拉菜单](http://www.udm4.com/)，它不仅是最跨浏览器兼容的层级菜单系统，也是最易访问的，因为它完全符合 W3C 的[网络内容可访问性指南 1.0](https://www.w3.org/TR/WAI-WEBCONTENT/) 。

在他为这本书贡献的章节中，James 涵盖了很多领域:他提供了 JavaScript 的介绍，主要集中在如何以及应该如何在现代最佳实践的 web 开发中使用它，而不是纠缠于诸如变量和`if-else`语句之类的平凡细节；他对框架和弹出窗口进行了非常乐观的处理，这些技术如今受到了不好的指责，但在特定的情况下，它们仍然在开明的开发人员的工具箱中占有一席之地；他重新审视了图像预加载这个“老话题”,并演示了如何使用进度条让用户了解幕后发生的事情；最后，他对 JavaScript 的性能进行了广泛的分析——既包括如何使它在各种浏览器中快速高效地运行(尤其是那些存在内存泄漏等问题的浏览器),也包括如何减少编写和维护代码的时间。

但是到目前为止，詹姆斯对这本书最大的贡献是在他个人的专业领域:构建复杂的界面，如下拉菜单系统，而没有复合的可访问性。在这本书的两个最大的章节中，詹姆斯从头开始构建了许多菜单系统，提供了他令人惊叹的 UDM4 产品的所有光滑和兼容性，然后坚定不移地解决了这些界面带来的可访问性问题，确保它们对于只使用键盘和屏幕阅读器的用户都是完全可访问的。在我看来，光是这份材料就值入场费。

合著者 **Cameron Adams** ，也被称为[的蓝衣男子](http://www.themaninblue.com/)给这本书带来了强烈的设计感，以及将 JavaScript [的能力推向极限的热情](http://www.themaninblue.com/writing/perspective/2005/10/05/)。

在 Cameron 的章节中，他介绍了文档对象模型(DOM)，并展示了如何克服其恼人的局限性；他展示了我所见过的在提交前验证表单的最优雅的方式；他深入研究了 DHTML 的所有内容，包括浏览器兼容性、CSS 的动态修改、事件处理程序和事件侦听器之间的区别，以及 JavaScript 驱动的动画技术；他通过引入 AJAX 请求、富文本编辑、所谓的“dropsheet”对话框和自动完成字段等前沿技术，将 JavaScript 推向了一个新的高度；他还展示了如何通过使用 JavaScript 的面向对象语言特性来管理这些技术产生的大量代码，并且全部都引用了大量的流行文化参考资料。

虽然有点不那么迷人，但处理数字、字符串、数组、cookies 和日期/时间的所有基础知识(和偷偷摸摸的技巧)也包含在这本书里。近 600 页的 JavaScript 选集将向您展示当今最前沿的 JavaScript 技术的全貌。[查看样本章节](https://www.sitepoint.com/blog/)——超过 150 页的免费 JavaScript 精华。
