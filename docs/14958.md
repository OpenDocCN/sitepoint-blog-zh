# Web 应用程序开发——成功指南

> 原文：<https://www.sitepoint.com/development-guide-success/>

**从过去使用`<BLINK>`标签的美好时光，到今天精通 XML 的 Web 服务，应用互联网语言已经走过了漫长的道路。这一点在 Web 应用程序开发领域表现得最为明显。**

随着互联网成为全球经济战线上的主要参与者，对其发展感兴趣的投资者数量也在增加。所以，你可能想知道，互联网是如何继续在通信、媒体和新闻中发挥主要作用的？关键词是:Web 应用项目。

Web 应用程序是通过使用用户、业务和数据服务在 Web 上实现的业务战略和策略。这些工具是未来的所在。在本文中，我将带您了解 Web 应用程序项目生命周期中的基本阶段，解释您有哪些选择，并帮助您为自己成功的 Web 应用程序制定计划。不过，首先让我们对 Web 应用程序做一个简单的概述。

##### 谁需要 Web 应用程序，为什么？

有许多实体需要 Web 应用程序——一个例子是企业对企业的交互。当今世界上的许多公司都要求通过安全的专用网络相互开展业务。这一过程越来越受到许多相互外包项目的海外公司的欢迎。从将资金转入银行帐户的简单过程，到部署大规模的 Web 服务网络以在全球范围内更新价格信息，采用 Web 应用程序基础结构对许多企业来说都是至关重要的。

##### Web 应用程序模型

与许多软件开发模型一样，Web 应用程序模型构建在三层之上:用户服务、业务服务和数据服务。该模型将应用程序分解为一个服务消费者和供应商的网络。

用户服务层为消费者与应用程序交互创建了一个可视化的网关。这包括从基本的 HTML 和 DHTML 到复杂的 COM 组件和 Java 小程序。

然后，用户服务从业务服务中获取业务逻辑和过程。这一层可以从 ASP/PHP/JSP 中的 Web 脚本到服务器端编程，如 TCL、CORBA 和 PERL，允许用户通过 Web 界面执行复杂的操作。

最后一层是数据服务层。数据服务在高层次上存储、检索和更新信息。数据库、文件系统和可写介质都是数据存储和检索设备的例子。然而，对于 Web 应用程序，数据库是最实用的。数据库允许开发者以系统和有组织的方式存储、检索、添加和更新分类信息。

##### 选择正确的项目

选择合适的项目类型是 Web 应用程序开发计划中极其重要的一部分。

评估你的资源、技术技能和出版能力应该是你的首要目标。考虑到这 3 层，设计一个可以明确分配给每层的所有可用资源的列表。

接下来要考虑的应该是成本。你有完成这个项目的预算吗？设计、开发和交付一个完整的项目并取得相当大的成功需要花费你多少钱？这些都是在你签署任何交易或合同之前应该回答的问题。

让我们看一个例子。一家名为 ABC 的公司需要开发一个 Web 应用程序来显示由不同销售代理创建的销售信息。所有 3 个服务层的数据每天都通过完全自动化的流程进行更新。客户告诉您，整个项目必须在 ASP/SQL server 中完成，您也应该托管应用程序。

在评估了您的所有资源后，您和您的团队得出结论，该公司无法每天进行数据备份。经过进一步的讨论，你意识到这对你的客户来说是一个非常重要的部分，你不应该在这个项目上冒险。下一次，当一个类似的项目摆在你的办公桌上时，你很可能会准备得更充分，所以你拒绝了这份工作，并推荐了其他有能力马上做这件事的人。

##### Web 应用程序项目中的各个阶段

Web 应用程序开发过程有 4 个阶段:

1.  预见项目的性质和方向
2.  设计计划
3.  发展
4.  测试、支持和稳定性

让我们更详细地看一下每一项。

***1。展望项目的性质和方向*T3**

在这个阶段，被分配到项目的管理人员和开发人员走到一起，建立解决方案必须实现的目标。这包括认识到应用程序的项目、时间安排和版本控制的局限性。在这个阶段结束时，应该有关于应用程序将实现什么的清晰文档。

***2。制定计划***

在这个阶段，您和您的团队必须确定应用程序“如何”运行。

什么脚本语言最合适，必须包含哪些特性，需要多长时间？这些是在规划阶段必须回答的一些问题。在这一点上，主要的切线是项目计划和功能规范。项目计划确定了事件和任务的时间框架，而功能规范详细描述了应用程序将如何运行和流动。

***3。发展*T3**

一旦项目计划和功能规范准备就绪，就为开始开发工作设定了基线。程序员或 Web 开发人员开始编码、测试和发布数据。这个阶段建立数据变量、实体和编码程序，它们将在项目的剩余部分使用。开发团队准备了一份里程碑文档，然后提交给管理层进行审查。

***4。测试、支撑和稳定性***

应用程序项目的稳定性阶段主要侧重于测试和消除可能导致应用程序失败的错误、差异和网络问题。正是在这里，为一个成功的支持系统建立了政策和程序。

##### 了解你的选择并明智地使用它们

好了，现在您已经了解了 Web 应用程序开发背后的架构和过程，让我们来看看您需要为开发过程本身考虑哪些技术选项。

***Windows 网络服务器***

微软在一个重要因素上建立了忠实的客户基础——他们易于使用的软件。Windows NT/2000/XP Web 服务器速度非常快，而且易于管理。操作系统是 Windows shell 的事实意味着管理员和作者可以很容易地允许 Web 服务器与其他软件和硬件应用程序进行交互，以便通过 Internet 发送和接收数据。用于 Windows 服务器的流行的服务器端脚本语言是 ASP/ASP。net、Java 服务器页面和 PHP。

***UNIX/Linux 网络服务器***

UNIX 一直以其可靠性著称。它是一个强大而健壮的网络服务器和操作系统。Unix 是许多需要内容管理系统或接收极高流量的大型网站的首选服务器。流行的 UNIX 服务器端脚本语言有 Java Server Pages、PERL、PHP 和 CORBA

每种脚本语言都有其优点和缺点。由于我不是在这里写书，我将使用 ASP 模型作为我的说明语言。当使用 Windows 服务器时，开发人员需要考虑几个重要的参数，包括安全性、可伸缩性、速度和应用程序设计。所以下面我将帮助你制定一个成功的计划来完成各种网络项目。

##### 规划一个成功的 Web 开发项目

为了极大地降低项目失败的风险，我总是按照下面的顺序处理我的应用程序开发项目。

***1。识别业务逻辑和实体***

从收集你所有的信息开始。如果您要使用数据库，首先要列举业务逻辑中将使用多少个实体。例如，如果您的程序实现销售数据，销售票将是一个实体。

一旦你确定了所有的实体，就为它们的关系建立一个清晰的指导方针。这可以通过演示、流程图甚至报告来完成。

***2。创建功能规范和项目计划***

这部分，在我看来，是这个项目最重要的部分。功能规范(或功能规格)是您希望特定 Web 应用程序如何外观和工作的地图或蓝图。规范详细说明了成品将做什么，用户交互，以及它的外观和感觉。

编写功能规范的一个好处是它简化了开发过程。它消除了编程过程中的差异和猜测，因为计划中的详细程度可以最大限度地减少通常与项目灾难相关的误解。参见 RayComm.com 公司精心编写的功能规范示例。

一旦完成了功能规范，就必须设计项目计划。项目计划是项目期间将要发生的任务和事件的时间表。项目或项目群经理通常是创建项目计划的人，他们的主要任务是细化任务注释，同时能够适应日程安排和资源信息。您可以在[Method123.com](http://www.method123.com/documents/p_plan.xls)下载项目计划的 Excel 文件样本。

***3。发挥应用模型***

如前所述，应用程序模型由三层组成——用户层、业务层和数据服务层，每一层都有重要的用途。

实际上，从数据层开始总是最好的，因为您已经确定了您的实体并理解了它们的关系。数据层可以是 SQL server 数据库、文本文件，甚至是功能强大的 Oracle。根据您选择的平台创建表、关系、作业和过程。如果数据是一个仓库(即数据已经存在并且不依赖于实时交互)，那么要确保新的和额外的数据能够以可扩展的方式安全地添加。

快速提示:[在 SQL server/Oracle 中使用视图](http://asia.cnet.com/builder/architect/db/0,39009328,39072342,00.htm)可以显著提高应用程序的生产率和性能。它们提高了速度，因为它们是没有物理存在的“存储查询”。

在我看来，业务服务层是应用程序的核心。它包括将业务逻辑实现到脚本或编程语言中。

在这个阶段，确保您已经为测试和调试设置了环境。在你的应用程序中，至少要测试两个实例，毕竟，对你来说很好的方法，在其他平台或机器上可能不太好。ASP、XML、PHP、JSP 和 CGI 是在业务服务级别使用的服务器端脚本语言的一些例子。无论您选择哪种语言，都要确保它能够处理功能规范中提出的所有业务逻辑。

最后是用户层，这对于应用程序中的交互和战略元素来说是绝对重要的。它通过将图像、图标、图形和布局元素放置在感兴趣的战略区域(通常基于管理研究),为用户提供了通向业务服务的可视化网关。如果你要自己开发用户层，一定要研究你的竞争对手。你最不需要的就是你的应用程序看起来和别人的一模一样。

***4。制定支持方案***

能够支持和稳定您的应用程序非常重要。为故障、事故甚至停工情况定义一个程序呼叫。让您的客户能够在出现与计划相关的紧急情况时联系您。

支持方案的一个很好的例子是票据跟踪系统。该系统允许用户归档与支持请求和支持团队相关的案例，然后使案例可跟踪。这意味着该请求可通过唯一的代码或号码来识别。虽然票务跟踪系统通常由托管公司或大型 ASP(应用服务提供商)使用，但它们仍然有助于保持应用程序的稳定性。

##### 交给你了…

现在你有了它——一个框架，你可以从它开始计划和开发你自己的成功的 Web 应用程序。

Web 应用程序将会存在很长一段时间。随着我们进一步走向未来，它们将变得更少人工操作，更多自动化。这将最终导致新类型的研究，但目前，我们可以高兴地看到，正是这一点推动了网络。
