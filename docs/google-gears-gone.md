# 谷歌 Gears 走了，走了，走了？

> 原文：<https://www.sitepoint.com/google-gears-gone/>

过去一两周，网络上充斥着关于谷歌 Gears 的谣言。似乎谷歌已经悄悄地放弃了该系统的开发。

Google Gears 于 2007 年 5 月作为浏览器插件发布。它提供了许多主要用于离线功能的有用工具:

*   脱机时缓存和服务应用程序的本地服务器
*   用于本地存储的 SQLite 数据库模块
*   一个工作池模块，允许类似 JavaScript 线程的异步处理
*   用于操作系统集成的桌面模块，例如创建快捷方式
*   用于创建位置感知应用程序的地理定位模块。

很棒的东西…但是一些问题已经浮出水面:

*   最近出现的版本是 2008 年 11 月的 0.5。谷歌项目的发布进度如此缓慢是很罕见的。
*   谷歌 Chrome 浏览器的 Mac 版本不支持 Gears。
*   采用齿轮的产品相对较少。

然而，取得稳步进展的最大原因是 html 5——新标准提供了许多由 Google Gears 引入的功能。谷歌工程总监莱纳斯·厄普森(Linus Upson)表示，Gears 的时代来了又去。出于兼容性原因，他们将继续支持该技术，但系统不会更新或改进:

> 我们非常专注于推动 HTML 5 向前发展，这也是我们投入全部精力的地方。Gears 很好地完成了它的使命，将这些功能融入到 HTML 5 中。
> 
> 设计 Gears 的团队也帮助设计了这些 API 的 HTML 5 版本。你几乎可以认为 HTML 5 中的内容，包括应用缓存、数据库和那些东西，本质上是 Gears 版本 2，这就是我们对它的看法。
> 
> 我们希望所有的应用程序都使用基于标准的 HTML 5 APIs，而不是专注于 Gears 的发展。现在我们是一个浏览器供应商，我们可以帮助 HTML 5 向前发展，不是作为一个插件，而是作为 Chrome 的一部分。

我认为这是一个明智的决定。当基于标准的替代方案可用时，很少有 web 开发人员会依赖第三方插件。然而，HTML5 的支持仍然不完整，也不能保证微软会在 IE 中采用它。web 应用程序中的离线功能最多还需要几年的时间。

谷歌降速是对的吗？您是否开发了使用该插件的应用程序？你相信浏览器厂商会在最早的时机完全实现 HTML5 吗？
