# XMLHTTP 本地化，以及其他 IE7 优点

> 原文：<https://www.sitepoint.com/xmlhttp-to-go-native-and-other-ie7-goodness/>

AJAX 开发者们，欢呼吧！微软[正式宣布【XMLHttpRequest 将成为 IE7 中的原生对象，这意味着即使浏览器出于安全原因禁用了 ActiveX 支持，它也能工作。这使得浏览器对远程脚本(AJAX 应用程序的核心技术)的支持达到了与 Firefox、Safari、Opera 和类似浏览器相同的标准。](http://blogs.msdn.com/ie/archive/2005/09/13/465338.aspx)

一旦 IE7 被广泛采用，这将有助于消除大量繁琐的浏览器检测代码，目前在所有流行的浏览器上实现 AJAX 都需要这些代码。

另一个同样受欢迎的声明是，select 元素(用于创建菜单和下拉列表)将是 IE7 中的一个轻量级组件，这意味着 HTML 元素(如 DHTML 弹出菜单)将能够覆盖这些菜单和列表，并在它们覆盖时正确地隐藏它们。目前在 IE6 和更早的版本中，菜单和下拉列表总是出现在所有其他 HTML 内容的顶部，除非你使用一些相当丑陋的方法来解决这个问题。

最后，火狐浏览器的 Web 开发者工具栏的粉丝们会很高兴听到微软即将发布他们自己的 IE6 或更高版本的 Web 开发者工具栏的测试版。看好这个空间！
