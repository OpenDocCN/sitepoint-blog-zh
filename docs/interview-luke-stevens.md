# 卢克·史蒂文斯谈数据驱动设计

> 原文：<https://www.sitepoint.com/interview-luke-stevens/>

卢克·斯蒂文斯是一名自由职业者，住在悉尼，在美国、英国和澳大利亚都有客户。卢克将在下个月的网络方向南方会议上发表演讲，主题是[数据驱动设计](http://south09.webdirections.org/program/design#data-driven-design)。SitePoint 的 Matthew Magain 就这个话题采访了他。

首先，你能向 SitePoint 的读者介绍一下你自己吗？

当然可以。我是一个普通的自由网页设计师，毫无疑问和许多 SitePoint 的读者一样。我已经做了三年半的自由职业者，在那之前我在公司工作了五年多。我主要设计和建立内容丰富的表达引擎网站。你听到的所有关于 ExpressionEngine 的好东西都是真的——它太棒了。

然而，最近我一直致力于一些想法，我希望这些想法能改变其他设计师和网络专业人士对网络设计的看法；这就是我将在 10 月份的 [Web Directions South](http://south09.webdirections.org/) 上谈论的数据驱动的东西。从设计师的角度来看，网络分析、A/B 测试等等。

我也在写一本相同主题的书，叫做[](http://lukestevensdesign.com/book/)*。已经写了大约百分之八十，我希望在接下来的几个月里发布它。*

 *你在今年的 Web Directions South 上就这个主题发表了演讲，也就是众所周知的数据驱动设计。“基于性能的设计”是什么意思？你是在说一边浏览 Photoshop，一边吃能量饮料和咖啡因药片吗？

不完全是！“数据驱动”*听起来*极其无聊。你知道，数学和统计，电子表格和 zzzz …

然而，真正重要的是性能。这样性感多了。运动员表演。他们还关注数据:如果他们以某种方式训练，或者采用不同的技术，会提高他们的成绩吗？他们能跑得更快或扔得更远吗？

如果他们想变得更好，他们不会随便做些事情。他们不断试验，不断优化自己的表现，使自己成为最好的。

就网络设计而言，性能取决于用户的行为。他们是购买、点击、转化、阅读、互动、注册、逗留，还是感到沮丧而离开？

我希望设计师和开发人员开始像运动员一样思考他们的网站，并开始使用数据来优化这种用户性能:使用数据来改善用户的生活，并解决他们面临的任何障碍。不是以某种蹩脚的、善意的、感觉良好的方式，而是以一种可测量的、可论证的、真实的、数据驱动的方式。

很长一段时间以来，我们只是假设我们作为设计师所认为的对用户来说是最好的。然而，有时这些假设可能是错误的，而且代价巨大。UIE 的 Jared Spool 有一个很棒的故事，讲述了设计师的错误假设是如何让一家大型零售商每年损失 3 亿美元的。

然而，这不仅仅是清除路障。YouTube 最近对他们的主页采取了数据驱动的方法。[他们运行了一个多变量测试，有 1024 个一些小变化和改进注册 15%](http://youtube-global.blogspot.com/2009/08/look-inside-1024-recipe-multivariate.html) 的组合，这令人难以置信。

在过去的十年里，设计师们主要关注于建立基于标准的网站。您认为您的基于性能的方法在哪些方面符合 web 标准运动？

我认为这是网页设计超越网页标准运动的自然演变。我们有基于标准的设计，我认为——或者至少希望——我们将转向基于性能的设计作为规范。

不要误解我——网络标准是绝对重要的，就像制定规范、倡导标准、教育设计师和让浏览器制造商负起责任所做的工作一样。在这方面，作为网络专业人士，我们都站在巨人的肩膀上。

然而，现在我们已经很大程度上弄清楚了网络标准是“如何”实现的。网络开始成熟，下一个合乎逻辑的步骤是*我们如何做某事……*什么表现最好*？*

 *这是我希望设计师能够想到的问题，因为它是基于性能的设计的基础。

网络是人类历史上最可测量和测试的设计形式，所以我认为作为设计师，我们不可避免地会在未来更加关注数据和性能。

在你演讲的简介中，你提到“数据并不意味着更少的创造力和实验，而是意味着更多。”你这话是什么意思？

当你开始谈论数字、度量和数据时，听起来就像是创造力的对立面。突然间，你变成了一个拥有电子表格的科学家，而不是一个运用想象力和直觉的设计师。

但事实并非如此。首先，网页设计者已经使用数据来指导他们的设计；屏幕分辨率和浏览器版本是网页设计师长期以来使用的数据点的两个明显例子。

其次，也是最重要的一点，当你采用数据驱动的方法，并可以衡量你的设计如何为用户执行时，你可以尝试一系列的设计想法——无论这些想法有多棒——否则它们就会作为封存的 PSD 留在你的硬盘上。

除了尝试你可能已经开发的想法，你现在知道会有绩效收益(这通常会转化为你的客户或老板的实际现金，最终是你自己)。因此，它推动你发挥创造力，想出你可能从未考虑过的新设计变化。

伟大的设计来自创造性地推动自己，数据驱动的设计让你做到这一点。然后你可以把这些想法推给访问者，他们使用这些想法并投票选出他们最喜欢的(也就是说，通过转换更多，或点击更多，或停留更长时间，等等)。

通过让用户测试你的想法，你可以进行更多的实验，承担更大的创新风险，因为数据是你的安全网。它告诉你什么可行，什么不可行。没有这个安全网，我们往往会更加保守，安于现状。我们不会尝试那些可能失败或者可能取得惊人成功的绝妙想法。但是通过数据驱动的方法，我们可以做到。

因此，数据驱动的方法不仅不会扼杀创造力，还能让你更加自由地进行实验。

这听起来很棒，但是网页设计师如何才能做到呢？

A/B 和多元测试是检验这些想法的方法，特别是使用[谷歌网站优化器](http://www.google.com/websiteoptimizer)。像 GWO 这样的工具被定位为“由营销人员使用，为营销人员服务”，通常与优化营销活动有关，特别是登录页面。然而，作为设计师，我们确实需要充分利用这些工具来满足我们自己的需求，这也是我将在 Web Directions 上讨论的内容。

也就是说，我强烈建议每个人都去谷歌网站优化器，熟悉这个工具，并自己尝试一下。真的很简单。谷歌让非技术人员很容易开始使用 GWO，有大量的帮助提示和支持文档，所以我相信 SitePoint 的读者不会有任何问题。

另一个令人感兴趣的工具是开源的 A/B 和多变量测试工具， [genetify](http://wiki.github.com/gregdingle/genetify) ，它采用了一种不同于 GWO 的方法来测试网页。

然后，一旦你开始测试，请博客的结果！作为一个网页设计社区，我们几乎没有什么数据可以证明什么是真的，什么是假的；所以我希望看到更多的设计师做测试，并公布结果供我们大家学习。

通过写博客和讨论，我们找到了基于标准的设计，我希望在基于性能的设计方面，我们也能这样做。

道格·鲍曼(Doug Bowman)曾引用他的前雇主谷歌(Google)测试“41 种蓝色调”的名言，作为他离开谷歌的原因之一。测试和信任设计师的直觉或经验之间的界限在哪里？

我在为数据驱动的设计 辩护的文章 [*中提出了“41 种蓝色阴影”的问题。*](http://design2-0.com/articles/in-defense-of-data-driven-design/)*[道格·鲍曼非常友好地在推特上链接了这篇文章](http://twitter.com/stop/statuses/1764814233)，说他同意我写的几乎所有内容，这很好。*

为了解决这个问题，相信你作为设计师的直觉和依靠数据之间的界限是非常有趣的。你不能测试所有的东西，如果你尝试了，你会陷入道格发现自己所处的荒谬境地。然而，如果你仅仅依靠你的直觉，你可能会放弃性能提升，就像前面提到的 3 亿美元的问题。

怎么办？

首先，你可能根本不需要做太多的改变。通常在设计过程中，大多数设计师会对一个主题提出几个不同的选择或变化。不要像我们现在这样抛弃这些想法，它们可以被进一步发展和测试。如果你的客户或老板对最终的设计有特别不同的想法，这很好。通过制定几个选项(一个是你喜欢的，一个是他们喜欢的)，你可以把你的钱用在你的嘴上，看看客观上什么是最好的执行者。

除此之外，还有一些经验法则需要遵循:

*   密切了解您的 web 分析数据，以便您可以发现问题和提高性能的机会。
*   测试真正的变化以获得真正的改进——不要浪费时间担心无关紧要的设计细节，要大胆！
*   一些最好的数据是定性数据；也就是来自用户的反馈，因为沮丧的用户不会留下太多的网络分析线索，所以一定要问。
*   测试重要的东西。你不必测试完全不同的设计。测试标题能带来如此大的不同，这是值得注意的；例如，通过测试不同的标题，37signals 将注册人数提高了 30%。
*   如果网站有一个商业目标(即达到一定数量的页面浏览量、转换率、注册人数或电子邮件查询)，你必须进行测试。就这么简单，所以开始尝试吧！
