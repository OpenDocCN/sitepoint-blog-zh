# 280 North 宣布 Atlas: Web App 设计永远改变了？

> 原文:[https://www . site point . com/280-north-announcements-atlas-web-app-design-changed-forever/](https://www.sitepoint.com/280-north-announces-atlas-web-app-design-changed-forever/)

今天在佛罗里达州迈阿密结束的[未来网络应用](https://events.carsonified.com/fowa/2009/miami)大会上，最令人瞠目结舌的技术演示奖(如果有这样一个奖项的话)将很容易颁给 Atlas。Atlas 演示不仅获得了众多热烈的掌声，而且当它在另一位演示者的演讲中被提及时，它甚至赢得了额外的一轮掌声，并且是会议主持人 Ryan Carson 和 Jason Fried 以及 Joel Spolsky 之间坐下来采访的讨论话题。“我觉得 web 应用程序设计发生了变化，”在 Atlas 会议结束后，敬畏的卡森回到舞台上说。

什么是阿特拉斯？Atlas 是一个新产品，预计将于明年夏天(北半球)首次亮相，来自北 280 号(T1)的人们，它将极大地简化设计过程，降低在 Objective-J 和 [Cappuccino](https://cappuccino.org/) 框架中创建应用程序的门槛。从本质上来说，Atlas 是一个界面构建器，允许设计师在不接触任何代码的情况下为 Cappuccino 应用程序(当然 Atlas 就是这样)创建功能性 ui。

280 North 的联合创始人弗朗西斯科·托马斯基在演示中说，设计不应该是写文字。设计师不需要知道如何编写 CSS 或 JavaScript 来让他们的 web 应用程序界面看起来和表现出他们想要的样子。Atlas 将设计与代码完全分离，允许设计人员创建丰富的 web 应用程序界面，这些界面的外观和行为符合他们的预期，甚至可以与应用程序的业务逻辑挂钩，而无需接触任何代码。

为此，Atlas 感觉很像 Adobe 的 Flash Catalyst([our coverage](https://www.sitepoint.com/adobe-debuts-flash-catalyst-at-max-conference/))或微软的 Expression Studio Blend。这三个应用程序之间有许多重要的差异，但是它们有一个共同的目标，即允许设计人员和非编码人员创建富互联网应用程序用户界面，而不需要进行任何实际的编码。

Tolmasky 用 Atlas 在不到 10 分钟的时间里构建了一个非常像桌面的 RSS 阅读器(功能齐全，尽管相当简单)。这个过程只需要将设计元素放到画布上，定义它们的行为，然后将它们与代码组件连接起来。这种方法完全是可视化的，结果完全令人震惊。

更令人印象深刻的是，Atlas 包括多平台支持——这意味着它可以用来为其他平台(如 iPhone)创建应用程序，就像它可以轻松地为标准网络浏览器创建应用程序一样，所有这些都使用相同的后端。因为 Cappuccino 是在 web 浏览器中编译的客户端，它是后端不可知的，所以应用程序的业务逻辑可以是 Ruby、PHP、Python 或任何你想要的语言。

Atlas 是非常令人兴奋的，它显然会对 web 应用程序设计以及 Cappuccino 框架和 Objective-J 语言的采用产生深远的影响。下面嵌入的视频是从 Atlas 网站上复制的托尔马斯基的 FOWA 演示。根据 Tolmasky 的说法，Atlas 的发布还有大约 5 或 6 个月的时间，但在此之前可能会进行封闭测试，开发者可以在 Atlas 网站上注册电子邮件更新。
