# ASP.NET 问题专家迪诺·埃斯波西托访谈

> 原文:[https://www.sitepoint.com/dino-esposito-asp-net-expert/](https://www.sitepoint.com/dino-esposito-asp-net-expert/)

迪诺·埃斯波西托警告说:“不要让 2.0 忽视了你。在 from.NET 开发者的热切期待下，ASP.NET 2.0 于去年 11 月发布。测试版真的让我们很开心，我们渴望得到 2.0 中承诺的一些新特性:母版页、数据源控件、提供者模型以及会话和缓存增强。

既然开发人员已经有机会使用一些工具，那么现在是从该领域的专家那里获得一些关于发布的反馈的好时机。迪诺·埃斯波西托是意大利的一名教练兼顾问，专门研究 ASP.NET 和 ADO.NET。他为《MSDN 杂志》撰写专栏，并撰写了大量书籍，其中两本是关于 ASP.NET 2.0 的。

##### 迁移到 ASP.NET 2.0

最近，我一直在考虑将我的一些 1.1 应用程序升级到 2.0。这是你会推荐的吗？我的应用程序在 1.1 中运行良好。升级对我有什么好处？

在 2.0 运行时环境中运行 1.1 二进制文件可以给你带来一些性能上的好处。例如，在 ASP.NET，只需切换引擎，就可以将视图状态的大小缩小 50%左右。然而，重写和升级可以工作并且不需要新特性的应用程序对我来说毫无意义。我的建议是等待，但是升级你的开发者。不要让 2.0 忽视了你。如果以后升级，您需要查看的内容数量可能会变得难以管理。

*很多。NET 开发人员已经习惯了代码隐藏的思想。现在，在 2.0 中，我们引入了代码旁置模型。哪个更好？内部代码呢？*

这是一个“无事生非”的例子。将您的代码放在一个单独的文件中会更容易管理。如果你没有像 Visual Studio 这样的工具在源文件中插入自动生成的代码，这个特性非常棒。但总的来说，ASP.NET 还有其他值得关注的重要进步。

*你怎么看待 VB 和经典 ASP 的未来？既然有这么多的人转而使用这些语言，那么这些语言存在下去的可能性大吗？网？*

这是进步。我明白安装应用程序的巨大基数，但是。NET 是一大进步，它已经存在了五年多了。现在，在软件开发这样一个大约 30 年前诞生的领域，5 年是一个地质时代。规划移植是最好的办法。不管你喜不喜欢，这纯粹是生意。

##### 脚本、标准和服务

AJAX 似乎正在席卷编程世界。NET 程序员不想被冷落。随着 AJAX.NET 图书馆和阿特拉斯图书馆的出现，你对图书馆的未来有什么看法？NET 和 AJAX？似乎每个人都在寻找一种在应用程序中使用 AJAX 的方法，不管是否需要。AJAX 会改变我们开发应用程序的方式吗？

事实上，我们正在经历一场范式转变。web 应用程序的工作方式正在我们眼前发生变化。从通过请求和替换页面来工作的应用程序，我们正在转向通过请求数据和更新当前页面来工作的应用程序。这是 AJAX 的生命周期。奇怪的是，现在每个人都在说 AJAX 将彻底改变 Web，但是 AJAX 的构建块可以追溯到七八年前。更令人惊奇的是，AJAX 的核心是一个组件——XMLHttpRequest——它是为 Outlook Web Access 创建的，并于 1999 年首次与 IE 5.0 集成在一起。同样值得注意的是，Mozilla 浏览器目前比 IE6 更好地实现了这个组件。(IE7 的事情会有所改变。科学(计算机科学也不例外)具有累积性，每个人都建立在他人之前所做的基础上。AJAX 无疑正在改变我们开发应用程序的方式，尽管大多数开发人员可能还需要几年时间才能完全理解它。这是让 Atlas 扎根并扩展今天的 ASP.NET 模式所需要的时间框架。

对 Visual Studio 和 ASP.NET 最大的抱怨之一是它制作的 HTML 太糟糕了。ASP.NET 可以制作优质的 HTML 你认为微软对网络标准有多认真？

我认为他们从 2.0 开始认真对待它，特别是它对 XHTML 的支持。然而，由于现有的代码，一些限制仍然存在。ASP.NET 2.0 的某些部分产生了与 XHTML 不兼容的代码，并且很难更新。就 Visual Studio 而言，2003 版重写了开发者的标记；幸运的是，这种做法在 2005 年的版本中结束了。

你已经写了几篇关于 web 服务和. NET 的文章。安全性和 Web 服务在 2.0 中的表现如何？

Web 服务和。NET 是不同的实体。Web 服务变得如此流行是因为。NET，因为。NET Framework 使得创建 web 服务变得如此简单。然而网络服务与。NET——至少在概念上是这样。因此，要解决 web 服务的安全性问题，我们不必看。NET:我们需要了解行业标准，以及公司如何开发软件来实现这些标准。从这个角度来看，我觉得 Windows Communication Foundation(WCF，原名 Indigo)是一个更好的目标。

##### ADO.NET、XAML 和开源

你是一个 ADO.NET 和 SQL 的家伙。ADO.NET 2.0 有哪些改进？

与该平台的其他部分相比，并没有太多的改进，而且已经做出的改进大多是不可见的，因为大多数改进都是为了提高性能。DataSet 和 DataTable 是团队最关注的类。DataTable 类实现了新的接口，使它能够像数据集一样处理序列化和 web 服务。最重要的是，这两个类的序列化算法已经修复，可以提供真正的二进制(与 XML 相反)序列化。对于移动大量数据的大型企业应用程序来说，这是一个关键问题。还有更多的改进，但这是最先引起我注意的。

对微软的可扩展应用程序标记语言(XAML)越来越感兴趣，你有什么想法？它的未来如何？

我不知道。目前，我对 WinFX 的兴趣仅限于使用 XAML 变体的工作流应用程序。无需进入太多的细节，XAML 下的模型(标记+代码隐藏文件)是现代应用程序的强大模型。

你对 ASP.NET 开源项目有什么想法，更具体地说，你有很多或任何开发 Mono 应用程序的经验吗？

我不是纯开源软件的忠实粉丝。我更喜欢有一个健壮可靠的平台，从头开始为可扩展性而设计，并对它进行完整的文档化，使编写扩展变得相当容易。我也喜欢软件背后有一家公司，关注用户在做什么，以及对它有什么看法。这或多或少是微软对 Office Live 的想法。很高兴看到这种模式将如何被接受。

##### 迪诺的作品

你有一本新书——《微软 ASP.NET 2.0 应用程序编程:高级主题【T2》——将于三月出版。它涉及到新框架中的一些高级主题，例如编译模型。你认为 ASP.NET 2.0 最好的新特性是什么？有哪些地方可以做得更好？

实际上，你提到的这本书是 5 月份开始的一个更大项目的第二部分。我发现写一本 1500 页的书太难了，不仅仅是对我来说。作为作者，这对于我来说很难处理，对于编辑和审稿人来说也是如此。最后，对于那些试图快速浏览页面有困难的读者来说，这在身体上很难处理。

因此，我和出版商微软出版社决定将原来的目录分成两本书。第一卷，*编程微软 ASP.NET 2.0:核心参考*，随着 2.0 框架的正式发布于 11 月出版。它涵盖了核心主题—基础知识。第二册是你提到的那一册，它涵盖了更高级的主题。我将 TOC 分成两个部分，查看每个主题可能的优先级。开发人员合理地想要或需要学习的主题首先在第一本书中找到；其余的都在三月份出版的书中。另一方面，有经验的开发人员以及已经通过 ASP.NET 2.0 完成大部分工作的开发人员(例如，参与测试程序并安装大量早期版本)，可能会直接阅读第二本书，该书涵盖了在现有文献和文档中很难找到的内容的细节。

我特别喜欢这样一个事实，即 ASP.NET 已经在内部进行了深度重构，以公开以前无法访问的可定制功能。提供者模型是构建许多特性(如安全性、成员资格、用户配置和会话状态)的通用模式。但是我看到这个模型也扩展到了编译机制。总之，我喜欢 ASP.NET 2.0 提供的可扩展性和定制化水平。很难找到可以改进的功能，即使 ASP.NET 2.0——像世界上其他任何东西一样——并不完美。我发现跟踪能力低于框架其余部分的水平。此外，服务器控件应该提供更丰富的脚本功能。但这些是 Atlas 接下来要做的改进。

##### 编程；编排

在你所有的文章和例子中，你都使用了恰当的面向对象编程技术。你一直是一个面向对象的程序员吗？或者你开始更多的是一个脚本编写者？

事实上，C++是我的第一语言。我是少数几个首先学习 C++和 OOP，然后继续学习 C 和 C 风格编程的人之一。用了这么多年的 C，然后是脚本语言，我已经失去了最先进的 OOP 技术，但用 C#我正试图赶上。

我一直是极限编程的粉丝，但我对我使用的方法很灵活。有你喜欢的编程方法吗？

我喜欢极限编程和敏捷开发的大多数方面，但是我通常不把方法论当作一种信仰。当我第一次了解敏捷原则时，我想，“哇，我一直都很敏捷——我只是不知道而已。”没有什么方法是我真正喜欢的，但是在我看来，任何看起来聪明的方法都是受欢迎和被采纳的。

测试驱动的开发已经存在了一段时间，但是人们对它的兴趣似乎还在继续增长。你对此有什么看法？有测试框架吗？你用的网？

我不怎么写真实世界的代码。但是自动化测试是关键。句号。

##### 变得个人化

既然我已经了解了你一点，我想更进一步了解你。你已经在事业上取得了这么多成就。你仍在努力实现哪些目标？

保持现在的水平就太好了。展望未来，我不做太多的咨询，我希望增加咨询，更多地站在第一线。可以说，我的确有一个梦想，并想把它变成一个目标:去南美、南非、澳大利亚、中国和远东进行一次环球旅行。

你的山羊胡让你得到了女人吗？

这仅仅是一个尝试，以更新外观…也许与秘密和绝对不承认的希望得到，嗯，更多的小鸡。但是不要告诉我的妻子…

作为一名. NET 开发者，我不得不说你是我钦佩和尊敬的人。你崇拜谁？

这么多人，我很荣幸的说，大部分都是朋友。一个名字？嗯……杰夫·里克特，他是一个好人，也是一个非常专业的人。我说杰夫是因为他间接地向我证明了看似不可能的事情有时会发生。他不止一次证明了这一点。

你喜欢做什么？你远足、旅行、写作、玩游戏等吗？？把货给我们！

这么多年来，我一直在工作，工作，甚至更多的工作。然后，去年春天，我 40 岁了，重新发现了健身的重要性。现在，我可以认真地说，“当不跑步或打网球时，Dino 用. NET 做了一些有益的工作，”我也有一个家庭，还有两个孩子——Francesco 和 Michela。我的几本书的读者都知道他们，看着他们在封底我们的经典画面中成长。

从个人或职业角度来看，你还有什么想分享的吗？

是的。这是一个简单的规则，却被所有时代的绝对天才之一——阿尔伯特·爱因斯坦——广为流传。当人们让我签名时，我通常会把这个想法写下来。保持简单。它几乎在任何地方都有效，让你更快乐。
