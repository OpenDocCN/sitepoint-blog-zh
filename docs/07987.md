路易斯:大家好，欢迎来到 SitePoint 播客的另一集。今天和我一起参加节目的是沃尔夫·贝瓦尔。你好，狼。

Louis: Wolf 是 HotGloo 公司的首席运营官，hot gloo 是一种在线互动原型制作工具。对于任何一个建立网站或软件的人来说，在真正开始设计之前，先勾勒出进入每个页面的交互类型。

我想请你谈谈 UX 设计、线框和原型在网页设计中的作用会很棒，因为这不是我们在节目中经常涉及的东西。我最近才开始使用 HotGloo，我真的很喜欢这个工具，所以我想知道谁能更好地谈论这个东西？

狼:嗯，非常感谢你邀请我。三年前，也就是 2009 年，当我们开始的时候，我们看到了对一种工具的需求，就像我们当时建立的那样。汉尼斯和我，汉尼斯是首席执行官和首席技术官，我们在一家互动媒体机构工作。他是那里的开发人员，我做项目管理，我做概念的东西。

我们意识到我们的很多工作流程都是基于——我们要去见客户。我们写下了规格。我们有我们的布局基本上是什么客户的想法是他未来的网站。

当时的情况是，实际上设计师会直接跳进去，启动 Photoshop，然后马上开始设计。那时候，有这样一种想法流过，然后我们跳出来说，“嗯，这不是正确的方法。我们实际上应该首先考虑整个站点结构，信息架构。然后，在所有这些流程都已规划好并正在与客户讨论之后，我们实际上应该开始着手设计。”

这是当时 HotGloo 的基本理念。当我们开始时，对我们来说非常重要的是，这整个过程实际上是一个真正的协作过程。这并不是说，在一个机构的 45 人团队中，每个人都独立工作，一个人只做设计，另一个人只做开发。

当然，在一个团队中有这些专门的角色，但对我们来说，团队中的每个人都在同一页上真的很重要。然后，当我们开始时，实际上开始做线框的人会让设计者、开发者，你知道，肯定每隔一天查看项目并带来他的想法和分享想法。

在项目的早期，团队中的每个人都知道项目将走向何方。你会让客户再来一次。你会给他们看，第一张草图，第一个线框。他可以自己进去。他甚至可以在任何东西被设计或开发之前体验他的站点。

所以，我们意识到，这整个过程，首先，对我们来说是一个迭代过程。结局越来越好。对于客户来说，这更容易，因为他从第一天起就在船上，你实际上可以通过一点一滴和一小步向他展示他的工作进展。在那之前，他会过来，开一个简报会，两三个星期后，他回来，我们会给他展示设计文件。然后，会有关于颜色之类的大争论。

**路易:**对。

**Wolf:** 首先，更快地完成项目，每个人都在同一页上。这整个迭代过程真的帮助我们做出了更好的东西。

路易:对，绝对没错。我们只是试图推动我们的发展。我在 flippa.com 工作，我们已经把我们的开发稍微推进了一点，让每个人都可以做出贡献。这是我开始使用 HotGloo 开发原型的原因之一，因为我在团队中的传统角色是后端开发人员。我做 Ruby on Rails。

但是，能够开始开发产品，对产品的外观有一个概念，并且能够从每个人那里获得意见……你显然是在谈论客户。我们正在处理一个不与客户打交道的应用程序。例如，我们甚至有支持人员，他们可以查看原型，他们比任何人都了解我们的客户，并且能够在早期提供反馈。

所以，我想，关于这些工具有趣的事情是，人们之间正在进行一场讨论，他们倾向于重视真正全面的设计构图，这些构图让你对网站的视觉设计有一个非常好的想法。有些人更喜欢用线框或草图的方式来展示每个页面的布局和内容。

然后，如果我理解正确的话，有一些像这样的工具真的更旨在概述某种交互，因为它们是交互原型，你可以点击查看页面如何链接在一起。

**狼:**完全正确，没错。

**Louis:** 在你的工作中，交互式原型制作是不是在你制作它之前就已经在做了，然后你发现那里的工具并不适合你的需要？或者这是你建造 HotGloo 的结果？

我认为这是建造它的结果。当我们一开始的时候，我们使用的是当时已经存在的其他工具。我们用的是 Axure 和 Balsamiq。

对我们来说，这些工具，当我们把它们展示给我们的设计师时，他们就像，“嗯，这是我可以在 Illustrator 中做的事情。这是我可以用 Photoshop 做的事情。”一位项目管理人员说，“嗯，这是我可以在 PowerPoint 或 Keynote 中完成的事情。”

当我们开始使用自己的工具时，我们想，“那么，你不能用这些工具做的区别因素是什么，这对整个过程至关重要？”这是建立互动的原型，而不是专注于高保真的视觉层次，因为这是一个完全不同的故事，建立一个像素完美的原型，完全设计和互动。

我们只是想确保构建线框的过程尽可能简单，但你可以在原型中灵活地进行真正智能的交互链接。对我们来说，这不仅仅是视觉层次，而是真正的视觉，重点和真正的高保真度，而是从非常粗糙的线框到添加越来越多的交互。从本质上来说，构建真正的交互式线框，用几个堆栈进行智能交互，并改变项目的状态。

Louis: 是的，这是吸引我使用这个工具的原因之一。我已经使用了一些不同的线框工具，其中很多都有从一个页面到另一个页面的链接。但是有时候，当你在一个论坛或者一个涉及状态变化的多步骤过程中工作时，就像你提到的，仅仅是页面之间的链接是很困难的。或者您必须为应用程序的每个可能状态创建一个页面。

然而，在你的工具中，你已经建立了一些简单的编程概念，给你更多的交互能力，比如改变一个状态。如果您从下拉列表中选择了一个选项，那么它会对某个状态产生影响，然后单击“下一步”按钮会根据该状态执行不同的操作。因此，你真的可以创建一个原型，它是真实应用程序的最终流程的非常准确的表示。

**狼:**是啊。

Louis: 至少我认为我还没有遇到过这样的情况，在 HotGloo 中，我需要的交互不能通过栈和变量的组合来实现。

**狼:**是啊。外面有几个，我们现在正在看。比如一些定时互动之类的。总的来说，这基本上是我们的想法。当你构建交互式线框时，可能会非常耗时。我从网上正在进行的整个讨论中了解到这一点。现在，人们说，“嗯，整个线框化的过程非常耗时。为什么不跳过这一步，直接进入 HTML 原型呢？'

是啊。对于熟悉编码的人来说，没问题。对我们来说，这是为了建立真正复杂的交互式线框，我们需要使这个过程尽可能简单。很多其他的工具，他们只是把互动中制作的图片连接起来。

但是对我们来说，重要的是当你使用导航时，你把整个站点地图带入导航，一旦你改变，例如，站点地图中的页面名称。然后导航中的页面名称也会改变，因为这也是交互式线框的整体概念。

在大师看来，一旦你换了一个大师，它就会自我更新。你放进去的每一页的主人都变了。交互式线框不应该是一个非常耗时的过程，以至于最后你会想，“哇，我本可以节省这些时间，直接进入 HTML 原型制作。”

**路易:**对。你简单地提到了人们希望直接用 HTML 制作原型而不是使用这种工具的想法。我想这里的论点是，当你完成了，你就有了成品的第一步。你觉得这对一些团队来说是一个有效的策略吗？或者，你真的非常相信在 HTML 之外创建一个原型，然后只在交互真正确定之后才开始编写代码的想法吗？

老实说，我深信每个人都有他的工具或工具箱。有些人正在投身于 HTML 原型制作，对他们来说，他们再也不会回头在第三方工具中重新制作线框了。

我只是看到有很多人来线框化之前没有想到这个过程。当我们从第一种更新开始时，我们看到，也有一种谈论它和这种教育事情的强烈需求。我们也试图用我们的 Posterous 博客和 Wireframe Wednesday 来粗略地想象一下，我们只是浏览了一下网络，并试图把我们找到的关于线框和原型的文章和这些有经验的话题放在一起。

因为我坚信有那么多的人和那么多的团队，他们甚至没有考虑过在实际开始构建之前尝试设计一些东西的过程。所以，我要说，对于那些人来说，拥有一个工具是非常明智的，因为在大多数情况下，他们不是开发人员。他们不懂任何编程语言。

有时他们是项目经理，设计师。我们也看到平面设计师投身于网页设计，他们必须做网页设计项目，他们找到我们说，“嗯，这是一个非常酷的工具，因为第一次我可以非常容易地把一些互动的，可点击的东西放在一起，我可以展示给我们的客户。”

这对我们来说非常重要。虽然我们认为自己是一个专业的工具，但实际上让第一次尝试这些过程的人参与进来，并且像那些已经尝试了一堆工具并且非常有经验的人一样容易地与工具相处。

路易:对，绝对没错。我发现的一件事是，如果你试图在 Photoshop 等复杂的设计工具中制作线框或原型，或者试图用代码来做，很容易不小心被卷入到细节工作中。很难让自己停留在“我在这里所做的就是确定互动”的思维框架中

我开始担心角落的半径和渐变的阴影，而不是页面上的单词和链接。

**狼:**是啊。有时候我觉得 HTML 原型制作的整个想法与我们之前的过程并没有太大的不同，直接跳到了 Photoshop 的设计中。正如你提到的，你被细节工作所吸引。之前发生的事情。在交互方面没有那么多，但是在设计方面。

但也有在 Photoshop 中设计或直接进入设计的时候，从第一次点击开始，你就进入了细节工作。你开始考虑渐变半径和颜色，你正在构建的项目的整个色域。

我们当时都同意，从设计开始并不是一个好主意。现在是这个关于从 HTML 原型开始的讨论。有时我看到这两个过程的相似之处。不是一般意义上的，而是在这种被吸进细节的过程中，有了线框图，你可以省略或者删减。

有人来找我们，询问我们的功能，或者试图获得反馈。很多反馈是，当你分析它时，它会越来越详细。人们喜欢更大的调色板。他们希望有更多的字体。他们希望所有的符号都可以旋转，诸如此类。

我总是提醒他们，我们对这个工具的看法是，它不是一个视觉设计工具。这是一个线框工具。对我个人来说，任何涉及太多细节的东西都像是进入了原型制作阶段。这是你可以在浏览器中做的事情。这是你要做的事情，或者你可以在设计页面的时候做，这不是你线框化时必须要做的事情。

路易:对。我们的团队在做这种交互式原型时发现的一件事是，它迫使人们更加关注每一页上的单词。不仅仅是内容，而是伴随应用程序任何功能的副本。然而，如果你开始设计，你会考虑像素和渐变，你会在其中加入一些 lorem ipsum。如果你从一个 HTML 原型开始，你可能会做同样的事情。

但是当你使用一个线框或者一个交互式原型时，当人们到达这个页面时，他们唯一能看到的是按钮上的一些文字。然后你思考这些词是什么，什么是最好的词，以及如何解释这个过程，特别是当你在处理一个复杂的过程时。

我为此做的第一项工作是重新设计我们的整个上市销售流程。这是一个巨大的——它最终成为一个有许多活动部件的七步向导。真正迫使我们注意页面上的文字是一种启示，我想这是来自一个团队，这个团队一直是“提出一个想法，然后立即开始构建它”，转而考虑更多的事情。

狼:那是我自己的事情，我会迷失在话语中。微型复制和警报信息复制之类的东西。有时候我会非常着迷。

是的，就像你说的，当你看一个线框时，你只会看到灰色阴影、方框和按钮。唯一突出的是文案，文案是网站上最重要的东西之一。尤其是在销售网站上，尤其是在像我们这样的网站上，在营销网站上，你描述你的工具，试图让人们测试它，并希望注册成为客户。

所以，是的，我会说任何事情，只要能把焦点固定在你真正想要的地方。当我们向客户展示产品时，你不必解释太多什么是无线框。我们只用一两句话简单地说一下他们在看什么，他们能从中期待什么。

**路易:**对。如果客户看到一个模型，他们甚至不会注意到上面写的是什么。如果他们不喜欢绿色的阴影，那是他们首先想到的。他们会说，“你能把它弄暗一点吗？”

用户测试如何适应你的开发工作流程？如果你必须做用户测试，你是用原型来做，还是先从原型开始构建功能，然后再看用户？

我们已经尝试了一些事情。我们试图用原型做用户测试，这需要一些前期工作。因为你必须向测试人员解释他们在看什么，他们应该执行哪些操作，他们实际上不是在看一个完成的网站，而是在看一个原型。

我们接下来要做的另一件事是，我们粗略地构建一些东西，比如第一次拍摄，然后把它放出来，然后只记录鼠标的流动动作。这也是我们做的事情。

我们在用户测试中做的其他事情是，我们试图预先获取信息。在我们制作 HotGloo 的第二个版本之前，我们试图让人们参与进来并接受采访。我们出去给人们展示了第一个网站，然后问了他们几个问题，他们对这样的工具有什么期望。在我们开始考虑第二版将会有什么变化之前，我们试着获取尽可能多的信息。

有时我会说这是一个持续的过程，你有用户测试，从做访谈和预先获得想法开始，然后用原型做可用性测试，进入迭代和再次测试。有时你只是提前获取信息，并试图非常快速地构建一些东西。拿出来，测试，途中迭代。真的要看项目了。

我认为这很大程度上取决于项目的复杂程度。一个大的项目，大的电子商务项目，往往比只有两页、两页、三页或四页的小型营销网站更耗时，需要更多的测试。

我们很快还会有一些新的互动。这也是我们的问题，“这个新的属性面板看起来会是什么样子？我们必须改变属性面板的整个外观吗？”这是我们在 HotGloo 实际建造的东西。当您看到您在工具中设计和构思的新特性，随后又用这些新特性进行更新时，有时会觉得有点可笑。

**路易:**对。所以，你稍微暗示了一下。未来你会在 HotGloo 上做些什么？

我们已经准备好了所有的要素。但是在我们更新所有的元素之前，我们有点发现，对于我们现在拥有的一组交互，它实际上没有意义。在移动原型中还有一些其他的交互方式，所以我们也开始了交互工作。

这基本上是今年的路线图，是新的互动。是 iPhone，iPad UI 小工具。我们希望改进反馈节点，因为在 HotGloo 的整个协作过程中，除了整个交互功能之外，我们最有潜力的地方就是反馈节点。

通过这些节点的通信，我们想使这个过程更容易。我们想建立一些东西，比如，一些你可以真正批准节点的需求步骤，这样一个团队就可以看到什么时候完成了什么时候一个元素或者一个网页需要做一些工作。

某种通知系统，可以帮助团队更快地绘制项目的线框，并保持对项目和项目过程的概述，即开发过程。这是我们今年关注的三件大事。
