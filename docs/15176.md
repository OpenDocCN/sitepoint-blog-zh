# 采访——杰弗里·泽尔德曼

> 原文:[https://www.sitepoint.com/jeffrey-zeldman-list-apart/](https://www.sitepoint.com/jeffrey-zeldman-list-apart/)

每个人一生中都会有一个只想尿裤子的时候。对不起，是真的；就像某种奇怪的强迫症。它往往发生在最奇怪的时刻。对我来说，这件事发生在一周前，当时我一边和妻子说话，一边看着电脑屏幕(这也是所有男人都完美的习惯)。

当我发现《泽尔德曼访谈》被拍卖时，我决定再也不穿那条卡其裤了。

那么，这位互联网之神，www.zeldman.com、www.alistapart.com、T2 等令人惊叹的网站的发起者，真的有什么要为自己说的吗？他值得努力吗？他会帮我付干洗费吗？遗憾的是，这些问题中只有两个问题的答案是肯定的(我会让你猜是哪两个)。

Jeffrey，非常感谢你有机会和我们交谈。整个世界都在想他们是如何把这些黏糊糊的东西放进卡拉米尔克巧克力棒的，然而在没有答案的情况下，我们有几个问题要问你。

首先，让我们了解一些背景信息。谁是“泽尔德曼”,你最大的激情是什么——你的信息是什么？

我是一名网页设计师、作家和顾问。自 1995 年初以来，我一直为客户制作网站，并在网上发布独立内容。在那之前，我做过其他类型的创意和专业工作:作曲家、制片人、记者、纽约市广告创意、未出版的小说家、未来的编剧。

但是这种媒介与众不同。我在网络上找到了自己的声音。许多其他人也是如此。你也可以。如果我有口信，那就是了。一个全球性的媒体正在从它的婴儿期出现。你可以成为它的一部分。动手吧。

SitePoint:泽尔德曼的世界有什么新变化？你最近最喜欢的项目是什么？我们能传播的关于你爱情生活的最新谣言是什么？给我们一些果汁！

5 月 31 日，[我的个人网站](http://www.zeldman.com/)将迎来七周岁生日。已经 49 年了，伙计。

去年，我为新手写了一本关于网页设计的书，我正慢慢地写一本新书，名为《向前兼容:用网页标准设计和构建》，我希望在我的孩子上大学之前完成。我还没有孩子，所以这为我争取了一些时间。

我爱上了一个非凡的女人，我们住在纽约市，周围都是高楼大厦和辛勤工作的人们。努夫说。

对许多设计师来说，你的话很有分量。你想过你会得到这样的认可吗——你是怎么做到的？

在我早期的网站上，我自娱自乐，并希望我所创造的东西(广告墓地，原谅我的图标)可以娱乐一些志同道合的人。创作和出版的自由让我兴奋，就像那些早期在传统媒体上的创作努力因预算不足、冷漠的编辑和唱片公司对营销他们无法标记的音乐不感兴趣而受阻的人一样。

在网络上，我没有要取悦的投资者，没有要争论的制片人，没有要扼杀我作品的编辑。我觉得有义务分享我对网络技术和设计的发现，这样其他人也可以同样地创作和发布他们自己的创造性作品。

1995 年，我发表了一篇在线教程“问 Web 博士”，分享我对网页设计和制作的一点了解。出乎意料的大量观众阅读了教程，并提交了我尽力回答的问题。

很快就有太多的问题让我无法处理，所以我与布莱恩·普拉茨共同创建了一个网页设计邮件列表，最终演变成了一个在线杂志，名为 [A List Apart](http://www.alistapart.com/) ，我继续编辑并每周发布。

然后在 1997 年末，4.0 浏览器出现了。它们非常强大，但是技术上不兼容。当格伦·戴维斯和乔治·奥尔森邀请我和其他人尝试做些什么时，我欣然接受了这个机会，于是网络标准项目诞生了。

在过去的七年里，通过这些教育和福音项目，我很幸运地在感知和创造网络的方式上做出了微小的改变。作为一个副作用，我建立了一个读者群，并在这个领域变得有点名气。

按照汤姆·克鲁斯的标准，甚至按照雅各布·尼尔森的标准，我并不出名，如果我打算出名，我肯定无论如何都会失败。我的动力来自于对新兴的、未被驯服的媒介的热爱，与其他网页设计者的友谊感，以及保护网页免受无意中伤害的利益集团的渴望。

如果你有热情并投入工作，你可以有所作为，并可能获得一些认可。如果你的主要动机是出名，网络可能不是最好的媒介。

##### 名单之外

好了，杰弗里，让我们谈谈个人吧。是什么让你开了一个清单，又是什么让你坚持了这么久？你对想经营在线杂志或文章网站的站长有什么建议？你是如何设法每周都获得原创内容的？

正如我提到的，它开始是一个邮件列表，而且是一个相当劳动密集型的列表，因为 Brian 和我编辑了数百个参与者的评论和问题，并将其中的最佳内容打包在一个紧凑的小摘要中，发给了 16，000 名订户(该列表是免费的，非商业的，就像它最终诞生的网站一样)。

我把一个列表看作是一个发布网页设计、代码和内容信息的机会，这些信息被类似的在线出版物遗漏了。我还将它视为另一种传播 Web 标准的工具——不是通过对浏览器公司大喊大叫，而是通过提供工作 Web 构建者可以理解和使用的 CSS、XHTML 和 DOM 教程。

鼓励原创内容不是问题。筛选所有提交的材料，找到时间编辑和设计每一期，并在需要时与作者合作——这就是问题所在。该网站是非商业性的。我花在这上面的每一个小时都不是在为客户服务，因此也不是在支持我的房东或国税局。

如果网站是成功的，那是因为它的内容是由它所服务的社区提供的…因为我们不卖任何东西…因为我们不强迫我们的作者去符合一种“房子风格”或者甚至是房子的偏见。我发表过我个人不同意的文章——但它们写得很好，它们反映的观点值得一听。

我们的一些作家才华横溢，但“难相处”我不介意。如果有必要，我会用电子邮件和他们争论。“路中间”就是死亡。“每个人最好的朋友”是编辑安乐死。一份独立的清单有一个人的声音:每个作者丰富的声音；首要的编辑焦点；超简约 CSS 布局的视觉环境告诉你你不再是在堪萨斯州了。

如果你在经营一个网络杂志，真实一点，人性化一点，做你自己。平淡无奇的公司布局和乏味有用的内容不会抓住任何人，不会针对任何人。

不要担心钱。在 90 年代末，网络上充斥着垃圾，因为许多人试图变得富有。如果你有话要说，有话要发表，就去吧！如果每一个编辑决策都是基于金钱，那就离开网络，开始印刷出版，祝你好运。

通常，一份名单上的文章可以被归类为“有争议的”。你有没有因为这个而做出不发表文章的决定？

我摒弃那些毫无意义的文章:那些没有新观点或没有新创新的文章。我会删掉那些包含好观点的文章，但把它埋在题外，尽管我会请作者与我和我的副主编艾琳·基桑合作，努力挽救那些文章。我很少仅仅因为一部作品的基调不好就把它扼杀掉。

我最近拒绝了一个有趣且可能有用的系列，因为它的作者对万维网联盟(W3C)怀有奇怪的怨恨。对一些 W3C 标准的有效的、潜在的有益的批评是一回事。但是这个家伙似乎对 W3C 有意见，原因他从来没有解释过。他的偏见妨碍了他的信息，我觉得最好让他在别处发表，而不是试图贬低他。

我不会发表对人的攻击。如果你不喜欢比尔·盖茨、约什·戴维斯或雅各布·尼尔森，并且觉得不得不说出来，那就在你的个人网站上说吧。它不适合单独列出(实际上我建议你不要在你的个人网站上做，但是，嘿，这是你的网站)。

同样，我也不会发表针对团体甚至公司的自由言论。如果你对一个公司或团体正在做的事情感到担忧，并且能够支持你所说的话，这可能完全适合我们的观众——正如克里斯·卡明斯基对微软“智能标签”的担忧，我们在 2001 年发表了[。](http://www.alistapart.com/stories/smarttags/)

但是如果你只是不喜欢微软，或者 Sun，或者 Macromedia，或者 Adobe，或者 IBM，或者 Apple，或者其他什么，网络是免费的，你可以自己发表你的观点。只是不在我的杂志里。

##### 标准

site point:WaSP 项目最近似乎已经销声匿迹了。有什么内幕消息，对于我们不知道的读者来说，什么是[黄蜂](http://www.webstandards.org)？

Web 标准项目(WaSP)是一个由设计师、开发人员和 Web 用户组成的联盟，致力于改善对 CSS、XHTML、XML、ECMAScript(JavaScript 的标准版本)和 W3C 文档对象模型等通用技术标准的支持。

我们将这些技术称为“Web 标准”，它们经过精心设计，旨在为最大数量的 Web 用户提供最大的好处，同时确保在 Web 上发布的任何文档的长期生存能力。

按照这些标准进行设计和建造可以简化和降低生产成本(这在当今非常重要！)，同时提供更多人和更多类型的互联网设备可以访问的网站。随着传统桌面浏览器的发展，以及新的互联网设备进入市场，按照这些思路开发的网站将继续正常运行。

WaSP 第一阶段不能再要求更多了，所以我暂时退出了这个网站(尽管它仍然可以在[http://archive.webstandards.org/](http://archive.webstandards.org/)使用)，重振了我们最初的指导委员会，并招募了额外的成员，我让他们负责编写和设计即将到来的第二阶段网站，它应该很快就会上线。

**SitePoint:在浏览器标准之争中，你们取得了哪些进展？你觉得浏览器厂商对你和你的组织足够重视，会在意你说的话吗？会不会有一种浏览器既完全符合标准，又足够友好，可以在日常生活中使用？**

1998 年至 2001 年间，WaSP 请求浏览器制造商在支持这些技术方面做得更好，最终他们做到了——部分原因是许多浏览器公司的工程师同意我们的观点，并将 WaSP 视为他们与管理层内部斗争的盟友。

从 2000 年开始，一个又一个领先的浏览器兑现了我们提倡的许多标准的承诺。目前市场领先的浏览器，以及它们的几个较小的竞争对手，为 HTML 4、XHTML 1、CSS、ECMAScript 和 DOM 提供了出色的支持——或者正在向这种兼容迈进。

由于这些改进，浏览器在解释网页内容方面做得更好。它们也更容易使用。例如，大多数当前的浏览器允许你调整文本大小，并且它们正确地处理标准的脚本行为，而不是阻塞它们。

多亏了这些浏览器，设计人员和开发人员终于可以自由地使用 XHTML 和 CSS 进行构建，并且在许多情况下可以将结构与表示分离，以最大化可移植性和可访问性。只要小心，设计人员还可以使用 W3C 标准 DOM 向他们的站点添加复杂的行为。

尽管今天的浏览器在很大程度上支持标准，但成千上万的设计人员和开发人员继续使用过时的方法，将结构与表示捆绑在一起，在某些情况下完全避免语义结构，并误用(X)HTML 作为设计工具。在第二阶段，WaSP 将提供教育资源，帮助我们的同行学习符合标准的方法，这些方法符合他们的利益，因为它们符合他们的观众和客户的最佳利益。

我们在第二阶段着手解决的另一个正在进行的问题是，许多网页设计专业人员使用的工具在支持当前标准方面做得还不错。例如，你不得不千方百计地使用 Macromedia Dreamweaver 或 Adobe GoLive 来生成有效的 XHTML。

Dreamweaver MX 和 GoLive 6 改变了这一点。

在幕后，WaSP 的 Dreamweaver 任务组(1.5 阶段？)与 Macromedia 合作改进 Dreamweaver 的标准和可访问性合规性，尽管 Macromedia 的工程师和管理人员应获得大部分荣誉。

我们无法组建一个合适的工作组来与 Adobe 合作，但他们似乎没有我们也做得很好。我们希望与这两家公司继续合作，因为他们将继续微调他们的产品并支持新兴标准。

##### 浏览器

随着 AOL 可能转而使用 Mozilla 作为其浏览器引擎的消息传出，许多人预测“浏览器大战”将会重新开始。你担心这会导致新一轮的特定于浏览器的标签和扩展吗，或者标准的信息现在足够强大来阻止这种情况发生吗？

这完全不会发生，因为 Mozilla/Netscape 6 都是关于 Web 标准的，IE6/Win 和 IE5/Mac 支持相同的标准(IE5/Win 支持其中的许多标准)。如果说有什么不同的话，那就是 3000 多万 Mozilla 新用户将强化这样一个信息:为任何一种浏览器设计都是不明智的。相反，使用 Web 标准并为它们设计。

如果你的网站不能在 IE 中运行，那你就是个傻瓜。如果它只在 IE 中有效，那你就是个傻瓜。真正的挑战是为可移植性和可访问性而设计(这并不意味着乏味的、最小公分母的设计)。

我们需要学习的东西不是网景特有的，也不是 IE 或 Opera 特有的。我们需要更好地理解现有的和新兴的 Web 标准，并为媒体而不是特定的浏览器进行设计，利用 Web 的真正力量来影响每个人。

你认为网景的反击是一件好事吗？你对整个 Mozilla 项目有什么想法，因为它在近 4 年后走向 1.0 最终版本？

我很高兴 Mozilla 快完成了。这是一个伟大的浏览器，具有广泛的标准支持。

我们都希望它能早点准备好，我相信没有人比 Mozilla 的工程师和 AOL/Netscape 的营销人员更希望这样。但我不会扮演纸上谈兵的项目经理。我不知道从头开始创建一个新的符合标准的浏览器需要多少代码。

在这条道路上——从头开始——网景失去了巨大的市场份额。一些用户甚至一些开发者似乎已经把他们作为玩家一笔勾销了。很高兴看到这种变化，很高兴看到他们赢得新用户，并恢复市场急需的平衡。

Opera 的市场份额也在增加，更小的替代浏览器也是如此。所有这些都强化了使用 Web 标准进行设计和构建的智慧。

##### 未来

SitePoint:你认为哪些即将出现的计划会对网络的未来带来最大的好处？

4.0 浏览器的最终消失，以及符合标准的浏览器的改进，将使我们能够抓住 CSS 在布局方面和 XHTML 在文档结构方面的真正力量。

这将导致网站通过拨号连接快速加载。可以移植到支持网络的手机和其他无线设备上的网站，不需要替代版本和昂贵的专有发布系统。

这些网站将更容易维护，更容易访问，并且持续时间更长(许多 90 年代末的网站在最近的一些浏览器中已经失败了)。基于共享技术的前向兼容性。这是一件美好的事情。它将解放我们，让我们花更多的时间在内容、设计和可用性上，花更少的时间在调试和版本控制上。

SitePoint:有你不同意的新兴标准吗？你觉得有什么需要被收养或创造的吗？

CSS 是一个很好的标准，但是它的一些部分还没有以对设计者实用的方式实现；最后 5%的遵从性是最难的。CSS 的某些部分看起来不必要的复杂和/或晦涩。

W3C 一直致力于解决这个问题。结果可能会在这篇采访发表之前公布。如果没有，它们将很快在 https://www.w3.org/上市。

有些事情用传统的(非结构化的)基于 HTML 表格的布局比用 CSS 更容易做到。有时这是因为浏览器对 CSS 的解释不同。但是其他时候，这是因为 CSS2 并不完全做你可以用表格做的所有事情，尽管它在许多方面超越了表格，并且允许你做你用传统的基于 HTML 的设计技术绝对不能做的事情。

XHTML 是 HTML 在 XML 中的重构。XHTML 1.0 非常出色，因为它甚至可以在较旧的浏览器中像 HTML 一样工作。但是这个事实本身并没有让一些开发者信服。W3C 已经开始模块化 XHTML，使其更加强大和可扩展，但是这些新的和即将到来的版本还不是浏览器友好的。

随着浏览器的进步，我们将会看到更复杂的 XHTML 在发挥作用，其好处将变得更加明显。现在转换成 XHTML 是个好主意，很多有远见的设计师都在这么做。当这种语言成熟并且浏览器支持它的高级功能时，更多的人会争相赶上。

SVG 是一种基于 XML 的矢量图形语言，具有有趣的功能和潜力。但是在目前的浏览器中，它需要一个插件。好吧，如果你打算使用插件，你也可以使用 Flash，它目前更加成熟和强大，而且[也变得更加容易使用](http://www.alistapart.com/issues/143/)。

结合其他形式的 XML，SVG 最终会做出惊人的事情，现在就值得学习。大卫·艾森伯格写了一本好书来帮助我们做到这一点。

##### 可用性

**SitePoint:您对 Macromedia 新推出的 MX 系列有什么看法，尤其是 Flash MX？就界面和页面而言，它会是网络的下一件大事吗？**

我看过 Jeremy Allaire 亲自演示 Flash MX，我的工作室里有一份该程序的副本。Flash MX 是一个成熟的创作环境，Flash 6 player 为动画驱动、高度交互式网站的设计者解决了许多问题。Flash MX 提供了对外观和感觉的绝对控制，并能够为任何拥有该插件的人创建复杂的界面。

Flash 的某些行为可以通过 W3C 标准 DOM 来模拟。但是动作将不会那么流畅，视觉效果将不会那么引人注目，旧的浏览器以及一些新的浏览器将无法呈现出它们应该呈现的效果。

对于设计密集型网站和某些商业和娱乐网站，Flash MX 非常有意义，尤其是如果您合并了可访问性增强功能(尽管这些功能需要成熟，而且肯定会成熟)。

对于程序员来说，Flash 支持大部分 ECMAScript 和大部分 XML，并且它可以与其他设计产品和后端技术顺利接口。妖怪从瓶子里出来了。

我不认为 Flash MX 会是下一个大事件。我认为自 20 世纪 90 年代末以来，闪存一直是一件大事，并将继续如此。

SitePoint:也就是说，你对纯 flash 网站的潜力、功能和整体可用性有什么看法？这将是一件好事的那一天会到来吗？

我们可能会看到主要由 Web 标准驱动的以文档为中心的网站和由 Flash 驱动的面向动画的网站之间的差距越来越大。

如果我在设计一个内容网站、社区网站、交易网站或信息网站，我可能会继续使用 XHTML、CSS 和 ECMAScript，用 Perl 或 PHP 来促进用户交互。

我将把这些标准用于打算持续很长时间的网站，以及打算随时间发展的每日或每周出版物。我将把这些标准用于鼓励读者通过论坛或评论做出贡献的网站。

如果我正在设计一个娱乐网站，特别是如果这个网站的预期寿命为两年或更短，我可能会选择 Flash MX，因为 Flash 带来了一种基于文本的网站所无法比拟的性感和兴奋。不过，这取决于项目和受众，也取决于设计师或设计公司的专业知识。

Flash 的每个新版本都解决了许多可用性投诉。Flash MX 支持书签和后退按钮，如果开发者足够聪明能够利用这些特性的话。

Flash 从 4.0 版本开始增加了一些可用性功能，包括复制和粘贴文本的能力，但实现这些功能取决于开发人员，就像 HTML 网站的开发人员需要考虑和适应不同的用户需求一样。

SitePoint:告诉我们你设计网站时遵循的常规。

我只和一个客户合作——从不和委员会合作。

我尽可能多地了解这个项目:它的目的是什么，它为谁服务，为什么它存在。如果网站有存在的理由，客户很清楚这个理由，并且客户明白网站应该为其受众服务，我就可以开始了。如果网站没有令人信服的存在理由，我可能无法继续工作。我就是做不好工作。

一旦我知道了这个网站是关于什么的，我会试着找到一个统一的想法来驱动它。设计，文字，甚至技术水平，都直接来源于这个想法。

天气好的时候。

在其他日子里，我只是在 Photoshop 里闲逛，有时玩颜色，有时玩网格，有时玩字体。

有时候我也会做同样的事情，只是用代码而不是 Photoshop。

这听起来很随意，但是有时候忽略对一个概念的探索是你发现这个概念的方式。有时候，当你在 Photoshop comp 中处理抽象的大小关系时，你的潜意识正在解决网站的概念问题。

有时候，当你在做一个个人项目时，你的潜意识开始发展客户的品牌。然后你出去吃饭，当你选择意大利扁面条的时候，这个想法突然出现在你的脑海里。

我不太喜欢正式的流程和集体思维(尽管这些对一些人有用)，我也不按小时计费。更高的功率解决了大多数设计问题；我只是一个渠道。我想对于大多数有创造力的人来说都是如此。我们是幸运的小船只。

在我的非商业项目中，我采用了类似的古怪的工作方法，知道我可能会搞砸网站的第一个版本，但是因为我是所有者和客户，我有自由随着时间的推移不断改进它。
