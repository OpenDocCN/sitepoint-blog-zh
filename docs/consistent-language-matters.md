# 为什么一致的语言很重要

> 原文：<https://www.sitepoint.com/consistent-language-matters/>

我们已经谈了很多关于在你的应用程序或产品界面中，以及在帮助页面、电子邮件和其他围绕它们的材料中使用一致的语言。

但是为什么一致性如此重要呢？

让我们举几个一致性的例子，看看它们给用户体验带来了什么。

## 品牌词汇

正如我们在讨论品牌词汇时所看到的，[围绕你所使用的语言的一致性](https://www.sitepoint.com/tell-the-story-of-your-brand-service-or-product-with-a-brand-vocabulary/)有助于用户理解你的产品。

如果你的目标人群是对你的产品或产品类别不熟悉的人，这一点尤其重要。例如，不断地将免费产品升级称为“奖励”,可以确保人们确切地知道你在说什么。如果你使用一系列术语——奖励、附加服务、免费升级、特别优惠——他们可能需要更长时间才能对你的产品感到满意。

因此，一致性有助于用户理解和预测你的产品。

## 标准措辞

如果你必须在很多地方给用户相同的指示或建议，假设上下文允许，我总是主张使用相同的措辞来提供这些指示。

原因很简单:如果它总是一样的，用户只需要学习一次那个短语或指令是什么。每当他们看到这个短语时，他们可以浏览一下，知道它说的是什么。

假设您有一个多页表单或向导，用户可以通过它工作。在底部，您可以通过链接前进到表单后面的页面，或者返回到前面的页面。也许您还有一个选项，让用户保存表单并在以后返回。

假设我将“下一页”指令写成“转到下一页，或者单击“保存”返回并稍后完成”。然后，我会在表单的每一页上，在解释如何完成表单的帮助内容中使用相同的短语，等等。

一旦用户看了一两遍说明，他们就不必通读就能知道内容。

所以一致性节省了用户的时间和挫折。

## 一致的行动呼吁

基于上一点，如果你需要一遍又一遍地重复同样的行动号召，每次都要使用同样的措辞(如果上下文允许的话)。

让我们考虑一个非常简单的例子:系统邮件。我的一个客户有一套关于他的产品的系统电子邮件，他想在每封邮件中包含一个邀请用户在有问题时联系支持部门的信息。

我们是否写了十次不同的行动号召，每封邮件一次？

不会。我们通过以下方式使该信息成为每封电子邮件的一致组成部分:

*   把它变成自己的一句话，放在邮件正文的末尾，与正文的其余部分分开
*   每次使用时，使句子结构(包括链接)完全相同。

这样，用户只需要读一遍句子，就知道它给他们提供了帮助。他们只需要在两封电子邮件的底部找到它，就可以期望，如果他们需要帮助，他们可以从我客户的业务中找到任何电子邮件的结尾。

一致性使用户更容易参与你的产品和业务。

## 列举 consist 语言的好处

所以语言的一致性给我们带来了三大好处。它:

1.  帮助用户理解和预测你的产品
2.  节省用户的时间和挫折
3.  让用户更容易参与你的产品和业务。

顺便说一句，这些都是视觉和体验的一致性给用户带来的好处。因为毕竟屏幕语言是视觉化的(对大多数用户来说)，是用户对你的产品或服务体验的一部分。
