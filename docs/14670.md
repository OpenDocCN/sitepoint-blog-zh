# 用 Linux 试水

> 原文：<https://www.sitepoint.com/testing-the-waters-with-linux/>

如果您是 Linux 新手，需要一个测试环境，或者不想在主工作站上设置双引导，有一种简单的方法来构建 Linux 服务器。简单地发现一些退役的硬件。

包括本文作者在内的许多用户继续在较旧的硬件上加载 Linux，这些硬件可能已经在砧板上或被 EBay 拍卖。如果你只是想试验一下操作系统，或者可能需要深入了解一下，一些最流行的 Linux 发行版可以在较旧的、性能较低的系统上运行。最适合 Linux 新人的三个是 [Mandrake](http://www.mandrakelinux.com/) 、 [SUSE](http://www.suse.com/) 和 [Fedora Core](http://fedora.redhat.com/) 。如果你有从 7.3 到 9 的任何红帽版本，这些也能很好地工作。

我目前有一台奔腾 200 MHz，256 MB 内存和一个运行 Red Hat 9 的 6 GB 硬盘。我另外加载了桌面环境(Gnome 和 KDE ),并在将它们转移到我的生产 Linux 桌面之前，使用这个系统测试任何新的应用程序和系统补丁。桌面应用程序可能不会像在最新型号的机器上那样作为 sprite 运行，但是测试 web 操作环境所需的 web 服务器和相关服务可以顺利运行。

这个环境可以为管理 Apache、FTP、SSH、邮件服务器(如 Postfix、QMail 或 Sendmail)以及 MySQL 和 PostgreSQL 数据库服务器提供一个学习空间。

因此，在拍卖或拆除另一个退役的系统之前，请考虑将其用于您的 Linux 追求的测试环境中。

还有其他的方法可以从引导光盘运行 Linux，不需要安装硬盘。用户在主工作站上的 Windows 中运行这些 CD，如 Knoppix。
