# CSS 字体大小:权威的字体大小指南

> 原文：<https://www.sitepoint.com/css-font-sizing-tutorial/>

CSS 中的字体大小听起来似乎很简单。直到你尝试它。许多开发人员使用原力；他们修补 CSS 字体大小属性，直到它看起来正确，却发现它在另一个浏览器中是不同的。一点点理解可以走很长的路…

## 字体大小属性

可以为任何 HTML 标签设置`font-size`属性(即使它通常不包含像`br`这样的文本内容)。可以为其指定各种绝对、相对或长度尺寸参数。

除非您覆盖它，否则元素将继承其父元素的字体大小。当您指定相对大小时，这尤其重要。

## 绝对字体大小关键字

有几个绝对字体大小关键字可用。字体大小由浏览器预设决定，元素不会继承其父元素的大小。

*   字号:xx-small；
*   字号:x-small；
*   字号:小；
*   字体大小:中等；
*   字号:大；
*   字体大小:x-大；
*   字号:xx-大号；

尽管大多数浏览器都支持这些关键字，但确切的大小会有所不同。它们是一种相当粗糙的字体大小调整方法，通常被大多数开发人员所避免。

## 相对字体大小关键字

有两个相对字体大小的关键字可用。字体根据其父元素调整大小:

*   字体大小:较小；
*   字体大小:较大；

例如，如果父元素的字体大小为“中”，则值“较大”会将元素设置为“大”。其他字体单位通常会改变 1.2 倍左右，但同样，没有标准，浏览器结果会有所不同。

## 绝对长度

可以给`font-size`属性分配一个绝对长度:

*   **毫米**:毫米，如 10 毫米。
*   **厘米**:厘米，如 1cm ( = 10mm)。
*   **英寸**:英寸，如 0.39 英寸(~ = 10 毫米)。
*   **pt** :点，其中 1pt 一般假定为 1/72 英寸，如 12pt。
*   **pc** : pica，其中 1pc 为 12pt
*   **px** :像素，如 14px。

总的来说，所有这些测量单位都有问题。毫米、厘米和英寸对于基于屏幕的媒体来说是不准确的。点数和十二点活字不可靠，因为系统可以使用不同的 <abbr title="dots per inch">dpi</abbr> 设置。Pixel 看起来是最合适的，但是它会导致可访问性问题，因为[文本不能在 IE](#textpagezoom) 中调整大小。

## 相对长度

可以给`font-size`属性分配一个相对于其父字体大小的单位:

*   **em** : 1em 相当于现在的字体大小，所以 2em 是现在的两倍。
*   **%** : 100%相当于当前字体大小，所以 200%是两倍大。
*   **ex** : 1ex 相当于当前字体中字母‘x’的高度。

很少有开发人员使用“ex ”,但在某些需要细粒度字体大小的情况下，它会很有用，例如 1ex 而不是 0.525em。

百分比和“em”大小相等，例如 50% = 0.5em、100% = 1em、120% = 1.2em 等。一些浏览器表现出细微的差异，但这很少是个大问题。如果您想保存每个字节，您可以选择最短的定义，即 50%比 0.5em 短，1em 比 100%短。

## 文本大小和页面缩放

这就是额外的复杂性出现的地方。大多数浏览器允许用户:

1.  增大或减小基本文本尺寸(图像尺寸不变)
2.  放大或缩小页面，以便所有文本和图形相应地改变，或者
3.  允许文本大小调整和页面缩放。

更复杂的是，Internet Explorer 不允许对以像素(px)为单位定义字体大小的元素进行文本大小调整。

如果你是一个从印刷背景转向网络的设计师，给用户这么大的权力是令人不安的。你的设计可能会被一个放大 200%但缩小到 50%的用户毁掉。而且——不——你无法阻止它。你也不应该。

## CSS 字体大小建议

普遍的共识是，在大多数情况下，“em”或“%”是最佳解决方案。[网页字体](https://www.sitepoint.com/anatomy-web-fonts/)可以相对于彼此进行精细缩放，并且支持浏览器文本大小调整。我还建议在 body 标签上使用百分比`font-size`;它可以在一些旧的浏览器中产生更好的文本大小。

当你在开发一个网站的时候，我会给你一些其他的建议:

1.  在测试之前，将所有浏览器的字体大小和页面缩放重置为默认值(这让我困惑了好几次！)
2.  在各种浏览器中尝试*合理的*文本大小和页面缩放的组合，以确保文本保持可读性。
