# 摆脱云托管的污名

> 原文：<https://www.sitepoint.com/getting-past-cloud-stigma-2/>

在过去的几年中,“云”已经成为一个主要的流行词汇。这已经到了一个地步，当人们听到“云”这个词时，他们会翻白眼。造成这种耻辱的因素有很多。

通常，不熟悉云技术但熟悉传统主机的人会说，“云就是‘互联网’，对吗？这只是另一个托管网站的地方。”

## 云方法

从表面上看，这是真的。云服务使用传统托管公司使用的大部分硬件。他们有服务器、路由器和交换机。云服务和传统技术的区别不在于硬件，而在于方法。它是关于以提供改进的可靠性、可伸缩性和性能的方式使用硬件。

在云计算开始爆炸之前，专用主机几乎是运行网站和网络应用程序的最佳选择。专用硬件有很多好处，尤其是在性能方面。对于大多数用例，专用硬件也具有合理的可扩展性。你真的可以用大量的内存、存储和处理能力来增强服务器。绝大多数网站和网络应用程序永远不会像现有的最强大的服务器那样需要如此强大的能力。但最终，专用硬件确实会限制其扩展能力。

虽然专用硬件在性能上很棒，但它也有弱点:可靠性。即使有最好的硬件(例如冗余存储和冗余电源)，您仍然会有单点故障。有很多方法可以解决这个问题，但是这些方法既复杂又昂贵。

随着越来越多的服务和应用程序(如智能手机应用程序)强化其使用互联网连接资源的方式，对可扩展性和可靠性的需求也在快速增长。这进而创造了一个市场，人们愿意为管理设置和维护超可靠硬件的所有复杂性的服务付费。

## 裁员

在专用服务器时代，重点是让硬件更加可靠。硬件开发人员找到了最常见的故障点(例如，电源和硬盘)，并开始加倍和三倍增加(这是一种过于简化的做法)，以便如果一个组件出现故障，机器可以继续运行。这实际上是一个非常激动人心的时刻，每个人都痴迷于创建一个永不停机的服务器。

但是随着时间的推移，试图创造无限可伸缩性和可靠性的人们意识到超冗余硬件永远不可能是解决方案。没有一台机器是永远不会出故障的。因此，他们决定最好的解决方案是反其道而行之，建立为失败而设计的系统。听起来很疯狂，对吧？

## 云可靠性

你可能想知道为什么构建失败的东西会导致无限的可靠性。答案其实挺简单的。物理定律表明，最终，每一个计算机硬件都会出故障。没有什么可以永恒。因此，设计硬件和软件系统的人决定改变他们将组件组装在一起的方式，而不是试图与自然抗争。在云中，组件的组装方式使得任何数量的组件都可以失败，而应用程序将继续运行。这并不是什么新想法。几十年来，飞机被设计成即使发动机完全失灵也能完成飞行。甚至我们自己的身体也是围绕这样一个概念设计的，即每天都有数十亿细胞死亡。这种情况在我们每个人的内心不断发生，在我们生命的大部分时间里，我们几乎没有注意到。

用于创建处理故障的硬件平台的确切方法过于复杂，无法在一篇短文中解释清楚。但重要的是，至少要知道云平台旨在处理哪些类型的故障:

1.  单个硬件组件的故障(例如，故障硬盘、电源、内存芯片、处理器)
2.  数据中心内的连接故障(例如，路由器、交换机、负载平衡器出现故障)
3.  整个数据中心本身的连接故障(例如，地震、龙卷风、恐怖袭击)

在过去五年中，云提供商在基础设施、系统和流程方面进行了大量投资，使他们能够将这些复杂的系统作为商品进行销售。他们提供的许多实际组件都不是新的。例如，每个云服务的中心模块之一是服务器虚拟化。服务器虚拟化已经存在很长时间了，它真的没有那么特别。正是模块和服务的强大组合使得云计算成为如此伟大的解决方案。

## 云需求规划

人们最初开始研究云产品时的一个误解是，云服务允许任何应用程序或网站迁移到云中，并享受无限的可伸缩性和可靠性。事实根本不是这样。为了让一个网站或应用程序具有真正的可伸缩性，它必须从头开始构建，以利用让云工作的所有功能。在许多情况下，这说起来容易做起来难。

这种误解很大程度上是由于云提供商营销其服务的方式，这可能是对云计算的少数有效批评之一——事实上，大多数云提供商宣传实施超可靠系统几乎不需要什么工作。事实是，它需要对构成云的所有组件有非常清晰的理解，并需要大量的奉献和努力来实现一个好的计划。然而，虽然它需要大量的努力，但它比使用传统的托管服务获得相同的结果需要更少的努力和费用。

我最近读了一个人的评论，他说云服务不会取代传统的主机服务。我相信这种想法来自于对云如何工作以及人们为什么迁移到云的误解。这让我想起了 90 年代我在家里安装第一条 ISDN 线路的时候。我花了一大笔钱来安装，每个月还要花数百美元来维修。许多人告诉我，他们永远不需要那样的速度，大多数人使用拨号上网就可以了(尤其是家庭用户)。当然，这个世界上仍然有一定数量的人使用拨号上网。但总的来说，高速互联网已经接管，提供商继续竞争提供更好的服务。我敢肯定，每个质疑我早期推动高速发展的人现在都在自己家里拥有高速互联网。

同样的概念也适用于云计算。传统主机是有限的。当然，大多数网站不需要无限的可扩展性。甚至可能会有一些人不在乎他们的网站偶尔宕机。但在大多数情况下，人们希望并愿意为提供更高可靠性和最佳性能的更好技术买单。

很多情况下，人们不会直接购买这些服务。但是越来越多的服务提供商(包括传统主机)将会找到利用云计算的方法来创建可以无限制扩展且永不停机的服务。

云计算远非完美，我很期待看到它在未来几年的发展方向。但有一件事是肯定的——我玩云技术比玩传统主机有趣多了。我的企业通过突破云的限制赚了更多的钱。
