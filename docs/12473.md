# jQuery 创建者 John Resig 访谈

> 原文：<https://www.sitepoint.com/interview-john-resig/>

约翰·瑞西格在 Mozilla 公司的角色是 JavaScript 布道者 T2。我挑战你发明一个更酷的标题！

他也是 jQuery JavaScript 库的创建者和主要开发者。Ara Pehlivanian 最近采访了他，谈论 JavaScript 的未来、在线隐私以及他正在进行的一长串项目中的一小部分。

微软和诺基亚最近[在](https://www.sitepoint.com/microsoft-nokia-adopt-jquery/)宣布他们将在他们的平台上使用 [jQuery](http://jquery.com) ，首先让我向你表示祝贺。我很好奇，jQuery 社区是否为这个结果进行了游说，或者说这个结果出乎意料？

肯定有很多 jQuery 开发人员也在 ASP.NET 工作。阅读博客，你可以看到他们对这一消息有多高兴。

据我所知，没有来自社区的官方游说——但是肯定有来自各个组织内部的游说来支持 jQuery 作为首选库。这两个声明都令人惊讶(因为我们没有预料到这些公司会来找我们)，但我们与他们进行了交谈，并努力确保任何形式的合作都进展顺利。

两家公司对合作都非常理性。我们列出了许多关于未来工作的条款，这样社区最终会受益(比如没有代码分叉、将贡献送回项目、开放交流等等)。

这对 jQuery 的发展会有什么影响？

绝对会变得更好。通过能够拥有更多的资源(因为我们将需要更多的东西:更多的 jQuery 测试、更多的补丁、更多的输入)，整个项目将会受益。最终，它只会迫使我们变得更加专业，拥有一个更加精细的开发过程——无论如何，这才是我们真正应该追求的。

作为 jQuery 的创建者和首席开发者，你对与之竞争的 JavaScript 库有什么看法，你认为随着时间的推移，前景会如何发展？

库经历周期(就此而言，浏览器也是如此)。有一段时间是竞争，然后是合作，然后是同质化。例如，对于所有库现在都实现的 CSS 选择器引擎，围绕其引擎的性能有很多竞争；因为每个人都在有效地一遍又一遍地实现相同的特性，所以必须有一些区别。然后，各库合作，提出了一套可靠的选择器来支持，并共享性能改进技术。最后，我们现在看到了代码库的同质化——统一在一个后端上(就像 Sizzle)。我绝对期望我们将会看到图书馆复制的其他特性。

库的最大好处是双重的:它应该让用户不用处理浏览器问题，它应该为用户提供一个 API 来简化他们的开发。图书馆要想成功，必须做到这两点。但同时，重要的是要认识到要求一个统一的库(不同库的某种合并)是没有意义的。一个库的独特之处在于它的 API 以及它选择如何解决浏览器带来的问题。把它变成一个平淡无奇的通用 API 只会导致用户不得不一遍又一遍地重复使用它。

什么是嘶嘶声？

Sizzle 是一个新的纯 JavaScript CSS 选择器引擎。Sizzle 是最终将成为 jQuery 新引擎的代码名称。我想要一个速度快(至少可以和其他引擎竞争)、体积小(文件小)、易于扩展、没有依赖性的引擎。

实现这一点的好处是，该引擎很可能成为许多主要库的一部分。我一直在与 [MochiKit](http://mochikit.com) 、[原型](http://prototypejs.org)、[道场](http://dojotoolkit.org)和 [TinyMCE](http://tinymce.moxiecode.com/) 团队交谈，他们都对使用 Sizzle 作为他们的代码库感兴趣。有一个统一的方法来处理 JavaScript CSS 选择器会很棒。

这些库经常填补 JavaScript 发展中的空白，但是对于它应该如何发展似乎还存在一些争议。例如，最近，道格拉斯·克洛克福特和布伦丹·艾希在是否需要将语言复杂化或简单化的问题上产生了分歧。你对这个问题有什么看法？

我不认为它们是相互排斥的。最大的争论似乎是(并且通过最近的【Harmony 合并相对解决了)该语言的新特性应该使用更小的构建块来构建。微软 JScript 团队希望语言以这种方式构建，我同意:如果要有一个`class{}`结构，那么它只需要是一个已经可以使用语言内置方法完成的任务的语法糖。从简单开始，然后逐步增加复杂性，这是一种非常有效的编写代码的方式。

鉴于浏览器对 JavaScript 1.5 之外的任何东西的支持都有点粗略，在你看来，想要体验该语言更高级特性的程序员应该怎么做？

我不太担心该语言的高级特性，而是更多地关注 HTML5 即将推出的特性。如果你想实际一点，能够使用你今天学到的东西，那么你很少会在 JavaScript 语言领域找到安慰。然而，因为 DOM 是很好的对象可检测的(`if (document.querySelectorAll)`)，所以很容易立即启动并运行，开始使用新特性。

让我们谈谈你现在正在做的一些事情。您网站上的项目页面列出了 76 个项目。有没有一两件让你特别兴奋的事情？

哦，天哪，有一大堆。我最喜欢的两个承诺是:

**[将浏览器环境移植到纯 JavaScript](http://github.com/jeresig/env-js/tree/master<br />
“>env.js</a></strong> – This was an attempt to <a class=) (运行在 Rhino 中)。它允许您完全离线运行 web 应用程序，包括像 jQuery 和 Prototype 这样的库。现在有很多人在研究它，并将其用于许多应用。**

 ****[processing . js](http://github.com/jeresig/processing-js/tree/master)**——这是我最近的一个[项目](http://ejohn.org/blog/processingjs/)，是处理可视化语言到 JavaScript(运行在画布之上)的一个移植。随着人们使用它来构建从游戏、实时可视化、图表到艺术的一切事物，所有这些都是用 JavaScript 制作的，这已经被广泛接受。

我的一个运行时间最长的项目是 RIT(罗切斯特理工学院，我所在的学院)的课程表生成器，你可以插入一些课程，它会计算出所有可能的课程表排列。利用这个，你可以找到理想的课程表。它从 2003 年初开始运行，[仍然有很多人在使用它](http://search.twitter.com/search?q=schedule.csh.rit.edu)。

过去我听你提过几次燃料，是什么？

这是我在 2007 年第一次加入 Mozilla 时承担的一个项目。它试图通过在复杂的 Mozilla APIs 前面放置一个简单的 JavaScript 层来简化它们。现在有很多扩展很好地利用了它(因为它随 Firefox 3 一起发布)，所以我很高兴看到它开始帮助开发人员的生活变得更简单。

除了你的编程项目，你还是一名作家。你定期在[你的博客](http://ejohn.org/blog/)上发帖，最近开始为 Ars Technica 撰写[文章。其他事情都摆在你面前，是什么促使你这么做的？](http://arstechnica.com/authors.ars/jeresig)

我想写一些与我的博客不同的东西——从一个更中立的角度来写。Ars Technica 在撰写技术文章方面享有很高的声誉，这些文章相对不偏不倚，但仍能提供大量信息。能够为他们写作是一个巨大的挑战，当然应该提高我的写作能力。

你还在写一本名为*[【JavaScript 忍者的秘密](http://jsninja.com/)* 的新书。进展如何？

慢慢来了。在过去的几个月里，我已经落后了，所以我需要重新赶上。看起来最早可能会在北美春季发行。

目标受众是谁？新手能从中学习吗？还是它更适合那些希望成为忍者的有经验的程序员？

肯定是更偏向有经验的编码员。有大量的书籍适合刚接触 JavaScript 的人，但我觉得目前渴望更多书籍的人的市场非常缺乏营养。

我们能从这本独一无二的书中期待什么？

它居然敢覆盖跨浏览器开发。几乎所有的 JavaScript 书籍都停留在 JavaScript 语言的安全领域，相对来说，这里几乎没有什么 bug 需要处理。一旦你进入 DOM 并解决困难的问题——图书馆处理的那种问题——这就是大部分工作发挥作用的地方。

我有几个与工作无关的问题要问你。你曾经在你的“关于我”页面上列出你的手机号码，作为一种开放的标志。已经不在了。怎么会这样

老实说，它很少被用于它的目的。我粗略地说，大约 70%的电话来自招聘人员——我不得不把他们派到别处；25%是 jQuery 问题，可以通过邮件列表更好地处理；还有 5%是其他问题，需要(至少)一封跟进邮件，以便我能更好地了解问题是什么。

我仍然会回复通过电子邮件或即时消息收到的问题(按照自己的时间表处理这些问题很好)；事实证明，显示电话号码太麻烦，没有太多好处。

这么说，你在写一本书，并定期在不同的网站上发表文章(包括你自己的网站)；你在会议上发言，在几个不同的项目上编码。你显然是个大忙人。我们能一瞥约翰·瑞西格的另一面吗？你做什么娱乐？你最喜欢的乐队/电影/游戏/书籍/杂志/网站是什么？

我有一个很好的女朋友，克莱尔，我和她一起度过了很多空闲时间。我喜欢去看表演(最近看了《渡渡鸟》、《预设》和《剪影》)，参观博物馆，通常和朋友出去玩。我也是一个好电视的爱好者:目前正在看《真爱如血》、《嗜血法医》、《广告狂人》,费城总是阳光明媚。

我目前正在看一些不错的漫画小说:刚刚看完了《守望者》和《T2》【蝙蝠侠:黑暗骑士归来】。我也无可救药地沉迷于 [reddit](http://reddit.com) 。

我喜欢在工作和私人生活之间保持清晰的界限。当我和朋友玩得开心时，我不会谈论工作或电脑。这非常令人放松——事实上，我现实生活中的大多数朋友对电脑知之甚少。当然，住在波士顿，很难忽略体育，所以我喜欢看一两场比赛，如果可能的话。
