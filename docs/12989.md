# 框架，到处都是框架

> 原文:[https://www.sitepoint.com/frameworks-frameworks-everywhere/](https://www.sitepoint.com/frameworks-frameworks-everywhere/)

框架是我最喜欢的话题。为了让每个人都在同一页上，[维基百科说框架](http://en.wikipedia.org/wiki/Framework)是“用于解决或处理复杂问题的基本概念结构”。更简单地说，在应用程序开发方面，框架是一组文件(代码)、约定和最佳实践，旨在帮助构建代码，以便于编程、协作和未来维护，并阻止开发人员在每个项目上重新发明轮子。

当我开始使用 ColdFusion 时，开发人员社区基本上不知道框架，只有一两个面向真正喜欢冒险的人的新兴产品。随着网络的发展，ColdFusion 已经变得更加复杂，以满足其不断增长的需求，可用框架的选择也绝对激增。

几周前，SitePoint 刊登了我对杰夫·鲍尔斯的采访，他是 FarCry 社区的“仁慈的独裁者”。虽然 [FarCry](http://www.farcrycms.org/) 更为人所知的是一个内容管理系统，但 FarCry 的构建模块充当了一个框架，允许开发者构建复杂的应用程序。就在上周，[就他的 COOP 框架](https://www.sitepoint.com/article/coop-coldfusion-framework)采访了约翰·法勒，该框架旨在实现快速原型开发和前端开发人员与后端开发人员之间的轻松协作。每次面试我都用同样的问题，这样就可以根据答案来比较不同的方面。他们试图帮助开发者决定一个特定的框架是否适合他们。

目前，我的头脑还在为同一套问题寻找答案， [Fusebox](http://www.fusebox.org/) 、 [Mach-ii](http://www.mach-ii.com) 、 [Transfer ORM](http://www.transfer-orm.com/) 和 [ColdBox](http://www.coldboxframework.com/) 也在努力工作，一旦这些答案出来了，我就会牢牢地盯着其他所有的框架开发者，无论他们是大是小(如果是你，请联系我——Kay 在 smoljak dot com)。在接下来的几个月里，我们可以期待看到每个框架的相似和不同之处。

但是首先，让我知道你的想法:你也是框架迷吗？你使用你自己的系统吗？或者你认为这都是一派胡言？让我知道你的想法。我会让投票持续一周。我很期待看到结果！
