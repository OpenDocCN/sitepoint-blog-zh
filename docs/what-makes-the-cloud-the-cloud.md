# 是什么让云成为云

> 原文：<https://www.sitepoint.com/what-makes-the-cloud-the-cloud/>

云计算是一个在过去几年里广为流传的术语。像“Web 2.0”一样，这是一个被公司采用并用于营销目的的术语，每个人都有自己的定义。在本周纽约 Web 2.0 博览会的一次会议上，联合广场风险投资公司的阿尔伯特·温格(Albert Wenger)阐述了他对什么使云成为云的想法，联合广场风险投资公司投资了一些云计算初创公司，如 [10gen](http://www.10gen.com/) 。

温格列出了云计算的四个原则，他认为这些原则定义了什么是真正的云。他的要求很简单，但相当严格，这让他得出了一个可能有争议的结论，即 [Google App Engine](http://code.google.com/appengine/) 是当今唯一真正可用的云计算平台。

## 4 云计算的定义原则

1.  不再有机器。Wenger 说，理想情况下，云计算是“后机器计算”。温格的意思是，在部署云基础设施时，你不必担心单个机器。也正是因为这个原因，他认为亚马逊的 EC2 是通往真正云计算道路上的一个过渡站。使用亚马逊意味着处理机器实例，所以它不是后机器计算。
2.  **代码超过配置。**云应该非常容易部署。设置应该基本上自己搞定，你没有理由浪费宝贵的时间去担心配置你的云设置。温格说，把你的时间花在编写实际的代码上，而不是建立你的云。
3.  没有失败的鲸鱼。云必须无痛苦地扩展，无论你增长多少，你的代码都应该像你第一次写它的时候一样工作。按照 Wenger 的定义，为了成为真正的云计算平台，代码需要在不对开发人员施加不合理期望的情况下进行扩展。
4.  它是可混搭的。根据 Wenger 的说法，云计算应该使 web 服务的整合变得非常容易。

因为谷歌应用引擎是目前唯一符合温格定义的平台，所以有时他在 Web 2.0 博览会上的会议感觉很像应用引擎的广告。温格滔滔不绝地说谷歌的平台几乎不需要任何配置，前 500 万页面浏览量是免费的。他展示了在该平台上构建的应用程序的例子，并吹嘘它们创建和展示的速度有多快。换句话说，他*真的*喜欢 App Engine。

但是开发者可以轻松上手 App Engine 并不是没有代价的。App Engine 几乎不需要配置的原因之一是它只适用于 Python。所以你得到了易用性，但是你付出的代价是缺乏灵活性。当我在展览大厅遇到亚马逊网络服务团队时，他们也是这么告诉我的。亚马逊犹豫是否将 AWS 与 App Engine 或任何其他云平台进行比较，但基本上说每个人都有不同的需求，App Engine 不会适合每个人。

## 结论

尽管温格显然喜欢谷歌的产品，但他也不希望看到谷歌主宰云计算。谁控制云是非常重要的，温格希望未来云是开放的，并且有一个健康的开发者生态系统致力于使它变得更好。

温格说，云计算仍处于早期阶段，未来它将“对我们的创新方式产生深远影响”。
