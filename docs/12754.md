# Isaac Dealey 关于 OnTap 框架的 10 个问题

> 原文:[https://www . site point . com/is AAC-dealey-on-the-on tap-framework/](https://www.sitepoint.com/isaac-dealey-on-the-ontap-framework/)

Isaac Dealey 是两个框架的作者:onTap 和最近的一个名为 DataFaucet 的 ORM(对象关系映射器)(未来框架访谈的主题)。以下是他对我的 10 个框架问题的回答。

见其他框架访谈:[杰夫鲍尔斯在 FarCry](https://www.sitepoint.com/article/geoff-bowers-farcry-framework) ，[约翰法勒在 COOP](https://www.sitepoint.com/article/coop-coldfusion-framework) 和[马克曼德尔在转会 ORM](https://www.sitepoint.com/10-questions-for-mark-mandel-on-transfer-orm/) 。

**1。嗨，艾克！向我们展示您的电梯间推介:用一两句话总结 onTap 的精髓。**

推销版本:onTap 框架为 ColdFusion 做了 ColdFusion 为 web 做的事情:简化了许多强大的工具。

不那么好卖的版本:onTap 框架是 ColdFusion 开发的 SOA 方法。

**2。让我们再深入一点:告诉我们更多关于主要特性的信息。**

版本 3.2 已经将其所有配置文件从现在的平面 CFML 包括转换为 CFCs，并包括一个新的 IoC 管理器。IoC 管理器和插件管理器为开发人员提供了一种结构，用于分发或销售可插入的、智能集成的应用程序和服务，这些应用程序和服务可以安装在浏览器中。在不久的将来，框架站点将托管一个 webservice，它允许那些“插件”应用程序直接从框架中安装，就像插件可以安装在 Eclipse 或 FireFox 3 中而无需离开 IDE 一样。

如果这听起来很吓人，不要担心。对于使用 onTap 框架开发应用程序来说，这些都不是必需的。框架本身对 OO 是不可知的，就像 Fusebox 框架是不可知的一样。您可以选择以“极端面向对象”的方式开发您的应用程序，使用 IoC 框架等等。或者，如果您喜欢，您可以只构建页面。我更喜欢 OO/SOA 方法。

该框架包括一个强大的 XHTML 库:一个强大的可扩展的模板引擎，它可以让您访问各种表单构建和 Ajax 工具。

我将这些 XHTML 工具与我称之为“分层包含”的独特核心架构结合使用。Fusebox 有 circuits 和 fuseactions，而 onTap 框架有一个“进程”，它类似于并实际映射到 URL 和文件路径。这种分层包含还为强大的品牌/定制功能提供了一些额外的基于目录的挂钩，并为任何需要或想要它们的人提供了一系列非常全面的国际化(i18n)功能。

这还没有涉及到框架中的各种其他零星内容，如内容缓存、符合 section-508 的 Ajax 小部件、一个基于 XML 的规则管理器工具，用于提供用户可以管理的强大的可配置业务逻辑(在其他框架中找不到)，等等。

**3。onTap 是如何产生的，创建它的原因是什么？你是怎么卷进来的？**

我是原作者。大约在我开始研究分层包含技术的时候，我已经用 Fusebox 3 做了很多工作。当然，这是一个不错的框架，最初我是它的忠实粉丝……随着时间的推移，我发现自己越来越不喜欢 Fusebox 3 中开关文件的重复。

分层包容的概念是我某一天突然想到的，我花了很长时间来完善它。对我自己来说，它总是提供了一个减少我写的代码量的好方法。框架的其余部分是后来才出现的，并且一直是对工具的无休止的追求，以使我自己的开发工作更容易——如果我找到了一种方法来创建一个通用的工具，在以后会有用，我会将它包含在发行版中。

**4。开发人员应该何时寻求 onTap？它有什么好处呢？**

onTap 框架的最大优势在于它能够实现无缝品牌和定制，以及插件应用程序之间的无缝自动化集成(两者都使用相同的技术)。

这些年来，我为许多 ASP 公司工作过，在我工作过的每一个 ASP 公司都有一个问题，那就是客户定制请求。客户总是想要系统没有设计提供的工具，然后找到一种方法来提供这些工具，这对于提供服务的公司来说总是有问题的，通常实际上在财务上伤害了公司。onTap 框架提供了一套全面的工具，能够很好地缓解实施客户端定制请求所带来的问题。

**5。使用 onTap 的先决条件是什么？有什么对新用户有帮助的先验知识吗？它是否使用了其他可以简化或复杂化事情的框架？**

了解一些 ColdFusion 显然会有帮助！唯一真正需要的是 ColdFusion 7，它应该可以在 Windows 和 Unix 操作系统上运行。我还没有尝试过更高版本的 BlueDragon 或 Railo。它不需要任何额外的框架，尽管如果您愿意的话，它包含了 ColdSpring 或 Lightwire 的内置集成。

说实话，onTap 框架背后的核心架构并不复杂，并且旨在非常简单易用，但是它在方法上不同于大多数其他 ColdFusion 框架。因此，如果您希望它像 Mach-II、Model-Glue、ColdBox 甚至 FuseBox 一样运行，您可能需要忘记一些复杂性，才能理解 onTap 框架是多么简单和强大。

当我在做自己的项目时，我花在一些事情上的时间不超过几分钟，而这些事情即使有一个像 ColdBox 这样的好框架，通常也会花去我几个小时的时间。我把这些额外的时间花在新问题上，而不是花在我已经解决了无数次的问题上，比如设计或验证一个表单。当我在日常工作中做项目时，我基本上认为我花在工作上的大部分时间是“浪费的时间”，因为我知道事情可以做得多快。

**6。是什么让 onTap 有别于其他框架？**

特别是三件事:
—cold fusion 开发的 SOA 方法
—定制和品牌化的强大工具(对 ASP 非常有用)
—简单

事实上，我觉得有趣的是，几年前我对几种 ColdFusion 框架进行了比较，当时没有多少框架，所以我只有 Fusebox 3 和 4、Mach-II 和 onTap 框架。当时，我真的对其他框架所需要的额外工作不再抱有幻想，特别是 Mach-II 所需要的许多行 XML。

在这期间，不仅仅是 Mach-II 和 Fusebox 采用了我的方法，其他框架如 ColdBox 也在不同的地方出现，它们都把我已经使用了很长时间的技术标榜为自己的优势。我这么说并不仅仅是为了自我安慰(尽管事实上我确实对此感到自豪)，而是要指出，onTap 框架中的几乎所有东西都一直“走在时代的前面”。我认为这应该与框架的寿命有关，我知道有些人担心这一点。

事实上，我认为 onTap 框架在许多方面更像 ColdFusion，而不是近年来创建的大多数其他 ColdFusion 框架。

7 .**。有哪些关于 onTap 的优秀例子？**

我必须诚实地说，还没有人告诉我任何消息。我真的希望我能说“啊耶，XXX 银行正在使用它”，但是我不能。

几年前，我用它为一家名为雷神的国防承包商实现了一个解决方案。我不敢说这是框架的一个例子，因为它还是一个早期版本。

我很兴奋得到这个消息！

**8。许可、社区、支持和文档呢？**

它以 OpenBSD 风格的许可证发布，比 LGPL2 的限制少一些，因为您不仅可以使用它创建商业软件(上帝知道我有自己的计划)，而且如果您愿意，还可以基于 onTap 框架加密您自己的软件。它只要求您注明您的发行版包含 onTap 框架，并且没有得到框架作者(目前只有我)的认可。

我目前正在非常努力地尝试和发展框架社区。我试图找到更好的方法来鼓励反馈和参与。我只是不擅长营销和人际交往技巧。然而，我对围绕这个框架发展一个更活跃的社区非常感兴趣，比如 Fusebox 或 FarCry 社区，它们有实际的开发。团队和活跃的论坛和邮件列表。我希望框架在某个时候有一个开发团队，而不仅仅是我。我希望它包括代码审查/批评和社区其他成员的代码贡献。我最近发表了一篇关于这个的博客，特别要求人们写下他们对这个框架的体验，好的坏的或者丑陋的。

框架的文档总是…嗯，冗长。我实际上已经删除了这个最新版本中的一些文档，因为我发现不仅有很多文档没有被阅读，而且这些年来我收到了许多开发人员的意见…他们不是抱怨，但他们会说一些事情，比如我在 cf.Objective 遇到 Fusebox 社区的 Jeff Peters 时，他提到了一个关于几天前查看文档并与其他人进行对话的轶事，其中的短语是“我必须阅读所有这些吗？!"被用过。

Model-Glue 的作者 Joe Rinehart 曾经在 cf-talk list 上发表了一条评论，大意是“感谢您为文档设置了如此高的标准”，因为他发现我的文档非常全面(读起来很长)。

**9。onTap 的未来会是怎样的？**

现在我正在努力让它回到自己的领域。我刚刚注册了 tapogee.com……很快我希望它能提供我之前提到的即时查找和下载插件应用的网络服务。我希望有了一个更活跃的社区，将会有许多基于插件的服务和应用程序可以一起工作，创建一个真正利用 ColdFusion 社区集体智慧的套件，这种方式是迄今为止其他框架所没有的(是的，甚至是 FarCry，我很大胆)。工作更少，成就更多。

为此，我也在考虑自己出版一本关于如何使用框架开发应用程序的小书，可能还有 t 恤衫和其他游击营销技巧。
