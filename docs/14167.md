# 机器的网络应用？

> 原文：<https://www.sitepoint.com/web-applications-for-machines/>

嵌入式系统已经存在很长时间了。最近，人们发现 Linux 对嵌入式系统的构建和部署非常友好，因为它具有灵活的许可、较低的入门成本，当然还有系统稳定性。

以一种迂回的方式，最近的 GNU GPL 法庭胜利指出了未来几年最终可能成为 web 开发者的主流市场——面向除了通常的 Internet/extranet/intranet 用户群之外的受众的 web 应用程序。

这是一个值得考虑的有趣领域，尽管它并不是全新的。一段时间以来，面向对象的软件开发技术和能力已经通过各种互联网语言渗透到 web 领域。现在，随着法律先例推动了对关键解决方案开源软件的审查和验证，互联网专业人士原本渺茫的垂直发展机会将进一步开放。

越来越多的“设备”由技术逻辑驱动，就像这个故事中处理汽车和控制其操作所有方面的逻辑一样。有些东西需要读取该行为，技术人员需要一个有能力的界面与之交互。

我们都曾开玩笑说，当牛奶不足时，冰箱会通知我们。现在我们差不多有了。请注意，很可能是 web 专业人员，他们能够开发出占用空间小的可伸缩系统，并且意识到系统资源，这种能力在这些环境中可能表现得最好。

技术人员和其他最终用户操作的嵌入式 Linux 系统可能最适合 kiosk 模式的基于浏览器的应用程序(我确信这在很多情况下已经被确定了)。

利用网络连接的知识和网络应用程序的跨平台特性(特别是开源解决方案)，网络老手可能会找到新的途径来营销他们的产品——并受到比预期更大的欢迎。
