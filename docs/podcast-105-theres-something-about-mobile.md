**Louis:**PhoneGap 的一个很酷的地方是，除了让你访问本地运行的 web 视图之外，它还可以绑定到一些更多的以设备为中心的 API，这些 API 是本地开发框架提供的。

**路易:**对。因此，如果有人想开始开发原生平台，你会建议先看看 PhoneGap，再看看一些原生平台；黄金时间准备好了吗？主要的问题是什么？

**路易:**对。所以我想这就结束了。我真正想让你们谈论的是一个为移动设备设计的问题，因为我想我们现在肯定正处于这一点上，它正变得非常主流，有很多设备和更便宜的设备，它们有很好的浏览器，所以你真的可以用它们开发全功能的网站。我想这是由杰里米·基思大约一个月前发表的一篇[博客文章引起的](http://adactio.com/journal/4367/)，这篇文章最近在 Twitter 上引起了一些评论，这让我开始思考这个问题。所以他的论点是，人们应该做的是对他们的常规网站进行响应性设计，但使用 CSS 媒体查询来使这些网站适应不同的屏幕尺寸。所以，我想这很有趣，因为他的论点是，有这么多不同的设备和这么多不同的大小，如果你想开发一个专门为移动设备服务的移动网站，然后开发一个完全不同的为桌面设备服务的网站，你会为平板电脑或上网本或 Kindle 或其他设备做什么？但我想你们可能对此有不同的看法，我想有四种方式可以选择，你可以在你的桌面网站上进行响应式设计，你可以开发一个只为移动客户服务的定制移动网站，你可以使用跨平台的 PhoneGap 之类的东西开发一个原生应用程序，或者你可以完全原生。我想知道你们，因为你们都参与了这个领域，你们对这些问题的看法是什么，因为我认为这是很多开发者现在开始关注的事情。

马克斯:是的，当然，我的意思是，我认为这真的取决于目标的类型，你想达到的目标和你的目标市场。我认为响应式设计很棒。

路易斯:也许有些听众可能不太熟悉，我知道我介绍得很快，但你能简单介绍一下基于媒体查询的响应式设计是什么意思吗？

**Max:** 是的，当然，响应式设计这个术语是指一种由[伊森·马科特在一篇开创性的文章中提出的技术](http://www.alistapart.com/articles/responsive-web-design/)，这个想法是你在 CSS 中使用媒体查询，你可能通常会看到使用，比如通过指定@media print 或@media screen， 但在这种情况下，您实际上可以查询像素尺寸或一系列其他内容，如 DPI 和用户屏幕的方向，然后根据设备的功能或屏幕大小为它们提供不同的 CSS， 这意味着你可以重新设计你的页面的样式和布局，这样它们就可以很好地适应任何特定的尺寸，从 320 像素宽的 iPhone 到 2700 像素宽的巨大的桌面屏幕，你可以调整你的布局，让它适应用户浏览器或设备的特定分辨率。

**路易:**对。因此，如果有人想看一看，我知道有一个网站是这种设计的展示，它位于媒体查询处，所以最后两个字母是域名，所以它是 [mediaqueri.es](http://mediaqueri.es/) ，有一堆非常好的设计，如果你在查看它的时候调整浏览器窗口的大小，你会看到它如何重新流动并适应一个非常窄的浏览器窗口，就像你在移动设备上看到的那样。但我想有些人对此有异议，所以有些人会说开发一个成熟的移动网站更好，你认为你为什么会想这样做而不是选择基于媒体查询的响应式设计？

Max: 当然，我的意思是，如果你想把基本相同的信息传递给你的手机用户，我认为媒体询问是好的，所以在某些情况下这是完全合适的；例如，我们的公司网站，我们并不真的需要在移动设备上比在桌面上给任何人更多或更少的信息，所以使用媒体查询很有意义。但是对于更强大的应用程序或 web 应用程序，人们试图实现某些事情，那么你可能想要约束或限制网站的功能，并限制人们可以做的事情的范围，使其更适合移动设备。所以 Twitter 是一个很好的例子，主要的 Twitter 网站相当复杂，功能齐全，但移动 Twitter 网站只是让你了解 Twitter 的核心，即查看时间线，更新最新的推文，查看朋友的内容；在主要的 Twitter 网站上你不会得到额外的东西。

**Myles:** 然后是 Twitter，你可以再举一个 Twitter 的例子，因为他们的本地应用是他们大部分流量的来源，所以这是一个很好的例子，Twitter 应用是一个本地应用，你可以在主屏幕上点击它，它就会弹出来为你做所有的事情，而不是你必须等待屏幕重新加载或在移动浏览器中运行 HTML。所以，是的，你说的对极了，这都是关于目标的适合性。

马克斯:对，我想没错。

路易斯:所以，在这种情况下，杰里米·基思似乎采取了非常强硬的态度，他说这真的永远不会是合适的；如果你在为网络设计，你必须为每一个可以想到的设备设计，这意味着要做响应式设计。你对此有什么看法，这是一个恰当的说法还是说得太过了？

迈尔斯:老实说，我觉得这有点过火了。我的意思是，你看——我能明白他的观点是什么，上帝知道关于不同屏幕分辨率和不同屏幕尺寸的喋喋不休持续了多久，现在我们有了不同的设备，但问题是你看一下类似平板电脑形式的东西，你可能有一个 7 或 9 或 10 英寸的屏幕，这些东西的分辨率与更小的桌面屏幕相当，所以通常你不需要为这些设备提供任何不同的东西；显然，手机的屏幕要小得多，你的空间要小得多，你需要更好地使用它，所以这里有一个论点，但任何超过 1024 x 768 屏幕的东西无论如何都是桌面，除非你有一些疯狂的 UI 交互东西，因为显然捏手势和其他手势会使其他东西不以同样的方式工作，但这是一个完全不同的论点。 这不是一个 CSS 媒体查询问题，这通常是一个 JavaScript 问题，我认为这实际上是一个更重要的问题。

Max: 是的，我有一半同意 Jeremy 的观点，我认为能够使用响应式设计原则来构建所有的设计是很棒的，但这并不总是现实。我认为响应式设计会很快变得复杂，这是你需要考虑的事情，如果你正在做一个响应式设计，你经常需要这样做，我只做了三四个，但是你需要适应你的——比如你的 HTML 的结构，这意味着你的桌面用户可能会得到比你本来会得到的稍差的体验，因为你试图适应每个人，我认为你需要看看；实际上，看看你的流量，看看你是否让人们在移动设备上访问，如果你不是，那么这是你可能不想考虑的事情，但如果你是，那么你一定要考虑它。

**路易:**对。我想其中一个因素是——有两个因素，其中之一是人们浏览的环境，所以我的意思是我今天早上在采访之前快速浏览了一下我之前谈到的媒体查询库， 看起来，大多数网站要么是博客网站，要么是代理组合网站，它们都是这种事情的演示，你知道，你想向用户提供的内容是完全一样的，不管它们来自哪里。

迈尔斯:对，没错。

**Louis:** 但我想有很多网站，比如我的银行网站，当我用手机访问时，它不是媒体查询改编版的完整网站，所有内容都挤在一个栏里，这是一种完全不同的体验，我只有三种选择:登录网上银行，找到一个网点，或者给我们打电话。

迈尔斯:是的，他们不会在你打手机的时候向你推销房屋贷款。

路易:(笑)完全正确。我就在公车站，就像哦，等等，你知道，我现在真的可以去申请抵押贷款(笑声)。

**Myles:** 是的，我认为它的核心是媒体查询从本质上重塑了相同的内容，而一个独立的移动网站真正针对的是特定环境中的用户。

**路易:**对。我猜另一个影响因素是带宽要求。有时媒体查询设计，取决于你如何做，但它可以，如果你做的图像适合 960 像素的分辨率，但它们在 CSS 中缩小，那么我的意思是，对于移动宽带的服务器来说，这可能是一个相当大的图像。

Max: 是的，我的意思是，你会看到一些非常大的网站，比如新闻网站，在这些网站上，你每次上头版都会下载价值 50 万英镑的内容。对他们来说，在手机上提供不同版本的网站是有意义的，因为你不想让人们在公共汽车站等着半兆的东西从网上下来。这是一个问题，一直以来都是，我的意思是这也不是一个新的争论，我们多年来一直有这种关于移动设备的东西，只是我们现在已经有了可以支持这种东西的实际浏览器，以前我们在移动设备上的带宽甚至更少，这甚至是一个更大的问题。但是，如果你要给手机用户提供同样多的东西，那么，是的，你做错了。

路易:对，很好。我应该采取下一步，所以说好吧，有人可能已经做出决定，在我的网站上我有一个单独的移动版本，我正在做一些服务端的恶作剧，以决定何时提供服务，下一步从网站到原生应用程序，你认为可能导致的决定因素是什么，或者使这成为一个好的判断，使这成为一个好主意？

**Myles:** 基本上你能从一个原生应用中得到什么，我的意思是这更进了一步，当你为一个你正在为之构建的原生应用而构建时，你现在需要考虑一大堆其他突发事件。第一个是，它需要在没有任何网络连接的情况下工作，或者至少优雅地失败，如果这是一个常规网站，你不必担心，因为他们无法到达那里。但我的意思是，你可能会注意到的第一件事是，你不会将你的博客网站转换为原生应用程序，这可能不太可能，所以你首先要考虑的是，它是否会引起人们的兴趣，并且首先成为原生应用程序是否有意义。但是如果需要的话，你必须考虑尽可能地使用本地存储来缓存东西，对网络故障做更多的处理，还有用户与应用程序交互的方式；当他们在网络浏览器中的时候，他们希望东西可以重新加载，当你点击链接的时候会有一点延迟。当你在本地运行时，他们想要更快一点、响应更快一点的东西，所以很多时候你可以把你在网上做的东西放到本地应用框架中，这可能不会那么容易发生，你必须做一些工作来重新调整你正在使用的一些东西。

是的，我认为你需要决定你是否认为人们会使用它是主要的事情。就像你说的，如果它是你的博客网站，那么人们可能会下载并安装在主屏幕上，每天使用。我的意思是，只有当你认为你的用户会很好地使用它，并且值得你花时间的时候，跳到一个原生应用才是值得的。

路易斯:对，没错。我想在某些情况下，你需要的功能是无法通过浏览器获得的。你之前谈到了 Twitter，我猜 Twitter 原生应用程序允许你上传照片和你的推文，这在网站上是做不到的，至少在大多数移动浏览器上是做不到的。或者直接从相机上拍照，我想这在手机上是不可能的。

**Max:** 我想说的是，在关于媒体查询的讨论中，我看到了一个很好的资源，叫做 Yiibu.com 的[，他们做了一些思考——他们写了一篇文章叫做](http://yiibu.com/)[反思移动网络](http://yiibu.com/articles/rethinking-the-mobile-web/)，我认为他们谈论的许多适用于媒体查询的东西非常有用， 它的基本思想是首先为移动设备设计，然后构建一套—为您的桌面用户提供更多信息和更重的设计，从而在一定程度上否定我们之前讨论的移动用户的网络访问和额外资产问题，具体做法是尝试为移动用户提供默认设置，然后针对桌面用户，而不是相反。 这是响应式设计最初被想到的方式，你不会强迫移动用户下载很多他们不会用的东西，但桌面用户会，这对他们来说不是什么问题。

**路易:**对。这确实带来了一件有趣的事情，我不记得我最近在哪里读到了这篇文章，但我确实记得看到一些网站的用户报告说，你知道吗，即使当我在桌面上时，我仍然更喜欢你的网站的移动版本，因为没有这些垃圾，我需要游过才能找到。

Max: 因为你把所有的广告都藏起来了(笑声)。

Louis: 这仅仅是广告，还是你认为简化的用户界面有什么值得一提的？

Max: 是的，我认为这绝对是为了简单。
