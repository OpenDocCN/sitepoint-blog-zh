# Java 要开源了？

> 原文：<https://www.sitepoint.com/java-going-open-source/>

在过去的几周里，Sun 一直在思考如何让 Java 更上一层楼。我没有重复我们大多数人在别处读到的内容，而是去找了一位该领域经验丰富的 Java 开发人员，他曾为通用电气(General Electric)以及金融服务和游戏行业开发过基于 web 的 Java 应用程序。

泰森·洛维里是 Teelo Technologies 的高级开发人员和所有者，他认为许多开发人员并不担心 Java 的地位，因为软件开发工具包(SDK)通常能满足他们的需求。当需要更多的时候。有许多库可以填补空白。

“Java 库提供了许多附加功能。洛维里说:“看看 Java 的核心能包含哪些功能会很有意思。”。例如，XML 解析器是一个独立的第三方库。我认为如果 Sun 将这些和其他类似的库包含到核心中，这将是一个帮助，因为开发人员不必安装和下载额外的 jar 来使用这些类。”

根据洛维里的说法，积极的影响可能是使 Java 的新特性在未来来得更快。他引用了 Java 5 中的类型安全枚举。

洛维里解释说，这是一种将对象放入数组列表或堆栈中的能力，当你从列表中检索对象时，你知道你得到的是相同类型的对象。

“多年来，程序员一直在开发自己的库和方法来解决这个问题。我认为如果 Java 是开源的，它很久以前就会发现自己是 Java 的核心了，”他补充道。

就他而言，洛维里认为开发人员可以帮助发现 Java 类的一些低效之处——但在语言的现阶段，他认为这些不足是很少的。

“但总的来说，我不认为这是大多数程序员的游戏规则改变者，”洛维里说。
