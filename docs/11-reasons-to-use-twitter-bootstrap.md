# 使用 Twitter Bootstrap 的 11 个理由

> 原文：<https://www.sitepoint.com/11-reasons-to-use-twitter-bootstrap/>

Twitter 的流行已经帮助它从用户管理的社交网络领域跨越到一个开发者埋头于重型代码并因此使世界旋转的地方。Bootstrap 框架是设计和开发领域的最新创新，使网站和应用程序的创建更容易、更快、更好。如果你还没有使用 Twitter Bootstrap，是时候看看了。Bootstrap 对于开发人员来说就像一个装满各种工具的糖果店。

那么为什么要使用 Bootstrap 呢？

## 1.节省时间

有了 Bootstrap，你要做的第一件事就是利用大量的时间。Bootstrap 库提供现成的代码片段，可以为网站注入活力。web 开发人员不必花时间费力地工作和编写代码，只需找到正确的部分并将其放入他们正在工作的结构中。除此之外，很多样式和设计方面的问题已经解决了，因为 CSS 是用更少的代码构建的。

## 2.可定制的

Bootstrap 的一个很好的方面就是你可以把它变成你自己的。你可以坐下来翻遍整个框架，保留你需要的，抛弃你不需要的。Bootstrap 的全部意义在于，它允许您考虑自己的需求，并相应地定制您的开发项目。这是许多开发人员觉得一些工具在 Bootstrap 上完全没用，而其他人却推崇这些工具的主要原因。

## 3.设计中的因素

### 网格会说话

创建页面布局需要一个像样的网格。使用平台的网格并不是强制性的，但是它确实让你的工作变得更加容易。在默认模式下，平台提供 940 像素宽的 16 列网格。一列容纳 40px，另加 20px 作为装订线。可以根据需要操作行和跨度。样式是在自动模式下完成的，开发人员只需将内容放入他们正在处理的 HTML 中就可以完成这项工作。此外，如果您需要处理列和嵌套，那么您已经有了合适的网格来完成这项工作。

### 少即是多

事实上，在发展圈里，很少有人暴跳如雷。有了它对 Bootstrap 平台的补充，您可以使用更少的混合和 CSS 操作来定制内置网格。事实上，每次变量转换后，更改都会自动更新，这也没有坏处。Bootstrap 采用了一些最流行的 CSS3 实用程序，并将其作为所有浏览器上的同质反应。

### Java Script 语言

Bootstrap 配备了 JavaScript 库，超越了基本的结构和样式。JavaScript 经常成为网页设计中不可或缺的一部分，开发人员必须将所有元素整合在一起，才能看到最终的形式蓬勃发展。使用 Bootstrap，开发人员可以轻松地操作模态窗口警告、工具提示、Scrollspy、Popover、Button、Typehead 等。然而，最好的部分是，Bootstrap 使您可以完全跳过编写脚本。

## 4.一致性

开发这个工具集的一个主要原因是因为 Twitter 发现从事他们项目的开发人员之间存在很大的不一致。这在开发方面和最终用户方面都引起了一些问题。因为 Bootstrap 使用一组集中的开发代码，所以它的结果在不同平台上是一致的。你会在 Internet Explorer、Chrome 和 Firefox 上看到同样的事情。

## 5.更新

这里以 JQuery UI 为例:用户一年可以获得两次更新。另一方面，Bootstrap 是不断发展的，更新会更加定期和持续。一旦 web 开发人员发现问题，Bootstrap 团队就开始着手修复它。

## 6.综合

如果你在一个已经上线的网站上工作，但仍然试图消除这些折痕，那么 Bootstrap 会有所帮助。例如，如果您使用表格样式，您所需要做的就是将您需要的样式复制到您正在使用的 CSS 文件中。Bootstrap 将立即以自己的风格投入使用，你需要做的就是链接你正在处理的文件或热链接到 Twitter(或多或少)。集成简单、快速且容易完成，一旦完成，您就可以随心所欲地玩您的设计。

## 7.响应性

自举是有反应的。如果你从笔记本电脑转移到 iPad，从 iPad 转移到 Mac，你就不必为你的工作烦恼了。Bootstrap 以超快的速度和效率适应平台的变化。

## 8.未来兼容性

Bootstrap 配备了许多被认为是未来设计的元素。例如，HTML5 和 CSS3 都是未来的大趋势。由于该框架考虑了设计和开发的未来，它有可能在未来几年成为 web 开发人员的衡量标准。

## 9.竞争力

Bootstrap 并不是唯一的同类产品。它与 JQuery UI 和 [HTML5 样板](https://www.sitepoint.com/introduction-html5-boilerplate/)竞争，但是一个更现实的竞争对手是 Zurb 基金会。Bootstrap 2 有一个工具集，基金会将很难与之匹配。第三方插件，主题，过多的功能，代码等。已经可以通过 Bootstrap 获得了——这对于基金会来说是不正确的。你们中的一些人可能会觉得 Bootstrap 有一种臃肿的感觉(Foundation 没有)，但这可以通过自定义 Bootstrap 下载构建器轻松消除。

## 10.医生会说话！

Bootstrap 的文档令人印象深刻。大多数较新平台的情况是它们没有适当的文档，但是 Bootstrap 为您提供了一个良好的开端，因为您不必学习所有内容，相反，您可以通过文档找到关于任何内容的信息。

## 11.对老板们来说

你不需要花很多时间来学习使用 Bootstrap，因为它已经采用了所有的模块化方法和良好的架构，并将它们结合在一起。从雇主的角度来看，如果你给一个 web 开发者一个 Bootstrap 框架，那么与 HTML 和 CSS 相比，他们不需要花费太多的时间去学习。Bootstrap 允许立即进行高质量的工作，这对于促进高效的、以结果为导向的设计交流非常有用。

## 结论

读完所有这些之后，如果你认为 Bootstrap 可以帮助你提高你作为一个 web 开发者的水平，那么看看这里的平台:twitter.github.com/bootstrap/.你可以下载一个编译版本或者获得 Bootstrap 源代码，里面有原始的 CSS 和 JavaScript 文档。

请注意，您确实需要 HTML 和 CSS 等 web 开发概念的工作知识来开始工作。平台附带的文档可以帮助您了解将要使用的不同工具和组件，例如基本 CSS、脚手架、JS 插件等。如果您还需要任何其他信息，请访问 Bootstrap docs。
