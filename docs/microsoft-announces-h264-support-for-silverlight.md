# 微软宣布支持 Silverlight 的 H.264

> 原文：<https://www.sitepoint.com/microsoft-announces-h264-support-for-silverlight/>

![](img/6be2a0da95efb053d3ab2ba41f30430b.png "silverlight-logo")在本周晚些时候于阿姆斯特丹召开的 IBC 会议之前，[微软今天宣布在其 Silverlight API 中支持【H.264 和 AAC。微软将展示一个支持 H.264 和 AAC 的 Silverlight 工作技术演示，尽管微软开发平台和工具总监 Brian Goldfarb 告诉我，它要到 Silverlight 3 才会对公众开放。Silverlight 2 预计将于今年秋季晚些时候发布。](http://www.microsoft.com/presspass/features/2008/sep08/09-09silverlight.mspx)

戈德法布告诉我，微软并没有放弃 VC-1 或 Windows Media，仍然支持它作为他们的视频编解码器的选择，而是他们认识到消费者需要一种选择。Adobe 开始在 Flash Player 9 中支持 H.264 视频和 AAC 音频，苹果公司长期以来一直依赖它们作为标准的视频/音频编解码器，这使得这对组合在整个行业都很受欢迎。

当我与戈德法布通话时，我问他关于 NBC 最近决定选择 Flash 作为他们的 NFL 周一晚上足球网络视频流应用程序，而不是 Silverlight，后者为他们的奥运会网站提供支持。读写网最近暗示，NBC 似乎“T0”改变了主意，并且已经回到 Adobe 和 Flash 的 NFL 应用程序上。上周末，当 NBC 网站在巨人队对红人队的揭幕战中首次亮相时，网络上的许多网站都重复了这一观点。《RWW 邮报》实际上是根据《硅谷内幕》的一篇文章[改编的，该文章暗示了类似的事情。](http://www.alleyinsider.com/2008/9/nfl-nbc-tap-adobe-s-flash-for-live-football)

当我提出这个问题时，戈德法布为自己辩护，称之为“愚蠢和无知”的观点。我被告知，NFL 实际上是在继续上个赛季在 Flash 流媒体技术上的投资，而不是做出新的决定来使用 Flash 而不是 Silverlight。我无法让戈德法布具体说明这个决定是 NBC 的还是 NFL 的，但我怀疑联盟在技术选择上有更多的发言权。上周，当我与 Adobe 谈论 NFL 网站时，我问了他们同样的问题，他们只回答说 Adobe 与 NBC 有着良好的关系。

微软告诉我，NBC 正在开发其他使用 Silverlight 的项目，将在未来几个月宣布。戈德法布还明确表示，他认为 Silverlight 或 Flash 将“接管世界”的想法是愚蠢的。他告诉我，两者都有存在的空间，他们可以一起工作。

就奥运会而言，微软认为我们把金牌授予 Adobe 可能为时过早。奥运会对微软来说绝对是一个净利好。在登陆 NBCOlympics.com 页面的 4000 万访问者中，微软告诉我，大约 50%的人在第一次访问该网站的几天内就安装了 Silverlight，如果他们还没有安装的话。该公司报告说，他们看到该插件“每天多达 150 万次下载”，并看到由于奥运会，美国市场增长了 30%以上。
