# 使用数据支持的人物角色来塑造 UX 的设计决策

> 原文：<https://www.sitepoint.com/use-data-backed-personas-to-shape-ux-design-decisions/>

使用人物角色来塑造网站设计决策的想法并不新鲜。

人物角色代表你网站的访问者；也就是说，他们是虚构的游客。每个人物角色都有一个名字、照片和个人简介，包括他们生活、工作或学习的地方，对他们来说什么是重要的，以及他们的动机是什么。参与网站建设的团队开始了解这些所谓的个人，就像他们最好的朋友一样。然后，当需要做出设计决策时，团队可以问:“朱莉会怎么做？”或者“这会对弗兰克有什么影响？”

人物角色在设计过程中实现了几个目标:

1.  它们为团队中的每个人提供了一个共同的基线，在此基础上讨论网站的访问者可能受到的影响。
2.  他们让设计团队*与现场观众*产生共鸣。通过将人物角色称为真实的人，目的是团队将认同每个访问者，并提供一个满足尽可能多的用户的设计解决方案。

## 为你的站点创建人物角色

我创建人物角色的过程结合了两种方法，我非常感谢他们背后的两位作者提供了这么好的起点。在为你的网站创建人物角色的最佳方法上有很多不同的观点，但是我在这里选择的是对我来说最有意义的一个。

我创建的人物角色基于以下两篇文章:

*   Andrea Wiggins，boxesandarrows.com
*   [开始构建人物角色](http://www.futurenowinc.com/resources/FutureNow_Getting_Started_with_Building_Personas.pdf)作者 Howard Kaplan，FutureNow Inc. (PDF，4.8MB)

我遵循的基本步骤描述如下。希望你会发现它们在创建你自己的角色时很有用。

1.  决定你要创建的人物角色的数量。在整个过程中，当你更仔细地检查你的分析数据时，你确定的数字可能会改变，但你可能已经知道可以识别多少类型的访问者。一些专家认为四五个就足够了。
2.  **分析 30 天内从您网站的匿名分析数据中获得的各种访问者特征的分布情况**—人们居住在哪个城市和国家，最常用的浏览器和操作系统是什么，网站的哪些部分是最常去的，哪些外部网站提供了最多的推荐，哪些短语构成了将搜索引擎访问者引向我们的最常见查询，以及新访问者和回头客的细分情况。我将这些类别中的最高结果分布在电子表格的五行中，当这些数据组合在一起时，就形成了我的五个角色。
3.  **分发出现在你的分析数据中的共同特征**—例如，如果你的访客中有 60%在美国，那么你的五个角色中有三个是美国人。然而，出于商业原因，您可能希望包含一些特征，尽管它们在您的站点数据中并没有得到很好的体现。例如，您的组织可能致力于满足具有特殊可访问性需求的用户，即使这些用户并不代表您的客户群的重要部分。因此，为了确保你“言行一致”，在你的人物角色中表现残疾用户可能是有意义的，即使他们是少数。
4.  一旦这些基础就位，**添加您可能从其他来源获得的任何额外数据**。例如，您过去可能进行过客户调查。有了这些调查的结果，你就可以推断出哪些新出现的人物角色最有可能在你的网站上表现出特定的行为——例如，订阅时事通讯或 RSS 提要，在社交媒体上关注你的组织，或者进行销售询价。
5.  叠加这两个完全不同的数据集可能是一个任意的过程，但没关系。我通常发现，通过加入这些额外的数据，我的人物角色才真正开始成形。有了这个基础，我就可以**推断出我的新代表用户团队在访问网站时可能正在做的某些任务**，比如寻求帮助、订阅时事通讯或在评论中提问。这些任务构成了我的用户的目标。
6.  如果你的网站包含一个电子商务方面，这下一步将是重要的(如果你不在你的网站上销售产品，你可能想跳过它)。**将霍华德·卡普兰的文章中描述的四种购买行为**——*竞争性*、*人性化*、*有条不紊*和*自发*—分布到你的人物角色中。虽然我不一定完全接受 Kaplan 和 FutureNow 团队在咨询实践中使用的“说服架构”概念(这是一个注册商标术语)，但我确信不同的人会做出不同的购买决策，这四种购买类型的描述引起了我的共鸣。当然，你可能无法知道这些性格类型在你的访问者中的分布情况——我通常在我的人物角色中创建偶数个性格类型。
7.  最后，**用有趣的信息**充实你的每个人物角色，给他们一个合适的背景故事和真实的个性。每个角色都应该有一个描述性的标题；例如，“足球妈妈”或“热情的学生”这是练习中最像创造性写作的部分，涉及到购买行为类型的一点改变，以匹配已经出现的个性。给你的人物角色一张脸——如果除了你之外的任何人会看到它们，记得使用你拥有版权的图像——给他们起一个可信的名字，符合他们的背景故事。我从[随机名称生成器](http://www.kleimo.com/random/name.cfm)中给我的角色取了名字，从 [iStockPhoto](http://istockphoto.com/) 中购买了一些图片，我的角色就完成了。

最终的结果是一套人物角色，无论是现在还是将来，当我在网站上工作的时候，我都可以在做任何设计决策的时候把它放在心上。显然，随着网站受众的发展，人物角色可能需要随着时间的推移而发展，但你可以看到准确的人物角色在帮助你描绘你为之创建网站的人时是多么方便。
