# 谷歌承认 WiFi 窥探

> 原文：<https://www.sitepoint.com/google-wifi-snooping/>

谷歌已经为从开放的 WiFi 网络收集数据样本道歉。当德国数据保护局(DPA)要求审计街景汽车获得的 WiFi 数据时，问题出现了。

在谷歌 2010 年 4 月 27 日的[原始声明中，该公司承认获得了:](http://googlepolicyeurope.blogspot.com/2010/04/data-collected-by-google-cars.html)

1.  街景照片。
2.  WiFi 网络信息，如 SSID 数据(网络名称)和 MAC 地址(分配给无线路由器的唯一编号)。这些数据用于改进基于位置的服务，作为 GPS 的替代方案。
3.  用激光获得的几何数据有助于改进 3D 地图。

然而，该公司否认收集有效载荷数据，即通过不安全网络发送的信息，如电子邮件内容。

经过进一步调查，谷歌发现有效载荷数据已经被下载。在[更新的帖子](http://googleblog.blogspot.com/2010/05/wifi-data-collection-update.html)中，工程&研究部高级副总裁阿兰·尤斯塔斯表示:

> 现在很清楚，我们一直在错误地从开放的(即无密码保护的)WiFi 网络收集有效载荷数据样本，尽管我们从未在任何谷歌产品中使用过这些数据。
> 
> 然而，我们通常只收集有效载荷数据的片段，因为:我们的汽车在移动；有人需要在汽车经过时使用网络；我们的车内 WiFi 设备每秒钟大约会自动切换五次频道。此外，我们没有收集通过安全、有密码保护的 WiFi 网络传输的信息。
> 
> 那么这是怎么发生的呢？很简单，这是一个错误。

为了纠正这种情况，该公司:

*   阻止街景车收集任何无线网络数据。
*   隔离数据，并与当地监管机构合作处理信息。
*   雇佣第三方审查软件、收集的数据以及是否已被删除。
*   开始对控制和程序进行内部审查。

爱尔兰数据保护局是第一个要求删除在该国收集的所有有效载荷数据的政府组织。

这是一个尴尬的问题——尤其是对一家依靠信任进行交易的公司而言。如果不是因为 DPA 的审计，谷歌会承认隐私泄露吗？
