# 微软改进 Internet Explorer 9.0 的 5 种方法

> 原文：<https://www.sitepoint.com/5-ways-microsoft-could-improve-ie9/>

不管你喜欢它还是讨厌它，Internet Explorer 将会一直存在。微软正在开发 IE9 的消息让人喜忧参半，而[在开发者中引起了激烈的争论](https://www.sitepoint.com/forums/showthread?t=647732)。

与其痛斥微软和他们的浏览器，不如让我们为 IE 开发团队提供一些建设性的建议。以下是我的 IE9 五大愿望清单:

## 1.改进标准支持

IE 远远落后于竞争对手，微软一直在追赶，而不是创新。

特别是，拜托，拜托，拜托我们能不能有原生 <acronym title="Scalable Vector Graphics">SVG</acronym> 和 canvas 支持。这个问题不断被问到，微软继续忽略它。我怀疑有一个重要原因:Silverlight。微软想增加 Silverlight 的追随者，可能认为 SVG 和 canvas 是相互竞争的技术。

来吧微软——让你的浏览器成为最好的基于标准的平台，Silverlight 用户就会到来。

## 2.做出关于 HTML5 的决定

过去，微软以身作则。然而，该公司对 HTML5 采取了观望政策，既没有完全接受也没有完全拒绝该标准。

HTML5 可能仍然是一个草案规范，但如果没有微软的参与和支持，它将永远不会成为一项可行的技术。微软——请做出这样或那样的决定，让我们逃离开发的困境！

## 3.坚持使用 Trident，但定期更新

一直有传言称，微软将改用 [WebKit](http://webkit.org/) 渲染引擎。这永远不会发生，也不应该发生。

我们已经有了两个主流的基于 WebKit 的浏览器:Safari 和 Chrome。如果 IE 改变了，WebKit 将成为世界上最主要的渲染引擎。这将使网络开发变得非常容易，但会减少竞争，限制创新。我们经历了 IE6 统治的许多年，任何组织都不应该再次拥有这种权力。

三叉戟无疑在进步，但它的发展速度非常缓慢，每隔几年就会更新一次。微软——让我们定期进行增量改进。

## 4.清理界面

IE 的界面一塌糊涂。IE6 干净简单，但 IE7/8 看起来像一堆反刍的 Windows 控件。它丑陋、混乱、浪费空间、在奇怪的地方有选项，并且妨碍而不是帮助用户。

部分问题在于微软坚持用时髦的营销名称添加不必要的功能。加速器和网页快讯有多成功？有多少人明白自己是什么？

微软:专注于基础，让网页浏览变得简单。用户应该能够忽略应用程序，而不需要*学习*它。

## 5.更好的开发工具

IE8 的开发者工具是对之前版本的巨大改进，但它们看起来仍然像是在最后一刻匆忙添加到浏览器中的。微软开发了一些世界上最广泛使用的开发工具，所以他们没有理由不提供高质量的浏览器跟踪和调试工具。至少，微软应该让第三方开发人员能够轻松地接入渲染引擎。

我错过了你对 IE 的最大愿望了吗？欢迎评论，但让我们保持积极的态度，伙计们…我们不需要 57 个*“我希望 IE 去死”*帖子！
