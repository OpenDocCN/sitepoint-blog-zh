# Visual Studio 2008 回顾

> 原文:[https://www.sitepoint.com/visual-studio-2008-review/](https://www.sitepoint.com/visual-studio-2008-review/)

我知道这有点晚，但正如我所承诺的，这里有一个关于 Visual Studio 2008 的回顾。我拥有它已经有一段时间了，但是我想等到我用它做了几个项目之后再发表我的意见。

总体而言，与 2005 年相比，这是一个坚实的进步。微软的家伙们已经做了一些很好的改变，实际上改善了开发体验，而不仅仅是添加了一些无用的东西。与 2005 不同，您不需要升级到最新的框架来使用它，所以您不必升级到。NET 3.5 刚刚好。IDE 通常比 2005 快一点，一些在 2005 年让我的团队头疼的特性已经变得有用了。两个最显著的改进是在设计视图中支持嵌套母版页，以及在表适配器编辑界面中的巨大性能改进。

我注意到的关于界面的第一件事是来自 Expression Web 的影响。设计器有一个新的拆分视图和几个新的 CSS 辅助工具。拆分视图在其他产品中已经存在很长时间了，比如 Dreamweaver，我通常不认为它非常有用，最初，我对 Visual Studio 中的拆分视图也有同样的感觉。在布局页面时，它确实没有多大帮助，但在编写代码隐藏时，它提供了一种更快捷的方法来查看布局和服务器控件属性。CSS 工具，很像 Expression，看起来对那些不熟悉 CSS 的人有用，但对我来说，它们更多的是阻碍而不是帮助。

最大的改进是 javascript 支持。对 ASP.NET AJAX 扩展的支持有了很大改进，但这是意料之中的。让我惊讶的是对 javascript 的广泛智能感知支持。它不仅选择内置的 javascript 函数，还为任何自定义函数和变量提供智能感知。虽然我的 javascript 技能一般，但我们专注于 AJAX 的人一直在谈论新的 javascript 支持有多好，这告诉我它非常可靠。新的 javascript 调试支持也很好，但它要求你在 IE 中打开脚本调试器，这使得除了调试应用程序之外的任何浏览都很麻烦。

Visual Studio 2008 是 Visual Studio 向前迈出的一大步。虽然有些人希望有更多的功能，但这些功能是可靠的，整体性能也有所提高。javascript intellisense 和调试本身就值得任何大量使用 AJAX 的人使用，性能调整也会满足像我这样没有耐心的程序员。有一些缺点(它仍然是一个内存猪),但改进是值得升级的。
