# 公共资金，私人代码

> 原文：<https://www.sitepoint.com/public-money-private-code/>

这是由合同程序员 Kurt Linghor 在澳大利亚发起的一项新倡议的名称——关注澳大利亚公共部门对开源软件的采用。

这项初出茅庐的努力是在 pm-pc.org 的网上发现的，旨在创建不同级别的政府组织使用开源解决方案的在线成绩单。林戈尔希望通过采访这些机构的 IT 经理来实现这一目标。

这些年来，倡导为开源社区做了大量的好事，然而，随着开源的成熟和成为可行的企业材料，我们也必须小心行事。我们不会在大规模专有供应商的脚下发抖，但是考虑到商业企业，我们寻求投资于我们的开源努力。尽管这可能令人不快，但在让过去的技术经理接受管理层的采纳方面，这仍然是一个感知和表达的问题。我们如何宣传和询问会影响那些持有公司支票簿的人对我们的请求的反应。

也就是说，林戈尔在他的媒体新闻稿中宣布公共资金和私人代码的范围时显得有些强硬。根据林戈尔的说法，运行我们社会的信息和系统不应该受到专利软件商的摆布。这可能会让一些技术经理失去兴趣，因为他们不能把这样的意见告诉他们的上级。

平心而论，林戈尔的项目基础是相当健全的。他认为争论停留在总拥有成本(TCO)的争论上(没错)。“这不仅仅是一个货币问题，”他说。他认为，继续推进的核心问题是开源软件的同行评审过程，这在很大程度上确保了安全和漏洞防范的适当修订。

我很有兴趣看到 Linghor 开发的报告。这与其他研究公共部门采用开源技术的组织是一致的，[早些时候在这里写了博客](https://www.sitepoint.com/blog/)。

显然，政府和研究的采用率将影响商业采用，因为开源解决方案已被证明是企业就绪的。或许林戈尔的努力会流行起来，我们会看到基于大陆或国家的公共资金、私人代码的卫星节点，这将开始建立一个“开放”的研究基地，供所有人使用。
