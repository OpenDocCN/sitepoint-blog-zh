# 谷歌 Chrome 网络商店将于 10 月推出

> 原文:[https://www.sitepoint.com/google-chrome-web-store/](https://www.sitepoint.com/google-chrome-web-store/)

谷歌 I/O 上首次宣布的 Chrome 网上商店将于 2010 年 10 月推出。它在网络上被称为*“应用商店”*，与苹果系统的比较是不可避免的。

网上商店有两个主要目标:

**1。轻松找到网络应用**
据谷歌称，网络应用发现很难。

在 I/O 期间，他们举了一个国际象棋游戏的例子:国际象棋有成千上万个在线版本，但找到一个好的同行评审的游戏很难(谷歌搜索应该承担责任吗？！)为移动设备寻找高质量的国际象棋游戏要容易得多。

我并不完全相信这个例子——在手机上更容易，因为可用的游戏更少。然而，当你考虑到谷歌 Chrome 操作系统的发布时，应用程序发现变得更加合理。该操作系统本质上是一个快速启动的网络浏览器，用户将能够查看应用程序的分类列表(想想大多数 Linux 发行版中的包管理器)。

**2。赚钱**
对于出售内容或软件的合法商人来说，在网上赚钱并不总是容易的。网络商店将使产品列表变得简单，并让谷歌管理丑陋的支付处理细节。它将支持:

1.  免费应用
2.  一次性付款
3.  按月或按年订阅
4.  免费试用，以及
5.  定制支付解决方案

有一个完整的许可 API，与苹果或其他支付网关相比，费用似乎合理。在一次性支付 5 美元的注册费用后，谷歌将抽取(传言中的)5%的收入。

## 你能卖什么？

谷歌为网上商店推荐了一些合适的产品:

*   Chrome 主题和扩展
    我不确定有多少人会为一些精美的图片或在其他地方免费的小工具付费。然而，这种模式在智能手机市场有效，Chrome OS 很可能会出现在平板电脑和其他移动设备上。
*   打包应用
    可下载的离线应用是可行的，在未来，谷歌本地客户端(NaCl)将允许快速的 32 位 x86 代码直接在浏览器中运行。快速动作游戏是可行的。
*   **可安装的网络应用**
    只需一点努力，一个网络应用就可以*安装*为 Chrome 应用。

这最后一个选项会让许多 web 开发人员兴奋不已。任何 web 应用程序都可以通过创建 JSON 编码的。描述其功能的 crx 清单文件，URL 和图标文件。安装后，应用程序图标出现在 Chrome 的“新标签”页面上，并打开它自己的纯图标标签窗口。还有几个额外的好处:

*   应用程序的窗口没有浏览器导航控件。
*   用户的偏好被保存，例如，他们仅被询问一次使用诸如地理定位和离线存储等功能的许可。

Chrome 网上商店听起来很有趣，赚钱的可能性可能会让它成功。有关更多信息，请参考:

*   [Chrome 网上商店](https://chrome.google.com/webstore)
*   [谷歌的可安装网络应用开发者指南](http://code.google.com/chrome/apps/docs/developers_guide.html)
