# 在这里获得您的 Java 6.0 版本！

> 原文:[https://www.sitepoint.com/get-your-java-60-builds-here/](https://www.sitepoint.com/get-your-java-60-builds-here/)

自从开源成为一个家喻户晓的名字以来，开发人员就一直呼吁 Sun 将 Java 平台发布到开源环境中。与此同时，Sun 坚定不移地维持着对该平台的管理，不折不扣地扮演着仁慈独裁者的角色。这并不是说 Sun 对 Java 不断要求更大的开放性无动于衷，最近的事件证明他们正在倾听！

就在一周前， **Sun [发布了 Java 5.0](http://java.sun.com/j2se/1.5.0/source_license.html) 的源代码，以一个新的、更简单的许可证供研究使用。**[许可证](http://java.sun.com/j2se/1.5.0/jrl_5.0-license.txt)将研究用途定义为“为了推进知识、教学、学习或定制技术或修改以供个人使用而进行的研究、评估或开发”

在这种许可下，您实际上可以获得 Java 基类的代码(不仅仅是使用它们的标准 Java APIs)，以及 Java 可以运行的每个平台上的 Java 虚拟机的原生 C 代码。现在，一直渴望改进 Java 工作方式的开发人员可以开始工作了。如果他们的努力证明是值得的，我希望 Sun 会考虑在正式版中加入任何改进。许可证只是确保开发者不能从这样的工作中赚钱。

好吧，我承认这对于像你我这样的 Web 开发人员来说有点无聊，这就是为什么当它发生时我没有报道它。然而今天， **Sun 宣布它将在整个开发过程中提供 Java 6.0(代号 Mustang)的源代码和编译版本，**从现在可用的[第一批快照](http://j2se.dev.java.net/)开始。Java 6.0 要到 2006 年才发布，但是 Sun 计划让我们一步一步地跟踪这项技术！

所以是的，如果你问我，Java 比其他任何“封闭”的技术更接近开源。问题变成了，这样够了吗？

在 Macworld UK 的一篇文章中，Onno Kluyt， [Java 社区进程(JCP)](http://jcp.org/) 的主席，让开发者参与编写未来 Java 平台版本的规范，被引用来问**“你认为[Java 的开源]现在人们不能做什么？”**

Javalobby.org 的讨论一直很激烈。Bruno Souza 的帖子给我留下了特别深刻的印象，他在帖子中描述了导致他的组织投资一个项目来独立生产 Java 的开源版本的许多因素。我认为他的观点特别有见地，即**政府不想依赖一项美国可能选择在未来对其实施贸易禁运的技术。**

不管你是否相信开源 Java 会给你带来好处，只要看一眼 Viva 网站，你就会相信在这个领域已经做了很多工作。无论 Sun 最终对其 Java 版本还是本网站上列出的某个独立项目采用开源开发模型，都会成为 Sun 版本的现实替代方案，**开源 Java 似乎是不可避免的现实。**你*会用它做什么？*
