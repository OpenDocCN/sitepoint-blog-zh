# 采访:Opera 软件公司的克里斯·米尔斯

> 原文：<https://www.sitepoint.com/interview-opera-softwares-chris-mills/>

![Chris Mills](img/04759ad16b50f92eb31506c4de513156.png "Chris and his excellent beard") **在 SitePoint 总部，我们都在为 [Web Directions South](http://south09.webdirections.org) 做准备——这是澳大利亚网络极客日历上*最大的事件*。今年的会议挤满了精彩的演讲和研讨会，包括我们自己的凯文·杨克关于 CSS 框架的演讲。我们很高兴成为此次活动的赞助商，我们每个人都期待看到一些我们最喜欢的话题被讨论、辩论和解决！**

我自己过去做过一点网络培训，我对克里斯·米尔斯、阿什·奥勒里和莱斯利·詹森-因曼主持的[教育方向](http://south09.webdirections.org/workshops#ed-directions)研讨会特别好奇。**克里斯·米尔斯**是 [Opera Software、](http://opera.com)的网络传道者和开发者关系编辑，也是 Opera 博客和 [Opera 网络标准课程的主要贡献者。我们和克里斯聊了聊他的工作、教育方向和网络教育的总体状况。](http://www.opera.com/company/education/curriculum/)

对于那些还没有认识你的读者，你能告诉我们一些关于你自己的情况吗？

没问题。在过去的两年里，我在 Opera(不是 Oprah——这是一个浏览器，不是脱口秀)工作，是一名网络布道者和教育工作者。我的工作涉及大量不同的事情，比如发表关于 web 标准技术的文章，为 Opera 产品编写文档，回答论坛上的问题，在会议上发言，联系网站所有者了解他们代码中的错误，帮助构思新的营销活动，以及校对其他人的工作。还叽叽喳喳。很多推特。没错，就是工作！

对于那些还没有睡着的人来说，我工作的两个主要且密切相关的焦点是传播开放标准和教育。我花了大量时间撰写相关主题的文章，并在大学演讲，以促进在课程中和学生中更好地使用 web 标准。我相信改善网络状况的最好方法是从那些新手开始。

就我是什么样的人而言，我是一个重金属男孩，混合了一点嬉皮士和一点伍基人。我在《钢铁征服》中打鼓，花很多时间在欧洲各地表演。德国人特别喜欢我们。

2008 年的 Web Directions North 非常重视教育，所以很高兴看到这一点在您的研讨会上得以贯彻。一个与会者会从你的研讨会上带回家什么？

研讨会的主要目标受众是任何想学习如何更有效地教授 web 开发或设计的教育工作者，尽管它对学生也很有用。首先，我们简要概述了网络教育的现状，以及我们认为的问题所在——行业与教育者之间的分歧。接下来，我们有四个会议，涵盖我们认为的四个关键主题:标记、CSS、JavaScript 以及可访问性和可用性。

在每一个会议中，一个被邀请的专家演讲者概述了主题背后的主要理论，以及它在整个 web 开发生态系统中的位置。然后我们讨论关于这个话题最难教的是什么，并提供解决这些困难的技巧以及有哪些免费资源可以提供帮助。

最后，我们以一个教育工作者小组结束了这一天:四五个教育工作者讨论了他们教授 web 标准的经验，并向大家传授了智慧。

我认为，如果你在该行业工作，但参与内部培训，或者如果你是一名经理，希望了解员工需要了解的最新信息，你也可以从研讨会中获得许多相关和有用的信息。我认为来自 web 开发社区的任何人都应该对 web 标准教育的状态感兴趣(或者至少应该感到有趣),并关心这反过来会如何影响他们的行业，但是有些人可能会认为它不如其他人重要。

我已经有一段时间没有学习 CSS 或 HTML 了——就像我们很多人一样，我是通过实践来学习的。你在学校和大学里听说过的一些错误是什么？

世界上有一些非常好的、进步的教育机构，如亚特兰大艺术学院(美国)和邓迪大学(英国)，但大多数在课程中有某种网络内容的地方都没有很好地处理这些内容。问题是网页开发是科学和艺术的混合体，很少有专门的课程直接涵盖它。相反，它会被附加到，比如说，计算机科学课程或平面设计课程的末尾。

计算机科学人士通常认为 web 技术不好，因为它们不遵循与所谓的适当编程语言相同的规则，例如具有非常宽容的解析器的 web 浏览器，以及不可预测的运行时环境。他们将从后端技术(如 ASP.NET 或 JSP)开始教授 web 开发，这与他们已经知道的一些东西有关，但很少考虑客户端在可访问性或语义方面的内容。

设计人员通常只想创建一个好的设计，并把它放到网上，而不担心它挂掉的代码。他们将创建一个设计，然后使用可怕的旧 Photoshop，切片，并将其转储到 Dreamweaver。同样，前端代码也会受到影响。

关于错误，我看到有标准意识的学生被扣分，因为他们没有像课程上说的那样使用表格和间隔 gif。我见过学生完全用 Flash 或一个大图片制作网站。我看到学生们在熟悉基本的标记理论之前就被教授了一个关于电子商务网站建设的模块。当一个学生问你一个问题，比如“你用什么做网站，XHTML 还是 Dreamweaver？”你真的知道他们被教导的方式有一个根本性的问题。

**当我进行网络培训时，我发现与那些*不是*初学者，但仍然坚持使用`font`元素和内联 JavaScript 的人打交道特别具有挑战性。有时，他们很难看出时间投资是否值得改进。你发现的鼓励这些人做出改变的最有效的方法是什么？**

“老狗，新把戏”类型的人是最难教的，因为你必须先改掉所有的坏习惯。然后你会得到的论点通常是这样的:“好吧，不管我用老方法还是新方法，我都会得到报酬，所以为什么要花时间和精力去学习呢？”对于这些人，你只需要不断给他们关键的论点，让他们尽可能的个人化，这样他们就更容易认同他们。

1.  你通常会认为代码更易移植、可维护、文件更小、更易访问、更适合 SEO。
2.  就学习难度而言，你不能告诉我 CSS 很难学，肯定！尤其是最基本的。向他们展示一些简单的例子，说明 CSS 有多么强大，可以让他们着迷。等到那个时候再提 ie6 bug！
3.  JavaScript 学习——对于这些人来说，你不妨从向他们展示 jQuery 开始——这将加速他们的工作，不管他们使用的是旧方法还是新方法。再次，让他们着迷，然后向他们展示更多的潜在细节。
4.  从可访问性的角度来解释它——诉讼胡萝卜，以及拥有残疾用户和移动用户的新用户市场
5.  从工作保障的角度来解释。向他们展示大量招聘前端开发人员的广告——越来越多的人现在要求至少具备 CSS、HTML、JavaScript 和最佳实践。他们还想在几年后有工作吗？

Opera Web 标准课程充满了好东西——总共超过 60 篇文章。你见过或听过哪些人们在课程中使用这种材料的例子？

我对这种理解非常满意。美国已经有许多学院使用它作为课程材料，英国也有一些大学，还有许多在线大学。现在有许多翻译项目:例如，HTML 和 CSS 的完整俄语翻译和几乎完整的中文翻译，而一所西班牙大学正在进行西班牙语和加泰罗尼亚语的完整翻译。

我听说的另一件好事是，许多有经验的 web 开发人员将它作为参考资料，并在客户和其他联系人被关于基本技术的乏味问题淹没时，将它作为一个指向他们的地方！

我坚信学生应该向从业者学习；如果您是一名开发人员，并且想开始教授 web 标准，您会从哪里开始呢？

我建议从帮助论坛上的人开始，给他们指出好的资源。当然，为此你需要一个好的资源库。维护您自己的个人资源黄金列表——当然包括 Opera Web 标准课程——并着手帮助人们。看看你的公司是否有机会将内部培训作为你工作的一部分。还要看看当地是否有学院或大学，你可以在那里做一些客座演讲或教授兼职课程。大多数教育机构都乐于获得他们能得到的任何资源。

当然，如果你发现自己正在计划一门 web 开发课程，你应该去看看 WaSP 的交互项目，它提供了课程结构、阅读列表、考试问题、能力等等。

既然你来了，有没有可能澳大利亚的征服钢铁之旅与 WDS 的 T2 之旅同时进行？

嘿，如果你们想赞助，我们会很乐意的！
