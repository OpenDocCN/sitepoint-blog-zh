# CSS 框架有什么不好？

> 原文:[https://www . site point . com/whats-so-bad-about-CSS-frameworks/](https://www.sitepoint.com/whats-so-bad-about-css-frameworks/)

![A boy rides a bicycle on a paved footpath. Training wheels are attached to the bike.](../Images/f166db52f623c1e8137b3de565334777.png "Little wheels")

CSS 框架绝不是突发新闻。几年来已经有好几个可供选择的了，其中最早也是最著名的是雅虎的网格 CSS T1；我们之前在博客中讨论过这个[。令我惊讶的是，许多人没有利用这些节省时间的好方法。快速浏览一下](https://www.sitepoint.com/warming-up-to-yahoo-ui/)[我们的 CSS 论坛](https://www.sitepoint.com/forums/forumdisplay.php?f=53)，会发现很多人都有同样的关于浮动布局和定位的令人沮丧的问题；我敢肯定，如果开发人员简单地使用从 CSS 框架中获取的经过尝试和测试的布局方法，这些是可以避免的。

嘿，等一下——把那些烂番茄收起来！我会解释的，我保证。

## 像训练轮子一样对待框架

经常针对 CSS 框架的一个批评是，它不鼓励帮助初学者学习的“试一试”的方法。当然，对于初学标记的猴子来说，自己学习 CSS 更好，但我认为使用框架有一些好处，远远超过这个特殊的陷阱。

你们中有多少人学会了骑带辅助轮的自行车？对于一个孩子来说，在学习平衡自行车和保持直立之前，这是一个很好的、有趣的方法，可以让他获得自信，掌握刹车和转向的基本原理。更有甚者，从自行车上摔下来擦伤膝盖，极其令人气馁。

训练轮子的目标是通过让新的骑手轻松骑自行车来建立信心:同样，以正确的态度使用 CSS 框架可以帮助指导新的开发者进入神秘且通常令人困惑的 CSS 布局世界。新手有机会看到正确工作的布局，而不会感到沮丧，这些框架的开放性质使人们很容易了解幕后发生了什么。后来，有了更多的信心，从框架中获得的技术可以在从头开始开发自己的布局时得到很好的利用。

所以，如果你是一个 CSS 新手，并且因为被告知对你的教育不利而避免使用框架，也许你应该重新考虑一下。试一些出来，看看你最喜欢哪一个，试试吧！请记住:你最终需要摆脱这些训练，所以一定要利用这些经验来帮助理解 CSS 是如何工作的。

## 鱼与熊掌，不可兼得

对 CSS 框架的另一个批评是关于框架是如何构建的。通常框架是基于一个网格或者一系列列的概念；相应地，这些专栏经常会有一系列的类以它们的表现形式命名。流行的 [960 网格系统](http://960.gs/)包含几十个奇怪的类，如`.suffix_10`或`.grid_3`，而同样流行的[蓝图 CSS](http://www.blueprintcss.org/) 则使用类名，如`.border`或`.span_5`。更重要的是，它们包含了大量的类，可以满足您可能需要的每一个可能的列宽或边距要求。然后，雪上加霜的是，他们中的一些人有三个或更多的单独的 CSS 文件，用于类型、布局、打印或 Internet Explorer。

绝对令人震惊！我们是否应该鼓励有意义的类名来描述元素的用途，而不是外观？我们应该避免膨胀吗？有人会想到浏览器吗？

当然，我们应该这样做，但是这并不足以成为完全抛弃 CSS 框架的理由。答案就像这样简单:你不需要使用所有这些东西，也没有人强迫你这么做。

对于大多数框架来说，只使用你需要的部分，根据需要修改它们，抛弃其余部分是完全可以接受的。事实上，选择一个具有编辑友好许可的框架，允许你只抓取你喜欢的部分是被鼓励的。大多数网络上的 CSS 框架都是在自由软件模式下许可的，这种模式允许你随意修改代码。例如，960 Grid 系统在[麻省理工学院许可](http://www.opensource.org/licenses/mit-license.php)和 [GNU 通用公共许可](http://www.gnu.org/licenses/gpl.html) (GPL)的双重许可下，两者都允许自由编辑，而 [Blueprint 的许可](http://wiki.github.com/joshuaclayton/blueprint-css/license)是同一麻省理工学院许可的修改版本。这些许可证中没有任何内容会阻止您对您的作品收费或主张您的版权权利。如果你打算向世界其他地方发布作品，比如在 WordPress 主题或免费模板中，你需要在类似的条款下这样做。

因此，如果你不喜欢它们的工作方式，完全可以重命名这些讨厌的类，重新安排它们，或者完全改变它们。你可以去掉任何对你的项目来说不必要的部分——我们甚至在 SitePoint 这里有一个叫做 [Dust-Me 选择器](https://www.sitepoint.com/dustmeselectors/)的整洁工具，它将帮助你做到这一点。真的，这样就好了。你实际上可以使用这些框架中的一个来启动你自己的个人框架。

一些开发人员把从头开始仔细构建每一个布局视为个人骄傲——即使这意味着他们一遍又一遍地做同样的重复任务。对我来说，我把找到更聪明地工作和及时回家吃饭的方法视为一件骄傲的事情。如果 CSS 框架可以帮助你在更短的时间内建立一个更坚实的布局，为什么不试一试呢？
