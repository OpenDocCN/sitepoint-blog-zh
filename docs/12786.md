# 新闻快讯:云将下降

> 原文：<https://www.sitepoint.com/news-flash-the-cloud-will-go-down/>

昨天，Gmail 关闭了几个小时。从推特和博客世界的[上的反应来看，你可能会认为世界末日到了。对于整天依赖电子邮件的工作人员和沉迷于网络的人来说，几个小时内可能会有这种感觉。但是，当你将自己置于基于云计算服务的支配之下时，这是你必须预料到的事情。](http://www.techmeme.com/080811/p127#a080811p127)

没有一个网站有 100%的正常运行时间——就像几乎没有一台电脑能正常运行一样。就像你的个人电脑有时会崩溃一样——拒绝你访问你的电子邮件——Gmail 和其他基于网络的服务有时也会崩溃。

ZDNet 的 Larry Dignan 有一篇很棒的文章提出了最明显的观点:云服务可能会关闭，那么让我们的数据保持离线同步难道没有意义吗？

“如果你依赖这些基于网络的应用程序，你需要一个备份计划，”思维一贯清晰的迪南写道。另一个头脑冷静的 ZDNet 博客作者(这些人正势如破竹)，我的朋友 Ryan Stewart 也[一针见血](http://blogs.zdnet.com/Stewart/?p=897)写道:“系统将会崩溃，这是生活的事实。重要的是当这些系统停机时要做好准备，这是电子邮件等系统应该内置某种离线访问的主要原因。理论上，我们会有一天云真的会一直存在，但我们还没有接近，而且这可能永远不会发生。”

他们都在倡导我一直在谈论的东西:[网络应用的桌面访问非常重要](http://www.readwriteweb.com/archives/why_we_need_web_apps_on_the_desktop.php)。我在 ReadWriteWeb 的文章中给出了两个主要原因，一个是 RIA 需要桌面访问，这有助于主流用户从桌面应用过渡到 Web 应用，另一个是浏览器中的多任务处理充其量只能算是粗制滥造。但除此之外，我们还可以添加第三个原因:**桌面同步让你在云关闭时可以访问你的重要数据——它*将*关闭**。

Dignan 想知道微软是否真的在他们的软件+服务愿景上有所作为。[通过微软](http://msdn.microsoft.com/en-us/architecture/aa699384.aspx):

> 未来是本地软件和互联网服务相互作用的结合。软件让服务更好，服务让软件更好。通过结合这两个领域的优势，我们为客户提供了最大的选择、灵活性和能力。在我们的行业中，我们将这种演进路径描述为软件+服务。"

”但迄今为止，这种努力一直受到清教徒阵营的嘲笑。这些云计算清教徒认为这不是真正的云计算。Dignan 说:“微软只是想保护它在软件领域的主导地位。

当然，微软并不是唯一一个认为软件+服务是 RIAs 真正未来的公司。我们几周前写道雅虎！也认为桌面 RIA 是未来。此外，Adobe 清楚地认为富互联网应用程序的桌面/离线访问是他们的 AIR 软件未来计算的主要部分。

RIA 的桌面访问和数据的离线同步是迈向云永远在线的神话未来的必要一步——正如 Stewart 所说，这是我们可能永远无法实现的未来。就目前而言，只有当你接受你的数据不会一直在你身边的事实时，完全依赖云才是好的。因此，将数据备份到其他地方是常识。

所以这里有一个非常重要的教训:

1.  云层会下沉，因此
2.  RIA 的离线/桌面访问非常重要。
