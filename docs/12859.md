# 微软向所有人开放 Live Mesh

> 原文:[https://www.sitepoint.com/microsoft-opens-live-mesh-to-all/](https://www.sitepoint.com/microsoft-opens-live-mesh-to-all/)

![](../Images/8f63f98ae179e486d4f2d61b276db759.png "livemesh-logo")微软在 4 月份的 Web 2.0 博览会上有限的技术预览中开放了其 [Live Mesh](https://www.mesh.com/) 服务，今天已经向所有人开放这项服务，[LiveSide](http://www.liveside.net/blogs/main/archive/2008/07/16/live-mesh-is-indeed-now-open.aspx)证实。现在，居住在美国的任何拥有 Windows Live ID 的人都可以使用微软的数据同步平台。国际用户如果将他们的 Windows 操作系统语言和地区更改为 EN-US，就可以加入。

虽然 Live Mesh 现在对任何人开放，但这项服务在很大程度上仍然是一个测试/技术预览产品。它只在你的个人电脑和云端之间同步数据——Mac 和手机支持仍然“即将推出”

## 什么是活网格？

Live Mesh 是一个自动数据同步平台，可以在设备之间同步用户数据——最终这将意味着基本上任何带有 1 和 0 的东西:手机、PC、Mac、视频游戏播放器等。—还有云。Mesh 是一个提要集合，可以表示为 ATOM、JSON、FeedSync、RSS、WB-XML 或 POX。用户创建的任何数据片段都作为信息呈现在提要中，然后根据共享该类型数据的规则(即隐私设置)传播到其他设备。

Mesh 遵循微软的“软件加服务”策略，包括 web 平台组件和客户端软件。目前，Mesh 是一个技术预览版，可以自动将 Windows PC 上的文件和文件夹与云中的网络存储空间同步。最终目标是让 Mesh 可以在任何数量的设备上运行，这样在工作时用黑莓手机拍的照片就可以自动备份到你的台式电脑、Macbook Pro、家里的 iPhone 和云端。

## 站台

Mesh 与其他数据同步游戏(如 [DropBox](http://www.getdropbox.com/) 或 [SugarSync](http://www.sugarsync.com/) )的区别在于，微软将 Mesh 视为开发者平台以及面向用户的服务。微软希望 Mesh 将成为设备和网络应用程序数据同步的事实平台。

微软在 4 月份向我展示了一个概念验证演示，其中一张刚刚拍摄的手机照片几乎立即被同步到 Flickr 和脸书上的相册、一对笔记本电脑、一台 UMPC 和云存储空间(整个过程不到 5 分钟)。当您将 Live Mesh 视为一个同步平台，而不仅仅是另一种数据备份服务时，出现的可能性是令人兴奋的。微软还希望同步与 Mesh 账户相关的所有数据，而不仅仅是文件。这意味着，例如，如果你以这种方式设置文件权限，你在脸书上对照片的评论可能会被复制到 Flickr 或回到你的手机，或者它可以将 Twitter 推文同步到其他设备或网络应用程序。

关于 Mesh 最有趣的事情之一是，客户端软件将允许使用该平台的 web 应用程序创建其应用程序的离线版本(作为 web 应用程序本身的桌面包装器存在)。这意味着，在某种程度上，Mesh 的目标可能是挑战谷歌的 Gears 和 Mozilla 的 Prism(在较小程度上，还有 Adobe 的 AIR)。每当我们听说网络是新的操作系统时，桌面仍然非常重要。

## 仍然只是预览

尽管 Live Mesh 拥有巨大的潜力，微软的平台仍然只是一个技术预览。由于目前只支持 PC 到 web 存储同步，很难判断 Mesh 会对我们交互、存储和同步数据的方式产生什么影响。微软展示的一些概念验证的东西令人兴奋，就像为网络应用程序、计算机和网络设备提供完整的同步平台的想法一样，但目前，几乎所有这些东西都还不存在于微软内部演示之外。

希望这个平台不会辜负宣传和潜力。与此同时，任何拥有 Windows PC 和 Windows Live ID 的人都可以在 www.mesh.com 查看 Mesh 的早期版本。试试吧，在评论中告诉我们你的想法。

看起来 Mesh 并没有完全对公众开放，只是比以前更加开放。技术预览版的用户数量仍然是有限的，这个数量已经翻倍了。所以，在你有能力的时候报名吧。 [Via LiveSide](http://www.liveside.net/blogs/main/archive/2008/07/16/live-mesh-is-open-with-a-cap.aspx) 。
