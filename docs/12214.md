# 下一代开发者在哪里？

> 原文：<https://www.sitepoint.com/it-developers-the-next-generation/>

我不想听起来像个老人，但我关心下一代开发者。越来越少的学生选择信息技术课程，计算机经常被“软”科目取代。

我认为自己很幸运，在 20 世纪 80 年代早期的 8 位时代，家庭计算刚刚出现。这个行业是革命性的，令人兴奋的，更棒的是，我们的父母对这项技术一无所知。硬件可能很原始，软件可能很糟糕，但是电脑是可以使用的。你可以买一个[辛克莱·ZX 谱](http://en.wikipedia.org/wiki/Sinclair_Spectrum)，然后在一个小时内编写程序(最初的 Speccy 附有一本优秀的基础手册)。

不幸的是，有证据表明，尽管大多数西方学校拥有非常棒的计算机设备，但学生们却在回避它。英国计算机专业学生在 2004 年至 2007 年间减少了 20%；任何科目中跌幅最大的。对 2，000 多名中学生的调查得出结论:

*   计算机作为一门学科已经失去了它的新奇因素
*   学生们认为计算机科学受到了极客形象的影响
*   许多人认为计算机太难了，或者不理解这门学科涉及的内容
*   计算被认为是“无聊”的
*   只有当你想在这个行业找份工作时，学习它才是必要的
*   计算机经常用于其他学科，所以没有必要学习它。

其他几个因素也很明显:

**1。游戏主机**
的兴起游戏是买电脑最重要的原因之一。自 20 世纪 90 年代初以来，专用游戏机提供了一种在家中玩街机质量游戏的廉价途径。虽然孩子们也可以使用电脑，但大多数人更喜欢玩他们的游戏机。不幸的是，这是游戏机唯一能做的事情；编程和实验是不可能的。

**2。杂念太多**
PC 很棒；你可以坐上几个小时，什么也不做。网上冲浪、和朋友聊天、摆弄照片会分散你做其他事情的注意力。当一个孩子想要的每一个应用程序都已经被提供时，他为什么还要学习编程呢？

**3。开始开发太难了**
许多程序员会为此斥责我，但是解释 BASIC 是一门很好的第一语言。这很容易开始，提供即时反馈，并教你的要领。大多数 8 位计算机引导到一个基本的编程环境，所以鼓励你从第一天开始就尝试命令和小程序。

现在学生能用什么？VisualStudio 和类似的 ide 对初学者来说太复杂了。基于表单的窗口 UI 开发和基于 web 的客户机-服务器编程对于除了最专业的开发新手之外的所有人来说都有一个陡峭的学习曲线。虽然有适合儿童的语言，但许多都过于简单，而且大多数都是很少有人知道的特殊环境。

你是计算机系的学生吗？你最先学的是什么语言？学科有用，教的好吗？随着选择计算机科学的学生越来越少，IT 行业会受到影响吗？还是说它无处不在，人们天生就具备所需的技能？
