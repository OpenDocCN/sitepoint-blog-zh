# 微软将放弃 ie 浏览器？没机会！

> 原文:[https://www.sitepoint.com/microsoft-to-drop-ie/](https://www.sitepoint.com/microsoft-to-drop-ie/)

互联网谣言制造者今天加班加点。几个网站声称，微软正处于放弃 Internet Explorer 的边缘，或者至少，他们将取代 Trident HTML 渲染引擎。我不确定这些谣言是从哪里开始的，但是有几个提到了 T2 信息世界的一篇博客文章。

谣言 1:微软将放弃 ie 浏览器
没有任何证据表明微软正在考虑此事。他们为什么要这么做？作为一个产品，IE 和 Windows 一样无处不在。

浏览器可能不会直接增加收入，但是制造商对互联网用户有巨大的影响力。2007 年，Mozilla 的收入达到了 7500 万美元，这主要是因为谷歌被用作 Firefox 的默认主页。

微软获得的要多得多:他们有各种各样的产品要卖，有成千上万的员工要付工资，有数百万的股东要满足。当大多数人仍然使用 IE 作为他们上网的门户时，他们为什么要放弃它呢？既然每个操作系统都需要浏览器，他们会用什么来取代它呢？

谣言 2: Internet Explorer 将采用 WebKit
这似乎源于史蒂夫·巴尔默在 2008 年悉尼开发者大会上的一句话。一个学生问:

> 为什么 IE 仍然是相关的，为什么当有开源引擎可以更快地响应 Web 标准的变化时，花钱购买渲染引擎是值得的？

这位微软的首席执行官回应说，这个问题是“厚颜无耻的”，但接着说:

> 开源很有意思。苹果已经接受了 Webkit，我们可能会考虑，但我们将继续为 IE 8 构建扩展。

史蒂夫·巴尔默是一个成功的商人，但他并不直接控制微软的开发团队或技术决策。他本可以声明开源浏览器是邪恶的，他打算将它们从地球上抹去——但这能有多好呢？

微软正在积极开发三叉戟，没有证据表明他们打算放弃它。

**传闻三:ie 浏览器将使用“瞪羚”**
[瞪羚](http://research.microsoft.com/apps/pubs/default.aspx?id=79655)是微软打造的实验原型。它使用类似操作系统的资源管理技术，从理论上讲，可以使浏览器更加健壮、可靠和安全。

微软可能会在 IE 中采用 Gazelle 或它的一些技术。至于 IE9 是否会出现这种情况，则是另一回事了。

不要相信这种宣传:IE 浏览器将会存在很多年… [淘汰 IE6 的时间已经够长了](https://www.sitepoint.com/the-death-of-ie6/)！
