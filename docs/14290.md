# A-Z 网站索引说明

> 原文:[https://www.sitepoint.com/a-z-indexes/](https://www.sitepoint.com/a-z-indexes/)

在搜索网站或内部网的内容时，A-Z 索引比搜索引擎要精确得多。考虑到 A-Z 索引给网站带来的价值，额外的花费是值得的，但是，如果你决定给网站添加一个索引，找到合适的人来做这项工作是很重要的。

##### 字母索引如何工作

A-Z 索引提供了一个按字母顺序排列的“入口点”主题列表，用户可以通过它进行浏览和选择。在书或手册后面的索引中，条目后面是页码。在网站上，入口点被超链接到适当的页面，并且通常被超链接到网页内的命名锚点，以在索引中获得更高层次的细节。

与书籍索引一样，一个站点索引可能包含多个条目，每个条目的措辞都不同，指向同一个页面或页面和锚。这种方法用于涵盖用户可能认为主题命名的所有不同方式，被称为“双重发布”它涵盖了同义词，如“汽车”和“汽车”，以及一个短语的不同词序，如“汽车发动机”和“发动机，汽车。”索引的可浏览特性解决了可能由不正确的或不同的拼写以及网站用户可能选择的单数或复数用法而引起的问题。

此外，通常还有第二层术语，称为“子条目”，在一些主条目下列出并缩进。

Web A-Z 索引通常是一个很长的 HTML 页面，尽管如果它非常长的话，可以为字母表中的每个字母分成单独的页面。在页面的顶部，通常会出现一个横向排列的字母表。用户从该列表中进行选择，并跳转到字母索引的适当部分。

A-Z 索引不是由机器创建的，而是由人类创建的，他们会小心翼翼地将索引条目添加到那些出现了关于主题的有用信息的页面上。通过这种方式，避免了对上下文中提到的主题词进行索引，提高了索引本身的整体相关性和质量..

A-Z 索引的一些例子可以在网上的 [A-Z 索引中找到:来自蒙塔古研究所评论](http://www.montague.com/review/AtoZ.htm)的注释样本。

##### 比搜索好！

我们不时会发现一篇文章或其他文档说站点索引是一件好事。时不时地，我们可能会在网站上看到 A-Z 索引，然后想，“嘿，这太酷了。”那么，为什么我们看不到更多呢？最简单的答案是，A-Z 索引的主要竞争对手——网站搜索引擎——通常实现起来更便宜(甚至可能是免费的)。但是，最终，你会得到你所付出的。

***站点搜索引擎可能检索不到足够的或任何页面。*T3】**

全网搜索引擎通常在页面数量上产生“令人满意”的结果，因为用户通常想要某个主题的“一些信息”，这通常可以在检索到的大量页面中的一些页面上找到。如果搜索引擎错过了许多好的页面，用户通常不会注意到或在意，因为已经找到了足够多的其他好页面。

然而，在一个网站中，页面的数量相对较少，所以一个简单的搜索引擎搜索可能不会产生足够的或任何结果，即使有关于该主题的好页面。这很可能是因为用户输入的搜索主题与页面文本中对该主题的引用的措辞不同。

***站点搜索引擎可能会检索到太多不相关的页面。*T3】**

全网搜索引擎通常在文章质量方面产生“令人满意”的结果，因为主要的搜索引擎公司已经为页面的检索和排序开发了复杂的标准和算法。站点中使用的现成搜索引擎并不复杂。他们经常检索到的页面只是简单地提到了搜索词，但根本没有真正关注主题。

搜索引擎经常不能满足搜索者在一个站点内进行搜索的需求。T3】

一个站点的搜索者可能想要一个站点关于给定主题的所有信息，而整个 Web 的搜索者只想要——并且期望——关于一个主题的一些信息。对一个站点的搜索可能也想更快地找到信息，因为他们可能同时在看许多站点。

最终，搜索引擎结果的质量反映了用户输入的搜索字符串的复杂程度，这是无法控制的。另一方面，在 A-Z 索引中，结果的质量反映了索引器的复杂程度，这是可以控制的。

##### A-Z 索引的其他优势

除了卓越的搜索准确性，A-Z 索引还提供了其他好处。

*   组成 A-Z 索引的大量标记良好的内部链接提高了链接页面的搜索引擎优化评级，从而提高了整个网站的搜索引擎优化评级。
*   没有不相关的页面检索使索引更有效，提高了网站的可用性。
*   浏览索引的能力使用户能够偏离主题，探索其他引起他们注意的主题，使他们在网站上停留更长时间。换句话说，索引可以增强网站的“粘性”。最后，A-Z 索引可以有效地在太小而不能与站点搜索引擎一起工作的网站上实现，例如 20-50 页范围内的网站。

***不比定制搜索引擎*** 更昂贵

当然，基本的网站搜索引擎可以通过修补变得更有效。它们可以被定制为只搜索元标签，然后可以为每个页面仔细地编写元标签。搜索可以限制在某些页面和/或页面中的区域。可以定制结果来显示“上下文中的关键词”所有这些定制都需要 Web 开发人员的专业知识，他们的时间并不便宜。搜索引擎能力的提高仍然无法与精心制作的 A-Z 索引相比。

另一方面，创建 A-Z 索引是一项简单的编辑任务，可以由自由职业的索引员来完成。索引器可以根据每个可索引页面的平均字数或索引中的条目数，在作业开始之前提供准确的报价。根据网站或内部网的规模，A-Z 索引的价格在 200 美元到 1000 美元之间。

##### 为工作找到合适的人

A-Z 指数做好很重要。尽管任何按字母顺序排列的列表一开始看起来都不错，但是如果用户不能快速找到他们要找的东西，他们会对索引和网站本身感到失望。没有索引可能比创建一个糟糕的索引要好。

你应该自己创建索引吗？创建索引比创建类别层次结构或分类法更复杂。要想胜任索引工作，确实需要适当的教育。然而，具有图书馆学背景和良好标签意识的信息架构师可能会从阅读关于该主题的好书中学会索引。另外推荐一个自动嵌入索引 URL 的工具，比如我比较喜欢的 [HTML 索引器](http://www.html-indexer.com/)。

那么，去哪里找网站索引器呢？如果你不想把时间和精力投入到自己学习索引中，那么最好的办法可能是签约一个自由职业的索引员。下面列出的大多数专业索引协会都有可搜索的自由职业索引数据库。将您的搜索限制在 HTML 或 Web 索引器。网站索引的好处在于，索引工作的样本通常可以在网上获得，所以你可以很容易地评估一个潜在的索引工作。

##### 索引维护

诚然，当页面被添加到站点或从站点移除时，手动创建的 A-Z 索引需要更新，但是站点地图也是如此。如果一个网站是如此的大和动态，以至于维护一个站点地图是不切实际的，那么创建和维护一个索引也是不切实际的。对于内容经常变化的页面，如公告页面，索引不应针对内容的细节，而应仅针对公告的一般概念，这样就不需要频繁更新索引。

如果索引是由签约索引器创建的，则需要就如何维护索引达成协议。可以保留索引器以备将来更新，或者索引器可以向网站管理员提供如何维护索引的书面指南。例如，在为一所学校的网站写了一个索引后，我确定了未来可能增加的页面，并为网站管理员写了指南，指出新页面可能被索引的条目/子条目。在这种情况下，新页面往往是单独的班级页面，每个页面最好在三个地方建立索引:在老师的姓氏下，按年级，在主题“教室页面”下。

##### 结论

A-Z 索引将增强网站的可搜索性、可用性和整体质量，其结果优于大多数网站搜索引擎。仅仅因为一个搜索引擎是免费的，一旦你添加了一个索引，不要试图把它保留在网站上。因为他们服务于相同的目的，搜索引擎和 A-Z 索引的同时存在，产生不同的结果，只会使用户困惑和浪费他们的时间。
