# 决议困境

> 原文：<https://www.sitepoint.com/resolution-dilemma/>

如果你把脸凑近显示器，你会看到它是由无数个小点组成的，它们共同作用形成一幅图像。用计算机的话来说，这些点中的每一个都被称为“像素”。显示器的分辨率越高，屏幕上的像素就越多。

**那么，显示器分辨率为什么如此重要呢？更高的显示器分辨率不仅意味着更高质量的图像，还意味着屏幕上一次可以显示更多的信息。**

作为一个网页设计师，你的目标应该是设计一个不需要用户从左向右滚动来阅读每一页的分辨率的网站。为此，只需指定一个只允许水平显示“x”个像素的表格宽度。

问题是这样的:假设您为 800 x 600 显示器分辨率进行设计，指定表格宽度大于 600 像素。每当有人将显示器设置为 640 x 480 时，他们就会看到一个水平滚动条。另一方面，如果你为 640 x 480 设计你的网站，任何人在访问你的网站时，如果显示器设置为更高的分辨率，就会看到大量的空白。

除非你的网站可以在掌上电脑或 WAP 手机等手持设备上浏览，否则只有三种选择:

【640 x 480 的设计:

优点:

*   每个人都可以浏览你的网站，而不需要水平滚动。

缺点:

*   对于分辨率较高的人来说，屏幕上会有很多空白。你能挤到屏幕上的信息量也将受到严重限制。

【800 x 600 的设计:

优点:

*   你会有更大的工作空间。而且，对于大多数设计来说，这个分辨率提供了足够的空间来呈现你的站点导航栏以及合理数量的内容。

缺点:

*   你会疏远使用 640 x 480 分辨率的访问者

**使用“灵活设计”:**

“灵活设计”使用百分比而不是精确的像素度量来指定表格宽度。因此，不是说表格宽度应该是 780 像素，而是指定它应该占据屏幕的 95%。网站访问者在 800×600 显示器上将看到 760 像素的水平信息，而在 640×480 显示器上，将看到 608 像素的信息。

优点:

*   永远不会有任何浪费的空间。您的站点将始终占据显示器上 100%的可用空间(或者您选择的任何百分比)。

缺点:

*   很难预测你的网站在各种各样的系统上会是什么样子。分辨率较高的显示器每行还会显示大量的文字，这会使文本难以阅读。

在做决定之前，请仔细权衡利弊。因为每个网站都有自己独特的受众，你应该弄清楚你的访问者使用的显示器分辨率——注册一个像[TheCounter.com](http://www.thecounter.com)这样的服务。在 Statmarket.com 的[可以找到关于普通网民使用何种分辨率的良好信息来源。](http://www.statmarket.com)

**你得到的空间比你想象的要少**

你可能会想，如果你正在设计一个 640 x 480 的网站，你只需要创建一个 640 像素宽的表格，然后把你所有的内容都放在里面。并非如此。浏览器的滚动条占据了大量的空间，进一步限制了你的工作空间。此外，像 Microsoft Office 工具栏这样的东西可以进一步减少屏幕空间。如果这还不够，一些用户不会最大化他们的浏览器窗口。下面我概述了在浏览器窗口最大化的情况下，在指定的分辨率下，一个表格应该有多少像素宽。

640 x 480 — 600 像素宽的表格
800 x 600 — 780 像素宽的表格
1024 x 768—1000 像素宽的表格
1280 x 1024 — 1200 像素宽的表格

那么，什么样的分辨率最适合您的站点呢？

不幸的是，对于你应该为哪个分辨率设计你的站点，并没有一个单一的答案。做出这个决定的最好方法是找出哪种分辨率最受目标受众的欢迎，然后设计您的网站来适应他们。如果你还没有开始设计，一个很好的经验法则是，如果你的网站是为技术人员或有钱定期升级电脑的人设计的，那么只为更高的显示器分辨率设计。
