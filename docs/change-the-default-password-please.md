# 请更改默认密码

> 原文：<https://www.sitepoint.com/change-the-default-password-please/>

最近关于 Windows 上的 MySQL 的争论很大程度上是由于那些安装了该应用程序并保留默认根密码 m(在 MySQL 的情况下，根本没有密码)的人。

当我在任何进入我的网络的设备上通过“默认”更改身份验证配置时，从路由器、交换机和服务器到软件应用程序和移动设备，我从未花时间考虑默认密码。

在过去的一周里，我开始做一些调查，看看一个漏洞部分是由于默认密码管理造成的。我在关于系统管理员标准实践的调查中没有发现太多硬内容，可能是因为依我的拙见，任何称职的系统管理员都应该在设置时修改所有的用户/密码组合。

我确实找到了许多网站，这些网站按品牌名称列出了数千种设备和应用程序，列出了默认的用户和密码组合，以及凭据支持的访问级别。一方面，当继承或重置旧硬件/软件时，这些数据很方便——另一方面，对于那些为了娱乐或盗窃而入侵他人网络的人来说，这是一个免费的破解凭证库。

MySQL 漏洞应该为新老系统管理员敲响警钟，并引发对您负责的所有设备/应用程序的全面盘点。审计认证最终应该提供一个详细的凭证和访问级别矩阵。

如果你锁上前门和窗户，让后门半开着，肯定会有人不请自来。
