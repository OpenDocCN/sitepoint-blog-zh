## 副本

Tim:

嘿，大家好，这是蒂姆·埃夫科…

David:

…这位是大卫·格林先生…

Tim:

…您正在收听的是版本控制播客的第一集。这是一个我们每两周坐下来讨论从开发到设计的网络产业的地方——一些人今天让它发生，并计划它在下一个版本中的走向。

所以今天，我们采访克雷格·巴克勒，他来自很多地方。他是 SitePoint 上非常著名的作家，也是英国的自由网络顾问。怎么样了，克雷格？

Craig:

进展很顺利。谢谢你邀请我。

David:

你已经做了这么多，我知道我们想联系你的原因之一是你已经为 SitePoint 写了四年多了，不是吗？

Craig:

七点了。我从 2009 年 2 月开始。

David:

太神奇了。你说得对，你在网上的资料我不认为有那么久远。你最初是怎么找到 SitePoint 的？

Craig:

嗯，我很早就开始使用 SitePoint 了。我认为它开始于大约 98、99 年，它是当时最好的网络资源之一——唯一的网络资源之一。它确实以一种非常深刻的方式涵盖了主题，并且有一些伟大的作者。然后在 2009 年，他们要求一些博主，我经历了一系列的面试和任务。

我和其他几个人成了常客——Alyssa Gregory 做商业方面的工作，Jennifer Farley 做设计，我做技术文章……我一直认为这对其他人有点不公平，因为我有大量的主题选择，同时我也做一些图形和商业方面的工作。

但是是的，那是七年前的事了，我还没有停下来。

David:

七年前。然后，看看你所涉及的话题范围！我们想问大家的一个问题是，既然这是版本控制秀:想想你的职业生涯，你现在的职业生涯是什么版本？你为什么认为它是那个版本？

Craig:

哦，哎呀，什么问题！实际上，我可能会说只是出于测试——因为我做的许多事情。但实际上我认为版本 1 可能是我在各种事情上做的开发。我的意思是，在早期，我并不热衷于网络，只是真正发现了我所热爱的东西——碰巧是编码和设计工作。

当网络出现的时候——我现在稍微展示一下我的年龄——感觉很棒。它结合了这两者。这是第二版，我直接进入了它。我观察了各种各样的公司，十年前我决定自由职业是一条出路。

从那以后我就喜欢上了它。我已经能够改变自己的命运，从事真正让我感兴趣的项目。在这方面我很幸运。

David [3:06]:

你所涉及的广泛内容——如何介绍自己，如何展示自己，你认为自己的重点是什么？

Craig:

哦，天啊。在早期，我会被称为<q>网站管理员</q>。这是一个广为流传的术语，因为你做了所有的事情。我是个多面手。我主要对前端感兴趣，但我也做很多后端工作。

我从事数据库工作，做一些设计，我喜欢这样。我喜欢这种多样性，有些人会说，是的，它让我成为一个万事通，一个都不精通。但是我认为在网络中，你需要在很多不同的技术上有一个良好的基础。

我感到幸运的是，我一开始就在那里，并逐渐学会了。如果是今天——如果我今天开始从事网络工作——我不知道我会去哪里，我可能会更专业一点。但是我喜欢我可以坐下来从头到尾写一份申请。

无论主要是在前端还是后端，都没有关系。拥有如此广泛的技能对我真的很有用。

Tim:

那真的很酷。那么，说到这个——你在网络上工作的时间长度——自从你开始工作以来，你注意到它的一些最大的变化是什么？

Craig:

追溯到很早以前，可能我做的第一个网站非常复杂。因为即使只是做悬停效果，你也需要一个 Java 小程序。我确实写了一个。这花了我很多很多天，你知道，这是我们现在用 CSS 在 30 秒内就能搞定的东西。

所以，这就简单多了。但是，生态系统也变得更大了。所以，在早期，你有 HTML，你真的有 Java 小应用程序，这就是一切——很快我们就有了 JavaScript，CSS，技术现在刚刚呈指数级增长。

甚至当我回想起 2009 年我开始在 SitePoint 上写作时，我们才刚刚真正走出停滞期。IE6 是最突出的浏览器。HTML5 和 CSS3 还不是真正的东西，或者肯定没有在许多浏览器中实现。

我们没有使用我们今天使用的工具，你当然不能非常容易地在服务器上编写 JavaScript。所以，是的，这只是快速的变化，它一直都是这样的，如果你不喜欢变化，那么，网络对你来说不会是一个有趣的地方。

David [5:56]:

那么，随着这些天所有的东西都在变化，你认为最大的关注领域在哪里，人们应该最关注正在发生的变化类型——因为现在你可以专注于 JavaScript 框架，或者你可以专注于节点，或者你可以专注于使用 Sass 的新方法。

你觉得哪里值得开发者关注那个变化，哪里只是噪音？

Craig:

如果我知道，我会成为一个有钱人。我觉得永远都是惊喜。这就是问题所在——你不知道下一件大事会从哪里来。

我只能说，你必须不断学习。你必须继续阅读这些东西。保持好奇。

因此，如果你是一名 PHP 开发人员，一定要使用 Node。如果你一直在尝试 Angular，那就试试 React。大概是挑战自己，每天学习新的东西。我还认为，如果在过去的七年里我学到了一件事，那就是记下你做的每一件事。记下一些事情，即使只是为了你自己。你不一定要在某篇文章中发表它。

但是如果你把它写下来，你就能记住它。而我已经到了连早餐吃了什么都不记得的年纪，更别提三年前学的一些诡异的 CSS 技术了。所以，有了这些笔记，我就可以回去看看我做了什么。

David [7:29]:

令人惊讶的是你一直保持在。当你通读你的文章时，你会对你所做的事情有一种自信。然而，你不可能同时精通所有这些不同的主题。其中有多少是边走边学？我很好奇。

Craig:

我总是边走边学。我选择我的主题是因为它是我当时正在工作的东西——这是你能真正工作的唯一方式。如果我说，<q>好吧，我今天就要学习 WebGL，</q>这会花去我一辈子的时间，我不会很擅长，我也不想发表一篇关于它的文章。

因此，每当我在做一个特定的应用程序，或者一个网站，或者一些 WordPress 主题时——不管是什么——如果我找到了我感兴趣的东西，或者我解决了一个特定的问题，并且我学到了一些东西，我就会写下来。无论如何，把它写下来是了解你自己的最好方法。

所以这就是我这么做的原因。我也对这个行业发生的事情感兴趣。所以我确实会写一些对话性的文章——比如浏览器趋势、新闻和事件——因为我认为进行这些讨论并听取其他人的意见是有好处的。

Tim [8:56]:

我的问题是——这些天你写了很多文章——你认为五年后你会写些什么，如果这是一个可以回答的问题的话？

Craig:

好吧，我写了一篇[网络预测文章](https://www.sitepoint.com/10-web-predictions-2016/)大部分时间，我得到了它可悲的错误。你在问*我*！*【笑】*

所以如果我不能预测未来 12 个月，我当然也不能预测未来 60 个月。我想，我所能保证的是，未来永远不会如你所愿。在事情被证明之前，不要相信技术炒作。

有几件事让我兴奋。当然是进步的网络应用。我认为我们正在进入一个阶段，我们可以在网络上复制本地、移动/桌面应用程序的大部分功能。五年内，每个应用程序都将基于浏览器吗？可能不会，但我们会越来越接近。

除此之外，我认为还有一些有趣的话题即将出现。虚拟现实目前是一个热门话题，我期待着有一天我可以戴上护目镜，在大型虚拟影院大小的 IDE 上编写代码。当然，物联网正在成为一个巨大的话题，当然，如果你阅读 SitePoint，我知道 SitePoint 上的 Patrick 写了许多奇怪而奇妙的设备。

虽然，我不知道它能走多远。前几天我读到了一些可以上网的衣夹，我认为这可能有点过头了。

David [10:48]:

这就是未来:你已经有了可以上网的衣服挂钩网站。今天就出去注册吧。

Tim:

我现在对未来感到恐惧。

Craig:

但是我认为这将会和我们预期的不一样。我们只能推断我们现在所知道的，通常是一些较小的事情——那些你认为没有任何重要性的事情——突然变得重要起来。

David:

所以，为了做这种预测，我很好奇你出去读什么。你遵循什么？你关注和阅读的人是谁？这些天谁对你有影响？

Craig:

哎呀，我的意思是一切。你必须一只眼睛盯着 Twitter，一只眼睛盯着 Medium，它有各种各样的人——这个行业的每个人——有太多的人了。

我不得不分享我的推特账户:上面有几百个，他们都是说过重要事情的人。或者我读过一篇文章，我觉得那很棒。当然，还有 SitePoint 上的所有人。我尽量阅读 SitePoint 上的每一篇文章。

而且要赶上变得相当困难，只是因为现在出版了这么多。但是这是一个很棒的社区，他们写一些我以前从未接触过的东西，他们做得比我好得多。我确实听了不少播客。大概最主要的是[商店脱口秀](http://shoptalkshow.com/)——克里斯·科伊尔和戴夫·鲁伯特。那很好。

但我也是一个老派，我仍然使用 RSS，尽管谷歌试图阻止我。所以我订阅了很多不同的网站——黑客新闻，所有常见的嫌疑人——并且尽可能地关注。因为我认为，就我写的文章数量而言，我已经读得更多了。

我只挑选我感兴趣的东西，幸运的是——或者不幸的是！—外面有很多令人着迷的东西，你可以花一生的时间来阅读文章。

David:

这是千真万确的——尽管令人印象深刻的是你不仅在阅读，而且还在出去做事。

当你开始接触这项技术时，是什么真正吸引了你？

Craig:

这真的是我喜欢的东西的组合。我想在早期——我刚开始工作的早期——这只是设计方面。当然，事实是，你可以发布一些东西，它可以立即被数百万人看到，而今天，可能会有数十亿人看到。

事实上，你有一个可以分发的网址，我来自一个网络很少的时代。我在 80 年代开始编码，如果你想在用户面前得到一些东西，你必须把它放在软盘上。你必须把它交给他们，让他们安装，并向他们展示你做了什么。那是不必要的，现在我们有了网络，它只是让事情变得容易多了。你可以立刻在某人面前得到某样东西。我认为这是最大的吸引力，也是为什么网络永远不会消亡——不管别人怎么说。因为你能给用户的只是即时部署和即时满足。

Tim [14:15]:

我非常喜欢听到网络永远不会消亡的说法。这实际上是我为 SitePoint 写的第一篇文章的主题。所以很好听。

有没有一项技术，或者一个项目，是你真正感到兴奋的，你现在正在做的？

Craig:

嗯，我用过很多不同的技术。我是从 Perl 开始的——或者肯定是在 web 上用 Perl 开始的。我做了经典 ASP，我做了。NET，PHP——我做了很长时间的 PHP。

在过去的一两年里，我实际上一直在使用 Node，这真的是一个新发现。我不得不承认，我是一个 JavaScript 迷。我喜欢它，即使它被认为是一个笑话，事实上你可以在客户端*和服务器端*编写 JavaScript，而且你不需要改变思维模式。

我知道您必须做不同的事情——您将与数据库通信，或者您将操作 DOM。但是语法相似的事实为我节省了很多时间。我不需要切换模式。

我也认为 Node.js 生态系统非常令人兴奋。NPM 真的改变了我的工作方式。像 Gulp 和 Express 这样的东西都是很棒的项目。

这是让我兴奋的时刻。当然，这只是在下一件事情发生之前。

Tim [15:52]:

是的，非常正确。你提到这一点很有趣，因为我现在正在启动一个相当大的节点项目，我注意到了你所注意到的，你在客户端和服务器端都编写了 JavaScript。这是同样的语言，但是你关注的是不同的——这种范式非常有趣，因为我习惯于更经典的<q>我需要在服务器上做一些事情；让我去看看如何用 PHP 实现 XYZ，然后把它发送给 JSON，供 JavaScript 使用和处理。</q>

Craig:

是的，我喜欢 PHP。我仍然在使用它，我仍然认为它有它的好处，尤其是在网站上添加简单的功能。而且非常容易上手 PHP。但是我认为 Node 的主要好处是——当然对我来说是这样，对你来说也是这样——它只是在语法上，我认为它也非常简洁。有了 JavaScript，你可以用非常简洁的方式做事。仅仅原型继承本身就使得设置对象变得非常快速和容易；你不必定义一个类。

所以我喜欢它的这一部分，你可以做一些非常快速的开发——即使 JSON 在客户机和服务器上都是本地的。没有翻译:你不是一直在使用 JSON 编码和解码。这样可以节省很多时间。

Tim:

哦是的。

David [17:20]:

我认为这个节点现在也越来越受到尊重。我记得当它开始的时候，很难让小公司同意在 Node 做一个项目。

我很好奇你是否在让人们参与进来时遇到了什么困难。

Craig:

我没有，但这部分是因为我倾向于做的工作是由我领导的。因此，如果我想做一点 Node，它往往会在 Node 中完成。是的，我能理解，这是一项相对较新的技术。我想它是从 2009 年才出现的，但实际上人们只是在过去几年才开始注意到它。

但就在几天前，我与一名招聘人员交谈，他从未听说过这项技术，当然也没有积极寻找节点开发人员。所以我认为这需要一点时间。甚至 PHP 可能也花了整整十年才出现 PHP 工作。

我想说的是——没有必要重写整个系统，因为 Node 已经出现了。但是，如果你正在做微服务之类的事情，如果你正在做一些新的事情，你当然应该考虑它，那么它当然值得使用 Node，并测试它，看看它对你自己来说是什么样的。它可能对你不起作用，但我不知道有谁不喜欢它。

这是一种新的思维方式，刚开始可能有点不寻常，只是因为你不习惯在服务器上使用回调之类的东西。但这是一个伟大的生态系统，我认为它将是未来几年最受欢迎的生态系统之一。

David:

我倾向于同意，并且我认为作为开发人员，我们有义务不断尝试所有出现的新事物。听起来你已经在职业生涯中找到了一个好的位置，因为你在自我管理。你在外面创造东西。你总是自我管理你的职业生涯吗，还是你有工作背景，然后进入这个职业？

Craig:

是的，这不是我积极管理的事情，我的事业。我想我已经陷入其中很多次了。我的意思是，即使作为一名自由职业者，我也不会说我是一个特别热衷于商业的人。我只接自己感兴趣的项目。作为雇员和自由职业者的一个奇怪的事情是，当你是自由职业者时，人们会听你的。你可以和员工说同样的话，但是人们不一定相信你说的话。我以自由职业者的身份参加会议，说我同意鲍勃说的任何话，然后人们就会注意到，这真的很奇怪。

David [20:18]:

这是一个非常有趣的观点，我认为这是我们的许多听众在如何管理自己的职业生涯方面可以借鉴的东西。

Craig:

我也这么认为我是说，自由职业并不适合每个人。我希望我几年前就这么做了，我喜欢这一切，因为它给了你想要的职业方向。有时候，我可能不得不出卖自己的技能，仅仅是为了把一些食物放在桌子上！但实际上，在过去的五六年里，我已经能够挑选我想做的项目，一切都很好。

你也可以和各种各样的人一起工作。如果某件事不是特别有趣，你知道这只是几个月的事情，然后你就会去做另一件事，你会遇到很多人。我认为这是很好的一部分。甚至我做的最后一个大合同实际上是和 Ruby 一起工作，这教会了我很多。

我以前从未真正使用过 Ruby，这有点出乎意料。所以你会有做很多不同事情的经历。我不知道有哪个自由职业者不喜欢这份工作。我知道有些人回去做全职工作，因为他们得到了一份好工作。但当然，如果你有好的技能，你永远不会失业。

Tim:

肯定！我完全同意。

克雷格，非常感谢你加入我们。人们怎样才能找到你，了解你，阅读你的作品？

Craig:

毫无疑问，[站点点](https://www.sitepoint.com/author/craig-buckler/)。我那里有不少文章。我想现在已经涨到一千五左右了。

Tim:

哦，我的天哪

Craig:

如果你想看我的文章，你可以在 SitePoint 找到我。但是推特，如果你看一下 [@CraigBuckler](https://twitter.com/craigbuckler) ，人们经常在那里和我开始对话，或者问一些关于特定帖子的问题。那可能是找到我的最好地方。

David:
