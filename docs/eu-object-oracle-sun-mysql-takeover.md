# 欧盟反对甲骨文收购 MySQL

> 原文：<https://www.sitepoint.com/eu-object-oracle-sun-mysql-takeover/>

在最近的调查之后，欧洲反垄断监管机构正式反对甲骨文收购太阳微系统公司。主要问题是 MySQL，世界上最受欢迎的开源数据库:欧盟认为这次收购会损害竞争。

2009 年 4 月宣布了甲骨文和太阳公司价值 74 亿美元的交易。尽管美国监管机构没有提出反对意见，但欧盟的调查可能会给合并计划带来风险。众所周知，欧盟竞争专员 Neelie Kroes 支持开源技术，甲骨文发布了一份声明回应她的担忧:

> [欧盟的反对意见]揭示了对数据库竞争和开源动态的深刻误解。
> 
> 了解开源软件的人都很清楚，因为 MySQL 是开源的，所以它不能被任何人控制。这就是开源的全部意义。

甲骨文和美国司法部都表示，数据库市场上至少有八家实力强大的公司，因此反竞争行为不太可能成功。然而，投资者已经对欧盟可能的反对做出了回应，Sun 目前的交易价格比甲骨文每股 9.50 美元的收购报价低了 12%以上。

## 甲骨文会伤害 MySQL 吗？

MySQL 不是甲骨文企业数据库的直接竞争对手，但开发者会怀疑该公司的动机。甲骨文可以拥抱 MySQL，通过销售咨询、培训、参考指南和数据库迁移工具来开拓市场。然而，他们也可能会剥夺 MySQL 的开发者资源，并移除 Sun 的支持网站和服务。

不幸的是，甲骨文对他们的 MySQL 计划守口如瓶。尽管 MySQL 的开源开发不依赖于该公司，但将数据库分流到多个系统中不太可能对任何人有利。最后，数据库市场真的有八大“强者”吗？MySQL 主导着网络，切换到另一个系统不是许多网站所有者和开发者愿意承担的任务。
