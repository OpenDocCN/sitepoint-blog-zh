# 如何避免项目预算井喷

> 原文：<https://www.sitepoint.com/how-to-avoid-project-budget-blowouts/>

我们都面临着工作看起来要失败的时候，但是没有什么比这种感觉更可怕的了，那就是你会搞砸项目预算。

灾难可能接踵而至——特别是如果你和客户没有特别亲密的关系，或者他们，或者他们的组织文化，决定了一定要有所指责。

当你意识到你努力工作的项目将会比你预计的花费更多的时间时，你会怎么做？显然，你的回答将取决于你的评估有多准确，以及你的项目范围有多清晰。

但我认为秘诀在于将报价和范围界定任务视为为您的项目和预算进度构建早期预警系统。

## 我的预警系统

在回应项目简报时，我通常会提供项目每个部分的估计时间——附带每小时的价格。

我通常将项目和评估分成几个部分，这样客户和我就知道哪部分工作需要更多时间，哪部分需要更少时间。我还增加了一些脂肪来避免超出预算。工作的每一部分都有自己的脂肪。

这种方法通常可以让我及早发现问题。如果我完成了一个项目的一大块，并且我已经用完了我为这一大块估计的所有时间，我知道我实际上已经用了*超过*的时间:我不仅占用了所有我期望使用的时间，而且还用完了 fat。

这促使我更仔细地研究这个项目，试图找出为什么它比预期的时间长。最近，我有一种感觉，一个项目的第一阶段会比类似的后续阶段花费更长的时间，因为最初客户和我会花时间来获得正确的内容基调。所以当我用完第一阶段的所有分配时间时，我有预感这就是原因。那就是:在调查中，我发现我花了比平时更多的时间在修改上，因为我们试图确定内容的个性。

## 预警反应

我的预警系统就是:一个预警系统。它能做的就是让我识别出什么时候某件事情花费了过多的时间。

我如何处理这些警告信息至关重要。

几乎在所有情况下，我都会在达到项目中一项工作的预算限制之前告诉我的客户。一旦我意识到我正在吃掉我在报价中构建的脂肪，我就告诉他们，这样他们就知道进一步的修改或请求对这部分工作意味着什么。

当我最近与我刚才提到的客户达到我的估计极限时，我告诉他们这一点，并重申了为什么我们预计这部分工作比其余部分需要更长时间的原因。在这种情况下，我们密切合作，我相信我们不会超出我的预算，但当我告诉他们这一点时，我解释说坚持项目这一部分的预算需要团队合作——他们和我都将负责保持事情正常进行。

当然，如果你的早期预警系统在项目的第一阶段就失灵了，你可能会考虑重新评估你对项目后期的评估。在我的案例中，我和客户预计第一阶段的工作会比其他阶段花费更长的时间，所以我们保留了对项目后期阶段的估计，最终我们的预算低于预算。

但是一些不可预见的问题可能会给你一个很好的理由来重新评估这些估计。一个常见的问题是签署。通常，客户告诉我他们会有一个快速、简单的签署程序，并且会很快将任何修改反馈给我。然后，到时候，企业中的每个经理都希望审查内容，进行修改，并审查—并可能对—实现的修改进行评论。

如果你在项目的第一阶段发现了这些不可避免的问题——或者其他客户的怪癖，当它们触发你的预警系统时，这是一个很好的理由来回顾你为项目的其余部分所做的估计，并为每一个增加一个比例缓冲。与客户讨论这一点，如果他们想保持较低的估计——或者甚至保持在预算内——他们可以选择简化未来项目阶段的签署程序。这是我避免预算超支的方法——到目前为止，这种方法对我很有效。您有什么基本流程来计划和应对项目预算超支？
