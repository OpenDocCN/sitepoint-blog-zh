# 挤压你的 PNGs 的最后一滴

> 原文:[https://www . site point . com/squishing-the-last-drops-from-your-pngs/](https://www.sitepoint.com/squishing-the-last-drops-from-your-pngs/)

我一直觉得 Fireworks 有 Photoshop 覆盖的一个领域是它的 PNG 导出工具。除了更加灵活和先进，它几乎总是产生较小的文件。

但是你能做得更好吗？也许现在你可以了。

早在 7 月，Sergey Chikuyonok 在《Smashing》杂志上写了一篇关于[优化 PNGs](http://www.smashingmagazine.com/2009/07/15/clever-png-optimization-techniques/) 的非常有趣的文章。基本上，Sergey 已经花了一些时间了解 PNG 的压缩魔法是如何工作的，然后找出了一些你可以用 Photoshop 来使你的图像更加 PNG 友好的技术。

聪明的东西，值得一读。但是，如果你只是想要即时的 PNG 满足感，而没有厚重的数学入门呢？

[![Puny PNG](../Images/948105828df64a3a91b159fe832b343b.png)T2】](http://www.gracepointafterfive.com/punypng)

没问题。令人高兴的是，GracePoint 的聪明人在五年后构建了一个免费的在线 PNG 压缩工具，名为 puny PNG T3，它无缝集成了 Sergey 的一些最好的 PNG 巫术。

我不得不承认，我怀疑我可爱的烟花伪造的 png 文件是否有任何值得修剪的脂肪，但我已经能够将大多数文件减少 15%左右——对于具有大量透明度的 8 位 png 文件来说甚至更多。

在一些随机测试中，我能够减少以下内容:

*   Mozilla 徽标:10%
*   SitePoint 徽标(最初是 GIF 格式):26%
*   Typekit 的标志:33%
*   谷歌的搜索页面标志:10%

好吧，那么在高速、宽带连接的时代，真的会有人*费心试图从 25KB 的 PNG 中挤出几个微不足道的 KB 吗？*

 *有些人可能会说这对于每一个随机使用的图形来说都是多余的，但是在某些场合下这确实很有意义。例如:

*   网站标志、标题和其他重复的图形:
    几乎每个访问者都会遇到这些图形，所以在准备过程中稍加小心，每次页面加载都会有回报。
*   CSS 图像精灵 :
    CSS 精灵是预先缓存翻转、动画和其他效果图像的好方法，(并减少服务器连接)，但它们依赖于用户接收一个大图像比接收许多小图像更快。事实上，像 YouTube 这样的网站几乎把所有的界面图片都放在一个大图片上。显然，你能做的任何事情都可以让这样的图像下载得更快，这对每个人来说都是一个巨大的奖励。

有趣的是，我无法从那个惊人的 13KB Youtube 雪碧中挤出一个*单个* KB。

无论如何，在你的一些 png 上测试一下，让我知道结果。
