# 浏览器大战:不偏袒任何一方

> 原文：<https://www.sitepoint.com/the-browser-wars-not-taking-sides/>

以下内容转载自《科技时报》第 151 期。

在过去的一周里，[的 ie 7](http://www.microsoft.com/ie/)和[的 Firefox 2](http://www.mozilla.com/firefox/) 都发布了。浏览器大战又开始了，这是我们作为负责任的开发者第一次有义务站在哪一边。

在对 SitePoint 上的 [IE 7 发布公告的回应中，我对](https://www.sitepoint.com/be-the-first-kid-on-the-block-running-ie7-final/)[评论](https://www.sitepoint.com/be-the-first-kid-on-the-block-running-ie7-final/#comments)发誓“坚持使用 Firefox”的比例感到惊讶，有些人甚至暗示他们根本不会费心安装 IE 7 更新！

由于其多功能的扩展系统，Firefox 很可能仍然是大多数 web 开发人员系统上的主要浏览器，我肯定会同意这一点。但是这些最新的版本绝不是我们在工作中把 Internet Explorer 当作二等公民的理由！如果说有什么不同的话，那就是我们应该通过将我们的工作建立在那些标准的基础上来拥抱任何优先支持 web 标准的浏览器。

在这方面，Internet Explorer 7 无疑是朝着正确方向迈出的一大步。我只是不明白人们怎么能声称 IE 7 有“可怕的 CSS 支持”。微软在修复 ie6 CSS 支持的最大缺陷方面做得非常出色，根据这两个浏览器 CSS 支持的最新进展，我发现自己开始怀疑 Mozilla 在这一领域保持领先微软的能力。见鬼，如果我今天只打算使用具有最佳 CSS 支持的浏览器，我会把我的马车拴在 Opera 上。

就其今天为支持标准所做的工作而言，微软是最棒的。微软有很多地方需要弥补，目前的这些努力是否会持续还有待观察，但如果我们考虑哪个浏览器供应商正在做的工作将最有利于日常 web 开发人员(相对于 Firefox 2 中出现的更深奥的功能)，微软可以说是目前浏览器开发中做得最好的！

Internet Explorer 7 是一个里程碑式的版本，因为它将允许我们在一年后回顾并思考(正如我们今天思考网景 4 过去强加给我们的限制一样)如果没有 PNG 透明、`min` / `max-height` / `width`和`:hover`等所有元素(不仅仅是超链接)，我们是如何度过的。

不要误会我的意思:Firefox 2 是一个令人印象深刻的更新，完成了 Mozilla 在三年内发布三个主要 Firefox 版本的帽子戏法。隐藏在它下面的一些令人兴奋的技术(特别是 JavaScript 引擎)暗示了一个令人兴奋的未来，在这个未来，它们将得到更广泛的支持。通过这次发布，Firefox 真正巩固了它在各地开发者桌面上的地位。

但多年来，我第一次相信，作为开发者，我们有责任让用户自己决定他们更喜欢哪种浏览器。通过建立基于标准的网站，确保浏览器制造商对改进标准支持的持续兴趣，我们可以退后一步，让浏览器之战不是基于哪个浏览器最正确地呈现网站(它们应该都呈现得很好)，而是基于哪个浏览器能够为其特定用户群提供最佳用户体验。
