# 著名的 Rails Screencast 获得更新

> 原文：<https://www.sitepoint.com/famous-rails-screencast-gets-an-update/>

几年前，当 Ruby on Rails 框架首次推出时，没有什么比现在著名的“在 15 分钟内创建一个博客”的截屏更能帮助它出名了。在这篇文章中，Rails 创始人 David Heinemeier Hansson 只用了 15 分钟就用 Rails 创建了一个博客引擎。那次演示是如此的强大，以至于它帮助 Rails 进入了 web 开发世界的一个主要框架，一个程序员仍然需要高薪工作的框架。

该演示非常成功，以至于许多其他 web 框架都模仿 Rails 截屏并发布了自己的版本。[阿克洛斯](http://www.akelos.org/screencasts)、 [CakePHP](http://www.cakephp.org/screencasts/view/3) 、 [CodeIgniter](http://codeigniter.com/tutorials/watch/blog/) 和其他人已经发布了受原始 Rails 视频启发的截屏。

最近，汉森颇具影响力的截屏[收到了 Rails 2.2 的更新](http://weblog.rubyonrails.org/2008/11/27/new-15-minute-blog-video-on-rails-2-2/comments/23491#comment-23491)。新的截屏是由 [Railscasts](http://www.railscasts.com/) 的 Ryan Bates 制作的，该公司每周都会发布免费的 Rails 教程截屏。除了贝茨带来的更好的制作价值之外，新的屏幕广播还通过在短短 15 分钟内创建一个包括评论、RSS 提要、API、管理界面、AJAX 等内容的博客应用程序而提升了几个档次。

新的截屏可能实际上有点太快，塞进了太多的信息，作为一种学习辅助工具非常有用(当然，这真的取决于你综合信息的速度和你个人的学习风格)。但是，在以一种快速且易于理解的格式展示 Rails 框架之美方面，它无疑提高了标准。

[官方截屏页面](http://rubyonrails.org/screencasts)还链接了之前的两个 Rails 博客截屏，这两个截屏是为框架的 0.5 和 1.0 版本创建的，对于那些有兴趣了解框架一年来的变化的人来说。

*注意:一定要看看 SitePoint 自己的[网络开发培训视频](https://www.sitepoint.com/premium/)！*
