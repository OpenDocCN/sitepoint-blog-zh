# 南方网络方向，第二天:人群 vs 社区

> 原文:[https://www . site point . com/we B- directions-south-day-two-crowd-vs-community/](https://www.sitepoint.com/web-directions-south-day-two-crowd-vs-community/)

也许是因为上周的 Web Directions South conference 是我参加的第五次会议，我比以前更难给人留下深刻印象。
[Web 方向南方的一天](https://www.sitepoint.com/web-directions-south-day-one-conversation-is-king/)并没有像我希望的那样真的跳出来打我耳光。这并不是说组织者没有做好工作，也不是说演讲者不是一流的。但是这些年来参加了很多会议，我可能已经被过去的高标准宠坏了。因此，我真的没有任何“啊哈！”时刻。

幸运的是，第二天弥补了这一点。

杰弗里·维恩以他独一无二的风格开创了局面。Jeffrey 克服了一个可能让他慌乱的技术故障，但他似乎完全不担心，因为他讲述了他为 Google Analytics 设计界面的一些经验，并挑战观众继续努力通过用户测试、耐心和坚持来实现伟大的设计。

digg.com[的创意总监丹尼尔·布卡](http://digg.com/)，对我来说也是一个亮点。看到 digg 评论系统的各种迭代，并对 digg 团队生产该界面当前版本所遵循的流程有所了解，真的很有启发。(就我个人而言，我总是发现发布到 digg 上的故事比随后的评论更有价值，这是一个该死的耻辱，因为该网站有一个非常好的对话界面。)

道格拉斯·克洛克福特也是一位演讲者，我很高兴我没有错过。Douglas 不仅挑战了标准机构的有效性，如 [W3C](https://www.w3.org/) 和 [ECMA](http://www.ecma-international.org/) ，还做出了大胆的声明，如“JavaScript 遭受的安全漏洞是 web 标准所要求的”，“web 现在是创新的障碍”和“web 开发需要掌握变通办法——但你不能绕过安全。”凯文和道格拉斯度过了一段美好的时光，所以期待在接下来的一周我们能更多地谈论 sitepoint.com。

我也非常自豪地看到这么多 SitePoint 作者、博客作者和工作人员在舞台上踢屁股——Kay Smoljak 对她开始自己的 web 开发业务的经历的描述挤满了房间，直到只有站着的地方，而 T2 Myles Eftos 对 OpenID 和 OAuth 的解释让房间里的人都在想“为什么我自己没有实现它？”

[纪娜·波顿](https://www.sitepoint.com/blog/)的[创造性感的样式表](https://www.sitepoint.com/article/jina-bolton-interview/)的演讲让每个自以为了解 CSS 选择器的人都在想“好家伙，我有一些[的读物要读……](https://www.sitepoint.com/books/csswrong1/)”而 SitePoint 总部自己的 [Lucas Chan](https://www.sitepoint.com/blog/) 举行了一场只能被描述为三大 web 框架有史以来最深入、最令人讨厌的辩论。我没能参加 Rails 博客作者 Tim Lucas 的 iPhone 网络应用会议，或者偶尔参加专家评论人 Derek Featherstone 的可访问性会议，但是我听到的都是积极的消息。干得好团队站点点！

结束这一天的是公开演讲老手马克·佩斯的闭幕主题演讲，他可能给自己设定了一个不可能的挑战，那就是在前两年的闭幕主题演讲中增加赌注。虽然对我来说有点短，但这并不反映他令人印象深刻和激动人心的演讲风格——只是反映了去年的演讲对我产生了多大的共鸣。他最后提出了挑战，要求其他人在明年挺身而出，我很高兴看到谁准备站出来。他无疑留下了巨大的空缺…

结合周四晚上的 [WebJam](https://www.sitepoint.com/webjam08-smells-like-geek-spirits/) 嘉年华和周五由 99designs 赞助的会议结束派对，Web Directions South 实现了我一直以来的高期望。悉尼为我们带来了最好的天气(保罗、亚历克斯和我周六利用这个机会在邦迪海滩快速泡了一会儿海)，我带着满脑子的想法、能量和对我们工作的这个奇怪、美妙的行业的热情回家了。

现在，开始考虑如何说服我的老板把我派到[网络边缘](http://www.edgeoftheweb.org.au/)或[网络股票](http://www.webstock.org.nz/)
