# 你的文本代码比重要吗？

> 原文:[https://www.sitepoint.com/text-to-code-ratio/](https://www.sitepoint.com/text-to-code-ratio/)

一个页面的文字与代码的比率是衡量内容的数量与结构的比率。例如，假设您的页面有 1000 字节。如果 700 字节用于 HTML 标签和嵌入的 CSS 或 JavaScript，那么 300 字节将是可读的内容。您的文本比例为 3:7 或 30%。有许多工具可以帮助你评估这个数字——最容易使用的工具之一是 [DOM Monster](https://www.sitepoint.com/dom-monster-bookmarklet-tool/) 。

但是有没有一个理想的文字与代码的比例呢？

## 搜索引擎优化

一些搜索引擎优化专家声称，更高的文本比例可以提高搜索引擎的排名。我甚至看到 42%被说成是一个神奇的完美百分比，这将提高你的 Google PageRank。然而，我怀疑这个数字是被一个痴迷于 *[《银河系漫游指南》](http://en.wikipedia.org/wiki/42)* 的 SEO 蛇油“专家”摘走的。

就我个人而言，我不认为文本与代码的比例对 SEO 有重大影响。也就是说，谷歌会忽略超过第一个 100Kb 的内容，这样更大的页面可以从更高的文本比例中受益。然而，如果你超过 100Kb，我建议将文档分成几个更集中的页面会是一个更有建设性的 SEO 练习。

## 码效率

一般来说，使用尽可能少的代码是最佳实践。不必要的标签会导致额外的页面权重、更慢的下载速度和更低效的浏览器渲染。这也使得你的代码更难维护。

如果你的页面是轻量级的，使用干净的，语义化的 HTML 和外部的 CSS 和 JavaScript 文件，你的文本代码比自然会下降。有几个例外:

*   较短的页面将具有较低的文本比率，因为您需要最少数量的代码元素来创建有效的 HTML 文档。
*   包含大量媒体的页面，例如包含图像或视频的图库，通常具有较低的文本比例。
*   Flash 或 Ajax 支持的 web 应用程序可能没有任何内容——但它仍然存在。

一般来说，超过 50%的文本代码比在大多数内容页面上都是可以实现的。

那么，作为有良知的开发人员，我们应该衡量我们的文本代码比吗？

如果你已经在使用好的编码技术，那就没必要这么麻烦了。你可以把它作为衡量效率的一个因素，因为一个好的开发者通常会有更高的文本代码比。然而，不应孤立地考虑比率。毕竟，一个在任何地方都能工作的 25%的页面通常比一个得分 50%但在大多数浏览器中失败的页面要好。

你考虑你的网页的文字与代码的比例吗？你用它来评价自己的表现吗？
