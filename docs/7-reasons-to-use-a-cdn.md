# 使用内容交付网络的 7 个理由

> 原文：<https://www.sitepoint.com/7-reasons-to-use-a-cdn/>

在过去的几年里，内容交付网络(cdn)彻底改变了网络托管。你可以在多个系统上分发文件和加载，而不是在一台服务器上托管你的网站。

## CDN 上可以托管哪些文件？

大多数 cdn 用于托管静态资源，如图像、视频、音频剪辑、CSS 文件和 JavaScript。你会在各种公共和私人 CDN 系统上找到常见的 JavaScript 库、HTML5 填充、CSS 重置、字体和其他资产。

## CDN 服务

谷歌、微软、雅虎和其他大型网络组织提供许多免费的 cdn。例如，当 YouTube 和 T2 的 Vimeo 提供令人惊叹的免费服务时，很少有人拥有自己的视频。类似地，如果您需要 jQuery，您可以使用以下命令在任何页面上加载它:

```
 <script src="https://ajax.googleapis.com/ajax/libs/jquery/1.6.4/jquery.min.js.js"></script> 
```

或者

```
 <script src="http://ajax.aspnetcdn.com/ajax/jQuery/jquery-1.6.4.min.js"></script> 
```

或者，你可以使用私有的商业 CDN，比如亚马逊 S3 T1、T2、微软 Azure T3 和 T4 MaxCDN T5 来存放你自己的文件。

## 使用 CDN 的好处

有几个原因可以说明 CDN 对你的网站和公司有好处。

**1。不同域**
浏览器限制单个域的并发连接数(文件下载)。大多数允许四个活动连接，因此第五次下载被阻止，直到完全检索到前一个文件。当从同一个站点下载许多大文件时，您经常可以看到这种限制。

CDN 文件托管在不同的域中。实际上，一个 CDN 允许浏览器同时下载另外四个文件。

**2。文件可能被预先缓存**
jQuery 在 web 上无处不在。很有可能有人在访问你的页面时已经访问过一个使用谷歌 CDN 的网站。因此，该文件已经被您的浏览器缓存，不需要再次下载。

**3。高容量基础设施**
你可能有很棒的主机，但我打赌它没有谷歌、微软或雅虎提供的容量或可扩展性。更好的 cdn 提供更高的可用性、更低的网络延迟和更低的分组丢失。

**4。分布式数据中心**
如果你的主要网络服务器设在达拉斯，那么来自欧洲或亚洲的用户在访问你的文件时必须进行多次跨洲电子跳跃。许多 cdn 提供更接近用户的本地化数据中心，从而加快下载速度。

**5。内置版本控制**
通常可以链接到 CSS 文件或 JavaScript 库的特定版本。如果需要，您通常可以请求“最新”版本。

**6。许多商业 cdn 提供文件使用报告，因为他们通常按字节收费。这些报告可以补充您自己的网站分析，在某些情况下，可能会提供更好的视频观看和下载印象。**

**7。提升性能并节省资金**
CDN 可以分配负载、节省带宽、提升性能并降低您现有的托管成本——通常是免费的。
