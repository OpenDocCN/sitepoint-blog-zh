# HTML 电子邮件–正确的富媒体方式

> 原文：<https://www.sitepoint.com/email-rich-media-right-way/>

**正如我们上次[在](http://www.promotionbase.com/article/571)讨论的那样，越来越多的电子邮件营销人员希望利用互联网的“力量”更好地与客户沟通。他们如何利用媒体的图形和互动功能？富媒体电子邮件。虽然这个决定可能很容易，但是一个好的富媒体电子邮件活动的执行可能很棘手。让我们考虑两种流行的格式，并解决每种格式的常见问题。**

##### 在你的 HTML 邮件中使用 JavaScript

由于 JavaScript 主要是一种独特和专有的元素(我的意思是你可以编写 100 种不同的脚本，它们基本上以不同的方式做同样的事情)，复杂的 JavaScript 总是会给你的 HTML 电子邮件带来麻烦。

由于安装了各种各样的电子邮件客户端、浏览器、安全设置、更新和服务包，很难预测脚本将如何针对任何给定的电子邮件客户端执行。JavaScripts 可以使浏览器& Outlook 2000 禁用电子邮件文档中包含的任何活动脚本(由于恶意脚本，电子邮件的安全性有所提高)。

明智的做法是测试您希望包含在邮件中的任何 Javascript。不是所有的电子邮件客户端都可以处理脚本，大多数基于 Web 的系统通常会禁用脚本，以防止恶意代码在系统上执行。可以使用 Javascript 导航表单(如跳转菜单表单)，但即使是这些也只能在非基于 Web 的电子邮件客户端中使用。大多数其他导航形式都不受支持，因为它们不能在大多数基于网络的电子邮件客户端上运行。然而，使用 GET/POST 方法的表单将在基于 Web 的电子邮件客户端中正确地呈现和测试，并且您可以经常使用它来解决 HTML 电子邮件中的许多 Javascript 问题。

记住！大多数基于网络的电子邮件浏览器本身就是一个“T0 ”,并使用 Javascript 以这样或那样的方式来增强他们的产品。因此，为了不干扰他们的产品，你的脚本必须是相当无害的。

##### 闪烁您的电子邮件:富媒体和 HTML 电子邮件

最近，人们对将 Flash 添加到日常的普通电子邮件中以给客户、潜在客户或时事通讯订户留下深刻印象产生了浓厚的兴趣。然而，事实是电子邮件 HTML 浏览器并不等同于他们的网页浏览器。各种各样的设置、首选项、安全更新、版本和第三方应用程序使得用户体验难以预测，这使得情况变得更加复杂。当你创建、设计和营销 HTML 电子邮件时，这是一个有趣的问题。你可能会讨厌我要说的话，但请不要开枪打信使！

除非你绝对确定你的收件人使用的电子邮件客户端可以处理 Flash 内容，否则你不应该在你的 HTML 电子邮件中使用 Flash 或任何其他富媒体内容。此外，您应该只将 Flash/富媒体内容发送给提出请求的人，或者与您有一致同意且非常了解的营销关系的人。第一次我不得不等了将近一个小时才下载到一封紧急邮件，那是在一个酒店的拨号账户上。一封电子邮件花费了我将近 10 美元和一个小时的时间，这可不是你想和你的顾客或客户建立的关系。

但是，如果您必须通过电子邮件发送 Flash 内容，请记住以下提示:

不要试图用活动脚本来控制你的 Flash。

由于安装了各种各样的电子邮件客户端、浏览器、安全设置、更新和服务包，您永远不知道脚本将如何针对特定的电子邮件客户端执行。JavaScripts 可以使浏览器、Outlook 2000 和修补的电子邮件客户端禁用电子邮件文档中包含的活动脚本(由于恶意脚本，电子邮件安全性有所提高)。

***考虑附加或发送链接。***

大多数基于网络的电子邮件客户端(Hotmail、Yahoo 等。)剥离 Flash 内容。因此，发送嵌入的 Flash 内容时，收件人在基于 Web 的客户端打开它，却什么也看不到，这种情况并不少见。同样，你不能依靠一个<noembed>来提供内容的替代链接。对于所有基于 Web 的收件人以及那些其系统、ISP、网络安全或其他变量干扰其查看 Flash 内容的人，请在您的 Flash 内容之前或之后包含一个文本链接。</noembed>

您最好附加或发送 Flash 内容的链接，而不是将内容包含在电子邮件中。通过将 Flash 内容作为附件或链接发送，您可以避开电子邮件客户端和浏览器的一些限制。Flash 内容将在浏览器中呈现，而不是在你的电子邮件中，如果收件人有插件，他们将能够查看 Flash 文件。

***确保您的文件不会立即开始播放。***

用 onClick、trigger 或其他事件控制你的内容。一句简单的“点击此处获取重要信息”就足够了。当观众准备好时，允许他们开始演示。如果在 Outlook 预览窗口中查看，开始流动的 Flash 或 Shockwave 片段将在打开电子邮件时再次开始。

这通常会造成接收者的声音系统混乱，扭曲你想要传达的信息。如果收件人的电脑在工作日突然发出意想不到的声音，没有什么能比这更快地删除你的邮件了。

##### 富媒体–正确的方法

如果你打算设计、发送和期待对快速邮件的回复，这些只是你应该注意的几件事。Flash 和其他富媒体可能都是 2001 年——网络浏览器世界的“前沿”。然而，不幸的是，你的普通 HTML 电子邮件浏览器似乎停留在 1998 年初。
