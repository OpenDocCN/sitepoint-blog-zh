# 审查–SiteWorks Professional 5

> 原文：<https://www.sitepoint.com/review-siteworks-professional-5/>

不久前，我决定创建一个 Web 开发产品评论网站。凭借我有限的(实际上，“有限的”可能会给我加分——假设不存在)编程知识，我决定查看一些可用的付费文章内容管理系统(CMS)脚本。其中一个引起我注意的是 SiteWorks Professional(从现在开始我们称它为 SWP)，由 SiteCubed 开发。

SWP 是一个基于 PHP 和 MySQL 的脚本，具有许多你期望从文章 CMS 中得到的特性。有一些不祥的遗漏，如缺少“给朋友发邮件”的脚本，但这些遗漏通常很小，不应该阻止你检查甚至使用 SWP。

除了实际的文章和类别(脚本的大部分)，SWP 还提供了一个“新闻”部分，在这里你可以标记其他网站的新闻文章并链接到它们，一个投票功能，时事通讯订阅脚本，以及一些现成的内容块和页面。它还允许您在索引页面上以“最近主题”块的形式进行一些基本的 vBulletin/phpBB 集成。

被这种产品的潜力所吸引，我得到了一份拷贝并开始工作。

##### 设置和安装

脚本的安装相当简单，正如您所期望的那样。SWP 有以下系统要求:

*   操作系统:Windows NT/2000、Linux 或 Unix
*   Web 服务器:Apache 或 IIS，取决于您的操作系统
*   PHP 引擎:PHP 4.1 或以上
*   MySQL: MySQL 3.23 或以上
*   浏览器:要管理您的网站，您必须使用 Microsoft Internet Explorer(MSIE)5.5 或更高版本。注意，你只需要 MSIE 5.5 或以上版本就可以编辑你的网站，不需要查看。SWP 支持的网站可以在任何浏览器中查看。

如果你已经安装了 phpBB 或任何类似的脚本，你应该不会有任何问题；SWP 文档帮助您完成这一过程。成功安装脚本后，您无疑会找到管理部分。

##### 管理

您可以从管理部分管理默认 SWP 发行版的几乎所有方面。还记得我一分钟前提到的那些内容块吗？这些包括一个“分支机构”块和一个“方便提示”块。您可以在管理部分编辑这些内容，但我决定我更喜欢将这种性质的元素硬编码到网站中。类似地，SWP 带有“关于我们”、“联系我们”等。可通过管理区编辑的页面。同样，我选择了硬编码版本。我打开一个模板文件，把“分支机构”部分撕下来，准备继续进行——没有任何错误或其他问题。

这是我喜欢 SWP 的原因之一。事实上，任何我不喜欢的，或者不适合我制作的网站类型的功能，都可以在几分钟内轻松删除。我在删除一些功能时遇到了麻烦，但在 SWP 支持团队的帮助下，我最终成功提取了这些功能。

SWP 有一个非常好的作者/用户管理系统，允许你创建任意多的用户。每个作者的字段包括名字、姓氏、电子邮件、用户 Id、密码、图片、简历(由一个方便的 WYSIWYG 编辑器表示)和权限。然而，如果你想让用户登录到管理区来提交文章，你需要有适当的许可——稍后会详细介绍。

您可以向数据库中添加任意数量的“文章主题”(或类别)，并且在这些类别中，您可以添加您的文章。您可以将以下内容与每篇文章相关联:

*   标题
*   文章类型(教程、评论、总结、提示或采访)
*   作者
*   摘要(显示为文章摘要的一小段文本)
*   主题(或类别——您可以为每篇文章选择任意多个主题)
*   相关文章(您可以从当前数据库中的文章中选择任意多的文章)
*   相关书籍(这利用了 SWP 的内置“书籍”区域)
*   论坛链接(硬编码链接)
*   最多 3 个相关链接
*   答。zip 支持文件(用于与文章相关的下载)
*   出版日期
*   文章是否应该可见

关于文章类型的一个快速注意事项——您不能用任何可以被描述为“轻松”的东西来添加、编辑或删除类型如果你想完全删除一个文章类型(就像我一样),这很容易通过从模板文件中删除它来实现。我被告知在 SWP 的下一个版本中文章类型将是可编辑的。

文章可以分成多达二十页。所有的页面都可以在所见即所得编辑器的一个屏幕上快速方便地编辑，并且你可以给每个页面指定自己的标题。这种功能无疑是 SWP 的一大优势。

最后，SWP 有一个评论系统，允许你网站的访问者对文章发表评论。然而，它并没有那么强大。除了能够手动批准和删除评论，审核系统是相当不起眼的。

##### 模板和定制

如果你选择从一开始就运行 SWP，你将拥有一个看起来相当不错的网站。它配有 3 个预制模板，每个都令人满意，但并不出色。尽管如此，我看不出你为什么不想至少稍微定制它们，所以模板的默认外观不应该决定交易的成败。

如果您曾经使用过 phpBB、DRAMS 或类似的模板系统，您应该能够定制您的 SWP 模板。我建议选择您最喜欢的模板，然后根据您的要求进行定制。你可能会遇到一些问题(就像我一样)，此时，在 SWP 支持论坛上发帖是明智的。

我对这个脚本的一个问题是，它没有提供大多数人所说的“搜索引擎友好”的 URL。默认情况下，文章的 URL 看起来像这样:http://www.url.com/articles.php?articleId=1.然而，在 SWP 支持人员的帮助下，即使是像我这样的编程新手也能够启用更漂亮的 URL，例如:http://www.url.com/articles/1.

现有的有限修改社区包含在 SWP 支持论坛中。

##### 支持

在很大程度上，我对为该产品提供的支持非常满意。有些回复缺乏时效性，但是对我所有的问题都做了回复，我能够完成我想做的一切，这是最主要的。

支持通过电子邮件、[知识库](http://www.siteworkspro.com/support/knowledge base)和他们的[支持论坛](http://forums.sitecubed.com/index.php?&c=2)提供。

##### 许可证混乱和下一个版本

SWP 的许可证结构有点混乱。单个用户的版本起价为 99 美元，11 个或 11 个以上用户的版本起价为 299 美元。

那么，到底什么才是合格的“用户”呢？您可以创建任意多的用户帐户/作者来关联文章。但是，当用户实际登录到您的管理区域并做任何事情时——提交文章，更改他们自己的信息，或者授予他们每一个权限(您可以在创建或编辑帐户时设置权限)——他们被视为“管理员”，这就是许可证的价值所在。当创建者提到“单个用户许可证”时，他们指的是单个管理员，而管理员则是登录到您的 SWP 的管理部分的任何人。

我所知道的大多数内容网站都接受电子邮件提交，所以这可能不是什么大问题。在这种情况下，你的站点的编辑(可能是你自己)是唯一需要登录的人，所以单用户许可是合适的。

奇怪的是，SWP 并不强制执行定价。也就是说，你可以允许 100 万人成为 99 美元版本的“管理员”,没有人会知道。当然，你会知道——这足以促使你采取合法行动。如果你购买单用户许可证，你应该是唯一一个你允许登录到管理区的人。不是因为你必须这么做，而是因为这是正确的。

SiteCubed 的人向我保证，在 SWP 的下一个版本中，这种怪癖将会消失，SWP 将从每用户许可切换到每域许可，这更有意义。下一个版本的价格预计会更高，但实际定价尚未透露。

##### 结论

毫无疑问，SWP 有它的问题，也缺少一些特色。但是，总的来说，单用户许可证的价格使它成为一个可靠的解决方案。如果你正在寻找一个可以让你创建一个类似于 SitePoint 的网站的文章内容管理系统，SWP 绝对是值得考虑的(尽管它显然功能不太丰富)。
