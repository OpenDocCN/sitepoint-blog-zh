# IE8 自动更新即将开始

> 原文:[https://www.sitepoint.com/ie8-automatic-update/](https://www.sitepoint.com/ie8-automatic-update/)

微软宣布将于 2009 年 4 月的第三周通过 Windows 自动更新发布 Internet Explorer 8.0(查看 IEBlog 公告)。自 3 月 20 日以来，该浏览器的最终版本已经[可供下载，但它的普及相当缓慢，可能仅限于网络开发者和早期采用者。](https://www.sitepoint.com/ie8-released-today/)

自动更新可能会显著改变这种情况。Windows XP、Vista、Server 2003 和 Server 2008 上的所有 IE6 和 IE7 用户都将收到 IE8 更新通知。在 XP 和 2003 中会标记为*【高优先级】*，在 Vista 和 2008 中会标记为*【重要】*。这一推出最初将针对一小部分受众，但在未来几周内将扩展到覆盖整个用户群。

但是，IE8 不会自动安装，欢迎屏幕会给用户提供安装、稍后询问或放弃升级的选项。该公司还发布了一个[拦截器工具包](http://www.microsoft.com/downloads/details.aspx?FamilyID=21687628-5806-4ba6-9e4e-8e224ec6dd8c&displaylang=en)，以便 it 管理员可以禁用更新。

尽管大规模推出，但大多数 web 开发人员需要知道的是:
*这会最终根除 IE6 吗？*

现有的 IE7 用户可能会迅速转换，到今年年底，浏览器将会在统计图表的底部萎靡不振。不幸的是，IE6 用户已经阻止了之前的 IE7 更新——他们没有理由突然感到安装 IE8 的紧迫性。许多 IE6 用户可能无法切换，因为他们:

*   在受限的公司网络上或依赖于仅支持 IE6 的 web 应用程序
*   使用旧版本的 Windows，如 98、ME、2000 或 XP SP1
*   使用禁用更新的非法 Windows 副本，或
*   对使用 Windows 自动更新系统无知或紧张。

或者他们更喜欢 IE6 提供的更简单的浏览体验？

不管是什么原因，微软的自动更新可能不会对现有的 IE6 用户产生重大影响。我怀疑我们会有一个奇怪的情况，IE6 和 IE8 的市场份额都超过 IE7。庆祝 IE6 的消亡可能为时过早。

IE8 自动更新会成功还是越来越无关紧要？
