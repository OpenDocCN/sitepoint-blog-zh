# 你准备好规模化了吗？

> 原文：<https://www.sitepoint.com/are-you-ready-to-scale/>

一个典型网站的发展可能看起来有点像这样:

1.  你建立了你网站的第一个版本，并将其部署到一个共享的托管环境中(比如 Dreamhost)。一切都好。
2.  您需要为不断增长的客户群开发更多功能。性能和正常运行时间也是一个问题，所以你把你的站点转移到一个专用的服务器上。盒子里的自由！
3.  更多的顾客。他们想要更多的功能，而且他们现在就想要！你如何应付这种负荷？你把你的数据库放在它自己的专用服务器上。我们又可以呼吸了。

这里我们使用传统的 2 服务器 2 层后端。一台服务器负责生成和提供网页，另一台服务器负责存储数据。在这样的设置上运行时，您可能会幸运地遇到以下一些问题:

1.  可用性:您现在有 2 个失败点。如果您的 web 服务器或数据库服务器出现故障，您就有麻烦了。当这种情况发生时，你有什么计划？
2.  绩效:你是如何衡量的？在您不再及时处理请求之前，您的站点还能处理多少并发用户？更重要的是，当你到达那个点的时候你会做什么？
3.  备份:我们如何在不使整个网站离线的情况下准确快速地备份所有客户数据？
4.  State & Tangle:你把用户和会话数据存储在文件系统的什么地方？将站点/应用程序的不同区域划分到不同的服务器上有多容易？

如果你已经到了网站发展的这个阶段，但是还没有这些问题的答案，那么是时候开始考虑解决这些问题了。
