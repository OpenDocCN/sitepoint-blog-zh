# Web 开发者工具栏来到 Chrome

> 原文:[https://www.sitepoint.com/chrome-web-developer-toolbar/](https://www.sitepoint.com/chrome-web-developer-toolbar/)

Chris Pederick 的 [Web Developer](https://addons.mozilla.org/en-US/firefox/addon/60) 工具栏是为 Firefox 开发的首批扩展之一。就我个人而言，我认为这是许多开发人员转向并推广 Mozilla 浏览器的主要原因之一。

我记得第一次尝试 WDT，陶醉于它的巨大力量。如果你在过去的五年中开始了 web 开发，你会对最初可用的低质量客户端编码工具感到震惊——诸如元素概述、动态 CSS 编辑和 cookie 检查之类的功能非常困难，甚至不可能。

虽然 [Firebug](https://addons.mozilla.org/en-US/firefox/addon/1843) 和[类似的开发者控制台](https://www.sitepoint.com/opera-dragonfly-open-source/)已经风靡全球，但它们受到了克里斯设计的许多功能和概念的影响。

WDT 仍然是我必不可少的火狐插件之一，好消息是它已经被移植到谷歌 Chrome 浏览器上。

![Web Developer on Chrome](../Images/d8735a87b0151b090e31e1001b304826.png)

你应该注意到这是 0.1 版本的代码，并且 [Chris 声明](http://chrispederick.com/blog/web-developer-0-1-for-google-chrome/)这是收集反馈的早期版本。有几个特性不见了，不起作用，或者不如 Firefox 的实现完美。Chrome 的扩展本质上是小网页；它的 WDT 版本永远不可能与 Mozilla 浏览器提供的深度浏览器控制相媲美。例如，像禁用 JavaScript 这样的功能在当前的 Chrome API 中是不可能的。

也就是说，Chrome 上的 WDT 是对浏览器的一个很好的补充。如果你最近刚从 Firefox 换过来，错过了一些更好的工具，帮自己一个忙，今天就安装 [Web Developer](https://chrome.google.com/extensions/detail/bfbameneiokkgbdmiekhjnmfkcnldhhm) 工具栏。

你在 Chrome 或 Firefox 上使用 Web Developer 扩展吗？你喜欢什么？你会如何改进它？
