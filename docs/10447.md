# HTML5 脏吗？

> 原文：<https://www.sitepoint.com/is-html5-dirty/>

上周，我写了关于谷歌的 mod_pagespeed。该模块的一些功能在评论中引起了一些负面反应；值得注意的是，mod_pagespeed 可以被设置为删除标记中属性周围的引号，并删除不必要的属性(比如 input 元素上的`type="text"`)。

评论者指出，他们对服务器输出无效标记感到不安，或者“破坏了他们的良好意图”

啊，但不是无效！或者至少，不需要这样。

HTML5 规范的一个更有趣(至少有点争议)的方面是它放宽了对标记精确语法的一些限制。想法是尽可能降低 HTML 文档的复杂性，同时保持向后兼容性。事实证明，浏览器总是支持未加引号的属性，并且在没有类型属性的情况下(或者在存在它们不理解的类型属性的情况下，它们总是默认文本的输入类型；这就是为什么像`"number"`或`"email"`这样的新输入类型是向后兼容的)。因此，完全向后兼容的最简单的输入属性是:

```
<input>
```

这适用于所有浏览器，并能正确显示文本输入框。这就是为什么 HTML5 规范的作者继续前进，并使这成为规范创建该元素所需的最低要求。仅当属性值中有空格时，才需要在属性值两边加引号；元素和属性名称不区分大小写，许多属性都有一个默认值，如果属性不存在，将采用该默认值(input 元素的 type 属性就是这种情况)。这就是 spartan HTML5 doctype 背后的全部思想:它是在旧浏览器中触发标准模式所需的最少字符数。

即使你理解了这一点，像`<INPUT type=text>`这样的标记看起来仍然是错误的，不是吗？但是，正如 Jeremy Keith [在他的 Fronteers 2010 主题演讲](https://vimeo.com/15755349)中非常好地论证的那样，这是一个编码风格的问题，规范应该是风格不可知的。

所以，回到 mod_pagespeed。如果它可以通过从代码中剥离一些不必要的内容来提高你的性能，并且每一个浏览器都可以毫无问题地解析它，那么我说我们就这么做吧。好消息是，HTML5 规范已经考虑到这一点，所以我们这些关心“以正确的方式”做事的人可以安心一些。
