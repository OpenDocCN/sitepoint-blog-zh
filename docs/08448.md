# 不要再提供这么多免费信息了！

> 原文:[https://www . site point . com/stop-giving-away-so-much-free-information/](https://www.sitepoint.com/stop-giving-away-so-much-free-information/)

我不知道你是怎么想的，但是一旦我开始讨论一个项目并发现潜在客户的梦想、目标和目的，我就开始发展某种情感投资……我对如何帮助他或她实现它感到兴奋。我认为这是我最大的优势之一。

但这也是我最大的弱点。在错误的时间，或与错误的人，带着所有的兴奋和激情，我最终放弃了农场。我提供了太多的信息，太自由，不期望任何回报。这对于推特(Twitter)或我的博客(T2)来说没问题。但那是营销。一旦我与自称是潜在客户的人面对面，营销就结束了。是时候达成交易了。

在本期[结束滥用客户行为](https://www.sitepoint.com/stop-client-abuse-of-web-designers-now/ "Stop Client Abuse of Web Designers Now!")中，我将解决我们最常出错的一个地方:为了达成交易而泄露太多免费信息。

# 行动步骤 2:推销“为什么”,就“什么”达成一致，忽略“如何”

我相信大多数销售我们的网页设计/开发或 SEO 服务的人本质上都是技术型的。当然，你可能是一个很棒的视觉设计师，但是你也知道你的 HTML 和 CSS 以便在多种浏览器中完成设计，不是吗？由于这种倾向，我们大多数人都喜欢谈论技术问题。所以典型的对话可能是这样的:

**前景:**我有一个网站，想在全国范围内销售我的产品。

你:我可以做到。首先，我会通过做一些关键词研究，用合适的关键词优化你现有的网站。然后我会通过各种方法建立高质量的反向链接来增加流量，比如文章提交和新闻稿。

潜在客户:你能帮我把这些都写进计划书里吗？

你:嗯，当然…

展示我们的专业知识是达成交易的必要条件。但是这种方法没有发现为什么他想在全国销售他的产品。你还没有解决他的痛苦、需求或欲望。相反，你给他提供了一份免费的白皮书，并称之为提案。还记得我如何反复强调[相互承诺的问题吗？为什么一个潜在客户应该从所有的知识和专业技能中受益，而不用给你一个承诺作为回报？](https://www.sitepoint.com/stop-the-abuse-7-steps-to-a-well-trained-client/ "Stop the Abuse! 7 Steps to a Well-Trained Client")

如果你遵循了我上一篇文章中的建议，那么在某个时候，你就决定了电话那头或桌子对面的那个人是否是一个可行的潜在客户。整个[销售过程](https://www.sitepoint.com/whats-a-sales-cycle-and-why-do-i-need-one/ "What's a Sales Cycle and Why Do I Need One?")可以归结为一系列的“是”他或她必须对你的解决方案说“是”，对你的流程说“是”，对你的价格说“是”，更重要的是，对你说“是”——你的能力、可爱程度和可信赖度。“不”是一个潜在的阻碍，除非它可以被协商和解决。(你应该事先知道什么是可以商量的，什么是不可以商量的。)一个成功的结论是，当对方从怀疑者到潜在客户再到客户，一路“是”。

但是我们的技术头脑倾向于使这个过程变得更复杂。要让潜在客户承诺与你做生意，你只需要确立并同意两件事:

1.  他想要完成的，他的“大图”目标
2.  你是帮助他完成任务的人

在这两件事发生之前，不要讨论技术细节(如何做)或提出写一份详细的建议书。在没有承诺雇佣你的情况下这样做，我们最终会给出太多的免费信息。所以停下来，已经！

当你不再讨论技术细节，而是开始询问目标、目的和购买动机时，奇怪的事情发生了——你会很快意识到你的潜在客户没有任何技术细节。你看，严肃的潜在客户可能并不总是知道他们想要什么，但他们可以清楚地表达他们为什么想要它。因此，他们也有购买意向。不认真的、别有用心的、或者玩你的潜在客户不会有这些。当他们不想回答你的问题，或者给出不明确或含糊的回答时，你就能看出来。

记住，讨论技术细节的时间是在你赢得合约之后，而不是之前。谈论他想要什么以及为什么想要，但是把深入的技术讨论留到之后*他同意你是这份工作的合适人选。*
