# 什么是好的 404 页面？

> 原文:[https://www.sitepoint.com/what-makes-a-good-404-page/](https://www.sitepoint.com/what-makes-a-good-404-page/)

[![3d_404_animation.png](../Images/32762a60aaaa458e23e7d26160b59b2b.png)T2】](http://idzr.org/404)

手边有 Safari 浏览器吗？甚至 iPhone 上的移动 Safari 也能工作。现在检查一下这个 404(“找不到页面”)错误页面:[http://idzr.org/404](http://idzr.org/404)T3】。

该页面上的流畅动画由 WebKit 提供的非标准 [3D CSS 转换驱动，这就是为什么它目前只在 Safari 中可见。Chrome 很接近，但缺乏合适的时机。页面中使用的唯一 JavaScript 是插入随机字符并对它们应用 CSS 样式；动画是纯 CSS。](http://webkit.org/blog/386/3d-transforms/)

虽然令人眼花缭乱，但它让我想到了 404 页。什么是好的“找不到页面”页面？游戏的名字是尽可能快速有效地让用户回到正轨，还是一个好的 404 页面应该奖励任性的用户一个像这样的小惊喜——只有从地图上掉下来的访问者才能看到的视觉享受？

当然， [SitePoint 的 404 页](https://www.sitepoint.com/404)看起来有点累了。你同意吗？如果我们要重新设计它们，你想在它们身上找到什么？最近有没有看到特别有效的 404 页面？留下评论指出来。
