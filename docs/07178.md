# PHP 和 I，第 3 部分

> 原文：<https://www.sitepoint.com/php-and-the-i-3/>

到目前为止，我们已经讨论了 PHP 如何成为面向 RPG 的商业系统的完美附属品，提供每个人都需要的高质量网页。在我在网上其他地方看到的文章中，重点总是放在如何将 PHP 连接到 RPG 上，反之亦然。从来没有人谈论过如何创建高质量的页面，它们应该是什么样子，应该如何构建等等。

原因很简单:我们不需要讨厌的指示！大家都知道怎么设计好屏幕(opps，pages)。这很简单，对吧？嗯，也许不是。我们当然不需要费很大力气就能找到不正确的例子——页面，如果它们的设计考虑到了效率和直观性，就很难判断它们在迎合什么样的心理。

当然，商业页面不同于我们为博客或网站开发的页面。并不是说页面的外观不重要，但输入字段的组织方式以及浏览页面和输入数据的效率对于业务页面非常重要。对于 I 用户来说尤其如此；他们习惯使用的绿色屏幕在硬核数据输入的速度和便捷性方面是无与伦比的。尽管它们可能很漂亮，但有时将用户切换到 GUI 屏幕会导致褒贬不一的评价。

因此，在这篇文章中，我将谈论一些值得考虑的事情，如果你想创建高质量、结构良好的页面(我们都这样做，对吗？).

## 开关数量

在我看来，设计商务页面最重要的是“切换”次数，即从键盘切换到鼠标、触控板或滚动条，然后再切换回来的次数。每次你这样做的时候，你必须移动你的手，改变你的焦点，没有什么比改变你的精神焦点更能导致错误和疲劳了。因此，我们希望尽量减少交换机的数量。

你可能会说“是啊，别开玩笑了。”这似乎是显而易见的事情。但是，如果你看看外面的许多页面，限制开关似乎不是第一优先事项。我想到的一个例子是输入日期。在绿屏世界里，你只要输入日期 20130130 就可以继续了。如果你输入 20130143，就会弹出一条错误信息。但在网络世界，当你有一个日期字段时，你会弹出一个日历，你可以用鼠标或触控板选择你想要的日期。总而言之，这需要三次点击和两次切换，如果你想要一个不在当前月份的日期，就需要更多。它完全改变了你思维的焦点，从看角色到处理图形表示。它很酷，尤其是有了新的 CSS3 功能，但是使用商业屏幕的目标是效率，而不是酷。

该示例还说明了业务系统环境与更普通的 web 页面环境有什么不同。计划度假的人确实可以从查看日历格式的日期中受益。对他们来说，额外的按键会减少错误，尤其是无法编辑的错误，比如在周三而不是你想要的周二离开。但在商业环境中，我可能会从纸质工作订单上看到“要求的交付日期”，而不需要看到任何讨厌的日历。我只需要一个可以输入 20130315 的字段。

这里的敌人正在转移注意力。尽管我们喜欢假装，但人类真的没有那么聪明。如果我们能一次集中精力做一件事，比如阅读文本，在屏幕上光标尽可能自动移动的地方输入数据，我们就没问题。这是一对一的映射，你不知道吗。但是当输入格式开始变化并变得混杂时，它变得更加困难，这反过来又使我们感到疲劳，更容易出错。

## 镜像任务，而不是数据库

很久很久以前，当恐龙统治地球时，在人们住在房子里之前，开发人员会设计一个商业屏幕来模仿它背后的数据库。我们会有一个负责产品主数据维护的功能，因为有一个产品主文件；另一个负责产品结构维护，因为这是一个文件；第三个负责设置工艺路线，告诉你如何制造产品。建立新产品的过程包括三个独立的功能。

随着时间的推移，人们开始看到真正重要的不是建立一个产品主记录，而是建立一个产品。因此，您可能希望开发一个功能来创建一个产品。是的，它更新了产品主数据，但它也允许您在许多其他文件中输入各种奇怪的字段，这些字段需要与产品主数据相关联，以便完整地定义新产品。换句话说，关键的事情变成了业务任务，而不是受影响的文件。

这在今天非常重要，因为员工在减少，越来越多的人开始负责跨学科的任务。当你的用户有一项业务任务要做时，你最不希望的就是他们必须通过两个或更多的功能来完成这项任务。作为技术人员，我们非常注重数据库，但你真的需要深入了解用户在做什么，以及你如何在单个屏幕或功能内完成这些。您必须了解他们输入什么数据，他们需要什么数据，他们输入或使用给定数据的频率，以及各种元素之间的关系。

你是怎么做到的？好吧，你可以问，但是正如犯罪现场的目击者对发生的事情通常是出了名的不可靠一样，许多用户也不太擅长记住他们真正需要的东西。你必须观察他们的行动，与各种各样的人交谈，对任务、其组成部分以及它与公司整体的关系有一个坚实的理解。

## 小心折叠

让数据出现在“折叠线以下”真的会影响它被看到的可能性吗？对于商业系统来说，问题变成了让数据出现在文件夹下面是否会影响屏幕的效率。

对于常规网站来说，直到几年前，在文件夹下有信息是被看到的一个障碍。如今，用户变得更加成熟，并且明白向下滚动是网络体验的一个重要部分。当然，商业页面略有不同。用户不能选择滚动或不滚动，他们必须向下滚动才能完成他们的任务。它变成了你不得不做的又一件事，转移了你的注意力，因此，这是一件讨厌的事…尤其是如果你不得不做不止一次的话。

如果你把额外的数据放在一个单独的页面上，那么用户仍然需要点击一个链接或按钮来访问它。那么我们能做什么呢？可惜不多。你能做的最好的事情就是把最重要的和最常用的数据放在页面的顶部，以减少用户向下滚动的可能性。

最重要的是，你应该注意你放在页面顶部的标题。我的银行喜欢让他们的标志和其他毫无价值的东西占据半页，这意味着每次我查看余额或进行转账时，我都需要向下滚动。你觉得这不会让我生气吗？

## 摘要

如果你已经建立了一个或两个网站，并且做得很好，你可能会说除了商业页面之外，这里的一切对常规网站都很重要。你可能是对的。也许真正的区别是，一个令人愉快的外观可以鼓励用户在网站上掩饰面向数据的问题，但在商业页面上，一切都摆在你面前，你必须把它做好。
