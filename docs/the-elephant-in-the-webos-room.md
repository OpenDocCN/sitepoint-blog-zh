# WebOS 房间里的大象

> 原文：<https://www.sitepoint.com/the-elephant-in-the-webos-room/>

WebOS 的未来似乎不确定，所以当倾听全球开发者关系副总裁的机会来敲门时，你应该打开门。昨晚我走进悉尼的机构，参加 WebOS Connect 活动时也是如此。

### WebOS 连接

理查德·蔡秋凤上台不到十分钟。他很友好、谦逊、风趣，花了一点时间介绍新的 WebOS Pivot 杂志，并强调他们是一家以开发者为中心的公司，然后答应给我们几个演讲者和几杯“成人饮料”。

最能说明问题的是，理查德在结束时坦率地表示，不会有正式的问答，原因同样简单明了，“我无法回答你的问题”。所以平心而论，WebOS 房间里的大象是被承认的。向前看。

### WebOS 内部

Rod Whitby 是 [WebOS Internals](http://www.webos-internals.org/wiki/Main_Page) 的创始人，他很聪明地在幻灯片中加入了 [WebOS 系统架构](https://developer.palm.com/content/resources/develop/overview_of_webos/overview_of_webos_palm_webos_architecture.html)的图表。他的第一点:WebOS 架构是完全开放的，不需要越狱，你甚至可以用 Konami 代码进入开发者模式。

故事是这样的，随着[预软件](http://preware.org/)和[自制软件](http://www.precentral.net/homebrew-apps)，一个应用程序、补丁、主题、服务和内核扩展的世界正在等待着你。相当古怪的东西，但是有令人惊讶的好文档，提供了改进性能和检修 WebOS 设备 UI 的能力。

还有一件事，有一个应用程序。WebOS 内部在 WebOS 应用程序目录中有一个[预软件自制文档](https://developer.palm.com/appredirect/?packageid=org.preware.docs)应用程序，解释了以上所有内容。他们还没有给设备砌砖，他们的一些修补工作已经被纳入正式的 WebOS 版本。

总之，围绕 WebOS 有一个规模合理、成熟的开源风格的开发者社区。到目前为止，这个社区被赋予了其他平台所没有的权力和影响力。

### WebOS 一等公民

对于 BuildMobile 读者来说，约翰奥尔索普应该不需要太多介绍。约翰是一名作家、开发者、会议组织者、电影明星和全能的 T2 艺术家。他今晚演讲的焦点是触摸板上基于 WebKit 的名为“Web”的应用程序。

当他对着幻灯片台说话并推进幻灯片时，观众将他们的惠普触摸板指向[演示应用](http://bit.ly/mQYa4L)，幻灯片在现场演示的同时在手边的设备上推进。好吧，正如在所有现场演示中所预料的那样，出现了故障，但理论是合理的。

这里的要点是，随着我们最近目睹的平台爆炸，以平台碎片和无数编程语言的形式出现了开发人员的挑战。然而，这些事情中不变的是在大多数平台上一流的 web 技术支持。

引用 John 的话，“WebOS 是第一个让 web 技术成为关键开发工具的平台”。正如 Rod 在他之前提到的，高性能 Javascript 服务器 [Node.js](http://nodejs.org/) 装载在 TouchPad 上。添加到本地存储，CSS3 选择器和 CSS3 动画，等等。

### 向前

虽然这个月和下个月销售 WebOS 应用程序可能会赚一点钱，但我不认为从长远来看会有可持续的收入。在这个阶段，这个平台的未来并不确定，问题比答案还多。

关于 HTML5 的爆炸，如果 WebOS 设备可以搭载一个有能力的 WebKit 浏览器，那么可以想象任何其他平台也可以。值得注意的是，在这个阶段，久负盛名的 PhoneGap 可以让你的网络应用程序 WebOS 本地化，但我敢打赌这些家伙会支持任何重要的平台。

楼上关于“成人饮料”的讨论无疑是乐观的。但人们必须记住，我们被喂养，然后在欢乐的气氛中递给美酒和异国情调的雪茄。一些人得到了触摸板。身材丰满的女士没有唱歌。不要忽视理查德·蔡秋凤的这条推文。
