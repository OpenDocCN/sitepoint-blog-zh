# SXSW 第一天综述

> 原文:[https://www.sitepoint.com/sxsw-first-day-roundup/](https://www.sitepoint.com/sxsw-first-day-roundup/)

又一个三月来了，随之而来的是另一个西南偏南(SXSW) 。我发现自己又一次和其他志同道合的网络极客一起朝圣，参加真正为所有其他人设定了标准(没有双关语)的网络会议。

这是我第四次参加这个会议，为了衡量我对它的评价，我通常会提供这个轶事:两年前，我休了一年的休假，在那段时间里，我没有赚钱——我没有以自己的方式环游世界，而是抵押了房子来支付旅行费用。尽管资金有限，我还是设法找到了在澳大利亚度假的钱，从悉尼出发，经过旧金山和丹佛，飞了一段有点长的路，到达德克萨斯州的奥斯汀。是的，这真的是一个证明这种疯狂的会议！

当我写这篇文章的时候， [SXSW Interactive](http://2006.sxsw.com/interactive/) (会议还有一个电影和音乐部分)刚刚开始。第一天的会议已经结束，每个人都在为开幕派对做准备。事实上，这在很大程度上是 SXSW 的内容，比我参加过的其他会议更是如此——这是一个非常社会化的活动，而且[晚上发生的事情](http://2006.sxsw.com/interactive/evening_events/)可以说与白天的会议一样重要，如果不是更重要的话。如果曾经有一个地方可以建立关系网，把虚拟人的名字和我们经常使用和访问的博客、网站、企业和服务联系起来，那就是这个地方。

SXSW 的开始让我大吃一惊。字面上。我很幸运地加入了莫莉·霍尔茨施拉格和 T2·安迪·克拉克的队伍，还有一群朋友，他们认识这个枪支最自由的州的一些“持枪者”。就这样，我们来到了射击场，度过了最难忘的会前快乐时光！这句话来自莫莉对安迪的评价:“他进去时是个温迪，出来时是个男人！”。安迪以前从来没有射击过任何东西，但后来证明是一个神枪手，用枪展示了与他在 CSS 和设计方面的精确度相匹配的精确度。当心现在任何胆敢批评马拉其作品的人！

那么，会议本身呢？我参加的第一个会议(从 5 个可能的选项中选择)题为“传统设计和新技术”，由(Razorfish)的[托尼·格里夫斯](http://www.tonigreaves.com/)、英国广播公司的[马克·博尔顿](http://www.markboulton.co.uk/)、T4 的【Khoi Vinh】、《纽约时报》的[利兹·丹兹科](http://www.bobulate.com/) ( [AIGA](http://www.aiga.org/) )和杰森·圣玛利亚[组成。会议围绕着人们对传统设计的期望，无论是印刷还是产品设计，以及在线设计。马克·博尔顿表达了他的沮丧，人们如何能从大学毕业并获得设计学位，却仍然没有对设计如此重要的概念的基本理解，例如如何设置铅字。他还表示，他不记得上一次从一个网站上获得类似的情绪反应是什么时候了，比如说，一辆汽车或其他一些设计精美的产品(“这就是为什么我们都买苹果电脑，对吗？”对此 Khoi 回答说“不是我们所有人！”).这被证明是一个话题，Toni 非常强烈地反对——她可以想到很多网站给她这种情绪反应，并回忆起早在 1997 年第一个这样做的网站(在互联网上，绝对是一个时代以前)。谁是对的——马克还是托尼？这表明，也是演讲者们随后讨论的，人们有着非常不同的期望，他们在网上看到的和现实世界中看到的也有着不同的期望。关于易贝的普遍丑陋似乎与其巨大成功不一致的古老观点出现了，马克建议说一点额外的设计投入不会出错，而托尼对长相非常简单的 Craigslist 得到一个拉皮条的外观的前景犹豫不决(我的话，不是她的！).尽管是第一次会议，观众们并不害怕提出有争议的问题(前一天晚上的社交显然没有杀死所有的脑细胞)。](http://www.jasonsantamaria.com/)

我参加的下一个会议是“[如何成为网页设计超级英雄”](http://2006.sxsw.com/interactive/programming/panels/?action=show&id=IAP060001)与[安迪·巴德](http://www.andybudd.com/)和安迪·克拉克。这看起来会很有趣，充满活力的二人组没有让人失望。舞台设置非常摇滚(“平克·弗洛伊德在演吗？”我在进来的路上听到有人评论)，演讲以一个非常半开玩笑的电影片头式的开场序列开始。两个安迪在超级英雄和网页设计师之间建立了联系，尽管这是一个不太可能的组合，但像你最喜欢的超级英雄一样努力追求伟大可以让你成为一个网页设计超级英雄。这听起来像一个古怪的想法，但这是一个有趣的方式来看待一个相当普通的主题，并很好地让每个人都放松到会议的其余部分。午饭后，事情会变得更加复杂。

我参加的第一个下午会议(后传统德克萨斯烧烤)是[杰里米·基思](http://adactio.com/)和[亚伦·古斯塔夫森](http://www.easy-designs.net/)的[如何在 Dom 脚本中虚张声势](http://2006.sxsw.com/interactive/programming/panels/?action=show&id=IAP060040)。我之前在伦敦参加了一个为期 1 天的 Clear:left AJAX 课程，其中包括许多不引人注目的 JavaScript 和 DOM 脚本，作为序言的一部分，在这个 SXSW 会议中，Jeremy 将那天的重要内容浓缩到了一个 1 小时的会议中，这被证明是非常有用的，作为什么是 DOM 脚本以及它应该和不应该用于什么的初级读本。Aaron 展示了 DOM 脚本的一些实际应用(包括一种样式化选择列表的方法)，他们展示了开始 DOM 脚本不一定是火箭科学。然而，关于在一些网页上使用 DOM 脚本方法的性能问题的观众问题表明，使用 DOM 以某些方式更新较大的页面可能并不总是最好的方法(我可以证明这一点，在一个特别繁忙的表格操作脚本上加载页面后，IE 在我身上冻结了近 10 秒钟，我最终放弃了这个脚本)。正如 Jeremy 所说，使用专有的`innerHTML`而不是 DOM 脚本就像使用大锤而不是精心制作的手术刀。在我看来，用手术刀进行一万次精确切割并不总是最好的方法。

那天我参加的最后一个演讲的题目是“[如何创造热情的用户](http://2006.sxsw.com/interactive/programming/panels/?action=show&id=IAP060014)”。我不知道主持人会是谁——我只被标题吸引住了。Kathy Sierra ( [的传记在这里](http://2006.sxsw.com/interactive/programming/panels/?action=bio&id=103421))用一场我能回忆起过去几年在 SXSW 上看到的最吸引人的演讲让观众惊叹不已，然而当我结束演讲时，我很难准确描述她做了什么让我们惊叹，主要是因为我不可能做到公正！实质上，她是在解释人类的大脑本质上是我们生活在洞穴中的时代的遗产，虽然我们可能想学习一些东西，但大脑基本上仍然停留在那个时代。举个例子:你正在为期末考试临时抱佛脚，你正在阅读一个并不令人兴奋的话题，无论你重读多少遍，它都不会被理解，因为大脑基本上在说“这东西不重要或没有生命危险”，但是当一个辣妹走过时(她的话！)，大脑开始注意。考虑到这一点，她尝试了各种技术，你可以用这些技术来欺骗大脑对事物产生热情，从而让你的用户对你所做的事情产生热情(或者更确切地说，是你让用户能够做的事情)。这些幻灯片非常精彩，非常有趣，与以往不同的是,“互动”环节真正做到了这一点，因为她三次让所有观众分成更小的讨论组。我能说的是，这个播客非常值得一试(当它可用的时候)，即使你没有得到所有使它如此有趣的视觉元素。

第一天会议的良好开端。现在剩下要做的就是继续晚上的社交活动。各位，明天会有更多报道。
