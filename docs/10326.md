# 2010 年五大网络趋势和技术

> 原文:[https://www.sitepoint.com/top-5-web-technologies-2010/](https://www.sitepoint.com/top-5-web-technologies-2010/)

现在我们已经到了 2010 年底，这是一个回顾过去一年有趣发展的好机会。以下是我认为的 2010 年改变了我们开发网站和应用方式的五大网络趋势和技术。这是我自己的观点，但是请随意插入你自己的想法和评论！

## 1.HTML5

虽然 HTML5 已经存在了一段时间，但 2010 年是采用率快速增长和媒体大肆宣传的一年。HTML5 成为最重要的 web 技术之一有很多原因:

*   这些规范即将成熟。他们仍然是草案，但所有的浏览器供应商都支持 HTML5，你今天就可以使用它。
*   [微软在 IE9](https://www.sitepoint.com/microsoft-reveal-ie9/) 中宣布支持 HTML5(并误以为是[干掉 Silverlight](https://www.sitepoint.com/html5-kills-silverlight) )。不管你对微软或他们的浏览器有什么看法，如果大多数网络用户不能使用这项技术，HTML5 就不会成功。
*   JavaScript shims 的开发允许开发者使用 HTML5 并保留对 IE6、7 和 8 的支持。
*   *“html 5”*已经成为主流媒体的科技流行语。你实际使用的是什么技术并不重要——称之为 HTML5，你会得到更多的媒体关注。

HTML5 前途光明。

## 2\. Web Fonts

字体嵌入技术已经存在了十多年，但受到版权问题和浏览器兼容性的阻碍。一年前，你很难找到一个使用 Arial、Verdana、Helvetica、Times New Roman 或 Georgia 之外的主要字体的网站。

这种情况在 2010 年有所改善，推出了字体代工厂，如谷歌的字体目录和 T2 的 W3C 支持的网络开放字体格式(WOFF)T3。今天随便挑几个网站，你就会看到网络字体在起作用。

## 3.REST 和 RESTful APIs

Roy Fielding 在 2000 年定义了表征状态转移的概念。今年，REST 成为分布式应用程序之间共享数据的最流行的方法之一。

简而言之，REST 用于实现 web 服务。客户端使用 POST(创建)、GET(读取)、PUT(更新)或 DELETE(删除)方法发送 HTTP 请求。服务器分析数据，执行适当的操作，并返回消息——通常以 HTML、XML、JSON 或图形格式。

虽然没有官方标准，但是 REST 的简单性是它最大的优点。与基于 SOAP 的 web 服务不同，不需要复杂的库或 XML 翻译。它是 web 应用程序 API 的理想选择，Google、Twitter、脸书、Digg、Yahoo 和 Flickr 等公司已经加入了 REST 的行列。

也许是时候为您的应用程序考虑一个 REST API 了？

## 4.移动应用

移动应用是 2010 年的成功故事。智能手机已经成为 PC 的一个可行的计算替代品，许多人预测它们将在几年内成为主要的网络接入设备。

苹果的 iPhone 和 App Store 提升了移动应用的形象。用 Objective-C 编写的 iOS 应用仍然很受欢迎，但许多公司正在转向基于网络的移动应用。这些通常使用库来实现类似 iPhone 的界面，比如 [jQTouch](http://www.jqtouch.com/) 。

此外，平板电脑已经开始影响网站的设计方式。2011 年可能是平板电脑优化网络应用的重要一年。

## 5.一个公平的浏览器竞争环境

今年有两大亮点——IE 的市场份额跌破 50% 和 [Chrome 达到两位数](https://www.sitepoint.com/google-chrome-market-share-10-percent)。但这才是真正重要的:*没关系*。

这可能花了 20 年的时间，但在 2010 年，我终于相信我们可以不用担心我们的访问者使用的浏览器了。网络总是被认为是设备不可知的，但是当浏览器供应商积极竞争控制互联网时，这个梦想从未实现。

今天，任何主流浏览器都足以处理我们扔给它们的功能。当然有差异和怪癖，但大多数是次要的或只影响更新的尖端技术。你很少会遇到无法解决的问题。

当我们开始下一个十年时，让我们为网络而不是浏览器创建网站和应用程序。

祝你 2011 年快乐！
