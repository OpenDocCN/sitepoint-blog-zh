# 用 Sitespring 管理 Web 项目

> 原文:[https://www.sitepoint.com/web-projects-sitespring/](https://www.sitepoint.com/web-projects-sitespring/)

据说一个村子才能养活一个家庭，我想说一个村子也能养活一个网站。

想想看:对于一个网络项目，你有 8 到 15 个人紧密地参与到与设计、编程、内容和管理相关的决策中。在现实世界中，你知道一个网络项目可能有 6 到 12 周的期限。再久一点，您正在处理的数据可能已经过时了。当然，你可以购买一些应用程序来帮助管理你的项目，但是没有一个现成的软件能够解决 Web 项目的细节问题。网络是关于快速、高效和即时的传递。那么，你如何在保持沟通渠道畅通的同时管理客户的期望呢？

这就是 Macromedia 推出杀手级应用 Sitespring 的原因。我们已经知道 Macromedia 生产最好的 Web 软件解决方案(只要问一问使用 Flash 或 Dreamweaver 的人就知道了),我们已经开始期待他们提供优秀的软件。然而，与其他 Macromedia 软件不同，Sitespring 不是一个设计包。相反，Sitespring 帮助控制电子商务经理和他们的客户之间的交互，任何给定项目的任务和开发周期，以及项目所需的任何文件的版本控制。最终结果是一个完整的生命周期管理程序，在正确的时间为正确的用户提供正确的内容。最终结果是:缩短了执行时间，提高了客户满意度，降低了任何给定项目的总体成本，所有这一切都通过一个 Web 界面实现。

##### 准备好了，快！

除非安装快速且易于使用，否则任何 Web 软件管理工具都不会成功。Macromedia 满足了这些要求。首先，Sitespring 的安装时间不到 5 分钟(目前它可以在 Windows NT 4 或 Windows 2000 服务器上运行)。软件成功安装后，浏览器窗口会将你直接链接到 Sitespring 管理工具。一个很大的好处是管理工具都是通过一个 Web 界面管理的！所以 Sitespring 可以在任何地方被管理，只要你能看到它运行的网络服务器，无论是公司的内部网还是外部网。

在管理窗口中，您需要做的第一件事是为 Sitespring 的用户设置访问权限。Sitespring 有两种不同类型的用户:客户端和活动用户。区别在于，如果你有一个用户需要访问 Sitespring 的个性化功能来管理项目、任务和版本控制，那么他们将被归类为活跃用户。如果他们只是被要求提供内容和审查网络项目，那么他们就被认为是客户。

##### 客户需求分析

电子商务经理或客户经理的职责是从客户那里收集一份完整的商业计划，或有时被称为“需求分析”的东西。一个有计划的“需求分析”将允许 Web 团队获得对项目、范围、目标受众以及项目出资人的全面理解。Sitespring 为您提供了收集和存储这些数据的工具。使用 Web 界面，Web 组的客户经理可以会见客户，并准确记下客户的需求。

任何 Web 开发项目的重点都应该是业务需求，而不是技术。正如我们在过去五年中看到的，随着网络的进步，技术会很快过时。然而，商业需求会持续更长时间。

Sitespring 使客户的业务需求对开发团队中的每个人都可用。这是至关重要的——在任何时候，Web 开发团队成员都可以查看这些数据，以了解他们的角色如何适应更大的愿景。它可以帮助团队成员理解看似神秘的任务，更重要的是，增加交流的流量。

##### 任务

一旦业务需求分析完成，下一步就交给项目经理了。他们进行“需求分析”,并在客户经理的帮助下，将项目的各个部分分解成任务，然后分配给 Sitespring 的活跃用户。

每个用户通过电子邮件接收他们的任务，并可以通过网站访问他们任务的详细信息。通过提供两种内容交付方法，项目经理能够保持沟通渠道的畅通。

每封邮件都提供了一个指向 Sitespring 网站的直接链接。点击此链接的用户会立即被引导回该站点。登录后，他们会看到一个个性化的网页，以满足他们的需求。他们可以从四个主要选项卡中进行选择:项目、任务、讨论和报告。

正如您所猜测的，“项目”选项卡列出了用户参与的项目。用户可以在任何时候点击一个项目来查看它的文档，包括任何客户联系信息。

第二个选项卡标识特定项目中涉及的所有任务。打开一个任务可以让用户确定该项目需要做什么。

沟通对于当今日益分散的开发团队的成功至关重要。“讨论”选项卡允许围绕特定项目进行公开讨论。这非常有帮助。电子邮件可以用来讨论项目，但是，通过电子邮件跟踪特定项目的讨论轨迹可能非常困难。Sitespring 让它变得简单。

最后一个选项卡是“我的报告”。该选项卡是高级报告的清单。

用户可以一目了然地看到他们正在进行的项目、需要完成的任务以及当前的热门话题。

##### 每个人都是项目经理

网络的工作方式确实不同:时间尺度大大缩短了。因此，开发人员、设计人员或内容所有者(除了项目经理之外)能够为项目创建任务非常重要。考虑这个场景:

一个由三名开发人员(一名服务器端脚本编写人员、一名 DBA 和一名 JavaScript 专家)组成的 Web 开发团队正在纽约等待圣何塞的一名设计师来完成网页模板的合成。团队的项目经理知道开发人员需要这个模板来继续他们的工作。项目经理与设计人员一起工作，给他们分配创建模板的任务。

设计师工作到晚上十点才完成设计(项目经理回家喂狗 5 小时后)。知道这个项目的时间线很短，她进入 Sitespring，检查她的任务是否完成。反过来，她为纽约的服务器端开发人员创建了一个新任务。

第二天早上 7 点，服务器端开发人员从家里登录到他的计算机上。他在附有模板的电子邮件中看到了新任务。他立即将模板转发给纽约的其他两位开发人员，并在 Sitespring 中创建了三个新任务:

1.  第一个是为自己创建服务器端脚本，以将模板上的动态内容链接到数据库中保存的内容；
2.  DBA 的第二项任务是为服务器端脚本创建数据库模式；
3.  最后一项任务要求 JavaScript 开发人员向设计者创建的模板页面添加一个弹出窗口命令。

到美国东部时间上午 8:00，JavaScript 开发人员已经在 Dreamweaver 中创建了弹出窗口，并将页面返回给项目经理进行审查；到上午 10:00，DBA 已经为页面创建了简单的数据库(原来 DBA 已经完成了，但是直到 9:30 才查看她的电子邮件)；到 10:30，服务器端开发人员已经使用 UltraDev 为页面创建了服务器端脚本。

每个成员通过 Sitespring 向项目经理发送任务已经完成的通知。项目经理在太平洋时间上午 9:00(纽约时间上午 11:00)到达，看到不仅所有的工作都已经完成，而且还可以通过 Sitespring 跟踪流程。每个团队成员都必须是项目经理。

当然，这也可以通过电子邮件来完成。但是 Sitespring 允许所有的动作都被容易地跟踪。这一点尤其重要，例如，当看到谁签署了项目的部分时。此外，还可以为您的客户生成一份报告，确定所有已完成的工作、由谁完成以及花费了多长时间。

##### 版本控制

有多少次你创建了一个网页，并且不止一次地编辑了同一个网页？

现在，问问你自己，有多少次你编辑了那个网页，只是希望在某个时候可以回到旧版本？如果这没有发生在你身上，那么你是完美的！另一方面，如果你像我和数以百万计的其他 Web 开发人员一样，这种情况可能已经发生过无数次了。Sitespring 来救援！

Sitespring 内置的一项令人难以置信的新技术是版本控制系统。与 WebDev 或微软的 Visual SourceSafe 不同，Sitespring 不使用数据库方法进行版本控制，也不局限于基于文本的文件。Sitespring 的版本引擎位于文件服务器之上，该服务器托管项目开发过程中使用的文件。您可以对任何文件应用版本控制，包括 Flash 电影、图形和视频，以及基于文本的文件，如 HTML、CSS 和 XML。

版本控制的应用如下:

1.  一个文件被创建或保存到 Sitespring 文件服务器(实际上，它的外观和行为与公司网络上的任何文件服务器一样)。让我们假设这个文件是一个名为 starting.jpg 的图形文件。
2.  设计人员打开 starting.jpg 并进行一些更改。偶尔他会保存文件。
3.  每次他保存一个文件时，都会在文件中添加一个扩展名:例如，文件的前四个保存位置是:starting-1.jpg、starting-2.jpg、starting-3.jpg 和 starting-4.jpg。

所以每次保存文件时，Sitespring 都会创建文件的新版本。对于像 AVI 电影这样的大文件来说，这会占用很多空间，所以 Sitespring 提供了一个“清理版本”工具来管理庞大文件的版本和存储。

版本控制是一个好处。可以轻松有效地跟踪每一项变更。但是，真正奇妙的是，当设计者将 starting.jpg 文件转发给另一个用户时，Sitespring 知道将文件的所有权更改给这个新用户。当文件的所有权改变时，Sitespring 创建一个主文件的里程碑版本，这些里程碑表示一个任务的完成阶段。

##### 什么是“完成”？

对于任何 Web 项目来说，一个棘手的问题是定义项目将在什么时候完成——什么时候“完成”我不止一次听到人们说“告诉我，做了什么？网站不是一直在被添加和修改吗？”这是真的，但是没有一个明确的项目结束的定义，一个网站可能永远不会完成。

Sitespring 将“完成”定义为 Web 团队的项目经理或客户经理向客户展示完整项目的时刻。同样，使用 Sitespring，客户可以检查项目并签字。Sitespring 捕获数字签名、谁签名、在什么时间签名以及为哪个项目签名。只有在这一点上，站点才能移动到临时服务器和主生产服务器上。至此，您的 web 项目已经“完成”

通过使用 Sitespring，Web 团队可以捕捉客户的确切需求，定义和分配项目的相关任务，并确定项目完成的准确时间。这减少了花费在 Web 项目上的时间、成本和精力，同时提供了互联网项目管理所必需的灵活性和实用性。客户会很高兴——当他们回来继续工作，或者推荐其他客户到你的公司时，你也会很高兴。
