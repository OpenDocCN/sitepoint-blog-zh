### 文本属性

有许多 CSS 属性可以控制文本的视觉外观——但实际上只有七个属性以单词“text”开头。

这些是:

*   `text-align`
*   `text-decoration`
*   `text-indent`
*   `text-overflow`
*   `text-rendering`
*   `text-shadow`
*   `text-transform`

我们在*[第十集:`justify`](https://www.sitepoint.com/atoz-css-quick-tip-justify-text)*中比较深入地看了一下`text-align`；它用于控制屏幕上文本的对齐，通常设置为`left`、`right`或`center`。

`text-decoration`通常设置为`none`以去除超链接下方添加的`underline`。也可以设置为`underline`、`overline`或`line-through`来产生删除线效果。

`text-indent`允许我们控制缩进。我经常使用一个大的负值来删除页面上的文本，这样它在视觉上是隐藏的，但对于屏幕阅读器这样的辅助设备来说，它仍然存在于文档中。

允许对溢出其容器的任何文本进行样式化处理——例如一个单词太长而不适合。我们可以用`clip`来隐藏这个溢出的文本，允许它可见，或者添加一个省略号来显示这个单词继续，但是没有空间了。

`text-rendering`是 SVG 规范中的一个属性，但允许浏览器优化文本的渲染，以提高渲染速度、几何精度或可读性。将`text-rendering`设置为`optimiseLegibility`将在某些浏览器中启用某些字体的字距调整和连字。支持不是很好，显然使用它会有一些副作用，所以我建议在使用之前检查一下。

`text-shadow`有点像`box-shadow`，但用于文本。它有四个属性:x 轴偏移量和 y 轴偏移量控制阴影的位置；模糊半径控制阴影的柔和程度，最后，颜色设置阴影的颜色。

`text-transform`可以用来改变文本的大小写。它可以用来设置所有的字母大写，小写或每个单词的第一个字母大写。我的一个 bug——发现 HTML 中写了很多全大写的单词——从 CSS 中控制这种视觉样式要灵活得多。

### 字体属性

我们还可以对文本进行各种各样的操作和样式化，还有一系列的 CSS 属性让我们可以控制我们的排版。

`font-family`设置要使用的字体。这可以设置为“堆栈”，浏览器将选择列表中第一个可以成功加载的字体。

`font-size`允许我们改变文本的大小。

`font-style`可以创建斜体或斜线文本，而`font-weight`属性用于创建粗体文本。`font-weight`也可以用来创建一个带有数值的权重范围。该值是 100 的倍数，范围从 100 到 900。`font-weight:normal`映射到数值 400，`font-weight:bold`映射到 700。

CSS3 中有一些非常漂亮的新`font`变体设置，可以在可能的情况下使用小型大写字母和连字——我们没有时间在这个视频中详细介绍，但如果你有兴趣了解更多关于这些功能的信息，可以搜索“CSS 中的开放类型功能”。

最后，`font`有一个简写属性，允许一次设置多个`font`属性和`line-height`。语法有一个非常具体的顺序，我永远也记不住，所以我倾向于不经常使用这个属性。但是，在一行中设置`font-family`、`font-size`和`line-height`非常有用，如下所示:

```
p {font: 12px/24px 'Avenir', 'Arial', sans-serif;}
```

### 其他属性

我们已经详细了解了`text-`和`font-`属性，但是还有一些其他的属性在设计页面上的文字时也可以派上用场。

我们看了*《第十二集:`line-height`《T4》里的`line-height`。该属性控制文本行上下的间距。我们也可以用`letter-spacing`控制单个字母之间的间距。这可以设置为较大的正值以增加间距，或者设置为较小的负值以收紧所有内容。*

我们还可以用`word-spacing`属性控制单词之间的间距，用`word-wrap`控制文本的换行。

还可以使用`white-space`属性来控制空白的行为。这是一个有用的属性，可以控制预格式化文本的视觉输出，比如代码片段——我经常这么做。
