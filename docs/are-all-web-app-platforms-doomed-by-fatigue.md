# 是不是所有的 Web App 平台都注定会疲劳？

> 原文：<https://www.sitepoint.com/are-all-web-app-platforms-doomed-by-fatigue/>

一个新的热门 web 应用平台的生命周期大概是这样的:1。该平台在大肆宣传中推出，2。广告商和投机者趁虚而入，3。平台上充斥着各种应用程序，其中大部分是愚蠢的或无用的。结果，好的应用变得模糊不清，人们失去了兴趣，因为发现变成了一个大问题。

一个热门的新网络应用平台推出后，不可避免地会出现大量应用，这导致了我们所说的应用疲劳。一旦应用程序的数量达到饱和点，人们就会开始对给定的应用程序商店或应用程序库中的应用程序数量感到不知所措。这反过来导致选择性增加，并有可能让人们将整个平台视为一堆琐碎的时间浪费。

这就是脸书的遭遇。当这个平台推出时，围绕着广告商和应用程序开发者的潜力产生了巨大的反响。随后，该平台上创建了大量的应用程序，用户面临着越来越多需要他们关注的应用程序。最终结果:app 疲劳。根据 Adomonics 的数据，在平台推出不到一年的时间里，我们开始看到应用程序疲劳，当时总共只有 15，000 多个应用程序——现在脸书平台上有超过 53，000 个应用程序。

脸书已经做了一些事情来试图解决应用疲劳和发现问题——通过设计调整和关于应用如何传播的规则，但迄今为止没有一件事做得很好。最终，那些曾经为脸书平台的成功投入巨资的人们实际上宣布它已经死亡。

同样的事情看起来可能会发生在大肆吹嘘的 iPhone 平台上。

11 月份，我们报道了 iPhone 平台的应用程序数量已经突破了 1 万个大关，但是我们想知道有多少是真正值得你花时间去做的。

“与脸书平台的比较是不可避免的，”我们在 11 月写道。“尽管明显存在许多差异(例如，iPhone 开发者可以对应用程序收费，而 iPhone 用户由于支付了手机和每月服务费用，总体上可能更愿意为应用程序付费)，但也有相似之处，因此这种比较是有道理的。随着越来越多的开发者涌入这些平台试图赚快钱，噪音水平不可避免地上升。我们在脸书看到了这一点，那里的应用数量继续稳步增长，但用户参与度已经稳定下来。”(注:根据 Adomonics 的数据，自 11 月 30 日我们发布那篇帖子以来，脸书平台上大约有 5000 个新应用。)

目前 iTunes 应用商店中排名第一的付费应用是 iFart Mobile ( [iTunes](http://itunes.apple.com/WebObjects/MZStore.woa/wa/viewSoftware?id=293760823&mt=8) )，这是一款模拟放屁声音的应用。显然，这个 0.99 美元的应用程序肯定有一些市场，但它也是一种愚蠢/无用的应用程序，在推出好东西的同时提高了噪音水平，最终可能会引领 iPhone 应用程序平台走上脸书走过的路。

Compete 最近的一项调查发现，45%的 iPhone 用户已经从 App Store 添加了 11 个或更多的应用。Compete 随后(11 月)总结道，“应用程序的部分吸引力可能在于找到它们并将其添加到 iPhone 上是多么容易。”通过控制应用程序的分发，苹果让用户更容易在手机平台上找到并安装应用程序。这是一个其他人开始效仿的模式——现在有来自微软、黑莓、谷歌/安卓、诺基亚、雅虎的应用商店。、O2 和 Palm。

然而，申请的泛滥已经成为一个问题。“随着平台、商店和应用程序的多样化，发现实际上正在成为这些天的大问题，”移动广告新闻写道。

对于这些平台来说，最大的问题不是有多少应用程序值得你花时间——在成千上万的应用程序中，无疑有大量好的应用程序——而是如何从垃圾中剔除有价值的应用程序。在如此高的噪音水平下，信号正在丢失，疲劳开始出现，导致用户失去兴趣。

我们不确定解决方案是什么，但很明显，对于 iPhone 用户来说，应用程序疲劳已经初露端倪，并且已经困扰着脸书用户。我们在 10 月份预测，随着开发人员确定快速赚钱的机会已经过去，无用的应用程序将开始在脸书平台上逐渐消失，最终有用的应用程序将取代它们，形成一个更小但更有价值的应用程序生态系统。这是一个很好的理论，但是到目前为止还没有任何事情让我们认为它会成为现实。

然而，也许这只是任何新的网络应用平台的必要演变——启动、炒作、抢占地盘、过载、疲劳、跌落……然后，重生？我们走着瞧。
