# 利用访问过的链接获得创意

> 原文：<https://www.sitepoint.com/getting-creative-with-visited-links/>

许多 web 开发人员本能地通过在一条语句中声明链接和已访问链接的样式来开始一个新的样式表:

 `a:link, a:visited {
/* rules here */
}`

这样做避免了为被访问的链接选择一个微妙不同(但同时视觉上明显)的颜色的棘手问题，但也扼杀了所有现代浏览器中嵌入的一个重要的可用性特征。如果你的网站大于几个页面，让用户快速查看他们是否访问过某个链接可以真正帮助他们浏览你的网站。

Jakob Nielsen，这位人人都讨厌的可用性专家，已经写了很多关于这个主题的文章。[改变访问过的链接的颜色](http://www.useit.com/alertbox/20040503.html)和[可视化链接的指南](http://www.useit.com/alertbox/20040510.html)是他关于这个主题的最新帖子。如果你发现很难认真对待 Jakob 的建议，因为他的网站上完全缺乏设计敏感性，你可能会喜欢 Fire 的[为可用性专家](http://www.designbyfire.com/000094.html)设计的设计之眼，它重新制作了*可视化链接的指南*，并展示了好的设计可以提高可用性的程度。

人们经常忘记的是，除了简单地使用颜色，还有其他方法来区分访问过的链接。我在我的另一个博客上有一个边链接集合，我称之为“Blogmarks ”,它使用下面的 CSS 声明来显示访问过的链接，并有一条线穿过它们。对于一般的网站导航来说，这是一个坏主意，但是在更专业的领域却非常有效:

 `ul#bmarks a:visited {
text-decoration: line-through;
}`

不过，要想找到真正有创意的替代方法，只需看看西蒙·科利森的[打勾](http://www.collylogic.com/index.php/weblog/comments/40/)技术就行了。Simon 在块级链接元素上使用背景图像，在已经访问过的链接旁边显示一个图形化的“勾号”,并扩展了这个想法，提供了一个简洁的悬停效果。背景图片是目前为止 CSS 设计者可以使用的最强大的视觉设计工具，很高兴看到它们以更有创造性的方式被使用。
