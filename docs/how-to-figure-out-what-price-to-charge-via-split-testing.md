# 如何通过分割测试确定收费的价格

> 原文：<https://www.sitepoint.com/how-to-figure-out-what-price-to-charge-via-split-testing/>

spivey 在最近的一篇评论中问及如何知道一个产品的价格。这个博客假设他是在询问通过网络销售的产品，尽管这里概述的过程适用于几乎任何类型的产品销售。

首先，一些背景。在网络出现之前(对于那些太年轻而不记得那段简单时光的人来说)，直销专业人士开发了一系列的流程，这些流程现在影响着许多基于网络的营销和销售。

因此，假设猫王收藏版的营销人员想知道“限量版”猫王收藏版的价格是 19.95 美元、29.95 美元还是 39.95 美元。他(或她)将进行如下三种方式的对比测试:

1.向同一个列表发送三封直邮邮件(例如，参观过格雷斯兰并在宾客登记簿上签名的人)。只发送足够的名字进行统计测试(通常 10，000 个名字就够了)。

2.邮件上只改变一件事:价格。将邮寄的 1/3、1/3 和 1/3 分割到同一个列表中，但是以不同的价格点提供盘子。

3.收集回应并确定哪个报价得到了最佳回应(例如，扣除所有费用后的最大利润)。

这种方法比焦点小组、访谈或其他任何方法都要好得多，因为它可以快速测量实际的回复率。

网络营销人员已经采取了这种方法，并将其应用于网络。不知道收多少钱？做到以下几点(此处简化；有许多变化):

1.将您的横幅广告设置为转到三个 URL 之一。这在 Google Adwords 上非常容易，你也可以在 Doubleclick 等服务上做同样的事情。

2.创建三个不同的 URL(例如`www.yourproduct.com/test1`——但称之为更优雅的东西，如`/yahoo1`)，每个测试不同的价格点。

3.同样，收集响应，并确定哪个 URL 获得最佳响应。这会告诉你该收多少钱。

网络上最好的营销人员使用这种方法来测试不同的广告服务、价格、报价(例如，是否提供 3 次轻松支付)和不同的横幅广告。

请张贴您对这一通用方法所做的任何改进…..
