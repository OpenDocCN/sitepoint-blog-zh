# HTML 会被取代吗？

> 原文:[https://www.sitepoint.com/will-html-ever-be-replaced/](https://www.sitepoint.com/will-html-ever-be-replaced/)

伊恩·希克森*(“Hixie”——WHATWG 规范编辑、CSS2.1 联合编辑和谷歌的 W3C 代表)*最近发表了[一篇关于 Google+](https://plus.google.com/107429617152575897589/posts/SiLdNL9MsFw) 的有趣帖子。偶尔有人联系他，向他推荐 HTML 的更好替代方案，但在所有情况下，都没有人接近。Ian 指出，任何技术要取代现有的 web 技术，都需要满足至少五个目标:

1.  没有许可证要求。
2.  保持供应商中立，接受每个人的意见。
3.  与设备和媒体无关；它应该可以在个人电脑、电视、手机、平板电脑、屏幕阅读器和任何未来的硬件上运行。
4.  保持内容中立，不局限于文档或应用程序的类型。
5.  在各方面都比现有的网络更好；更快，更可用，更多的功能，更容易开发，更容易赚钱，等等。

HTML 可能达不到目标二和目标三。诸如 XHTML2 和 XForms 之类的技术只满足了一个和三个。Java 和 Flash 在所有领域都很挣扎——我还想把谷歌的 Dart 加入这个列表。

让我们面对事实:网络技术从来没有也永远不会是完美的。总会有缺点和妥协。毕竟，本地视频支持花了 15 年才出现，供应商们仍未就实现细节达成一致。

然而，尽管微软、谷歌、苹果、Adobe、Sun 和甲骨文提供了替代产品，但网络技术的弹性令人难以置信:

*   HTML 起源于 SGML，就像它在 20 世纪 90 年代早期一样。
*   CSS 设计于 1996 年，保留了相同的选择器/属性语法。
*   JavaScript 被开发人员(不应该地)嘲笑，但它是世界上最常用的编程语言。自 1995 年 Brendan Eich 在几周内设计出这种语法以来，它只接受过微小的修改。

所有这些都幸存了下来，因为它们是第一个无需支付版税就能使用的实用网络技术。更好的选择可能会出现，但很难想象那些会得到通用供应商协议并有商业利益来抵消取代 HTML 所需的大量投资的选择。

不管开发人员和供应商的想法如何，HTML、CSS 和 JavaScript 仍将存在一段时间。我猜想，在它们作为核心开发技术被取代之前，将会有一个巨大的转变——可能是一个具有不同基础设施的更好的网络的发明。

但也许你知道一些我不知道的事？凝视你的水晶球，让我知道 HTML 是否有一个长远的未来，或者更好的替代品就在眼前…
