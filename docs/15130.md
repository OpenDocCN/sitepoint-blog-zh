# 为您的站点利用 CVS 的力量

> 原文：<https://www.sitepoint.com/harness-power-cvs-site/>

 **##### 我把文件放在哪里了？

在您的互联网之旅中，您可能会遇到缩写词 [CVS](http://www.cvshome.org ) ，从事开源项目的软件开发人员狂热地使用它。

如果你不好意思问，CVS 代表 Concurrent Versions System(不要与 CSV 混淆——数据文件的逗号分隔值格式),是控制任何类型文件的版本/修订的工具。CVS 背后的基本思想是有一个中心位置(或存储库),在那里存储文件的所有修订，以及关于它们是什么和每个新版本有什么变化的注释。然后，您可以从您喜欢的任何地方访问该存储库——无论是您家里的 PC、办公室的 PC，还是在世界另一端工作的其他开发人员。因此，CVS 确保参与项目的每个人总是使用文件的最新版本。

为什么你会对此感兴趣？如果你最近开始制作你自己的网页或者你的第一个 PHP 脚本，仅仅是为了好玩，版本/修订控制的想法可能看起来有点过头了。但是如果你除了涉足网站建设之外还做了其他事情，最终以下一件或多件事情会发生在你身上:

你在两台电脑上工作。首先你要在网页上做大量的工作，然后把它添加到你的网站上。然后，您在另一台计算机上做了更多的工作，并将其添加到站点，却发现您在第二台计算机上使用的是旧版本的页面，因此您之前在第一台计算机上做的所有工作都丢失了。

您对网页进行了一些重大更改，并将其添加到您的站点中，却发现有一个重大错误—您的站点陷入停顿。你不顾一切地争夺页面的前一个工作版本，却无济于事；你写多了。就像是在暗示一样，电话开始响起，第一个愤怒的顾客…

你彻底检查了你的站点，或多或少地对所有的东西进行了修改，用所有的新文件更新你的站点花了你三个小时(在这段时间里你的站点是关闭的)。

你开始和一些其他的网站开发者合作，但是当你花了 95%的时间去跟上其他人的变化时，你的工作很快就停止了。

在 Linux 中尝试使用`rm -r *` 命令，或者在 Windows 资源管理器中轻点鼠标，都会出现可怕的错误，你会从网络服务器上删除整个网站。你个人电脑上的备份已经过期，你的网络主机说“当然，我们会从备份中恢复你的网站。明天。”

CVS 可以帮助防止这些情况的发生。更好的消息是:它是免费的(开源的)！因此，无论你是一名图形设计师还是用 PHP 编写最新的 dB 抽象类，希望这篇文章将有助于揭开 CVS 的神秘面纱，并向你展示如何充分利用它。简历是一个很大的主题，所以我们必须远离细则，以避免陷入困境，但你可以随时找到相关的链接来进一步阅读。

##### 绝对的初学者

如果您曾经在任何分布式计算环境中工作过(并且，当您正在阅读这个网页时，您已经工作过了)，那么您很可能遇到过某种形式的版本控制。也许你在工作中试图从中央 nt 服务器更新一个 Word 文档，却被告知它是只读格式的，因为有人正在处理它。或者，您可能不得不处理数据库上的“记录锁定”,并且有过两个站点管理员同时更新同一篇文章的糟糕经历。不管怎样，您可能遇到过某种形式的版本/修订控制，并希望您能找到一种更好的方法来处理它。现在你可以了:有了简历。

本质上，CVS 提供了两个功能:**记录保存**和**协作**，以一种旨在解决您所有头疼问题的方式。CVS 特别的第一点是它**不**锁定记录。相反，它跟踪谁在做什么，允许每个人以他们自己的方式工作，但注意潜在的*冲突*，两个人试图用他们自己的个人修改更新同一记录。

但是在我们继续之前，是时候介绍一些我们需要使用的术语了，如果这个解释有意义的话。你会看到这些术语在互联网上随处可见，只要涉及到 CVS。

**记录:**我将在这里用它来指代你正在处理的任何“对象”,可以是. gif 图像、Shockwave 文件、PHP 脚本、Word 文档……或者你想要的任何东西。

**修订:**在“工作进行中”时，对*记录*(或组或记录)的更改。例如，您创建了一个 PHP 脚本。后来，你回来对它做了一些修改，从而创造了剧本的下一个*版本*。

存储库:这是您正在处理的项目的“母船”。你所有的工作都存储在这里，并与*修订*历史完全跟踪，让你*检查出*最新的*记录*。假设你有一个新客户，“Buildmysite Inc .”想让你帮他建网站。您将创建一个名为“buildmysite”的*存储库*，其中将存储所有的 HTML 页面、图像、PHP 脚本、文档以及用于创建 MySQL 数据库的查询。

**工作副本:**指的是您(或其他开发人员)正在处理的任何*记录*。一个*的工作副本*已经从*库*中*取出*，当你用 Photoshop、Ultradev 等编辑它时，它位于你自己的硬盘上。

**检查:**从*库*中获得*记录*的*工作副本*需要做的事情。一旦您的*库*被创建并且您所做的所有工作都被存储在其中，您就可以通过*签出*最新的*修订版*来开始每个工作日，这样您就可以继续工作了。

**提交(又名。签到):**你一直在创造一个*记录*，目前已经做得够多了(是的！是喝咖啡休息的时间了！)，您*将*记录*提交*回*存储库*，这样最新的*修订版*就存储在那里了。另一个开发人员，稍后*检查*记录*的*得到最新的*版本*来工作。

**日志消息:**每次*提交*一条*记录*时提供的评论；通常用一两句话来描述你做了什么。然后，该日志消息可供一般人查看，这样每个人都可以看到发生了什么变化。

**Update:** 这将使用来自*库*的所有最新变更来更新您的*工作副本*。例如，你通过在你的计算机上*检出*一个新创建的*记录*来开始你的一天，并且在 HTML 模板上一直工作到中午 12 点。然后你的好朋友 Bob 打电话来说他对 CSS 文件做了一些更改并且*将它们提交*到*库*，他认为 Janet 可能也修改了一些图像。所以你很快执行了一个*更新*来使你的*工作副本*与其他人的工作保持一致。一旦你看到你的 HTML 模板看起来有了新的工作，你*检查*中的它*到*库*。*

**冲突:**假设鲍勃和珍妮特*签出*同一个 PHP 脚本，不知道对方也在同时处理同一个文件。在做了一些修改后，珍妮特*将*她的工作提交给*库*；没问题。Bob 随后试图*提交*他的更改，但是将要覆盖 Janet 已经完成的工作。CVS 发现了问题并警告 Bob*冲突*。接下来，Bob 和 Janet 需要找出将他们的工作结合起来的最佳方式。这似乎是处理这种情况的一种奇怪方式，但这是 CVS 强大的原因之一。对于有许多开发人员的大型项目，或者开发人员必须通过互联网协作的项目，这种方法允许更大的灵活性。

此外，如果您正在处理的*记录*是文本形式的(比如网页或 PHP 脚本)，两个人有可能在不同的部分处理同一个*记录*(不会与另一个发生冲突)。然后他们可以*提交*他们各自的变更到存储库中，在那里 CVS 将第二部分与第一部分合并，产生最新的整个*修订版*。

**Tree:** 指 CVS 下存储的所有东西。在给定的 CVS 服务器上可以存储多个*存储库*。整个结构被称为树(就像硬盘上的任何目录结构一样)。

你可能已经意识到 CVS 是基于“客户机-服务器”的基础上运行的。例如，您有一台存储 CVS 存储库的服务器(这通常是一台具有大量磁盘空间和快速网络连接的机器，它可能运行某种 Unix 风格，如 Linux，或者某个版本的 Windows NT)。你通过网络使用客户机访问服务器，客户机运行在你自己的工作站上(几乎每个操作系统都有客户机支持:Unix、Windows 和 Macintosh)。这使得 CVS 非常强大——如果您的 CVS 服务器连接到互联网，您可以在世界任何地方查看项目的最新版本！

好的——这些是你进入 CVS 需要的基本术语和概念。不要慌！您需要知道它们是什么，但是我们稍后将向您展示实际的简历，这会让事情变得更加清晰。

##### 拥抱树

没错。说够了:是时候把你扔进深水区了！我们将安装一个 CVS 客户端，并在硬盘上签出一个存储库。对于这个例子，你需要运行任何版本的 Windows。

***大安装***

CVS 听起来很严肃，对吗？你认为安装客户端会很难吗？嗯，振作起来…

1.  前往 [cvshome Windows 下载](http://www.cvshome.org/dev/codewindow.html )，点击 [Win32](http://ftp.cvshome.org/win32/cvs-1-11-2.zip ) 链接(在“平台”一词下面)。将压缩文件保存在硬盘上的某个地方。

*   –现在将文件(cvs.exe)解压缩到您的 Windows“system32”目录中(通常是 C:/Windows/system32)

就这样:安装完毕！嗯…差不多了。我们已经安装了 DOS 命令行 CVS 客户端，它为您提供了一个简单的 CVS 接口，您可以直接使用。

但是我们不要就此打住——抓住时机，查看您的第一个 CVS 库！我们将检查一下 [phpBB2](http://www.phpbb.com) 的 PHP 源代码。phpBB 是一个在 [Sourceforge](http://sourceforge.net) 托管的[开源](http://www.gnu.org/philosophy/free-sw.html)项目，你可以在这里看到[。因为是 Sourceforge，每个人都可以匿名(只读)访问项目 CVS 树。phpBB 树的细节可以在](http://sourceforge.net/project/showfiles.php?group_id=7885)[这里](http://sourceforge.net/cvs/?group_id=7885)找到，如果你点击[“Browse CVS Repository”](http://cvs.sourceforge.net/cgi-bin/viewcvs.cgi/phpbb)你会看到实际上有两个存储库:phpBB 用于代码的旧版本 1，phpBB2 用于最新版本。所以让我们(从字面上)检查一下…

打开一个 MS DOS 提示符(通常在你的开始菜单的某个地方，可能在“附件”中)并键入以下内容。请注意，您需要连接到互联网才能工作:

```
cd   

mkdir cvs_root  

cd cvs_root   

cvs -d:pserver:anonymous@cvs.phpbb.sourceforge.net:/cvsroot/phpbb login    

cvs -z3 -d:pserver:anonymous@cvs.phpbb.sourceforge.net:/cvsroot/phpbb   

co phpBB2 
```

这是我们刚刚做的，一行一行:

1.  `将目录切换到 c:驱动器的“根目录”` 
`*   一个名为 cvs_root 的目录*   转到 cvs 根目录*   登录 phpBB CVS rver——当它要求您输入密码时，只需按 return 键*   检查(co)phpbb 2 代码——这是区分大小写的，所以要小心！`

 `将出现文件列表，这些文件将被复制到您的硬盘上。列表中的每个条目如下所示:

```
U phpBB2/common.php
```

U 代表“更新”,意思是它正在从 phpBB2 库签出文件，当快速检查您的硬盘发现没有 common.php 的现有版本时，它正在 c:CVS _ rootphbb 2 中创建一个

现在，如果您查看 c:cvs_root(您可能更喜欢在 Windows 资源管理器中完成这一部分)，您会发现一个名为 phpBB2 的子目录，其中包含所有最新的 phpBB2 代码。恭喜你！您刚刚签出了您的第一个 CVS 存储库！

现在，下一个魔术。使用您喜欢的文本编辑器，打开文件 c:cvs_rootphpBB2common.php，并在文件的顶部输入“这是一个测试”，然后以相同的名称保存文件，并关闭它。另外，出于好玩，删除文件 c:CVS _ rootpbb 2 install . PHP

接下来，回到 MS DOS 提示符下，从 c:cvs_root 目录键入以下命令(如果不确定自己在哪里，请键入 cd):

```
cvs -z3 -d:pserver:anonymous@cvs.phpbb.sourceforge.net:/   

cvsroot/phpbb update phpBB2
```

这一次，您将看到或多或少与您之前签出存储库时相同的东西，但是有一些细微的区别。

```
M phpBB2/common.php
```

M 表示在您的硬盘上找到的文件 CVS 与 CVS 存储库中的文件不同，因此它将保持不变。

```
U phpBB2/install.php
```

CVS 找不到 install.php，所以它报告说它正在创建一个。

然后剩下的看起来像这样:

```
cvs server: Updating phpBB/admin
```

这告诉您在硬盘上找到的文件与 CVS 存储库中的修订版相同。

不，再次进入你的文本编辑器，打开 c:CVS _ rootpbb 2 common . PHP——你的小消息还在文件的顶部吗？是的，它是！

然后在 Windows 资源管理器(或其他)中查看 c:cvs_rootphpBB2 目录——你能看到 install.php 吗？是的，你可以。它又回来了，尽管你删除了它。

就是这样！你刚刚完成了你的第一次更新！现在您已经看到了它的作用，记住更新工作的“方向”。“更新”是指从 CVS 库中**更新你自己计算机上项目的工作副本。为了用您在计算机上完成的工作来“更新”存储库本身，您执行了一个*提交*，我们稍后将对此进行研究。**

***开始感到温暖了吗？***

即使是匿名只读访问，我们也有一个非常强大的工具。以 phpBB 项目为例。对于标准安装，您只需用 MySQL 服务器等的本地设置更新文件 phpBB2/config.php。但是，感谢 CVS，您可以使用我们上面做的更新来跟上最新的 phpBB2 代码，同时保持您在 config.php 的本地设置不变。你可以在你的网站上安装 phpBB2 并定期更新，甚至不用眨一下眼睛。

把它转化为更新你的网站。您和一组开发人员对您的站点进行了一次大检查，更改了数百个文件。你把所有的工作都放在你的网站之外，让旧的工作保持原样。一旦你对新版本完全满意，你就可以登录到你的 live 网站，使用 CVS 在几分钟内更新所有的新作品——主要的大修让你的 live 网站宕机不超过五分钟！因此，看看一个典型的网站所有者是如何看待在他们的网站上使用 CVS 的简易性的，试试这篇文章。

我们安装的 CVS 客户端是 DOS 的命令行版本。但是如果您不喜欢命令行，不要绝望！有一个 GUI 版本叫做 WinCVS(或者有时叫做 cvsgui)，你可以在这里下载[用于 Windows、Macintosh 和 GTK (GTK 是为那些 Linux 朋友准备的——尽管他们可能已经对 CVS 了如指掌)。安装 WinCVS 客户端可能有点复杂，所以我将把安装和使用的解释留给](http://ftp.gnu.org/non-gnu/cvs/binary/stable/x86-woe/) [文档](http://www.wincvs.org/doc.html)。

CVS 有许多身份验证机制。在上面的例子中，我们使用了 pserver 方法(密码认证服务器)。这对于匿名访问来说是没问题的，但是，像 ftp 一样，它在互联网上以明文形式发送你的用户名和密码——任何有嗅探器的人如果碰巧在监听，都可以窃取它们。为了安全起见，对于比只读访问更重要的事情，通常会使用 ssh(安全 shell)通过互联网访问 CVS 存储库。Windows 和 Mac 用户应该看看 [WinCVS-SSH-Guide](http://sourceforge.net/project/showfiles.php?group_id=10072) 了解更多相关信息。

##### SitePoint 成员点击 Sourceforge

这个故事是 100%真实的，名字*没有*被改变以保护有罪的人！

很久以前，一个 PHP 程序员，在某些圈子里被称为 HarryF，决定写一个 PHP 应用程序(当它最终完成时，它的名字将在某个遥远的未来公布！).当他疯狂地为自己的作品编写代码时，另一位程序员和 Photoshop 大师，被一些人称为 Glenplake，好心地提出帮助 HarryF 设计他的应用程序的用户界面。

因此，两人开始热情合作，但很快就发现一切都不顺利。哈里夫的每周压缩文件包含了他的最新作品，与格兰普莱克的更新混淆了，陆地和海洋相隔很远，这只是增加了混乱。脱发随之而来。

幸运的是，在开源的圣山中矗立着 Sourceforge 的[神殿，所有美好的事物都会从这里发出。在](http://sourceforge.net)[注册一个新项目](http://sourceforge.net/docman/display_doc.php?docid=774&group_id=1)的仪式之后，我们两个强壮的 PHP 程序员能够进入 [CVS](http://www.cvshome.org) 的喷泉，从而最终成功合作。事情回到了正轨，项目蒸蒸日上，编码人员保留了他们的头发。

Sourceforge 要求美国开发人员使用 ssh 方法访问它。如果您运行的是 Windows，那么设置一些环境变量以使生活变得更简单是值得的，所以在我们继续之前:

1.  从[http://download.sourceforge.net/sfsetup/](http://download.sourceforge.net/sfsetup/)下载 ssh。该文件将被命名为类似 ssh-1.2.14-win32bin.zip 的名称

*   解压缩到一个目录:C:program filesssh*   现在打开 MS DOS 提示符，键入以下命令创建目录 c:program filesssh.ssh (Windows 资源管理器不允许您创建以句号(或句点)开头的目录):

`cd progra~1ssh  
mkdir .ssh`

*   更新您的系统环境变量:

*   右键单击“我的电脑”并选择“属性”
*   单击“高级”选项卡
*   点击“环境变量”

*   用 ssh 安装目录更新“path”变量(用于从系统的任何地方运行程序)

*   用鼠标选择“系统变量”下的“路径”变量，然后单击“编辑”
*   在您正在编辑的框的末尾，添加“c:progra ~ 1sh ”,并在前面加上一个分号，即“；c:progra ~ 1sh "

*   将您的 HOME 变量设为 CVS 查找 ssh 应用程序所需的变量

*   单击“新系统变量”
*   将新变量命名为“HOME”
*   将变量值设置为:“C:progra ~ 1sh”
*   单击“确定”

*   让 CVS_RSH 变量告诉 CVS 将使用什么认证方法

*   单击“新系统变量”
*   将新变量命名为“CVS_RSH”
*   将变量值设置为:SSH
*   单击“确定”

*   创建 CVSROOT 变量，告诉 CVS 客户端使用哪个 CVS 树

*   单击“新系统变量”
*   将新变量命名为“CVSROOT”
*   将变量值设置为:":ext:loginname @ CVS . your project . SourceForge . net:/CVS root/your project "
*   单击“确定”

*   单击“确定”,直到所有“我的电脑”属性窗口都关闭。

假设你已经安装了 cvs.exe，你现在可以访问你的 Sourceforge CVS 树了。

回到 HarryF 和 Glenplake 承担国际合作的艰难任务的时代，HarryF 就像这样设置他的环境变量。完成后，他将目前完成的所有工作导入到 CVS 中，如下所示:

```
cd MyWorkthe_project    

cvs import the_project vendor start
```

他一行一行地写道:

1.  他切换到他想要添加到 CVS 的项目所在的目录。

3.  他将该目录下的所有工作导入到 CVS 存储库“the_project”中。在这一点上，他不得不毫无疑问地使用“供应商启动”。

一旦它被导入到存储库中，HarryF 将目录“c:MyWorkthe_project”重命名为“c:MyWorkthe_project_old ”,这样它就不会在签出新存储库的第一个副本时丢失:

```
cd MyWork   

cvs co the_project
```

就像变魔术一样，项目被恢复到了原来的位置！注意，在项目的每个子目录中都有一些额外的目录，名为 CVS。这些目录包含 CVS 用来跟踪你的工作的本地信息——除非你知道你在做什么，否则不要乱用它们。

HarryF 随后发现了 CVS 的一个更黑暗的秘密:CVS 不能很好地处理二进制文件，包括可执行文件(。exe)、图像(。gif，。png，。jpg)，以及 Shockwave(。swf)文件等。默认情况下，CVS 假设所有文件都是文本文件，并试图在其中放置修订代码之类的东西，这往往会对二进制文件产生灾难性的影响。所有的。HarryF 制作并导入 CVS 库的 gif 图像现在被打乱了，毫无用处。他能做什么？

嗯，一般来说，在第一次导入项目之前，建立一个 [cvswrappers](http://www.cvshome.org/docs/manual/cvs_18.html#SEC164) 文件来告诉 CVS 哪些文件类型是二进制文件是个好主意。或者，当您向现有项目添加新的二进制文件时，您也可以使用“cvs add -kb myfile.gif”来告诉 cvs 该文件是一个二进制文件。

哈里夫通过狂热地查阅 cvs 管理手册以及如何从这场灾难中恢复过来发现了这一点。他实现了它——嘿——很快！—他的二进制文件恢复了往日的辉煌。

最终，HarryF 对 CVS 库感到满意，他给 Glenplake 发了一封电子邮件，告诉他一切都很好，是时候将他完成的工作与主代码合并了。哈里夫随后决定他需要休息一下，并出去开始了另一场与应该修理他摩托车的机械师的争论。

与此同时，Glenplake，作为一个经验丰富的 Linux 类型的人(也是一个 Photoshop pro——他的才华是无与伦比的！)，登录到他的 Linux Web 服务器，并快速设置环境以使用他们的新 CVS 树。

大多数常见的 Linux 发行版已经安装了 ssh 和 cvs 客户端(Glenplake 的肯定安装了),所以您可以像我们在 Windows 中那样使用以下命令来设置环境:

```
export CVS_RSH=ssh     

export CVSROOT=glenplake@cvs.theproject.sourceforge.net:/    

cvsroot/the_project
```

格伦普拉克比那更聪明，他编辑了文件。bash_profile，并添加了:

```
CVS_RSH=ssh     

CVSROOT=glenplake@cvs.theproject.sourceforge.net:/cvsroot/the_project    

export CVS_RSH    

export CVSROOT
```

这意味着他每次登录时，都会正确设置环境变量，而不需要重新输入它们。

现在，移动到他的服务器上的 Web 根目录:

```
cd /home/glenplake/www
```

…他将安装在那里的代码版本重新命名为:

```
mv the_project the_project_old
```

接下来，他从存储库中抓取了最新版本:

```
cvs -z3 co the_project
```

成功！Glenplake 现在有了一个存储库的工作副本。

他迅速地将他完成的工作从旧目录复制到新的工作副本中:

```
cp the_project_old/updated_file.php the_project/    

cp the_project_old/new_image.gif the_project/
```

这些命令的作用如下:

1.  复制了一个 HarryF 的文件，Glenplake 已经更新到工作目录

3.  将 Glenplake 创建的新图像复制到工作目录中

然后他进入工作目录:

```
cd /home/glenplake/www/the_project
```

…并向 CVS 库介绍了他的工作。对于他更新的文件，他使用了:

```
cvs commit -m "Fixed some bugs" updated_file.php
```

…将文件提交到存储库。他使用-m“修复了一些错误”来添加一个*日志消息*到 updated_file.php 的修订历史中，这样以后两个开发人员都可以看到文件发生了什么。

然后，对于他创建的新文件，Glenplake 键入:

```
cvs add -kb new_image.gif    

cvs commit -m "The project logo" new_image.gif
```

…将新映像添加到存储库中。“cvs add”告诉 cvs 计划在下次执行提交时将该文件添加到存储库中。“-kb”告诉 CVS 该文件是二进制的，必须小心处理。“cvs commit -m”项目徽标“new_image.gif”实际上是将新图像提交到存储库，并带有*日志消息*“项目徽标”。

在 Windows 上，Glenplake 也可以使用“cvs.exe”客户端愉快地执行相同的 cvs 命令。

现在他完成了，格兰披拉出去从他当地的面包店偷了一些百吉饼。

与此同时，在一个很远很远的地方，哈里夫回到了家，继续他的项目。

因为他一直在关注文档，所以他知道从“cvs 更新”开始是一个好主意，以检查他是否拥有最新版本的存储库。所以他输入了:

```
cvs update
```

CVS 的回应是:

```
U the_project/updated_file.php    

U the_project/new_image.gif
```

CVS 检查了 HarryF 是否有最新版本的存储库。然后，它发现了 Glenplake 所做的更新，并将其下载到 HarryF 的工作副本中，给他最新的文件进行处理。

因此，格兰普莱克和哈里夫继续前进，超越了他们所有的个人期望，完成了一个相当大的项目。

##### 包扎

如果你能原谅这篇糟糕的文章，这最后一个例子应该能让你了解 CVS 是如何工作的，特别是当你和别人合作的时候。这只是 CVS 能力的一个简单演示，我确信没有必要告诉你它能为你做更多的事情。

首先，我们还没有真正了解 CVS 如何跟踪修订，以及您如何查看日志、查看差异(一个修订与另一个修订之间的差异)、恢复您的工作的旧修订、解决冲突以及一系列其他事情。但是以你现在所知道的，你已经足够开始使用 CVS 了。随着您对它的深入研究，您会发现理解文档变得容易多了，并且可以快速找到您需要的内容。

***得到帮助***

也许最好的起点是 CVS 书籍,虽然很长，但需要时间来解释清楚概念。如果您只是快速浏览一下目录页，您将会对 CVS 可以做的一些更高级的事情有所了解。或者，你可能想试试[网站项目的 CVS 版本控制](http://www.durak.org/cvswebsites/howto-cvs/index.html)，它覆盖的范围较小，但更适合我们这样的网站建设者。“官方”文件可以在 http://www.cvshome.org 的[或这里的](http://www.cvshome.org)[找到。当然，如果你不喜欢这些资源中的任何一个，在谷歌上快速搜索会找到许多替代的解释、教程和指南。你绝对不是唯一一个有简历的人。](http://www.loria.fr/~molli/cvs/doc/cvs_toc.html)

***CVS Server for Windows***

正如我们在本文开始时提到的，CVS 不仅仅适用于*Nix 类型。正如您已经看到的，它被广泛使用并得到了很好的支持。它在许多公司也很流行，通常用于软件开发。毫无疑问，你可以从简历中获得的任何技能对你的简历来说都是好消息。

所以，Windows 用户，去 http://www.cvsnt.org/买一份吧。你会发现替代的逐步安装指南[这里](http://www.devguy.com/fp/cfgmgmt/cvs/cvs_admin_nt.htm)和[这里](http://w1.858.telia.com/~u85831169/InstallCVSNT.html)。CVSNT 服务器已确认能够在安装了 Service Pack 6 的 Windows NT4 服务器/工作站、Windows 2000 Server/Professional 和 Windows XP-Pro 上运行。还有一个能够在 Windows 95/98 上运行的精简版本，所以总而言之，你应该能够在某个地方安装服务器。

***在线查看简历***

最后，还有基于 Web 的 CVS 查看器，允许您在线浏览您的 CVS 树——非常好地了解了树包含了什么，以及做了哪些更改。三个例子是用 PHP 编写的 [Chora](http://www.horde.org/chora/) 、用 Python 编写的 [ViewCVS](http://viewcvs.sourceforge.net/) 和用 Perl 编写的 [cvsweb](http://stud.fh-heilbronn.de/~zeller/cgi/cvsweb.cgi/) 。
