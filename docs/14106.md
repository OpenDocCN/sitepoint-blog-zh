# 对开源的新看法

> 原文：<https://www.sitepoint.com/a-fresh-look-at-open-source/>

有时候，只需要从技术领域后退一步，就能有一个全新的视角。

在过去的几周里，我一直沉浸在一个 100%不开源的大型企业项目中。恰恰相反——自上而下构建在全面专有的平台上。我很高兴地宣布它会做得很好(当然，我正在努力——对！？).然而——它也让我更清楚地看到了开源在企业中能够并将成功的地方——以及它所面临的障碍。

注意:我从 1998 年开始使用开源软件——然而，从 1992 年开始使用专有平台——坦白地说，双方都支付了费用。；>)

现在——在 NDA(对细节守口如瓶的奇特缩写)下，我不会透露太多。然而，这是一个很好的项目，它将美国各地的众多数据源整合到一个数据集市中，并将最终为报告和其他有趣的活动提供一个非常好的整合 web 应用程序。首先，该公司希望每年节省数百万美元与以前管理这些不同数据源相关的成本。

也就是说，通过开源技术，采购、规范、原型和开发的成本可以大幅削减。首先——所使用的平台不具备某些开源选项的成熟度(我敢肯定你现在已经猜到了),也不具备社区文档和知识。选择开源选项将缩短到达原型阶段的时间，并开始微调可用性、功能规范和用例。

其次，由于当前所选平台的**经验证* *方法有限，该团队不得不在与集成到这一复杂 web 解决方案中的一些供应商的对接中构想出无止境的变通办法。

现在，在我们开始争论做出改变之前——我是在平台设置、初始需求编写和程序员聘用后的几个月被带进来的。这个项目已经陷入困境，我正在努力重新回到正轨。因此，基于承诺、投资和期限，它在这一点上不一定是“可转换的”。然而，在我的要求下，iterim 进行了事后分析，作为探索所有可能选项的补充。

虽然对使用开源软件的选择有些惊讶，但它被认为是小孩子的游戏，对于这个“规模”的项目是不可行的。考虑到像 NASA、国家安全局和许多在核心生产环境中部署开源的企业，我发现这有点令人困惑。除此之外，我浏览了赞成和反对的论点。我发现问题显然是语义上的——打破对 Linux 和开源的宣传和先入为主的观念，理解它的功能。

很难理解一大群非常有能力的技术和商业人士对开源没有一个坚定的理解——但这是千真万确的。这些人非常聪明能干，在没有利用 Linux 和开源工具的情况下取得了很多成就。

所以，回到开始(读者中的音乐家们的尾声)，偶尔后退一步，意识到世界在继续旋转，商业在进行，太阳在没有看到开源的情况下升起和落下，这是有好处的。这应该鼓励我们不仅要继续改进我们使用的开源工具，而且要投入一些精力来更清楚、平静和持续地向越来越多的观众解释我们的宇宙。

顺便提一下，我工作的组织正在探索他们如何开始一个测试平台来检查开源的可能性。尽管我很清楚这一点，但它与美元没有什么关系。它与灵活性、社区资源和上市时间有更多的关系。需要考虑的事情。
