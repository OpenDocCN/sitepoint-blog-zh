# 域名淘金热第 7 部分 Gets 获得 ICANN 批准

> 原文：<https://www.sitepoint.com/wls-gets-icann-approval/>

**正如域名淘金潮系列第六部分[中所预测的，WLS 已经被批准了！经过长时间的拖延，](http://www.webmasterbase.com/article/640) [ICANN 最近批准了 VeriSign 的“等待列表服务”(WLS)在](http://www.icann.org/minutes/prelim-report-23aug02.htm)的引入。**

对于那些不熟悉 WLS 的人来说，这是一个注册级别的，先到先得的订阅服务，用于自动注册被删除的域名。这在很大程度上是基于 SnapName 成功的 SnapBack(TM)补充订单系统，事实上，VeriSign 已经与 SnapNames 合作设计和生产 WLS 系统。

一旦 WLS 推出，任何删除。com 或者。网络域名(。org 名称将不包括在内)将不再被 ICANN 注册服务商和各种域名抢注服务“争夺”。相反，它将自动分配给该名称的 WLS 职位持有者。

换句话说，对于某些域名来说，注册服务商抢注域名的日子将会结束，取而代之的是先到先得的反向订购系统。只有那些还没有被订阅的名字才会被删除。

那么这个新系统什么时候实施呢？这个问题还没有明确的答案。作为批准该系统的条件之一，ICANN 规定:

<q>*订阅在 WLS 下。com 和。在这些顶级域名中所有已删除域名的赎回宽限期实施至少六个月后，VeriSign 全球注册管理机构服务方可接管. net 顶级域名。*</q>

由于“赎回宽限期”尚未实施，看来我们离推出 WLS 还有许多个月的时间。

ICANN 附加的其他条件也令人感兴趣。特别是:

<q>*不得因为任何注册服务商级别的预订服务而给予与 WLS 相关的优惠或排斥。*</q>

这意味着注册中心不会自动将 SnapBack 订阅转换为同名的 WLS 订阅。但是，SnapNames 及其注册合作伙伴打算将快照转换为 WLS 订单。

换句话说，一旦 WLS 启动，您的 SnapBack 订阅不会过时。SnapNames 给出的最低保证是，他们会将每一个 SnapBack 订单免费转换为 WLS 订阅。他们会尝试使用 SnapBack 上的名称，如果不可用，您可以选择不同的名称，就像今天的 exchange 保证一样。

应该指出的是，WLS 只是在 12 个月的试验基础上被批准的。然而，考虑到注册管理机构、注册服务商和消费者在转换为 WLS 时所要求的变化，以及该系统将不可避免地形成的势头，您不必是天才也能看出 WLS 可能会在未来很长一段时间内与我们共存。

虽然许多域名投机者将他们的商业模式建立在域名丢弃过程中错综复杂的域名抢注上，并对 WLS 竭力反对，但大多数消费者会发现这是一个更直接、更公平、更容易使用的系统。他们将知道，在未来，如果他们订阅了某个特定的域名，而之前的所有者没有续订，他们将获得该名称。没有比这更公平的了。

作为一个花了无数时间试图向困惑的客户解释当前域名丢弃系统如何工作的人，我期待着这个新的对消费者更加友好的系统。

带来勇敢的新的过期域名世界！
