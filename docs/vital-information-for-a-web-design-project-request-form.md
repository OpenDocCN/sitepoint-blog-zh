# 网页设计项目申请表的重要信息

> 原文：<https://www.sitepoint.com/vital-information-for-a-web-design-project-request-form/>

当我开始一个新的设计项目时，我的接收过程的一部分是要求客户完成一份项目申请表，即使我们已经提前进行了电话交谈。我发现，让客户花一些时间思考这个项目，并写下(或输入)细节是充实工作的一个很好的练习，包括他们希望和不希望网站完成的内容。这些信息成为确定项目范围和创建项目计划的工具。如果提供了正确的信息，可以在整个项目中节省时间和资源。

挑战在于让表格足够长，以包含你想问的所有问题，同时避免让表格太长而让人不知所措。提交的方法也可以促进这一进程。我倾向于以适合每个客户及其偏好的方式定制我的请求格式。我使用的一些格式包括微软 Word 表格(电子邮件、传真或回邮)、在线表格、将问题复制并粘贴到电子邮件中，甚至口头列出清单，有时自己写下答案。

以下是我打算从新客户那里收集的一些信息:

**联系方式**

*   名字
*   公司名称
*   邮寄地址
*   电话号码
*   电子邮件地址

**商业信息**

*   企业成立的年份
*   企业名称是如何选择的
*   业务性质
*   目标受众
*   主要竞争对手
*   任何现有品牌(徽标、颜色、字体)

**网站信息**

*   域名
*   网站的目的
*   他们会提供什么信息
*   他们最喜欢的设计风格
*   期望的功能/特殊特性
*   他们喜欢的其他网站，为什么

**其他**

*   预算
*   时间表
*   交流偏好
*   联系人
*   他们可能需要的其他服务

在他们提交表格的时候，我要求他们给我发送他们所有的网站信息:内容、徽标、照片、其他图形、营销材料、托管帐户信息。通常，我从转发的电子邮件、邮寄的硬拷贝或各种附件中零星地获得这些信息，但我会将收到的所有信息归档并记录下来，因此它们会成为客户文件的一部分。
