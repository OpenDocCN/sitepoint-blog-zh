# YUI 3.2.0 预览版有什么新功能？

> 原文:[https://www . site point . com/whats-new-in-the-yui-3-2-0-preview-release/](https://www.sitepoint.com/whats-new-in-the-yui-3-2-0-preview-release/)

雅虎用户界面(YUI)开发团队最近[宣布了](http://www.yuiblog.com/blog/2010/07/26/3-2-0pr1/)即将到来的 YUI JavaScript 和 CSS 库 3.2.0 版本的开发者预览版。虽然许多开发人员(包括我自己)在进行 JavaScript 开发时更经常使用 jQuery，但 YUI 仍然是一个有趣的选择；3.2.0 预览版中的这些新特性可能值得一试。这是一次大范围的更新，提供了一系列新的产品，所以我在这里只讨论最突出的几个。要查看完整的分析，请查看[的博客公告](http://www.yuiblog.com/blog/2010/07/26/3-2-0pr1/)和完整的[变更日志](http://yuilibrary.com/projects/yui3/report/38)。

首先，YUI 3.20 增加了对触摸事件的原生支持，因此你可以随心所欲地浏览页面。jQTouch 插件已经为 jQuery 开发者提供了类似的功能，所以很高兴看到 YUI 加入竞争。

接下来，YUI 增加了对使用 CSS3 转场的动画的支持。这意味着现代浏览器中的简单动画将能够利用硬件加速，同时加载更少的 JavaScript 代码。当然，所有的浏览器差异都解决了，所以你需要做的就是插入你想要的动画的值。

3.2.0 中实现的另一项下一代技术是[离线存储](http://yuiblog.com/sandbox/yui/3.2.0pr1/examples/cache/cache_offline.html) : YUI 的缓存实用程序已经更新，允许数据在支持时在浏览器会话之间持久存储。

YUI 加载器虽然算不上什么惊人的功能，但仍然令人感兴趣，它已经过更新，改进了缓存，并增加了基于浏览器功能加载代码的能力。这意味着该团队已经能够在整个库中将纯 IE 代码与现代浏览器代码分开。然后，加载程序可以根据用户的浏览器来确定需要哪些代码。因此，使用现代浏览器的访问者会发现自己加载了更少的 JavaScript，即使你的应用程序没有任何改变。

最后，还有一些新的小部件。其中一个特别有趣的是 [ScrollView](http://yuiblog.com/sandbox/yui/3.2.0pr1/examples/scrollview/index.html) 小部件，它模拟了苹果 iOS 设备上的滚动列表，在列表的顶部和底部有一点反弹。

这些特性似乎有一个共同的主线:HTML5 和移动平台。我相信这不是偶然的，看到更多更好的开发工具进入这个领域真是太好了。正如我所说的，不仅仅是这些功能，所以如果你是 YUI 的粉丝，或者如果你还没有去看看，那就去拿一份预览版来玩吧。
