# Opera 11 中终于出现了扩展

> 原文:[https://www.sitepoint.com/opera-11-extensions/](https://www.sitepoint.com/opera-11-extensions/)

尽管在其他浏览器上取得了成功，Opera 是唯一不支持第三方插件的主要供应商。粉丝们热情地争辩说 Opera 不需要扩展，因为它提供了比任何竞争对手更多的内置功能和配置选项。其他人声称 Opera 不适合他们，因为它缺少一个基本特征。

争论应该随着 Opera 11 的发布而结束——第一个支持扩展的浏览器版本。开发人员将能够使用标准 HTML5、CSS、JavaScript 和支持的 API 来添加或扩展浏览器的功能。扩展将基于最终可能成为开放标准的 W3C Widget 规范。

Opera 的扩展将类似于 Chrome 和 Safari 提供的扩展。这是明智的——它们很容易创建，移植现有的附加组件应该相对容易。在第一个版本中，开发人员将能够运行后台进程，添加按钮，创建菜单，并操纵界面控件，如选项卡和浏览器窗口。Opera 保持它的简单，这样他们就可以专注于做好它，但是随着系统的发展，会增加更多的 API 控制。

附加安装将是一个轻松的点击确认过程，或者你可以将一个扩展文件拖到 Opera 中。你可能不需要重启浏览器。

Opera 11 的 alpha 版本将很快推出。扩展开发教程、指南和参考资料将在[dev.opera.com](http://dev.opera.com/)发布。

一旦 Opera 有了扩展，你会考虑使用它吗？这个功能能提升 Opera 的形象和市场份额吗？还是为时已晚？
