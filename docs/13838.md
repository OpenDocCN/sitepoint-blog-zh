# 你的链接信誉如何？

> 原文:[https://www.sitepoint.com/whats-your-link-reputation/](https://www.sitepoint.com/whats-your-link-reputation/)

“你好，高德，谷歌更新了我们新开发网站的 Pagerank。在工具栏上，我们现在是 5，但我们没有看到好的排名或谷歌参考。我谷歌不出来。他们想要什么？”

大众观点需要很长时间来适应新的现实。在搜索引擎优化领域，谷歌的 Pagerank 计算具有神话般的品质，但大众的理解落后于谷歌现在真正在做的事情。搜索引擎已经改变了它计算排名的方式。无论你是开发人员/作家/搜索引擎优化/网站管理员还是营销人员，了解主题敏感的 Pagerank 对于确保你的网页被认为是相关的是至关重要的。如果你想从其他网站的链接中得到很好的提升，你最好理解“链接声誉”的概念

![1509_prmania](../Images/777f36f4e38ffdc8aa273f16c28da8fc.png)

本文用简单的术语描述了一个非常复杂的信息检索过程。然而，这种复杂性不会阻止我们试图获得某种理解。谷歌之外没有人真正了解谷歌每个算法组件的私密细节。有很多猜测。你不需要一个信息科学或人工智能的学位来理解像谷歌这样的搜索引擎试图实现什么。他们希望研究文档之间的关系，以便从每天抓取的数十亿垃圾链接中区分合法链接。我们想知道的:谷歌是如何分析链接的？

##### 研究链接中的潜在含义

谷歌使用深度分析技术来研究文档和它们之间的联系，因为它努力理解它们讨论的主题。是的，谷歌阅读网页，统计标题标签、正文和链接中的关键词。它从字里行间理解它在收集的文档中找到的链接的上下文，并分析它们的主题相关性。这就是术语“语义”的由来:“潜在语义”简单地指内容的潜在含义，这正是谷歌所追求的。

这意味着，你今天看到人们购买的所有付费文本链接可能对排名没有任何影响，只是因为这些链接最终可能出现在错误的网站上的错误上下文中:它们可能跑题了，或者广告商的网站有其他问题阻止这些特定链接提高目标页面的排名。

目前，谷歌的排名系统过度依赖出站锚链文本。垃圾邮件制造者和 SEO 充分利用了这一事实，因此搜索引擎通过评估文档的“主题相关性”来继续挖掘。是的，Pagerank 仍然是相关的，但只是在它如何与关键词主题相关。

谷歌使用语义分析等方法来更深入地解读链接的含义，并正在将 Pagerank 应用于关键词本身。事实上，谷歌甚至在某种程度上将其应用于同义词和相关词。这可以让搜索引擎更加确定你的网站是合法的聚焦在一个给定的主题上，并且链接是合法的。

因此，尽管你的主页页面排名可能是 7，例如，当涉及到一个特定的关键词短语时，它可能只排名 2。为了创建和维护你的排名，你需要了解谷歌是如何评估链接的，以及如何评估单个关键词的 Pagerank。

虽然大多数 SEO 都知道链接的声誉，但很少有人会想得比在链接锚文本中加入关键词更远，并希望得到最好的结果。而不是担心精确的工作，他们只是出去买了一吨的关键字加载链接。有时这种“肌肉”方法确实有效，但更多时候结果令人失望。看起来搜索引擎并没有被大量的链接所迷惑，这些链接似乎都表明你的网页/网站是该关键词主题的顶级资源。但是 Google 如何过滤掉这些链接的效果呢，这些链接很多都是来自其他网站的高 Pagerank 页面？

这是 SEO 和信息科学的一个开拓性领域。做一个搜索，你会发现宝贵的少数资源讨论链接信誉分析(或链接引用分析)或任何关于搜索引擎如何收集和处理链接信息。很少有人知道 Googlebot 在爬行时是如何收集和存储信息的，或者事后是如何处理的。更糟糕的是，一些研究论文使用的是基本上什么都没说的花哨 IS 语言。它是一个诡计吗？

##### Google Pagerank:从简单到多向量

最初，简单的 Pagerank 占主导地位的算法[给了局部不相关网站之间的链接同等的权重](http://dbpubs.stanford.edu:8090/pub/showDoc.Fulltext?lang=en&doc=2002-6&format=pdf&compression=&name=2002-6.pdf   )。然后，高 Pagerank 网站的所有者开始使用他们网站的高 page rank 来提升其他网站的排名，这使得搜索结果不那么相关。

为了缓解这个问题，谷歌(和其他搜索引擎)超越了“流行度”系统，开始研究链接的潜在意义和每个链接出现的上下文。例如，他们意识到，如果一个营销网站的链接指向另一个营销网站，它可能比一个非营销网站的链接更相关。

2002 年，伯克利的毕业生[塔赫尔·哈维里瓦拉提议用一组 Pagerank 向量来取代单一的 Pagerank 向量系统](http://www.stanford.edu/~taherh/papers/topic-sensitive-Pagerank-tkde.pdf)，这将有助于分配与关键词相关的 Pagerank，从而在查询时为页面生成特定于查询的重要性分数。这是一个简单的基于流行度的搜索引擎的第一个进步。

在过去的 5 年里，分析超链接的含义取得了巨大的进步。[潜在语义索引](http://www.seobook.com/archives/000657.shtml)被引入到算法中，以超越显而易见的内容，来“过滤”相关链接信息，从而可以发现与有价值的网页最相关的链接。这意味着内容正在被更深入地分析。LSI 的引入是为了控制网站所有者试图歪曲链接所指向页面的内容。随着 LSI 变得越来越强大，网站将不得不关注在网站的副本和链接中使用相关的单词、同义词和词干变化。

是的，[谷歌确实分析链接](http://www.google.com/corporate/tech.html)。这是商业的核心，T2 影响着有机搜索结果以及 AdSense 广告内容。每个链接包含的意义远远超出了我们在所涉及的两个页面上看到的内容。[最重要的计算是包含链接的网站的声誉或权威。如果它是一个受信任的站点，有很多链接指向它，并且是同一个主题，那么它可能有很高的权威评级。它的出站链接将被认为是有价值的，因此，将在排名中有影响。](http://citeseer.ist.psu.edu/cache/papers/cs/22164/http:zSzzSzwww.www10.orgzSzcdromzSzpaperszSzpdfzSzp474.pdf/bharat01when.pdf)

不过，事情是这样的。即使主题相关性元素在增加，网站内容仍然可以被操纵来增加主题相关性，来自其他主题无关网站的链接仍然可以在提升排名中发挥作用。因此，搜索引擎必须超越两个网页和它们之间的链接。它必须考虑的不仅仅是链接之间流动的原始页面排名:它还必须对这些链接说了什么得出结论。你的网站应该与你的主题“社区”中的其他网站有良好的链接，特别是那些被认为是社区中心的网站:“权威网站”。巧合的是，权威网站垃圾邮件的趋势越来越明显，这也是谷歌不得不应对的问题。因此，当你在热门网站上建立或购买链接时，你应该研究该网站的热门声誉，以了解你网站的链接是否真的会提高你的排名。网站的 Pagerank 也无济于事。

##### Pagerank:数字货币

如果你不熟悉 Pagerank 排名系统，这个类比应该可以让事情变得清楚:Pagerank (PR)就像一种货币。它是流动的，可以授予任何站点。这是一个通过计算得出的简单数字，因此很容易存储在数据库中并记入特定的网页。

谷歌将这种基于点的系统应用于其网页集合索引中的所有网页。一个链接被认为是对它所指向的网站的“投票”，有点像给那个网站一些钱。可以说，那些拥有最多入站链接的网站获得了最多的“钱”。这种货币流入网站的 Pagerank 银行账户。网站所有者可以用他们选择的任何方式使用这些“钱”或 Pagerank，例如，提高他们拥有的另一个网站的排名。你可以看到这对谷歌的搜索结果质量有多不好。

现在，有了话题敏感或话题相关的 Pagerank，我们就有了多种形式的货币；然而，这些货币形式中的一种不容易转换成另一种。一个关键词短语的高 Pagerank 并不意味着你在另一个不同的短语上会有高排名。

##### 人气系统死了！数量不再重要

在 90 年代后期，Pagerank 是一个基于流行度的系统，仅仅基于链接的数量，以及哪些网站被链接得最多。很少考虑从一个或另一个网站的链接是真正的“说”。链接确实会说话:它们传达了它们所指向的页面的一些信息。但是，在数十亿个垃圾网页上，有数十亿个误导性的、不相关的链接，它们只是为了转移搜索引擎而创建的。问题也不仅仅是去除没有价值的链接。对于谷歌来说，找到最好的才是真正的问题。我们习惯于认为搜索引擎专注于最好的内容，但谷歌现在专注于寻找最好的链接。

主题敏感过滤系统试图专注于网页的主题社区。因为它们专注于一个主题——查看该主题在网站上的使用和存在——这些系统可以过滤掉伪装成其他东西的无关页面。这样，搜索引擎就能读出字里行间的意思。他们甚至不需要依赖页面链接中的关键词。他们可以阅读链接周围的材料，评估链接是如何写的，并考虑它是否使用了与主题本身相关的单词。

所以，当你今天在谷歌工具栏看到一个网页的 Pagerank，这是一个独立于链接声誉的估计。实际上，它测量的是允许通过过滤器的页面的 Pagerank。这可能是沙盒过滤器，或其他过滤器，阻止谷歌所谓的无关链接。

你可能见过 PR 值为 7 的网站，但指向它们的链接相对较少。在这种情况下，这些链接被认为是相关的和正确的，谷歌非常重视它们。其他网站的 PR 值可能是 4，但是却有成千上万的链接指向它们。那么，为什么公关水平低呢？这些链接显然已经通过主题敏感过滤器被去除了大部分或全部影响。如果发布链接的站点对相关的关键词没有“声誉”，那么小的 Pagerank 就会被传递给链接的站点。

主题敏感过滤器是搜索引擎的一个巧妙而高效的工具。过滤器是控制垃圾链接最有效的方法之一。然而，你可能想知道一个过滤器能否被击败，如果是这样，你的竞争对手是否在利用这一知识。有可能你的竞争对手正在利用这个漏洞，但可能更多的是出于偶然，而不是有意为之。

主题敏感的 Pagerank 是一个系统，在这个系统中，你网站的 Pagerank (PR)的每一小部分都与某些关键词相关联。如果你的网站有很高的关键词声誉(如关键词“讨债”)，一个单独的公关必须适用于这些关键词，并记录在谷歌的数据库中。这种方法需要大量的计算和存储。谷歌只能存储它搜索到的网站的信息，以及在搜索后的处理活动中的信息。

##### 存储了多少链接信息？

我们知道 PR 通过一千个链接，但是话题信息能通过链接传播多远呢？如果谷歌有足够的处理能力，它可以为找到的每个链接存储链接信誉信息。这意味着第一个网页的链接信誉将会随着网页排名通过一千个网站传递到链接链中的最后一个网站。在这个系统中，一个链接的真正意义在整体上被评估，并与相关的 Pagerank 一起记入链中的最后一个站点。

然而，实际上，谷歌和其他搜索引擎并不具备进行这种分析的计算能力。因此，尽管 Pagerank 是主题敏感的，但只有少量关于页面主题或主题的信息可以被收集并用于算法中。拥有庞大网站网络的公司可以利用这一弱点。一家大型媒体公司可能有 150 个网站，并可能将它们链接在一起以最大化它们的排名。通过这种方式，Pagerank 可以被集中并有策略地发送到各种目标站点。

![1509_phpnet](../Images/a7457dda6686c900d18bff66d0b482a8.png)

如果你在谷歌搜索中输入“PHP 网络开发者”，你会注意到[php.net](http://www.php.net)排名第一。为什么？“developers”这个词在主页上只出现了 4 次，所以很明显有些事情正在进行中。如果你[查看反向链接](http://www.google.com/search?sourceid=navclient&ie=UTF-8&rls=GGLG,GGLG:2005-37,GGLG:en&q=link:http%3A%2F%2Fwww%2Ephp%2Enet%2F)，你可以看到 331，000 个链接指向 point。这可能会让你认为是 Pagerank 决定了网站的排名。看看[Zend.com](http://www.zend.com)，一个链接到 php.net 的重要网站，你会注意到这个网站是关于 PHP 开发的。Zend 网站中有 60，000 页出现了“developer”一词，884，000 页出现了“PHP”一词，77，000 页出现了“Web”一词。单词[“company”在语义上与“developers”](http://www.google.com/search?sourceid=navclient&ie=UTF-8&rls=GGLG,GGLG:2005-37,GGLG:en&q=%7Edevelopers)联系在一起，单词“company”出现在 zend.com 的 216，000 个页面中。因此，来自 Zend.com 的链接大大提高了 php.net 在短语“PHP Web developers”上的排名。

你可以看到一个网站的关键词声誉是如何让它有能力提升另一个网站在类似主题上的排名的。Zend.com 的反向链接显示了来自强大的“权威类型”网站的链接，如[www.infoworld.com](http://www.infoworld.com)、[www.phpindex.com](http://www.phpindex.com)和[www.phpindex.com](http://www.phpindex.com)。查看这些网站上指向 Zend 的链接，你会看到频繁使用的关键词“PHP”和/或“Web”和/或“developers”，以及与这些词语义相关的词。

在某种程度上，你在谷歌工具栏中看到的 Pagerank 将几乎无关紧要，因为排名的实际决定因素将隐藏在一个巨大的数学方程中，其中包括主题敏感的 Pagerank。

##### 收集链接信誉

当谷歌机器人抓取网页时，链接信誉被收集和存储。由于积累的链接信誉数据量如此巨大，谷歌需要一种方法来综合所有这些数据。它通过传递关键字短语的简化版本来做到这一点，并且“链接信誉数据位”连同成千上万的其他链接信誉数据位一起通过可能数百个站点连续传递。这就是为什么从一个或两个关键字短语的高声誉网站(如计算机)获得链接比从三个或四个关键字短语的高声誉网站(如廉价的笔记本电脑)获得链接更好。一个或两个关键字的链接信誉值在 Web 上更为普遍，更多的两个关键字的特定值将被传递到您的站点上。

##### 结论

话题敏感的 Pagerank 计算非常复杂，涉及到链接锚文本和链接页面本身的关键词内容。该系统使用关键字以及同义词和相关词。这也可能涉及谷歌分配网站的信任等级因素。尽管谷歌工具栏的 Pagerank 显示被认为排名靠前，但事实是谷歌分析的是指向特定页面的链接。当它们在主题上时，这些链接将高水平的主题敏感的 Pagerank 传递给目标页面。然后，网页会将该特定短语的搜索结果排在最前面。

在用于改进搜索查询结果排名的原始 PageRank 算法中，使用 Web 的链接结构来计算单个 PageRank 向量，以捕捉网页的相对重要性，而与任何特定的搜索查询无关。
