# 采访恶棍——詹姆斯·爱德华·格雷二世

> 原文:[https://www . site point . com/interview-rogues-James-Edward-gray-ii/](https://www.sitepoint.com/interviewing-rogues-james-edward-gray-ii/)

## ![jeg](../Images/7efe1528d33f072669ad555347197d6a.png)Ruby 知识库

每周都有一群 Ruby 爱好者聚集在一个虚拟的圆桌会议上，探索/讨论/争论 Ruby 语言或 Ruby 社区的某些方面。他们允许我们其他人通过 Ruby Rogues 播客收听他们的讨论。该播客可以在 iTunes 或网站上访问，并提供了一个“墙上的苍蝇”视图，让一大群精通技术的人分享他们的想法和意见。

在这一系列的文章中，我将分享我对这个特殊群体的每个成员的采访结果。

詹姆斯，我想问的第一件事是告诉我们一些关于你自己的情况。

詹姆斯:我是一名程序员。那是我的日常工作。我主要使用 Rails 来构建 web 应用程序。我在这个社区已经很久了。你可能会说，我是在 Ruby 第一个时代的末期来到这里的。第一个时代，在 Rails 登陆之前，我们只有少数人在玩这种语言。突然间，Rails 变得流行起来，我的一项爱好技能变得很有价值。

那是多久以前的事了？

Ruby，实际上作为一种语言，从 1995 年就已经存在了。但是当时在日本以外没有多少人玩它。[迪夫·托马斯](http://en.wikipedia.org/wiki/Dave_Thomas_(programmer))发现了它。我不记得最初的[鹤嘴锄书](http://pragprog.com/book/ruby/programming-ruby)是什么时候出版的，但我怀疑是在 21 世纪初。我想，我是在 2004 年夏天进入这个社区的。

**Thom:** 与我们许多人通过 Rails 发现 Ruby 不同，您实际上是反过来发现的。在 Rails 出现之前，你首先发现了 Ruby，对吗？

詹姆斯:是的。Rails 开发的时候，我还在玩 Ruby。

透姆:那么你对 Ruby 做了什么？桌面开发和脚本？

詹姆斯:我主要把它用作一个脚本环境。在我当时工作的一个大项目中，我们到处都在使用 Perl 来进行大量的数据操作。这是一家拥有庞大产品基础的公司。他们会将产品转换成报告、静态网站等等。我们只是用 Perl 完成了所有这些。然后我开始在那些类似的应用程序中使用 Ruby。最终，在 Rails 流行起来之后，我开始尝试并弄明白它。我就这样爬上了梯子。所以你是对的，我是从另一条路来的。

**Thom:** 这很有趣，我认为这可能(正如我所说的)不是常态。我们中的许多人发现了 Rails，它把我们带到了 Ruby。

我问的另一个问题是[查尔斯·麦斯武德](https://www.sitepoint.com/interviewing-rogues-charles-max-wood/)，尽管我是从他那里得知这个故事的，我还是想听听你对此的看法；[红宝石盗贼](http://rubyrogues.com/)是怎么开始的？

詹姆斯:这是个有趣的故事。我听了另一个播客，是关于一个完全不相关的主题的小组讨论。我只是一直喜欢这种形式，在这种形式中，几个人相互交谈，讨论想法，而不仅仅是单独的采访。我不记得是我还是[查克](https://www.sitepoint.com/interviewing-rogues-charles-max-wood/)最初发微博说，“我想看看关于 Ruby 的小组讨论。”但是他和我同时都在想同样的事情。所以我们中的一个人发了微博，然后另一个人说，“我也一直在想这件事。”

这就是查克和我接触的原因。我们成了前两个。我们开始充实它，然后播客的原始成员都是我们的朋友；亚伦·帕特森、彼得·库珀和大卫·布雷迪只是我们认识的人，我们问他们是否愿意这样做。他们做到了，这就是我们最初把它放在一起的方式。

然后，随着时间的推移，人们来了又走，我们选定了今天这个团队。

**Thom:** 你和 [Chuck](https://www.sitepoint.com/interviewing-rogues-charles-max-wood/) 在推特上互相关注，你们是在一次会议上认识的还是怎么认识的？

是的，我想我们那时见过面。我想我们在一次会议上见过。我去参加了在犹他州盐湖城举行的 [MountainWest Ruby 会议](http://mtnwestrubyconf.org/)。我可能在 2009 年去过那里。我在那里发言。我想我的妻子也做了一个小报告。

Thom: 你的妻子是技术人员吗？她也是程序员吗？

是的，她为我工作了几年，做 Rails 开发，但是现在我们有了一个小宝宝，她花了更多的时间到处追逐她。

汤姆:我明白。但是我很羡慕！

詹姆斯:她仍然在业务的非编码方面提供帮助，但年轻的她在编码方面有点吃不消。

正如我所说，我想我是在[mountain west Ruby Conference](http://mtnwestrubyconf.org/)上认识 [Chuck](https://www.sitepoint.com/interviewing-rogues-charles-max-wood/) 的。那时我已经见过了所有最初的[流氓](http://rubyrogues.com/)，除了彼得。然后乔希和阿夫迪上场了，我没有见过他们两个。

**Thom:** 你提到了一些现在在 Ruby on Rails 社区中家喻户晓的名字，但我猜当时他们可能不是。他们只是来参加一些会议并发表演讲的人？

詹姆斯:这是个好问题。我不确定那个时候 [Aaron Patterson](http://tenderlovemaking.com/) 对 Rails 做了多少，可能是一些。但是他已经因为《T4》和《T5》等电影而出名。在那之前，他已经制造了许多工具。人们就是这样认识他的。彼得·库珀那时已经有了他的 Ruby 每周邮件列表，但没有他今天的庞大帝国。

是的，你可以说，“我认识他的时候……”

詹姆斯:对。我认为这是导致他们两人最终向前看的原因，因为他们现在的时间承诺要大得多。

我想是 Aaron，当他有一周休息的时候，他说，“嘿，我们应该让 Josh 上节目，”我们试着让他上。有人提到了 Avdi，或者我知道 T2 的 Avdi，我们有过几次这样的经历。这非常合适，我们都很喜欢。所以，随着其他人的离开，我们让他们全职工作，从那以后，这个团队就变得很棒了。

**Thom:** 我认为 [Chuck](https://github.com/cmaxw) 在讲述 [Katrina](https://github.com/kytrinyx) 的故事时说得很好，她的客串很受欢迎，你们和她相处得很好，以至于在讨论中根本没有足够的空间容纳另一个人，每个人都举手说，“好吧，我放弃我的位置！”

是的，事情大概就是这样发生的。

Katrina 非常谦虚，我真的读过[你对她的采访](https://www.sitepoint.com/interviewing-rogues-katrina-owen/)，她称自己是初学者之类的。如果你相信这一点，你就不是《T4》节目的忠实听众。

她拥有擅长代码的罕见品质——能够思考代码之类的东西——还能够解释代码。这是我最喜欢她的一点。她可以用一种非常简单的方式描述一些东西。我认为这在程序员中很少见。我认为我们不擅长交流我们的想法是出了名的。

汤姆:伙计，你说得完全正确！

我知道作为一个听众，我对她印象深刻。卡特里娜似乎不常说话，但当她有话要说时，通常很有见地。

是的，她和[阿夫迪](http://avdi.org)都很安静。他们不会跳进去多管闲事。我是最大声的那个。我是那个他们永远无法闭嘴的人，对吗？他们不怎么说话，但每当他们开口问问题时，都会很有见地，通常会引发一场精彩的讨论。

你也看不到人们在幕后做了多少工作，尤其是卡特里娜飓风。她曾经是一名转录员，因此，当我们坐在那里谈话时，我们倾向于使用许多可怕的东西，如首字母缩写词或奇怪的词，如“Threequel Operator”或其他会让转录员发疯的东西。[卡特琳娜](https://github.com/kytrinyx)正坐在后面的频道里，按我们通常所说的那样打出来。

她在那方面给了我们很多支持，最近她也在预定节目和其他东西。她和[内尔·沙姆瑞尔](http://www.nellshamrell.com/)一起在正则表达式上建立了[节目，这真是一集很棒的节目。所以即使她不提问，她也会帮很多忙。但我倾向于享受休假一周的假期。没有我总是插嘴，我想其他人会说得更多，我总是喜欢这些插曲。](http://rubyrogues.com/105-rr-regular-expressions-with-nell-shamrell/)

**Thom:** 所以[红宝石盗贼](http://rubyrogues.com/)已经跨越了 **100 集**，这是一件相当大的事情。

詹姆斯:是的，我们很惊讶。

Thom: 是不是感觉已经过了 100 岁，或者在某些方面，感觉过得更久？

詹姆斯:你知道，我想当我们达到 100 集的里程碑时，我们都很惊讶，我们已经走了这么久。就在几天前，我们意识到我们的邮件列表已经有一年的历史了，这令人震惊。

**Thom:** 哦，好大的合流。

詹姆斯:我想我们都对此感到惊讶。我敢肯定，当筋疲力尽的时候，或者我们觉得我们已经谈论了所有我们想谈论的事情的时候，总有一天会到来。但是我可以告诉你，从我们目前的清单来看，我们有很多事情想谈，但是还没有谈。我的意思是，就像你说的，我们已经有 100 集了，正则表达式是我在 Ruby 中最喜欢的东西之一，我们刚刚做了这一集。

汤姆:太令人兴奋了。

詹姆斯:我也对做这个感到兴奋，尽管我认为它有更多的问题，像往常一样，我说得太多了，因为这是我最喜欢的主题。

**Tom:**您一直是 Rails 的忠实贡献者，自从您提到您在 Rails 出现之前就已经涉足 Ruby，对一个已经非常重要并得到广泛认可的框架产生如此大的影响，您感觉如何？

**James:** 事实上，我只对 Rails 做过几次小的贡献。我已经做了很多教学方面的工作，但是就源代码本身而言，我只做过一些小的提交。
[乔希](http://blog.hasmanythrough.com/)，就贡献而言，其实做的比我多不少。

Thom: 好吧，我误解了，但是你写了很多宝石。

詹姆斯:我写了很多宝石，我不知道有多少最终有用…

我们都有这种感觉。

**詹姆斯:**对，对。 [FasterCSV Gem](http://fastercsv.rubyforge.org/) 可能是我用过的最成功的一个，它最终成为了 Ruby 1.9 等版本中的 [CSV 标准库。](http://www.ruby-doc.org/stdlib-2.0/libdoc/csv/rdoc/index.html)

当然，这是值得骄傲的事情。

是的，我很高兴看到这一点。我在格雷格·布朗的帮助下写的 [HighLine Gem](http://highline.rubyforge.org/) ，我相信它仍然被广泛使用。这可能是我写过的影响最大的两篇。

**Thom:** 你之前提到过，我知道这是一种自嘲式的幽默，但是你提到过你总是在说话和插嘴。但是说实话，你认为你在这个有着不同观点的多元化团队中的角色是什么？你是否认为自己在其中提供了某种特定的角色或指导？

**詹姆斯:**[乔希](https://github.com/joshsusser)曾经在电视节目中扮演各种角色——[电视节目](http://tvtropes.org/)网站有一个电视节目叫做“[五人乐队](http://tvtropes.org/pmwiki/pmwiki.php/Main/FiveManBand)”,之后的一个星期，我们都在试图找出哪个角色是我们。

但说真的，我认为我在[盗贼](http://rubyrogues.com/)中带来的东西之一并不是太多技术之类的东西，我认为其他人在这方面比我更好。[乔希](https://github.com/joshsusser)尤其有很强的 Rails 背景， [Avdi](http://avdi.org) 对模式之类的东西非常在行。卡特里娜，伟大的沟通者，就像我之前说的。大卫比我有趣，查克是使这一切成为可能的粘合剂。

但是我对 Ruby 的各个方面有着几乎百科全书般的了解，这主要来自于三年来运行的 Ruby 测试。每个星期，人们都会通过电子邮件将所有这些程序发给我，我必须通读它们，并充分理解它们，这样我才能在那个星期写的文章中谈论它们。解决编程问题的人使用了书中所有肮脏的元编程技巧。所以，在我监督那次比赛的时候，我了解了 Ruby 和 API 的许多黑暗角落。我不知道我脑子里实际上有多少，但这是一个非常高的百分比。

Thom: 太好了。

**詹姆斯:**只是因为我已经玩了很多次，必须弄清楚这些东西。当我们做[正则表达式讨论](http://rubyrogues.com/105-rr-regular-expressions-with-nell-shamrell/)时，我们一度陷入为什么替换字符串`sub`和`gsub`的恐惧之中。为什么我必须开始加倍我的反斜杠，而最后我得到了四个或八个或其他什么？我就像，“哦，那是因为，”你知道，我开始解释这是为什么。

我想这可能是我带来的东西。我了解 Ruby 及其工作原理。

Thom: 最后一个问题。有没有什么你目前正在研究的东西是你想推广或者告诉全世界的？

詹姆斯:这是个好问题。我是那种从不为想法伤脑筋的人。我每天醒来都有更多的想法，我把这些想法保存在我的电脑里。在任何给定的时间，我总是在做 3-500 个。

**Thom:** 那是 3 比 500 而不是 300 比 500？

詹姆斯:对，差不多是这样。我肯定我妻子会用“太多了”来形容它。我从来没有因为想法而受伤，但是，有些变成了其他的东西，有些我最终用在了工作项目上，有些我因为各种各样的时间原因放弃了，或者意识到这是个坏主意或者其他什么。我不知道哪些最终会成为现实。我想我们都有这个问题。有几件我正在做的事情，我希望它们能像我想的那样酷，但我不认为我已经准备好分享它们了。

我将说一件我们正在做的事情，这是听众感兴趣的。我们在过去提到过很多次，由于各种原因，我们有很长一段时间没有谈论它，主要是让它全部工作，但我们正在编写 [Ruby Rogues](http://rubyrogues.com/) 的第一本书，这将是“Ruby 最佳实践模式”，这是来自 [Kent Beck](http://en.wikipedia.org/wiki/Kent_Beck) 的 [SmallTalk 最佳实践模式](http://www.amazon.com/Smalltalk-Best-Practice-Patterns-ebook/dp/B00BBDLIME/ref=la_B000APC0EY_1_5?ie=UTF8&qid=1369437373&sr=1-5)的 Ruby 移植。

汤姆:哦，这是写一本书的好主意。

我想说的是，作为[红宝石流氓](http://rubyrogues.com/)中的一员，我感到非常幸运每周能和乔希·苏斯、[阿夫迪·格林](https://www.sitepoint.com/interviewing-rogues-avdi-grimm/)、[大卫·布雷迪](https://www.sitepoint.com/interviewing-rogues-david-brady/)、[查尔斯·麦斯武德](https://www.sitepoint.com/interviewing-rogues-charles-max-wood/)和[卡特琳娜·欧文](https://www.sitepoint.com/interviewing-rogues-katrina-owen/)这样的人在一起。

**Thom:**James，感谢你抽出时间为我们提供见解。

詹姆斯·爱德华·格雷二世可以找到，除了每周播出的[红宝石流氓](http://rubyrogues.com/)，
