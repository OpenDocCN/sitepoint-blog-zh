**CSS 网站设计**

任何网站的发展都是从设计开始的。通常，你会从你的客户那里得到一份声明，或者至少在你的头脑中有一个粗略的想法，关于网站的预期功能。如果你是一个照章办事的开发人员，这甚至可能采取详细规范文档的形式，它可能描述站点需要支持的用例(即访问者可以做的事情)，站点必须遵守的官方规范和建议，以及必须能够访问站点的浏览器和平台的列表。

在这个阶段，设计师通常会创建一系列模型，从纸上草图，到图形应用程序中的原型设计，再到 HTML 中的实际网页。如果你有一些传统网站设计的经验，你可能甚至在制作你的第一张草图的时候就已经想到了最终会在屏幕上复制这些布局的 HTML 代码。

当您从使用表格转向使用 CSS 作为主要的页面布局工具时，您将不得不学习一套全新的设计原则，作为您最初的模型的基础。在本书的这一章和接下来的几章中，我将通过这些原则来指导你，这样你就可以掌握新的限制，并让你的想象力在新的可能性中驰骋。

抗拒改变是人的天性。当你遇到 CSS 做不到的事情时，你会倾向于紧紧抓住基于表格的设计所提供的沉重的控制，而不是勇敢面对 CSS 布局的新世界，在那里一百个页面的布局可以依赖于一个规则。在这一章中，我将通过解释基于 CSS 的设计的一些“大图”优势来努力哄你走出舒适区，并展示一些已经冒险并正在收获 CSS 布局回报的网站的成功故事。

##### CSS 设计的优势

在过去的几章中，我已经谈到了使用 CSS 进行网站布局的强大功能和原因。在这一节中，我将形式化这些论点，并把它们全部集中在一个地方。我不仅希望让你相信 CSS 的优点，还希望给你一些工具，让你可以向别人推销这项技术。

在残酷的自由职业网站开发世界中，你经常会被要求解释为什么你会比其他投标相同项目的开发者做得更好。如果 CSS 布局是你的网页设计工具之一，你建立的网站将会从这里的优势中获益。这些优势不仅仅是易于开发，而且直接转化为客户的额外价值。让他们知道这一点——这可能会决定是赢得合同还是输给一个以桌面设计为生的设计师。

***增加了文体控制***

CSS 最明显的卖点，也是大多数 Web 开发人员首先选择涉足该技术的原因，是它让您可以控制站点外观的许多方面，而这些方面您用纯 HTML 根本无法控制。例如，由于 CSS 的引入，从超链接中移除下划线并用其他一些样式区分(如粗体或彩色文本，或者当鼠标悬停在链接上时添加下划线)来指示它们的时尚正在消退。有关可以用 CSS 控制的样式属性的完整参考，请参见附录 c。

除了数量众多的可控样式属性之外，CSS 还允许您将它们更统一地应用于可用的 HTML 页面元素范围。以 HTML 为例，如果您想在页面的某个区域周围放置一个可见的边框，您需要使用表格来实现，因为您只能给表格添加边框。CSS 不仅能让你更好地控制边框的外观(实线、浮雕、点线或虚线、粗细、红色或绿色等)。)，还允许您向任何页面元素添加边框，而不仅仅是表格。CSS 背后的设计理念是给设计者尽可能多的选择，因此任何存在的样式属性通常可以应用于任何可能有意义的地方。

CSS 比 HTML 提供了更多的样式属性，可以应用于更多的页面元素。如果你不得不在 CSS 和 HTML 之间选择一种方式来指定你的网站的设计，仅仅基于哪一种能给你提供最直观的控制，CSS 会轻而易举地获胜。尽管如此，通常的做法是尽可能使用 HTML 进行设计，并在需要 HTML 无法产生的效果时求助于 CSS。虽然用这种基本原理设计的网站的视觉外观也一样好，但是你错过了 CSS 的所有其他优点。

***集中设计信息***

正如我已经解释过的，在网站设计中使用 CSS 的最好方法是编写一个或多个。css 文件来存放你所有的样式代码，然后用 HTML `<link>`标签把它链接到适当的页面。使用这种方法，所有与网站外观相关的东西都可以在一个地方找到，而不会与网站内容混在一起。

这个想法是你应该能够改变你的网站的内容而不影响它的外观，反之亦然。在传统的网页设计中，HTML 标签和属性被用来指定事物在浏览器中的外观，这两个方面的代码是混合在一起的，所以任何想修改其中一个的人都必须理解这两个，否则在修改另一个的时候会有破坏其中一个的风险。据说网站的外观和内容是结合在一起的。

这种将服务于不同目的的代码保存在不同位置的原则在编程界被称为解耦。如果站点的样式和内容是分离的，Web 设计人员可以通过编辑。css 文件，而内容编辑者可以通过编辑。html 文件。

比促进组织和团队合作更重要的是，这种代码分离减少了代码重复。在基于 HTML 的设计中，如果你想让网站上每篇文章的标题以红色大字体显示，你必须在网站上每篇文章页面的相关`<h1>`标签内放置一个`<font>`标签。使用基于 CSS 的设计，你可以在一个地方指定`<h1>`标签的字体属性，这样可以节省你的输入。如果你决定改变这些标题的外观，你只需要修改 CSS 文件，而不是每一个 HTML 文件，这样你就不会糊涂了！这些差异如图 4.1 所示。

***语义内容标记***

当你使用。css 文件来分离你的网站的内容和外观，正如我刚才所描述的，奇怪的事情开始发生在你的 HTML 代码上。因为 CSS 让你可以完全控制页面元素的外观，你开始选择标签是因为它们描述了页面元素的*结构*和*含义*，而不是你想要它们看起来是什么样子。除去大部分或所有的表示信息，HTML 代码可以自由地捕捉站点内容的语义。

有很多原因可以解释为什么这是一种令人满意的状态，尤其是当你对网站内容进行修改时，你可以很容易地找到一些东西。识别 CSS 网站最简单的方法是使用浏览器中的“查看源代码”功能——如果你能在 10 秒钟内弄懂那里的代码，很可能你面对的不是一个使用基于表格的布局和其他非语义 HTML 的网站。

语义 HTML 极大地帮助了搜索引擎优化(SEO ),因为搜索引擎在分析您的网站时需要处理的表示性标签越少，您网站的关键字密度就越大——这是决定您网站排名的一个重要指标。正如我们将看到的，CSS 允许您控制元素在浏览器窗口中的位置，很大程度上独立于它在 HTML 文档中的位置。所以，如果你有一个新闻订阅表单或者其他一些对搜索引擎来说不太重要的冗长的 HTML 代码，可以随意把它的代码移到你的 HTML 文档的末尾，并使用 CSS 来确保它显示在浏览器窗口的顶部。

现代浏览器越来越多地支持 HTML `<link>` tag1 的一个特性，它允许你限制一个链接的样式表，这样当它被某种类型的浏览器或显示器显示时，它只影响页面。例如，你可以连接三个。css 文件到页面:一个文件定义页面在桌面浏览器上的外观，另一个文件规定页面打印时的外观，还有一个文件控制移动设备(如联网的个人数字助理(PDA ))上的显示。只有通过使用语义标记，并允许 CSS 处理显示属性，这种内容再利用才是可能的。

最后，但肯定不是最不重要的，是网站通过使用语义标记可以获得的可访问性的巨大改进。我们将在下一个
部分详细讨论这些。

***可达性***

如果你有机会观察一个有视觉障碍的人浏览网页，我强烈推荐这种体验。或者，给自己买一些屏幕阅读器软件，关掉显示器，自己看看是什么样子。

当体验一个网站最自然的方式是自上而下地听它大声朗读时，大量设计的网站使用表格、图像和其他非语义 HTML 进行布局是非常难以使用的。在实际内容开始之前，现代网站经常会出现 30 秒或更长时间的嵌套表格打开和关闭、未标识的布局图像以及其他无意义的内容。现在，如果你认为这听起来有点烦人，想象一下你访问的网站的每一页都要听它！

语义 HTML 几乎消除了这种听觉垃圾，因为它确保了文档中的每个标签都具有对查看者(或收听者)有意义的结构意义。听觉浏览器忽略了 CSS 中定义的视觉格式属性，所以用户不必费心去听它们。

例如，在使用语义标记的网站上，有视觉障碍的用户永远不会怀疑一个单词是否因为更重要而加粗，或者仅仅因为这样看起来更好。出于设计原因以粗体显示的元素将使用 CSS 分配该属性，而听觉浏览器永远不会提到它。需要额外影响或强调的元素将使用语义上有意义的`<strong>`和`<em>`标签进行标记，这些标签在可视浏览器中默认显示为粗体和斜体。

有一套完整的指导方针适用于那些希望让他们的网站对残疾用户更容易访问的开发人员。建议所有 Web 开发人员阅读 Web 内容可访问性指南 1.0 (WCAG)，指南 3 处理了避免表示性标记而支持语义标记的想法。

***符合标准***

WCAG 并不是提倡使用 CSS 作为 HTML 文档表示属性的唯一规范。事实上，最新的 HTML 标准本身就是考虑到这一点而编写的！

万维网联盟(W3C)是负责发布与 Web 相关的建议(事实上的标准)的机构。以下是 W3C 关于使用语义标记和 CSS 的一些建议:

HTML 4
[https://www.w3.org/TR/html4](https://www.w3.org/TR/html4)

HTML 推荐标准的最新(也是最后的)主要修订版将所有非语义元素和属性标记为不推荐使用 2。例如，`<font>`标签在本标准中被明确标记为不推荐使用。在不赞成使用的元素的描述下，该建议是这样说的:

一般来说，作者应该使用样式表来实现风格和格式的效果，而不是 HTML 的表现属性

XHTML 1.0
[https://www.w3.org/TR/xhtml1/](https://www.w3.org/TR/xhtml1/)

作为 XML 文档类型的 HTML 4 的重新表述，XHTML 允许您使用 HTML 标记和属性，同时还受益于 XML 的特性(混合标记语言、自定义标记等)。).

该建议包括与 HTML 4 相同的标签和反对意见。

网页内容无障碍指南 1.0
[https://www.w3.org/TR/WCAG10/](https://www.w3.org/TR/WCAG10/)

如“可访问性”一节所述，WCAG 建议强烈推荐在网页设计中使用 CSS 和语义标记来提高可访问性。我会让推荐说明一切:

*“为了呈现效果而误用标记(例如，使用表格进行布局或使用标题来改变字体大小)使得具有专门软件的用户难以理解页面的组织或在其中导航。此外，使用表示标记而不是结构标记来传达结构(例如，使用 HTML PRE 元素构建看起来像数据表的内容)会使其他设备难以理解地呈现页面…”*

许多 Web 开发人员认为，严格遵循标准是一个理想化的目标，很难实现。这本书的主要目标之一是证明这不再是真的。今天的浏览器为 CSS 提供了强大的支持，当它们被提供符合标准的代码时，会产生更加一致的结果。虽然错误和兼容性问题仍然存在，但它们并不比依赖不兼容代码的设计人员所面临的错误更难以克服。
