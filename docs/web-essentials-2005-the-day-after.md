# Web Essentials 2005:后天

> 原文：<https://www.sitepoint.com/web-essentials-2005-the-day-after/>

在我和 Web Essentials 2005(WE05)之间度过了一个非常需要的放松和睡眠的周末，我现在可以事后回顾这段经历，看看我带走了什么。

我从会议中得到的主要好处是接触到一些致力于让网络变得更好的最杰出和最有影响力的思想家。像[坦泰克·切利克](http://tantek.com/)、[莫莉·霍尔茨施拉格](http://molly.com/)、[杰弗里·维恩](http://www.veen.com/)、[埃里克·迈耶](http://www.meyerweb.com/)和[道格拉斯·鲍曼](http://dbowman.com/)这样的人都很容易接近，脾气也很好，既扮演主持人，也扮演观众。瞥见他们正在解决的问题，以及他们认为理所当然的事情，为我在自己的工作中努力树立了一个强大的基准。

由于我整天都呆在网上，我已经对所涉及的大多数技术主题有了很强的理解。我原本希望在前沿的、实验性的工作方面多一点，但呈现的水平更接近“艺术状态”，或“最佳实践”。结果，我在会议上没有多少纯教育的机会；但是对于那些对 AJAX 和 T2 微格式一无所知的人来说，这次经历将会给他们真正的启发。

我从这次会议中得到两个深刻的印象。第一个是在网页设计者的日常考虑中作为一等公民的可访问性的愿景。我从来没有实际见过屏幕阅读器，更不用说用它来填表了，我发现我对可访问性专家今天所做的一些工作感到困惑。

我承认，我原以为专业的可访问性专家只不过是“标准检查员”，他们会对 WCAG 的指导方针指手画脚，并对过高的咨询费嗤之以鼻。相反，我遇到了许多务实的、有技术头脑的、精通代码的专家，并向他们学习，他们整天都在探索 W3C 指南和现实世界的辅助技术(如屏幕阅读器和屏幕放大器)之间的区别。最好的例子应该是[的德里克·费瑟斯通](http://boxofchocolates.ca/)令人印象深刻的、可访问的纵横字谜，这是 CSS 和 DHTML 魔法的一个壮举——来自一个可访问性专家！

总之，这是令人振奋和大开眼界的两天。如果说我从 WE05 中学到了什么，那就是对自己盲点的认识。通过从各个角度全面审视网页设计的艺术状态，我立即知道我必须回家学习什么，这样才不会落后。物有所值吗？是的，但也许不是因为你能向雇主证明的原因。
