# phpDay 2013:新闻和亮点

> 原文：<https://www.sitepoint.com/phpday-2013-news-and-highlights/>

在成为更好的开发者的过程中，会议扮演着重要的角色。当你参加一个会议时，你不仅有机会聆听语言或技术专家的演讲，还有机会认识其他开发者。如果需要的话，这对于拓展你的人际网络来寻找下一份工作是非常理想的。

上周，我参加了在维罗纳举行的 2013 年 phpDay 会议，以下是我参加的会议和我学到的一些重要内容。

## 第一天

会议以 Julien Pauli 的主题演讲开始，他是将于今年 7 月发布的 PHP 5.5 版本的发布经理。他首先关注开发人员如何通过开发特性和报告错误来为 PHP 的开发做出贡献。然后，他回顾了下一个主要版本中的新特性。

5.5 中的一个预期特性是新的密码哈希 API。API 的目标是帮助程序员正确存储密码(即加密！)，并提供了以非常简单的方式生成和验证加密密码的方法，因此您将不再需要生成和存储 salts。默认情况下，API 使用 Bcrypt 生成散列，因此解密所需的所有信息都存储在字符串本身中。虽然是一个重要的概念，但我不禁质疑这个 API 的效用；它可能很容易使用，但是如果一个开发人员不关心使用 salt 存储加密的密码或者使用 MD5，他会关心使用这个 API 吗？你有什么想法？

介绍的另一个特性是引入了用于异常处理的`finally`子句。我很高兴看到这一点，因为我一直不明白为什么它最初没有与 try/catch 一起引入。它帮助开发人员以适当的方式管理不同的情况。

我想从他的演讲中提到的最后一点是对 OPCache 的介绍。真的是除了 Zend 的 ZendOptimizerPlus 之外什么都没有了，被解放了，改名了。Pauli 说，它将随 PHP 5.5 一起发布，并作为一个必须在编译时激活的扩展发布。他指出，从这个版本开始，你应该认为 APC 已经死了。

在泡利的主题演讲之后，我参加了超级巨星 Jordi Boggiano 的演讲，他是 Composer 背后的人(更多关于 Composer 的依赖管理)。他的演讲集中在一些 Composer 用例上，例如如何使用分叉项目作为项目的依赖项，以及如何安装/更新项目的依赖项。

Composer 已经在 SitePoint 上讨论了几次，所以我将继续，但我想分享一下我问 Jordi 的项目是否会有自动更新功能(针对 Composer 本身，而不是项目)。他的回答是一个大大的否定。他说很多用户可能不会满意。我不知道为什么人们没有兴趣尽快知道一个修复了错误并具有新功能的新版本何时可用。你有什么想法？

第一天我想讨论的最后一个话题是 Volker Dusch 的“从开发到运营及其他——完成工作”。这是第一场讲座中我最欣赏的演讲，因为它的内容和 Volker 的精彩演讲，他讲述了 DevOps 方法和他的经验。我真正欣赏的是他给了我一个新的视角，让我了解软件是如何完成和部署的。他谈到了与公司中不同角色的人沟通对于产品成功的重要性。此外，他建议了很多工具，你可以开始使用它们来改进你的开发过程，比如 Jenkins、Trello、Graphite、Puppet 等等。

## 第二天

第二天的精华从 Jonathan Klein 和他的演讲“高性能 PHP”开始，他给出了几个加速 PHP 应用程序的技巧和诀窍。在演示期间，他展示了几种语言和框架的[性能比较表](http://www.techempower.com/benchmarks/)(这是我几天前在这个网站的评论中看到的，high five for us！).他首先讨论了为什么我们不应该真正关心微优化，甚至是不成熟的优化。然后，他教了我这个非常重要的教训，我想与你分享:如果一个网站在大约 2 秒钟内加载，就被认为是快速的，但鉴于 80%的加载时间是在客户端，你只有 400 毫秒的时间来从你的服务器端进程中输出所有代码。网络时间大约是 100 毫秒，所以你只有**300 毫秒**来完成你的处理！

他还教授了如何使用 JMeter 等工具分析代码，以及如何使用 APC 和 Memcache 进行缓存。让我印象深刻的是，简单地升级你的 PHP 版本有时可以获得高达 70%的速度提升，另外通过使用操作码缓存还可以获得 30-40%的速度提升！

Luka Kladaric 展示了“LAMP Scaling 101 ”,这是针对任何达到单个服务器(VPS 或类似服务器)极限并面临从该服务器扩展的挑战的人的。我很喜欢他的演讲，因为它是用现代工具 reveal.js 制作的，字数很少，所以人们可以专注于他的话。Luka 的一些建议是:将静态内容与动态内容分开，并使用 CDN 为前者服务；PHP 内容的缓存；扩展数据库(复制、集群、分片等)；和扩展存储。

第二天我想提到的最后一个演讲是埃利奥特·林德的演讲，他谈到了 FBMock，这是一个由他工作的地方脸书创建的开源 PHP 模仿框架。他描述并展示了一些例子，说明与 PHPUnit 等其他框架相比，它如何简化模拟的创建。这似乎是一个非常有前途的项目，但我不能给它一个诚实的评估，因为我没有尝试过它，也没有太多的嘲笑经验。

没有足够的空间来写，但也值得一提的是 Richard Tuin 与 Behat 和 Mink 关于自动化验收测试的谈话，以及 Jacopo Romei 的最后主题演讲。后者强调学习和自我提高的重要性，不仅与编程技能有关，还与开发人员对待工作和生活的方式有关。

## 最终注释

我发现令人惊讶的是，虽然世界其他地方都在拒绝二维码，但在会议上我看到了很多二维码。这很有趣，因为网站[www.wtfqrcodes.com](http://wtfqrcodes.com)很好地总结了普遍观点。

但作为结论，我想给出我对演讲幻灯片的看法。尽管许多主题都非常有趣，但似乎没有像选择和研究主题那样精心准备幻灯片。所有的演示都只有很少甚至没有图像。此外，除了几个案例，他们完全缺乏幽默的图像，而这些图像对于让用户暂停 5 秒钟并让他们与演讲者建立联系是非常重要的(更多关于[好的演示](http://www.slideshare.net/bigfishpresentations/5-killer-ways-to-open-up-your-next-presentation?sf13079690=1))。在 HTML5 和 JS 库占主导地位的时代，我认为不应该再有 90 年代风格的带有项目符号的演示了。

当然，这篇文章只是对我在 phpDay 2013 上的经历和学习的一个简短回顾，但是希望它能让你了解一些你应该知道的主题，以及你可以从哪些方面着手来提高你的 PHP 技能，甚至激发你参加附近的一个会议！
