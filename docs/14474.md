# 用户模式 Linux 第 0 天

> 原文:[https://www.sitepoint.com/user-mode-linux-day-0/](https://www.sitepoint.com/user-mode-linux-day-0/)

好了——终于找到了托管的[用户模式 Linux](http://user-mode-linux.sourceforge.net/) (UML)帐户。已经选择了一个提供商，并且车轮已经开始转动…

为什么是 UML 而不是专用服务器？首先，“经济学”通常看起来更好。此外，在虚拟服务器中，我应该是真正的“上帝”——可以安装我想安装的任何东西——它毕竟是虚拟的——当你是一个控制狂时，这是一个好消息。而且(理论上)它更稳定——在出现故障的情况下，应该可以很容易地将虚拟服务器从一台物理机转移到另一台物理机(当然，主机需要提供这种功能)。

有趣的是，与你典型的共享主机账户相比，你面临着更多的决定，而不仅仅是“需要多少磁盘空间/带宽？”。你还需要考虑多少内存(我打算从 64Mb 开始，这应该够用了，但我怀疑对于一个需要大量流量的网站来说是否足够)和其他问题，比如你想要什么样的 Linux 发行版。

除此之外，看看我中等的 Linux 技能是否足以管理一个用户模式的 Linux 帐户将会很有趣。我们将会看到…

发行版方面，可能会以 Fedora Core 结束，但你会选择什么(应该可以请求其他发行版)？

我非常了解 Suse，至少在运行我自己的 PC 时是这样，但是 Suse 和 Fedora 在为最新的 PHP 版本提供包时似乎都相当保守(这次我不想编译自己的包),当然我需要 PHP5。我也看不到自己在互联网上运行 X，所以不需要考虑 Gnome / KDE，所以一个更小的 disto 可能更适合。

也许是 Debian，我对它有一点经验。Jason 最近也让我对 Gentoo 产生了兴趣(应该很快就会在家里安装另一台电脑)，这可能是另一个选择。
