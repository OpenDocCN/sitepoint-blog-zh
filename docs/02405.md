# 面向前端开发人员的 20 多份文档和指南(第 9 位)

> 原文:[https://www . site point . com/20-docs-guides-front-end-developers-9/](https://www.sitepoint.com/20-docs-guides-front-end-developers-9/)

又到了该学习的时候了！和以前一样，我收集了许多不同的学习资源，包括指南、文档和其他有用的网站，以帮助您快速了解前端开发的不同领域。

因此，请欣赏我们的文档和指南系列第九部分,不要忘记在任何其他人的评论中告诉我我还没有包括的内容。

## [1。JavaScript 标准样式](http://standardjs.com/)

这主要不是一个学习指南，而是一个模块，您可以通过命令行安装和运行它，根据一组 JavaScript 语法规则测试您的代码。它也可以作为文本编辑器插件使用。然而，作为一个指南，你可以阅读一下[规则分解](http://standardjs.com/rules.html)，这对初学者和其他人来说应该是一个很好的了解一些通用 JavaScript 最佳实践的方法。

[![JavaScript Standard Style](../Images/889f564adf91857eda137887211b8773.png)T2】](http://standardjs.com/)

## [2。Webpack:简介](https://angular.io/docs/ts/latest/guide/webpack.html)

“Webpack 是一个流行的模块捆绑器，它是一个将应用程序源代码捆绑成方便的块并将代码从服务器加载到浏览器的工具。”该指南位于 Angular 官方网站上，因此该指南旨在将 Webpack 与 Angular 2 应用程序配合使用。

[![Webpack: An Introduction](../Images/11762f30de272629bae03bff60d326ad.png)T2】](https://angular.io/docs/ts/latest/guide/webpack.html)

## [3。HTML 元素的听觉 UI](https://thepaciellogroup.github.io/AT-browser-tests/)

"屏幕阅读器如何支持 HTML 元素."由四个数据表组成，涵盖了 Windows 10 上 Firefox 的 JAWS、OSX 上的 VoiceOver 和 Safari 9，以及 Windows 8.1 上的 NVDA 和 Firefox，还将有更多测试。

[![Aural UI of the Elements of HTML](../Images/d2ee846459ff8b53d47b03bf59564d4d.png)T2】](https://thepaciellogroup.github.io/AT-browser-tests/)

## [4。键入术语](https://www.supremo.tv/typeterms/)

这更多的是针对设计者而不是开发者，但是对于那些想要更加熟悉排版术语的人来说，这是一个非常好的设计和有用的交互工具。由总部位于曼彻斯特的设计机构 Supremo 的人制作。

[![Type Terms](../Images/24820268489f9e2c41b0a0f860692097.png)T2】](https://www.supremo.tv/typeterms/)

## [5。电子邮件工具箱](http://email-toolbox.com/)

这是一个广泛的链接资源，主要集中在设计和编码 HTML 电子邮件。各种类别下的许多东西，包括要关注的人、课程、要阅读的博客、工具和电子邮件服务提供商。

[![Email Toolbox](../Images/3c88168a94b60b01b5ab745c4521e782.png)T2】](http://email-toolbox.com/)

## [6。几乎完整的 flexbox 指南(无 flexbox)](https://kyusuf.com/post/almost-complete-guide-to-flexbox-without-flexbox)

有这么多不同的 flexbox 指南和工具，但这里有一点不同。本指南向您展示如何使用传统方法在布局中实现类似 flexbox 的效果。很高兴在这样一篇文章中看到它们，并附有代码示例。

[![Almost complete guide to flexbox (without flexbox)](../Images/874c1684bd71cd2208293cfc9251caa4.png)T2】](https://kyusuf.com/post/almost-complete-guide-to-flexbox-without-flexbox)

## [7。角度 1.x 样式指南(ES2015)](https://github.com/toddmotto/angular-styleguide)

这是 Telerik 的开发者倡导者 Todd Motto 的“团队风格指南”。托德还提供关于 AngularJS 发展的课程。该样式指南“已经针对 ES2015 进行了彻底的重写，Angular 1.5+中的变化是为了将来将您的应用程序升级到 Angular 2。”

[![Angular 1.x styleguide (ES2015)](../Images/caa1bcfc16efac3b6759f6f1cfdbc6f2.png)T2】](https://github.com/toddmotto/angular-styleguide)

## [8。CSS 清除](http://www.csspurge.com/)

这个网站给你统计了流行的网站和框架在高效、可维护的 CSS 方面的排名。它显示 CSS 文件大小、特异性分数和 CSS 属性使用的数据。

[![CSS Purge](../Images/df7f3df6372188afb955ae5fed416ab0.png)T2】](http://www.csspurge.com/)

## [9。谷歌 Chrome 的 CSS 文件](https://chromium.googlesource.com/chromium/blink/+/master/Source/core/css/html.css)

感谢 Umar Hansa [在推特上发布了关于这个](https://twitter.com/umaar/status/735744535179845632)的消息。这是 Chrome 用来为 HTML 元素提供一些基本默认样式的核心 CSS 文件。浏览它很有趣，因为里面肯定有一些奇怪的东西。

[![Google Chrome's CSS File](../Images/6cdd1cfe3726aa485bfd1516e7b04f20.png)T2】](https://chromium.googlesource.com/chromium/blink/+/master/Source/core/css/html.css)

## 10.`<head>`备忘单

"一份可以放入你的文件中的所有东西的清单."我喜欢它，因为它不仅列出了标准元素，还列出了我们许多人看到但可能不完全理解的特定于供应商的内容。

![`head` Cheat Sheet](../Images/cc40d73bb6450d4f0b47785caca0b8ae.png)

## [11。生成器和迭代器](http://greim.github.io/gen/dist/00-intro.html)

由 Greg Reimer 撰写的包含 ES6 发电机的八部分教程。“生成器是现代 JavaScript 中一个不被重视的特性。我认为这是因为生成器只是由生成器和迭代器两部分组成的整体的一半，学习其中一部分而不学习另一部分是不完整的。至少对我自己来说，啊哈！直到我缩小范围，从整体上看迭代器和生成器，时刻才到来。”

[![Generators and Iterators](../Images/d8fadd1bf2e94f9f89e15d7c735b4c77.png)T2】](http://greim.github.io/gen/dist/00-intro.html)

## [12。Node.js ES2015 支持](http://node.green/)

从技术上讲，这是后端，但与 JavaScript 开发人员相关。这是一组适用于 ES2015 的兼容性表，但仅适用于 Node.js。

[![Node.js ES2015 Support](../Images/c12f1d66d9a97d88ba1b1056f1fb43f7.png)T2】](http://node.green/)

## 13。HTML5 可访问性

我在以前的帖子中包含了这个，但它经过了重新设计，看起来很棒。该网站测试主流浏览器支持哪些新的 HTML5 特性。这包括它们是否可通过键盘访问、是否映射到平台可访问性 API，以及是否支持任何与可访问性相关的功能。”

[![HTML5 Accessibility](../Images/d8a4abd5cb1188a7753d244fec4c442a.png)T2】](http://www.html5accessibility.com/)

## [14。MaintainableCSS](http://maintainablecss.com/)

“编写 CSS 时不用担心过分热情、预先存在的样式会导致问题。MaintainableCSS 是一种编写模块化、可扩展，当然也是可维护的 CSS 的方法。”这是亚当·席尔瓦的 12 部分指南。

[![MaintainableCSS](../Images/052456d9b84abc77df12034833b00a83.png)T2】](http://maintainablecss.com/)

## 15。简化的 JavaScript 行话

最初由 Hugo Giraudel 发起，这是一个社区驱动的尝试，用一些简单的短语解释构成当前 JavaScript 生态系统的许多流行词汇。“我们的想法不是要取代单个文档，而是充当某种易于参考的术语表。”

[![Simplified JavaScript Jargon](../Images/67fc27583416b123450cf4842d138a5b.png)T2】](http://jargon.js.org/)

## 16。自举 4 备忘单

来自 HackerThemes 的 Alex 的 Bootstrap 4 的备忘单。该网站有一个非常好的交互式布局，在视窗底部以代码片段和演示的形式显示被点击的项目。你也可以点击突出显示版本 4 中的所有新内容。

[![Bootstrap 4 Cheat Sheet](../Images/6a06ed5dcbe93bd18a1b7f8ec9da2808.png)T2】](http://hackerthemes.com/bootstrap-cheatsheet/)

## [17。AngularJS 备忘单](https://egghead.io/articles/angularjs-core-services-directive-definition-object-and-ui-router-cheat-sheets)

“我们制作了一些备忘单学习指南，帮助你克服最初的 AngularJS 学习曲线，并为你的日常工作提供参考。”包括三个备忘单:AngularJS 核心服务、AngularJS 指令定义对象和 AngularJS ui 路由器。

[![AngularJS Cheat Sheets](../Images/cdc5d385a6df02ef14b82c3635ffa939.png)T2】](https://egghead.io/articles/angularjs-core-services-directive-definition-object-and-ui-router-cheat-sheets)

## 18。React 让你难过

Dan Abramov 为那些在使用 React 时难以理解不同概念的人提供的流程图。该流程图提供了一些建议，帮助您简化事情，以便更好地理解库。

[![React Makes You Sad](../Images/4a2c90ac82b82e3698d6df73928f91c3.png)T2】](https://github.com/gaearon/react-makes-you-sad)

## [19。柔性盒模式](http://www.flexboxpatterns.com/)

“Flexbox 很棒，但它引入了许多新概念，可能会使它难以使用。这些交互式示例将向您展示使用它来构建 UI 组件的实用方法。它们从简单开始，到最后变得越来越复杂。”

[![Flexbox Patterns](../Images/c7ea664589fb7068993512af8a75ee70.png)T2】](http://www.flexboxpatterns.com/)
