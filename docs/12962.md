# 网络方向政府:让电子政务成为现实

> 原文:[https://www.sitepoint.com/web-directions-government-2008/](https://www.sitepoint.com/web-directions-government-2008/)

Nathanael Boehm 在澳大利亚堪培拉的 2008 年政府网站上为 SitePoint 报道。

这是一个寒冷的开始，气温徘徊在零度以上，我们在旧议会大厦的用餐区等待，手里拿着咖啡，听参加早餐会的人告诉我们杰森·瑞安的演讲有多好。Jason 目前是新西兰国家服务委员会的沟通经理，他发表了题为“政府 2.0:公共管理挑战”的演讲。不幸的是，我没有机会见到 Jason，但是我和他在国家服务委员会的同事 Rowan Smith 谈过，他参与了新西兰政府网络标准项目。

上午 9:00，约翰·奥尔索普宣布会议开幕，并谈到这些会议的重点是如何扩展的，不再是可访问性和其他技术层面的问题。虽然这些现在仍然像 5 年前一样相关和重要，但在这些领域采用和整合最佳实践已经达到了可以进入下一件大事的地步，对于本次会议来说，这就是电子政务:政府如何在内部、与其他机构以及与公众和公民进行在线沟通和协作，以及在线提供服务。他强调说，我们作为网络行业的参与者和活跃成员，是推动这一变革和实现政府与政府所服务的公民之间双向沟通的关键。

来自 W3C 的 José Manuel Alonso 谈电子政务标准

当天的第一位演讲者是 José Manuel Alonso，他是万维网联盟(W3C)的电子政务负责人，在 W3C 的西班牙办公室工作。José详细介绍了在 web 上部署政府服务和信息的技术挑战的许多方面。他引用的研究表明，29%的人更愿意通过互联网与政府沟通，但需要做更多的工作，包括改善政府网站的内容和设计。

José谈到了互操作性框架、开放标准以及这样一个事实，即即使使用相似或相同的模式模型，在使用 XML 的系统之间传输数据可能也是不够的。他解释说，在欧洲，存在一些语际问题，对单词的含义和解释存在差异，因此即使不同国家的组织同意使用单一的 XML 模式，对共享信息的解释仍可能存在混淆或差异。

然后，他对一站式商店的想法表示怀疑，并引用了一份报告，该报告称，2007 年在谷歌搜索五大新闻故事时，博客的排名高于《纽约时报》，这表明了人们查找和获取信息方式的变化。José还引用了洛杉矶消防局(LAFD)的 Brian Humphrey 和 Ron Myers 的话说“我们再也不能以政府的速度工作了。我们有责任让公众尽快了解信息，以便他们能够做出关键决定”。如果你不能用你的数据和信息做一些有用的事情，把它放在别人能做的地方！然后，他列举了几个组织与第三方合作以及第三方转换和混合数据和信息的例子，包括国会图书馆与 flickr 的项目、USPOT 的同行评审在先艺术项目、FixMyStreet、Follow the Money、MapLight、OpenCongress、GovTrack 和 TheyWorkForYou(在 José的演示结束后，我去告诉他，澳大利亚现在有一个即将推出的 TheyWorkForYou 版本，名为 OpenAustralia)。

他在结束演讲时探讨了混搭和数据共享带来的一些挑战和机遇，如隐私和安全、语义技术、关联数据(如 RSS 和 RDF)以及确保和保证数据权威性的重要性，并且不要忘记为移动平台开发！

W3C 向其成员提出了一项电子政务提案以供审查，该提案于今天结束——审查结果将在几周内公布，如果(如预期的那样)W3C 成员为 W3C 的电子政务计划开了绿灯，那么将很快成立一个兴趣小组，开始更详细地研究该提案，并研究围绕电子政务建立标准。

最后，José指出，一旦网络服务成为政府的一个组成部分，不再被视为一个可选的附加组件，而是政府如何运作、交流和提供服务的一个核心组成部分，最终应该从电子政务中删除“e”。这就是我们的目标。

**珍妮·特尔福德澳大利亚统计局公开政府数据**

接下来是澳大利亚统计局产品和服务总监 Jenny Telford，她详细介绍了该局在过去几年中所做的工作，以及从(你能相信吗)对在线访问人口普查数据的人收费到如今他们拥有大量工具、报告和数据格式以及一些非常强大的数据提取配置功能的转变。

可用的报告功能之一允许高级用户访问和汇总普查数据的单个“单位”。尽管没有暴露姓名和地址，但汇总数据并与其他来源进行比较的能力可能会允许人们获得私人信息，因此他们不得不采用一些高级算法来保护人们的隐私。

她承认，该局仍有一段路要走，进一步开放数据，提高地理空间报告能力，并审视他们的版权和知识产权(IP)保护政策。目前，一些报告有 500 个单元格的限制，作为防止大规模数据收集的基本手段——她承认这一措施需要研究。

珍妮演讲后提出的一些问题包括 Y 一代的文化转变和对披露个人信息越来越冷漠，以及这可能如何反映在未来的人口普查中，以及关于利用社区验证和增加统计局数据库中的信息的问题。

GovDex 的拉尔夫·道格拉斯

Ralph 接下来做了一个关于 GovDex 的 25 分钟的简短介绍(我错过了一半，因为我正在和 W3C 堪培拉办公室的 Amit 谈论加入联盟的电子政务兴趣小组)。GovDex 是一个由政府运营的安全(归类为机密，计划在年底推出受保护版本)在线协作工具，允许内部、跨部门和政府-社区团体使用维基设施和成员注册/信息数据库在线联系和分享想法。不幸的是，演示有点轻松和枯燥；虽然这个应用程序看起来很有用，但我并没有从中获益。

**可用性场景七中的 Lisa Herrod**

Lisa Herrod 的关于可用性的演讲“不仅仅是肤浅的”，这是我在去年悉尼的 Web Directions South conference 上看过的一个演讲，但第二次也是一样好，它有一些实用的想法，关于如何对可用性采取一个整体的方法，它如何与代码和可访问性联系起来，自动的可访问性合规性检查是不够的，以及作为你的角色开发的一部分的可访问性用户配置文件的想法。将残障人士视为主要用户群体可确保他们的需求在整个网站开发过程中得到考虑，而不是在项目结束时软弱而仓促地尝试改造已完成的网站以符合 WCAG 标准。

【SMSMT 的马修·霍奇森谈社会计算

Matthew 接下来要谈的是政府中的社会计算，这是一个非常好的演讲，他以一个项目的故事开始，这个项目是他前段时间参与的，目的是在一个不知名的政府部门引入一个 wiki。其目的是使该部门内不同部门之间能够协作，以解决他参与的维基实施小组所发现的严重的知识共享和效率问题。不幸的是，一旦高级管理层发现了维基百科，并发现该部门的一些部门没有通过适当的渠道批准和签署信息以供传播(即使只是内部传播)，维基百科就被关闭了。

他把一些组织比作达斯·维达——需要控制，锁定一切。为什么普通人会知道他们想要什么信息？他还提供了一个有趣的统计数据:1 亿小时的知识进入了维基百科……相当于美国人每个周末集体观看电视广告的时间！如果人们能更好地利用他们的时间，通过像维基百科这样的合作项目或像可持续发展维基百科这样的生产性和解决问题的项目来贡献全球知识。

人是社会性的动物——他们需要满足这种社会需求，在工作环境中有满足这种需求的方法(马修提到了马斯洛的需求层次)——这不是浪费时间与朋友社交。社区和网络不一定是地理上的——如果是，那么你会和你网络中的人面对面交流(就像我们在会议上一样)。然而，对于那些建立在兴趣或其他基础上的人来说，这些网络在网上运行的事实并不能消除这些互动仍然需要社交元素的事实。

来自 Miskeeto 的 Robert Hoekman Jr .谈可用性和极简设计

当天的最后一个演讲是由来自亚利桑那州凤凰城的小罗伯特·霍克曼(Robert Hoekman Jr. 做的，他在演讲开始时宣布，网络上仍然有许多糟糕的网站——这些网站让用户感到愚蠢，无法快速直观地知道如何使用网络界面，网站如何将所有这些功能都塞进去，这并没有让它们变得更好，只是让它们变得更复杂，更难使用和学习。

他的经典台词是“理解用户，然后忽略他们”——继续解释你需要理解用户的需求，但人们并不总是知道他们想要什么，或者说一些不能真实代表他们在现实生活中可能会如何使用的话。他举了一个快餐连锁店的例子，该连锁店推出了一种低碳水化合物奶酪汉堡，在用户调查中，这种汉堡听起来很受欢迎，但当它出现在菜单上时，效果并不好。网站需要支持一项活动或一系列活动，而不是受众类型。你的用户研究应该确定你的用户的基本需求，但不要过度审查你的用户，并试图迎合每一个可能的用户类型的需求。

罗伯特接着谈到了极简设计——只建造绝对必要的东西:“一旦你开始在你的树屋里装上真正的玻璃窗，你就失去了树屋本来应该是什么样子的意义”。曾经厌倦使用锤子吗？锤子有一个明确的目的，你不太可能拿起锤子指望它做别的事情，这也意味着锤子永远不会令人失望，因为它做的正是它被设计用来做的事情——仅此而已。

他提到了几个案例研究，包括 WordPress 主页和 Squidoo 网站，以及页面设计的变化如何使其更加简单和明显，对网站使用产生了真实的、可测量的影响。

他给出了一些更好的建议，包括“把初学者变成中间用户”。立即”(指的是网站应该如何直观地使用，并立即赋予用户权力)、“设计的统一性、一致性和意义”以及“减少、减少、减少和精炼”……只包括哪些元素是必要的，并有意识地决定页面上每个方面和功能的位置。

**终于……**

约翰随后上台总结了会议，再次感谢 2008 年南方政府[网络方向的赞助商，包括 SitePoint 和主持人。我询问了一些人对这次会议的看法，他们的反应是积极的，尽管一些人说他们期待更多的政府关注——我同意这一观点，尽管很难从任何人那里了解更多的政府关注可能意味着什么。后来，我与 Maxine(会议组织者之一)进一步讨论了这个问题，得出的结论是，人们希望更多地强调政府的背景；在根深蒂固的头重脚轻的文化、程序和文化中，在一个强大的层级中工作的特殊挑战，创新的谨慎，尤其是社交计算。这可以通过更多的政府项目案例研究和更多有政府内部工作经验的人的发言纳入会议，这些人可以理解那些试图做好工作的“内部”人的痛苦，建立可访问和可用的网站，在设计、用户研究和测试中应用最佳实践。](http://gov08.webdirections.org/)

在我错过的四场演讲中，后来我有机会与斯科特·格莱德希尔和大卫·海沃德交谈，了解他们演讲的概要。Scott 的文章是他在 Digital Web Magazine 上发表的一篇文章的续篇，该文章讨论了公司 Web 标准以及在一个时间和效率优先的环境中实施 Web 标准的可悲现实(或有趣的挑战)。David 以新南威尔士州政府空间信息交换 [SIX Gateway](http://six.maps.nsw.gov.au/wps/portal/) 网站的开发为例，介绍了地理空间数据的利用。

总之，由约翰、玛克辛和全体工作人员组织和举办的另一个伟大的网络方向会议将在未来的许多天里激励我(我希望其他人也是如此)。
