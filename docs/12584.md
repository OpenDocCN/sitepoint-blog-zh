# 谷歌披露应用引擎产品地图

> 原文：<https://www.sitepoint.com/google-reveals-app-engine-product-map/>

谷歌宣布了其云计算服务[应用引擎](http://code.google.com/appengine/)的暂定产品路线图。昨天与应用引擎文档一起发布的[路线图](http://code.google.com/appengine/docs/roadmap.html)相当模糊，谷歌工程师承认日期可能会推迟，但该文件确实给了我们一些来自谷歌的指示。

到 2009 年第一季度末，谷歌预计将为其应用引擎产品增加以下功能:

*   存储和提供大文件的服务
*   大型数据集的数据存储导入和导出实用程序
*   计费:开发者可以为更多的资源使用付费
*   支持新的运行时语言
*   正常运行时间监控网站

该文档缺乏细节，所以我们只能猜测 App Engine 计划支持的新运行时语言可能是什么。最近有很多关于印度的报道，根据社区的反馈，Java 将成为下一个应用引擎语言。

然而，谷歌否认任何事情即将发生，并对他们正在支持的语言含糊其辞。“虽然我们没有任何立即的计划或公告，但我们正在为 App Engine 开发其他语言。据 ZDNet Asia 报道，该公司在一份声明[中表示:“安全性和性能是我们对任何新的语言运行时的首要关注点，要做到这一点需要时间。”。](http://www.zdnetasia.com/news/internet/0,39044908,62047603,00.htm)

如果谷歌确实支持 Java，那么 [Java 虚拟机语言](http://en.wikipedia.org/wiki/List_of_JVM_languages)似乎也可以工作。这意味着人们可以在 App Engine 上使用 JRuby 和 Rhino。

该路线图虽然令人发笑地含糊不清，但却包含了大量重要的更新。其中最值得注意的是宣布即将增加计费。为了让开发人员将 App Engine 视为其他云计算平台(如亚马逊平台)的重要替代品，确保客户永远不会收到超过配额的消息无疑是一个关键。
