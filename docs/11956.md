# FullCodePress 采访:蒂姆·康纳，新西兰队

> 原文:[https://www . site point . com/full code press-Tim-Connor-interview/](https://www.sitepoint.com/fullcodepress-tim-connor-interview/)

来自两个国家的两个团队要为两个慈善机构完成两个网站。24 小时后。观众注视着每一个动作。这就是 2009 年 5 月在 CeBIT 举行的 [FullCodePress](http://www.fullcodepress.com/) 背后的想法。

Tim Connor 是新西兰团队的程序员，该团队完成了[获奖的彩虹青年网站](http://fcp-nz.com/)。他友好地与 SitePoint 分享他的经验。

你或你的团队在比赛前做了什么准备吗？

没有我应该有的那么多！我的大部分准备工作包括调查技术选择和了解我的队友。在我看来，比赛前我和凯特、考特尼一起喝的啤酒以及和球队在一起的一整天是理想的准备。

你以前和其他团队成员合作过吗？

在 FullCodePress 之前，我见过的唯一一个团队成员是 Courtney，但那与其说是专业的，不如说是社交的。我们很幸运彼此相处得这么好。

你选择 browser CMS——一个相对不为人知的基于 Rails 的测试产品——作为内容管理系统？影响你决策的关键因素是什么？有没有造成什么问题或者限制？

我使用 Rails 已经有一段时间了，所以我倾向于任何基于 Rails 的东西。我听说过 BrowserCMS 的优点，并在准备期间使用过它。我最初的偏好是不使用 CMS，但是在我们与客户会面后，很明显需要一个 CMS。我们可以使用 WordPress 或者其他基于 PHP 的解决方案，但是因为我从来没有使用过 PHP，这不是最好的主意！

我不认为 BrowserCMS 给团队带来了很多问题，但我还是有点纠结。产品使用 Rails 而不是补充它；“铁路方式”和我通常的工作方法并不总是合适的。

你用什么工具进行开发？IDE？源代码管理？测试套件？

我用 Textmate 进行代码编辑，用 Git 进行源代码控制。我运行了一个我们在开发过程中使用的本地 MySQL 数据库。

在 CeBIT 的主要时间里，我们发现我们的互联网访问几乎没有用，所以我在我的笔记本电脑上运行了一个完整的环境，团队的其他人也可以访问。

最大的挑战是什么？

有两大挑战:有限的时间和保持清醒。我们只有 24 小时来建立网站，而大量的时间都花在了不太理想的心态上！凌晨 3 点到 7 点对任何团队成员来说都不开心。

什么比你想象的要容易？

客户。Clint 和 Daniel 非常棒，乐于助人，很容易共事。他们整晚都和我们在一起，让我们的工作变得容易多了。

**您是否需要放弃或简化某些功能，因为它们无法在 24 小时内实现？**

非常喜欢。Clint 和 Daniel 从一个巨大的需求列表开始，所以我们需要确定优先级。甚至我们实现的功能也必须简化——例如，捐赠过程本来可以是无缝的，但是没有足够的时间。尽管如此，我们创造了尽可能多的功能。

尽管你取得了胜利，但你的竞争对手在哪些方面击败了你？

我认为澳大利亚团队更好地管理了他们的项目范围。我们努力实现了许多功能，但也许我们应该更多地关注核心需求。澳大利亚的网站似乎是干净的，完成了。

事后想想，你会做出什么不同的事情吗？

我可能已经抛弃了 CMS，推出了我们自己的产品。使用 Rails，我可以轻松地构建一个简单的 CMS 来满足客户的需求。BrowserCMS 功能齐全，但相当复杂，对客户来说可能有些困难。我正在考虑写一个定制的管理界面来简化网站维护。

你学到了什么可以用在日常工作中的东西吗？

我知道我在早上 7 点左右会有第二次呼吸，但是我不认为我会在我的日常工作中使用它！我向自己重申，我不喜欢内容管理系统！

您会再次参加 FullCodePress 活动吗？

当然可以。我度过了一段美好的时光，交了一些很棒的朋友，我相信车队的其他人也有同样的感受。我们希望在惠灵顿有一场 07 对 09 的比赛，这样我们就可以挑战之前的冠军。

谢谢你，提姆。

Tim Connor 是新西兰惠灵顿的一名网络开发人员。他在新西兰和英国的银行、政府和在线游戏等多个行业拥有长期的企业软件开发背景。现在他是一名独立的 IT 顾问，专门研究 Ruby on Rails。如果你想和蒂姆·雷格拉丁咨询公司或网站开发公司的人谈谈，可以在 info@connorsoftware.com 和他联系。
