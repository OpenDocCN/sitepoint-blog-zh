##### 十大误解

***1。PHP 就像一个开源的. net .***

你不能拿 PHP 和. NET 做比较。NET 是应用框架(构建和执行应用的环境)*不是*编程语言；这就像把卡车比作公路。PHP *可以*，然而*可以*比作运行在。NET，比如 ASP+或者 C#(简称“语言。网“此处)。

***2。。NET 将会是互联网的未来！***

。NET 是微软所有技术的未来平台——它不仅仅针对互联网。事实上，它很可能第一次在公司内部网上大规模亮相，与 Office.NET 和。现有 Windows NT 服务器软件的. NET 版本。

***3。。NET 是建立网站的最佳技术。***

PHP 和“语言。NET”是构建 Web 的可行解决方案。无论是交付从数据库生成的动态网页、发送电子邮件，还是与支付网关对接，两者都完全能够胜任。在评估它们作为构建网站的技术时，唯一显著的区别是价格。试着问问你自己“那么托管我刚刚写的这个 ASP.NET 页面要花多少钱？”

***4。PHP 不被编译，除非在运行时被解释，而。NET 享有被编译的所有好处。***

这是新闻。网络人群——你终于赶上了！微软对“编译”一词的使用。NET 不是 C++程序员所说的“编译过的”。PHP 项目的公司代表 [Zend](http://www.zend.com) ，已经用 [PHP 加速器](http://www.zend.com/store/products/zend-accelerator.php)做了一段时间了(免费的等价物，试试[php-accelerator.co.uk](http://www.php-accelerator.co.uk))。对 Zend 来说不幸的是，他们选择将 Accelerator 的功能描述为“高级缓存”(阅读更多关于[Zend 所说的“缓存”](http://www.zend.com/store/products/accelerator-faq.php#5))。要了解更多技术细节，请在 SitePoint 论坛上试试这个帖子。

***5。ASP.NET 得到了应有的支持。PHP 没有。***

除了由 [Zend](http://www.zend.com/support/) (实际上是 PHP 的创造者) [NuSphere](http://www.nusphere.com/services/index.htm) 、 [ThinkPHP](http://www.thinkphp.de/) 和[提供的支持服务，其他公司](http://www.zend.com/partners/locate_isv.php)也为 PHP 提供支持。由于不断扩大的 PHP 社区，你不需要它，但是如果你的老板想找一个可信的人来责备，有足够多的选择！

***6。。NET 支持多种语言。PHP 没有。***

试试 [Java for PHP](http://www.php.net/manual/en/ref.java.php) 或者即将推出的 [Python](http://www.csh.rit.edu/~jon/projects/pip/) 扩展。显然，这与 C#或 J#不太一样，但有一点需要说明——PHP 非常灵活，为互联网提供了*有用的*解决方案。不要期望你会发现自己在 Cobol.NET 写了一个成功的网站——这很有可能。NET 语言最终会“适应”特定类型的应用程序开发。

***7。Language.NET 具有优越的面向对象性。***

对于那些了解 PHP 及其局限性的人来说，你可能会对此感到好笑。NET FAQ——听着耳熟？事实上，随着 PHP 4.3 的出现，当你考虑到常见问题时，PHP 可能会声称拥有更好的面向对象支持。
