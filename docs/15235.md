# 2 个快速 PHP 技巧

> 原文:[https://www.sitepoint.com/quick-php-tips/](https://www.sitepoint.com/quick-php-tips/)

现在你是一名精通 Web 编程的程序员，涉足著名的开源语言 PHP。唯一的缺点是这种语言允许程序员粗心大意，在不知不觉中降低了应用程序的性能。

在我的公司，我们每天都在使用 PHP，并且我们尝试使用许多不同的风格和规则来使我们的(相当大的)应用程序高效和无 bug。让我们来看看三年前我希望在学习这门语言之前就知道的两件最重要的事情。

##### 立即打开错误报告

我告诉使用 PHP 的人的最重要的一件事是将错误报告最大化。我为什么要这么做？一般来说，错误报告的级别会隐藏许多细节，例如:

*   提前声明一个变量，
*   引用该段代码中不可用的变量，或者
*   使用未设置的定义。

这些因素可能看起来没什么大不了的——直到你用函数和类开发结构化或面向对象的程序。通常情况下，编写错误报告率很高的代码会耗费您的时间，因为您会发现由于变量拼写错误或不可访问而无法运行的长函数。

在这种情况下，PHP 不会告诉您任何事情——它只会为您创建新变量并将其初始化为零。补救方法是在开发时，在每个 PHP 文档的顶部添加下面一行:

```
error_reporting(E_ALL);
```

它只是强制错误报告处于最高级别。试着将这一行放到其他 PHP 程序中，您通常会收到大量警告消息，指出代码中所有潜在的错误元素。

##### 单引号和双引号非常不同

我从来不建议在用 PHP 编程时使用“(双引号)”。除非您需要“(双引号)的功能，否则请始终使用“(单引号)”。您可能认为将代码编写为:

```
echo "Today is the $day of $month";
```

但是，使用单引号会强制变量在引号之外；相反，您必须使用句点(。)来组合字符串。它有助于更快的编码，但是对于其他程序员来说可能更难阅读。让我们看看如果在前面的代码中放入一个关联数组值会发生什么:

```
echo "Today is the $date['day'] of $date['month']";
```

您将会收到一个解析错误，并且对于其他团队成员来说更难阅读。编写这一行代码的两种正确方法是:

```
echo 'Today is the ' . $date['day'] . ' of ' . $date['month'];
```

和

```
echo "Today is the {$date['day']} of {$date['month']}";
```

这些代码可能看起来不如原始代码漂亮，但是从语法上来说，它们都是正确的。此外，我认为第一种方法，加上单引号，更容易阅读。

单引号和双引号的使用也适用于关联数组。考虑以下代码:

```
$SESSION[team] = $SESSION["old_team"];
```

这一行代码中存在一个主要问题。左侧的关联条目团队需要用单引号括起来；否则，PHP 会认为这是一个 define，并给你一个警告消息(仅当错误报告最大时)。我建议代码应该是这样的:

```
$SESSION['team'] = $SESSION['old_team'];
```

当我第一次学习 PHP 时，我希望我知道单引号和双引号之间的区别，因为它们与字符串有关。

##### 额外提示！

**使用 Dreamweaver 4 获得 PHP 的语法高亮显示**

当我使用 Dreamweaver 4 作为文本编辑器用 PHP 编程时，我遇到了一个方便的小技巧。您可以欺骗 Dreamweaver，让它认为页面上的代码是 JavaScript，这样它就可以像对 JavaScript 一样对您的 PHP 应用相同的语法高亮规则。

JavaScript 和 PHP 都来自相同的语言，包括 C 和 Java，因此有许多共同之处。自己试试看。将此代码添加到 PHP 文档的顶部，在 Dreamweaver 4 中打开它，然后进入文本编辑器(您可以单击键盘上的 Ctrl-Tab 来切换视图):

<q>/* >用 Dreamweaver 得到 PHP 语法高亮显示的代码
<脚本语言= " JavaScript ">(david@superupdate.com 提供)*/</q>

代码所做的是关闭页面上最近的左括号(通常从`<?php`开始),并将页面剩余部分的脚本语言设置为 JavaScript。代码本身不会影响 PHP，因为它是使用`/*...*/`注释系统注释掉的。

我希望这些 PHP 编码的小技巧能让你的开发时间更有效率，你的结果代码更有效！
