人们常说，最好的用户界面是看不见的，或者说“最好的界面就是没有界面”。一个简单的独立工作的特性——不需要与界面交互——是完成一个动作的理想方法，而那些需要多次交互的特性不是最佳的。

**智能向导**是 Sketch 的一个功能，它通常不需要与画布进一步交互，当你在画布上拖动它时，它会向你显示一个层相对于另一个层的距离*。对此没有*接口*。*

Sketch 也有一个简单易记的键盘快捷键列表，比如矩形的 **R** ，使得用户界面非常小。虽然与 Photoshop 和 Illustrator 等应用程序相比，Sketch 可能看起来有点简陋，但它是*的任何东西*但有限。如果你学会了相当于每次交互的键盘快捷键，你可以让自己进入一个非常高效的工作流程，尽管如果你用鼠标工作得更好，你不会觉得素描很难。你唯一真正的限制是你的创造力。

## 一览:比较 Sketch 和 Adobe 的应用程序

尽管 Adobe 早在 2015 年 12 月就为其用户界面引入了全新的平面设计皮肤，但与 Sketch 相比，这些界面仍然相当笨重。Sketch 有一个图层列表，一个检查器，当然还有一个工具栏，而 Adobe apps 可以选择在工作空间中添加和删除工具。Sketch 有一个单一的默认工作空间，更易于学习者掌握。

Sketch 所提供的一切都可以在屏幕上立刻看到:不需要优化工作空间。

## 画布

每当你第一次打开 Sketch，你都会盯着一个空白的**画布**，它本质上是一个白色的背景，你无法与之互动。这是一个创造的空间，仅此而已。

您可能已经注意到，空白画布是无限的；在创建画板之前，没有任何限制。画板是一种不同类型的工作空间；它们有*预定义的*尺寸，它们被*插入*画布。我们将在下一章更详细地讨论画板。

**层**是构成设计的独立组件(可以是一个形状、一些文本或某种矢量)。我们将在本书中广泛讨论它们。

页面是 Sketch 中的另一个概念，每一页都包含另一个无限的画布。在每个画布上，您可以有无限个画板/屏幕，并且层是在其中创建的。

**页面→画布→画板→图层**。

### 多个画布

草图没有选项卡式窗口。任何新文档( **cmd + N** )都会出现在一个新窗口中——这看起来可能有点尴尬，但实际上使用 Sketch 的无限画布来显示与某个设计相关的所有屏幕是很平常的事情。在画布中，您可以根据需要拥有任意多的画板和屏幕。在不太可能出现的图层过度拥挤的情况下，您可以将文档拆分为多个页面，但您永远不会需要多个窗口。(我们稍后会详细讨论这一点。)

### 画布缩放

当使用 Sketch 的无限画布时，您可能会同时在屏幕上显示相当多的画板。幸运的是，Sketch 有多个用于放大和缩小的快捷键:

*   放大: **cmd +加号**
*   缩小: **cmd +减号**
*   缩放至实际画布大小: **cmd + 0**
*   缩放至画布中心: **cmd + 1**
*   放大选中的图层: **cmd + 2**
*   在屏幕上居中选择: **cmd + 3**

如果你记不住这些，不要担心。我将在整本书中重申键盘快捷键，你可以随时轻弹到书的末尾来查看草图快捷键的完整列表。

## 图层列表

图层和图层组显示在应用程序窗口左侧的列表中。我们称之为**层列表**。它与 Photoshop 的图层列表没有什么不同，只是在 Photoshop 中它出现在右边。如果一个层出现在列表的顶部，那么它会出现在画布中任何其他层的顶部。我们称之为**层层次**。

您可以拖移列表中的层来对它们重新排序，但是使用键盘快捷键更容易。只需按住 **cmd + option** 并上下移动箭头键即可编辑层的层次。

### 形状层

**形状**是最常见的图层类型。我所说的形状是指矩形、椭圆形和许多其他可以插入的形状，比如星形、三角形、圆角矩形等等。以下是一些方便的键盘快捷键:

*   椭圆形: **O**
*   矩形: **R**
*   圆角矩形: **U**
*   行: **L**

通过菜单栏的**插入→形状**可以访问更多的形状，尽管你不会经常需要这些形状。选择形状后，拖动鼠标在画布上绘制它。获得正确的尺寸并不是一件大事，因为我们可以很容易地设计形状和其他类型的层。

![A simple rectangle](../Images/7510d92fafa978e9915d4ecd93eecd64.png)

A simple rectangle (“Rectangle 1”) as shown on both the canvas and in the Layer List

### 文本层

与形状类似，可以通过使用键盘快捷键 **T** 并拉出图层来创建文本。当你输入完文本后，你可以使用键盘上的 **esc** 退出编辑模式并再次移动图层。

![A simple text layer](../Images/f5a9a01c35ff2871a0d91ca37a505e34.png)

A simple text layer (“hello”) as shown on both the canvas and in the Layer List

### 选择和隐藏图层

您可以通过在画布中点按文本层或从层列表中选择它来选择文本层。您可以通过将鼠标悬停在层列表中的层上并单击出现的“眼睛”图标来隐藏层。再次点按它以再次显示它。

### 重新排序层

现在我们有了两层——一个形状层和一个文本层——我们可以改变层次结构。选择文字图层，运行键盘快捷键 **cmd + option + ↓** 。我们的文字层消失了。

它没有被隐藏，但是它*在*视图之外，因为我们在层列表中将其下移了一级，矩形层现在挡住了它的视图。按 **cmd + option + ↑** 使其在列表中上升一级。现在该层显示为之前的样子。

![Text layer hidden behind a rectangle layer](../Images/436a7fbc073f328c7dae7d7da7529e56.png)

Here, the text layer is hidden behind a rectangle layer because it appears lower in the list

### 复制和重命名图层

复制图层也可以通过快捷键 **cmd + D** 来完成——这比 Photoshop 的 **cmd + J** 更有意义。

重命名图层是 **cmd + R** 。

## 组

我们现在知道如何创建，选择，隐藏，重新排序和复制层，但整个组件呢？在这种情况下，**组件**是组成更大的东西的两层或更多层。例如，按钮背景(矩形图层)和按钮文本(文本图层)将组成一个按钮组件。

可以将层组合成单个组件，以便可以同时移动和对齐几个相关的层。这些单个组件被称为**组**。

你可以通过按住 **cmd** 同时在层列表中点击它们的名称来一次选择多个层(如果你是从画布上这样做的话，添加 **shift** 到混音中)。按下键盘上的 **cmd + G** 将这些选定的层组合成一个组，然后该组采用自己的层次，并成为另一个层次的*部分*。

可以选择和移动一个组，就像它是一个单层一样。

![Rectangle and text layers wrapped in a group](../Images/d1076539382f67bb0612f5accbf51336.png)

Our rectangle and text layers have been wrapped in a group, aptly named “Rectangle 1 + hello”

工具栏中还有一个**分组**和**解组**按钮。

### 访问嵌套(或“子”)层

**嵌套层**是包裹在组内的层。要选择群组内的图层，首先需要选择群组，然后双击群组内的图层；否则，您需要从层列表中手动选择它。图层可以嵌套在组中，而组又可以嵌套在其他组中，这最终会使图层选择变得非常烦人。

我们可以通过选择群组并在选择 UI 右侧的**检查器**中的复选框时检查**点进来绕过双击的需要。**

你可以在点击的同时按住 **cmd** 来完成同样的事情。它将允许您临时直接点击到嵌套层，但由于该功能的简单性，通常最好一直这样做。

## 第
页

最终，你的画布将开始建立，你将负责许多不同的层和组，以及画板(我将在下一章谈到)。正如我前面提到的，没有必要把一个设计分成不同的文件。为了保持整洁，我们使用书页。

在图层列表的最顶端，你会注意到它写着“第一页”。按下图层列表右上角的图标(方框箭头),然后再次打开一个新的“页面”。例如，如果你想把你的主页从“关于我们”部分中分离出来，这将是一个理想的方法。保持一个整洁的文件需要团队、画板和页面的共同努力。

### 重命名页面

当聚焦于层列表中的层或组时， **cmd + R** 可让您轻松地对其进行重命名。但是，这在 Pages 选项卡中不起作用，所以您必须双击来重命名它们。为了区分主页(例如)和“关于我们”部分，我们可以将“第 1 页”和“第 2 页”分别重命名为“主页”和“关于我们”。

如果你正在按照我到目前为止所描述的步骤进行，请确保在我们继续之前返回到“主页”,因为这是我暂时要关注的。

## 检查员

我在上面提到了一个叫做**检查器**的东西，它出现在草图窗口的右边。不，不是很牛逼的漫画超级英雄；这是我们设置图层样式和为画板和组声明可选设置的地方。如果你熟悉在“设计视图”中构建网站，你会发现这是完全一样的，只是没有生成代码。

不同类型的层有不同的可用选项。群组和画板根本没有任何样式选项，但它们有*设置*。我们将在第三章更深入地探讨这个名字令人惊叹的检查员。不要消失！

## 工具栏

在**工具栏**(上方)你会看到一排…嗯…工具。其中的每一个都实现了一个特定的动作——我们将在本书的各个章节中讨论其中的大部分，有些我们已经讨论过了(比如分组、解组、前进和后退)。你可能已经注意到**也插入了**，这是插入形状、矢量和位图的另一种方式。

## 隐藏用户界面

有时候，尤其是如果你像我一样使用一台小小的 11 英寸 MacBook Air，你会希望在没有用户界面的情况下客观地看待你的工作。您可以使用快捷方式打开和关闭 UI 的各个方面:

*   图层列表: **cmd + option + 1**
*   检查员: **cmd + option + 2**
*   工具栏: **cmd + option + T**

你可能已经注意到了，我已经在一些截图中这样做了，所以它们更干净，更容易理解。

## 上下文菜单

也被称为“右键菜单”，**上下文菜单**根据你点击的位置改变它的选项。它就像一个不断变化的工具菜单，为您提供与焦点对象相关的工具。

例如，如果你右击一个层，你可以通过点击上下文菜单中的**复制 CSS 属性**选项来提取 CSS 样式标记。上下文菜单中的许多可用操作也可以从工具栏中访问，或者通常是一个方便的键盘快捷键。

在极少数情况下，你可能必须通过应用程序的原生菜单来访问**文件**、**编辑**、**图层**、**类型**、**排列**、**插件**或**视图**。然而，这里列出的选项和工具并不被大多数设计者普遍使用。Sketch 的上下文菜单旨在让一些“隐藏”的工具更容易使用。

选择使用上下文菜单还是菜单栏取决于您。这主要取决于你的需求有多复杂。

## 总结

Sketch 如此奇妙的多功能性在于，它的许多功能都可以在上下文菜单、工具栏*和菜单栏*中找到，更不用说它们也可以通过键盘快捷键访问。你可以用适合*你*的方式使用素描。让我们总结一下 Sketch 用户界面的核心方面:

*   页面:在*文档*中的一块新的空白画布
*   画布:在*页面*中的一个空白的、无限的工作空间
*   画板:插入到*画布*中的可选模板
*   图层:一个*画板*中的形状、文本和位图图像
*   层列表:按层次排序的组和层的列表
*   检查器:用于指定样式和设置的界面
*   工具栏:一排鼠标操作的设计工具
*   菜单栏:不常用工具的原生应用菜单。
