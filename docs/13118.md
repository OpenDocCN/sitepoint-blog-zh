# 雅虎！待售:开发商会被留下吗？

> 原文:[https://www . sitepoint . com/Yahoo-for-sale-will-developers-be-left-been/](https://www.sitepoint.com/yahoo-for-sale-will-developers-be-left-behind/)

除非你最近一直生活在岩石下，否则你会知道[微软正在寻求收购雅虎！](https://www.sitepoint.com/microsoft-offers-42-billion-for-yahoo/)420 亿美元。[雅虎！据报道，T4 拒绝了这一提议，目前正在与包括新闻集团和美国在线(时代华纳)在内的其他潜在买家探讨选择。](https://www.sitepoint.com/yahoo-says-no-genius-or-suicide/)

关于雅虎可能会发生什么，已经说了很多。如果微软的收购继续进行的话，微软的各种资产——搜索、电子邮件和广告。但作为一名开发人员，我个人对雅虎会发生什么很好奇(T2)的开发者拓展计划。

自从两年前推出，[雅虎用户界面(YUI)库](http://developer.yahoo.com/yui/)和[雅虎设计模式库](http://developer.yahoo.com/ypatterns/)为文档完善、可重用的 JavaScript 和用户界面设计资源树立了黄金标准。这些资源只是所有优秀作品的两个突出例子。在 [Yahoo！开发者网络](http://developer.yahoo.com/)。

我甚至可以说，雅虎！是当今商业界唯一一家我信任的拥有网络未来的大型网络公司。他们不断努力让 web 开发人员的生活变得更好，他们愿意提供开放的 API 让数据进出他们的应用程序，以及他们最近对像 [OpenID](http://openid.net/) 这样的开放计划的投资，所有这些都表明一家公司了解 Web 如果开放开发的潜力。

与之形成鲜明对比的是，谷歌甚至懒得在它的[主搜索表单](http://www.google.com/)上提供一个合适的`DOCTYPE`声明(没有`DOCTYPE`声明的网络上访问量最大的页面？).

至于微软，前几天我在浏览微软下载网站寻找我鼠标的驱动程序。很自然，没有鼠标驱动，我只能通过键盘浏览网站。至少我*是*，直到一个新的 Silverlight 下载网站的弹出广告覆盖了页面——有两个非键盘可访问的按钮可供选择！

看着雅虎的名单！想到这些公司会对雅虎做些什么，我就不寒而栗的努力，使网络成为一个更好的地方，为开发人员。乐观者可能会寄希望于雅虎美国文化传播给它的收购者，但我不会屏住呼吸。

当雅虎！分析师们似乎一致认为，现在的问题是“何时”而不是“是否”被收购，还有其他大公司准备站出来带头吗？
