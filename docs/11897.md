# 如何挑选完美的编程编辑器

> 原文：<https://www.sitepoint.com/pick-perfect-program-editor/>

对于外界来说，开发人员可能对他们的代码编辑器有一种不健康的迷恋。这并不奇怪——我们大多数人花在 IDE 上的时间比花在家人身上的时间还长。肖恩最近发布的免费编辑器列表引发了关于什么是最好的编码工具，什么不是最好的编码工具的过多争论。

有几十种代码编辑器可供选择，其中许多是免费软件或开源软件。最终，选择是你的，无论是 Windows 记事本，VIM 还是 VisualStudio。然而，我向所有 web 开发人员推荐一些关键的 IDE 特性。

## 基本特征

如果编辑器不提供以下功能，它将不会保留在我的电脑上:

速度和稳定性
编辑必须快速启动并保持反应灵敏。抱歉，日蚀和阿普塔纳。

**多文件/ MDI 接口**
单个源文件打开的日子早已过去。选项卡是我首选的界面，它们绝对必须支持中键关闭！

**多语言支持**
即使是最基本的 web 项目也需要 HTML 和 CSS。一个简单的应用程序可能会添加 JavaScript、SQL、XML 和一种服务器端开发语言，如 PHP、C#、VB、Java、Perl、Ruby 或 Python。你的编辑器应该能够毫不费力地处理你选择的每一种代码。

换行和编号
这可能是不好的做法，但长代码行有时是必要的。编辑器应该允许您在不水平滚动的情况下查看代码，因此换行选项是必不可少的。当不可避免的错误出现在第 4321 行时，您需要能够快速找到那个 bug 所以忘记任何不显示行号的编辑器。

可配置的颜色编码
缺少引号或标签会造成混乱，但是颜色编码可以大大降低你的错误率。任何提供完全可定制的配色方案的编辑器都会获得奖励积分。

**自动完成**
微软 VisualStudio 凭借智能感知树立了黄金标准。很少有编辑能达到那个水平，但是如果我添加一个`<p>`标签，你可以保证我想要一个结束`</p>`。请不要让我打字！

**会话存储和恢复**
当编辑器重新启动时，它应该会恢复您正在处理的文件，以正确的顺序打开它们，最好是在您正在编辑的最后一行。

**文件浏览器**
选择打开当前项目中的其他文件可以节省时间。

**跨文件查找和替换**
你重命名过一个类或函数多少次？编辑应该提供方便快捷地修改所有参考文献的工具。

## 理想的特征

如果可以的话，以下选项非常好:

**调试工具**
调试设施的范围从绝对必不可少(。NET 或 Java)，到有用的(PHP 或 Perl)，到不必要的(JavaScript——除非比 Firebug 好！)确保你的编辑器提供了你需要的调试工具。

如果你可以标记单独的行，那么在同一个文件中的函数之间跳转会更容易。

**多视图**
同样，将一个文件“分割”成多个窗口也很有用，这样就可以同时编辑不同的片段。

代码折叠
折叠或展开代码段的功能可以让你专注于重要的代码。

代码资源管理器
visual studio 再次设立了标准，用类检查窗口显示所有属性和方法的细节。然而，按字母顺序或源代码顺序排列的方法/函数名称的简单列表也同样有用。任何处理 JavaScript 中多种类型的函数声明的编辑器都有加分！

**良好的字体支持**
一般来说，我更喜欢使用比例字体编码。我可能是少数，但请不要逼我用等宽！赛特提供了最好的解决方案；您可以使用 Ctrl+F11 在等宽字体和等比字体之间切换。

**快捷键、插件和宏**
如果您使用特定的键盘组合或功能，可配置的快捷键、宏和宏插件可以帮助您迁移到另一个编辑器。

## 还有不好的功能…

有时，编辑器会提供一些你并不真正需要的可疑功能。

**所见即所得设计视图**
虽然所见即所得对开发新手来说很有用，但你很快就会发现手工编写 HTML 和 CSS 比任何设计包都要快。如果您的应用程序需要 DOM 操作，这也是必不可少的。如果提供了一个所见即所得的视图，确保它不会在你意外点击“设计”按钮时重构你精心制作的代码。

**突兀的项目管理**
有时候，你只需要打开一个文件，做一个小改动——请不要强迫我们创建一个新项目！

**愚蠢的 HTML 生成工具**
我不需要一个占用宝贵屏幕空间并插入`<p>`标签的工具栏图标。而且我当然不需要坚持添加不推荐使用的`<b>`标签的图标！

你在代码编辑器中寻找的基本特性是什么？有没有什么“特色”让你特别烦？
