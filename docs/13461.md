# VS 2003 的记忆之旅。

> 原文:[https://www . site point . com/a-trip-down-memory-lane-with-vs-2003-oh-how-I-never-miss-thee/](https://www.sitepoint.com/a-trip-down-memory-lane-with-vs-2003-oh-how-i-never-missed-thee/)

我从事的一个项目围绕着一个年度事件。而且，对其网站的视觉部分进行一次重大改革也变得势在必行。不幸的是，该网站是一个. NET 1.1 应用程序。当然，这比 ASP 3.0 应用程序要好。但是这需要回到 Visual Studio 2003 的世界。而且，在 Visual Studio 2005 中呆了大约六个月之后，我感到震惊。

一开始，让项目的 webby 部分加载甚至花了半天时间。最后，我不得不重新创建原始应用程序的奇怪的文件夹结构。请记住，在 Visual Studio 2003 中没有文件类型的网站，也没有内置的 web 服务器。一切都需要通过使用 FP 服务器扩展的 IIS 连接起来。敏捷万岁。

在克服了最初的障碍，赢得了与 Visual Source Safe 的斗争之后，我设法打开了一个文件。不幸的是，我在设计器中打开了一个很好的、符合 XHTML 的控件。哦，VS，为什么你觉得

*   标签应该是*   Without matching off? At least VSS is compatible, so I can roll back to the checked-in version.

然而，回忆之旅也有一些快乐的时刻。首先，这个应用程序真正利用了声明性标记和配置文件来管理大部分前端。当我们第四次改变它的用途时，它真的真的出现了。此外，整个布局是 CSS 驱动的。这最初实现起来非常痛苦；但在未来的道路上，回报是巨大的。我们已经交换了两次模板，但从未在 visual studio 中打开应用程序。或者触摸任何保存 CSS 文件和相关图形的东西。

所以，回顾一下:

1.  与更现代的环境相比，VS 2003 糟透了。Lowlights 包括挑剔的 web 应用程序模型和疯狂的设计者，他们以新颖有趣的方式处理代码为乐。
2.  视觉源安全也能咬人。
3.  使用声明性和配置驱动的代码，结合 CSS 进行布局，可以在不打开 Visual Studio 旧版本的情况下做很多好事。

我迫不及待地想回到 Visual Studio 2005 更友好的世界中的当前应用程序上。
