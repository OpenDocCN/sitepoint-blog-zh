# 本周的有用资源

> 原文：<https://www.sitepoint.com/this-weeks-useful-resources/>

HTML 和 XHTML 都规定了哪些元素可以包含在哪些其他元素中。这些规则的权威指南是管理不同风格 HTML 的各种 dtd(文档类型定义)。不幸的是，dtd 读起来并不友好。进入 Cameron Adams 的出色的 [DTD 映射器](http://www.themaninblue.com/experiment/DTDMapper/)，它呈现了一个易于导航的 DTD 树形视图，并且很容易看出哪些元素在哪里被允许。卡梅隆的[博客条目](http://www.themaninblue.com/writing/perspective/2004/05/21/)有更多。

已经有很多关于 CSS 盒子模型的文章(去年我曾尝试解释它)，但是很少有人讨论盒子模型的各层在三维空间中的交互方式。乔恩·希克斯发布了一张漂亮的 [3D CSS 盒子模型](http://www.hicksdesign.co.uk/journal/2004/05/3d_css_box_model/)的图表，清晰地展示了 CSS 这一鲜为人知的方面。

我很不擅长选择配色方案，但谢天谢地，网络上有很多工具可以让这变得更容易。我最喜欢的仍然是 Eric Meyer 的[颜色混合器](http://www.meyerweb.com/eric/tools/color-blend/)，它可以计算两种颜色之间的中点数量，但是 Pixy 令人印象深刻的[配色方案](http://www.pixy.cz/apps/barvy/index-en.html)也非常有用。戴夫·谢伊最近关于[配色方案](http://www.mezzoblue.com/archives/2004/05/14/colour_schem/)的文章有一些很好的提示，安迪·克拉克也有一些关于使用 Macromedia Fireworks 创建一致调色板的[有用提示](http://www.stuffandnonsense.co.uk/archives/creating_colour_palettes.html)(尽管他的技术可以很容易地移植到其他应用程序)。

**更新:**优秀杀手办公室的 s . g . Chipman[在创纪录的时间内完成了](http://slayeroffice.com/index.php?c=/scripts/comment_scripts/comments2.php&id=114)一个安迪·克拉克调色板技术的 [JavaScript 实现](http://slayeroffice.com/tools/color_palette/)！
