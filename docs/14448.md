# PHP i18n 要改进？

> 原文:[https://www.sitepoint.com/php-i18n-to-improve/](https://www.sitepoint.com/php-i18n-to-improve/)

今天 PHP 中的 i18n 很大程度上是一团糟，以至于 Joel Spolsky 去年用；

> 当我发现流行的 web 开发工具 PHP 几乎完全忽略了字符编码问题，轻率地使用 8 位字符，使得开发良好的国际 web 应用程序几乎不可能时，我想，够了。

请注意 Scott Reynen 的回复,查看用 PHP 实现的解决方案。

不管怎样，看起来情况有可能会好转。在 Zend 上查看[这条](http://www.zend.com/lists/php-dev/200303/msg00656.html)有趣的消息…

这看起来非常像问题中的开发者，他使用了 L0t3k 这个手柄，(并且看起来很难在网上找到一个永久的家)是认真的——工作似乎正在 http://cvs.voltex.jp/php-i18n/的 T2 进行。如果有人能帮他解决一些网络空间的问题，发送邮件的地方应该是 T4 hotmail . com。

【http://sourceforge.jp/projects/php-i18n/】T2；

> 当前的 PHP 通过 mbstring 支持 i18n 特性。但是，ZendEngine 和许多 PHP 函数本身并不支持 i18n 特性。本项目旨在原生提供 i18n 特性。结果应该是后来合并到 PHP 项目。
