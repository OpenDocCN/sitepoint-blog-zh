# 试图破译 EULA 的密码？最好有博士学位

> 原文:[https://www . site point . com/trying-to-decipe-the-EULA-better-have-a-PhD/](https://www.sitepoint.com/trying-to-decipher-that-eula-better-have-a-phd/)

本周，在围绕谷歌 Chrome 浏览器的大肆宣传中，有一个关于谷歌新玩具 EULA 的小争议。EULA 要求用户“给予谷歌永久的、不可撤销的、全球范围的、免版税的、非排他性的许可，以复制、改编、修改、翻译、发布、公开表演、公开展示和分发你在服务上或通过服务提交、张贴或展示的任何内容。”

对许多用户来说，这听起来是不祥之兆——为什么谷歌有权在使用 Chrome 时为所欲为？事实证明，谷歌尽可能多地重复使用其 EULA 语言，因此偶尔会有不适用的东西溜进来。谷歌迅速作出回应，修改了令人不快的文字，现在的内容是，“你保留你在这些服务上提交、发布或展示的内容的版权和其他权利。”

不过,“争议”凸显了一些重要的东西:EULA 和隐私政策等内容的文本对用户来说是必要的，但往往它们太长，难以理解，更有可能被跳过。公平地说，谷歌的 Chrome EULA 实际上被认为是最容易理解的之一，但它仍然很长——超过 4000 字——或者用 Times New Roman 12 point 表示大约 10 页。大多数用户不太可能会去读它，而且冒犯性的文字只是在科技记者注意到它之后才被曝光(也就是说，不是你的普通用户)。

BNet 的 Erik Sherman 决定看看我们每天遇到的技术法律文件是多么容易阅读和理解。谢尔曼查看了 23 家科技公司的隐私政策，并按照 0-19+的等级给它们打分。下表按分数显示了可比较的阅读材料，因此您可以大致判断阅读这些政策文件的感受。

| 烟雾等级水平 | 阅读材料示例 |
| 0-6 | 肥皂剧周刊 |
| seven | 真实的忏悔 |
| eight | 女性家庭杂志 |
| nine | 读者文摘 |
| Ten | （美）《新闻周刊》 |
| Eleven | 体育画报 |
| Twelve | 时代杂志 |
| 13-15 | 纽约时报 |
| Sixteen | 大西洋月刊 |
| 17-18 | 哈佛商业评论 |
| 19+ | IRS 代码 |

结果是平均隐私政策为 3442 个单词，平均阅读水平为 15.77(谢尔曼平均得分为 3 分，包括烟雾)——或介于《纽约时报》和《大西洋月刊》之间。这很难让理解这些文档成为火箭科学，但也远远超出了你对有效地与广大用户交流的预期。这表明，这些公司通常不关心他们的用户是否能理解(或愿意费心阅读)他们为了使用所提供的产品或服务而被期望同意的法律文件。

谢尔曼名单上最差的是 Insight Communication，得分高达 20.78——这意味着阅读他们的隐私政策就像阅读美国税法一样。或者换句话说:没有人会真的去打扰。最可读的是雅虎！但它有 5500 个单词，也是最长的。或者换句话说:没有人会真的去打扰。

谷歌的 Chrome EULA 实际上表现不错，根据谢尔曼的方法，它的可读性平均得分为 11.77。这比谢尔曼的隐私政策清单上的任何东西都容易理解，但 10 页的长度仍然让普通用户望而生畏。谷歌 Chrome 隐私政策其实在一个 12.15 和 1100 字就挺好的。可读性和相当短。

在过去的几个月里，我们在科技界看到的最好的面向用户的法律文件来自 Cuil。他们的[隐私政策](http://www.cuil.com/info/privacy/)只有 520 个字长，使用谢尔曼的方法得到了非常可读的 12.29 分。另一个伟大的隐私政策是比尔·蒙克的[政策。它只有 733 个单词，得分为 10.01——这可能是网上最容易阅读的法律文件之一。(最搞笑的政策颁给了](https://www.billmonk.com/about/privacy)[一些糟糕的事情](http://www.somethingawful.com/d/feature-articles/website-privacy-policy.php)——他们严肃的版本也不错。它在理解测试中得分很低，但不到 200 个单词，非常容易掌握。)

教训:让你的隐私政策、EULA 和其他面向用户的法律文件尽可能简短易懂。你的用户会感谢你(你甚至可以通过逆势而为，发布一份普通人也能理解的法律文件，赢得一些免费的媒体报道)。
