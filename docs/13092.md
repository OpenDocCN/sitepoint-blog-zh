这是相当平静的一周，因为我猜想 ASP.NET 世界的大部分人都在忙着准备从今天开始在维加斯举行的 MIX08 聚会。说到这个，我自己也要去拉斯维加斯了。与普遍的看法相反，我不咬人。至少在第二杯酒后。所以，如果你要去的话，那就再见了。如果你不愿意，好吧，看看博客。我会在突破，空地和赌桌之间。

现在,“相当”并不意味着“死了”,所以这里有一个作者发现深刻的帖子和思考的快速汇总。首先是 Rob Conery 对 LINQ 的连续报道。首先，他介绍了在 LINQ 的查询中创建[的方法和原因；然后他展示了一些整齐的](http://blog.wekeroad.com/2008/02/27/creating-in-queries-with-linq-to-sql/) [LINQ 体操](http://blog.wekeroad.com/2008/02/27/linq-gymnastics-creating-a-predictive-query-with-linq/)。继续 LINQ 的主题，不要错过马特·伯塞思进军[建立 LinkedIn 风格的通讯录](http://mattberseth.com/blog/2008/02/building_a_linkedin_style_addr.html)。如果你在线程池中玩，一定要意识到。净 3.5。

首先，GridViewGuy 为我们提供了一个很好的指南，指导我们如何用 JQuery 处理拖放持久性。Devin Rychetnik 出席了黑帽 DC 2008 的新闻发布会，他发布了一个[有趣的活动摘要](http://geekswithblogs.net/devdevin/archive/2008/02/26/120004.aspx)。最难处理的事情之一。NET 处理非托管代码——通常是以 Excel 的形式。来自罗拉的小伙子们开始了一个很有前途的导入 excel 电子表格的系列。完成博客之后，我在此授予 Jesse Ezell 本周的标题，因为他的文章“[抽象:你的代码的避孕套](http://weblogs.asp.net/jezell/archive/2008/02/27/abstraction-a-condom-for-your-code.aspx)”撇开所有的笑话不谈，他提出的概念非常重要，值得了解和喜爱。

从微软出来的时候，有一些值得注意的东西。首先，[IIS.net 已经改版](http://blogs.iis.net/mailant/archive/2008/02/27/introducing-the-new-iis-net-web-site.aspx)。其次， [WSS 3.0 和 MOSS 2007 Service Pack 1 热修复补丁已经发布](http://blogs.msdn.com/sharepoint/archive/2008/02/27/should-i-install-sp1-what-if-i-was-told-to-wait-post-sp1-rollup-and-black-out-rollup-now-available.aspx)。第三，[Sql Server Notification Services 在 Sql 2008](http://coolthingoftheday.blogspot.com/2008/02/notification-services-in-sql-server.html) 正式 DOA。如果您正在等待 Service Pack 1 部署 Windows Server 2008，[您不需要等待](http://blogs.msdn.com/iainmcdonald/archive/2008/02/15/windows-server-2008-is-called-sp1-adventures-in-doing-things-right.aspx)。最后，别忘了报名参加您当地的 [Heroes Happen Here](http://www.microsoft.com/heroeshappenhere/) 发布活动。
