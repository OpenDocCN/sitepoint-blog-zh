# 你会使用谷歌新的页面速度服务吗？

> 原文：<https://www.sitepoint.com/google-page-speed-service/>

谷歌宣布了他们新的页面速度服务。本质上，它是代理服务器、内容交付网络(CDN)和网页优化器的结合，谷歌称这将为大多数网站带来 25-60%的[速度增益](https://code.google.com/speed/pss/gallery.html)。

这项服务免费提供给有限的网络开发者。试用期结束后，Page Speed 将向所有人发布，尽管没有任何细节，但*“定价将具有竞争力”*(来源:[官方谷歌代码博客](http://googlecode.blogspot.com/2011/07/page-speed-service-web-performance.html))。

要使用这项服务，只需注册并添加一个新的 DNS CNAME 记录到您的域名。除了为静态文件提供 gzipped 代理服务器之外，该服务还可以根据 web 性能最佳实践重写您的页面:

*   CSS 文件可以被合并、最小化和移动到 HTML `head`
*   使用谷歌的闭包编译器，JavaScript 文件可以被合并和最小化
*   图像可以缩放和优化

所有特性都是可选的，例如，如果闭包编译器破坏了您的 JavaScript 代码，您可以禁用它。

谷歌在 www.webpagetest.org/compare 提供页面测试对比服务。据估计，SitePoint.com 的主页速度将提高 13%——我怀疑这主要是由于 JavaScript 文件连接。

## 巨大还是麻烦？

根据价格，页面速度服务可能是运行在慢速服务器上的低效静态页面的理想选择。可能比花钱做进一步开发或者托管更划算。

不幸的是，也有一些缺点:

*   不支持裸域，即您必须使用 www.domain.com 而不是 domain.com。真遗憾——我已经从我的网站上去掉了“www”。
*   不支持 HTTPS 页面。
*   不支持 Flash、流式音频、流式视频和超过 50MB 的文件。
*   不支持大于 2MB 的 POST 请求。
*   在运行服务器端代码的 web 应用程序上，您不太可能体验到显著的速度提升。
*   不支持 Blogger、Google Sites 或 Google App Engine 托管的域。

作为一名 web 开发人员，这项服务让我有点不舒服。像许多人一样，我通过合并文件、最小化代码、减少 HTTP 请求和尽可能使用 cdn 来确保我的网站得到优化。为了让页面速度有吸引力，我不想失去控制，配置必须简单，我不想重写代码，价格必须比升级主机便宜。

风险是另一个需要评估的因素。页面速度会提供额外的冗余还是两个故障点？我怀疑这将取决于你网站上静态内容和生成内容的数量。

最后，你愿意把你的网站钥匙交给谷歌吗？他们的服务比大多数更可靠，但这是一种新产品，可能会遇到初期问题。阴谋论者也会将此视为谷歌走向全球统治的又一步。谷歌搜索考虑页面速度因素，那么该公司会成为一个全能的网络主机，破坏不使用其网络的网站吗？

从技术上来说，Google Page Speed 是一个惊人的解决方案，可以提高大多数网站的下载速度——尤其是那些低效编码的网站。然而，我不相信许多优秀的 web 开发人员会采用它。糟糕的开发人员会理解这项服务或足够关心推荐它吗？

时间会证明谷歌的页面速度服务是否成功。请让我们知道你的意见…
