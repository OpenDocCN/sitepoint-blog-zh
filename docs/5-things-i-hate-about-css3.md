# 我讨厌 CSS3 的 5 点

> 原文：<https://www.sitepoint.com/5-things-i-hate-about-css3/>

你知道会这样。是的，我仍然热爱 CSS3，每天都在使用它——但这并不意味着我对一切都很满意。为了平衡 Viki Hoo 关于 CSS3 的 5 个我喜欢的事情，这里有 5 个让我感到沮丧的事情…

## 1.属性重载

编写 CSS2.1 代码时，您查阅手册或参考资料的频率如何？学完基础知识后，我怀疑你只需要偶尔粗略浏览一下。

那些日子已经过去了。也许这只是我的看法，但是新标签的绝对数量是压倒性的。即使我知道一处房产存在，也不可能回忆起它的价值。有人知道所有的背景渐变和边框图片选项吗？

虽然 CSS3 确实节省了时间，但是你需要每隔几分钟查阅一下在线手册或工具。

## 2.供应商前缀

供应商前缀是绝对必要的。我使用它们，并接受这是尖端技术的代价。这并不意味着我必须喜欢他们。其他开发者也不知道——否则我们不会走向 CSS3 灾难。

充其量，前缀导致笨拙的重复。最坏的情况是，它们使用完全不同的语法。有 JavaScript 和服务器端预处理器可以帮助减轻负担，但没有一个是完美的。毕竟，如果一个属性在一个特定的浏览器中不工作，这是你的 CSS、浏览器或者你正在使用的自动代码创建工具的问题吗？

## 3.开发者工具

任何 IDE 供应商都很难跟上 CSS3 开发的快速步伐。有谁知道一个编辑器支持所有新的属性和语法以及所有供应商前缀的差异吗？我不知道。

也不全是坏事；有许多很棒的在线 CSS3 工具。然而，你仍然不会找到许多像样的验证器。

## 4.浏览器支持和 W3C 政治

浏览器厂商有不同的优先级和发布时间表。虽然很容易挑出 IE (IE9 不支持文本阴影)，但即使是 webkit 也不是一直领先于游戏(Chrome 不支持伪元素的单独动画)。一个浏览器总是比另一个更强大，但是炒作和营销经常会阻碍发展。你有多经常假设一个浏览器支持属性 X，写了一些代码，发现它不工作？

主要的供应商都属于 W3C，并且一起工作来达成一致的标准。事实上，他们是竞争对手:

*   苹果被指控实现了 iPhone/iPad 特有的功能，这些功能被宣传为“HTML5 标准”，但从未提交给 W3C。
*   在非 webit 浏览器中支持 webkit 前缀的提案可能会打破我们所知的 CSS3。
*   标准审批过程的缓慢凸显了分歧和政治斗争。

令人沮丧。开发人员陷入了困境，但没有简单的解决方案。而且，在任何人再次建议之前，放弃除一个渲染引擎之外的所有渲染引擎不是一个选项，并且会阻碍创新。

## 5.愚蠢的恶作剧

CSS3 圆角在每个项目中都节省了数小时的时间。不需要复杂的布局或多个背景图片。不幸的是，它们不能在 IE8 或之前版本的浏览器中运行。因此，您有三种选择:

1.  忘记 CSS3，使用传统的开发技术。
2.  接受没有两个浏览器呈现相同。IE8 已经有三年历史了，是在行业赶上 HTML5 潮流之前发布的。在 IE8 中浏览的页面看起来与 IE9、Firefox、Chrome、Safari 和 Opera 不同(也更难看)。
3.  尝试使用 JavaScript 填充程序使 IE8 看起来更好，这些填充程序通常实现旧的特定于 IE 的过滤器。

我推荐选择二。你可能需要教育你的客户，但他们的成本会降低，最终，这可能会说服用户升级他们的浏览器。

不幸的是，我一直在寻找添加过多垫片的网站，徒劳地试图实现像素的完美。虽然页面可能看起来更好，但它会导致庞大的代码，IE8 通常会变得令人沮丧地慢。我不能相信他们的客户对结果感到满意。

## 概括起来

CSS3 很棒。就实用性而言，它比 HTML5 或各种 JavaScript APIs 进步更快，也更有用。

浏览器兼容性是一个问题，但 Viki 相信“它们统一的那一天终将到来”。我不相信。当 CSS3 完全实现的时候，我们将会到达一个点，但是到那时我们将会抱怨 CSS4！
