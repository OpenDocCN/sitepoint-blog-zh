# 零碎东西

> 原文:[https://www.sitepoint.com/odds-and-ends/](https://www.sitepoint.com/odds-and-ends/)

上周我发现了一些有趣或有用的链接:

[使用 JavaScript 在光标处插入](http://www.alexking.org/blog/2003/06/02/inserting-at-the-cursor-using-javascript/)

这是提高表单可用性的一个非常有效的技巧，这也是我最近的 SitePoint 文章的主题。它对于内容管理系统的文本输入字段特别有用——例如，为用户提供一个“添加图像”小部件，提示他们输入 URL 和 ALT 属性，然后在他们当前的光标位置插入相应的 HTML。Alex King 的代码示例可以在 IE 和 Mozilla 中运行，这两种软件都提供了不同的访问当前光标位置的方法。

IFR:FIR 替代方案

你可能听说过图像替换(如果不是这个[戴夫·谢伊的精彩总结](http://www.mezzoblue.com/tests/revised-image-replacement/)应该能让你跟上进度)。IFR 是肖恩·因曼的想法的有趣转变，它使用 Javascript 和 Flash 从页面标题中提取文本，并用 Flash 渲染的等效内容替换每个标题。这可以实现高级文本样式效果，还可以让您使用主机操作系统上没有的标题字体。

[ALA 和 JavaScript——五个月后](http://www.quirksmode.org/oddsandends/ala_javascript.html)

PPK 是[QuirksMode.org](http://www.quirksmode.org/)的维护者，也是 Javascript 及相关技术的发言人。在这篇文章中，他回顾了来自[的六篇关于 Javascript 的文章，对文章的质量和它们所支持的技术的有用性进行了评级。我本来打算单独链接到 ALA 的文章，但 PPK 的评价给原文增加了额外的价值。](http://www.alistapart.com/)

[分离行为和结构](http://www.digital-web.com/columns/keepitsimple/separating_behavior_and_structure.shtml)

PPK 的另一篇文章，这次是一篇拥护智能 Javascript 和不引人注目的脚本的优点的专栏文章。他插入了我的用 JavaScript 增强结构化标记的文章，这至少让人放心！PPK 的核心信息是**停止在 XHTML 文件中编写 Javascript 指令**——这几乎把我对 JavaScript 最佳实践的想法浓缩成了一句话。

[XHTML 1.0 严没准备好黄金时间？](http://www.7nights.com/asterisk/archives/xhtml_10_strict_not_ready_for_prime_time.php)

D.基斯·罗宾逊在他的博客上主持了一些精彩的讨论，这是其中之一。实际上，我不同意 Keith 最初的帖子:如果我要使用 XHTML，我会毫不犹豫地使用 XHTML 1.0 Strict，因为我不需要 XHTML Transitional 中的任何特性，而且到目前为止，我已经请求避免在这样做时出现任何奇怪的问题。对于一般项目，我实际上更喜欢 HTML 4.01 Strict，但这可能是另一篇文章的主题。无论如何，Keith 博客上接下来的讨论很值得一读。
