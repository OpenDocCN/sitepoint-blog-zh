# 针对超文本的犯罪

> 原文：<https://www.sitepoint.com/crimes-against-hypertext/>

就像*信息高速公路*和*网络空间*一样，术语*超链接*和*超文本*似乎已经过时了。尽管如此，超文本仍然是网络的基础——它的核心特征之一。不起眼的 HTML `<a>`标签是产生将纯文本转换成超文本的超链接的唯一元素。当这样一个网页的核心特征似乎被认为是理所当然的时候，我仍然感到惊讶和恼火。糟糕的链接实践很常见——编辑上的和视觉上的——这对可用性有直接的负面影响。

## 流氓画廊

是时候揪出这些不法之徒了，所以这里是我的超文本流氓库——他们都是因危害可用性罪而被通缉的罪犯！

**重言式**

重言式链接并不是一个新的创新，但它可能会突然出现在一个超文本块中，没有任何征兆。你可能见过类似这样的超文本:

```
You can find two great examples <a href="/">here</a> 
and <a href="/">here</a>. <a href="/">Click here</a> 
for another one!
```

超链接意味着通过单击链接可以到达的目的地；同义反复链接只是用来陈述显而易见的事实！

**休闲套装**

休闲西装的联系只是模糊地指向。如果你想跟随它，你只能靠自己。这里有一个例子:

```
In fact, you can <a href="/">find</a> <a href="/">so</a> 
<a href="/">many</a> <a href="/">examples</a> of poor 
linking you may think the world no longer cares.
```

懒得写有意义的超文本？把一大堆单词链接起来，让用户自己处理就行了。

**中间人**

一些作者喜欢把他们的超文本承包给一个中间人。URL 缩短服务是有用武之地的，比如像 [Twitter](http://twitter.com/) 这样的网站，但是没有理由在你的超文本中使用它们。在没有任何其他上下文的情况下，用户可以退回到检查链接的 URL。通过使用 URL 缩短服务来消除这一点，就相当于让他们在没有桨的情况下顺流而下。

**干扰**

你经常在类似于[维基百科](http://wikipedia.org/)和 [Everything2](http://everything2.com/) 这样的网站上看到这种情况——这些单词链接到与周围文本几乎没有关联的信息。例如:

```
Hyperlinks are the <a href="/a_page_about_glue">glue</a> 
that holds the Web together.
```

**逝者**

对抗逝者，超文本永远没有机会。你有没有注意到，很多主流新闻媒体网站在文章正文中没有提供链接就引用了其他网站？这叫超文本吗？简直是屠杀！

**跟踪狂**

当你在它上面的时候，你只会看到一个跟踪者链接！这种情况越来越少见了，但偶尔还是会看到例子。跟踪者链接在外观上与周围的文本非常相似，你甚至不会知道它在那里。

**电话推销员**

超文本的祸害，电话销售链接包括一个悬停框，显示目标网页的缩略图预览。弹出的链接目标的小而模糊的缩略图会掩盖周围的文本吗？那些双下划线的广告链接呢？用户不应该经历这些。

## 好的超链接

低质量的超文本是可用性的灾难，会导致烦恼、困惑和焦虑。用户期望链接，并且链接是相关的和有用的。一个好的超链接应该与周围的文字相关，并为用户提供足够的信息，让他们做出明智的决定，决定是否离开当前页面，继续前进。

下面是一个有用的超链接示例:

```
For excellent examples of finely crafted hypertext 
look no further than <a href="http://kottke.org/">kottke.org, 
the online home of Jason Kottke</a>.
```

链接的文本必须有相关性，因为这是用户收到的关于链接性质的第一个提示。好的链接文本的测试是它是否能独立在网页上，在超文本之外，仍然有意义。

链接的风格也必须不同于周围的文字。它们可以是除了蓝色以外的另一种颜色，只要它不同于普通文本，并且页面中的所有链接都是相同的颜色，所以它们会清晰可见。

一个`title`属性是可选的，但是应该独立于提供一个上下文来使用，因为工具提示只在使用鼠标时出现。如果链接文字足够，反正也没必要。

精心制作的超文本易于阅读和使用，坦率地说，易于创建！你犯了针对超文本的罪吗？
