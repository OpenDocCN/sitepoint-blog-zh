# Mozilla Jetpack:附件主管 Nick Nguyen 访谈

> 原文：<https://www.sitepoint.com/mozilla-jetpack-interview-nick-nguyen/>

Jetpack 是一个新的扩展系统，允许开发者利用他们现有的网络技能来创建 Firefox 插件。我采访了 Mozilla 的 Nick Nguyen，讨论该项目的未来。

Craig Buckler: Firefox 插件可能是有史以来最成功的扩展系统。一个巨大的社区已经开发了超过 12，000 个附加组件，那么为什么还需要 Jetpack 呢？

Nick Nguyen:正如你提到的，我们的附加平台非常成功。话虽如此，附加组件社区已经发现了许多需要改进的地方，包括开发的容易性、工具的缺乏和调试的困难，我们认为可以用 Jetpack 来探索这些地方。

Jetpack 的愿景是扩大附加开发人员的数量，以包括任何有想法的 web 开发人员——因此强调 Web 技术。

你认为 Jetpack 是 XUL 分机的替代品吗？

NN:正如无处不在已经被 Mozilla 社区所接受一样，Jetpack 是一个实验平台，将由社区成员和 Mozilla 实验室共同驱动。因为我们正在尝试以前没有做过的事情，所以我们期望一些事情会成功，而另一些事情需要修改。这在新的实验平台上比在现有平台上更容易也更安全，所以我们将继续发展基于 Jetpack 和 XUL 的扩展。

我们认为 Jetpack 是对现有平台的补充——我们从 Jetpack 中学到的一些东西可能会移植到现有平台中，或者 Jetpack 可能会继续发展，成为快速扩展开发的一个独立途径。

**CB:是否存在混淆或割裂插件开发者和用户的风险？**

NN:混淆的可能性总是存在的，但是与我们的社区交流对我们来说很重要。我们已经努力澄清，Jetpack 不是当前系统的大规模替代，我们将继续这样做。

当前的 Jetpack API 提供了对浏览器标签、通知和状态栏的访问。它最终会像 XUL 一样提供对 chrome 界面的完全访问吗？

NN:我们正在向社区寻求帮助，并且正在考虑向 Jetpack 开放哪些 UI 元素。我们正在探索开发的简易性和平台灵活性之间的必要平衡。我们的目标是让 Jetpack 具备足够的功能来实现大部分的 Firefox 扩展，从而让开发人员能够灵活地构建自己的想法。

你计划为 Twitter、Delicious 和 Google Maps 等系统提供外部 API 库。你的选择标准是什么？任何网站开发者都可以要求加入吗？

NN:我们已经尝试识别一组常用的库，但是我们确实想达到这样一个程度，即任何 Javascript 库都可以很容易地被引用并包含在 Jetpack 扩展中。

CB:扩展可能会引入恶意的浏览器代码。Jetpack 将如何解决安全问题？

NN:在当前开发的早期阶段，Jetpack 的安全性是值得期待的。我们希望探索该平台通过架构提高安全性的内在方式。例如，我们希望通过使用库来最大化代码重用，并将定制扩展代码的数量限制到最低的行数。我们认为，至少这使得代码审查更加容易，并且为以编程方式理解扩展运行所需的访问级别提供了机会。随着平台的发展，如何向用户和评论者公开这些信息是我们计划回答的另一个公开问题。

Mozilla 会提供一个允许用户搜索扩展的 Jetpack 库吗？

正如[addons.mozilla.org](http://addons.mozilla.org/)帮助扩展开发者分发和度量他们的插件一样，我们将很快推出一个画廊，为 Jetpack 开发者做同样的事情。这将是什么样子，它将被称为仍有待确定。

火狐的更新可能会破坏现有的插件，或者要求开发者更新他们的代码。Jetpack 能做些什么来缓解这个问题？

NN:由于 Jetpack 在附加组件和底层浏览器代码之间提供了一个抽象层，维护跨浏览器版本的兼容性就变成了更新核心平台而不是单个扩展的问题。

火狐插件经常被指责使浏览器变慢。你在 Jetpack 中解决这个问题了吗？

NN:糟糕的插件性能通常是没有从对 Firefox 平台的深入了解中获益的实现的结果。这部分是由于当前扩展开发的灵活性和“裸机”特性。

通过促进代码重用和抽象层，从而消除需要这种知识的负担，我们可以通过提高底层平台的性能来更容易地提高所有扩展的 Jetpack 性能。

**CB: Jetpack 目前本身就是一个附加组件。最终会集成到 Firefox 中吗？**

NN:喷气背包是一个实验。Jetpack 的各个方面可能会成为核心平台的一部分，或者 Jetpack 可能会沿着一条独立的道路发展。现在确定还为时过早——就像我们在 Mozilla 做的所有事情一样，社区的参与和反馈将有助于推动这一决定。

Jetpack 会进入其他 Mozilla 项目吗，比如 Fennec(移动浏览器)或者雷鸟？

NN:当然有可能，但是现在说还为时过早。Firefox 扩展开发的许多方面同样适用于 Mozilla 产品，以改进 Firefox 附加组件开发为目标的 Jetpack 将有助于了解如何改进所有 Mozilla 产品的附加组件。

CB: Jetpack 扩展可以跨浏览器兼容。有没有其他浏览器供应商对这个项目感兴趣？

NN:是的，我们很高兴看到扩展开发在浏览器领域产生了如此大的兴趣。至于细节，我将把它留给其他供应商来透露他们的兴趣。

Jetpack 团队正在寻求开发者的参与。到目前为止，反应和反馈如何？

NN:反响非常好。我们看到了 Jetpack 讨论组中的积极参与，我们计划随着我们在平台中构建更多功能，继续发展该社区。

Jetpack 的工作是什么时候开始的，最终的发布日期是什么时候？

NN: Jetpack 的灵感来自于社区反馈以及类似于 FUEL 的项目，这些项目都有改善开发者体验的类似目标。实际的集思广益和实施开始于 2009 年初，但我们没有最终的发布日期。我们将以实验性的速度和节奏发布，直到社区感觉我们已经接近终点。

CB:谢谢你，尼克。祝喷气背包项目好运。
