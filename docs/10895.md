# 网页设计师应该编码吗？…民意调查结果

> 原文:[https://www . site point . com/should-we B- designers-code-the-poll-results/](https://www.sitepoint.com/should-web-designers-code-the-poll-results/)

我们最近的调查问到:*一个网页设计师应该具备什么水平的 HTML 和 CSS 编码知识？*在[中详细讨论过，设计师应该有编码技能吗？](https://www.sitepoint.com/should-designers-have-coding-skills/)投票结果如下:

*   70%的受访者表示，网页设计师应该具备良好的 HTML 和 CSS 编码知识
*   25%的人表示基本知识足够
*   5%的人认为一个好的设计师可以在没有编码知识的情况下创建一个伟大的网站设计。

也有一些很棒的评论…

> 知道在浏览器中可以完成什么，用户如何与设计交互，以及如何使用 CSS、图像和其他元素实现设计，这是非常明显的优势。

<cite>BPlunkett</cite>

问题的关键在于你对“网页设计师”的定义。这是主观的，取决于你自己的经历。对一些人来说，这是一个图形艺术家谁创造网页设计模型。对于其他人来说，它是一个能够创建具有 CSS 效果和 JavaScript 交互的高质量布局代码的人。大多数人会有一个介于两个极端之间的观点。然而，我有点惊讶地发现，相当大比例的受访者表示良好的编码技能是必不可少的。在某些情况下，这取决于组织的规模和类型。大公司可能有特定技术的专家，而个体贸易自由职业者通常需要更广泛的技能，不可避免地要学习 HTML 和 CSS。最后，看你的志向了。许多有才华的图形艺术家不想编码。很多程序员对设计兴趣不大。然而，两人都可能自称是网页设计师，因为这是整个行业都在使用的一个术语。

## HTML 和 CSS 比你想象的要难…

有一种普遍的误解，认为 HTML 和 CSS 很容易。从 IT 人员的简历中抽取一份样本，你会发现有高得离谱的数量将 HTML 作为一项技能。这些人中的许多人也知道国际象棋的规则——但是有多少人可以自称是大师呢？虽然 HTML 和 CSS 不是编程语言，但它们有微妙的复杂性，除非你有相当的开发经验，否则你不会理解。图形艺术家依赖于精确的定位，程序员依赖于严格的编码惯例，然而这种精确程度在浏览器世界中很少出现。很少有人精通 HTML 和 CSS，无论他们是从设计还是编程学科迁移过来的。我们是否应该期望每个人都对浏览器的怪癖、可用性、语义、可访问性、渐进增强等有深入的了解？

## …但这不是不理解媒体的借口

很难跟上网络技术和最佳实践技术的发展。然而，设计师没有理由不理解网络作为媒介的基本原理。同样的错误不断出现，令人沮丧。以我的经验来看，最好的图形艺术家理解网络的基本概念，并应用良好的设计实践。最好的开发人员知道如何建设性地分析设计，并将其转化为高质量的代码。你很难找到能两者兼得的人……*真正的*网页设计师是罕见的。继续阅读:[网页设计师新手常犯的 10 个错误…](https://www.sitepoint.com/top-10-web-design-mistakes/)
