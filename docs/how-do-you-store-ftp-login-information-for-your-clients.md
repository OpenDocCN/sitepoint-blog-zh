# 你如何为你的客户存储 FTP 登录信息？

> 原文：<https://www.sitepoint.com/how-do-you-store-ftp-login-information-for-your-clients/>

存储 FTP 访问信息的一种常见方式是在您的 FTP 客户端中。毕竟，那是你使用它的地方，所以它应该在那里。但是，如果 FTP 应用程序崩溃或者您的配置文件损坏了，会发生什么呢？一个 FTP 程序是否足够安全，可以成为你存储客户端登录信息的唯一地方？

在去年永久更换为 Mac 电脑之前，我在 18 个月的时间里经历了三次电脑崩溃。我知道所有关于数据故障的事情，并且做了 Craig Buckler 在他的“ [8 个从电脑故障中幸存的技巧](https://www.sitepoint.com/8-tips-for-surviving-pc-failure)”帖子中列出的所有事情。成功了。我保存了几乎所有的数据。但是，即使使用双重备份并将我的 FTP 配置文件导出到我的备份数据(这是我当时使用的 FTP 客户端的手动过程)，我也无法自动重新创建我的所有登录。自从上次备份数据以来，要么是密码被更改，要么是新的网站被添加到组合中，我不得不在数千封电子邮件中挖掘，试图找到我需要的信息。

我还在我的地址簿中存储了密码和登录信息，作为每个客户的记录。但是这也不是万无一失的。备份数据时丢失了一些笔记，当我转移到新的联系人管理系统时，更多的笔记丢失了。

我用过或考虑过的一些其他选项包括:

*   密码管理工具，如 [KeePass](http://keepass.info)
*   基于网络的 FTP 程序，在线保存您的数据，如 [AnyClient](http://www.anyclient.com)
*   Excel 电子表格
*   在线(甚至纸质)笔记本

目前，我将我所有的客户数据(包括 FTP 登录)存储在一个基于 web 的项目管理系统中( [Intervals](http://www.myintervals.com) )，该系统是安全的，我的分包商、客户和任何需要访问数据的人都可以访问。如果每个人都使用系统中的数据，将它放在一个中心位置还可以消除处理过期密码的问题。

但是我为拥有 SSL 的项目管理服务支付了高昂的月费，它可能比这类问题本身所需要的更健壮。幸运的是，我需要该服务提供的许多额外功能来有效地运行我的业务，但它可能不是每个需要存储有限信息量(如 FTP 登录)的人的选项。

有没有存储这些信息的最佳方式？如何管理多个客户端的 FTP 登录，使它们易于访问和安全？
