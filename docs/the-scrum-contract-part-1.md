# Scrum 契约(第 1 部分)

> 原文：<https://www.sitepoint.com/the-scrum-contract-part-1/>

以下是摘自我们的书《Scrum:新手到忍者，作者是 M. David Green。世界各地的商店都有出售，或者你可以在这里买到电子书。

在本书的前半部分，我们已经抽象地讨论了 scrum。我们讨论了它是什么以及它是如何工作的。我们把它分解成角色、仪式和艺术品。但是 scrum 不仅仅是关于细节。Scrum 是关于人的。

在下一部分，我们将看到 scrum 的实际应用。我们将看看 web 和移动开发团队如何使用 scrum 的各种特性。我们将从已经介绍过的团队开始。

### 初学者的思维

开始 scrum 会感觉有点不确定。有人质疑它将如何影响我们的工作方式和互动方式。还有一些关于 scrum 本身如何与组织的层级互动的问题。正如我们在第二章中看到的，有些人带着先入为主的观念进入 scrum，这些观念会限制 scrum 的潜力。

在后面的章节中，我们将围绕 scrum 团队可能出现的问题进行一些故障排除。但是在这一章，我们将讨论同意积极参与 scrum 过程意味着什么。没有真人的参与，Scrum 无法发挥其全部潜力。Scrum 依靠每个人来履行他们角色的职责，scrum 尽可能地让这些角色变得轻量级和易于支持。

我喜欢称之为 scrum 契约。与我们之前讨论的工件不同，这不是一个物理契约。这是一种在接近 scrum 时对人们有用的心态。这不需要抛开健康的怀疑态度，但需要用开放的心态取代先入为主的观念。

scrum 契约围绕着几个关键概念:

*   尊重角色
*   拥抱透明
*   保持界限
*   充满希望地迭代
*   共享定义

任何试图在团队中有效工作的人都熟悉其中的一些概念。其中一些特别适用于 scrum 的工作方式。所有这些都依赖于为共同目标而合作的意愿。

### 尊重 Scrum 角色

正如我们在第三章中看到的，scrum 定义了许多人们可以担当的角色。这些角色在 scrum 的上下文中是有目的的，每个人都需要理解它们，不管它们与一个人在组织中的职位或头衔有什么关系。Scrum 不是要取代组织层级。事实上，scrum 试图尽可能地独立于组织层级。

Scrum 希望人们能够区分他们的工作和他们在 scrum 过程中的角色。这并不意味着 scrum 免除了你的工作义务。scrum 提供的是你正在做的工作的另一个视角。当你在 scrum 项目中担任一个角色时，你同意积极参与到一个共享的过程中来，以促进每个人的工作。

在 scrum 团队中承担角色的一部分意味着愿意在 scrum 仪式中出现的问题上与你的同事平等合作。任何参与 scrum 的人都应该自由地分享他们对项目和过程的观点，即使这些观点与主流相左，或者挑战组织层级中职位更高的人的权威。当冲突发生时，scrum 主管有责任帮助调解和促进对话。

更基本的是，参与 scrum 假设你愿意成为团队的一员。scrum 的角色和仪式的建立是为了让每个人都有机会和义务参与进来。团队中不分享观点和想法的成员会让团队的其他成员失望。作为 scrum 团队的一员，不管你在公司的职位如何，你的目标都是帮助支持每个人的过程。因为小组是小而亲密的，而且参与的规则是明确定义的，scrum 为害羞的人提供了一个安全的空间来尝试表达自己。

由于典型的公司会议和 scrum 仪式之间的区别，人们可能会不时地对人们应该如何表现感到困惑。当出现这样的问题时，scrum master 应该是最终的仲裁者。由每个团队自己定义什么是最有用的行为标准，并适当地执行它们。

最终归结为尊重你的同事，尊重过程，尊重你在过程中的位置。如果你有问题，或者你认为有些地方不太好，总有机会在回顾会上提出来。不要错过积极参与的机会。

### 提供透明度

为了让 scrum 有效地工作，团队中的每个人都需要愿意让其他人知道他们在流程的所有阶段都在做什么。有人称之为透明。它允许每个人都知道其他人在做什么，这样每个人都可以对项目的状态有一个更大的了解，以及他们可以如何提供帮助。

透明度有多种用途。scrum 的部分价值在于，它使得公司中的任何人都可以看到关于项目状态的大量信息，因此从事项目工作的人不必被打扰。通过为团队保留一段非常短、非常明确的时间来每天互相更新，scrum 避免了每个工程师面对那些只想知道他们正在做什么的人的干扰的可能性。

透明性还允许工程团队之外的人跟踪对他们特别重要的问题的状态。Scrum 以这样一种方式定义 sprints，即它们不会被新的故事打断，但是客户和组织中的其他人经常有他们想要引入的变化，并且他们可能认为这些变化是紧急的。scrum 的透明性允许他们一目了然地看到团队现在正在做什么，以及为什么这些事情被优先处理。

scrum 支持的透明性与微观管理无关。scrum 的优势之一是它能够为团队中的个人创造尽可能多的非管理时间来管理他们自己。没有人比从事一个故事的工程师更了解完成这个故事需要什么。Scrum 提供每日站立作为透明度的接触点，这样工程师可以在更新团队状态时分享有价值的见解，然后在最少干扰的情况下工作。

Scrum 假设参与过程的每个人首先都有意识地选择在团队中工作。在团队中一起工作意味着让每个人都知道你在做什么，并且愿意分享信息和资源。scrum 在 web 和移动开发中如此有效的原因之一是，对于这些类型的项目所涉及的工作类型来说，为了一个共同的目标一起工作通常比孤立个人并试图在事后协调他们的工作更有成效。
