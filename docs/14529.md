# 分开但平等？

> 原文：<https://www.sitepoint.com/separate-but-equal/>

曼联足球队最近因其网站的[无障碍版本](http://www.manutd.com/access/)受到抨击，该版本提供各种无障碍小工具，但包含的内容远不如他们的[常规网站](http://www.manutd.com/)多。SitePoint 的特伦顿·莫斯[首先提出了这个问题](http://ecademy.com/node.php?id=25505)，并在他的最新文章[中将其作为例子](https://www.sitepoint.com/article/blunders-big-player-2)。Accessify 也有[的报道](http://www.accessify.com/archives/2004_06_20_news-archives.asp#108789829699055748)，来自 W3C 的 Matt May 也发表了一篇[杰出的长篇演说](http://www.bestkungfu.com/index.php?p=507)来反对这种糟糕的可访问性实践。

从上面可以清楚地看到，内容驱动网站的独立“可访问”版本是不被可访问性专家所接受的。然而，这同样适用于 web 应用程序吗？一个很好的例子是谷歌新的网络邮件服务 Gmail T1。Gmail 是我用过的最流畅的网络应用之一，这要归功于它对 JavaScript 的广泛依赖，从无需完全刷新就能加载额外的页面信息，到几乎所有常见操作的键盘快捷键。不幸的是，光滑的界面代价很高:网站[打破了书中几乎所有的可访问性规则](http://diveintomark.org/archives/2004/04/10/gmail-accessibility)。

谷歌提供一个可访问的 Gmail 替代版本有意义吗？目前，如果没有现代的桌面浏览器，这项服务是无法使用的。另一个入口点仍然可以提供对 Gmail 帐户中包含的数据的完全访问，但没有非 JavaScript 用户代理无法支持的许多界面增强。当考虑替代的网络邮件界面时，似乎如此令人信服地反对曼联的无障碍版本的论点远没有那么相关；毕竟，多年来，多个电子邮件客户端都可以与同一个 IMAP 服务器进行对话。

我要向上面提到的几个可访问性专家发出 ping，看看他们是怎么想的。
