# 蒂姆·伯纳斯·李爵士:抱歉有斜线

> 原文：<https://www.sitepoint.com/sir-tim-berners-lee-http-slashes/>

被认为发明了万维网的欧洲粒子物理研究所科学家蒂姆·伯纳斯·李爵士为出现在每个网址开头的双斜线(//)符号道歉:

> 真的，如果你仔细想想，它不需要双斜线。我可以把它设计成没有双斜线。
> 
> 当时这似乎是个好主意。

蒂姆爵士解释说，当他第一次设计网络地址系统时，他没有预料到双斜线会引起麻烦。发现的问题包括:

*   对于网络新手来说，这看起来太专业了。
*   很多用户误将其称为“反斜杠-反斜杠”。
*   这是 URL 语法错误的最大原因之一。
*   难以想象的大量打印机墨水和纸张被浪费在不必要的字符上。

幸运的是，在过去的几年中，斜线已经不再是一个问题，因为大多数浏览器会在任何非完全限定的地址前添加“http://”。

然而，我想我们都可以原谅蒂姆爵士。也许他本可以设计出一个更简单的称呼系统，但是如果不是他 30 年前的远见和发明，我们中很少有人会有工作。

双斜线有问题吗？你向客户解释 URL 符号有困难吗？蒂姆爵士真的需要道歉吗？
