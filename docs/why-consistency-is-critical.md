# 为什么一致性至关重要

> 原文：<https://www.sitepoint.com/why-consistency-is-critical/>

拉尔夫·瓦尔多·爱默生说，“愚蠢的一致性是心胸狭窄者的心头之患”。

许多开发人员会全心全意地同意这种观点——特别是如果他们已经被过分热心的可用性人员强加了“一致性”的话。

##### 用户需要一致性

毫无疑问，一致性对用户来说很重要。一致性使网站更容易使用，因为访问者不必在四处走动时学习新的技巧。

网站应该内部一致:应该建立标准和惯例，并在所有内容中应用。例如，一个用户在一个页面的右上角遇到“搜索”，如果它被随意移动到网站其他页面的不同位置，就会出现问题。

各站点还需要外部一致，即与一般实践一致。用户会倾向于应用他们在别处学到的规则，即使这些规则实际上并不适用于当前的网站。他们把自己的经验和期望带到你的网站上。如果你忽视了这一点，你就有引起困惑和疏远的风险。

##### 一致性是有意义的

一致性具有商业意义。它允许您为页面和其他设计元素开发一小组模板，并在整个站点中应用它们。它使编写、发布和维护更容易，并支持 XML、CSS 和其他依赖于结构化内容的技术的实现。

虽然一些开发人员认为一致性是一个束缚，但它也有许多好处。它消除了一次又一次做出相同决定的需要(我们的字段标签是左对齐还是右对齐？)，而且它支持使用模板和预定义的代码片段。

##### 一致性的各个方面

一致性适用于网站中的许多元素。让我们考虑其中的一些，理解它们为什么重要，并学习我们如何在应用它们时保持一致。

***语言***

“双顶置凸轮”和“双顶置凸轮轴”一样吗？我的汽车修理工知道，但我不知道。对同一事物使用多个术语是造成混乱的一个重要原因。您可以通过只使用认可的术语来减少这种混淆。

例如，如果你在销售电话，你可能会选择“移动电话”，而不是“手机”、“移动电话”或“手机”等同义词。当然，用户仍然可以通过这些术语中的任何一个进行搜索，理想情况下，甚至可以通过它们的拼写错误进行搜索。例如，电信公司的产品索引应该列出“移动电话”、“手机”和“手机”。

你也要有一致的“声音”。不要从一页上的热情友好变成另一页上的法律和威胁。如果你在网站的某一部分称用户为“你”，避免在另一部分使用“我的电子邮件”这样的术语。

如果没有对你的内容进行编辑性的概述，保持语言使用的一致性是非常困难的。至少，要确定一个人负责在发布前审查你的网站内容。

一致的语言会让你的网站看起来更简单、更权威。

***UI 元素***

人们往往倾向于对用户界面元素做一些巧妙的事情——通常是出于好意——这导致了一些问题。例如，当用户选择一个选项按钮(或单选按钮)时，一些站点执行一个动作。如果是意外的，这种行为会非常令人困惑。牺牲一致的行为来保存一次点击可能不是一个好的策略。

UI 元素一致性的最基本要求是按照最初的定义来应用它们。例如:

*   选项按钮(单选按钮)仅用于互斥项目，因此用户只能选择一个选项
*   复选框，如果用户可以选择一个或多个选项

避免应用不必要的图形处理，以免让用户困惑，让他们使用他们正在使用的 UI 元素类型。选项按钮应该是圆形的，复选框应该是方形的。可编辑字段应该有白色背景；不可编辑的字段应为灰色。

作为一个例子，考虑一下[谷歌“高级搜索”页面](http://www.google.com/advanced_search?hl=en)，它包含文本字段、按钮、下拉框、选项按钮、链接和各种其他元素，所有这些都清晰可辨。

相比之下，万宝龙网站的 Flash 版本虽然视觉上吸引人、时尚且有品牌效应，但使用和导航相对困难。

***布局***

如果你的网站是世界上第一个，你可以完全自由地设计你的网页。你可以决定主导航应该在屏幕的右边，次导航在底部，内容在左上角。

然而，现有的一般实践限制了您可以做的事情。当然，你仍然可以选择一个不寻常的布局，但是这样做，你可能会让你的用户感到困惑，他们在互联网上的一般经验会让他们期待某些约定。

例如，公司徽标应该出现在页面的左上角。[汇丰银行](http://www.hsbc.com)、[有线](http://www.wired.com)和[无标识](http://www.nologo.org)都是非常不同的组织，拥有非常不同的网站，但每个组织都将标识(带有相关主页链接)放在左上角。其他一些公认的惯例是:

*   顶部或左侧导航(或两者都有)
*   主导航下方的面包屑(如果使用)
*   页面中央的内容
*   右侧是相关材料和促销信息
*   在右上角搜索(尽管有很多例外)

在为你的网站选择布局选项时，花一个小时浏览类似的网站，感受一下一般的做法。除非你有充分的理由，否则不要采用这种常规做法。

人们对位置有很强的记忆力，你的设计可以利用这一特点，为屏幕元素(导航、搜索、登录、内容)保留特定的位置，并一致地应用它们。在可用性测试中，我们已经看到用户完全被看似相同，但实际上已经改变的导航元素所迷惑——用户根本没有注意到这种改变，也找不到他们想要的内容或功能。

***功能***

一致性的另一个方面集中在你的网站支持的功能上。例如，公共交通网站可能会提供时刻表、票价信息和行程规划。快速浏览一下这样的网站，例如柏林的 BVG、T2 的墨尔本的 Metlink 和三藩市的 T4 Bart，就会发现旅行计划或链接会出现在主页上。

类似地，电影院网站可能会提供订票系统。如果你的网站不能提供这种预期的功能(例如，由于预算原因)，你至少应该提供同等功能的信息(例如预订的电话号码)。

***视觉处理***

视觉处理通常是开发者拥有最大自由度的领域。然而，一旦你选择了一种视觉语言，坚持下去是很重要的。当用户浏览你的网站时，视觉处理让他们确信他们仍然在同一个地方，给人一种专业和可靠的整体印象。

视觉处理不仅适用于标识和导航等明显的元素，也适用于内容元素、字体和背景。CSS 的使用可以帮助确保这些元素的一致性。

有关一致的视觉处理和布局的示例，请参见 [Red Hat 网站](http://www.redhat.com)。相对少量的强识别元素提供了整个站点内容的一致呈现。主导航是显而易见的(并加强品牌)。二级和辅助导航可以立即识别。列的应用是一致和合理的(密集拷贝使用单个列；具有多个链接的概览页面的两列或三列布局)。所有表格都使用相同的布局、颜色和字体。

##### 一致性可以成为你的朋友

一致性是你可以利用的优势，而不是束缚你的束缚。它可以帮助您最小化开发和维护成本，使您不必担心相对琐碎的设计问题，并为您的客户提供增强的用户体验。
