# 微软准备紧急 IE 补丁

> 原文：<https://www.sitepoint.com/microsoft-emergency-ie-update/>

继法国和德国政府建议[用户应该停止使用 Internet Explorer](https://www.sitepoint.com/german-government-stop-using-ie/) 之后，微软决定在 2010 年 2 月 9 日下一次更新之前发布一个紧急浏览器补丁。

IE 漏洞被认为是最近来自中国的对谷歌 GMail 和其他系统攻击的主要目标之一。这次被称为“极光”的攻击导致几个政府和安全公司发布了关于 IE 的警告，并建议用户更换浏览器，直到微软推出补丁。*(尽管一些人采取了更谨慎的态度，声称攻击很少发生，切换浏览器可能会给用户一种错误的安全感。)*

微软继续否认存在重大问题，但他们无法在这种情况下真正获胜。由于没有发布补丁，公众会质疑 IE 的安全性，并可能促使企业和个人放弃该浏览器。通过在正常发布时间表之前发布补丁，微软似乎承认这个缺陷和报道的一样严重。

[微软安全响应中心声明](http://blogs.technet.com/msrc/archive/2010/01/19/security-advisory-979352-going-out-of-band.aspx)包括:

> 鉴于此问题引起的高度关注、客户如何保护自己的困惑以及不断升级的威胁环境，Microsoft 将针对此漏洞发布带外安全更新。

在我看来，安全问题是次要的还是难以利用的都无所谓。你可以保证全世界的程序员和黑客都在调查这个漏洞，因为围绕着它的宣传。微软正在做正确的事情，一旦通过公司的测试程序，补丁将会发布。

许多人会争辩说，IE 一开始就不应该有这个缺陷，或者应该在过去十年的某个时候修复它。所有那些产生完美无 bug 代码的人现在可能都在嘲笑微软！…
