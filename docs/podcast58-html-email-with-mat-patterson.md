凯文:2010 年 4 月 23 日。HTML 电子邮件的烦恼，以及 Campaign Monitor 如何帮助将这些烦恼转化为健康的收入流。我是凯文·杨克，这是 SitePoint 播客#58:马特·帕特森的 HTML 邮件。

最近我一直在为 SitePoint 写一本书。

马特:是的，没错。很奇怪。我认为这是一群人的想法，我认为它来自网络标准，这很奇怪，但这是一种想法，电子邮件是*的意思是*是文本，因此它应该总是文本，我们不应该改变这一点。当我做演讲的时候，我喜欢先问别人，如果你讨厌 HTML 电子邮件，请举手，我通常得到 50/50。所以那里还有一些工作要做。

这实际上是一个非常奇怪的情况，因为，当然，它并不意味着网络上有应用程序，或者甚至意味着有商店，你可以从亚马逊买东西，对吗？

凯文:对，对。

这一切都是科学家发明的，这样他们就可以交换科学信息，但不知何故，我们在网络上成功地超越了这一点，但在电子邮件中却不是这样，至少在人们的脑海中是这样。

是的，在某种程度上，对我来说，这来自于这些东西被发明的顺序。当电子邮件被发明的时候，网络并没有真正作为一种技术存在；如果是的话，那也是某人眼中的闪光。它还没有证明自己是一种会被如此普遍接受的文档语言。

**Mat:** 真的。

凯文:但是当我们看看像电子书这样的技术，现在电子书的 EPUB 标准，标准机构采用 HTML 作为电子书的基础语言是非常自然的。没有人会说“书应该是文本！”

**Mat:** 是的。我自己也用了这个例子，实际上，即使是不使用等宽字体的文字书籍，他们也使用标题、间距和漂亮的字体，这是因为这样更容易阅读，这也是我们在竞选监督中的立场，就像你个人可能不想收到那样，这很好，因为你应该总是有选择，但其他人想发送漂亮的电子邮件， 它们更容易阅读，如果你不设计它们，如果你作为他们的网页设计师基本上说，嘿，我不想碰它，那么它不像他们不会出去。 它只是让某人的营销秘书将获得 10，001 个剪贴画，并将它们粘贴到 Outlook 的一封邮件中，然后发送出去。

凯文:我想给他们一点免责声明，让他们知道。我的意思是，我们今天与 Matt 交谈的原因之一是，SitePoint 使用 Campaign Monitor 的服务来发送其所有的电子邮件简讯，Campaign Monitor 已经足够友好地为 SitePoint 免费提供这些服务，以换取你可能会说的促销考虑。我们在与我们的时事通讯相关的任何地方都提到竞选监督，今天让 Matt 上节目是这种安排的一部分，但我当然不想让这个播客成为竞选监督的广告。我希望我们能够挖掘一些真正有趣的东西，就像我觉得我们目前已经在做的那样。

我们喜欢 SitePoint 的 Campaign Monitor(甚至在我们成功说服你免费为我们提供服务之前)的原因之一是，你确实解决了这些难题之一，我们真的很高兴我们不必自己解决。

你能告诉我一些竞选团队关注的问题吗，你们觉得自己正在解决这些问题，并为此感到自豪？

我想这也是一个常见的问题，尤其是对于那些试图向客户推销电子邮件营销服务的设计师来说，我为什么要为我可以免费做的事情付费呢？

凯文:是的，我可以打开 Outlook，开始加粗。

**Mat:** 是的。但实际上这涉及到大量的工作，特别是如果你在 SitePoint 上处理任何大小的列表，你会发现那里有相当大的列表。但是，当你发送这么多邮件时，实际发送的时间是一个很大的因素。许多公司来找我们，因为他们自己一直在做这件事，然后他们发现我们花了两天时间试图把时事通讯发出去，我们希望每周都这样做。这是一个巨大的时间吮吸，而我们显然有能力在一个小时内发出来。

我们管理来自发送大量电子邮件的一切:当某人不在办公室的事情变得疯狂时，处理硬反弹和软反弹，处理退订请求，以便您不会意外地再次发送给某人，当您意外地发送给某人时，处理垃圾邮件投诉，或者由于某些原因有人投诉，然后所有关于这些电子邮件中有多少被反弹，谁收到了电子邮件的报告， 谁没有收到，谁打开了它，他们点击了什么，他们对什么感兴趣——所有这些你想知道的信息，如果你把它们考虑在内，会使你的电子邮件更有效，但如果你试图自己做所有这些事情，这些信息真的很难恢复。

它基本上只是围绕电子邮件的时间、成本和技术要求的一个巨大的纠结球，你可以抽象出方式，你可以花时间担心内容和设计应该是什么，而不是实际发送电子邮件的所有技术细节。

凯文:因此，你尽可能地尝试批量设计和发送丰富的电子邮件，你试着将设计师必须做的工作限制在他们设计网页时必须做的工作上。这样说公平吗？

是的，只要有可能，基本上处理好实际处理电子邮件的所有技术细节。正如我们所知，任何白痴都会发邮件，我们总是从白痴那里收到邮件。但是发送一封得体的或者有用的电子邮件是另一回事，设计师真的应该花时间让设计变得有效，并与他们的客户合作，以确保他们的客户发送的东西实际上是有用的，人们想要的，而不是花时间担心他们是否已经删除了上次反弹的人，这是永久的反弹还是暂时的反弹，以及他们是否应该再次发送。有很多信息要处理，如果你能摆脱那些不是你专长的东西，坚持做那些你能比别人做得更好的东西，这是一个赚钱的好方法。

Kevin: 我想，我们对 SitePoint 的 Campaign Monitor 的一个钦佩之处是，你们超越了你们的赚钱业务。与你的竞争对手相比，你们更加努力地推动 HTML 电子邮件相关标准的发展，努力解决一些让 HTML 电子邮件变得困难的问题，努力让这个领域成为设计师们的天堂。你能谈谈这些努力吗？

当然可以。很长一段时间以来，几乎整个时间，竞选监测已经存在，这是五年多一点，它出来的公司之前，竞选监测，同样的两个家伙开始竞选监测，有自己的网页设计业务。所以他们真的知道不得不处理电子邮件和所有渲染问题的痛苦，你的电子邮件在 Outlook 2007 和 Outlook 2003 中看起来不一样的痛苦，以及试图处理并非真正针对设计师的程序的痛苦。所以我想这很大程度上是因为他们个人的理解和他们试图解决的难题。这已经成为公司文化的一部分，那就是我们不应该仅仅因为存在痛苦而建立业务，而是应该尽可能地消除这种痛苦。对我们更好，对我们的客户更好，最终对接收电子邮件的人更好，因为人们越容易实际完成技术渲染部分，他们就越需要花更多的时间来使电子邮件变得有用。

我们正在做的几件事:早期，我们做了很多关于你在哪些特定的电子邮件客户端支持哪些特定的 CSS 的研究，这是多年来非常受欢迎的报告,只是为了发现，好吧，例如，这个 Outlook 2007，它不支持背景图像。因此，与其(像我们一样)一直试图找出它为什么没有出现，你可以到这里来看看，说看，它就是不被支持。

凯文:大概有五种不同的方法可以让你成功，然后当你下午快结束的时候，你会发现，哦，没有一种方法是成功的。

没有一个有效，或者更令人讨厌的是，其中一个有效，但只在某些情况下有效，这一点并不清楚，只是看起来很随意。为什么这封邮件会删除我所有的风格？如果你不知道这是怎么回事，很多事情都非常耗时。因此，我们在这些领域进行了大量的研究——哪些电子邮件客户端会在默认情况下屏蔽图像，如果我用电子邮件发送，表单是否有效……所有这些都需要花很长时间来测试。因此，我们已经走出去，这样做，只是坚持所有在报告中。所以你可以找到它，并节省自己很多时间。

凯文:作为一名设计师，你正在处理一份 HTML 邮件工作，你想让它尽可能的丰富和漂亮，那么第一步是阅读这些报告吗？你一定要看那些报告吗？

如果你只是想发送一封基本的电子邮件，而你并没有做什么花哨的事情，我想如果你从 1999 年开始设计电子邮件，或者设计网页，如果你在 1999 年设计它们，你可以用你 1999 年的设计技巧继续创作电子邮件，那就没问题了。

**凯文:**只要不需要背景图片。

Mat: 对，没错。如果你在 2000 年做了什么，你可能要重新学习一点如何用老方法做事。但是，是的，你会很快遇到限制和棘手的事情，你不确定，它是否应该工作，我是否做错了什么，我的代码是否有问题，你只是为自己节省了大量的时间，这样做或者你可以去拿一个我们已经做了所有测试的免费模板。

知道这些报告是什么肯定是值得的，这样当事情确实出错时，你至少可以发现是你的错还是微软的错。

凯文:说到微软，你们开展的大型活动之一是 fixoutlook.org[活动。你为什么不告诉我们，因为我很高兴看到你们这样做，但我真的很好奇幕后发生了什么，因为在一天结束时，你试图做的是非常温柔地扭转微软的手臂。这些年来，我采访过微软的人，也和微软的人谈过话，就像每一个网页设计师这些年来不得不面对微软做出的决定一样，你会有一种感觉，你对他们施加的压力越大，他们就越不愿意和你一起工作。](http://fixoutlook.org/ "Outlook’s broken—Let’s fix it")

**Mat:** 是的。当你试图做出一些改变时，这总是有风险的。我猜它来自于[电子邮件标准项目](http://www.email-standards.org/ "Email Standards Project")，这是整个活动开展的旗帜，我们开始单独进行活动监控，我们的想法是，我们可以从其他电子邮件服务提供商和类似的人那里获得支持，参与该行业的人，然后不仅仅是一个活动监控的事情，这仍然是一个想法。但我们发现，基本上每个人都在抱怨这种情况，比如为什么为电子邮件设计如此困难，为什么让它成为可能，但却因为在所有不同的电子邮件客户端上有各种各样的支持而令人难以置信地沮丧。

凯文:以及为什么随着时间的推移会变得更加困难。

凯文:不，我没有。

Mat: 这是一个更简单的方法，但 Gmail 有一些问题，如果你使用内嵌样式，它会工作得很好，但如果你不使用内嵌样式，它会把它们都去掉，你的电子邮件看起来很糟糕，这看起来很奇怪，因为从技术上来说，他们没有理由不能像大多数其他电子邮件客户端一样，只是采用那些样式，以任何他们需要的方式使它们安全，然后应用它们。

凯文:是的，这似乎是一种非常短视的行为……就像 Gmail 团队中的一些人在第一天就提出了一些简化的假设，一些我们可以用来简化工作的捷径，他们实际上并没有考虑捷径*是否会*让他们的工作变得更容易，他们认为这是我们可以尝试的事情，所以他们这样做了，而且他们从来没有真正回顾过。

是的，没错。所以他们设计他们需要的东西，情况总是这样；人们设计他们认为产品应该放在哪里，很明显 Gmail 内置了一些假设。我们基本上没有机会与 Gmail 的任何人取得联系，甚至问他们为什么这样做，因为可能有一些我们不知道的原因，我们希望进行这样的对话，但我们没有任何成功。所以 Gmail 鬼脸项目基本上是我们让世界各地的一群网页设计师给他们自己拍一张照片，当他们看到自己的邮件在 Gmail 中的样子时，他们会做鬼脸，然后有一些非常有趣的照片回来了。
