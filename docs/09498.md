# 如何描述云的特征

> 原文:[https://www.sitepoint.com/how-to-characterize-the-cloud-2/](https://www.sitepoint.com/how-to-characterize-the-cloud-2/)

现在，您已经对云是什么以及它是如何工作的有了基本的了解，让我们列举并深入了解它的一些最有用的属性和特征。在花了几年时间在公共论坛上谈论 Amazon Web Services 之后，我发现在传达 Amazon Web Services 的本质以及它能做什么的时候，表征往往比定义更有效。

# 一般特性

以下是亚马逊网络服务的一些一般特征。

### 弹性的

云允许根据需要扩大和缩小资源使用。增加或减少使用量所需的时间是以秒或分钟计算的，而不是以周或月计算的。

### 规模经济

云提供商能够利用规模经济，并能以尽可能好的价格采购房地产、电力、冷却、带宽和硬件。因为提供商将基础架构作为商品提供，所以随着时间的推移降低成本对其最有利。该供应商还能够雇用敬业的员工，他们拥有在世界范围内运营所需的有时难以捉摸的技能。

### 现收现付

这是一个普遍特征，而不是一个业务特征，原因很简单:有了基于云的服务，技术人员现在将做出资源分配决策，这些决策将对资源消耗和总体成本水平产生直接影响。高效运营企业成为每个人的工作。

# 商业特征

从面向业务的角度来看，以下是 Amazon Web Services 的一些定义性特征:

### 没有前期投资

因为云计算是为了满足对资源的按需使用而构建的，所以在实际需求出现之前没有必要进行大量的一次性投资。

### 固定成本变得可变

云计算允许资源消耗实时变化，而不是承诺在合同期限内(通常是一年或三年)使用特定数量的资源。

### 资本支出变成了 OPEX

资本支出是长期的，反映了使用特定数量资源的多年承诺。运营支出基于云供电系统的实际使用情况，并将实时变化。

### 分配是细粒度的

云计算支持时间和资源的最小使用量(例如:服务器使用小时数、存储字节数)。

### 业务获得了灵活性

因为没有对资源的长期承诺，所以企业能够快速响应业务量或业务类型的变化。

### 提供商的业务重点

云提供商的业务是提供供公众使用的云。因此，it 部门有很强的动力来提供可靠、适用且经济高效的服务。云反映了提供商的核心竞争力。

### 成本是相关的

由于云的灵活资源分配模型，购买和运行 100 台服务器一小时与购买和运行一台服务器 100 小时一样容易。这为大规模问题划分方式的创新思维打开了大门。

# 技术特征

从技术角度来看，以下是亚马逊网络服务的一些定义性特征:

### 缩放速度很快

新硬件可以在几分钟内上线，以处理内部(大型计算任务)或外部(网站流量)的意外需求变化。或者，当不再需要时，资源可以返回到云中。

### 无限的可伸缩性是一种错觉

虽然不完全正确，但每个消费者都可以将云视为提供了近乎无限的可扩展性。没有必要提前做好准备；应对需求的激增和增长是云提供商的问题，而不是消费者的问题。

### 资源是抽象的，无差别的

云计算鼓励关注相关细节——结果和可观察的性能——而不是所用硬件的技术规格。底层硬件会随着时间的推移而改变和改进，但是供应商的工作就是掌握这些问题。不再需要亲自了解特定动态资源的详细信息。

### 云是积木

云将 IT 资源作为独立、单独定价的原子级构建块来提供。消费者可以选择不使用、全部使用或部分使用云提供的服务。

### 实验是廉价的

云消除了实验的经济障碍。您可以访问临时资源来尝试新想法，而无需对硬件做出长期承诺。
