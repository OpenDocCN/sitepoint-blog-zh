# 10 年的 Java…为了什么？

> 原文：<https://www.sitepoint.com/10-years-of-java-for-what/>

*更新(12 月 3 日):*首先应该指出这里表达的观点是我个人的，我对它们负责——它们不是 Sitepoint 的*或* Maarten 的。回想起来，我应该听从我的第一感觉，把这个意见(这就是全部)留给自己。

上周三晚上，我与马腾一起开设了 Sitepoint 在 T2 的办公室；).我告诉了 Maarten 这个有趣的(对我来说)Java 狂言，我很想贴出来，但觉得它会产生太多不好的影响。但是作为一个 PHP 爱好者，Maarten 也觉得这很有趣，并告诉我“去做吧！”–~~所以怪他~~。本着落井下石的精神(或者在你自己的位置上呼叫空中支援)…

回到今年五月， [Javaworld](http://www.javaworld.com/) 正在庆祝[Java](http://www.javaworld.com/javaworld/jw-05-2005/jw-0523-anniversary.html)十周年。从那以后, [Ruby on Rails](http://www.rubyonrails.org/) 已经引起了很多关注，以至于人们[开始谈论](http://www.servlets.com/blog/archives/000068.html)[Java 的](http://headrush.typepad.com/creating_passionate_users/2005/12/never_underesti.html)[死亡](http://www.onjava.com/pub/a/onjava/2005/11/16/ruby-the-rival.html?page=1)。

这让我想到“如果 Java 死了，它将如何被后代记住？”。Java 的遗产是什么？从现在起 20 年后，人们还会使用哪些 Java 应用程序，并说“哇，他们那时真的知道如何编写代码”？

我突然想到——我想不出任何一个流行的 Java 应用程序是“每个人”(非书呆子)都在使用的。

如果你排除了为书呆子准备的工具，比如 [Eclipse](http://www.eclipse.org/) 或【在此插入应用服务器】以及用 Java 定制的网站(与现成的 web 应用相比)，还剩下什么？

如果我把自己限制在“有用的”(对真实的人)而不是“受欢迎的”，两个(但只有两个)应用程序最终跃入脑海——Sitepoint 自己的 [Editize](http://www.editize.com/) 和 [JAlbum](http://jalbum.net/) 。

也许我漏掉了一些东西——比如这个(我写这篇文章并在谷歌上搜索“java 杀手应用”时发现了它——以前从未听说过它)，但我并不信服。**Java 的杀手级应用在哪里？**。

10 年来，IT 行业被 Java 营销淹没了。我们已经有了[无休无止的书籍](http://www.amazon.com/exec/obidos/search-handle-url/field-keywords%3Djava/)、[诉讼](http://www.pcworld.com/news/article/0,aid,39130,00.asp)、一大堆[的规范](http://www.jcp.org/en/jsr/all)以及一代人在大学里教授“Java 之道”，引得他们继续嘲笑[有经验的同事](http://developers.slashdot.org/comments.pl?sid=165966&threshold=3&commentsort=0&tid=156&mode=thread&cid=13845934)。谁知道有多少美元被烧掉了？

所有这些都引出了一个问题:你们到底在做什么？
