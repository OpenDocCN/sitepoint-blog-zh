# 没有频繁更新的内容也能盈利

> 原文：<https://www.sitepoint.com/profiting-without-frequently-updated-content/>

今天有人问我，我是否认为有可能运行一个不经常更新的成功网站。我的回答:绝对！

除了博客和论坛之外，有许多类型的内容网站，最常见的两种是我称之为网络杂志或电子杂志，以及参考或资源网站。

SitePoint 是一个网络杂志，他们有大量的回头客，期待新的内容，就像这篇博文。在 SitePoint 的主页上，你会发现最近的内容获得了最多的曝光，这是因为这是访问者想要看到的，新的内容，因为他们有可能以前读过旧的内容。

现在我倾向于运行参考网站，因为我发现它们更容易。[Wilderness-Survival.net](http://www.wilderness-survival.net)是我的参考网站之一，它已经存在了大约 5 年。如果你访问它，你会看到没有真正的特色内容，没有最近的内容可言。取而代之的是，内容被组织得像百科全书或其他参考书(因此有了参考网站这个术语)。事实上，这个网站自从 5 年前启动以来就没有更新过内容。

那么，什么类型的网站更适合你呢？这真的要看题目和你自己的个人能力。如果你很了解你的网站的主题，并且你可以定期写文章(当你的网站是新的时，你不能像 SitePoint 那样让作者为你写文章)，那么你可以尝试走杂志路线。

然而，如果你不太了解这个话题，那么你可以考虑参考路线。参考网站仍然需要大量的内容，事实上，除了最老的电子杂志网站之外，他们往往比其他网站拥有更多的内容，但他们也需要在所有或大部分内容已经写好的情况下推出，所以一开始需要做大量的工作，但后来几乎没有任何工作。

就我个人而言，当我在陆军 ROTC(后备军官训练团，一个针对未来军官的大学项目)时，我参加了这个野外生存课程，这就是我网站的灵感来源。所以我对这个话题并不是完全无知，我对它很感兴趣，也至少读过一本书，但我还远远没有资格每周写一篇关于它的文章。我也不知道我是否有足够的资格来主持一个论坛，这是该网站还没有论坛的唯一原因…但稍后会有更多。所以，选择是明确的，我应该做一个参考网站。如果这是一个我觉得写起来更舒服的话题，我可能会做一个杂志网站。

有些人想知道，如果没有更新的内容，你还能成功吗？嗯，正如我在这篇文章开头所说的，当然。不过，有些问题你应该关心。事实是，你不会像一个杂志网站那样获得那么多的重复访问者，除非你有一个巨大的内容库，没有人能真正读完它，或者在没有多次回访的情况下读完它。然而，随着十亿互联网用户的出现，总会有新的访问者能够提供足够的流量来盈利。

另一个有利于参考网站的问题是，不需要经常性的新鲜内容向你施压，你可以更专注于一个内容领域，这可以带来巨大的受欢迎度。例如，你可能没有资源来制作一个像 SitePoint 这样的普通主题的网站管理员帮助网站。如果你想在这个领域建立一个新的网站，最好是专业化，所以你选择了像 AJAX 这样的新技术，并决定专注于此。AJAX 的范围是有限的，虽然您可以暂时提供常规内容，但最终它可能会变得多余。因此，你可以不采用杂志格式，而是采用参考格式，试着在百科全书的范围内介绍 AJAX，或多或少，编程参考书变成了网站。使用这种类型的格式，即参考格式，人们不会根据新的内容来判断你的站点，而是根据你的库的深度来判断，所以即使你不定期更新，你也可以成为获取 AJAX 信息的站点。

然后，当然有整个工作量的问题，与我的大多数参考网站，我所做的就是现金支票，这给了我更多的时间来制作新的网站。真的挺方便的。

您还可以尝试两全其美，将论坛与参考站点结合起来实现。你甚至可以更进一步，通过在文章或内容类别的底部显示相关的论坛线索，进行大量的内容和论坛整合。这样，你的网站总是有新鲜更新的内容，但你不是写它的人。你所要做的就是保持适度，甚至这种责任最终也可以传递给一个值得信任的社区成员。

所以，不要认为你需要一个定期更新的网站才能成功，只要你推广它，并且它有足够的内容让人们觉得它有用，那么无论你多久更新一次，你都可以赚钱。
