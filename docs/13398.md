# sIFR 2007:综述

> 原文:[https://www.sitepoint.com/sifr-2007-the-roundup/](https://www.sitepoint.com/sifr-2007-the-roundup/)

[sIFR](http://www.mikeindustries.com/sifr/) ，由[肖恩·因曼](http://www.shauninman.com/)、[迈克·戴维森](http://www.mikeindustries.com)和[马克·乌本](http://novemberborn.net/sifr3)开创的基于闪光灯的文本图像替换技术，在 2004 年末以大量的泡沫和气泡出现，但在过去的 12 个月里似乎失去了一些知名度。我想我们应该花点时间来看看这个地区发生的一些新的事情。
 **1) sIFR3:偶数 sIFRer！**

最新版本—[sifr 3——于 2 月](http://novemberborn.net/sifr3)发布，包含了一系列新花样，包括:

*   同时使用粗体和斜体
*   使用多种颜色
*   支持行距、字距调整和不透明度
*   支持闪光过滤器，混合模式和抗锯齿
*   支持像素字体
*   改进的 IE 性能

Reid Beels 也想出了一个不错的变化，它可能会很好地融入到 sIFR 的主要版本中——支持渐变。

里德的网站实际上是根据他当前的 Twitter 状态来选择标题文本的渐变。有用吗？也许不是，但无论如何这是一个绝妙的想法，Reid 在他的网站上对他的梯度法做了详尽的解释。

**2)制作无闪光闪光**

sIFRs 被广泛采用的一个主要原因是它对闪存的依赖——不是从最终用户的角度，而是作为生产周期的一部分。如果您没有 Flash 创作产品(很多 web 开发人员也没有),那么在生成包含您的字体的简单 SWF 文件时，您的选择会很有限。

不会了。伊恩·普顿构建了一个简单、免费的在线服务,它可以将你上传的 truetype 字体转换成 sIFR 可用的 SWF，然后通过电子邮件发送给你。实际上，我们不久前讨论过写这样的东西，所以很高兴看到有人完成了它。

伊恩的是一个没有虚饰的服务，我可以想象谁可以得到一个更优化的 SWF 你手工制作的，但服务是快速和有效的。很好。

**3) swf 图像替换:sIFR 为您的图片**

swfIR 是一个很好的例子，它注意到一个好主意，并看看它还适合哪里。它利用了 sIFR 背后的核心概念——即抓取绑定的 HTML 元素并将其加载到自定义的 Flash 电影中，以赋予它们新的超能力——但它应用于图片而不是文本。

我特别喜欢的一点是能够在你的布局中以随机的角度倾斜你的图像，但这项技术也提供了完整的边框、角落和阴影效果。当然，这是一个渐进的增强，所以没有 Flash 和/或 JavaScript 的用户不会因为标准的 IMG 标签被正常渲染而变得更糟。

记住最近出现的越来越多令人印象深刻的 Canvas 和 SVG 实验，我无法想象我们很快就会看到一个完全基于 JavaScript 和 SVG/Canvas 的等价物。
