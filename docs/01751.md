以下是我们的书的摘录，由 Alexis Goldstein、Louis Lazaris 和 Estelle Weyl 撰写的《真实世界的 CSS3，第二版。世界各地的商店都有出售，或者你可以在这里买到电子书。

在前一章中，我们向您介绍了 HTML5 中的一些语法变化，并涉及了一些与验证相关的问题。让我们进一步扩展这些概念，以便您可以更好地理解验证页面是如何变化的。

HTML5 验证器不再关心代码风格。您可以使用大写或小写，省略属性中的引号，排除可选的结束标记，并且尽可能不一致，您的页面仍然有效。

所以，你会问，对于 HTML5 验证器来说，什么样的*算错误？它将提醒您元素的不正确使用、元素被包含在不应该出现的地方、缺少必需的属性、不正确的属性值等等。简而言之，如果您的标记与规范冲突，验证器会让您知道，所以在开发页面时，它仍然是一个有价值的工具。*

为了让您更好地了解 HTML5 与过于严格的 XHTML 的区别，让我们来看一些细节。这样，您就可以理解在 HTML5 中什么被认为是有效的:

基于 XHTML 的语法中需要的一些元素不再是文档通过 HTML5 验证所必需的；例子包括`html`和`body`元素。这是因为即使您排除了它们，浏览器也会自动将它们包含在文档中。

*   Void 元素(即没有相应结束标记或没有任何内容的元素)不需要用结束斜杠结束；例子有`meta`和`br`。

*   元素和属性可以是大写、小写或混合大小写。

*   属性值两边不需要引号。例外情况是使用了多个空格分隔的值，或者 URL 显示为一个值，并且包含一个带有等号(=)字符的查询字符串。

*   在基于 XHTML 的语法中需要的一些属性在 HTML5 中不再需要。例子包括`script`元素的`type`属性和`html`元素的`xmlns`属性。

*   一些在 XHTML 中被否决从而无效的元素现在是有效的；一个例子是`embed`元素。

*   没有出现在任何元素中但直接放在`body`元素中的零散文本会使 XHTML 文档无效；HTML5 就不是这样了。

*   一些在 XHTML 中必须关闭的元素可以保持打开，而不会在 HTML5 中导致验证错误；例子包括`p`、`li`和`dt`。

*   `form`元素不需要有`action`属性。

*   表单元素，比如`input`，可以作为`form`元素的直接子元素；在 XHTML 中，需要另一个元素(如`fieldset`或`div`)来包装表单元素。

*   `textarea`元素不需要有`rows`和`cols`属性。

*   XHTML 中以前不赞成使用链接的`target`属性。它现在在 HTML5 中有效。

*   正如本章前面所讨论的，块级元素可以放在 link ( `a`)元素中。

*   如果&符号字符在页面上显示为文本，则不需要将其编码为`&amp;`。

这是 XHTML strict 和 HTML5 验证之间的差异的一个相当全面的列表，尽管还不够详尽。有些是风格选择，所以鼓励你选择一种风格并保持一致。我们在前一章中概述了一些首选的样式，欢迎您将这些建议融入到您自己的 HTML5 项目中。

### 注意:更严格的验证工具

如果您想使用更严格的准则来验证标记的语法样式，有一些工具可以帮助您。Philip Walton 的 [HTML Inspector 就是这样一个工具。](https://github.com/philipwalton/html-inspector)要使用它，你可以在开发阶段将脚本包含在你的页面中，然后在开发者工具中打开浏览器的 JavaScript 控制台并运行命令`HTMLInspector.inspect()`。这将在控制台内显示许多关于如何改进标记的警告和建议。HTML 检查器还允许您更改配置，以根据自己的需要定制工具。

## 摘要

到目前为止，我们已经了解了 HTML5 中所有新的语义和语法变化。有些信息可能有点难以直接消化，但是不要担心！熟悉 HTML5 的最好方法是使用它——从你的下一个项目开始。尝试使用我们在上一章中提到的一些结构元素，或者我们在本章中看到的一些文本级语义。如果您不确定一个元素应该如何使用，请返回阅读关于它的部分，或者更好的是阅读规范本身。虽然语言肯定比书中的文字枯燥(至少，我们希望如此！)，规范可以更完整地描述给定元素的用途。请记住，HTML5 规范仍在开发中，所以我们所涉及的一些内容在新的 HTML5.1 版本中仍会有所变化(或者在 HTML5“生活标准”中，如果你遵循 WHATWG 的定义)。规格将始终包含最新的信息。
