# 用于许可的知识共享软件

> 原文：<https://www.sitepoint.com/creative-commons-software-for-licensing/>

昨天我在 Creative Commons 网站上看到一个博客条目，提到了 Windows 和 Mac OS X 的最新版本 [ccPublisher](http://wiki.creativecommons.org/wiki/CcPublisher) 。现在我是一个大的 Creative Commons 许可粉丝——然而我从来没有听说过这个小应用程序，直到我的 RSS 订阅中差点错过它。

不幸的是，尽管在上面的链接中有一个路线图的框架开端，但几乎没有文档记录——对其使用的概述。

ccPublisher 只允许用户删除一个音频或视频文件(该程序假设你拥有版权或授权许可的权利)，并允许你选择和应用一个创作共用许可证的文件，并上传到你自己的服务器或可选地上传到 archive.org(互联网档案馆-一个值得浏览一些时间的目的地)。

这本身并不一定是 web 开发人员最感兴趣的。然而，在不得不追逐免版税、版权管理和其他类型的内容(更不用说你自己的产品——无论是副本、图像、软件等)。)，在 CC 许可下变得可用的内容的世界当然是令人感兴趣的。

从第二个角度来看，除了使用 CC 许可的资产之外，开发人员也许可以期待或贡献一个使用 ccPublisher 作为原型的扩展许可工具。

虽然这一产品肯定还处于起步阶段，但它是使内容创建者、开发者和其他处理者能够在资产(软件、音频、视频、文本等)内实现许可的方向上非常需要的一步。

作为一个框架，它展示了开源社区如何应用一种方法来标记他们制作的作品，就像专有内容创作者用数字版权管理标记数字资产一样。

想象一下，一个应用程序包含来自开源计划和知识共享的当前许可，并且可以在发布或部署之前嵌入或标记您的数字资产。

更进一步，考虑一个分析工具，它包括所有批准的许可证的参数和一个选择过程，在该过程中，资产所有者运行一个问题矩阵，最终将您的工作与适用的许可证匹配起来。

在我最近的一篇专栏文章中，Eric Raymond 提到了一个许可选择工具，这是开源倡议的目标。我也收到或无意中发现一些构建开源许可矩阵的尝试，以帮助选择过程。

确保像这样的工具发展势头的一种方法是以某种形式或方式做出贡献——无论是文档、特性请求、bug 和补丁测试、开发、资金还是其他活动。当然，如果开发者和用户对开源许可工具的需求增加，他们就会来。
