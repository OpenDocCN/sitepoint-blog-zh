## 采访记录

凯文:2009 年 10 月 23 日。在今天的节目中，收集了在悉尼 2009 年南方网络方向上录制的采访。这是 SitePoint 播客# 33:WDS 09 团队歌剧。

凯文:我是凯文·杨克，为你播报 SitePoint 播客。我正在悉尼参加 2009 年 T2 南方网络方向大会，我想我应该和一些与会者谈谈，看看他们对什么感兴趣。

**男 1:** 旧的微格式终于开始起作用了……地图成为主流。

凯文:所以位置感知的东西真的变得有用了。

男 1: 是啊。

凯文:是啊，真酷。

男 2: 我现在正热衷于位置感知，但一切都开始超越它自己的小应用程序，一切都捆绑在一起，这真的很好。

男 3: 是的，我真的对社交网络的主流化感到非常兴奋，你知道，推特是如何真正起飞的。我也对移动网络的影响感到非常兴奋，这意味着能够在一个小设备上提供应用程序，比如 iPhone。所以这让我很兴奋。

凯文:好的。到目前为止，你有没有挑选出你最喜欢的、不打算错过的演讲？

卡梅隆·亚当斯，永远是卡梅隆·亚当斯。

凯文:卡梅隆亚当斯……[谷歌浪潮](http://wave.google.com/)！

男 3: 对。

凯文:好的。嗯，我一定会赶上他的。谢谢伙计们。

凯文:那你兴奋什么呢？

男 4: Adobe 前两天宣布他们将有一个 Flash 到 iPhone 应用程序的导出功能，所以这听起来真的很酷，因为截至目前，我还不能编写 iPhone 应用程序，但我知道如何编写 Flash。就像是因为那个声明，我可以写 iPhone 应用程序。

凯文:是的，这也是本·加尔布雷斯在他的演讲中提到的。他谈到了一个框架，可以让你从 JavaScript 导出到 iPhone 应用程序。

**男 4:** 电话间隙。

凯文:耶，耶， [PhoneGap](http://phonegap.com/) 。

男 4: 这也是我的建议，我以前没听说过。

凯文:是的，这是将网络带到这些平台的能力，否则这些平台将被关闭。

男 4: 是的，还有我作为一个 web 开发人员，能够编写真实应用程序的能力。

凯文:你兴奋什么？

**男 5:** RDFa。

凯文:好吧，告诉我们那是什么。

男 5: 这是让我兴奋的事情。我真的很兴奋。[笑声]这样我们就可以转圈了。

凯文:它有什么令人兴奋的地方？

**男 5:** 我会认真的。有很多事情。它允许你使你发布的信息更加精确，或者允许机器解释你发布的信息。我们的网页经常包含很多有用的内容，比如地址或书评等等。人类很容易理解这一点；你看着这一页，你可以读它。通过用 RDFa 更精确地标记它，我们可以帮助搜索引擎，我们可以帮助应用程序，我们可以帮助用户界面更精确地处理信息。

凯文:好的，那么这个百万美元的问题——你认为 RDFa 会比微格式更受欢迎吗？

男 5: 我想说，从很多方面来说，它已经发生了。如果你看看谷歌最近宣布处理 RDFa 的事情，它标记了关于图像的许可信息或关于视频的格式信息，有一个新的倡议叫做 [GoodRelations](http://www.heppnetz.de/projects/goodrelations/) ，这是一个完整的词汇表，允许你标记出售的产品，商店的营业时间，你知道，所有这类事情——都使用 RDFa 和雅虎！正在处理地图格式，谷歌看起来也会这样做。

我认为微格式是一个非常不同的东西，我认为——我希望——人们用微格式做的一些事情将会进入 RDFa 世界。微格式是一种很好的方式告诉人们，看，这里有一组术语，如果你使用这些术语，将会有好处，我认为在某些方面，这就是 RDFa 需要做的。语法是这样的，很好；但是这里有一堆术语，如果你标记——就像谷歌说的评论——如果你用这些术语来标记评论，我们可以做一些聪明的事情。

凯文:谢谢你。

凯文:伙计们，詹姆斯·比提，现在你们对网络上的什么感到兴奋？

詹姆斯:画布元素。我只是认为这是网络的一个有趣的进步，你知道，能够在浏览器中用标记做所有这些图形的东西，而不是预先制作和预先编译图像文件和图形文件。

凯文:是的，你经常玩它吗，或者它是你期待的东西吗？

詹姆斯:不，我已经看过它的演示，但我希望今天我能得到足够的东西，然后我自己就能开始做了。

凯文:太好了。太不可思议了。伙计们。

**男 6:** 大概是 IE9。

凯文:IE9。

男 6: 哦，是的，我很前卫。[笑声]

凯文:我没听说过 IE9！

男 6: 没有，嗯，没有多少人有，但这正是我兴奋的地方。

凯文:你有没有去微软的展台看内幕消息？

男 6: 我现在还不想和那些家伙说话。[笑声]

凯文:我是凯文·杨克，在 2009 年悉尼南方网络会议上为我们播报 SitePoint 播客。我和来自 Opera 开发者关系团队的拉克伦·亨特、克里斯·米尔斯和丹尼尔·戴维斯在一起，对吗？

不完全是，我在核心，但他们还是派我去了。

**凯文:**核心。核心是什么？

这是核心技术，就像浏览器中的实际渲染引擎和 JavaScript 引擎之类的东西。

凯文:好的，拉克伦·亨特，你做了很多标准化工作，你在 HTML5 工作组工作，诸如此类。

拉克伦:是的，我做 HTML5，我在 Web Apps 工作组工作，我负责选择器 API……

凯文:好的，克里斯和丹尼尔，你们的关注点是一样的，还是有什么不同？

Chris: 所以重点的不同实际上，Daniel 和我都在开发人员关系团队。我讨厌用营销这个词，但我们确实在做营销，但专门针对开发者和那些真正知道我们在说什么的人。所以我们在 Opera 的超技术人员、开发者和最终用户之间形成了一个反馈回路和一座桥梁。这主要是为了形成一个焦点，但我们也和最终用户谈了很多。重要的是要保持下去，帮助人们尽可能多地学习如何使用我们的新技术，但也要关注开放网络标准，这是我们工作范围的一个非常大的部分。

**Daniel:** 这样做也是一种发展趋势，因为我们在过去的一年里迅速增加了我们的开发者关系团队，而 Palm 现在已经有了一个羽翼未丰的开发者关系团队。所以我认为我们看到了更多这种非市场营销；这是一种间接营销，我们不提倡使用——嗯，我们*说*使用 Opera 】,但是我们是以这样一种方式来做的，如果你建立在标准之上，如果你建立在开放的网络之上，那么间接地，你正在使 Opera 受益，也使整个网络世界受益。

Kevin: 好的，这就是我真正感兴趣谈论的 Opera 选择如此直接地接近开发者社区的原因。我的意思是，你真的没有从其他浏览器供应商那里看到那么多。火狐有点——这些天来，它有点像是开发者为开发者开发的浏览器，所以如果仅仅是因为构建浏览器的人往往是 web 开发人员，那么它们就相当普遍。但是，你知道，你看不到…你可以指责苹果和微软在开发者舞台上相当安静；当他们有话要说时，通常是因为他们刚刚推出了一款新的浏览器，他们希望你看一看，而在其他三年的开发周期中，他们相当沉默。而歌剧，伙计们，这至少是第二年你们有一个重要的代表团参加这个会议——这个伟大的会议——但这是在悉尼的一个小会议，真的。

那么为什么 Opera 如此重视与开发者的交流呢？

克里斯:这是因为我真的认为开发者是最重要的一类受众，他们会考虑如何使用我们的浏览器。像许多其他已经开始发布像样的开发工具的浏览器供应商一样，你知道——显然你已经启动了 Firebug 这一趋势，但我们所有人现在都希望沿着这条路走下去，我们所有人都开始形成这些——嗯，我们大多数人，有几个人没有——但我们大多数人开始建立这些强大的开发人员关系团队，例如，他们教人们如何使用标准，教人们如何在开发中采用更好的实践。除了标准的开发者关系之外，ee 在教育方面也做了很多。除了关于高级开发技术和未来标准(如 canvas 和其他 HTML5 材料和 CSS3)的高级文章、教程和会议讲座，我们还将回归基础，制作类似于 [Opera Web 标准课程](http://www.opera.com/company/education/curriculum/)的东西，帮助学生和爱好者，以及任何希望正确从事 Web 开发的人，以便有一个良好的开端。

有大量的工作要做，我们几乎是在尝试一种包罗万象的方法。

我们需要开发者明白我们是来帮助他们的。我们希望他们在 Opera 中测试他们的网站，并确保他们使用标准和材料。这就是为什么我们像他说的那样专注于课程。

丹尼尔，我听说你是日本的风云人物。所以你在日本是一个人吗？还是除了你自己，Opera 在日本也有一席之地？

丹尼尔:是的，我想我们现在在东京办公室有将近 50 人，比很多人预期的要多，但是我们和任天堂有大合同，我们为 Wii 和 DSi 提供浏览器，也为内部电话公司提供浏览器，KDDI 是一个大公司。因此，我们有许多工程师，他们采用拉克伦团队的做法，然后为日本合作伙伴进行定制和个性化。

Kevin: 尤其是 SitePoint 的很多观众都在北美，在北美，Opera 仍然是一个非常小众的浏览器，但在欧洲市场，我们听说 Opera 的存在要大得多。亚洲的歌剧怎么样？

拉克伦:我们在俄罗斯很受欢迎。那里有巨大的市场。

丹尼尔:是的，这取决于俄罗斯的哪些地区——对不起，是亚洲的哪些地区，就像这取决于欧洲的哪些地区一样。俄罗斯和东欧很大。在亚洲，越南，印度尼西亚，我们非常非常大。不幸的是，在日本，情况并非如此，但我现在的工作就是提高这一点。

Kevin:Wii 上的 Opera 浏览器曾经是付费产品，现在变成免费了，是这样吗？

丹尼尔:没错，是的。以前是 500 分，但他们已经免费了，我想，从这个月开始，他们会以给他们一个价值 500 分的免费游戏的形式来回报那些支付了 500 分的人。

凯文:哦，天啊，我得拿回我的 500 免费积分。

说到开发人员，Lachlan，我曾经知道你是一个标准极客，永远不会选择一个浏览器，永远不会宣誓效忠于一个或另一个浏览器，因为这会毁了你在标准社区的信誉。你加入了歌剧——是什么让你超越了那条线？是因为与开发者社区的合作吗？

**拉克伦:**是的，这是他们参与制定标准的原因，这也是我去那里的主要原因。在我去那里之前，我参与了 HTML5，我和 Opera 团队的很多人一起工作，然后我在找工作，他们说来为我们工作吧。所以我这样做了，我仍然在做我喜欢做的事情，当涉及到开发标准和东西的网站时，我仍然是非常中立的。我和歌剧合作是因为他们对什么感兴趣。

凯文:对，Opera 了解你。

拉克伦:嗯是的。

凯文:[笑声]听起来真好。Opera 10 显然已经过时了，但是你们已经在讨论你们的渲染引擎的下一步了。很明显，你们在考虑至少提前一个版本。开发者可以期待的事情有哪些？

克里斯:我们的目标是，你知道，保持竞争力，跟上其他浏览器正在做的许多事情，所以我们有 CSS3 的功能，如框阴影、过渡和转换，所有这些东西都在工作，如果不是已经工作，至少在一个公开发布的版本中。很有意思的是，听到前 Mozilla 的 Ben Galbraith，现在 Palm 的 Ben Galbraith，这个星期几次谈论未来的事情，比如真正快速的 JavaScript 引擎，就像 Opera，我们很快就会推出 Carakan。他实际上在他的演讲中为歌剧做了大量的宣传，他非常高尚。但是话说回来，你知道，像 Opera 中的许多人一样，他是另一个真正关心开放标准胜过一切的人。

回到之前的观点，实际上，我认为 Lachy 谈论供应商中立性是完全正确的，因为这是一个更有说服力的故事，让人们知道我们支持开放标准，而不仅仅是试图说“支持 Opera”。这就像我们做的许多“打开网络”活动，基本上涉及到去各个公司说，你知道，你有这种愚蠢的浏览器嗅探东西，这真的很糟糕，因为它通常使你的浏览器无法在 Opera 和 Safari 中工作，但有时它是 Opera、Safari 和 Firefox——这是一种非常糟糕的老式 IE 嗅探东西。但是这不仅仅是为了支持 Opera，而是为了正确地支持标准。在一天结束的时候，至少所有参加这种会议的开发人员都是如此。但是，当然，我们也试图进入这些更多的——我该怎么描述呢——斯图尔特·朗里奇曾经称他们为“暗物质开发者”——的思想和内心，这些家伙只是，你知道，不参加会议，他们只是在黑暗的房间里辛苦工作，不必费心去正确地制定标准。但我认为也有适合这些人的会议,“的未来”会议似乎有很多这样的人。有趣的是，我们也开始深入他们的大脑。

凯文:好的，谢谢大家抽出时间。

克里斯:谢谢你邀请我们，凯文。
