# Digg 的设计师泄露设计细节

> 原文：<https://www.sitepoint.com/interview-daniel-burka-digg/>

Daniel Burka 常驻旧金山，是加拿大网络开发公司 silverorange 的创始合伙人之一。他还是新闻网站 [Digg](http://digg.com/) 的首席设计师，也是 [Pownce](http://pownce.com/) 的联合创始人。

丹尼尔在上周的 [Web Directions South](http://www.webdirections.org/) 会议上花了几分钟时间与 SitePoint 的 Matthew Magain 聊了聊关于设计用户界面、回应用户反馈、Web 2.0 美学等问题。

SitePoint:首先让我们向 SitePoint 的读者介绍一下你自己——你的背景，以及你是如何走到今天的。

当然可以。我在 98 或 99 年开始涉足网络。我当时在高中，我的哥哥，一些朋友和我都被网络吸引住了。所以我们开了一家公司，一年后我们和附近的另一家小网络公司一起参与了一个大项目。这个项目进行得非常顺利，我们认为应该合并我们的两家公司，于是 silverorange 就诞生了。在过去的八九年里，silverorange 已经发展成为一家非常好的精品网店。

我在 silverorange 工作了多年——远在加拿大东部——与 Mozilla 这样的团队一起工作。我在 Mozilla 视觉识别团队工作，所以我帮忙画了火狐的 logo，真的很有趣。我在白板上画了草图，然后和我一起工作的史蒂夫·德斯罗彻斯用铅笔画了草图。然后乔恩·希克斯做了渲染。不要误解我——乔恩·希克斯获得了 90%的荣誉，他太棒了。但是参与这个项目很有趣。

然后我在《mozilla.org》中工作，这对我来说是一个大项目。看到这些结果后，凯文·罗斯雇佣了 silverorange 来开发 Digg，这个项目真的启动了。我参与了 Digg 的所有工作，由于我已经在 silverorange 上工作了很长时间，转移到 Digg 对我来说是一个很好的过渡，没有与 silverorange 断绝任何关系。我开始越来越多地在 Digg 上工作，在 silverorange 兼职的同时成为一名员工。现在我是 Digg 的创意总监，这是我的全职工作。

大约一年半前，凯文、莉娅·卡尔弗和我创办了 Pownce，这占据了我所有的业余时间。

我们的许多读者渴望为自己工作，放弃他们朝九晚五的生活，这样他们就可以成为自己的老板。对你来说，放弃你帮助成长起来的东西，去为那个人工作，是一个艰难的决定吗？

可以说，我不是为那个男人工作。我是 Digg 的第一批员工之一——我是参与这个项目的第四个人，所以我对这个项目有一些所有权。我当然不是创始人，我也不预设这么说。但是我觉得我已经获得了相当程度的所有权，我对网站的发展方向有很大的发言权。你知道，这是一个很有影响力的大网站。当你在这样一艘船的舵柄附近时，那是非常有趣的。推出一项新功能，并让 3000 万人使用它，这是相当疯狂的。这很奇怪。

我是 Pownce 的联合创始人，所以我在那里有很多所有权，我不觉得我在为那个人工作，不。

你在设计中有什么具体的哲学吗？

我不认为它可以被表达为一种哲学。通过与聪明人一起工作，我学到了很多东西，所以我真的很幸运能在 silverorange 与 Steven Garrety、我的兄弟 Nick Burka、Steve DesRoches 和一些非常优秀的设计师一起工作。

我们都没有上过设计学校——这些都是我们年轻时从经验中，从彼此的碰撞中获得的。上学很棒，我猜，但我觉得根本没必要。你会看到顶级设计师——丹·塞德霍尔姆、道格·鲍曼和 T2。他们没有去设计学校。这些聪明人发现了这一点。如果年轻设计师向我征求建议，尤其是那些去设计学校的设计师，我会强烈鼓励他们找到有相同兴趣的人，开始一起工作。构建虚拟的项目，尝试一些东西。

我的意思是，这就是网络的美妙之处——就像狂野的西部。你可以看看别人的源代码，可以借鉴，可以看到他们的接口。知道什么有用，什么没用。听听他们在会议上的谈话。如果你想走那条路，如果你是一个聪明、有动力的人，教室是不必要的。

除了你提到的那些名字，你还能从哪里找到灵感？

我的很多灵感来自我们的用户，以及观察人们如何使用我们的系统。我今天谈了一点这个，但是我们的用户是迷人的！他们做各种奇怪的事情。把他们正在做的事情拿来改编真的很有趣。

当然，我们这个领域的其他网站也很鼓舞人心——比如像脸书这样的网站。我认为下面的工作人员正在做一些非常有趣的 UI 工作。即使他们是从 Digg 借的。新脸书 feed 上的一些新用户界面——他们借鉴了一些想法。这很有趣。并不是说我们没有从他们那里吸取一些想法。

其他一些社交网站也在做一些有趣的项目。我认为 [last.fm](http://last.fm/) 正在做一些有趣的 UI 工作——汉娜·多诺万在那边。我们也是这样开始的。我会看着[37 个信号](http://37signals.com/)，然后想，“他们在做什么？”如果你够聪明，并且注意的话，你会发现像 [Basecamp](http://basecamphq.com/) 这样的 37signals 应用不仅仅是漂亮。你可以看到天才在哪里。你不需要完全复制它，但是你可以模仿它，向它学习，然后自己开始做。我认为那是最好的学习方法。网络上发生了太多的事情，而且都是公开的。

网站互相影响，用户影响设计的现象——平面设计师可以把任何东西带到桌面上的日子已经过去了吗？

嗯。我读杂志，当然，我在一定程度上受到了印刷的影响。但是我没有印刷背景。我大部分时间都在看网络。

当然，我们看到了印刷影响力的复苏？像马克·博尔顿这样的人谈论了很多关于网格的话题，我们也听到了很多关于优雅的印刷字体的话题……所有这些传统的印刷概念都回到了争论中。

也许吧，但这些不是我经常关注的事情。我更关注用户交互设计，尤其是 Digg。别误会，不是说排版不重要。但是在这样的网站上，用户输入的内容太多了，而且在很多方面都不可预测。我从来没有专注于让它看起来像某种方式。它更加自由和灵活，我可以接受它有点脏，只要人们能够推着它到处跑，用它做一些疯狂的事情。

对于习惯于静态内容设计的设计师，你有什么建议？

放开我。当然，有一些事情我们很明显是封闭的——你只能做这么长的标题，你只能在一个描述中输入这么多字符。但最大的是不为之烦恼的哲学理念。很酷！例如，我曾经在 Digg 上错误地设计了一些东西，因为我没有预料到在一定时间内会有一定数量的 Digg；我存储了前十个故事…它打破了布局。你知道，我们修好了！没什么大不了的。类似的事情经常发生。放开控制，不要紧张。从你的用户正在做的事情中获得灵感，而不是强迫他们用一种方式去做。

那么以用户为中心的设计会影响或限制你对网站的设想吗？

以用户为中心的设计是我唯一考虑的事情——这是设计 Digg 或 Pownce 或我工作的任何网站的关键方面。我所关心的是人们目前如何使用这个网站，以及他们将如何使用它，这取决于我做什么。

那我们还需要想法吗？或者我们应该让我们的用户告诉我们应该完全做什么？

不，不，我们的用户不会告诉我们应该做什么。我们的用户会引导我们找到问题所在。他们有时会提出很好的建议，但通常他们是在暗示一个具体的问题。我们有责任找到问题的解决方案——就像我今天在演讲中提到的亨利·福特的格言:“如果我问我的客户，他们会告诉我他们想要一匹更快的马。”超出用户的想象。有感觉让逻辑跳跃到下一个水平。这才是真正的天才所在。

**SP:先说一下 Digg 的视觉审美。有人说“Digg 的设计有一些渐变，它有一些圆角——这是 Web 2.0”，这是很常见的。你对此有什么看法？**

我过去常常取笑凯文说“Web 2.0”，那是在它还是一个普通术语之前。这是一件如此空洞、乏味的事情。尤其是“Web 2.0 美学”我的意思是，当然，我也参与其中——比如 Mozilla 网站。

但我为此责怪卡梅伦·亚当斯！他做了[themaninblue.com](http://themaninblue.com/)，用了一堆渐变。我绝对是被那个影响的！所以当我做 Mozilla 的时候，我到处做渐变。所以我认为他启发了很多人开始使用它们！如果他知道我在责怪他，我想他不会喜欢的…

但是我认为这是友好的概念，因为我们非常关心人们的参与，我们希望他们感到轻松。所以你最终会有一种玩具般的轻松感:圆角、柔和的颜色——这些东西变得很常见，因为我们想让人们放松，以鼓励他们的参与。所以不仅仅是审美。

这种方法绝对是我对 Mozilla 的心态。之前的设计是黑色、红色和棕色——非常刺眼。几乎是野蛮的风格，而且是故意的。他们有整个社会主义的东西。但从品牌的角度来看，这是非常难以接近的。这并不友好——你知道，一个两个孩子的母亲是不会安装 Mozilla 的(基于那个网站的设计)。当时，Firefox 试图打入主流市场，所以这就是我的想法——明亮、友好的颜色，漂亮、简单的审美——这基本上就是我的想法。这也带入了 Digg。对我来说，这些项目互相影响，所以我就是这种心态。

在你的演讲中，你谈到了很多关于指标，以及显示人们如何真正使用你的网站来影响设计的信息。你对调查有什么看法？我们应该运行它们吗？我们应该如何处理结果？我们应该给这些回答多少权重？

某种类型的人会填写一份调查，所以你应该自动把它放入你的权重中。但是调查可能非常有用。我们在 Digg 的推荐引擎上做了一个实验，得到了一些好的数据。但是我不会把调查作为从人们那里获得反馈的唯一手段。

调查给人的印象是相当不干净的，所以调查需要做大量的解读工作。我认为引入一组有针对性的用户更有益，比如用户测试、基于任务的分析和焦点小组等方法。但这也不仅仅是一项调查。

你在工作中使用什么样的工具？你是素描还是画缩略图，还是直接跳到 Photoshop？

我不是一个纸上谈兵的人——我真的很糟糕。我用白板写下大而广的想法。例如，凯文和我最近为 Digg 做了一些约会计划，我们最近做了一些白板，这真的很有用。所以我和他会消失在一个房间里使用白板。

但我会直接进入 Photoshop。我用 Photoshop 真的很舒服。我仍然喜欢制作非常逼真的构图，看起来好像我真的在发展。我的大脑就是这样工作的。

并且我使用 [Coda](http://www.panic.com/coda/) 作为我的编码工具。基本就是这样。

Digg 生成的数据量让你可以享受可视化的乐趣，比如 [BigSpy](http://labs.digg.com/bigspy/) 、 [Swarm](http://labs.digg.com/swarm/) ，以及其他一些 Flash 的东西。

是啊，都是靠[雄蕊设计](http://stamen.com/)打造的。我们雇佣了那些人，他们是我们的好朋友，在离 Digg 几个街区远的地方工作。

你能告诉我们一些关于 [Digg labs](http://labs.digg.com/) 的事情吗？

我不能，对不起！我们把它放在胸前。

最后一个问题，你有什么书想推荐或与我们的读者分享吗？

是啊！有几本书我非常喜欢，它们不是关于网页设计的，但是和网页设计有很多关系。一本是斯图尔特·布兰德的《建筑如何学习》。说真的，这是一本很棒的书。第二本是简·雅各布斯的《T2:美国大城市的生与死》，她几年前不幸去世。一本精彩的书，是关于城市设计的。例如，它解释了最佳行为的人行道应该有多宽，以便孩子们有足够的空间玩耍——但如果人行道太宽，他们就会离他们的监护人太远，所以危险的事情可能会发生。如此多的这类想法被推广到网络上。

当然还有爱德华·塔夫特:[设想信息](http://www.amazon.com/Envisioning-Information-Edward-R-Tufte/dp/0961392118/)。至于专门关于网络的书，戴夫·谢伊的[关于 CSS Zen Garden](http://www.amazon.com/Zen-CSS-Design-Visual-Enlightenment/dp/0321303474/) 的书很棒。凯文·考克斯写的一本非常老的书，叫做《T4 用户界面设计》。

你最近发现了哪些你喜欢并想分享的博客？

发布印刷和工业设计的加拿大设计资源博客是一个很好的网站。来自 boston.com 的[大图](http://www.boston.com/bigpicture/)，很惊艳。他们已经开始每两三天发布一次照片，但它们是 600 像素宽的巨幅图片。他们为奥运会做了一个，他们为地震做了一个…他们太棒了。每两三天就会有巨大的，华丽的图像。把它放到你的 RSS 阅读器上。他们总是出现在 Digg 的头版。

丹尼尔，谢谢你的时间。

非常感谢马特，那真的很有趣。
