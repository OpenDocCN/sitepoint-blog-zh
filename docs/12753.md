# 微软发布 Photosynth:仅仅是新奇？

> 原文：<https://www.sitepoint.com/microsft-release-photosynth-just-a-novelty/>

自从 Blaise Aguera y Arcas [去年在 TED 大会上展示了 Photosynth】以来，大多数看过它的人都想得到这个软件。正如 TED 网站所说，这个演示简直令人瞠目结舌。Photosynth 能够拍摄从 Flickr 收集的数百张巴黎圣母院的照片，并通过识别每张照片中教堂的特定方面与其他照片的关系，使用它们创建一个虚拟的 3D 环境。每张照片都被准确地放置在拍摄地点的空间中，允许观众在大教堂周围缩放，并从各个角度获得它的准确照片。](https://www.ted.com/index.php/talks/blaise_aguera_y_arcas_demos_photosynth.html)

上周，微软发布了免费的 Windows 专用浏览器插件和应用程序 [Photosynth](https://photosynth.net/) ，以及 20GB 的免费在线存储空间来存储你的“synth”这个插件应该可以在 Firefox 3 和 IE 7 上运行，但是我只能在 IE 浏览器上运行。Photosynth 符合微软“软件+服务”的理念，即桌面软件通过基于网络的组件得到增强。

一旦我让它工作起来，Photosynth 就非常容易使用。只需选择您的照片，将它们添加到 Photosynth creator 应用程序，然后单击一个按钮即可制作您的 synth。这个过程需要几分钟，照片就会上传到你的 Photosynth.net 账户。该软件会对你的作品的“相似度”进行评分——我收集的这是一个数字表示，表示你的主题覆盖范围有多好，以及你的照片拼接得有多好。我的第一次尝试——用大约 40 张照片拍下 SitePoint 办公室的整个一楼——彻底失败了，只有 32%的“吻合度”我的第二次尝试，描绘了办公室的休息区，嵌入在下面，利用 45 张照片获得了更好的 61%的“synthiness”分数。

理想情况下，合成器应该有 150-200 张照片，不少于 20 张。

虽然 Photosynth 的速度和易用性确实令人印象深刻，但不幸的是，我发现实际的 synth 并不令人印象深刻。部分原因可能是因为我创建测试合成器的主题不是这项技术的最佳素材——它擅长将从单个对象的多个角度拍摄的照片进行空间关联，而我的“合成”尝试使用从基本相同的地点拍摄的大部分空白空间的照片。

然而，即使是微软员工和国家地理网站上的演示也没有最初的 TED 圣母演示给人留下深刻的印象。也许这是因为劣质的照片集，或者也许是由于浏览器的轻微变化(例如，我发现缩小并获得整体“3D”图片更难，即使使用本应完美的合成器，如泰姬陵和狮身人面像)。

微软的 TED 演示从 Flickr 搜集了数百张图像，并能够将它们在空间上联系起来，以创建一个对象的 3D 表示。但是现在，Photosynth 是一项个人技术——没有简单的方法来协作和从外部来源收集多张照片。你只能用你电脑上的图片创建合成器，大多数人不太可能拥有创建高质量合成器所必需的全面的照片集。微软[暗示更多的协作工具即将到来](https://www.liveside.net/main/archive/2008/08/21/photosynth-released-as-a-worldwide-software-services-download-to-become-part-of-msn.aspx)。当这种情况发生时，这项技术将不再新奇，因为人们将(希望)能够向公共 synths 贡献照片，并不断改进它们。
