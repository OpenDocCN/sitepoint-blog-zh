我们一直沉浸在您的反馈中，回顾发展趋势，并与全球的合作伙伴和开发者会面，以告知我们下一步将如何发展 EdgeHTML。在本次会议中，我们将向您介绍我们 2016 年的计划，从我们即将发布的产品开始，最后是一些长期调查和对网络平台未来的投资。

这篇文章是微软技术倡导者和工程师关于实用 JavaScript 学习、开源项目和互操作性最佳实践的 web 开发系列的一部分，包括[微软 Edge](http://blogs.windows.com/msedgedev/2015/05/06/a-break-from-the-past-part-2-saying-goodbye-to-activex-vbscript-attachevent/?wt.mc_id=DX_838514) 浏览器和新的 [EdgeHTML 渲染引擎](http://blogs.windows.com/msedgedev/2015/02/26/a-break-from-the-past-the-birth-of-microsofts-new-web-rendering-engine/?wt.mc_id=DX_838514)。

我们鼓励您在包括微软 Edge(Windows 10 的默认浏览器)在内的各种浏览器和设备上进行测试，并在[dev.microsoftedge.com](https://dev.windows.com/en-us/?wt.mc_id=DX_838514)使用免费工具，包括[status.microsoftedge.com](https://developer.microsoft.com/en-us/microsoft-edge/platform/status/?wt.mc_id=DX_838514)，这是一个门户网站，提供微软 Edge 和其他浏览器(包括 Internet Explorer)中可互操作的网络平台功能的最新实施状态和未来路线图。此外，[访问 Edge 博客](https://blogs.windows.com/msedgedev/?wt.mc_id=DX_838514)，从微软开发者和专家那里获得最新信息。
