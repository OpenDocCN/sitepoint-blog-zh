# PHP 5.1 的黑仔特性？

> 原文：<https://www.sitepoint.com/php-51s-killer-feature/>

PHP 4 直到它的第一个主要更新版本 4.1 才在现实世界的服务器支持方面真正火起来。PHP 5 也是如此吗？它的第一个点版本现在正处于早期测试阶段。

PHP 5.1 Beta 1 已经出来十天了，Beta 2 已经[刚刚上市](http://news.php.net/php.internals/16814)。这两个版本都还没有在 php.net 的[上公布，也没有官方的改动列表公布，但是如果你特别好奇，你可以阅读由开发者维护的](http://www.php.net/)[的详细列表](http://cvs.php.net/co.php/php-src/NEWS?r=1.1945)。

在 2002 年初发布时，PHP 4.1 对语言做了一些重要的修改。这是第一个建议关闭 register_globals 以提高安全性的版本。像$HTTP_GET_VARS 这样用于安全访问提交值的长名称变量被替换为更短、更方便的超级全局变量[如$_GET。总的来说，PHP 4.1 对开发人员使用 PHP 的方式产生了比 4.0 更大的影响。](https://www.sitepoint.com/article/write-secure-scripts-php-4-2/2)

在这个阶段，PHP 5.1 看起来并不那么引人注目。对 Perl 正则表达式的一些增强，一些额外的数组函数…没什么值得大书特书的。但是在 5.1 中 PHP 有一个显著的增强: **PHP 数据对象** (PDO)。

PDO 库允许您使用一组函数访问任何广泛选择的受支持的数据库。因此，您不仅只需要学习一组为多个数据库编写的函数，而且可以将一个应用程序转换到另一个数据库，而不必在整个代码中更改所有函数。

PHP 开发人员多年来一直在使用数据库抽象层来实现这一点，首先是自主开发，然后是开源的 T1，最后是 T2 官方的 T3。然而，这是第一次这样的抽象层被完全内置到 PHP 中，而不是一个 PHP 代码层。

当 PHP 5 在现实世界中的采用仍然滞后时，这可能是吸引开发人员使用 PHP 5.1 的杀手锏吗？如果你想尝试 PDO 来自己决定，一个好的起点是[首席开发者的博客](http://netevil.org/node.php?nid=186)。
