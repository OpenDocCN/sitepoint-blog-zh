# Joyent 收购谷歌应用引擎相当明智

> 原文：<https://www.sitepoint.com/joyent-acquires-reasonably-smart-to-take-on-google-app-engine/>

开发人员在选择云托管环境时会面临两个问题。要么他们不得不担心管理他们的基础设施，这意味着当他们需要快速扩展时会有额外的工作——就像亚马逊的 Web 服务堆栈或 Rackspace 的 Mosso 一样。或者，他们不得不担心技术和供应商锁定——就像谷歌的应用引擎(目前限制开发者用 Python 编写应用程序)或微软的 Azure(目前希望开发者使用. NET)一样

Joyent 的加速器云计算平台的工作方式很像以前的各种云平台。但是今天，[公司宣布收购开源云计算平台](http://www.joyeur.com/2009/01/14/joyent-acquires-reasonably-smart)[的](http://www.reasonablysmart.com/)，这使得他们的方法更像是混合方法。

Joyent 创始人兼首席执行官大卫·杨将合理智能描述为“一个自动扩展的平台即服务；它是谷歌应用引擎的直接、开源竞争对手。”目前处于 alpha 阶段的开源平台为开发人员解决了缩放和架构问题，同时希望他们使用 JavaScript 编写后端代码。rational Smart 添加了扩展来完成 JavaScript 最初没有被设计来做的事情，比如存储。

虽然 JavaScript 通常不被认为是一种服务器端语言，但 Young 写道，他预计它“将成为网络的主流语言。”预计该平台还将在未来几个月内增加对其他语言的支持。

Joyent+rational Smart 与 App Engine 或 Azure 的不同之处在于，因为 rational Smart 是开源的，并且将保持这种方式，所以开发人员不必太担心供应商锁定。Joyent 将通过他们的加速器提供合理的智能(几个月后开始)，但开发者可以随时下载并在其他任何地方运行它——这意味着他们可以随时离开 Joyent(或完全跳过 Joyent)。

收购价格没有披露。
