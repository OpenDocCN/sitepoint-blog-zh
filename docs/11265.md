# 灵活你的可及性

> 原文:[https://www.sitepoint.com/flex-your-accessibility/](https://www.sitepoint.com/flex-your-accessibility/)

![Adobe Flex Logo](../Images/4de6cad62d670178cb6604c9566121e2.png)
我们 Flex 相关系列教程中最新的一篇关注了所有网页设计者和开发者都应该接受的东西:[可访问性](https://www.sitepoint.com/article/accessibility-in-flex-apps)。

作者 Toby Tremayne(高级开发和数据驱动交互设计方面的专家)首先向我们清楚地介绍了什么是可访问性，以及它为什么重要。他接着概述了四种主要的用户障碍类型(视觉障碍、阅读障碍、运动障碍和认知障碍),它们最有可能影响可访问性，并给设计者和开发者带来挑战。

Toby 将话题转移到 Flex 应用上，向我们展示了如何通过修改 Flex 配置来实现可访问性。在谈到屏幕阅读器、字体和颜色以及键盘访问之前，他概述了 Flex 中的可访问组件。

在这篇文章的结尾，Toby 认为从应用程序构建的一开始，可访问性就是一个值得解决的问题。他还提到了几个有助于进一步阅读该主题的网站。
