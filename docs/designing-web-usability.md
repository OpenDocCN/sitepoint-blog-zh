# 回顾——设计网站可用性

> 原文：<https://www.sitepoint.com/designing-web-usability/>

自称为网站可用性专家的雅各布·尼尔森，被无数人喜爱，也被同样多的人厌恶。在读了他的书《设计网站可用性》 (New Riders 出版社，1999 年)后，许多网站设计师会认真地重新考虑他们的设计方法，更多的人会干脆把这本书扔进壁炉。

尼尔森的信息很简单:网站的存在是为了帮助人们完成一项任务，而不是炫耀设计师的技能。令人遗憾的是，他为实现这一目标而建议的一些步骤过于激烈，而且没有确凿的证据支持。

在整本书的许多地方，尼尔森做出了毫无根据的陈述，更多的是观点而不是事实。尽管他非常出名，并且能够为可用性评论收取六位数的费用，但他所说的不应该被视为绝对真理。其实应该是完全相反的。毕竟，任何专家都应该有可靠的研究来支持他们的观点，而且，鉴于他的建议被广泛阅读的事实，尤其是尼尔森应该能够证明他的观点。

然而，我要称赞他一件事——他的书将让设计师走出自己的鞋子，走进用户的鞋子，那些试图通过设计师的创作来实现目标的人。这本书不乏关于写作、导航、空间使用、滚动、下载时间、搜索和跨平台设计的指南，促使你通过用户的眼睛来看待自己的网站。

然而，指导方针本身似乎陷入了时间扭曲。作者似乎认为我们生活在 1997 年，在这个时代，28.8 个调制解调器和 14 英寸 640×480 的显示器是标准配置。对于今天的许多网站来说，将设计限制在如此严格的规格是不合理的。看起来唯一能满足本书提出的理想的网站应该是无图形的。既然[他自己的网站只符合自己原则的 75%](http://users.cwnet.com/adunn/Other/CDES%20215/useit_points.html)，那我们其他人还有什么希望？

回到书的内容。前四章分别侧重于页面设计、内容设计和网站设计，发人深省。视觉例子的自由使用是非常有用的，然而在许多情况下，它们打破了文本的连续性。有时，描述屏幕截图的标题出现在不同于屏幕截图本身的页面上，这需要您来回翻动。其他时候，四分之三的页面除了分解内容的屏幕截图之外什么也没有。在我看来，这本书本身可以使用一点可用性工程。

我个人发现*设计网页可用性*有 400 页，相当罗嗦，尤其是与标题为*不要让我思考*的同一主题的简明扼要的书相比。一本更简洁的书，后面有一份研究参考书目会好得多。

这让我想到了另一个话题——雅各布·尼尔森的预言，也被称为“未来预测”,其中有整整一章的内容。读完整章后，我仍然不明白这和手头的主题有什么关系。就我个人而言，我对作者的未来预测能力不像对网站可用性主题那样感兴趣。难怪尼尔森被一些人贴上自大狂的标签，全神贯注于自吹自擂。

读完所有这些，你可能会认为我不会向每个人推荐这本书。你说得对。如果你从来没有读过关于网站可用性的书，跳过雅各布·尼尔森的*设计网站可用性*，得到*不要让我思考*。然而，如果在读完那本书后，你还想看更多，那就去读读雅各布的书吧——但要小心。他的规则，如果不折不扣地应用，会导致一个非常沉闷的网站。在某个地方有一个中间地带，但它留给你去寻找，因为作者肯定没有亲自提出来。

**评级:** 3/5
