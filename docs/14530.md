# 我衣橱里的骷髅

> 原文：<https://www.sitepoint.com/the-skeleton-in-my-closet/>

首先，感谢所有对我之前博客的友好欢迎的评论！当我看到一些最初发布的问题时，我想我应该开始分享一个深刻、黑暗的个人秘密。我承认，尽管有太多关于 CSS 的好文章(甚至是一本书！)在 SitePoint 上…靠得更近… *我仍然使用表格进行布局。*(喘息！)

一旦你们集中精神，我会进一步解释:

在取悦客户的现实生活中，我发现当面临选择时，大多数客户不愿意为了说“我们的网站符合所有当前的网络标准”而牺牲他们的低端用户/客户群(当我说“低端”时，我指的是*低*——一些日志显示人们使用*网景 3。*)即使是一个体面的网站，至少仍然可以被*阅读*，对于那些积极追求电子商务销售并希望他们的网站对 95%的客户“看起来不错”的公司来说也是不够的。另一方面，对于那些不想花钱让我制作 *x* 数量的样式表和 *y* 行 JavaSCript 代码的廉价客户来说，我拥有的唯一真正的资源是使用表格设计基本布局，遵循 CSS 1 对字体、背景颜色等的定义，并挂完全的可访问性。(难怪 Trenton Moss 在他的[文章](https://www.sitepoint.com/article/blunders-big-player-2)中发现，大多数“大玩家”在很多方面都无法满足所有当前的可访问性指导原则……甚至在基本方面。)

不要误会我；牢记可访问性——这意味着大量使用 CSS 将帮助您更好地构建您的网站，并尽可能多地分离内容和布局。从长远来看，尽可能多地学习 CSS 和当前标准肯定会有回报——不仅对你，对你的观众也是如此。这就是为什么我喜欢阅读 SitePoint 的 CSS 和可访问性专家的文章；他们追求纯粹的、更高的标准。但是我有一种务实的倾向，我需要和那些有使用网景 4 的客户的客户合作。

所以我的建议:
1。了解 CSS、可访问性和有效代码。练习一下！

2.彻底调查你的客户或观众群。请注意您的服务器日志以及所使用的浏览器和系统的百分比。

3.然后，画一条线(“没有网景 3！”)并做好失去部分观众的准备。

4.当你构建你的网站时，尽可能多地使用 CSS 和可访问性概念，同时在低端浏览器中不断测试。

5.如果这意味着使用表格，那就这样吧。但是要做好准备，在接下来的几个月或几年里重新设计你的网站，以兼容未来(和现在)的技术(例如，手持设备等)。

有人有更好的或者同样合理的折衷方案吗？请分享！
