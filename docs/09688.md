# RC9au 采访基里尔·拉德济霍夫斯基

> 原文:[https://www . sitepoint . com/rc9au-interview-with-kirill-radzikhovsky/](https://www.sitepoint.com/rc9au-interview-with-kirill-radzikhovskyy/)

在 RailsCamp 的第一个晚上，我有幸见到了杰出的 Kirill。他告诉我他非常多产，每个人都知道他，他是绝对正确的。他是一个很有个性的人，他所有的名声都在他之前。

基里尔，给我们介绍一下你自己吧？

我叫 Kirill，是来自墨尔本的 Ruby on Rails 开发人员之一。墨尔本的每个人都认识我，澳大利亚的每个人，任何一个 Rails 开发者都认识我，因为一些疯狂的事情，比如我是唯一一个喝醉的俄罗斯人。你不想和我喝酒，因为，我会毁了你。

稍微好一点的是，我知道 Ruby 中的语言分析。我给 Rails 3.1 打了补丁，以支持公元前日期，现在这个日期已经可以用了。我会说四种语言，不幸的是我每天都要混合使用它们，大量的混合。

你会说什么语言？

从出生开始，我就被乌克兰俄语所诅咒，我仍然必须用俄语和我的家人说话，我和我的日本女友住在一起，并且曾经住在日本，所以我在家里说日语，必要的时候明显会说英语。

在上次的 Rails 夏令营中，我做了一个关于我的日语解析器的演讲，你可以给它任何日语句子，它会计算出单词的结束位置，并把它们重新组合成字典形式。我应该给你看看。昨天有人找到我，指出我们做的谈话，他们想移植算法，因为它可以很容易地应用到中国。

你从事 Ruby 项目有多久了？

大概三年了。

你还用过其他什么编程语言？

我 11 岁开始编程，在乌克兰用 Pascal 写作。Pascal 实际上是一种令人惊奇的语言，直到今天国际编程比赛仍然用 Pascal 语言举行，我想是法国人在组织它。从那以后，很明显，你必须知道，因为否则你不会明白，计算机到底发生了什么。在大学，Haskell，Haskell 是我非常喜欢的另一种伟大的函数式编程语言。

显然我不得不做 Java，不是一个大粉丝，商业上我不得不做。在我的业余时间，每个人似乎都对此感到惊讶，在我加入 Ruby 之前，我曾经是一个大的 Python 开发者。每次我这么说，人们都会说“嗯，这太神奇了，因为通常 Python 的人讨厌 Ruby，而且永远不会改变立场”。这就是为什么每当我看到，我现在为 Rails 团队辩护，每当我不得不与 Python 开发者争论时，我希望我能与他有更好的争论，因为我确切地知道 Python 方面是什么。现在，我仍然不得不用 Python 做一些事情，我喜欢它。

有趣的是，一个很好的例子，我必须在大学做一个语言识别项目，而 Ruby 是我的第一个语言选择，它可以在 30 分钟内运行测试数据。您可以将相同的算法移植到 Python，它将在 8 分钟内运行。鲁比，我爱你，但你太糟糕了。总之，我不想谈我和露比的关系，她不会喜欢的。

你现在在哪里工作？

我在 [Silverpond](http://silverpond.com.au/) 工作，我们是一个小工作室，现在不在克雷莫纳，尽管它仍然在我的名片上写着。说真的，最低订单是 1000 个 75 美元，我们现在实际上是在城市，在市场街。我们在汽车行业有很多客户，最近为澳大利亚邮政和墨尔本大学做了一份工作，所以我们得到了更大的客户和更大的名字。

显然，我们已经在 RailsCamp 了，你觉得 Ruby on Rails 有什么有趣、吸引人或好玩的地方吗？

啊，Ruby on Rails，有趣的是，上周我不得不向我的女朋友解释这一点，所以她知道我总是去日本参加 RubyKaigi，所以她显然经常听到 Ruby 这个词，现在是 Ruby on Rails，我不得不向她解释 Ruby 和 Ruby on Rails 之间的区别，毫无羞耻地，我要对你做同样的事情。

我们都喜欢 Ruby，我们都喜欢 Rails，因为我们不得不这样做。有一个可怕的区别，因为 Ruby 是我能找到的最人性化的语言，我喜欢用 Ruby 写作。我讨厌网络，网络一团糟，所有不同的浏览器，所有不同的标准，请求响应会议，应用程序的状态，有这么多的问题，但 Rails 让它变得更好。我爱露比。

对于对 Ruby/Ruby on Rails 感兴趣的人，你有什么至理名言吗？对你来说，开始变得有意义的最重要的时刻是什么？

这又是一件有趣的事情。在 Python 中，它是一种最好的编程语言，它有一种计算机科学的感觉，它是一种语义机器学习语法来表示一切。请记住，做任何事情都有一个语法，我总是看到 Ruby 有很多方法可以实现任何事情。一开始我不喜欢它，但现在我发现它实际上更好。

虽然有许多方法，但有一个最好的方法来解决它。问题是，不要介意在 Ruby 中有多种方法，总有一种最好的方法。Ruby 最好的一点是，因为我们都是人，所以它是最直观的做事方式。所以很多时候当我不知道怎么做一件事的时候，不是给我文档，你只是猜测。你只要猜猜，就能成功。真的是可以预见的。七月有多少头牛出生？输入“七月出生的奶牛”,就有了。它非常实用，通常是解决问题的最显而易见的方法。

基里尔非常有趣，对自己的工作充满热情。如果你有机会和他说话，抓住机会。
