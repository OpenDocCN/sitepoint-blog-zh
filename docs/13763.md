# SXSW 2006–结束了！

> 原文:[https://www.sitepoint.com/sxsw-2006-its-a-wrap/](https://www.sitepoint.com/sxsw-2006-its-a-wrap/)

这是奥斯汀 SXSW 音乐节的最后一天(或者至少是互动部分的最后一部分)，自从四年前来到这里，我第一次真正发现自己准备好回家了。的确，这里真的是一个派对之城，尤其是当这么多网络极客像他们一样来到这里的时候，但是偶尔睡一会儿还是有好处的！

今天早上，有不少人看起来也有些萎靡不振——昨晚的博客和 Adaptive Path 赞助的活动看起来有些影响。不管怎样，研讨会的房间仍然挤满了想要学习更多网络相关知识的人。

我又一次玩了猜谜游戏，决定参加什么样的小组讨论，并以关于 Web 标准的会议开始了这一天，或者给它一个合适的标题“[说服你的公司接受标准](http://2006.sxsw.com/interactive/programming/panels/?action=show&id=IAP060065)”。这让我特别感兴趣，因为几年前我曾试图这样做，现在我发现自己处于一个幸运的位置，与一群真正“理解”它的人一起工作(尽管在你通过验证器运行它之前，它并不完美！).小组成员也来自有影响力的公司，如 AOL 和 Netscape。什么？标准？而 AOL 呢？网景呢？！是的，是真的。AOL 确实有一些非常优秀的人在为他们工作(试着去[主页](http://aol.com/)并调整文本大小几个等级，你可以看到那里有一些智能 cookie 在工作，我不是指 JavaScript 中的 cookie！);从网景的角度来看，你必须记住，他们是那些举起手说“好吧，网景 4 烂透了，我们要把它干掉，重新开始”的人。如果那件事没有发生，事情会有多么不同。

正如我所料，有很多建议对我和许多观众来说并不陌生——向唱诗班布道等等。我很感兴趣地听说了一个与 AOL 高管会面的故事，一位盲人用户使用屏幕阅读器演示了 AOL 网站的各个部分。那是 AOL 的“可访问性第一天”,真正得到了需要的支持，这完全反映了几年前我在 Nationwide 的经历。简而言之，这种尴尬的技巧绝对有效！关键是要确保在向赞助商销售 web 标准时，不管他们是谁，你不是在销售 web 标准本身——你是在向他们销售好处。换句话说，不要告诉他们你要用酷炫的 x 技术做什么，你要告诉他们你能实现什么，然后告诉他们你如何能实现这些目标。 [CSS Zen Garden](http://csszengarden.com/) 再次被强调为解释正确编码方法论的关键工具，但我很高兴看到的是一个禅宗花园化的例子(他们的说法！)一个现有的 AOL 页面。不了解 CSS 的人通常不会真正理解当他们不认识的网站时，将引用切换到样式表是多么基本，但是在他们知道的页面上看到相同的效果——这个例子是 AOL 登录页面——真的让他们明白了。

然后我参加了“[设计下一代网络应用](http://2006.sxsw.com/interactive/programming/panels/?action=show&id=IAP060004)”的会议，会上讨论和/或演示了 Flickr、各种博客系统和可视化工具。雄蕊设计公司的埃里克·罗登贝克在可视化前沿展示了一些真正令人惊讶的东西，包括[看起来很时髦的 IRC 后台频道](http://backchannel.stamen.com/)，呃，可视化和 [cabspotting](http://www.stamen.com/clients/cabspotting/index.html) 。观众得到了一些对 Flickr 变化的预览，毫无疑问，观众中的一些成员捕捉到了这些变化，然后立即上传到他们的 Flickr 账户。完美！很多人都在谈论这类应用程序的快速发展，以及运行它们的最佳方式(引用其中一张幻灯片上的话:“Writely 的发布次数超过了微软 Word 在十年内的发布次数”)。然而，在我看来，有太多的注意力放在了少数网络应用上，而没有足够的时间讨论即将出现的其他东西。但我也能理解为什么会这样——没有人会在这样的观众面前透露他们的伟大想法，从而危及他们的未来。

下午，我去参加了“RSS:不再仅仅是博客”的会议，我在这里了解到*一件事:那就是我真的不知道 RSS！我对 RSS 的理解很像标题 aludes to——一种订阅网站更新或博客评论等的方式，因此我渴望了解更多。我的问题是技能水平。主持人和观众显然在上面某个地方，而我在下面某个地方，只是不能得到概念。有太多 RSS 黑客和很酷的做事方式的例子，以至于我真的不确定这个怪物*到底是什么*。它似乎有数百种不同的使用方式，从简单的网站更新通知(我已经习惯了)到日历分享方式、公告栏、内容管理等等。所以我不再记笔记，而是发誓要回到 RSS 学校，努力弥合我和他们之间的知识差距。很能说明问题的是，我可以在我面前的某人的笔记本电脑屏幕上看到这样一句话:“如果你知道 RSS，那你已经太晚了。”这说明了很多问题。对于那些想找到一些讨论过的例子的读者来说，Adina Levin 已经创建了一个 del.icio.us 页面，所有的例子都在 T7 上。*

所以这就是我在西南偏南的基本情况，我跳过了当天的最后一场演示(布鲁斯·斯特林的传统总结)，毫无疑问，我会被告知这是非常棒的，有史以来最有趣的一场，每个人都得到了一吨的免费东西，但嘿，我有一个专栏要写，然后再一次在奥斯汀热闹的夜生活中挣扎。

正如我所预料的那样，从头到尾都是一场狂欢，但在某些方面却超出了我的期望。就数量而言，它已经从“可怕到更可怕”(这是组织者的说法)，我不知道如果不做一些改变，它还能扩大多少。我没有解决办法，但我作为一个见证了它日益快速增长的观察者提到了它。我仍然认为这是最好的网络会议，并强烈推荐它，只是希望它有时会轰动。我要去放松最后一个晚上了。各位晚安。
