# 可用性工程 101

> 原文:[https://www.sitepoint.com/usability-engineering-101/](https://www.sitepoint.com/usability-engineering-101/)

尽管万维网上的最新创新被大肆宣传，但归根结底，我们不能忘记，人是中心角色，是网站要取悦的用户。因此，我们应该努力满足用户的需求，其他所有的都应该放在第二位。

不幸的是，这是一个写起来比实际付诸实践容易得多的任务。许多所谓的“网页设计师”现在正在大量生产 HTML，就像它已经过时了一样。出色的图形和大量漂亮的小工具无法取代可用的导航。无论网站看起来有多好，除非你给你的访问者一个可行的系统来浏览你的网站，否则都是无用的。一个精心设计的导航系统会给访问者留下深刻的印象，如果你的内容足够好，访问者会一次又一次地回到你的网站。

现在我们到了紧要关头。我们知道好的导航对网站的成功至关重要，但是这个术语到底是什么意思呢？在实际编写任何 HTML 之前，你应该在设计阶段考虑一些成功导航的基本原则。实现一个成功的导航系统没有正确或错误的答案，只有快乐或沮丧的用户。然而，总的来说，有一些基本原则似乎是有效的。

##### 一致性

通过可预测性(注意这并不意味着无聊)，你可以给用户一种网站组织的感觉。在这种情况下，可预测性意味着，如果你在一个页面的左上角放置了一个蓝色的“主页”图标，那么这个按钮在整个网站的其他页面中也应该是蓝色的，并且也可以在左上角找到。你的导航设计必须提供有意义的链接，这样他们的行为就可以通过他们的图形表示或文本链接被直观地理解(注意一些用户可能关闭了他们的图像，所以你必须为所有的图像提供一个文本替代)。记住——保持一致！web 上缺乏用户界面标准，这意味着新手用户在浏览网页时没有经验可以依靠，因为网站与网站之间的设计差异很大。你至少可以在你自己的站点中保持一致性。网络为你在内容中跳跃提供了强大的力量。始终提供一个安全的锚，链接到预期的地方，并建立一个熟悉的景观，用户可以随时返回。一个很好的例子是商定某种定义标志或徽标，可以显示在您的网站上，以区别于其他网站，即 Macromedia 徽标。

##### 接近

您必须为用户提供多种访问特定信息的方式。仅仅因为你有一个标题为 index.htm 的首页并不意味着用户会首先看到那个页面。他们可能会从其他人的链接进入你的网站，这些链接把他们带到了你的一个子部分，因此除非你给他们一个机会，否则他们永远也看不到你的主页。就把你的网站想象成一组连接一个城市的道路。如果一个人从你坚持要扫描的那张烦人的宠物照片走了一条通往你简历的路，那么你必须为他们提供另一条路，让他们在以后的某个阶段回到那张照片，如果他们愿意的话！你必须提供替代方案。

##### 等级制度

目标是以最少的步骤和最短的时间提供必要的信息(即用户想要的内容)。要做到这一点，需要一个简单、有组织的层级。实现这一点的解决方案是逻辑。虽然这听起来很简单，但比你想象的要难。对你我来说合乎逻辑的东西，对下一个访问你的网站的人来说可能不合乎逻辑。这是很难实现的，你可能永远不会让每个人都满意，但是，你可以和大多数人相处得很好。保持简洁明了。用户应该总是能够弄清楚他们在事物发展过程中的位置。他们需要知道自己在你的信息结构中的位置。当用户浏览你的网站时，你必须给他们视觉上的指导。

##### 组织

信息必须以这样一种方式来组织，即它们可以被分成几个逻辑单元。人们从网上阅读的方式不同于从纸上阅读。平均而言，一个人从电脑屏幕上阅读文本的时间要比从书本上多 30%。用户不会阅读大量的信息。把它们分成更小的可读的块，只需要最少的滚动。我们网络用户很懒，尽可能避免接触滚动条。根据你认为信息的重要程度，使用你的层级来分解信息。主页面应该在下一层，而需求较少的信息可以在你的层次结构中构成一个子部分。等级制度非常重要，因为人们了解它们。从一个层次化的系统中，可以更容易地进入更多的全局导航设计，在页面之间有更多的横向移动，但仍然保持层次结构。

成功的网站导航没有所谓的“公式”。相信你的直觉，运用你的常识，希望你不会彻底出错。测试，测试，再测试。根据用户的反应，你很快就会意识到你是否犯了一个严重的导航错误。渐渐地你会为你的网站找到一个合理的解决方案；给用户他们需要的东西，他们会真正回报你。

"啊，当我们第一次练习欺骗时，我们编织了一张多么复杂的网啊！"沃尔特·斯科特爵士
