# 拉斯马斯·勒德尔夫:PHP 框架？再想想。

> 原文:[https://www . site point . com/rasmus-lerdorf-PHP-frameworks-think-again/](https://www.sitepoint.com/rasmus-lerdorf-php-frameworks-think-again/)

这是我第一次听拉斯马斯·勒德尔夫演讲，至少可以说很有趣。换句话说，我喜欢听到真实的意见，不会退缩——拉斯姆斯不会退缩。

简单介绍一下背景，拉斯马斯·勒德尔夫是 PHP 的创始人，并且仍然是 PHP 项目的核心开发人员。

**PHP 框架**

在他演讲中，他选择强调 PHP 框架(Drupal 也未能幸免)以及它们在性能上有多差。他们不仅速度慢，而且他们“万事通”的态度导致开发人员没有使用最适合工作的东西，从而误入歧途。他继续说，PHP 开发人员确实需要考虑性能，不仅是因为可伸缩性的原因，也是因为环保的原因。如果程序效率更高，将会减少数据中心的数量，从而降低能源需求。在我们新出现的能源意识时代，这确实成为一个重要的方面，我很高兴他正在提高意识。

回到框架，他首先讨论了他创建的一个数据库密集型 Twitter mashup。这需要大量的数据库调用和大量的后台工作。通过手动调谐，他能够达到 280 req/sec 的数量级。相比之下，Apache 提供的只有“Hello World”的简单 HTML 页面的请求速度仅为每秒 600 次。好了，舞台设置好了(顺便说一下，这是在他的本地机器上测试的)。

**你好世界**

PHP 框架在“Hello World”测试上的成绩如何？没有数据库调用，只是用框架的母语输出 Hello World。结果并不太好，最快的一个刚刚超过 120 请求/秒，最慢的是 8 请求/秒。这是一个巨大的差异，当然也突出了他对性能的论证。Drupal 哪里得分了？刚好在 50 请求/秒以上。所以不是最好的，但是他确实指出了 Drupal 并不是传统意义上的真正框架。这是一个可以快速扩展的 web 内容管理系统。

那么，有没有不烂的框架呢？拉斯穆斯确实提到他喜欢 [CodeIgniter](https://codeigniter.com/) ，因为它更快、更轻，而且最不像框架。

**如何让 PHP 变快**

“嗯，你不能”是他的快速回答。PHP 只是不够快，无法扩展到雅虎的水平。PHP 从来就不适合这类任务。“任何基于脚本的语言都不够快”。为了获得真正大规模 web 系统所需的速度，你必须使用编译过的 C++扩展来获得真正的、可伸缩的架构。这就是雅虎所做的，也是许多其他 PHP 重量级公司所做的。

**RDF、语义网和猴子**

Drupal 中的 RDF。Rasmus 特别强调了在页面中嵌入结构化元数据的重要性。RDFa 允许您将数据嵌入到 web 页面中，还允许您创建定制的词汇表，或者更好的是，重用现有的词汇表。你为什么想这么做？Searchmonkey 将对这些内容进行索引，并开放一个丰富的搜索 API，允许您进行智能查询。远远超出了传统搜索的能力。

随着丰富的搜索，你也可以得到增强的搜索结果。我以前在博客上写过这个，所以来看看。这真的是很酷的东西，我会在会议过程中更详细地讨论它。

**推销语义网**

如果所有的 Drupal 站点都嵌入了 RDFa 标签会怎么样？首先，雅虎会非常高兴。这将直接发挥雅虎新语义网战略的优势。他们试图用语义数据做一些有趣的事情，但是他们当然需要数据——典型的先有鸡还是先有蛋的事情。

拉斯穆斯提到，雅虎的语义数据存储可以扩展到网络的规模，因此邀请是开放的。

**Drupal 的未来**

这就是我在 Drupalcon 的工作重点，推动语义技术在 Drupal 中的采用——我觉得这里的势头将使这成为现实。有很多人感兴趣，一个语义网 BoF 会议挤满了有一些很酷的想法的人…

更多即将到来。
