# 反对 web 2.0 的理由

> 原文:[https://www.sitepoint.com/the-case-against-web-20/](https://www.sitepoint.com/the-case-against-web-20/)

Via [西蒙](http://simon.incutio.com)，[吉娜·特拉帕尼](http://ginatrapani.org/)在[反网页 2.0](http://ginatrapani.org/spun/posts/2006/07/24/the-anti-web-20) 中问了一些*非常*相关的问题…

> 我仍然更喜欢我的文本文件和我自己的 MySQL 数据库，而不是别人的，我不知道为什么我在这种情绪中如此孤独。

你肯定不是一个人，但是博客，一般来说，只会加强积极的言论，通常是不加批判的。你在这里也步入了政治，这在科技博客上不太好传播。尽管这一次我打破了自己的习惯，我还是要插一脚，以轶事的方式。

当我在大学学习三明治课程时，我在伦敦的 IBM 工作了一年。当时我的一位经理在某种程度上是 IBM 的高材生——一个有趣的家伙，但我一直不知道该对他说些什么——代沟、对长辈过于尊敬等等让我说不出话来。总之，长话短说，我们一整天都在参加 IBM /客户会议，讨论我们参与的产品，他让我搭车回家。大部分时间都是尴尬的沉默，对我来说，我不知道如何开始一个可持续的对话。然后，看似突如其来的；

> 你知道哈利；三年后，权力腐败。我一次又一次地看到同样的事情——谁有权力或者他们有多少权力并不重要——三年后，权力总是会导致腐败。

…这让我更加无话可说，但从那以后就一直困扰着我。三年在这里是否是一个神奇的数字，我不能说，但我确实认为这个原则是正确的。这就导致了一个结论:权力本身就是问题，不管谁拥有它，我们在分配它的时候都应该非常谨慎。

好的——如果你不是在为政治逃命——那么这和 web 2.0 有什么关系呢？嗯，简单地说，通过将我们所有的数据分发到中央存储库，不在我们的控制之下，也不对我们完全开放，我们将大量的权力掌握在少数人手中。有一些关于后果发生的讨论，但是远远不够。

以 del.icio.us 为例(并不是说我觉得 del.icio.us 有什么问题)。对于个人用户，del.icio.us 有一个很好的政策，已经实现了一些功能，如果你想离开，可以导出你的书签。但是他们没有给出的是“大图”(部分原因是很难以可扩展的形式实现)——作为一个普通用户，你只能查询这么多。但假设我是一名雇主，我正在寻找一名炙手可热的 Javascript 开发人员——我可以(理论上)找到一名开发人员的一种方法是追踪 del.icio.us 用户，他们一直是 del.icio.us 上最先收藏热门链接的用户之一——这很可能是至少在 Javascript 方面领先的人(也许不是一名理想的员工，但仍然是)。从技术上讲，您可以执行这个查询，因为它只有一个标签，但是您会避开 API 滥用的边界——从 del.icio.us 术语；

> 4.提要和 API
> 
> [……]您不得使用这些或任何其他功能或网站本身来显示 del.icio.us 数据库的大部分内容，也不得复制、复制或拷贝 del.icio.us 网站。[…]

如果我想在多个 del.icio.us 标签上进行这种搜索，或者开始分析在 del.icio.us 上建立的一些社交网络，我肯定会滥用 API 并产生一些重要的 HTTP 流量。但是 del.icio.us 拥有这些查询，这可能会给他们机会将它们作为一项高级服务出售——这就提出了一些关于谁在购买的重要问题…

好的——就我个人而言，我*不*认为这种类型的数据视图应该以某种方式受到保护，并被保密。相反，我认为这是关于它对*任何想要它的人*开放，而且(理想情况下)我希望能够看到我的数据是如何被使用的。这是一个[免费数据](http://www.freeourdata.org.uk/)的事情。

总之足够散漫。针对 Gina 的兴趣和未来预测数据，Mark Pilgrim 关于[长期备份](http://diveintomark.org/archives/2006/05/08/backup)的帖子(以及所有评论)值得一读，但不是决定性的。从更广的角度来看，这都是关于(难以解决的)实现细节——我们如何真正做到一个*去中心化*的 web，同时保留 web 2.0 的所有优点？
