# 为雅虎热身！用户界面

> 原文：<https://www.sitepoint.com/warming-up-to-yahoo-ui/>

凯文的在雅虎上写了一对[夫妇](https://www.sitepoint.com/javascript-libraries-and-patterns-yahoo-does-ajax/)或[时代](https://www.sitepoint.com/yahoo-ui-and-dp-libraries-updated/)！的用户界面库。随着每周不断有新的 DHTML 和 AJAX 库和框架发布，你对另一个不感兴趣是情有可原的。

但是浏览 [Simon 的](http://simon.incutio.com)[XTech](http://xtech06.usefulinc.com/)[talk](http://simon.incutio.com/archive/2006/05/18/yahooui)(PDF[here](http://simon.incutio.com/slides/2006/xtech/yui-slides.pdf))的幻灯片，特别是看看他们用 [Grid CSS](http://developer.yahoo.com/yui/grids/) 东西做了什么(对于像我一样苦于 CSS 布局的人来说，这似乎是一个重大突破)，我发现自己真的很感兴趣。浏览 Simon 在他的演讲中提供的代码和例子给我的印象是，他们真的成功地在做得太多和太少之间找到了平衡 yui 有真正的价值，但仍然“低脂肪”。

我想这里还有一个有趣的故事，关于把 yui 放在一起的开发团队。在某些地区，雅虎！似乎在[摸球](http://groups.yahoo.com/group/yws-maps/message/1904)的同时阅读 [Jeremy 的博客](http://jeremy.zawodny.com/blog/)，得到一个公司被中层管理吸干的[印象](http://jeremy.zawodny.com/blog/archives/006524.html)。尽管如此，yui 团队似乎已经完成了一些令人印象深刻的事情，这里是对他们的一些阅读，这里是对托马斯·沙(主角)的采访。

无论如何——只是刚刚开始试验/攀登学习曲线，所以没有真正的经验提供，但如果任何人(不是受雇于雅虎！)有什么笔记可以分享的话，很乐意听听。
