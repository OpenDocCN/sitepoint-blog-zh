# HTML 不确定的未来

> 原文：<https://www.sitepoint.com/htmls-uncertain-future/>

以下内容转载自《科技时报》第 152 期。

**如果你负责下一个版本的 HTML，你会改变什么？你会为文档中需要标记的内容添加新标签吗？你会去掉你从不使用的标签吗？什么是一个好的 HTML 标签，你如何适应那些不需要在 HTML 中添加标签的特殊需求？**

所有这些问题都被 W3C 的成员问过和回答过，但并不是所有的答案都经受住了时间的考验。在上个月末的一篇具有里程碑意义的博客文章中，W3C 主管蒂姆·伯纳斯·李承认 XHTML 未能兑现良好格式的可扩展标记的承诺，主要是因为浏览器继续毫无怨言地处理老式的“标签汤”HTML。

结果，他宣布 W3C 将修改它的 HTML 方法，以一个新的工作组开始恢复长期被放弃的 HTML 规范的开发。这个小组将并行地给 HTML 和 XHTML 添加新的特性，独立于继续努力开发 XHTML 的下一个版本[的工作。](https://www.w3.org/TR/xhtml2/)

如果我们可以期待这件事会有什么结果，那就是一系列新的 W3C 认可的 HTML 标签。关于这些标签应该是什么的分歧已经出现。

一方面，W3C 自己也在默默无闻地研究 XHTML 2.0(T1)，它提出的建议依次是[合理的](https://www.w3.org/TR/xhtml2/mod-structural.html)、[有争议的](http://www.alleged.org.uk/pdc/2003/xhtml2-cite.html)，以及[奇怪的](https://www.w3.org/TR/xhtml2/mod-handler.html)。

与此同时，失望的各方厌倦了 W3C 坚持网络是文档的媒介(排除了应用程序),他们成立了 T2 网络超文本应用技术工作组(WHAT WG )( T3 ),开始开发新版本的 HTML，他们混淆地称之为“HTML 5 ”,尽管它由两个规范组成:[网络应用 1.0](http://whatwg.org/specs/web-apps/current-work/) 和[网络表单 2.0](http://whatwg.org/specs/web-forms/current-work/) 。这些规范包含了他们自己的一套关于新 HTML 标签、[的](http://www.whatwg.org/specs/web-apps/current-work/#scs-dynamic)[建议，其中一些](http://lachy.id.au/log/2006/10/fixing-html#fixhtml-whatwg)甚至已经开始在最新的浏览器中被采用。

尽管这些问题已经酝酿多年，但蒂姆·伯纳斯·李的声明重新点燃了人们的讨论。一些有影响力的社区成员已经开始[发布他们的愿望清单](http://www.456bereastreet.com/archive/200610/new_w3c_working_group_to_improve_html/)，其他人[质疑 W3C 在 HTML 中选择合理补充的记录](http://blog.fawny.org/2006/10/28/tbl-html/#TBL-HTML-not-the-problem-2006.10.28)，还有一些人反对这种被认为是倒退的做法，[呼吁重新关注 XHTML](http://cafe.elharo.com/xml/why-tim-berners-lee-is-wrong/) 。

在这个阶段还不清楚的是，如果有的话，工作组将如何参与 W3C 内部开发 HTML 的新工作。Berners-Lee 的声明只是间接提到了该小组的工作，承认 [Web Forms 2.0](http://whatwg.org/specs/web-forms/current-work/) 将“告知”W3C 关于 HTML 表单的新工作。

令人失望的是(对于 W3C 来说), WHAT WG 在 Berners-Lee 的声明之后做出了第一个有意义的举动，发布了[邀请 web 开发者分享他们的想法、需求和关于 HTML 未来的问题。不希望被排除在这个过程之外，看起来这个组织急于证明它愿意让广大开发者社区参与进来——这是 W3C 一直未能做到的。](http://www.webstandards.org/2006/11/07/have-your-say-about-the-future-of-html/)

我心中的理想主义者想知道 WHAT WG 是否会自然地成为 W3C 中开发者社区的代言人。如果 W3C 不能与开发人员建立有意义的联系，也许 WHAT WG 可以。这需要资金来支付 W3C 会员费和相关人员的时间，但私人捐款和专业组织的贡献可能会对此有很大帮助。

最后，这些公告的影响在开发者的日常工作中被感受到还需要几年时间，但是我需要在这里同意 WHAT WG 的观点:如果 HTML 有一个新的开始，它将需要我们现在就参与到这个过程中，从一开始就确保这不仅仅是另一个错误的开始。
