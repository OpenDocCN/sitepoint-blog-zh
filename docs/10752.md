# 你能在 10K 建立一个全功能的网络应用吗？

> 原文:[https://www . site point . com/10k-apart-we B- application-competition/](https://www.sitepoint.com/10k-apart-web-application-competition/)

对于所有构建占用大量带宽的页面的开发人员来说，这是一个挑战——你能在 10K 或更短的时间内构建一个有用的、功能齐全的 web 应用程序吗？如果是这样，请在 2010 年 8 月 25 日之前前往[10k.aneventapart.com](http://10k.aneventapart.com/)提交您的参赛作品。

这个由微软运营的比赛是一个独立的事件，其灵感来自于 2000 年至 2002 年间运营的[the5K.org](http://www.the5k.org/)*(是的，那确实是很久以前的事了)*。即使在 Firefox、Ajax、Web2.0 和 HTML5 时代之前，也有一些优秀的作品，包括一个可播放的 [Wolfenstein 3D 克隆](http://www.wolf5k.com/)。

新的挑战提高了所有 HTML、XML、文本、CSS、JavaScript 和图像文件的 10K 总量限制。但是，您可以包含 jQuery、Prototype 或 Typekit 库，它们不会计入您的 10K 限制。我个人认为这是一种耻辱。我希望评委能给那些没有使用图书馆的作品加分——尽管很少有人会这么做。

有趣的是，规则中没有明确提到跨域 web 服务——尽管它确实说如果你有许可，可以使用第三方数据。我不明白为什么你不能有一个`script`-注入的 Ajax 调用到一个数兆字节的基于服务器的进程？好吧，这可能不在规则里，但这违背了比赛精神——如果你被取消资格，可别怪我！

所有提交的应用程序必须在 IE9 开发者预览版、火狐浏览器、Chrome 浏览器、Safari 浏览器或两者同时运行。没有提到 Opera，但是我希望评委们能给那些跨越更多浏览器边界的应用程序加分。

在太平洋时间 2010 年 8 月 25 日下午 3 点的截止日期之前，您最多可以提交 3 份参赛作品。这些申请将由行业专家杰弗里·泽尔德曼、埃里克·迈耶、杰里米·基思、惠特尼·赫斯和妮可·沙利文进行评审，然后将 10K 奖金分发给幸运的获奖者。网站上还有一个由投票者选出的人民选择奖。

如果你能充分利用每一个字节，访问[10k.aneventapart.com](http://10k.aneventapart.com/)和[条款页面](http://10k.aneventapart.com/Terms)。名利双收！

我认为这是个好主意。这将证明你可以构建一个不堵塞互联网管道的鼓舞人心的网络应用程序。你会提交参赛作品吗？让我们知道…
