# JBuilder 寻找美好家园

> 原文:[https://www.sitepoint.com/jbuilder-seeks-good-home/](https://www.sitepoint.com/jbuilder-seeks-good-home/)

在整个网络上，头条新闻都在宣布 JBuilder(Borland 的老牌 Java IDE)的消亡。

事情的真相一如既往，并不是那么一目了然。Borland 宣布打算出售其 IDE 业务，包括所有负责生产和销售 JBuilder 的开发人员和员工，以及 Delphi、C++Builder、C#Builder 和其他软件。该公告引用了该公司首席产品宣传员关于该产品在“新公司”光明前景的许多令人鼓舞的话。

这样的公司何时会出现，以及它是否有预算和动力让 JBuilder 继续参与竞争激烈的 Java IDE 游戏(目前由 Eclipse 和 NetBeans 主导，IDEA 是一个时髦的竞争者)，还有待观察。

作为 JBuilder 的长期用户，我一直很喜欢这个产品丰富的特性集。似乎只要值得做，JBuilder 就能做到。这并不总是简单的 T1 或吸引人的 T3，但是你可以做到。Borland 最近发布了 JBuilder 2006，这是 IDE 的第 12 个主要版本，其中的主要变化(除了支持新的标准和技术之外)是在团队开发领域:有了新版本，两个开发人员可以在各自的桌面上共同开发一段代码。IDE 为开发人员提供了聊天和控制共享代码视图的工具。

但是随着 JBuilder 在过去几年中的发展，我注意到越来越多的焦点放在产生像这样的新的“外围”特性上，而日常编码的核心体验相对来说保持不变。由于其创新的竞争对手已经部署了丰富的编码辅助工具(如 IDEA 中的源代码静态分析和 Eclipse 中的持续集成)，JBuilder 在这些领域的努力在某些情况下似乎没有经过打磨。

几个月前，我决定转向 IntelliJ IDEA，它向像我一样从事开源产品的开发者提供免费许可。虽然我错过了 JBuilder 的一些特性(比如在构建过程中对 JAR 归档签名的支持)，但整体环境和工作流感觉更加完善，更加仔细，而 JBuilder 总是感觉只是一个令人眼花缭乱的特性集合。

直到这个声明，JBuilder 发展的下一步是在 Eclipse 平台上的[重生，代号 Peloton，一些基础组件是开源的。无论哪家公司收购 Borland 的 IDE 业务，这款正在开发的产品的权利都将归属该公司，但在此之前，新公司宣布其意图，可以肯定地说 JBuilder 的未来——以任何形式*处于危险之中。*](https://www.sitepoint.com/some-ide-news/)
