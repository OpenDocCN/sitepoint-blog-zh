# HTML 5:机不可失，时不再来？

> 原文:[https://www.sitepoint.com/html-5-now-or-never/](https://www.sitepoint.com/html-5-now-or-never/)

在 SitePoint，我们已经开始思考 HTML 5，以及是否是时候出版一本关于它的书了。为了帮助我们做出决定，我们询问了一些网络名人的想法。他们的回答既多样又有趣。看一看，自己决定:是时候开始学习 HTML 5 了吗？

SitePoint 的《CSS 的艺术&科学的合著者 Jonathan Snook 采取了观望的态度。“你只能实现浏览器支持的东西，目前，HTML 5 支持的东西很少，”他说。事实上，*我们接触的所有专家都一致认为，HTML 5 的大部分新功能还没有得到最新的网络浏览器的支持。*

SitePoint 的《终极 CSS 参考资料》的合著者汤米·奥尔森(Tommy Olsson)至少可以说是持怀疑态度的:“我现在对 HTML 5 没有任何真正的兴趣。我所负责的网站的绝大多数访问者都使用不支持 HTML 5 的浏览器，这种情况可能还会持续几年。”

也就是说，有许多真正有用的 HTML 5 特性，你*今天可以*使用。用于动态绘制图形的 [`canvas`元素](http://dev.w3.org/html5/spec/Overview.html#the-canvas-element)就是一个很好的例子；作为 HTML 5 中的新功能，它已经被除了 IE 浏览器之外的所有主流浏览器所支持(IE 也有[的变通方法)。对于受人尊敬的网页设计师](http://code.google.com/p/explorercanvas/ "the explorercanvas project at Google Code")[戴夫·谢伊](http://mezzoblue.com/)(因构建 [CSS Zen Garden](http://csszengarden.com/) 而闻名)，`canvas`是 HTML 5 最令人兴奋的特性。“原生矢量图形。终于！SVG 仍然感觉不到它已经到达了今天的 web，尤其是在移动浏览器中；另一方面，`canvas`已经可以在少数手机和大多数桌面浏览器上使用，我认为可以肯定的是，其他浏览器很快就会赶上来。”

不太为人所知的 HTML 5 特性，如离线数据存储、跨文档消息传递和对后退/前进堆栈的访问，这些主要是 JavaScript 开发人员感兴趣的，也在最新的浏览器中出现，包括 IE8。

SitePoint 的《终极 HTML 参考》和[使用 HTML & CSS](https://www.sitepoint.com/books/html2/) 以正确的方式构建自己的网站》的作者 Ian Lloyd 指出，如果你准备指定自己的 CSS 样式，你甚至可以在很少或没有当前浏览器支持的情况下使用 HTML 5 功能。“如果你使用像`section`这样的 HTML 元素，它*实际上可以*用 CSS 来样式化，即使浏览器不理解元素是什么(但它需要一个 [JavaScript hack](http://remysharp.com/2009/01/07/html5-enabling-script/) )。随着 JS 做好自己的工作，它可以让 HTML5 的采用变得更容易接受。”Ian 甚至在他的 [Markup Maker](http://accessify.com/tools-and-wizards/developer-tools/markup-maker/) 工具中加入了 HTML 5 支持，它将根据你指定的结构为你生成网页的 HTML 框架。

设计师安迪·克拉克是《超越 CSS 的 T4》一书的作者，也是美丽网页的创造者，他认为现在还不值得使用 HTML 5 元素，但是他正在为下一步做准备。“我不打算很快就用 HTML 5 开发，但我确实认为以某种方式为他们的新语义做准备是很重要的。这就是为什么我修改了自己的命名约定，以包含 HTML 5 元素名称(`class="header"`、`"section"`、`"article"`、`"aside"`等)。)."尽管如此，Clarke 认为，如果只是为了给潜在客户留下深刻印象，学习 HTML 5 是值得的。“与任何重要的新技术一样，设计者和开发者应该尽可能早地理解 HTML 5 的细微差别，以便他们能够充分意识到它的优势。当客户询问 HTML 5 时，这也很重要，因为了解内情的人肯定会这样问。”

戴夫·谢伊建议你选择一种适合自己工作类型的方法。“我认为这是开始学习和尝试的好时机。有些人走得更远，已经用 HTML 5 发布了完整的网站，但我认为这最适合个人和实验性网站。我现在会用 HTML 5 重新设计我的博客吗？当然，我可能会试一试。我会用 HTML 5 做客户端工作吗？天哪，不。”

汤米·奥尔森可以为当今的设计师们想出更好的打发时间的方法。“我认为开发者首先应该集中精力学习 HTML 4.01。太多的人甚至不理解这一点。无论如何，要密切关注 HTML 5，但它还需要好几年才能被普遍使用。”即使你真的了解你的 HTML 4，安迪·克拉克似乎也同意可能有更好的地方来集中你的注意力。“我坚信网页设计和开发还有其他更有用、更重要的方面值得我们关注，尤其是 T2 的微格式。微格式的准入门槛很低，建立在现有的语义之上，在 2009 年提供了比 HTML 5 更有用的机会。

这涵盖了现在，但是这些有影响力的人对 HTML 5 的未来有什么看法呢？安迪·克拉克怀疑 HTML 5 会对他的工作产生任何真正的影响。“我希望一切顺利，但我确实感觉到，当 HTML 5 成为我们目前使用的主流替代品时，我将只是谷歌搜索的一个幽灵结果。1865 页，是的，我会在那里。”

可以预见的是，汤米·奥尔森希望它永远见不到天日。“我认为这是令人憎恶的，但我可能是少数。如果/当 HTML 5 变得流行时，我正在认真考虑转行。它嘲弄了我认为在网络上重要的一切:语义、可访问性和内容与表现的分离。最糟糕的是，它重新定义了现有元素类型的语义，以至于十多年来语义正确的标记突然变得毫无意义。”

无论是哪种情况，乔纳森·史努克都更倾向于把事情往积极的方面想。“虽然有些人担心 HTML5 的方向，认为它的进展不够快，但我很高兴看到任何进展，浏览器至少实现了其中的一些功能。”

你怎么看？SitePoint 应该出版一本关于 HTML 5 的严肃书籍，还是应该暂时忽略它，希望它消失？
