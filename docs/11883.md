# 漂亮的网址:非常简单！

> 原文:[https://www.sitepoint.com/pretty-urls-pretty-easy/](https://www.sitepoint.com/pretty-urls-pretty-easy/)

当我写第三版 [*使用 PHP & MySQL*](https://www.sitepoint.com/books/phpmysql4/) 构建你自己的数据库驱动的网站时，将所有的代码放在一起生成一个网页到一个文件中，然后使用一个 URL 命名该文件来访问页面，这仍然是可以接受的(尽管肯定不流行)。

PHP 已经向前发展了。如今，以这种方式构建网站的开发人员不太可能找到工作(或许除了报酬很低的工作)。开发人员被期望将他们的代码组织到多个文件中，包含像 index.php 这样的文件名的 URL 肯定要被淘汰了。

有经验的开发人员将使用精心制作的框架和 web 服务器配置来转换所谓的漂亮的 URL，如 http://www.example.com/thingy/. No。php”广告事实上，该网站是建立在 PHP，没有具体的文件名。像这样的 URL 被设计得尽可能简单和容易记忆。

对于本书的第四版，我希望开发人员早点养成这种习惯，而不是强迫他们摆弄他们的服务器配置，或者学习复杂的 PHP 框架。

我所能找到的制作漂亮 URL 的最简单的方法是利用内置在每一个配备 PHP 的 web 服务器中的一个特性:一个指向服务器上一个目录的 URL 将自动运行位于该目录中的一个名为`index.php`的 PHP 脚本。

记住这一点，以下是本书从第 3 章开始的所有例子的结构:

*   **一个文件夹**
    所有的东西都放在一个文件夹里，文件夹的名字是 URL 中唯一可见的部分。因此，如果示例显示“thingies”，您将首先创建一个名为 thingies 的文件夹，示例的 URL 将是 http://www.example.com/thingies/
*   **一个控制器，index.php**
    当浏览器请求例子的 URL 时，服务器会自动运行文件夹中包含的 index.php 脚本。这个文件被称为**控制器**，包含所有处理浏览器请求和执行响应任务的逻辑。
*   包含文件，比如 db.inc.php
    将在不止一个例子中有用的代码片段从 index.php 中提取出来，放在单独的 PHP 文件中。对于 PHP 老手来说，这是基本的东西，但这是一种有价值的编码技术，在第三版的第 12 章中有所介绍。现在在第三章。
*   **模板文件，像 thingy.html.php**
    一旦控制器(在一个或多个包含文件的帮助下)做了所有的工作，它必须准备一个网页发送回浏览器作为响应。第四版没有把所有的 HTML 代码都塞进 index.php(这是我们在第三版中一直做的)，而是为每种类型的页面使用一个单独的 HTML 代码文件。这些模板文件主要由纯 HTML 组成；它们包含的唯一 PHP 代码是用来输出控制器创建的 PHP 变量的值。这些文件中的代码非常简单，因此可以交给可能根本不熟悉 PHP 的设计人员。此外，使用单独的文件使得从单个控制器生成几种不同类型的页面(即几种不同的模板文件)变得容易。
