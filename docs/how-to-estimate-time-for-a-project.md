# 如何估算一个项目的时间

> 原文：<https://www.sitepoint.com/how-to-estimate-time-for-a-project/>

无论你是按小时还是按项目向客户收费，所有项目的一个必要步骤是估算所需的时间。客户不仅想知道他们将要花多少钱，而且还需要围绕一个估计的时间表进行计划。你需要能够确保你有必要的时间和资源来完成这个项目。

取决于许多因素，包括你对你正在做的工作类型有多少经验，你是否使用分包商，以及你从客户那里得到的信息，估计一个项目的时间可能是困难的。以下是我在界定新项目的时间承诺时使用的流程。

**确定可交付成果**

第一步是确定主要项目(即网站重新设计)，然后确定与项目相关的具体交付内容。例如，重新设计完成后，您将通过 FTP 向客户提供新设计的网站文件，并向客户发送包含工作文件的 CD 或 USB 驱动器。

**分解一下**

接下来，我将项目分解成由组件分隔的简单任务——越具体越好——这将使我们获得可交付成果。以下是任务的一个示例:

计划图编制

*   与客户初次会面，评估项目范围
*   向客户提供项目信息表，以获得更多关于他们喜欢/不喜欢现有网站的信息
*   审查/分析现有网站和客户表单
*   制定需要进行现场变更的区域列表
*   获得客户的批准

设计

*   设计网站模型
*   获得客户的批准
*   代码页
*   创建新导航
*   将内容重新组织到新页面中
*   针对 SEs 进行优化

测试

*   跨浏览器测试
*   验证代码
*   检查链接
*   测试表格

**加起来**

下一步是估算每项任务的时间，四舍五入。如果你使用分包商，你需要先得到他们的时间估计，并把他们算进你的时间里。然后把所有任务的总时间加上一个缓冲区。缓冲剂可以是任何东西，尽管我通常坚持 10-25%的添加量。这样可以应对可能出现的任何意外情况或挑战。

**需要牢记的事情**

你做的时间估计越多，你就越准确。当您创建自己的公式时，您可能要考虑的一些其他因素包括:

*   项目管理时间
*   审查分包商工作的时间
*   项目期间发生的假期或休息日
*   客户周转时间
*   排除故障
