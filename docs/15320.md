# 纯文本还是 HTML？让你的用户决定

> 原文:[https://www.sitepoint.com/text-html-let-users-decide/](https://www.sitepoint.com/text-html-let-users-decide/)

在我的[上一篇专栏文章](http://www.ecommercebase.com/article/516)中，我讨论了用 HTML 和文本格式发送电子邮件简讯的利弊。我收到了许多反馈，询问如何知道发送哪种格式是正确的，以及更多的具体信息。

首先让我说这是一个很有争议的话题。有些人会信誓旦旦地说你永远不应该(我是说*永远不要*)发送 HTML 格式的电子邮件。其他人(比如我自己)说，HTML 电子邮件是一个强大的工具，因为它允许你在收件人的电子邮箱中开发更深层次的品牌识别，并更有力地展示你的产品。

##### 基本原则

也就是说，让我陈述一下这个主题的一个基本原则:允许用户选择他们自己的偏好。一旦他们做出选择，尊重他们的选择，不要把你自己的偏好强加给他们。

虽然尊重访问网站或订阅电子邮件简讯的人的偏好似乎是常识，但有时雄心勃勃的营销人员会选择无视受众的偏好。这样做的主要原因是，从技术上来说，确定某人的计算机是否能阅读 HTML 电子邮件非常容易。当那个人打开他或她的电子邮件时，web 服务器提供了一个图形，很明显，那个人的计算机可以显示 HTML 电子邮件。这是一个被称为“嗅探”HTML 的过程。然而，这是关键，仅仅因为有人使用 HTML 格式的电子邮件程序并不意味着他或她想收到一个特定的 HTML 格式的简讯。

有许多人更喜欢纯文本格式的电子邮件——即使他们的软件能够处理 HTML 邮件。这些人通过两种方式表达他们对纯文本的偏好:一种是通过在订阅表单上做出这一选择，选择文本而不是 HTML，另一种是当他们注册一份仅以文本模式提供的简讯，并在收到几期后继续订阅。

有时，网站管理者会试图将简讯订阅者的偏好从文本格式改为 HTML 格式，因为研究表明 HTML 格式的简讯点击率更高。人们很自然地会认为，改变用户的偏好会增加回应。

然而，这不是一个好主意。

##### 不要强迫你的格式

互联网行业的人们倾向于接受 HTML 时事通讯，而时事通讯出版商通常能够成功地自动改变该群体的订阅者偏好。然而，当涉及到发送简讯给这个行业以外的人，这是一个不同的故事。当订阅者的时事通讯偏好未经许可被转换时，他们会发尖叫的电子邮件向出版商投诉。

虽然尊重访问网站或订阅电子邮件简讯的人的偏好似乎是常识，但有时雄心勃勃的营销人员想无视他们的观众的偏好。

这绝不是个好主意。

##### 倾听你的用户

作为一名营销人员，我知道关注行业平均水平来验证我与自己客户的关系是错误的。我应该询问并倾听这些客户告诉我他们希望如何接收内容和优惠。不要试图通过查看调查来猜测客户的接受程度，而是直接询问客户的偏好。

时事通讯应该有一个偏好问题，允许电子邮件订阅者设置他们自己的格式偏好。毕竟，并不是每个能收到富媒体邮件的人都想收到——他们可能更喜欢纯文本。

此外，为什么不让你的顾客告诉你他们想要什么？
