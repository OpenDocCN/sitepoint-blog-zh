# 为什么浏览器升级的呼声永远不会停止

> 原文：<https://www.sitepoint.com/no-end-to-browser-upgrades/>

Web 开发者很少会开心。我们对自己的工作吹毛求疵，对别人的批评更是如此，我们永远不会满足于我们所拥有的工作工具。

浏览器是我们最大的障碍。人们从来不会使用正确的浏览器，也不愿意升级，即使是为了他们自己的利益。当如此多的用户停留在过去时，我们怎么能期望推动网络前进呢？

Internet Explorer 6.0 是 web 开发人员生活中的祸根。尽管有两个更新的版本，该应用程序在推出九年后仍在使用。我们为什么要为了少数误入歧途的个人和公司而奋斗？

事情是这样的:这部分是我们的错。我们基于 web 应用程序可以解决与桌面程序相关的分配和管理问题这一点来推销 web 应用程序的想法。员工可以使用浏览器，不再需要任何软件或升级。不幸的是，我们经常忘记提到浏览器本身总是需要更新的。这对我们来说可能很容易，但新手不太了解，对于有数千台桌面需要支持的公司来说，这是一个很头疼的问题。

IE6 的问题部分是我们自己的懒惰造成的。我们走了捷径。我们专注于有趣的东西，避开世俗的东西。我们编写的应用程序在 IE6 中运行良好，但在所有后续浏览器中都失败了。

我敢肯定你在想…啊，但是 IE6 不一样；微软的主导地位和懒散导致了我们今天遇到的许多问题。因为 IE6 是唯一的浏览器，所以只编写 IE6 应用程序。这是真的，但是同样的问题在其他情况下也会出现。

考虑网络标准。就在你以为厂商在朝着一个共同的目标前进的时候，他们开始出现分歧，实施自己的想法。浏览器已经与 HTML5、CSS 动画、视频格式和离线功能等技术产生了分歧。再过几年，开发者会哀叹火狐对 X 的不支持或者 Chrome 对 Y 的糟糕处理吗？

那么 IE6 的结束会让你我还有其他人开心吗？十年前，我们呼吁网景 4 去死(如果你认为 IE6 很糟糕，NS4 会让你大吃一惊！)。IE6 的死亡可能迫在眉睫，但我们会开始妖魔化其他浏览器吗？
