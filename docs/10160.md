在[的上一篇文章](https://www.sitepoint.com/article/microsoft-web-app-gallery-part-1)中，我们从[的 Windows 网络应用程序库](http://www.microsoft.com/web/gallery/)中选择了 nopCommerce 来构建一个功能正常、可定制的电子商务网站，并将其配置为接受通过 PayPal 的支付。为了让这个过程变得更加简单，我们使用 [WebMatrix](http://www.microsoft.com/web/webmatrix) 来下载、安装和管理我们从图库中选择的应用。

WebMatrix 可用于深入编码领域，一旦您开始使用它，您几乎肯定会想要使用它，但我们将在下一次使用它。在本文中，我们将使用 WebMatrix 来了解 Web 应用程序库在**内容管理系统**方面提供了什么。在这个过程中，我们可能会发现微软的态度正在发生一些变化。

## **内容管理系统**

内容管理是一个与所有类型的存储文档、记录、对象、工件和数据相关的术语。收集的内容越多，就越需要一个用于聚集、存储和检索内容的系统。

A Web Content Management System (WCMS) is a tool, or set of tools, specifically for managing the content of a website. Since we’re focusing exclusively on web content in this article, we can safely stick with just calling it a CMS.

以下几点有助于我们定义 CMS:

*   允许添加、编辑和删除网站内容
*   在线存储网站内容
*   提供基于浏览器的访问
*   不需要详细的技术知识

除非您特别想要一个开箱即用的模板，否则您可能会寻找一个 CMS，它还:

*   允许定义、配置和编辑可视化网站设计和网站结构

这将让您为您的内容创建一个功能强大、结构合理、视觉愉悦的环境，而且更重要的是，将让没有经验的开发人员和设计人员进入动态网站的世界，而不是静态网页。

## **选择您的 CMS**

让我们回到 Web 应用程序图库所提供的内容，并注意这些应用程序在图库中排名的共同特征:

*   它们在 Windows 服务器上都可以很好地运行。
*   它们都是免费的——至少作为入门版或社区版。
*   在某种程度上，它们都是受欢迎的，并且经过了市场的检验。
*   他们都有一个用户社区来提供支持。

根据您对内容的了解以及 Web 应用程序图库中提供的描述，您至少可以减少一些选项。即便如此，你可能需要一些东西来帮助你进一步缩小选择范围。

CMS Matrix 是一项免费的在线服务，允许您将内容管理系统与一系列非常全面的标准进行比较，其中大多数标准都是有经验的开发人员所熟悉的。

对于 DotNetNuke，我们将遵循相同的路径。对于**选择数据库**选项，点击 **SQL Server 2005/200 (Express)文件**单选按钮。这很可能已经被选中，因为 SQL Server Express 已被检测为已安装。

类似地，如果安装了 SQL Express，**服务器**选项将被预填充为。SQLExpress，所以让它保持原样。

默认情况下,**文件名**字段也将预先填入 Database.mdf。您可以将其更改为您选择的数据库名称，但是在本练习中，我们将保留它。

对于 nopCommerce 安装，您可能记得我们选择使用集成的 **Windows 身份验证**而不是 **SQL Server** 帐户用户名和密码。我们将在这里做同样的事情，因为它将使用我们现有的 Windows 登录(您的计算机名和登录名)，而不必使用单独的用户名和密码。这是使用您的 Windows 帐户身份使生活变得简单的又一个例子。我们所要做的就是确保选中**集成安全**复选框。

请注意，如果您取消选中此框，系统会提示您输入一个**用户 ID** 和**密码**来访问数据库。如果您在网络上工作，这可能是必要的，请咨询您的系统管理员。

如果您选择这样做，您会发现您还可以取消选择**作为数据库所有者**运行，这意味着您将以**用户**而不是**所有者**的身份访问数据库。同样，如果您在网络上工作并访问网络服务器上的数据库，而不是您自己的硬盘上的数据库，这可能是必要的。

这个屏幕上的最后一个字段也是最有争议的。DotNetNuke 引入了**对象限定符**作为一种识别由两个不同的 DNN 安装使用的单个数据库的方法——这是一种旨在从托管帐户中获得最大价值的解决方案，该帐户可能仅限于支持一个 SQL Server 数据库，但支持多个域。通过给一个数据库单独的 OQ 前缀，DotNetNuke 可以支持两个 DNN 设施。

有很多 DNN 迷不喜欢这个选项，因为它可能会令人困惑，一旦选择就很难更改，而且当您可以选择支持多个 SQL Server 数据库的托管计划时，它并不是真正必要的。

然而，当您在 DNN 和一个单独的应用程序之间共享一个数据库时——比如 nopCommerce 这样的电子商务安装——有必要考虑防止在一个应用程序中使用的特定用途的表之间的冲突。

不管怎样，我更愿意将我的 DNN 数据库和 nopCommerce 数据库完全分开，所以我建议将**对象限定符**字段留空。你的情况可能不一样。

## **新方法**
