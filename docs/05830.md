# 2014 年最佳 PHP IDE 调查结果

> 原文：<https://www.sitepoint.com/best-php-ide-2014-survey-results/>

整整一个月前，我们公布了 2014 年最佳 PHP IDE 调查。4000 个词条之后，是时候和大家分享成果了。

## 最流行的 IDE

本文将只关注 IDE 结果。在对数据进行更大程度的清理之后，我们将在以后的文章中对 PHP 社区进行总体分析。请注意，这些是初步结果，还没有进行太多详细的筛选。数据仍将被处理和额外验证。大概的范围是正确的，但是不能被认为是精确的(在每个类别中可能会有几十个偏差，不足以影响最终结果)，因此图表中只会显示百分比值。具体数字见原始数据。

在之前，我们已经报道过 PhpStorm [，由于其社区的力量，它获胜并不令人惊讶，但是在超过 4000 个有效响应的样本中，像这样的优势肯定是没有预料到的。PhpStorm 是 IntelliJ IDEA(Jetbrains Java IDE)的产物，基本上是一个嵌入了 PHP 支持的精简版本。由于这种基于插件的性质，PhpStorm 可以轻松地支持其他语言，允许您在同一环境中开发 NodeJS、Dart、Go 和其他语言的应用程序——这是一个无价的好处。PhpStorm 投票者最流行的论点之一是，考虑到它的大小，它的速度非常快，支持开箱即用和通过插件的许多语言和框架，并且是真正的多平台，允许您像指向云托管的引用一样容易地共享单个配置文件。](https://www.sitepoint.com/phpstorm-top-productivity-hacks-shortcuts/)

Sublime Text 理所当然地排在第二——它是免费的(以 WinRAR 的方式),多平台的，并且速度很快。它在不到一秒的时间内加载，即时响应命令，支持项目，并拥有丰富的插件文化和一个非常活跃的社区。它支持最流行的语言，并通过插件添加更多的语言，从头开始配置机器时，它不需要大量的设置。

PhpStorm 的免费替代品 Netbeans 位于第三位。它具有几乎相同的功能，但是支持更少的语言，速度稍慢，而且需要更多的资源。尽管如此，一个优秀的 IDE 值得任何人关注。

进一步的结果显示，Zend Studio、Eclipse with PDT 和 Notepad++也很受欢迎，尽管没有一个能接近前三名所吹嘘的百分比。

### 一些参与者评论

当我说有数百个令人惊奇的答案时，我不是在开玩笑——是的，我确实读了其中的大部分。选择最显著的数据与其说是确定性的，不如说是随机性的，所以如果你有兴趣浏览其余的数据，这些数据都由你支配。

> Jeremy Dove 选择了崇高的文本:我只构建很小很少的页面应用程序。我需要一些跨平台和轻量级的。

这是有道理的。当您只在小项目中工作时，没有必要带着 PhpStorm 这样的庞然大物到处跑，即使它是多平台的。

> Lichai Cohn 选择了 PhpStorm:使用 IDE 总会有速度和功能的问题。我不认为 PHPStorm 是完美的 IDE，只是最不糟糕的一个。在我尝试过的所有想法中，这是最快的。如果速度如此重要，为什么不用文本编辑器呢？IDE 提供了如此多的额外功能，以至于我愿意用一些速度来换取功能。PHPStorm 的主要问题是 bug 没有解决，而新的特性却在增加。这最终会让我再次尝试其他 ide，但目前没有一个 ide 更好。

我同意他的观点——PhpStorm 确实非常快，而且确实在 bug 上花了不少时间。但这就是如今软件的现状——看看浏览器吧，它是目前使用最多的软件类型，仅次于操作系统。每一家都在添加新功能方面与另一家竞争，但几十年来都没有修复错误——击败其他供应商的愿望是如此强烈，以至于错误被简单地掩盖起来。开发人员意识到新特性远比缺陷修复更引人注目，可悲的是，这正是他们关注的焦点。

> Joe Campo 选择了 Netbeans:我过去使用 DevPHP，但是当我需要一个有版本控制等功能的成熟的 IDE 时。我试过 Netbeans。我开始适应了。因为我已经尝试了其他 ide，有些甚至有更好的特性，但是我很难切换。我发现自己回到了 Netbeans，因为我感觉很舒服。

舒适确实起了很大作用——我能理解。在为我的第一个雇主做一个大型 Zend 项目时，我已经使用 Zend Studio 6 很多年了。这成了第二天性，似乎完全足够了。在一些同事的要求下，我尝试了 Netbeans，并立即被迷住了。在持续使用、快速响应、丰富的插件架构、卓越的键盘快捷键的日子里，没有一次崩溃。从那一刻起，我一直在寻找更好的东西，从未停止。走出舒适区是最难的一步，但也是最重要的一步。

> Sherwin 选择了 Dreamweaver:出于几个原因，我们在工作中更喜欢使用 Dreamweaver。第一个原因是我们同时在 PC 和 Mac 上开发。第二个原因是我们喜欢 IDE 使用不同的颜色来表示函数、类等。有很多方法可以配置 DW 来适应我们的编程方式。第三个原因是对多个开发者的支持。它有一个简单的入住/退房系统。我们希望进入 GIT，但在我们的工作环境中，我们几乎没有时间研究替代方案或资金。我们不用 DW 的“自动编码”。我们使用 CodeIgniter 作为框架从头开始构建东西。DW 的“自动编码”太臃肿了，它不能做我们需要的一切。我们主要把它当做“类固醇上的文本编辑器”。
> 
> 我们试过 Notepad++很棒，但不幸的是它只在 Windows 中可用。它也没有一个易于使用的入住/退房系统。我们也试过 TextMate，但它只适用于 Mac。我们第二接近的首选 IDE 是 Aptana。它不仅是免费的，而且做 DW 做的事情，尤其是在管理缩进和代码自动完成时。不幸的是，它没有一个易于使用的入住/退房系统。

更有趣的评论之一——这一条感觉像是读到有人试图证明 WordPress 对于大型项目来说“足够好”。老实说，我对此感到惊讶——Git 最多花一天时间来学习，至少是基础知识，而且每个像样的编辑都支持它。如果您没有使用 DW 的 WYSIWYG 功能，那么它有什么好处呢？每个编辑器都有代码高亮显示，许多都有多平台支持和插件。Notepad++通常被认为是 Sublime Text 的小弟弟，Sublime 是真正的多平台，从零开始为每一个人构建，无需在 VM 中运行，这就是为什么我觉得这个答案特别有趣。

> Chuck Burgess 选择了 Eclipse with PDT: Eclipse 是我在 PHP 之前做 Java 时最初选择的自由/开源软件。随着时间的推移，我尝试的其他 ide 在工作流方面差异很大，以至于我对 Eclipse & PDT 的熟悉程度超过了它们。我想不起 Eclipse 的任何一个方面阻碍了我高效地工作。

另一个与舒适度相关的响应。有趣的是，这种反应在 40 岁以上的老开发人员中很常见。随着年龄的增长，我们是否正在失去对新技术的适应能力？我们会不会变成“老古董”，为了保留我们习惯的舒适而拒绝任何新事物？我们是否变得如此满足于我们所拥有的，以至于失去了尝试新事物的欲望，即使从长远来看有可能会更好？也许我们觉得我们已经过了“长期”了？

> [布莱恩](https://twitter.com/GameGamer43)选择了 Vim: Textmate，Eclipse，Xcode。继续回到 Vim，因为它随处可用(在服务器上等)。

许多“Vim”答案使用相同的论点——“随处可用”。事实上，这种广泛的可用性是一个很好的好处——完全掌握一个编辑器，然后在默认情况下几乎在任何地方都可以使用它，这一定会极大地提高生产效率。我个人对 Vim 还不够熟悉，无法适应，但是我可以想象 Vim 专业人员的开发流程非常流畅，尤其是在切换机器时。

> Mohammad Alhobayyeb 选择了 Eclipse with PDT:我喜欢 perspectives，因为我也在 Android 上使用它。所以我只想学一个适合所有人的 IDE。

透视图是一个很好的特性，但是看起来大多数人都在寻找 ide 和编辑器，当他们找到一个“足够好”的时候就停下了脚步，而不是继续寻找“完美”。如果“一个人统治所有人”真的是唯一的参数——有许多替代方案，其中最强大的是 IntelliJ IDEA。

你怎么想呢?为什么我们如此执着于我们的编辑和 IDEs，拒绝给其他人一个机会，即使我们内心深处知道有一个新的孩子可能会做得更好？在下面的评论里告诉我们，大家一起讨论吧。

## 结论

ide 的选择是很棒的，但是我们希望我们现在已经使它变得更容易了。没有竞争就没有进步，希望这些结果能鼓励其他厂商在功能和社区参与方面有所提高。非常感谢大家的参与——我们已经通知了所有获奖者，由于大量的回复，我们选择了九位而不是三位获奖者。
