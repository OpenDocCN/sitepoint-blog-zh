# 密码:大多数人做错了

> 原文：<https://www.sitepoint.com/passwords-most-people-do-it-wrong/>

快:你的密码是什么？是 123456 吗？是密码吗？是 abc123 吗？这是你的名字吗？令人惊讶的是，对于大量用户来说，这些都是用来保护私人账户的词。毫不奇怪，这是一件坏事。

大约一周前，开源论坛项目 phpBB [的网站遭到黑客攻击](http://area51.phpbb.com/phpBB/viewtopic.php?f=3&t=29973)。该网站用户的大约 20，000 个密码被公布到互联网上。虽然这肯定不是一件好事，但对于安全研究人员来说，这提供了一个独特的机会来研究真实用户如何创建密码。

上周，Dark Reading 的罗伯特·格雷厄姆[发表了一些关于被黑密码使用模式的发现](http://www.darkreading.com/blog/archives/2009/02/phpbb_password.html)。phpBB 数据集中的前 20 个密码列表并不令人鼓舞。超过 3%的账户使用的头号密码是“123456”名单上的第二个是“密码”第三个是“phpbb”事实上，几乎所有最常用的 20 个密码都是这些简单主题的变体:顺序排列的数字、键盘组合(如“qwerty”)或常用单词或名称。

格雷厄姆发现，65%到 94%的密码是常用的字典单词(后一个数字是包含常用专有名词的字典，如“Xbox”或“Pokemon”)，平均而言，这些单词往往是简单的单词，如“apple”或“orange”，而不是更复杂的单词。

16%的密码与一个人的名字相匹配。14%的密码是键盘上的图案。4%是单词“密码”的变体。5%的人提到了流行文化，4%的人在选择密码时可能描述了附近的事物(如“三星”、“优派”或“康柏”)。

被黑的 phpBB 账户的密码似乎展示了两年前 MySpace 类似入侵的密码中发现的类似模式，当时大约有 34，000 个用户名和密码被公开。来自攻击前二十名包括 password1、abc123、myspace1、password、qwerty1、123abc、123456、jordan23 和 iloveyou1。

这意味着，对于那些有这种倾向的人来说，如果他们使用这种不安全的密码，闯入许多用户帐户可能比人们想象的要容易得多。再加上 2008 年 1 月数字通信机构@www 的一项研究，发现 61%的人对他们拥有的每个账户使用相同的密码，你可以开始意识到这是一个潜在的巨大问题。

不安全密码和反复使用相同密码的原因可能是相同的:[密码疲劳](http://en.wikipedia.org/wiki/Password_fatigue)。人们现在需要记住如此多的账户凭证，这近乎荒谬。为了跟踪这么多不同的帐户，似乎大多数人一遍又一遍地重复使用相同的密码，并且经常选择容易记住的、不安全的短语。这是不好的形式，可能会使他们的帐户更容易受到威胁。

当然，一个解决方案是 OpenID。如果 OpenID 真的被普遍接受，那么人们*需要记住的就是一套登录凭证。当然，这也意味着如果你的账户被入侵，它会让攻击者一次访问所有内容(尽管考虑到目前人们展示的使用模式，这已经是事实了)。*

然而，如果人们接受安全密码实践的教育，更难辨别随机的字母、字符和数字串，并经常更换它们(每月？)，OpenID 可以使这一过程变得更加容易和可行。想法？
