# 建议:处处提供支持

> 原文:[https://www.sitepoint.com/advice-offer-support-everywhere/](https://www.sitepoint.com/advice-offer-support-everywhere/)

最近，我在电子邮件营销公司 [Feedblitz](http://www.feedblitz.com/) 的创始人兼首席执行官菲尔·霍洛兹的博客上看到一篇文章，建议网络初创公司删除他们的支持论坛。表面上看他的文章实际上是有道理的。 [Hollows 认为](http://feedblitz.blogspot.com/2009/02/support-forums-just-say-no.html)“论坛让公司摆脱了终端用户的痛苦和随后的改进机会。当然，可能会有好的论坛，但是监督一个论坛并不等同于提供支持，这确实是一条非常危险的路。是外包。”换句话说:论坛不够个性化，不足以用来提供支持，抛弃它们会迫使你为用户提供更好的支持。

但是 Hollows 真的忽略了一个关键点。您提供的支持质量与您使用的工具无关，而是与您如何使用它们有关。论坛只是一个工具，但它们可以用来为你的客户提供强大的支持，就像电子邮件和电话可以用来提供可怕的支持一样(任何曾经在我的当地有线电视公司等待过的人都知道)。

如果你能从言外之意看出，霍洛思论点的主旨是好的。虽然他的建议在愚蠢的“沟论坛”咒语中变得混乱，但他真正的观点是:提供良好的支持。当然，这是个很好的建议。根据 Hollows 的说法，一家公司“最有价值的市场资源是非常具体的、专家的客户反馈”，只有当你在个人层面上与客户互动时，你才能得到这种反馈。

我不明白为什么他认为你不能通过论坛得到这些。也许 Hollows 与一家通过论坛提供可怕的、非个人化支持的公司有过不愉快的经历。不过，说真的，公司应该接受 Hollows 一半的建议，忽略另一半。是的，专注于提供巨大的支持。不，不要扔掉你的论坛。

事实上——增加更多的论坛。公司应该在尽可能多的地方与客户互动(也就是说，不要降低你所能提供的支持的质量)。让你的顾客在他们更舒服的地方找到你寻求帮助。这意味着你应该通过电子邮件、论坛和电话提供支持，但也要通过 Twitter、脸书、MySpace 与你的客户互动，并获得满足感。

Hollows 的一位评论者似乎非常理解这一点，并提供了我认为最终要好得多的建议。

> 我觉得这是把婴儿和洗澡水一起倒掉。有很多公司(和其他组织)在论坛方面做得很糟糕，但这只是公司的反映，而不是工具。我同意单靠一个论坛很少是最好的支持形式，但如果做得好，即由公司监督和积极参与，它会非常有效。与其他形式的支持相结合，它可以为出色的支持体验做出巨大贡献。—杰夫·科布

好的支持不在于使用什么工具，而在于如何使用它们。只要你能继续提供高水平支持，你就应该提供尽可能多的不同联系方式。你*可以*通过论坛提供良好的支持体验，你或许应该这么做。此外，你不仅应该在论坛上提供支持，还应该在任何地方提供支持。
