# 普适可用性——为不确定的未来做计划

> 原文：<https://www.sitepoint.com/planning-uncertain-future/>

可用性是一种现象，它极大地改变了包括网站在内的产品的设计和制造方式。

传统上，产品开发周期中的可用性是人为因素或人类工程学专家的责任。今天，由于对可用产品的爆炸性需求，许多产品工程师、开发人员、设计人员和技术交流人员不得不在他们的组织中承担可用性工程的主要责任。在很少的情况下，这种责任的转移比在网站开发中更容易和成功地实现。

网站设计的每个阶段都是提升或削弱网站可用性的机会。因此，即使你面临一个紧迫的截止日期，重要的是要确保测试和修复网站所需的时间被纳入网站的生产过程。在测试上妥协会导致低效和无效产品的产生。

有许多原则支配着网站设计的可用性。设计方法可能不同，但本质上，所有以用户为中心的过程都以产品的可用性为主要标准。然而，重要的是，设计过程支持可用性，而不给必须忍受可用性的 Web 开发人员带来过重的负担。因此，研究表明，迭代开发的产品，利用主动的用户设计和反馈机制，明显优于标准产品。一个这样的过程是普适可用性模型。

##### 什么是普适可用性？

正如这个术语本身所暗示的，普适可用性提倡在设计过程的每个阶段应用各种方法来评估设计的可用性，牢记项目的目标和用户的需求。

普适可用性从其他可用性测试方法中脱颖而出，因为它可以在产品的整个生命周期中进行，而不仅仅是在初步开发阶段。这也是独一无二的，因为这种方法涉及的步骤非常简单，对那些进行测试的人来说不那么费力。

##### 普适可用性测试的三个步骤

普适可用性测试的步骤如下。

***第一步。解析***

测试仪分析:

*   用户需求
*   目标可用性需求
*   从用户和商业的角度来看网站的目标
*   网站的现有版本(如果有)，并评估提议的更改
*   竞争者的网站
*   用户访谈和调查

每个网站都必须从设计过程的一开始就解决所有这些问题。进行访谈、实地研究和焦点小组讨论，以收集关于用户和企业利益相关者需求的信息。通过这种方式，分析阶段通常会消除大多数关于站点需要什么结果的疑虑。

相反，从长远来看，对这一阶段收集的信息的曲解会损害你网站的可信度。因此，确保对分析阶段给予应有的关注，以确保在随后的阶段中出现更少的问题。

***第二步。概念化***

*   在抽象层次上概念化网站设计
*   确保在开始设计流程之前，也详细规划了站点的建筑布局
*   进行任务分析，以确定网站的关键功能，并将它们放在网站架构的显著位置
*   创建网站的可视(实体模型)或交互式(原型)表示
*   通过焦点小组、用户测试和认知走查对原型进行评估
*   重复这个过程，直到评估结果和站点的目标重叠

概念化阶段清除了对网站设计和导航的任何疑虑。由于原型和模型是在这个阶段创建和测试的，用户能够向设计师提供关于功能和他对网站的期望的有价值的反馈。然后，经过广泛的头脑风暴，可以根据导航流程编写和重新排列信息。

用户界面的设计和用户界面的技术实现是不同的活动，需要非常不同的技能。可以根据设计者和开发者责任的完成情况来设定优先级。信息架构师在这里的角色非常重要，因为他们在这个阶段决定的导航结构将为设计人员和开发人员的个人任务设定指导方针。在这一阶段，分析性思维和用户同理心在决定网站命运方面起着重要作用。第二阶段的成果将被转化为最终产品:一个可以托管的功能性网站。

***第三步。最终设计、托管和维护***

*   根据评估得出的原型，创建最终设计
*   再次进行可用性测试来评估网站的功能
*   即使在网站上线后，也要根据用户的反馈保留进一步完善网站的空间

除了在这个阶段完成的修饰性变化，设计师和开发者的工作现在被整合到一个单一的产品中，即最终的网站。功能性和美观性现在都到位了，最后的调整是在两个方面进行的。网站的交互性是以这样一种方式设计的，它能留住访问者，并在他们需要的时候为他们提供他们想要的东西——这也可能包括我们希望用户想要的东西！

建立网站并最终托管它并不能完成工作:维护是最重要的。随着互联网的不断变化，网站所有者通常希望用最新的信息来更新网站。这需要你努力输入最新的信息，并且在必要的地方，在几个小时内改变网站的设计和/或功能。所以，从一开始，对于普遍可用性方法来说，重要的是你要建立你的网站，这样它才有成长的空间。

##### 可访问性要素

显然，上面提到的遵循普遍可用性方法的指导方针必须考虑到可访问性问题。例如，使用不被普遍接受的区域标志和符号会降低您的网站在全球受众中的可接受性，就像提供没有文本描述的音频文件会排除听力受损者一样。

一旦你决定增加你的站点的覆盖范围，你的布局应该是这样的，它只需要在这里和那里做一些小的改变来融入新的设计和功能，同时满足世界上不同能力的用户的需求。

##### 普适可用性—变形媒体的解决方案

随着这种变化成为当今互联网最持久的特征，确保你的网站的基本特征不丢失是很重要的。网站设计的改变不应该让访问者感到困惑，或者更糟:让他们寻找另一种选择。

随着不断变化的环境前进，但同时，确保你为访问者创建的基本路径在你所有的网站设计或迭代中保持通用。这是影响网站寿命的最重要的问题之一。

可用性的集成方法通过在设计生命周期的早期发现问题来节省资金，在这一点上可以更便宜地实现更改。任何将这些面向用户的原则放在心上，并实现了普遍可用性原则的组织，都将在创建可用产品的道路上走得很好，并取悦大量满意的客户。归根结底，最重要的是用户！
