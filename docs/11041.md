# 你用哪个浏览器做网页开发？

> 原文:[https://www.sitepoint.com/primary-development-browser/](https://www.sitepoint.com/primary-development-browser/)

在过去的十年里，浏览器的前景发生了巨大的变化。10 年前几乎没有选择——你需要在 IE *(或者网景 4，如果你真的不走运)*中开发和测试你的代码。5 年前，Firefox 成为了开发者的首选武器:它提供了最好的调试工具，并促进了 Web2.0 应用的兴起。

今天，情况有了很大的改善，你们这些幸运的开发者被宠坏了:

*   Firefox 仍然拥有最广泛的工具和 [web 开发插件](https://addons.mozilla.org/firefox/browse/type:1/cat:4)。
*   Safari 和 Chrome 提供了 webkit inspector，如果这还不够的话，[像样的扩展](https://www.sitepoint.com/chrome-web-developer-toolbar/)正在谷歌的浏览器*中出现(请关注 SitePoint 上即将发布的一系列 Chrome 扩展文章)*。
*   Opera 有更好的 JavaScript 控制台之一 DragonFly 和一些 T2 网络开发工具 T3。
*   最后，IE8 提供了强大的开发工具和一些[插件](http://www.ieaddons.com/)，它们可能会帮助你的编码工作。

但是，2010 年使用最广泛的 web 开发浏览器是哪一种呢？我怀疑 Firefox 会保住桂冠，但 Chrome 的人气正在迅速增加。还有人信歌剧。如果你正在开发公司内部网应用程序，使用 IE 有意义吗？
