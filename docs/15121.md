# 采访–YaBB SE 的 Jeff Lewis

> 原文：<https://www.sitepoint.com/interview-jeff-lewis-yabb-se/>

杰夫·路易斯是 YaBB SE 的首席开发人员，之前是最初 YaBB 团队的首席管理员和程序员。

YaBB 是“又一个公告板”的首字母缩写，由一位名叫 Zef Hemel 的程序员于 2000 年 7 月开发，是第一批免费的基于 Perl 的 BB 脚本之一。最初的 YaBB 团队受益于几位经验丰富的程序员的技能，包括克里斯蒂安·兰德和马特·米查姆。Matt 后来继续编写他自己的脚本 Ikonboard，现在领导着 Ibforums 的团队。

当杰夫开始在 YaBB 团队工作时，他后来开始了一个将 YaBB 转换成 PHP 和 MySQL 的项目。该项目被命名为 YaBB SE(“SE”代表“分裂版”)，并于 2001 年秋季向公众发布。该产品是在开源协议下发布的，就像最初的 YaBB 一样。

在帮助我的一个客户定制 YaBBSE 的时候，我碰巧和杰夫谈了一些事情，他告诉我的关于 YaBBSE 目前的进展似乎很有希望。对杰夫的采访包括:

*   YaBB 的背景
*   YaBB 的诞生
*   开发团队和资源
*   他们在开发产品时面临的挑战
*   开发计划和对新语言的支持
*   收购和接管
*   竞争对手的产品和市场方向

##### YaBB SE 简史

***你是怎么进入软件开发的？在开始 YaBB 项目之前，你在哪些领域工作？***

当我开始用 Visual Basic 为我的在线梦幻棒球联盟创建应用程序时，我真正进入了软件开发:开发工具来创建在线草稿、团队管理等等。在那之前，我在我的老 C64 上用 BASIC 编程，在我高中的时候用 Pascal 编程。

因此，在 YaBB 之前，我在业余时间从事我的棒球项目，并且全职为一家制药公司做系统管理员。我编写了登录脚本，允许人们更新球队新闻、交易板块等——这些都是用 Perl 完成的。我也在写自由职业项目(包括为该地区的零售店创建一个应用程序)。

然后，我开始为我的联盟寻找一个新的论坛，并遇到了 YaBB(最初是由 Zef Hemel 发起的)。我参与其中，做了一些工作，并最终在 Zef 决定是时候继续前进的时候接管了他。那时，YaBB 只有 Perl 和平面文件。

但是大多数主机开始不再允许使用使用 Perl/平面文件组合的论坛脚本。与此同时，我真的很喜欢 PHP，所以很自然地去了 YaBB 大学。实际的想法是由一位名叫约瑟夫·冯的程序员提出的。他提到他已经想到了这一点，我说“是的，让我们做吧”。所以一个周末他过来了，我们把大部分 Perl 版本转换成了 PHP。

最初的 YaBB 是互联网上使用最广泛的文字之一，YaBB 社区论坛就是一个证明，拥有任何公告板社区中最大的多语言用户群。当你作为管理员和项目负责人加入 YaBB 时，你给当时的剧本开发带来了什么变化？当时你在推进剧本的过程中遇到的最大挑战是什么？

YaBB 正面临艰难时期。有一些内部争吵和一些争议，以及普遍缺乏方向，这是可以理解的，因为 Zef 试图后退，专注于其他事情。YaBB 也需要一点资金注入来支付托管费用。

我做的第一件事就是为我们准备了一些托管空间。这为我们节省了资金，并有助于缓解团队成员的焦虑。然后我想专注于传播 YaBB 的名字。在这样做的过程中，我们成功地将站点许可卖给了几家大型制药公司，以及几家使用 YaBB 作为其基础系统的董事会主机。

我们还需要吸引更多的开发者，因为一些原始成员已经离开，YaBB 的开发已经开始放缓。最大的挑战是找到一些好的程序员，然后留住他们——我最讨厌的一件事就是当人们加入后就消失了，或者有太多的项目要做而无法真正专注。

因此，一旦一些优秀的编码人员参与进来，我们就开始开发下一组 YaBB 脚本，试图减轻服务器的负载并增加更多有用的功能。

##### YaBB SE Today–团队和资源

***有多少人在做 YaBB SE？他们有什么特色菜？***

第一版 SE 的大部分内容是由两个人写的:我自己和 Joseph Fung。其他长期在 YaBB 工作的人也做出了贡献，比如 Zef Hemel 和 Christian Land。

随着 SE 的继续前进，它主要与约瑟夫和我一起继续，约瑟夫承担了工作的主要部分，直到我们增加了几个其他人。目前，格雷格·罗布森正逐步走上领导岗位，而我和约瑟夫则后退一步。他做了大量的数据库模式工作，以确保 SE 2 的性能达到最佳。

我们还有另外两位来自荷兰的出色的程序员:Francis 专门研究安全，Compuart 是一位出色的程序员。当然，每个人都是志愿者，因为这是一个开源项目。

在开发协议方面，YaBB SE 团队是否与 YaBB 团队平行工作？两个团队是否遵循相同的目标/指导方针？

不，这两个团队不能并行工作。“SE”代表分裂版，我们最初“分裂”的原因有很多。虽然已经就产品的未来发展进行了小组讨论(包括他们的 YaBB 2，这已经进行了一年左右)，但没有任何实质性的进展。

***具体有哪些资源，在电脑、上网、托管、资金、软件开发等方面。YaBB 和 YaBB 的 SE 团队现在都有员工吗？有没有一个办公室里住着不止一个团队的成员，或者所有的事情都在网上完成？***

我真的不能再为 YaBB 说话了，因为当我们分开去做《YaBB 之歌》时，我几乎完全离开了那个项目。由于我们对它的发展方向不满意，我们决定从不同的哲学开始。

大多数人听说过 YaBB 是一个资源猪，它很慢，等等。这些抱怨都是基于 YaBB 的 Perl 版本，我们已经尽了最大的努力去改变这种思维方式(事实上甚至讨论过用不同的名字发布)。所以我不能评论 YaBB 方面的事情。

至于 YaBB SE，大多数事情都是在网上完成的。然而，Joseph 和我住在同一个城市，所以我们偶尔会聚在一起做一些编码，或者见面讨论一些事情，但我们也会在其他项目上合作。

YaBB SE 在一台专用服务器上运行，这台服务器是由我和约瑟夫以及彼得·克劳奇和泰德·苏兹曼共同运营的公司捐赠的。因此，资金是从路易斯媒体(我现在工作的公司)带来的，用于 YaBB SE 项目。现在它自我维持，这一直是我们的目标。

##### 迄今为止的成功…

迄今为止，YaBB SE 得到了新用户和 YaBB 老用户的大力支持。你会给出什么理由来解释《YaBB SE》自推出以来在剧本界获得的成功和支持？这是否源于你决定支持 MySQL——这有助于减少服务器负载(在最初的 YaBB 中这是一个常见的抱怨)?这是团队对产品采取的全新方法吗？

自 2001 年 12 月推出以来，YaBB 证券交易所的表现相当不错。由于 Perl 版本消耗了服务器资源，YaBB 过去(现在仍然)遭受着严重的映像问题。尽管 YaBB 并不是唯一一个由于平面文件结构而出现问题的产品，但它可能是第一个被注意到的资源贪婪者。但最终，即使是“惊人的”Ikonboard 也因为同样的原因被从服务器上撤下。

使用 SE，我们基本上将原始产品转换为 PHP 代码。现在，这个产品是建立在最初的 YaBB 之上的，所以它仍然有一些缺陷，但是由于 PHP 和 MySQL，它有了一个良好的开端。这种最初的速度爆发和服务器资源使用的放松带回了一些用户，但这将是一个漫长的过程。

在每个后续版本中，我们都致力于尽可能地优化。我们已经发布了版本 1、1.1、1.3 和现在的 1.4 —都是从 12 月开始的。在最新版本中，我们进行了更多的优化。

YaBB 公司也采用了新的方法和新的团队。Joseph Fung 是一名出色的程序员，他和我一起创办了 SE。他的编码和解决问题的技能是首屈一指的，如果有什么东西增加了一个查询或几分之一秒的过程，他就在它的顶部-改变它以确保更好的性能。我们决定，SE 需要清理 YaBB 的行为，所以我们专注于创造一个伟大的产品，将很容易在服务器上。我们已经在 SE 2 的优化上有了相当大的进步，并计划几乎重写产品。

当人们还在摆弄 1.1 和 1.3 时，我们已经开始将 SE 2 放在一起，添加了目前没有其他主板使用的高级功能。最重要的是，我们设法减少了 70-80%的查询数量。因此，速度提升和不同的方法极大地帮助了产品，尽管我们没有大肆宣传，但它拥有强大的追随者。

你认为 YaBB 用户正在逐渐转向 YaBB SE 吗？

是的。YaBB 的用户已经为 YaBB 2 等待了很长时间，因为它经历了延迟，重写，团队似乎永远不想发布任何关于它的状态的信息。人们已经不耐烦了，甚至 Perl YaBB 的铁杆粉丝现在也开始迁移过来，可能是因为许多主机不喜欢基于 Perl 的系统。

YaBB 和 YaBB SE 之间的基准测试是否在大型论坛上进行过？

为了回答这个问题，没有对任何 YaBB 软件进行过真正的基准测试。据我们所知，迄今为止唯一的基准测试是由 Ikonboard 完成的。在这些测试中，我们在速度方面表现良好，击败了 vB 和 phpBB，但同时测试表明我们需要解决我们正在使用的查询数量——我们已经开始这样做了(在 SE 2 中已经将数量削减了 70-80%)。

YaBB SE 是否正在任何大型社区中使用？

YaBB SE 已经在一些流量相当大的网站上使用，但是到目前为止，我还没有在任何给定时间在线人数超过 300 人的网站上看到它。不过，一些较大的网站已经选择使用 se 作为他们的沟通方式。一个特殊的曲棍球网站在任何时候都有超过 150-180 人在线。一个使用 YaBB SE 的 Mac 网站徘徊在接近 300。

这些只是少数；但是由于我没有访问很多网站，所以我并不了解所有的网站。

目前，YaBB SE 提供一次性付费的可选“站点许可”，这使得该脚本属于“付费脚本”类别。你能解释一下这个概念最初是如何想到的吗，它是否已经赢得了用户的支持？

站点许可证是在 Perl YaBB 时代形成的，作为公司被允许移除版权的一种方式——因为一些公司不想在站点上显示它。许可证的费用是 250 美元，这笔钱用于托管和其他与 YaBB 相关的费用。对于 SE，我们提供同样的服务，但我们决定采用 150 美元的价格。

自然，这对普通用户来说没有太大的意义，他们只是一个爱好者，并且只是从寻找自由软件开始。然而，一些公司和个人已经利用了他们网站的许可证，包括德国的一家大型制药公司和美国的一家大型计算机公司。

该许可证不是面向普通用户的——他们倾向于提供捐赠，而不是花费一大笔钱来移除版权。任何人都不需要购买这个许可证，除非他们希望从他们的模板中删除版权。

##### 未来

未来有计划对 YaBB SE，甚至是 YaBB SE 脚本本身的支持收费吗？

不，目前没有收费的计划。YaBB 的 Perl 版本对优先支持收费，其他系统也对一些支持收费，但是我们没有计划这样做。我们会一直在论坛上提供免费支持。

至于对脚本本身收费，不:它将永远免费。我们在 GPL 下提供我们的脚本，不像现在的其他系统。

最近，Ibforums 发布了他们的第一个稳定版本，并且有一个 Perl API 系统，允许网站管理员和脚本开发人员实现脚本的某些特性。PhpBB 支持多种数据库和不同语言已经有很长时间了。YaBB SE 是否计划在未来提供对更多数据库和语言的支持，以及类似 API 或您自己的独特功能？

几个月前，我们提供了一个 SSI.php 文件(我看到 Ibforums 现在也有一个)，允许用户在他们的站点上实现部分论坛。例如，他们可以添加登录框，最近的职位等。

SE 自问世以来就获得了全面的语言支持，Perl YaBB 也是如此。据我所知，我们已经支持了 11 种语言。用户可以选择他们的特定语言，因此他们不需要回复管理员设置的任何语言。

至于数据库，我们现在为 SE 2 提供了一些支持多种数据库类型的东西(到目前为止，我们有 MS SQL、mySQL、PostegresSQl 和 ODBC)。

##### 收购、合并等等

有没有人接触过你，想要购买 YaBB SE 的版权，或者你有没有接触过任何其他的脚本产品来购买或与他们合并？

马特去 Ikonboard 的时候，我们从来没有想过这个问题。马特和我讨论了一些其他的事情，但我们从来没有通过最初的讨论——即使这样，我们的谈话也几乎没有任何意义。那是在开始的时候，后来我确实去找过 Matt 问他这件事。当时，YaBB 还没有采取任何行动，还有一些内部的事情正在进行，我只想摆脱这些事情。

Matt 基本上想获得 YaBB 用户，而 YaBB 团队将维护 ikon board 2(ikon board 团队当时正在开发 iB 3)。这是导致交易失败的原因——YaBB 团队不想维护一个旧产品。

如果由我来决定，今天可用的几个论坛包将会合并。这些团队中有如此多的人才，如此多的好想法——结合起来，他们真的可以做出一些伟大的东西。但是，可悲的是，每个人都有自我，他们都想得到荣誉。

已经有两家公司与我们接触，他们想购买 YaBB SE 的版权，但是我们从来没有听取报价。YaBB (Perl)也收到了一些请求，包括两个非常重要的邀请。在这两种情况下，我都签署了保密协议，虽然我仍然受到这些协议的约束，但我会尝试解释这两种情况。

第一家公司已经提供了一些论坛。他们相当认真:交换了数字，讨论了未来的道路，这一次似乎就要发生了。但是，当他们未能理解我们许可证的概念时，谈判破裂了。

第二个是一家想进入编程领域并提供论坛的公司。尽管我们的讨论进行得相当深入，但 YaBB 的内部问题阻碍了这笔交易。

***你在哪里看到 YaBB 的未来 SE？目前剧本有什么重大的改进吗？***

我们认为 YaBB SE 是五大论坛应用之一。目前有三个真正的大公司，YaBB 仍在努力消除它早年受到的一些“负面新闻”。它所遭受的命运是所有其他基于 Perl 的板所遭受的。

因此，SE 将继续努力重建 YaBB。目前正在编写 SE 2，重点强调性能，并且已经显示出重大改进。虽然这是我们对 SE 2 的主要关注点，但我们也增加了更多的功能，其中一些还没有在论坛上看到。

商业脚本用户中有一种观念，认为免费脚本不值得安装，因为它们不能产生有益于软件应用项目工作人员的资产。很多人认为像 VBulletin 或 UBB 这样的商业脚本已经存在了很长时间，并且因为缺乏支持而没有遭受任何重大损失。

虽然大部分免费脚本会逐渐消失，但你会有一个特定的群体会留下来，并设法抓住一个体面的观众。phpBB、YaBB 和 Ikonboard 都是这类产品的例子——而且它们不会很快就消失。同样是那些两年前说这些论坛会消失的人，现在看着论坛一直在激增。

人们，我敢说大多数北美人都属于这一类人，遵循着“一分钱一分货”的哲学。在商业中，我们每天都在和雇主们斗争，看我们工作中的 PHP 和 MySQL。我们的雇主想要支付成千上万的美元来购买使用开源软件和技术可以轻松获得的解决方案。

我总是使用开源程序，对于一些共享软件，我也支持作者自己。这种反馈鼓励开发人员继续前进——这是一种额外的“奖励”,这不同于开发人员自己承认他们喜欢帮助别人，喜欢做他们正在做的事情。我认为这些动机不会改变。

##### 选择您的论坛软件

最近出现了几个新的论坛脚本，这给已经拥挤的市场带来了更多的竞争。对于正在寻找好的论坛脚本的新手和专业站长，你有什么建议？那些想把论坛作为生意来经营的人应该在论坛软件中寻找不同于那些想把论坛作为爱好来经营的人的标准吗？

对于新手和专业人士，我会说:检查几个脚本。弄清楚你的需求是什么，然后从那里开始。对于您的需求来说，有些软件包可能有点臃肿，有些则可能刚刚好。这些脚本中的大部分都很容易安装，而且不管别人怎么说，它们都非常相似。

如果您想定制外观，请确保您选择的软件允许您轻松完成。有些产品要求您编辑一个文件来更改外观；其他人会让你换几十个。

我也建议你不要在一个产品自己的论坛上征求它的意见。我了解到的关于大多数用户的一件事是他们有可怕的偏见。所以你要自己研究你喜欢的产品。

至于商业和业余爱好的使用，免费的脚本和任何商业的一样好，除了 VBulletin，它可能仍然在所有脚本的顶端。除此之外，四处寻找一些有历史的东西，这样你就知道它不会在某一天无声无息地消失。

感谢您的宝贵时间，杰夫！我们期待 YaBB SE 软件的下一个版本。欲了解更多信息，请访问 [YaBB SE](http://www.yabb.info) 网站。
