# 设计师应该具备编码技能吗？

> 原文:[https://www . site point . com/should-designers-have-coding-skills/](https://www.sitepoint.com/should-designers-have-coding-skills/)

这场争论已经在 SitePoint 上激烈进行了一段时间。Elliot Jay Stocks 在一条简单的推特上提出了这个问题:

> 老实说，我很震惊在 2010 年我仍然遇到不会自己设计代码的“网页设计师”。没有借口。

## *对……的争论*

设计师应该具备编码技能有几个原因。

### 中等升值

艺术家利用他们正在使用的媒介，无论是纸张、泥土、布料、木头、大理石还是网络浏览器。一个不了解网页是如何构建的设计师几乎肯定会犯根本性的错误，例如

*   不考虑浏览器视窗尺寸、字体堆栈、页面重量或编码实用性
*   提供基于 Illustrator 的 300 dpi 模型，这是不可能在屏幕上复制的*(为什么那条线不能是三分之一像素宽？)*

### 这不是火箭科学

没有人建议设计师应该能够开发 RESTful 数据库驱动的应用程序。但是一点 HTML 和 CSS 标记知识会大有帮助。了解基础知识、优势和局限性可以帮助设计师创造更好的网页设计。

### 普通通信

当您与开发人员讨论问题时，编码知识会有所帮助。你在说他们的语言，会理解他们遇到的问题。

### 你是更好的资产

你会雇佣谁:不会编程的伟大设计师还是会编程的伟大设计师？

## *反对……的理由*

有人认为设计师不需要编码技能。也许最好的设计师从来不会用腐败的编程弄脏自己的手？

### 伟大的设计师做出伟大的设计

一个有才华的设计师可以创造出有用的设计——不管它最终用在哪里。

### 时间是宝贵的

为什么一个设计师要花宝贵的时间学习编码和跟上 HTML/CSS 的发展？把时间花在提高设计工作的数量和质量上更好吗？

### 你不需要成为一名水管工

设计房子的建筑师不需要是管道、电力或木工方面的专家。这些任务可以由该领域的专家来处理——就像设计师可以将作品交给 HTML/CSS 编码员一样。

### 无知是福

那些不了解网络的人不会被它的局限性所束缚。他们可能创造出一个看似“不可能”的设计，这个设计突破了界限，带来了新的想法。

## 有机会充分发表意见

以我的经验来看，最好的设计师都涉猎过编码。他们可能不理解 HTML 和 CSS 微妙的复杂性，但他们知道如何创建一个包含这种媒体的可用设计。但我的观点可能有所偏颇:他们是最容易共事的人，不需要关于像素、字体和浏览器问题的冗长培训！
