# Zembly:社交应用的社会发展——100 个邀请

> 原文:[https://www . site point . com/zem bly-social-development-for-social-apps-100-invites/](https://www.sitepoint.com/zembly-social-development-for-social-apps-100-invites/)

当我第一次听说 T2 的时候，我有了错误的印象。我认为赞布莱是雅虎！管道或社交应用的 Dapper。我的期望是，Zembly(它仍处于测试阶段，自称是一个“一起创建社交应用的地方”)将允许我——一个非编码者——通过简单的拖放操作快速制作出一个脸书或 OpenSocial 应用。然而，当我收到网站的测试邀请时，我发现我错了。Zembly 不是我所期望的，但它仍然是一个非常酷的服务。我们通过这篇文章底部的小工具为 SitePoint 用户提供了 100 个邀请。

Zembly 不是让创建社交网络应用程序的过程变得更容易，而是让*协作*创建社交网络应用程序的过程变得更容易。Zembly 是一个在线环境，允许你在社交环境中为脸书、OpenSocial、meebo、iPhone、iGoogle 和其他小工具创建应用程序。

虽然有一些自动化(该服务可以在您选择的平台上为您建立一个标准的空白应用程序)，但这不是像 Yahoo！管道。您需要了解一些 CSS、JavaScript 和 XHTML 来创建一个使用 Zembly 做任何有用事情的应用程序。但这是他们的目标受众——已经知道如何开发的开发人员，他们只是在寻找一种共同开发的方式。

显然，这意味着我不是他们的目标受众，所以从开发者的角度来测试 Zembly 有点困难。Zembly 提供了一个适合创建社交应用的开发环境。用户可以在其应用程序上邀请合作者，并创建可以共享、重用和从其他应用程序调用的服务。

![](../Images/f4cd8396a169f62e16aa0620a6098f7d.png "zembly-screen")

Zembly 还致力于成为一个可重复使用的社交代码库。Zembly 上的公共应用程序可以被克隆，允许开发者从已经存在的片段中拼凑出社交应用程序。“Zembly 的全部意义不仅在于重用和组合 Zembly 其他人创建的东西，还在于依赖网络上任何地方的 API 和数据，”该服务的网站说，该网站已经集成了来自亚马逊、Dapper、del.icio.us、脸书、Flickr、谷歌地图、Meebo、Twitter 和雅虎的 API！、YouTube、Zillow 和 Zvents。

由 Sun 支持的 Zembly 声称，在其后端基础设施上创建和托管的应用程序将能够扩展到数百万用户。该网站承诺:“当你在 Zembly 创建一个应用程序时，你的应用程序将透明地跨我们高度可扩展的基础设施栈进行扩展，而不会让你为必须使其全部工作而头疼。”。目前，该网站在测试期间是免费的。
