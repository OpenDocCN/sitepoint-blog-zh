# ColdFusion:物有所值！

> 原文:[https://www.sitepoint.com/coldfusion-worth-the-cost/](https://www.sitepoint.com/coldfusion-worth-the-cost/)

从我上一篇文章的许多[评论来看，ColdFusion 的许可成本对许多人来说是一个主要问题，因为有许多免费和开源的替代方案。这并不是一个新的问题——开发者一直在问“值得花钱吗？”自从 ColdFusion 出现以来。SitePoint 的博客作者 Eric Jones 早在 2004 年 6 月就写了一篇文章，在](https://www.sitepoint.com/coldfusion-myth-busting/)[为 ColdFusion](https://www.sitepoint.com/making-case-coldfusion/) 辩护中谈到了这个问题。快进到 2008 年，似乎人们还在问。

但首先，ColdFusion 的实际成本是多少？

标准版许可证——设计用于在一台服务器上提供多个应用程序或站点——平均价格为 1299 美元。在天平的另一端，企业版许可证——用于多台服务器或插入现有的 J2EE 安装——可以花 7499 美元购买。企业版有一些标准版中没有的额外功能，顾名思义，这些功能是在企业级环境中工作的开发人员最感兴趣的，并且扩展了支持的平台和数据库的范围。

需要注意的一个重要事项是，ColdFusion 的完全免费的开发人员版本一直是可用的。该版本在功能上模仿了企业版，但是限制了对本地主机和两个外部 IP 地址的访问。这使得它非常适合个人开发者和小型团队在本地使用。

对于较小的项目，共享主机是可用的，根据 Ben Forta 网站上的 ColdFusion 主机列表，每月 10 美元起。和所有的主机一样，你花钱买什么，在大多数情况下，更贵的主机会更可靠、更安全、功能更多。

因此，开发是免费的，但除此之外，部署成本也不是微不足道的。然而 ColdFusion 比以往任何时候都更受欢迎。这种语言有一些——实际上是几个“东西”——让开发人员回来。

**1。开发速度**
ColdFusion 真正将“快”放在了快速应用程序开发中。询问任何 CF 开发者，他们很可能会告诉你，他们可以用比其他语言少得多的代码行编写与数据库、LDAP 目录、电子邮件服务器等交互的代码。更少的代码行意味着更快的开发和部署，在一个以小时咨询费为标准的行业，这意味着更低的成本，抵消了服务器本身的成本。

**2。易学**
ColdFusion 也很容易学。它主要是基于标签的——这是许多核心 web 脚本编写人员嘲笑的来源——但这意味着它很容易理解，对于可能已经开始使用 HTML 的开发人员来说学习起来也很快。这也转化为更低的应用程序开发成本，特别是对于那些在需要额外开发帮助时可能首先寻找内部资源的组织。值得注意的是，CFSCRIPT 块允许几乎所有的编码都使用类似 ECMA 的脚本而不是标记来完成，因此更习惯于 PHP、ASP classic、ActionScript 或 JavaScript 的开发人员也可以很快熟悉。

**3。Java 因素**
从版本 6 开始，ColdFusion 就一直基于 Java。事实上，ColdFusion 一度被宣传为构建和部署 J2EE 应用程序的最快方式。拥有现有系统或需要与现有 Java 应用程序交互的组织发现这很有吸引力，主要还是因为开发速度。

所以你有它。我不想说 ColdFusion 是所有工作的最佳解决方案——事实上，当涉及到小型网站时，在我自己的业务中，我经常选择 PHP，因为它无处不在并且托管成本低。但是，每当需要大量的自定义编程或与其他系统集成时，开发的速度和简易性使 ColdFusion 成为我的首选。
