# 来自 Elance Pro 的教训

> 原文:[https://www.sitepoint.com/lessons-from-an-elance-pro/](https://www.sitepoint.com/lessons-from-an-elance-pro/)

我刚刚在 Elance 上完成了一个项目，我不得不说结果对于价格来说是极好的。事实上，Elance 已经被证明是一个非常好的人才来源，尽管我不得不到处花钱买一些无用的东西。

我在 Elance 找到了一个很棒的产品设计师。

现在，这家印度公司在一个网站上做了一件了不起的事情，费用大概是美国开发商的 1/10。

但这不是这篇博客的重点。

他做了每一个 web 开发人员在项目结束时都应该做的两件事，不管是不是在 Elance 上:

1.他感谢了我的客气话，给了我发票，问我是否不介意马上付款。

2.他要求我提供一份对他有利的证明，我照做了。

他还做了一些我喜欢的事情:当我们到达项目的末尾时，他的开发人员表示他们将“马上”修复所有遗留的问题。他们通过即时消息和我一起工作了一个小时来解决这些问题。因此，他们没有花几天的时间来完成最后的细节，而是马上就完成了。

这令人耳目一新，开发人员还指出，如果出现任何问题，我当然可以在接下来的 30 天内回来。

现在他确实做了一件你们大多数人都会觉得讨厌的事情，我也是:在发票到期的最后，他问我想付多少钱(相对于我们最初的协议)。他暗示奖金是合适的。在某些方面，我同意他的观点，因为他超越并提供了巨大的价值。但他真正的奖金将来自我给他的 3 个额外的项目。要求原始协议之外的东西让我心烦。

然而，在认为这种技术令人讨厌之前，请注意，当与客户谈判时，在项目开始时(当它更合适时)使用它可能是一种好技术，尽管它不能在像 elance 这样的投标网站上完成。谈判研究发现，在几乎敲定一份报价后，再要求多一点，效果会很好。例如，汽车经销商(开发商和汽车经销商之间没有任何比较)在收到潜在客户的可接受报价后，会要求多 100 或 200 美元。你也可以这样做。

我希望上面的两个段落不要把这个条目的观点带得离要点太远:

1.立即出示发票，要求立即付款。

2.询问每个项目的证明。

3.快速完成项目。不要让他们逗留。
