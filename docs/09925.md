# 糟糕网页设计的五大烦恼

> 原文:[https://www.sitepoint.com/top-5-peeves-of-bad-web-design/](https://www.sitepoint.com/top-5-peeves-of-bad-web-design/)

我猜这篇文章更长的替代标题是:“或者为什么我最有可能在页面呈现的前几秒离开你的网站…以及如何避免你的用户出现这种行为。”

我已经详尽地介绍了[在网上获取类型](https://www.sitepoint.com/getting-type-to-the-web/ "DesignFestival.com: Getting Type to the Web")、[易读性&可读性](https://www.sitepoint.com/achieving-good-legibility-and-readability-on-the-web/ "DesignFestival.com: Achieving Good Legibility and Readability on the Web")，以及更多小众主题，如[引用样式](https://www.sitepoint.com/typography-of-quotations-citations/ "DesignFestival.com: Typography of Quotations & Citations")，所以在下周继续深入讨论之前，是时候来点轻松的了。

大多数经验丰富的网络用户都有一些讨厌的东西，这可能会让我们离开我们偶然发现的任何网站。虽然我们花越多时间处理一个糟糕的网站(并小声咒骂),越重要和越难找到的信息在其他地方，但每个人都有一个门槛，在那里我们把手举到空中并放弃。

如果你是一名网络老手，你可能知道如何避免或即时改善你所在网页的质量，但大多数用户——包括那些很少使用网络甚至一般情况下很少使用电脑的人，他们可能对过分的设计和可用性有更高的容忍度——会放弃，或者更重要的是*去别的地方*。

这些问题可以大致分为两大类:

1.  剥夺用户的权力
2.  抑制糟糕的设计和排版

让我们直入主题:

## 1.从一开始就是音乐

(又名:*如果我发现你住在哪里…* )

我认为这可能是最糟糕的，并且在使用选项卡式浏览器时会过度恶化。

![A screenshot of two Google Chrome browser windows under Mac OS X with lots of tabs open a tab highlighted as an offending music-playing page.](../Images/8d071a37b7baa65853eb27a044377ecf.png)

你正在浏览，打开许多新的标签页，突然有一个你还没打开的新页面开始发出声音。你立刻非常烦恼地去寻找那个讨厌的标签。当然，由于糟糕的设计，播放器控件要么难以定位，要么完全被忽略，迫使你一起关闭最近打开的标签，因为你不知道哪个是罪魁祸首。

我几乎总是在网上工作的时候听音乐，所以这首歌是我的首选。

除非你想让人们焚烧你的肖像，否则不要自动播放任何东西。

## 2.干扰浏览器行为

使用标签式浏览器时最烦人的是，你打开一个新标签，突然你的窗口晃动了一下，所有的东西都调整了大小。最糟糕的是，当你把窗口大小调整到你喜欢的尺寸后，每次你在网站中加载一个新的页面时，网站或网络应用程序都会一次又一次地这样做。

一个仍然流行的相关抱怨是在新页面中自动打开链接时粗俗地使用`target="_blank"`。正如雅各布·尼尔森在《T2》中指出的:

> “设计者打开新的浏览器窗口的理论是，它能让用户留在他们的网站上。但是，即使不考虑接管用户机器中隐含的用户敌意信息，这种策略也是弄巧成拙的，因为它**禁用了返回按钮**,而这是用户返回之前网站的正常方式。用户通常不会注意到一个新窗口已经打开，特别是如果他们使用的是一个小显示器，窗口被最大化以填满整个屏幕。所以试图返回原点的用户会被变灰的后退按钮弄糊涂。”

(强调我的。)

任何让用户脱离他们自己环境的东西都应该被重新考虑，并带着极大的怀疑；这真的有必要吗？我们会收到一堆要求我们不要这么做的恐吓信吗？这可能会扰乱或迷惑用户吗？我敢打赌，可能的答案是肯定的，如果是，那就不要做。

## 3.“启动”主页

不，只是不。这是当一个网站第一次加载一个带有某种图形(通常是动画)和一些模糊的“继续”或“进入”链接的入口页面时，需要视觉定位并按下以继续任何实际内容。

任何一个这样做的网站都制造了一个额外的步骤，从而为所有通过主页进入的用户设置了一个额外的障碍。

Flash 网站在这方面绝对是臭名昭著的，在大多数情况下，我不会去打扰；再见；`Ctrl` + `W`。

## 4.易读性和可读性差

不像前两个那样令人讨厌，但是被动地令人沮丧，因为我们不能阅读任何东西——至少不能舒服地阅读和阅读。常见示例:

*   字体太小:这可能是所有印刷错误中最常见的——这里的文字太小了，读起来不会有压力。令人欣慰的是，将文本设置为愚蠢的小尺寸的趋势在很大程度上得到了遏制，并且在早期的 Internet Explorer 中变得更加糟糕，因为如果文本是以像素为单位设置的，它就无法调整页面的文本大小(这是 web 上流行的调整大小的方法，现在仍然如此)。
*   几乎没有或者根本没有引导:由于缺乏垂直间距，行与行之间似乎融为一体——难以阅读。
*   **太长或太短的小节:**要么是小节太长，眼睛找不到下一行，要么是只写了几个字就在列中快速曲折地移动。为了克服这一点，用户必须调整浏览器窗口的大小或增大字体大小，而这可能也无法实现，这取决于事情的大小。
*   **对比度差:**这种现象在一些网站上很普遍，在这些网站上，设计师认为亮灰色的文字在纯白的背景上看起来很时尚或者“现代”；甚至“更好”，如果在 10 像素显然。这对于许多用户来说是非常麻烦的，因为通过混合使用`Ctrl`和`+`来增加字体大小并不容易——改变呈现页面的字体颜色需要使用 web 开发插件。

像[可读性](https://www.readability.com/)这样的服务存在是有原因的:有太多的网站的设计者很少或者根本不考虑他们文本的易读性和可读性。尽管我喜欢可读性这样的好服务，但在理想情况下，它提供的易读性和可读性功能应该是冗余的(快速保存和存储待阅读的内容当然非常好)。让我们努力使我们放在网上的文本读起来像最好的期刊和引人入胜的小说一样诱人和舒适。

### 但是字体呢？

你会注意到我完全忽略了字体选择。一个网站可以使用各种各样的字体，每种字体都有好有坏的美学属性，但它仍然可以保持易读性和可读性。以上这些问题更加普遍，需要首先解决——正如杰夫·克罗夫特所说:*“排版不仅仅是选择一种‘酷’的字体。”*

## 5.缺乏明显的等级制度

继续排版——书籍有标题页、索引、章节标题、标题、副标题、带标签的表格和图表、强调、页脚和脚注、词汇表等等。这是一个由良好的排版和立体网格支持的信息层次，非常适用于网页设计，但经常被遗忘。

![An example of relatively plain text to a stytlized typographic hierarchy.](../Images/6c018e3adc6fa11e3a0456d74fbaf7a1.png)

颜色可能在标记上，文本可能清晰可辨，CSS 和标记可能有效，但内容似乎仍然可以转储，页面上各种内容类型之间几乎没有或根本没有划分；区块报价看起来像其相邻的引用段落，标题看起来像加粗或斜体的单行，等等。

好的内容需要一个坚实的、一致的——为了额外的价值——令人愉悦的视觉层次来帮助“可浏览性”。我们大多数人不会在线阅读完整的页面；我们浏览，寻找我们想要的信息的单个部分、段落甚至一行(想想电话号码、日期、电子邮件地址等等)。

## 关闭

当然还有更多令人烦恼的事情(将内容分散在多个“页面”上，除非 JavaScript 允许，否则根本不加载文本内容，完全没有空白，公然违反基本设计惯例，纯 Flash 网站，没有上下文“单击此处”链接，无意义的漫画 Sans 设置等等)，但经过快速思考，以上可能是我最喜欢的五个。

在网页设计中，你最讨厌的五件事是什么？

## JavaScript 补遗

多年来，这些恼人的设计选择困扰着我(除了提供的安全性之外)，这让我不得不屏蔽所有 JavaScript(甚至是来自我访问的根域的第一方脚本)。

我用一个叫做 [NoScript](https://addons.mozilla.org/en-US/firefox/addon/noscript/ "Mozilla.com: Addons page for NoScript (US)") 的 Firefox 插件来做这个。它允许我在浏览时设置白名单，为我提供了一个简单而又经过微调的更安全的网络浏览环境(JavaScript 当然是恶意网站用来跟踪和利用浏览器运行恶意代码的头号途径)。

如果你对插件之类的感兴趣，我已经在[上写了我如何浏览网页](http://klepas.org/how-i-browse-the-web/#notebook)。
