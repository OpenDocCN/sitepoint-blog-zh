# 哪种服务器端语言适合你？

> 原文:[https://www.sitepoint.com/server-side-language-right/](https://www.sitepoint.com/server-side-language-right/)

如今，服务器端 Web 开发解决方案的市场非常拥挤！Perl、ASP、PHP、JSP、Cold Fusion 和许多更专业的选择比比皆是。一个新人应该如何知道该学什么和用什么？选择一个平台来构建您的第一个主要的数据库驱动的 Web 应用程序与选择一个 HTML 编辑器完全不同——您不能只是下载半打候选文件，然后在一个慵懒的下午带着它们进行测试。这些语言中的一些可能需要一个没有经验的 Web 开发人员几个月的工作才能真正理解并舒适地使用。除非你有时间休息一年，蜷缩在一堆书里，否则你将需要一些指导！

在这篇文章中，我将谈到今天网络上的每个主要选项。对于每一个，我将解释主要卖点，不可避免的缺点，相对成本和必要的妥协。我还将对这些技术的目标开发人员进行描述；也就是说，我将很快解释当这些技术背后的天才们第一次创造它们时，他们想到了谁。如果这些简介中的任何一个让你想起你自己的情况，那应该是一个很好的迹象，表明你已经找到了正确的技术。

服务器端编程语言的选择一直是当今互联网上激烈争论的焦点。就像 HTML 编辑器一样，每个人都有自己喜欢的脚本语言，大多数人都有很好的理由做出选择。然而，服务器端开发的本质 Web 浏览器兼容性不是一个问题——允许两种以非常不同的方式解决相同问题的技术取得同等程度的成功！因此，这篇文章不是要决定哪个平台是最好的，而是哪个平台最适合你。嘿:即使你在读完这篇文章后仍然不能做出决定，或者如果事实证明你已经做出了错误的选择，至少你会对你面临的困难有所了解！

##### CGI/Perl

Perl 比万维网本身存在的时间还要长。版本 1 发布于 1987 年 12 月 18 日，比本文中涉及的其他语言出现整整早了*十年*！如果你是这方面的新手，你*可能*想知道服务器端 Web 开发语言在 Web 出现之前是如何存在的。你看，Perl 不是作为一种 Web 开发语言*创造的*。事实上，Perl 是一种多用途脚本语言，旨在处理涉及大量文本操作的任务。Perl 非常擅长将各种来源的文本片段组合起来形成完整的文档，这正是服务器端 Web 开发所涉及的内容。

自创建以来的 15 年中，Perl 一直在通过**模块**进行扩展，这些模块是一些复杂的功能，可以插入到语言中以适应各种特定的目的。其中一个目的是通过大多数 Web 服务器支持的通用网关接口(CGI)创建动态 Web 内容。CGI 是一种标准方法，通过它 Web 服务器软件可以将浏览器对网页的请求传递给任何程序。说实话，如果有必要的话，CGI 程序可以用 C/C++编写(有时已经这样做了)。然而，由于它对文本的方便处理，Perl 总是更适合这项工作，现在 Perl 自带了内置的`CGI.pm`模块，处理 HTML 表单提交等事情的大部分细节都为您处理好了。最后一个好处是，[全面的 Perl 归档网络(CPAN)](http://www.cpan.org/) 的建立是为了提供一个集中的资源，用于插件模块、示例脚本，以及在某些情况下完全开发的 Web 应用程序。

今天，Perl 的版本可以在所有主要的操作系统上免费获得(从主流的 Windows 到最晦涩的 Unix 变种)，如果您想让它做一些复杂的事情，很可能有人已经编写了一个模块来简化您的工作。这种语言唯一的缺点是，因为它不是专门为 Web 开发而构建的，所以它没有针对速度、可伸缩性或在 Web 服务器环境中的易用性进行优化。在 Web 开发领域，有很多更容易学习的语言可以做同样多的事情，但是如果你需要的话，Perl 还可以让你做更多的事情，而不仅仅是 Web 开发。

****优点:****

*   作为一门有着十多年历史的成熟语言，Perl 比其他任何服务器端语言更不容易遇到错误。
*   这是免费的，不过一本向你展示诀窍的好书可能会让你花费一点。
*   大多数好的 Web 主机都支持 Perl，所以你可能不需要为主机支付额外的费用。
*   遍布世界各地的专门 Perl 开发人员的庞大网络随时听候您的调遣。他们不仅会在无数的 Perl 讨论组和邮件列表上回答您的问题，而且 CPAN 档案馆(以及其他机构)会给您提供大量的例子。

****弊端:****

*   尽管有一些选项可以改善这种情况，但是 Perl 在繁忙的服务器上不能很好地伸缩。如果你想创建一个能在短时间内获得大量点击量的 Web 应用程序，你可能会想到别处去看看。
*   众所周知，Perl 有五种不同的方法来做最简单的事情，如果其他人没有使用您所熟悉的方法，那么从他们的工作中学习会很困难。此外，Perl 开发人员因寻找编写任何给定程序的最短方法而臭名昭著，因此您可能会惊讶地发现，一个完整的电子商务购物车脚本是用三行代码编写的，其中大部分是斜线和反斜杠。
*   Perl 针对创建它的 Unix 平台进行了优化，因此如果您希望在 Windows 服务器上工作，您可能有更好的性能选择。
*   Perl 是开源软件，这意味着它没有正式的支持。尽管你可以获得巨大的支持社区，但如果出了问题，你唯一可以依靠的人就是你自己。

****目标开发者:****

今天仍在使用 Perl 的大多数 Web 开发人员往往是 Unix 管理员和用户，他们也将 Perl 用于他们系统的其他领域。对于这些人来说，Perl 是一个令人欣慰的“去任何地方，做任何事情”的工具，可以一直依赖。如果你是一个缺乏经验的开发人员，面临着维护一个最初用 Perl 编写的网站的任务，那么就拿几本[好书](http://www.webmasterbase.com/article/435)准备在你的武器库中添加一个真正强大的工具。

##### 动态服务器页面

在我们继续深入之前，让我们为纯粹主义者们做些准备。微软的动态服务器页面(ASP)是一种语言。更确切地说，ASP 是一个框架，它可以让你将众多脚本语言中的一种(VBScript 和 JavaScript 是最流行的选择)与一组可扩展的软件**组件**结合起来。当您选择的脚本语言用作 ASP 框架的一部分时，这些组件被视为对象。本质上，ASP 的强大之处在于它能够将 VBScript 和 JavaScript 等相对简单的脚本语言与 C/C++开发人员通常使用的强大软件组件结合起来。

好了，这就是微软的销售策略。现在让我们面对事实:对于所有实用的目的，ASP 也可以是一种语言。你只需在编写 VBScript ASP 或 JavaScript ASP 之间做出选择。由于大多数示例代码都是用 VBScript 编写的，所以您的选择几乎是为您量身定做的。因此，ASP 通常被认为是一种使用 VBScript 开发动态网站的语言，在这种情况下，它表现出色。ASP 易于学习，对于大多数主流的服务器端 Web 开发来说足够强大，并且性能方面相当好。

ASP 与 Windows 操作系统(ASP 功能所依赖的大多数软件组件都是特定于 Windows 的)和 Microsoft Internet 信息服务 Web 服务器软件紧密相连。由于这些原因，在除了基于 Windows 的服务器之外的任何服务器上运行基于 ASP 的网站既不实际也不可取。如果您正在建立自己的 Web 服务器，这意味着您必须投资购买一份 Windows 2000 Server(尽管 ASP 的大部分运行在 Windows 2000 Professional 上，但 IIS 在该平台上的可伸缩性较差)。如果你为虚拟主机付费，那就意味着要为基于 Windows 的服务器付费，这通常会更贵。不管你怎么看，ASP 都需要你花钱来运行。有一些版本的 ASP 可以在运行于 Linux 下的 Apache 服务器上运行，但是除非你的具体目标是将现有的 ASP 网站迁移到 Linux 服务器上，否则没有什么意义。

ASP 开发人员关心的另一个问题是 ASP 附带的组件相对有限。如果您的应用程序只需要数据库连接、搜索引擎、电子邮件处理服务、基本的表单处理和会话跟踪，那么 ASP 将非常适合您。但是，任何不常用的功能通常都需要您在服务器上安装额外的软件组件。有许多公司生产用于 ASP 的组件，这些组件生成 PDF 文件，与 FTP 服务器交互，处理二进制文件上传和许多其他事情。不幸的是，这些组件，像 ASP，往往是商业解决方案，会让你花钱。

所有这一切的好处是 ASP 是一项得到很好支持的技术，有微软开发社区的支持。 [MSDN](http://msdn.microsoft.com/) ，微软开发网络为 ASP 开发人员提供了丰富的技术文章和参考资料，微软支持人员将很乐意调查您在开发过程中遇到的任何问题(收费)。如果在任何时候你绝对需要一个问题的答案，你可以放心，那里会有人给你答案。

****优点:****

*   虽然需要编程，但相当容易学会。
*   内置于 Windows 2000 Server 中—没有安装问题。
*   提供专业支持。

****弊端:****

*   这不是免费的(除非你已经在使用 Windows 服务器)
*   专门化的功能可能需要您购买商业组件，或者自己用 C/C++或 Visual Basic 开发它们。
*   如果你习惯了 C 风格的语法(C/C++、Java、Perl、PHP 等共享。)，您可能会发现 VBScript 的特性很难掌握。

****目标开发者:****

如果您曾经使用 VBScript 或 Visual Basic 进行过开发，或者曾经在 Windows 中开发过 COM 对象，那么这绝对是您的 Web 开发平台。此外，ASP 对于非程序员来说是一种编程语言所能达到的最友好的程度了(但是如果你真的讨厌编程，看看 Cold Fusion 吧，接下来)。如果你愿意在你的 Web 服务器平台上使用 Windows，并且能负担得起相关的费用，或者如果你工作的公司只是要求微软能提供的支持水平，那么 ASP 是一项很好的学习技术。

##### 冷聚变

对于以前从未编写过程序的 Web 设计人员来说，选择服务器端 Web 开发平台可能是一个可怕的选择。无论你走哪条路，你都要面对一门庞大、邪恶的编程语言，你的头开始疼了……Cold Fusion 最初是由 [Allaire](http://www.allaire.com/) 在 1995 年创建的，最近被 [Macromedia](http://www.macromedia.com/) 收购，它旨在与众不同。Cold Fusion 不是一种脚本语言，而是给了你一组需要学习的标签。是的，标签——就像那些你已经熟悉并喜爱的 HTML 标签一样！

让我们以在网络上发布存储在数据库中的信息为例。对于绝大多数的服务器端语言，您需要向 Web 页面添加代码以打破 HTML 模式，连接到数据库，从数据库请求您想要显示的条目，并处理数据库发送给您的结果以将其显示为 Web 页面的一部分。在 Cold Fusion 中，这个常见任务由两个标签完成:一个`<CFQUERY>`标签，其属性为查询提供一个名称，并指示从哪个数据库中检索什么信息。第二个标签`<CFOUTPUT>`引用查询，让您使用标准 HTML 来格式化和显示查询结果。

对于有经验的程序员来说，这种基于标记的系统听起来可能有限制和不灵活，但在最新版本中有超过 300 个标记的内置库，并且能够使用更传统的编程语言(如 C/C++和 Java)添加定制标记，Cold Fusion 绝对不是限制性的。Cold Fusion 5.0 包括从数据库数据自动生成图表的标签，在您的网站上无缝地提供搜索工具，甚至生成带有完整 JavaScript 验证代码的 HTML 表单。对于被迫学习服务器端技能的设计者来说，Cold Fusion 带来了乌托邦式的承诺，只需要学习一些新的标签！

像 ASP 一样，Cold Fusion 也是一个商业服务器平台，这意味着无论你是建立自己的服务器还是在虚拟主机提供商那里租用空间，运行它都需要花钱。然而，与 ASP 不同，Cold Fusion 不依赖于 Windows 平台。Cold Fusion 可以像与 Windows 2000 上的 IIS 集成一样，轻松地与运行在 Linux 下的 Apache 集成。

****优点:****

*   极其简单易学——无需编程！
*   功能强大且非常可扩展(最新版本支持基于软件和硬件的服务器集群)。
*   Macromedia 的专业支持。
*   跨平台。

****弊端:****

*   安装自己的服务器成本高昂:Cold Fusion Server Professional 5.0 的成本为 1295 美元
*   程序员可能不喜欢基于标记的开发方法。

****目标开发者:****

熟悉 HTML 但需要在不学习编程语言的情况下增加服务器端技能的 Web 设计人员。从一般开发者的角度来看，学习冷融合就像学习一些新的标签一样。非设计人员也很可能对使用标记的服务器端编程的整洁感兴趣，所以如果您厌倦了调试服务器端脚本，Cold Fusion 可能值得一看！

##### 服务器端编程语言（Professional Hypertext Preprocessor 的缩写）

作为服务器端开发领域的相对新手，PHP 以其熟悉的语法和全面的函数库在程序员中赢得了大量追随者。此外，它相对简单的语法和低，低价格(这是免费的！)已经使它成为微软 ASP 的一个有吸引力的替代品。

PHP 被一些人戏称为服务器端脚本世界的“Perl 杀手”。出于 Web 开发的目的，PHP 可以做 Perl 能做的几乎所有事情，而且通常做得更快更容易。Perl 通常作为 CGI 程序运行，由 Web 服务器调用来处理每个页面请求，与 Perl 不同，PHP 可以与您的 Web 服务器集成，因此它的运行效率要高得多。

PHP 是一种非常强大的语言，它有大量的内置函数可以做任何事情，从跟踪用户会话到动态生成图形，甚至是动态生成 PDF 文件！无需安装模块，无需购买商业插件… PHP 自己处理一切！事实上，PHP 唯一的弱点是，要扩展这种语言来添加内置函数无法处理的非标准功能相对比较困难。这样做需要 C/C++编程知识，并且不适合胆小的人！幸运的是，你必须想出一个*非常*深奥的 Web 应用程序来要求 PHP 中没有内置的功能。

PHP 是免费的、跨平台的开源软件。它集成了所有主要操作系统上的所有主要 Web 服务器。这对于预算有限的开发人员来说是一个巨大的福音，因为他们可以建立一个完整的基于 Linux 的 Web 服务器，并有 PHP 的支持，而不用为软件支付一分钱！当然，这样做的缺点是，对于使用 PHP 可能遇到的任何问题，没有正式的支持。此外，由于 PHP 是一项如此新且正在积极开发的技术，在最新版本中仍会偶尔发现一些错误。但是，不要让这些琐事影响你——PHP 有一个非常活跃的用户社区(其中很大一部分可以在我们自己的 [SitePoint 论坛](http://www.sitepointforums.com/)中找到！)总是渴望帮助你解决任何绊脚石，PHP 的当前版本相当稳定和可靠。

****优点:****

*   相当容易学习，特别是对于有 C/C++、Java 或 Perl 经验的开发人员来说。
*   活跃的用户社区愿意帮助你开始。
*   这种语言包含了所有的东西。
*   免费的！
*   跨平台。

****弊端:****

*   这种语言被设计成对程序员友好的，不幸的是，这使得它对非程序员不太友好。

****目标开发者:****

喜欢简单编程(例如 JavaScript)或者不介意学习如何编程的预算有限的 Web 开发人员。PHP 以相对简单的代码结构为您提供了强大的功能，但可能不适合 PHP 内置特性不支持的非常高端的应用程序。

##### Java 语言(一种计算机语言，尤用于创建网站)

Java 可以说是当今服务器端 Web 开发最强大的平台。从处理网页请求的小型 Java 程序(Servlets)到将 HTML 与定制标记和 Java 代码结合在一起的 Java server Pages(JSP ),甚至到 Enterprise JavaBean s(EJB ),这些软件组件具有在服务器和网络之间无缝传送信息以适应分布式应用程序的复杂能力，Java 确实是一项热门技术。

一个有经验的 Java Web 开发人员每年可以轻松获得超过 100，000 美元的薪水。正如您所料，这是有原因的:Java 不容易学！与 Perl、ASP 和 PHP 等脚本语言不同，Java 是一种完全成熟的编程语言，完全有能力编写大型程序！有一点编程经验的人可以期望在一本好书的帮助下在一两个月内掌握 Java 语言的工作知识，并且在另一个月内能够掌握用 Java 进行简单的服务器端编程的基础知识。现在，如果你对从事服务器端编程感兴趣，那么这可能看起来没什么，但是如果你只是想让一些东西快速工作，像 PHP 这样简单的语言可以让你在一周或不到一周的时间内建立并运行起来！

Java 因其“一次编写，随处运行”( WORA)的理念而被重新命名，事实上，Java 可以在所有主要的 Web 服务器和操作系统上运行。Java Web 应用程序也可以打包成一个标准的 Web 应用程序归档文件(。WAR)文件，这些文件可以安装在任何支持 Java 的 Web 服务器上，不管是什么平台。Java 的制造者计划将这种机制用于分发商业 Web 应用程序，以便在 Java Web 服务器上使用；然而，根据我的经验，Java Web 应用程序往往是复杂问题的定制解决方案，而不是可以在线购买的可销售解决方案。

许多不同的供应商生产插件，这些插件使 Apache 和 IIS 等 Web 服务器能够运行 Java Web 应用程序，虽然大多数插件对于个人使用和开发是免费的，但大多数插件必须付费才能用于商业网站。

****优点:****

*   极其强大和可扩展。
*   跨平台。
*   大多数 Java 服务器插件对于个人和开发目的都是免费的。

****弊端:****

*   学习 Java 需要下很多功夫。如果你不认真学习面向对象编程，就不要开始。
*   大多数 Java 服务器插件如果要用来托管商业网站，就必须付费。

****目标开发者:****

认真的开发人员，他们希望他们的 Web 应用程序具有终极的功能、灵活性和可伸缩性，并且不介意为此付出汗水、眼泪和开发时间。

##### 摘要

在本文中，我们研究了目前可用的所有主要服务器端开发选项的相关特性、优点和缺点:

*   Perl:一种成熟的跨平台语言，可能很难理解
*   ASP:微软强大的基于 Windows 的框架
*   冷融合(Cold Fusion):面向非程序员的强大的、基于标签的选项
*   PHP:一个跨平台、开源的选择，内置了很多特性
*   Java:为严肃的程序员提供终极的能力和灵活性

希望掌握了所有这些选项的内幕，你会感觉更好地准备应对重大决策，即在下一个项目中学习和使用哪些选项。如果您仍然需要一些指导，请务必访问我们的服务器端互联网开发论坛，网址为 [SitePoint Forums](http://www.sitepointforums.com/) 。我们很乐意帮忙！
