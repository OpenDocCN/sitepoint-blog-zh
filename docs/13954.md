# Web Essentials 2005 上的站点点

> 原文：<https://www.sitepoint.com/sitepoint-at-web-essentials-2005/>

当我写这篇文章时，我正赶着为在悉尼举行的 2005 年 Web Essentials 做准备。由于总部位于澳大利亚，SitePoint 很少参加会议，所以参加这次会议尤其令人兴奋。你不能亲自去吗？[跟着播客一起听。](http://we05.com/podcast/)

从我自己参加会议的丰富经验来看(哈哈)，任何像这样的大型聚会都很容易被社会氛围所淹没，并忘记你从整件事中得到了什么(除了一个很棒的包，里面有一本免费的 SitePoint book)。因此，在《科技时报》第 123 期上，我总结了我目前的想法，以及对我计划参加的每次会议的期望。为了你的阅读乐趣，我在这里复制了那段疲惫:

2005 年网络状况

Molly 是我们今天发布的 Dreamweaver 8 书籍的专家评审，鉴于她在这本书上的工作以及她在 [WaSP](http://www.webstandards.org/) 指导委员会的职位，我希望 Molly 会从标准合规的角度来看待网络。

也就是说，现在的网络标准比以往任何时候都要酷。基于 CSS 和文档对象模型(DOM)等标准的 Web 设计技术已经达到了狂热的程度，这些标准提供的稳定基础使开发人员能够推动 Web 不仅仅是格式化超文本的集合，而是交付丰富应用程序和体验的平台(通常称为 Web 2.0)。

有意义的 XHTML 的元素

要么这将是 XHTML 元素的一次运行，以及它们应该如何被用来实现语义标记(例如，块引用应该用于引用，而不是缩进)，要么这将是一次跨所有浏览器真正支持 XHTML 对 Web 开发人员意味着什么的讨论。我希望是后者，但期待是前者。

**超越可用性:设计完整的用户体验**，杰弗里·维恩

随着 AJAX 慢慢接管已知的世界，当涉及到构建 Web 用户界面时，Web 设计人员被宠坏了。过去，衡量一个网站的可用性就足够了，而今天的网站设计者必须考虑如何让网站的使用体验尽可能的有凝聚力和舒适。希望维恩先生能给我们展示一些非常好和非常差的用户体验。

从 Comp 到 Code:把一份清单拉在一起

这个都在标题里了。CSS 大师 Eric Meyer 将带领我们完成新设计的实现过程。我希望如此，看到新的和旧的 CSS 黑客，但可能比最终结果显示的要少。如果没有别的，看看另一个 CSS 专家如何处理实现网站设计的日常事务会很有趣。

**无障碍形式**，罗杰·哈德森

键盘快捷键，以及

<fieldset>、<legend>和<label>标签无疑将成为本次会议的议事日程。但是，当您在 Web 表单中加入了诸如客户端验证、自动完成和动态修改表单字段
之类的先进功能时，如何维护它们的可访问性呢？祈祷吧，我们会得到比“不要那样做”更实际的建议</label></legend></fieldset>

**SVG–力量与激情**，迪安·杰克逊

多年来，可缩放矢量图形(SVG)已经准备好为网络图形带来革命。随着 Firefox 1.5 将这种格式引入主流，是时候看看它能做什么了。我期待的不仅仅是标准的[SVG 示例库](http://www.croczilla.com/svg/samples/)，比如[将 SVG 混合到 XHTML](http://developer.mozilla.org/en/docs/SVG_In_HTML_Introduction) ，但是如果我能看到如何在不牺牲文档语义的情况下实现这种事情，我会非常兴奋。

从战壕中冲出道格拉斯·鲍曼

除了在不牺牲标准的情况下获得一些网页设计的视觉享受之外，我真的不知道还能期待什么——对此我一直很期待。

**网络标准工作流程**，莫莉·霍尔茨施拉格

在 W3C 网站上通过 XHTML 和 CSS 验证器运行你完成的网站远不足以确保符合标准。你把事情拖得越晚，以后解决的时间就越长；许多可用性和可访问性标准甚至不能用验证器来检查。毫无疑问，Molly 将在本次会议中就如何在整个网站开发过程中考虑标准给出大量建议。

**标准快速设计原型**，Eric Meyer

刚刚完成使用 Dreamweaver 8 构建自己的符合标准的网站的工作，我知道 Dreamweaver 现在是一个非常实用的工具，可以快速组装符合标准的网站的原型设计。很好奇这位 CSS 大师会推荐什么工具和方法。

**无障碍设计:超越基础**，德里克·费瑟斯通

大会网站给这次会议的“极客评分”是 5/5，所以我的期望很高。希望 Derek 将超越 W3C 关于可访问性的指南，并提出一些不明显的可访问性问题。如果他能证明我们的网页设计选择如何影响真正的残疾人，我会很高兴地离开。

阿贾克斯，蒂姆·卢卡斯

每个人口中的首字母缩写词，AJAX 可以用作一个简单的可用性调整，或者整个应用程序的使能器。在我看来，我想看看 AJAX 的技术水平，公平地说，它每天都在变化。随着 Web 服务器技术[开始适应](https://www.sitepoint.com/jetty-60-to-provide-new-architecture-for-ajax-apps/)来满足大量使用 AJAX 的需求，我希望这个会议能够关注这些真正革命性的应用程序的架构。

**标题属性——有什么用？绝对没有！**史蒂文·福克纳

从本次会议的标题来看，我觉得标题属性并不完全是为了可访问性的目的。这可能是因为大多数视力受损的人使用的屏幕阅读器不支持这个属性吗？这是我最好的猜测，但我准备完全错了。

**JavaScript 和 DOM** ，卡梅隆·亚当斯

卡梅隆现在正在写我们的续集 *[DHTML 乌托邦](https://www.sitepoint.com/books/dhtml1/)* ，所以当谈到 JavaScript 和 DOM 时，我知道卡梅隆很在行。有传言称，他为那些喜欢看起来像桌面应用程序的网络界面的人设计了一个令人印象深刻的演示。

微格式:进化网络

微格式是标准语言(如 XHTML)中灵活特性的特殊应用，这些语言本身已经成为事实上的标准。微格式的一个简单例子是 [Technorati 标签链接](http://microformats.org/wiki/reltag)，它允许网页被加上主题标签。在这里，对当前微格式的调查将是强制性的，但我很好奇还能收集到什么新的见解。

**设计下一代网络应用**，杰弗里·维恩

毫无疑问，这将是[全天研讨会](http://webessentials.org/veen/)的浓缩版(我不会参加)。我不确定这是否真的很夸张(例如，“下面是如何构建下一个大型 Web 应用程序！”)，或者对 Veen 将在他的第一天会议中提出的主题进行实际探索。期待后者，为前者做好准备。

请在接下来的几天里关注 SitePoint 博客，了解进展如何！
