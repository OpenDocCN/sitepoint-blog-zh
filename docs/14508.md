# AllMusicGuide:网络标准案例研究

> 原文:[https://www . site point . com/all music guide-a-web-standards-case-study/](https://www.sitepoint.com/allmusicguide-a-web-standards-case-study/)

昨天发布了一个新的音乐指南设计，这是音乐界对 T2 互联网电影数据库的回应。自 1995 年以来，AMG 就一直存在，并且已经成长为网络上最大、最有用的音乐资源。这是一个好的内容胜过糟糕的设计的经典例子:你会发现很少有人喜欢旧的设计，但它足以提供对内容的访问，这对大多数人来说是足够好的时期。

我称这个条目为案例研究，因为 AMG 的新设计是一个关于如何*而不是*处理 2004 年重新设计的案例研究。直到几个小时前，当在其他浏览器中查看时，该网站还显示着一条显著的“该网站针对 Windows 的 Internet Explorer 5.5+进行了优化”的消息(他们现在似乎已经删除了它，但我测试的浏览器仍然显示出明显的视觉故障)。对旧网站最大的抱怨之一是大多数链接需要 JavaScript 才能运行——这个问题太大了，以至于我的一个朋友[写了一个书签小程序](http://www.holovaty.com/blog/archive/2004/06/13/2303)来解除对它们的屏蔽！令人惊讶的是，这些链接仍然出现在新网站上。

事实上，最突出的新“特性”是标题中一个华丽的导航小部件，用 Flash 实现。导航链接位于一个漂亮的 3D 树状结构中，当鼠标触摸时会旋转。乍一看，它可能是像音乐等离子体这样超级酷的东西。然后你意识到每次都是相同的六个链接，事实上这正是你在 1998 年左右在一些自命不凡的“酷”网站上看到的导航。有人记得 Boo.com 吗？
