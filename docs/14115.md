# 啊啊……闭嘴，问一个问题！

> 原文：<https://www.sitepoint.com/aaarghshut-up-and-ask-a-question-already/>

所以我今天通过电话和 5 个潜在开发者中的一个交谈。这个家伙一直对我说:“这就是为什么我们是最好的。这就是为什么我们的开发人员是最好的。我们的目标是取悦他人。我们会给你你想要的。我们的价格很好，因为我们在这里外包。我们用 PHP 开发，比。净…”

但是他从来没有花时间问我任何关于我想要达到的目标的问题——除了阅读我提供给他的一些高水平的规格。他确实问了一些问题，关于预算和我对他的长期承诺。但他的问题都是关于他和他的需求，而不是关于我作为一个潜在客户想要实现什么。

然后，当我问他一些问题时——比如他如何估计项目的时间，他可能知道什么样的开源模块，如何处理变更请求——他变得有所防备，好像我在烦他。(而且他是公司的总裁)。

帮你的潜在客户一个忙。不要对他们说教。除了基本的资格确认问题，还要问他们更多的问题。他们如何定义成功？他们的担忧是什么？他们对技术的适应程度如何？他们的目标是什么？他们在开发规范或选择技术平台时需要什么帮助？他们是否考虑过项目成功所需的所有要素？

你们都知道这一点，但我惊讶的是，it 专业人员打电话时忘记是多么常见。

作为一个潜在客户，这种事情让我非常沮丧，但对你来说却是一个很好的机会。但你知道这个。
