# 复习:学习 eZ publish 3

> 原文:[https://www.sitepoint.com/review-learning-ez-publish-3/](https://www.sitepoint.com/review-learning-ez-publish-3/)

如果你曾经探索过用 PHP 构建的开源应用程序的财富，那么你很有可能听说过 [eZ publish](http://ez.no/ez_publish) ，这个 CMS/应用程序框架拥有你正在寻找的所有特性。

eZ publish 甚至多次出现在 SitePoint 上；回到 2002 年 11 月，eZ publish: PHP 的黑仔应用程序(涵盖 2.x 版本)，以及对 eZ systems 的博德·法斯塔德的采访。后来，在 2003 年 6 月，Sebastian van der Vliet 对版本 3.x 进行了[评估。](https://www.sitepoint.com/article/version-2-version-3)

eZ publish 版本于 2003 年 3 月发布，是一次彻底的革新，与以前的版本只有最模糊的相似之处，但它的设计考虑到了增长，并根据开发人员对 2.x 版本的经验而定。一年半后，我们已经到了 3.4 版本，随着功能列表的增长，新的设计开始“安定下来”。与此同时，eZ systems 从[变得越来越强大](http://ez.no/ez_publish/references)，一路上不断壮大[吴镇男·雷森斯](http://www.derickrethans.nl/month-2004-05.php?item=200405240535#200405240535)([XDebug](http://www.xdebug.org)的作者)。

##### 一个第一！

也许这是 eZ Publish 的成熟和规模的标志，它现在是一本书的主题:[学习 eZ publish 3:构建内容管理解决方案](http://www.amazon.com/exec/obidos/tg/detail/-/1904811019/)(本文结尾提供了完整的细节)——这是第一个用 PHP 编写的应用程序(除了[综合 PHP PEAR::DB](http://www.amazon.com/exec/obidos/tg/detail/-/1411603613/) )。

从历史上看，文档一直是 eZ publish 最薄弱的环节，原因和任何软件项目团队都会提到的一样。结果是，那些精通 PHP 的人，如果有时间，就必须搜索源代码。对于其他人来说，这意味着抓耳挠腮，希望[社区](http://ez.no/community)能够伸出援手。

尽管 eZ Publish 的[在线文档](http://ez.no/ez_publish/documentation)自 2.x 版本以来有了显著的改进，这主要归功于社区的努力和贡献，但仍然存在差距——提供的材料介于常见问题和参考手册之间；“大局”缺失。

输入:“学习 eZ publish 3”，由 eZ publish 社区的一些知名成员[撰写，他们熟悉 eZ publish 的内部机制，并且拥有使用 eZ publish 构建和部署站点的第一手经验。](http://ez.no/community/stat)

这本书是围绕技术主题组织的，每个主题都按照读者的学习经验进行了深入的处理。最后两章是案例研究，用来完善前几章的内容，为读者提供一个心理框架，帮助构建一个使用 eZ publish 的站点。

##### 第 1 章:安装 eZ publish

这本书直接探讨了 eZ publish 的安装，大概是为了给读者一个环境，让他们可以在后面的章节中练习。

这里假设了一个基本的知识水平:作者期望你有足够的经验来设置 PHP、Apache 和 MySQL(或者有一个工作环境)，并且你熟悉像 cron 这样的工具(尽管这些知识都不难找到)。

安装描述集中在 eZ publish 附带的基于 Web 的安装程序上，解释了呈现给你的选择的重要性。有了这些信息，再结合一些关于安装 eZ publish 时可能遇到的常见问题的提示，您应该可以在安装过程中安然无恙。

请注意，这个阶段的材料似乎是面向 eZ publish 3.3 的，在 3.4 版本中过程已经发生了变化。否则，在一个关键点上，本章占据了大量的空间去安装程序的屏幕截图，其中一些可以被省略，文本描述是绰绰有余。

##### 第 2 章:使用 eZ publish 进行内容管理

eZ publish 3.x 有一个方面可能很难理解，对初学者来说也很吓人，那就是它处理内容的方式。它使用的结构对内容本身和存储内容的数据库模式都是抽象的。

本章提供了基础知识。它以提问开始，“什么是内容管理？”然后研究 eZ publish 的内容模型中的关键术语和概念:“内容对象”的概念和它们的蓝图“内容类”,以及它们与“节点树”的关系。还介绍了授权、权限和角色，以及工作流、模板和基本的站点管理员职责，目的是让读者有一个概述。

虽然有很多理论需要学习，但本章的结尾将带您构建一个简单的示例站点，并让您实际体验到目前为止所讨论的要点。

##### 第 3 章:用 eZ publish 模板显示内容

在使用 eZ publish 构建网站时，大部分工作都是通过原始(X)HTML 代码来设计和定制用户界面模板。

“Learning eZ publish 3”用了整整一章来探索模板引擎，从有用的概念讨论和有用的框图开始，这些框图根据用于生成页面的模板来表示页面。然后这一章继续定义模板语言:操作符、变量范围、循环等结构，以及总结 eZ publish 向模板引擎公开的“内核”函数。

像模板性能和缓存这样的问题得到了关注，也许最有用的是让读者快速移动，提供了一个“常见模板任务”的集合，说明了如何构建功能，如面包屑导航，并使用模板代码的相关片段处理日期和时间的显示。

##### 第四章:核心一瞥

如果你想为别人而不是你自己建立网站，了解 eZ publish 在模板和管理工具的“幕后”做了什么是理想的，因为它将帮助你评估性能需求和可能需要你的支持或干预的领域。

本章旨在向读者介绍 eZ publish 作为 PHP 应用程序的设计。这是 PHP 源代码片段第一次出现在文本中，同时还有 UML 图来说明类之间的关系以及它们在 eZ publish 中的角色。

假设您了解 PHP，并且讨论集中在一些特定于 eZ publish“内核”(核心 PHP 代码)的主题上，比如如何处理权限，以及 PHP 对象如何存储在数据库中。

虽然这一章确实是理解 eZ publish 核心的一个有用的起点，为您提供了一个开始自己探索源代码的地方，但是我发现这里的讨论有点太具体了，缺乏整体背景或者与其他系统设计的比较。对 eZ publish 的架构进行更高层次的讨论可能会有所帮助，参考相关的设计模式，也许还能洞察为什么要做出特定的设计决策。

也就是说，不一定要完全理解 eZ publish 的内部结构才能用它来构建一个成功的网站，这里提供的信息成功地为下一章奠定了基础…

##### 第 5 章:扩展 eZ publish

eZ publish 的设计考虑到了可扩展性，尽管它已经包含了许多功能和越来越多的[社区贡献](http://ez.no/community/contributions)，但您可能会发现自己需要一些 eZ publish 还没有的东西。

本章提供了用 PHP 开发自己的 eZ publish 扩展所需要知道的基本细节。它从用于组织文件系统中的扩展的结构开始，然后讨论如何在 eZ publish 中处理浏览器重定向和模板处理等常见任务。像 eZ publish 和工作流这样的主题是从 PHP 开发人员的角度来研究的，这一章最后研究了 eZ publish 中的 SOAP 和 RSS 支持。

再一次，这里有很多信息要吸收，很多信息你在开发的时候更有可能参考，而不是试图一次掌握所有的信息。这些片断开始与下一章相吻合。

##### 第 6 章:扩展开发

本章从另一个概念框架开始，帮助您“在心理上构建”构建 eZ publish 扩展的过程。然后，本文将继续讨论构建“WorldPay 扩展”的实际例子，这将允许一家在线商店通过 WorldPay 处理支付。解释很清楚，并引导读者一步一步地完成构建。该示例还有效地扩展了 eZ publish 工作流和触发器的概念。

第二个例子探索了新的 eZ publish“数据类型”的开发(eZ publish 用“email”这样的数据类型扩展了 PHP 中的基本类型，作为一种组织验证等任务的方法)。在带领读者浏览扩展开发的各个阶段时，进一步关注 eZ publish 的持久性 API，作为如何在 eZ publish 中使用数据库的有用说明。

本章的其余部分将讨论扩展与 eZ publish 的整体集成，并研究代码重用的机会和策略，通过构建一个使用 Google 搜索服务的 SOAP 客户端来说明这一点。

本章中提供的例子都是经过精心挑选和解释的，让读者接触到他们在构建扩展时可能需要用到的 API 和 eZ publish 的元素。结合前两章，这一章证明了这本书的价格，填补了在线文档不完整和缺乏细节的空白。

##### 第 7 章:部署 eZ publish

第 1 章集中讨论了如何尽快安装并运行一个有效的 eZ publish 系统，本章讨论了在真实环境中部署 eZ publish 的相关问题。

本章从讨论一般部署“策略”开始，强调了主要考虑事项，从硬件要求到安全问题以及您应该尽力保护的敏感数据。虽然这里的一些评论是有价值的和发人深省的，但其他的评论显然是居高临下的。该文本还包含一些空的“表格”来帮助您定义您的需求，这些表格看起来可疑地像空格填充符。

本章的后半部分通过研究技术问题来弥补第一部分的不足，比如数据库备份的管理和 PHP 加速器，它们已经被证明可以与 eZ publish 一起工作。所提供的信息应该对那些 PHP 新手有所帮助，但是我不能说有什么我个人觉得特别有启发性的东西，而且所提到的主题范围太广了，以至于无法解决任何有用的细节。

总的来说，我不得不说这是书中最薄弱的一章。就我个人而言，我希望看到更多关于 eZ publish 特有的技术问题和“风险”的讨论，特别是关于 eZ publish 的*维护*的讨论，以及开发人员如何处理诸如从数据库导入和导出内容以及克服 MySQL 中的外键“约束”等问题。

##### 第八章:墨尔本理工大学设计中心案例研究

两个案例研究中的第一个研究了皇家墨尔本理工大学设计中心网站的重建。最终结果可以在[http://www.cfd.rmit.edu.au/](http://www.cfd.rmit.edu.au/)看到。

本章首先从 eZ publish 开始。首先，它探讨了项目的设计要求，以及界面设计的各个阶段。与客户一起工作的过程也叙述了，随着成功和奋斗的道路上。

在对项目进行了定性研究后，作者继续关注 eZ publish 的具体实施细节，并给出了真实的结论、经验教训以及对实施过程中达到各种里程碑所需时间的评估。

这一章为我写了这本书。它不仅提供了对 eZ publish 的有价值的见解，而且还提供了一个成功的“诀窍”,来处理客户和使用该技术构建网站的各个阶段。对于刚刚开始使用 eZ publish 的团队来说，应该可以概括这里使用的策略，降低他们自己的失败风险，并避免被 eZ publish 提供的所有功能淹没。

##### 第 9 章:创建符合标准的 eZ 发布站点

本书的最后一章，也是一个案例研究，考察了用 eZ publish 替换现有的伦敦大学学院(UCL)地理信息工程系的静态网站:[http://www.ge.ucl.ac.uk/](http://www.ge.ucl.ac.uk/)。

这项研究的主要重点是使用 eZ publish 模板构建一个符合 XHTML 的用户界面，并使用 CSS 将样式与内容分开。引言提供了对 XHTML 和 CSS 的简短介绍，其中大部分对于经常使用 SitePoint 的人来说并不新鲜，但还是值得一提的

将项目放在上下文中，讨论转移到用户界面和内容结构的需求，并将它们转化为 eZ publish 模板。提供的模板示例应该证明是有用的，作为进一步的片段，读者可以从中获得想法；它们也很有价值，因为它们为 eZ publish 提出了可行的 CSS 策略，包括命名约定和首选标识符。

本章最后谈到了性能问题、缓存方法以及 eZ publish 性能的优化。

虽然这一章不能像第八章中的案例研究那样概括为一个整体策略，但是关注模板设计是有价值的。它提供的见解应该帮助读者更快地掌握模板语言，并理解它如何适应网站的大环境。

##### 谁应该读它？

“学习 eZ publish 3”是面向那些已经决定 eZ publish 本身满足他们的一般要求的技术头脑。它并不试图“推销”eZ publish，而是侧重于你作为开发者、设计者或管理员将面临的技术和战略问题。

根据您正在阅读的章节，需要对 PHP 有所了解，理想情况下，读者应该能够管理基本的平台相关任务，比如使用 UNIX 命令行和 MySQL，以及配置 Apache。与此同时，作者明智地避免重复生产材料，可以发现，深入，在其他地方，这使得文本停留在 eZ publish 的重点。

##### 风格和感觉

尽管在叙述中，这篇文章采用了第一人称复数“我们”，以此来称呼读者，并有一种直截了当的风格，这种风格在交流问题时很有效，尽管有时会有些枯燥。

正如所有由多个作者撰写的技术书籍一样，“Learning eZ publish 3”为读者提供了从不同作者的观点和见解中获益的机会，代价是主题和章节之间的脱节。令人欣慰的是，“学习 eZ publish 3”结构良好，其章节组织，以避免重复，并允许读者从早期的材料进展。

一个小小的实际问题是，这本书没有太多的空白或清晰的视觉结构，这使得在“翻阅”时找到自己的位置有点棘手。当然，白纸是要花钱的，所以好的旧页码也是可以的。

##### 通常

总的来说，我认为这是一本好书。它包含许多有价值的信息，其中大部分要么是在线文档中没有的，要么是太“特别”而无法确保清楚理解。这本书设法凝成一个清晰的结构，与 eZ publish 的工作人员相关。

如果你想最大限度地利用这本书，就要准备好 eZ publish 安装程序，以便在阅读的时候随时进行黑客攻击。

有些地方，比如第 7 章，还有改进的空间，但是希望这本书能随着 eZ publish 本身继续发展。其他一些值得一看的东西(取决于出版成本)将是 eZ publish API 参考，沿着 O'Reilly 坚果壳书籍的路线，以及对 eZ publish 站点维护的更多见解(处理升级、管理数据库等)。).在网站有了一些在线里程后，案例研究的后续工作也将是有价值的。
