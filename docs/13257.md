# ColdFusion 打破神话

> 原文:[https://www.sitepoint.com/coldfusion-myth-busting/](https://www.sitepoint.com/coldfusion-myth-busting/)

我想自我介绍一下:我叫 Kay Smoljak，是 ColdFusion 的开发人员。我将不时地在这里发布与我选择的开发语言相关的主题。

我在 CF 中写代码已经有一段时间了。我早在 1999 年就开始了——第 4 版——当时网景导航器 4 统治着网络，PHP 处于第 3 版，网络泡沫刚刚开始变得有趣。

从那时起，情况确实发生了变化。经过四个版本和用 Java 完全重写的过程，Allaire ColdFusion 变成了 Macromedia ColdFusion，然后在 Adobe 找到了新家。没有改变的是，CF 在一般的 web 开发社区中仍然不知名，许多错误的信息定期在个人博客和大型技术出版物中传播。

所以，我将尽我所能去驱散那些讨厌的神话(第一:不，ColdFusion 没有死！)并重点介绍 ColdFusion 社区中正在发生的一些很酷的事情。当然要涵盖的内容很多:从框架到开源项目，从报告、图像处理和 Ajax 控件等内置特性到第三方工具和 ide。更不用说与 Flex、Flash、Acrobat 和 AIR 等其他 Adobe 产品的集成了。

好像这还不够，尽管第 8 版推出还不到三个月，在几周前的 MAX 北美会议上，Adobe 展示了第 9 版即将推出的内容。对于 ColdFusion 开发人员来说，激动人心的时刻即将到来！

现在，轮到你了:你有没有特别想看的报道，你想看的探索的问题，或者你最喜欢的需要打破的神话？留下你的评论，我会看看我是否能回答，或者更有可能的是，找其他人讨要回应。
