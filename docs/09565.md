# 泽维尔·谢伊访谈

> 原文:[https://www.sitepoint.com/interview-with-xavier-shay/](https://www.sitepoint.com/interview-with-xavier-shay/)

### 介绍你自己/告诉我们关于你的情况

我让电脑做奇妙的事情，摇摆舞蹈，变戏法，冥想，享受锻炼，攀岩，纯素食生活，我还教其他人如何做同样的事情。

自去年以来，我一直在墨尔本的一个新的非营利性媒体网站 [theconversation.edu.au](http://theconversation.edu.au) 上与 Rails 合作，并且即将在三藩市与[squareup.com](http://squareup.com)开始一个新的演出。

### 你从事 Ruby/Ruby on Rails 项目有多久了？

从 2007 年开始，早在 rails 1.x 时代，我就是专业的。Redbubble 是我的第一次演出，从那以后，我参与了许多其他大型项目。

### 你还深入接触过哪些语言？

我以前对 Java 很熟悉，但现在已经相当生疏了。我记得我在自学如何用 Spring 构建网站，花了很多快乐却毫无收获的时间在工厂里埋头苦干，然后尝试 Rails，在几个小时内就找到了工作，然后想“就这些吗？我想我现在需要做一些真正的工作。”

我曾经用 C++写了一个 3D 引擎，非常酷。哦，还有汇编程序中的引导加载程序。最近我一直在进入 Haskell 来提高我的函数式编程技能。

### 你最喜欢 Ruby/Ruby on Rails 的哪一点？

至此，熟稔。我对语言和库了如指掌，所以我使用它的效率比其他任何东西都要高一个数量级。当然，这个生态系统支持快速开发，自从 rails 的 15 分钟博客视频发布以来就一直如此。我可以在一个小时内从一个想法到一个部署的应用程序(在 Heroku 上),这仍然让我感到惊讶。

我喜欢社区对自动化测试和部署等最佳实践的支持，尽管我觉得它倾向于快速开发而不是可持续开发。我觉得还没有好的资源可以用来维护大型 Rails 应用程序，尽管已经有很多了。作为一个相对较新的框架和语言，也许作为一个社区，我们还没有集体经验。

### 当你学习 Ruby 的时候，有没有什么大的领悟可以帮助其他人学习？

太久了我都不记得学过了！

加入你当地的聚会团体，参与到社区中来，即使只是出现在那里和人们一起喝啤酒。这会让你接触到人们在用这种语言做什么，以及他们是如何做的，并且会节省你很多时间去寻找死胡同。

开始为你使用的库做贡献，即使只是反馈、复制 bug 或编写文档。如果你是初学者，你是贡献文档的最佳位置，因为维护者和专家通常已经对他们的代码太熟悉了，以至于不能写出好的说明。这也将迫使你多走一步，弄清楚事情是如何工作的，这对你的 Ruby 教育有好处。
