# 甲骨文公开承诺使用 MySQL

> 原文：<https://www.sitepoint.com/oracle-mysql-commitment/>

自 2009 年 4 月以来，甲骨文收购 Sun 的计划一直被搁置。尽管美国司法部批准了该交易，但欧盟委员会正式反对。本周，MySQL 创始人 Michael Widenius 也紧急呼吁[将 MySQL 从甲骨文](http://monty-says.blogspot.com/2009/12/help-saving-mysql.html)的魔爪中拯救出来。他的理由是:

*   甲骨文没有与欧盟合作，而是联系了数百名客户，要求他们给欧盟写一封“无条件接受”的信。
*   一个强大的 MySQL 好处不多，而一个较弱的 MySQL 对甲骨文来说价值超过 10 亿美元。
*   Oracle 没有承诺将 MySQL 全部置于开源许可之下，保留当前的支持定价结构，定期发布新版本，与社区合作，或添加使其与企业数据库更具竞争力的功能。
*   Oracle 不情愿地更新了 InnoDB 引擎，Sun 最终接手了这个项目。

作为回应，该公司最终发布了一份新闻稿，概述了他们对 MySQL 的意图。甲骨文的五年公开承诺包括:

1.  维护和增强 MySQL 的可插拔存储引擎架构和文档。
2.  改变 Sun 当前的政策，这样第三方存储引擎供应商就不需要在 GPL 下发布代码。
3.  在相同的条款和条件下，向商业许可证持有者提供与 Sun 的协议延期，直至 2014 年。
4.  继续增强 MySQL，并在 GPL 下创建后续版本。企业版和社区版将获得相同的增强功能。
5.  Oracle 支持服务不会成为强制性的。
6.  MySQL 的研发经费将会增加。
7.  在 6 个月内，甲骨文将成立并资助一个客户顾问委员会，其中包括最终用户和客户。它将对 MySQL 的优先级和其他重要问题提供指导和反馈。
8.  在 6 个月内，Oracle 将创建并资助一个存储引擎供应商顾问委员会，就 MySQL 的优先事项提供指导和反馈。
9.  Oracle 将继续维护、更新和提供免费的 MySQL 参考手册。
10.  购买支持服务的客户可以按年或按年续订。

迈克尔·维德纽斯的顾问弗洛里安·穆勒称甲骨文的声明是“表面文章”。然而，欧盟竞争专员尼莉·克罗斯(Neelie Kroes)表示，她“乐观地认为该案会有一个令人满意的结果”，甲骨文的承诺是“正在进行的诉讼中需要考虑的一个重要的新因素”。

欧盟可能会要求进一步的担保，但甲骨文-太阳的交易似乎有更大的成功机会。这可能会在年底前发生。

甲骨文的声明让你放心了吗？你和迈克尔·维德纽斯一样担心吗？
