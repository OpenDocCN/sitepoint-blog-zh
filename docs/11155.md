# 想要更好的浏览器？让他们看起来不错

> 原文：<https://www.sitepoint.com/want-better-browsers-make-them-look-good/>

web 行业支撑着一个有趣的生态系统，它凸显了浏览器制造商和我统称为 web 专业人士(包括 web 开发人员和设计人员)之间的微妙关系。浏览器制造商想为他们的客户制作最好的浏览器，网络专业人士想为他们的客户制作最好的网站。因此，所有 web 用户的最终体验最终是浏览器制造商和 Web 专业人员之间的合作。

这种合作通常并不容易。当浏览器制造商认为他们有优势时，他们试图强迫网络专业人员让网站使用专有功能。当专业人员认为他们有优势时，他们试图迫使浏览器制造商支持更多的网络标准。在近代历史中，我们已经看到跷跷板向两边倾斜。

然而，web 专业人员真正得到他们想要的东西的唯一方法——普遍增加标准支持——是有一个健康的浏览器生态系统。实现这一点的唯一方法是让网站在所有浏览器中尽可能地看起来和运行得更好。

简而言之，如果你想要更好的浏览器，开始使用他们的东西吧！

CSS 布局、Ajax 和现在的 HTML5 都是如此。不断试验，应用渐进增强；给每个浏览器一点点爱，让浏览器在用户眼里好看。

如果浏览器增加了对特定属性的支持，就使用它，即使这意味着只有该浏览器的用户才能看到它。如果一个浏览器有一个增加你的网站可用性的新特性，也使用它。例如，sitepoint.com 使用了 [IE 网页快讯](http://www.microsoft.com/windows/internet-explorer/features/easier.aspx)功能，在我们最近的网站搜索项目中，我们添加了一个 [OpenSearch 描述](http://www.opensearch.org/Specifications/OpenSearch/1.1#OpenSearch_description_document)文件。这使得在支持该功能的浏览器中添加 sitepoint.com 作为搜索提供商变得很容易。

一些专业人士可能会感叹这意味着更多的工作，而其他人可能会对为特定浏览器应用增强功能感到畏缩。但你需要不断挑战极限，因为浏览器生态系统的健康取决于此。
