# 德里克·费瑟斯通:可访问性不仅仅是合规性

> 原文:[https://www.sitepoint.com/interview-derek-featherstone/](https://www.sitepoint.com/interview-derek-featherstone/)

在上周的 *Web Directions South* 会议上，SitePoint 的凯文·杨克有机会与[德里克·费瑟斯通](http://boxofchocolates.ca/)交谈，他是一位著名的 Web 开发者、作家和可访问性倡导者。

Derek 的主题是“超越法规遵从性的可访问性”，在这次采访中，他透露了更多关于他教育其他 web 设计人员和开发人员真实世界的可访问性的动机。你可以[下载这次采访的音频](https://i2.sitepoint.com/audio/2008-09-27_derekfeatherstone.mp3) (MP3，11.1MB)，或者阅读下面的文字记录。

我是来自 sitepoint.com 的凯文·杨克，我正和德里克·费瑟斯通坐在这里参加 2008 年南方网络方向大会，他昨天做了一个关于网络无障碍的精彩演讲。你好德里克。

嘿凯文。

德里克是那种在无障碍环境问题上不只是说说而已的人，他会行动起来——或者参加铁人三项赛，视情况而定。你想谈谈你的那个项目吗？

是啊，当然。我和妻子开始做铁人三项。我们希望有一个网站，我们可以在上面分享我们的旅程，与我们的朋友和家人分享，所以我们决定创建一个博客，不仅分享我们的故事，还分享我们的照片和一些数据，如果你愿意的话，来自我们的培训。

所以我们会记录下我们所有的训练跑步和骑自行车的情况，记录下所有这些，然后上传到网站上，并尝试以一种有点创意和有趣的方式展示出来。我们使用谷歌地图绘制了我们所有的路线，我们有这些数据，它会跟踪我们的位置，它还会显示我们的心率和海拔以及其他各种关于我们训练骑行的精彩数据。

**这是你第二次在 Web Directions South 演讲，上次你在这里的时候，每个人都在谈论填字游戏的例子——你如何以一种非常直观的方式展示一个填字游戏，同时也让盲人用户能够理解并感兴趣。**

没错。

今年，谷歌采用了谷歌地图，让这些用户更容易使用。你是主动去寻找这些问题，还是它们似乎主动找上了你？

他们中的许多人，他们…这种类型的问题只是通过我们的日常生活呈现给我们。我们有需要解决的问题，无论是客户的问题，还是我们看到的特定解决方案的可访问性存在问题的问题，或者像谷歌地图这样的问题。

我们想让它们比默认情况下更容易访问。所以我们看着它，决定，好吧，让我们解决这个问题——不一定为每个人解决它，但至少让它比现在更容易获得。所以，这些机会出现在我们面前有点意外，但通常是为了特殊的需要。

话虽如此，但肯定有这样的时候，我们去寻找一些东西，我们知道 X 有一个问题，让我们去找出解决它的方法。我们会在这方面花些时间。

你有多关注 HTML5 等标准团体中的可访问性争论？

可以说，我在外围跟着他们。我不想卷入其中。我想……老实说，我宁愿把时间花在做其他事情上，而不是在细枝末节上争论不休。在我看来，已经有足够多的人在做这件事了，所以我宁愿把时间花在修理东西和构建酷的东西上，而不是争论一些细枝末节。我不是说这不重要，只是这不是我想花时间的地方。

您是否注意到了善意的开发人员所谈论的可访问性和您在谈论可访问性时所面临的问题之间的区别，这些开发人员可能没有应对这些挑战的个人经验？

我认为我发现的一件事是，有很多人确实是好意，但他们不一定有与真正的残疾人一起工作的经验。所以通常他们的感觉是基于，或者他们的观点是基于他们的假设，而不是他们的所见所闻。我当然不是说我们做的一些事情不是基于假设和观点，但我们确实尽可能多地与真实的人一起做，以便我们有那种观点。

因此，一个常见的假设是，可访问性意味着屏幕阅读器，例如？

是啊，没错。是的，对于使用屏幕阅读器的人来说，可访问性是很重要的，但是也需要考虑其他的技术。我们需要考虑可能使用语音识别软件的人的需求，或者他们可能使用屏幕放大软件，他们的需求可能与使用屏幕阅读器的人略有不同和/或完全不同。

我注意到，语音识别是你今年演讲中的一个新元素，我上次见到你时，你并没有提到它。这是一种正在兴起的东西，还是在过去几年中新采用的，还是你刚刚注意到的东西？

我认为这绝对是一个正在上升的趋势。和我交谈过的人，对重复性压力损伤的讨论越来越多，使用这种技术的人似乎越来越多。这是我特别选择的，因为正如你所说，当我们谈论可访问性时，大多数人会想到屏幕阅读器。

我认为在这种情况下和在这些论坛上强调这一点非常重要，这不仅仅是这些，我们需要展示其中一些技术是如何工作的，以便人们可以带着对使用这种类型软件的人将面临的挑战类型的更多理解和更多知识离开。

是的。语音识别，我认为它作为一种辅助技术特别容易使用，因为它有一些令人惊叹的因素，即使是非残疾用户也可能会玩它，并在日常工作中实际使用它。

绝对的。

谈到富 web 应用程序，我注意到很多为可访问性而写的指南…很多都是基于这样的假设而写的，即 Web 将是一个相互链接的文档、页面的集合。

我们现在看到的是，越来越多的文档和应用程序混合在一起，以及介于两者之间的东西。网络的发明是为了本质上的可访问性，而在某种程度上，我们正在偏离这一点。这些新的挑战是网络发展方向失误的结果吗？

有意思。我认为，可以说，最初作为文档存储库的东西已经发展了。我认为这可能是从未被设想过的事情，因此我认为这是非常困难的，你知道，因为我们现在正在建设的东西正在继续发展我们使用网络的方式和网络工作的方式，很难拥有和应用一套在我们一直在思考的任何东西出现之前就被设想的指导方针。

所以我认为这里有点脱节。我不确定这是不是一个失策。我认为我们可以建立一个完美的可访问的、丰富的网络，但是我们仍然依赖于在我们试图使用和创造丰富的网络体验的技术之前发明的工具和技术。所以我认为这只是有一个滞后，不一定是有失误，可以这么说。

**举个例子，ARIA 是一个正在开发的新标准，旨在为这些丰富的 web 应用程序带来可访问性……你对它带来的巨大变化感到乐观吗？**

我是。我认为在很多情况下，这正是我们所需要的。在某些情况下，我认为我们不一定需要艾瑞亚。所以，我乐观地认为，我们将会实现这一目标；我只是不太乐观，不会很快。标准过程是出了名的缓慢——这并不是说 ARIA 的完整规范将是一个缓慢的过程。有很多人对此非常感兴趣。他们希望这是会发生的事情，所以如果这将以相当快的速度继续下去，我不会感到惊讶。

我认为决定性的一步是该规范从仅仅是一个规范到成为一个在任何地方都被实现和支持的规范需要多长时间。这就是我现在担心的，因为我觉得 ARIA 有很大的前景，人们会对此感到非常兴奋，我们现在就可以开始实施其中的一些。

但是，要过多久我们才会没有这个部分支持、部分不支持的过渡阶段呢？我们需要在一个没有咏叹调的世界里呆多久？这是我们永远都无法忘记的事情吗？我们会不会遇到这样的情况，我们可以假设 ARIA 支持将一直存在，或者我们仍然要在向后兼容性方面向后工作？

在昨天的演讲中，杰夫·克罗夫特谈到了类似的东西。他说的是，在 CSS 中，你应该用绝对像素度量，还是用相对度量来指定字体。他指出，像素测量，当然，你不能在 IE6 中调整它们的大小。

但是目前大多数浏览器——包括 IE——都提供了页面缩放功能，让你不用修改字体大小就可以放大页面。所以他说，他在自己的工作中选择了像素字体，因为像素字体更容易开发，并假设受该问题影响的人会转向支持他们所需页面缩放的新浏览器。

你对此有什么想法？在什么情况下，我们可以说，好吧，我们把它留在后面，由你来使用最新的工具，给你最好的体验？

是的，我认为……现实和——我不想说幻想，因为我不认为这是幻想，但这可能是我们对浏览器前景的理想看法。所以，在我看来，在 IE6 消失之前，我基本上还是要用相对单位。

我不知道那会是什么时候。我知道我用来衡量是否支持某个特定浏览器的一个标准是，微软基本上说他们不再支持该浏览器或任何技术，无论是浏览器、操作系统还是其他任何东西。当他们停止对它的支持时，那就是我也不再支持它的时候。

所以，我…你知道，杰夫的假设是，需要调整文本大小的人会选择更好的工具，对我来说，这仍然是一个跳跃。我愿意相信，真的。不过，我认为，就最终用户教育而言，我们还有很长的路要走，因为人们不…可悲的是，有很多人使用电脑，他们不一定知道他们可以更换浏览器。他们使用他们电脑上的东西，这就是故事的结尾。

现在，话虽如此，我们现在的处境，我的意思是，我有三个孩子；他们在四岁、七岁和九岁时对电脑的了解可能比我十九岁时还多，只是因为我年轻时我们不用电脑。所以，时代确实在变。我们仍然有一群人不是在电脑的环境中长大的，他们不一定会感到舒适。这是我们的问题吗？这个问题真的很好，我也不确定。

但是我认为总的来说，作为开发者，我们越能让我们的问题代替用户的问题，就越好。

我处理事情的方式通常是谨慎为上。我们的很多客户会谈论一些具体的事情，我会和他们说，“你知道，这不一定是一个可访问性的问题，这是别的事情。”他们会回来对我说，“我们知道这不是一个严格意义上的可访问性问题，但我们的目标是考虑到我们所有的用户和他们的需求，无论他们是什么。”

因此，如果我们想在他们的网站上提供一些*浏览功能*，那么他们就会这么做，因为他们的目标不一定是满足可访问性指南，而是确保尽可能多的人能够消费他们的内容。所以当我看着 BrowseAloud 并认为这不一定是我会在我的网站上使用的东西时，他们使用它是因为他们想把他们的信息传递给每个人。所以我认为这是谁的问题，以及谁来解决这个问题之间的相互作用。

今天，如果你有 15 分钟的时间来关注网站的建设，你想教他们什么来提高网站的可访问性？

哦，哇。如果我有 15 分钟……如果我有 15 分钟，我会想办法让每一个 web 开发人员和设计人员通过各种方式体验各种残疾人将会经历的事情，如果可能的话亲自体验。因为当一个残疾人在使用他们时，看到他们并体验我们工作中的缺点的那种经历…那是一种变革的经历，真的是。

所以，这才是我真正想要的。如果我只有 15 分钟的时间，我会带人们经历这些，这样他们就可以体验这些，并从中学习。所以他们学会了——我想让人们知道键盘交互不是为了键盘交互；它的存在是为了让人们可以在应用程序中使用这些网站，我想这就是我所关注的，对人们的影响。

好吧。我是凯文·杨克。这是德里克·费瑟斯通。他在 [boxofchocolates.ca](http://boxofchocolates.ca/) 写博客。非常感谢。

太好了。谢谢你凯文。
