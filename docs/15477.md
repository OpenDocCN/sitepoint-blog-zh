# 数据库您的网站

> 原文:[https://www.sitepoint.com/databasing-site/](https://www.sitepoint.com/databasing-site/)

在过去的三个月里，我越来越频繁地收到来自网站管理员的电子邮件，询问我他们如何为他们不断增长的网站建立数据库。这篇文章不会一步一步地告诉你需要做的所有事情。相反，它将向您介绍在这一努力中可用的技术和选择。

##### 1.)需要什么？

我首先要说的是:这并不容易。除非你有编程和数据库设计的诀窍，并且有很多经验，否则你要么不得不雇佣一个数据库程序员(50 美元/小时)，要么花数百个小时学习如何自己做。在你开始这么大规模的项目之前，问问你自己是否真的有必要。你在网站数据库上投入的时间和/或金钱会在几个月内得到回报吗？你能不能用几个服务器端包含调用来代替？请记住，如果您想要添加诸如实时库存状态、大量产品目录或个性化等功能，您别无选择，只能使用数据库。

##### 2.)我有哪些选择？

你有几个选择:

自己动手
如果你有 PERL 之类语言的编程经验，你应该不会有很多网站数据库化的问题，尽管你可能需要花几十个小时学习使用 mySQL 和设计数据库。

另一方面，如果你很少或没有相关的编程经验，学习如何设计一个数据库，使用 mySQL/mSQL/Access 和用 PHP 这样的语言编程将消耗相当多的时间。

***二世。)自己做一部分，外包一部分***
如果你时间紧迫，并且有一些编程经验，你可能会考虑自己做一些简单的任务，将其余的外包。要知道，好的数据库程序员并不便宜。准备支付 50-250 美元每小时。

***三世。)雇人帮你做一切***
如果你手头有些花不完的钱，就雇一个专业人士帮你做这一切。根据你的需要，你最终会为一个完整的数据库支付 5000 美元以上。

##### 3.)你现有的技术

你需要做的第一件事是决定站点是托管在 NT 还是 Linux/Unix 服务器上。这里就不赘述利弊了。我会这样总结:在 NT 服务器上做数据库比在 Linux 上做数据库编程容易，但这会让你付出更多的代价。NT 也不如 Linux 健壮(请不要发恐吓信。甚至微软也在他们的一些网站上使用 Unix/Linux)。

***i.) Linux/Unix 选项:*
PERL/ePerl**——对于一个较小的站点，您可以编写一个数据库，使用 PERL 脚本来访问驻留在服务器上的文本文件。另一个选项是 ePERL，它允许您在 HTML 文件中嵌入 PERL 命令。ePERL 和 PHP 很像(下图)。

PHP 是一种简单易学的语言，可以嵌入到标准的 HTML 文档中。PHP 代码非常擅长连接到 mySQL 数据库来获取数据，然后可以格式化并显示这些数据。

***二世。)NT 选项:*
Cold Fusion**–Cold Fusion，由 Allaire 开发，是一种强大但相对简单易学的数据库开发语言。您需要将您的站点托管在安装了 Cold Fusion 扩展的服务器上，以便您的 CFML (Cold Fusion 标记语言)代码能够工作。Cold Fusion 服务器扩展也可用于在昂贵的 Sun 服务器上运行的 Solaris 操作系统(一种 Unix 变体)。显然冷融合 4.5 也可以在 Linux 上运行。

**动态服务器页面**–ASP 与 PHP 非常相似。ASP 页面可以使用 VBScript、JavaScript、Microsoft 的 JScript 或任何其他脚本语言编写，前提是安装了该语言的脚本引擎。请确保您的服务器启用了 ASP，也就是 Active Server Pages，并且它们允许您实际运行它们，因为一些虚拟主机公司可能会收取额外费用或限制它们的使用。还要找出你将使用的连接类型(DSN，DSN-less，OleDB)和后端是什么(MS SQL Server，Oracle，Access)。

##### 4.)链接 2.0 可能就是你需要的全部

Links 2.0 是一组高度可定制的 PERL 脚本，允许您维护一个 Yahoo 风格的目录。它允许您通过 web 界面添加/删除/编辑/批准网站，添加/创建类别和子类别，等等。

该脚本非商业使用免费，商业使用 150 美元。该脚本的 SQL 版本也可用于更大的目录，价格为 450 美元。
