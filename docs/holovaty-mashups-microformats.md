# Adrian Holovaty 谈混搭和微格式

> 原文：<https://www.sitepoint.com/holovaty-mashups-microformats/>

**在 [Web Directions South](http://www.webdirections.org/) 大会上，我与 Adrian Holovaty 坐在一起，他是[chicagocrime.org](http://chicagocrime.org/)的创建者，第一个谷歌地图混搭，Django 框架的共同创建者。我渴望得到他对在线隐私概念、网络结构化数据的潜力，当然还有他的最新冒险的独特观点。**

让我先引用你的话。在你的演讲中，你说，“结构就在我们周围；你不需要像我这样的怪胎才能找到它。”你能详细说明一下吗？

基本上我认为，正如我在演讲中所说的，我们开始期待更多的可浏览性。我们开始期望，如果你查看随机事实 X，它在给定网页的侧边栏中，你将能够点击它并发现更多的价值。

你似乎记录了很多关于你自己的信息——你坐过的每一次航班，住过的所有酒店。我们是否应该为了更大的利益记录我们的生活，以便有一天有人可以把它混合起来？

不，不，我绝对不会这么说。我并不是建议我们应该着手构建我们还没有收集到的信息。但我们肯定应该分析我们已经收集的信息，并为其添加结构。因为如果我们没有在这一点上收集信息，很可能是因为一个好的理由而没有收集。就像我的酒店住宿——人们不收集这些信息是有充分理由的，因为这是浪费时间。

但是如果你的组织出版了时事通讯，或者你的医院出版了信息，而你已经出版了，那么你应该找到它的结构，并鼓励浏览。

我猜想，关起门来的数据比公开的多 100 倍。你认为我们会到达这样一个点吗？在这个点上，这些数字是颠倒的，有信息可以被利用和组合。世界是不是更加开放了？

绝对的。要找到这个问题的答案，最好的方法就是看看脸书和 MySpace，以及年轻一代，看看他们在分享一切方面有多开放。

SP:那么，隐私是不是就在门外呢？它还存在吗？

我不认为它已经完全离开，但我认为它的定义已经发生了变化，人们对他们的数据——他们给出的个人数据——的满意度正在上升。他们更愿意说“我可以接受这些照片，我喝醉了，精神失常，被未来的雇主看到。”我认为，沿着这条路走下去，这将成为一个非问题。

我们会看到，也许在 10 年或 20 年后，竞选美国总统的人可能会在 2007 年有一个脸书账户，上面有一些犯罪照片，但我认为，与我们今天可能看到的相反，在那里你会因此受到批评，在未来，每个人都会说，你知道，“我也这样做了。”拥有所有在线信息的永久记录具有非常深远的意义。

SP:那么，给我讲讲每个街区吧。为什么我们需要另一个新闻服务？

我不能说太多，因为我们还没有发射，我们想轰轰烈烈地发射。但本质上，我们试图解决一个真实的问题，一个真实的人都有的问题，这个问题就是，有这么多的信息，“我如何找到只适用于我非常具体的位置的信息？”具体来说，新闻，以及作为新闻的信息。

在这种情况下，我对新闻有一个非常自由的定义——不仅是你可能在报纸上发现的传统类型的东西，也包括新的东西。例如，Flickr photos:如果有人在你家附近贴了一张照片，你可能会想看看，因为你就住在附近，你很好奇。所以我们的目标是回答这个问题:“我怎样才能轻松地发现我周围正在发生的事情？”无论是正在发生的，还是已经发生的。

来自新闻行业，你觉得在一家小公司工作怎么样？

哦，天啊！这太有趣了，我玩得很开心。我们有四个人:我们睡懒觉，我们努力工作，这非常有趣。

SP:我们来谈谈 Django 吧。框架针对的是谁？我需要了解很多 Python 才能使用它吗？

Django 是一个快速 web 应用开发的框架。为了使用 Python，你确实需要了解它。我想你可以使用 Django，从没用过 Python……但是你必须像学习 Django 一样学习它。这就像 Ruby on Rails:为了使用它，你真的必须学习 Ruby。它是那种语言的工具。

你在简历中提到你喜欢逆向工程。
接下来是什么？你打算把你的 iPhone 拆开吗？

我想我的动机是发现人们还没有做过的事情，并去做那些事情。如果事情很简单，或者其他人已经做了，对我来说就没那么有趣了。就下一步而言，我喜欢识别那些很难使用的信息——导航性很差的网站，通常是政府网站——然后将其公开，就像 chicagocrime.org 一样，还有我在 washingtonpost.com 做的一些事情。所以我喜欢这样打开数据。

我也喜欢技术层面的逆向工程。就像当谷歌地图出来的时候，我是黑客团队中的一员，在 API 出来之前，我想出了如何把地图放在其他页面上，所以…我想这有点淘气，但是管它呢？

这是诱惑的一部分吗？

是的，我不得不说它是。

你的第一个混搭，chicagocrime.org，描绘了芝加哥发生的犯罪。然后在你的陈述中，你提到了一些你正在做的关于伊拉克士兵死亡的数据…你正在混合的数据似乎变得越来越暴力。你会故意寻找暴力数据吗？

哈哈！不，不。那纯粹是巧合——我不是一个有暴力倾向的人，也不是一个喜欢电影中的暴力或任何东西的人。那只是巧合。

不久前,“结构化博客”的概念很流行。微格式、结构化博客和其他结构化数据在哪里？

博客有各种各样人们没有利用的伟大结构，比如你从哪里发帖。不过要看博客了。例如，三四年前，我在自己的博客上对报纸网站做了网站评论，在那里我抨击了他们在可访问性和可用性方面做得很差的地方，以及他们的标记是多么糟糕。

因此，我每周会发布几次关于世界各地某个特定新闻网站的条目——它总是有相同的
结构:它总是关于某个特定网站，评级，他们做得好的地方，他们做得差的地方……我认为博客作者往往有那些重复出现的小特征， 给博客添加结构的问题是，它只是一个大盒子——使用任何博客软件，你都有一个标题，然后是一大块，那是你放条目的地方。

我希望看到更多的人利用添加更多的元数据，而不仅仅是一个大 blob。很棒的是，这些工具已经到了人们可以轻松发布博客的地步，比如 Blogspot (Blogger)，任何人都可以发布，而不必知道代码或拥有 FTP 访问权限或其他任何东西。这是一步，但我们需要进入下一步，拥有更多特定内容的发布系统。

你认为 HTML 的作用是进化以包含更多的标签和表达更多的语义吗？

嗯，不，我认为这个结构需要存在于较低的层次。需要在数据库或者某种定制的 XML 语言中，然后 HTML 可以作为交付机制。但是我确实非常喜欢微格式……不过我确实对微格式有所保留。

它们是什么？

好吧，看看一种在网络上广泛使用的格式，比如 hReview 或 hCalendar。如果你想用所有的数据做一些事情，你必须重新实现 Googlebot。如果你想说“哦，我将为事件制作权威网站，我将通过搜集人们的事件微格式数据来做到这一点”，你必须手动去所有这些地方，自己搜集。因此，虽然它是分散的，这很酷，但对于想用它做事情的人来说也是禁止的。

另一方面，如果你有一个孤立的领域，像你只是想刮 upcoming.org 或一个特定的网站，显然这是容易得多。但是如果你想在大范围内做一些事情，你必须成为 Technorati 或者 Google。

非常感谢你的时间，我很感激。好好享受你在澳大利亚的剩余时光吧！
