# 开源资助网站 microPledge 呼吁帮助

> 原文：<https://www.sitepoint.com/open-source-funding-site-micropledge-calls-for-help/>

在开源世界中，“软件奖金”是一个熟悉的概念——为开发人员实现新功能、编写文档或开发新软件筹集资金。自从去年 8 月推出以来，位于新西兰克赖斯特彻奇的 [microPledge](http://www.micropledge.com/) 一直在为开源软件项目提供一个管理和收集软件奖金的免费平台。但是，除非他们能够筹集到足够的资金对他们的支付系统进行重大改革，否则他们可能不得不关闭。

microPledge 从 1200 多名用户的 280 多个项目中获得了约 3 万美元的承诺软件奖金，自推出以来，它一直在利用 PayPal 进行支付处理。创始人 Berwyn Hoyt 告诉我，PayPal 最近联系了他们，让他们知道以信托形式持有资金并在以后分发，正如他们所做的那样，违反了 PayPal 的服务协议条款。霍伊特说，当他们开始这项服务时，他和联合创始人布莱恩·霍伊特无意中忽略了服务条款中的这一部分。具有讽刺意味的是，正是 microPledge 无意中向 PayPal 透露了他们正在做的事情，当时他们向自己的系统报告了被盗信用卡的抵押情况。

因此，PayPal 冻结了 microPledge 的账户——包括每个人的承诺资金——等待官方审查，microPledge 暂时没有支付处理解决方案。霍伊特告诉我，他们目前正在评估替代方案，但事实证明，很难找到一个能让 microPledge 接受资金并将其托管的支付处理器。

转换也不会便宜。当它推出时，人们的设想是 microPledge 将通过从在封闭源代码或专有许可证下开发的项目中抽取一部分奖金来赚钱(该网站对开源项目是免费的)。不幸的是，情况并非如此，该网站从未盈利。保持它的开放已经成为一项爱的工作。

为了资助替代支付解决方案的开发，microPledge 可以说是在“吃自己的狗粮”，并要求用户承诺 4000 美元以帮助保持网站开放。到目前为止，已经筹集了[$ 515](http://micropledge.com/projects/keep-micropledge-open)，在目标实现和新的解决方案实施之前，用户不会被要求付款(毕竟，这是那些捐款*的人能够*实际汇款的唯一方式)。

霍伊特说，如果没有达到 4000 美元的目标，他们可能会被迫关闭网站。在接下来的几周里，microPledge 计划退还每个人的钱，不管他们被冻结的 PayPal 账户中的资金状况如何，即使这可能意味着自掏腰包和亏本经营。

microPledge 对开源社区来说是一项有用的服务，我们不希望看到它倒闭，因为它可以解决支付问题。
