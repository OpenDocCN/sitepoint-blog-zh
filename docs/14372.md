# SOAP 遇到了现实世界

> 原文：<https://www.sitepoint.com/soap-meets-real-world/>

关于 O'Reilly 的有用文章:[不要害怕丢掉 SOAP](http://www.perl.com/pub/a/2004/09/30/drop_the_soap.html) 讲述在使用 Perl CMS 的 [Bricolage](http://bricolage.cc/) 时遇到的经验和问题。也许不需要说，但无论如何都要去做:如果你想要真实，就去和那个 Perl 的家伙谈谈。

我对 SOAP 得出的结论是，它真的不适合互联网，甚至可以说不适合外联网，而是不适合用几代遗留技术在公司内部集成不同的系统。即使这样，您也应该首先考虑 XML-RPC。我认为 SOAP 最大的优点是每个人都同意，所以可以在各种平台上找到实现。关于“委员会设计”，我们或许可以从中吸取教训。
