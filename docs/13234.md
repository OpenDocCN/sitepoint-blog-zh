# 行动中的缺失:Mac 版 Java 6

> 原文:[https://www.sitepoint.com/missing-in-action-java-6-for-mac/](https://www.sitepoint.com/missing-in-action-java-6-for-mac/)

Java 开发者对最近发布的 Mac OS X 豹和苹果公司突然对 Mac 上 Java 6 的未来保持沉默感到愤怒。不仅 Leopard 没有搭载 Java 6，而且苹果已经悄悄地关闭了 Java 6 的开发者预览版，据报道，苹果开发者论坛正在删除帖子，询问原因。

过去几年参加过开发者大会的人都可以证明，作为一个群体，开发者是苹果的最佳客户。要是苹果公司也同样尊重它的开发者就好了。

在 Java 游说团的一篇名为[再见苹果的帖子中。派对结束了，迈克尔·厄本很好地总结了这种情况:](http://www.javalobby.org/java/forums/t102936.html)

> 不仅 Leopard 没有发布 Java 6，而且苹果公司，以典型的方式，显然认为它没有义务通知客户为什么计划改变了，什么时候改变的(或者即使现在改变了？)苹果将会有一个 Java 6 的工作版本。

现在，很明显苹果并没有完全放弃 Java。正如许多开发者指出的那样，苹果确实为 Leopard 版本的 Java 5 做了大量工作。例如，本·加尔布雷斯(Ben Galbraith)最近根据[苹果(Apple)在 Leopard](http://developer.apple.com/releasenotes/Java/JavaLeopardRN/) 上发布的 Java 5 的发布说明，给出了他最喜欢的 Leopard 中的 Java 改进。

但是，尽管 Java 社区持续了两周的狂热，苹果公司还没有关于 Mac OS X 上 Java 6 的状态的消息。

一方面，根据苹果公司在 Mac OS X Tiger 发布几周内发布 Java 5 的历史，有[猜测它的发布即将到来](http://stuffthathappens.com/blog/2007/10/28/os-x-java-definitive-timeline/)。另一方面，苹果为发布 Leopard 对 Java 5 做出的重大改进让我怀疑苹果最近是否在 Java 6 上做了*任何*工作。

如果苹果公司简单地打破其冷漠的沉默，这一切都不会令人沮丧。阻止新产品的发布以最大化其营销影响是一回事，但是让你最重要的盟友——开发者——对 Java 等主要技术的未来一无所知不是做生意的方法。
