# 谷歌和吃豆人过去的幽灵

> 原文：<https://www.sitepoint.com/google-and-the-ghosts-of-pacmans-past/>

这是网络最大的讽刺之一。尽管谷歌拥有网络上最具标志性的页面——谷歌首页——但他们花费了大量的时间和精力给我们提供完全绕过它的方法。我知道我的大多数搜索是直接从 Chrome 地址栏进行的，每天都有更多的搜索来自任意数量的搜索栏、浏览器扩展、手机应用、插件和工具栏。

这让一件事变得非常清楚:在谷歌，是结果页面在买单。

然而，回顾过去 10 多年的[致敬涂鸦](http://www.google.com/logos/)，我认为谷歌的心脏和灵魂仍然住在它的首页。在这段时间里，我们已经看到了从摩尔斯电码到燃烧的人再到乐高玩具的所有东西，昨天，如果你错过了，[~~25~~<ins datetime="2010-05-24T07:33:24+00:00">30</ins>吃豆人](http://www.google.com/pacman/)周年纪念。

只是这一次，他们很可能超越了自己。不满足于简单的视觉致敬，这一次谷歌人把他们的标志编码成一个工作的 Pacman 游戏的迷宫。点击“插入硬币”按钮，你会突然玩到一个 80 年代经典的 chomper。

出于多种原因，这是一项令人印象深刻的努力。

首先，他们设法从普通的 HTML divs、图像精灵和 JavaScript 构建了整个射击比赛。令人印象深刻的是，他们甚至没有求助于任何新奇的画布或 SVG，更不用说 Flash 或其他第三方技术了。

其次，据我所知，这是谷歌第一次将声音纳入致敬，有一个辉煌的声景“gloop-gloops”，“peeps”和凄厉的警笛为我们的小黄人的冒险唱小夜曲。然而，尽管这无疑是一个技术上的胜利，但公平地说，它并不是完全没有用户体验问题。

发布后不久，谷歌、T2、Mozilla、T4 和雅虎的帮助论坛开始收到越来越多的投诉，称他们的电脑发出莫名其妙的警报声。

虽然有些用户在启动浏览器时只是不知不觉地发出了声音，但其他人有更严重的潜在问题。显然，流行的[酷预览](https://addons.mozilla.org/en-US/firefox/addon/2207/) Firefox 扩展中的一个 bug 允许 Pacman soundscape 在他们离开谷歌页面后仍然存在。

现在，我喜欢吃豆人，但这可能会很快过时。

Cooliris 很快修补了漏洞，但这些东西需要一段时间才能进入野外。你可能会说这不是谷歌的错，但当你在网络上尝试声音时，这可能会让你停下来思考一下。

无论如何，如果你错过了吃豆人访问谷歌首页，不用担心。幸运的是，他在 http://www.google.com/pacman 找到了一个永久的家。

嗯..这是我听到的生产力下降的声音吗？
