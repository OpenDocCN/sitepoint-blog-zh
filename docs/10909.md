# PHP 今天十五了！

> 原文：<https://www.sitepoint.com/php-15-birthday/>

PHP 于 1995 年 6 月 8 日由拉斯马斯·勒德尔夫发布。如果你想研究一个来自网络时代的计算人工制品，他在 usenet 上的原创文章仍然可以在网上找到。我们许多人的职业生涯都要归功于这门语言，所以这里有一段 PHP 的简史…PHP 最初代表“个人主页”，Rasmus 在 1994 年开始了这个项目。PHP 是用 C 语言编写的，旨在取代他在主页上使用的几个 Perl 脚本。很少有人会老到记得用 Perl 进行 CGI 编程，但这并不有趣。你不能在 HTML 中嵌入代码，开发又慢又笨拙。Rasmus 添加了他自己的表单解释器和其他 C 库，包括数据库连接引擎。PHP 2.0 诞生于 15 年前的这一天。PHP 在 1998 年 6 月发布 3.0 版本之前有一个适度的追随者。解析器完全由 Andi Gutmans 和 Zeev Suraski 重新编写；他们还把名字改成了递归的“PHP:超文本预处理器”。批评者认为 PHP 3.0 不安全，语法混乱，没有提供标准的编码规范，比如面向对象的编程。今天，有些人会引用同样的论点。然而，尽管 PHP 缺乏优雅，但它使 web 开发变得非常容易。编程新手可以在他们的 HTML 页面中添加代码片段，专家可以使用开源技术开发完整的 web 应用程序，这种技术在 2000 年 5 月 22 日发布的 web hosts.PHP 4.0 中得到了广泛应用。它提供了基本的面向对象，并解决了几个安全问题，比如禁用 register_globals。脚本崩溃了，但是为新平台改编应用程序相对容易。PHP 4.0 取得了巨大的成功，今天你仍然可以在网络主机上找到它。即使平台开发已经停止，WordPress 和 Drupal 等流行系统仍然可以在 PHP 4.0 上运行。最后，我们来看看 2004 年 7 月 13 日发布的 PHP 5.0。这种语言的特点是更加健壮的面向对象编程以及安全性和性能的增强。由于 PHP 4.0 的成功和竞争框架(如 ASP.NET、Ruby 和 Python.PHP)的引入，PHP 4.0 的发展变得更加平稳，但它有不一致和语法混乱的地方，但你很少会遇到一种语言，它可以安装在几乎任何操作系统上，由大多数 web 主机提供，并提供类似水平的生产力和社区帮助。无论您对这种语言的看法如何，在过去的 15 年里，PHP 已经为服务器端编程和 web 应用程序开发提供了坚实的基础。愿它长久持续下去。
