# 八月阅读材料。网络世界

> 原文:[https://www . site point . com/August-reading-materials-from-netland/](https://www.sitepoint.com/august-reading-materials-from-netland/)

经常有人问我怎么知道这么多。我告诉他们我受到了神的启示，但那是谎言。事实是我读了很多书。可能太多了。或许应该多出去看看太阳什么的。但是我很讨厌蓝色的大房间。

我读这么多的部分原因是有很多。网络资源。但是信噪比有时会有点低。无论如何，我想我会分享一些最近的项目以及一些常规来源:

*   我发现了一张非常棒的 ASP.NET 2.0 页面生命周期图。我一直在做大量的自定义控件开发，这张图片是我的救星。在页面的生命周期中有很多事情在进行，这使得判断什么在做什么以及下一步做什么变得非常容易。
*   戴夫·里德给了我们[有史以来最惊人的视图状态解释。](http://weblogs.asp.net/infinitiesloop/archive/2006/08/03/Truly-Understanding-Viewstate.aspx)嗯，可能除了一些尚未发布的微软备忘录。阅读它，学习它，热爱它。记住，视图状态是你的朋友。
*   [Scott Guthrie](http://weblogs.asp.net/ScottGu/) 的[ASP.NET 提示、技巧和亮点页面](http://weblogs.asp.net/scottgu/pages/ASP.NET-2.0-Tips_2C00_-Tricks_2C00_-Recipes-and-Gotchas.aspx)是纯信号。除了其他重要的事情，Scott 碰巧管理着 ASP.NET 团队。所以当他说话时，他说的是事实。立即订阅此订阅源。
*   Scott Mitchell 的《深入 ASP.NET 2.0 网站导航的五部分》是一篇必读的文章，如果你正在构建一个非简单类型的定制网站地图提供商的话。当你在那里的时候，记得订阅 4guysfromrolla 的 RSS。
*   Phil Haacked 有一篇关于如何在 ASP.NET 实现[模型-视图-展示者模式的精彩文章。对于那些喜欢可测试 UI 逻辑的人来说，这是一本非常有趣的读物。他是一个频繁的，非常好的海报。所以今天就订阅这个吧。](http://haacked.com/archive/2006/08/09/ASP.NETSupervisingControllerModelViewPresenterFromSchematicToUnitTestsToCode.aspx)
*   你会问，我在哪里偶然发现了这些东西？为什么，我只是偶然发现了 DotNetKicks.com。就像掘墓一样。NETLand，踢一脚。不用说，如果你真的能戒掉的话，早点回来看看。
*   此外，知道不做什么可能与做什么一样重要，如果不是更重要的话。所以我盯着[每天的 WTF](http://www.thedailywtf.com/) 。好吧，我承认，看到“企业化”的人如何制造东西是很搞笑的。

这就是我最近在读的东西。享受吧。踢我什么的。
