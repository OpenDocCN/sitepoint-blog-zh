# HTML5 公布完成日期

> 原文：<https://www.sitepoint.com/html5-completion-date-announced-by-w3c/>

如果你因为 HTML5 的规范是一个草案文档而一直回避它，你会很高兴听到 W3C 宣布了一个完成日期。将 2014 年 7 月标记为 HTML5 最终成为 W3C 官方推荐的日期。*(注意我指的是 HTML5 是一项真正的技术，而不是无所不包的“HTML5 是 CSS3 是 JavaScript 是 Flash 杀手”的营销废话！)*

然而，一个更重要的日期即将到来。2011 年 5 月 22 日已经被确认为[“最后一次呼叫”](https://www.w3.org/2005/10/Process-20051014/tr.html#last-call)——在此之后，HTML5 规范将不会添加任何新特性。

在接下来的三年中，W3C 将:

1.  查看所有评论。该联盟期待从最后一次呼叫中获得大量反馈，这很可能导致对 HTML5 规范的进一步修订。
2.  创建一个 HTML5 测试套件来帮助供应商满足 W3C 的标准。官方套件也将帮助开发者构建 HTML5 内容和应用。我们将能够依赖独立的实现结果，而不是来自供应商营销部门的夸大的功能旋转。测试套件的早期版本现在已经可以使用了，但是它还在进行中，你不应该依赖它的结果。

## 完成日期重要吗？

从纯开发的角度来看，完成日期没有任何区别。大多数浏览器已经支持一些 HTML5 特性。也不能保证你想要的功能会在 2014 年 7 月后完全实现。开发者总是小心翼翼地选择技术，并在必要时，为不支持它的浏览器使用垫片、替代内容或优雅的降级。无论在规范上贴上什么标签，这种情况都不会改变:[你今天可以使用 HTML5】。](https://www.sitepoint.com/5-reasons-why-you-can-use-html5-today/)

也就是说，完成日期可能会让那些认为 HTML5 是蒸汽软件的人放心。在软件工程术语中，“草案规范”等同于*“没有准备好生产。”*然而，当提到 W3C 文档时,“草案”标签意味着该规范不一定完整，但是每个特性已经在至少两种浏览器中实现，并且可能已经是可行的。这种困惑弥漫于整个行业，也是 [WHATWG 放弃 HTML 版本号](https://www.sitepoint.com/html5-is-dead-long-live-html/)的原因之一。

## HTML 的未来

HTML 是一个不断发展的标准。W3C 成员已经在考虑对工作名称为“HTML.next”的未来版本进行改进，所以不要忽视未来三年的 HTML5 到 2014 年，该行业将进入下一个迭代！
