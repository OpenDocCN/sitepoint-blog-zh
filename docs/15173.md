# 不要禁用右键！

> 原文:[https://www.sitepoint.com/dont-disable-right-click/](https://www.sitepoint.com/dont-disable-right-click/)

在努力完成一个设计、图像或文章后，你可能想使用 JavaScript 来保护它，它可以禁用右键单击，同时警告访问者该内容是受版权保护的。

这听起来不错，但这通常不是一个好主意。为什么不呢？

##### 烦死了！

直到你不能使用它的时候，你才意识到你用了多少右键！它打开的短菜单中有很多有用的功能。虽然大多数也可以在主浏览器菜单中找到，但使用右键菜单通常会快得多——这通常是你能找到的最近的菜单。

##### 毫无意义！

如果您认为禁用右键单击可以保护您的源代码或图像，请三思！任何决定复制您的内容或代码的人都会这样做，不管他或她是否有能力打开浏览器上下文菜单。如果他们想要你的源代码，那么只需从主菜单中选择“查看源代码”即可。可以突出显示和复制文章文本，可以从缓存中检索图像和媒体演示，还可以录制流媒体。

禁用右键只会让人们更坚定地了解你到底在隐藏什么。这可能会适得其反，因为你的图片和源代码会吸引不必要的注意力。不仅如此，您还可以在启用了 JavaScript 的浏览器上禁用右键单击:访问者只需关闭浏览器选项中的 JavaScript，就可以完全忽略该脚本！

##### 它是残废的！

实现更快导航的鼠标手势开始成为浏览器的一个特性:Opera 有，Mozilla 刚刚增加了支持，Internet Explorer 采用它们只是时间问题。禁用在这些浏览器中查看的页面上的右键单击也禁用了使用鼠标手势的能力，所以你不会受到那些经常使用它们的人的欢迎！

即使你不使用鼠标手势，它们的缺失仍然会严重影响你浏览互联网的能力。我倾向于在右键菜单的新窗口中打开我喜欢的页面的任何链接，这样我就可以阅读和比较两个页面并返回，而不必使用后退按钮。虽然您可以在新窗口中通过按住 shift 键单击链接来打开链接，但许多人发现使用右键菜单中的选项更容易。禁用右键单击，你会很快疏远这些用户。

##### 太不专业了！

问问你自己:你会从一个每次使用右键时都会提醒你它的图片受版权保护的网站上买东西吗(即使你的指针不在图片附近)？我想不是！禁用右键意味着用户缺乏专业素养。

##### 太侮辱人了！

你的大多数访问者会来到你的网站，希望购买一些东西，或者寻找信息。只有一小部分的访问者会登陆你的网站，并试图窃取它。通过试图保护自己不受少数人的伤害，你实际上侮辱了大多数人，他们会出于合理的原因使用右键菜单。当你给访问者“有用的提醒”暗示你不能信任他们时，你真的指望他们会信任你吗？

当然，有些人会想看你的源代码——但是不要认为每个看了它的人都会想在他们的网站上使用它！精明的访问者知道一些不道德的网站所有者使用的欺骗技术，他们可能只是想检查链接是否会将他们带到他们认为会带他们去的地方。那他们是小偷吗？

##### 内容保护解决方案

对于保护您的图像或源代码的问题，还有其他争议较少的替代解决方案。

***保护您的图像***

**1。仅在图像上禁用右键**

如果你真的必须禁用右键单击，那么限制恶化，并使用一个只在图像上禁用它的脚本(一些脚本也阻止 IE6 图像工具栏出现)。当然，有时人们会想使用右键点击，将鼠标放在图像上，所以我建议你稍微改变一下信息，以反映只有图像会受到影响的事实。

**2。标记图像**

另一种识别图像是你自己的方法是以某种方式标记它们。阻止小偷的最好方法是在图像中最有趣的部分做一个视觉标记。诸如您的徽标、您的网站地址，甚至是纯色或半透明的“样本”字样等标志，都清楚地表明该图像不是用于一般分发的。这种涂鸦风格的策略在提供艺术服务的网站上效果很好，也有助于阻止热链接。

**3。数字水印**

数字水印是在文件中嵌入可识别信息的实践。通常，这是唯一的 ID 代码或图像创建者的联系方式。这种嵌入的信息对于普通的浏览者来说是不可见的，但是可以借助图形程序插件或专业的独立解码器来查看。数字水印通常比物理标记更强大，因为当您查看图像时，它是否具有附加信息并不明显。话虽如此，这种方法并不能阻止小偷，尽管从理论上讲，这种方法可以让追踪你的作品(在某些情况下还有衍生作品)变得更加容易。

**4。窗户**

如果你想在高质量的图像中显示细节层次，而又不“免费”透露太多，那么你可以使用我称之为“窗口”的方法。想象你通过一张纸上一平方英寸的洞看到完整的图像；您可能看不到最终图像的大部分内容，但是您可以看到图像的质量和细节，而不必显示完整的图像。我在一些图片库看到过用 Java 实现的这种方法(允许你移动可视窗口)，尽管使用基于原始图像的裁剪图像可能更简单(对你的访问者来说也更快)。

***保护你的 HTML***

使用脚本来禁用右键单击并不是使源代码难以阅读的最有效的方法。大多数用于保护 HTML 源代码的技术都使用 JavaScript，所以如果访问者碰巧关掉了 JavaScript，网站就会变得无法使用。

有三种主要的 JavaScript 技术用于保护页面的源代码:

*   对源代码进行编码，使其不经解码就不可读
*   将大部分源代码放在 JavaScript include 中
*   在新窗口中打开页面，禁用所有菜单(不推荐)

让好奇的人更难破译你的 HTML 源代码而不必求助于 JavaScript 的唯一方法是删除不必要的空白。这样做的效果是将您的代码变成一行或多行代码，这样任何在浏览器中使用查看源代码功能的人都会发现，如果不花时间整理一下，很难阅读。

##### 你会保护什么？

显然，您选择使用的替代方案将取决于您需要保护什么以及它的保护有多重要。你应该记住，任何决心偷你的东西的人都会找到方法来偷——即使这意味着要截屏或打出一篇你禁用了文本突出显示的文章！
