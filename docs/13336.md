# OSCON 2007:风车:Ajax Web 应用程序的自动化测试

> 原文:[https://www . site point . com/oscon-2007-windmill-automated-testing-of-your-Ajax-web-applications/](https://www.sitepoint.com/oscon-2007-windmill-automated-testing-of-your-ajax-web-applications/)

Mikeal Rogers 是开源应用基金会的 QA 开发工程师，在 semanticmikeal.com T2 有一个博客

Adam Christian 也是开源应用基金会的 QA 开发工程师。他的博客在 t0asted.com 的

(声明:我在 OSAF 工作，是风车项目的负责人。)这个演讲从亚当展示使用[风车](http://windmill.osafoundation.org/)测试 OSAF[钱德勒服务器](http://chandlerproject.org/)的 Ajax Web UI 开始，包括日历视图的拖放测试。

会谈继续进行，Mikeal 描述了为什么要构建一个全新的框架，而不是使用 Selenium 这样的现有工具——因为无法调试被破坏的测试而感到沮丧。Windmill 被设计成允许测试被暂停和交互调试。

他们随后简要概述了 Windmill 的 Python/JavaScript 架构，并以跨浏览器记录器的演示结束，该记录器可用于在不编写代码的情况下从 Windmill IDE 创建测试。

Mikeal 和 Adam 在网上[这里](http://windmill.osafoundation.org/trac/attachment/wiki/Miscellaneous/OSCON2007-Windmill.pdf)有他们的幻灯片。

他们有一些截图和截屏可以在网上找到。
