# Mozilla 视觉的东西

> 原文:[https://www.sitepoint.com/mozilla-vision-thing/](https://www.sitepoint.com/mozilla-vision-thing/)

Via [Mozillazine](http://www.mozillazine.org/talkback.html?article=4584) ，一篇由 Javascript 之父兼 Mozilla 首席架构师 Brendan Eich 撰写的[精彩帖子](http://groups.google.com/groups?as_umsgid=40724396.1000303@meer.net)。这是个有趣的东西。Brendan 基本上展示了他对未来五年控制未来应用程序开发和部署平台的“战斗”的看法，主要双方是微软+ Longhorn + XAML 对“开源”+ Mozilla (Gecko) + XUL。

随着 XUL(见[介绍 XUL](https://www.sitepoint.com/article/xul-nets-biggest-secret-1) )现在有了“工作”的东西，Mozilla 在这个领域绝对领先。尽管如此，仍有一些障碍需要克服。

作为一个在网络上部署的平台，XUL 仍然受到实际问题的困扰——安全问题和易用性之间的妥协尚未达成，这阻止了，比如说，PHP 编码人员像我们生产 HTML 那样生产基于 XUL 的应用程序。尼尔·迪肯在这里很好地总结了这些问题。

从战略角度来看，Mozilla 也处于商业软件和开源软件之间的灰色地带，这要归功于他们的 Netscape / AOL 过去。这意味着一方面他们没有钱强迫每个人接受他们的技术，另一方面，他们在开源圈子里缺少强大的追随者，与 Linux 和 PHP 这样的技术相比，他们不算是一场“运动”。吉多·范·罗苏姆(Python 之父)今年在 OSCON 很好地阐述了这一点(但我已经失去了链接)。不过，这种情况可能会改变。来自微软“竞争对手”的支持，如 IBM、Novell 或 Sun 可能有助于推广 Gecko 并扩大其安装基础。与此同时，自去年 7 月从 Netscape / AOL 分离以来，独立的 Mozilla 基金会似乎在开源方面获得了更多的认可，Brendan 说他希望 Mozilla 与其他组织如 Gnome T1 合作。

同时，微软拥有微软一直拥有的所有优势。XAML 推广已经开始，但仍处于“试验”状态。当 XAML 最终上市时，它很可能是一项全面的技术，有 Visual Studio 的支持，以及无休止的广告和专栏文章来推销它，正如我们已经看到的. NET。但它可能不会一帆风顺。

根据微软过去的表现，我们所说的 Longhorn 至少还需要两年时间，甚至可能更长，甚至可能是 2008 年。微软已经发现更难说服客户从旧版本的 Windows(和 Office)升级的价值，如果 Longhorn 不仅仅意味着许可费，还意味着购买最新的硬件来运行它，即使它发布了，吸收也可能很慢。这意味着 XAMLs 的“安装基础”可能需要很长时间才能增长。

与此同时，多亏了 IBM，Linux 在主流 IT 领域获得了更多的关注，这可能会导致客户对跨平台技术的需求(XUL 就是这种情况)。XUL 现在“工作”了。如果它能在 XAML 成为现实之前加快步伐，它将面临激烈的竞争。

反正够了[占卜](http://www.hyperdictionary.com/dictionary/soothsaying)。已经开始了。
