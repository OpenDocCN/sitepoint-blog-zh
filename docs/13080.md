# 微软和 IE8:让我们继续前进，让它变得更好

> 原文:[https://www . site point . com/Microsoft-ie8-let-move-on-and-make-it-great/](https://www.sitepoint.com/microsoft-ie8-lets-move-on-and-make-it-great/)

在 SitePoint 上写任何涉及微软或 Internet Explorer 的东西都会引起一些麻烦。有很好的理由——即使是我，微软 Shill，也不能说微软自 2000 年以来在浏览器创新、安全，尤其是网络标准支持方面有糟糕的记录。我可以理解为什么很多 SitePoint 的读者对 IE 和微软很反感，不愿意，可以说，信任路西法。

但这将是他们的损失——巨大的变化正在发生。这不是你哥哥的 IE，也不是你哥哥的微软。微软自己也意识到，在这个新的互联时代，任何不支持开放标准的人都注定会变得无关紧要，因为生态系统围绕着可通信、可操纵、可调整和可攻击的应用程序发展。许可独家桌面软件似乎是一条死路。鲍尔默和奥齐在本周的 MIX08 上明确表示了这一点。

我知道我们 Web 开发社区中的许多人都不愿意相信 Lucifer 的代表。但是有很多其他证据表明，一个开放的时代正在向我们走来。谈话是廉价的，但人员流动是永恒的。嗯，也许不会永远，但至少这是一个需要花费几万到几十万来解决的问题。微软的开放标准和开放倡导者似乎正在获胜。斯科特·格思里，[可能是有史以来最开放、最平易近人的项目经理，他被提升为公司副总裁](http://www.microsoft.com/Presspass/press/2008/feb08/2008ExpandedLeadershipPR.mspx)并负责整个项目，我认为这并非偶然。NET 平台。在较低层次上，微软雇佣了一些最公开、最直言不讳的人。净开源乡亲左右，即[菲尔哈克](http://www.haacked.com/) ( [潜台词](http://www.subtextproject.com/)的仁慈的独裁者】，[斯科特汉塞尔曼](http://www.hanselman.com/blog/)(博客/播客杰出者)和[罗布科内利](http://blog.wekeroad.com/) ( [亚音速](http://www.subsonicproject.com/)的仁慈的独裁者)，以帮助 ASP.NET 变得更好，更开放。在微软之外，他们与米格尔·德·伊卡萨(Miguel de Icaza)合作并接受他的单声道(Mono)和月光(Moonlight)的方式几乎令长期观察者震惊。我觉得这些行动比营销部门的话要响亮得多。

在这一点上，你可能会说“哎呀，怀亚特，这一切都很好，但这与我作为一个(某种类型的 webby)有什么关系？”此时此刻——相当多。不管你喜不喜欢 IE，你都必须和浏览器打交道，因为它将会是你访问者中非常非常重要的一部分。这个经常被嘲笑的平台的新版本正在开发中。不同的是，这可能是我们，威比社区，第一次能够对微软内部产生实质性的影响。不信他们在听？好吧，我提醒你，我们已经说服他们做出一个根本性的战略改变，这确实冒着长期持有的“不要破坏现有网络”的风险，这一观点已经统治了微软几十年，它选择根据像你这样的标准化者的反馈，将[标准模式设置为默认渲染模式](https://www.sitepoint.com/microsoft-backflips-on-browser-version-targeting/)。

我为什么相信这个？这主要是因为，在“微软非常、非常认真地对待开放标准”的主题下，有一股来自微软各个项目团队的非常强烈的暗流，非常想知道您的困难以及他们如何解决它。这些开发者并没有失去开放的力量，他们中的许多人都是在开源应用的陪伴下成长起来的。现在他们有权限听了。

不要忘记网景 5 的寓言——创建一个新的渲染引擎是一个相当冒险的努力，最终导致了网景本身的灭亡。IE 团队是第一个承认新的渲染引擎目前有点缺陷和古怪的。但他们也率先表示，只有更广泛的 web 开发人员才能确保它符合我们多年来一直要求的可靠行为和标准。所以不要抱怨了，今天就参加 IE8 测试版反馈计划或者接受结果。
