# 七个屏幕阅读器可用性提示

> 原文:[https://www.sitepoint.com/screen-reader-usability-tips/](https://www.sitepoint.com/screen-reader-usability-tips/)

不幸的是，仅仅确保你的网站对屏幕阅读器用户来说是可访问的，并不足以保证这些用户能够以一种合理的快速有效的方式找到他们想要的东西。即使你的网站可以被屏幕阅读器用户访问，它的可用性也可能很差，以至于他们一开始就不需要弯腰查看。

幸运的是，有很多简单易行的指导方针可以遵循。以下七个简单的提示将极大地提高一个网站对于屏幕阅读器用户以及其他访问者的可用性。

##### 技巧 1:使用描述性标题

对屏幕阅读器用户来说，页面标题的使用是最重要的可用性特征之一，因为它帮助这些人更容易理解页面结构。尽管对于视力正常的用户来说，页面上的文本可能会显示为标题，但为了让屏幕阅读器用户知道它是标题，它必须在 HTML 代码中被标记为标题。

屏幕阅读器不看网页:他们读 HTML 代码。如果一段文本被识别为 HTML 代码中的一个标题，屏幕阅读器将宣布它是一个标题。否则，屏幕阅读器用户将无法区分你的标题和文本的其余部分。

使用适当的标题标签还有另一个不太明显的可用性好处:屏幕阅读器用户能够调出页面标题列表，并跳转到他们最感兴趣的页面部分。如果你的页面标记正确，屏幕阅读器用户会发现导航比没有使用正确标签的页面容易得多。在最流行的屏幕阅读器之一 JAWS 中，BBC 主页的标题框是这样的:

![1487_jawsheadings](../Images/3aa8827dff263a6994650bbcd844dff0.png)

这种工具的工作方式与视力正常的网络用户通过浏览标题来浏览网页的过程非常相似。如果标题描述了标题下的内容，屏幕阅读器用户就更容易找到他们需要的信息。

##### 技巧 2:编写描述性的链接文本

屏幕阅读器用户可以通过调出页面上的链接列表，并激活他们最感兴趣的链接来浏览网页。因此，非描述性的链接文本，如“点击这里”应该不惜一切代价避免:如果脱离了上下文，它是没有任何意义的。JAWS 中 BBC 主页的链接列表对话框显示如下:

![1487_jawslinks](../Images/4388cec87704c4a50a920b4f108c9bbd.png)

好消息是描述性链接文本的使用为每个人提供了可用性优势。当视力正常的用户浏览网页时，其中一个突出的项目是链接文本。“点击这里”这几个字对于浏览网页的网络用户来说完全没有意义；我们必须搜索周围的文本来找到链接的目的地。

##### 技巧 3:在列表中提供信息

在 HTML 代码中使用列表对于屏幕阅读器用户来说非常有用，因为屏幕阅读器会在读出列表项之前宣布每个列表中的项数。这有助于这些用户在听到一系列项目(如网站导航)时知道会发生什么。

这种设备的工作方式与电话答录机非常相似，它会在播放之前告诉你有多少条留言。一旦你知道你有多少条信息，你就知道会发生什么。如果只有一两条信息，你可能还记得它们；再多一点，你可能就要拿笔和纸来做笔记了。

列表的使用(用`<li>`标记)实际上只是代表了代码的幕后变化；它不需要影响你的网站的视觉外观。

##### 技巧 4:采用逻辑线性化

屏幕阅读器用户通常必须从头到尾、从上到下、从左到右地收听网页。另一方面，视力正常的网络用户几乎可以随意浏览网页，无论重要信息出现在网页的什么地方，他们都能发现。因此，重要的信息应该总是放在页面的顶端。因此，举例来说，当你创建一个表格时，要确保避免在页面底部找到如何填写的说明。

将重要信息放在页面顶部实际上对每个人都有好处，因为重要信息会被放在“折叠上方”，放在视力正常的用户和屏幕阅读器用户最先发现的位置。

##### 技巧 5:使用简短、简洁的替代文本

替代文本是为网站上的图像提供的文本内容替代。替代文本是向屏幕阅读器用户朗读的，所以任何提供基本辅助功能的网站都会提供这种替代文本。然而，一些网站试图过度解释图像所传达的信息，迫使屏幕阅读器用户不得不听取大量不必要和不相关的信息。

屏幕阅读器用户通常比视力正常的网络用户花费更长的时间来浏览网站，因此简洁的 ALT 文本有助于使他们的浏览时间更容易。

##### 技巧 6:写短的、前置的段落

在“前置”段落中，结论放在第一位，然后是“什么”、“为什么”、“何时”、“何地”和“如何”。通过把结论放在第一位，你可以让屏幕阅读器用户立即理解段落的内容。然后，他们可以决定是继续听这个段落，还是跳到下一个段落(他们可以通过屏幕阅读器轻松做到这一点)。如果段落很短，用户可以向前跳，知道他们不会错过额外的信息。

前置内容显然对所有用户都有好处，因为你的网站访问者不再需要四处寻找每一段的要点。

##### 技巧 7:写描述性的页面标题

页面标题是屏幕阅读器用户到达一个网页时听到的第一件事，所以标题必须是对页面的描述。同样，这对每个人都有好处:所有访问者都可以使用页面标题来定位自己，并确认他们在他们期望或希望出现的页面上。对于使用拨号连接的 Web 用户来说尤其如此，在拨号连接上，页面标题比其余内容提前几秒钟显示。

##### 结论

我们已经探索了一些相对简单、轻松的步骤，可以用来提高屏幕阅读器用户对站点的可用性。幸运的是，几乎所有这些都提高了所有 Web 用户的可用性，因此每个人都受益！对你的网站做这些改变，你的用户会感谢你的。
