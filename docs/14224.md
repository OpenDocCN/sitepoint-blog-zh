# 爪哇和杜松子酒

> 原文：<https://www.sitepoint.com/of-java-and-gin/>

一场反常的 [风暴](http://www.theage.com.au/photogallery/2005/02/03/1107409974398.html) [去年](http://www.theage.com.au/photogallery/2005/02/03/1107409987268.html) [周三](http://www.theage.com.au/photogallery/2005/02/04/1107472871605.html)墨尔本迎来了有记录以来单日最多的降雨。第二天早上，整棵树都被淹没在耶那河上，我去听爪哇之父在孙举办的活动上的演讲。

讨论是由与会者提前提交并即时提问的问题推动的。问题从技术到政治，从深奥到平凡。

第一个问题是:如果他是第一次载人火星飞行的乘客，他会希望飞船在爪哇还是在火星上运行？NET，为什么呢？尽管他认为。NET 是可预见的，他沮丧地承认，他会担心他的生活取决于他必须做的任何事情。

当然，这可能正是高斯林先生的发展方向，因为他目前的工作主要集中在[实时 Java](http://www.rtj.org/) (他承认，这让他对当前的“热门话题”如企业 Java 一无所知)。

当被问及他的工作时，他说实时编程是一个可怕的领域，大致定义为如果出了问题，就会有人死去。当前面临的一个主要挑战是，像汽车这样的实时系统开始网络化，网络固有的延迟是一个需要应对的真正挑战。

他接着描述了他参与的一个研究项目，在这个项目中，一辆汽车被赋予了无线功能，因此车主可以通过手机与它进行通信。“你甚至可以问汽车，‘你在哪里？’”他利用车载 GPS 接收器说道。他咧嘴笑着，开玩笑地描述了他如何在几个场合使用该功能，当他在晚上“穿过杜松子酒薄雾”找到他的车时——这是几个提到他明显选择的饮料之一。

关于开源 Java 的话题，Gosling 先生大声问那些呼吁“开源 Java”的人想要什么，而他们还没有。“从第一天起，从 Java 1.0 的第一个版本开始，源代码就已经可用了，”他指出。唯一的限制是许可证，它旨在确保兼容性和互操作性。“要发布它，你必须通过测试套件。所以我们有来自地狱的测试套件，”他挖苦地解释道。他似乎对最近几个开源团体许可测试套件持肯定态度，但仍然察觉到了一些文化冲突。“我认为开源开发者喜欢兼容性测试，但他们不喜欢*承诺*兼容，”他笑着说。

当被问及 Swing vs. SWT 时，他带领观众回顾了一下 Java 的历史，以说明当前形势的讽刺性。“Swing 在很大程度上是由 IBM 创造的，”他指出。" Swing 是 IBM 的梦幻 UI 套件."这就是为什么他觉得很奇怪，以至于几年后，IBM 的一个团队会发布 SWT，他将其描述为带有所有相同问题的 AWT 的倒退。他称之为“奇怪的工业政治”。

下一个议程是 Sun 和微软之间长达 10 年的技术联盟。观众中的一名开发人员回应了 Java 社区的大部分问题，他问道:“这意味着什么？”高斯林先生似乎与这种困惑感同身受，但他能够对由此产生的一些工作进行一些阐释。

他详述的主要例子是电子邮件服务器标准之间的分歧——开放的 IMAP 标准与微软的 Exchange。根据该联盟的条款，Sun 能够获得有关 Exchange 的专有信息，以便在其产品中构建与此类服务器的兼容性。不幸的是，任何封闭的东西都必须保持封闭，因此 Sun 产品中的任何微软专有技术都必须以封闭许可证的形式发布。

Sun 和微软正在共同探索的另一个领域是身份管理系统的统一，如开放的 LDAP 标准和微软的 Active Directory。这方面的工作正在以“自由项目”的名义进行，但方向变得清晰需要时间。他确实提到了微软最近决定停止将 Passport 作为其客户的解决方案。尽管他仍然不确定他们在想什么，“他们似乎在做正确的事情，”他说。

高斯林先生在最近的一篇杂志文章中发表评论后，被问及关于软件专利的问题，这个话题最近在欧洲的新闻中经常出现。高斯林先生深情地回忆起他和一群 Sun 工程师曾经不得不参加的一场非正式比赛，看谁能让最愚蠢的专利获得批准。他提到了他获得的一项专利，是关于电灯开关的接线。公认的获胜者获得了一项通过应用液氮清除电路板上胶水的专利。

简而言之，他总结道，专利审查员让任何旧的东西通过。他继续解释说，虽然他认为软件专利作为一个概念是一个好主意，但他看不出有什么方法能让这个系统在实践中发挥作用。不过，他预计，如果有解决方案的话，就需要废除目前的体系，从头开始。

总之，这是一个了解 Java 世界中一位传奇人物思想的绝佳机会。如果你有机会听詹姆斯演讲，我强烈推荐你去体验一下。
