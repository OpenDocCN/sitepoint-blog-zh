# 复习 PHP 4 圣经

> 原文:[https://www.sitepoint.com/review-php-4-bible/](https://www.sitepoint.com/review-php-4-bible/)

最初，我对 *PHP 4 圣经*非常感兴趣。我听说 Tim Converse 和 Joyce Park 合著的这本书很好地涵盖了这个特定的主题，不像大多数其他 PHP 书籍，它们通常是在考虑 PHP 3 的情况下编写的。

然而，在阅读了 IDG 世界图书公司的出版物后，我非常失望。作为一名有 PHP 入门经验的程序员，我理解这本书的最初几章。但是，我怕如果没有这之前的知识，我早就迷失了。

##### 最初的几章

在 *PHP 4 圣经*中你会发现的第一件事是 PHP 的基本解释。作为对这一描述的“辅助”，文本通过一个过于复杂的例子来处理数据库、复杂的 HTML 代码和 CSS，显然是为了迷惑读者(我可以向你保证，对于新手来说，它做得很好)。当作者试图解释基本的 PHP 时，他们快速地浏览变量，甚至在他们解释控制结构和循环之前，他们已经试图向读者介绍 OOP 了。前 7 章写得非常糟糕，任何没有基本编程知识和对 PHP 有所了解的读者无疑已经迷失了。

我认为，在这一点上， *PHP 4 圣经*只会变得更好。谢天谢地，这本书的字符串和数组章节写得相当好，尽管我仍然有一些不满。例如，`printf()`和`sprintf()`函数的内容缺乏高质量例子的广度，而这些例子对于读者真正掌握这两个函数的用途和语法是必要的。此外，我惊讶地发现，在字符串一章的结尾，作者试图塞进正则表达式。这是一个高级话题；为什么这么快就盖住了？也许他们害怕在书中增加新的章节和更多的页数？= >操作员的解释也非常令人困惑。读者可能期望在字符串和数组章节的结尾，或者在书的结尾有一个好的参考，但是他们会再次失望。

我对 *PHP 4 圣经*感兴趣的一个主要原因是，从目录来看，这本书包含了大量关于数据库的信息——这是 PHP 的主要优势之一。然而，没有数据库经验的读者很容易在探索数据库的过程中迷失方向(如果他/她还没有的话)。不管这是一本 PHP 书籍，它应该包含基本的数据库信息和技术。事实并非如此。

##### 不完整的参考文献

这本书里我最喜欢的两章是关于 Cookies 和会话的，而且令人惊讶的是，这两章非常有用。不幸的是，他们在面向对象编程的主题上没有取得这样的成功。PHP 的初衷是成为一门简单的语言，因此 OOP 支持自然不会被认为是过于重要的。但是这本书中涉及它的方式使得 OOP 显得无关紧要。尽管有整整一章专门讨论 OOP，但它实际上并没有包含关于这个主题的有用信息。作为指导，作者给出了例子，让观众自己去解读。

作为许多计算机书籍的读者，我发现在整本书中散布扩展的例子通常有助于使它更有效，并使关键的知识深入人心。如果 PHP 4 圣经包含了一个扩展的例子，结合了本书试图探索的所有 PHP 技术，那么它作为参考或教育工具将会更有价值。

##### 判决

总之，我不会推荐这本书，只是因为我无法想象它能满足谁的需求。这不是一本适合初学者的书，因为它需要预先的编程知识和基本的 PHP 经验。但是它也不能称之为一本深入、先进的 PHP 4 书籍，因为尽管它试图解决许多主题，但它几乎涵盖了所有的主题。

PHP 4 圣经也不能构成一本参考书，因为它只涵盖了 PHP 函数全部内容的 5%,而且缺乏全面的参考。尽管 [PHP 文档](http://www.php.net/manual/en/)是一个很好的免费参考资料，但你可能会期望，如果你付费阅读一篇文章，你会得到一个更全面的参考资料，有更好的解释和例子。如果这是你的期望，不要为 *PHP 4 圣经*而烦恼。

**2/5 明星**
