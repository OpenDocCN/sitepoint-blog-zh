# Chrome 扩展可能在 5 月推出

> 原文：<https://www.sitepoint.com/chrome-extensions-likely-by-may/>

我们知道[扩展](http://dev.chromium.org/developers/design-documents/extensions)是 Chromium 团队的首要任务(Chromium 是支持谷歌 Chrome 网络浏览器的开源项目)。几个月前，谷歌工程师在一份设计文件中写道，他们希望创建一个扩展基础设施，能够支持“一个开放的 API 列表，如工具栏、侧栏、内容脚本(用于类似 Greasemonkey 的功能)和内容过滤(用于父母过滤器、恶意软件过滤器或类似 adblock 的功能)。”

然而，尽管扩展是 Chrome 问题跟踪器上迄今为止[请求最多的功能，但该文档从未提及交付这样一个附加系统的任何具体时间表。](http://code.google.com/p/chromium/issues/list?can=2&q=&sort=-stars&colspec=ID%20Stars%20Pri%20Area%20Type%20Status%20Summary%20Modified%20Owner)

我们在 12 月份注意到，许多用户没有从 Firefox 转向 Chrome 的主要原因之一是缺乏这样的扩展系统。SitePoint 和 Chrome 官方讨论列表上的许多用户都表示，他们真的很喜欢 Chrome，但没有他们已经依赖的某些 Firefox 扩展，他们就无法生活。所以 Chrome 没有插件支持是不行的。

对于 Chrome 来说，最好的早期目标用户群是早期用户——那些已经在使用 Firefox 并且可能已经有一两个不想放弃的扩展的人。“一旦谷歌将计划中的扩展框架添加到 Chromium 项目中，[我们]认为我们可以期待看到它的市场份额大幅上升。它不会在一夜之间超越火狐，但它将成为一个相当重要的次要玩家——我们的意思是:小心火狐、Opera 和 Safari，”我们在 12 月写道。

现在看起来我们可能已经有了 Chrome 延期的暂定日期:不晚于 5 月 27 日。这一天，谷歌年度开发者大会 Google I/O 将在三藩市开幕，眼尖的开发者[尼古拉斯·魔线](http://www.nick.pro/2009/02/04/google-chrome-extensions-coming-out-by-may/)发现的[会议之一叫做“为谷歌浏览器开发扩展”](http://code.google.com/events/io/sessions.html)

“了解 Google Chrome 如何使用您已经了解的 web 技术轻松编写扩展。该讲座将涵盖扩展系统的基础知识(分发/打包、安装、更新)，以及通过浏览器增强的不同 API，”是对 I/O 站点上的会话的描述。

很可能，这意味着到 5 月 27 日——可能更早——谷歌的 Chrome 浏览器将会有扩展。或者至少是开发人员创建它们的方法。谷歌操作系统博客[指出](http://googlesystem.blogspot.com/2009/02/google-chrome-will-have-extensions-by.html)Chrome 的开发者版本已经支持用户脚本，这可能是更广泛的扩展系统的先驱。

浏览器大战即将更加白热化。
