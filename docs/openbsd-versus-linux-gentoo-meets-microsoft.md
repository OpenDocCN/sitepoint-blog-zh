# OpenBSD 对抗 Linux，Gentoo 遭遇微软

> 原文：<https://www.sitepoint.com/openbsd-versus-linux-gentoo-meets-microsoft/>

OpenBSD 创始人 Theo de Raadt 在接受《福布斯》杂志采访时公开抨击 Linux 和 Linus Torvald 对操作系统的监管质量低下。

Raadt 也不关心像 IBM、HP、Sun 和其他从开源社区的免费贡献中获取利润而不给予更多回报的公司。这可能有一些实质性的东西——尽管这些公司和其他一些公司在赔偿客户和释放一定程度的专利供免费使用方面采取了积极的行动。

也许看到开源社区以某种广泛的联盟方式进行更多的投资，以提高开源趋势、使用模式、投资回报等方面的研究数量，这将是一件好事。然而，我们需要记住，这些公司确实承诺员工在带薪工作时间从事 Linux 项目，作为对社区贡献的一部分。

我在多平台环境中花费了太多的时间，所以我不像其他人那样对 Linux、OpenBSD、Windows 和其他 Unix 变体等抱有宗教信仰。然而，作为一个个人和服务器端的重度 OS X 用户，我非常喜欢 OpenBSD、OS X 和 Solaris 的 BSD Unix 基础。尤其是从安全角度来看。

我在工作中使用的主要操作系统是 Linux 和 OS X，我仍然是这两个系统的粉丝。我对质量控制糟糕的混乱系统没有相同的看法，也不认为 Linux 是 Raadt 评论的“廉价的小黑客”的集合。

有一点 Raadt 让 rings 有点类似于我们在开放源码中讨论过的 Linux 商业化和集成到主流技术中。如果 Linux 社区中的一些人花更少的时间来憎恨微软，而花更多的时间来为开源建立论据，收集可靠的研究成果，并在 Linux 和应用程序中编写伟大的代码——我们可能会比今天更进一步。

这也引发了一则新闻，Linux 创始人和前首席架构师 Daniel Robbins 受雇于微软，帮助该公司更好地理解开源作为一个社区、一种商业以及它在商业中的部署和集成。

至少对我来说，不一定是完全的震惊。开源项目的参与者也需要吃饭。我在一个专有的世界里赚了一部分钱。

虽然这可能有一些阴谋——我认为这主要是因为微软看到市场份额被 Linux 蚕食。此外，许多已经在混合环境中采用 Linux 的商业客户可能最终会向微软施压，要求了解如何与它一起部署。还有一个事实是，微软一直在玩弄开源，向开源社区发布自己的代码，并探索如何利用社区参与。

众所周知，Windows 上有相当多的开源项目，如果微软看到了与我们一起参与开源的市场，无论我们喜欢与否，它都会这样做。
