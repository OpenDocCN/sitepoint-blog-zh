# 写伟大的代码

> 原文:[https://www.sitepoint.com/write-great-code/](https://www.sitepoint.com/write-great-code/)

我们所有人都渴望写出伟大的代码，不仅功能强大、没有错误，还能充分利用环境的各个方面来高效运行。兰德尔·海德希望从名为[写伟大的代码，理解机器](http://www.nostarch.com/)的四本书系列的第一本开始，让这条路变得更容易一些。

Hyde 正在着手一个继续处于所有开发的核心的主题，即计算机体系结构的基础。在这种情况下，Hyde 主要面向 x86 平台上的用户，尽管基本原理也适用于 Power PC 和其他平台。在本系列之前，人们将通过培训课程、操作手册和试错法开始编程。

然而，作者希望他能为程序员提供相当于大学系统架构课程的东西，而不需要先学习汇编语言。

在万维网成为专业开发的核心元素之前，程序员走过了学习过程语言的传统道路，可能是在大学或通过自己的试错，例如 c。然而，自 20 世纪 90 年代后期以来，许多开发人员的第一次代码尝试是 Perl、PHP 或 Python。

在某些情况下，缺少编程的功能构建模块以及代码与处理器、内存等硬件元素的交互可能会成为编写优秀代码的无形障碍。以我自己的经验来看， [Code Complete](http://www.amazon.com/exec/obidos/ASIN/0735619670/qid=1109901759/sr=2-1/ref=pd_bbs_b_2_1/104-4939488-3605544) 是一本类似于海德作品的书，它把我带到了在写代码之前思考代码的下一个层次。海德的书在一个全新的层面上挑战着我。

根据 Randall Hyde 的说法，这是他的书旨在触及的新一代开发人员——“因为这是一群可能从未在其他地方见过这些材料的程序员，”他告诉 SitePoint。

在一些人可能认为这些书可能偏离 web 开发语言很远之前，Hyde 说四卷中的三卷将直接面向选择使用 web 解释语言的开发人员。

“第一卷、第三卷和第四卷将适用于那些想继续学习[Perl、PHP 等]的人。第二卷没什么意思——因为它涵盖了编译器如何生成代码，这对脚本/解释语言来说是不适用的，”海德补充道。

第一卷涵盖了基本的机器数据类型、基本的机器体系结构、I/O 和其他相对独立于语言的主题。

“这本书使用了 C、Pascal、汇编和 Basic 语言中的例子，但熟悉任何命令式语言的人都会明白发生了什么，”海德说。

虽然第一卷没有大量涉及 LAMP 程序员感兴趣的面向对象开发，但是讨论了对象如何在机器级别实现以及它们的成本。

对我来说，重要的是专注于提供对计算机如何执行代码的理解——这使人停下来考虑自己的应用程序的组织。

海德的临别评论:“无论程序员使用什么样的语言工具，理解机器如何表示数据、执行代码和处理 I/O 对那些想写伟大代码的人来说都是至关重要的。”
