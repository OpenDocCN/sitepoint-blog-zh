# 如何在 CSS 中使用操作系统样式

> 原文:[https://www.sitepoint.com/css-system-styles/](https://www.sitepoint.com/css-system-styles/)

CSS2.1 的一个鲜为人知的特性是能够定义底层操作系统主题所使用的字体和颜色。这在您需要更紧密的操作系统集成的情况下非常有用，例如 HTML 帮助文件、Adobe AIR 或离线 web 应用程序。

在我们开始之前，有一些警告:

*   不要依赖这些属性在所有操作系统/浏览器组合中工作。如果您的应用程序必须在 BeOS 上的 Opera 中运行，那么我建议您首先进行测试！
*   在 CSS3 中，这些属性已经被弃用，取而代之的是[外观值类型](https://www.w3.org/TR/css3-ui/#system)(尽管目前浏览器支持非常有限)。
*   没有什么可以阻止用户在他们的操作系统中定义不寻常的、冲突的或难看的配色方案。页面会反映他们的选择，而不是你的设计师的选择。

## 系统字体

使用“font”属性分配系统字体。请注意，族、尺寸和样式都是根据需要指定的，例如

```
 body
{
	font: caption;
} 
```

以下字体值可用。“示例”栏显示了操作系统当前设置的字体。

| 财产 | 描述 | 例子 |
| 标题 | 控制字体(按钮、下拉框等。) | ABC abc 123 |
| 图标 | 图标标签字体 | ABC abc 123 |
| 菜单 | 菜单字体 | ABC abc 123 |
| 消息框 | 对话框字体 | ABC abc 123 |
| 小标题 | 小控件标签 | ABC abc 123 |
| 状态栏 | 状态栏字体 | ABC abc 123 |

## 系统颜色

系统颜色可以分配给任何需要颜色值的属性，例如

```
 body
{
	color: WindowText;
	background-color: Window;
	border: 2px solid ActiveBorder;
} 
```

以下是可用的颜色值。为了清晰起见，它们显示在[茶色盒](http://en.wikipedia.org/wiki/Camel_case)中，但是任何大小写都是有效的。“示例”栏显示了操作系统设置的颜色。
