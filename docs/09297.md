# Chrome 14 有什么新功能

> 原文：<https://www.sitepoint.com/chrome-14-whats-new/>

你注意到 Chrome 这个周末的更新了吗？你当然没有。浏览器会自动安装，无需你动手。选择“工具”>“关于谷歌浏览器”,你应该会看到版本 14——如果没有，更新将在该点发生。它在所有平台上都可用。

谷歌为期六周的发布时间表通常不会为许多闪亮的新功能提供时间，但浏览器已经增加了一些有趣的功能…

## 错误修复、增强和速度改进

我在 Chrome 上没有遇到过很多问题，但是一些 SitePoint 用户报告了一些问题。32 个 bug 已经被压扁，虽然没有一个被认为是严重的，但它可能会解决你的稳定性问题。

打印预览现在可以在所有平台上自动运行。它由内置的 PDF 阅读器驱动，看起来运行良好——尽管一些 OS X 用户报告了不寻常的问题。Mac 用户也会很高兴听到全屏模式有效；按住 Ctrl+Shift+F。

正如你所料，Chrome 14 速度更快。也就是说，它已经是最快的浏览器之一，除非你运行基准测试，否则你不太可能注意到改进。

## 网络音频 API

一般来说，网络上的音频要么不存在，要么需要插件，要么使用不当。现在这种情况可能会改变，开发者可以得到 W3C API T1，它允许在 JavaScript 中进行声音处理和合成。通常，您可以应用复杂的滤镜和空间效果来增强游戏或应用程序界面。

谷歌提供了一系列令人印象深刻的例子。你一定要检查鼓机、摇摄/混响和多普勒频移。除非你有几个小时的空闲时间，否则不要犯我玩 8 球的错误！

## Native Client (NaCl)

Chrome 14 是第一个开箱即用启用 NaCl 的浏览器版本。NaCl 是一个浏览器插件，它直接在浏览器中下载和运行本地编译的 C/C++代码。谷歌的好处是显而易见的:他们的网络浏览器和操作系统可以支持复杂的游戏、视频编辑器和其他通常在桌面上运行的高端应用程序。

NaCl 代码运行在安全的沙箱中，目前，它只对[网络商店应用](https://chrome.google.com/webstore)可用。谷歌正在为其他浏览器创建插件，然而，这项技术是试验性的，所以不确定它是否是 Java、Flash 和 Silverlight 的可行替代品。

为了解更多信息，产品经理 Henry Bridge 制作了一个 [NaCl 介绍视频](http://www.youtube.com/watch?feature=player_embedded&v=nP8Mo0jGQDk)。你也可以从 code.google.com/p/nativeclient-sdk/的[下载 SDK。](http://code.google.com/p/nativeclient-sdk/)

谷歌流畅的更新流程和创新的功能赢得了开发者和用户的青睐。如果微软和 Mozilla 想要阻止 Chrome 成为世界第一浏览器，他们还有一些艰苦的工作要做。
