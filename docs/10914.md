# 关于 2010 年混音的思考

> 原文：<https://www.sitepoint.com/reflections-on-remix-2010/>

上周，SitePoint 的一个小团队参加了澳大利亚的 REMIX Australia(T1 ),在那里，我们自己的[Matty Magain(T3)做了一个非常受欢迎的关于匿名访客数据的演讲，题目是“我们对你的了解”](https://www.microsoft.com/australia/remix/agenda/schedule.aspx#Matthew_Magain)

这是我总结的最好的部分。

这次为期两天的会议的主题是“分享网络之爱”，虽然微软可能不是你第一个与“自由之爱”联系在一起的公司，但发言人名单显然比我想象的要多。

老詹姆斯——微软和 jQuery:一个真实的爱情故事。

第一天，我们看到[微软的网络布道者 James Senior](https://www.jamessenior.com/post/jQuery-Slides-Demo-from-my-Remix-Australia-Talk.aspx) 展示了微软鲜为人知的 jQuery 项目的背景故事。

从 2002 年开始，微软一直在开发他们自己的内部 JavaScript 库，以包含在他们的许多产品中。然而，在 2008 年，当 jQuery 显然是一个更好的解决方案时，他们开始用 ASP.NET 和 Visual Studio 来代替它。从表面上看，这是一个非常冷静的商业决策。

然而，根据 James 的说法，微软自己的 Ajax 脚本库确实有一些 jQuery 目前无法复制的特性。约翰·瑞西格同意了。

因此，今年 2 月，James 和一个小型 MS 团队与 jQuery 团队坐下来，计划如何在 jQuery 的下一版本中帮助实现这些特性。微软现在有全职开发人员和支持人员致力于 jQuery。

微软会因为自己的原因试图劫持或颠覆 jQuery 吗？一点也不。微软的贡献经历了任何贡献者都会面临的完全相同的同行审核过程。他们是开放合作的积极参与者，大概也看到了好处。耶。

James 继续用一个漂亮的 jQuery 模板演示展示了他们的一些劳动成果，该演示将 Twitter 搜索查询拉到一个页面中。由于我们在 SitePoint 上使用了类似但更冗长的代码，很难不对这一新代码的优雅印象深刻。

当然，每个人都有一个快乐的故事。

Windows Phone 7:皇帝有了新线索——尼克·伦道夫

尽管毫无疑问，iPhone 和 Android 目前正在赢得思想份额之战，但随着 Windows Phone 7 的推出，微软终于给了开发者一些他们真正感到兴奋的东西。

第一天午餐前， [Nick Randolph](https://nicksnettravels.builttoroam.com/) 带领我们完成了一个典型的 Windows Phone 7 应用程序创建项目，在 Visual Studio 和 Expression Blend 之间来回切换。虽然这种技术超出了我的日常范围，但这是一种感受平台的好方法。

虽然我不得不承认，到目前为止，我倾向于将 Silverlight 视为没有安装基础的 Flash，但 Windows Phone 7 几乎将成为 T2 的食物和呼吸。微软还能再次成为移动 OS 市场的大玩家吗？这仍有待观察，但如果他们这样做，Silverlight/Expression 混合技能将会非常受欢迎。

顺便说一句，如果你对 Windows Phone 7 的开发感兴趣， [Shane Morris 最近为我们写了一本很棒的入门书，涵盖了关键概念](https://www.sitepoint.com/article/expression-blend-behaviors)。这是一个很好的起点。

**深入了解 Internet Explorer 9–Tatham Oddie**

第二天，Tatham Oddie 带我们在闪亮的新 Internet Explorer 9 预览版中进行了一次奇妙的旅行。如果你还不了解这一点，尽管“外壳”仍然很原始，IE9 渲染引擎显示了非常有价值的车辆的早期迹象。

Tatham 演示的亮点包括:

*   完全的 W3C 支持
*   原生 SVG (VML 终于被埋葬了)
*   CSS 背景中的 Alpha 透明支持(即`background-color: rgb**a**(124, 78, 13, .5);`中的“a”)。Unllike CSS 不透明。这让你可以使一个 DIV 半透明，而不用使里面的所有东西都半透明。
*   一些新的高级 CSS 选择器包括`first-child`、`last-child`和`even`奇数和`even`。

最终，尽管有很多我喜欢的，但我还是觉得有点苦/甜。虽然，我真的很高兴 IE9 看起来像我们都想要的微软浏览器，但 IE7 和 IE8 看起来像是错过了更早到达那里的机会。啊，没错。

**建立开放的网络——拉克伦·哈迪**

我在第二天看到的最后一个环节是拉克伦·哈迪(lach LAN Hardy )( T1 ),他详细讲述了选择开放标准而非专有代码的冷酷无情的商业理由。

正如你可能知道的，拉克伦长期以来一直是标准的倡导者，但听到他对不太相信这一论点的观众讲话，比平时更有优势。

**总结**

虽然甲壳虫乐队的歌曲和迷幻漩涡可能不是我们大多数人对微软的看法，但我认为你真的应该给予一些信任。

标准社区和 Redmond 并不总是意见一致，这不是什么秘密，最简单和最安全的做法是向他们的 MS 友好人群推销纯粹以 MS 为中心的内容。就不会有什么抱怨了。

相反，他们选择展示像拉克伦、约翰·奥尔索普和马蒂这样的演讲者——他们谈论的是技术，实际上是微软正常对话范围之外的意识形态。

最后，我认为演讲者、观众和微软都明白其中的巨大好处。
