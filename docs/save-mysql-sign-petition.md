# 注册拯救 MySQL？

> 原文：<https://www.sitepoint.com/save-mysql-sign-petition/>

甲骨文-Sun-MySQL 的争论似乎还远未结束。开源数据库的创始人 Michael“Monty”wide nius 一直强烈反对 MySQL 落入 Oracle 的魔掌。然而，时间已经不多了，最初犹豫不决的欧盟委员会(T2)很可能会在甲骨文(Oracle)最近公开承诺声明(T4)之后批准收购。

作为拯救数据库的最后努力，Widenius 于 2009 年 12 月 28 日推出了 helpmysql.org。该网站热情地为 MySQL 辩护，并提出了一些有趣的问题，例如

*   为什么甲骨文如此坚持收购 MySQL 作为收购 Sun 的一部分？如果数据库被传递给另一个组织，监管者不会感兴趣。
*   如果一个开源产品的所有权并不重要，为什么甲骨文想要购买 MySQL 而不是将其分叉？
*   MySQL 会掉牙吗？为什么 Oracle 会以可能影响其商业市场的方式操纵数据库？

该网站要求所有相关的贡献者、开发者、经销商[签署一份请愿书](http://www.helpmysql.org/en/petition)，以确保未来与 MySQL 相关的创新，并保护数据库作为一个主要的竞争力量。你的时间不多了:请愿书将于 2010 年 1 月 4 日正式提交给监管机构、其他政府机构、议会和记者(尽管更新将在未来几个月公布)。

根据撰写时的[统计，迄今已收到超过 13，500 个签名。](http://www.helpmysql.org/en/stats) [Widenius 的博客](http://monty-says.blogspot.com/2009/12/help-keep-internet-free.html)报道称，在目前收到的回答中，只有 0.7%的受访者表示他们信任 Oracle 的 MySQL。剩下的 99.3%有所顾虑，要么希望甲骨文或 MySQL 进一步向合适的第三方转让。

如果你担心甲骨文对 MySQL 的所有权，[在请愿书上签名，说出你的意见](http://www.helpmysql.org/)。

监管机构会被这份请愿书左右吗？甲骨文应该被允许拥有 MySQL 吗？收购会损害数据库竞争吗？欢迎评论…
