# 微软在浏览器版本定位上的后空翻

> 原文：<https://www.sitepoint.com/microsoft-backflips-on-browser-version-targeting/>

当我今天早上上传博客的时候，我不得不再次确认今天的日期。

一旦我确认事实上*不是*4 月 1 日，我就继续阅读，呆住了。Internet Explorer 的总经理 Dean Hachamovitch 宣布[微软已经改变了针对](http://blogs.msdn.com/ie/archive/2008/03/03/microsoft-s-interoperability-principles-and-ie8.aspx)浏览器版本的想法。

> 我们已经决定，默认情况下，IE8 将以最符合标准的方式解释网页内容。这个决定与我们之前发布的有所不同。微软最近发布了一套互操作性原则。根据这些原则思考 IE8 的行为，以最符合标准的方式解释 web 内容是一件更好的事情。

除了引用微软最近对互操作性的公开承诺之外，他还引用了来自社区的反馈作为决定的一个因素。他小心翼翼地提醒开发者为什么微软最初确信默认 IE7 的标准呈现模式是唯一的选择——这是在与[网络标准项目](http://webstandards.org/)和像[网络标准之王本人](http://www.alistapart.com/articles/minorthreat/)这样的领军人物协商后做出的决定。

微软做出这一决定的事实表明了几件事:

*   他们认真对待互操作性。大谈开放和标准是一回事，但同时要求开发人员在标记中添加专有标签以触发标准模式是完全虚伪的。
*   网络真正属于人民——不是微软，不是 WaSP，也不是一个单独的列表。Web 标准项目可能已经决定将 IE7 的标准模式默认为 time eternal 已经足够好了，但谢天谢地，足够多的社区成员通过评论、博客帖子和像 Jeremy Keith 写的文章这样的[文章表达了他们的担忧，让他们关注并重新考虑手头的所有问题。](http://www.alistapart.com/articles/theyshootbrowsers)

等等——你听到那个声音了吗？

我的朋友们，这是世界各地的 web 开发人员发出的喜悦的呼喊——web 开发人员仍然记得 IE6 使他们的工作变得多么痛苦，他们在被要求添加专有标签以满足一个现代浏览器的需求时保持沉默并受到侮辱。

微软做出了正确的决定，赢得了巨大的赞誉。你可以像猴子一样跳来跳去，尖叫着[开发者，开发者，开发者！](http://video.google.com/videoplay?docid=6304687408656696643)所有你想要的。但正是这样的行动让我们站起来倾听。

在这种特别放心的开发商的情况下，鼓掌。
