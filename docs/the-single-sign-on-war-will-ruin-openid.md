# 单点登录战争会毁掉 OpenID

> 原文：<https://www.sitepoint.com/the-single-sign-on-war-will-ruin-openid/>

就在[微软宣布计划](https://www.sitepoint.com/microsoft-jumps-on-openid-bandwagon/)让超过 4.2 亿 Windows Live ID 账户兼容 OpenID 的两天后，谷歌也加入了进来。[谷歌宣布](http://google-code-updates.blogspot.com/2008/10/google-moves-towards-single-sign-on.html)从今天开始，Gmail 账户可以作为 OpenIDs 使用。随着谷歌的宣布，这意味着世界顶级网络资产-谷歌，雅虎！微软、美国在线、MySpace——现在都是 OpenID 的提供商，或者即将成为提供商。这对 OpenID 来说应该是一个巨大的胜利，但不幸的是，虽然这些公司口头上支持单点登录的想法，但他们仍然没有真正支持 OpenID 的想法。

OpenID 网站这样描述这个想法:“OpenID 消除了跨不同网站使用多个用户名的需要，简化了您的在线体验。您可以选择最能满足您需求的 OpenID 提供商，最重要的是，选择您信任的提供商。”

OpenID 应该是一个完全开放的系统，允许任何人成为 URIs 的提供者。OpenID 应该在品牌身份系统之上，因为不管你的提供商是谁，你的凭证应该在任何接受 OpenID 的地方都有效。不幸的是，这不是大网站所认同的愿景。

相反，雅虎！和谷歌——可能很快还有微软——陷入了一场成为事实上的 OpenID 提供商的战斗。通过拒绝成为依赖方(即，拒绝对来自其他提供者的 OpenIDs 在他们自己的属性上进行身份验证)，单点登录实用程序对用户来说是完全失去的。目前，我仍然需要一套单独的凭证来登录 Gmail，MyYahoo！、和 Windows Live Messenger(我使用的所有服务)。

正如克里斯·梅西纳写的那样:“虽然我赞同更多的 OPs 对网络更有利的观点，但如果没有一家大型百货商店接受 Visa 卡，我不认为它有什么用处。”

雅虎！谷歌进一步掩盖了他们的真实目标，他们试图通过鼓励开发者添加“登录雅虎！/Google”按钮，并把他们自己独特的“旋转”放在 OpenID 上(正如一些开发者已经开始注意到的，Google 今天宣布的并不是一个纯粹的 OpenID 实现)。也没有为用户提供独特的，声称 OpenID URIs。相反，它们使用通用 URL 作为 API 起点，指导用户使用更传统的用户名和密码模式登录。这本身并不是一个坏主意。

***编辑:**技术上来说，雅虎确实为用户提供了一个独特的 URI，但是它们并不是非常用户友好，用户也不清楚在哪里可以找到它们——可能是因为雅虎我更希望开发者实现一个“登录雅虎！”按钮。*

两个字段“用户名”和“密码”的方法在用户心中根深蒂固，以至于许多人在看到 OpenID 登录表单时感到困惑，不知道如何进行。用户在最近一次[雅虎！可用性测试](https://www.sitepoint.com/yahoo-mainstream-has-no-idea-what-openid-is/)证实了这一点，许多人报告说，当他们没有看到他们习惯的密码框时，他们感到困惑。使用电子邮件地址代替 URIs 作为 OpenID 是 T2 的 Chris Saad 在八月份谈到的事情。

然而，谷歌和雅虎！(很可能微软也会步其后尘)最终会相互竞争，成为网络的品牌单点登录解决方案。对用户来说，好消息是通过使用相同的底层技术，大多数依赖方将能够相当容易地打开对任何新的 OpenID 提供者的支持。对用户来说，坏消息是，由于没有一个主要的提供商也是依赖方，在这些网站上使用服务仍然需要多个帐户。此外，一个登录框中有 100 个不同提供商的 100 个不同的徽标，这不是一个好的用户体验。
