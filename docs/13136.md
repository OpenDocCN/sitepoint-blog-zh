# CSS 重置已重新加载。又来了。

> 原文:[https://www.sitepoint.com/css-reset-reloaded-again/](https://www.sitepoint.com/css-reset-reloaded-again/)

去年，埃里克·迈耶记录了他对重新设定网页风格到一个共同基准的想法。对于渴望像素完美布局的设计师来说，每个浏览器应用的默认风格之间的差异可能会令人愤怒，这是一个非常好的方法来平衡所有浏览器之间相同的渲染。

他在首次发布风格重置规则时的评论是:

> …而不是“每个人都必须以某种方式不加改变地使用这些风格”。我也不是说每个人都必须使用它们，否则就会陷入黑暗。记得在那之前，我把这些称为“我对重置风格的主题的看法”(重点是后加的)。你的观点可能会有细微的或很大的不同。

然而，他最近发布了一个[修改版本](http://meyerweb.com/eric/thoughts/2008/01/15/resetting-again/),修改了其中的一些风格，并承认这真的可以成为一个一致的基准，从这个基准开始开发在所有浏览器中都可以预测的风格规则:

> …随着时间的推移，我逐渐意识到这不仅仅是一个一次性的开发工具。这实际上是基线样式表的开始。(或者可以。)像粗体和斜体这样的东西是读者会看到的最明显的文本效果，让重置样式在不同的浏览器中以不一致的方式对待它们是没有意义的。

感谢 Eric，他深入到令人沮丧的浏览器特有风格的世界中，并将其剔除，这样我们就不必这么做了！
