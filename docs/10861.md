布莱德:是的。另一个主要的特性，这是一个稍微有点技术含量的特性，但是如果你是一个开发者或者你真的很喜欢设计和定制 WordPress，你会喜欢它的，这就是定制的 post-type 增强。所以自 2.9 以来，自定义的文章类型就出现在 WordPress 中了，但是它们一直都在代码端——没有附加的用户界面，所以你当然可以使用它们，但是这并不容易。好吧，3.0 改变了这一点，现在通过一个简单的函数调用，你可以注册一个 post 类型，它本质上是 WordPress 中的另一种数据类型。想象一下你有帖子和页面，它们都是帖子类型，现在你可以制作一个电影帖子类型，如果你做电影评论或者汽车帖子类型，如果你卖汽车或者你是一个汽车经销商，或者其他什么，一个播客帖子类型，我们可以制作这个节目的播客。一旦你注册了文章类型，WordPress 会自动添加所有管理界面的东西，比如菜单，管理，你可以点击添加新电影，为电影创建新内容，它完全独立于你的文章。而在此之前，你可能会欺骗 WordPress，让它拥有一个播客类别，该类别中的任何内容都会被稍微不同地对待。你不必再那样做了；你实际上可以为你的播客制作一个帖子类型，并有一个完整的部分专门用于它，它不会弄乱你的博客帖子或类似的东西。因此，它真的有点像是将 WordPress 移入真正的 CMS 类型平台，这是朝着正确方向迈出的重要一步。
