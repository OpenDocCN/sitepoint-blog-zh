# 开源商业化 Redux

> 原文:[https://www . site point . com/open-source-commercialization-redux/](https://www.sitepoint.com/open-source-commercialization-redux/)

我们之前已经在开源上讨论过很多次 Linux 商业化的利弊。这包括建立一个服务和支持系统，类似于开源的专有表兄弟(商业世界已经习惯了)，引入基准评级(已经是一个增长的运动)和验证代码的方法，以防止合法和恶意的法律行动。

很明显，这是过去几年里的一场运动，现在几个倡议正在走向成功，不管是好是坏。总的来说，付费支持已经可以通过几家 Linux 发行版制造商获得，然而，越来越多的新兴公司基于付费模式成功浮出水面。

事实上，在今天的帖子中，引领新潮流的是一个风险投资基金的出现，该基金专门致力于在未来几年推出六家或更多的开源公司。看到一项业务投入超过 1000 万美元来指导新的开源产品进入商业组织，对于从事非专有项目的新晋企业家来说是一个巨大的推动。

这肯定会鼓励那些有项目的人不要仅仅在社区网站上分享项目，或许还可以试着看看一些商业投资的融资时机是否成熟。

同样，[黑鸭](http://www.blackducksoftware.com/)(我[在](https://www.sitepoint.com/blog/)之前报道过的一家公司)与[开源软件研究所](http://www.oss-institute.org/)合作，成为公司成员。这将为组织及其成员带来公司的法规遵从性和许可专业知识。

他们还将担任美国海军研发项目的技术顾问。对于开源和 Linux 平台来说，这又是一个重要的举措，进一步检验了平台在复杂环境中的能力。

为什么要结合 web 应用程序开发人员的思维来讨论大型企业问题？首先，我们所有在 SitePoint 工作的人都处于不同的开发阶段，有些小，有些大。第二，Linux 和开放源码解决方案在最高层变得越成熟，web 解决方案的提案在各个层面变得越合理。想想小型企业和他们对购买微软解决方案的看法。大多数人认为它(微软)是技术的水印标准。这没什么不好——这是一个大市场，每个人都有空间。然而。我们中的许多人都在以某种形式或方式兜售 Linux 和开源——这些小新闻为我们提供了更多弹药，表明该平台已经成熟并准备就绪。

随着我不断转向越来越大的项目和环境，我发现，你可能会惊讶地发现，你很快就会向我们讨论的一些公司寻求解决方案。即使在最小的公司，合规性问题也变得至关重要，我们对代码完整性、许可帮助和防止可能的法律诉讼的需求变得非常现实。

Linux 已经表明它可以通过社区模式取得成功。现在，当它确立自己作为主要参与者的地位时，它需要保留这个群体。
