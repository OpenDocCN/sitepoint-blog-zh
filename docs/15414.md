# 职业阿帕奇人

> 原文：<https://www.sitepoint.com/professional-apache/>

Apache server 无疑是当今互联网上最受欢迎的网站服务平台。虽然我会第一个说，看到开源运动的产品成为流行的选择，而不是流行的替代品，这是令人耳目一新的，但我只希望它不是如此难以使用！正如彼得·温赖特在本书中所展示的，事情并不一定如此。

Apache 的强大之处在于其无限的配置和扩展能力，可以满足任何需求。正如任何一个不得不在需要配置的情况下处理标准 Apache 安装的人会告诉你的那样，这也是它失败的原因。试图通过阅读在线参考资料来学习如何使用 Apache 就像试图通过阅读字典来学习说英语一样。在线参考只是 Apache 可以识别的所有命令和配置指令的参考。实际上，将大量的命令描述组合成程序的工作理解几乎是不可能的。难道没有什么手册可以帮助用户学习如何使用这个神奇的软件吗？嗯…现在有了。

用作者的话来说:“这本书旨在让您超越描述 Apache 指令列表的在线文档和书籍，并为您提供控制 web 服务器所需的信息。”在这个目标下，这本书取得了巨大的成功。

[![Get Your Copy of Kevin Yanks Book NOW!](img/b0efd5d43fb184b59453aa181c0a55de.png)](https://www.sitepoint.com/books/?bookid=kevinyank) 这本书的前半部分讲述了基础知识:按照你需要的方式设置 Apache。它涵盖了从因特网及其协议的介绍(除了几个容易混淆的首字母缩写词(TCP/IP 不代表“传输通信协议”, UDP 不是“Unigram 数据协议”)到服务器端包含(SSI)的配置和交付动态内容的脚本语言的所有内容。一路上，Apache 的所有主要配置指令都用丰富的例子进行了说明。

本书的后半部分讨论了性能调优和服务器安全性这两个非常大的主题。与其他书籍和在线教程中对这些主题的敷衍处理形成鲜明对比的是，作者深入探讨了一些问题，如调整 Apache 的性能、设置代理服务器和缓存、提高集群服务器的容错能力、监控 Apache 的活动日志以及设置 SSL 服务器(我一直在寻找这方面的好指南！).

为了检验这本书，我决定尝试在我的网络服务器上建立一个虚拟主机——直到现在，我还不知道该怎么做。嗯，这本书不仅向我详细介绍了设置基于名称的虚拟主机的过程，而且还向我提供了几个备选解决方案，如果我打算使用更强大的解决方案，即多个服务器实例和基于 IP 的虚拟主机，我可能会考虑这些解决方案。对动态虚拟主机的讨论结束了这一章，让我大吃一惊！

虽然你可能认为 49.99 美元的封面价格有点贵，但这本书所释放的巨大力量远不止这些。无论您是每天使用 Apache 服务器，还是第一次安装它，“专业 Apache”都是您不可或缺的工具。
