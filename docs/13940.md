# ESPN 要归巢了吗？

> 原文:[https://www.sitepoint.com/is-espn-returning-to-the-nest/](https://www.sitepoint.com/is-espn-returning-to-the-nest/)

回想两年半前——互联网时代的一代人。基于表格的布局仍然是王者，但是有很多关于 CSS 的讨论。有什么优势？桌子有什么问题？它能被认真考虑用于现实世界的布局吗？

ESPN 大步走进了这片风景——一个闪亮的、具有商业头脑的世界品牌，拥有大量不同的受众。他们做了数学计算，并决定 CSS 布局是去。正如迈克·戴维森(因 [sIFR](http://www.mikeindustries.com/blog/archive/2004/08/sifr) 而出名)当时解释的那样，[新的 ESPN 将会更快、更轻并且向前兼容](http://devedge-temp.mozilla.org/viewsource/2003/espn-interview/01/index_en.html)。

这在当时是一个分水岭。当然，许多以技术为导向的网站已经表明这是可能的，但这可能是第一次这样一个迎合“Joe Sixpack”观众的核心在线玩家说桌子不利于商业。人们真的注意到了，ESPN 成为了网络标准运动的代表人物。

那里发生了什么？

![The de-styled ESPN website.](../Images/47209923489c9fa30d17d87efb1ea9f9.png)

在这里，我使用了 Web Developer Toolbar 去样式化并在普通页面的表格上放置边框。正如 Matt Grover([Resource Interactive](http://www.resource.com/default.php))向我指出的，它大量使用了布局表格——实际上是 4 层嵌套。

现在，我不是在这里指手画脚或啧啧啧。他们是一家大企业，每天都做出他们认为合适的决定。然而，我想听听理由。

显然迈克很久以前就离开了，所以这不太可能是他的方向。

难道是:

*   旨在安抚网景 4/IE4/Opera 2/Mosaic 用户抱怨的新政策？
*   CSS 布局不再像以前那样重要了？
*   他们发现很难雇用精通 CSS 的开发人员？
*   没有桌子，他们无法完成他们的设计想法
*   ESPN 的开发者现在可以随心所欲地以他们认为合适的方式制作每一页了吗？
*   违背了官方的网站理念，但是由不了解更好的开发人员编码，并且没有被链上的任何人采用？

看看 web.archive.org，这似乎是随着时间的推移而悄悄发生的事情。我猜这是自然的员工流动带来的，但我当然有兴趣听听“知情人士”的意见。
