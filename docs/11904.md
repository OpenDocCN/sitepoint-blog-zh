这篇文章写于 2009 年，至今仍是我们最受欢迎的帖子之一。如果你热衷于学习更多关于 PHP 和 MySQL 的知识，你可能会发现这个[最近关于 PHP](https://www.sitepoint.com/php-security-cross-site-scripting-attacks-xss/) 的文章集很有意思。

## PHP 和 MySQL 都变了。

早在 2001 年，当我写这本书的第一版时，读者惊讶地发现，你可以创建一个充满网页的网站，而不必为每个网页编写单独的 HTML 文件。PHP 从众多编程语言中脱颖而出，主要是因为它对几乎任何人来说都很容易学习，并且可以免费下载和安装。同样，MySQL 数据库为一个问题提供了一个简单而免费的解决方案，在此之前，这个问题只能由有公司预算的专业程序员来解决。

那时候，PHP 和 MySQL 很特别——见鬼，它们简直是奇迹！但多年来，他们赢得了大量快速发展的竞争。在一个任何拥有免费 WordPress 账户的人都可以在 30 秒内建立一个功能齐全的博客的时代，对于像 PHP 这样简单易学的编程语言来说已经不够了；像 MySQL 这样的数据库免费也是不够的。

事实上，当你坐下来阅读这本书的时候，你的雄心可能已经超出了你使用免费的网络点击工具所能实现的范围。你甚至可能会考虑自己开发一个令人兴奋的新点击工具。毕竟，WordPress 是使用 PHP 和 MySQL 构建的，所以为什么要把你的视野限制在其他方面呢？

为了跟上竞争，以及更高要求项目的需求，PHP 和 MySQL 不得不不断发展。与 2001 年相比，PHP 现在是一种更加复杂和强大的语言，而 MySQL 是一种更加复杂和强大的数据库。今天学习 PHP 和 MySQL 打开了许多大门，这些大门对于 2001 年的 PHP 和 MySQL 专家来说是紧闭的。

这是好消息。坏消息是，就像黄油刀比瑞士军刀更容易辨认一样(而且不太可能造成自伤！)，所有这些令人眼花缭乱的新特性和改进无可争议地增加了 PHP 和 MySQL 初学者的学习难度。

更糟糕的是，PHP 已经完全放弃了几个初学者友好的特性，这些特性在 2001 年给了它竞争优势，因为它们被证明是过于简化的，或者可能导致没有经验的程序员构建具有巨大安全漏洞的网站。如果你是一本关于 PHP 和 MySQL 的初学者书籍的作者，这是一个问题。

PHP 和 MySQL 已经发生了变化，这些变化使得写这本书变得更加困难。但是他们也让这本书变得更加重要。路径越曲折，地图越有价值，对吗？

在本书中，我将为您提供使用 PHP 和 MySQL 构建数据库驱动的网站的实际操作。如果你的网络主机提供 PHP 和 MySQL 支持，你的情况就很好。如果没有，我会告诉你如何在 Windows、Mac OS X 和 Linux 电脑上安装它们，所以不要担心。

这本书是你通向每个初学者今天学习 PHP 和 MySQL 必经之路的地图。抓起你最喜欢的拐杖；让我们去远足吧！
