# 灵活的设计都去哪了？

> 原文：<https://www.sitepoint.com/flexible-designs-dying-out/>

有没有人注意到灵活/流动和有弹性的网站设计越来越少了？虽然无法确定数字，但灵活设计似乎不像几年前那样受欢迎或普遍。

这是一种不寻常的情况，因为灵活的布局利用了 web 作为设计媒介的优势:

*   灵活的站点可以在多种分辨率下工作。没有必要对用户屏幕尺寸进行激烈的辩论或研究。况且屏幕分辨率统计是个神话；并不是每个人都全屏运行浏览器，很多人都有工具栏、侧边栏或其他占用宝贵屏幕空间的小部件。
*   用户——而不是设计者——决定了浏览器窗口的尺寸。灵活的设计把控制权还给用户，不需要水平滚动。
*   iPhone 等手机和游戏机正在成为网络浏览的可行替代品。一般来说，这些设备的分辨率较小，可以受益于灵活的网页设计。
*   CSS `min-width`和`max-width`允许设计师实现弹性布局(SitePoint.com 就是一个很好的例子)。灵活性受到限制，以确保列不会变得太宽而不可读，或者太小而被隐藏。如果需要完全支持浏览器，IE6 也有变通方法。
*   一个灵活的布局可以用一两个 CSS `width`属性切换到固定布局。从固定转换到灵活可能会带来更大的重新编码工作量。

然而，为灵活性而设计更加困难。许多设计师制作固定宽度的网站，因为它们在图形包中更容易理解和模拟。高度图形化设计的趋势也会导致布局需要像素级的精确定位。

或者也许是经济问题？灵活的设计需要更长的时间来编码和测试，尤其是考虑到今天可用的浏览器的数量。感受到的用户利益可能不值得更长的生产时间和更高的成本？

或者也许我错了？是否有很多灵活的网站设计，但我错过了它们？

流体设计正在消亡吗？你的客户喜欢固定宽度的网站吗？灵活性给你的用户带来真正的好处了吗？
