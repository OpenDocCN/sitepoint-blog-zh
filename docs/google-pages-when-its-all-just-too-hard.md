# 谷歌页面创建者:当一切都太难的时候

> 原文：<https://www.sitepoint.com/google-pages-when-its-all-just-too-hard/>

厌倦了为你的家人和朋友建立快速和肮脏的网站，这样他们就不会产生自己的首页怪物？告诉他们试试 Google Page Creator。

页面创建者 big 的最新“测试版”web 应用程序使普通人也能通过 CSS 布局和(大部分)语义标记制作页面。

当然，Google 从来都不是符合标准的 Web 开发方法的拥护者，Page Creator 也不例外。丑陋的地方:

*   它会生成`<font>`标签(哎呀，老的呕吐反射又来了……)。
*   它使用`<b>`和`<i>`代替语义替代。
*   它非法地把`<div>`放在像`<h1>`这样的标签里面。
*   它不把`alt`描述放在图片上(不是通过*默认*，而是根本不放！).

从好的方面来看，它不使用表格进行布局，它允许用户在浏览器中用大量的“皮肤”(CSS 样式表)创建一列、两列和三列布局，具有非常可用的 AJAX 风格的界面。

作为一个关心标准的开发人员，最让我沮丧的是，上面所有丑陋的部分似乎都是谷歌有意识决策的结果，而不是简单的无知。谷歌*本可以*轻松打造出没有上述任何缺点的产品，但他们没有。

Google 正在积极征求对 Page Creator“测试版”的反馈，所以如果标准对你很重要，[让他们知道吧！](http://www.google.com/support/pages/bin/request.py)
