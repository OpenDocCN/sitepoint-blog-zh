# 10 个简单的 WordPress 安全提示

> 原文：<https://www.sitepoint.com/10-easy-wordpress-security-tips/>

![WordPress security](img/f0306b6ad1da2efe6b394371c53917ba.png)

*本文是与 [SiteGround](https://www.siteground.com/go/article-sp) 合作创作的系列文章的一部分。感谢您对使 SitePoint 成为可能的合作伙伴的支持。*

WordPress 并不是天生不安全的，开发者努力确保漏洞被迅速修补。不幸的是，WordPress 的成功让它成为了众矢之的:如果你能破坏一个 WordPress 的安装，数百万个网站可能会向你开放。即使 WordPress 是安全的，也不是所有的主题和插件都是以同样的谨慎开发的。

一些人会攻击 WordPress 来进行挑战或者恶意破坏。这些很容易发现。最坏的罪犯将链接偷偷放入您的内容中，将钓鱼网站放入您的文件夹结构深处，或者使用您的服务器发送垃圾邮件。一旦您的安装被破解，可能需要删除所有内容并从头重新安装。

幸运的是，有一系列简单的方法可以提高安全性。以下安全修复都不会超过几分钟。

## 1.切换到 HTTPS

HTTPS 防止第三方监听或修改客户端和服务器之间的通信的中间人攻击。理想情况下，你应该在安装 WordPress 之前激活 HTTPS，但是如果你以后添加它，也可以更新 WordPress 的设置。

HTTPS 也可以提高你的谷歌页面排名。像 SiteGround 这样的主机提供免费的 SSL 证书，你可以获得高达 65%的折扣。

## 2.限制 MySQL 连接地址

确保您的 MySQL 数据库拒绝外部人员和系统连接到您的本地服务器。默认情况下，大多数托管 web 主机都这样做，但是那些使用专用服务器的主机可以将下面一行添加到 MySQL **my.cnf** 配置文件的`[mysqld]`部分:

```
bind-address = 127.0.0.1
```

## 3.使用强数据库凭据

在安装 WordPress 之前，[创建 MySQL 数据库](https://www.sitepoint.com/how-to-create-wordpress-mysql-databases-on-cpanel/)时，使用一个强的、随机生成的数据库用户 ID 和密码。这些凭证在 WordPress 安装过程中使用一次来连接到数据库——你不需要记住它们。您还应该输入一个不同于默认的 **wp_** 的表前缀。

用户 ID 和密码可以在安装后更改，但是记得相应地更新 wp-config.php 的配置文件。

## 4.使用强管理员帐户凭据

同样，对安装过程中创建的管理员帐户使用强 ID 和密码。任何使用 ID **admin** 和密码**密码**的人都应该被黑。考虑为日常编辑任务创建另一个权限较少的帐户。

## 5.移动或保护 wp-config.php

**wp-config.php**包含你的数据库访问凭证和其他有用的信息，供企图闯入你系统的人使用。大多数人把它放在 WordPress 主文件夹中，但是它可以移到上面的文件夹中。在许多情况下，该文件夹位于 web 服务器根目录之外，HTTP 请求无法访问。

或者，您可以通过配置您的 web 服务器(如 Apache **)来保护它。htaccess** 文件:

```
order allow,deny
deny from all
```

## 6.授予用户尽可能低的角色

用户是任何系统中最薄弱的一点——尤其是当他们可以选择自己的弱密码，并愉快地将凭证传递给任何询问的人时！很少人需要管理权限。WordPress 提供了一系列的角色和功能。在大多数情况下，用户应该是:

*   一个**编辑**:可以发布和管理自己和他人帖子的人
*   作者:可以发布和管理自己帖子的人，或者
*   a **投稿人**:可以撰写和管理自己的帖子但不能发布的人。

这些角色都不能配置 WordPress 或者安装插件。

## 7.通过 IP 地址限制访问

如果您有几个带有静态 IP 地址的编辑器，您可以通过添加另一个**来限制访问。htaccess** 文件到 **wp-admin** 文件夹:

```
order deny, allow
allow from 1.2.3.4 # user 1 IP
allow from 5.6.7.8 # user 2 IP, etc
deny from all
```

## 8.隐藏 WordPress 版本号

WordPress 的一些版本有已知的漏洞。任何人都很容易发现你使用的是哪个版本，因为它显示在每个页面的 HTML `<head>`中。通过在主题的 functions.php 文件中添加以下行来删除这些信息:

```
remove_action('wp_head', 'wp_generator');
```

## 9.明智地选择第三方插件和主题

WordPress 插件和主题拥有用户梦寐以求的力量！一个糟糕的插件可能会影响性能，泄露私人数据或授予另一种访问方法。除非绝对必要，否则避免安装代码。在进行实时安装之前，验证插件的真实性并在本地服务器上进行测试。

## 10.定期更新 WordPress 和插件

WordPress 会自我更新，但是主要版本需要一次点击激活过程。当然是在备份了你的数据库和文件之后。同样，定期检查主题和插件的更新。

在更新实时系统之前，风险承担者应该在复制的测试服务器上检查更新。也就是说，WordPress 的更新过程和向后兼容性很少引起问题。
