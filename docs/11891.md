# GMail 终于完成了吗？

> 原文:[https://www.sitepoint.com/gmail-beta-over/](https://www.sitepoint.com/gmail-beta-over/)

谷歌素有从不完成项目的名声。GMail 已经存在五年了，但还没有失去它的“测试版”标签。Docs、Calendar 和许多其他谷歌服务仍然是测试产品，尽管自最初发布以来已经发生了巨大的变化。

该公司对无休止的测试周期的理由从未得到明确解释，谷歌一直坚持一旦产品准备就绪，标签就会被移除。我怀疑谷歌测试版标签的动机是以下几个方面的结合:

*   它提醒用户 web 应用程序是实验性的，正在开发中。
*   它降低了用户对早期网络系统的期望，给了我们一个惊喜，让我们找到了功能齐全的在线产品。
*   当应用于 web 应用程序和服务时，版本号很少有意义。与收缩包装的磁盘分布式软件不同，web 解决方案可以每天进行增量改进。除了开发人员，很少有人会关心它是 7 版还是 57 版。

然而，谷歌最终认识到,“测试版”标签在商界有不同的含义，这可能会损害他们的前景。许多商业用户对此不感兴趣，因为他们将这个术语与不完整或未经测试的软件联系在一起。

谷歌的其他产品可能会效仿 Chrome 的做法。这款网络浏览器的测试期只有 100 天，尽管它在 2008 年 9 月才发布，但现在已经是第二版了。谷歌的测试习惯会结束吗？

谷歌对测试标签的坚持有没有让你的客户感到困惑或厌烦？
