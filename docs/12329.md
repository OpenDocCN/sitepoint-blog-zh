# IE8 足以拯救 IE 浏览器吗？

> 原文:[https://www . site point . com/will-ie8-be-full-to-save-internet-explorer/](https://www.sitepoint.com/will-ie8-be-enough-to-save-internet-explorer/)

谈论 Internet Explorer 的消亡可能还为时过早——毕竟，三分之二的网络用户仍在使用某种版本的网络浏览器。但这比四年前的 91%有所下降，那时 Firefox 还没有被我父母这样的人使用。[据网络应用](http://marketshare.hitslink.com/browser-market-share.aspx?qprid=1&qpdt=1&qpct=4&qptimeframe=M&qpsp=97&qpnp=25)报道，Internet Explorer 已经连续 6 个月失去市场份额，并且已经两年没有超过 80%。

原因？IE 在多条战线上面临激烈的竞争。在过去的 6 个月里(Chrome 是 5 个月)，排名第二到第四的浏览器都获得了市场份额。Firefox 现在稳稳地超过了 20%，Safari 正在稳步增长，并趋向于 10%，谷歌的新 Chrome 浏览器已经连续两个月超过 1%的市场份额。即使是排名第五的浏览器 Opera(它在欧洲比在全球其他地方拥有更多的追随者)，在过去的两年里也有了相当大的增长。

与此同时，IE 继续下滑。上周，微软发布了 IE8 的第一个候选版本，这意味着他们无处不在的浏览器的下一个版本很可能就在眼前。这对微软来说是潜在的成败时刻——不仅仅是 IE 的存在。

Internet Explorer，以及微软丰富的互联网应用程序和 web 平台技术，如 Silverlight、Azure 和 Live Mesh，对公司未来的成功至关重要。谷歌制定了引人注目的网络操作系统三管齐下的战略，其中包括 Chrome 浏览器。谷歌——以及 Mozilla 和 Adobe 等其他公司——正试图为基于云的应用创建一个平台，如果成功，这将意味着底层的计算机操作系统对大多数用户来说将变得基本无关紧要。

这将给微软带来灾难，微软的大部分收入来自 Windows，除非他们能与自己的网络操作系统竞争。如果他们遵循谷歌的模式，微软的网络操作系统可能会主要围绕 Internet Explorer (Chrome)、Silverlight(JavaScript/Native Client)、Azure(应用引擎)和。NET (Python)。诚然，在微软赚取大部分利润的企业环境中，Web 操作系统的可行性还有很长的路要走，但微软仍有理由担心 ie 浏览器市场份额的暴跌。

我们注意到 SitePoint 的访问者也有类似的趋势。基于最近三年的浏览器趋势，[我们预测 IE 将在 2013 年](https://www.sitepoint.com/internet-explorer-extinct-by-2013/)灭绝。当然，这是假设下降速度不会减慢——这是不太可能的。我们在 8 月份也问过 SitePoint 的读者，他们是否会考虑转用 IE8 [65%的人说不](https://www.sitepoint.com/would-you-switch-to-ie8/)。如果我们可以推测，至少有一部分赞成票是计划升级的当前 IE 用户，这对微软来说是一个非常坏的消息——没有多少人从 IE 转而计划转回。

看起来 IE8 真的不足以拯救微软。即使它明显优于之前版本的 ie 浏览器，它也不会比 Firefox、Chrome 或 Safari 好到足以说服叛逃者改回来。微软最大的希望可能是阻止背叛的浪潮，并在浏览器市场保持微弱的多数——但在我们看来，这似乎也不太可能。

微软可以采取的拯救 ie 浏览器的一个激进措施是开源。目前蚕食其市场份额的三种浏览器都是开源的或者基于开源项目，这可能不是巧合。
