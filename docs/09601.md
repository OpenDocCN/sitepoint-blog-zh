# 面向移动设备的设计——付诸实践

> 原文：<https://www.sitepoint.com/design-for-mobile-putting-it-into-practice/>

我们已经讨论了在为移动设备设计时的一些更广泛的考虑，所以现在让我们来讨论一下我们的应用程序的业务。首先，我们需要一个计划。我们的客户给了我们一些高级的用户场景，我们需要更详细地检查这些场景，以弄清楚每个场景对我们的应用程序意味着什么。准确地决定我们打算向谁提供哪些特性是至关重要的。只有在我们搞清楚这一点之后，我们才能确保我们的应用程序的外观和感觉能够真正实现我们想要的任务。那么我们该如何着手呢？

### 胸怀大志

为了形成完美的特性集，我们需要考虑我们的用户*可能*想要的所有特性。如果我们不检查我们可能想要包含的所有东西，我们几乎肯定会错过一些可能成就或破坏我们应用程序的想法。不要介意列表是否很长；最好从宏大的可能性开始，以后再缩小我们的范围。StarTrackr 是一个名人识别应用程序，因此我们的综合功能列表可能包括:

*   按位置查找目击事件
*   按名人查找目击事件
*   按流派对名人进行排序
*   搜索特定的名人
*   通过地址找到附近名人出现过的地点
*   在 GPS 上找到名人出现过的附近地点
*   喜爱/追随名人
*   收藏/追踪一个位置
*   当某个特定的名人出现时得到通知
*   当某个特定地点有名人出现时得到通知
*   查看最近的目击事件
*   添加一个名人景点
*   添加一张名人照片
*   添加名人目击视频

### 整理用户资料

正如我们前面谈到的，用户是我们应用程序规划中最重要的考虑因素。他们是谁？我们对他们了解多少？他们在做什么？他们想要什么？对他们来说最重要的特征是什么？他们为什么想要使用我们的应用程序？他们什么时候会使用我们的应用程序？他们可能对什么感兴趣，不太可能对什么感兴趣？这样的问题并不总是容易回答的，但是如果用户对 StarTrackr 的一般概念感兴趣，我们可以对它们做一些假设。我们知道他们:

*   比如名人
*   对名人八卦感兴趣
*   更可能是女性而不是男性
*   更有可能是年轻人而不是老年人(比如 14 到 25 岁)
*   可能位于一些名人热点附近，如好莱坞或伦敦

在现实世界中，我们的客户通常会向我们提供他们当前用户是谁以及他们如何使用网站的信息。出于我们的目的，我们将做一个有根据的猜测:我们的典型用户是生活在大城市的年轻女性，喜欢名人八卦。

### 决定核心功能集

从简单开始。优秀的移动应用程序主要关注用户想要完成的任务。事实上，我们应该对所有的项目都这样做——不管是不是移动的——但是对移动 web 应用程序来说更重要。在我们的应用程序中包含更多的特性是很诱人的，认为这样会使它变得更好。事实上，在谈论移动设计时，情况并不总是这样。

我们有一个可以包含的功能列表，我们知道谁是我们的用户，他们想要什么。接下来，我们需要将我们的特性集削减到最基本的元素。这看起来很简单，但是决定哪些特性应该被忽略却是一个挑战。考虑最小可行产品:对于我们的应用程序来说，哪些特性是必不可少的。如果我们残酷地将我们的功能列表削减到最基本的，我们可以专注于三个主要特性:

*   按位置查找目击事件
*   按名人查找目击事件
*   添加一个名人景点

其次是:

*   添加一张名人照片

至少对于第一次迭代，这给了我们所需要的一切:名人与位置相关联，我们能够过滤这些数据类型中的任何一种——此外，我们的用户可以在旅途中添加他们自己的景点。查看这些信息并将其浓缩成我们应用程序的一个单一的使命陈述是很有用的；然后，我们可以在整个开发过程中返回到它，以确保我们不偏离轨道。对于我们的应用程序，这可能是:“一个让人们找到并添加他们最喜欢的名人的照片的应用程序。”
