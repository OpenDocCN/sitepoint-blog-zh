# 开始 PERL

> 原文：<https://www.sitepoint.com/beginning-perl/>

Perl 最初于 1987 年作为一种脚本语言发布，作为使 UNIX 程序协同工作的媒介，自此以后，Perl 已经成为几乎任何一种成熟的编译语言都无法胜任的情况下的首选语言。自动执行 UNIX 系统维护任务、为数据库驱动的网站提供动力，以及介于两者之间的任何事情，在当今的计算机世界中，Perl 几乎无处不在。

除了是现存最通用的语言之一之外，Perl 也是最令人困惑的语言之一。在《Perl 入门》一书中，Simon Cozens 认为，Perl 唯一令人困惑的地方是它采用了一种“自然语言”的编程方法——对于任何熟悉其他 C 风格语言(如 C/C++、Java 或 JavaScript)的人来说，这种方法都是陌生的。他还指出，出于某种原因，Perl 程序员倾向于试图找到最短、最有效的方式来表达他们程序中的一切——这在许多情况下导致了相当晦涩的代码。

本书没有回避 Perl 的这些方面，而是正面解决了它们。通过停下来解释 Perl 和其他流行语言之间的主要差异，并解释常用的编码技巧，它提供了一个完整而深入的语言描述，既适合编程新手，也适合经验丰富的老手。

“Perl 入门”以介绍性章节开始，描述了该语言的起源及其吸引人的特性，并提供了在基于 UNIX 和基于 Windows 的平台上下载和安装该语言的完整指南。准备工作结束后，Perl 语言在第一章开始时被暂时搁置一边，读者将从总体上了解编程的描述。如果你以前从未写过程序，这一章应该是这本书的主要卖点之一。有经验的程序员可能会喜欢作者巧妙的例子和类比，但是可以安全地浏览一下如何创建和运行一个简单的 Perl 程序的描述。

[![Get Your Copy of Kevin Yanks Book NOW!](img/b0efd5d43fb184b59453aa181c0a55de.png)](https://www.sitepoint.com/books/?bookid=kevinyank) 接下来的三章涵盖了基本的语言特性:数据类型、变量、操作符和控制流。这必然是学习任何语言中最不投入的阶段，但作者的对话语气成功地保持了节奏。熟悉其他语言的读者可能会倾向于略读本书的这一部分，但最好仔细阅读。Perl 的一些特性看起来像是其他语言中的类似结构，实际上包含了本书所强调的细微差别，并且对后面章节中的例子至关重要。

有了基础知识，第 5 章到第 8 章介绍并解释了高级语言特性:强大而灵活的文本操作的正则表达式，文件和数据处理工具，处理复杂数据结构的变量引用，以及将程序组织成可重用代码块的子程序。这些章节的质量都很高，没有遗漏细节，因为它们涵盖了使 Perl 成为适合许多不同任务的强大而优雅的语言的特性。

第 9 章值得特别一提，因为它标志着 Perl 语言旋风之旅的暂停，而是把重点放在如何让 Perl 程序员的生活更轻松上。本章提供了使用您可以使用的工具和 Perl 语言本身的特性来纠正常见错误消息的描述和建议。对于初学者来说，特别有价值的是 Perl 错误消息和常见原因的列表(以及解决方案！).

在涵盖高级语言特性(特别是模块和面向对象编程)的几个章节之后，本书介绍了 Perl 的两个最常见的应用:CGI 编程和数据库操作。进入高速档，这本书提供了一个完整的(但简洁的)概述，如创建动态网站，填补 CGI 脚本中的安全漏洞，安装关系数据库(特别是 MySQL)，并从 Perl 程序中访问该数据库等主题。诚然，这些章节在如此有限的空间(仅 100 多页)内传授的知识量令人印象深刻，但由于涵盖了所有材料，因此没有空间来进行贯穿本书前几章的友好握手。如果你熟悉服务器端脚本和数据库的概念，那么这些章节将提供一个很好的指南，告诉你如何在 Perl 中完成这些事情。然而，初学者如果想学习相关的概念，最好是拿起一本单独的书。

这本书的最后一章“Perl 的世界”由一系列 Perl 高级应用的快照组成。像上面两章一样，这是高级用户原始信息的集中来源。如果你是一个初学者，不要指望只使用这里介绍的一些知识就能学会如何编写图形用户界面或加密安全系统。

简而言之，这本书是学习 Perl 语言的好方法。它的非正式和有趣的风格不仅为编程新手提供了诱人的体验，而且作者精心指出了 Perl 和其他流行语言之间的主要区别，这也使它成为熟练程序员的优秀入门。然而，重要的是，当你读这本书时，不要把你的期望定得太高。Perl 的大多数实际应用超出了本书的范围，需要读者多做一些研究。
