## 流畅的浏览

根据营销宣传，Fire 将运行 Silk——一种全新的网络浏览器，既可以在平板电脑上运行，也可以在云中运行:

> Kindle Fire 引入了一种全新的模式——一种“拆分浏览器”架构，通过使用亚马逊网络服务云的计算速度和能力来加速移动设备硬件的功能。结果是更快的网络浏览体验。
> 
> 大多数繁重的工作都从您设备上的处理器转移到了我们强大的 AWS 服务器上。访问这种闪电般快速的 CPU、扩展的内存和巨大的网络连接允许 Amazon Silk 的性能超越您的本地设备的能力。

本质上:

1.  Silk 是一个定制的 webkit 浏览器。
2.  页面请求通过 Amazon 代理服务器路由。
3.  代理将为 Fire 优化内容。资源将被缓存在 EC2 服务器上，因此 DNS 查找和文件下载被最小化。此外，可以调整大图像的大小以适合设备。
4.  亚马逊监控使用模式。例如，如果它注意到很大一部分 SitePoint 用户访问了 [HTML5 开发中心](https://www.sitepoint.com/welcoming-the-html5-dev-center/)，它会将该页面预推至设备，以便可以立即呈现。
5.  浏览器支持 Flash — *不支持简直是疯了！*

[观看 YouTube 上的丝绸介绍视频……](http://youtu.be/_u7F_56WhHk)

亚马逊将 Silk 称为“革命性云加速网络浏览器”。

很抱歉扫了大家的兴，但是 Opera 这么做已经很多年了——远在“云计算”这个术语被发明之前。Opera 服务器预渲染页面，并将优化的字节码推送到 Opera Mini。这就是为什么它能在普通手机上运行*(也是为什么苹果允许它在他们的设备上运行，而他们通常不允许其他浏览器运行)*。

Opera 的其他版本也支持 [Turbo](http://www.opera.com/browser/turbo/) ，如果你的网速很慢，它会压缩网页和图片。

虽然丝绸可能不是革命性的，但它确实很有趣。假设没有安全或隐私问题，它应该提供更快的浏览体验，同时允许客户端 HTML5、CSS3 和 JavaScript。

## 是时候点燃你的 iPad 了吗？

几篇文章报道亚马逊已经赢得了平板电脑战争。

我不相信。

像标准的 Kindle 一样，Fire 不是一台通用计算机。这是一款消费设备，面向那些想从亚马逊应用商店购买电子书、音乐、视频和游戏的人。

iPad 功能更强大。它的价格可能是原来的三倍，但苹果的顾客并不以价格敏感著称。此外，还有更便宜的安卓平板电脑。

我怀疑亚马逊会削弱苹果的销售，但我们不要低估他们的营销影响力。Kindle Fire 非常适合那些想要彩色屏幕和更强计算能力的电子书阅读器的人。Fire 也吸引了那些想要平板电脑但无法证明苹果价格溢价合理性的人。

假设没有技术或销售障碍，Kindle Fire 几乎肯定会成功。平板电脑市场将因此而增长。

## Web 开发人员应该如何准备？

你在测试你的电子墨水 Kindles 网站吗？我表示怀疑。目前的设备使用刷新缓慢的黑白实验浏览器。

然而，Fire 和类似的小型平板设备将在未来 1-2 年内变得越来越重要。幸运的是，web 开发技巧和技术今天已经存在…

*   在定义 JavaScript 事件处理程序时考虑触摸设备
*   确保你的网站可以在 Chrome 和 Safari 等其他 webkit 浏览器中运行
*   尝试响应式布局和媒体查询
*   考虑采用移动优先的设计方法*(敬请关注 SitePoint，稍后将有更多文章)*
*   在 1024 x 600 以及更重要的 600 x 1024 的分辨率下测试您的设计。
