# 2013 年 10 大网络预测

> 原文:[https://www.sitepoint.com/2013-web-predictions/](https://www.sitepoint.com/2013-web-predictions/)

新年快乐 2013 年已经到来，虽然我们可能没有悬浮滑板或月球一日游，但技术和网络的发展速度比以往任何时候都快。我对 2012 年的预测与[有点偏差](https://www.sitepoint.com/2012-web-prediction-results/)，但没关系。灵魂在和我交流。啤酒和葡萄酒也是…

## 1.浏览器使用模式将保持稳定

你关心访问者使用 Chrome，IE，Firefox，Safari 还是 Opera 吗？你不应该。我们已经达到了一个点，五大浏览器之间几乎没有区别。它们都支持核心的 HTML5 技术，都很快，都很稳定。为什么要换？

不幸的是，这并没有阻止开发人员做一些愚蠢的事情，比如优先使用 webkit 供应商前缀。如果你发现自己发布了*“这个网站最适合…*的消息，控制住自己，修改你的代码。浏览器兼容性比以往任何时候都好。

## 2.浏览器的选择将变得更加有限

不幸的是，选择你喜欢的浏览器已经成为过去。供应商们正赶着 iOS 的潮流，规定你可以在你的设备上安装什么:

*   苹果强迫你在 iPhone 和 iPad 上使用 Safari。iOS 版 Chrome 是 Safari 皮肤。Opera 相当于功能手机上使用的 Mini 版本；它不会直接在设备上运行 JavaScript。
*   微软在 Windows RT 和 Windows Phone 上授予 IE 更高的操作系统权限。其他供应商无法在同一个竞技场上竞争；竞争将会很困难。
*   你可以猜猜 Chrome OS 和 Firefox OS 都提供了哪些浏览器。

## 3.IE11(或 10.1)将会发布

微软历史上的两年浏览器发布时间表一直是个笑话，但它不再好笑。IE10 是第一个实现自动浏览器更新的版本，所以我希望公司有计划使用它。让我们也期待 IE10+的黄金版在几个月内登陆 Windows 7；我们不能等待世界升级到 Windows 8…

## 4.一个重要的 Windows 8 更新将会到来

说到这一点，SP1 将带回许多微软从操作系统中剥离出来的商用电脑友好特性。我不会惊讶地看到对手势废话的依赖减少了，一个更小的地铁/不管它叫什么名字的屏幕看起来可疑地像一个开始按钮。

## 5.Node.js 将成为一项颠覆性的技术

这是一场赌博。诸如 Ruby 和 Python 之类的服务器语言可能在开发人员中很流行，但是与 PHP 相比，它们是小众技术。NET 甚至 Java。Node.js 也是如此，但它有几个优势:

1.  它使用了 web 开发人员已经理解的 JavaScript。
2.  开发工具、站点和资源正在迅速变得可用。
3.  它很流行，并得到了一些大型行业参与者的支持。

虽然 Node.js 在商业应用中站稳脚跟还需要时间，但它在全球范围内取得成功的可能性要大得多。

## 6.2013 年是响应式网页设计年

好的，所以我认为 RWD 会在 2012 年成为主流技术，但是我预测 2013 年一些重要的网站会采用这些技术。对于设计师来说，从固定布局转向(更)流动的布局是一个很难理解的概念，但是工具正在变得可用。

RWD 可能并不完美，但它是让你的网站在各种设备上运行的一种划算的方式。如果你现在不为客户提供服务，做好失去一些业务的准备。

## 7.将决定响应的图像标准

RWD 的主要绊脚石之一是图像。虽然您可以根据任何屏幕尺寸缩放图像，但将较小的图像发送到较小的设备会更有效。或者更具体地说，在带宽有限的设备上使用较小的图像。

可以使用 CSS 媒体查询或 JavaScript 来实现响应式图像解决方案，但是基于浏览器的纯 HTML 选项会更有用。有几个提案有望在年底前汇聚成 W3C 规范草案。

也就是说，你真的需要那个占用 1Mb 带宽的图像吗？也许是时候找一个懂网络的设计师了！

## 8.触摸屏设备将走向成熟

触摸屏设备并不像你想象的那样普遍。你可能在任何地方都能看到他们，但那是因为你和科技行业的其他人在一起。2013 年，你的父母将会用一台新的平板电脑取代他们老旧的 Windows 98 电脑。

然而，我并不是在预言个人电脑的死亡。真正的键盘、鼠标和显示器对于任何从事实际工作的人来说都是必不可少的。但是你的老板需要一台 27 英寸的 iMac 来每天回复三封邮件吗？

## 9.原生应用与网络应用的争论将会继续

2012 年的一个观察让我目瞪口呆；公司被原生智能手机应用蒙蔽了双眼。愚蠢的假设包括:

1.  每个人都用智能手机做一切事情
    现代智能手机非常适合快速发送信息、拍照和娱乐，但这并不意味着人们会用它来写小说或计算年度税单。语境决定一切。
2.  开发和发布原生应用更容易
    编写原生应用并不比在 Windows、Mac OS 或 Linux 上编写应用更容易。如果说有什么不同的话，那就是由于有限的硬件资源、笨重的开发工具、更广泛的设备和目标操作系统，这变得更加困难。
3.  **原生应用很容易赚钱**
    真的吗？对于每一个蓬勃发展的本地应用，都有数百个赚不到钱的应用。以 Rovio 为例:他们开发了许多游戏，但在《愤怒的小鸟》和《T4》成功之前，他们濒临破产。

对于某些项目来说，网络应用可能是更好的解决方案，但是公司仍然想要本地应用。或许我们都应该转向目标 C，然后拿钱走人？
