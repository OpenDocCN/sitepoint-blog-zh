# Safari 通过了严格的测试

> 原文:[https://www.sitepoint.com/safari-passes-the-acid-test/](https://www.sitepoint.com/safari-passes-the-acid-test/)

吹响号角，放飞鸽子！

Safari 赢得了成为第一个通过浏览器合规性测试的浏览器的竞赛。干得好，戴夫·海厄特。

如果你错过了关于 Acid2 的演讲，以下是执行摘要。

从根本上说， [Acid2 测试](http://webstandards.org/act/acid2/)是一个页面，由[网络标准项目](http://webstandards.org/)设计并提供——但不是任何页面。

不，事实上，ACID2 正是你妈妈让你的浏览器远离的那种页面。它巧妙地将标准允许的各种新式 CSS 符号、PNG jiggery 和边缘 HTML 小部件一层层堆积起来。然后，为了更好地衡量，它堆积在一个无效的 CSS 的慷慨帮助，所有这一切都是精心设计的失败，失败“丑陋”。

要通过测试，浏览器首先需要有足够的知识来理解所有的页面符号。如果它得到了控制，那么它只需要能够处理错误，而不会在弹簧和螺栓的冰雹中爆炸。一个“通过”为你赢得了这个有点尴尬的笑脸。

![](../Images/b8ed3bf734d7982159a8fcacc81fab8b.png)

或许，Acid2 最大的优点是面部的每一条“扫描线”都是独立的测试。理论上，如果一个失败了，它不应该对其他测试有任何影响——更好的浏览器会呈现更好的笑脸。事实上，页面有一个极其详细的逐行分解，按顺序解释了每个测试。

那么，假设你不够幸运，不能使用最新的 Safari 版本，你最喜欢的浏览器会怎么样呢？

很高兴你问了。这是我之前准备的一些！

![](../Images/f7f4b27ccb938546792772b3b4fa26ad.png)

所以，说到底，这一切有什么好处呢？

我相信它有两个目的，最终都将使我们的工作更容易。

1)它为浏览器开发者树立了一个高度可见的灯塔。

2)它提供了一把尺子来衡量每个浏览器的竞争力。

在过去，浏览器开发者很容易耸耸肩说'*是的，好吧，也许我们不支持<feature a="">，但是，嘿，那边的那些家伙不支持<feature b="">。到底，谁是对的？</feature></feature>*’。Acid2 给了我们这个问题一个明确的答案。

当然，Acid2 并不是测试浏览器的唯一方法——嘿，它甚至可能不是最好的——但是只要它继续获得关注，它将是最重要的。

事实上，IE7 团队甚至可能不会公开承认 Acid2。但是我很确定这将会是一个非常激烈的内部讨论点。
