# 这一切都是为了谷歌

> 原文：<https://www.sitepoint.com/its-all-go-for-google/>

自 20 世纪 90 年代中期以来，web 开发取得了令人难以置信的进步。在那个黑暗的时代，尝试服务器端应用程序开发通常需要令人讨厌的 CGI T2、一点 Perl、记事本编辑和很多运气。

今天，开发人员被 PHP、Java、C#、VB、Ruby、Python 和许多其他平台和语言的选择宠坏了。显然，谷歌认为这还不够，他们创造了自己的新编程语言，命名为*‘Go’*。

Go 作为谷歌 20%的项目之一开始。谷歌开发人员被允许每周花一天时间在他们选择的项目上——这一想法导致了 GMail、AdSense 和其他成功的系统。Go 团队的目标是构建一种快速、有趣且高效的开发语言。它必须提供编译程序的性能和安全性，以及解释语言开发的简易性和速度。

从语法上讲，这种语言让人想起面向对象的 c 语言。下面是常见的“Hello，world”示例:

```
 package main
import fmt "fmt" // package implementing formatted I/O.

func main() {
	fmt.Printf("Hello, worldn");
} 
```

Go 提供了函数、循环、条件表达式、指针(不含算术)和你能想到的所有其他结构。有几个有趣的特点:

*   行尾分号是可选的。
*   类似 JavaScript 的匿名/ lambda 函数和闭包。
*   类似 Python 的数组切片和映射。
*   使用“goroutines”支持并行性(并发执行)。仅此一项就可能是一个有吸引力的选择；很少有开发人员钻研多线程编码，而 Go 似乎让这变得很容易。

## 收集

Google 提供了两个编译器:6g 用于 64 位系统，8g 用于 x86 系统。两者都设计得很快，中等规模的应用程序可以在一秒钟内编译完成。即使包含 120，000 行代码的整个包库在标准笔记本电脑上也能在不到 10 秒的时间内编译完成。还有一个基于 <acronym title="GNU Compiler Collection">GCC</acronym> 的替代编译器叫做 Gccgo。

不像 Java 和。Go 编译成本机二进制代码，而不是托管或中间代码。这是一个不同寻常的决定，许多人会认为这是一种倒退。理论上，托管代码更安全、更可移植，并且可能更快，因为它可以针对运行它的处理器进行优化。实际上，很少有 Java 和。NET 应用程序以其速度著称。

## 围棋有前途吗？

Google 正在 BSD 许可下发布 Go 源代码，并希望一个热切的开发社区将会发展起来。

这种语言是实验性的，不推荐用于生产环境——没有集成 IDE，标准库很少，代码示例也很少。然而，这个项目看起来是成熟的，开发者用它来建立运行项目官方网站[golang.org](http://golang.org/)的网络服务器。

有了谷歌的支持和资源，你可以放心 Go 有未来。就我个人而言，我认为这种语言很有潜力，但它还需要一段时间才能吸引开发者离开他们目前选择的平台。还有一个选择谷歌正在考虑，这是一个有点可疑…见[为什么谷歌不应该给 Chrome 开绿灯](https://www.sitepoint.com/google-chrome-go-plugin/)。

你想试试围棋吗？是未来还是另一种小众发展语言？
