# 开源成熟

> 原文:[https://www.sitepoint.com/open-source-maturing/](https://www.sitepoint.com/open-source-maturing/)

本月浮出水面的一份尘封已久的旧备忘录可能引发了开源和闭源之间的宗教战争。然而，开源社区(商业)也有成熟的迹象，其中包括两项对 web 架构和开发界来说是好兆头的业务。

其中一家公司 Specifix 由前红帽高管金·克努蒂拉(Kim Knuttila)和埃里克·特罗恩(Eric Troan)共同创立，并将包括来自 Linux 和 GNU 氛围的其他公司。

Specifix is 很有趣，因为它最初专注于提供工具来管理跨多种配置和运行 Linux 的系统的单一代码库。这包括对 Linux 发行版源代码的更改，以满足基于 web 的应用程序中的定制需求。

引入稳定性和保证的开源产品可以成为网络公司向客户推销开源平台的核心卖点，促进单一可管理的代码库(不仅是操作系统代码，还有运行解决方案的系统和软件，如 Apache、Tomcat、MySQL 等)。虽然这似乎比引起人们对 web 应用程序的兴趣和销售 web 应用程序更重要，但 IT 经理会关注整个包，包括硬件、软件和系统集成，即使这不是 web 合同的义务的一部分。

马萨诸塞州 Needham 的开源顾问提供咨询和技术服务的独特组合，就硬件和开源软件的获取、维护和升级向客户提供建议。

这两家公司并不孤单，我们肯定会看到许多公司出现，试图在 Linux 发行商和致力于 Linux 和开源的硬件/软件制造商之间找到合适的位置。

网络公司的关键是关注标准化、咨询服务和开源分析评论的组织的兴起。由此产生的数据成为辅助营销材料，以促进网络公司客户和潜在客户的开源解决方案的稳定性和寿命。
