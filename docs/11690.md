# Microsoft Office Online:针对受支持浏览器列表的案例

> 原文:[https://www . site point . com/case-against-official-browser-support/](https://www.sitepoint.com/case-against-official-browser-support/)

如果你的爱好包括抨击微软，那么继续读下去。该公司宣布，他们的新 Office Web 应用程序将正式支持 Windows 上的 IE7、IE8 和 Firefox 3.5 以及 Mac 上的 Safari 4。没有提到谷歌 Chrome 或 Opera，所以可以理解的是，这些浏览器的许多粉丝正在让自己陷入偏执的反垄断狂潮。

Office Web Apps 博客声明其他浏览器*可能*也能工作，用户应该尝试一下。浏览器不会被屏蔽，微软的目标是确保广泛的兼容性和覆盖面。

这就引出了一个问题:为什么你、微软或任何其他公司要承诺支持*【官方】*浏览器列表？

正如我在上一篇文章中所讨论的，网站和应用程序应该是为网络而不是浏览器而写的。我们应该努力开发与设备无关的系统，而不是针对特定的操作系统和浏览器组合。

微软博客说*“不是所有的浏览器都一样”*是正确的，但是现代浏览器足够符合标准。很难找到一个不可能实现的功能，微软引用的唯一例子就是文本复制。Firefox 的默认安全设置不允许 JavaScript 将文本复制到剪贴板，因此，单击 Office“复制”图标将显示以下消息:

![Office Firefox copy alert](../Images/d99e1e14bdbdd7414a74cd2ceaf5aa83.png)

对我来说，这是一个荒谬的解决方案。如果您的浏览器不支持复制，为什么要显示图标？警告消息没有帮助，也没有提供键盘选项；它折磨用户并暗示火狐低劣*(这是微软的意图吗？)*

我们目前有五种主流浏览器，许多分支，以及 Windows、Mac OS 和 Linux 的多个版本。跨操作系统和浏览器的每一种组合测试 web 应用是不可能的。公布官方支持的浏览器列表只会招致批评，并暗示其他负面含义:

*   微软在开发过程中没有测试过几种流行的浏览器。在发布后修补产品比在主要开发阶段进行全面测试要困难和耗时得多。
*   Office 在 IE、Firefox 或 Safari 的更高版本中还能工作吗？网络应用要到 2010 年才推出——Firefox 4 和 Safari 5 肯定会推出。

如果微软没有公布他们支持的浏览器列表，会发生什么？非常少…

*   Office web 应用程序尚未发布，可能会进入测试阶段。
*   Opera 和 Chrome 用户占市场的比例很小。
*   只有一部分用户会尝试这些应用程序。
*   他们中很少有人*应该*会遇到兼容性 bug。
*   只有一小部分人会向微软报告问题。

微软可能会收到一些支持请求，但是当他们试图解决兼容性问题时，这些信息可能是非常宝贵的。

如果您正在开发下一个大型 web 应用程序，最大限度的浏览器兼容性应该是优先考虑的问题:

1.  将代码组件化，并在各种浏览器中进行单元测试。
2.  尽早测试，经常测试。
3.  不要追求不切实际/完全相同的跨浏览器功能。
4.  当浏览器支持时，使用渐进式增强来添加功能。

浏览器支持列表比它们的价值更麻烦。
