# 与 Mat Marquis 一起寻找网络职业的意义

> 原文:[https://www . site point . com/versioning-show-episode-2-with-mat-marquis/](https://www.sitepoint.com/versioning-show-episode-2-with-mat-marquis/)

在这一集的版本控制节目中，Tim 和 David 与 Mat Marquis 进行了对话，Mat Marquis 是一名网页设计师，以其在响应式图片、波士顿环球报重新设计和 CSS 容器查询方面的工作而闻名。在这本关于开始网络生涯的搭便车指南中，Mat 谈到了网络标准、在网络上成名以及寻找网络生涯的意义。

## 副本

Tim:

嘿，大家好，这是蒂姆·埃夫科…

David:

…这位是大卫·格林先生…

Tim:

…您正在收听的是版本控制播客的第二集。这是一个我们每两周坐下来讨论从开发到设计的网络产业的地方——一些人今天让它发生，并计划它在下一个版本中的走向。

David:

那么，让我们开始这个版本吧。

Tim:

今天，我们有一位特殊的客人，马特·马奎斯。马特，怎么样了？

Mat:

很好，很好，很高兴来到这里。

Tim:

所以，对于那些不知道你是谁的听众，你介意告诉我们一些关于你自己的事情吗？

Mat:

当然，我是 [Bocoup](https://bocoup.com/) 的一名开放 Web 工程师。我是[回应问题社区小组](http://ricg.io/)的首席发言人——我的意思是，我不是特别聪明或特别有才华，但我非常大声，这主要是我在该小组中的角色。我已经做了几个[的活动片段](http://aneventapart.com/)，我有一本书即将出版，我刚才肯定没有提到……但是请留意我从未提起过的那件事！是的，我一直很忙。

David:

你没有提到你的书，但我想我们会问你更多关于它的问题。但在此之前，我们想问每位嘉宾一个问题，因为这是版本控制秀:在你目前的职业生涯中，你是什么版本，为什么？

Mat:

哦，天啊，这是一个沉重的问题。

我认为职业生涯的开始至少是我个人的第二个版本。就职业生涯而言，我已经四五岁了——我的意思是，我想这就像是 Gmail 的永久测试版。但是，至少这个版本会随着时间的推移而改进。

David:

我喜欢这个 Gmail 永久测试版。我认为从某种意义上说，我们都处于 Gmail 永久测试阶段。

Mat:

我是一个活的 bug 修复版本:珍贵的几个功能；随着时间的推移，很多损坏的东西会减少。

Tim [1:55]:

这很有趣。你想谈谈你是如何以 web 开发为职业的吗？

Mat:

是的，我走了一条非常传统的路线，那就是通过搭便车的方式进行网络开发，跟随一个失败的职业木工生涯——我认为这对我们大多数人来说是一个非常典型的故事。是的，我当时 20 出头……差不多。那是很久以前的事了，我现在几乎都忘记了。

我开始做木工。从那种情况下开始，我在零售行业干了几年，这是最棒的！*【扭曲语气】*我当过几次流水线厨师；我是一个倒霉轮班的保镖。在我职业生涯(零售业)的末期，我在一个商场的移动电话亭里工作，当你走过的时候，他们会冲你大喊——<q>嘿，我们有免费的 Razr 手机！</q>

是啊，所以就是那个时代。有一天我放弃了。就像，我住在一个非法的两居室公寓里，我们一共五个人，我靠一张张的薪水勉强度日，然后我就辞职了。我只是举起双手，然后我就像这样——我什么都没有准备好，我不知道我要去哪里，但它不在这里。我不干了，我不干了，剩下的我以后再想。

所以我放弃了，在两周的周末，我买了一个大的绿色背包，开始向南走。我有个朋友住在佛罗里达州的珊瑚角。那是我所知道的最远的地方，所以我开始步行。

我在东海岸搭便车旅行了大约两个月。我去了那里，最终为他当时约会的一个女人做了一个网站，因为她就像是——你知道计算机之类的东西，对吗？

我说，嗯，我在 [CompUSA](https://en.wikipedia.org/wiki/CompUSA) 做过一段时间的销售，所以我会打字。还有，<q>你能给我做个网站吗？</q>

我当时想，只有一个办法可以知道。所以我做了这个粉红色的，基于表格的噩梦网站。家乡的一群人凑钱让我飞回来，因为他们说——<q>你肯定会在回来的路上被捅死</q>……严格来说这不是假的。

是的，我回来了，我说，我不会做木工。我不能就这么捡起来重新开始木工生意。我不会再做零售了，就这样。但是我曾经做过一个网站，所以也许我能做到。

逛了一段时间 craigslist 网站。我没有联系他们，我只是看了描述，造了就扔了。只是靠我在佛罗里达建的网站赚的一点点钱生活。

David [4:28]:

那很有趣。那么，就你的技术技能而言，你基本上或完全是自学的？

Mat:

完全，是的，没有学位，什么都没有。几本奥莱利的书。

David:

这证明你可以从公共媒体和现有资源中学到很多东西——仅仅是从书本和自学中。但我很好奇你是如何让自己走到这一步的，你实际上拥有可以卖给别人的技能。

Mat:

对我来说，这也是非传统的。在 craigslist 的几场假装演出后，我开始为钱接几场演出，他们就像——安装这个 PHP 脚本，等等。

其中一个是同类装置之一；这就像是几个小时的工作，他们在事后联系我说——<q>你看起来像是初级水平，你是在为学校学分寻找实习机会吗？</q>

我说，当然可以，我的学校非常希望你能给我一个实习机会，让我得到我没有去过的学校的学分。

*【笑声】*

所以，是的，我在那里呆了大概三个月，在结束的时候，他们就像是——我们不是应该签些什么吗——不是有一些文件吗？

我说，是的，不——关于那件事，那都是谎言。你想雇我做什么吗？

他们说，是的，好的，当然。这一切都很好，为什么不呢？

Tim:

大胆的策略。

Mat:

是啊，运气太差了。所以，我在那里呆了几年。做了几年自由撰稿人。灯丝组好几年了。现在，我不知道，一年半了。

Tim [6:00]:

从你最初的故事来看——至少对我来说——走出去为网络做些有益的事情是很有动力的。它帮助人们开始职业生涯以及诸如此类的事情——这样人们就可以把它作为一种工具来做好事。我注意到在你的职业生涯中，你为网络做了很多好事，无论是响应图像还是性能或可访问性。

你想谈谈这个吗？是什么让你在这方面坚持下去的？

Mat:

老实说，这可能听起来有点唯利是图，在工作中需要大量的解释才能理解这种对网络的哲学，但我是在蓝领阶层长大的。在那之后，我从一份工作跳到另一份工作，得到我能得到的任何薪水。我把这当成一份工作。我对打字并不是特别有激情。我花了一段时间才理解这个想法。就像我出现了，它就像，好吧，我可以每天打字八个小时，然后回家，让它结束，只是这样做，日复一日，一旦新鲜感消失。我在领薪水，一切都很普通。但是所有这些东西都足够新，以至于在任何行业都不容易建立一个名字——让你的名字出现在那里，并在这个、那个和其他方面获得一点影响力。

但是这些东西都是很新的，每个人都还在摸索中。没有人来自传统的背景。这主要发生在我做自由职业者的时候。我从小就有这样一种感觉，每当我与从事投资银行或其他工作的人交谈时，那是一个我不被允许去的星球。那是我没有意义的生活。

这一切都是新的，每个人都来自世界各地，这就像，你知道吗，也许这就是它。也许我可以给自己找份坐办公室的工作。也许我可以借此成名，现在回想起来，我做得太过了。我每天工作 14 个小时，我累坏了我的背，一大堆是特权，一大堆是运气，但我最终在一个相当不错的地方。所以，我从这里得到的第一个意思是，我要在某个地方取得成功，我要以某种方式证明我的教养是错误的。

但是一旦你开始做得不错，那种感觉就消失了，你什么也没有，你只是一天敲八个小时。所以在那一点上，它变成了— <q>我该怎么处理它？这有什么值得做的？现在我有了一个小平台，现在我可以制造一点噪音，人们会经常听我说话，那么这里有什么值得做的呢？</q>

这就是我的结局:开发者的网络标准，性能，用户的可访问性。这就是我决定要表达的意思。因为否则我会在正确的地方制作 div 节目，这不是非常令人满意的工作。触及事物的边界——不是你每天能做的最有意义的事情，但我可以做一些东西，这个星球上的任何人——人们拿起他们的功能手机，第一次连接到网络——我可以做一些他们可以阅读的东西。那是相当大的。

David [9:21]:

嗯，在我看来，你的职业生涯已经走过了唯利是图的阶段，进入了更多的科技慈善模式，在这种模式下，你实际上是在努力帮助人们。我不确定人们是否也很熟悉——他们知道通用术语“可访问性”——但是你能更详细地描述一下你正在做的工作吗？

Mat:

大约在那段时间，我对这些东西越来越感兴趣。我觉得这是我希望事情发展的方向，这是我个人在职业生涯中希望的方向。但我大部分是在细丝集团学的。所以，我去了那里，感觉很棒。

我去面试，约翰·瑞西格坐在离我两英尺远的地方，吃着肉丸三明治，所以我很尴尬，我说，*【耳语】* <q>我也做 jQuery！</q>

*【笑声】*

是的，我去那里，我说我真的很喜欢可访问性，我真的很喜欢渐进增强。性能在当时并不是一个大问题——现在已经变成了一个大问题——但我不会放弃对它的兴趣，所以当我在那里工作时，感觉很好——每个人都说同样的语言。

我们都专注于可访问性，我们都专注于渐进增强，然后，稍后，响应性的东西。因此，在早期，我为 jQuery UI 团队做了一点工作。我为 jQuery Mobile 团队做了更多的工作，然后，最终，我们来到了波士顿环球报网站，这是一个很大的网站——嘿，我们证明了 responsive 可以在大规模的网站上工作。这间接地导致了整个 responsive images 的努力，因为那时我们才刚刚开始考虑更智能的资产交付。

我们知道如何使它具有弹性，但是我们如何使资产上下文合适呢？

David [11:02]:

你能多告诉我一点吗？因为很多人都听说过响应式 web 开发，一般来说就是响应式 web 设计。响应式图像可能是一个有些人不太熟悉的术语。

Mat:

是啊，这是相当新的东西。回到在地球上工作的时候，让图像有弹性——非常容易。你知道，`max-width: 100%`，然后它从它需要的任何尺寸到它需要的任何尺寸。

但即使追溯到全球，这种一刀切的方法并没有真正解决问题。

David:

嗯，我认为他们必须支持 Internet Explorer 6，所以这是行不通的。

Mat:

是的，我们在那个项目上支持了大量的东西——大部分是偶然的，这总是好的。

但是，是的，不仅仅是在性能方面的缺点——通过网络向每个人发送一幅巨大的图像，无论他们是否看到任何真正的好处——但是全球员工没有足够的控制力，他们有时希望裁剪出更适合较小布局的图像，而不是缩小一幅巨大的图像。

所以，我们想出了——我称之为黑客，而不是脚本——使用间隔 gif 和 cookie(不骗你！)根据图像的上下文交换图像。它从较小的图像开始，在较大的显示器上将它换成较大的图像，然后设置一个 cookie 说，<q>总是给这个较大的图像。</q>

它完全崩溃了——就在 Globe 发布后——只是因为浏览器中超级激进的预取。就好像，在它获取图像之前，我们无法应用任何逻辑。

我们最后坐在一个小组里，我们有几个像保罗·爱尔兰人、杰森·格雷斯比这样的人——一群聪明人。我们开始思考，如果我们有一个基于标准的解决方案，会是什么样的呢？经过一两周的反复讨论，我们想，嘿，我想我们已经有了一个很好的想法。这是布鲁斯·劳森的，最初是他提出的。

我们的计划是，<q>我们现在把它交给标准机构，他们会对我们满意的！</q>

David [13:09]:

当然啦！他们的大门是敞开的——他们只是在等待有人进来给他们标准！

Mat:

绝对的！他们的座右铭是在别处发明的，我相信这是他们的主要动力。

所以，是的，我们满面笑容地走进去——嘿，我们想出了这个东西！

他们立即拒绝了我们，并说，你没有资格给我们提供解决方案。你应该给我们带来问题，让我们屈尊为你服务。

我不再为这件事或任何事耿耿于怀。

*【笑声】*

但是很快，在两年的邮件列表争论和讨论之后，我认为我们最终达成了最初的 responsive images 提案的第三个版本，它本身成为了一个规范，我们在技术上不被允许作为一个社区团体发布，但无论如何，作为<q>海盗电台</q>。

我们最终在浏览器中得到响应的图像。因此，我们有办法解决所有我们试图解决的问题，早在全球项目时就有了，而且还在继续。这意味着——我想是 Tim Kadlec 对一大堆网站进行了研究——在 Alexa 排名前 100 位的网站中，实现其中一些意味着大约 72%的传输节省，或者类似的情况。

这对于网络本身来说是一个相当大的冲击。就传输大小而言，图像仍呈上升趋势，但速度正在放缓。能够成为一个团体的一员，对网络的构建方式有这样的影响力——这是一件非常重要的事情。

David:

那么，这是标准之外仍然存在的东西，还是最终开始进入标准的东西呢？

Mat:

现在 100%标准化了。它与 HTML5 规范合并了——我认为两个版本都是。目前，除了拥有大部分功能的微软 Edge 之外，所有浏览器都完全支持它。

你知道，其他的很快就会出现。你可以用聚脂填料。您不必使用 polyfill，这是一个很好的位置，因为它有一个固有的后备模式。

Tim [15:20]:

所以，Mat，说到你为了把这个东西推出去所做的努力。对于那些感觉自己可能面临某种类似挑战的人，他们试图提出一些他们认为对他们的工作或总体网络或他们正在进行的项目更好的东西，但似乎面临大量阻力，你有什么建议吗？

Mat:

当然可以。就标准而言，我很难说，<q>当然，参与标准，这很有趣，</q>因为它并不那么有趣。但是，几乎比我们有弹性的图像更多的是，我们现在已经建立了一个先例。除了[什么工作组](https://en.wikipedia.org/wiki/WHATWG)本身之外，没有多少先例可以让一群志同道合的开发人员聚在一起，挑起一场战斗，并在 web 标准方面取得一些进展。

社区团体的模型并不完美，但我们从中做出了一些贡献。懂我意思吧?因此，如果你是一个注重标准的开发人员，你可以创建一个社区团体，你可以加入一个现有的相关社区团体，你可以出去推销你的东西，让别人听到你的声音。

WICG——网络孵化器社区团体——现在是一个致力于倾听开发者声音的社区团体。他们有一个[话语论坛](https://discourse.wicg.io/)。所以你不会被锁在 1985 年的邮件列表中，试图完成工作。你实际上可以加入这个网站，你会收到电子邮件通知之类的东西，它只是一个论坛。

你不需要 1000 美元的订阅费，也不需要成为 W3C 的会员。你可以说:嘿，我有个主意，我有个好主意。谁也认为这是个好主意？

我有一个演讲台——很多都是特权。我可以通过 DM 问别人，嘿，帮我把这件事说出去。但是那里的情况已经有所改善。而且，最糟糕的情况是，如果你是一名开发人员，并且你需要对一个潜在的标准发表一些意见，那么让我成为你的代言人。

David [17:32]:

那太好了。你仍然有一个很大的演讲台，我很好奇你认为你想接受的下一个挑战是什么。

Mat:

o-脚跟转动，100%！我要变得超级邪恶。

*【笑声】*

完全腐败，彻底破坏网络标准，然后一些有事业心的年轻开发者将不得不把我赶下台——我想这可能就是计划。

David:

悬停效果都将是 Java 小程序，对吗？

Mat:

我的计划是，尽可能多地引入任务关键型闪存。

Tim:

这只是第一步，对吗？
