# Opentape 用开源的 Muxtape 把它粘在 RIAA 身上

> 原文：<https://www.sitepoint.com/opentape-sticks-it-to-riaa-with-open-source-muxtape/>

随着网络音乐爱好者最喜爱的 [Muxtape](http://www.muxtape.com/) 由于“RIAA 的问题”目前退出使用，一个开源的——如果没有更多合法的——替代方案出现了: [Opentape](http://opentape.fm/) 。Opentape 将自己描述为“一个免费的开源包，让你可以在网上制作和托管自己的混音带。”或者，换句话说，一个免费开源的 Muxtape 克隆。

演示 mixtape 基本上是 Muxtape 网站的直接克隆。软件使用 PHP 5，Apache，需要 curl。几天前，0.1 版本在网上发布了。

为什么会有人想要创建一个开源版本的应用程序，而这个应用程序由于美国唱片业的法律纠纷而关闭了呢？在[黑客新闻](http://news.ycombinator.com/item?id=286601)上，我第一次听说 Opentape，人们一致认为这个想法将成为 RIAA 的另一个眼中钉。越多的人运行他们自己版本的 Muxtape，理论上 RIAA 就越难关闭他们。

Opentape 并没有解决任何版权问题，它只是让 RIAA 更难提起诉讼。然而，正如黑客新闻上的其他人所指出的那样，在 P2P 流量方面，RIAA 并不羞于追查个人。

几周前，我们[发布了关于](https://www.sitepoint.com/8tracks-the-legal-web-mixtape/)一项名为 [8tracks](http://www.8tracks.com/) 的新服务，其运作方式与 Muxtape 类似——允许用户创建 8 首歌曲的混音带并与朋友分享——但计划通过向 SoundExchange、ASCAP、身体质量指数和 SESAC 支付版税来合法地这么做。

Opentape 会成功吗？这很难说，但有两个潜在的问题阻碍了它的发展。显而易见的一个是合法的:有多少人愿意冒着激怒 RIAA 的风险在他们的服务器上发布一个 mixtape？第二是内容。Muxtape 之所以伟大，是因为它能够浏览其他混音带，并找到优秀的新音乐。如果没有一种方法来聚集和发现人们用 Opentape 制作和发布的 mixtapes，它作为一种分布式服务的吸引力将会降低。
