# 。2 月 15 日至 22 日:Silverlight 2 是甜蜜的

> 原文:[https://www . sitepoint . com/net-on-the-net-Fe B- 15-22-Silverlight-2-is-sweet/](https://www.sitepoint.com/net-on-the-net-feb-15-22-silverlight-2-is-sweet/)

首先，一份简短的 Windows 2008 服务器之旅报告:它在太多方面都令人敬畏，无法一一列举。现在，让我们进入网络世界。

今天斯科特·格思里给了我们一个印象深刻的 Silverlight 2 第一印象。不要错过最后关于构建 Digg 应用程序的教程。在 IE 8 方面，Dean Hachamovitch 提到了一个里程碑，并宣布 [IE 8 测试版将于今年上半年推出](http://blogs.msdn.com/ie/archive/2007/12/19/internet-explorer-8-and-acid2-a-milestone.aspx)，Eric Lawrence 向我们提供了[测试版的用户代理字符串](http://blogs.msdn.com/ie/archive/2008/02/21/the-internet-explorer-8-user-agent-string.aspx)。对于工具，微软 Visual Web Developer 团队发布了 ASP.NET MVC 的 [MIX 预览版的新工具特性预览。在 IIS 7 上，请务必查看 MSDN 杂志关于扩展 IIS 7 管道的文章。](http://blogs.msdn.com/webdevtools/archive/2008/02/17/tooling-features-overview-of-asp-net-mvc-framework-for-mix-2008.aspx)

继续谈架构，微软应用咨询和工程团队的 Alik Levin 探讨了为什么需要分布式架构。不要错过网上一些最大网站的案例研究链接。乌迪·达汗发表了一篇关于制作 [CRUD apps 域驱动](http://udidahan.weblogs.us/2008/02/15/from-crud-to-domain-driven-fluency/)的尖锐文章。我们都知道依赖注入和控制反转非常流行。但是 SitePoint 论坛自己的 CompiledMonkey 提醒我们[有时候依赖是没问题的](http://weblogs.asp.net/cstewart/archive/2008/02/15/but-i-like-my-dependencies.aspx)

在编码方面，ASP.NET MVC 是新的热点。DotNetSlackers 团队的 Kazi man zur Rashid[向世界展示了 Kigg，](http://weblogs.asp.net/rashid/archive/2008/02/18/kigg-a-digg-like-application-developed-in-asp-net-mvc.aspx)一个看起来很有前途的类似 Digg 的应用程序。不仅是开源，Kazi [开始了一系列关于构建 Kigg](http://dotnetslackers.com/articles/aspnet/KiggBuildingADiggCloneWithASPNETMVC1.aspx) 的文章。当我们谈论 MVC 的时候，Chris van de Steeg 发布了使用 jQuery 实现 ASP.NET MVC 的第二部分[。Matt Berseth 帮助美国开发人员](http://www.chrisvandesteeg.nl/2008/02/16/jquery-for-aspnet-mvc-part-2/)[构建光滑、柔软和简单的数据网格](http://mattberseth.com/blog/2008/02/building_sleek_soft_and_simple.html)。最后但同样重要的是，一定要看看 Gustavo Duarte 发布的[ASP.NET 运行时备忘单](http://duartes.org/gustavo/articles/Asp.net-Runtime-Cheat-Sheet-HttpRequest-HttpRuntime.aspx)——它是 phpinfo()；我怀念过去的时光。
