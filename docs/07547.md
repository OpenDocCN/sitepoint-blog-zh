## 采访记录

凯文:好了，欢迎来到 Sitepoint 播客。我是凯文·迪斯，我的搭档是迈克尔·金萨尔。欢迎来到这个节目。

迈克尔:你好，非常感谢。

凯文:我不得不说，我喜欢你的声音。你听起来比我专业多了。

迈克尔:这些年来，我已经和它结缘了，谢谢你。

凯文:很高兴听到这个消息，你没有和任何人分享。这是一种非常播客式的声音。你可以看出我的演讲并没有想象中的那么雄辩。

迈克尔:我觉得很好。目前为止我很喜欢。

凯文:谢谢你。你做播客，对吧？

迈克尔:是的。我一直在做[网站开发电台](http://webdevradio.com/)，显然这不是电台，这只是一个播客。实际上从 2005 年开始，到现在已经有七年多了。7 月 4 日周末，刚开始做。我有时采访人，有时只是我在信口开河。

我曾经在 iTunes 上看过一篇评论，有人说，‘我不明白。这看起来就像一个老人。他说了这样的话，“这就像一个喜欢抱怨的老家伙。”他碰巧遇到了一个问题，我真的在抱怨 rails，我在它上面遇到的一些问题。我通常不会抱怨那么多，但有时我会发泄。

凯文:那很有趣。我觉得在深入对话之前不让听众知道你到底是谁，是对他们的不礼貌。我觉得自我介绍一下会很好。迈克尔，你过去几年都在做一个会议，你也是一名 PHP 程序员。也许你能告诉我们一些关于这两件事的情况，然后我们可以从那里开始？

迈克尔:哇，是的。我该如何开始？有很多背景。

我一直从事 PHP 工作，幸运的是在很多方面，我从事软件工作已经很长时间了，但我在 96 年很早就进入了 web。在那之前我一直在 telnet 和 gophering，但是进入 web 开发领域后，我发现了这个叫 PHP FI 的东西。我记得是 96 年 2 月，96 年 1 月。

我感到非常幸运，因为我在职业生涯的早期就进入了 web 开发领域。我不需要忘记很多概念。我大部分时间都在使用 PHP。我做过 Perl，ASP，一些 Java。最近我非常喜欢 Grails 和 Groovy，但是 PHP 可能已经占据了我职业生涯的 70%。这对我很好。观察它随着时间的推移而成长和演变是非常有趣的。

你提到了这次会议。我组织了一次会议，这是 IndieConf 的第三年，主要是为像我这样的网络自由职业者。如果你做自由职业者 Druple，WordPress，PHP，。NET，Java，不管是什么。如果你在网上工作，你是自由职业者，你会有和我想回答的同样的问题。我把它放在一起。

凯文:你知道，我去年去了那个会议，我不得不说它真的很棒。我对我的会议很挑剔，我喜欢两种。很小的那种，或者不是超小的那种，你懂我的意思吧。它小到你可以见到人们，和他们交谈，因为我们都在一个小组里。然后你有这些超级大的像西南偏南。

我真的很喜欢你的活力。每个人都有自己的对话。你有机会与演讲者见面并交谈，这在会议的规模中起着很大的作用。这个系列正好符合它的要求，我认为你去年的尺寸非常棒。我知道你想种那个。

迈克尔:一些。去年我们有大约 130 人。我希望今年在日程安排和诸如此类的事情上有大致相同的数字。如果有 200 个人敲门，那就太好了，我们可以满足他们。

从组织上来说，我们现在没有准备一个 500 人或 400 人的会议，400 名与会者。如果需求增长到那种程度，我很乐意支持，但我很欣赏你对规模的看法。

对我来说，如果你想往那个方向漫谈的话，我会简单讲讲这是从哪里开始的。

凯文:是的，绝对是。

迈克尔:2007 年，我又开始了自由职业。2007 年，2008 年，这个小小的全球经济崩溃发生在世界各地。这确实不是一个做那件事的好时机，但我设法坚持了下来。通过大量的网络和诸如此类的东西，我终于在这个地区扎下了根。我觉得我现在在这里有了一个不错的关系网。

到了 2009 年、2010 年，我开始经常有人问我，‘你是怎么做到的？你如何获得客户？你如何得到报酬？你是怎么做到的，你是怎么做到的？你的合同是什么样的？诸如此类的事情。

我有答案。我可以给出答案。有些问题，我的答案肯定对某些人有帮助，但不一定对每个人都是正确的答案。每个人的背景都有些不同，他们的财务需求不同，他们的心理安全需求也不同。

我开始询问一些我认识的有更多经验和不同观点的人，并意识到没有人可以去的地方。有在线资源，有很多好的论坛，但就面对面的事情而言，那里有不同的动态。这是 2010 年两年前 IndieConf 的真正动力。

召集能够回答这个问题的人，但是尽量把重点放在软件上。当然，水管工和木工也可以来参加，他们会从中得到一些东西，但与会者的焦点主要是在网络上工作的人。开发者，设计师，也许还有内容作者。

即使你不是开发人员，也许你只是为人们做 Druple 网站，但你不是编码，你仍然在网络上工作。如果你在为自己工作，你可能仍然会遇到很多和我们很多人一样的问题。这就是背后的简单故事。

Kevin: 这是我去年非常喜欢它的一点，我参加了很多关于 CSS 代码和设计的会议。它总是有一个主题点，那就是新技术，下一件大事是什么。

有了 IndieConf，就务实多了。你可以学以致用，到时候就可以用了。

迈克尔:我们有像你这样的人。你可能会是一个更技术性的演示者。凯文·迪斯将展示实用的 WordPress 工作流程。这将是你可以带回家并在那里运行的技能之一。我并不指望你会对 PHP 基准和 CSS 调试之类的东西感兴趣。

我告诉你，甚至在 2007 年 IndieConf 之前，我随机地提交了一些东西给加州的 Oz Con. Mega 会议。我被选中发言，我出去讲了讲 Solr。太棒了。我喜欢 Solr 这个搜索引擎，它很棒，我玩得很开心，我在那里遇到了一些很酷的人。做了几个播客。

总的来说，我精神上超负荷了。我出去了，只是一个接一个的技术。我记得回来的时候，我哭了一下。我并不是哭了几个星期，而是在回来的路上，当我坐在那里，在回来的路上，我哭了一点，因为意识到，我将回到一个工作岗位，而我所接触的技术中，我将能够实现的是零。

我在做维护。等到我被允许实现它，或者被允许将它集成到我的工作中时，它就已经过时了。随便啦。有一种意识。

我参加科技会议。Code Stop 是一个我喜欢参加的非常棒的会议。以前去 Code Mash，可能还能进去，但是现在很难进去了。它们的代码量很大，很适合极客使用。对于那些为自己工作的人来说，这并没有把合同放在我的桌子上。

当你和大多数客户打交道时，他们并不在乎你使用的是什么版本的 CSS。还有一系列不同的问题。我喜欢钻研技术，但这些会议并没有满足我的迫切需求。

那是 2007 年，我当时并没有想过这个问题。这是我们在 IndieConf 不关注技术的一部分，因为它是为技术工作者服务的，但它与技术没多大关系。你可以参加 50 个会议来学习 Javascript 和 CSS，这很棒，我认为你需要参加这些会议，但你也需要去 IndieConf。

凯文:对。像奥兹国和西南偏南有两个部分。这是关于技术的，我认为这些会议更倾向于初创企业文化，这是一种新的时尚，人们正在 Node 中构建他们全新的网络应用。我知道这是两年前的事了，现在它仍然是一个东西，但现在它是 Python 或其他什么东西。

总是有新的，旧的技术现在变成了新的技术，对吗？它循环往复，这是一种创业文化。另一部分也是你从会议中获得的网络和联系。

这就是我认为的原因，我并不想让这变成一个会议演讲。。。

迈克尔:我们马上就要搬走了。

凯文:我喜欢大型会议的一点是，你可以获得这些联系，你会问，‘嘿，创业界在做什么？’你真的可以得到拼图的那一部分，并真正建造它，然后你可以去这些较小的地方，实际上找到一些有用的东西。

所有的新技术对于那些将要写博客并谈论他们将如何实现它的人来说都是很棒的。有些事情你可以。我知道有一年在西南偏南，打字是件大事。Typekit 刚刚问世，Font Deck 刚刚问世。你可以在网上使用这些字体，字体嵌入终于出现了，法律问题终于解决了，我们可以做这件事了，这很棒。

这是一个很大的势头，它继续下去，实际上意味着什么。在那些大型会议上，并不是每个演讲都是这样的。而较小的，就像我说的，你仍然有相同的联系，但我认为这是更深层次的联系。

这只是表明，就像现在的这个对话，我们去年见过面，现在我们又开始对话了。就在会议之前。

迈克尔:对。我在大大小小的会议上遇到过一些很棒的人。我可以毫不夸张地说，我在会议上遇到的一些人最终让我的生活变得更好。

我在 2005 年开始我的播客。2008 年初，我一月份去了俄亥俄州的 Code Mash，是的，那是另外一个故事了。我知道那里有几个人，但是那里很大，当时有几百人。

我走到餐桌旁，明确地坐在一张我谁都不认识的桌子旁。大约有八到十个人坐在桌子旁边，我在和某个人说话。桌子对面的那个家伙正看着我。他们中的一些人，他们会看着我，然后他们会看向别处，然后他们会再次看着我。

大约十分钟后，他们说，“我们知道你的声音，但不知道你是谁。”我自我介绍说，“哦，你做那个播客。”并不是说我名气大，但是有些人认识我，他们知道我的声音。

我们开始交谈，是一个叫卡尔·埃文斯的人。PHP 社区的人们，如果你们不知道他的名字，现在应该知道了。去 calevans.com。他是 PHP 社区的一个极好的资源，一个巨大的倡导者。他在 Zend dev zone 工作了几年，开始做这个。

当时他在 Zend。那是 2008 年 1 月。奥古斯特，我接到一个电话，他说，“嘿，我们在找一个可能想做一些培训的人。“你对此感兴趣吗，”当然可以。

通过认识他，我成为了 Zend 的一名培训师，已经做了四年了。我见过很多很酷的人，有很多很酷的学生，只是通过 Zend 接触了一些很酷的技术。这太棒了，所有的一切都来自于去参加那个会议，坐在那张桌子旁，有人认出了我的声音。

我有几十个这样的例子，但这是你必须走出去和网络的想法。特别是作为一个自由职业者，但我认为即使你不是自由职业者，当人们去上学或选择职业，从别人那里获得职业建议时，走出去并让你的职业有一个社交方面的想法经常被忽视。不管你的职业是什么，很少有人谈论社会或人际关系方面。

也许更确切地说是技术，因为你是一个代码极客，你喜欢做这些东西。坐在电脑后面，继续工作一天，一周，一个月，而不是离开是非常容易的。

凯文:是的。我喜欢我们在这里谈论的东西，因为它确实谈到了我认为可能是成功的程序员。他们往往是建立联系的人。

我知道在我自己的职业生涯中，就像我周围的人一样，我在格林菲尔德的一个叫 Co-Work 的地方工作。这是人们之间的合作。仅仅是和其他人在那栋楼里，因为我是自由职业者，甚至仅仅是和其他自由职业者在那栋楼里，即使那也是一种社交。基本上在那里是值得的。

它的作用不止于此。光是在大楼里我就有很多工作要做。我记得有一次，我在之前的播客中谈到过这个，有一周我生病了，我决定学习如何制作 HTML5 画布视频游戏。我去上班，瞧，我正在制作视频游戏的原型。

这是你必须同时做的事情之一。你必须建立东西，尝试新事物，这样你才能和你要见的人谈论它们。你不能只是出现在现场，然后说，‘哦，我就是这个人，你是做什么的？’“嗯，我只是遇到了一些人，”

有一个平衡。我要说的是，这种联系是有道理的，我很高兴你提到了这一点，因为我知道这对我很有价值。只是回过头来看，所有对我来说是养家糊口的事情都被网络蒙上了阴影和伏笔。

迈克尔:当然可以。有些事情，我已经说了好几年了。我没有在你的播客上说过，所以我可以在这里重复这些。在我的脑海中，我已经能够将这一点更加具体化了。我并不是 web 开发领域的先知或元老，我只是在软件领域做了很长时间。

我已经做了足够长的时间，我已经看到了足够多的技术变化，十年前热门的东西，甚至五年前热门的东西，都不太一样了。我们用技术解决的问题，有些是不同的。有时我们有新的技术，移动的想法。这是一个全新的消费空间和商业空间，在 10 年或 15 年前并不存在，肯定不会达到现在的水平。

总的来说，我们正在解决的这类问题不是通过新的热点、新的热门技术解决的，而是通过理解如何交流解决的。同事、客户、消费者、顾客，无论他们是什么。我喜欢说的一件事，我想我是从哈尔·赫尔姆斯那里得到的，或者从这位广为人知的冷聚变大师那里得到的，他曾经做过一个很棒的播客。我想他不再需要了。

我记得这个，我想是他说的，他怎么说的？他没有因为某人不知道如何写出文件，或者因为他们不知道如何连接数据库而导致项目失败。项目失败的主要原因是缺乏沟通，缺乏理解，缺乏同理心，缺乏对问题的认识。

凯文:当然。

迈克尔:有些人说了些什么，但他们的意思是别的。我参与过几个项目，它们被严重延迟，因为事实上我们无法可靠地连接到数据库，因为我们有驱动程序问题，但这些都是暂时的。你可以解决这些问题。你可以找人来解决这些问题。

很难找到一个人能从客户那里推测出他们真正的意思，或者当他们不接电话或不回邮件时，如何从他们那里获得信息。你是怎么处理的？

这些是人类的问题。你可以通过经验了解这一点，也可以通过与其他人交流来学习如何处理这些问题。如果你是一个自由职业者，你可以在 IndieConf 上做，那是我的小插件。即使没有，去参加演讲会，学习如何与人交谈。参加表演课程，这样你就能理解如何伪装他人，而不是移情别恋。了解其他人对同样情况的反应。站在他们的角度想想。

你可以做很多非技术的事情来让自己在技术领域变得更有价值，而且这些事情不一定与学习节点或异步编程或其他什么有关。我不是在抨击 Node 或 Async 或其他什么，它们都很棒，但它们对不同的人来说是不同的。

凯文:是的，我完全同意。当我回顾过去失败的项目时，甚至不是我失败的事情。这只是缺乏沟通或缺乏对目标是什么的理解。

作为开发人员、设计人员或项目负责人，很多时候很容易将责任推给客户。你没有告诉我你想要什么，或者你告诉我这个，你想要那个，你改变它，你改变它，你改变它。

我发现很多时候都是因为这个原因，不久前我和 Paul Boag 讨论过这个问题，这个以客户为中心的网页设计。这归结为缺乏视角，也许客户有，但他也是开发商兼设计师。至少我知道我喜欢从用户的角度看问题，以及什么对用户来说是最好的。

很多时候，在所有情况下，真的，这是关于业务和什么将增长那个人的业务。用户是其中很大的一部分，但归根结底，这仍然是一项业务。不仅能够看到和理解你的用户需要做什么，因为我认为作为开发人员和设计师，我们做得很好，但是当理解业务和业务是什么时，我认为这就是很多缺乏沟通发生的地方，也是它崩溃的地方。

因为顾客想要实现某些目标。如果他们是非盈利性的，他们试图赚更多的钱或者帮助更多的人。他们试图实现某个目标，所以我认为很多时候我们会陷入这个界面，技术，选择正确的模式或它是什么。

我们太沉迷于那个领域，以至于忘记表达我们为什么要做这个，以及为什么让客户考虑这个问题很重要。然后也站在客户的角度，就像你说的换位思考，对吗？我该如何理解你？

你是说你有过类似的经历。

迈克尔:嗯，我发现这是一个艰难的平衡过程。有时根据客户的不同会更容易。在我看来，尤其是作为一名自由职业者进入某人的企业并与他们一起工作，在了解他们在做什么的同时也带来你的经验并说‘这个问题应该这样解决’之间需要有一个平衡。

不得不，战斗是一个太强烈的词，但与'嗯，不，我们的人这样做。'有可能，而且他们这么做可能有充分的理由。有时候，很多时候，他们在生意中做一些事情的原因是因为他们从来没有机会做其他事情。他们从未得到合格 IT 人员的支持，或者他们根本没有了解可用资源的专门 IT 人员。

凯文:没错。

迈克尔:你需要这样做。你真正的问题是让人们进行实时交流。你想这么做，但是你知道另一项技术吗？如果我们这样做，这可能会更快，这可能会更便宜，这可能会更贵。但是，如果我们这样做，这就是你可以让你的企业做到的。不会解决你的问题，但是会帮你解决问题。

另一件让我头疼的事情是，进去后有人会说，‘我们的业务是独一无二的。’我做这个已经够久了，所以我不经常相信。我相信他们的一些术语是独特的，他们可能有法律问题，要求，报告他们必须做的事情。

在一天结束的时候，你的工作是找到有问题的人，你为问题服务，你从他们那里收钱。你在满足市场的需求，这就是你的业务。你没那么独特。每个企业都是这样做的。

我意识到我有点过于简单化了，但是没有太多。。。

凯文:我们都有基本需求。
