# 在没有 Mac 的情况下在 Safari 中测试

> 原文：<https://www.sitepoint.com/testing-in-safari-without-a-mac/>

如果在我转而使用 Mac 电脑之前，我会把这项服务卖给我奶奶。它允许任何人输入一个网站的 URL，并下载该网站的截图，就像在 Safari 中看到的那样，其他 Mac 专用浏览器也在路上。与[浏览器摄像头](http://www.browsercam.com/)不同，这是一项免费服务！

对于不太准确，但更灵活的替代 PC 用户可能会想看看 KDE 3.2。上周发布的这个广受欢迎的 Linux 桌面环境的新版本包括 Konqueror 的更新版本，该版本整合了 Safari 开发团队对 KHTML 引擎的许多改进。Konqueror 当然不会为 Safari 兼容性测试提供一个完美的环境，但总比没有好。最棒的是，一旦 [Knoppix](http://www.knopper.net/knoppix/index-en.html) 更新到包括 KDE 3.2，PC 用户将能够测试最新版本的 Konqueror，而不必先安装 Linux。
