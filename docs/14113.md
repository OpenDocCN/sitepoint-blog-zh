# 谷歌的 TrustRank 商标

> 原文:[https://www.sitepoint.com/googles-trustrank-trademark/](https://www.sitepoint.com/googles-trustrank-trademark/)

正如[广泛报道的](http://blog.searchenginewatch.com/blog/050425-131316)和[讨论的](http://slashdot.org/articles/05/04/26/1157212.shtml?tid=217)，谷歌已经注册了“TrustRank”商标

在商标背后，我们发现了斯坦福数字图书馆项目中常见嫌疑人的一篇研究论文。本文讨论了一种通过使用可信网站的“种子集”从搜索结果中清除垃圾邮件的方法。

如论文中所述，TrustRank 使用类似于 PageRank 算法的方法来确定给定网站的可信度，或者换句话说，该网站可信的可能性有多大。

当搜索引擎申请专利或商标，收购拥有有趣技术的公司，或者当与搜索引擎远程关联的个人发表研究论文时，SEM 社区总是有兴趣。

保持对这些事情的洞察力是很重要的，因为所有主要的搜索引擎都雇佣了很多人来做研究和发明东西。仅仅因为一项专利已经被申请或被授予，并不能自动转化为搜索引擎在他们的主要搜索结果中实现该专利。

TrustRank 似乎有点不同，因为商标实际上必须使用才能得到维护。我听到有人说，如果谷歌已经注册了商标，他们必须计划使用 TrustRank。这是否意味着谷歌将实现类似于这篇研究论文中描述的 TrustRank 算法的东西？有可能，或者他们可能只是在决定做什么的时候谨慎行事。

无论如何，像 TrustRank 这样的创新将在网络搜索的未来扮演重要角色。无论搜索引擎使用手动编辑的“好”网站的“种子列表”、用户反馈还是其他手段，搜索引擎都有强烈的动机来减少其搜索结果中的垃圾邮件数量。

既然其他人都在猜测 Google 会如何实现 TrustRank，我就提出一个想法。当然，这可能是个坏主意。我会让那些实际上必须处理实际执行的人来决定…

正如在 [SitePoint SEM Kit](https://www.sitepoint.com/books/sem1/) 中所讨论的那样(你可以在样本章节中读到)，PageRank 算法模拟了一个“随机网络冲浪者”，他偶尔会对自己的随机点击感到厌倦，并在一个新页面上重新开始。在 PageRank 算法中，我们的随机冲浪者“感到无聊”的几率被表示为一个“阻尼因子”

实现 TrustRank 而不丢弃 PageRank 的一个可能方法是根据我们对某个网页的“信任”来调整阻尼因子。如果你的信任度很低，那么调整阻尼因子，增加我们的随机冲浪者对该页面感到“厌烦”的可能性。这意味着信任度较低的页面会向网站的其他部分传递较少的 PageRank 而不是惩罚它们所链接的网站，只是减少这些链接的好处，包括网站本身的内部链接。

谷歌实际上会怎么做？你的猜测和我的一样好，除非你是那种相信他们实际上实现了他们申请专利的每一个过程的人。
