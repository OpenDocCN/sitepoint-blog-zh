# 如何与优柔寡断的客户共事

> 原文：<https://www.sitepoint.com/working-with-indecisive-clients/>

你按照客户的规格制作了一个令人惊叹的模型，并提前两天提交给了客户。你只知道这是一个扣篮；你希望得到批准并马上开始开发，所以你要确保你的时间表已经准备好了。

但是两天过去了…然后是一周。你已经跟进了两次，客户说他还在仔细检查，试图决定这是否是他想要的。两个星期，七封电子邮件和两次电话交谈后，你仍然没有得到客户的批准，或任何修改要求。这个项目被正式搁置了。

听起来熟悉吗？这在商业中是一个非常常见的场景，可能会非常令人沮丧。但是有办法处理。

## 了解潜在因素

往往优柔寡断是因为缺乏自信。如果客户犹豫不决，不能为他的不确定性提供一个具体的原因，他可能只是担心他正在进行的新的努力。完成你为他做的这个项目意味着他离他的目标更近了一步，对于一个不完全信任自己的人来说，这可能是一个可怕的情况。

如果客户没有花必要的时间来概述他的计划，确定他的目标，并考虑全局，他也可能面临犹豫不决。虽然你的模型可能正是他所要求的，但他可能甚至没有意识到这一点，因为他仍然落后两步。

客户端可能处于静止状态还有许多其他原因。问一些能触及问题核心的问题，然后提供一个解决方案——无论是将项目分成更小、更容易理解的部分，还是给客户时间退后一步重新组合。

## 提供有限的选择

如果你用大量的选择轰炸客户，可能会让人不知所措。另一方面，有一种选择会让人觉得受到限制。给客户一个比较和做出选择的机会(感觉他们对这个过程有更多的控制权)，和给他们太多或太少的选择，这是一条微妙的线。

处理这个问题的一个方法是为客户提供一个解决方案(最接近他所说的他想要的那个)，然后提供 1-2 个对该解决方案的修改建议。这为您提供的解决方案提供了一定的灵活性，也为客户请求修改提供了方向。如果他能设想出一个替代方案，他就能更容易地决定什么最能满足他的需求。

## 使用时间线

在项目开始时，你可能与客户讨论了一些工作日期。但是讨论之后有多少结构呢？保持事情进展的一个方法是使用一个时间表，不仅列出项目不同部分的截止日期，还列出各方的责任。确保客户有一份这份文件的副本，并在你工作时参考每项任务的截止日期。

虽然这不是保持项目进行的万无一失的方法，但更有效的方法是制定一个带有里程碑的时间表，以及错过最后期限时的后果(即项目启动将被推迟)。有时候，一个具体的日期和一个总体构想是客户做出决定所需要的。
