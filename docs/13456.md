# 成为第一个运行 MS Ajax 1.0 Final 的孩子！

> 原文:[https://www . site point . com/be-the-first-kid-on-the-block-running-ms-Ajax-10-final/](https://www.sitepoint.com/be-the-first-kid-on-the-block-running-ms-ajax-10-final/)

刚刚发布的消息——MS Ajax 1.0 最终版已经发布！[在这里下载](http://www.microsoft.com/downloads/details.aspx?FamilyID=ca9d90fa-e8c9-42e3-aa19-08e2c027f5d6&DisplayLang=en)或查看[斯科特·格思里的博客文章](http://weblogs.asp.net/scottgu/archive/2007/01/23/asp-net-ajax-1-0-released.aspx)了解全部细节。请注意，在撰写本文时，ajax.asp.net 已停机，因此 Scott 的下载链接可能不起作用。据推测，该程序集将很快访问您的共享 web 主机。

这次发布最酷的部分可能是**他们也将发布服务器端源代码**。这有几个重要的含义:

1.  想知道他们是如何制作更新面板的吗？好了，现在你不用借助反射器就能明白了。想看看 MS 如何编写旗舰产品的 web 控件吗？现在你可以了。
2.  您可以编译自己的非 GAC 版本，用于您自己的项目。我怀疑这仍然需要完全信任，但它仍然有助于更多的自包含部署，这是我喜欢的。

源代码的发布也让我提出了一个有趣的问题:为什么不为 ASP.NET 堆栈的其他部分发布源代码呢？我希望能够浏览页面生命周期，或者观看我的 IHttpModule 在调试器中的工作。我认为这将有助于 ASP.NET 开发人员理解在被概括为“页面生命周期”的相当复杂的一系列事件中实际发生了什么
