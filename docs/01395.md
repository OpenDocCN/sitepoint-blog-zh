SVG 动画不是动画的替代品。SVG 动画是动画，它使用相同的工具。但是您对 SVG 有不同的关注。如果你正在操作一个`<div>`或者一个位图——就像你习惯于处理 png 或者 JPEGs 格式的图形一样。问题是它们是平的。如果你必须移动 JPEG 的特定部分，比如说你有一个海洋场景，里面有一艘船，你必须把船从海里切出来，然后分别移动它们，诸如此类。

它变得有点粗糙。你得做一些定位。也许你在包含单元上有一个`position: relative`。你有一个`position: absolute`在里面让船移动。SVG 是一种具有可导航 DOM 的图形。它是用数学建造的。有一种能力可以让你一头扎进去，抓住它的一部分，然后以一种静态图形无法做到的方式移动它。它为无障碍环境打开了大门。你可以像玩一根绳子一样玩它，因为你实际上有这些贝塞尔曲线命令，你可以用`requestAnimationFrame()`来操纵它们。

你可以用它做很多很酷很有趣的事情。你所知道的关于 CSS 动画或 JavaScript 动画的所有东西都可以应用到 SVG 动画中。它们不是两个不同的东西。但是 SVG 的世界为如此多的东西打开了大门，这些东西要么很复杂，要么不可能用静态图形来处理——特别是对于 UI/UX 动画。如果我们谈论的是`<canvas>`，那完全是另一回事，但是如果你谈论的是在页面上操作东西，SVG 真的很强大，而且它们超级小，所以它们真的很有性能。
