# 出乎意料的方便

> 原文:[https://www.sitepoint.com/unexpectedly-handy/](https://www.sitepoint.com/unexpectedly-handy/)

在过去的几个月里，我收到了数量惊人的关于与 Paypal 整合的请求，尤其是 Joomla/Virtuemart 和 Paypal 之间的整合。

在很大程度上，Paypal 集成往往很容易配置。然而，Paypal 的欺诈部门最近收紧了欺诈预防规则，使得我们的生活变得更加困难，因为来自“禁止”国家的单次登录会导致整个账户立即进入受限模式。一旦账户被限制，交易就受到限制，账户所有人必须经历一系列痛苦的验证，有时包括等待 PIN 通过邮件送达。

幸运的是，我们在公司网站而不是客户网站上工作时偶然遇到了这个问题。我们处理 Joomla/Paypal 工作的团队位于罗马尼亚——根据 Paypal，这是一个受限制的国家。尽管我们过去从未遇到过问题，但罗马尼亚的一次登录导致我们整个 Paypal 集成瘫痪。更糟糕的是，我们有 6 周的 Paypal 相关工作积压，需要立即投入生产，我们不能冒险导致客户的账户被自发限制。

我们在罗马尼亚的团队如何登录 Paypal 开发者网络并处理测试交易等？当他们国家的整个 IP 范围被禁止的时候？如果海上开发和墨菲定律之间的关系有一个经典的例子，那就是这个！

经过深思熟虑，我们注册了我们能找到的最便宜的 Windows 2003 VPS，为开发人员创建了来宾帐户，并要求他们使用该服务器进行所有与 Paypal 相关的活动。开发人员使用远程桌面在 VPS 上建立一个会话，然后简单地启动一个浏览器并从那里登录 Paypal。Paypal 看到一个国产 IP，大家都乐了。VPS 是最低的，因为它实际上并不服务于单个 www/ftp 站点，而且可以以每月不到 30 美元的价格租用。

起初，我对不得不每月支付 30 美元来使用一个没有被禁止的 IP 地址的浏览器感到不满。令我惊讶的是，Windows VPS 变得非常方便。

在测试服务器时，我意识到远程桌面方案在我连接速度很慢并且需要执行带宽密集型任务的情况下可能非常有用，比如通过 ftp 备份一个站点或者更新一长串 Joomla 站点。远程桌面会话不需要太多带宽，即使用户断开连接也能继续运行，这是通过移动电话连接操作时的一大优势。由于我要出差几个星期，我决定在 VPS 上安装 Outlook、Office 和全套 web 开发工具，并在我出国时将它用作我的远程办公室。

几天后，当我们从加州前往曼谷时，一场大地震袭击了台湾。我很快了解到，为泰国提供大部分连接的海底光纤网络已经受到影响，大部分连接已经中断。甚至我们的卫星连接也慢到了 56k 左右。

突然之间，30 美元/月似乎成了一笔划算的交易。当其他人沮丧地看着垃圾邮件和大附件导致他们的电子邮件客户端超时时，我建立了一个到我的 Windows 服务器的精简(640×480 + 256 色)会话，并且能够全速运行。因为我只是浏览桌面而不是下载邮件，所以没有垃圾邮件或附件会影响我的速度，我可以照常工作。

在离岸开发业务中，一个成功的公司总是为这种事情的发生留有很大的余地。当问题出现时，保持开放的心态和创造性来确定解决方法是很重要的，并且总是寻找机会来学习一些东西或者从前进道路上的障碍中受益。

我们的连接仍然慢得令人痛苦，但我能够毫无困难地发布这篇博客，而且我一直在想新的方法来利用这个 Windows VPS。我意识到，即使在最好的情况下，web 开发(尤其是离岸开发)也是复杂且容易出问题的，每一个障碍最终都会导致一个更好的长期解决方案。

我希望每个人都有一个愉快的新年庆祝活动，我期待着一个成功和快乐的 2007 年为我们所有人！新年快乐
