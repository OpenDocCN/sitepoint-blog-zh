# 回到马鞍上

> 原文：<https://www.sitepoint.com/back-in-the-saddle/>

嗯，我的日程中有一些时间出乎意料地空了出来，我重新致力于为 Sitepoint 写博客。你在我上一篇博客中说的友好的话是一个因素，我感谢你的支持。也感谢 Sitepoint management 的支持。所以让我们回到正题。

今天我们来讨论一个由我的一个咨询客户提出的案例研究。

他发现他 20%的设计/开发业务是为小客户服务的，而 80%是为大客户服务的。他雇了一个人来接待小客户，但是随着越来越多的小客户来找他，他发现自己的服务质量在下降。他该怎么办？

我的建议是:

1.他有一个好问题，不是一个坏问题。需求过多是好事，会带来很多机会。

2.每个专业人士，如果他们的业务发展得好，都会有机会转向更大的客户或提高价格。

3.提供糟糕的服务，即使是对小客户，也不是一个好主意。消息传得很快，会传到更大的客户那里。

4.我向他建议，既然他已经明智地雇佣了一个人来管理小客户(对他来说是一个杠杆，假设他有利润)，他应该保留这项业务并提高价格。这将减少需求，提高盈利能力，并在需要服务的客户减少的情况下，改善他对客户的服务。

5.他应该确保为员工提供一套一致的标准和流程，以取悦这些客户。

6.他应该推荐任何不想为竞争对手支付更高价格的前客户或潜在客户，不管有没有介绍费。关键是给竞争对手送去利润较低的客户，留下更多空间来接纳好客户。

7.他应该专注于他 80%的业务，这些业务是由更大、更有利可图的客户组成的。他的定位是什么？他怎样才能更好地利用他在那里的时间？他怎么能主宰自己的小众呢？

对于我的建议，还有其他建议或挑战吗？
