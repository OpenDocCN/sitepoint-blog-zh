# 微软的 Linux 实验室？

> 原文：<https://www.sitepoint.com/microsofts-linux-lab/>

ZDNetAsia 在本月早些时候发表了一篇有趣的文章，反映了微软内部对开源的更成熟的看法。

很明显，对开放源码的焦土政策对微软不起作用，因为他们在之前试图用煽动性言论贬低 Linux 后发现了这一点。因此，该公司已经开始建立一个内部沙盒，在其中探索从 Linux 到 Solaris 再到苹果 OS X 的 Unix 变体。Windows 制造商正在使用比尔·希尔夫(Bill Hilf)，一位终身的 Unix/Linux 专家来管理一个实验室，提供对竞争对手 OS 的秘密访问。

我发现建立实验室的考验和磨难很有意思——希尔夫在文章中揭示了这一点。它还表明这项工作可能会产生积极的结果——微软的 Linux 实验室可能会对开源应用程序做出贡献，微软可能能够并愿意改善其系统与竞争对手的操作系统的交互方式。这对他们来说是不可避免的灾难，因为许多微软的大客户也同时运行 Unix 变种和 Windows 系统。

Hilf 也可能是该公司最近与[开源开发实验室](http://osdl.org) (OSDL)就[Windows 和 Linux](http://www.zdnetasia.com/news/software/0,39044164,39251063,00.htm) 进行共享研究的驱动力之一。然而，乍一看，我不得不同意 OSDL 领导人 Stuart Cohen 在同一篇文章中的反应:

“就与微软合作的一项研究而言，微软可能会在一份 100 页的研究报告中找到一条关于 Linux 的负面消息，即它将花费 1000 万美元进行营销，而忽略其他 99 页。”
