# 我们已经变得太依赖谷歌了吗？

> 原文:[https://www.sitepoint.com/too-dependant-on-google/](https://www.sitepoint.com/too-dependant-on-google/)

最近谷歌宕机影响了他们的几项核心服务，包括搜索、GMail、YouTube、分析和广告系统。故障持续了大约一个小时，谷歌估计 14%的用户访问缓慢或中断。在此之前，今年早些时候 GMail 出现了几次中断。

我们大多数人都经历过某种服务器硬件或软件故障。无论您制定了多少计划或应急措施，服务中断都是不可避免的:意外事件总是会发生。然而，我们中很少有人负责占全部互联网流量 5%的服务。

公平地说，在十多年的持续服务中，谷歌很少经历过宕机。不幸的是，当谷歌真的倒下时，它可以带走其他系统。许多人在使用业务关键型网站(如网上银行)时遇到了问题，因为它们依赖于分析等服务。

看看你自己的投资组合。即使你不使用谷歌的网站平台，如谷歌网站或 Blogger.com，你会依赖分析、谷歌账户登录、谷歌结账、谷歌地图、YouTube 视频、AdSense 或 Feedburner 吗？还是你的业务依赖于谷歌文档、GMail、AdWords、新闻、阅读器或趋势？

在停机期间，谷歌表示，人们可以切换到竞争系统。如果用户无法访问谷歌搜索，他们可以很容易地使用雅虎、Live.com、Ask 或许多其他搜索提供商之一。但是 GMail 用户如何转换服务呢？一个使用 Google Checkout 的网店怎么可能一两个小时就切换到 PayPal。

情况可能会变得更糟。谷歌的商业模式是零成本提供伟大的在线工具和服务；我们许多人都受益于这项政策。然而，你现在会选择开发一个类似分析的网络统计系统或谷歌免费提供的任何其他服务吗？微软和雅虎都尝试过，但都不是特别成功。谷歌的工具更好，而且免费:五年后会有任何分析竞争对手吗？

互联网可能是一个分布式网络，但谷歌正迅速成为一个单点故障。

您的站点是否受到停机时间的影响？谷歌是否已经变得过于强势？我们是否过于依赖他们的服务？谷歌的失败会在整个互联网引发海啸效应吗？
