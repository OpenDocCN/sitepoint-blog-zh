# 2005 年 PHP 预测

> 原文:[https://www.sitepoint.com/php-predictions-for-2005/](https://www.sitepoint.com/php-predictions-for-2005/)

今年快结束了。我计划回顾 2004 年，但是…时间和动机不一致。

简而言之，2004 年:可能最重要的两件事是去年 7 月 PHP5 的发布和今年 1 月 PHP 星球的到来，这更像是博客成为 PHP 社区主流的象征。除此之外，它一直在向[那座山](http://www.php.net/usage.php)冲去。

**更新:**完全忘了(已经一年了)。下面是[吴镇男的](http://www.derickrethans.nl/)有趣的(无疑也是有争议的)总结(谢谢提醒)。作为品酒师，这里有 [2003](http://www.derickrethans.nl/month-2004-01.php?item=20040101#20040101) 和 [2002](http://www.derickrethans.nl/month-2002-12.php) 。

更有趣的是在 2005 年进行一些疯狂的尝试…

–随着 5.0.6 在 2005 年 7 月的发布，PHP5 将开始让人们真正感受到它的存在，向 web 主机推出它的方式，而更知名的 PHP 应用程序将开始利用 PHP5 特有的*功能*。

–在 PHP5 增加的 13 个新的[保留字](http://www.php.net/manual/en/reserved.php)中，轶事证据表明在现实世界中使用的不超过一半。

–2005 年 1 月，PHP 将获得“2004 年年度编程语言”奖

–动态四者(Perl、Python、PHP 和 Ruby)之间的[圣战](http://c2.com/cgi/wiki?HolyWar)将最终枯竭，来自所有四个社区的开发者将开始分享想法并寻找互操作的方法，偶尔的[抱怨](http://www.parrotcode.org/)提醒事情即将到来。导致…

–数量惊人的用其他语言编写的新项目[生成 PHP](http://codegeneration.net/tiki-read_article.php?articleId=19) ，利用了它的易部署性和[不断增长的](http://www.php.net/usage.php)安装基数，再加上它始终是一种模板语言的事实。

–PHP 将在构建 [P2P](http://en.wikipedia.org/wiki/Peer_to_peer) 网络中扮演一个小而有用的角色。

——“PHP”和“安全”这两个词将在智能讨论和 [FUD](http://en.wikipedia.org/wiki/FUD) 之间掀起旋风，受到进一步的[蠕虫](https://www.sitepoint.com/blog/)和 [PHP 粉丝](http://lerdorf.com/fanmail.txt)的刺激。神奇的数字 0644 将阻止世界末日的到来，ModSecurity 将开始在网络主机中传播。与此同时，PHP 源代码分析工具方面的努力将会得到加强，PHP 的平均工资将会提高…

–另一个 PHP 驱动的热门话题将会让“(广告)证明我的薪水”的人群谈论“新”范例:一个 [REST](http://en.wikipedia.org/wiki/REST) y、 [SOA](http://en.wikipedia.org/wiki/Service-oriented_architecture) y、 [dynamic](http://www.activestate.com/Company/NewsRoom/whitepapers_ADL.plex) y、敏感、感性的东西；丰富，但薄，有很好的，柔软的边缘。不管怎样，真正的工作还是要继续。

–至少有一个知名的 PHP 应用程序会弹出一个完全用 [XUL](http://www.xulplanet.com/) 呈现的后端管理界面。另一个将推出 Firefox 扩展。

谷歌将在 XUL 包装他们的火狐主页。

–Javascript 将开始被许多人认真对待。

–你的母亲会给你提供她对一个你从未听说过的火狐扩展的看法。

我似乎偏离了 PHP。有人愿意提供一些(更大胆的)预测吗？否则，明晚新年快乐。
