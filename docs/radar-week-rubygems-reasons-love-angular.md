## RubyGems

![Source: PSD100.com](img/7c1a927ee222253d6b71f76d275bb079.png)

资料来源:PSD100.com

RubyGems 是 Ruby 编程语言的一个包管理器，它为发布 Ruby 程序和库提供了一种标准格式(以一种称为“gem”的自包含格式)。

如果你曾经想从零开始创造自己的宝石，这个[两个](https://www.sitepoint.com/create-ruby-gem-jquery-plugin-basics/) [部分](https://www.sitepoint.com/create-ruby-gem-jquery-plugin-advanced/)教程将带你走过这个过程，并给你一堆方便的提示。

这里有一个很棒的列表，列出了用于 Ruby on Rails 代码优化和清理的[gem](https://www.infinum.co/the-capsized-eight/articles/top-8-tools-for-ruby-on-rails-code-optimization-and-cleanup)，以及用于促进与黑客新闻 API 交互的[gem。](https://github.com/O-I/hn_api)

## 本周流行语:渐进增强

![157-progressive-enhancement](img/5a07f1b933000beebd02deb15e829149.png)

渐进式增强是在构建网站时采用的一种做法，设计师从内容开始，然后向内容添加增强层(例如，使用 JavaScript 或 CSS)。这导致内容对所有人可用，而不管在其上观看内容的设备的能力如何。

渐进式改进真的很重要，所以我很高兴看到一个新的系列来解释其背后的理论，以及 T2 如何将这些想法付诸实践。

一旦 JavaScript 被禁用，许多网站就会停止工作。情况不一定总是如此。以下是如何使用 JavaScript 创建逐渐增强的 UI 元素。

不久前，Craig Buckler 提出[离线优先可能是下一个渐进增强技术](https://www.sitepoint.com/offline-first-next-progressive-enhancement-technique/)。下面是[的一篇更现代的文章，详述了为什么这仍然是一个好主意](http://nolanlawson.com/2014/10/03/offline-first-is-people-first/)。

这是一位设计师[在权衡渐进式改进的利弊](http://notlaura.com/progressive-enhancement/)。这篇文章包含了一些很好的类比和许多值得思考的东西。

同样，这篇文章问了“[适应性设计:它是什么，我为什么要使用它？](http://www.digitalartsonline.co.uk/features/interactive-design/adaptive-design-what-is-it-why-should-i-be-using-it/)
