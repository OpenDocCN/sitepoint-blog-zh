# 什么是链接失效，如何防止它

> 原文:[https://www.sitepoint.com/how-to-prevent-link-rot/](https://www.sitepoint.com/how-to-prevent-link-rot/)

当网站上的链接断开时，就会发生链接损坏。很简单:链接停止到它们应该去的地方，用户通常会得到某种 404 Page Not Found 错误。在网上很常见，不是吗？甚至有一些漂亮的 404 页面让迷失在网络上的访问者更容易接受这个错误。

但结果是一样的:访问者找不到他们要找的页面。这可能意味着失去销售，退出你的网站，以及不专业的印象。

当然，如果外部网站的链接很差，而这并不是你的错，你也无能为力。但是，在许多情况下，您可以控制链接在发布后是否保持活动状态。你当然可以防止链接腐烂的情况发生。方法如下:

## 明智地命名页面

如果你有一个大的站点，如果所有的东西都是随机命名且没有组织的，那么实际上是不可能跟踪什么是什么的。使用反映网站结构的类别，然后对每个文件使用通用的命名约定，如日期、关键字和/或缩写。

这也可能是一个好主意，特别是如果你有一个不是基于 CMS 的大型网站(是的，他们仍然存在！)，以保存一个正在使用的文件命名约定的主列表，这样您就有一个快捷方式来查看特定链接应该指向哪里。

## 检查你的网站统计

你的分析软件可以告诉你很多关于断链的信息。您应该能够看到访问者使用什么 URL 来访问您的站点，从而导致 404 错误。使用此信息来修复页面、创建重定向和解决链接问题。

例如，如果你使用 Google Analytics，你可以[修改你的跟踪代码](http://www.google.com/support/googleanalytics/bin/answer.py?hl=en&answer=86927),在你的报告中包含丢失的页面名称和引用的 URL。

## 使用重定向

如果你出于某种原因重组、移动或更改页面名称，使用 [URL 重定向](http://en.wikipedia.org/wiki/301_redirect)将站点访问者从失效的链接移至你的新页面。请记住，即使你更新了所有的内部链接，转到新的网页，同样的情况可能不是外部网站链接到你。一个永久的重定向将修复这些损坏的链接，即使链接网站没有。

## 定期检查你的链接

虽然验证所有链接都是活动的并不能保证站点访问者得到了他们认为他们得到的页面，但是定期检查站点上的所有链接绝对是一个好习惯。有许多链接验证器可以让你自动完成这个过程。或者你可以使用像 W3C 链接检查器这样的工具。

## 对网址缩写要挑剔

网址缩写特别容易受到链接损坏的影响。如果你使用短 URL 链接到你的页面(现在谁不这样做呢？)，选择一个有良好记录的服务，并不时检查自己的链接，以验证它们是否有效。或者更好的是，[创建你自己的网址缩写](https://www.sitepoint.com/article/get-shorty-coldfusion)。

## 使用永久链接

如果你有博客或论坛，启用永久链接可以大大降低链接损坏的可能性。当您选择 permalink 格式时，请仔细考虑选择过程(如果您的平台提供此功能)，以便它适用于您现有的文件结构。

## 给网站访问者一个举报的途径

你可能会惊讶于网站访问者报告断开链接的意愿……如果你让它变得容易的话。添加一个选项，直接从您的 404 页面报告错误。您还可以创建一个报告表单，获取访问者试图访问的页面的 URL，这样只需点击一两次就可以报告问题。
