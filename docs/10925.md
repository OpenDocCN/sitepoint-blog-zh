Craig Buckler:请简单介绍一下你自己、你的工作以及你是如何加入 web directions @media 的。布鲁斯·劳森:我为 Opera 工作，传播开放网络。2009 年 2 月，我重新设计了我的博客，使用 HTML5(据我所知，我是第一个这样做的)。这真的是一种自学的方式，因为我发现通过阅读说明书来学习更容易。似乎我引起了很多人的兴趣，所以约翰*【约翰·奥尔索普，网络方向联合创始人】*和玛克辛*【玛克辛·舍林，网络方向联合创始人】*请我讲话。CB:你以前参加过几次大型活动。你是在刺激中茁壮成长，还是仍然紧张不安？这仍然令人伤脑筋；我一般上台前都会吐。CB:你的演讲题目是“HTML5:结构、语义、风格和性感”。HTML5 真的那么性感吗？BL:嗯，有些部位很性感。画布和视频很性感。事实上，有些人认为 HTML5 =动画和视频(它们只是规范的一部分)。但后来其他人认为 HTML5 是地理位置，CSS3 或 SVG 这是完全错误的。我是一个标记极客，所以我觉得这 28 个新元素、新形式和增强的可访问性潜力很有吸引力。我希望你也是，因为这就是我要展示的。 **CB:你认为今天开发者可以使用 HTML5 吗？** BL:是的，有些部分。如今，画布、视频和本地存储都是可用的。目前有如此多的创新，每个月都有更多的浏览器支持。但是需要强调的是，你不一定非得使用 HTML5。您当前的网站将继续运行，HTML4 和 XHTML1 仍然可以在任何地方使用。如果你需要 HTML5，尽一切办法学习它——但不要重新发明轮子。 **CB:在你看来，HTML5 最好和最差的方面是什么？BL:它在保持向后兼容的同时推进了网络。这很好，但是当你看到一些丑陋的 API 和标记时，它会导致一些*“WTF”*时刻。例如，拖放 API 是特别明了的——但它现在可以在浏览器中工作。如果 HTML5 采用了一个新的通用标题< h >(正如夭折的 XHTML 2.0 所规定的那样)，那么分段算法就更容易理解了。但是这不是向后兼容的，所以我们保留了< h1 > … < /h6 >。那不是 bug，是特性。在不破坏网络的情况下推进网络是一个非常不同的循环。你希望我们在你的演讲中避免问什么尴尬的问题吗？！我希望没有人问起关于我和康妮·胡克在按摩浴缸里的传闻！非常感谢布鲁斯。** *见 Bruce 于 2010 年 6 月在伦敦 [Web Directions @media 的演讲](https://secure.webdirections.org/wdatmedia/aff/SitePoint)。注册时使用促销代码 **SitePoint** 可获得 100 英镑的折扣。*
