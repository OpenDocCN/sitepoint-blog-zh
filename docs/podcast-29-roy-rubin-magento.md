## 显示抄本

布拉德:欢迎来到 SitePoint 播客的另一集。我是主持人布拉德·威廉姆斯，今天和我一起的是广受欢迎的开源电子商务软件 Magento 的创始人罗伊·鲁宾。

欢迎来到节目，罗伊。

罗伊:嘿，布拉德，很高兴来到这里。

布莱德:有你在真好。我知道有很多关于 Magento 的传言。这确实是目前最热门的电子商务开源包。所以我很高兴你能来，因为我自己也有很多问题，我们也有很多来自社区的问题，所以我们可以一起了解 Magento。

所以让我们直接进入正题，因为我们有很多内容要讲。我想从最开始说起，实际上，我的第一个问题来自 SitePoint 成员 IceMan90 和 IceMan，他们想知道，既然 Zen Cart 和 OScommerce 在开源购物车市场上统治了多年，是什么让你决定是时候让这个市场领域的另一个主要参与者出现了？

Roy: 当我们在 2007 年初创建 Magento 时，我们当时正在与 OScommerce 合作；我认为禅宗车少，但更多的 OScommerce，作为一个服务机构，多年来。随着我们的业务增长，我们的客户对我们有了更复杂的期望，互联网也在不断发展，我们非常清楚，OScommerce 和 Zen Cart 及其众多分支的时代已经真正结束了，是时候让一个更成熟、更先进的平台真正进入市场，满足商家的需求了，因为技术在不断发展，市场在不断发展，我希望我们和 Magento 一起做到了这一点。

你今天在 Magento 中看到的许多功能来自于我们多年来在 OScommerce 和 Zen Cart 等方面的经验。所以这真的是它的下一次进化。我认为我们的背景真的有助于我们设计出一款能真正表达这一点的产品。

**布拉德:**你提到了 OScommerce 最初我听说你计划发展 OScommerce，但是你实际上放弃了这个计划，决定从头开始，事情是这样发展的吗？

罗伊:不，事实并非如此。

**布拉德:**好奥莱维基。

罗伊:是的，你知道，这会是一个好故事。但是没有，我们从来没有真正想过使用 OScommerce 代码库。我认为，当我们开始真正整合项目的愿景时，我们知道架构将会完全不同。OScommerce 今天提供了什么，现在还在提供什么，我们根本无法与之合作。所以，从一开始，我们就知道我们必须整合一些不同的东西。

布拉德:你会说这比你预想的要难吗？在我看来，当我想到电子商务时，电子商务涉及的内容太多了。因此，从头开始创建一个电子商务网站，所涉及的金额让我大吃一惊。比你想象的容易，比你想象的难吗？结果如何？

罗伊:我认为这比我们任何人想象的都要复杂。从功能集的角度来看，我认为我们今天所处的位置和两年前我们认为的位置是完全不同的。这是一个巨大的工程。我们投入了令人难以置信的时间，由一个令人难以置信的热情和敬业的团队。

事后回想起来，至少可以说，这是一件没有预料到的事情。

Brad: 当你刚开始的时候，Magento 的实际开发团队有多大？

罗伊:我说过，我们在 2007 年初就开始了。我们大约——我想至少有三四个人开始思考建筑会是什么样子。那确实是我们工作了大约 5-6 个月的核心团队。之后，我们开始壮大团队。在 1.0 发布之前，这个团队增长到大约 35 人。它最终成为了一个非常非常大的内部项目，我们将所有的人都转移到了 Magento，并确保我们提供的包不仅仅是一个技术包，而是真正为用户创造了正确的体验，该网站真正提供了他们做出明智决策所需的信息，我们整个公司都围绕着这个项目，显然，随着我们越来越接近 1.0，团队变得越来越大。

Brad: 我想澄清一下，你所说的公司是 Varien，是一家软件开发公司；对吗？

罗伊:是的，从历史上来看，就是这样。我在 2001 年创立了这家公司。今天，你知道，我们显然是专注于 Magento 这是我们的核心产品，也是我们真正关注的焦点。在 2007 年末到 2008 年初之前，我们是一家传统的咨询网站，做很多不同的事情。

**Brad:** 在这么短的时间内，它从像你说的那样是一个经典的开发商店变成了现在你开发的这个软件包 Magento，这真是太棒了，它是所有的嗡嗡声，真的很鼓舞人心。实际上，我还经营着一家软件开发公司，所以我可能会把自己放在那个经典的桶里。我们做传统的开发工作。所以看到你在很短的时间内做了什么是非常简洁的。Magento 已经存在两年半了，你的下载量已经超过一百万了，是吗？

罗伊:是的，现在下载量已经超过了 110 万次，而且我们的下载量和增长率都非常高。所以这是非常积极的，增长仍在继续。我们非常兴奋。

也许我可以谈谈从咨询公司到产品公司的转变，我可以告诉你，这从来不是一个容易的转变。这是我们仍需解决的问题。我想我们已经到了尽头。我认为这是许多咨询公司渴望的，拥有自己的产品，并以此为基础。我非常鼓励它；我认为有很多……当你看到像这样的产品，你的产品，真正进入市场并开始被一些人采用时，会有很多回报。很刺激。

布莱德:太棒了。我真的很尊重这样一个事实，你不仅创造了这个伟大的产品，而且你还以开源的方式发布它，在那里你可以很容易地保持它的所有权并收取许可费，但你没有。你在开源许可版本 3 下发布了它。我想知道你是否能说一下——它总是从第一天开始，你计划发布这个开源吗，或者它只是进化成那样？

Roy: 从第一天开始，当我们开始这个项目时，我们就非常清楚这将是我们的战略。我们来自开源的根源，这是我们所熟知的。我们过去非常相信，现在仍然相信这个模型。我们看到其他玩家和其他垂直市场继续表现良好。我们看到了 MySQL，我们看到了 SugarCRM，Alfresco 和其他商业开源播放器，我们明白在电子商务领域有一个真正的机会，这真的促使我们相信有一个围绕分发自由软件的商业模式。我认为两年后的今天，我们仍然非常相信这个模型。

布莱德:太好了。就需求而言，Magento 运行在 Linux 平台上，对吗？

罗伊:的确，是的。

Brad: 我在看需求，但没有列出 Windows。它能在 Windows 上运行吗，或者它完全不能在 Windows 上运行？

**罗伊:**你知道，它可以在 Windows 上运行；我觉得配置有点棘手。我们一直在与 PHP 公司 Zend 合作，实际上，如果你在他们现有的 Zend 服务器产品上运行 Magento，事情会变得容易得多。这对我们来说是一个很大的优先事项，所以我们希望继续与 Zend 合作，让那些喜欢 Windows 而不是混合类型环境的组织可以访问 Magento。希望这能回答这个问题。

Brad: 有没有计划真的发布它，让它可以直接安装到 Windows 上，像 WAMP 堆栈一样运行，或者它总是需要 Zend？

Roy: 再说一遍，我认为今天你可以在 WAMP 堆栈上运行它。Magento 中没有任何内容禁止您这样做。我们正努力使这个过程变得简单一点。这个想法是用一个 Zend 服务器(有一个免费的社区版本的 Zend 服务器)在 Windows 中安装几乎是无缝的。

今天有点棘手，但我们希望简化这个过程。

Brad: 我想谈谈 Magento 背后的许可，我之前提到过它是在开源许可(OSL 3.0)下许可的。我想知道你为什么选择那个许可证，而不是更传统的许可证，比如 GPL 或 LGPL？

罗伊:OSL 许可证实际上是开放软件许可证，而开放软件许可证是一个封闭所谓 ASP 漏洞的许可证。ASP 的漏洞基本上是——开源社区对什么是分发有很大的争论，通过 Magento，我们把网络上的分发视为 GPL 下的经典分发。

那是什么意思？也就是说，如果你创作了 Magento 的衍生作品，你只是把它放在一个网站上，并通过网络以数字形式发布，这仍然被认为是一种发布，因此，由于 OSL 许可是一种互惠许可，你的作品必须在 OSL 许可下获得许可并公开。

我们为什么选择这个？我们之所以这样选择，是因为我们觉得我们在 Magento 上投入的工作意义重大，并且我们希望社区能有所回报，这样如果有衍生作品，Magento 社区的其他人也能从中受益。

当我们选择 OSL 许可证时，它是唯一一个认证开源许可证的 OSI(开源倡议)组织，是唯一一个 OSI 批准的堵住 ASP 漏洞的许可证。从那以后，LGPL 许可证进入了市场，现在这两个许可证，也许还有其他的，都是可用的，但在我们选择 OSL 的时候，它是唯一一个 OSI 批准的封闭 ASP 漏洞的开源许可证。

Brad: 开源许可可以……这是一个有趣的话题。这也是一个很难理解的问题，尤其是当你进入实际案例中，什么是允许的，什么是不允许的。所以人们可能会有点困惑，但在它下面发布是件好事。我说了什么？开源许可证…我指的是开放软件许可证(OSL)。所以它实际上并不是 GPL，这可能是很多人所期望的。

罗伊:当然，这不是 GPL，两者有很大的区别。我们试图就此教育市场。在这方面，我们可能还有很多工作要做。但是这是一个非常非常不同于 GPL 的许可，它有很多好的理由。

**Brad:** 我们来说说 Magento 的特性。显然 Magento 有很多特性。如果你去 MagentoCommerce.com 的(这是 Magento 的官方网站)，就在产品下面，你可以去功能，那里有一个很长的不同功能的列表。

Magento 有哪些特性？这是大多数人对电子商务商店的期望吗？有没有一些你平时看不到的、你认为很酷的附加功能？

Roy: 我认为 Magento 与众不同的首要原因不仅在于市场上的开源软件包，还在于许多专有软件包，甚至是企业级软件包，即它能够进行多商店零售。同样，这是我们在 2007 年初发现的一种趋势，我认为这绝对证明是正确的，零售商正在寻求建立多个在线商店，从 Magento 的一个中央管理实例运行多个在线业务。

这是内置于核心产品中的东西，是我们非常感兴趣的东西，它已经非常非常受欢迎。

还有什么？我认为我们的定价规则和折扣引擎也非常非常强大。创造真正复杂的折扣类型的能力已经得到了广泛的认可。也许我认为另一件引起很多关注的事情是我们的目录管理。我们为商家提供了真正灵活的能力来管理各种不同的目录类型。如果你有虚拟产品，如果你有实体产品，如果你在多个领域做生意，Magento 的目录结构和 Magento 建立目录的过程可能会支持这些。

因此，作为一个软件产品，我们试图置身于零售商之外，不告诉他们如何经营他们的业务，而是给他们一个平台和一个产品，让他们能够真正建立他们认为合适的一切。

Brad: 网站管理员，这是一个很酷的功能，我想我们能不能谈一谈。如果我理解正确的话，我可以在 Magento 的一个安装下拥有多个商店。这是否允许我拥有这些商店的多个域，或者它们仍然像主网站的子域或子目录一样进行映射？

罗伊:你要分手了，但我想我明白了其中的要点。你是绝对正确的，你实际上可以在不同的域下运行 Magento 商店；这绝对没有问题。如果更适合您，您可以在子域下运行它，或者在不同的域下运行，或者在同一域上运行，我们为您提供了完全的灵活性。

Brad: 它实际上把所有东西都分开了，所以是分开产品群、分开客户，还是把所有东西都放在一起？

罗伊:实际上，你可以在商店之间共享顾客，或者设置不同的顾客账户。你可以在商店之间共享购物车，也可以不共享。这是您可以配置的选项。同样，您可以在不同的域中运行您的商店。

我们的公司有多个品牌，并为他们的许多品牌运行一个 Magento instants。你甚至不能说这是同一个瞬间。所有品牌的用户体验完全不同。它要求每个品牌都有单独的注册和用户登录，然而，这一切都是通过一个 Magento admin 集中管理的。

布莱德:太棒了。老实说，我不知道 Magento 这样做，但这真的很酷，因为从我自己的经验来看，我知道当我们为客户建立电子商务商店时，很可能在我们完成后几个月，他们会回来并想要另一个商店。所以我们又建了一家店。但在这种情况下，我们可以使用站点管理，在他们现有的安装中创建另一个商店，这太棒了。

罗伊:对。你不受限制。如果你想拥有完全定制的用户体验，在这些商店之间拥有完全定制的功能集，这是你可以做到的。同样，我们真的置身事外，让您经营业务，并提供足够强大的平台来真正为您管理业务，而无需为您规定业务规则和逻辑。

布莱德:太好了。我试图调出 Magento 的路线图，看看我们在未来几个月和几年可以期待什么，那里有一个帖子解释说路线图已经-控制权已经移交给社区咨询委员会。我想知道你能否解释一下什么是社区顾问委员会，这对 Magento 的未来和即将到来的新功能意味着什么？

罗伊:当然可以。首先，也许我会设置场景。今天真的有两款产品。一个是 Magento 社区版，这是我们在 2008 年 3 月第一次推向市场的产品，1.0 版本已经推出。大约一年后，我们发布了企业版产品。

企业版产品由 Varien 控制。我们继续为它添加功能。我们与我们的合作伙伴和客户进行了大量的磋商，并真正将该产品面向那些在线业务量比社区版用户大一点的商家。

我们试图用社区版产品尽可能地向社区开放。因此，在许多层面上都有一些倡议。

一个是我们已经启动了社区顾问委员会，社区顾问委员会由我们的一名员工 Koby 领导，我们已经召集了来自世界各地的社区的许多不同的人，这些人是为企业版做出产品决策的人，他们在涉及到[MagentoCommerce.com](http://magentocommerce.com)时做出产品决策，他们是与当地社区联系并获得反馈的人，并在我们做出决策的社区顾问委员会会议上真正过滤这些反馈。

我们与社区顾问委员会一起做的第一件事是建立一个用户语音帐户，社区可以在其中向我们传达他们认为社区版中最紧迫的事情。

本月，我们还首次向外部贡献开放了我们的代码库。现在，我们基本上已经开发了 99.9%的 Magento 核心代码，我们希望慢慢开放这些代码，并引入第三方贡献者。社区中的很多人都表示有兴趣帮助我们解决这个问题。这就是我们正在做的。

在未来的一年、两年或三年内，我们希望看到一个由 Varien 领导的社区推动的社区产品，它从社区中获得核心贡献，Varien 在社区的协助下，将继续发展社区版产品。

企业版产品将继续走自己的路，在很大程度上也是由我们领导的，但同样，只是专注于不同的客户群。

Brad: 一旦路线图确定下来，就会再次公开发布，对吗？

罗伊:是的，是的。不发布 community edition 路线图的真正原因是因为我们不希望人们从可能尚未最终确定的信息中做出任何商业决策，无论我们要求人们耐心等待多少次，重要的是不要误导和提供任何错误的信息。一旦我们根据我们的机构群体咨询委员会做出最终确认，我们将公开该信息。

我们也有企业产品的公共信息，但那是在一个单独的层次上，当然。

Brad: 你能否解释一下 Magento 扩展是什么，它们具体能做什么？

罗伊:当然可以。我们建立了一个叫做 Magento Connect 的东西，这是 Magento 扩展市场，我相信今天有超过 1200 个扩展，相当多的商业扩展，相当多的开源扩展，范围从支付网关；我们现在支持产品中的 200 多个支付网关、运输集成、语言传真、主题、功能增强、后端功能增强、前端功能增强以及各种其他零碎功能。

所以，我们今天把 Magento 看作是一个平台，一个我们非常鼓励扩展的平台，我们只希望我们的扩展开发者成功，并有很多计划继续帮助他们成功，并希望把他们所做的工作公之于众。

**Brad:** 是的，Magento Connect 吸引我的一点是，你不仅有社区提交的插件，还列出了商业插件，以及价格和可以购买这些插件的链接。

我知道在开源社区中，有一半的人讨厌它，一半的人喜欢它，所以我认为你们显然喜欢它，所以你们有商业插件。这是在发布时出现的，还是后来才加入的？

罗伊:不，那是在发布会上。我们很大程度上来自开源社区。同样，这是我们过去几年的根基和传统，我们对此深信不疑。但这并不意味着我们可以告诉其他人如何运营他们的业务，当我们开始推出 Magento 并真正思考扩展框架会是什么样子时，我们非常明白，我们首先需要的是由最好的开发人员提供最好的扩展。为了做到这一点，真的需要一个适当的激励机制，让那些开发人员继续做出贡献，继续构建真正高质量的扩展。要做到这一点，你要鼓励他们，给他们提供一个设施，让他们可以为自己的工作收费。从长远来看，我们希望会有开源扩展，我们非常努力地推动这一点。

事实上，我们今天所有的扩展都是开源的；我们没有收取任何延期费用。这就是我们经营业务的方式。但是我们非常理解对开发高质量工作并期望支付帐单的第三方开发者的补偿的需要。对我们来说，这是开源的一部分。我们只是不认为零的价格标签从长远来看是可行的，我们希望这个项目将伴随我们很长一段时间。

Brad: 是否担心所有的扩展都是商业化的，因此没有任何免费的东西可供我下载和使用，所有的东西我都必须付费？
