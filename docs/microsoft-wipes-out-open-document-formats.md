# 微软清除开放文档格式

> 原文：<https://www.sitepoint.com/microsoft-wipes-out-open-document-formats/>

随着周四宣布致力于互操作性，微软成功地将 OpenOffice.org 的 OpenDocument 格式送进了坟墓，将自己的文件格式作为每个人都应该支持的行业标准。

好的，所以微软的`.doc`、`.xls`和`.ppt`文件格式对于文字处理文档、电子表格和演示文稿来说已经是事实上的标准了。但是随着史蒂夫·鲍尔默和雷·奥茨的声明，微软将开放定义这些文件格式的技术规范，微软已经迈出了一大步，将它们整合为数据交换的行业标准*。*

 *上周的大部分新闻评论都集中在微软明确不会因为开源开发者连接到微软软件而起诉他们的声明上。但在我看来，这里更大的故事是，通过明确地免费提供这些 API，微软本质上是将他们自己的文件格式变成了标准，消除了任何保持开放文档格式的真正理由，例如 OASIS consortium 的[开放文档格式](http://en.wikipedia.org/wiki/OpenDocument)(OpenOffice.org 套件的[使用的默认文件格式)。](http://openoffice.org/)

一堆问题浮现在脑海中，包括这个:微软准备在向开发者提供其规范方面走多远？(到目前为止，承诺的 30，000 份文件中只有大约 30 份可用。这仅仅是我们完成工作的最低要求吗，或者这种互操作性的承诺是真的吗？开放协议之后，开源微软任何应用程序背后的代码是下一个合乎逻辑的步骤吗？

你怎么看？微软的互操作性承诺是一个没有实质内容的空洞的公关活动，还是意识到了开放对于生存的重要性？ODF 还有机会吗？*
