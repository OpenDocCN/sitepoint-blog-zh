# 英国政府拒绝 IE6 升级申请

> 原文：<https://www.sitepoint.com/british-government-rejects-ie6-upgrade-petition/>

2010 年 2 月，我报道了英国公民可以签署一份网上请愿书，要求所有政府部门更新 Internet Explorer 6。6 月 6 日的最后期限已经过去，[政府已经发布了他们的回应](http://www.hmg.gov.uk/epetition-responses/petition-view.aspx?epref=ie6upgrade)。你不会高兴的——他们保留了 IE6。

这很遗憾，但我们不应该感到惊讶。这份请愿书只吸引了 6223 个签名，因此很难称得上是英国人民的授权。这是一个合理的网页设计者和开发者的数量，但是，因为我们是主要的受益者，没有人可以说这是公正的。

请愿书最大的错误是将安全列为主要关注点:

> IE6 有一些安全缺陷，让用户容易受到攻击。这两个政府(法国和德国)已经让他们的人民知道，升级将使他们在网上更安全。我们应该跟着他们。

这个问题太模糊了，可能会被指责为危言耸听。政府的回应是:

> 复杂的软件总会有漏洞，有动机的对手总会努力发现并利用这些漏洞。没有证据表明，从最新的完全补丁版本的 Internet Explorer 升级到其他浏览器会使用户更安全。定期的软件修补和更新将有助于抵御最新的威胁。政府继续与微软和其他互联网浏览器供应商合作，以了解 HMG 使用的产品的安全性，包括 Internet Explorer，我们欢迎微软继续努力提供安全解决方案，尽快部署到所有 Internet Explorer 用户。
> 
> 每个部门负责根据政府信息保障政策和国家信息保障技术机构 CESG 的技术建议管理其 IT 系统的风险。该建议的一部分是，定期的软件修补和更新将有助于抵御最新的威胁。应由各个部门根据这一明确的指导来决定如何最好地管理风险。

IE6 有更多的漏洞，但它也获得了十年的安全补丁。在欧洲，浏览器的市场份额已降至 3.5%以下，因此它不再是黑客的首要目标。最后，政府部门有严格的安全系统:当用户无法访问外部网络时，他们不容易被感染。

或许在经济不那么具有挑战性的时期，请愿会有更好的机会。政府回应的最后一部分强调了复杂性和纳税人的成本:

> 对于英国政府部门来说，升级他们系统上的 IE 版本并不容易。将这些系统升级到 IE8 可能是一个非常大的操作，需要数周的时间进行测试并向所有用户推广。测试英国政府各部门目前使用的所有网络应用程序可能需要几个月的时间，这对纳税人来说是一笔巨大的潜在成本。因此，在许多情况下，继续使用 IE6 并依靠防火墙和恶意软件扫描软件等其他措施来进一步保护公共部门的互联网用户更具成本效益。

英国新政府已经开始大规模削减成本。当成本可以与失业、护士工资、教育和国防预算直接比较时，公民不太可能接受花费数百万美元进行收益微不足道的 IT 升级。

我们的问题是，在互联网时代，12 个月是一段很长的时间，而且浏览器升级很容易。然而，大多数政府 IT 项目的时间跨度最短为 5 到 10 年，而且它们采用的技术是可靠的(它们已经过时了)。甚至那些正在升级的部门也只是刚刚转移到 IE7。这令人沮丧，但即使他们今天实施了 Firefox 3.6 或 Chrome 5，我们也会要求在几个月内进一步升级。

最终，你有一个简单的选择。如果你不想为 IE6 做准备，就不要从事有要求的工作。
