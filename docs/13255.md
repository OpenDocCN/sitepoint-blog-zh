# 为什么是可访问性？因为这是我们的工作！

> 原文:[https://www . site point . com/why-accessibility-因为这是我们的工作/](https://www.sitepoint.com/why-accessibility-because-its-our-job/)

当想到网页可访问性时，我会经历自满和怀疑的阶段。

在 SitePoint 总部工作，我相当幸运，因为这里的每个人在可访问性方面都是一致的；我们花时间来确保我们的网站是可访问的，我们的应用程序优雅地降级，我们的 JavaScript 不会给屏幕阅读器用户制造障碍。我们这样做是临时的，甚至不知道我们的目标人群最终是否需要它。没有任何理由，除了**这是我们的工作**。在我们出版的书籍、文章和其他内容中，最佳实践位于优先列表的首位。

但并不是我们社区的每个人都同意，正如最初的论坛对目标案例的讨论和最近 TechCrunch 上的争论所显示的那样。我认为这很可悲——不是因为这些人本身不同意(我更希望看到健康的辩论，而不是默许)，而是因为这么多反可访问性的论点是无知的言论，它们掩盖了这个问题，使每个人都更难进行有益的讨论。他们通常遵循一种熟悉的模式，采取一种独特的美国路线，支持市场力量决定一切，并批评任何理想主义的意图是政治正确变得疯狂。

随便啦。我不打算讨论这个特殊案例的是非曲直；这是一个深深触及我们所认为的“权利”核心的问题，这是一个我不想打开的难题。我知道我在想什么；你知道你在想什么；说够了。

但我将借此机会重申我所相信的我们职业精神的本质。**我们应该努力创造无障碍内容，[匿名](http://anonymx.com/ "anonymous")，因为这是我们工作的一部分。**而且坦白说，这并不需要太多的努力；不难。

没有人期待完美。没有一个头脑正常的人会认为人人平等，或者生活是公平的。人与人是不平等的，生活是不公平的。但这使得我们在遇到失衡时试图纠正失衡变得更加重要。

我们所做的不是火箭科学——我不会说这很容易，但也不是特别困难。 <abbr title="HyperText Markup Language">HTML</abbr> 的设计考虑到了可访问性，它提供了钩子和元信息来使内容更易访问:`alt`文本用于图像；`caption`、`summary`和表格标题；良好的标题结构；段落、列表和其他结构标记的语义使用。如果使用得当，我们的工具可以完成这项工作。如果使用不当，它们会造成障碍。

而技术是人类努力的一个领域，在那里这是完全不可接受的。技术不像物理世界，在物理世界里，有些东西永远无法接近，这有很好的、切实的原因。一个盲人永远不会赢得大奖赛；坐在轮椅上的人永远也爬不上乌鲁鲁。技术不像物理世界——技术可以采取任何形式。技术是我们的奴隶，我们可以让它做我们想做的事情。

如果我们称自己为专业人士，我们应该为我们的客户、他们的客户和我们自己做好我们的工作。厨师必须关心健康，建筑工人必须关心安全，我们必须关心可及性。
