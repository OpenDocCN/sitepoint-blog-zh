# 微软+网络标准= Expression Web 4

> 原文:[https://www . site point . com/Microsoft-we b-standards-expression-we b-4/](https://www.sitepoint.com/microsoft-web-standards-expression-web-4/)

在最近的[三篇文章系列](https://www.sitepoint.com/article/expression-blend-behaviors)中，用户体验专家 Shane Morris 向我们展示了使用 Expression Blend 设计 Microsoft Silverlight 应用程序的基础知识。在[今天的文章](https://www.sitepoint.com/html-css/)中，我们的最新作者 Tatham Oddie 向我们介绍了 Expression Blend 的姐妹产品 Expression Web，并在 Web 标准的背景下为我们概括了它的标记和样式功能。

Tatham 首先指出，Expression Web 不依赖于微软的服务器端产品，因此无论您使用的是 HTML、ASP.NET 还是 PHP，都值得研究一下。然后，他赞扬了向 web 标准构建，并提到了这样做的一些好处(例如，用 CSS 布局可以更快地加载)。

![](../Images/452a32591257e29f35666290c570947b.png "Expression Web 4")

随着我们的继续，我们将会看到在开始编写代码之前确定您想要的目标标准是多么重要，以及 Expression Web 是如何实现这一过程的。它的智能 HTML IntelliSense、内置的“兼容性检查器”和 HTML 风格的“查找和替换”功能都使 web 标准变得更加简单。
