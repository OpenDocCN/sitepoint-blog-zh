# Evri 试图公开内容连接

> 原文:[https://www . site point . com/evri-attempts-to-expose-content-connections/](https://www.sitepoint.com/evri-attempts-to-expose-content-connections/)

上周， [Evri](http://www.evri.com/) 发布了他们的内容发现引擎公测版，它[招致了一些顶级博主](http://www.techcrunch.com/2008/09/24/evri-unleashes-an-open-beta-but-falls-short-on-results/)的批评，认为没有足够的内容。这可能是真的——该网站处于测试阶段，目前的主题非常狭窄——但这也可能忽略了 Evri 试图实现的目标。本周早些时候，我与 Evri 首席执行官尼尔·罗斯曼进行了交谈，以了解内部人士对这家西雅图公司目标的看法。

Evri 是一个语义发现引擎。罗斯曼告诉我，他们认为搜索已经不足以挖掘大量的信息。虽然罗斯曼强调 Evri 并没有试图与搜索竞争——搜索做得很好，他说，在搜索中添加自然语言处理对最终用户没有真正的价值——但他不认为搜索在揭示它找到的内容中实体之间的关系方面做得足够好。

Evri 不是让用户搜索内容，而是使用自然语言处理来尝试理解它找到的内容并将其连接起来，从而创建该公司所谓的网络“数据图”。这个想法类似于脸书这样的网站为我们的社会联系展示的社交图，但适用于网络内容中提到的实体之间的联系。

例如，Evri 可能会看到一篇关于布莱思·丹纳的文章，发现她是一名演员，然后将她与格温妮丝·帕特洛联系起来，让用户发现丹纳是帕特洛的母亲。Roseman 告诉我，现在 Evri 理解了它发现的许多连接的*方式和原因*,但是他们还没有想出一个好方法来在 UI 端向用户公开这些信息。

当然，与大多数当前的 NLP 搜索或发现引擎一样，这项技术并不完美。该引擎似乎永远不会像一个见多识广的人那样了解一个主题，但它可以为用户指出算法搜索引擎可能很难揭示的联系。这基本上是当前的目标。罗斯曼希望用户能像往常一样在其他搜索引擎上进行搜索，然后来 Evri 进行更深入的挖掘，找到他们感兴趣的主题的相关信息。他甚至认为 Evri 页面可能会进入传统的搜索引擎结果。

目前，Evri 正在构建一个目标站点，重点是为探索其 NLP 引擎找到的连接创建一个简单而愉快的用户体验。最终，我得到的印象是 Evri 希望成为一种后端技术，人们可以在 API 的基础上构建语义搜索和发现工具。罗斯曼告诉我，他们正在建设的目标网站将作为开发者的技术演示——如果用户发现他们所创造的东西有价值，那么开发者就会想加入。

在这方面肯定有一些令人兴奋的未来前景。 [Evri 搜索页面](http://www.evri.com/mainline-ui/jsp/index.jsf)，提供了对后端 NLP 分析如何发现连接的窥视，也提供了对可能在最终的 Evri API 上进行的研究应用类型的观察。

例如，搜索“[Organization/Name]> acquire >[Organization/Name]”，会调出 Evri 已知的已经收购或买断其他组织的公司列表。

![](../Images/8c8e56d1081bcc2947a85508a250bc0f.png)

您会注意到结果中有许多重复，但这类事情暗示了 NLP 发现引擎可能的研究应用类型。暴露联系的查询可能有强大的研究应用——例如计算格温妮丝·帕特洛穿了多少次维拉王参加颁奖典礼，或者某个特定的政治家多久访问一个特定的州，或者某个乐队在特定的城市演出了多少次，等等。

然而，这里有一个规模的问题。现在 Evri 非常了解政治家和名人，Roseman 告诉我有计划将产品的知识领域扩展到商业、技术、健康和医学。但是添加知识需要人力——创建和寻找好的种子列表，开发和管理本体，等等。引擎自己会学习很多东西，但正如我前面所说，我认为它永远不会像人类专家那样了解特定主题。要扩展 Evri 的知识广度以涵盖谷歌能找到的所有内容可能是不可能的。

还有一个潜在的*物理*规模的问题。与关键字索引不同，自然语言本体通常比它们索引的文档大。例如，因为 NLP 引擎必须试图找出文档中的每个代词以及它所指的人，所以 NLP 本体必然很大。随着实体数量的增长，存在潜在的缩放问题。

## 结论

我一点也不失望，反而觉得埃夫里很有趣。虽然目的地界面有点笨拙，而且我不知道 Evri 是否会成为一个消费者目的地网站，但我对基于该公司的自然语言处理技术和连接索引开发应用程序的最终前景感到兴奋。

Evri 正在做的一件聪明的事情是为 bloggers 提供 [widgets，这些 widgets 从语义上分析帖子，并根据用户写作中发现的实体将用户指向相关内容。这不仅会让更多的人接触 Evri，还会给引擎带来更多潜在的内容。](http://www.evri.com/partners-and-bloggers.html)
