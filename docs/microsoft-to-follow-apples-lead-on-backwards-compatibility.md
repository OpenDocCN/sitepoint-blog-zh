# 微软将跟随苹果的脚步向后兼容

> 原文：<https://www.sitepoint.com/microsoft-to-follow-apples-lead-on-backwards-compatibility/>

通过 SitePoint [行业新闻](https://www.sitepoint.com/forums/forumdisplay.php?f=264)论坛，微软正计划效仿苹果的做法，向后兼容最早将于明年推出的下一代 Windows 操作系统，即 Windows 7。

根据 [Dev Corvin](http://thebetaguy.com/exclusives/?postid=1029344029&title=microsoft-windows-7-exclusive) :

> Windows 7 对组件化和向后兼容性问题采取了不同的方法；简而言之，它根本不考虑他们。Windows 7 将是 Windows 代码库的全新包装；部分源代码，但与以前版本的 Windows 不兼容。

Corvin 认为，这种方法将允许通过不必为了向后兼容而支持单片库来提高 Windows 的性能。为 Vista 和 XP 编译的应用程序将在虚拟机上运行，类似于苹果的 Mac OS X 运行经典应用程序的方式——这种方式在当时是有风险的，但鉴于 OS X 的成功，苹果已经获得了回报。

如果是真的，这明显背离了微软传统的向后兼容政策。从桌面到网络，标准的 Redmond 哲学是不惜一切代价向后兼容。当微软[上个月屈服于网络开发社区](https://www.sitepoint.com/microsoft-backflips-on-browser-version-targeting/)默认使 IE8 标准兼容时，给出的理由是它“最适合网络”。现在，微软似乎意识到，在桌面上采取更具前瞻性的方法实际上可能有利于他们的底线——以及监管机构。

> 这也让微软巧妙地避开了司法部和欧盟的反垄断裁决…虽然反对微软的人会声称这是不道德的商业行为，但技术用户会意识到这是一种提供新功能的极好方式，同时保持了与传统应用程序的向后兼容性。

比尔·盖茨最近透露，Windows 7 最早可能于明年推出。
