# 5 大 jQuery UI 备选方案

> 原文:[https://www.sitepoint.com/top-5-jquery-ui-alternatives/](https://www.sitepoint.com/top-5-jquery-ui-alternatives/)

当构建现代网络时，你经常需要创建有用的 UI 组件。无论你是否需要日历、滑块、图表或任何其他有助于改善或简化用户交互的东西，你的选择要么是自己创建，要么是利用现有的功能。

自己开发这些组件通常既耗时又复杂，除非你在做完全独特的事情，否则通常不会很好地利用你的时间。这就是 UI 库和框架发挥作用的地方。这些库简化了创建通用 UI 组件的过程。您可以利用这些现有的框架，并对它们进行定制以满足您的需求。

最大最广泛使用的框架之一是 [jQuery UI](https://jqueryui.com/) 。它是基于 jQuery 构建的一组扩展的小部件、效果和主题，被分成自己的一组组件。您可以下载一个包中的所有 jQuery UI 元素，也可以挑选您感兴趣的组件和功能。使用这样的集合可以让您为组件创建一致的外观，并让您以最少的麻烦开始运行。

虽然 jQuery UI 工作得很好，是一个很好的选择，但也有其他框架拥有令人惊叹的高质量控件。在这篇文章中，我将分析其中的一些，看看它们是如何叠加起来的。

## 剑道 UI

**付费框架**

剑道 UI 框架提供了一系列超过 70 个有助于加速开发的组件。这些组件是响应性的、主题化的、快速的和高度可定制的。

![Keno UI Framework Example](../Images/4237adea5c490924ece1cdd3d7334859.png)

基诺用户界面有几个让人喜欢的地方，以及它如何帮助你创建令人敬畏的交互元素。

首先，这些组件是由 Telerik 快速从头开始构建的。与其他一些框架不同，这些小部件完全是用 JS 从头开始构建的，根本不需要 jQuery。即使在移动设备上观看，组件本身也感觉快速、平滑和坚固。

当我们谈论移动设备时，这是剑道 UI 大放异彩的另一个领域。这些组件在构建时考虑了移动设备，根据查看位置提供了响应性和适应性的布局。大多数 widgets 会根据您是否在移动设备上进行相应的调整和更改控制。这是一个很棒的功能。这是滑块组件，它会根据你的屏幕大小自动调整。

![enter image description here](../Images/f3567b4f67ff144061b8d3b6b15d2f15.png "kendo_ui_demo.jpg")

从实现的角度来看，这些控制也是经过深思熟虑的。开发者既可以在 JS 中设置它们，也可以在服务器端配置它们(例如通过 PHP 输出)。除了剑道 UI 的 web 方面，还有这个框架的分支可以用于 [Android](http://www.telerik.com/android-ui) 和 [iOS](http://www.telerik.com/ios-ui) (如果你想在你的应用中使用它们的话)。

另一件有趣的事情是与 Angular JS 的集成组件。这是一个相当完整的系统，从头开始创建 UI 元素，以完美匹配您的 Angular 项目。如果你想利用 Angular，很高兴知道剑道已经接受了他们，并且你应该得到支持。

![Kendo UI Angular sample image](../Images/70c60256cc234872bb8251ae3115d96b.png "kendo_ui_angular.jpg")

需要注意的一点是[剑道 UI 并不是一个免费的框架](http://www.telerik.com/purchase/kendo-ui)。这是一个完全商业化的库，根据您的许可需求，它的价格可能高达几千英镑。这本身可能足以吓跑一些开发人员，但是，您得到的质量和支持是您所支付的。

## Webix

**免费/付费框架**

Webix 为开发者提供了一种快速简单的方法来开始构建通用的 UI 元素。它带有一系列数据可视化、布局、导航和编辑控件。虽然这个框架与 jQuery UI 共享一些组件(日历、手风琴、对话等)，但 Webix 扩展并超越了 jQuery UI 通常所能提供的。

![Webix main page UI example](../Images/6f709cbe42ce29af2a229dd57c055f69.png "webix_main.jpg")

你得到的文件令人印象深刻。所有控件都附带一个 API 参考指南，其中概述了所有控件的方法、属性和事件。此外，大多数控件都有指向几个示例的链接，向您展示控件的确切功能。拥有友好的文档非常重要，所以很高兴看到他们花时间在文档上。

![API documentation for the Webix Calendar control](../Images/f1a3e75408e88cf49a21cdabf1ef3355.png "webix_doc_calendar.jpg")

另一个值得考虑的因素是，他们经常在博客上发布内容，有几篇文章提供了有用的建议和实用的例子，比如这篇关于 UI 组件可访问性的文章。有大量的框架，所以找到一个提供新鲜内容的总是有价值的。

Webix 提供免费和付费访问他们的框架。他们的免费版本涵盖了你可能需要的大部分内容，可以访问常见元素，如[图表](http://docs.webix.com/desktop__chart.html)、[列表](http://docs.webix.com/desktop__list.html)、[树](http://docs.webix.com/datatree__index.html)和[日历](http://docs.webix.com/desktop__calendar.html)。

他们服务的付费版本带有额外的控件(如[多选](http://docs.webix.com/desktop__multiselect.html)和[日期范围选择器](http://docs.webix.com/desktop__daterangepicker.html))以及支持和帮助。

他们付费服务的另一个方面是访问一些非常详细的解决方案，例如他们的[日程安排](http://webix.com/scheduler/index.html)。

![Calendar screenshot for Webix](../Images/6c5878559f682ab6f04ae65bbd6ee51d.png "webix_scheduler.jpg")

这些额外的付费插件是非常详细的小部件，抽象出自己构建这些组件的复杂性，让您专注于自定义功能和样式。

总的来说，这些免费的小部件是可靠的，并且提供了大量的功能。考虑到你可以免费获得一些很棒的控件，他们服务的付费部分是可以理解的，也是合适的。你必须浏览他们的[小工具文档](http://webix.com/widgets/)，看看额外的控件和功能是否值这个价钱。

如果你正在寻找一个有良好文档的，大部分是免费的解决方案来满足你的 UI 需求，那么 Webix 可能正适合你。

## JQwidgets

**半免费/付费框架**

JQWidgets 是一个基于 jQuery 的框架，旨在让开发人员能够为网站构建响应迅速、健壮的 UI 组件。JQWidget 的组件被创建来利用 jQuery 提供交互式的、动态的和高度可定制的小部件。

![JQwidgets homepage intro image](../Images/63de5c44c2e3b4a998b37a03f7755e03.png "jqwidgets_main.jpg")

当你开始学习一个新的框架时，有一种方法与开发人员联系，询问问题，并感受一切是如何工作的，这是非常好的。JQWidgets 提供了一个易于使用的、似乎经常被监控的论坛。除了论坛之外，还有一个来自开发者的最近更新的博客(包含文章、发布说明和整体想法)。

![jQWidgets forums image](../Images/71ff83c406cf9d6f668dd021575b1645.png "jqwidgets_forum,.jpg")

文档是任何框架的核心，JQWidgets 也不例外。他们的文档概述了每个小部件的工作原理，最后还有示例代码和一个很好的演示。这里是 Popover 小部件的[文档。每个小部件都有一个简单易学的 API 参考指南，以及如何实现每个属性或方法的例子。](http://www.jqwidgets.com/jquery-widgets-documentation/documentation/jqxpopover/jquery-popover-getting-started.htm?search=)

还有一点值得一提的是，JQWidget 支持 [React](http://www.jqwidgets.com/react/) 、 [Angular](http://www.jqwidgets.com/angular/) ，甚至 [ASP。净](http://jqwidgets.com/asp.net-core-mvc-tag-helpers/)组件。如果你倾向于其中之一，那么这个框架可能会为你节省很多时间。

![JQWidgets Angular Components](../Images/a55ab6bba2cdefea4e321d7dfe1217de.png "jqwidgets_react_angular.jpg")

很高兴看到 JQWidgets 采用了这些通用框架。您可以选择单独使用 JQWidget，或者使用它的替代形式，比如 Angular。这是真正的双赢。

JQWidgets 有半免费和付费两种版本。收藏的非商业用途是可以的。然而，如果你想在商业上使用它(大多数人都会这样)，你需要购买一个许可证来保护你自己。

定价的差异与支持和公司规模有关。如果你是一个单独的开发者，你可以使用他们的低级选项，如果你是一个数字机构，你需要一个更接近顶级选项的许可证。总的来说，你支付的越多，你得到的支持就越多。这将由你来决定哪个选项是最合适的。

## EasyUI

**半免费/付费框架**

EasyUI 是一组扩展的 jQuery 小部件，用于创建交互性，以构建现代的、响应迅速的网站和 web 应用程序。使用 EasyUI，您可以通过直接的 HTML 标记或者 JavaScript 创建通用的 UI 元素。

![EasyUI Documentation Image](../Images/6ed5fb60dce07c67165b50a8530e2891.png "easyui_main.jpg")

EasyUI 的设计可能看起来与我们看过的其他一些 UI 框架不一致。它的部分设计看起来和感觉有点不一样。就像 2010 年设计的一样。

虽然默认的样式可能会让你没有灵感，但 EasyUI 附带了一个非常棒的实时主题编辑器，让你控制字体大小、文本颜色、背景颜色、位置和大多数其他常见属性。最重要的是，它给你直接的 CSS 访问，这样你就可以到处玩，建立一个适合你的主题。

这是“材料设计”的主题。

![EasyUI theme Customizer example](../Images/8c1e38523134d799cf2e48e44e4b470d.png "easyui_theme_builder.jpg")

EasyUI 的[文档简单直观。所有控件都显示在左侧，相关信息显示在右侧。每个控件都有一个示例屏幕截图、示例代码以及所有的属性、方法和事件。我在这里的主要抱怨是，这些网页上没有真正的现场演示。](http://www.jeasyui.com/documentation/index.php)

[主演示页面](http://www.jeasyui.com/demo/main/index.php)给你每个控件演示的快速链接。例如，[面板控件演示](http://www.jeasyui.com/demo/main/index.php?plugin=Panel&theme=default&dir=ltr&pitem=)展示了定制控件、嵌套面板和其他元素的外观。

![jQuery EasyUI Demo page, showing the Panel Tools control](../Images/3ff874619eb3160261ee5fb190eab7ba.png "easyu_demo.jjpg.jpg")

jQuery EasyUI 对于非商业用途是免费的，但是如果你是开发者，你需要一个许可证。价格列在他们的[联系页面](http://www.jeasyui.com/contact.php)上，总体来说相当合理(与其他框架的一些高端价格相比)。作为一个整体，这个库是一个很好的起点，它提供了一些你所期望的更高端/支持驱动的服务的附加特性，比如一个[相当活跃的论坛](http://www.jeasyui.com/forum/index.php)和一系列[扩展。](http://www.jeasyui.com/extension/index.php)

总的来说，它是一个可靠的框架，除了它看起来有点旧(网站和文档)之外，如果你正在寻找一个好的 jQuery UI 替代品，它是一个可靠的起点。

## 威莫

**付费框架**

Wijmo 是一系列用 TypeScript 编写的定制 JavaScript 控件[，用于创建快速、响应迅速且可扩展的 UI 控件。Wijmo 提供了广泛的控件，并支持 React、Angular 和 Vue 等几种 JS 框架。](https://www.typescriptlang.org/)

![Wijmo homepage image](../Images/a5b49eda7d1d95c493f357b984aba159.png "wijmo_main.jpg")

Wijmo 支持一系列的控件，其中大部分侧重于数据可视化和输入，比如图表、仪表、输入控件和网格。虽然其他框架提供了一系列基本控件，如对话框或布局，但 Wijmo 似乎专注于数据。

![Wijmo components overview](../Images/2558769df49d88642f44aa7b9310be4a.png "wigmo_controls.jpg")

需要指出的一个有趣的方面是，开发人员提供了对最新版本 Wijmo 5 和遗留版本 Wijmo 3 的支持。Wijmo 5 是围绕更现代的标准构建的，因此不能在 IE8 上工作。这就是旧版发挥作用的地方。Wijmo 3 基于 jQuery，所以它的支持可以追溯到 IE6。很高兴知道您有选择，而且开发人员正在支持旧的框架，以防您需要使用它们。

Wijmo 的文档和他们网站的结构令人困惑。他们所有的控件都列在他们的文档页面上，然而，它们并不友好或令人兴奋。例如，如果你想找到更多关于图表模块的信息，你可以进入[这个页面](http://wijmo.com/5/docs/topic/wijmo.chart.Module.html)。这些文档提供了关于类和接口的详细信息，但没有提供具体实现控件的信息，包括图片、代码示例、具体描述或演示展示。

他们的网站上有一个“资源管理器”部分，向你展示所有这些控件的外观，并让你调整设置。这是一个有趣的工具，有助于了解他们的外观和感觉。

这是一个通过 Flexchart 组件的折线图示例。

![Wijmo Flexchart Linegraph control example](../Images/b9445e1819698d12f08bd8cfeab38a08.png "wijmo_flexchart.jpg")

其他框架以更好的方式提供了文档，要么为每个组件提供一个单独的专用页面，要么提供几个子页面来概述您需要的技术组件、示例和描述。

在投资一个新的框架时，拥有可靠的文档是至关重要的。如果你想看看运行中的图表模块(并想看看代码示例)，你需要导航回[演示页面](http://wijmo.com/products/wijmo-5/#demos)并找到 [Flexchart 演示](http://demos.wijmo.com/5/Angular2/FlexChartIntro/FlexChartIntro/)来看看它是如何工作的。另一点值得注意的是他们的[最近更新了博客](http://wijmo.com/blog/)。这些看起来像是有用的提示、系统的亮点和各种想法的组合。这是一个不错的额外的东西。

![Wijmo flexchart documentation samples](../Images/e3c30c66b56a943bbc1d9c445e52a9fb.png "wijmo_flexchart_demo.jpg")

除了他们网站上的碎片，控件本身也很有趣。每个控件都允许您调整它的显示方式，为自定义功能提供回调，并且总体上允许您根据需要设置它们的样式。

Wijmo 为他们的产品提供试用期(类似于剑道)，但如果你想将其用于商业用途，你需要一个许可证。[购买页面](http://wijmo.com/purchase/)概述了您的选择。价格取决于你需要哪些元素，如果你只是想要 Wijmo 5 可以自行购买。但是，如果您需要。NET / Xamarin 您将需要高端企业许可。

由于 Wijmo 主要专注于数据可视化，很难不将其与诸如 [HighCharts](http://www.highcharts.com/) 之类的库进行比较，这些库可以说拥有一些最好的图形/图表视觉效果。

## jQuery UI 备选方案

总的来说，有很多优秀的框架是 jQuery UI 的完美替代或补充。这些框架中的大多数，比如我们所看到的，都需要付费，这是可以理解的。

说到底，如果您正在寻找一个解决方案，您要么需要支持一个体面的开源项目，如利用社区体验的 jQuery UI，要么购买一个公司的订阅，并利用您已经购买的体验。

如果你正在使用一个框架，并且受益于不用从头开始，这已经是一个胜利了。您需要权衡现有框架(比如我们今天看到的框架)的价格以及构建类似质量的东西所需的时间和精力。

有几个高质量的库，网站上的[‘Alternative To’列出了几个解决方案](http://alternativeto.net/software/jquery-ui/)，在创建一个令人敬畏的 UI 时，你可能想尝试一下。如果你能推荐任何其他好的 jQuery UI 替代品，请在评论中告诉我。
