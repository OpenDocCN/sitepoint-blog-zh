# 采访 Jedox 有限公司的克里斯蒂安·劳埃

> 原文:[https://www . site point . com/interview-with-kristian-raue-of-jedox-gmbh/](https://www.sitepoint.com/interview-with-kristian-raue-of-jedox-gmbh/)

在工作表服务器的[审查之后，我与](https://www.sitepoint.com/article/php-apps-excel-worksheet-server) [Jedox Gmbh](http://www.jedox.com) 的创始人 Kristian Raue 进行了一次有趣的聊天，从源头上了解更多关于工作表服务器的信息…

你能告诉我们一些关于你的 IT 背景和你为什么创建 Jedox Gmbh 吗？

在 2002 年 6 月开始 Jedox 之前，我是一家以 OLAP 为中心的软件和咨询公司 IntelliCube Software AG 的首席执行官。Intellicube 的旗舰产品是基于 Windows 的电子表格技术，适用于处理来自 OLAP 服务器的数据。当 IntelliCube 被卖给德国的 MIS AG 时，我看到了利用我的电子表格专业知识并将其应用于开发 web 技术的机会。到目前为止，微软对为微软 Excel 提供基于网络的基础设施没有表现出任何兴趣，所以创建工作表服务器对我来说似乎是一个合乎逻辑的步骤。

**你为什么选择 PHP 做工作表服务器？微软的 ASP 3.0 不是更好的选择吗？**

有几个原因:PHP 可以在几乎所有的托管服务器上使用，它非常容易学习，我特别着迷于 PHP 处理变量和数组的容易程度。

此外，随着微软每两年改变一次技术路线，我不想再依赖微软的技术了。

更一般地说，在我看来，今天的一个常见模式是微软鼓励第三方在 Windows 下开发软件产品，但是，一旦产品成功，微软就会提供与其操作系统或其主流产品捆绑的类似解决方案。这是另一个原因，为什么我决定不再为微软的特定软件开发投资更多的钱。

**工作表服务器开发了多长时间？**

我们在 2000 年开始开发。

Jedox 开发团队的其他成员是谁？

基本上，它是一个由 4 名开发人员组成的团队，分别位于弗莱堡和不来梅(德国)。

**根据 Jedox 网站上提到的客户和合作伙伴，Worksheet server 的营销似乎主要集中在讲德语的欧洲。你有计划扩展到说英语的市场吗？**

是的，我们在美国和加拿大已经有了第一批潜在客户，并且我们已经向澳大利亚和新西兰的客户销售了工作表服务器。但要在这些市场取得真正的成功，我们真的需要当地的合作伙伴。我们目前正在寻找软件和咨询公司，他们希望成为合作伙伴，特别是，但不限于，在英语国家。

**您的客户正在使用工作表服务器构建什么类型的应用程序？**

应用程序的范围和 Microsoft Excel 本身的使用一样广泛。最常见的是公司在控制、预算、销售预测等方面使用工作表服务器进行财务应用。但是你也可以在统计、技术和农业领域找到应用。

Worksheet-Server 适用于所有需要从不同地点或部门收集信息并将数据汇总到中央数据库的公司和组织。

一旦客户开始使用 Worksheet Server，您会得到什么样的回应？这对他们的工作方式有什么影响？Jedox 通常需要为新客户提供多少额外的支持和帮助？

到目前为止，我们从客户那里得到了非常积极的反馈。我们经常惊讶于他们使用 Worksheet-Server 所需的支持是如此之少。他们通常在没有我们支持部门帮助的情况下运行基本功能。只有在应用程序中集成用户定义的 PHP 脚本时，才需要一些支持。

请记住，对于我们的大多数客户来说，基于 PHP 的工作表服务器扩展是他们第一次接触 PHP。所以我们在这里做一些基本的 PHP 培训。

你最喜欢/不喜欢 PHP 的什么？

有很多我喜欢的。例如，它是一种弱类型语言，因此类似于 Excel 电子表格中的单元格。我也喜欢语言的完整性；一切都可用:XML、SOAP、邮件，一切都唾手可得。最重要的是，我欣赏 PHP 允许你创建数组的数组的方式……任何你喜欢的东西。

我不喜欢什么？如果你能在函数调用中省略参数就好了，比如 myfunction(23，，43)。也许 PHP 可以为缺失的参数插入一个空值。

客户通常对 PHP 有什么反应？对于一个有经验的 Excel 用户来说，用 Visual Basic 知识切换到 PHP 有多容易？

客户对 PHP 非常开放。我们发现我们的一些大客户已经在其他项目中使用它了，但是即使是初学者从 VBA 转换到 PHP 也没什么问题。

**您计划为工作表服务器提供哪些新功能？**

在即将发布的服务版本中，您将看到链接工作表服务器应用程序的可能性，即使它们驻留在不同的服务器上。这样就有可能创建相互依赖的工作表-服务器应用程序的整个网络。

我们不久还将发布一个 Microsoft Excel 的客户端应用程序，允许用户动态地将 Excel 单元格与单个工作表服务器单元格链接起来。每当 Excel 重新计算时，它将从 Worksheet-Server 中提取最新的结果，并在 Excel 中自动显示这些值。

如果给你足够的时间/金钱，你希望实现哪些(疯狂的)想法？

创建一个真正面向工作流的邮件客户端，并从我的桌面上删除 MS Outlook。

*感谢您的宝贵时间，克里斯蒂安*。
