# CFML 的局限性(以及解决方法)

> 原文：<https://www.sitepoint.com/the-limitations-of-cfml-and-ways-around-them/>

*…发帖人* [大卫·梅德洛克](https://www.sitepoint.com/forums/member.php?u=24385) *:*

ASP 和 PHP 开发人员比我们 CF'ers 更有优势。他们可以编写一个应用程序，把它放在一个漂亮整洁的包中，然后卖给客户，而不必面对让客户购买 PHP 或 ASP 许可证并安装和配置它等问题。PHP 和 ASP 托管很常见，很容易找到，而且非常便宜。

但是对于我们 CF 编码人员来说，我们必须担心一些不同的挑战。首先，到目前为止，我们的潜在客户群仅限于 ColdFusion 开发人员或已经在使用该技术的公司。这很好。不知有多少人已经在充分利用 ColdFusion 了。但是，难道我们不应该将我们的客户群扩展到那些已经拥有 ColdFusion 主机或服务器的客户群之外吗？答案是肯定的。显然有一种简单的方法可以做到。

现在，我最近才开始阅读 BlueDragon 和 Coral，所以我还没有这两个解决方案的所有细节。在接下来的几周里，我会更仔细地研究它们，并向这里汇报。与此同时，我建议每个人都去看一看来自新亚特兰大的蓝龙和 T2 的珊瑚。这两种解决方案对 ColdFusion 社区来说都很有前景，在不久的将来，我希望更全面地探索它们，并解释它们如何用于扩大我们的客户群，正如 [Vince Bonfanti](http://www.newatlanta.com/products/bluedragon/editorials.cfm#CFDJOct03) 所说，向他们出售解决方案，而不是技术。享受吧。:D
