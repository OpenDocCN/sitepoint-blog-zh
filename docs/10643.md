## 采访记录

凯文:2010 年 9 月 10 日。UX 专业人士回应对他们职称的攻击。我是凯文·杨克，这里是 SitePoint 播客#78: UX BS，马特·马金和詹姆斯·曼斯菲尔德。

欢迎来到 SitePoint 播客的另一集。一如既往，我是凯文·杨克，在这个访谈节目中，我想听到一些来自 SitePoint 总部的声音。我们最近对公司以外的人进行了很多采访，我想你知道吗，我们在 SitePoint 总部有一个非常好的房间，所以我今天邀请了 Matt Magain 和 James Mansfield 与我们聊聊用户体验设计。Matt Magain 在 SitePoint 团队工作，他现在是一名 UX 设计师，James Mansfield 在 99designs 做着几乎相同的工作。嗨，伙计们，最近怎么样？

马特:嗨，凯文。

**詹姆斯:**嗨，凯文，你好吗？

凯文:所以我们在这里谈论一篇由瑞安·卡森在 Think Vitamin 上发表的博客文章，他对 UX 设计师和他们选择的职位有一些选择的话，但在我们开始之前，我想听听你们到底是做什么的，这里是马嘴；对你们来说，什么是 UX，什么是 UX 设计师？

所以对我来说，UX 设计是一个总括术语；它包括一系列在项目的研究、发现和设计阶段发生的事情，视觉设计只是其中的一个方面，你知道，有人专门从事信息架构、信息设计、版权保护、交互设计、营销、用户研究、编辑和策划，这些只是我认为属于术语“用户体验”的一个例子。

凯文:好的，詹姆斯呢？

**James:** 谢谢 Kevin，是的，我同意 Matt 的说法，我认为用户体验是一个非常宽泛的术语，我认为这是我猜测 Ryan 使用这个术语的部分原因，它对其他人来说可能并没有太大的意义。对我来说，用户体验包括围绕用户研究的事情，从理解用户的动机或行为到最终的视觉设计，或最终的图形设计，我认为这里面有很广泛的东西，特别是对我来说，用户体验包括更多的研究和用户研究的事情，到现场去了解人们如何与软件交互。所以我说的是医疗设备之类的东西，并不总是特定于网络的。因此，用户体验涵盖了比网络更广泛的内容，也涵盖了比我所认为的更接近工作面的技能，比如图形设计和将其发展成工作界面。

凯文:好的，你们显然对 UX 设计有热情，但是你们选择了将这种热情应用到网络上。一个 UX 设计师是如何从事网页设计的，这对你们来说和网页设计有什么关系？我知道很多这些术语在我们的领域是模糊的，但是 UX 设计在一个网站团队中是网站设计的一个子集还是一个独立的工作，对你们来说是什么？

对我来说，网页设计是用户体验的一个子集；用户体验是它的一个更广泛的方面，正如我所说，我认为用户体验包括它的研究部分和它的用户测试部分，对我来说，网页设计是一种活动，或者创造它将成为什么，所以它是一个更广泛的设计过程的一个方面。我想，我不知道，网页设计，设计是一种活动，也不是一种结果，所以我不太确定我在说什么，但网页是媒介，设计是一种活动。

凯文:所以我们有网页设计，你做的事情，你有网页设计师，做这个的人。UX 设计师是网页设计师的一种吗？或者这太简单了？

用户体验设计有很多方面，其中之一就是视觉设计。我认为，当人们想到网页设计师时，他们会自动想到在 Photoshop 中创建漂亮的用户界面模型的男孩或女孩，然后根据你的多面手程度，可能会用 HTML 和 CSS 标记他们，甚至可能会编写一些 JavaScript。我会考虑最终产品，但是用户体验设计会后退一步，看看我们是如何走到这一步的，就像詹姆斯说的那样，像研究、创建线框和原型这样的事情，在我看来，所有这些发生在你打开 Photoshop 之前的活动都属于用户体验设计的范畴。用户测试也是其中很大的一部分，应用程序的可用性和验证市场，这些都是与产品营销和市场研究密切相关的领域，它们都是创造最终产品的宝贵投入。

是的，我再次完全同意马特所说的。几周前，我在澳大利亚 UX 参加了一个非常棒的会议，SitePoint 也赞助了这个会议，Matt 也参加了，那是一个座无虚席的活动，我们有 100 人参加，所以 UX 是一个非常知名的领域，或者肯定是一个非常健康的领域。我认为对我来说，如果我问，我觉得如果我问参加会议的人他们是否是一个网页设计师，我想他们中的很多人会不愿意称自己为网页设计师；对我来说，参加会议的大多数人都处于研究阶段。我认为，如果你可以在“用户体验”这个术语中有两个领域，我认为有研究部分，然后有设计部分，我认为在那个会议上的大多数人会把自己放在研究类别中，这种研究是在设计之前进行的研究，以及研究现有的东西，它是如何工作的，它有多好，它有多有用。

**Kevin:** 作为一名 UX 的研究人员，他们的工作是了解用户的想法，无论他们最终会建立什么样的网站或网络应用程序，或者已经建立了什么样的网站或应用程序，但却没有为他们服务。

詹姆斯:是的，绝对是。甚至更广泛地看它将被使用的环境，比如如果你想一想医疗、医院或其他地方的机器，它们将如何在昏暗的光线下使用，什么噪音，什么警报，那些突然出现的东西，所以我想对用户体验的研究也包括一些物理事物，一些物理环境方面。

Kevin: 今天让我们聚在一起的是 thinkvitamin.com 的运营者 Ryan Carson 的推文，他也是 Carsonified 的幕后人，Carsonified 运营着几个与网络相关的会议。他前几天在推特上说“‘UX 专业人士’是一个 BS 头衔。这只是向天真的客户要价过高的一种方式，所有的网页设计师都应该是 UX 的专业人士。”他[对他的立场](http://thinkvitamin.com/opinion/ux-professional-isnt-a-real-job/)进行了更长形式的辩护，但那是——他被指控链接诱饵，但我觉得他似乎真的相信他在说什么，毫无疑问，这是一个有争议的立场，但我想也许从你看到这个的第一反应开始。

Matt: 我的第一反应是 Ryan 显然是信口开河；他正在推特上写东西，时间可能有点不太合适—

凯文:告诉我们时间。

Matt: 嗯，他在 Clearleft 在英国布莱顿举办的 dConstruct 会议上发了推文，在 Ryan 发推文的时候，Andy Budd 和他的团队正试图聘请一名 UX 设计师，所以这可能对主持会议的人来说是一个小小的打击，也许这是故意的；也许不是。我认为瑞安肯定喜欢有争议，所以如果这是一种链接诱饵的形式，我不会感到惊讶。

凯文:我的意思是，我们会，你知道，五十步笑百步，我们有时会在 SitePoint 网站上为某个博客标题选择最耸人听闻、最有争议的措辞，这是我们的过错；显然，这是吸引流量到你的网站的标准策略。他博客的标题是“‘UX 专业人士’不是一份真正的工作，”我现在坐在两位 UX 专业人士旁边，你们是做什么的？这看起来像是一份真正的工作吗？

**James:** 首先，我想回顾一下，Kevin，支持 Matt 的说法，我认为这是一个明显的链接诱饵，我认为他希望引起一些争议、一些流量和一些讨论，但我认为进行一些讨论是健康的。我认为“用户体验”这个术语在业界是一个相当新的术语，不是很多人都习惯使用它，或者不是很多用户体验之外的人都理解它的含义。今天早上我参加了一个幼儿园的父亲节活动，当人们问我是做什么的时候，我发现说我是一个网页设计师比说我是一个用户体验设计师更容易。所以我认为这是有道理的，如果人们不理解什么是用户——如果一般市场不理解什么是用户体验设计师，那么，是的，网页设计师这个术语更容易描述。

Matt: 我认为我对 Ryan 在这个问题上的立场的两个主要问题是，我不同意他对用户体验的定义。

凯文:对，他接着定义了它是什么，它是一系列技术的列表；HTML，CSS，yada，yada。

**马特:**正是。我认为 Ryan 认为用户体验是设计的一个子集，包括一些实际操作的东西。正如我之前提到的，几周前，James 和我参加了 UX 澳大利亚会议，会上有些人不会写任何 HTML 或 CSS 或 JavaScript，但他们都是各自领域的专家，无论是信息架构还是用户研究还是其他，就像 Ryan 所说的，这些人不会增加任何价值吗？我的意思是这基本上就是他的描述。

凯文:嗯，我读了 Ryan 博客上的一段话，他说，“如果你不是一个经验丰富的网页设计师，没有参与设计、构建、测试、营销和更新网页项目的日常工作，你就不可能成为‘UX 专业人士’。”这和我们几个月前在播客上讨论的有些类似，人们认为每个网页设计师如果想称自己为网页设计师，至少需要对 HTML 和 CSS 以及如何机械地建立一个网站有所了解。对于一个从事网络工作的 UX 设计师来说，这是必要的吗？我们能给他吗？

没有。我和很多有才华的用户体验设计师合作过，他们是我非常尊敬的人，但他们不知道如何创建 HTML 和 CSS。我认为 Ryan 的背景可能很小，或者他谈论的范围很小。他提到他之前为三家设计机构工作过，我和 Ryan 在小机构、大组织和大公司工作过类似的时间，也为金融机构工作过。现在，那些金融机构有营销团队；有专门从事用户测试和用户研究的专业公司，所以我认为说一个网页设计师应该能够做所有这些事情是有点短视的，是的，我认为这回到了他的链接诱饵和寻找东西。

但是就像你提到的，詹姆斯，你知道用户体验设计师的技能可以移植到其他媒体，所以如果有一个用户体验设计师正在开发 iPad 应用程序或桌面软件或 Windows Phone 7 的东西，你知道，他们不一定要编写 HTML 或 CSS，这可能与他们无关，因为他们是创建线框或进行用户测试或进行人种学研究或他们擅长的任何其他事情的专家。

詹姆斯:我认为对我来说，用户体验的一个共同点，或者说我与用户体验这个术语紧密联系的一点，就是实际的交互设计。我认为每个自称为用户体验设计师的人都会专门做可用性和交互设计，他们可能不会继续做视觉设计或图形设计，也可能不会继续做建筑，但我认为交互设计是我认为用户体验所包含的关键内容，可能是所有用户体验设计师都应该具备的技能。

**Kevin:** 我怀疑听到这个消息的很多人从来没有与 UX 共事过，他们习惯于在小型网络团队中工作，团队中有设计师和开发人员，这是 Ryan 在他的博客中经常提到的那种团队。我在想，你知道，我可能问错了人，但是没有专门从事 UX 工作的人，一个好的网站能建立起来吗？

**Matt:** 我觉得总有有才华的个体；Ryan 在他的博客中提到肖恩·因曼是一个多面手的例子，他在设计和开发领域都有惊人的天赋，根据所有人的说法，看了肖恩的一些作品——我是他的粉丝——他对用户体验和原则有很好的理解。但是，我的意思是，你知道如果瑞安显然认为 UX 是重要的，并认为网页设计师应该把这作为课程的一部分；如果有人在那个领域有专长，为什么不应该反映在他们的职称上呢？

Kevin: 所以如果有人在听这个，并且他们认为自己是前端设计师，他们对 jQuery 了如指掌；如果有人给他们一个 Photoshop 模型，描述如何根据用户与他们的交互方式进行改变，但他们没有信心自己做出这种决定，他们只想让它发生，有人告诉我在这个页面上会发生什么惊人的事情，我会负责让它发生。我听起来好像 Ryan 在说那个人不是专业的网页设计师。

如果有一个人是 jQuery 专家，有那么多工作要做，他们所做的就是编写 jQuery 代码，那么我绝对同意他们应该称自己为 jQuery 专家，而不是网页设计师，不做更多的一般技能。我似乎有一种感觉，Ryan 不相信专业化，如果你进行比较，例如在医学界，从前只有外科医生，从来没有心脏外科医生或脑外科医生或神经外科医生。Ryan 是说外科医生应该能够做所有的事情，而不应该专攻某一方面吗？这种类比可能有点牵强，但用户体验领域正在成熟，网页设计这个职业也在成熟，因此专业化也在发展，我不太确定称它们为 BS 真的有什么意义。

是的，绝对是，我认为我同意这个领域正在走向成熟，我认为人们变得更加专业化。我认为围绕界面、交互和体验设计的行业正在变得成熟，也在变得支离破碎；我想在过去的几年里，我确实看到很多人出来自称为前端开发者，他们专攻 HTML 和 CSS。这是一个正在成长的新兴领域，我认为这也是理所当然的；有许多库，有许多技术和方法需要跟上并保持领先，我当然不能保持领先。我想这样做，我也希望能够这样做，但是随着我的成熟，我发现自己会被更多的用户研究吸引，所以我认为市场肯定会分裂，我认为我们需要接受这一点。我认为用户体验这个术语是一个宽泛的术语，用来描述一个人可能拥有的一系列技能，但我认为知道市场正在分裂是很重要的。

Kevin: 我想超越 Ryan 的博文，因为我确实认为很多人不同意他的观点，但由此产生的很多对话以及人们不同意的不同观点是目前最有价值的事情。但我认为表面上我认为我同意你的观点，马特，他反对专业化；他说“网站的核心应该是网页设计者和开发者的产品，很明显，在较大的项目中，你需要增加不同的人，因为工作量对于两个人来说太多了。然而，增加这些人员是出于后勤原因，而非战略原因。”所以，是的，就好像每个人都是网页设计师或网页开发者，这本身就是一种专业化，你知道，这两种角色在网络诞生时曾经是一个人。所以，是的，一些正在进行的对话是几个用户体验博客，[其中之一 Cennydd bow les](http://www.cennydd.co.uk/2010/the-pollution-of-ux/)；他似乎在谈论为什么人们能够质疑用户体验设计的价值？为什么有人看不到这个东西的内在价值？显然，你知道你已经有了一个包含用户体验设计师的成品，你已经有了另一个不包含用户体验设计师的成品；如果这是一个有价值的贡献，那么这两个产品之间的差异应该是显而易见的。你应该能够指着它说*，如果没有用户体验设计师，我们永远不会有*。这里混乱的来源是什么？为什么甚至有人会质疑这样的人能给一个团队带来的价值？

Matt: 所以我的观点是，有些人是摇滚明星，他们可以第一次就搞定他们的设计，当然他们可能会将一些用户体验融入其中，或者他们可能只是第一次就搞定了，但对于我们这些没有这种天赋的人来说，我很高兴地举手说，我的设计技能来自于迭代、进化、测试和验证，直到我实现我已经证明有效的东西，而不是仅仅有一种直觉。

凯文:我认为测试和验证是特别重要的，因为即使你是这些天才中的一员，能做出一些大家都退后一步认为是了不起的东西，你怎么能证明它是了不起的东西，如果没有一些纪律，一些严格的 UX 设计实践，它就没有更好的空间？

当然有一些摇滚明星可以在一个非常小的团队或者在他们自己的实践中创造出很棒的用户体验。我认为用户体验这个术语更合适的地方是当我们再次谈论更大的组织或更大的用户市场时。用户体验设计师的一个很好的特质是同理心，对产品的最终用户有同理心，而你获得这种同理心的方法是与他们进行持续的对话，并测试你的产品或与他们一起测试体验，以了解他们在想什么，他们有什么感觉，他们没有发现什么，他们在非常基础的层面上发现了什么。所以我认为，如果我们谈论一个更大的组织或银行的背景，用户体验设计师有助于为该组织带来对最终用户的同理心，我认为任何在这些组织中工作的人都知道同理心不是许多人都有的品质，许多人在这些组织中有非常强的想法和强有力的概念，他们不一定意识到他们是坏想法或没有经过很好的思考。所以我认为用户体验设计师对我来说更像是一个团队，向他们展示并传达最终用户的想法和感受，以及他们对某些事情的反应，这样他们就可以继续帮助设计最终产品，但我认为他们需要，是的，我看到用户体验这个术语在更广泛的组织意义上应用，而不是在一个小团队中。我认为在一个小团队中，你需要一个有同理心的网页设计师，他能做设计，然后为你构建；如果你有一个小团队，这正是你所需要的，但在一个大组织中，有更多的人要交流，有更多的人要参与到一个想法中来，你需要有人能够——你需要一个用户体验设计师来帮助建立对用户反应的意识，以帮助支持他们的事业。

Matt: 公平地说，Ryan 并不是说用户体验设计没有价值，你知道，他承认它很重要，事实上，他是说如果你是一名网页设计师，每个人都应该能够做到。但关键是，这个过程，就像变得更好一样，有专业化的空间，有人专攻，我和詹姆斯参加了一个专门的会议。Ryan 是在暗示 UX 澳洲是一个 BS 会议名称吗？

凯文:所以，我想到了一些事情，这是我个人生活中的一些事情，在我的业余时间，我做的事情之一就是与一个即兴剧团一起表演，关于即兴表演的事情是你做得越好，它就变得越隐形。如果你在即兴表演方面做得很好，你在晚会结束时得到的不是对你的即兴表演技巧印象深刻的观众，而是不相信你在即兴表演的观众；他们认为，哦，没有任何错误，他们做得很好，显然他们没有编造，这一定是事先写好的。我想知道这是否在一定程度上适用于 UX，你在某个网站的 UX 设计中做得越好，感觉就越自然；你看到的是问题和错误，如果没有错误，你的工作就会变得透明。

是的，Jared Spool 在 UX 的一次会议上很好地总结了这一点，好的设计是看不见的，你不会谈论它，也不会注意到它，我认为谷歌搜索是一个很好的例子，你不知道发生了什么或它如何工作，或者你不会谈论界面，它不是一个很有吸引力的界面；它有效，你得到结果，你想要的结果，相关的结果很快，每个人都对此赞不绝口，所以这是一个很好的体验。我认为，是的，我认为好的用户体验是无形的。我想稍微跑题一下，回到 Matt 的观点，我认为 Ryan 说的有些道理，我可能在五年前就称自己为网页设计师了。

凯文:在聚会上，你有时还会用到这个词。

是的，我知道，因为我认为人们理解它；这样大家就明白多了。但是我称自己为用户体验设计师已经有几年了，这是因为我的兴趣更多的是在进行用户会议和做用户研究，以及那些在过程中更早期的事情。我已经不再——我编码，我编码过，但我正在远离它，我努力跟上趋势和技术以及正确的方法，我的兴趣更多的是在设计过程的早期，所以我称自己为用户体验设计师。我也做视觉设计，所以我认为用户体验这个词符合我所掌握的技能，我认为它适合做平面设计，以及做研究、测试和上下文查询等类似的事情，你可以去现场与用户交谈。

凯文:我想提出的另一个问题是，如果你的团队中有一个人的工作是 UX 职业球员，这是否会减轻其他人担心 UX 的责任，这本身会是一个问题吗？我再一次试着去想，这可能是瑞安无意中提出的一个观点，但是，是的，如果你是 UX 的人，而没有其他人关心 UX，这有用吗？

詹姆斯:我认为每个人都应该关心用户体验，项目中的每个人都应该对此有自己的看法。我认为我们都是设计师，每个人天生都会设计，我们都喜欢提出问题的解决方案，所以我认为一个好的用户体验设计师会倾听项目中每个人的意见，并对他们的想法和反馈持开放态度；我当然与两位开发人员有着非常密切的合作，我非常尊重他们对用户体验的看法，我当然已经开始了一个想法，他们也改变了我的想法。一个好的用户体验设计师需要能够思考，好吧，这是我的目标受众，这个想法不一定来自他们，但是一个好的用户体验设计师需要能够挑选他们知道适合这些受众的想法，并且客观地说，嘿，这不是我的想法，但是这是一个更好的想法，并将其融入到这个过程中。因此，一个健康的团队每个人都应该对用户体验有一个看法，每个人都应该表达他们对用户体验的看法，但我认为对我来说，用户体验设计师需要是在一天结束时打电话的人，因为他们是在采煤工作面上与产品的最终用户理想地交谈的人，他们是对他们在体验中寻找什么有深刻理解的人。

**Matt:** 他们也是理解流程的人，对吧，所以他们将是团队中确保“用户体验”中的用户是关注的焦点，并且决策是基于用户的需求而不是意见冲突做出的人。

凯文:我想再深入探讨一下。我们在这里还有一点时间，所以我想知道，因为在开始时我们谈到了 UX 是什么，它取得了什么成就，你能谈谈实用性吗？作为一名 UX 设计师，你在 SitePoint 的一个团队中做了一些其他团队成员不会做的事情；一些实际的过程、练习和设计步骤，老实说，在我们有像你们这样有职称的人之前，我们从来没有在 SitePoint 做过。

詹姆斯:我可能会专门谈谈我进入 99designs 之前参与的一个项目，因为我认为这是一个用户体验设计师应该具备的技能的好例子。所以，我是一个政府组织，他们的内部网，他们知道它运行得不是很好，他们也不知道为什么，所以我参与帮助他们做一些用户研究，以了解这种多样性是什么，该组织内不同范围的人想从他们的内部网实现什么，他们想知道什么，他们转向那里是为了什么。此外，他们认为目前 it 的哪些方面是好的和不好的，所以我去采访了从个人助理到科学家到管理层的一系列人，了解了他们希望 it 能够做什么，目前做得好的和不好的不同观点。 所以这给了我和我作为设计师的组织一个很好的基础去理解我们应该设计什么样的关键东西，以及我们应该从我们承担的任何重新设计项目中获得什么。 然后我们想出了一些概念，这些是原型，粗略的线框，我们再次与来自该组织的用户坐下来，再次，组织中不同级别的人担任不同的角色，并在他们身上测试了特定任务的界面， 这有助于进一步宣传设计，我们从那里对它进行了更改和调整，然后我们继续进行视觉设计以进行交流，我们与他们的营销部门合作，以确保它传达了该组织的品牌和品牌价值，然后我们继续将其交给一家公司，然后从那里进行开发。 因此，我认为这很好地概述了我如何看待用户体验设计师的角色，即在做任何线框图之前理解并从头开始，然后从那里进入。

从我的角度来看，还有一些其他的工具也是这个过程的一部分，我认为它们是非常宝贵的；其中之一是调查。我们最近一直在 SitePoint 上调查客户对我们课程的看法，以及我们从这些调查的反馈中收集的数据——显然需要有所保留，因为只有特定类型的人会对调查做出回应，但当然如果你想要一些定量数据，那么调查是实现这一目标的一个很好的方式，另一件事是使用人物角色和场景；人物角色是代表客户群中典型用户的用户配置文件，场景是这些用户可能在您的站点上执行的任务。我认为当 James 之前提到用户体验设计师有义务向整个团队传达用户体验时，人物角色和场景是实现这一点的好方法。一般来说，你给这些人物角色一张真实的照片，你可能只是从股票图片网站上抓取某人的图片，给他们一个背景故事，让它变得有趣一点，你知道，给他们一只狗和一所房子以及一些个人问题，人们会同情和认同这些用户简档，他们开始了解并喜欢他们，就像他们是真实的人一样， 然后，用户体验渗透到整个团队的任务已经开始，因为人们甚至下意识地，如果他们正在做一个开发决策，如果他们脑海中有拉里、亨利、约瑟夫和伊莱恩，他们可能会想，嗯，这对伊莱恩有什么影响，我知道她有这个特殊的要求，所以我确保我考虑到了这一点，如果这种情况发生，那么你作为用户体验设计师的工作就是成功的。

詹姆斯:是的，绝对是。我认为调查非常重要，人物角色和场景也非常重要，Matt。我想快速补充的另一件事是分析和流量统计，作为用户体验设计师，我们经常深入了解发生了什么，用户通过网站的一般路径是什么，他们在哪里下车，以帮助确定我们可以微调的方式。一个例子是关于 99designs，我们从网站流量统计中了解到，很多人在开始参加比赛之前会浏览网站上现有的比赛，所以我们会尽我们所能帮助那些真正活跃的， 将竞赛放在显著位置，让我们的用户更容易找到这些竞赛，并了解这些竞赛中发生的事情，这样他们就可以更深入、更快速地了解这些项目中涉及的内容，所以，是的，我认为分析也是设计师的另一个非常重要的研究工具。

最后，用户测试是一件不容忽视的事情。当然，自从 James 加入进来后，我已经从他那里学到了记录用户测试会话与确保涉众真正观看会话之间的价值。一般来说，对编写 Silverback 的 Clearleft 人来说，这是我们在 SitePoint 使用的游击队用户测试应用程序。如果你记录了一个用户测试会话，说服你的老板实际观看整个记录的机会是非常小的，所以如果你能找到一种方法让他们在会话发生时实时监控会话，那么 live element 是有吸引力的，你会发现人们被吸引住了，然后你在整个组织中渗透用户体验的工作就完成了。

詹姆斯:是的，我认为这就像一场篮球赛或足球赛，我认为观看上周末比赛的录像是不太可能的，你会得到与现场观看比赛完全不同的体验。所以我认为对于用户测试来说，让利益相关者参与进来是非常重要的，让他们更加同情他们的用户是关键，让他们真正更好地了解他们的用户，理解他们的用户的观点，因为我认为这在任何规模的组织中都很容易迷失，这是非常重要的，非常重要的。Jared Spool 说，专注于 UCD *【以用户为中心的设计——KY】*的成功组织的三个关键因素之一是组织花在观察、交谈和观察用户与其业务互动的方式上的时间，我认为这非常非常重要。
