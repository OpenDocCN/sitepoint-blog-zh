# 据我们所知，PHP 是一个框架。

> 原文:[https://www . site point . com/last-we-checked-PHP-is-a-framework/](https://www.sitepoint.com/last-we-checked-php-is-a-framework/)

当谈到 web 编程语言时，PHP 可能保持着受到整个社区批评的记录。与 Ruby on Rails 和 Python/Django 等替代方案进行比较是常见的；PHP 的捍卫者很快批评了语言和框架的比较。但最终，开发人员会使用 Ruby on Rails、Python/Django 和 PHP。就 PHP。对于大多数 PHP 应用程序来说，这种语言是完美的，因为 PHP 在某种程度上是框架的 T2。

PHP 是为网络设计的。你可以将普通的 Ruby 或 Python 插入一个 web 服务器，然后很快启动并运行起来。但是，至少在基本层面上，你需要一个框架来处理 web 开发的常见问题。在 PHP 中，你只是刚刚开始。PHP 和 Apache 解决了请求数据、输出处理等问题，开箱即用。( [PHP 也掌握部署。](https://www.sitepoint.com/what-php-deployment-gets-right-ian-bicking-nails-it-at-last/) ) David Heinemeier Hanson，Ruby on Rails 框架的创建者，[称之为 PHP 的直接性](http://www.loudthinking.com/posts/23-the-immediacy-of-php)。

现在，考虑“普通的”PHP 框架。它们帮助你处理请求数据，管理你的输出，控制应用程序流——本质上，扩展了 PHP 的内置功能。因此，它们是 PHP 框架上的 PHP 框架。PHP 提供了大量现成的功能。但是当你想按自己的方式做事时，它就不碍事了，这在构建简单的数据库前端之外的东西时非常重要。

有多少分布广泛的 PHP 应用是建立在第三方框架上的？MediaWiki？这里没有框架。WordPress？这里也没有。Drupal？你明白了。每一个应用程序都有自己的框架，到目前为止，开发人员为他们的代码构建了一个适合他们试图创建的结构。这三家以及无数其他网站显然做对了一些事情——MediaWiki 甚至成为了世界十大网站之一。当构建这些“框架”时，PHP 一直在帮助他们。

当然，Cake、Symfony、CI——它们都可以帮助你构建 PHP 应用程序。但是与 Ruby 框架或 Python 框架不同，没有它们，编码是完全可以忍受的。当然，大多数开发人员倾向于在开发过程中创建他们自己的框架——我称之为 PHP 的 *DIY 框架*思维，在这种思维中，您构建堆栈中的最后一级，并且通过扩展，您确切地知道在引擎盖下发生了什么。

这又回到了经常被引用的“这项工作的最佳工具”的论点——但是作为某种“半框架”, PHP 只是在 web 环境中帮助你。我确信 Rails 对于无数的应用程序来说是非常棒的；Ruby 语法也有一点帮助。但除了基础知识，语言变得越来越不相关，一个框架将迫使你根据最初开发者做出的设计决策来构建你的应用。另一方面，PHP 允许你以你想要的方式快速组装你的应用程序，在基于 web 的方面帮助你，并且不需要打破语言/框架来实现你的目标。本质上， *PHP 是没有框架包袱的 web 层。*

Twitter [最近因为使用 Ruby on Rails](https://www.sitepoint.com/did-rails-sink-twitter/) 出现了一些宕机。现在，Ruby 或 Rails 没有任何问题。但是 Twitter 是一个消息系统，RoR 是一个应用框架。Rails 可以被变通以适应 Twitter 吗？也许吧。如果 Twitter 使用了 PHP，他们能避免这些问题吗？大概吧。Twitter 的开发者指出，他们并没有完全预料到这个网站会如此成功。如果他们是用 PHP 构建的，他们可能会在早期遇到其他(类似的)问题，在它仍然可行的时候重写整个应用程序，并构建一个功能强大的系统，可以很好地扩展(很可能是用 C 语言的消息传递平台)。使用 PHP 可能会有所帮助，但不是作为一种具有语法和库的语言，而是作为一个 web 开发平台，让开发人员可以自由地设计他们想要的系统。([他们也可以试试二郎](http://www.facebook.com/notes.php?id=9445547199)。)

所以下次你在寻找一个 PHP 框架来构建你的杀手级应用时，考虑使用最好的框架:PHP。它会给你自由去实现你想要的，你想要的方式和结果。快速破解？查看手册，可能是一行程序。一个大规模分布式消息平台？全速前进，用你的方式建造它。对于介于两者之间的任何东西，都有多种选择，但是如果您有时间，PHP 就是您需要的所有框架。
