# 所有访问过的链接都在哪里？

> 原文：<https://www.sitepoint.com/where-are-all-the-visited-links/>

我最近意识到的一件事是被访问链接状态几乎消失。在最近对六个与行业相关的政府网站的比较审查中，只有一个网站显示了访问过的链接。即使这样，它也只是在内容文本链接上，而不是在主网站导航上。我不得不承认，这真的让我大吃一惊；毕竟，这不是一个技术上先进的网站功能！

但是它比一些政府网站更普遍…

在过去的几个月里，我查看了一堆 Mc Farlane 奖的网站，以及最近的 T2 奖和 AIMIA 奖的网站，我惊讶地发现很少有网站包含访问过的链接。

我们不要忘记，这些网站正在提交他们的作品，以接受“卓越网页设计”(麦克法拉尼奖)、“展示行业最佳创新和项目”(AIMIA)的评判。

现在，我并不是说仅仅因为这些网站没有突出显示被访问的链接，就推断它们不属于“优秀”类别。但这是怎么回事呢？

在[我的上一篇文章](https://www.sitepoint.com/usability-for-developers/)，[詹姆斯·爱德华兹问我]( https://www.sitepoint.com/usability-for-developers/#comment-111095)关于“可用性和可访问性的交叉”的想法，我认为被访问的链接是一个很好的例子。

从可用性的角度来看，访问过的链接是有用的，因为它们向我们展示了我们去过的地方，并帮助我们了解我们可以去哪里。在大型网站上，访问过的链接是必不可少的，因为很难记住我们已经访问过的确切页面。它使浏览和搜索内容更快、更有效、压力更小。

从可访问性的角度来看，被访问的链接也是如此。想象一个患有后天脑损伤的用户，或者一个有认知障碍的人。通过突出显示访问过的链接来减少用户的内存负载，可以提高网站的可访问性，这也是 W3C 网站内容可访问性指南之一。

那么这些没有被实现的原因是什么呢？这纯粹是一个设计决策，还是它们已经过时了？

随着越来越多的开发人员和设计人员了解以用户为中心的设计实践，以及用户测试的增加，也许我们会看到被访问链接的重新出现…或者这有点太 1.0 了？
