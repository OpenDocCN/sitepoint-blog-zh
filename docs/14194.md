# 闭源 PHP

> 原文：<https://www.sitepoint.com/closed-source-php/>

对于一些开发人员来说，PHP 的一个大缺点是它没有提供发布闭源二进制 web 应用程序的官方方法。出于各种原因，商业应用程序的开发人员通常希望保持其源代码的私密性。

但是，有一些第三方解决方案。从给你带来 PHP 解释器的人那里来了 Zend 编码器。这个编码器将你的 PHP 转换成二进制形式，Zend 称之为“Zend 中间代码”。Zend Encoder 对小开发者的主要缺点是它的价格标签，目前是 2400 美元。使用 Zend 编码器创建的二进制文件可以使用免费的 [Zend Optimizer](http://www.zend.com/store/products/zend-optimizer.php) 来执行，它安装在许多 LAMP 托管帐户上，使得使用 Zend 编码器创建的二进制文件具有一定的可移植性。

Zend 编码器的竞争对手是 ionCube 的 [PHP 编码器](http://www.ioncube.com/index.php)。PHP 编码器的价格从 199 美元起，这让它更加实惠。要使用 PHP 编码器生成的二进制文件，您需要在服务器上安装他们的 [PHP 加速器](http://www.php-accelerator.co.uk/)，这不像 Zend Optimizer 那么常见。因此，它可能不太便于携带。

另外还有 [Turck MMCache](http://turck-mmcache.sourceforge.net/index_old.html) 。最近版本的 MMCache 包含一个 PHP 编码器和加载器，允许 PHP 脚本以中间二进制形式分发。价格合适——mm cache 是免费和开源的。它也可能对托管公司更有吸引力，因为它可以与 Zend Optimizer 同时安装。

存在不分发应用程序的编译版本的替代产品，因此可能不需要在服务器上安装解释器。然而，这类产品往往依赖于代码混淆或源代码加密。这两种方法的问题是，源代码可以被具有编程技能的人访问，即使这可能很难。

代码混淆将函数和变量的名称更改为奇怪的、类似垃圾的名称，并删除所有格式。这使得理解代码变得非常困难，尽管如果有人真的想借用它或者修改它，他们是可以的。

源代码加密的问题是，您还需要提供一种方法来解密源代码和密钥。这给了用户解密你的源代码所需要的一切。用户需要安装二进制可执行文件(比如 PHP 扩展)或 PHP 脚本来解密文件，用户可以通过逆向工程获得源代码。解密过程也会降低性能。

有些人可能会声称以二进制形式将 PHP 应用程序作为“闭源”分发是一件坏事，因为这对 PHP 本身所来自的开源社区是有害的。然而，PHP 团队本身也乐于看到 PHP 被商业化使用——[PHP 许可非常宽松](http://www.php.net/license/),以便“尽可能地帮助 PHP 变得流行”。我认为，如果能够以二进制形式发布他们的应用程序只会让更多的开发人员转向 PHP，这对 PHP 来说是一件好事。
