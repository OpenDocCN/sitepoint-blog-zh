太棒了。它的方法是让我们把 Javascript 看作计算机科学。因为很多人错误地认为 Javascript 只是一堆移动的 DOM，如果你真的是开发新手，DOM 基本上是文档对象模型，书呆子的缩写。但它基本上意味着你所有的 HTML。所以，如果你在右边这里有一个红框，你把它移到左边，你可以用 Javascript 说，当我点击这个东西的时候，把它移到这里。这是 Javascript 可以做的一件重要的事情，就是交互和移动 HTML 和 CSS，并呈现 HTML 和 CSS，但这最终只是 Javascript 的一小部分。我认为从像雄辩的 Javascript 这样的东西开始，它专注于语言本身，直到第 10 章(我认为是最后一章)才真正进入 DOM，这是一个非常好的开始。

那么我想你知道摆弄 jQuery 或任何其他库或框架没有什么错。JQuery 是一个非常强大的工具集，非常有用。使用 jQuery 不需要知道 Javascript 的每个小细节。最终，如果你除了从别人的站点复制和粘贴 jQuery 之外什么都不做，我不认为你会理解 Javascript 的强大。我认为你无法处理现实世界中的问题。我认为这是两者的混合。穿插，实际上训练你自己的语言，做一些像 jQuery 或 coffeescript 这样非常抽象的东西，至少我是这么认为的。
