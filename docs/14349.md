# 重新发现 Java

> 原文：<https://www.sitepoint.com/rediscovering-java/>

对于 Java Web 开发人员来说，这是一个激动人心的时刻。 [Java 5](http://java.sun.com/j2se/1.5.0/) (Java 2 标准版 5.0)刚刚发布，带着一堆[的新语言特性](http://java.sun.com/developer/technicalArticles/releases/j2se15/)迷惑那些已经习惯于对语言了如指掌的自满开发者。主要的 Java Web 应用服务器发布了支持 Java Web 开发最新标准的新版本( [J2EE 1.4](http://java.sun.com/j2ee/) )。并且所有的[主要开发工具](http://www.borland.com/jbuilder/)都在更新以支持这些标准的强大特性，比如 [JavaServer Faces](http://java.sun.com/j2ee/javaserverfaces/) 。

与此同时， **Java 在日常 Web 开发人员中的名声越来越差。越来越多的人意识到 Java 不再是*简单的*，你必须为一家预算只有一个小国家的公司工作，在这个平台上进行 Web 开发才有意义。像微软这样的竞争平台。NET 因为做了 Java 已经做了多年的事情而赢得了大量的好评，而 Java 被许多人认为是一个停滞不前的庞然大物。**

这种情绪很大程度上来自于这样一个事实:Java Web 开发是分层次发明的。首先是这种很酷、容易学习的编程语言:Java。有人想用它动态生成网页，于是 Servlets 被发明出来，它们运行在 Java 上。有人*还有人*想在所有 Java 代码中不丢失 HTML 代码的情况下构建动态网页，于是 JavaServer Pages (JSP)被发明了，他们产生了 Servlets。

几年后，Java Web 开发的前沿位于至少六层技术之上，作为一名新手，你会很快感觉到**迷失在流行词汇的森林中！**所以现在我们有爱好者询问如何用 Enterprise JavaBeans (EJB)建立一个基于网络的图片库——这相当于买一辆拖车把你从街角商店买的面包和牛奶带回家。

随着这个博客的发布，我在这里为我们其余的人写关于 **Java 的文章。**在接下来的几个月里，我将帮助你梳理那些流行词汇，了解你真正需要知道的东西，你真正可以使用的东西，以及你可以放心地留给跨国公司的东西。

最重要的是，我来这里是为了重新发现我过去在使用 Java 时找到的快乐。希望我也能帮你找到。
