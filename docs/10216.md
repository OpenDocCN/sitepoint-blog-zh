# HTML5 已经死了。HTML 万岁。

> 原文:[https://www.sitepoint.com/html5-is-dead-long-live-html/](https://www.sitepoint.com/html5-is-dead-long-live-html/)

Web 超文本应用技术工作组(WHATWG)——开发 HTML5 的组织——宣布他们将放弃版本号。HTML5 完成了。新标准只是“HTML”

该公告是由规范编辑伊恩·希克森发布的。他的[博客文章](http://blog.whatwg.org/html-is-the-new-html5)指出 HTML 标准已经成为一个活的文档:

1.  该规范将被称为“HTML”，其 URL 为[http://whatwg.org/html](http://whatwg.org/html)。
2.  WHATWG HTML 规范现在可以被认为是一个“生活标准”它比迄今为止任何版本的 HTML 规范都要成熟，所以称它为“草案”没有什么意义规范开发的快照模型已经被抛弃了。

## W3C 呢？

WHATWG 的声明是在 W3C 发布他们的 HTML5 标志后两天发布的。

术语“HTML5”不太可能消失，因为 W3C 标准批准过程是围绕一系列版本编号的技术规范构建的。目前，W3C 仍然致力于 HTML5，他们不一定会跟随 WHATWG 的领导。

然而，新的标志已经把水搅浑了，W3C 似乎把 CSS3、SVG 和 JavaScript APIs 归入 HTML5“品牌”之下。尽管他们在他们的 [logo FAQ](https://www.w3.org/html/logo/faq) 上后退了一点，但是当它自己的标准团体已经加入到市场潮流中时，将 HTML5 称为一种特定的技术变得越来越困难。

## 我们需要版本号吗？

许多开发者对 WHATWG 的想法表示惊讶和恐惧。它围绕着版本控制。没有这些数字，发展将陷入混乱。我们如何知道浏览器何时达到合规性？我们如何针对特定版本的 HTML 测试代码？

不要惊慌。放弃版本号可能感觉不自然，但是 web 开发一直并将永远基于一组完全不同的模糊技术。

有一种普遍的误解，认为 W3C 编写了供供应商遵循的规范。这导致许多开发人员得出这样的结论:当这些文档被标记为“草稿”时，HTML5 还没有准备好。事实上，浏览器是不断更新的。添加了 HTML 特性，如果它们被认为足够好，它们会被其他厂商分享或复制。只有当两个或更多的供应商实现了相同的特性时，它才会被记录在规范中。在大多数情况下，一个特性在被记录之前是可用的。

HTML 版本号造成了混乱。基于一个“草稿”的 HTML v5.0 规范写代码感觉不对。但是考虑到这一点:XHTML 1.0 是“最终版本”，但是直到版本 9 才在 Internet Explorer 中得到支持。类似地，CSS2.1 是完整的，但没有浏览器提供完整的无错误实现。最后，如果你使用的是稳定的 CSS3 特性，比如圆角，为什么 HTML(5)会有所不同呢？

今天，所有的浏览器供应商都声称 HTML5 兼容。然而，这是非常不同的水平，没有人支持一切。桌面、平板、移动、游戏和其他平台上有许多浏览器。每一种都有不同的特性，并沿着自己的道路发展。您可以自由地等待规范完成并被所有供应商完全实现的神奇的一天，但是您将会等待很长时间。即使那一天到来，浏览器也将进入下一个技术层面。

这是有争议的，但我同意 WHATWG。无所不包的版本号规范已经变得太大、太笨拙而不实用。幸运的是:

*   没有哪家厂商垄断了浏览器，他们也不能(轻易地)按照自己的方向改变标准。
*   主要的浏览器厂商都在互相合作。他们独自创新，但必须与他人分享想法，以确保某项功能成为公认的标准。
*   HTML 进化(大部分)是向后兼容的。例如，所有的浏览器都接受新的`<input type="email">`标签，但是旧版本会恢复到`<input type="text">`。
*   供应商很少从他们的浏览器中移除技术。谷歌正在从 Chrome 上剥离对 H.264 视频的支持，但是，总的来说，HTML 代码将在它最初工作的浏览器中保持可操作性。
*   我们总是需要特征检测。你永远不能依赖每一个 HTML 标签、CSS、JavaScript、SVG、Canvas、Flash 或任何其他技术的可用性。必要时，您可以检测它的存在并提供替代内容。这种情况不会改变——即使 HTML5 规范已经完成。

一个活的 HTML 规范反映了当今 web 技术的现状。
