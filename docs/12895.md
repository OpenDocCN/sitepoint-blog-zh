# 为什么多任务处理是浪费时间

> 原文:[https://www.sitepoint.com/multitasking-is-a-waste-of-time/](https://www.sitepoint.com/multitasking-is-a-waste-of-time/)

你经常听到人们赞美多任务处理的优点:<q>你做更多的事情</q>，<q>这是未来的方式。</q>

**丸子。**

如果你想在很长一段时间内做很多不太好的事情，多任务处理是很好的选择。你当然可以:在查看邮件、发微博、写报告、尝试新的网络应用和在脸书上聊天之间切换。你忙吗？大概吧。你有效率吗？大概不会。

在我看来，这里有两个关键问题:

1.  做非必要的任务
    由于拖延和自我分心，你通过做除了该做的事情之外的其他事情来推迟某些任务。
2.  任务转换
    另一个经常被忽视的问题是任务转换——在一个任务和另一个任务之间转换。这是我想在这篇文章中引起你注意的问题。

## 任务切换

你的意识思维有效地像单线程处理器一样工作。我们以线性方式做事最好。一件事接着一件事。多任务处理就像试图让自己成为多线程处理器。不幸的是，进化还没有跟上，所以你还停留在你的老式单线程大脑。当然，你可以在某种程度上进行多任务处理，但这有点像黑客，而且不太管用。

多任务处理的问题是你不断地从一件事到另一件事来回变化。这意味着你需要反复地将你的思想从一个任务转移到另一个任务，然后在你实际做任务之前，反复回忆每个任务的所有相关信息。这都需要时间。当你进行多任务处理时，你是有效的(用计算机术语来说)，打开一个文件，用它做一些事情，关闭它，打开另一个文件，用它做一些事情，关闭它并再次打开前一个，用它做一些事情，再次关闭它，等等。这显然是低效的。事实上，这取决于你在不同任务之间切换的频率。你可以[浪费 40%以上的工作时间](http://www.psychologymatters.org/multitask0306.html)仅仅是在任务切换上！

## <q>可以做得更好</q>

还有一个更好的不做的理由:质量。事实证明，与单独完成每项任务相比，试图同时做多件事情通常会导致每项任务的表现更差。

当然，说了这么多；以线性的方式处理事情并不总是可能的，但是一般来说，如果你想完成更多的事情并且达到更高的标准: ***停止多任务*** 。TTFN。
