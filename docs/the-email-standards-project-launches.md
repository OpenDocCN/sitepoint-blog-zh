# 电子邮件标准项目启动

> 原文：<https://www.sitepoint.com/the-email-standards-project-launches/>

![email standards project](img/d01eb757d2aae50e2c41386332e5a7e7.png)正如 SitePoint 之前报道的[，来自](https://www.sitepoint.com/the-email-standards-project-moderating-the-email-wars/)[fresh view](http://freshview.com/)([竞选监视器](http://campaignmonitor.com/)的制造商)的团队已经承担起改善桌面和基于网络的电子邮件客户端对 HTML 电子邮件的标准支持的任务。

昨天，随着[电子邮件标准项目](http://www.email-standards.org/)网站的正式启动，这一目标又向前迈进了一步。

Freshview 的 Dave Greiner 在上个月接受 SitePoint 采访时总结了我们为什么需要推动标准支持，以及电子邮件客户端和网络浏览器中的标准支持之间的相似之处:

> 我们所做的是我们建立了一个标准的基线，我们认为这至少应该是一个好的开始，让每个人都能达到。我们目前正在研究所有主要的电子邮件客户端，整理出一个愿望清单——前 10 个。信息是，“如果你能做任何事，做这 10 件事，它会让你非常接近我们需要的地方。”

阅读[为什么我们需要电子邮件标准支持的背景](http://www.email-standards.org/why/)和[你能做些什么来帮助](http://www.email-standards.org/what-you-can-do/)，然后去阅读更多关于这一运动的信息或在他们的[新博客](http://www.email-standards.org/blog/)上查看电子邮件标准项目的下一步(完成[电子邮件客户端的运行记录和每个](http://www.email-standards.org/clients/)基于他们如何处理[酸性测试](http://www.email-standards.org/acid-test/)的评级)。
