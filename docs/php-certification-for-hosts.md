# 主机的 PHP 认证？

> 原文：<https://www.sitepoint.com/php-certification-for-hosts/>

认证的话题不时在 PHP 圈子里被提起(通常稍后又会消失)，作为向潜在客户/员工证明你作为 PHP 程序员的勇气的手段。程序员证书是否在总体上“有效”,是一个长期争论的话题。现在，我不认为有任何组织有足够的信誉和资源为 PHP 开发人员开发一个有效的程序。无论如何，可能没有足够的需求使其可持续发展。

也就是说，最近我一直在想认证是否适用于虚拟主机公司——那种提供“启用 PHP”的每月 5 美元账户的公司？我自己在过去有过共享主机的糟糕经历，也听说过其他人有类似的问题，似乎常见的是那些运行服务器的人对 PHP 本身知之甚少或一无所知——这只是添加到列表中的东西；

> –PHP 安装的
> –1 个 MySQL 数据库
> –10 个电子邮件帐户

有多少 web 主机向潜在客户提供了比“PHP 已安装”更多的细节？你很少能找到细节，比如它是作为 Apache 模块还是 CGI 可执行文件安装的，有哪些扩展可用，他们使用的是哪个版本的 PHP，php.ini 设置能被修改吗。htaccess 文件，启用/禁用了哪些 PHP 功能，有哪些升级策略/程序等。等等。如果你正在做比用 PHP 写邮件更重要的事情，这些事情是必须知道的。

此外，当您考虑安全性时，主机方面的无知对每个人来说都是危险的。有多少主机知道[的意义——比如 disable-url-fopen-wrapper](http://www.php.net/manual/en/install.configure.php#install.configure.disable-url-fopen-wrapper) ？

我认为托管公司的认证计划可以改善很多人的生活，既可以提高客户的 PHP 托管质量，防止主机出现令人讨厌的安全问题，又可以给 PHP 一个更好的名声。

就实现而言，认为主机要获得认证，应该有两个主要活动；

–向客户公开 PHP 安装的确切性质(即在网页上),并发布明确的升级政策/程序(例如，在我们对次要版本(如 4.2.x 到 4.3.x)进行升级之前，您会得到 3 个月的警告)以及对扩展的承诺等。基本上为 PHP 开发人员提供了所有必要的信息，让他们确切地知道自己在做什么。

–至少有一名工程师通过了一项考试，该考试侧重于安装和配置 PHP 的细节，并提供如何使用 PHP 的基本理解，以便主机真正了解关闭 register_globals 的后果，并可以建议客户如何继续。它不需要涵盖面向对象编程，但是像安全性这样的主题应该是一项要求。

托管业务不断变化的性质(通常是小公司)可能意味着重新认证是必要的，也许是每年一次。

虽然它最初可能不会在托管公司中流行，但如果它是他们可以宣传的东西，它就会成为质量的标志，将那些试图经营真正业务的人与牛仔区分开来，它将吸引业务。

至于应该由谁来运行这个，我不得不说是 Zend 或那些提供商业 PHP 支持的公司之一，比如 T2 Tap Internet 或 T4 ThinkPHP。我的猜测是，它或多或少可以自筹资金——500 美元/年的认证费用可能足以支付运行该项目的成本。

不管怎样，这只是我的奇思妙想。现在，我很庆幸自己不需要为 PHP 寻找共享主机。一段时间以来，我一直想花钱买一个用户模式的 Linux 账户，这似乎是运行 PHP 更明智的方式。如果我有机会，我会分享我的经历。
