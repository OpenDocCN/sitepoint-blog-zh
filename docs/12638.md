# 道格拉斯·克洛克福特谈网络标准和 JavaScript

> 原文:[https://www . site point . com/Douglas-crock Ford-on-web-standards-and-JavaScript/](https://www.sitepoint.com/douglas-crockford-on-web-standards-and-javascript/)

![Douglas Crockford at Web Directions South 2008](../Images/bce1fcb2c53fcb703d2f53a1f880608d.png)

**去年，当我编写简单的 JavaScript[](https://www.sitepoint.com/books/javascript1)时，我成了一个 JavaScript 爱好者，因此，几周前，在 [Web Directions South 2008](http://south08.webdirections.org/) 大会上，我与[道格拉斯·克洛克福特](http://www.crockford.com/)——可能是世界上最大的*JavaScript 爱好者——坐在一起，谈论我们对 JavaScript 的共同热爱，这尤其令人激动。***

JavaScript 最令人惊讶的事情之一是，这样一种优雅、微妙而强大的前瞻性语言可能诞生于 90 年代中期所谓的“浏览器战争”的激烈竞争创新。实际上，Netscape 要取得成功所需要的只是一种可以在浏览器中运行的简单脚本语言，但不知何故，它最终构建的是这种能力惊人的小编程语言。我问道格拉斯·克洛克福特这是怎么回事:

> 他们真的很幸运。考虑到创造这种语言的过程，我们应该得到更糟糕的东西，因为他们没有仔细设计需求。他们肯定没有给它的设计或实现足够的时间。他们拿了一个原型，只是为了证明概念，这就是他们运送的东西。并且它有所有你期望这样的实现会有的问题。这就是我们所拥有的。部分是基于这种实现，这种语言得到了它所拥有的可怕的名声。许多这样的缺陷仍然存在于语言中。

在会议上的发言中，Crockford 概述了一些基本的安全问题，他希望看到这些问题随着 JavaScript 的发展而得到解决。撇开问题不谈，也许 JavaScript 作为一种语言最大的优势在于它对初学者来说是多么容易理解。

随着 JavaScript 的发展，我想知道，我们是否能够保持这种低门槛，使 JavaScript 成为你的第一语言，并在一两天后感到自信？

> 我想是的，我认为我们需要。我认为如果我们不保持语言的简单性，我们将会犯一个悲剧性的错误。我想对这门语言做的大部分修改都是为了让它更简单。有一些垃圾，有一些吸引人的讨厌的东西，我们不需要，人们会依赖它们。没有它我们会过得更好。
> 
> 不幸的是，关于网络的事情是，一旦一些不好的东西进入其中，需要几年的时间才能把它弄出来。Ajax 直到 2005 年才出现，但是我们开发 Ajax 所需的所有技术在 2000 年就已经存在了。这五年的大部分时间都花在了从市场上移除旧浏览器上，直到 IE6 上有了足够多的受众，Ajax 才成为一个可行的应用程序平台。

克罗克福德演讲的主要部分是他如何确信第二次“浏览器战争”——尽管前景可能很可怕——正是将 JavaScript 和网络的发展踢回正轨所需要的。

克罗克福德认为，从根本上说，网络标准在引领网络创新的努力中失败了:

> 例如，CSS2 是不可实现的，最终它不得不被修改为 CSS2.1，这是一种尝试，将 CSS2 削减到人们实际上能够理解如何实现的程度。这个顺序完全颠倒了——或者说它开始是颠倒的，但最终他们做对了。让我们看看什么实际上可以工作，并制定一个标准，然后让每个人都赶上对方。我认为这是标准的恰当角色。
> 
> 我现在看到的 HTML5 发生的事情令人震惊。有一些东西我真的很喜欢:我真的很喜欢他们发现了 HTML 解析的规则。太棒了。早就该这样了。你可以看到他们正在做的任何一个单独的功能，然后说，“是啊，这很有意义。”但是有太多的东西，没有一个好的权衡，没有一个复杂的预算。这不是出于真正的需要，而是委员会面前闪闪发光的东西。
> 
> 所以，我想找到一种方法，在这个过程中注入更多的纪律，我认为一种方法是把它变成一个评估和描述的过程，在这个过程中，我们将观察在野外发生的事情，并记录最好的事情。
