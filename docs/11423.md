很多关于已经去世的名人的事情，很明显这很有趣，迈克尔杰克逊排名第一。排在第二位的是实际上没有死的杰夫·高布伦，然后是帕特里克·斯威兹，然后是排在第五位的坎耶·韦斯特——死了！有趣的是，谷歌搜索并不是一个很好的事实来源。分手，在澳大利亚非常普遍。很多乐队解散了，但第二个是 Telstra 解散，这是我们在澳大利亚的大型电信公司。它被分成了更小的片段，所以更多的人在搜索关于澳大利亚电信分手的信息，而不是除了绿洲乐队之外的任何其他名人分手或乐队分手的信息。绿洲名列第一。

 **凯文:是的，这是一个新的数字电视频道。

凯文:第二个故事是公共 DNS。而我们今天一整天都在说的这家大公司也在发布域名服务。这是您的互联网服务提供商提供给您的信息，您可能知道也可能不知道，但每当您在浏览器中键入网址或使用任何程序访问互联网上的服务器时，它通常会将名称转换为数字地址。所以 sitepoint.com 映射到一系列特定的数字。

所以你的电脑不断地询问你的互联网服务提供商这个名字的数字地址是什么，谷歌已经决定这是目前网络上最慢的东西，所以他们要自己做。就像他们对 Chrome 所做的一样，他们认为所有其他浏览器都太慢了，所以他们打算建立自己的浏览器，向人们展示快速浏览器是如何实现的。他们已经决定让网络更快的下一个最容易实现的目标是让 DNS 查询更快。

我调出了 PCMag.com 的一些速度测试，他们实际上对他们的默认 DNS，谷歌 DNS，然后开放 DNS 做了一些基准测试，这是另一个有点像谷歌的提供商，这只是毫秒的差异，我的意思是这是你永远不会注意到的事情。

凯文:他们是在测试受欢迎的网站还是更不知名的网站？

凯文:好的，因为我在看谷歌的技术文档，关于他们做了什么，为什么要这么做，还有一些，他们说，只是数量庞大的名字慢慢淹没了服务器。因此，如果您的互联网服务提供商在 5 年前建立了他们的 DNS 机器，很可能今天它处理的域名比 5 年前多得多，因此，正如他们所说的那样，它供应不足，他们跟不上，所以，就像现金大小和足够大的有用之类的事情。

他们所做的一部分只是确保他们有足够大的集群来以有效的方式处理今天的 DNS 需求，但他们在这方面最具创新性的事情是预取结果。特定名称还具有与其相关联的生存时间和到期时间。

当我检查 sitepoint.com 的地址时，记录说，“这是目前的地址，但这可能只在 30 分钟内有效，”所以，如果我们改变到一个新的服务器，理论上只需要 30 分钟新地址就可以在互联网上传播。这意味着，如果在过去的 30 分钟内，no 向你的互联网服务提供商的 DNS 服务器询问 sitepoint.com 的地址，那么当你询问它时，它将不得不向权威服务器询问那个地址，而不是给你一个缓存的结果。

谷歌正在做的是记录哪些域名是受欢迎的或最有可能被请求的，并确保即使最近没有人要求他们的地址，它也会提前与权威服务器进行核对，以获得最新的 IP 地址，这样当你询问谷歌的 DNS 时，它可以立即给你答案，而不必询问服务器。

这似乎是这里的核心创新，他们说，在最糟糕的情况下，如果您访问一个最近没有被请求的不知名网站，那么特定的 DNS 查找可能需要数百毫秒的时间，如果该网站在其他域上托管 CSS 文件、JavaScript 文件和图像文件，那么这些查找可能会堆积起来。我认为这可能是最坏的情况，但是他们的技术页面上的图表显示了五个序列化的请求。因此，在 11 秒的页面加载过程中，它必须执行五个请求，总共需要 7 秒钟，因此在理论上，对于这种特殊的最坏情况，如果他们真的快速加速 DNS，他们可以从 11 秒的页面加载中节省 7 秒钟。
