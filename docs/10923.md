## 采访记录

凯文:2010 年 6 月 4 日。CSS 万事通 Russ Weakley 和我谈论网络教育、完整代码出版社等等。我是凯文·杨克，这里是 SitePoint 播客#64:与拉斯·维克利一起学习网络。

本周我和拉斯·威克利在一起。Russ 你拥有并经营 [Max Design](http://www.maxdesign.com.au/ "Max Design: engaging, user-focused web design, development, CSS training") ，对吗？

鲁斯:没错。

**凯文:**那么什么是 Max 设计？

我想你可以说这是一个网页设计行业，但是我们做各种各样的事情，从设计到开发网站和教授 CSS。

凯文:是的。所以我的印象是，你就像——你是那里的全职员工，你自己做很多工作，但你也有朋友，你可以在任何给定的项目中利用他们；你知道人们有各种各样的长处。基本上是这样的吗？

是的，这是一个孤独的世界，办公室里没有其他人。

凯文:(笑)

**Russ:** 但是，我们使用所谓的即插即用模式，如果我们需要一套技能，幸运的是，我有很多可以联系的人，我可以很快让人们加入，所以我们根据需要建立一个团队，而不是现有的或已经存在的团队。

凯文:我想很多人在网上都是这样工作的。

鲁斯:是的。

凯文:人们对你的期望因项目而异，我经常猜测客户会给自己带来什么，对吗？

鲁斯:绝对是。几年前我们学到的一件事是，如果你有一个强制的解决方案，那么它适合一些客户，但不是所有人。而即插即用模式意味着它是完全可适应的；你可以扩大或缩小一个团队，你可以根据需要引进技能。所以，是的，这只是一个更宽松的模型，它非常适合网络，因为它们也可以存在于不同的城市和不同的国家，这没关系。

Kevin: 我们今天和 Russ 谈话的原因是因为你说作为一名网页设计师，尤其是像你这样的自由职业者，这是一个孤独的世界，所以我们让 Russ 离开他的办公室一段时间，Russ 将在七月初参加 [SitePoint 的 CSS 直播课程](https://www.sitepoint.com/courses/css-live/)。

7 月 5 日。

凯文:7 月 5 日开始，紧接在 [PHP Live](https://www.sitepoint.com/courses/php-live/) 之后，从 6 月中旬开始，我将负责指导罗斯的课程。所以在接下来的一小段时间里，我们都要上课。

Russ 这不是你第一次教人们网络，对吗？

不，不，我已经做了很长时间了；太久了。

凯文:(笑)嗯，你是这么说，但你非常渴望接手这个新项目。

鲁斯:绝对是。我真的很喜欢它。为人们做项目，实现网站的目标是一件很棒的事情。但是没有什么能真正比得上教人们并让人们得到一些东西，这真的，听起来不老土，这是一个非常丰富的经历。

凯文:我知道你的意思。我想我第一次见到你是通过 Web 标准组织，这个组织主要是澳大利亚的，但也有一些在世界各地涌现。网络标准组织是做什么的？

Russ: 嗯，它主要是——它的目的最初是一个骗局。我们真正想做的是把网页开发者和网页设计师聚集在一起。在早期，web 标准的概念对人们来说是非常新的东西，也是一个非常好的起点。所以我们刚刚成立了一个叫做网络标准组织的组织。但主要是让设计师和开发人员——给他们一个空间，他们可以在邮件列表中聊天，但更重要的是，他们可以在那里见面，进行演示和分享，你知道，诸如此类的事情。所以它已经持续了很长一段时间，它们是突然出现又消失的会议，所以正如你在澳大利亚所说的那样，但在新加坡、俄罗斯、伦敦和新西兰也有一些会议。因此，各种不同的国家在不同的时间举行了会议。

凯文:罗斯在悉尼，我在墨尔本。所以我只知道你是网络标准组织背后的主导力量，但是当然我们在墨尔本也在做我们自己的事情。不过，我最近才真正了解你，当时你邀请我去澳大利亚参加 CSS 课程——对不起，不是 CSS，是 JavaScript。这方面的材料成了我上个月主持的 [JavaScript Live](https://www.sitepoint.com/courses/javascript-live/) 课程的核心，那非常有趣。

你做工作坊有多久了？

**鲁斯:**嗯，大概五年左右吧。

凯文:五年了。

Russ: 我参加了两种不同的研讨会；一个是我们一起做的，有点像我们在不同国家旅行的系列。但我的很多东西实际上只是现场培训，一家公司会让我进去，有三到十名开发人员或设计师，我只教他们一天，这也很有趣。

凯文:嗯。所以这类事情，网络标准组织，研讨会，对你来说都差不多是同时开始的。

鲁斯:是的，我想是的。我猜可能像你一样，当我开始学习 CSS 时，这是一个非常令人沮丧的经历，当时没有太多的资源，而现在有书，一切都在那里。所以它真的开始于，对我来说是痛苦的，如果我没有成功，那就太好了，你知道，如果我真的能停止一些痛苦。

凯文:嗯。所以作为一个网页教育者，我想你会不断地接触到那些第一次学习网页设计的人。我发现我很纠结。在 SitePoint 工作，在一个被生活和呼吸着网络的人们所包围的办公环境中，每隔一段时间，我不得不停下来提醒自己，并不是每个人都像我周围的人一样了解这些东西。你知道我们五年前写的那篇关于 CSS 入门的文章可能不再是学习 CSS 的最好方法了。

鲁斯:嗯。

凯文:对我来说，以一种新的形式回到这些概念是令人兴奋的，它刷新了我的记忆，让我知道第一次学习这些东西意味着什么，也让我看到自从我上次从初学者的角度看它以来它发生了如此大的变化。对你来说，现在的网络对初学者来说变化很大吗？

哦，是的，我完全同意。首先，你的第一点我认为，是的，我们必须不断地把自己放在那种心态中。我认为有很多文章写得非常好，但它们通常是从了解网络的人的角度来写的，所以我们回到“回到”，这听起来很贬义，坐在一个你可以讨论对每个人都有意义的事情的水平上，这是非常重要的。就它如何变化而言，是的，它在不断变化。当我在黑暗时代学习 CSS 的时候，真的很少，我想大概有三四个网站在做，所以那里的资源很少。你知道，有一本埃里克·迈尔的书或者类似的东西。

凯文:当你看到“这样做的网站”时，你是在说 CSS，全 CSS 布局设计吗？

Russ: 哦，不，没有，可能和你一样，当你开始转向 CSS 时，没有大的网站在做这件事。有——我的意思是有解释 CSS 如何工作的资源。

凯文:哦，对了。

Russ: 过了一段时间，第一批全 CSS 网站出现了。我记得当《T2》连线《T3》时，当道格·鲍曼推出《连线》时，这可能是一个真正的转折点，人们突然意识到，嘿，这现在已经成为主流了。但是在那之前有一段时间，你知道，很多人谈论它，它更理论化；“我们应该这样做。”

凯文:嗯，是的，CSS 多年来都是可选的额外服务。

鲁斯:是的。这是一件令人沮丧的事情，因为你知道，你正在与网景 4 竞争，可能你的许多听众不知道它是什么，但基本上它是一个痛苦的捆绑。

凯文:(笑)

Russ: 但是，是的，它已经彻底改变了。我认为最大的转变是，在某些方面，初学者现在正在处理信息过载的问题，而当我学习的时候，你知道，是信息的缺乏。所以现在不是能不能找到它的问题，只是有这么多的信息在那里，这更是一个我从哪里开始的问题？有许多文章提供了大量信息，但其中许多文章并没有从基础层面开始，说这是一些基本的构建模块，让我们从那里开始构建。

凯文:当你谈到信息过载时，你知道，如果你接触到如此广泛的资源，你会发现许多混杂的信息和许多相互矛盾的信息。

鲁斯:是的。绝对的。我认为其中一个经典的例子是 CSS 重置的概念，对于那些没有听说过的人来说，我想这是关于将 CSS 放回到一个公平的竞争环境中，在那里你可以强制所有浏览器以相同的方式呈现页面，然后以此为基础构建你的 CSS。网上有很多有争议的问题和观点。是的，如果你是新手，你可以读一些东西，觉得有道理，然后，你知道，五分钟后看到一篇文章，断然否认另一篇文章中陈述的一切。所以，是的，它是；有很多东西不仅要解决，而且要弄清楚你在学习时所处的位置。

凯文:我知道，你知道，当我设置 JavaScript 课程时，我回去了，因为我们有这本书，这是我最近写的，但我想尽可能多地引入我多年来收集的旧示例和资源。看着我以前做的一些东西，我想，哇，今天这真是个糟糕的建议，它仍然存在。

鲁斯:是的。我跑的每一条路都是这样开始的，顺便说一句，我已经在身后留下了一条毁灭之路，我想我们所有人都是这样；我们承认与否是另一回事，但是你经常改变你的方法，并且回顾你一年前、两年前做的事情。希望你看着我，然后说，我能明白我为什么这么做，但是我会用稍微不同的方式去做。但是你越往回走，你就越开始走，我的上帝，我在想什么，所以我认为这很重要。

Kevin: 这是我们在 SitePoint 努力做到的事情。你知道，我们有这样的书——书名中有“正确的方法”这样的词的书。

**鲁斯:**是的。

凯文:它们都是很棒的书，但特别是在过去几年的网络上，我觉得事情开始变化得如此之快，你真的不能，我的意思是你今天可以说这是做这件事的最佳方式，我可以证明为什么今天是这样。但是真的很难挑出哪怕是最简单的技术——目前用于建立一个网站，并说即使在六个月的时间里这仍将是最好的方法。所以我觉得我不得不停止责备那些听从坏建议的人，这真的需要改变方法。

我完全同意你的看法。我的意思是在 Web 标准组织的运动中很早就开始的事情之一，我的意思是显然在我出现之前就有人了，但我过去真正感到担心的事情之一是人们会用非黑即白的术语谈论简单的事情；这是做这件事的方法，那不是做这件事的方法。我总是发现，第一，对于刚开始的人来说，这是一个很大的障碍，因为当他们从技能角度、软件角度或 CMS 角度可能无法做到时，对他们说你必须这样做，这不是一个好方法。但是就像你说的，你知道，现在正确的在将来可能不正确。

另一件事是，有时这是完全值得商榷的。我想很早的时候，我学到的第一课就是，我认为这是 [SimpleBits](http://simplebits.com/ "SimpleBits") 做一些快速测试，会问你如何标记这个解决方案？

凯文:哦，哇。

人们提出了各种各样的方法，并激烈地争论这是最好的方法，很快就清楚了，即使像语义标记这样的东西也包含了太多的个人观点。

凯文:是的，有时候这很主观。

**鲁斯:**是的！

**Russ:** 什么卖给我的？嗯，我想对我来说最大的事情是我在人们面前做了很多事情，站在人们面前谈论 CSS。但是这提供了一些全新的东西。有很多在线课程，也有很多研讨会，但我很少遇到在一个地方涵盖所有范围的东西。当你向我描述课程时，你知道，我当场就被说服了；你可以开设基于一天或一节课的课程，它们涉及一系列不同的媒体，因此它们可以包括视频、文章和演示。最重要的是，你可以让人们在课后直接去聊天，分享他们的想法。我的意思是，它很好地提供了你需要的一切，然后你谈到了 Q & A 会议，最重要的是，他们可以与主持研讨会的人现场交谈，所以这是一个令人惊叹的设置。

凯文:是的，我现在谈论 JavaScript Live 时用的是过去时，但事实是我们以 Live 的形式进行课程，有三千多人注册了，他们一起参加了为期三周的课程。当课程进行时，门是关着的，我们不接受任何新人加入课程。但在过去的一周，我们重新推出了 JavaScript Live，作为一个非 Live 版本。所以所有这些材料，12 节课加上三个 T2 问答环节和一个私人论坛，你现在可以购买这些材料，并开始按照自己的速度学习。

哦，不，不要告诉别人。你很专业。你把一切都安排好了。

凯文:是的。我发现这些课程的社会方面意味着，即使不是每个人都能理解相同的解释。如果你想出了 CSS 定位上下文的完美解释，从空气中拉出一些东西，即使在你最好的情况下，你也只会让 90%的人完全点击你的解释方式。

鲁斯:绝对是。
