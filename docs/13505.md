# 第 2 集:真实世界的正则表达式

> 原文:[https://www . site point . com/episode-2-real-world-regular-expressions/](https://www.sitepoint.com/episode-2-real-world-regular-expressions/)

让我们马上开始吧:我喜欢正则表达式。真的，我知道——它们是文本处理的瑞士军刀，任何有名望的开发人员都离不开它们。

当然，我们都知道他们有多危险。一如既往，权力越大，责任越大。

尽管如此，如果你知道如何——何时——以及为什么——使用正则表达式，它们是必不可少的。所以本周，正则表达式将是我们的主题。

下面是五个正则表达式。它们中的每一个都匹配一个真实世界的字符串；也就是说，您可能希望从更大的文档中提取一段半结构化的文本。这里有一个问题示例，可以让您了解我的意思:

0.  `[0-9]{5}`

当然，这是美国的邮政编码。

那么，这些正则表达式匹配什么“东西”呢？在这个测验中，我们假设 regex 引擎以不区分大小写的模式运行:

1.  `[A-PR-Y0-9]{3}-[A-PR-Y0-9]{3}-[A-PR-Y0-9]{4}`
2.  `&(?!(w+|#d+);)`
3.  `(-?(?:0|[1-9]d*))(.d+)?([eE][-+]?d+)?`
4.  `([da-f]{2}:){5}([da-f]{2})`
5.  `<[^>]*?>`

当然，既然我们在这里讨论正则表达式，如果我不以一个问题的代价给你两个问题，那我就错了。

在每种情况下，正则表达式都有问题。例如，上面的邮政编码正则表达式与目前许多地址使用的 ZIP+4 格式(即 66044-0034)不匹配。
