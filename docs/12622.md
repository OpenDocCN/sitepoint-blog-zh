# SourceForge 增加托管应用

> 原文：<https://www.sitepoint.com/sourceforge-adds-hosted-applications/>

成立 9 年的开源应用程序源代码控制和项目管理服务公司![](img/97c3bee36fbe62b54fd6bde1b7f6712a.png "sourceforge-logo")[SourceForge.net](http://sourceforge.net/)，为其服务阵容增加了一项新功能:托管应用程序。Blogger [ Sarah Perez 今天早上注意到了](http://www.readwriteweb.com/archives/sourceforge_announces_hosted_apps.php)的托管应用功能，尽管它显然是在大约一个月前通过 SourceForge 支持论坛为开发者试运行的。

托管应用的目的是向项目团队提供第三方开源应用的虚拟化版本，以便团队可以利用有用的开源软件，而不必担心设置问题。应用程序将被托管在 SourceForge.net 几个月前开放的新芝加哥数据中心。SourceForge.net 计划扩大其最初的产品供应，包括 phpBB(论坛)、MediaWiki(维基)和 LimeSurvey(调查)。本质上，这意味着这个新特性实际上是 SourceForge 开发人员可以获得的无限数量的新特性。

托管应用程序背后的想法是为开发人员提供一种轻松部署和使用流行应用程序的方式，而无需设置它们。开发人员可以自由地专注于构建他们的应用程序，而不必担心项目基础设施。因为 SourceForge.net 上的任何应用程序都可以成为托管应用程序，所以托管应用程序功能也可以用于该网站上开发的应用程序的演示目的。

SourceForge.net 运营总监 Jacob Moorman 在[论坛帖子](http://sourceforge.net/community/forum/topic.php?id=3358&page)中列出了新托管应用功能的以下优势:

*   我们新的托管应用产品消除了部署受支持应用的开销；只需选择加入并立即开始使用该应用程序。不需要处理配置文件和安装程序。
*   我们的托管应用程序由独立于 project web 服务器的专用数据库和 web 服务器池提供服务，因此您无需应对 project web 共享托管环境的安全限制，或者 project web 的出站邮件和连接限制。
*   我们维护托管应用程序的应用程序代码，并将在供应商提供更新时进行部署。这将降低托管应用中发现的漏洞带来的风险，并消除项目的主要管理负担(安装更新)。
*   我们会定期备份托管的应用程序数据，但您也可以根据需要轻松进行应用程序备份。
*   我们执行应用程序测试、调整和监控，以确保托管应用程序正常运行。如果出现服务故障，我们会做出响应并解决问题。如果在应用程序中发现缺陷(通过我们自己的测试或通过最终用户报告)，我们将修复缺陷或将缺陷提交给供应商进行修复。
*   所有托管的应用程序都使用我们的集中认证基础设施(用户使用他们的 SourceForge.net 用户名和密码登录)，但保留应用程序的权限(授权)(例如，MediaWiki 的现有用户会发现权限处理完全符合他们的预期)。
*   由于该产品是集中管理的，我们对基础设施或托管应用本身所做的任何改进都将立即适用于所有项目。托管应用程序产品减少了我们为产品添加主要新功能的开销，因为所有应用程序共享公共集成点和公共基础架构。
*   由于托管应用在开源许可下可用(顺便提一下，我们向社区发布我们对供应商提供的代码所做的任何补丁)，这种集中式服务有可能快速增加开源应用的用户群，并为这些应用的进一步改进提供高质量的反馈。
