## 这是什么？

我们的目标是什么？实际上非常简单:我们将构建一个 HTTP 服务器。

更准确地说，我们将使用 Celluoid 库构建一个用 Ruby 编写的非常不完整但易于扩展的 HTTP 服务器。

不要指望它会像 Apache 或 Thin 一样——但是，在这个过程中，我们会学到很多关于赛璐珞和 HTTP 的知识，这(可以说)是开发人员需要理解的重要网络协议。

## 关于 HTTP 的一点点

当然，如果我们想要构建一个 HTTP 服务器，我们需要知道 HTTP 是什么以及它是如何工作的。

如果你不知道的话，HTTP 是一种用于从服务器向客户端传输 HTML 的协议(这并不总是如此，但却是最常见的)，也是最常见的，用于客户端告诉服务器做某些事情。

该协议基于请求。因此，客户机可以向服务器发出一个“GET”请求，以获取某个页面的 HTML。此外，客户端可以发出“POST”请求，向服务器提供一些数据。

关于 HTTP 很重要的一点是协议本身是*无状态的*。这意味着每个请求都不知道任何先前的请求。

我们将只实现 GET 请求；这有两个原因。首先，它是 HTTP 服务器的核心功能。第二，非常容易实现。

但是，我们将编写我们的代码，以便它足够模块化，可以很容易地添加其他方法。
