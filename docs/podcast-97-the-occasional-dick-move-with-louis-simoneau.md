## 采访记录

凯文:2011 年 1 月 28 日。SitePoint 的首席技术编辑谈论 WebM、HTML5 和 web 标准的未来。我是凯文·杨克，这里是 SitePoint 播客#97:路易斯·西蒙诺偶尔的鸡巴动作。

今天加入我们的 SitePoint 播客，我非常高兴我的受访者和我在同一个房间。路易斯·西蒙诺加入了我们。你好路易斯。

路易:您好。

凯文:路易斯是 SitePoint 网站的首席技术编辑。什么意思，你在这里做什么？

**Louis:** 所以，我们所有的书籍和在线出版物都要经过两个阶段的编辑过程，首先是技术编辑，我从作者那里拿到手稿，确保代码有效，符合标准，是最佳实践，确保代码的所有解释对该领域的初学者来说清晰易懂，然后会交给语言编辑，他负责语法、拼写、标点和风格。所以，是的，我负责我们所有书籍和在线出版物的所有技术方面。

凯文:我想你会说他是技术书呆子气的头儿(笑声)。是的，当 SitePoint 还是一个小公司的时候，我曾经做过很多这样的事情，是的，你是我们在这里发布的所有东西的技术质量的最后一道防线。

路易:这是一个公平的评估，是的。

凯文:好的。嗯，我想让你加入的原因是，当你不编辑书籍时，你也会偶尔为 SitePoint 博客写稿，上周播客中我们谈到的话题之一是谷歌决定从谷歌浏览器中删除 H.264 视频。我知道你是个谷歌迷。

路易:的确是的。我不会说我是谷歌的粉丝，我不认为这是一个准确的表述；在这种情况下，我实际上——我对这个问题的看法有点像 Mozilla 粉丝，我想。

凯文:一个 Mozilla 粉丝。

我认为这种做法对 Mozilla 有利，我是火狐的粉丝，一直在使用它，我还没有赶上 Chrome 的潮流。

凯文:好的，所以你没有使用 Chrome。虽然我注意到你在机器人上。

路易斯:我用的是安卓手机。

凯文:是的。我们的常规面板上实际上没有任何人使用 Android 手机，所以这是我想让你上的原因之一，另一个是你写的这个帖子；我认为，新闻发布的那天，网上肯定有几十个，如果不是几百个帖子，在某种程度上大喊犯规，说“谷歌已经疯了，他们疯了，他们正在从他们的浏览器中删除对最流行的视频格式的支持，这是怎么回事？”你的帖子是我看到的少数几个积极的帖子之一，我想我会说是一个浏览器供应商中立的出版物，所以你知道我们看到了 Opera 和 Firefox 开发人员以及 Chrome 开发人员支持这一举动的可预测帖子，但作为一个独立的渠道，你的帖子似乎认为这是一个好的举动。让我们从这里开始，你为什么高兴看到这种情况发生？

**Louis:** 所以，我在 SitePoint 博客上报道这个问题已经有一段时间了，所以过去有几篇帖子，曾经只是介绍了不同编解码器的想法，当时它不是一个有争议的问题，然后当谷歌宣布开源 WebM 时，我写了另一篇文章，包括 VP8 编解码器。所以，我对此的看法是，除非你有一个人人都能实现的标准，否则你不会有一个标准。现在 Mozilla 和 Opera 可以讨论这个问题，但是他们似乎不认为他们能够在浏览器中实现 H.264。

凯文:你说的有能力是指他们不能——这与他们的许可不相容？你是这个意思吗，还是它太贵了？

**路易:**我觉得很可能是两者的结合。Mozilla 已经站出来非常强烈地说，他们不会这样做，也不会发生；我不太了解 Opera 在做什么。还有一个明确的问题，例如，Mozilla 是许多 Linux 发行版的默认浏览器，如果他们在浏览器中包含解码器，这将与许可不兼容。我不确定下载到你的 Windows 机器上的 Mozilla 发行版是否一定如此。是的，所以我仍然认为，我们需要一个视频编解码器，任何浏览器制造商都可以免费使用并分发。

凯文:我想这是为什么呢？就像如果我们在谈论包容性和开源操作系统发行版的许可，这里有一个非常明确的论点。如果我们认为这是一个既定的事实，这是一件好事，我们希望获得开源许可的操作系统能够开箱即用，假设这是我们肯定希望实现的事情，那么你肯定需要一个视频编解码器，至少有一个合法的开源解码器软件。

路易∶耶。

Kevin: 是的，它与开源许可证兼容，基本上你不需要付钱给任何人来使用这些代码，你可以把它取出来，放入你的浏览器中，只要你给出正确的说明，你就能做到，当然，取决于我们谈论的是哪种许可证，还有很多微妙之处， 但这实际上是 H.264 作为一种格式，如果你想让你的浏览器播放它，没有开源代码可以做到这一点，当然也没有合法发行的； 如果你想回放，你需要去许可机构注册一个许可，这是否是一个免费的许可取决于你如何使用它，这些微妙之处，这些许可限制都会干扰你将它包含在像 Ubuntu 这样的操作系统中的能力。所以你使用桌面 Linux 已经有一段时间了？

是的，我使用 Ubuntu 大概有三四年了，这是我主要的桌面操作系统。我仍然在工作中使用 Mac，但我更喜欢 Ubuntu，这也是我个人使用的。

Kevin: 所以撇开开源操作系统不谈，有什么好处呢？我们已经在播客上讨论了开放的不同定义，我们刚刚讨论了一个你可以获得免费软件来播放它的定义，不管你是谁或者你想用它做什么。还有开放的定义，包括所有的 W3C 技术，HTML，CSS，这些都是在一个标准机构中开发的，有一群感兴趣的团体签署合作，没有人对这个规范有独占的控制权，任何人都可以引入一个变化，然后必须投票表决，当它到达规范的最后一步时，公众可以权衡并提出异议；这是一个缓慢的过程，这是一个深思熟虑的过程，有些人可能会说是它成就了网络。像 WebM 这样的格式不具备标准化、开放标准化的好处，这让你担心吗？

路易:老实说，没有。我认为我们在网络上有一堆技术，可能会遵循或不遵循这个过程。在这种情况下，在我看来最重要的事情是有一些免费的东西，可以很容易地整合到任何浏览器或任何类型的——不仅仅是浏览器，而且从编码方面来说也适用于网站，如果我正在做一个类似 YouTube 的项目，如果是出于商业目的，我希望能够对视频进行编码；目前，如果我想使用 H.264，确实需要向 MPEG-LA 支付许可费。所以之前的争论是，好吧，我们已经有了 Ogg Theora，它是预先存在的开源视频编解码器—

凯文:是的，在 WebM 被谷歌收购并开放之前，Firefox 和 Opera 最初就决定押注于此。

路易∶耶。

凯文:人们说这很糟糕。

路易斯:人们说它很烂，基本上就是这样。

Kevin: 我们在 Ogg Theora 的课程中尝试了视频编码的实验，我可以证实:这很糟糕。(笑)

Louis: 没错，只是没有编码所需的吞吐量。你只是得到了比 H.264 大得多的相同质量的文件。另一方面，使用 VP8 编解码器的 WebM 是可比的，你知道，有人对微小的细节吹毛求疵，但它在性能上与 H.264 相当，这是每个人都很快接受的东西，所以 Chrome 最初当然支持它，但 Firefox 和 Opera 在开源后的几天内就增加了对他们的夜生活或测试版的支持。是的，我认为当你阅读谴责谷歌从 Chrome 中放弃 H.264 支持的博客帖子时，这很有意思。许多人似乎在说，这在某种程度上改变了格局，或者增加了碎片化，我只是——我不明白这是怎么回事，因为现有浏览器市场的相当大一部分已经获得了与 Chrome 相同的支持水平。

凯文:对。谷歌的举动可能与其他人的举动方向相反，但这并没有让他们变得更糟，就视频播放而言，他们还不是最差的浏览器；此举的结果是，它们不比 Firefox 差，不比 Opera 差，当然也不比 Internet Explorer 差。这一举动并不是盲目的“我们正在移除它”；他们正在移除它，用他们希望看到其他浏览器厂商取代它的东西来取代它。

**Louis:** 是的，正如我所说的，其他浏览器已经达到了这种支持水平，所以你已经得到了浏览器市场的这一部分，因为 Mozilla 和 Opera 都声明“我们不会从这里向任何方向前进”，要么是因为它与我们的许可证不兼容，要么是因为我们真的无法负担作为一个组织支付的许可费， 接下来将要发生的是，要么我们集体转向免版税的东西，要么我们开始在所有东西上使用 H.264，然后 Opera 和 Firefox 得到 Flash，你知道，就我个人而言，我不认为这是一个合适的长期解决方案。

凯文:嗯，我认为长远的愿景是 Opera 和 Firefox 将被迫或以某种方式被迫实现 H.264 视频，即使它是一个附加组件，你必须在事后下载，添加对该格式的“原生”支持；H.264 永远不会成为像 W3C 这样的标准组织认可的格式，因为它必须经过许可，但是如果它成为事实上的标准，他们必须找到一种方法来支持它，我想这是希望所在。

路易斯:希望，是的，或者恐惧，这取决于你站在通道的哪一边。是的，我意识到这是一种理想的立场，我们将尝试并希望我们将有一个完全免费的解决方案。

如果没有像谷歌这样的大公司说好吧，拿走我们的钱，这一切会发生吗？有人必须为网络支付费用以获得一个开放的格式，这个人就是我们，让我们开一张大额支票，然后我们都可以继续前进。如果谷歌没有这么做，你认为其他人最终会这么做吗？还是我们会被卡住？

路易斯:嗯，我认为在这个特殊的案例中，因为软件专利带来的困境，我可以，你知道，像 Ogg Theora 这样的东西，一群开源黑客会提出一种格式，一是如果他们试图避开专利，它不太可能是好的，二是他们可能不会做好避开专利的工作， 而谷歌，当他们做这样的事情时，他们有一个律师团队，他们可以仔细检查代码，并说“我们确信我们可以”

凯文:十指交叉。

路易:十指交叉。“如果 Mpeg LA 决定就这个专利问题对我们穷追不舍，我们至少还有一个案子。”因此，因为软件专利是一个非常复杂的问题，而且很难保护自己免受其害，所以我认为像谷歌这样的公司才能想出最终可能成为网络解决方案的东西。

凯文:这种争论大多是假设和最坏的情况。我回想起围绕 GIF 图像格式的问题；我记得在 1997 年的某一天，有人告诉我“嘿，这家公司声称拥有 GIF 压缩技术的所有权，他们试图让每个人都为他们在网上使用的 GIF 文件付费”，有一两天，人们变得歇斯底里，人们说，“哦，我们必须用 JPEGs 来取代我所有的 GIF，因为压缩，它们会占用太多的空间，看起来很粗糙。”我试图回忆是否有人真的取消了对 GIF 的支持，我不认为他们这样做了；有传言说他们在追逐浏览器供应商，也就是当时的微软和网景，我甚至不确定这是否真的会上法庭。但在我看来，开发者社区的反应似乎特别迅速和有效，而当发现这种格式肯定受到了污染，人们试图从中赚钱时，开发者社区开始转向 PNG。

路易:对。

凯文:浏览器落后了几年，直到我们看到了一个合理的 PNG 支持，但它来得非常快，肯定足够快，没有人需要为 GIF 许可支付任何费用，现在我们在这之后，甚至没有人记得 GIF 有问题，我想在这一点上，我试图记住相关的日期，但如果它还没有过期，我们很快就会过期，如果它还没有过期，它将很快成为一个问题在我看来，视频格式也会发生同样的情况，无论是 H.264，如果标准机构——或者不是标准机构——如果拥有它的许可公司决定突然反悔并开始向人们收费，或者如果 WebM 突然有人以一种没有预料到的方式声称拥有该技术的所有权，如果这些事情发生了，这不会是 Web 的终结，我们只会转向其他东西，我认为开发人员社区会聚集起来，建立新的东西或回到安全的东西。

**Louis:** 是的，我认为这是公平的，但我认为巩固 WebM 的这一步，据我们所知是一个免版税的视频编解码器，是保护浏览器免受类似事情发生的一步，因为你已经有了这个东西，人们会开始入侵它，解码器和编码器会在浏览器中提出不同的实现方式，所以如果 H.264 出现问题，将会有对替代标准的广泛支持。

凯文:它有着巨大的公众形象，你知道，我们不可能在五年内让每个网站都迁移到这种视频格式，然后一些公司过来说，咳咳，对不起，对不起，你们都欠我钱；如果这种情况发生，他们会说“五年前当我们把它作为标准时，你在哪里？”好吧，那么你对硬件解码有什么想法？

路易斯:嗯，好的。(笑声)

凯文:好吧，听着，我们口袋里都有智能手机；我的手机上恰好有苹果的标志，因此只能播放硬件的 H.264 视频。

路易斯:我相信我相当确定我的只能在硬件上播放 H.264 视频，我不认为 WebM 支持是在 Android 2.3 之前添加的，我还没有升级。

凯文:即使这样，它也会是软件，所以，如果你长时间观看 YouTube，它会消耗你的电池。所以我想，当我们拥有这些将伴随 H.264 多年的 I-设备时，我们将如何前进；有人认为，即使在谷歌最好的情况下，你也需要几年时间才能说服苹果在他们的设备中内置一个 WebM 硬件解码器。

路易:绝对是，我想那可能是真的。正如我所说的，我认为对于谷歌和像我一样的开发人员来说，我认为我们正在玩要么全有要么全无的游戏，这是我们需要的，我认为这是值得努力的，至少要尝试一下，谷歌已经尝试过了。 我认为，在谷歌、Mozilla 和 Opera 的合力下，甚至在较小程度上还有微软，因为他们已经在 IE9 中添加了一个插件，增加了 WebM 视频支持，这是一个相当简单的插件安装，我认为这只是时间问题，你必须等等看，但我不认为竞争环境与我们几周前宣布这一点之前相比有多大变化。 我认为我们仍然在同一个地方，是的，如果你想要 HTML5 视频，你必须进行双重编码，这种情况可能会持续一段时间。

凯文:是什么促使一个懒惰的开发者编写他们的第一个 WebM 视频呢？因为我知道我没有——我认为大多数人编码 MPEG2 只是因为这是他们知道的，而且很方便，直到这些 i-devices 出现，迫使他们转向 H.264 他们要么坚持他们所知道的，要么坚持他们选择的视频制作工具中自动选择的。我只是想了解一下这种变化有多快。五年、十年后，我们还会坐在这里说“你知道每个人都使用 H.264，但每个人都知道 WebM 是我们应该前进的方向”吗？

路易斯:谁知道？我认为这是那种坐以待毙的事情，我们会看到会发生什么，我的意思是显然谷歌和 YouTube 已经开始在 WebM 中编码，所以你已经看到了大量的内容。

凯文:是的，YouTube 可能是谷歌的王牌，因为如果他们将 YouTube 全部转换为 WebM，这是很好的第一步，这是一大块可以在 WebM 中获得的网络视频，也许他们可以推出手机、平板电脑、移动设备等类似的东西，这些东西可以原生播放 WebM，并使用 Flash 或 H.264 的一些软件解码。将 YouTube 放到线上，也许这就足够了，这样他们就可以—

Louis: 我认为这是完全可能的，我的意思是你已经看到了——所以，正如我所说的，最新版本的 Android 2.3 确实在浏览器中包含了对 WebM 视频的支持。有很多硬件制造商，我在 WebM 博客上看到一个视频，是一家中国制造商演示一款平板电脑，这是一款在芯片组中内置了硬件解码的 Android 平板电脑。

**Kevin:** 对于 WebM？

**路易:**为 WebM。

凯文:哇。

路易:那就是演示。所以我认为，对于许多硬件制造商来说，人们似乎认为这不会很难，随着技术产品的生命周期，人们可以将这种支持添加到硬件中，你可能会比你想象的更快地看到更多的硬件设备提供这种支持。

Kevin: 我怀疑许多对此表示强烈反对的人是 Mac 上的开发人员，他们使用 Chrome 作为他们的首选浏览器，因为，你可能意识到或没有意识到，在 Mac 上播放 Flash 电影，在 Mac 上播放视频，传统上是次优的，你知道，有原生视频支持，你知道很多人， 特别是 Windows 用户习惯了 Windows 上相对快速和可靠的 Flash 播放器，他们可能会说有什么大不了的，我不在乎我的视频是本机播放还是通过 Flash 电影播放，只要它能播放，而在 Mac 上，至少直到最近，通过 Flash 播放器播放的视频可能会暂停，可能会减慢机器上其他事情的速度，可能会消耗电池，导致 CPU 风扇疯狂旋转。

Louis: 虽然在 Mac 上很糟糕，但我可以向你保证，在 Linux 上更糟糕。
