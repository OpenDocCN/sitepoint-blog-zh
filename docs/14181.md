# JBuilder 即将过时的迹象

> 原文:[https://www.sitepoint.com/signs-jbuilder-is-on-the-way-out/](https://www.sitepoint.com/signs-jbuilder-is-on-the-way-out/)

随着 [EclipseCon 2005](http://www.eclipsecon.org/) 最近在加州的 Burlingame 完成，Eclipse ，Java IDEs 中正在出现的事实上的标准，现在无处不在。

JavaLobby 刚刚将会议报告发送到它的邮件列表中。作为一名 [JBuilder](http://www.borland.com/jbuilder/) 用户，一个大新闻是 Borland 正在打破其在 Java IDE 前端长期保持的独立性，并将 JBuilder 的一些最好的特性移植到 Eclipse。

但是这对 JBuilder 意味着什么呢？

> 谈到 JBuilder 的未来，我无法得到一个明确的答案。当我问 Borland 是否会继续支持 JBuilder 时，我被告知“在可预见的未来”。当我问那是多长时间时，我被告知“大约一年”。当我问这是否意味着 Borland 实际上计划最终淘汰 JBuilder 时，我得到的回答是“我不能对此发表评论，但客户需求将发挥作用”。所以我的印象是 Borland 肯定在考虑淘汰 JBuilder。
