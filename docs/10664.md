# H.264 免版税:会成为 HTML5 视频标准吗？

> 原文:[https://www . site point . com/h264-视频-免版税-html 5-标准/](https://www.sitepoint.com/h264-video-royalty-free-html5-standard/)

MPEG LA，H.264/AVC 编解码器的授权组织，已经宣布使用该格式编码的视频文件的网络流是免费的。该集团此前表示，它将在 2016 年前保持免版税，但这一期限已被取消。MPEG LA 没有给出改变的具体原因，但免费编解码器的到来，如 [WebM](https://www.sitepoint.com/google-announces-new-webm-video-standard-open-sources-vp8/) 会影响他们的决定。

大问题:*这对 HTML5 视频会有什么影响？*

尽管 HTML5 规范包含了一个用于原生浏览器回放的<video>标签，但它并没有指定编解码器…</video>

*   Google、Mozilla 和 Opera 支持 Ogg Theora、Ogg Vorbis，并在新版本中引入了 WebM。他们拒绝支持 H.264，因为该专利可能会向供应商或用户收取许可费。
*   苹果的 Safari 浏览器使用 Quicktime 支持的任何编解码器，但 H.264 是最常见的格式。他们的移动设备包括 H.264 硬件解码，以确保视频播放保持流畅。苹果是唯一一家采用 WebM 的供应商——它应该可以在安装了编解码器的 Mac 上运行，但不能在他们的移动设备上运行。
*   微软 IE9 支持 H.264 和安装在用户 PC 上的任何编解码器。这包括 WebM，但它不会随 Windows 或浏览器一起发布。

编解码器混乱影响了想要采用原生视频的网络开发者。为了确保跨浏览器的视频播放，您必须以三种格式对剪辑进行编码:Ogg Theora/Vorbis(或最终的 WebM)、MP4 H.264 和用于旧浏览器的 Flash 视频。

到目前为止，我预计 WebM 将成为“标准的”HTML5 视频编解码器。苹果将成为唯一不支持该格式的厂商，但 iPhone 和 iPad 用户没有 Flash——他们已经习惯了无视频网络。

但是 MPEG LA 的声明会颠覆这个行业吗？Mozilla 和 Opera 是否免费添加 H.264 支持？谷歌可能会支持 WebM，但没有什么理由阻止他们将 H.264 加入 Chrome。H.264 最终会成为最可靠的 HTML5 视频格式吗？

也许我有点愤世嫉俗，但我对此表示怀疑。H.264 只在交付时免费——当你观看视频时。最初的编码、服务器技术和浏览器解码软件招致向 MPEG LA 支付版税。即使供应商为解码器付费，H.264 也不能直接在开源产品中实现，因为源代码必须对任何人免费开放。Mozilla Firefox、Google Chromium 和 Linux 发行版会出现许可问题。

相比之下，制造商可以将 WebM 添加到视频编码软件、服务器解决方案、操作系统和浏览器中，而没有限制或版税支付。假设在 Windows 上安装编解码器很容易，到明年这个时候，很大一部分网络用户应该能够观看 WebM 视频。

Web 开发人员不想浪费时间、金钱和带宽来支持多种视频格式。虽然我欢迎 H.264 的发布，但是对于开放网络来说，这个许可可能还是太严格了。WebM 有潜力，但在未来几个月内，对它的支持仍将是不完整的。如果你只想对视频进行一次编码，目前只有一种技术可以在大多数浏览器中提供可靠的视频播放——Flash。
