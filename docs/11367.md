# SVG 终于要来 ie 浏览器了吗？

> 原文:[https://www.sitepoint.com/internet-explorer-svg/](https://www.sitepoint.com/internet-explorer-svg/)

我对 IE9 的最大愿望之一会实现吗？据 IEBlog 报道，[微软已经加入 W3C SVG 工作组](http://blogs.msdn.com/ie/archive/2010/01/05/microsoft-joins-w3c-svg-working-group.aspx)！这是工业工程团队的高级项目经理帕特里克·登勒的一句话:

> 作为微软对互操作性和标准支持的持续承诺的一部分，昨天我们提交了加入[万维网联盟(W3C)](https://www.w3.org/) 的[可缩放矢量图形(SVG)工作组](https://www.w3.org/Graphics/SVG/)的请求。我们很高兴能够参与确保 SVG 规范的未来版本能够满足开发人员和最终用户的需求。

Web 开发人员多年来一直强烈哀叹 IE 缺乏 SVG 功能——它是唯一不支持图像标准的主流浏览器。微软可能没有宣布在 IE9 中实现完整的 SVG，但是迹象看起来不错。

我一直怀疑 Silverlight 是微软避开 SVG 的主要原因。该公司希望增加 Silverlight 的追随者，并可能考虑将图像格式作为竞争对手。我不相信是这样，即使是这样，良好的 SVG 支持也有更大的潜力来减少对 Adobe Flash 的依赖。这最终可能对 Silverlight 的未来有利。

然而，这可能仅仅是因为微软之前对 IE 团队的投资不足，使得 SVG 的优先级较低。毕竟，首先修复 IE 的渲染错误更重要。

不管是什么原因，让我们希望微软在 Internet Explorer 9.0 或更快的版本中实现一个完整的基于标准的 SVG 渲染引擎。明天到来 SVGs 对网络重要的 9 个原因。
