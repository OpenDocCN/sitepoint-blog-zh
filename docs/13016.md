# 20 种让客户回头客更多的方法

> 原文：<https://www.sitepoint.com/ways-keep-clients-coming-back/>

无论你是自由职业者还是 500 人的开发团队，留住客户都很重要。对于每一个新客户，你必须解释你提供的服务，花时间讨论他们的项目，启发他们项目的可能性——本质上，投入资源赢得客户。web 服务行业的客户流动率尤其高，在这个行业，价格竞争几乎是不可能的。

现在，如果你所有的客户都是回头客，你可以花更多的时间管理项目，这不是很好吗？在这篇文章中，我将找出 20 种方法来提高你的客户保持率，并让客户再次光临。

##### 留住客户的重要性

布兰登·辛克莱在 SitePoint 的[网页设计商业工具包](https://www.sitepoint.com/kits/freelance2/) ( *Psst:直到四月底，你可以用一个的价格买两个[！艾德。](https://www.sitepoint.com/kits/sale/)*)。以下是我对此的看法:

最好的客户是你现有的客户。当你在经营一家企业时，即使是作为一名自由职业者，确保你有源源不断的工作的最好方法之一就是留住现有客户，而不是寻找新客户。你可能会对现有客户能给你的工作量感到惊喜，即使他们没有任何未来的工作给你，如果他们高兴，他们仍然可能会把你推荐给朋友或商业伙伴。不幸的是，你不能把这样的建议视为理所当然——尤其是如果你是自由职业者。

我曾经为一家构建简单 web 应用程序的 web 开发商店做过一些咨询。他们的工作并不十分创新，但它完成了工作，客户对产出也很满意。业务蒸蒸日上，工作源源不断，但有一个问题——公司一直在寻找新客户。当我联系一些过去的客户时，大多数人对这项工作很满意，后来又有了其他项目，但他们把这些项目交给了公司的竞争对手。为什么？一条评论总结道:“我们想试试其他人。”

客户流失可能是一个严重的问题，但有一个简单的解决方案:投资保持你现有的客户！如果你的客户对你过去为他们所做的工作感到满意，那么他们将来会考虑来找你也是合乎逻辑的。但是对他们来说仅仅开心是不够的——他们必须真正被打动。要做到这一点，你必须小心地控制你的项目——这是一个相当科学的挑战。

##### 保持回头客的 20 条建议

让我们来看看你可以用来提高客户保留率的策略。我将这些技巧分成三个关键领域——与客户打交道的技巧、项目管理的技巧以及与业务相关的技巧。

***客户端***

当你在管理你的业务和处理所有进来的工作时，很容易忽略真正的客户。这里有一些管理客户及其满意度的技巧。

**1。选择你的最佳赌注。**

这可能是你全年做出的最佳商业决策之一。随着时间的推移，你会发现 80%的工作来自 20%的客户。所以，从更广泛的角度来审视你的业务，找出哪些客户构成了那 20%，并专注于他们。当然，不要忽视你的其他客户，但要集中精力监控这 20%的项目，并确保一切顺利进行。

**2。放弃行话。**

当你通过电子邮件、即时消息、电话或面对面与客户交流时，不要使用行话。除非客户特别要求技术细节，否则不要谈论技术细节。使用专业术语会疏远客户，造成误解，导致沟通中断。我曾经和一个客户交谈过，他认为 AJAX 指的是 Web 2.0 风格的网站上常用的动画脉冲图标(有人知道吗？).放弃行话让你和你的客户站在同一立场上，这样你就可以清晰有效地沟通。

**3。表明你已经做了功课。**

当你为你的客户开发一个 web 应用程序时，这个应用程序很可能与他们的专业领域有关，而不是你的。他们可能是这方面的专家，你可以从他们那里了解背景，但是如果你做足了功课，你会给他们留下深刻的印象。在你们第一次见面之前，先研究一下这个地区；维基百科总是一个很好的起点，而在线文章可以让你很好地了解这个行业的现状。采访现有客户是一个很好的方向来源。通过进行这些基本的调查，你的客户会感激你带着这个领域的经验来参加谈判，并且更有可能信任你在这个领域的进一步工作。

**4。让他们参与进来。**

即使你已经做了功课，你也需要把你的客户作为你的参考点。你不是这个领域的专家；他们是。当你计划你的应用时，最好的方法是通过和你的客户交谈来了解这个领域所涉及的内容。任何非技术性的项目规划阶段都应该让你的客户大量参与；简单地过一遍草稿，通过电子邮件提出问题，定期会面以确保你在正轨上。

**5。了解你的客户。**

了解你的客户是值得的。我说的不是他们的业务或行业(尽管这也有帮助)，而是客户自己。了解一下他们——他们可能有在线传记，或者你可以告诉他们喜欢什么。迎合他们的自身利益可以走很长的路，通过了解你的客户，你可以确定如何最好地吸引他们。例如，如果他们喜欢赛马，也许你可以安排他们在当地的赛马场见面。你的体贴会留下持久的印象，他们会很高兴的。

**6。获取反馈。**

你的客户会不断地对你的进步、你的总体表现、你的服务质量等等形成看法。那么为什么不利用这些意见呢？下次你会见客户时，问他们对你目前为止的项目工作有何看法。这是从他们那里获得一些反馈的合理的、非威胁性的方式，它可以帮助你在重要的领域磨练。

7 .**。让它变得有趣。**

在一天结束时，从你的客户那里得到结果的最好方法是确保他们玩得开心。让会议变得有趣和愉快。幽默总是有帮助的。让你的客户放松会帮助你从他们那里获得更多有见地的评论和有用的反馈。一旦你的客户真正喜欢参加你的会议并讨论项目，你就可以真正开始行动了。

***项目***

世界上有许多项目管理方法。每一种都适合不同的情况、管理风格和人。我说，如果你想要结果，把它们都忘掉，找出最适合你的——试试这些想法。

**8。过度记录项目**

我怎么强调都不为过。无论你在项目中做什么，都要做好记录。实际上，不存在文档过多这种事情——你永远不会有足够的有用文档。不要为了一页又一页地写下去，但是如果项目中出现了一些事情——例如，做出了一个重大的设计决策——将它记录在文档中，以及某种简明的项目日志中。最后，让你的客户和员工访问这个文档，并鼓励他们使用它。您的客户会很高兴能够获得关于他们项目的知识，当每个相关人员都知道项目的情况时，您可以避免一些常见的问题。

**9。坚持约束。**

项目安排有一个黄金法则:你只能有“好”、“快”和“便宜”中的两个。我认为这是任何项目的三个约束，成功安排(和满意的客户)的关键是找出客户的约束是什么。例如，一家大型跨国公司可能希望一个项目做得又好又快，预算控制在可控范围内，而个人可能只是希望项目在预算内快速交付。

在项目结束时，你可能会面临这些限制中的一个，当你的客户在这方面有足够的灵活性时，和他们一起解决问题就容易多了。大多数人可以处理项目完成的轻微延迟，而有些人宁愿花钱雇佣一些自由职业者，按时完成工作。无论哪种方式，你都要避免以伤害客户的方式将他们推向极限。

10。隐蔽的计划。

项目规划是棘手的。如果多人将参与最终产品，避免让他们互相讨论项目计划。每个人的意见都很重要，但是当你的开发人员开始讨论一个项目的时候，他们的意见会开始倾向于集体意见。不同的开发者有不同的想法；有些可能是好的，有些可能是坏的，但都算。让你的团队成员分别制定计划，然后单独接近他们，讨论他们的想法，这样你就可以挑选出最好的想法。

**11。欠排，过排。**

给客户多一点点。除了在截止日期前完成任务，给软件增加额外的功能，或者只是提供额外的有用文档，没有什么更好的方法来打动客户了。超越，哪怕只是一点点，将向你的客户表明，你不仅掌握了工作，有能力管理项目，而且你做得很好，甚至可以在预算内提供一些额外的服务。

**12。一天见面两次。**

无论你想怎样安排你的一天，都要留出开会的时间。无论你是两个人的商店，还是有一整层楼的开发人员，定期开会有助于信息的流动，并确保每个相关人员都知道项目的进展情况。每天会面两次——清晨和快下班时都是不错的选择——让每个人花一分钟时间讨论他们一天都做了什么，他们一路上做的任何决定，以及他们在项目中的位置。会议不需要很长，甚至不需要很正式；他们只需要把每个人聚集在一起，让每个人都了解情况。

13。开发者才是明星。

如果你是一个项目经理，很可能你不会在你销售的实际产品或服务上做很多工作——你的开发人员会做。因此，当与客户讨论初始需求时，为什么你应该是帮助客户做决定的人呢？对于任何涉及设计决策的会议——例如应用程序前端的外观，如何管理管理，或者有哪些菜单项可用——带上一名开发人员，即使他们只停留五到十分钟。

凭借他们当前的实际经验，您的开发人员知道什么是可能的，什么是不可能的，什么是可行的。他们可以与客户分享这些知识，真正让项目的规划有一个良好的开端。很有可能你的开发者的建议对他们来说是如此显而易见，以至于他们无论如何都要实现；这样，你的计划会让你对最终产品有一个更清晰的了解。你的客户也会喜欢在项目规划方面得到帮助。

**14。和所有人说话。**

通常情况下，你的客户只是一个联系人——一个更大组织的代表，以及该组织内更大的用户群。如果你知道一个特定的部门或一群人将会使用你的最终产品，和每个相关的人谈谈他们想从最终产品中得到什么。最好与每个人见面，但对于较大的群体，你可能需要发送一份简短的问卷。在从大多数最终用户那里得到反馈之前，不要写任何一行代码。对用户群进行详细、全面的调查有助于你识别用户真正想要什么，哪些功能是重要的，哪些可用性方面是重要的，等等。

***商业***

当然，在一天结束的时候，你是在向企业出售产品或服务。不管是某个人的生意，还是一个有一千名员工的公司，这个生意的目标应该是你正在开发的产品或服务中最重要的。如果你提供的东西促进了他们的商业目标，客户将更有可能回来找你做进一步的工作，因为他们会知道你物有所值。

15。质量第一，最佳实践第二

与普遍的看法相反，客户会为高质量的网络服务付费，但是首先你必须确保他们能够重视这种质量，最佳实践不算数。如果你提供的东西和离岸外包公司的产品功能完全一样，客户为什么要雇佣你呢？你的客户不会意识到你在代码中有一个严格的变量命名约定——这对他们来说毫无意义。然而，他们会欣赏你的界面易于使用，或者你的页面加载速度很快。他们通常不会认可高质量的工作——确保你在项目汇报中指出最终产品的有用特征。区分质量和最佳实践是很重要的——有许多最佳实践是有帮助的，但是对于小型的、一次性的项目来说，有些是不必要的，是浪费时间，尤其是如果他们是开发人员的新手。以一种他们能欣赏的方式给你的客户质量，但是要清楚地表明你正在这样做。

16。记住等式两边谁是老板

每个项目都有两个方面:开发和商业。双方都有专家:一般来说，你是开发方面的专家，客户是业务方面的专家(实际上是他们的业务)。当你计划你的项目时，请记住这一点。让客户确定业务需要什么，而开发人员研究如何实现它。项目经理可以是中间人，决定什么会成为最终产品，什么不会。

这里的关键是避免模糊界限。例如，您的开发人员可能建议您将项目与现有系统集成，但是除非有这样做的商业案例，否则这可能是浪费时间，并产生自己的问题。另一方面，也许客户想为了 AJAX 而加入 AJAX——除非你认为这是值得的，否则就避开他们。让您的客户和项目经理决定业务需要什么，让开发人员决定如何实现它，然后就不管了。

**17。严格控制成本**

作为项目经理，您需要保持对项目的严密控制。如果要按时完成，许多(如果不是大多数)软件项目需要的时间比预期的长，或者需要增加预算。避免此类问题的关键是详细计划项目，并监控项目，以确保尽可能严格地遵循计划。

例如，定期获得状态报告。当您计划项目和确定子任务时，将这些任务分解，这样您的开发人员就可以在每天结束时向您提供某种状态更新。简短的一行就够了——一封“仅供参考，我已经完成了任务 39”的邮件就足够了。当事情没有按计划进行时，确保你知道这件事，并做些什么。如果你遇到延迟，很快发现你没有赶上进度，你也许可以在这里或那里删减一些，或者外包某些任务。无论如何，如果你发现了延迟，不要等到为时已晚再去处理它。

延误和成本激增会严重损害客户对你业务的信心。尽可能地控制和控制项目。这将提升你的客户对你管理未来项目能力的认识。

18。继续说。

你在这里从事一个行业。它不是一个静态的行业。这是一个动态的、不断变化的行业，你的客户往往对此知之甚少。所以继续说。每当你与客户讨论项目时，让他们了解进展情况；例如，在规划设计时，提及当前的界面设计原则是什么。从背景信息到成功案例，让你的客户了解信息将有助于他们在计划时与你合作，并强化你是你所在领域的真正专家的印象。

**19。拿到维修合同。**

当然，在担心客户生意的同时，你也要把自己的事情记在心里。每个应用程序都需要某种维护，无论是小的更新、功能添加还是内容更改。在你结束项目之前，提出维护合同的问题。我通常发现电子邮件效果最好。让客户知道你认为产品需要维护，你可以提供这种维护，并简要说明你的支付结构。一般“快速修复”的固定年费可以为最少的工作投入创造额外的收入流。然而，更重要的是，一旦你完成了最初的产品并负责维护，客户会更倾向于雇佣你做未来的工作。

20。继续卖。

最后，继续销售你的产品和服务。许多客户对未来的购买感兴趣，如果你遵循这些建议，他们会非常乐意向你推荐。问题是很多客户不知道他们在追求什么。例如，现在他们已经有了一个客户管理系统，也许他们需要一个项目管理系统与之配套？确保你的客户不仅知道你提供哪些服务，还知道他们会雇佣你做什么。

你可能会发现你的客户有一个问题，可以很容易地用软件解决，但他们只是没有意识到你可以帮助他们解决这个问题。如果你看到一个机会，向客户提及潜在的项目；如果你在过去已经建立了一个类似的系统，检查它的工作情况，并给他们一个测试的机会。如果他们看到一些真正有用的东西，你就可以再进行一次销售。

##### 让他们回来

让你的客户回头客真的是你能为你的生意做的最好的事情。回头客很好对付，可以说他们“深谙此道”。

当你管理你的客户时，记住我们在这里讨论的建议，你会留下一个持久的印象。当你和一个客户合作过一次，你就会明白如何最好地管理那个客户和他们的项目，你就能交付成果——和更高的利润率。让您的客户保持正确，你就成功了！
