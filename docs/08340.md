## 剧集摘要

Louis 和 Tim Boronczyk 谈论了 phpmaster.com 的启动，PHP5.4 中的新特性，包括 traits 和会话上传进度指示器，PHP 的优点和缺点，包括最近关于这个主题的详尽的博客文章，以及 PHP 社区的 DIY 方法。

路易斯:大家好，欢迎来到 SitePoint 播客的另一集。这周我们有一个访谈节目，和我一起上节目的是蒂莫西·博伦奇克，或者蒂姆，我可以叫你蒂姆吗？

路易斯:好吧，我们和蒂姆一起去吧。Tim Boronczyk 是 phpmaster.com 的执行主编，这是 SitePoint 的最新项目之一，所以这是一个关于 PHP 奇妙世界的博客。嗨，蒂姆，欢迎来到这个节目。

路易:有你在真好。你从一开始就在管理 PHP Master 吗，或者说从一开始就在管理 PHP Master 吗？

蒂姆:对，从一开始，大概八个月，六个月，七个月，八个月左右。

路易:好的。那么，你是否想对那些不熟悉 PHP Master 的听众，或者那些没有机会看到这个网站的人，说一下你倾向于发表什么样的东西，事情进展如何，有什么想法？

蒂姆:嗯，这是所有事情的一点点，我们每周发布三次，分别在周一、周三和周五，所以如果你还没有启动你的 RSS 阅读器，那就启动吧。我们试图让每个人都有一点点东西。我们有一些初级水平的文章来帮助新程序员提高他们的技能并成为一名 PHP 高手，我们有一些中级的文章给那些已经在 PHP 学徒期工作过并希望提高他们技能的人，我们还有一些真正深入的文章给那些已经认为自己是 PHP 高手的人。

**路易:**对。所以我想，既然我有机会邀请你参加这个节目，我们可以就 PHP 的总体情况谈一点，它的现状和发展方向，首先，PHP 的最新版本是 5.4 版，这是一个相对较新的版本，对吗？

蒂姆:是的，它可能已经上市几个月了。

Louis: 我对 5.4 版本的 PHP 只是略知一二，我也接触过一点 PHP，但主要是 Ruby 和 Rails，所以你能谈谈 PHP 5.4 中真正酷的特性吗？

5.4 版本中有很多新特性让你感到兴奋，对我来说，可能有三四个我个人认为非常好的特性，第一个是会话上传进度指示器；当你上传文件的时候，PHP 开发人员已经把信息嵌入到会话中，所以你不需要做任何奇怪的，黑客的，flash 的，奇怪的事情——只是肮脏的代码；您可以提交表单，将您的文件发送到服务器，并在后台派生出一个 Ajax 请求，该请求将使用会话变量来监控进度，即已经为您传输的文件量。

**Louis:** 好的，这样就可以更容易地向用户显示一个有效的进度条，显示他们上传文件的进度？

是的，没错，这是一个内置在 PHP 中的解决方案，你不必使用第三方库或一些奇怪的变通方法。

路易:对。

**Tim:** 还有一个是特质；如果你是一个优秀的 OOP 程序员，并且 traits 很有趣，你可能对它们很熟悉，我相信 Ruby 中也有，它们是你会在一个类中使用的一组功能，但它们不在一个类中，它们只是自己坐在一旁，你可以将这组功能导入到类中，这样就使得代码重用更容易，你不会有奇怪的复杂的类层次结构试图不重复你的代码。

Louis: 是的，这绝对是我在 Ruby 中喜欢的非常非常有用的东西，可以混合一些不一定适合继承层次的小功能片段；我不需要所有东西都从这个主类的后代继承来获得这种功能，我可以把它放在一个小模块中，在 Ruby 中它们被称为模块，当我需要它的时候就把它放在我需要它的地方，所以在 PHP 中看到这种功能也很棒。

蒂姆:对，没错。此外，我们现在在 5.4 中有一个内置的 web 服务器，有些人真的很兴奋，他们喜欢它，或者他们对它有点冷漠，但我认为它很酷，很高兴拥有它。

路易斯:那么这主要是为了发展吗？如果我只是在我的本地机器上玩一些 PHP 脚本，我可以下拉到命令行并启动这个小的开发服务器，它可以提供这些文件，而不必安装 Apache 和 MOD PHP 以及所有的设置工作。

是的，它让初学者很容易掌握 PHP，而不必担心配置 Apache 或 Nginex 服务器。现在，我相信，从 5.0 版本开始，你已经将 SQLite 作为 PHP 的一部分，你真的不需要 MySQL 了；如果你正在做一个简单的 web 应用程序，你现在可以把所有的东西都内置到 PHP 中，只要启动内置的服务器就可以了。虽然有些担心是因为这是对 Ruby 社区的回应，因为这个内置服务器似乎只适合本地测试，我还没有看到任何人用它做任何真正酷的事情，我相信有人会的；只是没见过而已。但批评是 PHP 和 MySQL 和 Apache 是如此多产，每个人都已经安装在他们的托管环境；为什么要有一个内置的测试服务器，它与您在部署中使用的配置不匹配。

路易:对。我的意思是，当你说这看起来至少是从 Rails 借来的时候，我认为这是一个有效的观点，因为 Rails 有一个非常相似的方式，只是启动一个快速的小服务器进行测试，但在 Rails 中这几乎是一个要求，因为要建立一个真正的测试环境有很多障碍，因为服务器设置不像安装几个应用程序那么简单；在 PHP 的例子中，如果你有 MySQL 和 Apace，它会自动设置。

蒂姆:的确如此，我认为这与语言本身背后的驱动力有关，PHP 确实是作为一种 web 语言而诞生的，所以它从一开始就紧密地集成到 Apache 中。你所有的 post 和 Git 变量都是作为语言中的功能公开的，就像我刚刚谈到的检查会话密钥来监控你的文件传输一样，这也是嵌入到语言中的，如果你有一个像 Python、Perl 和 Ruby 这样的通用编程语言，你必须有提供网络交互、接受输入数据和刷新输出的框架。

路易:对。无论如何，我的意思是，我认为这绝对是一件有趣的小事，特别是当人们可能会在 Apache 以外的环境中部署 PHP 时，你提到的 Nginx，设置起来可能有点棘手，我可以看到能够运行一个非常快速和肮脏的开发服务器是多么有价值，这样我就可以摆弄你的脚本，然后在你开发完它们后再担心部署它们。

Tim: 我个人最喜欢的是新的短数组语法。几年前，当有人第一次向内部列表提出它时，我看到它被上下左右地投票，我很高兴看到它最终成为 5.4

**路易:**对。这基本上是 PHP 一直以来都有这种特殊的声明数组的方式，如果你想给一个变量赋值一个新的数组，你可以使用带括号的关键字 array，然后在括号里输入数组的内容。

Tim: 是的，能够键入一个左方括号和一个右方括号并结束今天的工作真好。

Louis: 是的，所以它更像是数组的文字表示，类似于你在 Ruby 或 JavaScript 中所做的。

Tim: 无论如何，你都要用方括号来访问数组元素，所以没有什么奇怪的概念上的跳跃。

路易斯:对，对，没错。尤其是当我同一天在 Ruby 和 PHP 代码之间来回工作时，例如，声明数组总是让我感到困惑，因为我会突然在 Ruby 代码中键入 array 或试图在 PHP 中这样做；我想现在这在 PHP 中可以工作了，所以至少我的两个错误中的一个会被避免。

是的，你的 Ruby 代码现在可以在 PHP 中工作了(笑声)。

路易斯:嗯，这些都是很酷的新功能。那么在性能方面做了很多工作吗，我知道过去的 PHP——嗯，我的意思是 PHP 已经很快了，所以。

是的，我真的不知道，我还没有机会看任何基准测试。

路易斯:酷。是的，真的很令人兴奋，你已经开始在日常工作中使用 5.4 版本了吗？

**Tim:** 是的，当它第一次出现时我就看到了，我看到了第一段，我想好了，这是——好了，他要说什么，我开始向下滚动，一直向下滚动，向下滚动，向下滚动，向下滚动，他一直逐点滚动，他提出的一些观点就是我们刚刚谈到的数组问题，但是 PHP 5.4 现在通过使用数组文字来解决。它还提到了没有真正的 Unicode 支持，我相信在 PHP 6 中有 Unicode 的计划，但这些年来已经知道或重复了很多，函数的命名约定不一致，有些已经改变，有些将会改变，有些可能永远不会改变，但我不得不赞扬他的长度和热情。

路易:对。我的意思是，在任何语言或框架中都有很多抱怨，显然这是一个人对这种特定语言的抱怨的非常非常详尽的编目，正如你和我看到的其他对此的批评中所指出的那样，其中一些抱怨要么不再有效，要么指的是过去存在于 PHP 中但现在不再存在的问题。但是，是的，显然 PHP 有它的优势，你已经提到的优势之一就是对于初学者来说学习它是多么容易。作为学习 web 编程的第一语言，特别是对于那些从前端 web 设计开始并希望开始构建和访问数据库的人来说，只需创建一个文件并将其直接放入您的 Web 中，这一步或多或少会让您受益匪浅，因为您已经有了 Apache 和 Mod PHP。

蒂姆:是的，老实说，你只需要写一个文件，如果它有正确的扩展名，它就能工作，你知道，从初学者的角度来看；你不必建立一个神奇的框架，或者你不必建立一个神奇的服务器。

另一件事，我想问的是，如果我让你上了节目，你有没有什么很酷的脚本或库或框架，你可以分享或给听众？
