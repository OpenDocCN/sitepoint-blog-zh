# 你使用哪些 PHP 库？

> 原文:[https://www.sitepoint.com/which-php-libraries-do-you-use/](https://www.sitepoint.com/which-php-libraries-do-you-use/)

出于兴趣，想知道人们使用什么 PHP 库？我所说的*使用*是指你真正信任并愿意使用/已经在实时网站上使用的库。

让球滚动起来，一些我以前用过；

邮箱:[PHP mailer](http://phpmailer.souceforge.net)–百分百开心。也是帮助初学者理解 OOP 的基本目的的好地方

DB: [ADOdb](http://adodb.sourceforge.net/) 与 Oracle 一起工作时，API 比原生 Oracle 扩展友好得多

XML-RPC:[Incutio XML-RPC Library](http://scripts.incutio.com/xmlrpc/)—“现成的”版本有一两个在不太常见的情况下出现的错误，但这是 PHP 中最完整和易于使用的 XML-RPC 实现之一，IMO。我最喜欢的是它充分利用了 PHP 的动态特性，而不是需要大量的粘合代码来映射类型。

HTTP client:[PEAR::HTTP _ Request](http://pear.php.net/HTTP_Request)–这是我想修复的两件事情中的一件，但总的来说，这是我可以忍受的一件大事。

forms:[PEAR::HTML _ quick form](http://pear.php.net/HTML_QuickForm)用于一次性的独立脚本。 [WACT](http://wact.sf.net) 否则。

RSS 一代:曼努埃尔莱默斯的 [RSS 作家](http://www.phpclasses.org/browse/package/560.html)

缓存:[PEAR::Cache _ Lite](http://pear.php.net/Cache_Lite)——赶时间的时候。不过，通过一个分析器运行它让我考虑编写自己的、针对特定问题的程序。
