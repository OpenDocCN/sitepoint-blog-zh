# 进行跨浏览器测试的另外两种方法

> 原文:[https://www . site point . com/two-more-ways-to-do-cross-browser-testing/](https://www.sitepoint.com/two-more-ways-to-do-cross-browser-testing/)

几天前，我们在 Meermeer 上给你带来了一个[的偷偷高峰，这是 Adobe 即将推出的一个应用程序，它将使开发人员跨浏览器测试网站变得更加容易。Meermeer 将允许开发人员直接从 Dreamweaver 向一组运行不同操作系统和浏览器的虚拟机发送代码，并几乎立即收到在这些不同配置上呈现的网站截图。这比大多数开发人员使用的当前系统好得多，当前系统通常是使用多个测试床系统、本地虚拟化软件、依赖朋友和像 Browsershots.org 这样的缓慢批处理服务的组合。](https://www.sitepoint.com/adobe-meermeer-will-change-the-way-you-test-web-sites/)

Meermeer 要到明年年初才能面向消费者推出，但对开发者来说，谢天谢地，现在有几个服务可以提供类似的有用测试功能。

## [石蕊](http://litmusapp.com/)

Litmus 是一个跨浏览器和操作系统的测试工具，像 Meermeer 一样，它可以抓取网站的截图，并几乎立即返回结果。与 Meermeer 不同，Litmus 还测试 HTML 电子邮件，它有一个令人印象深刻的列表，包含 23 个受支持的网络浏览器和 16 个电子邮件客户端。Litmus 强调持续的测试和组织，并允许您跟踪所有的测试，在浏览器中标记页面通过或失败的时间，以便您可以掌握网站上需要做的工作。

Litmus 应用程序还可以发布关于您的产品的 web 浏览器或电子邮件客户端兼容性的公共报告，您可以向客户展示这些报告。Litmus 截屏有窗口和全屏两种格式(这样你就可以看到访问者在页面加载和调试时看到的内容)。

Litmus 落后于 Meermeer 的地方是缺乏洋葱皮视图(它有助于覆盖两个浏览器截图)，并且无法呈现未发布的代码来创建截图。但是，您可以将石蕊指向受密码保护的私有生产服务器。

Litmus 的免费版网站支持 IE 7 和 Firefox 2，电子邮件支持 Outlook 2003 和 Gmail。付费计划解锁了更多的浏览器和电子邮件客户端，允许更多的用户，并取消了每月测试次数的上限。

## [CrossBrowserTesting.com](http://crossbrowsertesting.com/)

截图很棒，但是我们在本周早些时候的帖子评论中经常听到的对 Meermeer 和其他基于截图的测试工具的批评之一是，你不能通过图片来测试功能。您可以使用截图来测试布局，但是您的 Javascript 或动态 CSS 元素的跨浏览器和操作系统功能如何呢？为此，你需要真正与页面互动。

这就是像 CrossBrowserTesting.com 这样的服务有帮助的地方。该网站提供对许多不同的[虚拟机](http://crossbrowsertesting.com/configurations.php)的访问，这些虚拟机跨越一系列操作系统、浏览器，并且安装了各种插件。使用一个名为 VNC 的程序，用户可以通过 Java 小程序在浏览器中加载这些虚拟系统，然后使用不同的浏览器和操作系统配置测试他们的网页。

在测试时，用户可以立即捕捉他们所看到的屏幕截图，以便他们可以记住在特定浏览器配置下可能存在的问题。

CrossBrowserTesting.com 以每分钟 0.20 美元的价格对使用他们的系统进行收费，并有批量折扣。该网站提供免费的五分钟测试时段，但付费客户优先，因此免费时段仅在系统有空闲周期时可用。
