我们的下一个故事是 Google 开源了它的 JavaScript 工具集，命名为 Closure。Google Code 博客在过去的一周发布了一篇名为[‘介绍闭包工具’](http://googlecode.blogspot.com/2009/11/introducing-closure-tools.html)的帖子，它谈到了这样一个事实，即他们为 Gmail、Google Docs 和 Google Maps 等产品编写了如此多的 JavaScript 代码，以至于他们决定将这些东西开源，并提供给其他 web 开发人员，你和我，让他们使用和他们一样的 JavaScript 工具。他们已经发布了 Closure 编译器，它将缩小你的 JavaScript 代码，使它尽可能的小，并且加载更快；他们发布了 Closure Library，这是一组可重用的软件组件，使用谷歌的代码，而不必自己为常见任务重新发明轮子；和闭包模板，我还是不太明白它是什么。就像你在 PHP 中构建一个可重用的页面模板一样，这个页面将显示你的站点上的所有文章，闭包模板是用 JavaScript 实现这一点的一种方式。它是一种模板语言，应该由客户端的 JavaScript 或服务器端的 Java 来解析。因此，使用相同的模板语言，您可以编写将内置于客户机-服务器两端的网页中的模板。我认为你必须做一些非常特别的事情来找到闭包模板的用途，但是如果你正在构建下一个 Gmail 的竞争对手，那么你已经有了——谷歌已经帮你省了一些工作。
