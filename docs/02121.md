# AtoZ CSS 截屏:CSS Quotes 属性

## 副本

写代码时单引号和双引号经常出现。

然而，当显示任何引用的文本时，如`q`元素或使用撇号时，我们应该使用智能引号。

这些通常是由浏览器添加的，但是我们可以从 CSS 中控制它们的外观和字符的选择。

在这一集里，我们将了解:

*   哑引号和智能引号的区别
*   国际上使用的不同风格的报价
*   以及未被充分利用的 CSS `quotes`属性

### 聪明还是愚蠢？

当我们在代码编辑器中创建一串文本时，我们使用“哑引号”。

这些要么是‘单引号’要么是‘双引号’，都是直的。在字符串的开头和结尾使用相同的字符。

智能引号是浏览器中显示的任何引号或撇号的正确印刷选择。它们通常是卷曲的或倾斜的，并且开始和结束引号通常是不同的。

如果您的文档使用了`utf-8`字符集，您可以在 Mac 上使用以下键盘快捷键将智能引号直接添加到 HTML 中:

```
‘ ⌥+]
’ ⌥+⇧+]
“ ⌥+[
” ⌥+⇧+[ 
```

在 Windows 上:

```
‘ alt+0145
’ alt+0146
“ alt+0147
” alt+0148 
```

或者，可以使用字符实体:

```
‘ &lsquo;
’ &rsquo;
“ &ldquo;
” &rdquo; 
```

### 国际行情

在研究这一集的时候，让我有点吃惊的是，不同国家的引号风格有如此多的差异。

英文引号看起来“像这样”——左右双引号(或单引号)。

法语引号使用左右角引号。

和丹麦引号使用相同的字符，但反过来。

保加利亚语、捷克语、爱沙尼亚语、格鲁吉亚语、冰岛语、俄语、斯洛伐克语、斯洛文尼亚语和乌克兰语使用底部双引号作为开始引号，使用右侧双引号作为结束引号。

其他国家使用这些风格的组合，在维基百科上可以找到引用标记的[表格。](http://en.wikipedia.org/wiki/International_variation_in_quotation_marks)

### `quotes`在 CSS 中

在 CSS 中，有一个`quotes`属性，允许我们控制浏览器如何为像`q`这样的元素生成`quotes`，这是为了内联引用。

```
q {
  quotes: "“" "”";
}
```

两个空格分隔的字符串指定了用于开始标记`quote`和结束标记`quote`的字符。

可以使用两组字符串来指定嵌套的`quotes`如何出现。

```
q {
  quotes: "“" "”" "‘" "’";
}
```

有了这个代码片段，任何嵌套的`quotes`都将使用单引号而不是双引号。

> "报价包含一个'嵌套'报价"

`q`元素将有浏览器生成的引号，但`blockquote`元素没有。我们可以使用伪元素和`content`属性添加`quotes`。

```
blockquote {quotes:"“" "”" "‘" "’";}
blockquote: before {content:open-quote;}
blockquote: after {content:close-quote;}
```

这些可以是具有任何其他 CSS 属性的样式，以实现所需的设计结果。

最后，我们可以通过根据文档或部分文档的语言更改`quotes`属性，将国际引用字符的知识与 CSS 结合起来。

我们可以使用`:lang`伪类来做到这一点。

```
:lang(en) q {quotes: "“" "”";}
:lang(fr) q {quotes: "«" "»";}
:lang(pl) q {quotes: "„" "”";}
```

现在，通过更改`html`元素的 language 属性，我们的引号被更正为相应的语言。
