# IE8:标准模式选择加入起死回生

> 原文：<https://www.sitepoint.com/ie8-standards-mode-opt-in/>

2006 年 10 月 Internet Explorer 7 的发布让许多人大吃一惊:微软已经齐心协力遵循网络标准。这款浏览器可能没有 Firefox 或 Opera 那么先进，但它正朝着正确的方向前进。不幸的是，它破坏了成千上万的网站和系统。

可以说，错误是 web 开发人员的责任，他们的系统继续使用非标准代码。不幸的是，这些系统中有许多是专门为 IE6 设计的——尽管有缺陷——并且多年来都运行良好。微软陷入了交叉火力之中，从 IE6 中脱离出来的举动充其量也是平静的。

2008 年 1 月，[微软宣布了他们对网站崩溃问题的解决方案。IE8 将提供两种渲染模式:](http://www.alistapart.com/articles/beyonddoctype)

*   默认情况下，网络上的每个站点都将使用 IE7 的布局引擎来显示
*   如果你使用了 web 标准—*,并且真的使用了 web 标准—*——你可以添加一个 meta 标签，强制 IE8 显示页面…
    `<meta http-equiv="X-UA-Compatible" content="IE=8" />`

明显的好处是，每个在 IE7 中工作的站点将继续完美地工作。即使 IE57 发布了，你今天创建的网页也会退回到最初测试的渲染引擎。

标准社区一片哗然:

*   采用网络标准应该有助于网站变得独立于设备和经得起未来考验。
*   浏览器应该尽自己最大的能力来呈现页面，而不是求助于之前的版本。
*   这项提案的长期前景如何？开发者需要列出所有兼容的浏览器和版本吗？
*   为什么我们需要更新每一个符合标准的页面，以便它选择加入或退出特定于浏览器的定向？

微软听取了他们的意见，并于 2008 年 3 月 3 日宣布他们将放弃这个计划。所有人都欢呼起来。

尽管取得了这一小小的胜利，但故事并未就此结束。最近的一篇微软博客文章描述了 IE8 中的[兼容性视图改进:](http://blogs.msdn.com/ie/archive/2008/12/03/compatibility-view-improvements-to-come-in-ie8.aspx)

1.  当用户在 IE8 中遇到网站问题时，他们可以单击“兼容性视图”按钮恢复到 IE7 渲染。
2.  该 URL 被发送给微软，微软汇编了一个 IE8 不兼容网站列表。
3.  这个列表被发送给 IE8 用户，这样网站就可以为每个人自动切换到 IE7 模式。
4.  **如果您的网站被修复或意外添加到列表中，您可以添加一个 meta 标签来禁用兼容模式！**

虽然这可能没有微软最初的提议那么糟糕，但最实际的解决方案是添加 meta 标签，以确保您的网站永远不会被添加到兼容模式黑名单中。我们又回到了起点！

微软的提议合理吗？这会对你的网站产生更好还是更坏的影响？
