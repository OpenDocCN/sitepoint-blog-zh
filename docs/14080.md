# 日蚀潮流中的另一位乘客:闪电侠？！

> 原文:[https://www . site point . com/another-passenger-on-the-eclipse-band wagon-flash/](https://www.sitepoint.com/another-passenger-on-the-eclipse-bandwagon-flash/)

Macromedia 已经加入了越来越多的人的行列，忙着将他们的高端开发工具移植到 [Eclipse](http://www.eclipse.org/) 平台上。

在一系列的新闻发布和网站宣传中，Macromedia 本周宣布了 [Flash 平台](http://www.macromedia.com/platform/)，这真的不是什么新东西——只是一个，呃，*华而不实的*提醒人们 Flash 可以做的一切。

Java 开发人员特别感兴趣的*新消息是[消息](http://www.macromedia.com/macromedia/proom/pr/2005/eclipse_flashplatform.html)，取代 Macromedia 定制的 [Flex Builder](http://www.macromedia.com/software/flex/flexbuilder/) IDE 的下一代开发工具(代号“左恩”)将基于 Eclipse 平台。*

Flex 是一个用于快速开发 Web 应用程序的平台，这些应用程序使用丰富的、基于 Flash 的接口，在服务器端动态生成，并与服务器端应用程序实时交互，这些应用程序通常用 Java 编写。尽管这是一个非常巧妙和强大的开发环境，但我认为可以肯定地说，Java 开发人员的接受速度很慢，他们通常不愿意放弃自己选择的开发工具去尝试定制的 IDE。

将 Flex 开发转移到 Eclipse 平台不仅会让数百万 Java Web 开发人员直接看到这项技术，还会让他们在用 Flash 构建富互联网应用程序时继续使用他们最喜欢的 [Eclipse 插件](http://eclipse-plugins.info/)。随着 Borland JBuilder】也即将被 Eclipse 取代，喜欢这个流行 IDE 的开发者也将有机会在他们选择的 IDE 中发现 Flex。

用 Flash 界面设计 Web 应用程序是否是一件好事情是另一天的辩论，但是 Macromedia 肯定会给这项技术提供蓬勃发展的机会。
