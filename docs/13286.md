# 雅虎！收购 Zimbra

> 原文:[https://www.sitepoint.com/yahoo-acquires-zimbra/](https://www.sitepoint.com/yahoo-acquires-zimbra/)

以下内容转载自《科技时报》第 174 期。

雅虎！收购所有最酷的公司。首先是 [Flickr](http://flickr.com/) ，然后是 [del.icio.us](http://del.icio.us/) ，现在是打包 [Zimbra](http://zimbra.com/) (据报道价值 3.5 亿美元！)，我敢说它是这个星球上拥有最多 Ajax 技术的公司(谷歌可能是个例外)。

**![](../Images/4ab62f10944e9cfc752baff7b04ec361.png) [“雅虎将如何换 Zimbra？”在 SitePoint 论坛](https://www.sitepoint.com/forums/showthread.php?p=3562528)**

如果你还没听说过的话， [Zimbra](http://zimbra.com/) 是一个开源的电子邮件、地址簿和日历服务器，它基于开源技术，用一堆 Java 代码捆绑在一起。它与 Microsoft Exchange 竞争，最著名的是它丰富的 web 界面，大量使用了 Ajax。我们在 SitePoint 使用 Zimbra。

在谷歌让 Zimbra 变得更酷之前，它一直在做 Ajax 驱动的电子邮件。该公司还向世界贡献了大量的 Ajax 技术。Zimbra 发布了第一批工业级 Ajax 库之一[，它帮助领导了](http://www.zimbra.com/community/kabuki_ajax_toolkit_download.html) [OpenAjax Alliance](http://openajax.org/) ，并发布了大量[博客帖子](http://www.zimbra.com/blog/)，这些帖子揭示了现实世界 Ajax 开发中有时令人困惑的水域。

现在，Zimbra 属于雅虎！我只希望看到这些趋势继续下去。雅虎！最近在代码和知识方面为社区做出了惊人的贡献，尤其是通过[雅虎！开发者网络](http://developer.yahoo.com/)。这种开放和共享的 web 开发方式似乎是两家公司的共同点。

至于 Zimbra 产品(相对于 Zimbra 公司)可能会发生什么变化，这是许多人[对](http://www.zimbra.com/blog/archives/2007/09/business_as_usual.html)更加紧张的事情。雅虎！看起来——至少从我的角度来看——在服务器端开发上更喜欢 PHP 而不是 Java，据报道，它正在寻求利用对 Zimbra 的投资来打入新市场。这一战略可能会给 Zimbra 带来哪些变化？
