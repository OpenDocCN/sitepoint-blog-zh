Louis: 那么，你想不想告诉正在听的人，我意识到大多数人可能至少对 Git 和 GitHub 有些熟悉，但是对于没有熟悉的人，你想不想告诉我们什么是 Git，什么是 GitHub，为什么它有这么奇怪的名字？

Tom: 所以，Git 是一个分布式版本控制系统，这意味着软件开发人员使用它来版本化他们的代码，并与其他人共享。现在，它的分布式部分意味着每个拥有代码库副本的人都拥有包含其所有历史的整个库，所以你可以在本地使用它，在飞机上，所有的操作都很快，因为它访问你的本地库；这与 Subversion 等系统不同，Subversion 有一个集中的地方存储所有文件，要做任何事情都必须同步到那个存储库。因此，在 Subversion 中，即使像做日志这样简单的事情也非常慢，因为它是通过网络进行的，但是在 Git 中，它非常快，因为它是本地的。Git 也有所谓的分支，因此它使得创建新的分支和在相同的代码基础上做实验工作变得非常容易，并且它使得将这些分支合并到主开发分支中也非常容易，这是其他版本控制系统不能很好处理的。这就是 git，Git 是由 Linus Torvalds 为 Linux 内核创建的，当他对替代方案不满意时，他将其命名为 Git，这背后的故事据说是 Linus 说他喜欢以自己的名字命名项目，先是 Linux，现在是 Git，这很有趣，因为在英国“Git”的意思是一种混蛋或某个自以为是的人。这就是 Git，这就是 Git 的来源，也是它擅长的。现在，GitHub 显然是以版本控制系统 Git 命名的，它的“中心”部分就是我们的全部，我们要与你的朋友分享这些 Git 库。所以在 g itHub 出现之前，我们开始使用 Git，这是在早期，当时除了 Linux 内核之外，没有多少人使用它，我们发现具有讽刺意味的是，Git 是分布式的，这意味着与其他人一起使用您的存储库应该非常容易，分享您正在进行的更改应该非常容易；据说这是它最擅长的。但事实是，除非你在某个地方有一个 Linux 服务器，你可以为你的朋友设置用户帐户，否则实际上分享这些代码并不容易，你需要在某个地方把它放在你可以来回传输的地方，这样你就不必直接连接其他人的计算机，因为防火墙之类的东西，这真的很难。所以我和另一个联合创始人克里斯·万斯特拉斯，我们来自网络开发背景，我们说嘿，我们想使用这个 Git，这个版本控制系统，Git，我们开始使用它，它真的很棒，用于开发和做分支之类的，但是它的共享方面不是很好。我们说，嗯，我们是 web 开发人员，如果我们做一个网站，让我们共享这些库，在线查看它们，查看人们正在做的那些提交，跟踪分支的历史，以及它们是如何分支和合并在一起的，如果我们做一些事情让这变得非常容易，会怎么样？这就是我们所做的，这就是 GitHub，一个你可以非常容易地共享你的代码并使用我们从一开始就建立的许多工具与其他人合作的地方。

**Louis:** 我想有趣的事情之一是 GitHub 已经取得了惊人的成功，它已经从几乎每个人都在使用 Subversion，有一些人在 Rails 世界的边缘，就像你说的 Linux 内核在使用 Git，而现在它几乎感觉像 Git 的主干，就像每个发布开源库的人都会把你链接到 GitHub 库，这发生得非常快。

汤姆:是的，我们刚刚在三年前的四月建立了这个网站，所以这个月，我想是四月十日<sup>三年前我们公开发布了这个网站。</sup>

路易:太疯狂了。

**Tom:** 是的，我认为我们所做的与最初不同的一件大事是，我们没有对人们在 GitHub 上放置什么进行任何限制，所以不像一些早期的系统那样，他们会让你获得许可来放置项目，他们会有一个批准过程，我们说这有什么意义，人们应该共享任何性质的代码， 它不一定是一个合法的项目，它可以是小实验，甚至是你的点文件，现在 GitHub 上有成千上万的存储库，只包含人们的命令行提示符的配置文件。 我们通过引入用户命名 URL 的概念来做到这一点，所以 GitHub URL 看起来像 github.com/username/projectname,，这意味着你不必担心用尽项目的有价值的名称，因为它都在你的用户之下。这就是为什么人们可以真正自由地发布内容，越多的人发布内容，就越有可能有人无意中发现，并说嘿，这个人正在解决我的问题；即使那个人可能不认为它对其他任何人有用，GitHub 上一些最受欢迎的项目是配置文件，谁会想到这真的是人们想要分享的东西，嗯，在 GitHub 出现之前没有多少人，但事实证明，偶然发现别人发现有用的东西，即使它不是为了公共消费，这是一件非常强大的事情。

路易斯:当你开始的时候，你有没有想过人们可能会开始分享这些随机的小片段，而这些片段可能不是成熟的项目？

汤姆:是的，我认为这是我们想要做的事情，因为我们对其他网站感到失望，他们需要等待数天才能创建一个新项目，这看起来有点傻。我们是非常大的开源人士，我们有各种各样的存储库，它们的代码并没有真正准备好以一种合理的形式发布，它们只是我们一直在玩，我们开始了这个库，这个 Ruby gem 或什么的，我们只是想把它放在某个地方，这样首先它是安全的，我的意思是它充当备份， 其次，当我们想向其他人展示它时，我们只需向他们发送 URL，就这么简单，我们不需要通过电子邮件向他们发送大型代码库的 zip 文件，不，我们只需向他们发送 URL，并说嘿，这可能会解决您的问题，我不再研究它了，但也许您希望获得这个代码库并从中受益，因为我不会再这样做了。 所以这是我们想亲自做的事情，这就是我们这么做的原因。因此，我认为我们知道，在没有大量数据支持的情况下，我的意思是，我们这样做是因为我们想这样做，所以这就是我们在 GitHub 所做的主要工作，我们构建我们想要的东西，满足我们的需求，事实证明，我们非常像许多开发人员和其他人一样，喜欢我们做的同类事情。

**路易:**对。我可能只是想稍微转换一下话题，你简要地谈到了 Ruby Gem，我有来自 Ruby Source 的 Dan 和我们在线，我只是想让你们聊一下它是什么，我的意思是我猜你在开始 GitHub 之前是一个 Ruby 的人，所以在 Rails 中构建 GitHub 的决定对你来说是很自然的？

汤姆:对，绝对是。我的意思是我在为一家叫做 Powerset 的公司做工具，这家公司做维基百科搜索，在我开始做 GitHub 之前，我就在那里做工具，因为我开始做 GitHub 的时候，我还在那里全职工作。在那之前，我已经做了几年 Ruby，对 Rails 很着迷，经常在三藩市的 Ruby 聚会上闲逛，遇到了很多做 Ruby 的人，在那里我遇到了克里斯和另一个联合创始人 PJ。所以做这个决定是，我的意思是没有决定，只是嘿，我想做这个网站，我们开始，创建 Rails 应用程序并开始编码。

**路易:**对。所以当你开始的时候，GitHub 现在的工作方式是，我想，它有一个有点新颖的定价结构。所以你免费提供无限的公共库，所以任何人都可以注册一个帐户，如果他们想做的只是放一些东西给大家看，或者他们有一个他们想托管的开源项目，他们可以免费这样做，然后如果你想拥有私人库，只是与你的团队分享它们，那么这些都是付费计划。你从一开始就有这样的定价结构吗？

**Tom:** 在很早的时候，我和 Chris 就我们希望如何定价进行了长时间的讨论，我从我以前的一个创业项目 Gravatar 中学到的一个教训是，你可以在任何地方看到跟随你的小头像，我这样做了，我创造了它，没有任何商业模式，我只是因为我认为这很有趣。 这是回报博客生态系统的一个很好的方式，我最终了解到，如果你没有办法从你的副业项目中赚钱，最终它会变得非常昂贵，并开始占用你越来越多的时间，然而你并没有找到谋生的方式，所以现在你基本上有两份全职工作，你只是没有从其中一份工作中获得报酬。 这是我从那些日子里学到的一个很大的教训，所以当我们开始 GitHub 时，我说这看起来可能是一个很酷的兼职项目，但是如果我要在这方面投入很多时间，那么我也希望能够从中赚钱，所以如果它真的变得很受欢迎，那么它也应该赚很多钱，这样我就可以全职做它。因此，这肯定是一个从一开始就有意识的决定，让我们找出一种收费的方式，我们就此谈了很多，最有意义的事情是，谁将能够支付这笔费用，谁将拥有这笔钱？开源的人不会愿意为这样的东西付费，因为它是开源的，他们是在边上做的，他们做这个是为了好玩，我们希望鼓励人们把他们的代码放在这里并分享它，这是我们如何获得曝光率的一种方式，所以让我们对他们免费，我们可以向谁收费？嗯，你知道谁有钱，公司有钱，所以让我们向他们收费，对吧，让我们针对有钱的人定价，这是有意义的，对吧？那么，我们可以对那些公司想要而开源开发者不需要的东西收费，那就是隐私。所以这个模型真的变得很好，让我们对私人储存库收费，这就是我们三年前向公众推出的时候；我们推出了这种定价模式，自推出以来，这种定价模式从未改变过。

路易:没错，这对你们来说效果很好。

汤姆:是的，效果非常好。我是说每个人都很开心，对吗？开源开发者免费获得，公司有他们的隐私，他们有钱支付，所以他们很乐意这样做，我认为这对每个人都是双赢的。

**Louis:** 鉴于发布到 GitHub 上的代码数量如此之多，每个人的东西都在那里，你是否发现有足够多的人为私人回购付费，从而能够为其他人提供免费服务？

汤姆:哦，是的，当然。我们构建架构的方式和运营业务的成本绝对在实现这一目标的范围之内。我的意思是我们现在，我想我们刚刚雇佣了第 27 名<sup>第</sup>名员工，所以本质上来说，这不是问题。

**L** **是的:**好吧，很明显你成功了。

Dan: 我个人知道，作为一名开发人员，我会把我愿意分享的东西放在那里，特别是在一开始价格很低的时候，开始把你的专业项目放在那里也是一件很容易的事情。老实说，我甚至没有想过要订阅白金计划并坚持到底，所以它对我来说真的很好。

路易:是的，我也是。我不得不说，我也有一个付费的 GitHub 帐户，所以我想我不应该对人们在那里为足够多的人付费这一事实表示任何震惊。我想，这不仅是公司，而是很多——你有定价计划，只对日常开发者开放，好吧，我正在做一些开源工作，我想让每个人都看到，但我也为客户做一些项目，而不是只把它们放在我的机器上，我可以把它们和其他东西放在一起。

汤姆:对，没错。该计划每月起价 7.00 美元，因此个人开发人员很容易进入，拥有咨询公司或其他机构的人，大概会随着客户数量的增加而增加，对吧，你有一群客户，他们向你付费，所以你拿一小部分，向我们付费，让我们构建这个很棒的工具，你可以用它来让你的生活更轻松，你可以让你的客户在个人的基础上访问这些存储库，这对很多人来说都很好。所以，是的，你肯定可以在不涉及很多钱的情况下进入它，这是另一个有意识的决定，嗯，我们需要人们付钱给我们，虽然三年前没有多少人使用 Git，所以我们必须使价格足够低，人们可以说每月 7 美元，我可以摆动，这不是一件大事；我喜欢 Git 足够支付这些家伙为我做真正伟大的主机。我们一直将这一理念保留至今，即帮助人们参与进来，对吧，只是制定非常好的低成本计划，在这些计划中，你仍然可以完成真正的工作，这真的是一种入门；让他们更容易参与进来，因为如果我们的第一个计划是每月 100.00 美元，那么没有人 Git 的新用户，他们只是想尝试一下，他们只有几个存储库，他们不会支付这个费用，所以用这种好的方式进行定价效果非常好，我们确实得到了教训。当我们推出组织功能时，这是一些关于团队和组织人群的功能，并基于此进行许可，我们从旧计划中有了一个相当大的跳跃，所以下一个计划的成本与下一个计划的成本有 50.00 美元的差距，结果是这个数字太高了，人们不愿意做这么大的跳跃。因此，我们回应说，好吧，让我们在中间添加另一个计划，以便人们可以有一个更简化的升级过程，人们非常喜欢这一点，每个人都说，哦，这将工作，这太棒了。因此，你必须对这些事情做出反应，并了解你的客户群。我们意识到，我们的客户主要是小企业，我们有一些较大的人，但在大多数情况下，我们的核心客户是大量的小企业，我想我们暂时忘记了这一点；我们说，他们都是大企业，对，他们有大量的资金，但这些小企业不一定，他们只是刚刚起步，或者他们是咨询公司，没有太多的利润，他们只是想混下去，我们想为这些人服务，他们是我们的客户群，所以我们需要能够为他们提供真正的服务。一旦我们意识到这一点，并制定了新的计划，那么从那时起一切都很完美，这就是正确的解决方案。
