# PHP 是 ASP 3.0 / VB 6 的自然升级途径吗？

> 原文:[https://www . site point . com/is-PHP-the-natural-upgrade-path-for-ASP-30-v b-6/](https://www.sitepoint.com/is-php-the-natural-upgrade-path-for-asp-30-vb-6/)

冒着引发一场战争的风险，我得把这件事说出来。

[Joel Spolsky](http://www.joelonsoftware.com) 最近在[上发表了一篇他对微软如何输掉 API 战争](http://www.joelonsoftware.com/articles/APIWar.html)的感想；一篇目前正在各地掀起风暴的文章(很多人都听乔尔的)。无论你的立场如何，都值得一读。

在许多观点中，他提到了 VB.NET 如何不支持 VB 6，他总结说；

> 新的 API 是 HTML，应用程序开发市场的新赢家将是那些能让 HTML 歌唱的人。

我想知道的是 ASP 3.0 开发者对 ASP.NET 的感觉如何？ASP.NET 是软件架构理想应用，这很好，但我想知道 ASP 3.0 开发人员，这种类型的人可能不太关心 OOP，更感兴趣的是按时完成项目，对 ASP.NET 的感觉如何？

“将 ASP 应用程序移植到 PHP”这个短语最近似乎越来越多地出现在雷达上，PHP 支持不同的编程范式(例如，过程化的、面向对象的、 [functional](http://en.wikipedia.org/wiki/Functional_programming) )，而不会强加给你任何东西。在某些方面，PHP 与 ASP 3.0 非常相似，甚至有一个 [ASP 2 PHP](http://asp2php.naken.cc/) 转换器，它并不完美，但至少可以帮助初学 PHP 的人入门。

促使这一点的是兰迪·霍洛韦斯(T2 在微软工作)博客上的匿名评论，兰迪本人(T4)回应了乔尔(T5)；

> 这篇文章最终足以让我的老板相信微软并不关心 VB6 用户。没有人想学更多的 VB6。VB.net 就像 C#一样复杂。搬到 VB.net 比仅仅用 PHP/HTML 重做我们的整个系统更昂贵。

比较一下从 PHP 4 迁移到 PHP 5 的文档[；](http://www.php.net/manual/en/migration5.php)

> PHP 5 和集成的 Zend Engine 2 极大地提高了 PHP 的性能和功能，但也尽可能少地破坏现有代码。所以把你的代码从 PHP 4 移植到 PHP 5 应该很容易。大多数现有的 PHP 4 代码应该已经准备好运行，无需修改…

火焰在…；)
