# 注意贝宝，亚马逊对支付很认真

> 原文:[https://www . site point . com/watch-out-paypal-Amazon-gets-serious-about-payments/](https://www.sitepoint.com/watch-out-paypal-amazon-gets-serious-about-payments/)

去年 8 月，亚马逊推出了他们的灵活支付系统，允许初创公司使用亚马逊的后端收款。当时，它被誉为贝宝的竞争对手，在某些层面上，它的确是——它是一个商业账户的替代品，可以联合品牌，允许开发者从他们的网站上支付。然而，它远不是一个简单的支付产品。当它推出时，亚马逊的杰夫·巴尔警告说，使用 FPS 不适合心脏虚弱的人。“FPS 是专门为开发者设计的，不应该被轻视，”他说。"这是一个非常丰富的服务 API 文档长达 250 多页."

今天，亚马逊通过推出简单的支付和结账扩展了他们的支付产品。这两种服务都可以通过亚马逊的支付页面获得，新服务面向小企业，但不需要太多的开发者知识。

亚马逊发布了他们三种商业产品的对比图。FPS、Checkout 和 Simple Pay 之间的主要区别在于，FPS 需要“使用 API 的高级编程技能”才能实现，而其他的只需要 HTML 或脚本知识。简单支付和结账之间还有进一步的区别——结账是更完整的产品，在亚马逊照顾你的订购过程方面，而简单支付只处理支付，并为消费者提供更多选择，在*如何*支付方面。

两者的费用结构是相同的，与 PayPal 竞争。对于超过 10 美元的付款，卖家将支付 2.9% + $0.30，批量折扣从每月 3000 美元的销售额开始，对于低于 10 美元的付款，费用为每笔交易的%5 + $0.05。

《纽约时报》指出，许多卖家可能对使用亚马逊的支付服务持谨慎态度，因为他们将亚马逊视为竞争对手，不希望这家总部位于西雅图的零售巨头获得他们的销售数据。“许多大型零售商对与亚马逊合作表示担忧，因为亚马逊是他们最大的竞争对手之一，他们不想让亚马逊知道他们的信息，”ChannelAdvisor 首席执行官 Scot Wingo 告诉*纽约时报*。“另一方面，易贝和贝宝不会利用销售数据去采购产品，因为他们不是零售商。”

但是考虑到易贝最近向固定价格商品的转变，比如他们今年早些时候与 Buy.com 的广泛合作，卖家可能也会变得越来越警惕。
