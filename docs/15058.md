# 审查-电梯尼尔森诺曼集团版

> 原文:[https://www.sitepoint.com/nielsen-norman-group-edition/](https://www.sitepoint.com/nielsen-norman-group-edition/)

近来，网页可访问性已经成为许多网页设计师的重要考虑因素。美国政府下属的网站现在必须遵守 Section 508 可访问性指导原则，而 W3C 已经为所有的网页设计者制定了自己的一套指导原则。但是，除了这些技术标准之外，还有很多其他的事情可以做，以使网站对残疾人更有用，正如尼尔森诺曼集团最近的报告所概述的那样:[超越替代文本:使残疾用户更容易使用网络](http://www.nngroup.com/reports/accessibility/)。

如果您使用 Macromedia Dreamweaver/UltraDev 4 . x 或 MX 进行网页设计工作，[LIFT–Nielsen Norman Group edition](http://www.usablenet.com/products_services/lfdnng/lfdnng.html)(LIFT–NN/g)是一个强大的工具，它将标准和 Nielsen Norman Group 报告中的所有建议捆绑到一个自动化工具中，可以融入您的工作流程。

Dreamweaver 扩展 LIFT–NN/g 为程序添加了一个可停靠窗口调色板。这些窗口允许您通过以下两种方式之一检查您的工作是否存在辅助功能问题:

*   评估:这种模式允许你检查一个或多个页面(或你的整个网站)，并在一个方便的向导中纠正任何问题，如果你需要的话，它会详细解释每个可用性问题。
*   监控(Monitor):这就像评估模式一样，但是它实时监控你的工作，而不是一次检查所有的工作。当您将问题列在一个不显眼的列表中时，系统会向您指出这些问题，这为您提供了相同的向导界面，以便在您方便的时候修复它们。

![LIFT - Nielsen Norman Group main interface](../Images/aa4da57cba5dee37f41ebd9b3b560d93.png)

LIFT–NN/g 检查的指南集是完全可配置的。您可以有选择地启用和禁用单个指南，选择指南集(W3C，508，可用性等。)，或者创建您自己的自定义集。自定义的指南集可以保存并复制到其他计算机上，这使得 LIFT–NN/g 非常适合在设计团队成员之间协调可访问性指南。

不像这种类型的其他工具那样乏味和不灵活，我发现 LIFT NN/g 非常适合我的工作流。不要问我怎么做，但是 LIFT–NN/g 能够根据上下文对可访问性问题提出适当的修正建议。例如，它能够判断图像标签何时是间隔图像，而不是提示图像的描述，它提供插入一个空的`alt`属性(`alt=""`)。

![LIFT - Nielsen Norman Group fix wizard](../Images/087e01af8c3357708272e8973e186308.png)

像`alt`属性编辑器和报告生成器这样的便利特性只是锦上添花，前者为您提供了一个统一的界面，可以从一个窗口编辑文档或站点中的所有`alt`属性，后者可以输出 HTML 或 XML 格式的可访问性报告。如果您使用 Dreamweaver 并且可访问性是您关心的问题，那么 LIFT–NN/g 是必备的。

我对这个软件唯一的不满是它不灵活的界面。每个面板都有固定大小的内容。例如，上面的“Monitor”选项卡可以变小或变大，但问题列表和按钮的位置保持不变；因此，让它变小会导致界面不可用，让它变大只会浪费屏幕空间。发现讽刺之处。这些大小问题可能是用于创建 LIFT–NN/g 的 JavaScript 扩展技术的限制，但无论如何，它们都不仅仅是有点分散注意力。

LIFT–NN/g 现已在 UsableNet 网站上以 549 美元的价格出售。Macromedia Dreamweaver 的 LIFT(只检查 W3C 和 Section 508 可访问性，缺少一些更高级的 LIFT 向导–NN/g)售价 299 美元。微软 Frontpage 的 LIFT 也是 299 美元。

最近，我有机会在一次采访中与尼尔森诺曼集团的 Jakob Nielsen 交谈，并特意向他提出了几个关于 LIFT–NN/g 的问题。请继续阅读，了解他的观点…

##### 雅各布·尼尔森和 LIFT-NN/g

尽管有新的法律和标准，为什么今天的网站“残疾用户比其他用户难使用三倍”？为什么可访问性不总是优先考虑的？

JN:网站在过去甚至更糟糕，所以已经有了一些小的进步。我没有确切的数字，但是有许多网站对于残疾用户来说更加难以使用，因为他们根本无法使用。例如，全图形网站，没有替代文本。

在过去(比如说，五年前)，有很多设计项目根本不关心可访问性。大多数项目经理认为盲人无论如何都不能使用网络，因此不能使用正在开发的特定网站。我们现在已经消除了这种误解，大多数互联网管理者都知道他们必须关心残疾用户。

在大多数项目中，可访问性的优先级相当低，因为项目经理低估了受设计问题影响的人数。他们认为他们只是失去了少数客户，而事实上他们可能会拒绝数百万客户，特别是老年人，他们构成了一个庞大而富有的群体，每天都在网上变得越来越活跃。对于一个设计团队的人来说，很难真正理解这样一个事实，即许多其他人与他们的网站的交互方式与他们自己非常不同。

项目总是有有限的资源，并且由于可访问性的优先级相当低，完成不了多少工作。有两种方法可以解决这个冲突:

*   提高这个问题的优先级。提高对受可用性差影响的具有可访问性问题的用户数量的认识，以及
*   减少为这些用户提供更好设计所需的资源。

我用第一种方法写文章和演讲，用另一种方法写这个新软件。

该软件将如何帮助纠正这种不平衡，修复歧视残疾用户的网站？

JN:我们已经接受了网页设计师也是人，有人类的错误。他们和用户一样没有完美的记忆。因此，我们不再依赖设计师在网站开发的每一步都记住每一条指导方针，而是将内存负担转移到电脑上(电脑当然非常擅长保持警惕，不断检查大量细节而不会感到疲劳)。

新软件基本上使网页设计者更容易记住残疾用户指南。该软件还可以通过提供一些问题的自动修复来使遵守指南变得更容易，这再次降低了做正确事情的障碍。

SP:为什么这款产品注重可用性(而不仅仅是易访问性)？为什么以用户为中心的设计很重要？

JN:技术上的可访问性不足以让一个网站易于使用。这从一个简单的事实就可以看出，没有残疾的人仍然很难使用许多网站。

真正的问题是，用户能否在合理的时间内从一个网站上得到他们想要的东西，访问对他们来说是否愉快。如果在你得到你想要的东西之前，你不得不坐下来听那些没完没了的废话，那么也许有可能使用这个网站，但是它肯定是不可用的。此外，因为网站是导航设计，如果听觉用户体验太难，用户会跟随错误的链接并迷路，这是一个很大的风险。

传统的可访问性假设用户有完美的记忆力，卓越的智力，并且没有比这更好的方法来投入所有的精神资源来解决网站的问题:只要让所有的信息都是可访问的，用户就会解决问题。并非如此。残疾用户也是人，和其他人一样需要简单易用的用户界面。

包含可用性指南的自动化软件有什么好处？这将如何改变设计师和开发人员的工作方式？

JN:主要的好处是计算机不会感到疲劳:因此它会一直检查每一个指导方针的每一个设计细节。此外，该软件提供了一个链接，从任何潜在可用性问题的发生到相关的研究材料，解释为什么它是一个问题，以及在决定是否保留或修复有问题的设计元素时应该考虑什么问题。这比在书中查找答案要快得多，事实上，这是超文本最初承诺的一个很好的例子。

主要的变化是，设计师和开发人员将会对可用性原则有更多的了解，因为他们会及时被提醒这些原则。当研究结果与你自己的工作直接相关时，阅读这些研究结果要比阅读厚厚的文件中的一部分更有动力，尽管你不得不费力地寻找你想要的东西。

这个产品进行了什么样的研究和用户测试？

JN:我们采取了一种不同于大多数其他可访问性项目的方法——我们将我们的指导方针建立在对与各种网站交互的各种用户的直接观察之上。这意味着我们观察真实的人在真实的网站上做了什么，并确定他们在哪里有最大的问题或最大的成功。

我们还在美国和日本进行了测试，以获得国际视野。我们纳入了失明、弱视和有运动技能障碍的用户。

对于这个项目，我们没有测试听力有困难的用户，因为声音目前对使用网站的能力不是很重要。如果你是聋子，你会错过一些没有隐藏字幕的视频，对于一个视频丰富的网站来说，添加字幕是一件好事。但是绝大多数网站对于有听力障碍的用户来说都非常好用。也许将来会有所不同，我们将不得不添加更多关于如何处理音频的指南。

SP:怎样才能带来可用性的巨大改进？在建设无障碍网站方面，像谷歌这样的网站会比法律有更大的影响力吗？

JN:谷歌在两个方面有很大的影响。首先，通过例子证明了简单可以获胜，用户会喜欢一个快速简单的网站。第二，因为谷歌把最高优先级放在最好的网站上，它会倾向于把用户导向可用性合理的网站。谷歌当然不会把可用性作为在搜索列表中排名的唯一标准，但是可用的网站确实在谷歌评级中得分较高。

尽管如此，要真正在可用性方面取得巨大进步，我们需要超越谷歌，让公司高管相信，在设计过程中给予用户需求更高的优先级，而不是满足他们自己的突发奇想，这对他们自己有利。你不是为自己设计，而是为顾客设计。这听起来是一个显而易见的观点，然而大多数网站仍然没有把用户需求作为设计的驱动力。
