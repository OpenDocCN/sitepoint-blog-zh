# 如何请求推荐，第 2 部分

> 原文：<https://www.sitepoint.com/how-to-ask-for-referrals-part-2/>

周一的博客谈到了在你的合同/提案中加入对推荐人的讨论。

这篇博客讨论了何时以及如何向客户寻求推荐。(下一篇博客将讨论从非客户那里获得推荐的策略)。

首先，IT 专业人员在寻求推荐时犯的最大错误是混淆了“口碑”和推荐。口碑就是当你不在房间里的时候，依靠你的客户的善意来谈论你。它依赖于偶然性，以及你的客户——尽管他们正在做着一切——会记得向别人吹嘘你的可能性。

如果你受益于口碑，恭喜你！但是你仍然有很大的机会让客户推荐你。而口碑是被动的，要求引荐是主动的。它将事情置于你的控制之下。

当你证明了自己的价值或改善了业务关系时，才是向客户寻求推荐的最佳时机。这些时间包括:当你卖出合约的时候，当客户感谢你的时候，当你完成一个交付品并且客户告诉你他们喜欢你的工作的时候，当客户给你一个推荐的时候，当你改正一个错误的时候(信不信由你，这是最好的时间之一，因为你已经做出了弥补并且为一个错误承担了责任，这在大多数专业人士中是很少见的)，当你超越了职责的时候，当你和客户克服了一个棘手的障碍的时候，等等。

或者，你可以简单地通过问“到目前为止，你最喜欢或发现我们合作中最有价值的是什么？”

一旦你创造了合适的要求引荐的氛围，你就得用正确的方式去问。首先请客户坐下来和你一起集思广益。“谢谢你的美言。您是否介意花几分钟时间讨论一下还有谁会从我的服务中发现类似的价值？”

然后将客户的记忆集中在特定的推荐类别上。不要只问一般的问题。例如，如果你的客户是当地商会的董事会成员，你可能会问，“还有谁对你的董事会感兴趣？”如果客户向你可以服务的企业销售产品，询问那些可能从你的服务中受益的客户的名字。如果他们打网球，找可能想和你一起打网球的伙伴。诸如此类。

如果你和客户关系很好，你甚至可以要求查看他们的联系名单。

当你得到一些名字时，问客户他们是否想做一个介绍，或者你是否可以跟进。一起制定行动计划，这样你就尊重了他们的关系。然后从那里开始。

让客户知道发生了什么。毕竟，他们是在代表你冒险。不管结果如何，感谢他们的支持和帮助。如果你通过推荐找到了一个客户，送他们一份大礼。

同时，一定要问清楚你的客户他们在寻找什么样的推荐。获得推荐的最好方法之一就是给他们！

这就是如何要求引荐。我可以建议你挑选你的前 5 名客户，并制定一个计划，要求他们推荐本周？让我知道进展如何。
