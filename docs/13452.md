# Rails 1.2:不再有借口！

> 原文:[https://www.sitepoint.com/rails-12-no-more-excuses/](https://www.sitepoint.com/rails-12-no-more-excuses/)

如果你一直生活在另一个星球，并且错过了它，这个月见证了 Rails 1.2 的发布，这是框架的一个重要里程碑。

为了配合这次发布，SitePoint 本周宣布了我们自己的 [Ruby on Rails 图书](https://www.sitepoint.com/books/rails1/)、*由 Patrick Lenz 撰写的《构建你自己的 Ruby On Rails Web 应用程序*。这是对 SitePoint 库的一个极好的补充，是专门为 Rails 1.2 编写的。如果你还没有发现这到底是怎么回事(以及为什么 Rails 让 web 开发人员如此兴奋)，那么你就没有更多的借口不去冒险了！

[![Build Your Own Ruby On Rails Web Applications](../Images/617a3e55b5013d361f055b8135558355.png)](https://www.sitepoint.com/books/rails1/) 尽管有其长处，但是 [Rails 可能会让初学者望而生畏](https://www.sitepoint.com/php-developers-most-likely-to-switch-to-rails/)，因为掌握框架还需要学习一系列与最佳实践 web 开发相关的技能和概念。

因此，初级程序员会欣赏 Patrick 对核心主题的关注，如模型-视图-控制器模式、迁移和持续测试，以及他对面向对象概念的解释和对 Ruby 语法的深入探讨。

作为这本书的技术编辑，我也为这本书对细节的关注感到非常自豪——尤其是安装过程；根据您的平台，安装 Rails 可能是一件不需要动脑筋的事情，也可能是一场令人沮丧的噩梦。

比如，我没有看到市面上有其他 Rails 书籍提到 Ruby 1.8.5 与 Rails 断点客户端不兼容的事实(尽管[Rails 核心团队推荐使用 1.8.5](http://www.rubyonrails.org/down) ，最新版本的 [MacPorts](http://www.macports.org/) 和 [InstantRails](http://rubyforge.org/projects/instantrails/) 也都安装了 1.8.5)。在我看来，使用这个不可或缺的工具的能力使得坚持使用 Ruby 1.8.4 非常值得——我已经在 Java 世界中崭露头角，我无法想象在无法设置断点的情况下试图跟踪代码中的 bug。

然而，最精彩的部分无疑是你将在本书结尾时构建的示例应用程序——流行社交新闻网站 [digg](http://digg.com/) 的克隆版(出于厚脸皮，我们称该应用程序为 *Shovell* ),配有 Ajax 投票、标签、每用户投票历史等功能。耶宝贝，从这里开始一切都是 Web 2.0！

我们已经发布了几个章节作为 Ruby on Rails 教程，和往常一样，前几个章节可以以 PDF 格式供[下载。](https://www.sitepoint.com/show-modal-popup-after-time-delay/)

因此，如果你已经对下一件大事有了一段时间的想法，这本书可能会告诉你如何将想法变成现实。
