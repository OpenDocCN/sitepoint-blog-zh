# 新 Linux 用户的配置文件

> 原文:[https://www.sitepoint.com/profile-of-the-new-linux-user/](https://www.sitepoint.com/profile-of-the-new-linux-user/)

Tom Adelstein 写了一篇关于 T2 新 Linux 用户改变社区面貌的有趣文章，这让我想到了一篇漫无边际的深夜博客。

在与 [Jules](https://www.sitepoint.com/blog/) 交谈时(很抱歉，但最终决定不做完整的文字记录——应该做笔记——我的啤酒模糊记忆让我失望了…)他“承认”最近他的桌面从 Windows XP 切换到了 Linux (Fedora Core)。让他做出切换的是[跨平台](https://www.sitepoint.com/blog/)开源，即[火狐](http://www.mozilla.org/products/firefox/)、[雷鸟](http://www.mozilla.org/products/thunderbird/)和一个(未命名)跨平台文本编辑器。除此之外，在使用 PHP 时，他能够在 Linux 上“生存”,这要归功于他将 Linux 作为服务器(即 shell)的经验。

我不得不承认我的故事非常相似——只是在火狐和雷鸟的列表中添加了 [SciTE](http://www.scintilla.org/SciTE.html) 。如果我没记错的话，大约一年前，我才开始使用 Linux 作为我的*桌面*(Suse 8.1)，尽管自从我还是一名物理学学生以来，我已经与各种风格的 Unix 发生过无数次冲突(在那里，我被告知 Fortan 是多么有用)。

我想，我这一代“英国书呆子”是从辛克莱·ZX 的光谱(事实上在那之前有一台 ZX81)开始的，随着 DOS 和 Windows 3.1 的发展壮大。在桌面上安装 Linux 让我想起了 IBM PC = DOS 时的情况。我仍然对微软的 Win 95(以及更高版本)和它提供给内部的“简化”界面心怀怨恨——我喜欢控制。

无论如何，回到汤姆·阿德尔斯坦的评论，特别让我感动的是:

> 他们礼貌地问了很多问题。他们也很快理解，并且不介意让你知道他们想要改变操作系统。这些用户去支持论坛，提出有趣的情况。他们不想显得愚蠢，即使是在论坛别名的面具后面。当被追问时，他们会告诉你任何你需要知道的来帮助他们。他们也很快学会如何通过邮件列表和论坛解决问题。

当我开始在 IT 行业全职工作时(1993 年),普遍的态度是“保护你所知道的/保护你的工作”。问别人一个类似“为什么？”或者“怎么做？”回应是典型的敌意咆哮。

互联网改变了这一切——通过谷歌传递一些错误信息，10 次中有 9 次问题得到解决。

在可能的情况下，我会尝试与尽可能多的人分享我所知道的东西，这样他们就可以在以后提醒我已经忘记的东西(因此有了 Sitepoint 的文章——让我不必填满我的大脑空间)。

从我的角度来看，你在 IT 领域“优秀”的标志是*而不是*你知道什么，而是你的态度——在我看来，记住无数事实的能力在很大程度上已经无关紧要了——更重要的是有信心*尝试*，以及在处理“未知”时解决问题的逻辑方法。

很高兴听到 Linux 用户的“新品种”,他们有信心提出问题并承认无知。在我看来，在 Linux(实际上是 Unix)环境下生存其实就是知道正确的东西。比 Windows 更是如此，在 Windows 中,“真相”经常被伪装成“用户友好”的错误消息。

当然，关于 Linux 和在其上运行的软件，有几乎无穷无尽的东西需要了解，它继承了 Unix 的过去。这可能令人望而生畏，但是只要你不沉迷于记住所有的东西，你就可以成为一个快乐的 Linux 用户。好消息是，一旦你学会了，它很可能会保持相关性——我敢打赌，你可以从 20 世纪 70 年代末找一个相当有经验的 Unix 用户，把他们放在 Linux 前面，他们会过得很好。
