# 作为消费者的社区

> 原文：<https://www.sitepoint.com/communities-as-consumers/>

这一年让我对社区的本质有了有趣的了解。我已经从经营一个非常大的、已建立的社区，发展到从头开始建立四个衍生社区。这些新观众是非常不同的，当谈到了解他们需要什么时，我仍然在摸索。我所指的社区是 DesignFestival、BuildMobile、CloudSpring 和 RubySource。

每周我都会花一个小时左右的时间来分析我们在脸书和推特上的统计数据，以便发现任何正在形成的模式。我观察我们不同的利基社区的表现，并据此制定策略。不追随率高吗？是什么推特触发了他们？脸书互动比平时高吗？他们对什么样的事情有反应？他们更喜欢个人问题还是自动反馈？有没有我能提供的针对某个特定领域的东西？

## 活跃的社区

如果你很好地理解了一个活跃的社区，那么它很容易维持，但是如果一个社区是稳定平坦的，那么你能做些什么呢？有很多刺激活动的选择，但是在什么时候你决定投资回报与花费的时间不相平衡呢？什么时候是够了？

几个月前，我决定打这个电话，关闭 RubySource 脸书页面。有 160 名粉丝，但他们似乎不需要或不想要任何互动，只是让页面停滞不前让我很不舒服。感觉有点像我们错过了观众，这似乎很天真。我现在想知道这个决定是否为时过早。

## 没有两个利基是相同的

与其失败，也许我应该从不同的角度来思考这个社区。仅仅因为他们不像我们的 SitePoint 社区那样互动，并不一定意味着我做错了什么——也许他们表明有新一代的社区。缺乏直接互动意味着他们只是消费者。统计数据显示，新成员数量稳步上升，互动数字保持稳定。他们点击和喜欢，但他们没有声音。看起来他们只是来跟踪我们的。我想，如果你花足够的时间在脸书，这是通过电子邮件订阅的一个公平的选择。

因为 SitePoint 最初是一个论坛社区，所以它本质上是交互式的。这件事传到了脸书和推特上，在很长一段时间里，我只知道这些。一般来说，公司使用脸书网页来建立一个营销基地。我们的利基页面不是为这个目的服务的，所以我发现我在试图复制我在 SitePoint 页面上做的事情，而没有意识到社区性质的差异。现在我有了，它带走了一些表演的压力，我正在学习接受社区作为一个消费者，而不是一个人们来寻求帮助和分享想法的地方。

如果这是社区行为新趋势的开始，我很好奇这对未来社区经理的角色意味着什么。
