# 我 2011 年的三个网络愿望

> 原文：<https://www.sitepoint.com/3-web-wishes-for-2011/>

在我回顾了 2010 年五大网络趋势和技术之后，是时候展望 2011 年了。如果我有三个*现实的*网络愿望，这就是我在来年想要的…

## 1.微软将在 Windows XP 上发布 IE9

微软显然对 HTML5 感到兴奋。他们的会议、文章和产品都在推动这项技术成为下一个“大事件”

那么，为什么要开发一款 60%的用户都无法安装的 HTML5 浏览器呢？

如果微软不为他们最成功和普及的操作系统发布一个兼容的浏览器，HTML5 可能会被推迟几年。不在 XP 上发布 IE9 的主要理由是:

1.  **IE9 采用 Windows Vista/7 渲染技术。**我不怀疑，但是没有根本原因不能移植到 XP 上。DirectX 不应该是解决操作系统和硬件不兼容的解决方案吗？此外，所有其他浏览器供应商都支持 XP，没有抱怨——有几家提供类似 IE9 的视频加速。如果别人有资源支持 XP，微软当然有。
2.  XP 是一个十年前的操作系统，对它的支持正在被淘汰。的确如此，但它是目前微软最受欢迎的操作系统。如果他们真的想放弃 XP，他们可以停止销售操作系统，发布仅支持 Vista/7 的 Office 版本！

如果你认为 IE6 的开发现在很艰难，那就想想 2018 年支持 IE8 会有多糟糕。

## 2.服务器端 JavaScript 的广泛可用性

桌面应用程序开发人员很容易做到。他们选择单一的开发语言，并可能为复杂的系统添加少量的 SQL。然而，一个还算过得去的 web 开发人员必须学习 HTML、CSS、JavaScript、一种服务器端语言，比如 PHP、SQL，甚至可能还有 XML。

如果能在服务器上使用 JavaScript，减少工作量，岂不是很棒？node.js 可能是最好的解决方案，但它还没有获得广泛的吸引力，与 PHP 和 ASP.NET 的可用性相比相形见绌。或许这种情况会在 2011 年改变？

## 3.网络开发者会放弃占用带宽的网站

有一种恼人的网络发展趋势，认为带宽不重要。为什么有些网站坚持几兆的页面？为什么文件总大小比用来渲染的浏览器大？

我有些同情那些开发复杂 web 应用的人，虽然借口很少。谷歌和其他供应商可以提供几百 Kb 的完整在线办公套件，因此很少需要更大的应用程序。

但是对于纯内容网站来说，这是完全不同的事情。带宽不一定是廉价或无限的——尤其是对那些使用移动设备的人来说。削减大部分或者吊销你的网络开发许可证！

## 幻想的愿望…

这是我现实的愿望清单。以下是在 2011 年或任何其他年份都不太可能发生的不切实际的事情…

1.  地球上的任何地方都可以使用免费的开放式 wi-fi。
2.  SEO 牛仔找到自己的良心。
3.  PHP6 的发布完全支持 Unicode。

最后，图形设计师和网页开发者将会抛开他们的分歧，拥抱在一起！

新年快乐——2011 年再见！
