# 易访问性的秘密好处第 1 部分:增加可用性

> 原文：<https://www.sitepoint.com/accessible-usable-website/>

网页可访问性有这么多好处，我真的很好奇为什么这么多网站的可访问性如此糟糕。其中一个主要的好处是增加可用性，根据可用性专家 Jakob Nielsen 的说法，这可以将网站的销售/转化率提高 100%，流量提高 150%。

在这一点上，你肯定会问，“所以如果我让我的网站变得可访问，它的可用性会增加，我会从中赚更多的钱？”不完全是。一个可访问的网站不会自动变得更有用，但是有很多重叠的地方。现在让我们来看看它们。

##### 1.描述性链接文本

视障网络用户可以通过在链接之间切换来浏览网页，并收听链接文本的内容。因此，可访问网站中的链接文本必须始终描述其目的地。

同样，视力正常的网络用户不会逐字阅读网页，而是浏览网页寻找他们想要的信息。读这一段:

这是一些文字，很多很多可爱的文字。现在，这里有一个带链接的句子:要阅读更多关于我们的小工具，请。接下来是更多的文字，很多很多可爱的文字。还有一句话，包含更多的文本来说明这一点。

你认为它与这一个相比如何？

这是一些文字，很多很多可爱的文字。现在，这里有一个带链接的句子。当您访问我们的网站时，请阅读关于我们的小工具的

第一段的可访问性和可用性很差，因为无论是视力正常的还是视力受损的网络用户浏览该段时，都不会从“单击此处”这几个字中获得任何意义。第二段包含有效描述其目的地的链接文本，更容易浏览——用户无需阅读周围的文字就能理解链接的目的地。

##### 2.描述性表单标签

看看这两个复选框。它们可能看起来相似，但第一个是不可访问的，因为提示文本没有分配给框。为了使其可访问，我们可以使用`<label for>`标签，就像我们对第二个复选框所做的那样:

```
<input type="checkbox" id="good" /><label for="good">This is good</label>
```

以这种方式使其可访问有一个非常大的可用性优势:文本也变得可点击。在示例中，单击单词“这是坏的”。什么都没发生，对吗？现在试着点击“这很好”。其复选框被选中。复选框很小，即使是最稳健的手也很挑剔。增加可点击区域，人人受益！

##### 3.分割大块的信息

许多技术可以用来提高网站对视觉障碍用户的可用性。视障用户必须听每一页上的信息，并努力记住它。通过将信息组织成小的、可管理的组，可以实现更快的理解和更容易的回忆。

实现这一点的方法包括:

*   使用副标题来分解正文内容
*   使用 fieldset 命令对表单项目进行分组
*   使用列表

不需要天才就能意识到这些策略也增加了视力正常的网络用户的可用性！

##### 4.提供站点地图

站点地图对于视力有障碍的用户来说是一个有用的辅助工具，因为它们提供了一个到站点主页的直接链接列表，中间没有任何混乱。当然，网站地图对每个人都很有用，因为它提供了一种快速找到网页的方法，并帮助用户可视化网站的结构。

##### 5.简单易懂的语言

从可访问性的角度来看，这对于有阅读和/或认知障碍的人以及母语不是你写作语言的网站访问者来说很重要。从可用性的角度来看，它对每个人都有帮助。

从电脑屏幕上阅读会使眼睛疲劳，比从纸上阅读慢 25%。因此，写作风格越简单，网站访问者就越容易理解你的智慧之言。尽可能缩短你的句子。例如，用“apply”代替“make an application”或者用“use”代替“make use of”。

##### 6.一致导航

跨页面的一致导航也是最大化网站对阅读和/或认知障碍者的可访问性的一个重要方式，但同样，每个人都受益。每次你访问一个新的网站，你都需要几秒钟来适应这个页面独特的布局和用户界面。想象一下，如果你每次点击一个新页面的链接时都必须这样做！

事实上，大多数网站在大多数页面上都有一致的导航。违反这一准则的罪魁祸首是主页，一些网站的结构与网站的其他部分非常不同。通过在整个网站上使用一致的界面，我们可以立即找到页面内容，而不必四处寻找。

如你所知，导航一致性是网页设计的一个基本原则，对于注重易访问性的网站来说并不重要。这是支撑网页设计的基本要素。

不管你的客户是对易访问性还是可用性感兴趣，这些原则仍然有效。所以如果你有疑问，参考这些原则！他们通常很容易接近。

##### 7.没有未经宣布的弹出窗口

对于我们这些使用屏幕阅读器的人来说，弹出窗口可能是一个真正的可访问性麻烦。屏幕阅读器读出位于“顶部”的任何一个窗口的内容——在所有其他窗口之上。

显示在主网站顶部的弹出窗口总是在实际网站内容之前被首先读出。对于视障用户来说，这可能会令人沮丧——他们可能没有意识到他们听到的不是“真正的”网站。

弹出窗口不利于可访问性。至于可用性，我相信你和我一样讨厌弹出窗口！许多工具栏，如[谷歌工具栏](http://toolbar.google.com/)，现在都打包了一个弹出窗口拦截器，允许你在网上冲浪，而没有新窗口弹出的烦恼。

##### 8.使用 CSS 进行布局

基于 CSS 的网站通常有更大的内容与 HTML 代码的比率，所以它们更容易被屏幕阅读器和搜索引擎访问。使用 CSS 布局的网站也可以通过车载浏览器、网络电视和掌上电脑访问。不要低估这一点的重要性！到 2008 年，预计全球将售出 5800 万台 PDA。

除了提高可访问性，基于 CSS 的网站还有一个很大的可用性优势:提高下载速度。宽带并不像你想象的那样普及。例如，在英国，只有四分之一的网络用户接入了宽带。提高你的网页下载速度，你就能比你的竞争对手获得更大的可用性优势。

##### 9.使抄本可用于音频

有特殊访问需求的一组网络用户是听力受损者。这些用户需要音频内容的书面等价物。事实上，提供抄本对所有用户都非常有益。许多网站访问者可能懒得等你的 3Mb 音频文件下载并开始播放。他们可能更喜欢音频内容的快速概要。

通过提供一份按小标题划分的文字记录，并突出显示关键术语，非残疾网站访问者可以快速浏览以获得内容的大致想法。然后，他们可以做出关于等待音频文件下载的更明智的决定。

##### 10.避免屏幕闪烁和移动

一些癫痫网络用户必须小心避免 2 到 55 Hz 之间的屏幕闪烁。有阅读和/或认知障碍的 Web 用户以及使用屏幕放大镜的用户将很难跟上滚动文本(如果您确实有滚动文本，请确保提供一种机制来阻止它)。

除了对可访问性来说是个坏主意之外，闪烁和滚动文本对可用性都没有好处。前者可能会在你试图阅读时分散注意力，你会看到有东西从眼角闪过；后者并不理想，因为你必须等待内容慢慢出现。

想一想:当你看到滚动的文本时，你会放下所有的东西，这样你就可以在它逐渐具体化的时候阅读它吗？还是直接无视？

滚动或改变文本的另一个缺点是，你可能会看到你想点击的东西，但在你知道之前，它就不见了。然后你要等 30 秒让它再次重新出现！

##### 结论

鉴于网站可用性和可访问性之间的重叠，没有理由不在你的网站上实现基本的可访问性。除了道德上的争论，让你的网站具有可访问性还有很多原因，其中一个主要原因是它的可用性将会提高。没人能否认这一点！

敬请期待！明天，本系列的第 2 部分将揭示改进的可访问性的第二个隐藏的好处。
