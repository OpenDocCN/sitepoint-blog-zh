# 采访-来自 vBulletin 的 John Percival

> 原文:[https://www.sitepoint.com/john-percival-vbulletin/](https://www.sitepoint.com/john-percival-vbulletin/)

约翰·帕西瓦尔高高地坐在导演的位子上，他有很多值得骄傲的地方。作为 Jelsoft 不可或缺的一部分，以及 vBulletin 的创建者，John 已经将一个梦想变成了一个令人惊叹的现实。

世界上许多网站——事实上有数千个——使用 vBulletin。它功能丰富，用户友好，可扩展性强，可以说是当今市场上最好的 PHP/MySQL 论坛软件。

vBulletin 是如何产生的？为什么被创造出来？未来有什么计划？约翰·珀西瓦尔揭开了几乎所有…

首先，你能介绍一下你自己吗？你是如何进入软件开发的？在你开始研究 vBulletin 之前你在做什么？

当然可以。我是约翰·珀西瓦尔，我已经为 Jelsoft 工作了 5 年。我最初是为了帮助创始人兼合伙人詹姆斯·利姆(James Limm)运营一个面向 Visual Basic 开发人员的网站。这包括为网站写文章、代码片段和评论，以及保持我们的问答论坛活跃。我一直这样做，直到 2000 年 2 月，我们发现现有的留言板不够用，于是决定用 PHP/MySQL 开发一些新的留言板。

最初是什么激发了你写 vBulletin 的灵感？你花了多长时间拼凑出第一个版本？

正如我提到的，我们现有的留言板在压力下无法支撑，詹姆斯和我都不熟悉留言板的代码，所以我们无法修改它以将其与网站集成。我们决定用 PHP 开发一个全新的系统，这样我们就可以将它与 VB 世界网站的其他部分更紧密地集成在一起。

我已经用 PHP 为 VB-World 编写了一个类似的内容管理和反馈系统，在构建后来的 vBulletin 时，我已经有了很多代码。我们的新论坛很快引起了公告牌世界的兴趣，我们被说服进一步开发并发布它。从第一次决定编写软件到第一个版本发布的总时间大约是 2 个月。

**SP:你为什么挑 PHP 和 MySQL？你还考虑过哪些组合？**

我选择 PHP 和 MySQL 的主要原因是因为我知道如何使用它们。我没有太多的 Perl、C ++或 Java 经验，所以它们不是真正的选择。我以前曾使用 PHP/MySQL 组合编写过 VB-World 的内容管理系统，所以我相信他们会完成任务。

此外，PHP 和 MySQL 有非常活跃的开发社区，所以我们保证它们在可预见的将来会得到支持和开发。最终，我们没有认真考虑任何其他选择，因为 PHP/MySQL 是当时最适合我们的——直到开发的主要部分完成后，我们才考虑向公众发布它。

你能告诉我们一些关于 vBulletin 的编程环境吗——包括物理环境(你的办公室)和屏幕环境？

当然可以。我倾向于在相当安静的环境中工作，尽可能少分心。背景音乐很好，虽然我经常过于集中注意力，以至于没有注意到 CD 结束:-)在我最初开发 vBulletin 的时候，我往往工作到很晚，每次连续工作几个小时，然后休息一段时间，睡上一觉。我发现在固定的、集中的时间里，我比在这里或那里的小片段里更有效率。

vBulletin 的大部分设计工作都是在电脑上完成的，但我总是在手边准备一支笔/铅笔和纸，以防我需要做笔记或画草图。大部分开发工作都是使用 TextPad (textpad.com)完成的，我现在对我所有的 PHP 工作都深信不疑。这是一个优秀的，方便的小文本编辑器，做我需要的一切，没有过于复杂，过于漂亮，或笨重。

测试工作是在我的本地机器上完成的，Apache 和 MySQL 在 Win98(现在是 Win2k)下运行。PHP 的优点是不需要昂贵的开发工具——只需要一个简单的文本编辑器，PHP/MySQL/Apache 都是免费的，所以这是一个非常低成本的解决方案。

当你推出 vBulletin 时，UBB 是领导者。你是如何将 vBulletin 引入由 UBB 主导的市场的？你有没有看到过自己真的在和他们竞争，并且变得比他们更好？

当我第一次开发 vBulletin 时，它只是一个在 VB-World 上使用的个人项目。然而，一些 UBB 用户担心他们软件的发展方向，并对我们提供的新鲜空气感到满意。他们与我取得了联系，并说服我，如果我们发布它，它可能会有市场。这就是我们所做的。这样做我们不会有什么损失，而且我们非常惊讶它发展得如此之快。它一炮而红，很快就赢得了一大批直言不讳的忠实用户。

这些人和“黑客”社区为 vBulletin 的成功做出了巨大贡献。最初的一小段时间，我们没有营销预算，只是通过口口相传来推广。我们从未想过自己会成为市场领导者(目前还没有——我将把这个决定留给独立读者)，成功的速度几次让我们大吃一惊。如此快速的扩张是一个挑战，但我们已经接受了，而且我认为我们已经成功了。

你认为 vBulletin 的成功归功于什么？

有很多事情我可以立即指出，但我认为最重要的是人:我们的客户在帮助我们改进软件方面发挥了宝贵的作用，他们不断提供建议和错误报告，并通过口头传播有关 vBulletin 的信息。此外,“黑客”社区对 vBulletin 来说是一个巨大的奖励。

这些人对普通的 vBulletin 软件进行代码修改，以添加新功能或改进现有功能。vBulletin 发布时附带了完整的源代码，因此客户可以自由地获取代码，并根据自己的喜好对其进行修改。人们非常喜欢这一点，因为他们可以定制它以适应他们的情况，而不是必须使用一刀切的解决方案。尽管如此，我们已经尽最大努力创造最全面的解决方案。

vBulletin 2 是在不到一年前发布的，这使得 vBulletin 更安全、更稳定，并大大增加了我们的功能库。vBulletin 的模板系统也大受欢迎——特别是能够从在线 Web 控制面板中修改论坛的完整外观，如果用户愿意，还可以使用 HTML。

vBulletin 有大量的管理工具，使它更容易维护，也为用户提供了许多功能，这使它在将社区聚集在一起和将网站的焦点集中在论坛上时很受欢迎。用户喜欢这种体验，所以管理员也喜欢，所以他们会告诉其他人——我认为这是 vBulletin 成功的关键。

SP:论坛软件已经涌入市场，人们对开源软件的兴趣也一直在增长。在未来，你们会继续向用户收取许可费吗？或者随着市场竞争的加剧，你们会寻找其他创收方式吗？

vBulletin 比开源项目有巨大的优势，因为我们可以付费给人们提供支持。因此，如果您购买了许可证，我们保证会尽我们所能为您提供支持。这可能包括登录到人们的服务器，通过电子邮件提供帮助，以及与服务器管理员联系以提高性能。

尽管市场竞争越来越激烈，但 vBulletin 仍继续受到欢迎。我们没有坐以待毙，而是继续 vBulletin 3 的激动人心的发展。

从业余爱好者到大型企业，网络上有各种各样的网站。vBulletin 如何适应这些不同类别客户的网站？

从技术角度来看，如果硬件合适，vBulletin 可以处理各种规模的网站，从最小的公告板到最大的社区。我们有几个客户，他们的数据库中有数百万个帖子，所以不存在缩放问题。这在普通的硬件上也是可以实现的。

我们的低价使该产品对中小型网站非常有吸引力。我们还致力于为我们的企业级客户提供更多的解决方案，包括更多的支持选项和对其他企业级数据库引擎的支持。

我知道你正在为 3.0 版本重写 vBulletin。您刚刚重写了 vBulletin 2.0 的核心引擎，为什么还要重写呢？为了提高 vBulletin 在版本 2 中的性能，您做出了哪些牺牲？

vBulletin 3 不会像 vBulletin 2 那样完全重写。我们正在看我们在创造 vBulletin 2 中学到的东西——我们将保留最好的部分并使它们变得更好，并将不太好的部分重新加工。代码的设计考虑了性能和可伸缩性，同时尽可能保持模块化。为了提高性能，我们所做的唯一牺牲就是开发人员的时间——如果要包含一个特性，我们要确保在它上面花费额外的时间，确保它能在所有不同的条件下运行。

我们能够对 vBulletin 2 进行的最大改进是 SQL 查询——这是 vBulletin 和 PHP 与数据库的接口方式。我们能够改进这些查询的方式，以及系统可以处理的查询数量，这样代码就快了很多。

如果 PHP/MySQL 开发人员想学习如何针对高性能条件优化他们的代码，比如 vBulletin 经常遇到的情况，您有什么推荐的资源吗？

MySQL 网站是个好地方，而[保罗·杜布瓦的 MySQL 书籍](http://www.amazon.com/exec/obidos/ASIN/0735709211/webmasterresou05)也是个不错的参考。然而，最好的体验是通过亲自尝试和修改代码来获得的。看看什么样的代码可行，什么不可行，通过尝试和改进来学习。你很快就会开始学会一些技巧。我们在为各种环境开发 vBulletin 的过程中学到的一件事就是尽可能减少页面上的查询。

在整合用户要求的功能和优化产品性能之间，你如何划分界限？

用户请求的特性通常根据它们的受欢迎程度来包含。如果一个特性被证明是非常受欢迎的，我们将开始寻找将它包含在 vBulletin 代码中的方法，这种方法不会显著影响性能。

几乎总是有几种不同的方法来实现一个特性，我们总是寻找性能最好的方法，而不是最容易编程的方法。如果有可能人们不想要某个特定的功能，或者对某些人来说它可能有点太耗费资源，我们就加入一个选项，让董事会的管理员有最终的选择权。

你能和我们分享一下 vBulletin 3.0 中一些尚未公开的新特性吗？

我们现在没有透露太多，因为很多还在进行中，所以我们还没有完整的细节。从管理员的角度来看，控制面板正在修改，包括一个更好的模板系统，和一个更全面的安全/权限系统。从最终用户的角度来看，界面将得到很大的改进，可以选择消息的线程视图。用户控制面板也得到增强，但我们还不能说更多！

你永远不知道，但不久你可能会发现 vBulletin 驱动你的整个网站，而不仅仅是你的论坛！；-)

SP:一个小问题:同时使用 vBulletin 系统的最大用户数量是多少？完成这一壮举需要什么样的硬件？据你所知，运行 vBulletin 的最大社区是哪个？

不幸的是，我不能提到我们客户的名字或网址。然而，我们看到有超过 1100 个用户同时在线的论坛，另一个有超过 170，000 个注册成员，另一个有超过 300 万个帖子。所以它应该可以处理大多数日常论坛而不会有太多麻烦！

不过，要启动这么大的论坛，确实需要相当大的服务器设置。我认为他们可能有两台服务器，一台用于数据库，一台用于 Web 服务器。每台可能都是双 P3 1Ghz 的机器，2GB 内存，SCSI 硬盘。这只是一个猜测，但是随着服务器成本的下降，对于一个大型网站来说，这样大小的服务器并不是不合理的。

**SP:vBulletin 的未来有什么计划？你会坚持使用留言板和相关系统吗，或者有计划扩展到其他产品或服务吗？**

我想我在其他问题中已经给出了一些提示，所以回去再读一遍！我们计划在不久的将来继续使用网站开发工具，与我们目前所做的类似。在过去的几周里，我们一直在与 ISP 进行一些交易，所以请关注 vBulletin 网站，很快就会发布公告和更多详细信息！

vBulletin 将继续积极开发，我们将努力保持技术的前沿，确保我们的客户得到充分的支持，并以低成本向他们提供最全面的公告板。
