# @ GDD 07:Google Gears——让服务器和客户端更加紧密

> 原文:[https://www . site point . com/GDD 07-Google-gears-bring-server-client-even-closer/](https://www.sitepoint.com/gdd07-google-gears-bringing-server-client-even-closer/)

这篇文章来自澳大利亚悉尼的[谷歌开发者日](http://code.google.com/events/developerday/au-sessions.html)，Aaron Boodman 刚刚完成了他关于谷歌对网络应用程序的新增强功能“[谷歌齿轮](http://gears.google.com/)”的演讲。由于悉尼在地球的右边，你首先在这里看到了它！

Google Gears 的目标是让丰富的、大量使用 AJAX 的 web 应用程序在线上和离线时都变得完全可用。交付这种能力将是谷歌成为主导应用平台的愿望的重要一步。

为了解决这个问题，Google Gears 将本地数据库存储引入到您的 web 浏览器中，并提供离线同步服务来保持本地和远程数据同步。

所有的数据交互都应该在本地进行，这有助于提高用户界面的速度，同时只要有连接，后台进程就会处理同步任务。

在存储方面，Gears 将一个 [SQLite](http://www.sqlite.org/) 数据库服务器嵌入到一个浏览器扩展中，提供 API 挂钩来查询和更新数据库，就像使用任何其他数据库一样。

Aaron 对同步方面的细节有些轻描淡写，这很不幸，因为这是离线< ->在线问题的症结所在。任何试图将各种日历和地址簿与手机同步的人都可以证明同步技术的变幻莫测。很难。

想象一下让您的网上银行应用程序离线的复杂性。您会信任 Firefox 扩展中的 SQL db 来保存等待与您最喜欢的金融机构同步的支付细节吗？

尽管这个目标很高，但谷歌采取了非常开放的方式来鼓励采用谷歌 Gears 作为开放标准。如果这项技术要起飞，网络浏览器制造商的全面采用和网络应用程序开发者的支持是至关重要的。代码是完全开源的，在 BSD 许可下，现在[就可以获得](http://gears.google.com/)。
