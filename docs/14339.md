# 现代网页设计的四大要素

> 原文:[https://www.sitepoint.com/essentials-modern-web-design/](https://www.sitepoint.com/essentials-modern-web-design/)

网络是一种年轻的媒介:它只有大约 10 年的历史。考虑一下 10 岁时的电视或电影。那时候，这些媒体还很原始，还在摸索中。他们正在探索新技术的极限，但巨大的发展尚未实现。

如今，网络专业人士也处于同样的境地。我们是先锋，仍在探索新媒体的极限。毫无疑问，我们的子孙后代在回顾我们所做的事情时会觉得这很有趣。关于网络能做什么，以及如何使用它，我们还有很多有待发现。

进入:会议如 [Web Essentials 04](http://www.we04.com) 。

Web Essentials 04 是一个为期两天的会议，于 2004 年 9 月 30 日至 10 月 1 日在澳大利亚悉尼举行，会议详细探讨了 Web 的基本支柱。演讲者包括[戴夫·谢伊](http://www.mezzoblue.com)、[道格拉斯·鲍曼](http://www.stopdesign.com)和[拉斯·威克利](http://www.maxdesign.com.au)，仅举几例。

会议关注的四个基本做法是:

1.  有效代码(符合标准)

3.  可访问代码

5.  语义正确的代码

7.  内容和表现的分离

在本文中，我将讨论这四个问题的具体细节，因为它们是由不同的演讲者提出的。

第一个重要的区别是，拥有有效的代码本身并不一定能让你的网站具有可访问性；内容与表现的分离也是如此。这四点中的每一点都是谜题的一部分，但没有一点本身是完整的解决方案。现在，让我们来看一下关于这些要点都说了些什么。

1.  **Valid Code**

    大多数制成品都是按照某种标准制造的。你会买一辆不符合设计规则的车吗？你会购买不符合当地建筑规范的房子吗？不符合基本安全标准的烤面包机呢？

    毫无疑问:标准的存在是有原因的，忽视它们是不专业的。用 W3C 的 HTML 验证服务来验证你的页面应该是你质量保证的一个标准部分。在现实世界中，会有一些无法验证的页面示例。

    主要的一点是:如果你的页面没有通过验证，那应该是因为别的原因，而不是因为你懒得去尝试。

3.  **Accessible Code**

    除非你听说过一个网站是通过像 [JAWS](http://www.freedomscientific.com/fs_products/software_jaws.asp) 这样的屏幕阅读器呈现的，否则你真的不知道网站对使用辅助设备的人来说有多难。通常，可能导致问题的是一些基本要素:

    1.  元素应该包含特定页面的描述，而不仅仅是公司名称。

    3.  页面上的`<h1>`元素也应该这样做。

    5.  确保你能用键盘浏览你的网站。如果你有需要“onclick”或“onhover”的导航，确保你也包含一个标准的`<a href>`链接。

    7.  类似地，“选择并运行”菜单也不可能通过屏幕阅读器使用。如果你必须使用组合框导航，使用一个单独的“Go”提交按钮来激活选择。

    9.  对所有图片使用替换文字，对表格使用摘要文字。表格摘要很像图片的替代文本。用它向看不见的人描述表格，而不是向能看见的人展示工具提示。工具提示风格的注释应该用在图像(和其他元素)的标题和表格的标题中。当我们讨论表格时，确保使用`<th>`元素，因为屏幕阅读器可以导航这些元素。

    11.  使用高对比度的颜色来适应其他视觉障碍。

    基本的可访问性并不难实现，但是可以避免一些用户的巨大痛苦。这是关于成为一个好的网络公民，为网络的“全球化”做出积极的贡献。

5.  **Semantically Correct Code**

    这个词经常被使用。但这意味着什么呢？

    语义学研究符号和标志之间的关系以及它们所代表的意义。

    要把它翻译成英语，你使用的`(X)HTML`元素应该描述它们包含的内容。你的(X)HTML 标记应该解释文档本身，而不是它的外观。

    同样重要的是要注意收益递减规律确实适用于检查你的语义。弄清楚几个最基本的方面，你就能极大地提高你的文档的意义。然而，你也可以越过这一点，花几个小时来实现微小的改进！

    你应该注意什么样的事情？

    1.  `<title>`元素。描述特定的页面，而不是整个网站使用同一个标题。

    3.  `<h1>`到`<h6>`应该用来把你的文档“分块”成有意义的小块。宣布以下段落将涉及的主题。

    5.  列表是对相关信息进行分组的好方法。即使您不希望您的演示看起来像一个列表，也要考虑这些信息是否适合该演示，如果需要，以后可以使用 CSS 来更改它的外观。

    7.  对于图像，使用 ALT 向看不见的人解释图像，使用 TITLE 向看得见的人提供工具提示。

    9.  桌子——每个人的最爱！使用表格进行布局当然已经过时了(而且有充分的理由！)，但是表格对于表格数据的显示仍然是完全有效的。表格有助于建立信息片段之间的关系，所以如果使用表格有意义，就不要使用浮动的元素！

    *   用 SUMMARY 向看不见的人解释表格，用 CAPTION 向看得见的人解释。
    *   `<thead>`、`<tfoot>`和`<tbody>`帮助解释表格的部分内容，并为后面的样式提供良好的 CSS 挂钩。
    *   使用`<th>`而不是`<td>`来传达标题单元格的额外意义。
6.  将表单分组为有意义的块。使用`<fieldset>`和`<legend>`不仅能帮助那些有残疾的人，对那些没有残疾的人来说也是一大优势。使用`<label>`为表单元素添加标题，并帮助将文本与这些元素关联起来。

8.  `<div>`和`<span>`元素没有真正的语义，所以要谨慎使用。我不是说“不要用它们”，而是如果有另一个解释内容的元素，那就用它代替吧！

语义本身并不能为你赢得可访问性或认可——它们只是大局的一部分。语义正确的代码确实提供了一个可靠的平台，可以在现在和将来的替代设备上使用。

*   **Separation of Content from Presentation**

    内容和表现的分离到底意味着什么？简而言之，它建议您的 HTML 不包含“表示”标记:没有描述外观的东西。所有“它看起来如何”的说明都应该是你的 CSS 的一部分。

    没有任何语言能比一个工作实例更清楚地描述这一点的重要性，因此, [CSS 禅宗花园](http://www.csszengarden.com)诞生了。探索一些设计，并且记住您一直在看完全相同的 XHTML 文档。

    抛开所有关于节省带宽、加快下载速度和提高可访问性的完全有效的论点，想想下次客户要求重新设计时，你的工作会变得多么简单。没有更多的重新编码整个网站！

    道格·鲍曼(Doug Bowman)解释了内容与表现分离的另一个好处，他讨论了自己重新设计[博客](http://www.blogger.com)的经历。一旦他和 Blogger 开发团队就基本的 XHTML 结构达成一致，他就可以在不干扰开发团队工作的情况下通过 CSS 文件来改进设计。设计师和开发人员可以独立地同时在网站上工作——这是将内容和表现分开的另一个令人信服的原因。

    当然，CMS 可能会妨碍一些人，但是如果你有能力将你的内容和表现分开，实际的好处远远超过最初的学习曲线。

##### 总结

采用 Web 开发的这四个基本支柱基本上就是采用最佳实践。如果你认真对待你的工作，那就是你应该去的地方。

即使你知道你没有任何残疾的潜在客户；即使你知道没有人会在除了 Internet Explorer 之外的任何地方查看你的站点；即使你喜欢收取巨额费用并花几个月时间做下一个重新设计的想法；考虑一下这个:

至少有一个盲人用户你需要适应。他的名字叫谷歌。他什么也看不见，但是忽视他是危险的。
