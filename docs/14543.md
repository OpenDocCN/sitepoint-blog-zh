# 从 Linux 命令行监控

> 原文:[https://www . sitepoint . com/monitoring-from-the-Linux-command-line/](https://www.sitepoint.com/monitoring-from-the-linux-command-line/)

系统管理可能是一项耗时且困难的工作，许多 web 开发人员专门将这些职责分配给员工或合同系统管理员，以确保服务器以最佳方式运行。

然而，并不是每一个网络公司在经济上已经准备好雇佣管理人员并自己处理这些任务。第二，不是每个网页设计者或开发者都有系统管理员的背景，并且经常是在飞行中学习。

命令行可能令人生畏，但在需要监控服务器的性能和控制服务器上正在运行的进程时，它也非常方便。对于那些通过 telnet 或安全外壳(SSH)访问服务器的人，下面列出了两个命令行功能，它们可以帮助您对 web 服务器进行日常监督。

监控性能

top 和 ps 命令提供了关于服务器上正在运行什么以及正在使用什么资源的有价值的信息。

top 工具(通过在命令行中键入 man top 可以获得大量信息)列出了正在运行的顶级进程以及它们正在使用的资源(CPU、RAM、空间)。该命令具有对结果进行排序的选项，并且是交互式的，不断更新。

ps 命令提供了正在运行的进程的列表，以及哪个用户启动了该进程或服务、资源使用情况以及该进程已经运行了多长时间。

我倾向于使用 ps -aux，它列出了所有用户正在运行的所有进程，包括那些没有终端的进程(例如，从服务器启动开始的守护进程)。

该列表包括一个进程 ID (PID ),如果您需要停止一个失控的进程或识别一个正在运行的未经授权的服务，它会很有帮助。

kill 命令使用 PID 来停止(或终止)进程。请注意，kill 异常中止了正在运行的进程，所以在使用它之前，请确保您希望暂停该进程或应用程序。传统上，你需要超级用户权限来发布 kill 命令(或者作为根用户)。

我已经包含了在我的 OS X 系统上运行的 top 命令的屏幕截图，作为所提供的统计数据的示例。当手动监控服务器很长一段时间时，我经常让终端开着 top running。
