# 结束求职——招聘者不会告诉你的 3 个秘密

> 原文：<https://www.sitepoint.com/secrets-recruiters-wont-tell/>

显然，IT 行业最近在伦敦过得很艰难……就此而言，在其他地方也是如此。自从众所周知的网络泡沫破裂以来，众所周知，工作机会越来越少，而想找工作的人却越来越多。

就在几天前，我在酒吧和一个很有才华的自由程序员聊天，他刚刚在伦敦金融中心的一家大银行找到了一份全职工作——在失业 6 个月后。

他告诉我，他和其他六个聪明、有才华的人坐在一起，等待这个职位的第五次面试，他们都没有工作过几个月(至少没有)。所有人都是自由职业者——就在一年前，这种人会被四处搜寻，也会礼貌地拒绝全职工作。

现在看来，伦敦正被无聊、失业的自由网络开发者和程序员所困扰。然而，我注意到并不是所有的自雇网络开发者都在谈论厄运和悲观。

在目前的环境下，我的几个朋友实际上做得更好。但是怎么做呢？是什么让他们成功，而其他人注定要在接下来的六个月里看白天的电视，吃烤豆子？

然而，在这些人前进的同时，无数的开发人员(他们比我年轻很多，并且已经掌握了大量的技术知识)不断进入市场——然而他们似乎没有得到工作。

这种情况让我思考什么才是真正的优秀 Web 开发人员。

##### 清单

我认为一个好的起点是看看我对一个优秀的 Web 开发人员的期望。

显然，良好的 HTML 和 Flash 技能是与生俱来的:我甚至不会考虑没有这些技能的人。此外，在另一个领域的一些经验是有用的，无论是数据库开发，XML，PHP…

但是当我真正想到和我一起工作过的人中谁是最好的，技术技能开始退居二线。

让我解释一下。我在一家中小规模的广告公司工作，这可能是开发人员面临的最苛刻的环境之一。截止日期很短，你必须快速完成并不总是理想的设计，并且成品的质量必须非常高。

这让我在招聘时寻找三样东西:速度、对细节的洞察力和幽默感(尽管我的一些更糟糕的同事可能会说，我不会根据这些标准招聘自己)。

你不会在很多招聘广告中发现这些要求，但是我向你保证，我不会雇佣不具备这三个条件的人。

***速度***

速度显而易见。我偶尔发现自己只有不到一天的时间来编写一个微型站点(对于那些现在问“什么？”—微型网站是我们为特定活动或产品建立的小型网站的名称。

不完成工作不是一个选项。这些微型网站通常与第一个广告通常在第二天出现在报纸上的新闻活动相吻合。

***准确度***

对细节的关注也非常重要。作为为一个活动制作一套材料的团队成员，每个人都有责任确保最终产品的质量。这样，每个人都会检查其他人的输入，不会有任何漏网之鱼。

当我创造“新媒体”材料时，我不仅仅是编码。我在看副本。我在几个不同的场景下检查。我要确保每一页上的每一个“t”都被划了叉，每一个“I”都被点了。我们不断创造高质量的图形材料，我们的工作标准非常高。如果当你转到下一页时，那个按钮跳了一个像素，那么*就与*无关了。我们不能就这样丢下它，希望没人注意到！

我想这是一件值得骄傲的事情:对自己工作的骄傲会带来好的工作。这真的很简单。你需要这样的心态，无论你在做什么，你都要尽可能地让它看起来更好。你必须保持这种心态，即使你暗地里对整个项目有点厌倦。

***幽默感***

最后，幽默感至关重要。这听起来可能是一个奇怪的要求，但我想我真正的意思是你必须有正确的心态。

这在很多领域都很重要。正如我已经说过的，我们的环境有时相当紧张，任何在严肃、高周转的广告公司工作过的人都会证明这一点(我不是指 Soho 区的一些小的、时髦的、有三个客户的机构——有争议，但在我看来这不能相提并论)。让我给你举几个例子，说明一个 Web 开发人员会遇到的高压情况。

首先，和设计师一起工作——不一定是网页设计师。与其他人一起为网站设计提供信息本身就是一种技能。通常很难取得平衡。正如一位艺术总监前几天对我说的，“我还没有遇到一个只会说‘是’的 Web 开发人员。”我想她是在开玩笑。但这是真的:我很少对每件事都说“是”，因为我必须让该死的东西工作。所以设计元素通常会被牺牲掉。

不被看作是一个只会说“不”的人的诀窍是尽可能保持设计的完整性，但永远不要走向另一个极端，对所有事情都说“是”。不要因为这是设计工作的唯一方式，就不得不使用那些肯定会疏远访问者的技术来建立网站。如果是这样的话，那么你一开始就不应该批准这个设计。

还有一个:现在是下午 4:30。一场活动明天开始。大约 12 小时后，第一份报纸可能会出现在人们的早餐桌上。你的合同是 9-5:30 工作，但你还远没有完成。你是回家还是留下来完成工作？

这么说吧:如果你回家了，明天早上你就不用来了。不要误解我，这种情况并不经常发生，但它确实会发生——如果你不得不通宵工作，那你就必须通宵工作。先不说别的，如果你是一个足够关心自己工作的人，那么你就是我想要雇佣的人。

或者这个怎么样？你已经完成了一个网站(第二次，由于客户改变主意)，你被问及是否可以“调整”背景颜色。你知道，这将涉及到在 Photoshop 中重做右手边所有漂亮的小图像，使它们与颜色变化保持一致。又一次，它发生了。

如你所见，幽默感真的很重要。

##### 当一切归结起来

许多 Web 开发人员发现自己失业了，尽管他们拥有出色的技术能力。这让我得出一个结论，尽管成为一名优秀的 Web 开发人员有一半是技术能力，但如果没有其他方面——灵活性、个性、速度和对细节的关注——你就不会做得很好。

毕竟，我们现在处于一个饱和的市场。任何人都可以得到一本关于 HTML 的书，并学习如何去做(他们越来越多地这样做——有多少次我看到有人在地铁上读“傻瓜的 HTML ”?).因此，你的编程技能有多好越来越不重要，更重要的是你能多好地融入一个团队。

如果我可以在基本技能和正确的心态之间做出选择，或者在无懈可击的技术简历和客户名单以及灵活性之间做出选择，我每次都会选择前者。
