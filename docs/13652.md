# 将微软技术与 ColdFusion 结合起来

> 原文：<https://www.sitepoint.com/uniting-microsoft-technologies-with-coldfusion/>

周五我有机会参加了 CFUnited。前几年我可能不会参加这个活动(除了一些聚会)，但是今年情况发生了很大的变化。在一次重大的策略转变中，微软在大会上占了很大的比重，并举行了一些有趣的会议。我在那里的一些重要经历:

**Atlas + ColdFusion = Neato**

Atlas 是一个非常强大的 Ajax 框架，通常与 ASP.NET 相关联。但是 [Brad Adams](http://blogs.msdn.com/brada/default.aspx) 主题演讲，[用 Atlas](http://www.cfunited.com/topics.cfm#603) 开发更好的用户体验，展示了如何将 Atlas 的 JavaScript 库与 ColdFusion(或任何其他服务器端技术)结合使用。现在，我不是一个 CF 的家伙，但[西奥拉什](http://therush.mxdj.com/)是。他[参加了会议](http://therush.mxdj.com/will_atlas_rule_the_ajax_world.htm)并认为这种互连有一定的潜力。此外，Brad 非常友好地张贴了幻灯片和示例应用程序。[查看此贴了解详情](http://blogs.msdn.com/brada/archive/2006/06/29/649944.aspx)。

IIS7:阿帕奇黑仔？

IIS6 是一个非常非常可靠的 web 服务器。但是它也不是没有问题。其中一个最普遍的问题是，配置通常要求用户对服务器所在的机器拥有 root 访问权限，这在许多共享主机场景中并不是一个真正的选项。该系统的另一个缺陷是它不是最具可扩展性的包。是的，你可以编写 ISAPI 扩展和过滤器，但是我认识的所有编写过的人都不想再走那条路了。我将在下面讨论可扩展性，但首先是配置。

微软新的网络应用服务器 IIS7 的端到端概述，Drew Robbins 解释了 IIS 的下一版本将如何解决这些问题。与以前版本的 IIS 截然不同的是，用户可以在一个人类可以理解的 Xml 文件中操作配置。此外，用户可以用每个子站点的 web.config 文件覆盖单个站点中的 web 服务器配置设置。不要担心服务器管理员—您可以指定哪些设置(如果有的话)可以被覆盖到单个节点和属性。而且，如果不想钻研 Xml，微软还提供了一个图形界面来管理服务器。

现在，任何已经部署了任何重要的 spent 应用程序(或者，想想看，任何对 IIS 有意义的东西)的人，可能已经花了相当多的时间来跟踪开发环境和登台/部署环境之间的配置差异。经常在拔头发的时候，还可能酗酒。有了 IIS7，这种情况将基本消失，因为人们可以将配置设置嵌入到应用程序的配置中。就像人们如何使用？htaccess 文件为 LAMP 应用程序调整 apache。

这些新特性似乎直接针对 Apache 的一些主要优势——具有基于文本的、可继承的配置文件，以简化维护和特定于应用程序的配置问题。同时也涵盖了最大的弱点之一——需要理解冗长的 httpd.conf 文件的血淋淋的细节才能对服务器做更多的事情。

**IIS7:吻别我的 ISAPI】**

另一个大的新特性是不需要钻研 C++来扩展服务器。IIS 7 上的任何站点或虚拟文件夹都可以通过编写一个实现 IHttpModule 的类并将其正确注册到服务器来扩展。我可以看到它的一些巧妙用途，比如轻松地在静态网站上实现访问者跟踪。

这并不是说以前存在的、功能性的 ISAPI 扩展将不再被支持——它们显然会被支持。今天的整个演示是使用当前 ISAPI 扩展在 IIS 上运行的 ColdFusion 进行的。

IIS7:弗雷布是我的好朋友。你的也是。

可能最简洁的特性是内置的失败请求事件缓冲区(FREB)。它允许管理员(或开发人员)定义各种标准，比如 Http 响应代码和页面执行时间，并记录关于这些失败请求的大量信息。有一个非常简洁的特性。这个事件缓冲区也可以记录你的自定义跟踪事件。我可以看到，当您调试某些只用于生产的竞争条件时，这个特性非常方便。

FREB 就像 IIS7 中的所有东西一样，转储到一个 Xml 文件中，以便于解析成您用来分析或跟踪这类东西的任何东西。

**嘿！我想玩 IIS7**

运行 IIS7 需要 Windows Vista Beta 2。不幸的是，预览计划现在已经关闭，所以如果你没有注册，你就不走运了，至少目前是这样。

要了解 IIS7(或 IIS 的所有内容)的最新信息，请查看[IIS.net](http://www.iis.net/)。
