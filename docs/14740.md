# IT 认证:好处和陷阱

> 原文:[https://www.sitepoint.com/it-industry-certification/](https://www.sitepoint.com/it-industry-certification/)

你是否在考虑通过 IT 认证来帮助你找到工作？也许你认识的某个人获得了一两个证书，并设法找到了一份理想的工作。或者，也许你的一个朋友完成了认证，但还没有得到工作面试的机会。在这篇文章中，我们将探讨如果你想站在正确的一边，你应该考虑的问题。

获得 IT 认证听起来很明智。你得到了市场上最热门的认证，从衣柜里找出那条幸运领带，让面试小组大吃一惊，你就准备好了，对吗？如果运气站在你这边——非常好——这可能行得通。然而，实际上，事情远不止如此。

解决，甚至简单地承认一些基本问题将会增加你成功的机会，并消除你对 IT 认证的任何误解。让我们深入思考一个常见的问题，如果思考不当，可能会让你从一开始就跌倒。

##### 哪种认证适合你？

你是一个程序员、网络员还是网页设计师？例如，即使你知道自己是计算机编程类型的，你需要的是 Sun Microsystems 的 Java 认证还是微软的？网认证？

如果你已经做了这个决定，你知道在你所在的地区有哪些公司雇佣具备这些技能的人吗？你准备好搬家了吗？花时间思考这些问题的答案，可能需要你暂时推迟对认证的追求。但是深思熟虑总比陷入可怕的“当时看起来很酷”综合症要好。

令人惊讶的是，有这么多的人投身其中，开始追求他们认为最受欢迎的认证。受欢迎程度不一定能很好地表明证书对你有多大用处。事实上，一些证书变得如此受欢迎，以至于它们开始失去曾经拥有的“让你与众不同”的品质。但是，即使你决定参加这些“风靡一时”的认证之一，仍然有可能通过正确的认证组合让自己脱颖而出——稍后会有更多的介绍。

##### 认证有影响力吗？

为了让一个证书开始值得你付出汗水，它必须被你希望给人留下深刻印象的人所认可，并对他们有用。如果最终你会发现自己在一个面试室里和一群 UNIX 人在一起，那么为微软认证工作就没有什么意义了。当然，你对你想为之工作的人了解得越多，就越容易发现一个证书有多大的影响力。

打几个电话给潜在的雇主和招聘机构，感受一下你选择的证书在业内的认知度。例如，我的“Sun Java 认证”得到了广泛的认可和尊重——它甚至是一些工作的要求。但是我的“Macromedia Coldfusion 开发人员”证书实际上没有任何分量。据我所知，至少有两个经常雇佣 coldfusion 类型员工的雇主甚至不知道它的存在！作为“Macromedia 认证培训师”的一部分，我需要这个认证，以防你想知道我为什么要这么麻烦。

##### 认证与胜任

如果你获得了认证，这是否意味着你将真正有能力完成你的工作？如果你还没有这种想法，那就尽快吧——这是一个重要的考虑因素，也是另一个潜在绊脚石的基础。

不要认为“认证”和“胜任”是同一个意思。现实情况是，大多数认证只是设定了一个基本的能力标准，并证明你已经达到了这个标准。如果你通过了考试，自动获得了从事相关工作的能力，把它当作一个额外的奖励——这并不总是认证工作的方式。

你在认证过程中的能力在很大程度上取决于培训供应商或组织设定的能力标准有多高。例如，我通过了“Sun Microsystems 认证 Java 程序员”考试，这是一个真正的挑战——我绝对必须了解我的东西。相反，我也参加了“Macromedia Certified cold fusion Developer”考试，我和坐在我旁边的两只黑猩猩都顺利通过了考试。

低能力标准不仅让人们在没有真正胜任的情况下勉强度日；它们还降低了资格的声望，最终降低了资格的有用性和可信度。不幸的是，有很多我认为属于这一类的认证。一点研究应该可以剔除无用的东西。

##### 保持专注！

通常，人们很难获得认证，但一旦他们获得认证，就会意识到世界基本上是一样的。工作机会都在哪里？

在这一点上可以犯更多的错误。人们很容易丧失信心，放弃认证的想法。许多人有相反的反应——他们全力以赴去获得尽可能多的认证。这两种方法通常都不太理想。

你可能不想听，但很可能你需要追求额外的认证——一个认证是远远不够的。如果你决定寻找额外的资格证书，试着只考虑那些:

*   与您的原始认证密切相关
*   原始认证的专门版本
*   由不同供应商或组织支持的相同认证。

我看到的许多简历让我相信，它们的作者将认证视为某种数字游戏。不要落入这个陷阱！注重质量，而不是数量。是的，你可以忽略这个建议，尽可能在简历中添加更多的证书，但是如果你在简历中添加下面这句话，肯定会更容易节省你的时间、金钱和精力:“我不知道我想从生活中得到什么，”或者，“我什么都知道一点，但不是很多。”

另一方面，一个适当考虑的认证组合可以有效地让你与众不同。这不就是重点吗？

##### 当心“非理性供应商投入”综合症

这里还有一个需要注意的问题:“非理性供应商投入”综合症。当一个人开始相信他们的认证道路必须围绕一个供应商或组织时，这种令人讨厌的痛苦就发生了。

例如，我认识一个人，她打算参加所有 Macromedia 认证，这样她就有了“全套证书”有数不清的其他证书可供选择，但每一个都与她想要的工作类型更相关，这并不重要。我认识的另一个人彻底迷上了微软，尽管他不太确定自己是网络管理员、办公室专家、系统工程师、桌面技术员还是数据库管理员。一个非常聪明的人，他只是需要一个称之为家的地方。

注意，你选择的认证不是简单地宣传你对某个特定公司的忠诚；它应该宣传一套真正有凝聚力的技能。

##### 比赛呢？

为了便于讨论，我们假设你遵循了正确的认证途径，并以高分通过了认证。到目前为止，一切顺利。但是它真的能帮你找到工作吗？这是我们需要考虑竞争的地方。

你的行业认证与大学学位相比如何？用人单位更喜欢哪个？这些问题会经常出现，答案很大程度上取决于雇主和你想要的工作类型。有些雇主不会雇用你，除非你有学位，不管你有什么行业认证或多少年的经验。

另一方面，一些雇主更喜欢一系列紧密相关的行业认证，而不是大学学位。然而，以我的经验来看，当你陈述你追求基于行业的学位认证的理由时，很多人都愿意倾听。你最大的敌人当然是两者兼得的人。

根据你选择的领域，甚至可能没有涵盖你想学习的主题或技术的大学学位。毕竟，我们首先拥有行业认证的原因之一是填补“真实世界”和大学提供的东西之间的差距。所以你的研究可能会发现这是一个多余的问题。

##### 自定进度学习还是正规培训？

另一点需要注意的是，一些雇主只接受通过注册培训机构获得的行业认证。

我曾经处于 Java 程序员证书变得毫无用处的境地。一个特殊的雇主不喜欢我自己准备并通过了考试，没有在培训机构定期上课。当然，我一点也不喜欢这个——我认为我的蒸汽质量非常高！

然而，这个事实确实支持了我之前提出的一个观点——证书并不能证明你可以做一份工作。有时候，雇主也想知道你是否接受过某种形式的正式培训。据我所知——我也问过周围的人——这不是经常出现的事情。即便如此，如果你的蒸汽受到挑战，也要准备好东山再起！

##### 经验

还有一个小的经验问题需要考虑。通常，如果你没有经验来支持它，一个证书几乎没有它所写的纸的价值。你可能需要寻求一些创造性的方法来获得经验——即使这意味着极其廉价或免费的工作。而且，如果你没有准备好不择手段地获取经验，我建议你可能一开始就在认证上浪费了时间和精力。

IT 认证已经成为一项庞大而复杂的业务，因为有太多的认证可供选择。研究是成功的关键。
