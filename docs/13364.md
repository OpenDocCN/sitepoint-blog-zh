# Twitter 为用户测试招募会员

> 原文:[https://www . site point . com/Twitter-recruiting-members-for-user-testing/](https://www.sitepoint.com/twitter-recruiting-members-for-user-testing/)

不管喜欢还是讨厌， [Twitter](http://twitter.com) 已经在[在线社交网络](http://en.wikipedia.org/wiki/Social_network#Social_networking_.2F_Internet_social_networks)空间引起了轰动。

简而言之，Twitter 允许用户通过大量的[小部件、混搭和插件](http://franticindustries.com/blog/2007/04/04/all-twitter-tools-and-mashups-in-one-place/)向 intarweb 发送无限量的(通常是)自我驱动的噪音，这使得事情变得更加容易。看看用于 Mac 的[Twitter fic](http://iconfactory.com/software/twitterrific)或用于 Windows 的[Twitter licious](http://www.ejecutive.co.uk/projects/twitterlicious)——这两个应用就是这么做的。

现在，让我对 2.0 应用和网站的泛滥感兴趣的一件事是[永久测试版](http://en.wikipedia.org/wiki/Perpetual_beta)的概念，“这个术语用来描述永远不会离开测试版开发阶段的软件或系统”。对我来说，这是一个非正式用户测试的完美例子，应用程序的设计团队能够直接从应用程序或通过网站上的反馈机制获得用户的反馈。

有趣的是， [Twitter](http://twitter.com) 刚刚宣布，他们正在招募已经成为[会员两个月或以上的用户参与一些用户测试](http://twitter.com/help/usertest)。

> 你可以通过参与用户测试来帮助我们改进 Twitter。您的反馈将直接影响未来功能的设计。

但是，在你兴奋得发邮件参与之前(像我一样:|)，“你必须能够自己提供去加州旧金山办公室的交通工具”。您还必须:

*   使用 Twitter 至少 2 个月了
*   超过 18 岁
*   我从未为 Twitter 的竞争对手工作过

因此，如果你有兴趣从用户的角度体验一些用户测试，并且住在旧金山(或者准备自己飞到那里)，到月底你很可能会有 50 美元的收入！
