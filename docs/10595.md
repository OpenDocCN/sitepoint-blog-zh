# PostgreSQL 9.0 发布

> 原文:[https://www.sitepoint.com/postgresql-9-features/](https://www.sitepoint.com/postgresql-9-features/)

PostgreSQL，或者它的朋友们称之为 Postgres，是 web 数据库世界的失败者。尽管 MySQL 是一个健壮的、功能齐全的、开源的关系数据库，并且拥有一个充满激情的社区，但是这个系统还是被 MySQL 所掩盖。MySQL 的高采用率几乎没有什么技术原因:它首先到达，并与 PHP 开发人员建立了强有力的联系。

然而，不管甲骨文的保证如何，MySQL 的未来似乎不太确定。PostgreSQL 没有单一的所有者，它非常适合利用这个数据库巨人带来的任何复杂性。

PostgreSQL 9.0 最近已经发布，其中有一些改进和新特性，可能会吸引更多的追随者:

*   热备用:用于只读查询的第二个备用数据库。它是通过重放主数据库的二进制日志文件创建的。
*   流式复制:辅助服务器可以随时连接到主服务器，并接收它们需要的记录。滞后时间很短，主服务器上的负载最小。
*   增强的存储过程支持
*   改进的权限管理
*   列触发器:仅在特定列被更新时触发的触发器。
*   当条件触发器
*   可推迟的惟一约束:现在可以批量更新惟一键
*   聚集中的分类
*   不必要的连接移除优化
*   匿名函数
*   增强的存储过程支持
*   解释 JSON、XML 或 YAML 格式的可用输出
*   64 位 Windows 二进制发行版
*   支持就地升级的新模块

雅虎、Skype、reddit.com 和 MySpace 等组织都使用 PostgreSQL。你会吗？

有关版本或数据库的更多信息，请参考[postgresql.org](http://www.postgresql.org/)。
