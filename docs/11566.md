# 你达到电子邮件帐户霸王了吗？

> 原文:[https://www.sitepoint.com/email-account-overload/](https://www.sitepoint.com/email-account-overload/)

有没有人开始忘记他们使用了多少个电子邮件账户？以前要容易得多。几年前，我有三个主要账户:一个是工作账户，一个是个人账户，还有一个垃圾邮件账户。

我现在至少有五个独立的主要账户。然后又有十几个账号转发了这些。最后，我已经记不清自己注册了多少个邮箱账户了。我曾经认为这些地址是“一次性”的，但是它们变得越来越重要。例如，任何使用谷歌系统(如 Docs、Analytics 和 AdWords)的人几乎肯定都会有一个 GMail 账户。

我怀疑这是主要由自由职业者和小企业主遇到的问题。从各种网站开发销售软件或服务的微商很常见。我也经常被客户要求成为他们在线系统的技术联系人(或者过去被称为*网站管理员*)。然而，我确信电子邮件超载也正在成为主流用户的一个令人沮丧的问题。

像微软 Outlook 或 Mozilla Thunderbird 这样的应用程序会有所帮助，我喜欢认为自己相当有条理。我所有收到的电子邮件都有颜色编码，或者被转发到一个合适的文件夹。垃圾邮件很少成为问题，因为它至少在三个级别上被过滤。最后，当任务完成后，我喜欢清空收件箱，删除邮件。但问题依然存在——一些网络邮件账户必须手动检查，我仍然有太多的电子邮件用于太多的目的！

简单的答案是取消一些账户，但这并不那么直接。通常有必要从一个特定的地址发送电子邮件，因此需要多个身份来展现专业形象，并且不会混淆收件人。

但是这是一个普遍的问题吗？是否需要一个解决方案？许多网络邮件服务提供电子邮件聚合功能，但很少提供离线客户端的配置选项，如 Outlook 或 Thunderbird。还有一个风险是，将你的电子邮件集中在网上会产生单点故障，你会屈服于第三方对隐私、安全和财务的担忧。

我想通过提供一个完美的解决方案来结束这篇文章，但我不相信有这样的解决方案存在。因此，我将这些问题抛给 SitePoint 的读者…

您监控多少个电子邮件帐户？你正遭受电子邮件超载的困扰吗？找到好的解决方法了吗？或者你采取了什么措施来纠正这个问题？
