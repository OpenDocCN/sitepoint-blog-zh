# HTML 电子邮件样板

> 原文：<https://www.sitepoint.com/html-email-boilerplate/>

Web 开发人员经常抱怨不得不支持五种主流浏览器、一些移动设备和某些应用程序的古怪行为。如果你认为我们过得不好，想想那些创建 HTML 电子邮件的人。他们必须应对:

*   几十个电子邮件客户端和版本。
*   多个渲染引擎。
*   没有 HTML 电子邮件标准。
*   很少支持 CSS。
*   繁琐的测试程序。
*   某个流行的电子邮件客户端[放弃了浏览器渲染，转而使用文字处理引擎](/microsoft-breaks-html-email-rendering-in-outlook/)。

忘记样式表、浮动、负边距、定位、背景图像、动画 gif、png 或任何其他有趣的节省时间的技术。如果你认为在今天的 IE6 中做一个网站很难，HTML 邮件必须像 1998 年那样编码。这意味着表格、内嵌样式和图像。

幸运的是，Sean Powell 从 Paul Irish 的 [HTML5 样板](https://www.sitepoint.com/introduction-html5-boilerplate/)中获得了灵感，并创建了 [HTML 电子邮件样板](http://htmlemailboilerplate.com/)。它包括行业领导者发现的各种修复方法，如[竞选监督](http://www.campaignmonitor.com/)和 [MailChimp](http://www.mailchimp.com/) ，以产生一个防弹电子邮件模板，可以在 Outlook、GMail、Hotmail、Yahoo Mail 和其他流行的客户端上使用。

Sean 承认这不是即插即用——你需要亲自动手编写代码——但这是很好的第一步，解决了开发 HTML 电子邮件时遇到的许多常见问题。

HTML 电子邮件样板代码包括两个 HTML 文件:一个带有注释说明，另一个没有注释说明，可以作为电子邮件的基础。它是在 MIT 许可下发布的，是免费的，可以用于任何商业或非商业项目。

你会失去什么？请告诉我们你是否尝试过样板文件，更重要的是，它在你的电子邮件应用中是否有效。
