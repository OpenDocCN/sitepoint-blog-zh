# 快速分享代码的五大场所

> 原文:[https://www . site point . com/top-5-places-to-share-code-quickly/](https://www.sitepoint.com/top-5-places-to-share-code-quickly/)

协作开发或从你的同事那里获得在线开发帮助要求你与他人共享代码，但这并不总是可能的。如果你在 IRC 上，将巨大的代码块粘贴到公共聊天频道对其他人来说是很烦人的，通过即时消息或电子邮件分享代码是笨拙的，缓慢的，难以阅读的(因为这些工具缺乏代码格式和行号)。

这就是垃圾桶出现的原因。Pastebins 允许开发人员在可共享的、一次性的 URL 上发布代码片段，以便轻松地相互交换代码。有大量的 pastebin 站点，它们之间的差异通常很小。然而，下面列出了我们最容易快速共享代码的五个地方。没有特定的顺序。

## [馅饼](http://pastie.org/)

Pastie 不是目前功能最丰富的 pastebin 网站，也不支持大多数语言，但它是最优雅的网站之一。Pastie 在特色上的不足，在简单和美丽上得到了弥补。该网站提供私人粘贴，粘贴不会过期。更多功能丰富的代码粘贴，请查看 [Paste2](http://paste2.org/) 、[Pastebin.com](http://pastebin.com/)、 [Lodge It](http://paste.pocoo.org/) 、 [Pastebin.ca](http://pastebin.ca/) 和 [GnoPaste](http://nopaste.info/) 。

## [要旨](http://gist.github.com/)

Gist 是互联网上最强大的粘贴工具之一。它与其他垃圾桶的区别是什么？版本控制，私有粘贴的 SSL 安全，支持在其他站点嵌入粘贴，以及使用 Git 推送和拉取粘贴的能力怎么样？像母网站 GitHub 一样，Gist 上的公共贴子(毫不奇怪，叫做“gists”)可以被其他用户通过点击一个按钮来获取。

GitHub 的活跃用户社区也通过创建不断增长的扩展生态系统为 Gist 做出了贡献，例如[自动发布到 Tumblr](http://github.com/blog/137-gist-on-your-tumblr) 和 [Textmate 支持](http://github.com/blog/233-gist-support-for-textmate)。你甚至可以从 Gist[那里得到 Greasemonkey 脚本](http://github.com/blog/302-gist-for-greasemonkey)。非常酷。

## [TinyPaste](http://tinypaste.com/)

如果你渴望粘贴更加简洁明了，那么也许 [TinyPaste](http://tinypaste.com/) 适合你。TinyPaste 是一个准系统的 pastebin。输入你的密码就可以了。不要在名称、描述、选择语言等方面浪费时间。不幸的是，这也意味着语法高亮已经过时了。但是如果你只是想粘贴并离开，那么 TinyPaste 可能就是你要找的。该网站还有一个 Firefox 插件，允许你从任何文本源(在浏览器中)粘贴文本，并在不访问 TinyPaste 网站的情况下获得粘贴 URL，它还有一个[API](http://tinypaste.com/api/)。

另见 [Snipt](http://snipt.org/) ，它被宣传为 Twitter 的 pastebin。理论上，这意味着它的功能应该和 [TwitPic](http://www.twitpic.com/) 处理图像的功能差不多——不幸的是，现在它似乎只是一个普通的粘贴网站，带有粘贴的短 URL。希望他们能增加自动发布到 Twitter 的功能，以及通过@ replies 向特定用户发送贴子的功能。

## [以太网路径](http://etherpad.com/)

EtherPad 实际上并不是一个粘贴网站，但如果你粘贴的原因是因为你正在与合作者一起解决一个编码问题，那么它可能比这个列表中的任何其他网站都好。EtherPad 允许用户在文本文档上实时协作。每个用户都加入一个工作区，可以实时观察彼此的变化——如果用户 A 输入了新的内容，用户 B 会看到它实时出现在他的屏幕上。

因为 EtherPad 不是一个真正的 pastebin，它不支持许多语言的语法高亮显示，但是添加它应该是相当简单的。不过，它确实有行号，所以即使没有语法突出显示，它也应该适合以静态 pastebin 无法企及的方式协作解决编码问题。

## [Snipplr](http://snipplr.com/)

像 EtherPad 一样， [Snipplr](http://snipplr.com/) 也不是一个真正的 pastebin。相反，Snipplr 是一个公共代码片段库，也是最好的库之一。如果你想和更多的人分享你的代码，Snipplr 是一个很好的地方。这是一个设计非常好的片段网站，有超过 10，000 个片段。他们有一个 Textmate 捆绑包，一个 WordPress 插件，Firefox 和 Safari bookmarklets，甚至还有一个更加晦涩的 Gnome 文本编辑器 Gedit 的插件。

也可以看看另一个大型公共片段网站。
