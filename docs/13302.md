# 把你的苹果扔出窗外！

> 原文:[https://www.sitepoint.com/throw-your-mac-out-the-window/](https://www.sitepoint.com/throw-your-mac-out-the-window/)

在我尊敬的同事重述他为什么如此热爱他的 Mac 电脑之后，我想我应该提出一个不同的观点。

好吧，所以我实际上并不主张你把你的 Mac 扔出窗外。这东西还没有完全生物降解，你可以把它当作显示器支架。此外，可能会有人路过(除非那个人是史蒂夫·乔布斯，然后，无论如何——把它扔掉)。

但我确实认为，鉴于人们对 OS X 省时、直观和简单漂亮的赞誉有加，有必要探究一下 Mac 用户界面中明显出错的一些方面。我最大的不满是界面的一个非常糟糕的基本元素——键盘快捷键。

声明:我既是 Mac 用户，也是 PC 用户，已经很久了。这是我的牛肉。

在 Windows 上，应用程序中的键盘快捷键:

1.  提供全面覆盖
2.  很容易被发现
3.  是直觉的
4.  是一致的，而且
5.  一般不要踩到对方的脚趾

我在 OS X 的经历让我得出这样的结论:Mac 应用程序中的键盘快捷键不属于上述情况。让我通过查看一些流行应用程序中的常见命令来说明我的观点。

我们先来看看 Adobe Photoshop。

在我的电脑上，执行**图像** > **图像大小**命令相当快(只需三个键):

`ALT`–`CTRL`–`I`

Mac 上也有类似的快捷方式:

`Cmd`–`Option`–`I`

到目前为止，我们有一个相当公平的竞争环境。不过，就我个人而言，我不喜欢同时按住三个键，我更喜欢从键盘上执行这个命令的另一种方式，稍后我会谈到这一点。

那么，如果我想利用**图像**菜单上的其他项目呢？例如，**裁剪**功能(我在为我们的书准备截屏时经常使用的一个命令)。

如果我们看一下下拉菜单，您会发现在这两个平台上，都没有列出指定的键盘快捷键。在这种情况下，我们只能听任操作系统的摆布，让事情变得更简单。![A comparison of Photoshop's Image dropdown menu, as viewed in Windows and on a Mac](../Images/172135b327db75eca3655c74506bd488.png)

Windows 通过允许用户按住`ALT`键来参加聚会。这样会突出显示可用于触发每个菜单项的按键:

`ALT`–`I`，`P`

第一次`P`有点误导，因为人们会直觉地认为**裁剪**菜单项将由`C`执行。然而，由于`C`键已经被同一菜单上的另一个选项占用，这是一个相当优雅的妥协。因为当我们按住`ALT`的时候，这个键是带下划线的，所以我们一眼就能看出这个快捷方式是什么，并执行它，所有这些都在一个动作中完成。

对于 OS X 来说并非如此。

在 Mac 上，你有四个选项:

1.  使用操作系统提供的按键导航菜单(`Ctrl`–`F2`、`I`、`Enter`、`C`、`C`、`Enter`)
2.  在操作系统级别应用您自己的键盘快捷键
3.  录制一个宏来执行这个函数(对于这样一个简单的函数来说，这是大材小用)
4.  使用鼠标

考虑到属于这一类别的菜单项的数量，这些都不是特别可口。第一种选择对我来说更糟糕，因为我有一台 MacBook，由于笔记本电脑的紧凑键盘，我需要输入`Fn`–`Ctrl`–`F2`。不可能指望有人用一只手快速打出`Fn`–`Ctrl`–`F2`。

这让我想到了我的下一个问题:**冲突**。以 Photoshop 和 OS X 的内置文档搜索 Spotlight 为例。这两款应用都使用了`Cmd`–`Space`组合键:对于 Spotlight 来说，`Cmd`–`Space`是在桌面上搜索电子邮件、图片和文档的按键。在 Photoshop 中，它是放大文档的命令。**其实这是每个 Adobe 程序放大的命令！**

每当我想放大 Photoshop 图像、Illustrator 图形或 Reader 中的 PDF 时，Spotlight 搜索栏就会出现，这让我非常恼火。当然，有时它是可行的(如果你记得按住`Space`然后按下`Cmd`，你可以避免这种烦人的冲突)，你可以重新映射这些按键，但是**为什么 Adobe 和苹果还没有把它们作为默认选项呢？**

这让我想到了一致性。

我在 Mac 上使用最多的一些应用程序是 Java 应用程序。对于原生 Cocoa 应用程序，使用箭头键的`Cmd`和`Option`光标导航与 Java 应用程序中的光标导航是**完全不同的。这种导航文档或突出显示文本的基本模式并不一致，这一事实让我感到困惑，每次我启动 Java 应用程序时，我都会诅咒不得不重新编程我的思维模式，仅仅是为了选择文本或上下移动页面。我不知道这是 OS X 的错，还是孙的错，还是单个应用程序开发人员的错，但事实是，我在 Mac 上使用的每个 Java 应用程序都存在这个问题。**

是的，毫无疑问 OS X 很漂亮。很稳定。就是这样。除非你依赖键盘来提高工作效率——这对于像我这样的超级用户来说绝对是至关重要的。

键盘快捷键可以使应用程序对残疾用户更易访问，对健全用户更易使用。在这些赌注中，Windows 只用了几个灵巧的击键就把 Mac 的鼻子蹭进了土里，甚至连水银都救不了它([但阿尔弗雷德可以](https://www.sitepoint.com/holy-hacking-batman-create-alfred-workflows-in-ruby/))。
