# 创建触觉界面

> 原文：<https://www.sitepoint.com/creating-a-tactile-interface/>

随着过去几年新技术的出现，典型的网络界面已经发生了变化。对于任何 web 开发人员来说，这可能并不奇怪，但是许多人已经意识到，web 上可用技术的灵活性越来越大，这往往是一件令人头痛的事情，也是一件好事。我的团队最近接受了一项任务，要重建我们公司的主要产品，让它的功能不那么像一个网站，而更像一个 web 应用程序。

这当然看起来是个好主意。有成千上万的博客帖子在谈论它，成千上万的博客作者不会错。不幸的是，web 应用程序真的没有一个清晰的定义。变化太多了，很难确定是什么让一个网站看起来更像一个应用程序。

似乎大多数开发人员都专注于使用 AJAX 来改进数据处理和减少页面刷新。虽然这对于大多数网站来说绝对是一个很好的改进，但它并不能让网站看起来更像一个应用程序。应用程序仍然有加载时间；当您更改视图时，文件夹仍然会部分刷新，桌面用户可能已经习惯了。桌面应用程序不仅处理数据更快，它们通常比 web 应用程序更有触觉，但是桌面的触觉本质还没有很好地过渡到 web。

我们开始建立一个用户在桌面上熟悉的触觉界面机制的列表，并寻找将它们应用到我们的应用中的方法。经过长时间的讨论和大量的研究，我们提出了一个桌面应用程序中常见的界面特性列表，我们希望将这些特性转化到我们的 web 应用程序中。
 **1。**拖拽
最常见的鼠标技术之一就是拖拽。在网上已经有了很多解决方法，但是这确实是一个桌面的核心界面机制，在网上没有被充分利用。我们开始让元素可拖动，而不是从下拉列表中选择类别。这使得界面感觉更加舒适，并且使组织数据的过程更加容易。这个特性可以很容易地添加到任何符合类别、文件夹或其他组织/分类系统的数据中。这种想法也可以很好地转化为从标签云中选择多个标签或向搜索添加多个标准。一些 web 应用程序已经很好地实现了这一点。看看 Yahoo Pipes 和 Flickr 对拖放的一些真正伟大的应用。实现一般也不会太差。许多可用的 javascript 库都支持跨浏览器拖放。

**2。组选择**
在我们有了一些可拖动的元素后，我们意识到下一个逻辑步骤是创建选择选项。将项目拖动到文件夹或类别中并没有多大用处，除非你可以一次拖动多个项目。在桌面上，有几个选择多个项目的选项。您可以按住 shift 键点按、control 键点按或通过拖移项目周围的框来选择项目。javascript 可以处理 Shift 点击和 control 点击，但是要让它在不同的浏览器上都能很好地工作还需要一点努力。像许多关于鼠标事件的信息一样，shift 和 control 键的状态也是可用的，但是在每个浏览器中的不同位置。我们发现 http://www.javascripter.net/faq/ctrl_alt.htm.的这篇文章非常有用
 **3。可定制的界面**
我们决定添加到应用程序中的最后一样东西是一个更加灵活的界面。大多数桌面应用程序都有一定程度的界面定制。Photoshop CS3 是这种灵活性的完美例子。大多数菜单都有折叠模式，只显示图标，而不是整个菜单或面板。这是在不限制功能的情况下最大化屏幕空间的好方法。菜单和面板也可以移动和重新排列。这两个定制选项为更广泛的受众提供了更好的用户体验。初级用户可以打开菜单，使用最少数量的面板或功能。高级用户可以折叠项目，同时在屏幕上显示更多选项。

这三种技术可以让你的 web 应用程序更有触感。在很大程度上，这些都很容易用许多 JavaScript 库实现。我们使用 Scriptaculous，它支持拖放、可排序的元素和基本的 AJAX 工具，只需要很少的额外代码。
