# 少做

> 原文:[https://www.sitepoint.com/do-less/](https://www.sitepoint.com/do-less/)

所以我被说服为 Sitepoint 写博客，这是我的第一篇文章。我将主要写 PHP，因为这是我的主要能力范围，但也可能会涉及相关的主题。无论如何，为了不把这里变成另一个“Hello World ”,我想我应该思考一下 PHP 的特别之处。

## PHP 不做任何事情

基本上可以归结为一件事:PHP 不会为你做任何事情。

看一看 PHP 默认自带的大量扩展——或者大量的核心函数——看起来 PHP 做了很多事情，但事实并非如此。不完全是。这种功能大多是被动的；当你看语言核心时，它对你将要构建的应用程序做了很少的假设。相反，它提供了这样做的工具。

这就是开源的传统。Linux 以拥有众多桌面环境、shell 解释器等等而闻名。PHP 遵循同样的组织。虽然它只有一个支点。语言本身只有一个官方实现。嗯，实际上是两个，但是希望[最终能够修复](http://www.php.net/index.php#2007-07-13-1 "PHP 4 end of life announcement")。

## PHP 没有框架..

如今，应用程序框架风靡一时。简而言之，框架是关于如何编写应用程序的规则体，由一组支持这种特定方式的库提供支持。这些规则在构建应用程序时为您提供指导，并且这些库可以更好地集成，因为它们可以对您的应用程序做出假设。

Rails 是将它从一种时尚提升到炒作的人，但是 PHP 也有自己的份额。考虑到 PHP 扩展和库的多样性，不像 PHP 的大多数竞争对手，没有一个单一的定义框架是不足为奇的。这有点问题，因为框架往往是互斥的——至少在某种程度上是这样。

## ..也不会！

有些人可能会说，统治他们只是时间问题。我更相信，PHP 的独特本质就在于它的多样性。这种多样性是通过缺少与语言紧密集成的单一应用程序框架来保证的。

这并不是说，没有框架的空间，但它将永远是复数。这使得它们从根本上不同于 Ruby 或. NET。这是令人振奋还是令人沮丧是个人脾气的问题，但这是无可争议的。

这应该会引发一些评论。
