# 哇！Windy Wellington 的网络股票总结…

> 原文:[https://www . site point . com/wow-windy-wellingtons-web stock-wrap-up/](https://www.sitepoint.com/wow-windy-wellingtons-webstock-wrap-up/)

对于我们这些未能参加上周在新西兰举行的[网络股票会议](http://www.webstock.co.nz/)的人来说，界面设计[卢克·乌鲁布莱夫斯基](http://www.lukew.com/about/luke/)做了一件令人钦佩的工作，他在博客中记录了许多演讲的要点。

这里有一个选择:

*   杰森·圣玛利亚谈[好的设计不容易](http://www.lukew.com/ff/entry.asp?634)。Jason 思考了一个事实，与 Web 不同，印刷设计有固定的限制，如页面大小和不变的字体大小和布局。然而，这些限制是有利的，因为它们允许设计师利用黄金比例和“三分法”——在这种情况下，约束可以提供创造性的优势，而不是劣势。
*   **Molly Holzschlag** 谈[为什么网络标准没有](http://www.lukew.com/ff/entry.asp?630)。Molly 回顾了最近在墨尔本召开的 Web 标准小组会议上[讨论的一些问题——我们目前拥有的并不是一套真正的标准，而是一套浏览器供应商可以选择实施(或不实施)的最佳实践。W3C 规范只有在以完全透明的方式发展，并且结果没有歧义的情况下，才会成为真正的标准。](http://webstandardsgroup.org/meetings/index.cfm?event_id=148)
*   **彼得·莫维尔**谈[的环境发现能力](http://www.lukew.com/ff/entry.asp?635)。Peter 将 findability 描述为“随时随地找到任何人或任何东西的能力。”鉴于我们淹没在信息中，我们面临的挑战不是更大声地交流，而是让事情可以被发现(我喜欢这句名言“在一个更大的干草堆的世界里，我们如何制造更大的针？”)他的答案是:回归元数据，改进搜索，以及更加结构化的信息组织。
*   [JavaScript 的艺术&科学](https://www.sitepoint.com/books/jsdesign1/)合著者**西蒙·威廉森**关于 [OpenID 和去中心化社交网络](http://www.lukew.com/ff/entry.asp?636)。Simon 讨论了 OpenID 的用户体验、潜在问题，以及 OpenID 在构建分散式在线社交网络中的作用(Brian Suda 去年在 SitePoint 上讨论过这个话题)。

…还有更多来自演讲者，如**凯利·戈托**、**凯西·塞拉**和**丹·塞德霍尔姆**。鉴于会议上发言人的高水平，我希望播客即将推出。

谢谢卢克。
