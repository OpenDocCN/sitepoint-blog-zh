# 什么是开放网络？

> 原文:[https://www.sitepoint.com/just-what-is-the-open-web/](https://www.sitepoint.com/just-what-is-the-open-web/)

这是来自 AJAX 开发者布拉德·纽伯格和 AJAX 编辑迪翁·阿尔梅尔和 T4 的本周问题。“开放 Web”是一个在过去一年左右的时间里流传很广的术语，从语义上来说，它开始感觉很像 Web 2.0——一个每个人都在使用但没有人能真正定义的术语。

Neuberg [在四月亲自尝试了一下，用一个很长的定义将开放网络定义为“一套哲学”这些理念包括权力下放、透明度、可黑客性、开放性、支持礼品市场和传统自由市场、第三方整合、第三方创新、促进民间社会和对话、双向沟通以及最终用户可用性和整合。](http://codinginparadise.org/weblog/2008/04/whats-open-web-and-why-is-it-important.html)

如果这听起来很拗口，那是因为确实如此。Neuberg 定义这个概念的尝试已经走了很长一段路，以创建一个关于驱动开放网络的思想以及如何和何时应用该术语的具体定义，但它不是我们都渴望的简洁，容易重复的定义。本周早些时候，纽伯格为开放网络寻求一次电梯推介。他要求他的读者给出一个定义，而不是像他之前的定义那样罗列要点，只有几个句子长，并且不会限制未来网络的发展。清晰、简洁、紧凑、完整是开放网络需要的定义类型。

“我们反复讨论开放网络这个术语，但我从未见过它被简洁地定义。它是什么，我们为什么支持它？我相信这很重要，值得支持，但现在一切都有点模糊，”纽伯格写道。

在 2007 年的 SXSW 上，来自 Mozilla [的 Brendan Eich 用他自己的要点列表定义了开放网络](http://developer.mozilla.org/presentations/sxsw2007/the_open_web/):

*   不受阻碍的跨平台标准
*   开源/自由软件实现
*   没有单一供应商“锁定”
*   用户创新网络效应
*   分布式可扩展性

总的来说，艾希和纽伯格的清单定义肯定不够简洁。一个清晰、简洁的定义是很重要的，因为定义像开放网络这样的重要概念有助于巩固我们对未来以及如何实现未来的愿景。而开放的 Web *是*重要的。正如[凯文·丹古尔在四月份写的](http://www.sitepen.com/blog/2008/04/03/flash-silverlight-and-the-open-web/)，“最终，[开放网络]所有的能力都是为了确保网络以一种更好地服务于终端用户的方式发展。”

所以我们想把它交给你。在评论中，给我们你对开放网络的最好定义。
