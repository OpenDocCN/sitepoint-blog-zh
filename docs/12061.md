# FullCodePress 更新:CodeBlacks 押注 Beta CMS

> 原文:[https://www . site point . com/full code press-update-code blacks-bet-on-beta-CMS/](https://www.sitepoint.com/fullcodepress-update-codeblacks-bet-on-beta-cms/)

[![Tim Connor, developer for the CodeBlacks](../Images/9eb6a35cb522e3e2614161d5738fa699.png)](http://www.flickr.com/photos/fullcodepress/3525369628/) 在开玩笑说我会[破坏他们的互联网连接](http://www.fullcodepress.com/2009/04/30/fullcodepress-sponsor-interview-sitepoints-matt-magain/)并在整个活动中不断用糟糕的音乐分散他们的注意力后，我想我最好伸出橄榄枝，更好地了解一下新西兰 FullCodePress 团队。

我设法和 CodeBlacks 的程序员 Tim Connor(T1)聊了几分钟，向他询问了新西兰团队为他们的 FullCodePress 网站选择的 CMS。

首先，给我们讲讲 BrowserCMS。我从来没听说过。当我第一次在谷歌上搜索这个术语时，我只能找到一个 J2EE 产品的信息，附带 25，000 美元的许可费……这听起来不太适合你的非营利客户，是吧？

蒂姆:嗯， [BrowserCMS](http://browsercms.org) 是开源的。而且是免费的。它最初是 J2EE 的一个项目，最初是商业性的。建造它的公司认为 J2EE 不适合他们，他们想转向 Rails。所以他们这么做了，Rails 是一个开源项目，他们决定把整个项目都开源。

这是 BrowserCMS 的第三版吧？

没错。它实际上仍处于测试阶段。我想这年头很多事情都是这样，对吧？永久测试版！

那你经常玩它吗？

我必须承认，不是很多。我是一名 Ruby on Rails 的开发者，所以这是我使用它的主要原因。我真的不想走上使用 WordPress 或任何基于 PHP 的东西的道路。

那只是因为你偏爱红宝石吗？

是的，这给了我们很大的灵活性。它给了我们从 Rails 应用程序中获得的能力。这意味着我们可以解决更复杂的问题，并有希望提供一个更好的网站。

你考虑过其他基于 Rails 的 CMSes 吗？例如，打字错误？

我真的没有听说过。有博客工具，如 Typo，但据我所知，没有任何功能齐全的 CMSes。

那么，你对 CMS 是否能应对客户的产品化网站的挑战有什么担心吗？

没有。我见过它被部署。

**Rails 显然有一堆，找不到更好的词，“Web 2.0 功能”… [原型](http://prototypejs.org/)和 [script.aculo.us](http://script.aculo.us/) 已经安装…你们会添加“全栈”提供的任何东西吗？**

实际上，我们使用的是 jQuery。

**好的，那么你是否计划为这个网站的客户端带来一些闪光和用户体验的增强？**

这取决于时间。我们只有 24 小时，所以我们要完成核心工作。如果我们不太累，我们会看看我们能做什么。

在这次活动之前，你还做了哪些准备工作？

老实说，我一直在忙着工作，所以不要太多！除了下载很多我知道会在服务器上的源代码之外。有几件事要确保这项技术将为这个项目工作。但除此之外，不要太多。我有很多以前的作品。

在你彻夜未眠之后，我们还能期待什么样的复习时间呢？

很有希望，你会看到一个完成的网站！

**完了？一切都完成了吗？**

是的，理想状态！

我们期待着看到它。祝你好运，感谢你的时间，蒂姆！

没问题，谢谢！
