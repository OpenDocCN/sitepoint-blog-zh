## 处理 WordPress 安全问题:回顾

大多数人对 WordPress 安全这个词并不感兴趣。这是一个棘手的话题，通常伴随着恐惧。害怕怀疑你是否为你的网站做了足够的工作，是否做得正确，或者根本不做。

几周前，我们在 [SitePoint](https://www.sitepoint.com) 上与 [Chris Burgess](https://www.sitepoint.com/author/cburgess/) 举行了一次网络研讨会，讨论 WordPress 的安全性以及如何让你的网站变得安全。我们研究了:

*   常见的神话和误解。
*   是什么让 WordPress 成为容易攻击的目标。
*   攻击者的动机是什么以及他们如何攻击网站。

**最重要的是**我们还看了:

*   你可以做些什么来加强你的网站的安全性。
*   如何避免常见的 WordPress 安全风险？

我们都应该理解并从我们的网上研讨会中学到的一件事是*安全性至关重要*，它不是一个产品— **它是一个过程！**我们不只是让克里斯做所有的发言，你也参与进来了！很高兴看到聊天室里有这么多活动。观众问克里斯问题，观众回答彼此的问题。它变成了一个充满蓬勃发展的讨论的 WordPress 生态系统，所以让我们进入其中的一些。

## 你问克里斯的问题

### 问:手动硬化是什么意思？

Chris: 许多安全插件会做的是，对主机环境和服务器配置进行配置更改。这阻止了人们下载文件、查看文件和限制访问——诸如此类的事情。你知道这可以自己做吗？如果你知道要改变什么文件。如果你不知道，有很多流行的博客、知识库和食谱，人们用它们来加固 WordPress 站点。一个好的起点是官方文档，叫做 WordPress Codex。具体来说，一个致力于强化 WordPress 的[部分。](https://codex.wordpress.org/Hardening_WordPress)

我见过一些人说，*“看，我不信任安全插件，我更喜欢自己做。”*这很好，但你需要知道自己在做什么，并准备好投入时间。在我看来，安全插件在很短的时间内完成了大量繁重的工作。此外，它们还会给你带来其他好处。例如，他们可以为您提供审计和报告，如果您在团队环境中工作，拥有这些功能会有所帮助。您还需要考虑到安全插件正变得越来越复杂，以抵御越来越多的威胁，因此在幕后有很多功能。不过，如果你喜欢自己做这项工作，如果你真的想弄脏自己的手，你可以这样做！只要准备好投入时间。

### 问:即使隐藏了我的`wp-login.php`或`/wp-admin`位置，我的网站仍然会受到登录攻击。这些是机器人吗？我能做些什么来防止它们？

克里斯:这真是个好问题！这也是为什么 [WordPress Codex](https://codex.wordpress.org/Brute_Force_Attacks) 有关于暴力攻击的信息。

关于如何应对暴力攻击，有几个不同的思想流派。所有面向公众的网站都不断被探测，但大多数情况下，这些都可以使用流行的安全插件来阻止。安全插件可以被配置为在一定次数的不正确尝试后阻止某人，你可以增加这一点的敏感性，例如，如果不正确，你可以在尝试几次后就锁定某人。

您还可以在服务器级别做一些事情。还有 DNS 服务会过滤很多不良流量，也可以帮助阻止收割和垃圾邮件机器人。一些流行的 DNS 提供商将过滤一些不良流量，甚至在它到达您的服务器之前。这些服务通常也有助于提高性能。

### 当继承一个 WordPress 站点时，你应该采取的第一步是什么？

Chris: 我要做的第一件事是确保你已经阅读了 [WordPress Codex](https://codex.wordpress.org/) [关于强化 WordPress](https://codex.wordpress.org/Hardening_WordPress) 的指南，所以你已经涵盖了基础知识和所有基本的最佳实践。

这意味着:

*   没有“管理员”用户名，使用强密码，限制谁有管理员权限。
*   安装了一个安全插件(并运行全面扫描)。
*   WordPress(包括所有主题和插件)已经更新。
*   移除所有未使用的插件或主题。

我建议对网站进行审计，看看正在使用哪些插件。这有时可能有点主观，归结为偏好。我倾向于尽可能无情地使用插件——现在有太多插件了！尝试并坚持只使用你能找到的最有信誉的开发者的插件。这并不意味着它是一家公司。有些开发人员有很好的声誉，他们能够快速修复问题或者提供强大的支持。这些就是我们要寻找的东西。

当你继承一个站点时，你应该做的事情的清单:

*   您已经备份了您的站点，至少可以回滚到您获得它时的状态。这可能是最重要的事情。
*   遵循最佳实践将是我的首要任务，社区文档非常全面。
*   请确保您已经安装了一个安全插件，并且您已经了解了选项并正在正确使用它。
*   确保所有东西都是最新的，包括主题和插件。确保你有任何高级插件的许可证。
*   告诉客户风险，根据你管理的网站的价值来计划你的下一步。

我认为如果你继承了一个网站，你必须说，*“我们没有建立它，我们没有写很多代码。但我们会尽我们所能确保你现在得到很好的照顾。”*我知道这有点热情和模糊，但这确实是我们所能做到的最好的，因为有许多网站已经建成，并留给了网站所有者。一种— *“给你钥匙，回头见”*的方式。然而，我们大多数人已经认为网站是一个正在进行的**工作。**这是一个活生生的“东西”，所以当有人愿意握住他们的手，帮助他们度过他们不一定理解的事情时，客户会非常感激。你只需要试着教育他们，确保他们明白风险总是存在的。不仅仅是一个*“构建并转储”*；如果你想要一个强大的在线状态，这不仅仅是新的和闪亮的，还需要确保它的维护和安全。

### 问:有没有一个网站列出了 WordPress 安全插件的优劣？

克里斯:这可能会很主观。就像我们会在亚马逊或 app store 中寻找社交证明一样，这些东西会引导我做出决定:

*   在 WordPress 官方插件目录里吗？
*   它有星级吗？
*   它目前有多少活动安装？
*   最后一次更新是什么时候？
*   它有多少个支持查询？

请记住，没有什么比与其他开发者或网站所有者交流更好的了。除此之外，我还建议查看 WordPress 支持论坛，那里有一些非常有经验的人愿意帮助那些需要帮助的人，只要确保你已经[阅读了欢迎页面](https://codex.wordpress.org/Forum_Welcome)。

如果你是从零开始研究，只需寻找文章。你会发现有很多人写这方面的东西，并会真正深入到功能。

关于插件的快速说明。如果有东西两年没有更新，它会在官方插件目录上被标记出来。远离它们，因为从支持的角度来看，它们可能是有问题的，从安全的角度来看，它们也可能是潜在的噩梦。

> 更简单的方法是点击一个按钮，安装一个插件。请记住，最终这些网站将会存在很长一段时间，所以请确保我们处理的是可靠的东西。
