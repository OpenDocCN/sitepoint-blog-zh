# 你已经走了很长的路，宝贝…不。

> 原文：<https://www.sitepoint.com/youve-come-a-long-way-baby-not/>

尽管武断地谈论微格式、结构化标记和语义网很有趣，但在过去的一周里，有两件事让我明白了网络思维的前沿远远领先于《街上的乔》。

第一篇是 XML.com 大学的戴夫·西格尔的一篇文章。

> 网络被毁了，是我毁了它。有人说我毁了网络，对他们来说这是真的。网页不容易被搜索引擎看到，那些低端机器很难从我的网站上获得更多。在我的个人网站上，我甚至不放 ALT 标签，只是为了给那些没有图片的浏览者传递一个信息。我的生活是视觉化的。我喜欢博物馆。你想不想在关闭图像的情况下游览卢浮宫？

戴夫是网页设计的真正先驱之一，他的“[创建黑仔网站](http://www.amazon.com/exec/obidos/ASIN/1568304331/killersitesco-20/104-0917087-7023923)”是网页技术的早期经典之一。将他与当前的网络格局联系起来，如果坦特克·切利克是“*盒子模型黑客先生*，那么戴夫·西格尔就是“*先生。表&垫片 Gif* 。

这篇文章非常有趣，但有点令人沮丧，不是因为 Dave 支持 spacer GIFs，而是因为他实际上承认了他让我们陷入的问题，以及他早在 1997 年就写这篇文章的事实！

是的，没错，*差不多十年前，*‘[单像素 GIF 魔术](http://www.dsiegel.com/tips/)的发明者警告我们所有人，*不要为了一些表象上的巫术*而损害你的结构。他在我们这边(结构化标记这边)之前就已经站在我们这边了。

> 在此之前，我们将经历另一轮黑客攻击，将所有东西放入数据库，并从那里提供页面。对搜索引擎一点帮助都没有。这将花费数百万美元。这完全没有必要。别看我。看看网景。他们打破规则；我只是做该做的事。如果我毁了网络，我道歉。这是我一直以来的意图。许多人和我一样，把设计和内容放在结构之前，现在我们可以看到隧道尽头的一盏灯。网景公司挡住了去路，但他们可能会回心转意。网站设计师联合起来。为表达和结构而战。如果我们赢了，我们的未来会很光明，我们将不得不戴上墨镜。

大卫的愿景令人印象深刻，但当我们 2006 年坐在这里时，结果却令人沮丧。

当我注意到一种新的“圆角技术”在社交网络中心的一些网站上获得势头时，这一周就被整齐地结束了——漂亮的角落有整洁的标语*没有图像。没有 Javascript。没有绒毛*。”

顾名思义，Greg J 的 Spiffy Corners 大量借鉴了 Alessandro Fulciniti 的' [Nifty Corners](http://pro.html.it/esempio/nifty/) '(他坦率地承认了这一点)，使用多层风格化的`<b>`标签来构建一个柔和的角落。关键的区别在于，Alessandro 的`<b>`标签只在 JavaScript 说它们存在时才存在(所以移除是微不足道的)，而 Spiffy Corners 采取了将它们永久硬编码到标记中的方法。标记是这样的。

```

 <div>
<b class="spiffy">
<b class="spiffy1"><b></b></b>
<b class="spiffy2"><b></b></b>
<b class="spiffy3"></b>
<b class="spiffy4"></b>
<b class="spiffy5"></b>
</b> <div class="spiffy_content">
<!-- Your Content Goes Here -->
</div>
<b class="spiffy">
<b class="spiffy5"></b>
<b class="spiffy4"></b>
<b class="spiffy3"></b>
<b class="spiffy2"><b></b></b>
<b class="spiffy1"><b></b></b>
</b>
</div> 

```

现在，我对 Greg 或 Spiffy Corners 没有任何意见，他显然在建设和推广网站方面投入了大量工作。从我自己的角度来看，稍微令人沮丧的部分是，如此多的人似乎将此视为他们感知问题的解决方案——最后在一个知名的社交书签网站上查看了 2000 多个“铲子”。

虽然我可以理解 JavaScript 可能不是所有人的解决方案，但我必须承认我并不真正理解它所解决的问题。

*   图像有什么问题？至少你可以关掉它们。
*   这样怎么节省带宽？CSS、Javascript 和图像在第一次下载后都会缓存。**标签不行，所以每新一页出血多一点。**
*   如果我们想要更大的弧呢？每当我们想让曲线变大 1 个像素时，我们会添加一个新的类吗？

毫无疑问，这也不是针对 Frontpage 98 人群的。你至少需要熟悉 HTML 来让它工作。在我看来，整个网络标准的一些核心概念似乎还没有真正成为集体的网络意识。

啊戴夫，现在去哪儿？

Greg 功不可没，他消化了很多讨论(这里和其他地方)并基于 Thrashbox 方法开发出了一个更大更好的拐角生成器。你可以在这里亲自查看 Spiffybox。干得好，格雷格。
