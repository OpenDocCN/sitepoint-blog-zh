##### 拆分长输入表单

**设计博士，网站如何将一个长长的用户输入表单拆分并分散到几页上(continue… > >)最后一页有提交按钮？杰夫**

Jeff，我相信你知道，做这种工作需要某种服务器端处理(ASP/PHP/CGI/etc)。我将让您选择使用哪种语言，但是无论您选择哪种语言，一般过程都是这样的:

1.  让你的第一页提交到你的第二页，并像这样标记提交按钮:`<input type="submit" name="submit" value="Continue >>>">`

*   让您的第二个页面从第一个页面收集信息，确保它是有效的数据(如果不是，生成一个错误提示用户提供更多信息)*   让你的第二页提交给你的第三页，就像你的第一页一样*   起泡，冲洗，重复！

除了第一页，基本上每一页都首先收集和验证数据(名字是必需的，年龄必须是数字，等等。).然后，它让用户填写该页面的信息，然后提交给下一个页面。一般来说，最终页面会验证所有数据是否有效，然后将其放入数据库(或通过电子邮件发送给用户，或启动 StalkerBot 等)。).

##### IIS、ColdFusion 服务器和多种语言

嘿，医生，我在 ColdFusion Server 5 上安装了 IIS 5.1，用于运行 ASP 和 CFM，但我也需要处理 PHP 脚本。我对 web 服务器的了解有限，但我知道 PHP 运行在 Apache 上。有什么方法可以让这三种强大的服务器端脚本语言在我的机器上得到本地处理？提前感谢亚历山大

当然亚历山大，这是完全可能的，只要你被分配了一个以上的 IP 地址。原因是不同的 Web 服务器不能“监听”同一个端口，因为它们会发生冲突。您只需指派 IIS 监听一个 IP，并指派 ColdFusion Server 监听另一个 IP。要将 PHP 加入到这个组合中，我建议你使用 IIS 的 PHP 模块，因为你可能不想在这个组合中加入第三个 Web 服务器。

##### PHP 还是 ASP？

设计博士，我正在考虑我的网站应该使用 PHP 还是 ASP。哪个更好？—来自苏西

苏茜，这个古老的争论几乎和麦金塔与个人电脑之战一样激烈，答案往往归结为个人偏好。也就是说，PHP 和 ASP 确实有不同的优点和缺点，这可能有助于你下定决心。Kevin Yank 写了一篇关于选择服务器端语言的优秀文章，下面是一些摘录来帮助你。

***PHP***

**优点:**

*   相当容易学习，特别是对于有 C/C++、Java 或 Perl 经验的开发人员来说
*   活跃的用户社区愿意帮助你开始
*   语言中内置的一切
*   跨平台

**缺点:**

这种语言被设计成对程序员友好的，不幸的是，这使得它对非程序员不太友好。

***ASP***

**优点:**

*   虽然需要编程，但相当容易学会
*   内置于 Windows 2000 Server 中—没有安装问题
*   提供专业支持

**缺点:**

*   这不是免费的(除非你已经在使用 Windows 服务器)
*   如果你习惯了 C 风格的语法(C/C++、Java、Perl、PHP 等共享。)，您可能会发现 VBScript 的特性很难掌握
