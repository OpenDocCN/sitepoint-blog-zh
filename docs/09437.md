# 以用户为中心的界面设计的 5 个原则

> 原文：<https://www.sitepoint.com/5-principles-of-user-centered-interface-design/>

哦，谷歌地图…没有你我会在哪里？

可能现在还在去新墨西哥的路上。我只想去街角的商店。

但这不是一篇关于我缺乏导航技能的文章。这是关于我们都想知道如何在不迷路的情况下四处走动。当我使用网站、电脑程序、视频游戏或任何需要我与软件交互的东西时，我也想知道如何四处走动而不迷路。

幸运的是，我们生活在一个像用户体验设计和用户界面设计这样的学科已经足够成熟的世界，我们倾向于将它们简化为 UX/用户界面设计，人们仍然知道我们在谈论什么。然而令人惊讶的是，有多少次我仍然发现自己在一个网站上，并感到我的沮丧程度在上升，因为我不知道如何做对我来说似乎应该是一个简单的任务。有时候我真的很想去新墨西哥，但是我得到的都是去街角商店的方向。

我在下面列出了以用户为中心的界面设计的 5 个原则，这些原则帮助我们的网站和应用程序保持正轨。我最熟悉的是网站和移动界面，但这些原则确实可以应用于任何界面，或者任何人与数字或机械系统接触的时候。界面可以是导航菜单、搜索字段、对话框、选择指针、工具面板、拖放功能、web 表单、简单按钮等等。接口也是电脑上的电源按钮和音响上的音量旋钮。还有你 iPad 的触摸屏。界面是我们人类与机器互动的层面。在我的网页设计世界里，界面非常重要，因为不管网站的内容如何，如果有人不能轻松地浏览你的网站，那么伟大的内容可能永远也找不到。这个世界可能永远也不会知道你最可爱的小猫的视频。

乍一看，下面的原则似乎相当明显。即便如此，当谈到网页设计最佳实践时，再次强调这些显而易见的东西也无妨。有多少次你需要把一个网站推出去，并且想跳过为你正在使用的最后几个图片提供 alt 属性？抵制诱惑！有时，即使是看似显而易见的事情，在一开始也不是那么显而易见。如果这不是真的，那么所有的界面都会很容易使用，像我这样的人就没有必要写一篇文章来陈述这些显而易见的东西了。好吧，你已经等得够久了，关于原则…

# 原则#1:理解你的用户并支持他们的目标

Mitchell Kapor 早在 2002 年就写了一篇软件设计宣言，这篇论文中有一段话我至今仍喜欢引用:

“如果一个用户界面是‘事后’设计的，这就像是在发动机、底盘和所有其他组件和功能都被指定之后设计一个汽车仪表板。”

现在，乍一看你可能会认为没什么大不了的，一旦你建立了所有的核心组件和功能，就可以设计汽车的仪表板或网站的界面。但这通常会给你带来麻烦。为什么？因为你的用户并不直接与你的核心组件交互，他们通过你提供的界面来体验你的系统，以及它能为他们做的一切。如果这些界面是事后设计的，它们几乎不可能满足所有用户的目标。换句话说，你需要首先了解你的用户——他们的需求和目标——以便创建一个允许他们有效访问你的系统功能的界面。

在创建以用户为中心的界面时，重要的问题是:谁是用户？用户需要的主要功能是什么？用户为什么使用这个特定的软件/硬件，他们的目标是什么？不同经验水平的用户可以访问软件/硬件吗？用户与软件/硬件交互的最直观方式是什么？

# 原则 2:让你的界面简单易学，使用愉快

对于典型的 iPhone 用户来说，他们不关心创造他们的设备的软件和硬件工程的辉煌，他们只知道它容易使用，令人愉快。手指滑动，点击，双指捏，弹弓愤怒的小鸟。你可以打赌，苹果从来不会在没有首先考虑人们将如何与他们的产品交互的情况下创造任何东西，我猜想，在产品开发生命周期的所有阶段以用户为中心的设计理念是苹果成功的一大原因。

对于网站来说，创建一个令人愉快的界面可能看起来不太明显，除非你正在为 Flash 游戏创建一个复杂的界面，但是令人愉快可以像一个简单的导航结构一样直接和直观。即便如此，随着越来越多的应用程序迁移到云端，我们的浏览器成为我们的操作系统，为我们的网络产品创建有效、简单和令人愉快的界面变得前所未有的重要。我最近从基于桌面的线框软件(Axure，Omnigraffle)转换到一个名为 Lucid Chart 的在线应用程序，这种转换是无缝的，因为作者确保创建一个易于使用的 web 界面，并为那些从桌面程序迁移的人所熟悉。

# 原则 3:保持一致

帮助用户保持空间方向感和理智的一个重要概念是保持一致。从用户的角度来看，这不仅意味着系统界面具有一致的外观和感觉，还意味着在界面上执行的操作会产生预期的结果。如果重复相同的动作，系统需要以一致的方式响应。

我偶尔还会碰到一些奇怪的网站，它们会根据你在网站上的页面来改变导航栏或子导航栏的顺序。对我来说，这就相当于根据你是在高速公路上行驶还是在小路上行驶来切换你汽车的油门和刹车踏板。界面需要保持一致，这样它们就可以消失在背景中，而不需要更高的注意力来使用。

# 原则 4:与你的用户建立对话

我喜欢和我说话的界面。不是直接跟我说话(虽然那也不错)，而是一个界面，它使用有意义的术语，并在我需要时给我反馈。

为了确保我们不会迷路，并且总能找到我们要找的东西，给界面贴上正确的标签是很重要的。每个人都知道导航栏中的“主页”按钮应该做什么，但并不是每个菜单标签都这么明显。例如，假设您的网站上有一个多用户游戏，允许人们相互竞争，上下移动排行榜，聊天，组成小组等。如果这是你仅有的两个选择，你会把这个游戏放在导航中的“多媒体”或“社区”类别下吗？这时像卡片分类这样的技术可以帮助你找出最佳的导航结构。

反馈机制对于有效的界面设计也是至关重要的:用户需要感觉到他们的行为是有意义的。例如，我们都熟悉填写和提交在线表单，但是您是否曾经单击提交按钮，然后页面刷新并再次显示空表单？然后你被迫问你的信息是否通过，是否有错误，我是否应该再次填写表格以确认？不要在用户心中留下任何疑问——对他们的行为给予反馈。感谢您的询问，我们从星巴克回来后，会有人尽快回复您。

# 原则 5:没有问题

抑制快乐的最快方法是在简单的界面和导航问题上制造挫败感。如果你和我一样，测试从来都不是过程中最有趣的部分，但是它是至关重要的。这是一个竞争激烈的市场，如果你的系统界面有明显的缺陷，你可能会失去你的用户，也许是永远失去，不管你的内容有多好。然而，界面问题可能不仅仅是软件错误，因为设计不良的界面仍然是一个主要问题。测试你的界面的一个好方法是观察人们在真实场景中使用你的系统。他们能够相对容易地导航并实现他们的目标吗？你的界面对有经验和没经验的计算机用户来说都是直观的吗？

我会原谅一个网站有小的界面问题，可能会让我有点摇头，但时间是有限的资源，如果太多的问题干扰了我试图完成的任务，对不起，但我继续前进。

# 结论

我有一个秘密要分享。我是一个漂亮的网站吸盘。但事实是，在一天结束时，一个网站的视觉美学很可能与它的成功或失败无关。如果一个网站看起来不错，我的注意力就会被抓住，我会从设计中获得乐趣，但是很快就归结为这个网站是否能为我做我想要它做的事情。决定一个网站成功的关键因素是它的可用性和实用性。内容当然是王道，但是内容需要被发现和访问。

所以我希望你在当前和未来的项目中考虑这 5 条以用户为中心的界面设计原则。当然，除了这五条原则之外，还有很多其他原则，但这正是 UX 设计书籍的目的。我只是希望给你一些思考的食粮，或者再次强调你已经知道的东西，这样我们就可以一起努力建立一个有更好界面的世界！

以用户为中心的界面设计的伟大之处在于它永远不会静止不动。它融合了许多不同的学科——硬件和软件工程、人类工程学、心理学、社会学、语言学、计算机科学等。–这是一个不断发展和成长的主题，要完全掌握其中的复杂性意味着我们永远不能停止对它的研究和学习。
