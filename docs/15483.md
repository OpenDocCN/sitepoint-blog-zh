# 移动日:切换网络主机

> 原文:[https://www.sitepoint.com/day-switching-web-hosts/](https://www.sitepoint.com/day-switching-web-hosts/)

每个网站管理员至少都会遇到一次这种情况。你要么超过你的虚拟主机公司，他们不支持你需要的新功能，或者他们只是咬。所以你开始寻找一家公司为你的网站提供一个新家。经过几天乏味的搜索，你找到了一个完美的公司来托管你的网站。但是，如何确保平稳过渡到新服务器呢？遵循以下五个简单的步骤，确保转移顺利进行:

##### 1.设置新帐户并上传

当在新的托管公司建立帐户时，一定要告诉他们你不希望他们转让域名，直到你告诉他们这样做。

从 webhost 公司获取新帐户的登录名和密码。你将登录到一个 IP 地址(类似 209.207.44.3)，而不是像 yahoo.com 域名。

在新服务器上复制您当前的目录结构，并上传除 CGI/PERL 脚本之外的所有内容。您必须询问新的托管公司 PERL 在服务器上的位置以及您的站点所在的目录。如果它不同于你现在所拥有的(我打赌它会)，那么你必须在上传之前手动编辑脚本来反映这些变化。

提示:以 ASCII 模式上传你所有的 HTML 文件。这将防止以二进制模式上传时可能出现的“空白页”症状。查看 FTP 程序附带的说明，了解如何更改设置。

##### 2.测试几页

通过测试几个关键页面，确保上传顺利，域名转移后一切正常。这将需要你手动输入几个网址。它们看起来会像这样:

<q>http://209.207.44.3/faq.htm
http://209.207.44.3/contact.htm
等。</q>

只需将 IP 地址替换为托管公司分配给您的地址，将 HTML 文件名替换为驻留在您服务器上的实际文件名。

如果您单击测试页面上的任何链接，您将被转移到您当前的服务器，而不是您想要去的新服务器。唯一的例外是你使用相对路径而不是绝对路径来链接你的 URL。

<q>相对网址:contact.htm
绝对网址:http://site.com/contact.htm</q>

这也是测试您的脚本的好机会，尽管您必须将脚本中的域名引用更改为新主机上站点的 IP 地址。

##### 3.转让域名

一旦所有的东西都在新的服务器上并正常工作，给你的主机公司发邮件，告诉他们转移你的域名。通常，您会在 24 小时内收到网络解决方案的电子邮件，要求您确认转账。简单地回复邮件，然后等待。

一旦转移完成，你将不得不等待长达两周的时间来更新世界各地的 DNS 服务器。在此期间，维护您的网站的两个副本。即使在您的 DNS 服务器更新后，为了那些 ISP 的 DNS 服务器没有更新的人的利益，您可能会想暂时保留站点的两个副本。

了解您自己的 ISP 何时更新的一个好方法是将某个特定页面的稍微不同的版本上传到新服务器。当你输入:http://site.com/page.htm，看到不同的版本，你就会知道你的 ISPs DNS 服务器已经更新。

**热门提示:**一旦您站点的新副本启动并运行，请联系您当前的主机，要求他们更改 MX 记录，使其指向您新 web 主机上站点的 ip 地址。这将消除维护两个站点的需要。

##### 4.稍等片刻，然后取消您以前的帐户

在等待期间，您将有两个电子邮件帐户可以查看(如果您使用托管公司为您提供的 POP 帐户)。检查它们的最佳方法是使用新旧服务器上的站点 IP 地址，作为电子邮件程序中的 POP 服务器，而不是使用域名。

一两周之后，注销之前托管公司的账户。确定何时关闭旧帐户的一个好方法是，当电子邮件停止进入旧服务器上的帐户时。

##### 5.休息一天

如果一切顺利，给自己放一天假，好好工作！
