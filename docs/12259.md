# Gmail 的崩溃是否预示了“暗云计算”？

> 原文:[https://www.sitepoint.com/gmail-dark-cloud/](https://www.sitepoint.com/gmail-dark-cloud/)

今天像往常一样开始，我为 SitePoint 的读者搜索相关新闻。然后事情发生了， [Gmail 崩溃了](http://www.foxnews.com/story/0,2933,499106,00.html)。怀疑这是客户端的问题，我花了大半个小时试图解决它，这时一个熟悉的“服务器错误”消息出现了。停电发生后不到一小时，新闻就出现在各大媒体上，宣布可能的全球系统故障。这是[而不是](http://news.cnet.com/8301-1023_3-10068189-93.html)第一次【Gmail 失败了，但是今天的事件似乎传播得更广。有趣的是，这一事件对云计算的影响可能比对电子邮件的影响更大。

如今，云计算的反对者和支持者各占一半。多年来，谷歌和其他公司一直试图向主流市场销售像 Gmail 这样的“基于云的”应用。但是，今天的事件加剧了对许多运营来说，云计算根本不可行的批评。Gmail 目前是仅次于 Hotmail 和雅虎的第三大最受欢迎的在线邮件服务。据大多数[报道](http://www.email-marketing-reports.com/metrics/email-statistics.htm)，拥有 9500 万到 1.1 亿用户。

具有讽刺意味的是，谷歌利用 Gmail 进行大部分沟通，因此用户甚至无法有效地抱怨今天的问题。这项服务只关闭了几个小时，但不到一个小时，Twitter 就被投诉疯了，正如我所说的,[新闻](http://www.washingtonpost.com/wp-dyn/content/article/2009/02/24/AR2009022400850.html)像野火一样传遍了媒体渠道。我们不需要因为这些事件而抛弃云计算的想法，但即使是像这样大范围的小不稳定也需要仔细检查。就应用程序对崩溃的敏感性而言，电子邮件和许多其他云应用程序一样，是相对惰性的。至于其他更“敏感”的公司，看到谷歌崩溃至少会让能力较弱的公司感到惊讶。

好消息是 Gmail 已经恢复了，谷歌也就此事道歉了。毫无疑问，热情会在一两天内平息下来，一切都会恢复正常。但是云计算(特别是电子商务和企业类型)的更大图景受到了另一次打击。当然，影响有多大还有待观察，但要让“云”发挥其作为数十亿美元产业的潜力(尤其是长期产业)，冗余和其他保障措施必须得到更充分的解决。基本原理可能是这样表达的:“如果谷歌甚至可以宕机一个小时，那么小得多的企业云解决方案会发生什么？”今天的不幸是否让云计算的前景变得暗淡？它只代表一场柔和的冬雨，但它确实提醒我们，它可能会下雨。
