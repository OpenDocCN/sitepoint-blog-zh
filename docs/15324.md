当我们很多人听到“黏黏的”这个词时，我们会想到黏糊糊的小圆面包或含糖的奶油馅(好吃)。其他人认为是“GUI”，比如“图形用户界面”图形用户界面是指计算机用来显示和访问信息的图标、任务栏和其他对象的系统。我们中的一些人可能想知道 GUI 是怎么来的。我们记得 DOS 提示符和命令行交互的太平日子；然后，我们中的一些人吃一片阿司匹林，然后躺下。其他人仍然想知道我们到底是如何从绿色屏幕上深奥的 UNIX、CP/M 和 DOS 命令发展到可以玩漂亮的图片和彩色桌面的。

这篇文章讲述了 GUI 是如何产生的。它从克鲁马努·格鲁格如何意外开发 GUI(以及个人电脑)的故事(可能是虚构的)开始。然后，它带我们回顾了伊凡·苏泽兰的画板、施乐的 PARC 实验室、艾伦·凯的 Smalltalk，以及乔布斯的苹果公司和比尔·盖茨的微软公司之间的竞争(可能更杜撰)故事，这些故事给了我们今天的 Windows 和 Mac GUI 驱动的操作系统。在这个过程中，我们将了解 memex、第一个木制鼠标、“bit-blitting”、施乐之星、苹果丽莎，以及史蒂夫·乔布斯和他的公司手拿笔记本拜访 PARC 实验室的那个重要日子里到底发生了什么…

事实上，有些人认为 GUI 是史蒂夫·乔布斯赐予渴望的人类的，仅仅是因为苹果的 Macintosh 系列电脑是我们大多数人第一次接触到图形界面的地方。是给我的。我第一次使用 GUI 是在 1984 年左右，当时我的电脑朋友自豪地向我介绍了他的全新 Mac 电脑，包括一个 neato 视频游戏，其中有一个人在挖洞捕捉捕食性蜘蛛，还有一个非常酷的程序，名为 MacPaint。我至今记得我创作了一件精美的麦克派艺术作品，另一位朋友立即称之为“蒙德里安华夫饼”可惜我不知道怎么把它打印出来。

但是今天我几乎完全在 GUI 环境中工作；甚至在输入这个文档的时候，我点击一个图标化的按钮来创建字体效果或者保存文档。每次我在网上冲浪的时候，我都会利用别人的 GUI 创作。我点击带我去新地方的按钮；我欣赏有人创造的图形；我对设计糟糕的 GUI 结构嗤之以鼻。你做同样的事情，不管是在苹果电脑上，还是在 Windows 电脑上，还是在其他电脑上。毕竟是 GUI 世界。但这是史蒂夫·乔布斯送的礼物吗？还是故事更复杂？

##### 图形用户界面的起源

“我在施乐 PARC 技术公司看到的是穴居人界面，你指着它，然后咕哝。一次大规模的放松，远离语言，以解决用户的技术紧张。”T3】

 *——一名 IBM 技术人员痛斥苹果丽莎的图形用户界面*  *很久很久以前，在石器时代，住着两个穴居人，呃和咕嘟。Ugh 是一个英俊、活泼、喜欢户外运动的人，有着惊人的体格和水怪的智力。他的朋友格鲁格正好相反:紧张，牙齿，骨瘦如柴(除了他因为吃了太多外卖披萨而变得有点啤酒肚)，而且聪明。太聪明了。有一天，当 Ugh 出去打猎并为穴居女人摆姿势时，Glug 闷闷不乐地坐在他的洞穴里，心不在焉地抓着他的腰带，告诉自己他不嫉妒那个笨蛋 Ugh。在他头顶正上方，一块岩石从天花板上脱离时颤抖着。它啪的一声挣脱了，砰的一声直接落到了格鲁的尖脑袋上。

当咕嘟醒来时，是令人讨厌的水泼到他脸上的感觉，以及同样令人讨厌的 Ugh 须后水的气味——巨大的麝香。呃弯下腰看着他，他完美的蓝眼睛里充满了担忧。咕嘟好吗？岩石重击咕嘟。"

格鲁把他的朋友推开说:“给格鲁一点空间！必须思考！”格鲁格从洞里冲出来，冲进洞口外的草地。他气喘吁吁地抓起椰子壳碎片、一大堆高草、光滑平坦的岩石和几块早餐剩下的猛犸象骨头。他拖着行李走进洞穴，几乎是在事后才想起，捡起了之前把他打昏的那块石头。呃惊奇地看着他的朋友，然后走到外面盯着虫子看(他一直很着迷)。用一根尖尖的棍子，格鲁开始在泥土上画奇怪的图表。几个小时后，Ugh 偷看了里面，看到他的朋友全神贯注于他奇怪的洞穴绘画，耸耸肩，去了第三个洞穴，与他的女性朋友 Oohlala 共度夜晚。

几天过去了。Ugh 很快学会了呆在洞穴外面，因为如果他进入，Glug 会咆哮并向他扔石头和椰子壳。另外，和奥拉拉在一起更有趣。第四天，咕嘟第一次来到外面。他眯着眼睛看着刺目的阳光，然后示意呃进来。Ugh 把 Oohlala 从他的腿上推下来，跟着他的朋友进了山洞。

在角落里，藏在熊皮后面的是一个方形的物体。暗淡的光线似乎来自动物皮肤下面。令人惊讶的是，Ugh 接近了那个物体，准备在第一个奇怪的声音或威胁动作时逃跑。格鲁说，“啊，看哪！”从物体上撕下熊皮。

不管它是什么，它是由椰子壳和猛犸象骨头组成的，一个大的挖空的椰子壳坐落在大块扁平的岩石上，这些岩石早些时候已经被大脑吞噬了。一捆捆的草缠绕在一起，塞进各种物品的背后。一块扁平的矩形岩石上覆盖着细小的藤壶，位于物体主体的前方稍下方，用一条草编带与主体相连。黑暗的光线从中央放置的大椰子壳中倾泻出来。呃盯着可怕的恐惧在咕嘟的创作。

“你看，呃！”咕嘟嘟囔囔。"咕嘟咕嘟造奇迹！"当 Ugh 目瞪口呆的时候，他俯身在布满藤壶的平坦岩石上，开始戳它，显然是随意的。但是……在椰子壳的黑暗光线下出现了线条，细小的亮绿色线条。显然，咕嘟咕嘟越戳那些布满藤壶的扁平岩石，贝壳上出现的绿色线条就越多。

过了一会儿，呃开始变得不耐烦。毕竟奥拉拉在外面等着。“这东西有什么用？”他问。“不好吃。不适合在雪地里穿。也许对猛犸象有好处……”

“不不不！”咕嘟嘟囔囔。“能做事。数猛犸象。看好了。”他开始用又短又尖的戳戳藤壶板，一条一条的线出现在壳上。"一只…两只…这是你上个月杀死的一只…这是你追过悬崖的一只…这是被冲到河里的一只…"他不停地戳着，最后停了下来。“看到了吗？自从第一次解冻以来，已经杀死了 17 只猛犸象。很高兴知道，嗯？”

呃，他很感兴趣，尽管他自己。“Ugh 能用吗？”咕嘟示意他坐在壳前，把 Ugh 的脏手指放在藤壶上。“好吧，呃，注意了。第一，必须登录。”

“登录？”

“登录…我们创建一个用户名。我们称之为……呃。”越过 Ugh 的肩膀，Glug 戳了戳藤壶，让 Ugh 越来越困惑。“现在 Ugh 可以进入系统了。很好。现在我们转到 C: prompt…..”

“看到提示了吗？在哪里？唉糊涂了。”

“呃等等。很好，我们进去了。现在我们访问文件目录。”

“文件目录？用户名？见提示？”Ugh 推开自己远离奇怪的对象，站了起来，愤怒地耸立在咕嘟。“咕嘟咕嘟搞不清啊！弄得头都疼了！你想用奇怪的东西，给我漂亮的照片！鲜艳的颜色！指向并点击，而不是藤壶和看到提示！唉要走了！”

Ugh 把 Glug 推到一边，向洞口和外面走去。他没有注意到自己把格鲁格推得太用力了，可怜的格鲁格一头栽进了洞壁。一周内，格鲁格的头骨第二次撞上了坚硬的岩石。

格鲁格终于醒来时，外面已经黑了。呃，哪里也看不见。格鲁坐了起来，双手抱着受伤的头。“呼，头疼得厉害，”他喃喃自语。"但是格鲁有更好的主意！"他找到了他的尖头棒，又开始在泥土中画了起来。“他想要明亮的颜色，漂亮的照片，呃，他得到明亮的颜色，漂亮的照片。Glug 称之为…图形用户界面。但是不要告诉我。他像洞熊一样顽固，名字吓人。我叫它什么？”咕嘟咕嘟想了又想。最后，他决定采用简单直接的方法。“唉，让婴儿说话好吧，”咕嘟想。“坚持用婴儿的声音。叫它……GUI。黏糊糊的。甚至聪明到说出那句话。Gooooooeeeeeey。”

“没有理由会有人想要一台电脑放在家里。”T3】

<q>*—肯·奥尔森，数字设备公司总裁、董事长兼创始人，1977 年*</q>

这个故事是计算机世界神话的一部分。1979 年 12 月，一个普通的下午:年轻的电脑奇才兼企业家史蒂夫·乔布斯带领一帮家人来到施乐公司位于 PARC 的帕洛阿尔托研究中心。乔布斯和他的朋友们带着惊讶的眼神参观了工厂，给人留下了他们最好的诺曼·洛克威尔天才儿童的印象(“天哪，凯博士，能给我们签个名吗？”)，而在朴实的眼睛和黑框眼镜后面，正在记笔记和记忆图表。

乔布斯带领他的朋友们走出大楼，向里面友好的实验室极客们挥手告别，然后飞奔回他破旧的仓库，在那里，他和同事史蒂夫·沃兹尼亚克将他们能记得的施乐之旅中的每一个想法和过程都融入到他们的新产品 Macintosh 中。施乐被搞糊涂了，微软的比尔·盖茨被激怒了，苹果公司用一种新的舞蹈热潮——图形用户界面抢在了所有人的前面。“做图形用户界面”风靡了计算机世界，其他人都争先恐后地想得到这份肥缺。盖茨将乔布斯的偷窃行为超越了一步，并推出了苹果克隆版的 Windows，微软与当地司法部门上演双人舞以躲避苹果的诉讼，Windows 接管了世界，苹果在计算机行业的叛徒黑客和 Mac 痴迷者中被归入邪教地位。

给你的孩子读故事哄他们睡觉的好故事。它拥有一切:戏剧、犯罪行为、前同事之间无情的竞争，除了性之外的一切(反正刻板的电脑极客对性并不熟悉)。见鬼，它甚至可以成为一首很好的大卫·艾伦·科饮酒歌。唯一的问题是这不是真的。

##### GUI 的真实历史

图形用户界面的真实历史比简单的“它需要一个小偷”的概念更加复杂和交织。

***从小种子……***

1976 年，史蒂夫·乔布斯和史蒂夫·沃兹尼亚克在乔布斯的车库里创建了苹果公司。乔布斯和沃兹尼亚克在惠普相遇，并通过制造(沃兹尼亚克)和销售(乔布斯)“蓝盒子”非法设备开始了他们的合作生涯，这些设备从马贝尔那里骗取免费电话。两人都对当时的“原始”计算机感兴趣，并喜欢用焊料和试验板拼凑电子产品。最终，他们决定开一家公司，制造不会占用整个地下室、不需要超冷、也不需要一排排穿着连体裤的家伙来照顾他们的计算机。换句话说，他们设想为大众制造个人电脑。当然，乔布斯和沃兹尼亚克都不是第一个想到个人台式机大小的电脑的人(常识将这一荣誉给予了 MITS“Altair”，这是 1975 年基于套件的创造，运行微软的基本操作系统，并基于英特尔的 8080 芯片)，但那是另一个故事了。他们集思广益，决定将他们的公司命名为苹果公司。

1976 年 3 月，沃兹尼亚克制造了第一个苹果，Apple I。这是一个由电路板和 led 显示屏塞进一个木箱里的拼凑物，但它在计算界引起了足够的兴趣，激发了乔布斯和沃兹尼亚克在 1976 年愚人节创建了苹果公司，出售他们的小动物。乔布斯卖掉了他的大众迷你巴士，沃兹尼亚克卖掉了他的惠普科学计算器，来资助这家初创公司。他们只卖出了大约 200 台 Apple I，所以这家羽翼未丰的公司——现在由沃兹尼亚克、乔布斯和几个朋友/员工组成——用他们从 Apple I 销售中筹集的资金开始了 Apple II 的开发工作(据说沃兹尼亚克说他想开发 Apple II 的很大一部分是因为他为雅达利设计的经典视频游戏“Breakout”。沃兹尼亚克想为个人电脑编写程序)。1977 年，Apple II 首次亮相，其特点是光滑的塑料外壳(与容纳 Apple I 的“橙色板条箱”相反)、游戏手柄和视频显示器上的彩色图形。作为 Ugh 的后代，人们被明亮的颜色和闪烁的图像迷住了，Apple II 开始下架。

乔布斯意识到他已经开始做一件可能会迅速发展成为一个严重的商业问题的事情，于是他雇佣了更多的员工，扩大了工作空间，并全力以赴地去满足消费者对他产品的突然需求。当苹果在 1977 年添加了内置软盘(放弃了缓慢而笨拙的磁带存储设备)时，II 的销售真正起飞了，苹果突然处于个人计算兴趣的顶峰。不要介意许多新手买了一台苹果 II，却不清楚如何使用它……普通人能够拥有并操作一台“个人电脑”的简单概念就能抓住人们的想象力。

***40 年代——桂祖辈:老布什和恩格尔巴特***

让我们回到 1945 年(！)和一个叫万尼瓦尔·布什的幻想家。布什，一位科学家和未来学家，公开了他的“memex”的想法，这是一种计算设备，它将使用我们称之为超链接技术将信息带到每个用户的指尖。

布什的想法激发了一位名叫道格拉斯·恩格尔巴特的科学家的一些富有远见的想法。早在 1962 年，当乔布斯和沃兹尼亚克还在喝着阿华田，穿着睡衣看着周六早晨的卡通片时，恩格尔巴特就在创造一些个人电脑用户感兴趣的东西。他发明了第一个“鼠标”，他称之为“X-Y 位置指示器”，这是一个装在带轮子的木盒中的小发明，可以在桌面上移动，并在显示器上显示光标。恩格尔巴特将鼠标视为“图形窗口界面”的一个组成部分，并发明了他所谓的“窗口 GUI”，这让同事们着迷，但在实验室之外并不被认为有用。1968 年，Engelbart 创建了 NLS(在线系统)，这是一个超媒体群件系统，它使用鼠标、窗口 GUI、具有对象寻址和链接的超媒体，甚至是视频电话会议的早期版本，让斯坦福大学的一群技术人员、工程师和科学工作者惊叹不已。

然而，恩格尔巴特并不是 GUI 历史上唯一有远见的人。1963 年，伊凡·苏泽兰麻省理工学院的一名研究生提交了一个名为“画板”的程序作为论文，该程序使用光笔直接操纵 CRT 屏幕上的对象。

<q>*“Sketchpad 开创了图形计算的概念，包括存储对象的内存结构，线条的橡皮筋，在显示器上放大和缩小的能力，以及制作完美线条、拐角和关节的能力。这是第一个 GUI(图形用户界面)，远在这个术语被创造出来之前。”*T3】</q>

<q>*——摘自孙《微系统传》*</q>

在屏幕上直接操作对象的想法是图形界面概念的一部分。事实上，图形用户界面的想法来源于认知心理学，即研究大脑如何处理交流的学科。这个想法是，大脑使用图形图标和显示器比使用文字更有效——文字为交流过程增加了一层额外的解释。想象一下，如果你看到的所有路标都是统一的白色矩形，只有文字本身来区分不同的命令、警告和信息显示。当“停止”标志看起来与“恢复高速公路速度”标志没有什么不同时，处理标志信息的过程变得更慢、更困难，你会遇到比现在更多的事故。

将这一点与 Alan Kay 的“生物计算”概念结合起来，在生物计算中，计算机组件就像有机的“细胞”一样工作，无论是独立的还是在适当的时候协同工作，您就会对现代计算和 GUI 背后的思想有所了解。

***70 年代——small talk 和施乐***

<q>*—非正式的 PARC 口号*</q>

地下的嗡嗡声一直在地下，但施乐 PARC 工厂的创意人员没有忘记恩格尔巴特和萨瑟兰的创作。PARC 过去是(现在也是)，至少在某些方面，是一个计算“智囊团”，在这里，才华横溢和才华横溢的反复无常的头脑产生想法，并尝试在工作台上实现它们，取得了不同的成功。

在 70 年代早期，作为设想笔记本大小的超链接计算机的“Dynabook”项目(不幸流产)的一部分，Alan Kay 和其他人开发了一种交互式面向对象编程语言，称为 Smalltalk。Kay 之前曾与犹他大学的一个团队合作，开发了一个名为 Flex 的编程系统。这是一个灵活的模拟和面向图形的个人计算机的设计，许多想法来自挪威开发的 Simula 编程语言，另一种编程语言 LISP 和萨瑟兰的画板。凯还借鉴了一种叫做 Logo 的高度图形化语言的思想，这种语言是用来教孩子们编程的。Smalltalk 的特色是图形用户界面(GUI ),看起来与苹果和微软后来的迭代非常相似。

Smalltalk 并没有停止用户界面的创新:在 Sun 的人提出 Oak/Java、面向对象、重叠“窗口”和 bit-blt 或“bit-blitting”的第一个实例之前几年，它就推出了多平台虚拟机，后两个实例由 Dan Ingalls 贡献(st 中的面向对象语言实际上在 20 世纪 60 年代末出现在 Simula-67 程序中；“位块传输”或位块传输，简单地说，是一种可以操纵屏幕上的对象的协议。许多观察家认为，ST 干净、易用的界面至今仍未被超越。在 Smalltalk 下编写的第一个程序是 Pygmalion，它最著名的地方在于它演示了计算机编程可以基于图形而不局限于文本。用图标代表数据的想法在皮格马利翁中得到了体现。

第一个现实生活中可用的图形用户界面出现在施乐公司的 Alto 计算机上，它于 1974 年首次亮相，被设想为一个更小、更便携的主机替代品。Alto 没有你我习惯使用的 GUI，而是以图形驱动的应用程序为特色，它的大小和大众汽车差不多(嗯，不完全一样，但这东西很大)，对普通用户来说肯定没有用，尽管它最初显示的是芝麻街“饼干怪兽”的形象。Alto 的特色是位图显示，这对于显示图形和所见即所得打印是必不可少的。Kay、David Canfield Smith、Bill Verplank 和其他人也为 Alto 的各种程序开发了图标表示，最引人注目的是绘图程序“Markup”、文本编辑器“Bravo”和绘画程序“Superpaint”。

1981 年，诞生了 Alto 的设计和概念导致了更精简、更可用的施乐 Star 的开发和生产，这是第一台真正的 GUI 驱动的 PC。据后来为苹果工作的前施乐员工布鲁斯·霍恩说，Smalltalk 和 Star 的软件架构比 Mac 或 Windows 的同类产品复杂得多。虽然苹果机器融合了施乐的许多头脑风暴，但许多最具创新性和尖端的想法从未进入苹果，主要是因为苹果坚持降低成本。《明星报》展示了第一个“电脑桌面”，以及重叠的、可调整大小的窗口和复杂的 PARC 鼠标，这是一个没有移动部件的神奇小发明，使用激光束和金属网格来跟踪光标的移动(尽管员工发现鼠标在李维斯身上的效果和在金属网格上一样好)。这个界面被称为 WIMP 窗口、图标、菜单和指针。PARC 的共识是，一旦这些想法大规模实施，计算效率将大幅提高。

***1979 年——苹果造访 PARC***

苹果公司的项目经理杰夫·拉斯金(Jef Raskin)首先向乔布斯和沃兹尼亚克透露了 PARC 正在进行的研究。把这个场景想象成发生在某个废弃的停车场是错误的，拉斯金躲在阴影里，尽他最大的努力模仿深喉。更接近的情况是，拉斯金希望更直接地在 GUI 上工作，并向乔布斯透露了 PARC 正在做的 neato keeno 工作。乔布斯起初不愿意去，但最终拉斯金激起了他的兴趣。拉斯金早在 1967 年就在所见即所得的图形界面上写下了他的硕士论文，并看到他的一些想法被 PARC 的人们实现了。

无论如何，乔布斯在 1976 年第一次被拉斯金告知 PARC 发生的趣事，他决定要把一群苹果粉丝带到 PARC，看看是什么引起了这样的轰动——但同样，乔布斯像一个藏在衬衫下参观 Epcot 的孩子一样走进来的想法是错误的。苹果公司与施乐公司达成协议；作为对苹果股票的回报，施乐允许乔布斯和他的团队在 1979 年 12 月游览 PARC，做笔记，并在他们自己的创作中实施一些在 PARC 传播的想法和概念。我不知道施乐公司对苹果公司随后从他们那里雇佣了一半——也许是更好的一半 board 员工有什么感觉，但是这个过程相对来说是光明正大的；《盒子里的杰克》没有夜间绑架或贿赂。施乐公司允许苹果公司在他们的机器上使用他们的想法。正如沃兹尼亚克在他的网站上所说，“史蒂夫·乔布斯直接向施乐 PARC 公司的高管们证明，他们拥有伟大的技术，但苹果知道如何让这些技术变得足以改变世界。这非常开放。最终，施乐因为分享了技术而获得了一大块苹果股票。那不是赤裸裸的偷窃。"

“乔布斯之所以在以人为中心的计算领域获得如此辉煌的声誉，是因为他没有告诉 PARC 的任何人，他对图形用户界面等问题的洞察力来自他一两个月前在苹果公司与拉斯金的讨论。他巧妙地让这一切看起来像是他有生以来第一次遇到位图图形用户界面，而不是和一个亲自去过 PARC 的人讨论过 T3】

<q>*—尼尔·富兰克林*</q>

无论如何，乔布斯和苹果公司的人从他们的 PARC 之旅回来时，眼里都是星星。他们着迷于“窗口 GUI”的概念，喜欢 Smalltalk 的灵活性和强大功能。他们有了一个新的愿景，并决心尽快在计算机世界中实现它。苹果公司的“丽莎”立即开始了研发。

***1979 年——苹果丽莎的诞生**T3】*

丽莎本身就值得一两段文字。乔布斯和他的伙伴们设想 Lisa(以最初的总工程师的女儿命名，也代表本地集成软件架构)作为一个新的、基于 GUI 的 PC 家族的第一个，但开发她主要是为了商业用途。值得注意的是，新的产品线紧随 1981 年苹果 III 系列的失败而来，该系列缺陷太多，不得不被召回。苹果有一些地方需要弥补。Lisa 系列采用了军马摩托罗拉 MC68000 微处理器，主频为 5MHz，内存为 512K(可升级至 2MB)，拥有苹果设计团队可以在其中填充的所有铃声和哨声，价格超过 1 万美元。Lisa 相当大而且笨重，尽管许多 PC 大战的老手坚持认为她仍然是她所制造的同类机器中最有效和最有用的一个。

Lisa 的最初开发始于 1979 年 PARC 实地考察之前(拉斯金说，Lisa 最初被设想为一台与 Apple II 类似的文本驱动 PC)，但她直到 1983 年 1 月才出现在市场上。最终，更便宜、更精简的 Lisa2 出现了，但这两个兄弟都没有在市场上卖得好——它们太贵了，Apple II 系列仍然在市场上卖得很好，即使有其他机器的竞争，如 Commodore 64 和 VIC-20、IBM PC 和 Radio Shack TRS-80。甚至在后来，在麦金塔电脑开始席卷个人电脑市场之后，苹果公司决定通过将其重新包装为“麦金塔 XL ”,来卸下他们的 Lisa 库存。“买家没有被愚弄，许多 Lisas 最终被扔进了加州的垃圾填埋场。有趣的是，Lisa 展示了一套名为“7/7”的集成软件，包括文字处理器、电子表格、图表生成器、大纲管理器、项目调度器、绘图程序和调制解调器通信实用程序。7/7 可能是第一个完整的“工程”包。

<q>*“看了[施乐之星]几个月后，我们根据从中获得的想法对用户界面做了一些修改。比如我们之前的桌面管理器就完全不一样；它根本没有使用图标，我们也一直不太喜欢它。我们决定改变我们的图标基础。我想这可能是我们从 Star 那里得到的唯一东西。我们施乐的大部分灵感都来自闲聊，而不是明星。”*T3】</q>

<q>*—Lisa 开发团队之一*</q>

乔布斯和丽莎设计团队努力将他们获得的施乐/PARC 概念整合到自己的设计中。Lisa 的 GUI 确实是基于 Smalltalk 的，因为它在 Alto 上运行，但 Lisa 的大部分设计都是苹果自己的，包括点击拖动功能和下拉菜单——这是 Macintosh 设计团队的负责人 Jef Raskin 说的，他应该知道，但其他来源认为点击拖动和下拉菜单是 PARC 的功劳。这是 PARC 的想法在苹果公司实施的又一个例子，还是并行独立开发的例子还不确定。就像他们说的，当汽船时间到的时候，它就汽船。苹果还与心理学家、艺术家、教师和普通用户合作，改善他们的界面。一个著名的例子是，苹果公司为加州的一所小学提供免费的机器供每个学生使用。在夏天，苹果程序员与老师和孩子们一起工作来改进软件和 GUI，因为他们觉得孩子们对基本的界面问题做出了最真实的反应，例如，“这些菜单东西太酷了！”或者“那张照片烂透了！”

<q>*“[ Lisa]用户将能够通过简单地指向他想要做的事情的图片来执行许多功能，而不是键入指令。”*T3】</q>

***1983 年——Mac 抵达***

乔布斯不再是苹果公司中唯一的大男子主义者(如果他曾经是)。公司高管约翰·斯卡利被请来将苹果重塑为一个“成熟”的企业，因为乔布斯糟糕的项目管理技能，他将乔布斯从 Lisa 项目中带走，并让他专注于苹果的下一个项目，一个精简且便宜得多的 Lisa“女儿”，最终被称为“麦金塔”Mac 是以队长杰夫·拉斯金最喜欢的苹果品种命名的，但为了不冒犯音频制造商麦金托什，拼写有所不同。自 1979 年 9 月开发以来，Mac 失去了 Lisa 的大部分体积和价格标签(第一台 Mac 售价 2500 美元)，是第一台具有图形用户界面的流行 PC。Mac 还捆绑了 MacPaint 和 MacWrite，前者为普通用户带来了计算机“艺术”设计(同样重要的是，在鼠标上销售给普通用户)，后者是一款简单的文字处理器，是消费市场上第一款所见即所得的产品。

拉斯金于 1982 年离开苹果，但 Mac 团队继续努力，Mac 于 1984 年 1 月上市，这是由超级碗期间播出的著名的“1984”广告预示的，该广告描绘了苹果个人电脑摧毁灰色、不知名的 IBM 计算世界。预言。许多普通用户尖叫着逃离恼人的 DOS 命令行世界，转而使用友好的 Mac GUI，尽管超级用户和 DOS 爱好者认为 Mac 是 Playskool 产品，但 Mac 的用户友好界面在计算社区的各个层面都成了朋友。Mac 后来的迭代增加了动力不足的 128k RAM，给了它与保守的 IBM 机器竞争所需的进取心。1986 年，Aldus 为 Mac 发布了桌面出版应用程序 PageMaker，Mac 突然成为每个人的图形艺术和桌面出版的首选 PC。图形用户界面风靡一时(后来在 Mac II 中增加了彩色显示器，使其更加美味)，Mac 统治了 PC 世界，微软正在努力追赶，世界一切都好。尽管乔布斯在 1985 年 5 月被无趣的首席执行官斯卡利赶出了苹果，苹果还是如虎添翼。

据说有可靠的消息来源称，麦金塔电脑中所有原创的东西都是在 PARC 炮制的，然后被大规模移植到 Mac 电脑中；其他同样“可靠”的消息来源称，Mac 实际上是苹果公司自主创造的，几乎没有受到 PARC 概念的影响。这两种想法都是错误的；显而易见，Mac 是两种创造性来源强烈融合的产物。正如拉斯金所说，“许多有才华的人对麦金塔项目和其他项目进行了多年的研究、思考和实验，但很大程度上没有被报道，尽管这些研究、思考和实验带来了突破，使麦金塔和自推出以来构建的系统比以前有了很大的改进。面对这一复杂的现实，我们有一个强大的神话形象，乔布斯从一口充满知识的井里喝水，发出“啊哈！”体验，然后全力以赴回到苹果，创造一个梦幻般的项目。“事实上，Lisa 比 Mac 更应该归功于 PARC 的设计。Lisa 的许多功能都是从 PARC 大批量借来的，包括字体和命名。正如拉斯金指出的，“当我在运行 Mac 项目时，我们在某种程度上更纯粹。”

<q>*“未来在于图形窗口界面、鼠标光标控制、下拉菜单、对话框等[以及基于此类界面的计算机]注定要接管 IBM PC 和兼容的世界。”*T3】</q>

<q>*— W.F .扎克曼，1987*</q>

***与此同时在微软…***

与此同时，在太平洋西北部，一股邪恶势力正在蠢蠢欲动……哦，拜托。将微软及其掌门人比尔·盖茨描绘成大撒旦，或者是苹果勇敢的小霍比特人索隆，都是荒谬的。两位联合创始人，乔布斯和盖茨，相似之处多于不同之处。没有一个人是纯粹的利他主义者，只是想把个人电脑带给大众，也没有一个留着黑色小胡子的恶棍，一心想要破坏。虽然我怀疑乔布斯或盖茨是否会承认道德准则，但他们都不在十大通缉名单上。两人都想在个人电脑市场上为自己开拓一席之地，两人都愿意走捷径来得到他们想要的东西，而且两人都在他们所做的事情上取得了巨大的成功。

***1977 年——微软和苹果联手***

艾伦及时重返微软，为公司在阿尔伯克基的新办公室命名。1977 年初，微软以 21，000 美元的固定费用将“AppleBASIC”授权给苹果，这被证明是一笔划算的交易，因为苹果销售了 100 多万台计算机，AppleBASIC 在其中发挥了作用(沃兹尼亚克实际上为早期的苹果编写了 integer BASIC)。到 1979 年底，微软参与了将 FORTRAN 和 COBOL 语言移植到微型计算机的工作，搬到了华盛顿州，与日本的 ASCII 公司签订了协议，并扩展到了欧洲。这个两人操作的公司现在雇用了 40 人，收入超过 700 万美元。微软与苹果的友好关系一直持续到 20 世纪 80 年代，微软于 1980 年将 Z-80 软卡赠予苹果。软卡允许 Apple II 运行目前大多数小型计算机上的大多数 CP/M 程序。

有趣的是，微软正在研究他们和蓝色巨人之间一个新操作系统的秘密交易的细节，他们称之为 DOS(磁盘操作系统)。1981 年 8 月，MS-DOS 作为第一台 IBM 机器——IBM PC——的操作系统出现。由于盖茨坚持为他的公司保留 MS-DOS 的权利，他就能把这个操作系统授权给任何数量的“克隆”计算机和应用程序制造商。IBM 试图通过发布运行他们自己版本的 PC-DOS 的机器来控制 DOS，但是由于微软愿意将 MS-DOS 许可给所有来者，PC-DOS 就再也没有流行起来。直到 1993 年，IBM 还在试图将 PC-DOS 作为微软操作系统的可行替代品进行营销，但那时 DOS 的市场吸引力正在减弱——大众市场用户喜欢各种图形用户界面，并且对进一步的命令行界面没有什么用处)。1981 年底，史蒂夫·乔布斯拜访了微软，让他们看了看雏形的 Mac，并授权微软为新的基于图形用户界面的系统开发应用程序。从 1981 年到 1984 年，微软人遍布苹果实验室，与苹果技术人员一起开发 Mac 应用程序。在这个过程中，微软非常熟悉 Mac 设计的内部运作。

上面的一个注释:微软的 DOS 1.0 代码结构实际上是数字研究公司的 CP/M 1.4 操作系统的克隆…一个消息来源称之为“bug-for-bug”拷贝。数字研究公司(DRI)开始为 16 位计算机开发一个名为 CP/M86 的更新版本，用于采用 Intel 8086 处理器的机器；对 DRI 来说不幸的是，当 IBM 来寻找操作系统时，CP/M86 还没有准备好，他们用微软的 DOS 代替了。1982 年，Digital Research 最终发布了 CP/M86，并于 1987 年将其转换为他们自己的 DR-DOS 系统。Digital Research 因 CP/M-DOS 纠纷起诉微软，但诉讼失败了。一个对微软非常敌对的消息来源声称，微软在 DR-DOS 发布时竭尽全力破坏它，包括制造虚假的声明说 Windows 不能在 DR-DOS 下运行，以及将他们自己的 MS-DOS 更新推向市场，从 Digital Research 的产品中去掉腿，并使用非法营销手段迫使 PC 制造商使用他们自己的系统代替 DR-DOS。

当然，这不是这个故事的唯一版本，但赤裸裸的事实是，DR-DOS 从未像微软的竞争对手 MS-DOS 那样影响市场，至少部分是因为微软积极地、可能是不光彩地试图将自己的系统推过 DRI 的系统。数字研究公司后来在 1991 年将 DR-DOS 卖给了 Novell。在试图将它集成到他们自己的网络操作系统中并以“Novell DOS”的名称发布版本后，Novell 于 1996 年将它卖给了 Caldera，这是 Novell 最后一次尝试与 DR-DOS 合作后的三年。卡尔德拉将 DR-DOS 变成了一个开源产品，叫做 OpenDOS。卡尔德拉还起诉微软在 DR-DOS 事件中的非法营销行为，微软于 2000 年 1 月庭外和解。如果 IBM 打电话来时 CP/M86 已经准备好使用，微软可能永远也不会与 IBM 合作，将它推向辉煌，我们今天都会诅咒数字研究，而不是微软。谁能说呢？

1983 年 11 月，在 Macintosh 正式上市的两个月前，微软宣布它正在开发自己的基于 GUI 的操作系统(实际上，是一个在 DOS 操作系统上运行的“外壳”)，被称为“Windows”(盖茨想称之为“界面管理器”，但更聪明的人占了上风)。微软在 1983 年 4 月就已经引起了轰动，它给出了一个原型界面管理器的“烟雾-镜子”演示，使用重叠窗口来模拟同时运行的多个程序。IBM 高管对微软的小玩具不满意，因为他们正在开发自己的基于 DOS 的程序管理器，称为“Top View”盖茨曾多次试图让 IBM 对 Windows 感兴趣，但每次都遭到拒绝；IBM 认为对 GUI 的兴趣只是一个过渡阶段。《俯视图》于 1985 年发行，1987 年停产；它的图形界面影响了 IBM 更引人注目的 OS/2，尽管 GUI 驱动的版本从未公开。

1985 年 11 月，Windows 1.0 在发布近两年后正式亮相。苹果对 Mac 和 Windows 界面的相似性感到震惊，但由于几乎没有适用于 Windows 环境的应用程序(Aldus 的 PageMaker for Windows 是一个明显的例外)，Win 1.0 在消费者市场上没有太多宣传。Win 1.0 未能获得可观的市场份额，加上 Mac 销量停滞不前，这让一些人怀疑 GUI 热潮是否已经达到顶峰。具有讽刺意味的是，鉴于两家公司之间的嫌隙，微软的 Excel(一种基于 GUI 的电子表格，类似于其前身 VisiCalc，但更容易使用)在这个时候给了 Mac 急需的生存能力。

***1983 年起——其他图形用户界面上市***

市场上只有 Mac 和 Windows 图形用户界面吗？几乎没有。事实上，第一个面向消费者的、基于 PC 的 GUI 不是由这两家公司制造的，而是由 VisiCalc 的制造商 VisiCorp 制造的。它名为 VisiOn，于 1983 年 10 月在 Lisa 之前不久首次亮相，但由于缺乏在其上运行的流行软件而陷入瘫痪。DRI 的 GEM(图形环境管理器)也是如此，它出现在 1984 年 9 月，但很快就消失了，部分原因是它和 VisiOn 一样，没有运行 DOS 应用程序的能力，也没有自己的软件。GEM 的运气更差:苹果不喜欢 GEM 与 Mac 桌面的相似性，并威胁要起诉。DRI 没有抗争，而是改造了 GEM 桌面，让苹果不再纠缠不清。VisiOn 和 GEM 都有其支持者，但都没有在消费者市场上取得重大进展，消费者市场继续由苹果和微软这两大巨头主导。

还有 Quarterdeck 的 DESQView，这是第一个将多任务和窗口功能引入 DOS 环境的程序。DESQView 并不是一个成熟的 GUI 操作系统，但它在 DOS 上的 GUI“外壳”赢得了许多粉丝，并引起了微软许多人的兴趣，包括盖茨，据一些人说，他的第一次 Windows 迭代基于 VisiOn、GEM 和 DESQView，而不是 Mac 界面(这与苹果粉丝在篝火边流传的故事相矛盾，他们把盖茨描绘成一个小偷，对他的 Windows 团队咆哮说:“让它看起来像 Mac！”).Berkeley Softworks 的 GeoWorks (GEOS)是另一个值得注意的 GUI OS 它被用在 Commodore 64 和一些苹果的 IIs 上，并作为 PalmConnect 系统的软件以一种改变的形式存在。GEOS 被称赞为一个光滑、稳定的操作系统，但它缺乏软件——开发者软件在 GEOS 首次亮相后六个月才出现——这确保了大多数 PC 用户不再考虑它。

苹果对 Windows 一点也不满意。甚至在该系统上架之前，苹果公司就已经威胁微软，称其侵犯专利、窃取知识产权等等。在一个巧妙的举动中，微软与苹果签署了一份许可协议，声明微软不会在 Windows 1.0 中使用苹果的技术，但不会在 Windows 的后续版本中使用苹果的技术。苹果花了一段时间才意识到微软彻底欺骗了他们；由于 Windows 在消费者市场上的惨淡失败，这种意识过了很久才显现出来。

然而，苹果和微软都在推进他们自己统治世界的计划……呃，更确切地说，是他们拓展个人电脑市场的计划。不过，和往常一样，这两支乐队并不是游行队伍中唯一的乐队。1985 年，Commodore 推出了 Amiga 系列家用电脑，赢得了数百万用户的青睐。Amiga 是第一台真正将“多媒体”概念引入 PC-dom 的 PC，尽管由于大多数用户不知道如何利用新的多媒体功能，他们转而在上面玩游戏。好看的游戏。Amiga 先进的声音和视频功能与其复杂的 GUI 驱动的操作系统相得益彰(该操作系统还具有抢先式多任务处理、共享库、消息传递、脚本编写、多个同步线路控制台、真正使用鼠标右键以及当时的苹果和 IBM 所没有的其他功能)。雪上加霜的是，Amiga 还推出了苹果/IBM 界面仿真。喜欢旧界面的苹果或 IBM 用户可以让 Amiga 模仿这个界面，而不是自己的界面。

那么为什么 Amiga 没有把苹果和 IBM/微软都从 PC 市场上抹去呢？像往常一样，我们有各种各样的理由。最好的猜测是，阿米加犯了与塔克乘用车相同的错误……它走在了时代的前面，走得太快，没有利用自身的能力。Amiga 和 Atari 之间的激烈竞争对微软有利，Amiga 让经销商和客户满意的能力也不尽如人意。除了 Amiga 的问题之外，第一台机器未能采用单一的 GUI(一位 Amiga 用户告诉我，早期的型号根据运行的程序有不同的界面)。但不管是什么原因，阿米加是一只聪明的小狗，应该得到更好的命运——尽管今天阿米加既没有消失，也没有被遗忘；一种叫做“数字环境”的新操作系统被吹捧为图形用户界面驱动的操作系统的下一步。在一切都结束之前，我们可能会再次听到阿米加的消息。

80 年代中期 GUI 大战中的另一个竞争者是 Atari ST. Atari，它以其视频游戏而闻名，生产了一台装有 GEM OS 的个人电脑。像 Amiga 一样，ST 无法与大男孩竞争，也无法与 Amiga 争夺游戏玩家，但其复杂的声音处理能力为它赢得了音频编辑和音乐家的一席之地。

在 Amiga 首次亮相的时候，第一个 UNIX GUI 也出现了。许多 UNIX 主管长期以来对充斥消费者市场的头脑简单、过于复杂的操作系统和 playtoy PCs 嗤之以鼻。但是一些 UNIX 用户决定看看他们是否能在 UNIX 上覆盖一个 GUI，就像微软在 DOS 上覆盖 Windows 一样，于是 X 诞生了。X(有时被称为“X Windows”，有时被错误地称为“X for Windows”)出生于麻省理工学院，父亲是斯坦福大学的窗口系统 W，母亲是 Sun 的“SunView”环境。x 成为大多数基于 RISC 的 UNIX 操作系统的主要图形系统。虽然 X 是一个编写良好且易于操作的操作系统外壳，但它从未固定在一个特定的“外观和感觉”上，因此至少有三种不同的界面或“窗口管理器”为它而浮动。

这不是 X 从未在 UNIX 社区之外流行起来的主要原因，但这肯定是一个原因。x 仍然是一个可行的 GUI shell，并且有一个相对较小但活跃的追随者。x 正在 UNIX 用户中引起某种程度的复兴:“windows 管理器”之间的斗争已经结束，界面更加完善，更易于使用，并且对于高端计算机图形制作非常有用。x 也是大多数 Linux 图形界面的底层 GUI。“几个图形用户界面”更确切地说是各种*nix 窗口管理器，用户可以运行桌面环境，如 Gnome 或 KDE，以获得额外的功能。X 驱动的界面正在诸如 TiVo、Web pads 和 PDA 等非 PC 设备中出现，一位 X 用户推测，随着这些设备变得越来越普及，我们可能会看到 X 实际上比 Apple 或 Windows GUIs 使用得更多。

还值得一提的是，三河计算公司在 1981 年制造了一个名为 PERQ 的图形工作站，它集成了一个基于 UNIX 的 GUI，并由 ICL 在英国销售。这个 GUI 实际上比上述所有的都要早，包括 VisiOn，但是据我所知，它从来没有在个人电脑上出现过。

Completists 将指出，IBM 的 MVS(多虚拟系统)主机系统包括一个名为 ISPF(交互式结构化程序设计工具)的可选程序，它允许在终端显示器上支持分屏窗口。考虑到 ISPF 创建于 20 世纪 70 年代末，它是第一个可用的“类 windows”系统之一。当然，家庭电脑用户不太可能看到这种情况。

值得注意的是，许多图形驱动的应用程序是独立于上述任何系统发布的。最早的一个是 Bill Budge 的弹球构建系统，它于 1985 年出现在 Atari 上，并因其使用点击和拖动操作对象的复杂能力而在游戏玩家和程序员中迅速成名。像个人电脑这样的程序在随后进入个人电脑市场的操作系统上留下了印记。

<q>*“我对微软有着巨大的好感，因为只有它——不像施乐、苹果、Amiga 和其他许多在它之前尝试过的公司——最终在无处不在的廉价硬件上提供了可用的图形界面。微软通常不是第一家，它的软件通常也不是最好的，但毫无疑问，它以其他软件制造商无法做到的方式实现了个人计算的早期承诺。微软比任何其他公司都更为普通人打开了计算之门。为此我爱微软。”*T3】</q>

 *弗雷德·兰加*  *回到大人物。1987 年 12 月，Windows 2.0 发布，令苹果公司惊愕不已，但消费者却打了个哈欠。尽管 Win 2.0 看起来比以往任何时候都更像 Mac，图标代表文件和程序，层叠式窗口等，但 Mac 用户并没有离开苹果群体去购买 IBM/Windows 机器(特别是因为 hunky Mac II 已经摆满了货架)。苹果犹豫了几个月，最终对微软提起了长期威胁的诉讼，声称 Windows 窃取了 Macintosh 的“外观和感觉”1988 年，市场几乎忽略了苹果 IIGS 的 GS/OS，但 Mac 继续主导市场。到 1989 年，人们普遍认为 Windows 是一个巨大的失败。微软继续与 IBM 合作开发完全图形化的操作系统，但继续冲击 Windows，希望最终得到一个正确的版本。正如老话所说，“即使是一只瞎松鼠也会时不时地找到一颗橡子。”雷德蒙的橡子时间即将到来。

***90 年代的 Windows***

<q>*“我认为 Windows 3.0 会得到很多关注；人们会检查它，不久他们都会回到原始的 DOS。他们偶尔会出于某种特定目的启动 Windows，但很多人会把它和 Commodore 64 一起放在壁橱里。”*T3】</q>

<q>*—约翰·德沃夏克，1990*</q>

1990 年 5 月，一场盛大的、精心策划的喧嚣将 Windows 3.0 推向了市场。史蒂夫·鲍尔默带头高呼“Windows！Windows！Windows！”在华盛顿州雷蒙德市的微软总部；过去是，现在也是，微软的营销机器照顾了世界的其他地方。微软在发布新版 Windows 的同时，推出了数十款专为 Windows 编写的应用程序，现在推出了 OS/2 风格的“雕刻按钮”——这要归功于图标设计师 Susan Kare，她设计了更具吸引力的按钮风格；Kare 还在 Mac 上工作——更多的颜色支持，真正的多任务处理，以及一个大大改进的程序管理器，等等。这些新功能和新软件终于引起了市场的关注。由于自身兼容 Win 的 Word 和 Excel 版本以及许多其他第三方应用程序的流行，微软在发布的第一年就售出了超过 300 万份 Win 3.0，苹果感到了寒意。win 3.1(1992 年 4 月)增加了可扩展的 TrueType 字体支持和更好的多媒体功能，苹果公司当时正在努力。装有 Windows 系统的个人电脑销量首次超过了 MAC 电脑。被称为“工作组的 Windows”的 Windows 3.1.1 在企业界表现相对较好，同时也将其大部分设计留给了 Windows 的后续版本。

1993 年出现了 Windows NT(新技术)的第一个版本，它抛弃了其前辈容易崩溃的内核，而采用了一个新的、更稳定的内核。NT 最初是 IBM OS/2 系统的新版本，是微软和 IBM 合资企业的一部分。它最初被称为 OS/2 3.0 或 OS/NT，但在早期开发期间，微软和 IBM 分裂了，微软放弃了该程序，将 IBM 的 OS 概念与他们自己的结合起来，重写了代码，并最终在 Windows 保护伞下发布。问题是，微软的营销人员无法决定如何处理它。很明显，它对商业用途更有用，所以，作为微软，他们试图把它卖给除了商业用户以外的任何人。它很快成为众所周知的“不感谢 Windows ”,狡猾的观察家认为微软搬起石头砸了自己的脚。没那么快…原来很多使用 Unix 的人已经决定尝试一下 NT，并且喜欢他们所看到的。到 1995 年初，许多欧洲公司已经转移到 NT 的第二个“官方”版本 3.5。到 1995 年中期，NT 已经在技术和商业用户中确立了自己的地位，到“防弹的”(意思是:几乎防碰撞的)版本 3.51 问世时，Win NT 已经根深蒂固。NT 在公司和办公室环境中工作得很好，但在家庭中就不那么好了:它不是为运行旧的基于 DOS 的软件而设计的，这使它成为游戏玩家和没有尖端软件的人的错误选择。

诉讼也不顺利。苹果的策略是证明 Windows 非法抄袭了 Mac GUI 的“外观和感觉”,但在微软的律师指出这两个系统都“借鉴”了施乐最初的概念后，这一策略出现了一些重大漏洞。对于乔布斯的偷窃指控，盖茨做出了严厉的反驳，“不，史蒂夫，我认为这更像是我们都有一个叫施乐的富裕邻居，你闯进来偷电视机，你发现我先去了，你说，‘嘿，这不公平！“我想偷电视机，”“Windows 的界面设计看起来更像旧的 Alto GUI，而不是 Mac 的设计，这一事实对苹果的情况没有帮助。突然间，微软成了热门话题，而苹果似乎成了昨日黄花。

***苹果的火热之旅***

这场旷日持久的诉讼最终在 1993 年 6 月以有利于微软的方式解决，末日预言家认为苹果的命运已经注定。沃兹尼亚克和乔布斯早就走了。该公司陷入了财务困境(尽管即将破产的报道是错误的)。他们期待已久的“牛顿”个人数据助理失败了。管理层似乎更感兴趣的是内部斗争，而不是纠正公司。由于生产问题，订单没有兑现。1995 年 8 月，当微软发布其开创性的 Windows 95 操作系统时，一些人预测苹果将变得完全无关紧要。Win 95 是第一个利用英特尔强大的 32 位芯片的操作系统，也是 Mac GUI 的近似克隆，它似乎是注定最终将苹果永远赶出市场的不可抗拒的力量。苹果试图通过将其低端 PC(基本上是旧的，重新包装的 MAC)的“Performa”系列推向其高端“PowerPC”系列来弥补，但惨败——Performas 坐在苹果仓库里积灰，而买家发现很难(如果不是不可能的话)买到他们想要的 PowerPC。越来越多的首次用户选择 Windows 驱动的个人电脑，而不是苹果，这在很大程度上是因为 Windows 克隆制造商之间的激烈竞争使 Windows 机器的价格相对具有竞争力，而苹果无情地拒绝让其他人制造克隆产品(1994 年只是部分放松，1999 年又突然取消)，令人尴尬的质量控制问题，以及相对较高的标价，都让许多购买者对苹果这个名字产生了反感。微软决定在 Win NT 4.0 上安装 Win 95 界面的修改版，这一决定提高了 NT 平台的受欢迎程度，并从苹果公司的市场份额中进一步减少。

1994 年 10 月，IBM 试图用自己的操作系统 OS/2 Warp 的第三个版本夺取微软的部分市场份额，但没有成功。OS/2 最初是微软/IBM 的合资企业，但 Warp 是 IBM 自己的产品，具有类似 Windows 的 GUI。它设法维持下去并赢得了一些忠诚度，但对于不想使用 Windows 的 IBM 机器用户来说，它从来没有真正成为一个更弱的选择。当我们谈到弱替代品的话题时，现在是时候给微软 Bob 一个讽刺的点头了，这是“下一代”GUI，它骑在 Windows 之上，用一个快乐、健谈的虚拟助手 Bob“帮助”新手用户。鲍勃喝得酩酊大醉，成了电脑行家们的笑柄。他唯一的遗产是同样令人讨厌的 Office 助手，即目前困扰微软 Office 的“跳舞回形针”。

与此同时，苹果还没有去核。史蒂夫·乔布斯创办了一家名为 NeXT 的公司，虽然 NeXT 电脑在市场上失败了，但时尚而复杂的 NeXT 操作系统(一种基于 UNIX MACH 内核的操作系统，具有神话般的 GUI)对苹果公司非常有吸引力。苹果的 Mac 操作系统已经露出了它的灰色头发，苹果想要一些新的和耀眼的东西来对抗微软的单一产品。1996 年 12 月，苹果收购了 NeXT，从而收购了 NeXTStep，其元素将出现在新的 Mac OS Rhapsody 中。乔布斯搭上了顺风车，没过多久，他再次执掌了自己创立的公司。

苹果粉丝对他们英雄的回归并不高兴太久。1997 年 8 月，乔布斯宣布与微软正式合作，这让普通员工感到沮丧。微软购买了 1.5 亿美元的苹果股票，两家公司都同意一劳永逸地结束 GUI 纠纷。许多不满的苹果用户，已经被苹果持续无法制造足够的机器来满足订单，以及苹果未能授权苹果系统来克隆制造商所困扰，一劳永逸地跳出了苹果潮流；并非巧合的是，人们对 Be 和 Linux 等“另类”操作系统兴趣的新一轮高涨，与乔布斯认为苹果将被“出卖”给微软的时间差不多。当然，逃离家园的前苹果爪牙们并没有启动 Linux/Be/etc。但他们也做出了自己的贡献，尤其是当苹果公司撤回对 Be 平台的支持时，该平台原本是为 PowerPC 设计的。

不过，请注意，大多数苹果迷并不仅仅是因为苹果“出卖”给了微软。苹果社区中更有见识的人明白，微软的承诺意味着苹果更稳定，以及继续为 Mac 开发 MS Office。微软 1.5 亿美元的收购只是苹果股票基数的一小部分，而且微软的股票严格来说是没有投票权的。根据一些故事，苹果粉丝的反应类似于教堂会众的牧师宣布别西卜被任命为首席执事。但如此多苹果粉丝不满的真正原因要复杂得多。

随着 Mac OS 8“Platinum”的成功发布，苹果取得了进展，这是一款流行而稳定的操作系统。不幸的是，在 OS 8 之后的混乱中，一些基础已经丢失了。最初的想法是给苹果用户一个名为 Copland 的“下一代”系统。苹果公司没有在 1996 年年中发布它，而是压制了这个项目，支持与新收购的 NeXT OS 一起工作。苹果随后宣布了一个正在开发的新系统 Rhapsody，它将在 UNIX 核心之上结合下一代操作系统的元素。前面提到的 OS 8 出现在 1997 年 7 月，并采用了一些来自科普兰的更受欢迎的元素。Rhapsody 也没能实现，最终变形为 1998 年 5 月宣布的 OS X 计划。1999 年 10 月发布了“白金”系统的升级版 OS 9。下一代 OS X 终于在 2001 年 3 月发布，许多承诺给 Rhapsody 的功能都出现在这个系统中。

1998 年，苹果用现在无处不在的 iMac 彻底改造了个人电脑市场。iMac 凭借其可爱的配色方案和人性化的设计赢得了许多用户的心，也遭到了许多评论者的鄙视。“严肃的”苹果用户被越来越强大的 PowerMac 系列所吸引，但数百万人选择将可爱的小 iMacs 放在办公桌上。随着 PowerMac 和 PowerBook 笔记本电脑的适度成功，iMac 给了苹果急需的充电。iMac 安装了 OS 9 或 OS X。其他人模仿苹果的趋势仍在继续，大宇的 eMachines 因外观和行为像 iMac 而受到苹果的抨击。这一次苹果占了上风。

***今天……***

在雷德蒙，微软步履维艰。它在 1998 年 6 月推出的大肆宣传的 Windows 98 版本没有达到预期。当 Win 98 被证明是一个升级而不是一个突破性的新产品时，大多数被引导期待一个革命性新产品的用户感到恼火。人们对 Windows 2000 或“Win 2K”的感觉大同小异，这是 NT 系列的最新和最后一次迭代，对 Windows Millennium 的感觉更是如此，在大肆宣传之后，它只不过是 Win 98 的一次小升级。微软把自己操作系统的命运押在了即将推出的 Windows XP 系统中 9x 和 NT 系列的整合上。微软在最近的法庭诉讼中表现不佳，司法部做了苹果未能做到的事情，获得了一项裁决，即微软是垄断企业，其行为违背了市场和竞争的最佳利益(尽管在我撰写本文时，DOJ 已经放弃了拆分微软的尝试)。苹果公司继续推出他们最新的 Mac OS 版本 OS X，并承诺在你读到这篇文章的时候推出新版本 10.1。两家公司都准备在可预见的未来继续主导个人电脑市场，尽管 UNIX 的支持者依然坚定，Linux 也在稳步发展。

GUI 的未来呢？嗯，考虑到世界上超过 90%的用户使用一种或另一种图形用户界面操作系统，考虑到像 Be 这样的新的图形用户界面驱动的操作系统似乎正在流行，考虑到这一季的宠儿 Linux 通常与 Caldera、Corel 和 Red Hat 等公司的图形用户界面一起使用，图形用户界面的未来似乎是安全的。盖茨同志和其他人已经提出了一个更“复杂”的界面，有语音识别、触摸屏、视网膜和指纹安全扫描、全息表现和虚拟“化身”，与用户的互动比，比如说，该死的 MS Office 回形针更直接。我会很高兴的。我不知道格鲁会怎么想。
