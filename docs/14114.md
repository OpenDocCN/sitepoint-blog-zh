# 倾斜！桌子偷偷溜进后门

> 原文：<https://www.sitepoint.com/tilt-tables-sneak-in-the-backdoor/>

已经有很多关于使用 DOM 改造 HTML 以使其工作起来更好看的文章。我们不知道我们正在修补的黑暗力量。渴望简单的基于表格的布局，[迪米特里·格拉兹科夫](http://glazkov.com/blog/)想出了一个巧妙的方法，在父母不注意的时候把桌子塞给孩子们。

[TILT](http://glazkov.com/blog/archive/2005/05/02/476.aspx) 或[“表格注入布局技术”](http://glazkov.com/blog/archive/2005/05/02/476.aspx)采用一个健康的、结构良好的 DIVs 页面，并在其中重新调整一个老式的表格布局。

你可以在这里看到它的作用[。正常查看源代码，您会看到发送的标记。但是在 Firefox 中查看页面，点击 CTRL+A(全选)，然后右击选择“查看选择源”,你会看到他们试图看起来很无辜。桌子。](http://www.glazkov.com/Resources/Code/Tilt/Example.html)

我不得不承认我对此有点犹豫不决，但这确实非常聪明。甚至他对这种技术最初的名字也是一个笑话。*…肉毒杆菌素，一种快速抚平讨厌的老皱纹的方法，通过将通常有毒的物质注射到你页面的面部区域*。

但最终，在我看来，这就像是为轻微的鼻塞开化疗处方。这相当极端，可能有害，并带来一系列令人讨厌的副作用。这些包括:

1)可访问性问题:大多数屏幕阅读器都能理解 JavaScript，因为表格不利于可访问性，所以代码在“去除”好的标记方面做得很好。

2) DOM 结构问题:如果你的 CSS 或其他 JavaScript 依赖于一个特定的结构，对它进行大的动态修改显然会破坏东西。这也意味着如果你打算照顾你的缺乏 JS 的用户，你将需要两套 CSS..} '和' *div#page H3 {..}*’)。

3)源代码排序问题:TILT 似乎会强迫你按照代码中出现的顺序来呈现页面元素(就像 table 一样)，这剥夺了 CSS 的一个优点。

不过，这至少是一个伟大的实验——你不得不佩服 Dimitri 提出一种全新方法来解决成千上万开发人员必须面对的问题的能力。
