# 便携式社交网络:带上你的朋友

> 原文:[https://www.sitepoint.com/social-networks-take-friends/](https://www.sitepoint.com/social-networks-take-friends/)

你知道世界上 98.6%的人口都在脸书吗？这当然不是真的，但是记者会让你相信的！两年前，他们对 MySpace 的发展也说了同样的话。但是两年后，一旦你厌倦了脸书，你的朋友们催促你加入另一个新网站，你真的想从头开始吗？你真的想重新建立那些关系吗？信息已经存在；为什么不能跨转？

你可以的！这就是便携式社交网络的用武之地。

##### 什么是社交网络？

在我们能够使社交网络可移植之前，我们需要知道什么是社交网络。[维基百科将社交网络](http://en.wikipedia.org/wiki/Social_network)描述为:

<q>[……]由节点(通常是个人或组织)组成的社会结构，这些节点由一种或多种特定类型的相互依存关系联系在一起，如价值观、愿景、想法、金融交易、朋友、亲属关系、不喜欢等等。</q>

这是一个非常笼统的定义。许多人都知道斯坦利·米尔格拉姆著名的小世界实验——更普遍的说法是“六度分离”——米尔格拉姆假设(后来通过实验证明)你与地球上任何其他人的联系不超过五个人。人们之间的这些联系定义了社交网络——你与其他人的联系由你通过这些联系从最佳来源检索信息的能力来定义。

在准备这篇文章的过程中，我很快统计了我经常使用的社交网站的数量。至少有 20 家——这还不包括像脸书和 MySpace 这样更受欢迎的网站。尽管我有技术倾向，但我怀疑我是否能很好地衡量一个典型的人的社交网络习惯，但可以肯定地说，在一生中，一个普通的网络用户会加入许多不同的网站。随着时尚的改变，他们会想从一个网站迁移到另一个网站:他们会去他们朋友去的地方，在他们朋友聊天的地方聊天。

如果你曾经决定加入一个新的网络，你会知道从头开始是多么令人沮丧。你需要重新结交同样的朋友，联系同样的熟人。这样做几次，你会发现自己患上了严重的社交网络疲劳症。

最重要的是，你必须说服你的朋友也加入这项新服务；克服这种惰性所需的努力通常太多了。迁移数据，说服你的朋友，调整偏好…这一切都很麻烦，所以人们不会这样做。如果在额外努力和现状之间做出选择，他们会坚持现状。

##### 数据的自由移动

数据的自由流动是社交网络可移植的原因。从供应商和消费者的角度来看，轻松地将数据导入和导出应用程序的能力都是关键。

自由移动数据的能力有两个重要的含义。首先，你更有可能加入一个尊重你的时间和数据的服务。其次，它有助于保持公司的诚实。

我加入并享受 Flickr 的原因之一是，我可以随时从服务中取回自己的数据，然后离开。这不仅仅影响我的照片:它包括与我的朋友相关的数据，我的评论，以及丰富我的网络的所有其他元数据。如果 Flickr 变得邪恶或做出糟糕的决定，用户可以去别的地方。Flickr 的团队知道这一点，并确保公司关注用户的最大利益——否则，Flickr 就不会有太多的客户了！

拥有数据的对立面是供应商锁定。一些供应商阻止您自由移动您的数据。他们试图通过使用专有格式、冗长的合同或缺乏出口功能来强行留住你。你被阻止带走你的数据，希望移动数据的痛苦会超过他们的产品给你带来的痛苦，所以你留下来。

但是，供应商锁定和专有格式并不能保证是财务成功的关键——这是越来越多的公司每天都在利用的事实。随着我们将越来越多的生活、联系和数据放到网上，寿命和所有权已经成为社交网络应用成功的关键。作为我们数据的所有者，我们应该要求能够按照我们认为合适的方式将数据从一个地方移动到另一个地方。成为便携式社交网络的一部分意味着你可以把你的数据带到任何你想去的地方。你花费了大量的时间和精力来收集、整理、注释和组织这些数据——不要忘记这些数据是你的。

##### 让生活更轻松

获得新客户的最好方法是让他们很容易加入。较低的准入门槛使得注册更容易，这意味着会有更多的人尝试这项服务。在你的注册过程中，必须有尽可能少的障碍、要求和问题。您向客户展示的每一个复选框、下拉框、表单字段和按钮都会失去潜在的业务。有新的和旧的技术试图解决降低壁垒的问题，其中一些已被证明比其他更成功。

***一次登录***

最近一个简单注册技术的例子是 OpenID。简单地说，OpenID 是一个密码的分布式系统。它允许您维护一个单独的用户名和密码，您可以用它来登录任何理解 OpenID 的站点。没有许可费，没有安装费，任何人都可以接受 OpenID 登录。

当我访问 [Dopplr](http://dopplr.com) 时，我可以使用 OpenID 登录。 [Backpack](http://www.backpackit.com) 也有一个 OpenID 登录选项， [Magnolia](http://ma.gnolia.com) 也有，所以我可以对所有三个网站使用相同的 OpenID 登录细节。OpenID 很出色，因为我现在只需要管理多个网站上的一个用户名和密码。

为什么一个社交网站要允许客户用 OpenID 登录？因为这是降低加入门槛的第一步。当您使用 OpenID 创建一个新的社交网络帐户时，您第一次登录社交网站，OpenID 会发回一些数据:您的 URL、您的姓名(可选)、您的昵称和您的电子邮件地址。这些数据用于在社交网站上自动为您创建一个帐户。因此，花费在重新输入关于你的重复数据上的时间被最小化。

如果我希望我的 web 应用程序或社交网站得到尽可能广泛的采用，让人们更容易、更舒适地加入和创建帐户将是第一步。OpenID 还有一些非常有趣的特性，比如白名单。一个社交网络可以公开列出一个群组中的人的 OpenID URLs。其他社交网络可以使用 OpenID URLs 列表来“预授权”账户，进一步加快注册过程。

人们使用这种预授权功能来发表博客评论。通常情况下，如果我在没有账号的情况下在博客上发表评论，那么该评论在发布到博客之前需要得到授权。相反，如果我留下评论，并使用我的 OpenID URL 来验证我自己，该网站可以引用一个预先授权的可信 URL 列表，并让那些来自自动匹配来源的评论在没有明确授权的情况下发布。在你的网站中，OpenID 功能还有很多其他用途；Simon Willison 的一篇名为“[你可以用 OpenID](http://simonwillison.net/2007/Feb/25/six/) 构建的六个很酷的东西”的短文解释了其中的一些。

作为客户，OpenID 对我来说也很棒。如果我想更改我所有 20 个社交网站的密码，我只需通过我的 OpenID 提供商更改一次我的密码，它就会反映在所有这些网站上。这是因为他们不存储我的密码，只存储我的 OpenID 网址。它还允许我快速创建账户，看看这项新服务是否值得。我可能已经错过了很多寻找酷东西的机会，因为我懒得再输入我的详细信息了！

我建议你今天就研究并注册一个 [OpenID](http://openid.net) 账户。OpenID 是一个很好的开始，但是降低便携式社交网络障碍的下一步是带上你的朋友们！

***分享你的人脉***

当从一个社交网站跳到另一个网站时，重新邀请你所有的朋友并重建这些联系是一件痛苦的事情。怎么可能把我们所有的朋友和关系也带来呢？嗯，这实际上很简单——一切都归结于 HTML 属性、微格式和一种叫做 XFN 的东西的迷人世界。使用下面的技术，有可能创建开放数据，这有助于移植，而没有比我们今天拥有的基本 HTML 更新或更复杂的东西。

**XFN**

XFN 代表 XHTML 朋友网络。它是一个 rel 值列表，通过 URL 定义两个人之间的关系。这些联系从简单的“联系”到更复杂的关系，如“配偶”。Rel 值也可以用空格分隔，所以我们用像`rel="contact met friend neighbor"`这样的值来定义更复杂的关系。这些价值观开始成为便携式社交网络的支柱。

如果我们在 HTML 中定义关系，其他服务可以提取信息和语义，这样它们就可以在其他网络中使用。开放数据是一条双行道——用 XFN 标记的用于显示和导出的相同 HTML 文件也可以用于导入另一个服务。
示例 blogroll:

```
<a href="http://example.org/blog/joe" rel="friend met">Joe Morris</a> 

<a href="http://blog.terryj.com/" rel="friend met colleague">Terry Jacobs</a>
```

您可以在 http://gmpg.org/xfn/creator 访问 XFN 关系构建器。

**HTML 相关值**

除了已经创建的 XFN 值，HTML 规范还定义了几个本地 rel 属性，这里有一个[的完整列表](https://www.w3.org/TR/html401/types.html#h-6.12)。该列表包括帮助、术语表、索引和开始等值。在收集开放数据时，有趣的值是 Next 和 Prev。如果一个蜘蛛来到 XFN 数据所在的 URL，并遇到一个包含`rel="next"`的链接，这个蜘蛛就知道有另一个页面可能包含 XFN 数据。反过来也一样:如果蜘蛛已经在第 3 页着陆，一个`rel="prev"`链接告诉蜘蛛返回一页以获得更多的 XFN 数据。这些 rel 值对于分页非常有帮助。

Twitter.com 利用你朋友列表中的`rel="next"`值。你的每个朋友链接都用 XFN 微格式标记，描述你和那个朋友之间的关系。如果你有多个好友页面，那么`rel="next"`和`rel="prev"`属性可以让蜘蛛知道这些页面都是相互连接的，所有收集到的数据都可以汇集在一起。

**身份巩固**

`rel="me"`值是后来添加到 XFN 中的，它是一个强大的值，允许您执行身份整合。这项任务包括几个简单的步骤:

*   以你的博客为起点，链接到你所属的所有社交网站上你的页面。现在只需将`rel="me"`值添加到这些链接中。
*   转到您的个人资料页面，用相同的`rel="me"`值将链接添加回您的博客。

搞定了。你现在已经将这些网站称为“你”，当你在互联网上的不同网站上为自己建立一个单一的身份时，这很有用。GMPG(全球多媒体协议组织)向[提供了关于`rel="me"`和身份整合](http://gmpg.org/xfn/and/#idconsolidation)的更多信息。而 [Plaxo Pulse Open Graph 搜索工具](http://www.plaxo.com/info/opensocialgraph)可以帮助进行整合。输入一个开始网址，它将抓取 rel="me "链接，并开始整合关于你的信息。

**hCard**

在我之前的文章中，我提到了什么是 hCard，以及如何创建 hCard。简而言之，hCard 是在结构化(X)HTML 中表示人、公司、组织和地点的一种方式。

##### 开放数据

像 [Firefox plugin Operator](https://addons.mozilla.org/en-US/firefox/addon/4106) 这样的工具可以提取 hCard 和 XFN 数据，并以有用的方式将其与其他网站结合起来。

当将数据导入一个新的社交网络时，像 XFN 和 hCard 这样的格式允许你描述这个人，以及他或她与你的关系。XFN 是用来找到你认为是朋友的人的链接。hCard 用于提取这些朋友的结构化信息，包括 URL、电子邮件地址、组织机构和电话号码等，所有这些都可以通过使用开放数据迁移到其他服务中。

***允许导入数据的站点***

作为帮助用户避免一遍又一遍地重复输入信息的繁琐任务的一个例子，一个名为 GetSatisfaction.com 的网站允许他们根据 hCard 的 URL 创建用户档案。用户只需输入他们的 hCard URLs，而不是填写关于他们的姓名、地址、电话号码和电子邮件地址的乏味的公共表单字段。GetSatisfaction 将获取 HTML 页面，解析它以获取 hCard 数据，并填充相应的字段。几个网站已经有了标记为 hCards 的联系信息。例如，如果你有一个 Flickr 账户，你的个人资料页面是用 hCard 编码的。所以你可以输入你的 Flickr 个人资料的 URL，或者你博客上的一个页面。

理想情况下，GetSatisfaction 会记住这个 URL，如果你更改了 Flickr 个人资料页面上的数据，GetSatisfaction 也会更新它的信息。很像 RSS 的发布/订阅模式，GetSatisfaction 和其他人可以订阅你在 hCard 中编码的远程联系人资料数据。然后，您只有一个需要更新的单点数据，所有的更改都会在全局范围内得到反映。

新的微格式和社交网络相关工具不断涌现。例如，Dopplr 为用户提供搜索可能也在使用这项服务的朋友的能力。这个简单的工具可以帮助您查看您认识的人是否是该服务的成员；然后你就可以和他们交流了。这项服务的缺点是，我需要手动输入每个朋友的名字，以便进行搜索。我的地址簿不是很大，但是一个一个地搜索我的联系人还是太麻烦了。微格式拯救世界！Dopplr 让您能够输入 XFN 编码的页面的 URL。它将解析该页面中的任何 XFN 链接，自动在其数据库中搜索匹配，并告诉你有多少朋友是 Dopplr 的成员。它只是减少了我的工作量，从搜索数百个名字，一个接一个，到搜索一个网址。

不过，我并没有完全摆脱困境——大部分工作还在我身上。我需要保持一个 HTML 网页在线，其中有我所有的朋友在 XFN 标记的网址。这听起来好像刚开始就有很多工作要做，更不用说跟上时代了。但是由于可移植性的因素，我们可以使用其他社交网络的公开数据作为我们朋友的种子列表。例如，我的 Twitter 账户有一个朋友列表，恰好用 XFN 微格式标记。在 Dopplr 中，我可以指向那个 Twitter 好友 URL，作为我想要导入的好友列表。实际上，我正在用一个社交网络来启动另一个。同样，Dopplr 也有可能“订阅”我的 Twitter 好友 XFN 列表。因此，每次我在 Twitter 上更新/添加新朋友时，Dopplr 都会找到它，并检查新朋友是否也是他们服务的成员。这意味着我需要更新的地方更少了，因为所有的服务都是互相支持的。

允许通过 URL 导入数据的网站正在建立一个更开放的便携式社交网络。通过从其他网站导入数据，用户能够使用这些下一代网站快速构建他们的新网络。通过像 XFN 一样分享这些数据，社交网站成为其他网络可以从中提取数据的枢纽。HTML 的内在能力允许我们在不使用任何新技术的情况下创建这些关联和链接。

##### 反模式

并非所有能帮助客户启动社交网络或导入数据的东西都是好主意。出现了几种反模式，您应该尽可能避免它们。

一些网站要求你上传你的地址簿，这样他们就可以“帮你找朋友”。他们真正想要的是你朋友的电子邮件地址列表，这样他们就可以给他们发垃圾邮件了。隐私当然是一个灰色地带——你的朋友很少希望第三方代表你联系他们。在未经朋友允许上传他们的个人信息之前，你应该三思而行。当然，并不是所有的网站都会向你的朋友发送垃圾邮件，所以一定要阅读附属细则。

另一个你想避免的陷阱是那些要求你提供电子邮件账户登录信息的网站。这是一个巨大的安全漏洞。通过交出这些信息，你就给了一个随机提供者访问你所有电子邮件和朋友的权限，更不用说访问 API，他们可以通过 API 编辑和删除你的信息。而且，我们都不想承认，我们经常在许多不同的服务中使用相同的密码。向一个网站提供你的电子邮件密码，它的所有者不仅可以进入你的电子邮件，还可能进入你的银行账户(以及一系列其他服务)。你不应该把你的密码给任何人！创造隐私保证会让我们产生一种虚假的安全感——它让我们放松下来，认为每个人都可以信任，一切都会安全。这种不良行为正是网络钓鱼者喜欢利用的。

从另一个服务导入数据的能力是一个很好的工具，但是使用用户名和密码这样做是错误的。像 Flickr 这样的网站，通过它的认证 API、Google 的 AuthSub 和 Yahoo 的基于浏览器的认证，在认证方面做得很好。OAuth 是一种开放式认证的尝试，这将有助于整合不同网站的不同认证方法。

使用这些工具，您可以授权其他网站使用您设置的精细权限访问您的数据，而不是通过您的用户名和密码授予完全访问权限。如果你使用的是你的用户名和密码，那么社交网络开发者没有以正确的方式导入你的数据。

##### 走向真正的可移植性

OpenSocial 是谷歌的一项新发展，这是朝着正确方向迈出的一步。尽管它不允许你的社交网络的直接可移植性，但它在任何实现 OpenSocial API 的社交网络之间创建了公共挂钩。这意味着开发人员可以使用 People and Friends 数据 API 来获取用户的个人资料、朋友列表以及他们的个人资料。

尽管还没有导出按钮，但可以开发一个简单的小部件将你的数据导出为可移植的格式，如 XFN、hCard 和 FOAF。

OpenSocial 正处于非常早期的阶段，唯一实现有限 API 的网站是谷歌自己的 Orkut，但这是创建便携式社交网络的非常重要的一步。

##### 提倡良好行为

容易让你导入和导出数据的网站应该因其良好的表现而得到推广。你应该试着去光顾像这样的网站，并迫使其他网站跟随他们的脚步。

网络之所以有效，是因为任何人都可以链接到其他人。你现实生活中的社交网络之所以繁荣，是因为没有人告诉你可以和谁做朋友，不可以和谁做朋友。在线网络应该没有什么不同。网络是我的社交网络；我只是碰巧使用各种网站来帮助我管理它，而不是成为控制它的交通警察。开放的便携式社交网络解决了所有这些问题。

你的在线身份是你的，你可以随心所欲地控制。
