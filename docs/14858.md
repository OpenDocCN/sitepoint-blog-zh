# HTML 乌托邦:使用 CSS 设计无表格

> 原文:[https://www.sitepoint.com/utopia-designing-tables-css/](https://www.sitepoint.com/utopia-designing-tables-css/)

我们可以从许多上下文的角度来看级联样式表。

我更愿意把它们看作是对网络时代开始时犯下的一个根本性错误的纠正，这个错误可以追溯到 20 世纪 90 年代初，当时蒂姆·伯纳斯·李和第一批先驱网络建设者首次预见了网络的开端。

那是什么错误？

为了满足 Web 最初有限目的的要求，没有必要将内容和表示分开。即使有些人认为这是一个好主意，也没有真正令人信服的、实际的理由来承认这种区别。毕竟，网络的早期意图仅仅是让少数在不同地点使用不同系统的核物理学家共享重要的实验数据。

伯纳斯-李没有预见到从他 20 世纪 90 年代早期的核心思想中出现的大规模流行、疯狂商业化、广泛变形的网络——我怀疑任何人都不可能预见到。

所以，错误是缺乏远见，而不是疏忽。但这仍然是一个错误。

本系列文章开始详细讨论当今在 Web 上使用 CSS 的问题。它由 4 条组成:

本章首先探讨 CSS 技术是如何发展的，以及为什么它变得如此流行。我们将探索一些基础知识，包括 CSS 规则的各个部分，CSS 规则的不同类型，以及在哪里可以定义这些规则。

[第 3 章:在地表下挖掘](https://www.sitepoint.com/blog/)

第三章，本系列的第二部分，继续完成我们对 CSS 机制的回顾。这里，我们将讨论使用简写属性将相关属性的值分组到一个语句中。我们将深入研究样式表中继承的概念，了解样式的结构，并讨论何时以及如何在代码中使用 CSS 注释。

[第四章:CSS 网站设计](https://www.sitepoint.com/blog/)
