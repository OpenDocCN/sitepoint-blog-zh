# 回顾–Macromedia Contribute

> 原文:[https://www.sitepoint.com/review-macromedia-contribute/](https://www.sitepoint.com/review-macromedia-contribute/)

你希望能够通过友好的界面轻松更新你的网站，或者让你的客户有机会更新你为他们创建的网站，而不用担心他们破坏设计或服务器端代码？你想从使用一组内容作者来保持你的网站内容新鲜中获益吗？

如果您手动更新网站页面，并使用 FTP 客户端将文件上传到 Web 服务器，或者您希望避免使用多种不同工具在线创建、发布和更新内容的麻烦，Macromedia 声称它可以为您提供解决方案。

##### 这是什么？

Macromedia Contribute 是一种新的客户端工具，允许 Web 开发人员保持对站点的控制，同时让内容编辑人员有机会更新他们自己的内容，即使他们对 HTML 或 FTP 一无所知。

![946_logo](../Images/017c2eab48c6488321cea6b9b060e79c.png) Contribute 被设计为安装在具有可用互联网连接的 PC 上，并使用 FTP 将完整的页面传输到 Web 服务器。管理员可以设置将更新网站内容的用户的个人资料，也有权在他认为合适的时候限制他们的行为。用户和站点设置以 XML 文档的形式保留在 Web 服务器上，以便不同的 Contribute 安装可以实施这些权限和首选项。

用户只需安装 Contribute 并插入管理员为他们创建并通过电子邮件编码提供的特殊连接密钥。一旦他们插入了密钥，用户就可以开始浏览网站和编辑网站上的页面。连接密钥实际上包含所有适当的 FTP 和其他设置，因此用户可以从这些任务中解脱出来，并可以立即开始工作。

![946_browser](../Images/29961951db6a47898701f14e29d68507.png)
*Contribute 浏览器工具栏*

Contribute 的界面类似于 Web 浏览器。用户导航到他们想要更新的页面并点击编辑按钮。如果他们被授权进行这种操作，浏览器就会变成一个带有工具栏的编辑器，在某种程度上类似于 Microsoft Word。通过该工具栏，用户可以获得他们需要的所有文本格式选项，以及 HTML 表格创建选项、图像和超链接插入、拼写检查(在许多语言中)和对页面属性(如文本和链接颜色、背景图像和编码)的控制。

##### Contribute 的主要功能

Macromedia Contribute 的主要功能是:

*   轻松创建和更新包含文本和图像的页面
*   轻松将完整页面发布到实时网站
*   内容作者在没有开发人员或网站管理员帮助的情况下进行内容更新
*   内容贡献者团队之间的协作
*   服务器代码的保护
*   现场设计标准的实施
*   无障碍标准的实施
*   页面版本控制
*   离线编辑
*   与 Dreamweaver MX 集成

##### 优点

Macromedia 声称 Contribute 将 Web 从只读介质变成了读/写介质:Contribute 非常易于使用，发布内容不需要任何特殊知识。因此，在网上展示你的作品，你不必是一个网站管理员。

虽然在我看来，Contribute 确实是一个非常简单和方便的 Web 发布工具，但 Macromedia 并不是第一个想到创建一个工具来简化 Web 发布的人。现在已经有许多大大小小的内容管理系统和各种独立的发布工具，它们承诺在没有特殊编程知识的情况下实现 Web 发布。

不过，Contribute 确实有几个优点。首先，它消除了使用不同工具(HTML 编辑器、FTP 客户端)来更改拼写错误的电子邮件地址的麻烦。

其次，您可以让几个用户同时更新您的站点:Contribute 将自动检查用户试图修改的页面是否已经被其他人编辑，以防止页面版本冲突。此外，还有一个页面回滚功能，通过恢复到页面的稳定版本，可以轻松地从可能的问题中恢复。

除此之外，作为管理员，您可以确保没有用户破坏网站的代码或风格一致性。此外，您可以确保新的更新符合当前的可访问性标准。

此产品的另一大优势是，使用 Macromedia Dreamweaver MX 创建的模板也可以在 Contribute 中使用。Dreamweaver MX 识别使用 Contribute 设置的站点，允许设计人员和内容作者一起工作。这项功能是通过免费更新添加到 Dreamweaver MX 中的，该更新将 Dreamweaver MX 升级到 6.1 版。

最后，用户只需将 Word 或 Excel 文档拖放到 Contribute 的编辑器中，就可以将它们导入 Contribute。然后，只需轻触“发布”按钮，Contribute 就会将所有更改和文件发布到 Web 服务器。

##### 弱点

在选择 Contribute 作为您站点的更新软件之前，您肯定要考虑它的一些弱点。

首先，它是一个客户端专用的工具。因此，您可以忘记从网吧或没有安装 Contribute 的电脑进行更新。

Contribute 还要求每个用户都获得单独许可。尽管许可价格很低，但对于一个有很多贡献者的网站来说，这可能会很快增加，除非每个贡献者都购买自己的副本。

如果您有一个从数据库中提取数据的动态站点，或者有由 Web 服务提供的内容，您将不会从 Contribute 中获得太多。不幸的是(对 Macromedia 来说)，今天最活跃的网站是动态的。

我也觉得产品的发布工作流程进度系统有问题。Macromedia 声称，贡献者可以通过电子邮件发送新页面或页面更新供审查。当他们在等待答复时，他们可以暂时保存页面，并在收到肯定答复后发布。虽然我必须承认我没有充分发挥 Contribute 的作用，但我无法找到一种方法来阻止用户在没有管理员积极响应的情况下发布页面。

##### 谁会受益？

*   如果你正在建立自己的个人网站，Contribute 是你的救命稻草。
*   如果您有一个需要频繁更新的静态站点，Contribute 可以为您节省大量时间
*   如果您希望允许不同的内容作者协作，而不用担心有人破坏网站设计或代码，您会发现它非常有用
*   如果您的动态网站包含需要更新的静态部分，使用 Contribute 可以节省大量时间
*   如果你想让你的 Web 开发公司的客户更新他们自己的页面而不打扰你，例如，添加一个问号，给他们买一个副本！

##### 谁应该去别处看看？

Contribute 肯定不是用来处理非常大或完全动态的站点的。如果您希望能够对您的网站、保存在数据库中的内容、来自其他网站的内容源、强大的审批工作流进行基于 Web 的远程更新，或者通过应用程序对您的网站进行全面管理，Contribute 不会为您做太多事情。

##### 摘要

总的来说，Macromedia Contribute 是一个非常有趣的工具，可以节省您更新站点的大量时间，并提供了页面版本控制和团队协作等高级功能。我认为，如果 Macromedia 在 2 年前发布它，当动态网站不太受欢迎的时候，他们会卖出几百万。当然，这并不意味着它现在没有用了，它标价 99 美元，物有所值。

Contribute 目前提供技术预览版。你可以[下载它](http://www.macromedia.com/software/contribute/trial/)，免费试用 30 天，并在 2002 年 12 月可以购买时自己决定它是否是你所需要的。Macromedia 在其网站上有大量关于 Contribute 的[信息。他们甚至建立了一个](http://www.macromedia.com/) [Contribute 开发中心](http://www.macromedia.com/desdev/contribute/)，里面包含了从介绍性信息和操作文章到深入的视频教程的所有内容。如果您对 contribute 有其他问题，也可以使用特定的 Contribute 论坛。
