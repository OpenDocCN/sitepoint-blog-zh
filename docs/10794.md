# HTML5 在今天可行吗？

> 原文：<https://www.sitepoint.com/is-html5-viable-today/>

我们喜欢 SitePoint 上的新技术，所以很容易被 HTML5 的兴奋和潜力所吸引——因为“HTML5”现在是一个营销术语，包含了所有很酷的新网络技术。报道最新的技术是我们的职责，这样你就可以在你自己的项目中采用它们。然而，我们最近的调查— *“你什么时候开始使用 HTML5？”*——发人深省。尽管这个词对不同的人有不同的含义，但结果是:

*   23%的人现在正在使用 HTML5
*   28%的人打算很快使用 HTML5
*   46%的人会在 HTML5 稳定时使用它
*   3%的人表示他们永远不会使用 HTML5 *(为什么？？？)*

风险也许是最大的问题。该规范尚未最终确定，可能在未来许多年仍是草案。没有人知道会发生什么——你喜欢使用的`article`或`header`标签可能会被废弃。至少 HTML4 和 XHTML1.0 已经稳定了十几年了。浏览器支持是另一个问题。所有的供应商都加入了 HTML5 的行列，但是他们有自己的解释，并且还没有实现一套一致的技术。微软可能已经宣布在 IE9 中全面支持 HTML5，但该浏览器不太可能在明年出现，也不会在 Windows XP 上可用。开发人员将需要支持 IE8，并在一定程度上支持 IE7 和 IE6。最后，我怀疑许多开发人员不愿意花时间迁移到一种新的、很大程度上未经验证的、可能有问题的标记语言。也就是说，今天使用 HTML5 的一部分是可能的。例如，您可以使用 HTML5 doctype 并从`script`标签中删除多余的 *type="text/javascript"* 。还可以使用新的标签，如`header`、`footer`、`nav`和`article`来增加语义的丰富性。可以用 JavaScript shiv 让 IE 理解这些元素，或者你可以完全避免设计它们。但是这在现实世界中有什么好处呢？你的同行可能会印象深刻，营销部门可能会写不准确的 HTML5 推广 BS，但额外的努力值得吗？当诸如验证器之类的工具不完整或者不存在时，这尤其困难。我怀疑许多开发者愿意使用 HTML5，但是当你不得不为不支持 html 5 的浏览器实现变通方法和额外的代码时，又有什么意义呢？然而，我确实认为你应该认真考虑 html 5——尤其是对于新项目。继续阅读 [**“今天可以使用 HTML5 的 5 个理由”**](https://www.sitepoint.com/5-reasons-why-you-can-use-html5-today/) …
