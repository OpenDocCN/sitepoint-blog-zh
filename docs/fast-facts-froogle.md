# 关于弗罗格的快速事实

> 原文：<https://www.sitepoint.com/fast-facts-froogle/>

Froogle 是一个新的基于“产品”的搜索引擎，Google.com 目前正在开发，最近已经进入测试阶段。Froogle 试图根据您的搜索词找到最相关的产品。

事实上，Froogle 在寻找与你的搜索匹配的产品方面做得非常好。它仍有一些改进的空间，但在大多数情况下，您至少会发现结果是相关的。当您搜索产品的品牌或型号时，会获得最佳结果。

虽然它仍处于测试阶段，但 Froogle 有着巨大的潜力。作为谷歌的狂热用户——是的！我是“谷歌人”！我确信，很多像我一样的人很快就会在我们的头衔中加上“弗罗格勒”。

##### 它是如何工作的？

谷歌再次提出了一种复杂的算法，从其 Froogle 数据库中检索最相关的结果。

看起来 Froogle 不仅从独立的在线商店获取信息，而且它还在网上爬行来主动寻找产品。这些来源于网络的结果可能不太准确——有时这些结果只是价目表等。但是，从提交的提要派生的产品似乎非常准确。您可以将这些类型的结果与其他结果区分开来，因为它们可以通过该产品结果列表下方的链接进行访问。链接看起来像这样:

<q>**——查看来自**</q> 的所有结果

Froogle 的高级搜索功能增加了搜索结果的相关性，允许您根据价格、特定关键字、短语的出现次数以及类别来缩小搜索范围。Froogle 的人还通过将产品分类到层次目录结构中，使浏览变得容易。

稍后我将讨论提交提要的细节，但是首先，我将回答每个人心中的问题…

##### 这要花多少钱？

我非常惊讶地发现，在 Froogle 上的列表是完全免费的。好吧，我再说一遍，免费！

现在，我无法想象免费模式会永远持续下去，但通过尽早开始，你可能会享受先发优势，并为你的商店带来一些可靠的销售线索。

##### 接受您的数据馈送的要求是什么？

要让 Froogle 接受您的订阅源，您的在线商店或网站必须符合以下标准:

1.  位于美国
2.  内容以英语发布
3.  产品必须直接从你的网站销售
4.  不允许服务
5.  没有固定价格的产品是不允许的
6.  你的网站必须能被他们的蜘蛛谷歌机器人抓取

##### 如何提交数据馈送？

在您开始之前，您需要查看位于[此处](http://froogle.google.com/froogle/merchants.html)的商户 Froogle 信息页面。

一旦你确定你符合标准，提交是一个相当简单的过程。第一步是通过位于此处的[表格注册。在几天内(对我来说，只有两天)，Froogle 将回复一个用户名和密码，以及一个链接，通过它你可以同意该网站的使用条款。也就是说，如果你的网站被批准，你将得到所有这些。](http://services.google.com/froogle/merchant_email)

一旦您同意使用条款，您应该会在一天左右的时间内收到一封电子邮件，其中包含如何设置和上传您的订阅源的说明。这封电子邮件将包括一个 PDF 格式的详细说明，一个样本 Excell 电子表格，和一个样本制表符分隔的文本文件。

他们会将您的订阅源作为压缩文件(例如。zip，。焦油等。)，但是数据文件本身必须是以 Froogle 指定的格式设置的文本文件。虽然平面文件系统似乎有些过时，但我想这是他们最容易使用的方法。

格式相当简单，包括以下制表符分隔的列:

产品 URL name description price image UrlCategoryOffer ID

唯一需要解释的列是 Category 和 offer ID。类别是指您在商店中使用的类别结构，而优惠 ID 代表您可能使用的唯一标识符，如 SKU 或产品编号。

该服务还允许您在订阅源中添加高级栏，如产品是否有货、最低运输价格、品牌名称等。从这些扩展字段的外观来看，Froogle 可能会在未来添加更多高级搜索选项。

在说明的顶部，Froogle 提供了 FTP 信息，帮助您在数据馈送准备就绪后上传数据。他们还要求你每月至少上传一次你的 feed，否则它会从 Froogle 数据库中消失。

##### 如何创建自己的数据馈送？

以下内容非常具体地针对我的场景，但是我认为让您确切地知道我是如何创建我的数据提要的可能会很有见地和有帮助。

如果您不是程序员(我也不是)，创建数据提要可不是一件容易的事情，但是这并不像听起来那么难。起初，我决定走进我的商店，简单地复制和粘贴。我们有超过 150 种产品，所以在第 10 种产品之后，我决定一定要有更简单的方法。还好有(不然我老婆早就在 75 品左右跟我离婚了！).

我们的网站使用 MySQL 数据库，所以我可能会联系我们的程序员来编写一个小应用程序来自动化这个过程，但是，由于我急于开始，我决定自己来做。毕竟，这是周日晚上 10 点以后，我想在这一周有所突破——总是做出理性判断的最佳时间！

我们已经有一个小的 PHP 应用程序，列出了我们商店中的所有产品。唯一的大问题是它列出了所有的产品，而不仅仅是库存产品——而且它还遗漏了一些字段。我开始摆弄这个应用程序，过了一会儿，我让它列出了我需要的所有东西。唯一的两个问题是，它仍然列出所有的产品，无论它们是否有库存，而且它不是我需要的制表符分隔的格式。最后，我想出了如何至少在末尾添加一个字段，以指定产品是否是活动的。

接下来，我决定处理格式问题。我使用的应用程序在一个大的 HTML 表格中显示产品。因此，我删除了应用程序中的所有 HTML 标签，并放入管道(或“|”)来分隔每个字段。这被证明工作得非常好，现在我有了一个显示管道分隔的平面文件的页面。接下来的部分很简单。我将文本从浏览器中复制出来，放到一个文本文件中，然后导入 Excel。一旦我在 Excel 中有了所有的数据，我就删除所有没有库存的产品，并为数据输入输出一个制表符分隔的文本文件。

##### 你如何上传提要？

现在是简单的部分:上传您的数据馈送。使用 WS-FTP，我按照 Froogle 指定的方式上传了文件(如 username.txt ),并通过电子邮件告诉他们上传完成。文件没那么大，我决定不压缩，直接上传文件就好了。

我在凌晨 2:30 左右完成了这一切，到那天下午，他们已经批准了我的数据输入。该邮件称，该 feed 已获批准，并将在接下来的几个工作日内出现在 Froogle 上。不用说，在所有这些工作之后，我对这可能给我们的网站带来的潜在业务感到非常兴奋。我定期查看网站，到了晚上，我们的产品已经上市了。真快！

我意识到上传后，数据饲料中的一些图像是死链接。Froogle 很好地处理了这个问题，它自动为那些死链接添加了一个“图片不可用”的图形。我还应该提到的是，Froogle 将你所有的产品图像重新采样为 90 x 90 像素——他们在这方面做得很好。

总而言之，整个过程非常简单，我相信 Froogle 可能会成为电子商务的下一个大事件。如果它提到了谷歌，这表明他们是创新的，并且站在新兴市场的顶端。尽管我还没有看到 Froogle 的一单销售，但我仍然保持乐观。而且不管怎么说，饲料昨晚才出现！

祝你们这些未来的“蛙人”好运！
