# 开源和商业化

> 原文：<https://www.sitepoint.com/open-source-and-commercialization/>

最近，我花了一些时间与一位受欢迎的 Yankee Group 分析师进行了交谈，他负责美国的企业部门，重点关注开源以及未来几年的发展方向。

澄清一下，正如大多数行业观察家所做的那样，我区分了 Linux 和开源。虽然 Linux 是开源的，但主要的 Linux 发行商已经明白他们需要如何定位自己以获得成功，并开始像任何专有软件公司一样经营他们的业务。

Red Hat 和 SUSE 就是最好的例子，它们实现了长期成功和收入流的途径在于向更大的企业和机构证明自己的企业价值，已经转变了业务模式或被植根于企业的组织收购。

她的观点虽然在开源社区并不总是受欢迎。如果开源寻求广泛的采用和长期财务成功的永久席位，这是对的。

开源支持者需要接受一些障碍并继续前进:

1)对于一家公司来说，从 Windows 迁移到 Linux 的成本会更高，即使持续维护略有减少，安全性和正常运行时间有所提高。

虽然我并不总是同意成本会更高，但在过去将企业系统迁移到 Linux 后，他们的研究表明，在许多情况下这是真的——尤其是在迁移标准的 web 托管和电子邮件系统之外的时候。如果将重新认证驱动程序、应用程序完整性和培训考虑在内，成本会更高。

2)要真正成为一个可行的财务回报选项(意味着开源公司赚钱并创造就业)，向商业软件模式的转变是必要的。这并不意味着放弃开源，但是，这意味着为开发、发布、修补和支持开发一个符合公司 it 经理要求的结构，这些经理可能会在开源上投入大量资金。

归结起来就是，虽然开放源码确实革新了软件，并且在大大小小的国际公司中都可以找到，但是企业仍然选择软件，因为它提供了一个解决方案，而不仅仅是因为它是开放源码的。

它更便宜或免费的事实仅仅意味着用户将节省金钱，但这并没有赢得那些购买者的青睐，他们可能会向开放源码项目而不是专有软件制造商注入数百万美元。

我会用火鸟作为模型。在对 Helen Borrie 的一次采访中，她指出，由于许多财富 500 强公司都在使用 Firebird 这样的开源数据库，这充分说明了他们的项目和开源技术的成熟。

在我看来，这是因为 Firebird 被当作一个企业级的专有软件项目。他们有一个管理良好的开发人员社区和活跃的支持列表，通过与几家公司的合作伙伴关系提供商业支持，并为公司客户提供商业开发项目。

如果更多的开源项目关注 Borrie 的团队模型和开发与支持纪律，我们可能会看到更多的渗透，吸引更长、更有利可图的合同，并为 SitePoint 社区中像我们这样的人工作。
