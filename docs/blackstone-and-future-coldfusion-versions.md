# 黑石和未来的 ColdFusion 版本

> 原文：<https://www.sitepoint.com/blackstone-and-future-coldfusion-versions/>

黑石是 ColdFusion 下一版本的代号。网上有很多关于这个新版本的讨论。人们正在发表他们的想法，关于这个新版本需要改变什么，需要增加什么，应该删除什么。

我还没有像这些人中的许多人那样深入思考，所以我不能告诉你下一个版本有什么问题或者需要添加什么。然而，我确实找到了 Ben Forta 的一个帖子，他在其中谈到了一个应该出现在 CFMX 中但没有出现的未来功能。

他谈到了在没有源代码的情况下运行 CFML 代码的能力！正如他所说的“没有任何虚幻的东西，我真的写了 CFML 代码，编译了它，然后在没有本地源代码的情况下运行了应用程序”。

这多酷啊！没有源代码，我们的客户就无法进入并搞乱应用程序，他们也无法通过简单的剪切粘贴操作来窃取我们的时间。当然，你现在可以[对你当前的模板](http://cfhub.com/tutorials/cfencode/)进行编码，但是这里的想法不同。我想象更像 Java 的东西，我们可以编译我们的代码，然后发送。类文件或一个. jar 文件，大多数情况下一切正常。

你希望在黑石看到什么样的东西？
