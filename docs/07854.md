# WordPress v Joomla:赢家

> 原文:[https://www.sitepoint.com/wordpress-v-joomla-the-winner/](https://www.sitepoint.com/wordpress-v-joomla-the-winner/)

![](../Images/fea5db2e38c9d0092d6b579ac9a202bd.png "header6")

我们成功了！在过去的五周里，我们深入研究了目前最流行的两个内容管理系统。我们在不同的部门对 WordPress 和 Joomla 进行了批判性的分析。我们已经发现了每个系统的优点和缺点，并发现没有一个系统能做到完美。

今天，我将总结本系列每周得出的结论，并提供一些关于 CMS 战争的最终想法。

## 第 1 部分:[简介和内容结构](https://www.sitepoint.com/wordpress-v-joomla-introduction-and-content-structures/ "Introduction and content structures")

在第一周，我介绍了每个 CMS 并解释了当前的事态。得出的结论是，根据目前的统计，WordPress 是最受欢迎的 CMS，尽管 Joomla 比它的对手年轻几岁。

![](../Images/6336ba1e5f1d86d06755385057a2dd22.png "conclusion1")

我们认为 WordPress 对用户更友好，如果你是一个网络新手，需要的学习曲线更少。

我还建议，如果你开始一个博客/内容驱动的网站，WordPress 可能是你最好的选择，而 Joomla 可能更适合静态或企业网站，比如内部网，易贝是这方面的主要代表。

## 第 2 部分:[模板和主题](https://www.sitepoint.com/wordpress-v-joomla-templates-and-themes/ "Templates and themes")

第二周的讨论围绕 WordPress & Joomla 的模板。我们发现，在为任何一个系统开发定制模板/主题时，难度没有真正的区别。我们还讨论了这样一个事实，WordPress 的主题通常是有特定目的的；通常是内容监管/博客。Joomla 模板似乎被设计得更加通用。

![](../Images/539cf202a65e2b9da8fe04654a1c4560.png "conclusion2")

我觉得 WordPress 更适合极简设计，而 Joomla 在设计方面提供了相当多的灵活性和自由度。

这里没有明确的赢家——如果你找到一个适合你的模板/主题，就去做吧。设计是主观的。

## **第三部分:[插件、](https://www.sitepoint.com/wordpress-v-joomla-plugins-extensions-and-customization/ "Plugins and customisation")** [**扩展和定制**](https://www.sitepoint.com/wordpress-v-joomla-plugins-extensions-and-customization/ "Plugins and customisation")

 **在第 3 周，我们讨论了每个 CMS 可用的扩展种类。

我们讨论了每一个都有不同的扩展结构的事实。Joomla 的结构比 WordPress 更直观，一旦你掌握了它的窍门。

同样，这归结为你的网站的目的。我发现大多数 WordPress 插件是专门为博客类网站开发的，而 Joomla 扩展的种类要多得多。如果你找到了一个完美的扩展来支撑你的网站的核心功能，那么你应该使用这个扩展开发的 CMS。

## **第四部分:** **[搜索引擎优化](https://www.sitepoint.com/wordpress-v-joomla-search-engine-optimization/ "SEO")**

在这个分析中，我们比较了 WordPress 和 Joomla 的 SEO 能力。毫无疑问，WordPress 比 Joomla 更适合 SEO。虽然 Joomla 在 SEO 部门并不可怕，但要做到 100%正确可能有点麻烦。

![](../Images/d7745ede3f1627593130ed1f4361f2e2.png "conclusion4")

WordPress 真正的 SEO 能力来自于可以让你的生活更简单的优秀插件。这是一个明确的赢家，但 Joomla 并没有那么糟糕，你应该完全避免它的基础上搜索引擎优化。

## 第 5 部分:[支持和社区](https://www.sitepoint.com/wordpress-v-joomla-support-and-community "Support and community")

在第五周，我们有点惊讶地发现——与 Joomla 相比——WordPress 缺乏很多社区建设的主动性。它的网站平淡无奇(他们可能会称之为极简主义)，似乎是基于 WordPress 已经是最受欢迎的 CMS 这一事实。

另一方面，Joomla 确实在努力培养自己的社区。该网站非常吸引人和直观，它有许多伟大的举措，它的支持论坛是惊人的。

## 获胜者是…

我已经考虑这个决定很久了。不仅仅是在我写这个系列的时候，而是这些年来我一直在设计网站。

事实是，我只是找不到足够的证据来决定性地命名一个 CMS 是一个全面的赢家。我认为这两个系统对某些目的来说都是好的，而且这种想法似乎在我每次开始规划网站设计时都会被重复。我从来不知道当我接受一个客户时，网站是用 WordPress 还是 Joomla 来做——在我做出决定之前，我必须考虑所有上述因素(特别是与网站目的相关的可用扩展)。

本系列的目的是让您清楚地了解每个 CMS 的功能、优点和缺点。我希望你带着足够的洞察力和理解离开，做出真正明智的决定，什么是最适合你网站的内容管理系统。

一般来说，当我开发一个非常依赖内容的网站时——比如博客/内容管理网站或新闻网站——我通常会选择 WordPress。对于几乎所有其他的东西(通常是更复杂的站点)，我倾向于使用 Joomla。

我猜测这与 WordPress 的目的相匹配，WordPress 的目的是为*内容的轻松发布和分发提供一个平台。*

 **无论你决定使用哪种 CMS，你都需要花一些时间来学习系统的细节，尤其是如果你打算自己开发的话。是的，WordPress 比 Joomla 更直观一点，但是 Joomla 的“复杂性”使它成为一个更加通用的系统。

事实是这取决于个人喜好。这是一个主观的决定，这也是 WordPress 对 Joomla 的争论将会继续的原因。决定哪种系统适合你的目的，然后努力去做。
