# 雅虎！推出 BrowserPlus，并计划将其开源

> 原文：<https://www.sitepoint.com/yahoo-launches-browserplus-plans-to-open-source-it/>

经过几年的发展，雅虎！在雅虎的一次聚会上正式发布了他们的浏览器插件。今日校园。BrowserPlus 是 Yahoo！这使得他们所谓的“浏览器内桌面应用程序”成为可能，这是一种奇特的说法，它允许开发人员向基于网络的应用程序添加类似桌面的功能。BrowserPlus 方法与 Gears(谷歌主导的开源项目)和 Adobe 的 AIR 略有不同，前者是带有本地数据存储的 web 应用程序的桌面包装器，后者在桌面上重建了 web 环境，尽管两者有一些重叠。

BrowserPlus 为开发者提供了许多[服务](http://browserplus.yahoo.com/developer/services/)，包括持久数据存储——这是 Gears 和 AIR 的主要卖点之一，拖放，文件上传，IRC 聊天，运动传感器控制和桌面消息。

据雅虎报道！Lloyd Hilaiel 说，BrowserPlus 的目标是“修复 web 插件环境，使快速实验成为可能。”Hilaiel 认为浏览器插件很棒，因为它们允许开发者扩展浏览器来做一系列新的事情，但当前的系统是坏的。它们很难编写，安装它们也很痛苦(谁想在你能访问新内容之前重启你的浏览器呢？)，更新它们是笨拙的，保护它们是困难的，他在发布会上告诉与会者。

另一方面，BrowserPlus 将允许开发人员通过 web 应用程序向用户推送新的桌面功能，而无需重启浏览器，并且更新可以在后台推送给用户，而无需他们重新安装任何东西。对于最终用户来说，BrowserPlus 将是一个单一的安装，可以在后台启动并向他们推送附加的开发人员服务。目前，只有雅虎！他们的合作伙伴可以在 BrowserPlus 上推出新的服务。

雅虎！打算最终开放源代码浏览器 Plus。服务 API 应该在今年年底开放，其他的将在 2009 年中期开放。

“我们希望 BrowserPlus 便携、防碰撞、安全、小巧，”Hilaiel 说，他谈到了人们已经在讨论的大量即将推出的服务。“论坛上的人们都在谈论点对点的支持。人们建议使用屏幕捕捉技术来更好地报告错误。网络摄像头集成！轻松导入日历数据！Word 文档的拖放！Bittorrent！不缺乏想法，”他说，并指出他很高兴看到开发者一旦完全访问服务 API 后会提出什么。

早在六月，我们[在 SitePoint 博客](https://www.sitepoint.com/why-yahoos-browserplus-has-a-long-way-to-go/)上写道，BrowserPlus 是雅虎的重要组成部分！s web OS play。“雅虎已经运行了一些全球最受欢迎的网络应用程序；添加 BrowserPlus，他们不仅可以增强他们的应用程序，还可以在他们自己的平台上实现。有了 BrowserPlus，本地操作系统突然变得不那么重要了，”阿卡什·梅塔写道。
