# FullCodePress 采访:詹姆斯·法雷尔，澳大利亚队

> 原文：<https://www.sitepoint.com/fullcodepress-james-farrell-interview/>

来自两个国家的两个团队要为两个慈善机构完成两个网站。24 小时后。观众注视着每一个动作。这就是 2009 年 5 月在 CeBIT 举行的 [FullCodePress](http://www.fullcodepress.com/) 背后的想法。

James Farrell 被选为澳大利亚团队的程序员，该团队为 DDLC 新南威尔士州完成了一个伟大的网站。他友好地与 SitePoint 分享他的经验。

你或你的团队在比赛前做了什么准备吗？

TextPattern 是我们最初的 CMS 首选。我花了很多时间研究产品，安装相关插件。

我也熟悉了 Joomla。如果我们需要社区或多用户权限处理，这将是一个选项。

你以前和其他团队成员合作过吗？

直到前一天晚上，我才见到任何一个队员。我们的第一次团队会议是在悉尼烤肉店的晚餐。

你选择了 WordPress 作为内容管理系统。这是一个安全可靠的选择，但是影响这个决定的关键因素是什么，它有没有导致任何问题或限制？

在活动开始前选择一个 CMS 并不容易，因为你对要求一无所知。尽管 TextPattern 是我们的第一选择，但我们找不到插件来满足这个需求。Joomla 也被拒绝，因为我们都没有使用该产品的模板经验。我们后来改用了 WordPress，因为团队中的大多数人都用过它，它有很多插件，模板也很简单。

就局限性而言，也许我们的主要问题是我们不能在我们的 PC 上建立一个本地开发环境并连接到一个公共的数据库服务器。WordPress 将网站的 URL 保存在数据库中，虽然我确信我们可以绕过这个问题，但是时间很紧，我们必须继续前进。开发环境并不理想，可能会阻碍我们。

在过去的几个小时里，我们上传了很多内容，WordPress 引发了一些问题，比如丢失了更改。那段时间压力很大。

你用什么工具进行开发？IDE？源代码管理？测试套件？

我们使用 Subversion 进行版本控制，但是没有像我们希望的那样管理好我们的代码。

最大的挑战是什么？

主要挑战是我们工作的独特环境:

*   该活动在 Cebit 展览期间举行，因此随机的人会和我们聊天。
*   时间限制意味着我们能够交付的东西有限，并且当问题出现时会产生额外的压力。
*   在其他方面完成之前不能开始工作是令人沮丧的。例如，在设计完成之前，Elle 不能编写 HTML 和 CSS 代码。

什么比你想象的要容易？

整晚保持清醒比我想象的要容易。然而，集中注意力是个问题——在 24 小时里，我只有一次 15 分钟的休息时间。这并不理想，但在这么短的时间内有这么多事情要做。

**您是否需要放弃或简化某些功能，因为它们无法在 24 小时内实现？**

我们不得不放弃我们的时事通讯管理系统(Mailpress 插件)。在早期阶段，它一直在工作，但是在整合过程中，它突然失败了。我花了一段时间试图解决这个问题，但是没时间了。这太令人失望了。

你在哪些领域击败了你的竞争对手，他们在哪些方面击败了你？

我更喜欢我们的视觉设计，但这可能是我个人的偏好或偏见。亚历克西在激发情绪反应方面做得很好，同时传达了强烈的积极信息。

批评什么都难。这是一次令人精疲力竭的经历，所以我对另一个团队的成就充满敬意。这是一场竞争，但重要的是两个伟大的慈善机构收到了精彩的网站，这将彻底改变他们的日常运作。

事后想想，你会做出什么不同的事情吗？

虽然 WordPress 是我最喜欢的选择，但我后悔没有准备更多。那天我花了很多时间研究插件。

理想情况下，我应该事先制定出一些解决方案和平台。然而，事实上，当你在活动前的几天全职工作时，很难找到时间来准备。

你学到了什么可以用在日常工作中的东西吗？

绝对的。我现在很了解 WordPress，并且有一些插件的想法，我将很快开始。

您会再次参加 FullCodePress 活动吗？

是的，我会再做一次——尽管这是我经历过的压力最大的工作，可能会让我少活几年！我遇到了许多伟大的人，包括竞争者、组织者和支持者。

谢谢你，詹姆斯。

詹姆斯·法雷尔是悉尼的自由网络开发员。他专门从事高质量、价格合理的网站设计，并为中小型企业提供创造性的在线营销解决方案。他从 13 岁开始编程，精通网站界面布局、UI 设计、XHTML、CSS、PHP、Javascript 和 Flash。你可以在 http://twitter.com/james_farrell 的推特上关注詹姆斯。
