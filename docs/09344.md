# Mozilla、Firefox 和版本号混乱

> 原文:[https://www . site point . com/Mozilla-Firefox-version-number-arguments/](https://www.sitepoint.com/mozilla-firefox-version-number-arguments/)

Firefox 的快速发布时间表并不像 Mozilla 希望的那样成功。大多数 web 开发人员都认为这有利于 HTML5 特性的发展，但也不是没有问题:

*   附加兼容性。我们大多数人使用的扩展跟不上 Firefox 的开发进度。
*   加大努力。大多数 IT 部门必须先测试任务关键型应用程序，然后才能在整个企业中部署浏览器更新。
*   困惑。很少有人理解主要版本增量背后的基本原理。为什么火狐 6 不应该是 4.2 版？

Mozilla 正在复制谷歌的发布模式，但 Chrome 不一定会出现同样的问题。它的附加系统要简单得多；更类似于 bookmarklets 而不是集成代码。这款浏览器的遗留障碍也更少，而且从早期开始就一直在默默更新。使用 Chrome 的要么明白这个概念，要么不在乎。

Mozilla 考虑的一个解决方案是从 Firefox 的“帮助>关于”对话框中移除版本号。Mozilla 的逻辑:

1.  很少有用户理解版本号。
2.  移除会简化用户界面。
3.  当最后一次检查发生时，用户会被告知他们是否使用了最新版本，以及他们如何更新(如果 Firefox 没有自动更新的话)。
4.  如果您真的需要版本号，可以在 about:support 中找到。

Bugzilla 和[联合新闻组讨论](http://groups.google.com/group/mozilla.dev.usability/browse_thread/thread/fe75ec92c02be934#)引起了轩然大波。大多数回答者厌恶这个想法*(尽管当 Mozilla 的意图公开后出现了大量的咆哮和垃圾邮件)*。

该组织提出了一些合理的论点，但最终放弃了。Mozilla 的罗伯特·凯泽:

> 我们能关闭这个错误报告吗？
> 
> 软件中的版本号就像物理学中的坐标系:不相关，同时又是必要的——你怎么做完全不相关，但它们提供了必要的参考点。不多也不少。
> 
> 无论我们走到哪一步，我都不认为它会对版本号消息传递或让 Firefox 变得无用产生大的影响，所以我认为双方的愤怒都被高估了。

来自 VanillaMozilla 的回复:

> 完成…我很难找到一个认为这是个好主意的人。

争论变得激烈了，但是 Mozilla 的提议有很多缺陷:

1.  它违背了横跨操作系统和 20 多年 It 开发的既定 UI 惯例。可能有更好的方法，但是删除版本号不太可能是最好的解决方案。
2.  这个提议太简单了，而且没有解决火狐的快速更新问题。版本编号从来都不是原因，也不是解决方法。
3.  用户可能不理解版本号，但是删除它们不是问题。火狐不会突然变得更容易使用。
4.  Firefox 在野外有多个版本。有些有版本号，有些没有。没有一个旧版本会声明它们已经过时。
5.  版本号对于开发人员和 IT 支持人员来说非常重要。当有人报告特定浏览器出现问题时，您首先会问什么？

版本号在 Chrome 和 Firefox 中已经变得毫无意义。很少有人知道或关心他们运行的是什么版本的 Chrome。也许有一天，Firefox 也会如此——但我们还没有到那一步。

Firefox 是一个更老的浏览器，有着更多的包袱和一个庞大的热情的用户群体。Mozilla 最终听取了他们的要求，但是这个提议和随后的攻击对浏览器没有任何帮助。
