# 超越 MySQL–高端数据库解决方案

> 原文:[https://www.sitepoint.com/high-end-database-solutions/](https://www.sitepoint.com/high-end-database-solutions/)

如今，每个人都在建立数据库驱动的网站。几年前，这是搜索引擎、门户网站和像微软这样的大公司网站的主要奢侈品。但是现在，随着 MySQL 的流行，数据库驱动的网站已经成为主流。MySQL 提供了许多功能，它很快而且是免费的。然而，当你达到这个软件的极限，并尝试了每一个小的优化和调整，会发生什么？

你会放弃和抗拒成长吗？不，你迁移。MySQL 并不是唯一的数据库系统。您实际上有许多不同的系统可供选择——这些系统各有优缺点，并且提供了许多 MySQL 中没有的高级功能。这些特性包括存储过程、触发器、原子更新和外键完整性，它们提供了在搜索引擎和其他大型网站上进行快速搜索所需的可伸缩性。

并非所有这些解决方案都很昂贵，有些甚至是免费的。更昂贵的解决方案，如 Oracle 和 Microsoft SQL Server，可以由托管公司提供，这也降低了他们的维护成本和复杂性。这是 MySQL 的一大优势:因为它的简单，任何人都可以维护它。这些服务器中的许多都需要经过培训的全职数据库管理员(DBA)的关注，但是通过使用托管解决方案，您可以显著降低这一成本。

##### 一点 SQL 背景知识

SQL (ess-que-el)是数据存储标准。SQL 是结构化查询语言的首字母缩写，这种语言可用于从数据库和文件系统中访问数据。SQL 基于另一种类似的语言 Sequel。这在开发社区中造成了一些混乱，因为许多人认为 SQL 是旧语言名称的简单缩写。

Sequel 是由研究人员在 20 世纪 70 年代中期在加利福尼亚州圣何塞地区的 IBM 工厂创建的。这项工作是基于 E. F. Codd 博士在 1970 年 6 月发表的一篇名为“大型共享数据库数据的关系模型”的论文。

IBM 进一步开发了一个名为 Sequel 的研究数据库和 API。到 20 世纪 70 年代末，这项工作已经发展成为一种叫做 SQL/DS RDBMS 的产品。这个新系统基于后来成为 SQL 标准的东西，并被设想为使数据访问通用化。

当 SQL/DS 宣布时，许多竞争对手使用相同的 SQL 标准将类似的产品推向市场，IBM 被一家不知名的公司打败了。一家名为 Relational Software Inc .的小公司率先推出——一旦其软件获得成功，该公司就更名为甲骨文公司。迄今为止，Oracle 一直凭借其产品统治着 RDBMS 市场。但稍后会更多地介绍 Oracle。

尽管这些产品都是基于 SQL 标准的，但是每个产品都以不同的方式实现了某些特性，因此，一些东西在不同的包之间有很大的不同(听起来很熟悉吧？).它们都涵盖了 DDL(数据定义语言)和 DML(数据操作语言)的基础知识。然而，它们的主要区别在于它们的优化特性和 DCL(数据控制语言)。这对您来说是个好消息，因为基本术语是相同的:您的基本查询和表标记在不同的服务器上几乎是相同的(大多数人在日常开发中不太使用 DCL 子集)。

##### 常见术语和功能

让我解释一下这些特性是什么，以及它们如何提高您的性能。

*   **存储过程**–这些是用于修改查询结果或修改表中数据的代码。与脚本中的查询不同，这些查询以编译后的格式存储在数据库本身中，这样可以加快执行和响应速度。这些数据库都带有 SQL 之外的过程语言，允许您编写和运行存储过程。这比普通的查询节省了时间，因为 RDBMS 不必知道如何运行查询，它只需执行它。它还使您能够添加特性和修复 bug，而只需对前端代码做很少或不做任何更改。
*   **触发器**–这些是专门的存储过程。触发器是数据库服务器为响应您定义的事件(如添加或修改表中的记录)而自动运行的函数。
*   **原子更新**–这些通常被称为“事务”。这允许您将一系列查询收集到一个组中，以便将它们作为单个实体对待。这样，如果一个查询失败或被取消，其余的将被撤销，从而防止部分交易被输入到您的系统中。原子更新对于在线交易处理以及维护您的企业和客户的财务安全都是必要的。
*   **外键**–它们在关系数据库中强制实施关系。外键允许您确保数据是有效的，而不必对代码进行双重检查，或者担心重复条目。这些键是使用服务器中现有的索引和表查询方法来执行的，这使得它们非常快速和可靠。
*   **视图**–这些是特殊的表格。通过视图，您可以使用标准的 SQL 查询来创建表的外观。这允许您在检查索引和匹配行时跨表进行连接，而不会导致性能下降。视图为您维护所有这些匹配，以便在您需要时可以使用它们。视图也有利于向不同的人提供数据子集。例如，视图可以让您确保您的支持人员永远不会看到客户的信用卡信息——就视图而言，这些信息根本就不存在。

##### 一种数据库系统

Postgre 为 SQL 数据库世界带来了一些独特的特性。它允许开发人员创建关系对象作为数据库的基础。这些对象允许继承，这在数据库服务器中是一个相当高级的特性。Postgre 还包括 MySQL 缺乏的其他特性。

这些包括触发器、存储过程和原子更新。Postgre 基于加州伯克利大学开发的代码，在 GNU 公共许可证下发布。PostGre SQL 在 GPL 许可下免费发布。服务器无需修改即可在 Linux 或 Unix 机器上运行，客户端也无需修改即可在 Windows 上运行。

Postgre 继续在互联网开发前沿获得认可。这是因为它提供了许多其他更昂贵的解决方案所提供的功能，但仍然易于使用，并且是在 GNU 公共许可证下发布的。这给你很大的权力，很少或没有成本。由于 MySQL 的限制以及由更成熟的提供商生产的系统的高成本，许多网站和公司已经迁移到 Postgre。

Postgre SQL 是唯一支持存储过程和触发器等过程语言结构的开源数据库服务器。对于这些命令，它使用一种叫做 PL/pgSQL 的语言。PL/pgSQL 的格式和结构类似于 Oracle 数据库服务器中使用的 PL/SQL 代码。我确信这是正确的——如果你已经超越了 Postgre，它为开发者提供了一个很好的升级途径，只需要很少的转换工作。

我非常喜欢 Postgre 的一点是它的文档化程度，这与商业服务器中常见的文档化程度相似。每个不同的功能，即程序员或开发人员，在单独的 Adobe Acrobat 文件中可用，这使得功能的定位更容易。

总的来说，如果你需要一个开源的解决方案，并且 MySQL 已经无法满足你的需求，你应该考虑 Postgre 作为替代方案。它提供了小型企业解决方案、稳定的引擎和积极开发所需的特性。

##### IBM DB2

这是 IBM 进入高级数据库领域。他们称 DB2 为通用数据库，因为它应该在所有平台上运行。它甚至可以在 OS/390 等系统下的大型机硬件上运行。我个人没有使用过 DB2，但是在这里把它列为您可能会考虑的一个选项。较新的版本配有 IBM 的 Websphere 软件、ADO 和 ODBC 驱动程序、从 web 服务器连接到数据库的接口，以及允许您通过 PERL 的 DBI 模块连接到 PERL 代码的定制模块。

IBM 刚刚发布了该套件的版本 7，它在 IBM 硬件上很受欢迎，通常预装在较大的机器上。再加上强大的 Websphere 快速开发工具集和 Lotus Notes 的健壮特性，DB2 可以提供一个平衡的集成平台来满足大多数公司的数据处理需求。

通过这个系统，IBM 的目标是提供一个单一的解决方案，可以满足所有人的所有需求。该版本包括 XML 数据、多媒体文件处理、Web 驱动输出、Java 支持等特性。

***一小杯爪哇***

与当今许多可用的数据库服务器一样，IBM 已经将 Java 功能集成到其 DB2 产品中，这允许开发人员创建新的和独特的特性，并使用常见的日常工具来扩展 DB2 库。DB2 利用两种不同的 Java 技术，Java 数据库连接(JDBC)和嵌入式 SQL for Java (SQLJ)。两者都可以用来创建访问 DB2 的 Java 应用程序和小程序。

JDBC 通过访问数据库的命令行界面来访问 DB2 数据库中的信息。这可能导致访问速度比使用 SQLJ 的存储过程或触发器慢。SQLJ 使用 JDBC 类连接到数据库，但是将编译后的查询保存在字节码中，以便更快地规划和执行。

***定价混乱***

正如我前面说过的，DB2 被宣传为能够在任何平台上运行。这在很大程度上是正确的，但是有些模块只适用于特定的平台。DB2 附带了一个复杂的定价和许可方案，IBM 根据使用的模块来计算费用。虽然这确实能让你挑选你需要的东西，但是如果你想要一个完整的套餐，你可能最终会超出你的支付能力。

在我看来，DB2 需要一些改进才能成为“通用数据库解决方案”。但是，如果您使用其他 IBM 软件(比如 Websphere 和 Notes)来处理对 DB2 服务器的快速访问，那么您应该利用这一点，使用这个数据库。

在这种语言中，您可以在程序将结果返回给您的脚本之前操作查询的数据——这意味着更快的处理和更容易的格式化。想象一下能够在数据库级别应用模板。想象一下，确定姓名的敬语(即先生、夫人或博士)并自动将其添加到姓名中，即使该敬语没有存储在数据库中。想象一下，在不考虑拼写的情况下，为搜索引擎比较单词。

在结果被转交给脚本之前，您可以在 Transact SQL 中完成所有这些操作，由于它们是编译后的代码，所以执行速度快得多，没有任何延迟。

##### Microsoft SQL Server

SQL Server 是微软企业数据战略的基石。it 领域的技术被广泛应用于从交易所到。NET 架构。对于许多使用其他微软软件的公司来说，它是首选的数据库，因为它被设计成可以整合在一起。

您最常发现的 SQL Server 是由那些服务器使用 Windows 操作系统的托管公司提供的。SQL Server 并不便宜，但它将为任何类型的数据提供可伸缩的体系结构。微软的 SQL Server 最初是作为 Sybase ASE 的许可副本移植到 Windows NT 上的，这两个系统仍然非常相似，因为直到最近微软才开始对底层引擎进行大的修改。

SQL Server 被设计为可伸缩的。您可以在多台机器上共享一个数据库，并使用复制来维护数据完整性。每台机器可以支持多达 32 个处理器和 64 千兆字节的 RAM，以实现终极处理能力。与 Sybase 一样，SQL Server 也支持 Web，允许通过 HTTP 查询直接访问数据，并能以 XML 格式输出数据。

***T-SQL***

SQL Server 也有自己的过程语言，由于它的 Sybase 根，它是基于 Transact SQL 的。微软只是把名字简称为 T-SQL。T-SQL 允许开发人员快速创建具有完整互联网功能的企业级应用程序。SQL Server 客户端应用程序包含一个完整的调试器，使开发过程更容易。使用调试器，您可以在应用程序运行时控制和监视它，以便在查询和其他代码中创建最佳优化。

***SQL Server 的一些长处***

**全文搜索**–SQL Server 2000 的全文搜索功能使您不仅能够快速搜索表中基于字符的字段，还能够搜索存储在硬盘上的文本文件、格式化文件(如 Word 文档)等。

**。NET 集成**–这是一个. NET 服务器，因此它可以与任何其他有能力的服务器集成，并将直接交换信息。这使得从社区到在线商店的在线应用快速发展。

**点击流分析**–该功能允许您对每次访问者点击您网站中的网页时捕获的信息进行数据挖掘。你可以找到他们从哪里来，他们去哪里，并绘制随着时间的推移，以产生有效的预测你的流量。你也可以决定你的顾客是在星期三还是星期四买更多的红袜子，以及他们还会买什么来搭配这些袜子。这为您提供了强大的信息，使您能够简化业务和数据。

如果您的网站使用 Windows 服务器，或者需要使用 ActiveX 访问 Windows 服务，那么 SQL Server 是您显而易见的选择。SQL Server 为在 NT 和 Windows 2000 上使用提供了一个强大的、可伸缩的平台。大多数 Windows NT 托管公司也提供这种服务，要么免费，要么每月收取少量费用，这使您能够以很少的费用或免费使用高端数据库服务器。

##### Oracle 9i 发行版

Oracle 是第一个在公开市场上销售的商业关系数据库服务器。今天，就功能和特性而言，它仍然被认为是头号数据库服务器。大多数《财富》500 强公司都将 Oracle 作为他们首选的数据存储方法。这种声望和权力是有代价的:Oracle 也是最昂贵的解决方案。数据库软件的最新版本是 9i(I 代表互联网)。Oracle 随其数据库包提供了一个完全标准的 Internet Web 服务器。该服务器可以直接从数据库中访问数据，就像它是一个普通的 HTML 页面一样。

Oracle 数据库系统可以访问最大 1tb 的文件，并且可以支持成千上万的用户。开发起来轻而易举，因为它拥有完全可用的 PL/SQL 语言、对存储过程和触发器的强大支持、安全控制和许多其他特性。然而，这也要付出代价——很大的代价。Oracle 是这里列出的最昂贵、资源最密集的数据库服务器。Oracle 数据库的安装和维护也不适合缺乏经验的人。也就是说，Oracle 将提供强大的功能来支持最大的全球企业，并处理来自世界各地的数万用户和交易。

甲骨文的大部分优势来自它的速度。这种速度来源于这样一个事实，即一旦它运行类似查询或存储过程的东西，这些信息就会以编译格式无限期地保存在内存中。当用户登录并访问表时，这些信息也会被缓存在内存中，并且只有在事务被提交到永久磁盘存储时才会被更新。这为数据库提供了超快速的访问，因为硬盘访问(该过程中最慢的一步)被消除了。

从 Windows NT 到大型主机，包括 Linux 和 Unix，Oracle 为所有操作系统提供支持。他们的安装包和 Oracle 数据库附带的许多实用程序都是用 Java 编写的，这使得他们可以在每次安装时使用相同的代码，并使软件更易于支持。

除了内存缓存特性，Oracle 还支持与 PERL、PHP、C++、Java、TCL/TK 和许多其他语言的完全集成。它还有一个强大的排序实用程序，用于生成
唯一的 id，其内部编程语言是无与伦比的。

***PL/SQL***

Oracle 有许多特性，但我觉得最感兴趣的是它的网关服务、故障转移和事务日志。

**网关服务**—这是一系列允许 Oracle 从任何来源访问数据的应用程序，包括本文中描述的不同数据库。通过使用网关，Oracle 可以像直接存储在自己的表中一样访问数据。这允许您像在本机系统中一样访问数据。

**故障转移**–您可以设计您的应用程序，使它们始终连接到具有故障转移和复制功能的数据库。使用故障转移，您可以获得由复制提供的数据库的副本，如果第一个数据库离线，可以访问该副本。这使您能够全天候不间断地访问您最重要的资产:公司的数据。一旦主数据库离线或查询失败，应用程序将自动进行故障转移。这不需要在应用程序或脚本中进行特殊的编程，只需要在一个系统初始化文件中增加几行代码。

事务日志(Transaction Logs)–这些日志旨在维护您的数据，并且包含每个事务的快照。通过在备份之间维护这些日志，您可以在数据损坏的情况下恢复所有数据。这些日志按顺序应用，并恢复备份和损坏之间发生的一切，因此它们为您的数据提供了良好的故障保护。

Oracle 是 RDBMS 领域的头号产品。是别人与之竞争并试图超越的那个。然而，每一次，他们都失败了。Oracle 提供了不同于任何其他系统的功能、速度和效率，但是，如前所述，这些优势需要付出巨大的代价。Oracle 也是目前世界上最昂贵的计算机应用程序，但如果您的数据依赖于最大的完整性和安全性，那么它就是您的最佳选择。
