# dev connections Spring 2007 Day 0:ScottGu、AJAX++和实体框架

> 原文:[https://www . site point . com/dev connections-spring-2007-day-0-scottgu-Ajax-entity-frameworks/](https://www.sitepoint.com/devconnections-spring-2007-day-0-scottgu-ajax-entity-frameworks/)

DevConnections 2007 于今晚开幕，由 Scott Guthrie、ASP.NET 先生本人和 Sql Server 关系引擎组项目经理 Christian Kleinerman 发表主题演讲。

Scott 是第一个，他主要关注一些客户端技术。在标准的[ASP.NET 阿贾克斯](http://ajax.asp.net/)狗&小马(Northwind + GridView + UpdatePanel，对于外行来说)之后，他演示了[ASP.NET 控制工具包](http://www.codeplex.com/AtlasControlToolkit/)的一些特性。他非常强调地提到，微软对该工具包的开发结果非常满意，这是通过将该项目分为一个稳定的、由微软开发的基础(ASP.NET AJAX)和一个提供大量功能的更灵活的层(ASP.NET AJAX 控制工具包)来实现的。他们也很高兴微软开发者和社区贡献的结合使得工具包的开发持续进行。他指出他们计划在即将到来的项目中使用这种设计和开发模式。

走完 ASP.NET AJAX 之路后，Scott 介绍了 Visual Studio 的一些关键特性。网虎鲸。是的，孩子们，JavaScript 调试是非常真实的。似乎为 Expression 系列设计工具开发的许多技术已经迁移到 Visual Studio 中。无论如何，请继续关注“几周后”即将到来的新 CTP。

但亮点是“AJAX++上的演示，或者更确切地说，是 WPF/E——微软试图创造一个闪存杀手。它有几个关键的区别——主要是直接与。NET 代码。这个平台非常强大——只需[获得插件](http://msdn.microsoft.com/wpfe/),将你的浏览器指向【http://www.windowsvista.si/】的[即可。虽然该网站是斯洛文尼亚语，但在线 Vista 克隆版在任何语言中都相当流畅。根据 Scott 的说法，我们可以在夏天期待 WPF/E 的 RTM。](http://www.windowsvista.si/)

克里斯蒂安涉及了很多领域。在“我从未听说过的非常酷的东西”类别中，他提到了 [Sql Server 性能仪表板](http://www.microsoft.com/downloads/details.aspx?familyid=1d3a4a0d-7e0c-4730-8204-e419218c1efc&displaylang=en)。当与 Sql 2005 SP2 结合使用时，它可以将许多常见的数据库性能报告集成到您的 Sql Server Management Studio 中。第二项是 Sql Server 最佳实践分析器 V2(尚不可用)，它可以分析您的 Sql Server 设置，以确保它符合已发布的最佳实践。非常方便，因为 Sql 2005 的配置比 Sql 2000 复杂一点，对事物进行健全性检查&审计总是好的。最后，Christian 暗示他们正在显著缩短 Sql Server 的开发周期，而且 Katami 的初始 CTP——Sql Server 的下一个版本——已经不远了。

最后，ADO.NET Sql Server 的项目经理 Brian Dawson 做了一个关于[实体框架](http://www.microsoft.com/downloads/details.aspx?familyid=E9BA57AA-2A27-4658-AD04-4380A2DF836C&displaylang=en)的简短测试——微软 ORM 方法的一部分。技术上的困难阻止了计划中的演示，但是我肯定会在明天检查那些会议。敬请关注更多内容。
