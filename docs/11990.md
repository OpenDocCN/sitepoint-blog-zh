# AppJet 起飞后，乘客可能会被困

> 原文：<https://www.sitepoint.com/appjet-discontinued/>

AppJet 项目背后的团队宣布，他们将从 2009 年 7 月 1 日起停止免费托管服务。该公司将继续专注于他们的 [Etherpad](http://etherpad.com/) 协同在线文字处理器。

AppJet 是一个允许开发人员在浏览器中创建服务器端基于 web 的应用程序的平台。该公司的创始人包括两名前谷歌工程师，这项服务于 2007 年 12 月作为公开测试版推出。它提供:

*   免费应用托管和存储
*   在线集成开发环境
*   自定义域名
*   初学者教程、示例库和支持论坛

应用程序开发是通过运行 Java 的虚拟机、Rhino JavaScript 引擎和 Scala 库实现的 JavaScript 框架来实现的。客户端开发人员可以利用他们的语言知识，在一个稳定的环境中开发服务器端应用程序，而不用担心托管、备份或带宽。在线 IDE 也是一个新的激进的想法，最近被 Mozilla 用于他们的 Bespin 项目。

在发布会上，创始人之一 Aaron Iba 表示:

> 我们厌倦了设置服务器、配置 apache、安装 MySQL、php 和 mod _ 无论什么的麻烦，只是为了让一个简单的 web 应用程序上线。所以我们很大程度上是为自己做这个(AppJet ),这样我们就可以更快地开发小应用。我们还认为，如果编写 web 应用程序比编写桌面应用程序更容易，那就太好了。这将改变一些事情:如果你要写一个简单的计算机程序，那么不要把它写在你的桌面上，不与任何人分享，你可以把它做成一个网络应用程序，其他人可以从中受益。

该公司 6 月 1 日的公告称，Etherpad 旨在成为 Appjet 的技术展示，但它变得比平台本身更重要，他们收到了数百份专业版本的请求。该公司决定转移他们的资源，并取消对 AppJet 托管服务的支持。

该公司正在为希望保留其 AppJet 应用程序的开发者提供建议和文件。不幸的是，在其他地方托管文件可能需要相当大的努力，在某些情况下，还需要一个专用的或虚拟的私有服务器。

虽然 AppJet 平台可能还没有被大量的开发人员使用过，但是这个消息对于云中编码的未来来说并不是一个好兆头。当定制服务随时可能被撤销时，企业主怎么能依赖这种服务呢？
