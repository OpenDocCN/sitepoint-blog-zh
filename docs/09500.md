# 六种云使用模式

> 原文：<https://www.sitepoint.com/cloud-usage-patterns/>

现在让我们来看看一些常见的云使用模式。有了这些信息，您应该能够很好地决定您的应用程序或工作负载是否适合 AWS。尽管所有这些模式本质上都代表了一段时间内的使用情况，但还是有一些重要的细微差别。在下面的案例中，“使用”通常代表常见云资源的组合—服务器、存储和带宽。

### 1.随着时间的推移不断使用

常见于每天或每小时的使用量或负载变化很小的内部应用程序。

### 2.循环内部载荷

以可预测的周期运行的批处理或数据处理应用程序的特征，例如一天或一个月的下班时间；时间和预期资源消耗方面的负载是高度可预测的。

### 3.周期性外部载荷

通常适用于满足特定市场需求的网站；与娱乐和体育赛事相关的网站通常符合这种模式。

### 4.峰值内部负载

通常在研究人员或分析师可以提交大规模、一次性作业进行处理的环境中使用；需求通常是不可预测的。

### 5.峰值外部负载

当一个不知名的网站突然流行起来时，在网上看到，通常是在很短的时间内

### 6.随着时间的推移稳步增长

通常用于成熟的应用程序或网站；随着用户的增加，增长和资源也会相应增加。
