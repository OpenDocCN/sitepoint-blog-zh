在很多情况下，你可能会决定自己托管你的网站。你可能是一个自由职业者，一个企业家，或者经营一个个人的甚至是非盈利的网站。您可能希望将网站托管在地下室的一台永远在线的 PC 上，或者工作场所的一台小型服务器上。

不管什么情况，本地托管通常都不是一个好主意。在这篇文章中，你会解释一些原因，希望能为商业决策者和对互联网“运营”方面缺乏经验的开发者提供有用的信息或警告——在这种情况下，是托管网站。

## 自托管起步门槛更高

根据您的互联网服务提供商(ISP)和您的情况(无论您使用的是商务类服务套餐还是住宅类服务套餐)，您可能会遇到 ISP 阻塞端口 80 的问题，该端口是大多数互联网流量的默认端口。如果您在本地托管网站，您还可能会收到警告，甚至被关闭服务。这是最常见的，当你试图从你的家庭连接上托管一个站点，而 ISP 更喜欢你这样做的话，你购买一个商务互联网包，价格可能是它的三倍。

然而，一些互联网服务提供商仍然可以提供限制，甚至对商务类套餐，可以阻挠您的尝试，让您的网站启动和运行。此外，您还需要一个静态 IP 地址，这可能是额外的费用，取决于您的 ISP 计划如何工作。大多数家庭(和一些商业)连接使用动态 IP 地址，这意味着它们会随时改变。如果你的 IP 地址，也就是可以在线找到你的网络的方式，有时改变了，你将没有简单、可靠的方法将访问`yourdomain.com`的用户指向你的本地托管设置。

## 自行托管网站会带来安全风险

当然，你在互联网上做的任何事情都有安全风险，但在你的网络上托管一个网站会让你更加开放，尤其是在你不确定自己在做什么的时候。大大小小的网站都受到拒绝服务攻击和脚本攻击，有时数量惊人。你的网站可以是一个自由职业者的，一个小公司的，一个两百人的小公司的，或者一个企业机构的，它们都会吸引攻击——其中一些甚至可能是自动化的，甚至没有考虑到网站的内容。

如果你不小心的话，在你的网络上托管一个服务器，可能会给你的网络带来潘多拉盒子般的安全问题，如果你是通过云服务提供商而不是本地托管你的网站，你永远也不会遇到这些问题。你对 web 服务器了解多少(如果你的 web 服务器是 Linux，[你知道命令行](https://www.sitepoint.com/premium/screencasts/navigating-the-linux-terminal-and-commands))吗？

## 自主托管你的网站不太可靠

另一点要考虑的是，关于网站的自主托管，你有更少的选择来处理停机和服务中断之类的事情。

如果在典型的住宅或小型企业场所停电，您将无法控制接下来的情况。多长时间能出来？分钟？几个小时？甚至几天？这取决于具体情况。如果孩子碰了或损坏了他们不应该碰的东西，如果是在你家，怎么办？在办公室里，如果一个粗心的员工弄洒了饮料，或者拉错了线，该怎么办？

即使您将网站安全地保存在办公室专用房间的服务器机箱中的服务器上，您仍然可能遇到随机事件，如管道泄漏或互联网服务中断。如果您的带宽超出了分配的数量，您的 ISP 可能会限制或关闭您的连接。这些事情中有许多完全不受你的控制。

## 考虑自己托管网站的成本

让我们来看看自主托管网站的成本:

*   静态 IP——你需要一个静态 IP 地址，以便人们在浏览器中输入你网站的地址时能正确地找到你的网络服务器。这通常会在你的网络账单上产生额外的费用。
*   更好的网络连接——你可能也想要一个更好的网络连接，尤其是当你的网站流量开始增加的时候。这也会让你付出代价。
*   设备——你还需要设备来安装网站。主要(或唯一)任务是充当 web 服务器的服务器或计算机。如果你使用的是 Windows，你需要一个操作系统许可证，也许还需要其他软件。您可能还需要额外的外围设备，如交换机或路由器，否则您可能不需要这些设备。
*   备份–如果可能的话，您需要支付外部云备份服务的费用，并且您还需要设备、软件以及执行本地备份所花费的时间。您希望在现场和异地都有数据的备份副本，以便可以使用本地副本进行快速恢复，并在发生灾难时使用异地副本。所有这些都需要钱，需要时间来设置和测试备份，以确保系统正常工作。
*   支付顾问费用——最后，你应该假设你最终需要支付顾问费用。你的企业可能不具备处理某些网络或编程问题的本地专业知识，如果你是一家个体商店，那就更是如此。你不妨把它记为必需品，并在需要的时候有一个应急预算来处理它。
