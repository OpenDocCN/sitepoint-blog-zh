# 网络字体:做一些积极的事情！

> 原文：<https://www.sitepoint.com/web-fonts-do-something-positive/>

马克·皮尔格林今天发表了一篇名为[去他的代工厂](http://diveintomark.org/archives/2009/04/21/fuck-the-foundries)的帖子，点燃了网络字体争论的火焰。很遗憾，他选择用愤怒的咆哮来表达自己的想法，因为他的语气破坏了一些重要的观点。

## 简单地说，这场辩论

这场辩论并不简单，但也不复杂。

好消息是，真正参与这场辩论的人(而不是那些只是在一旁大喊大叫的人)，包括字体供应商、浏览器供应商和专业网页设计师，实际上都同意一些重要的观点:

*   网络字体不应该被 DRM 所拖累(万岁！)
*   应该有一种便捷的方式来许可在 Web 上使用商业字体

在浏览器厂商中，对于浏览器是否应该支持使用网络上现有文件格式的“免费”字体(对于“免费”的大多数含义)存在激烈的争论。事实上，这场辩论是一种干扰，如果真正的问题能够得到解决，这种干扰就会立即消失。

唯一有意义的一点是，相关人员不同意的是，用于网上许可的商业字体的文件格式是否应该包含嵌入其中的许可细节。如果是，必须开发这种格式并为浏览器添加支持。

微软已经建立了这种格式的雏形，但是还需要进一步的投资。同时，有许多人认为在所有主流浏览器中开发、许可和支持这种格式所需的时间和精力太多，或者几乎没有实际好处，因此是不合理的。

眼下，游戏处于胶着状态；微软和商业字体供应商认为新的文件格式是必要的，但无法用现有的时间和资金来开发它。其他人都认为像 OpenType 这样的当前字体格式是我们所需要的，但是他们并没有控制字体。

如果你玩过象棋，你就会知道没有什么比僵局更令人沮丧的了:每个人都怒气冲冲地走开了。这就是为什么最近字体争论中唯一真正的进展似乎是像马克·皮尔格林那样的愤怒爆发。

## 做一些积极的事情

作为一名网页设计师，无论你相信什么，你可能也会感到沮丧。试着对网络上的字体做出积极的贡献，而不是加入大喊大叫的比赛。

这里有一个重要的方法，你可以:**买一种字体。发送消息。**

也许，像 Mark Pilgrim 一样，你会很乐意付费在你设计的网站中使用商业字体，而且你不相信像你这样正直的网站设计者需要一种新的文件格式来让你知道适用于字体的许可条款。要是那些落后的字体代工厂肯收你的钱就好了！

你猜怎么着:他们会的！

如今，在网络上使用商业字体有种合法方式；它们只是不太方便。尽管如此，专业设计师每天都在使用像 [sIFR 和 CSS 图像替换](https://www.sitepoint.com/article/fixing-fancy-font-frustrations/)这样的解决方案在网络上使用商业字体。

下一次你设计网站时，使用这些技术中的一种来证明你愿意支持 Web 上的商业字体。简单地在你的设计中包含一两种商业字体，并确保你购买了使用这些字体的许可。一旦你做到了——这是最重要的部分——告诉全世界你做到了！

在你的网站上包含一个页面，列出你使用的字体，并声明它们是完全授权的。给设计字体的设计师或铸造厂发一封电子邮件，让他们知道你买了这种字体用于网站设计。

最重要的是，在你的笔记中提到，如果他们只提供允许你这样做的许可证，你会很乐意支付额外的费用(公平的)来使用你网站上现有的 OpenType 版本的字体文件。

所有这一切的好消息是，商业字体供应商并不像马克·皮尔格林的长篇大论所说的那样是一个铁板一块的实体。他们中的大多数都是小工作室，只有几个设计师，他们只想设计字体并获得报酬。我们不必一下子说服他们所有人。

这只是为解决一场迄今为止一直被消极情绪主导的辩论做出积极贡献的一种方式。你能想到其他人吗？
