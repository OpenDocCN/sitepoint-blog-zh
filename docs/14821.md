# Flash 与 CSS/HTML:你会选择哪个？

> 原文：<https://www.sitepoint.com/csshtml-which-will-choose/>

**Macromedia Flash。这是 Web 开发世界中最有争议的产品之一。**

在本文中，我们将了解闪存的优势和劣势。我们将仔细看看它能做什么，不能或不应该做什么。并且，通过这个过程，我们将 Flash 与其对应的 HTML 和 CSS 进行比较。

这篇文章肯定会引起争论…并给我带来很多额外的邮件！但是我写这篇文章的目的是让 Flash 新手了解这项技术。至少，本文应该允许您与潜在客户交流项目应该或不应该在 Flash 中完成的原因。

##### 大胆的声明

HTML 和 CSS 永远做不到 Flash 做的事情。仅凭这句话，有些人就推出了他们的邮件客户端，但在你撰写那封充满苦涩的电子邮件之前，请允许我解释一下这句话。

让我们来看看 Macromedia 的多媒体矢量产品 Flash 的优势。然后我们将检查它的弱点，并以 HTML/CSS 串联的比较结束。

有哪些 HTML 和 CSS 做不到的事情，Flash 做到了？

***闪光的长处***

*自由*

Flash 提供了其他地方没有的开发自由度。Flash 允许设计者将对象放置在他们喜欢的任何地方，而不必担心绝对/相对位置、框模型问题、浏览器间的不一致、z 索引、表中表、切片图形或屏幕分辨率。在 Flash 中，您只需将对象放在画布上您想要的位置，然后继续移动。终于自由了！终于自由了！感谢全能的上帝，我们终于自由了！

说真的，这个优势不可小视。作为设计师，这就是我们自网络诞生以来一直想要的:不受浏览器或编程语言限制的设计能力。有了 Flash，您不再需要担心您的页面在哪些浏览器中可以正确呈现，哪些浏览器会出现问题。您的页面在所有浏览器中都以相同的方式正确呈现。

*更强的互动性*

Flash 为更复杂的多媒体体验提供了一个交互式环境。我们不只是在谈论交换颜色或图像的翻转，顺便说一下，这在 Flash 中是小菜一碟，它们甚至可以包括声音。我们在谈论更大的交易。

Flash 允许您通过 mp3 和 wav 文件将声音整合到您的页面中，允许您使用例如语音或背景音乐。您还可以直接将数字视频导入 Flash，这样您就可以轻松地演示产品或展示产品系列。诚然，Flash 并不是将视频加入网页的唯一方式。还有其他工具，如 Windows Media Player、Real-Player 和 QuickTime。然而，Flash player 插件的渗透率现在约为 97%。与 Windows Media Player 插件的 59%、Real-Player 插件的 56%和 QuickTime player 插件的 41%相比，Flash 遥遥领先。

好吧，也许你对网站上的音乐和视频感到恼火。没问题！不加就是了。但是至少你可以选择将这些元素整合到客户端站点上，如果你使用 Flash 的话。HTML/CSS 根本没有提供这个选项。

*更大的控制权*

Flash 的矢量渲染过程使它能够根据浏览器大小调整整个显示大小，这样做可以保持图像和文本的整洁和不失真。随着人们开始更频繁地从台式电脑之外的来源上网，如 PDA、移动电话、汽车显示器或手表，这将变得更加有利。

*改进的集成*

Flash 允许您将任何多媒体文件格式集成到您的站点中。例如，位图图像格式(如 GIF、JPEG、PNG、PCT、TIF)、矢量图像格式(包括手绘文件、EPS、Illustrator 文件)以及前面提到的声音格式(WAV、AIF、MP3)都可以导入到 Flash 电影中。

*更简单的字体处理*

使用 Flash，您可以嵌入任何您想要的字体，并让它显示在客户端浏览器中，而不管他们的计算机上是否安装了该字体。您不再需要为了在不同的用户系统上显示而将字体转换为图像。

*复制帧*

Flash 可以复制 HTML 网站上框架的行为，因为当屏幕的其他部分滚动时，它可以将网页的某些部分(如导航按钮)保持在固定位置。

*独立电影*

Flash 允许您将电影作为称为投影仪的独立演示来播放。投影仪是带有嵌入式播放器的 Flash 电影，您甚至不需要网络浏览器来播放这些 Flash 电影！您可以将 Flash 投影仪刻录到 CD-r om 或 DVD 上。

*没有重装*

Flash 使您能够在不断变化的过程中显示数据，而无需重新加载或刷新页面！没错——数据可以在浏览器不动声色的情况下改变。

好了，我们已经介绍了这项技术的优势。现在，我们来看看 Flash 的弱点。

***闪光的弱点***

*需要插件*

Flash 电影仍然需要安装单独的插件才能在大多数浏览器中运行。许多公司不允许他们的员工在他们的计算机上安装应用程序，这自动拒绝一些 Flash 网站所有者访问他们的潜在用户群的一部分。

*视频灾难*

Flash MX 中的视频压缩和播放质量与一些专门为此目的开发的播放器(如 QuickTime)不同。例如，Flash 视频没有利用增强的视频驱动程序来优化放大后的播放。

*搜索引擎问题*

典型的网络搜索引擎(或蜘蛛)不能索引 Flash 电影中的内容。如果你创建一个 100%基于 Flash 的网站，如果你想让你的内容被搜索引擎索引，你可能想在你的网页上提供一些显示或隐藏的文本或 HTML。当我说隐藏时，我的意思是内容应该通过无脚本标签或搜索引擎可接受的其他方式可用。这可能是 Flash 最大的缺点，是的，这是个大问题。

*缺少屏幕阅读器支持*

虽然将 MSAA 兼容性集成到 Flash Player 6 中是一大进步，并且已经被辅助功能专家宣布，但是许多类型的屏幕阅读器还不支持 Flash Player 的 MSAA。

*打印问题*

如果你的网站是以向浏览者传递基于文本的信息为基础的，那么不要用不必要的动画来分散或延迟他们获得他们想要的东西。对于用户来说，在 Flash 电影中打印和选择文本通常不像在 HTML 网站上那样简单(或熟悉)。

##### CSS 逐步升级

如今，网络上日益增长的趋势是将设计与内容分开。因此，让我们感谢 CSS——毕竟，分离这两个元素是 CSS 的设想，也是它现在实现的如此之好。CSS 允许你创建一个网站的外观和格式，而不影响你的内容。作为开发者，你会受益，搜索引擎蜘蛛和文本阅读器也会受益。

此外，CSS 使网站维护更容易。只需对一个外部 CSS 文件做一个简单的修改，你就可以改变网站的多个页面。要深入了解 CSS，一定要查看一下 [SitePoint 的大量 CSS 资源](https://www.sitepoint.com/subcat/90)，包括他们的关于主题的[书籍。](https://www.sitepoint.com/books/css1/)

CSS 能够以最纯粹的形式提供内容，这使得这种技术非常适合向搜索引擎提供纯粹的内容，向纯文本读者提供易读的文本。结合 CSS 为网站所有者提供的通过改变单个文件来修改整个网站的能力，很容易理解为什么这么多人会爱上这项技术。

…但这一切都是美好的吗？别着急，CSS！

##### 向前闪

想一想 CSS 给你的所有这些东西——那些让你爱上 CSS 的好处，包括搜索引擎就绪的内容、文本阅读器可访问性，以及从单个文件修改整个网站外观的能力…

你猜怎么着？很有可能在不久的将来，Flash 也会为您提供这些功能。

FAST 网络搜索副总裁 Tim Mayer 解释说:“搜索引擎最初是为了索引和服务 HTML 文档而构建的，但现在网络的内容类型变得更加多样化，知道如何对待 Flash 和其他类型的内容对搜索引擎来说变得更加重要。”

Mayer 指出，“FAST 使用 Macromedia 的 Flash 搜索引擎软件开发工具包(SDK)构建了它的 Flash 索引功能。SDK 旨在将 Flash 文件的文本和链接转换为 HTML 以进行索引。”

然而，并不是所有的搜索引擎蜘蛛都有抓取或索引 Flash 的能力。据我所知，谷歌还没有像 FAST 那样包含 Flash-SDK 的索引设置。但是谷歌可以跟踪嵌入的链接。

Macromedia SDK 解决方案远非完美，还有很长的路要走。然而，我们可以看到搜索引擎正在努力拥抱 Flash，我相信在未来我们会看到 Flash 努力拥抱搜索引擎。

可以想象，用不了多久，搜索引擎将只会看到一个网站的内容——他们不会关心那个网站看起来像什么。内容将与电影分开，甚至可以用 HTML 标签或 XML 进行标记。这为设计者提供了让他们的站点看起来像他们选择的那样的能力，并且仍然让搜索蜘蛛享受内容。搜索引擎将能够做人类一直在努力的事情——他们将有能力透过外表看到真正重要的东西。

同样值得注意的是，当这一天到来时——所有迹象都表明这一天会更早到来——不仅搜索引擎会接近你的 Flash 内容，文本读者也会发现它更容易访问。

最后，Flash 和 CSS 一样，也允许用户修改单个外部文本文件，这样就可以对整个网站进行修改。因为 Flash 将电影的所有元素存储在一个库文件中，所以您可以轻松地对整个网站或多个网站(通过 Flash 的共享库功能)进行全面修改。
闪存的未来是光明的

还记得我当初说的“HTML 和 CSS 永远做不到 Flash 做的事情”吗？虽然这是真的，但很有可能在未来，Flash 将能够实现 HTML 和 CSS 所做的事情。上述闪存的缺点在不久的将来可能不会出现。可以想象，Flash 将能够把内容从设计中分离出来，为搜索引擎和文本阅读器提供纯粹的内容；毕竟，Macromedia 现在正在解决这些问题。

总有一天，一个典型的 Flash 网站会像它的 HTML/CSS 对应物一样对搜索引擎友好；就可访问性而言，它甚至可能超过 HTML 和 CSS 网站，原因很简单，它支持声音和视频。

Flash 至少有机会修复它的弱点，而且很可能会及时修复。但是 HTML 和 CSS 永远做不到 Flash 做的事情！
