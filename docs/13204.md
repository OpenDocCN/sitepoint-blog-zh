# 微软和 Mozilla 在浏览器安全性上意见不一

> 原文:[https://www . site point . com/Microsoft-and-Mozilla-disagree-on-browser-security/](https://www.sitepoint.com/microsoft-and-mozilla-disagree-on-browser-security/)

微软和 Mozilla 陷入了一场关于谁的浏览器有更好的安全记录的口水战。然而，所有的网络开发者似乎关心的是，Mozilla 刚刚发布了 Firefox 3 的第一个公开测试版，而微软还没有对 IE.Next 说太多

在 IE7 发布一年后，微软的 IE 博客发布了[IE7 的第一年](http://blogs.msdn.com/ie/archive/2007/11/30/the-first-year-of-ie7.aspx)，总结了浏览器的成就:

> 根据微软基于视觉科学公司数据的内部研究，有超过 3 亿用户使用 IE7 体验网络。这使得 IE7 成为仅次于 IE6 的第二大流行浏览器。IE7 已经在美国和英国排名第一，我们预计 IE7 将很快在全球范围内超过 IE6。

微软 Internet Explorer 项目组经理托尼·Chor 继续讨论 IE7 在安全领域的成就:

> 根据今天发布的一份[漏洞报告，IE7 在同一时间段内比之前版本的 IE 漏洞更少。更重要的是，报告显示 IE7 在第一年比我们比较的其他浏览器有更少的已修复和未修复的漏洞。](http://blogs.csoonline.com/blog/jeff_jones)

显然，这种说法肯定会激怒 Mozilla 的一些人。首先插话的是 Mozilla 的营销副总裁 Paul Kim，他指出微软引用的报告实际上是由微软准备的，在《T2 的谎言，该死的谎言》和《微软安全营销》中:

> 那真的应该说:
> 
> “根据微软安全战略主管 Jeff Jones 今天**发布的一份漏洞报告，在同一时期，IE7 比以前版本的 IE 有更少的漏洞。”**

这很快得到了来自 Mozilla 的其他声音的跟进，包括 Mozilla 的工程副总裁迈克·斯科洛普夫(又名施瑞普)，他揭穿了微软在[苹果、欧根斯和真相](http://weblogs.mozillazine.org/schrep/archives/2007/11/use_the_metric_which_suits_you.html)中的声明:

> 为了验证这些数据，我去了微软大张旗鼓发布的公共 IE 漏洞数据库，我遇到了这个:
> 
> [![Thank you for visiting the IE Feedback Site. The site is temporarily closed. It will re-open in the future.](../Images/7bc8ef9b749d106d3254efa98de5c3b4.png)T2】](http://people.mozilla.com/~schrep/Capture.JPG)
> 
> 这生动地提醒了我们，除了微软之外，任何人都没有办法确认 Internet Explorer 中曾经存在多少漏洞。

Schrep 接着展示了他自己对过去一年浏览器安全性能的衡量标准，描绘了 IE7 记录的一幅截然不同的画面:

> Bug 计数没有意义，重要的是你是否有风险。赛门铁克在 T1 之前就关注了这个问题，华盛顿邮报的 T2 布莱恩·克雷布斯也关注了这个问题。我最近发现[这份](http://www.webdevout.net/browser-security)关于 Secunia 的最新数据分析描绘了同样的画面。Firefox 比 IE 更安全:
> 
> [![](../Images/08a4cbf46509595df35ab97ad2075efb.png)T2】](http://people.mozilla.com/%7Eschrep/security-record.png)

我会让你自己决定相信谁，但作为 Web 开发人员，我们并不特别关心哪个浏览器的安全漏洞最少。我们真正想知道的是我们什么时候能期待下一个有新的开发者特性和改进的标准符合性的浏览器版本。

在这方面，Mozilla 显然也处于领先地位。Firefox 3.0 Beta 1 于两周前发布，现在，微软甚至还没有告诉我们 IE 的下一个版本将会是什么*叫做*。

IE 上的[内字](http://twitter.com/mollydotcom/statuses/470201552)。接下来是将会有信息在 [MIX08](http://visitmix.com/2008/) 上透露，但那要到三月份。看起来那些等待微软下一代浏览器的人可能会面临一个非常寒冷的冬天。
