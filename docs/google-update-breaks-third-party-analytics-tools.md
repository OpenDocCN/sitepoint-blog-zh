# 谷歌变革打破第三方分析工具

> 原文：<https://www.sitepoint.com/google-update-breaks-third-party-analytics-tools/>

几天前，博客作者 Michael VanDeMar 写道，谷歌似乎正在美国测试 AJAX 搜索。VanDeMar 注意到他的 Google 查询的 URL 从熟悉的“/search？= " to "/搜索？#= "对于用户来说,#是“页面变化”的视觉提示，并允许浏览器的后退和前进按钮进行 AJAX 调用。

在 VanDeMar 的帖子之后，很多人都在猜测 Google AJAX 搜索可能会带来什么后果。从破坏 Firefox 插件，到导致搜索排名检查器需要重写，到至少暂时停止自动 SERP 抓取器。但是，与范德玛本周发现的更广泛的含义相比，这些都显得苍白无力。Google going AJAX 意味着任何外部分析包都无法读取访问者访问你的网站所使用的搜索查询(至少是从 Google——当然，这是大多数搜索流量的来源)。

分析包通过读取浏览器发送给服务器的引用 URL 的文本来获取搜索查询信息，以告诉服务器您是如何到达该页面的。Google 的 AJAX 搜索打破这一点的原因是浏览器在# hash 标记后不发送任何信息。这不是分析包可以解决的问题——改变它的唯一方法是在浏览器级别重写——它对基于日志的分析软件的影响和对基于 JavaScript 的一样。

目前，它甚至影响了谷歌自己的分析包，显示来自谷歌 AJAX 搜索的推荐来自 google.com。但是分析软件制造商的担心是可以理解的。“如果这次更新对每个人都有效，这实际上意味着 2/3 的普通网站搜索将完全是一个谜。这是巨大的，”第三方分析提供商 [Clicky](http://getclicky.com/blog/150/googles-new-ajax-powered-search-results-breaks-search-keyword-tracking-for-everyone) 在他们的博客上写道。

谷歌为什么要做出这样的改变？嗯，一种可能性是，该测试只是一个测试，不会对所有人开放。在系统范围的变更之前进行测试的原因是为了了解变更的后果。然而，正如 Clicky 所说，另一个原因可能更“邪恶”。Google Analytics 就像其他的分析软件一样已经崩溃了，但也许不会永远。

Clicky 写道:“当然，我愿意打赌，无论如何，Google Analytics 仍然能够获得搜索词。”“我现在可以看到:‘哦，对不起，我们弄坏了 Clicky？别担心伙计们！来使用谷歌分析，一切都会好的！哦，我们在这个过程中彻底摧毁了我们所有的竞争对手？哎呀！"

搜索博客作者彼得·达·万佐也有类似的想法。他表示，如果网站所有者想要完整的搜索推荐信息，谷歌可能会将他们锁定在谷歌分析中。他提醒我们，谷歌以前也做过类似的事情。

“还记得站长的变化吗？谷歌推出了一种新形式的跟踪代码，无法被第三方工具跟踪。然而，这些数据可以在谷歌分析中找到，”他在 SEOBook 的一篇博客文章中写道。

谷歌发言人[今天](http://searchengineland.com/google-ajax-search-results-death-to-search-term-tracking-16431)表示，这个实验只是一个实验，只对部分用户可见。谷歌表示，目标是让用户搜索更快，打破推荐追踪并不是他们的本意。这位发言人说:“我们无意中断推荐追踪，我们将继续迭代这个项目。”
