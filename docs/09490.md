我非常满意地向大家介绍“从。NET”系列。在这篇文章中，我将解释你应该从这个系列中期待什么。

希望我会在这里呆一段时间，所以我会告诉你一些关于我是谁和我的背景。

## 这个系列是关于什么的？

这个系列是关于 Ruby 的。这个系列是关于 Rails 的。也是关于 Ruby on Rails 的。但是最重要的是，这个系列都是关于指导的。NET 开发人员可以轻松进入 Ruby、Rails 和相关领域。

对大多数人来说，如果你能把任何新东西和你熟悉的东西联系起来，学习起来会更容易。就像学习口语一样:当我们学习第二语言时，很自然地会在第二语言和第一语言之间画出平行线。

这个系列将充满 Ruby 和 C#、Rails 和 ASP.NET、Visual Studio 和…(嗯，在 Rails 世界中没有一个真正占主导地位的 IDE，但那是另一篇文章)。

它以一位前任的真实经历为特色。最近沉迷于 Rails 开发的 NET 开发人员。疼痛是真实的，但是药尝起来像泡泡糖！

## 这个系列是给谁看的？

主要有两种类型。NET 开发人员:那些始终坚持微软的人(只使用 MS 工具、框架等。)，以及那些主要目标是微软用户，但他们更愿意使用任何最终能完成工作的东西，不管是微软的、第三方的、开源的还是内部构建的。第一个群体拥有绝大多数的开发人员，而第二个群体在过去几年中一直在增长。

这个系列关注的是第二类人，他们不害怕尝试新事物。

第一组人呢？他们通常不愿意打破他们的舒适区，因为这只是他们的个性，或者其他原因，如政治或经济。不管情况如何，也许随着时间的推移，我们可以让他们中的一些人不时品尝不同的水。

如果你已经是一个有经验的 Rails 开发人员，你在这里唯一可以学到的就是什么样的困难。NET 开发人员通常会遇到 Rails。如果您是我们的顾问或培训师，这可能会有所帮助。网友们。

## 历史会重演

几年前，我是一名 Clipper 程序员。我对此很满意，也很有成效。然后来了一个和 FoxPro，Visual FoxPro 合作的机会。我接受了它，并找到了一个很好的方法来了解我的新工具:在浏览了一本针对初学者的小册子后，我开始在在线论坛上问大量的问题，并从许多优秀的人那里获得了帮助。就在我试图在网上帮助人们之后不久。我对它了解不多，所以我会看到某人的问题，然后出去研究这个话题。这样，我既能学到东西，又能帮助别人。最终我在 FoxPro 上变得很好，我又很舒服了…

然后来了. NET，我是怎么学的？我学到了一些非常基础的东西，有很多优秀的人帮助我，并且尽我所能帮助别人。最终，我能够胜任。网，我又舒服了…

然后是 Ruby on Rails。我做了什么才学会的？我学到了一些非常基础的东西，伟大的人帮助了我，现在…好吧，现在我正试着帮助其他人(希望是你！)，而我仍然有伟大的人帮助我(希望是你！！).
