# 设计竞赛 2.0 在没有服务器的情况下启动

> 原文:[https://www . site point . com/design-contest-20-to-launch-without-servers/](https://www.sitepoint.com/design-contests-20-to-launch-without-servers/)

SitePoint 总部的团队似乎对即将发布的改版 [SitePoint 设计竞赛](https://www.sitepoint.com)屏住了呼吸。经过几天、几个晚上和几个周末的紧张开发，该应用程序现在即将上线……几周之后，它的服务器就准备好了！亚马逊弹性计算云让这个小小的奇迹和其他几个奇迹成为可能。

如果计划在最先进的、多服务器可伸缩的 PHP 5 架构上从头重写现有功能，并使用完全符合标准的前端，这还不够大，随着时间的推移，该项目变得越来越雄心勃勃，在每个阶段都添加了新的功能和可用性改进。

同时，开发工具也在变化。仅举一个例子，团队花了一个周末的时间构建了一个持续集成框架，该框架可以更容易地将构建部署到登台服务器，并在应用程序的更改破坏了自动化测试时向团队发送电子邮件。

甚至基础设施也是一个移动的目标。在漫长的规划期间，一个闪亮的新专用服务器集群被组装在纸上，并在各种主机上销售，在此期间，该团队听到了一些基于目前处于测试阶段的[亚马逊弹性计算云(EC2)](http://www.amazon.com/gp/browse.html?node=201590011) 的成功故事。

简言之，EC2 让你可以在亚马逊的服务器上托管你的网站。该服务使您的应用程序看起来像是在您自己的专用物理服务器集群上运行，而实际上这些服务器是使用 CPU 虚拟化技术模拟的。

你只需上传一个硬盘镜像到 Amazon，然后你就可以添加和删除运行该硬盘镜像的虚拟服务器，以满足你网站的流量需求。最重要的是，它们的运行成本比真正的物理服务器低得多！

当然，所有这些服务器魔术都增加了管理方面的复杂性，因此该团队又花了一个周末来构建一个工具，以自动管理 EC2 服务器集群。

尽管 EC2 仍处于测试阶段，但这项服务已经从一个奇怪的辅助项目变成了 SitePoint 设计竞赛 2.0 的主要服务器平台！该网站实际上将在我们的新服务器集群准备就绪前几周(T0)推出，最初将完全在 Amazon EC2 上运行。假设一切顺利，我们的物理服务器将充当备份，以防 EC2 出现故障。

当然，在发布之后仍然有一个计划增强的洗衣清单，所以该团队决不是在它旅程的终点。然后是 SitePoint Marketplace 2.0 项目，也将在未来几周内完成并启动。

希望我能挤出团队的一些时间，写下他们一起创造的这个技术奇迹中更有趣的部分。同时，和我一起祝贺他们在设计竞赛 2.0 上线时的发布！
