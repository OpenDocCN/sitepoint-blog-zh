# 你的客户希望你如何处理他们的数据？

> 原文：<https://www.sitepoint.com/handling-client-data/>

无论您处理的客户数据仅限于 FTP 访问和密码，还是管理财务数据和其他敏感机密信息，拥有一个标准(安全)的系统来维护和备份这些信息都是至关重要的。很多时候，客户的偏好决定了如何处理。

有许多方法可以管理客户端数据。这篇文章介绍了管理客户数据的两种方法——离线和在线——以及每种方法的优缺点。

出于讨论的目的，让我们假设我们引用的数据是链接、登录信息、访问细节、其他密码和杂项注释的组合。假设这些信息可以改变，并且需要与他人共享。

## ![](img/3125d46caffbbf0c6352423b73195245.png "lock")离线数据管理

有些人更喜欢在本地维护所有数据，并根据需要共享。这可能意味着有一个主电子表格，或者甚至是一个手写的文档，可以根据需要进行更新，但它是唯一的主要版本。

虽然这种方法在许多方面似乎已经过时，但对于那些担心在线系统安全的人或那些喜欢将所有东西都放在家里和手边的人来说，这是一个很有吸引力的选择。您可以限制共享哪些数据，控制谁可以访问(至少在某种程度上)，并确保文档本身仍然可以访问。

这种数据管理方法最明显也是最大的挑战是共享。我想到的一些问题包括:

*   主文件是零碎解析出来的吗？
*   整个文件是否通过电子邮件来回发送？
*   主文件有多大？
*   有更新时会发生什么？
*   谁负责维护文件的完整性？

## 在线数据管理

在线数据管理涉及拥有一个或多个中央系统，用于维护、更新和共享数据。这可以通过项目管理系统，甚至是密码管理工具来实现。

这是一种非常有效的方法，可以管理数据并与多个地点的多人共享数据。每个人只需登录到协作网站，并在需要时获得他们需要的更新信息。

在线数据管理的挑战有两个方面:确保数据的可访问性和安全性。要考虑的问题包括:

*   储存场所的安全措施是什么？
*   如果网站关闭会发生什么？
*   谁控制对数据的访问？
*   谁管理数据的更新？

## 你说什么？

总的来说，我倾向于认为，在一个安全的协作环境中，在线访问信息更容易，这样每个需要访问信息的人都可以这样做。我还发现，只有一份信息副本是处理版本控制并确保每个人都使用相同信息的最佳方式。
