# 介绍站点点角色

> 原文：<https://www.sitepoint.com/sitepoint-personas/>

我们最近决定开发一些人物角色来帮助 sitepoint.com 的设计过程。这是我们第一次开发人物角色，我们选择通过结合一些方法来进行实验。以下是这次练习的结果。

在我向你介绍我们家庭的新成员之前，有几点需要注意:

*   当我们设计下一个版本的 sitepoint.com 时，这些人物角色无疑会对我们有用，但它们显然无法捕捉到每一个访问者的细微差别。如果你认为我们失去了很大一部分观众，那么我们当然想听听，但是如果这些*个人*的背景故事与你自己的大相径庭，请不要见怪。例如，我们知道不是所有的青少年都是麻烦制造者；毕竟这些都是虚构的人物。
*   关于创建人物角色的最佳方式，以及他们到底有多有用，有不同的观点。如果你对我遵循的过程感到好奇，你可以在这里阅读全部内容。
*   你想知道你最认同哪个 SitePoint 角色吗？参加我们的[趣味测试](https://www.sitepoint.com/quiz/personas/)来寻找答案吧！

好了，让我来介绍一下 site point Personas——一个由八个虚拟读者组成的小组，我们认为他们很好地代表了大多数访问者。他们被分成两组:*回头客*和*新访客*。如果你同意，或者你认为我们大错特错，请告诉我们。

##### 回头客

***前端编码器***

![The Front-end Coder](img/8531ecbdc840e2a8002cc00a9ed2aeeb.png)

艾琳·菊池(28 岁，美国)

Erin 住在洛杉矶，是一家大型电影制作公司的网站开发人员。每当一部电影上映，她的团队都会推出一个新的网站和电子邮件活动来推广这部电影。她认为自己是一个高级水平的客户端编码器和中级 PHP 开发人员。她和一只名叫弗瑞丝的白色波斯猫相依为命。

Erin 的团队主要由 Flash 开发人员组成，因此大部分 HTML、CSS 和 JavaScript 编码都落在了她的肩上。她经常去 sitepoint.com 查看参考网站[中代码的语法，但是还没有购买任何一个硬拷贝版本。](https://reference.sitepoint.com)

艾琳也关注着[教程](https://www.sitepoint.com/recentarticles/)，最近开始更加关注[博客](https://www.sitepoint.com/blogs/)。她不使用 RSS，所以只要有时间，她就会查看网站的更新。Erin 确实有一个论坛帐户，并且在她职业生涯的早期曾经与论坛社区互动。然而，这几天她缺少时间，所以已经有一段时间无法登录了。她访问的其他网站包括[、](http://alistapart.com)和[思考维生素](http://thinkvitamin.com)。

Erin 运行的是 Windows Vista，她选择 Firefox 作为浏览器，主要是因为 Firebug 插件很有用。

艾琳以一种**自发**的方式做出购买决定——例如，
她从亚马逊购买了一本 Ajax 书籍，纯粹是因为这本书是她最喜欢的博客作者之一写的，希望她可以将一些 Ajax 融入到即将到来的项目中。事实上，该项目并没有进行下去，但她很高兴与她的购买。

艾琳在访问 sitepoint.com 时问自己的问题包括:

*   如何找到 CSS border 属性的正确语法？
*   最近有没有和我相关的教程？
*   我怎样才能更多地了解这本书是否适合我？
*   关于我订购的图书，我如何联系客户支持？

***工作狂企业主***

![The Workaholic Business Owner](img/9fcca5c4c52bc719fba82b154e85b512.png)

塞缪尔·约翰逊(35 岁，美国)

Samuel 来自西雅图，但后来搬到了旧金山，创办了一家创业公司。虽然 Samuel 自己设计并构建了公司的原始网站，但现在他花在实际设计和编码上的时间更少了，而将更多的时间放在了业务扩展上。他对客户端和服务器端技术有中等程度的了解。塞缪尔和他的女朋友莱拉合住一套公寓。

塞缪尔雇佣了六个人，他的公司发展很快。这些年来，他购买了几本 SitePoint 书籍，骄傲地陈列在办公室书架上，供其他工作人员阅读和参考。Samuel 很乐意投资学习资源，如果这意味着他的问题会得到解答。

塞缪尔订阅了网站点*论坛报*和设计视图电子邮件时事通讯。他也阅读大量博客，包括 [TechCrunch](http://techcrunch.com) 、[读写网](http://readwriteweb.com)、[塞思·戈丁](http://sethgodin.typepad.com/)和[软件上的乔尔](http://joelonsoftware.com/)。自从他开始使用 FeedBurner RSS 阅读器，他开始访问 SitePoint，主要是点击进入一个教程或者在博客上发表评论，尽管他仍然是 SitePoint Marketplace 的一个相当频繁的用户。

Samuel 运行 Windows XP，并使用 Internet Explorer 作为他的浏览器。他[在 Twitter](http://twitter.com/sitepointdotcom) 上关注 SitePoint，但将 Twitter 比作“从消防水管中喝水”,并且由于时间限制，最近很少使用它。

Samuel 展示了一种**竞争性**购买模式。他根据逻辑和手头的信息迅速做出决定。如果一个产品看起来符合他的需求，那么他可能会当场做出购买决定。

Samuel 在访问 sitepoint.com 时问自己的问题包括:

*   我在哪里可以读到如何外包工作？
*   我在哪里可以找到关于使用社交媒体来推广我的业务？
*   我如何提交文章发表？
*   这些人经营的播客我可能会感兴趣吗？
*   这个博客的 RSS 源在哪里？

***论坛瘾君子***

![The Forum Junkie](img/865c8057fe44936ab093d0a64d9f76dd.png)

拉吉·苏曼德拉(19 岁，印度)

Raj 住在印度钦奈，刚刚开始在一家大型政府机构担任网页设计师。他的工作量相当轻，因此他可以腾出大量时间在论坛上发帖。Raj 热衷于优秀的设计，他喜欢偶尔参加 99designs.com 的竞赛。他是 Adobe Photoshop 的高级用户，但在编写标记和代码方面处于中级水平。Raj 和他的父母住在市郊的一栋高层建筑里。

Raj 是 SitePoint 论坛的长期会员，已经发布了 3000 多个帖子。他访问 sitepoint.com 的主要动机是感受一种归属感；他在社区中与主持人和顾问建立了许多友谊，并喜欢与他们所有人互动。他还定期阅读文章，这些年来自己也写了一些教程。

Raj 也活跃于[数字积分论坛](http://forums.digitalpoint.com)，尽管程度较低。

在工作中，Raj 使用 Internet Explorer 和 Windows XP。他在 Twitter 上关注 SitePoint。

Raj 以一种人文主义的方式回应在线广告——他重视与公司相关的社区，并钦佩品牌。也就是说，他讨厌 sitepoint.com 提供弹出式菜单，并在他的帖子中经常提到这一点。尽管如此，他在论坛上强烈支持 SitePoint 产品，几乎每本书一出版他就买。

Raj 在访问 sitepoint.com 时问自己的问题包括:

*   有哪些我可以帮忙的最新未回答的问题？
*   今天有哪些讨论话题比较热门？
*   最近有没有关于设计的教程出版？

马可和他的父母以及烦人的妹妹切丽住在悉尼。

他喜欢用 WordPress 建立网站，并且非常擅长编写模板和调整 PHP 代码，但是对好的设计缺乏真正的欣赏。他拥有一个盗版的 Photoshop，但不确定如何真正使用它。

Marco 喜欢在论坛上讨论最新的 web 开发技术和行业新闻(以及许多与 web 开发完全无关的话题)。不幸的是，他在论坛上写的很多东西都是无知的，而且是不必要的负面的。他与其他论坛成员建立了友谊，但同时他尖刻的评论和不成熟的行为也惹恼了其他论坛成员。

Marco 还没有真正在网上买过东西，因为他太小了，还没有信用卡。他可能会考虑让父母给他买一件东西，但迄今为止，他只是使用点对点的种子共享网络下载书籍和软件。

Marco 在访问 sitepoint.com 时问自己的问题包括:

*   我能在哪里找到一些免费的东西？
*   我收到过哪些私信？
*   我的朋友都在看什么论坛的帖子，评论什么？
*   SitePoint 有我可以联系的 Twitter 或脸书账户吗？
