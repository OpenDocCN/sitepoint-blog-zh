# IE6 的缓慢死亡

> 原文：<https://www.sitepoint.com/the-death-of-ie6/>

Internet Explorer 6.0 于 2001 年 8 月 27 日发布。2003 年，IE 的使用率达到顶峰，占市场的 95%；这一数字不太可能被任何浏览器超越。微软成功地创造了自己的浏览器垄断。

这种情况引起了一些负面报道。这导致了著名的美国司法部对微软的诉讼，微软被指控滥用职权。然而，无论你对微软的商业实践有什么看法，IE6 的成功很大程度上是因为:

*   它是免费的。与网景和 Opera 生产的早期软件不同，IE 没有花一分钱。
*   它与 Windows 捆绑在一起。不管用户是否想要，他们都会收到浏览器。
*   它与操作系统紧密集成。用户不能卸载 IE，开发者可以在他们的桌面应用中使用网络渲染引擎和通信库。
*   竞争很少。尽管 Mozilla 和 Netscape 发布了相互竞争的浏览器，但早期版本臃肿且漏洞百出。
*   IE6 速度快、稳定，是目前最好的浏览器。

这种情况也有利于 web 开发人员，因为几乎不需要在多种浏览器中开发或测试代码。当 IE6 *是标准的时候，谁会关心 W3C 标准呢？微软宣布 IE6 将是他们生产的最后一款独立浏览器。*

2004 年底 Mozilla Firefox 的发布改变了游戏规则。浏览器远远优于 IE，而且是开源的；Mozilla 几乎没有商业压力，也不会被微软收购或关闭。用户蜂拥而至，最近，苹果和谷歌被鼓励进入这个市场。IE 的统治结束了。

微软被迫采取行动，于 2006 年 10 月发布了 IE7。IE8 很快就要发布了，网络开发者祈祷它会成为 IE6 棺材上的最后一颗钉子。支持 IE6 是一件非常头疼的事情，但是在 IE6 首次发布八年后，仍有五分之一的人在继续使用这款浏览器(与 Firefox 用户的比例相当)。

IE6 的死亡极其缓慢，原因有几个:

1.  IE7 和 IE8 只能安装在 Windows XP SP2+或 Vista 上。许多人继续使用旧版本的 Windows 或避免自动更新。
2.  许多大公司的传统应用程序只支持 IE6。升级这些系统会产生巨大的成本，这可能是不合理的，尤其是在当前的经济环境下。因此，他们的员工无法切换到其他浏览器。
3.  许多 IT 新手不知道什么是浏览器，如何升级，或者为什么要升级。
4.  一些用户更喜欢 IE6，而不是 IE7 和与之竞争的浏览器。

IE7 的采用可能已经达到饱和点。尽管 IE7 用户有可能升级到 IE8，但它的发布可能不会对现有的 IE6 用户产生重大影响。虽然有几个高调的互联网运动来根除 IE6，但它的消亡正以令人沮丧的缓慢速度进行。

随着遗留系统得到修复，人们迁移到更新版本的 Windows(或 Mac/Linux)，浏览器将自然消亡，但庆祝 IE6 的最后一口气可能还为时过早。不幸的是，在未来一段时间内，很少有 web 开发人员会放弃对浏览器的支持。
