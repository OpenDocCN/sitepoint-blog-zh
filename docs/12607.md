# Opera:只有 4.13%的 Web 代码是有效的

> 原文:[https://www . site point . com/opera-just-413-of-webs-code-is-valid/](https://www.sitepoint.com/opera-just-413-of-webs-code-is-valid/)

Opera 的一项新研究发现，绝大多数网站没有充分支持网络标准。好消息是，与以前的研究相比，今天的有效网站比前几年多。坏消息是，在超过 350 万个网页的研究样本中，只有 4.13%的网址通过了 W3C 验证器。

结果来自 Opera 的“元数据分析和挖掘应用程序”(MAMA)，这是一个搜索引擎，“索引标记，风格，脚本和创建网页时使用的技术。”Opera 工程师和数据挖掘者可以问搜索引擎一些问题，比如“有多少网站使用 CSS？”(答案是 80.4%)或者“一般的网站有多少标记错误？”(是 47)。

Opera 对于他们将用这个工具做什么含糊其辞，除了继续分析数据和发布额外的发现，但是[一份新闻稿](http://www.opera.com/pressreleases/en/2008/10/15/)暗示了向网络开发者公开它的可能性。

Opera 写道:“MAMA 将帮助 Web 开发人员找到特性和功能的使用示例，了解趋势并收集数据，以向客户或经理证明技术的合理性。”“这也将鼓励标准机构考虑开发者关于现实中网络上发生的事情的建议，并将最终提高规范、网络和浏览器的质量和互操作性。"

Ars Technica 今天[发布了一份对调查结果的分析](http://arstechnica.com/news.ars/post/20081015-opera-study-only-4-13-of-the-web-is-standards-compliant.html)，它似乎也认为 Opera 报道的搜索工具将会公之于众。

今天发布的 MAMA 研究结果还显示，大约 50%标榜“W3C 有效”的页面实际上是无效的。Opera 推测，原因是对许多开发人员来说，保持页面的有效性并不容易，对许多人来说，跟上不断发展的标准也很困难。Opera 说，关键的一点是，“人们不擅长 HTML 这个东西。”不适当的标签嵌套非常普遍，错误拼写或放错位置的元素和属性名称时有发生。很容易犯愚蠢的、偶然的错误——我们都会犯。"

然而，一个问题是，人们用来创建网页的工具也会产生难看的、无效的代码。MAMA 还研究了与网页编辑器和内容管理系统相关的验证，发现除了苹果的 iWeb——令人印象深刻的 81.91%的 URL 通过验证——结果普遍令人沮丧。根据 MAMA 的数据，用微软 Frontpage 制作的网页只有 0.55%是有效的，用 Adobe Dreamweaver 制作的网页只有 3.44%。

对于内容管理系统，结果也好不到哪里去。WordPress 页面只有 9.0%的时间是有效的，对于打字错误只有 12.74%，对于 Joomla 只有 6.45%。谷歌的博客表现最差，只有微不足道的 0.30%的网址通过验证。

在[完整结果](http://dev.opera.com/articles/view/mama/)中有许多有趣的信息可以挖掘，Opera 承诺将发布更多结果，作为计划中的“漫长、多部分传奇”的一部分
