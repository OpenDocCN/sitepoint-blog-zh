# 开源许可细分

> 原文：<https://www.sitepoint.com/open-source-licensing-breakdown/>

自从开源出现并改变了我们对技术解决方案的思考方式以来，关于开源的讨论现在围绕着商业和法律问题，而不是技术问题。

最近的重大事件涉及知识产权、专利、版权和源代码管理。该列表的另一个重要主题是许可管理。

无论是为您自己的开发项目选择合适的开源许可证，还是为您的企业选择核心应用程序解决方案，理解开源许可证的含义及其影响都是至关重要的。当考虑包含开源、扩展开发和销售的 web 应用程序时，这变得更加重要。

普伦蒂斯霍尔的一本新书，作者是律师[劳伦斯·罗森](http://www.rosenlaw.com/html/lawrenceerosen.html)(他自己也是几个开源许可的作者)[开源许可](http://www.phptr.com/title/0131487876#)，应该是你开始成为一个开源许可大师所需要的一切。

罗森是一名律师，最近被选为黑鸭软件公司的顾问，他不仅仅是对每一个许可证进行大量的解释。相反，他首先介绍了知识产权法，然后剖析了开源许可模式以及该法律如何适用于它。

正是这种通过法学院 101 的介绍性旅行将开源许可证带入了生活——展示了他们如何处理版权、合同协议、用户权利(也就是开发者权利)、再分发等等。

也许对开发者来说最重要的是，这一章专门讨论了选择正确的许可场景、通用商业模式、从开源软件中赚钱等等。

涵盖的许可包括 Apache、BSD 和 GPL。LGPL，麻省理工学院，Mozilla 和其他几个。Rosen 还讨论了开源(即共享源代码)的替代方案，并对现在臭名昭著的 SCO 诉讼提供了一些评论。

他认为 SCO 的行为表明了企业界对 Linux 和开源的重视程度——重视到足以起诉并花费数百万美元打官司。罗森也相信它会过去，Linux 会生存下来。

我敢肯定，在 Linux 和开源软件的发展过程中，我们已经看到了成长的烦恼。
