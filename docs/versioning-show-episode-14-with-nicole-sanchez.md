## 副本

David:

嘿，大家好。这是大卫·格林先生…

Tim:

…这位是蒂姆·埃夫科…

David:

…您正在收听的是版本控制播客的第 14 集。

Tim:

这是一个我们聚在一起讨论 web 行业的地方，从开发到设计，一些人今天正在实现它，并计划它在下一个版本中的发展方向。

David:

今天，我们采访了 GitHub 的社会影响副总裁 Nicole Sanchez。我们为什么不开始这个版本呢？

妮可，你今天好吗？

Nicole:

我很好。谢谢你邀请我。

David:

因为这是版本控制秀，我们总是问我们的嘉宾一个哲学问题，你今天的哲学问题是:在你目前的职业生涯中，你是什么版本，为什么？

Nicole:

我喜欢这个问题。好吧，前几天我和我的一个戏剧极客朋友有过类似的对话，我告诉她我正处于人生的第三幕。她变得很沮丧，因为在她的世界里，只有三幕，但我没有想到这一点。在我的脑海中，有 5 个或 7 个或 20 个，我不知道，但她最终同意让我称自己为 3.0 版，因为我有童年，那是一个，有许多不同的迭代，然后在某个时候作为一个成年人和一个妈妈，那变成了两个，有许多不同的迭代。

因为我的孩子们现在都十几岁了，我能够——老实说——比过去 16 年更多地考虑工作。就像我说的，我有一个 16 岁的孩子和一个 13 岁的孩子。对我来说，这绝对是版本 3 的开始，因为我能够更快地迭代，就像我认为我以更有规律的速度发布更新一样。我对自己在 3.0 的稳定性比以前更有信心了。

这是我在版本 1 中进行创造性工作的地方，也是所有这些迭代的地方，所有这些迭代真正地将它整合到版本 3 中，但这并不意味着我在版本 3 结束时就完成了。可能有 25 个，我不知道。我们走着瞧。

*【笑声】*

Tim:

绝对不是，但我总是对投资某个东西的第三版更有信心。

Nicole:

好吧，这就对了，对吧？没错。然后，谁还没有准备好成为早期采用者，你可以等到妮可第四版，并投资于我。

David:

很期待。

Nicole:

谢谢大家！

David:

我对你的标题很感兴趣。并不是每个公司都有这样的项目，但是有一些公司已经实施了像你们这样的项目。

Nicole:

是啊。社会影响不仅仅意味着成为多元化和包容性的领导者。它涵盖了多样性和包容性，还包括我们的社区合作伙伴关系。因此，这意味着在技术社区内部，以及在我们的当地社区中，与技术没有任何必然联系的事情。第三件事是真正尝试与我们的工程团队和 GitHub 的其他工作人员合作，确保人们可以来到 GitHub 并利用他们的知识做好事——产生积极的社会影响——无论是建立一些解决实际社会问题的酷东西，还是为其他人正在建立的东西做出贡献。我们真的希望开始让人们更容易利用开源带来积极的影响。这就是社会影响团队所涵盖的内容。

David:

那很有趣。所以你的角色，既包括内部候选人和在公司内部工作的人，也包括使用你的服务的人。

Nicole:

没错。我们开始着手的主要工作之一是我们员工的多样性，我要说的是，大约一年前，我们非常清楚这一点非常糟糕，甚至没有达到行业标准。我们已经达到了行业标准，但从人口统计学的角度来看，整个行业有一些糟糕的统计数据。现在，我们的工作是超越它。

此外，我们正在努力实现开源用户和贡献者社区的多元化，因为如果没有这两方面的工作，我们就不会在开源、软件和技术领域获得我们真正需要的创新水平。所以，我真的很喜欢在这方面工作，这就是我们在 GitHub 正在做的事情。

David [4:24]:

当你说多样性的标准时，我很好奇。很多人可能对此没有清晰的理解，或者他们可能在脑海中有一个图像，但没有真正从你带来的角度来看待它。我很好奇你是否能分享一下你的意思。

Nicole:

当然可以。我认为人们总是关注的两件事是种族和性别。当你考虑劳动力多样化时，这些都是非常重要的人口统计因素，因为不幸的是，我们仍然处于这两个因素可以预测你的职业生涯、成功、学术成就、总收入、寿命的时代——不仅在美国，在全球范围内都是如此。这两件事对我们来说非常重要。

除此之外，我们还关心性别认同等超越二元的东西。我们非常关心性取向、宗教、年龄、体能、退伍身份，这样的例子不胜枚举。我们主要关心的是那些目前在技术领域代表性不足的人，但他们的人口背景过度预测了他们在技术、高等教育等领域的职业成功或失败。

这些都是我们没有人真正要求生来就有的东西，但由于偏见，由于偏见，由于种族主义，由于厌女症，这些东西对你在生活中的表现有着不成比例的影响。我们积极寻找来自代表性不足的背景的人来为 GitHub 工作和行动，因为这将使我们能够为更广泛的用户带来更高水平的创新和更好的平台。这就是我们正在谈论的。这回答了你的问题吗？

David:

是啊。那是有帮助的。我想你自己也认为自己的背景在某些方面被低估了。

Nicole:

完全正确。我是一个 40 多岁的墨西哥裔美国女人。我是一个母亲。有很多方法可以把它分割开来，我在职业生涯中发现，我的第一份多元化和科技工作是在 1999 年。对于从不太可能、代表性不足的背景中引进人才对该行业意味着什么，人们的误解程度令人担忧。我们现在认为这是令人担忧的，但是让我告诉你在 1999 年，我听到的那种事情！

我一直说的，今天仍然一直说的是，有这么多的天才，我们只是从来没有利用技术。有如此多的创造力。有如此多的才能。有如此多的渴望和好奇。由于获取技术或接受高等教育的人为障碍，我们失去了所有这些人才。我们失去了一代又一代的人才。

我写了一篇文章放在 Medium 上，讲的是我父亲和他在 60 年代初试图去加州大学伯克利分校，但由于其他人一些非常可怕但带有种族主义色彩的行为而被迫退学。我的父亲有一个聪明的数学头脑，他的同学们继续开发编程语言和我们今天继续使用的硬件。一些与他有多聪明无关的事情导致他从轨道上掉下来或者被推下来。

我讲述他的故事不是因为他的故事是独一无二的，而是因为它无处不在，每天都在发生，因为我们已经失去了成千上万——此时此刻，数百万——本可以为非常重要的科学发现做出贡献的人，因为我们没有在他们身上投资，因为我们让自己的偏见妨碍了他们，他们没有机会接触。这是我最关心的。我个人认为科技领域的代表人数不足。

Tim [8:12]:

谈到 1999 年，我不认为你会在技术对话中经常听到这个问题，但谈到 1999 年，你说那是你第一次开始接触技术多元化的时候。你是如何得到这个角色的？

Nicole:

1994 年，我从大学毕业，我的第一份工作是社区服务的多元化，特别是在波士顿。我加入了第一批美国军团项目中的一个项目，这是美国军团第一次真正发展成为全国性项目。

我花了将近五年的时间建立多元化的团队，在社区里做好事——在小学里，在医院里，建造操场，做非常有趣的事情——我们日复一日地证明，多元化的团队可以一起创造伟大的事情。这更好，因为我们是多样化的，不是尽管我们多样化，而是因为我们多样化。

我在波士顿遇到了一位企业家，他说:<q>我想让你在我的初创公司里做这件事。</q>现在，我不会说出这家初创公司的名字，我也永远不会透露，因为最终发生的事情是，我怀孕了，生了孩子，并因为生了孩子而被解雇，有人告诉我，我现在的首要任务应该是我的孩子，我的孩子，他们需要可以睡在办公桌下的睡袋里的人，而我还没有准备好迎接创业生活的挑战。

然而，在此之前，我们取得了一些惊人的成功。我们有一个主要由美国土著人组成的工程团队，这是我第一次看到群组模型的力量，如果你从一个代表性不足的背景中雇用了一个真正有才华的人，然后他们说，<q>我知道另外三个我非常亲近的人会很适合这个团队，</q>因为我们会在安全的团体中旅行，并且在团体中，这个美国土著人带了三个也是美国土著人的朋友，他们形成了我们工程团队的核心。他们期待着彼此的成功。

在那个时候，群体模型作为一种多元化招聘方式的力量是显而易见的，所以我们看到了一些非常有趣的事情。我们还确保我们的员工男女各半。当然，这是在我们开始真正公开谈论挑战性别二元制之前，但那时我们大约一半是男人，一半是女人。我们看到了一些很酷的事情发生。现在，当资金压力和来自投资者的压力来临时，它是第一批被扔掉的东西之一——这让我非常沮丧。我非常沮丧，只是说，<q>这不是一个真正关心这个问题的行业，</q>我离开技术行业有一段时间了。我就是这样进来的，第一轮就带着很不好的味道离开了。

Tim:

不，我绝对能理解。我们没有播放视频，但我听到这个故事时脸上有一种非常惊恐的表情。

Nicole:

你做到了。你脸上有一种非常恐惧的表情！你脸上有一种适当的惊恐表情。*【笑】*

Tim:

说到钱，我们经常在社区中听到——我先假定其他人是无辜的——有人可能怀疑多元化努力确实需要钱。你想谈谈你对此的感受吗？

Nicole [11:32]:

当然可以。我对此有强烈的感觉，我相信你能理解。多元化努力需要资金。确实如此。首先，它们很贵，我把它比作飞轮。一旦你让飞轮转了几圈，它就开始自己运转了。启动它是非常…就像任何创业一样。那是一段极其昂贵的时间。

现在，在回报方面，我们在过去的两年里得到了人们愿意接受的数据，我认为，总的来说，这些数据来自麦肯锡。它基本上是这样说的(这是跨部门的):如果你的公司主要由白人男性经营，你加入了白人女性，你的收入就会超过你的竞争对手 10%。如果你在此基础上增加种族和民族的多样性——我们说的是在所有职能领域的所有层面——你会增加 35%。

因此，尽管启动和运行的成本可能很高，但如果做得好，你很快就会获得回报。问题是，如果你做得很好，那么很多人会失败。一方面在前端，绝对要花钱。在后端，你会收获回报。中间是艰苦的工作，有很多人认为，<q>哦，好吧，所以我必须去找黑人和棕色人种，把他们带到我的公司，神奇的是，他们帮我扭亏为盈，</q>但事实完全不是这样。根本不是这么回事。

我认为这是公司失败的地方。这些天来，甚至没有太多的愿望去做这件事。好吧，我们甚至不知道从哪里开始。这是我见过最多的。

David:

了解从哪里开始——在某种程度上，我认为这违背了人们在高科技领域看到的精英管理的概念。我相信你可能对精英管理的概念有一些看法。

Nicole:

我知道。好吧，对于那些不知道精英制度起源的人来说，它实际上最初是作为一个笑话来写的。你可以决定谁是有价值的，或者不管这个词是什么，谁在客观上比其他人更值得。这不是一个植根于任何社会科学的概念——你无法证明任何东西。

科技和其他行业的人去做他们认为是客观的测量。<q>你的 <abbr title="grade point average">GPA</abbr> 是多少？你去了什么学校？你上的是什么大学？</q> —因为我将利用社会告诉我的所有有偏见的重要信息，并决定赋予这些信息更多的意义。

问题是在每一个系统中，都有相同的偏见。你只是把你的偏见放在别人的偏见之上。如果你说，<q>哦，你去了斯坦福大学，你在计算机科学上得了 4.0 分，</q>没有分析来自各种背景的女性和有色人种是如何被挤出计算机科学经历的，当然，你会有一群非常同质的人能够在斯坦福大学获得计算机科学 4.0 分，这与客观上人们有多聪明无关。

如果你说这是优点，我把它放在数字上，或者我说，<q>那个人比任何人都更快地完成了编码挑战，</q>你给看似客观的东西赋予了意义，但它们不是。这些都不是客观的。所以我会说——从我作为一名招聘经理的角度来看——<q>给我看一个计算机科学学生，他得了 3.0 分，做了两份工作，参与了校园活动，是他们家第一个上大学并坚持下来的人。</q>我会聘用那个在大学经历中每次都没有摩擦的完美 4.0 以上的人。

我喜欢 GitHub 的一点是，我们对任何工作都没有更高的教育要求。这对我们真的很重要——我看到你们都在谈论开源——这对我们来说是非常重要的，因为很多人不是通过学校学习开源，而是因为一个朋友向他们展示了开源，然后他们涉猎了一下，然后他们对此非常兴奋，然后他们在 GitHub 或其他开源项目上花了很多时间。这些都与学校无关。

我们这里的工程师实际上甚至没有从高中毕业，他们是我们的一些顶级工程师，因为这与他们有多聪明无关。在某些情况下，是社会经济因素。在某些情况下，这是因为学习差异。除了你上过大学，大学本身并不能证明什么。当人们说，<q>你有斯坦福大学的学位，妮可，</q>我说，<q>我有。</q>你所知道的是，我在斯坦福上了四年学，为了拿到那张纸，我做了他们让我做的所有事情。就是这样。你对我的斯坦福学位就知道这么多。没有什么优点可以转移到我在科技领域的成功上。

事实上，一些公司已经发现，随着时间的推移，当他们进行模式识别时，回报是递减的，他们试图重复说，<q>哦，在这个学校上学的这个专业的人，他们取得的成绩在我们公司一定很好。你把它作为一种模式使用的时间越长，它就不再是真实的。</q>

Tim [16:48]:

Nicole:
