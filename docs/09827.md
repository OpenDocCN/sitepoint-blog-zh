# 小屏幕第一:为 320 及以上尺寸打造

## 首先是 320 及以上的小屏幕

如果你还没有学会如何处理 [CSS3 媒体查询，](https://www.w3.org/TR/css3-mediaqueries/)我希望这是你的首要任务，因为 2011 年绝对是移动设备年。无论你是需要建立一个独立的移动网站，还是计划制作一个适合所有人的网站，你都需要及时更新网页设计技术。

开发人员在使用媒体查询方法时面临的一个常见问题是，您经常必须做大量的工作来让小屏幕浏览器满意；你会发现自己在调整大图片的大小，切换脚本，甚至隐藏大块的内容。虽然这项技术完成了工作，但它意味着小屏幕浏览器可能会加载大量他们永远不需要显示的内容。对于又慢又贵的移动连接，这意味着要为额外的数据包付费，这是我个人倾向于避免的。

听起来很熟悉？如果你关注 SitePoint 播客，应该会的。在上周的节目中，路易斯和杰里米·基思谈论了这个问题。你会发现这场讨论开始于 23:45 左右:

> Louis:……我想响应式网站设计的最大挑战之一是应对带宽挑战，对，如果我需要提供一个在 1024 或 1280 分辨率下看起来不错的图像，它还需要在 320 分辨率下看起来不错，到目前为止，大多数响应式网站设计的方法是使用一个非常大的图像，并在 CSS 中缩放它，但这确实给带宽带来了问题。然后出现的另一个问题是一些多余的内容，所以你已经谈了一点关于惰性加载的解决方案。因此，我想听听您对我们在应对这些技术挑战和处理带宽问题方面的发展方向有何看法。
> 
> Jeremy:谈到提供不同大小的媒体，比如图像，我认为这是关于布局的想法，而不是缩小以桌面为中心的布局以流入移动网站，如果我们颠倒过来，从窄版本开始，然后扩大到桌面，会怎么样？我认为这可能也是一个与媒体打交道的好方法；不要将大图像视为默认图像，并(问)我们如何缩小它以适应较小的屏幕尺寸，相反，将较小的图像视为默认图像，然后我们如何为具有较大屏幕的设备(笔记本电脑、台式机等)提供较大的图像。
> 
> 我的意思是，部分原因是因为更小的屏幕设备将包括一大堆功能不全的手机。我们现在已经有了一些很棒的移动浏览器，比如 Mobile Safari 等等，但是也有很多手机的浏览器有点糟糕。因此，安全的默认做法是考虑更低的带宽，考虑更小的图像，考虑线性化设计，然后为台式机或笔记本电脑上更强大的浏览器应用更大的图像、更多的内容和多列布局。

或者，正如卢克·乌鲁布莱夫斯基在 2009 年所说的那样:[首先为移动设备设计。](http://www.lukew.com/ff/entry.asp?933)你的基本设计是一种简单明了、没有花哨的体验，即使是最笨的手机和最小的屏幕也可以使用。对于支持媒体查询的更现代的手机和平板电脑，我们可以采用更多的样式，创建新的布局来适应它们的尺寸，或者利用更酷的 CSS 效果。最后，对于桌面浏览器，创建布局和使用资产，最大限度地利用屏幕空间。当然，有些桌面浏览器只是*有*要与众不同；对于这些浏览器，我们可以使用 polyfills 来包含这种支持，或者找到其他解决方法——或者如果您觉得不太好的话，可以让它们使用最简单的设计！这太简单了，以至于你想知道为什么我们没有从一开始就这么做。

然而，在你使用你的文本编辑器之前，考虑一下安迪·克拉克的新 [320 和 up](http://stuffandnonsense.co.uk/projects/320andup/) 模板。使用优秀的 HTML 5 样板项目作为基础，他添加了大量的 polyfills、helpers 和 bug 解决方案，例如 [Selectivizr](http://selectivizr.com/) 添加 CSS3 选择器， [Respond.js](https://github.com/scottjehl/Respond) 在 Internet Explorers 6 到 8 中支持媒体查询。样式表已经为五种不同的设备宽度准备好了——小屏幕、480 像素、768 像素、992 像素和 1382 像素——另外还有一个用于 iPhone 4 和最新 iPod Touch 的 Retina 显示屏。您可以为每个设计变体选择单独的样式表，也可以选择包含所有内容的单个样式表。

你还能做什么？当然，响应式设计不仅仅是挤压或扩大你的标志。有些界面可能需要您以特定大小显示或隐藏某些内容，而其他界面可能包含仅适用于某些大小的全部功能。手机的菜单可能需要与台式电脑的菜单有很大的不同，尤其是考虑到我们更多的人使用指尖导航。(是的，还有*款*仍然是非触摸智能手机。)你可能需要围绕[上下文感知图像大小做更多的工作](http://filamentgroup.com/lab/responsive_images_experimenting_with_context_aware_image_sizing/)你可能需要考虑你网站上的某些信息是否应该为移动用户强调。但是，无论您的需求是什么，您都应该发现 320 和更高的模板是构建下一个响应式布局项目的可靠、可靠和经过测试的起点。
