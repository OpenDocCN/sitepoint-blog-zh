# EC2 已出测试版:现在提供 Windows 支持及更多

> 原文：<https://www.sitepoint.com/ec2-out-of-beta-now-with-windows-support-more/>

![](img/c7d6c2dd6ca0b06a82a63b4155ba800d.png)

亚马逊今天宣布 [EC2 正在放弃“beta”状态](http://aws.typepad.com/aws/2008/10/big-day-for-ec2.html)，现在已经可以生产了。最重要的是，他们现在承诺实现 [99.95%的正常运行时间 SLA](http://aws.amazon.com/ec2-sla/) 。如果您还没有考虑将 EC2 作为传统托管/专用服务器托管的合法替代方案，现在是这样做的好时机。

在 SitePoint，我们设计和构建我们的应用程序，期望它们能够在早期出现故障并且经常出现故障的硬件上运行。我们对 EC2 的实际经验是，实例失败是一种相对罕见的情况。EC2 的美妙之处在于，从故障中恢复的成本(时间和金钱)远远低于我们使用传统(好吧，我姑且称之为“传统”)数据中心模式的成本。

今天还宣布了 EC2 对 Microsoft Windows Server 的测试版支持。您可以选择运行 32 位或 64 位安装，也可以选择运行 Microsoft SQL Server。启动 Windows 实例后，您可以使用 Windows 远程桌面客户端连接到它，一切就绪。捆绑 Windows 实例目前的工作方式略有不同，实际上是关闭实例，而不是在捆绑过程中保持在线的 Linux 实例。

接下来的几个月对于 Windows Server 的用户来说将是非常有趣的，因为它在云环境中运行和表现良好的能力变得更加为人所知，并且 Windows Cloud OS 的更多细节也将浮出水面。

然而，亚马逊的声明不止于此。将于 2009 年发布的是管理控制台、负载平衡、自动扩展和云监控系统。我预计这些特性将大大降低 EC2 新用户的准入门槛，因为管理云有时可能是一项艰巨的任务。

在 SitePoint，我们开发了自己的负载平衡器、集群管理和监控工具，这些工具让我们对 EC2 环境有了很好的了解，但它也是奇怪的头痛和战争故事的来源(边喝啤酒边问我)。开发您自己的管理系统会给您带来极大的灵活性和控制力，但这也意味着您需要重新开发它们，以便利用新的 EC2 特性(比如新的实例大小，或者像[弹性块存储](http://aws.amazon.com/ebs/)这样的新特性)。

凭借 SLA 和支持业务关键系统的新承诺，除了提供更广泛的功能和操作系统，亚马逊再次巩固了其在云计算领域的领导者地位。
