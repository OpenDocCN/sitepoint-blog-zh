# SVG 对网络重要的 9 个原因

> 原文：<https://www.sitepoint.com/9-reasons-why-svgs-are-important-for-the-web/>

尽管自 2001 年以来一直是 W3C 规范，但可缩放矢量图形从未得到应有的重视。这主要是因为大多数网络用户(使用 Internet Explorer)在没有插件的情况下无法查看图像。

[SVG](https://www.sitepoint.com/svg-101-what-is-svg/) 是一种基于 XML 的文件格式，它用颜色、渐变、图案、不透明度等来描述图形矢量，如线条、路径、方框、圆形、多边形和文本。必要时，位图也可以嵌入到 SVG 中。

SVG 非常适合 web 平台的原因有很多:

**1。W3C 标准**
SVG 是一个开放标准，开发者可以无限制地使用它。

**2。图像缩放**
SVG 可以缩放到任何大小，而不会导致像素化或细节丢失。

**3。较小的文件大小**
简单的图像，如徽标和图表，通常比它们的位图 JPG、PNG 或 GIF 文件小。

**4。SVG 工具已经可用**
SVG XML 代码可以使用从记事本到 [Inkscape](http://www.inkscape.org/) 、[OpenOffice.org Draw](http://www.openoffice.org/)和 [Microsoft Visio](http://office.microsoft.com/visio) 的各种现有工具来创建、验证、操作和压缩。

**5。服务器端生成**
SVG XML 可以使用 PHP 在服务器上创建和操作。NET、Python 或任何其他语言/框架。

**6。客户端生成**
SVG XML 可以在客户端使用 JavaScript 创建和操作，以创建动态效果和动画。事件处理程序，如 click 或 mouseover，可以应用于任何 SVG 元素。

**7。兼容性**
尽管 SVG 渲染引擎提供的功能可能不同，但这种格式是向前和向后兼容的。SVG 引擎会渲染它们能渲染的，忽略其余的。

**8。无障碍**
SVG 无障碍；文本和绘图元素是机器可读的，因此屏幕阅读器可以让其他设备解析图像。

**9。搜索引擎优化**
SVGs 提供改进的 <acronym title="Search Engine Optimization">SEO</acronym> 因为 Google、Yahoo、Bing 和其他搜索引擎可以索引图片的内容。

在某种程度上，canvas 抢了 SVG 的风头，因为它可以在 IE 中得到支持，而无需使用 VML 和 JavaScript 的插件。但是，canvas 不允许静态渲染(图像必须由代码生成)，并且绘制的元素在 <acronym title="Document Object Model">DOM</acronym> 中是不可识别的。

IE 是唯一不支持 SVG 的主流浏览器。如果微软在 IE9 中实现 SVG，它将为 web 开发者提供许多令人兴奋的机会，其使用将呈指数级增长。

不幸的是，即使 IE9 支持 SVG，这款浏览器也要到 2010 年末才能上市。微软不太可能在以前版本的浏览器中实现它，所以在 IE9 获得显著的市场份额之前会有相当大的延迟。但是让我们感谢微软正在考虑 SVG 我们已经等了很久了。
