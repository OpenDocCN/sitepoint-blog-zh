# MySQL 可视化数据库迁移工具

> 原文:[https://www . site point . com/MySQL-visual-database-migration-tool/](https://www.sitepoint.com/mysql-visual-database-migration-tool/)

没有比这更好的时机了。我现在正与一家公司合作一个项目，该公司选择将 30 多个不同大小和复杂程度的微软 Access 数据库迁移到 MySQL 4。

为了我自己和我们所有人的好运——MySQL AB 今天宣布免费提供 [MySQL 迁移套件](http://www.mysql.com/products/migration-suite/)。这个 GUI 工具集将允许从 Oracle 和 Access 进行全面迁移(随后是 Microsoft SQL Server、IBM DB 2 和其他工具),并在 GPL 2 许可下发布。

更好的是，它是用 Java 编写的，这意味着可以执行跨平台的数据库迁移(Unix 到 Windows、Solaris 到 Linux 等等)。该套件利用久经考验的方法和向导指南，涵盖了迁移的所有基础知识以及存储过程和触发器。

凭借其便捷的报告功能，可以在加速迁移时对迁移执行审计和尽职调查，以确保没有遗漏的环节被忽略。

我将在我的开发环境中进行一些测试迁移，因为至少可以说，将任何全新的东西投入生产是很有趣的。然而，这看起来节省了大量的时间和潜在的麻烦。

打破我不提及我使用的产品的习惯–通常我在迁移过程中使用 Navicat，因为它也跨越多个操作系统(特别是 Linux、Mac 和 Windows)。它出色地完成了它的工作——然而 MySQL 迁移套件是 GPL 开源的——我绝对要测试一下它的性能。

Navicat 7 即将发布，并增加了我认为第 6 版中缺少的几个功能，这可能会阻止我采用 MySQL 套件。但是，这不会阻止我在当前的 Access 项目上对它进行一次很好的基准测试！

对于那些也以商业身份使用 MySQL 的人(为 MySQL 服务或许可付费)，该公司还为他们自己的咨询服务提供相当有吸引力的 1500 美元折扣，以获得为期 5 天的 MySQL 5 部署。
