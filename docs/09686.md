**John:** So I wrote something in 2007, so quite a few years ago now. However, that doesn’t mean you can’t build your application using web technologies in conjunction with a solution like PhoneGap or Appcelerator and there are a number of others, and then deploy those apps built with web technologies as native applications through app stores directly onto devices. 路易:对。我想快速地说一下，假设我们承认，没有混合应用这样的东西，你要么有本地应用，要么有 web 应用，本地应用可以使用本地 SDK 或 Web 技术构建，然后打包，然后通过应用商店销售。鉴于此，你是否仍然认为，除了某些特殊情况，web 应用通常是可行的，或者当你说没有混合应用或混合应用是 BS 时，你是说这不算数，我们应该做真正的 web 应用，还是说这是另一种本地应用，本地应用在某些情况下是适用的？

Well, I think in theory and in the longer term I think people should invest a lot of their energy and thinking into Web technology, Web based apps, apps that run in the browser. I think there are tremendous advantages there that are over and above what are the disadvantages with a Web based app, most of the ones people cite such as performance, such as kind of the ability to run offline or to be installed directly into the homescreen of a device, they actually don’t apply; I mean most of the time people have a series of items about why web apps are not as good as native apps, and on the whole except to some extent edge cases around performance, basically that’s simply not true, the advantages that native apps apparently have, supposed to have, don’t exist. Probably the single great advantage most people think about, the advantage of a native app is that it is in an app store, that people can find it there and install it there very easily. I think that that’s a very overrated argument, I don’t think there is a lot of evidence for that, the overhead in someone locating and installing an application for an app store is pretty high, and I think a lot of developers essentially abdicate their responsibility to market their own applications by simply saying, well, I’ve got it in the app store, people will find it, they’ll install it, I’ll make a heap of money, right, which for most developers isn’t how it plays out. So I think in the longer term people should really be thinking about the browser as the platform to develop for, for a heap of reasons. But I think probably the prototype of the future is something like Chrome OS already shipping in various netbooks out at Google, basically the idea is you boot into a browser, the browser is the way the user interfaces with their device, and I think that’s a model for the future of computing. I honestly don’t think people, the average user, is really going to give a damn about what the operating system of the use is in the relatively short term; you know five years ago people used to care whether it was an Intel Celeron or whatever, people don’t care about the hardware anymore, they just do not care, it’s just no longer relevant, right. Most people don’t even know the chips that are in there let alone the GPU’s and the clock speeds of their Android device or whatever, people don’t care anymore. So I think that is also going to be true of operating system level in the relatively short term, and I think the Web is really just — Web technologies are going to be the way in which developers develop applications and the way in which users interact with all sorts of devices, tablets, phones, laptops, desktops, but everywhere there’s going to be a screen there’s going to be a Web technology sitting there. So I think as a developer if in the short term yes there are times when definitely very sensible idea to package up as a native app and deploy onto devices as a native application you should still be really investing heavily in your skills, HTML, CSS and JavaScript because I think it’s abundantly clear that that’s where the future of computing is headed, and whatever your background as a developer, whatever your skill-set, it will be very marginalized in the next three or four years I think, except in very niche applications, because really the great majority of demand for development skills will be around web technologies.
