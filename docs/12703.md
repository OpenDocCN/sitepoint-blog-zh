# 打开雅虎！将与谷歌友好合作，而不是竞争

> 原文:[https://www . site point . com/open-Yahoo s-will-play-nice-with-Google-not-competite/](https://www.sitepoint.com/open-yahoos-will-play-nice-with-google-not-compete/)

今天有很多关于雅虎的报道。■开放战略，即雅虎描述为利用其 5 亿独立用户来鼓励网络创新的一种方式。“雅虎！”美国的策略似乎是，我们越是让人们从雅虎内部与网络的其他部分互动。他们就会越来越多地回到雅虎！。这似乎是一个合理的提议——毕竟，这对于谷歌来说是有效的，因为谷歌的理念是让人们尽可能快地找到外部内容。

正如我们上周所写的, [AOL 现在也在使用](https://www.sitepoint.com/report-aol-to-revamp-homepage/)的开放策略。公司开始理解网络的价值在于信息和数据的自由流动——越开放越好。

雅虎！昨天向记者展示了他们的一些新的开放平台，展示了运行在其 Yahoo！邮件服务。还有雅虎！今天在加州桑尼维尔的园区举办了一年一度的黑客日，开发者们在这里开发运行于其平台上的应用。雅虎！表示将在雅虎上推出更多的第三方应用。在接下来的几个月里，随着 BOSS 和 Search Monkey 等最新项目的推出，他们已经证明了他们将会对所有的来者更加开放。

一些人建议雅虎！谷歌的开放战略是试图挑战谷歌。CNET 的 Stephen Shankland 写道:“不能保证这一发布会改善雅虎的财务困境，这是该公司称之为雅虎开放战略的关键一步。”。"但这一战略有望帮助雅虎，而不必试图击败谷歌."

但是雅虎！不是也不应该试图挑战谷歌。开放策略实际上与谷歌配合得很好。雅虎！并没有试图像谷歌那样创建网络操作系统——雅虎！正试图为 web 操作系统创建*应用平台*(请记住，当“OS”应用于 Web 时，它的含义与传统意义上的计算机操作系统大相径庭)。Web OS 竞赛实际上是一场三匹马的竞赛。不在竞争者之列。但是雅虎！*是*在为应用提供商提供平台和工具以构建操作系统的次级竞赛中的先锋。谷歌也在这场竞赛中，但雅虎！将自己定位为直接插入网络操作系统——不管是谁创造的。他们真的应该关注其他平台提供商，比如脸书。

雅虎！平台是一个修复雅虎的计划！那是我一年前建议的。让开发者接触到大量受众和丰富的社交图信息的能力是平台成功的头号驱动力(工具的质量是第二大重要因素)。可能没有任何人比雅虎更适合在这些领域提供服务。

雅虎没有理由。在这一点上应该尝试与谷歌竞争——谷歌已经赢了。他们的搜索广告交易似乎承认了这一点。但是雅虎！应该可以在谷歌的阴影下做得很好，为运行在谷歌网络操作系统上的应用程序开发提供一个平台(或 Mozilla 的，或微软的，或任何最终胜出的平台)。

最后，雅虎！据报道，今晚的 hackday 上有 [Girl Talk 在玩，这给了他们足够的风格点数来获得一点优势。但是比赛才刚刚开始。](http://badtofu.com/isaid/hackday-with-girl-talk)
