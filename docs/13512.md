# 可用性双重打击！

> 原文:[https://www.sitepoint.com/usability-double-whammy/](https://www.sitepoint.com/usability-double-whammy/)

可用性。

我们认为我们知道这个词的意思，但许多人努力定义或描述它。当然，我们总是知道什么时候某个东西是可用的，因为它感觉起来是“直观的”。我们当然知道什么时候某样东西不可用，因为它让我们非常沮丧。

我们可能偶尔会读到一些关于可用性的话题，或者在日常工作中实际应用一些以用户为中心的设计技术或进行结构化可用性测试。就我个人而言，在参加了一次[会议](https://www.sitepoint.com/sitepoint-at-web-directions-south/)后，我发现自己对这种东西变得非常[兴奋](http://themaninblue.com/writing/perspective/2006/10/11/)，但几天后，日常事务的实际现实意味着它被搁置了。

那么，当我们建立自己的网站时，我们实际上使用了多少可用性理论，又有多少被我们自己的判断或者我们知道在过去有效的东西抛弃了呢？你需要一个有学术证书的专家才能应用这些东西吗？有时甚至连专家都不同意，或者理论与实践相去甚远，或者它迫使我们牺牲美学来在每个盒子里打勾。让一个网站变得可用需要做些什么？

[![The Usability Kit](../Images/592cffacb4ba05fb2a7d71cb0d8b1042.png)](https://www.sitepoint.com/books/usability1/) 紧随[世界可用性日](http://worldusabilityday.org/)之后，我非常自豪地宣布由 Gerry Gaffney 和 Daniel Szuc 设计的[可用性套件](/books/usability1/)的发布。该可用性套件包含真实世界的示例、蓝图和最佳实践技巧，引导您了解卡片分类、纸张原型和亲和力图表等设计技术，并向您展示如何为您的客户网站进行焦点小组、用户访谈和大规模可用性测试。(查看[目录](https://www.sitepoint.com/books/usability1/toc.php)中的完整分类)。

作为该工具包的技术编辑，我可以担保该工具包包含的大量实用建议——你将获得诀窍*和*来建立能够*运行*的网站的信心，并且你将*知道*它们能够运行，因为它们已经在网站生命周期的所有阶段被用户所塑造。

与 SitePoint 的所有工具包一样，这个工具包附带了一堆好东西，例如包含工具包中解释的所有原理图的 CD-ROM。它还包括一些我们称之为磁性网络部件的东西——磁性按钮、文本字段、复选框和其他表单字段，您可以使用它们在会议室的白板上或厨房的冰箱上展示您的下一个网站！它们绝对是你所拥有的最酷的一套磁铁，让磁性诗歌看起来很 2001。此外，我们对所有书籍和工具包都提供退款保证。

那么这个双重打击的第二部分是什么呢？好吧，我能说的就是尽快在这些陷阱周围寻找一个了不起的可用性博客。我对这一个非常兴奋。敬请期待！
