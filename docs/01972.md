是的，这是版本控制秀，我们很好奇你是什么版本的，你是怎么做到的。

是啊，真有趣。现在我正处于重新开始的阶段。我现在工作的公司很小，我们还是隐身的，具体的我也说不出来。噼里啪啦。是金融方面的东西，我整天都在写 Java 代码。我们太小了，没有结构。我们都只是在编码，因为有很多事情要做。

David:

我喜欢这样。你又把自己从零版本重置了。

Hampton:

我们两三个一起去吧。我的意思是，我的天啊，我已经经历了漫长而奇怪的生活，从在南方腹地的一个小的宗教教养到生活在世界各地。我觉得我是无限版本。当我和我丈夫刚结婚的时候，我们互相承诺的事情之一就是我们永远不会停止一起改变。这就是我们的目标，永远不要停止鞭策自己或做一些恐怖或有趣的事情。在 Sass 或类似的事情之后，我很容易就能找到一份工作，成为谷歌或其他公司的首席 Sass 传播者，然后得到很多报酬。那可能会是美好的生活，但那不是我喜欢的。

David:

萨斯足够受欢迎，以至于我甚至确信它需要一个传道者。

Hampton:

我肯定他们会给一些头衔，比如首席创新工程师之类的。我肯定我可以得到某种喜欢…我不知道。

David:

大概吧。好吧，我想很多人会想知道的一件事是，回到萨斯出现的时候。我很好奇，你是怎么想到的？你是怎么创造出这样的东西的？

Hampton [3:54]:

在长期打零工之后，我在大约 23 岁左右找到了我的第一份技术工作。在 21 世纪初，如果你没有计算机科学学位，要找到一份计算机方面的工作有点难——或者如果你的简历中什么都没有。所以从大学退学并没有让我很快找到一份高薪的工作，所以我做了一些零工。

后来我在一家小咨询公司找了份工作，他们给了我一个机会。很快我就做得很好，看起来这是适合我的职业。我和这些人一起工作，尤其是安东尼·沃茨，他是多伦多的一名设计师。我会看着他写他的 CSS，那时我大部分是用 Ruby 写的，他会这样写…我不知道人们这样做有多久了，但在 Sass 之前，很多最好的 CSS 工程师都是这样做的——每个文件都要展平 CSS 行，所以每个规则都在一行上。

你应该输入`.main-page .button .red`。这将是一行，然后你把所有的规则放在右边。你会在页面上看到几列相同的单词。这就像—

```
.main-page …
.main-page …
.main-page …
```

——这太疯狂了。我会看到他们。我们会说，<q>哦，我们重组了一个页面，</q>他脸上会有一种非常悲伤的表情，他不得不去手动进入每个文件，手动更改每个选择器，这是永无止境的重复。所以从头到尾找一遍，换一遍。

是的，所以我和这个人一起工作，他非常…他是我遇到的第一个真正对他的 CSS 感兴趣的人。这是他的事。在那份工作中，我已经提出了 Haml，并介绍了它。然后萨斯有点——我看到他这样做，所以我想，<q>嘿，我已经创造了一种语言。为什么不做另一个，我们会让你可以筑巢。</q>这是最初的想法。然后他一说，<q>哦，我们要对它进行预处理</q>……顺便说一下，没有……我的意思是，这是第一台网络预处理机。

我记得当我第一次设计它的时候，我花了大部分时间没有解释 Sass 是如何工作的，而是解释什么是预处理器，因为他们就像, <q>O，它是动态的。</q>我喜欢，<q>不，不，不，这不是真正的动态。你编译它。</q>他们喜欢，<q>你是什么意思？我可以用查询参数动态改变颜色吗？</q>你就像，<q>这不是动态的。我意识到自己越来越专业了。</q>

无论如何，对于这个和我一起工作的家伙，他是一个真正的专业人士，我觉得他的工具真的很糟糕。所以我说服了娜塔莉·韦森鲍姆，她是比我更好的程序员；我们和圣地亚哥一起去了 T2 火车站，我向她提出了这个想法。我算是开始了这个项目，然后她投入进来，建造了它，并继续建造它。有个故事。

我觉得所有这些对我来说总是来自…我做的每件事总有一个朋友影响了我，并且是我为之建造的人。我的朋友梅丽莎是维基百科手机用户。我觉得她在抱怨她不能在手机上查看维基百科，所以我建了那个。后来就正式结束了。我最终在他们那里找到了一份工作。

Haml 是因为我和一个需要更好的结构化数据的团队一起工作。Wordset 是因为我给自己编了一本字典，然后意识到我们需要一本更好的。是的，那是我的哲学。总会有人。

David:

我想这里的每个人都一定听说过萨斯，很多人也听说过哈默尔。我不知道有多少人听说过 Wordset。那是一个有趣的项目。

Hampton:

是的，这可能是因为它不太成功，但是是的…

David:

多大程度上的成功？我是说，你是萨斯的发明者。

Hampton [7:16]:

几年前我做了一个名为[字典的 iPhone 应用程序！](https://itunes.apple.com/WebObjects/MZStore.woa/wa/viewSoftware?id=293283136&mt=8)有几十万人长时间使用它。我用的字典——源字典——真的很差。嗯，不是真的…是的，很糟糕。它叫做 [WordNet](https://wordnet.princeton.edu/) 。人们可能听说过[维基词典](https://en.wiktionary.org/wiki/Wiktionary:Main_Page)，它是维基百科的字典，但是我在维基媒体基金会工作过，这些数据很难使用。所以我意识到有一个缺口，没有一种结构化的方式来合作编写一本可以开源的词典。那是不存在的。

所以我丈夫和我创建了 Wordset，花了一年半的时间靠积蓄生活，并建立了这种开源项目。现在有一个 iPhone 应用程序，但我们认为它不会赚钱，事实也确实如此。它实现了所有的个人目标。它需要存在。我们没有太多的人来做这件事。

它仍在运行，人们仍在提供帮助，但它从未真正吸引足够多的英语书呆子来帮忙。但是一些非常有趣的设计。我对它的运行方式感到非常自豪。

那是 Wordset。但是大约九个月前，我们花光了自己的钱，然后意识到我们可能应该去做一些我们认为可以产生积极影响的事情，*和*赚钱。因此又是零版本。

Tim:

说到你一直在做的工作，开源在你发起的项目类型中有多重要？我注意到 Sass 是开源的。我不确定 Haml 是否是开源的，但我猜是开源的。开源对你有多重要？

Hampton:

非常！完成——继续前进！

Tim:

很好，问题回答了。

Hampton:

我从来不简短回答。有趣的是，开源对我来说很有意义，尤其是麻省理工学院的风格——非常喜欢分享。老实说，这只是我的个性。我确实做了一些没有公开所有源代码的东西。Wordset 本身作为一个服务器在技术上并不是开源的，但是所有的数据都是开源的。只是打了个电话想知道它是否有用。它并不是一个随机运行的工具。(如果你想帮忙，给我发个 Twitter 消息。)

是的，对我来说，如果你做了很酷的东西——我想帮助别人，我想分享东西。我是开发商。我想帮助和分享给其他开发者。这就是我活着的原因。在这里我可以针对个人。我不打算要孩子。我不用太担心钱的问题。我是旧金山的白人开发者。我这辈子要做什么？我只是想产生积极的影响。

我认为最好的方法之一是——比如，开源软件存在于你之外。它甚至存在于公司之外。我喜欢它一旦出现就不受你的控制。尤其是有了麻省理工的执照:我觉得我可以用它做任何事情。拿去吧，走吧，它是你的了。这真的让我觉得我正在做的事情很重要，即使没有人最终会使用它。但是开源最酷的部分是总会有人最终使用它。

我为一个从未发布过的 iPhone 项目做了一个愚蠢的东西，我想它叫做 [RBResizer](https://gist.github.com/hcatlin/180e81cd961573e3c54d) 或者别的什么。这就像一个盒子大小的图像。总之，我想我把它公开发布在 Gist 上了，我还发了微博。我想，<q>如果有人需要这个…</q>

太搞笑了。就像，那愚蠢的 200 行代码，就像一个要点，它甚至不是一件事，只是不停地在互联网上一遍又一遍地出现。这个文件就像，<q>哦，看看这个，用 RBResizer。对我来说这太酷了。这艘船沉入了大海，它正在结束自己的生命。</q>

David:

你看，这一定很令人满意，我认为有很多开发者在开源社区看到他们发布了一些东西，他们担心没有人会去碰它。他们真的不知道如何让人们对它感兴趣。他们考虑的是，<q>我如何向其他开发者推销这个？另一方面，你创造了这些项目，人们只是成群结队地跟随。</q>

Hampton:

每一次成功，都有一堆失败。有一吨。贾布尔从未离开。这基本上是一个咖啡剧本的事情，我个人缺乏执行或营销可能是最大的责任。我只是有太多的项目，我开始和没有完成。

Tim:

我认为区别在于你不断推出的内容数量。你的想法的数量…你不只是把它写在一本书的某个地方，就像我大部分时间做的那样。我只是想，<q>哦，这是个好主意，让我永远保存它，什么也不做。</q>你实际上在做这些事情。我在想，灵感从何而来？创意从何而来？是什么让你继续从事这些工作？

因为，我可以想象，在某个时候，你打开收件箱，你看到一千个请求，你知道，开源帮助，或者维护你 15 年前工作过的东西，但忘记了它的存在。是什么让你回来继续这项工作？

Hampton [12:04]:

对死亡的恐惧。这是一个公平的答案——来吧。

Tim:

今天额外的哲学！

Hampton:

耶！很多人都谈到了这一点——开源贡献者的责任感。人们在项目上筋疲力尽，因为他们觉得每个人都欠他们的……如果他们不去做，每个人都会发疯……我不知道。我的大部分项目我都工作一段时间，然后要么他们死了，要么另一个维护者介入。

萨斯由娜塔莉和 T2 保持了一段时间。我仍然谈论它。我仍然参与其中，但我不编码。我不会把整个周六都花在这上面。如果人们需要帮助，我会回复电子邮件。我试着教这方面的课程。我试图鼓励人们，但我不会在周六早上醒来，然后花一整天的时间写代码。

现在…嗯，这取决于我人生的哪个阶段，对吗？当我在上一份工作中攒了一点钱，然后花时间做了一年的 Wordset。我真的不认为这会赚钱。我们发布了一些开源代码。我不知道，如果有人现在有问题，我可能会尝试解决它…
