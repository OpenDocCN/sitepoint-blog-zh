# 没有' Function '语句，JavaScript 能运行吗？

> 原文：<https://www.sitepoint.com/shorter-javascript-function-statement/>

你是否厌倦了每隔几分钟就在 JavaScript 代码中输入“函数”？应该缩短吗？JavaScript 发明者 Brendan Eich 是这样认为的，他在上个月伦敦的主题演讲中透露了以下选项。

第一种选择是弗罗林符号:

```
 ƒ myfunction() { … } 
```

我挺喜欢的。它在代码中非常突出，看起来像函数中的 f。不幸的是，你不会在所有的编码字体中找到它，而且它很难输入(在 Windows 中，按住 Alt 并在数字键盘上输入 0131。Mac 用户应该尝试特殊字符调板)。

接下来，我们有λ符号:

```
 λ myfunction() { … } 
```

JavaScript 爱好者会认为这很酷！不幸的是，它遭遇了和弗罗林一样的问题，你甚至不能在 Windows 中输入它。

最后，布兰登给出了简单的老答案:

```
 # myfunction() { … } 
```

大多数观众投票赞成这个标志。它当然更容易输入，尽管我不确定它是否会在 JavaScript 代码中非常突出。

可能会出现其他选项，您应该注意，如果您喜欢，仍然可以使用“function”……您现有的代码不会中断。

除了减少您的打字工作量之外，节省的带宽也是巨大的。例如，缩小的 jQuery 库将变得小 4.3%。这可能只有 3Kb，但乘以它每天收到的数百万次下载。
