# 故地重游

> 原文:[https://www.sitepoint.com/revisiting-tidy/](https://www.sitepoint.com/revisiting-tidy/)

我最近遇到了一个情况，我在没有自己设备的情况下远程工作，不得不编辑一篇相当长的微软 Word HTML 文章。

虽然 Word 很容易被认为是文字处理领域最好的软件之一，但大多数人认为它作为 HTML 生成器实在是太差劲了。我被要求以尽可能符合标准的格式发布这篇文章，一想到手动删除数百行代码，我就紧张。

Tidy 来救援了！我们中的许多人可能在开始时单独使用过 Tidy 然而它和它的功能变体现在已经被构建到如此多的 web 开发工具中，我们认为代码清理是理所当然的。

我很快就能把这篇文章中的 HTML 源代码放到整洁的输出 HTML 中——节省了时间，也提醒了我为什么喜欢保持简单。Tidy 是 KISS 理论的一个很好的例子，也是为什么它适用于许多开源软件解决方案——保持简单，专注于利基市场，做你擅长的事情。

在不使用 Macromedia Dreamweaver 和 BBEdit 的时候，我碰巧把由[特里·蒂格](http://www.geocities.com/terry_teague/index.html)管理的 [Mac Tidy](http://www.geocities.com/terry_teague/tidy.html) 放在我自己的设备上。最初的 Tidy 驻留在 [W3C](http://w3c.org/People/Raggett/tidy/) 中，当前的项目已经移交，可以在 [Sourceforge](http://tidy.sourceforge.net/) 上找到。
