# 可移植的 Linux 虚拟机

> 原文：<https://www.sitepoint.com/portable-linux-virtual-machines/>

MetroPipe 发布的[技术预览](http://www.metropipe.net/ProductsPVPM.shtml)引起了不小的轰动。这个包运行在 [Damnsmall Linux](http://www.damnsmalllinux.org/) 上，使用 [QEMU](http://fabrice.bellard.free.fr/qemu/) (一个 CPU 模拟器)。

本质上是一个互联网通信系统，它可以完全运行在一个 USB 迷你驱动器(128 MB)和其他闪存媒体设备上，甚至是 iPods 上。它包括一个可启动的 Linux 操作系统、一个网络浏览器(Firefox)、电子邮件客户端(Thunderbird)、Enigmail GPG(用于电子邮件加密)和一个永久的主目录。

从 web 应用程序开发的角度来看，正是这最后一个特性引起了我的兴趣。这种便携式机器的核心特点是，虽然它可以在 Windows、Linux 和 OS X(后者即将推出)机器上运行，但其活动的所有痕迹都被分割在可引导闪存设备上，这也保留了一定程度的隐私。

这意味着 cookies、日志和其他数据痕迹不会作为残余留在用于引导设备的主机上，因为虚拟机作为自己的“计算机”存在，同时寄生连接到一台完整的计算机。

这种讨论主要针对那些寻求隐私和其他个人资料的人——我感兴趣的是 web 开发人员如何在他们为客户构建的应用程序和解决方案中找到这种适用性。

这是否为拥有大量移动销售人员的公司打开了销售窗口，这些公司需要在旅途中访问和更新 CRM(他们并不总是有地方将笔记本电脑连接到网络)？他们能简单地在现场完成销售，然后跳上机器更新数据吗？

是否可以为公司和员工部署一个 kiosk，让他们能够通过 USB 接入，下载和上传数据到 web 应用程序，然后简单地将他们的“PC”放入口袋？哪些 web 应用程序现在可以部署到生产环境中，而以前会被认为对生产工作区中的多台 PC 不利？

更重要的是，web 应用程序的演示或草稿现在可以交付或交给拥有完整运行环境的客户进行审查吗？

虽然目前只是一个技术预览，但这种设备有很大的潜力，可以被认为比笔记本电脑、PDA 和高端手机稍微耐用和移动。
