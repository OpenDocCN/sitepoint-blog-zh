# 网站曾经做过吗(这对销售网络服务意味着什么)？

> 原文：<https://www.sitepoint.com/is-a-web-site-ever-done-and-what-does-that-mean-for-selling-web-services/>

我即将推出一项新的基于订阅的网络服务(不久将会宣布，并特别向 Sitepoint 读者提供 beta 测试)。

在创建这个网站的过程中，有一件事变得非常清楚:一个网站永远不会被完成。每当我们修复一个 bug 或进行一项功能改进时，我们都会看到继续改进的机会。这就像升级房子里的一个房间，然后看到其他房间突然也需要升级。

在复杂和简单的网站上都是如此。每个 Web 设计/开发客户都不断需要更新和升级。

这对您来说既是机遇也是挑战:

机会是显而易见的:如果你在客户面前做得很好，你应该有一个长期的后续工作。

我的供应商面临的挑战是:当一个网站从未完成时，如何定义“完成”？你可以和客户设定什么样的里程碑，这样你们就能在交付成果实现的时间上达成一致。

例如，我和我的商业伙伴经常与我们的开发人员来回沟通。他们认为事情已经完成，即使还没有准备好。按照我们的标准，我们认为某件事情在黄金时间做好了就完成了。我们都无法准确定义“完成”因此，双方都不会完全满意。

这个问题我没有简单的答案。没有明确定义的可交付成果，固定价格合同对你来说很棘手。时间和材料合同会让你的客户非常恼火，他们厌倦了支付额外的时间来满足他们“明显的”市场驱动的标准。我尝试过奖金安排，在我们满意之前保留一部分。那也不行。

所以我要做的是关注这个机会，并从长远的角度看待这段关系中的整体机会。
