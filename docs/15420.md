# 损失加载时间，留住访客

> 原文：<https://www.sitepoint.com/loading-time-keep-visitor/>

速度慢的网站会赶走访客。就这么简单。你说，不是你的？觉得“值得等待”吗？

很抱歉告诉你，南瓜，不是的。你在回家的路上。你有两个选择。你可以选择距离最短但有很多红绿灯的路线，也可以选择距离稍长但没有红绿灯的路线。你选择哪个？

我选择#2，许多网上冲浪的人也是如此。为什么？因为继续前进更好。我想看看。我想看书。我想买。我不想为了下载一个页面而等待 67 秒，因为网站设计者认为用他的宠物雪貂戴着时髦的小贝雷帽的照片作为图像地图是个好主意。

我们都知道图形会拖慢网站的速度，那么有没有可能在不发布纯文本网站的情况下让访问者远离鼠标呢？

当然可以。这只是一个平衡审美情趣和加载时间的问题。我想讨论几种方法来做到这一点。不过，在我们开始之前，帮我一个忙，去一趟[网站车库](http://websitegarage.com)，免费调一下你想加速的页面。请务必记下不同调制解调器速度下的页面加载时间，因为稍后您将使用这些时间作为基准来衡量您所做更改的有效性。哦，别忘了“按原样”备份你的页面(和图片)，以防你对这些改变不满意，想把它恢复到原来的状态。一旦你做了这些事情，我们就准备解决两个方法来减少加载时间:减少页面上的图片数量和减少剩余图片的大小。

##### 减少页面上的图像数量

冷静、客观地审视你的网站。试着从游客的角度来看。哪些图形是必要的，哪些是多余的？你能用 HTML(粗体、斜体、字体)处理文本，而不是用文本图形吗？可以用文字链接代替可点击的图片吗？

削减消耗性图形可以大大加快页面加载时间。逐一评估每个图形，以确定它是该留还是该走；只保留那些必不可少的。

明智地使用 Flash 动画。毫无疑问，Flash 电影非常酷——如果做得好的话。然而，“因为它看起来很酷”并不是在你的网站上添加东西的理由。我想不出比毫无目的的 Flash 介绍更令人讨厌的事情了。你看过这个，是吗？当你点击一个网站的主页时，你会看到一条空洞的短信，比如“欢迎来到我的网站！”，你会看到它跳舞，扭动，变形，以其他方式让马自己变成你知道的样子。一个做得好的介绍有一个明确的信息/目的，它看起来很突出，访问者可以选择跳过它——这很重要。

##### 缩小剩余图像的尺寸

较小的图像加载速度更快。在你把你的图片精简到最基本的部分之后，是时候减少剩余部分的大小了。Y

你可以通过两种方式做到这一点:
1。减小图像尺寸(宽度 x 高度，以像素为单位)。
2。“优化”图像。

缩小图像的尺寸并不总是可行的选择。页面的布局可能取决于特定大小的特定图像(不过，页面的显示方式当然与屏幕分辨率有关)。如果您可以安全地缩小图像尺寸，那么就这样做，然后进行优化。如果您无法更改图像的尺寸，请仅进行优化。

“优化”图像意味着减小图像的文件大小。这主要是通过从图像中去除无关的颜色来实现的。例如，以 256 色保存的 GIF 通常可以缩小到 128、64 或 32 色——有时甚至更少——而不会影响图像质量。这可以大大减小图像的文件大小，从而(通常)减少几秒钟的加载时间。

[NetMechanic](http://www.netmechanic.com) 、 [MediaBuilder](http://www.gifoptimizer.com) 和 [Spinwave](http://www.spinwave.com) 都提供免费的图像优化。一旦你删除了多余的图片，优化了“管理员”，上传你的轻量级页面，回到网站车库，再次运行调整。希望您已经获得了页面加载时间的显著改善——或者至少已经下降到一个更容易接受的数字。我的首页从 22.05 秒的加载时间到 33.6 秒的 14.55 秒，仅仅是因为图像优化。仍然有点慢，但是嘿，7.5 秒的节省也不算太差！

确保你的网站尽快加载是一个好的网站管理员/网站设计师的一部分。从冲浪者的角度来看，一个高质量的网站会将更多的访问者转化为客户。
