# 火狐计划内置可用性平台

> 原文:[https://www . site point . com/Firefox-plans-built-in-usability-platform/](https://www.sitepoint.com/firefox-plans-built-in-usability-platform/)

尽管仍处于概念阶段，Mozilla 正在计划一个有趣的新平台，名为 Test Pilot，它将允许 Mozilla 实验室的每个项目直接向参与用户推出可用性测试。该平台将以 Firefox 扩展的形式开始，目标是收集 1%的 Firefox 用户代表样本——鉴于 Firefox 拥有数百万用户，这是一个艰巨的任务。

Mozilla 实验室的用户体验主管阿萨·拉斯金本周在博客文章中谈到了试飞员[。“不仅仅是 Firefox 需要一个可用性实验室。雷鸟需要一个。Seamonkey 需要一个。每个 Mozilla 实验室项目都需要一个。他写道:“试飞员是一个平台——从 Firefox 扩展开始——在这个平台上，任何东西都可以通过可用性测试训练营。”拉斯金说，这是必要的，因为目前 Mozilla 只能访问通过调查、轶事、用户反馈等收集的零散可用性研究。](http://labs.mozilla.com/2009/01/test-pilot-vision/)

测试试点将允许 Mozilla 在实际使用场景中收集真实用户的反馈，拉斯金将其视为 Mozilla 实验室正在建设的大规模可扩展开放可用性实验室的一部分。拉斯金说，试飞员不会经常记录数据，但只会记录足够回答研究人员提出的问题的信息。拉斯金概述了该软件将如何在用户端工作:

> 第一次运行 Test Pilot 插件时，它会问一些简单的非个人身份问题，将您放入一个人口统计桶中，例如技术水平、地区等。，并允许您选择附加的匿名检测。
> 
> 我们只会收集匿名数据，在开放内容许可下发布所有结果，并审查每项测试以确保您的隐私得到保护。偶尔，您可能会被要求参与一项基于您的人口统计数据的简短调查。如果您选择允许额外的匿名工具，实验可能会要求一些信息用于汇总研究。

虽然它将从 Firefox 开始，但最终计划是将测试试点融入到每个 Mozilla 实验室项目中。Mozilla 计划开放测试结果，一旦通过审查，将允许任何人创建研究测试并运行它们。

开源博客作者 Matt Asay 认为[测试将为 Mozilla](http://news.cnet.com/8301-13505_3-10146821-16.html) 提供一个主要的竞争优势。他写道:“如果 Mozilla 能够成功地从非技术性的普通用户那里收集信息，它将获得计算领域的圣杯:深入的客户反馈。”。“Mozilla 会如何处理这些反馈呢？为什么，当然是征服世界。”

但是我们有一些关于试飞员的重要问题，拉斯金的博客文章没有回答。第一:什么时候来？试飞员实际上是在 2008 年 3 月宣布的，当时第一个版本被承诺将在“几周内”发布。显然，如果 10 个月后它仍处于概念阶段，那就不会发生。但拉斯金再次谈论它的事实似乎表明，试飞员的发布要么即将到来，要么至少仍在积极开发中。

第二:有多少人会实际使用？就试飞员的有用性而言，这是一个很大的假设。即使这个平台构建得很好，并且成功地捕获了有意义的反馈数据，如果没有人使用它，它也没有任何价值。Mozilla 真的能说服 1%的用户加入他们的用户反馈小组吗？

第三:非 Mozilla 开发者可以使用试飞员吗？由于 Mozilla 软件的开源特性，测试平台将向其他开源开发者开放似乎是一个安全的赌注。但在拉斯金的帖子中没有具体提到这一点。
