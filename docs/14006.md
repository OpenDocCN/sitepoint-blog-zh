# 关于补偿你的设计师/开发者团队的想法

> 原文：<https://www.sitepoint.com/thoughts-on-compensating-your-team-of-designersdevelopers/>

一位读者询问向开发人员提供奖金的最佳方式。我不太喜欢奖金。首先，很难客观地判断开发人员或设计师应该获得多少奖金(与销售人员、投资银行家和其他做出可衡量的具体贡献的人相比)。第二，人们倾向于获得奖金，并期望每年都能拿到奖金。

这是一个理想的薪酬阶梯:

1.人们应该从承包商做起，按时、公平地获得高质量工作的报酬。不管对错与否，不管喜欢与否，我们正越来越多地转向自由经济。大多数人都是从这里开始证明自己的价值。

2.优秀的承包商有权成为雇员，享有福利和薪水。只有当你确定你将会 100%利用他们的能力时，才雇佣一个员工，与一群合同工一起上下浮动。

3.优秀的员工会获得许多非现金福利:挑选最佳任务、弹性工作时间、以各种个人和适当的方式得到感谢、成为有趣文化的一部分、竞赛、更有经验的设计师/开发人员的指导等。

4.核心员工——那些提供持续、可衡量的福利的关键员工——可以获得股权和/或奖金。股权属于为公司增加独特企业价值的员工。奖金发放给实现具体、可衡量目标的员工。

5.当公司业绩特别好的时候，时不时给所有员工发奖金是个好主意。但是你要明确这些都是一次性的。此外，考虑根据员工的资历或工资的百分比来分配这些奖金。

以上是思考薪酬的底线方法。你们很多人不会喜欢的。我现在已经习惯了:)
