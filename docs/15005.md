##### HTML 和 XHTML 有什么区别？

首先，HTML 代表超文本标记语言，XHTML 代表可扩展超文本标记语言。通过学习 HTML，你会学到大部分 XHTML。

XHTML 是 HTML 的严格版本。W3C(万维网联盟，可以说是这方面的管理机构)已经发布了 HTML 的最终版本 4.01，现在专注于 XHTML 的开发。

许多你已经知道甚至喜欢的 HTML 标签在 XHTML 中并不存在。这包括许多表示标签，比如字体标签，因为 XHTML 基本上要求使用层叠样式表(CSS)。

例如，XHTML 还要求关闭文档中的每一个标签，甚至是在 HTML 中不需要关闭的标签，比如图像标签。所有的标签名称和属性都是区分大小写的，必须用小写字母书写，而在 HTML 中这并不重要。

这些只是 HTML 和 XHTML 之间的许多差异中的一些，我无法在这篇简短的评论中给出所有的差异(有关这方面的更多信息，请参考 Nick Wilson 的 XHTML —简介)。然而，这应该让您了解为什么作者在一册书中讨论了 HTML 和 XHTML 以及这对初学开发人员有多大价值。

读完这本书后，你将拥有成功创建和编辑 HTML 文档所必需的知识。

##### 级联样式表

尽管这本书不断提醒你这是一本 HTML 书，但它确实提供了一些关于层叠样式表(CSS)的重要信息，正如我们之前所讨论的，这对想要构建符合标准、向前兼容的网站的开发人员来说是必不可少的。

对于那些不知道的人来说，样式表是你应该用来控制整体外观(背景、颜色、字体等)的工具。)的网页文件。XHTML 基本上要求在文档中使用 CSS。这本书教你如何编写自己的样式表并将它们嵌入到 HTML 页面中，让你能够根据自己的喜好定制文档的外观。

这本书可以被认为是对 CSS 所有基本概念的介绍。如果你想学习 HTML 和 CSS，但只想得到(或只能买得起)一本书，这本书是一个很好的选择！
