Matthew Mecham 是在线社区世界中最著名的人之一，也是使用著名的 Ikonboard(现在是流行的 Ibforums)脚本来为社区提供动力的网站管理员中最受欢迎的人。

马特曾开发过 YaBB SE 等产品，并与 YaBB 创始人泽夫·赫梅尔一起创建了“Ikonboard”，这是互联网上最受欢迎的社区脚本之一。现在，Matt 凭借他自己的服务团队 Invision 引起了轰动，该团队推出了 Ibforums。我们请马修花点时间回答几个一直困扰我们的问题…

##### ikon board——一点历史

***是什么让你开始写自己的论坛脚本，Ikonboard？在你冒险进入那个方向之前，你做了什么？**T3】*

在 1999 年下半年，我为图形和网页设计受众运营了一个网站(www.ikondiscussion.com)。该网站围绕着优秀设计的教程和技巧。和许多站长一样，我发现自己在寻找一个可以用来建立在线社区的论坛脚本，而且，和许多其他站长一样，我将网站作为一种爱好来运营，而预算却是众所周知的小本经营。

我偶然发现了一个不错的免费论坛脚本(字面上只有一个脚本)，叫做“BoardMaster”(现已不存在)。这是一个相当基本但坚实的小脚本，类似于 UBB(这是在 YaBB、vBulletin 等的时代之前)。它没有现代主板所具备的任何“必备”功能(比如私人信息、复杂的配置文件等)。)，但这足够经营一个小社区了。

我的站点需要一些定制脚本，当时选择的脚本语言是 Perl。我学习 Perl 的方法是错误的(专家会告诉你)，通过查看其他 Perl 脚本并弄清楚它是如何工作的。我很快就学会了，因为我曾经在 Amiga 上给 AMOS 编程(但那是另一个故事了)。在我准备好运行网站之后，我开始对论坛脚本做一些小的修改——比如改进导航(比如页面底部的论坛下拉框),允许不同的图片在不同的论坛中使用。我将这些“黑客”发布到 BoardMaster 社区，最终被要求主持“黑客”论坛。

任何一个网站管理员都会告诉你，一个网站永远不会真正完成，它只是在不断发展。我一直在寻找改进这个网站的方法，这种愿望让我想到了另一个叫做“董事会力量”的论坛脚本。与 BoardMaster 相比，Board Power 功能丰富，使用有趣，并且提供了(当时)复杂而强大的管理中心。我下载了一个副本，摆弄了几个星期，然后决定用 Board Power 替换现有的 live BoardMaster，这让我的社区非常高兴。没过多久，我就发布了一些技巧，并建立了一个致力于修改董事会权力的小型网站。

我写的最大的黑客是 Ikonboards“用户控制面板”的一个预运行程序。它允许你修改个人资料，将好友添加到联系人列表中，发送和接收私人信息(这远在私人信息流行并作为默认功能包含在今天的公告栏中之前)。对于董事会权力来说不幸的是，开发者似乎从来没有认真对待过董事会权力——一系列的分歧和不正当的商业行为迫使我放弃了董事会权力社区。在 Board Power 工作期间，我开发了一个评论脚本(标签为 Ikon Review ),允许公众提交网站、教程等。，并使管理员能够对它们进行评级和提交。董事会权力的内部争吵和缺乏好的替代方案让我认真考虑编写自己的论坛脚本。

随着 Ikonboard 的开发开始，我注意到街区里有一个名叫“YaBB”的新成员。我从来不会拒绝新的脚本，我下载了一个副本，玩了玩，不久我就有了一些小技巧。我暂停了 Ikonboard 的开发一会儿，心想如果这个脚本成型了，我可以用它来代替自己写。我加入了 YaBB 开发团队，并为其早期开发做出了贡献(如搜索引擎、主题视图和发帖时的主题评论)。虽然 YaBB 开始获得动力，但我仍然渴望完成 Ikonboard，并认为我不能同时参与 YaBB 和 Ikonboard，所以我在 2000 年夏天从 YaBB 团队辞职。

Ikonboard 的特性列表是由 Ikondiscussion 社区的建议驱动的。我想将个人信使和控制面板添加到基本脚本中，并增强许多其他标准功能，如公告、审核工具等。我认为如果这些特性是标准的，人们就不需要修改代码来添加这样的功能；我希望 Ikonboard 是一个完整的解决方案，并作为更多功能的坚实基础。2000 年 8 月初，我建立了一个“即将推出”的网页，其中有一个订阅邮件列表的部分，几周之内，我就有了 200 多个电子邮件地址。一个月后，也就是 2000 年 9 月，Ikonboard v0.9 测试版完成了，可以下载了。

每当一种新产品投放市场时，它都必须与同一个市场中现有产品的激烈竞争，同时证明自己是竞争对手。哪个剧本 did 是 Ikonboard 最大的竞争对手，在推出 Ikonboard 第一个稳定版本的时候，你还面临过哪些困难？T3】

当我发布 Ikonboard 的时候，并没有太多激烈的竞争，当然也没有今天那么多的论坛脚本。Board Power 和 BoardMaster 正在慢慢消亡，要么是因为愚蠢的决策，要么是因为缺乏开发(BoardMaster 开发人员消失得无影无踪)。vBulletin 相对较新，尚未推出 1.1 版本，YaBB 刚刚起步，phpBB 1 尚未发布，而 UBB 5 仍然是该产品的领导者。我从来不担心竞争和其他论坛软件公司在做什么，我只是专注于我自己的工作，尽我所能。

当 Ikonboard 发布时，一些 UBB 所有者声称 Ikonboard 只是一个重新标记的副本(显然不是)，并制造了一些噪音，我想这是很自然的，因为大多数板从前端看起来都一样。以我的经验，只有一小撮人试图挑起事端——你很少看到开发者卷入针锋相对的争论；事实上，我一直与 UBB 和 vBulletin 的作者和开发团队保持着良好的关系。理论上，UBB 是 Ikonboard 发布时最大的竞争对手，尽管我并不真的认为这是“竞争”——这并不是傲慢的意思，只是我从未试图与 UBB 竞争，因为我们服务于两个不同的市场。

Ikonboard 面临的最大困难——也是 Invision Board 仍然面临的最大困难——正在受到商业部门的重视，在商业部门，大多数人认为“免费”等同于“聊胜于无”。我总是努力说服人们不要总是这样，事实上，互联网上 3 个最常用的软件工具都是可以免费下载的。Apache、MySQL 和 PHP 都是各自领域的巨头，因为它们都是免费的。如果在使用 PHP 之前必须支付服务器端许可费，很难想象 PHP 会得到如此广泛的应用。考虑到这一点，我确实发现使用这些工具的自由软件在被贴上“聊胜于无”的标签之前不被允许证明自己是相当讽刺的。话虽如此，随着时间的推移，Ikonboard 确实为自己赢得了相当不错的声誉，并被一些相当大的网站使用，包括索尼和维亚康姆。

Ikonboard 最初是在什么类型的许可协议下发布的，你当前的脚本 Ibforums 是在什么许可协议下发布的？T3】

Ikonboard 和 Invision Board 都遵循免费软件模式，使用我们自己的专有许可:它们一直都是免费的，但它们不是，也从来不是开源的。很多人问我们为什么不把 Invision Board 作为开源(GPL)发布，我的回答总是一样的:开源很棒，PHP、MySQL 和 Apache 都证明了 GPL 模式是可行的。YaBB 和 phpBB 在开源许可方面也做得很好，但这不是我们想要做的。我以前发布过一些小脚本，在几乎所有的情况下，对大多数人来说，开源仅仅意味着“我可以移除版权，重命名它，然后作为我的脚本发布”，当然，事实并非如此——我厌倦了看到我的作品被重新标记。对于真正的开源开发者来说，这可能看起来是虚荣和自恋的，但是我总是用商业方法来推销我的产品。

Invision Board 可以免费下载，但是未经我们的书面同意，您不能重新分发或标记该产品。当然，你可以改变模板的颜色、图形，并根据自己的意愿对白板做任意多的修改(只要你不删除页面底部的版权和链接)。实际上，这允许你用开源产品做尽可能多的事情，除了修改后重新发布。我并不打算直接从 Invision Board 赚钱——这从来都不是我的初衷——但我们确实用它来为我们的网站带来流量，这些网站提供定制脚本、虚拟主机、论坛托管和优先级支持。这就是我们赚钱的方式——这就是我们开发产品并允许免费使用的方式。我们处理过的大多数企业更喜欢使用免费软件产品，因为开发商仍然承担一些责任，提供优先支持计划也给他们信心，我们这样做不只是为了好玩。我们有一个由大约 20 名核心成员组成的强大支持团队，他们的工作非常出色，总是受到客户的好评。

***你发布 Ikonboard 的时候，当时和你一起工作的是谁，你们用了哪些资源？**T3】*

当 Ikonboard 第一次发布时，我正独自从事代码、开发和支持工作。我自费支付虚拟主机的费用。然而，没过多久，我就开始招募一个团队来帮助支持该产品并主持支持委员会——这是一个新产品，我们都在同一条船上，必须快速行动并快速学习。我们很幸运有一些有才华的人与 Ikonboard 团队一起工作——我们的 NT 专家 Marcel(又名 Redmak)继续在 Neowin.net 工作，但几乎所有最初的“iTeam”都仍然与我在一起，将近 2 年后的今天。

当 Ikonboard 2 发布时，该产品已经开始获得动力，我们的虚拟主机增长已经超过了我们的增长——尤其是带宽。为了支付这些款项，我一直在扩大预算，但很明显，我们需要做出一些改变来实现增长。对我们来说幸运的是，Ventures Online 提供了一个非常慷慨的半专用主机服务，以换取一个横幅广告来帮助我们发展。当 JEG 收购 Ikonboard 后，我们转向了他们的内部网络。Ikonboard.com 坐在专用的 linux 服务器上(IBM Netfinity ),连接到为网络供电的双 OC48——不用说，带宽和资源不再是问题。

2001 年初，JEG 接洽并收购了 Ikonboard。那笔交易的细节是什么？T3】

贾维斯娱乐集团(JEG)在 2001 年第一季度找到了我。这时，Ikonboard 已经出来几个月了，已经站稳了脚跟。我刚刚开始 Ikonboard 3 的工作，我意识到，考虑到我也有全职工作，试图管理开发、行政和业务是太多了。在交易正式确定之前，我和 JEG 的首席执行官谈过几次，他承诺了很多事情，比如在他自己的数据中心进行专门的托管，帮助管理和业务发展，并向我保证他不想为 Ikonboard 收费。他很高兴让它发展并保持免费——毕竟，一个繁忙的网站可以获得良好的赞助协议，通过网络保持良好的流量给股东留下深刻印象。

经过深思熟虑，我签了合同，交易完成了，现在 Ikonboard 归 JEG 所有，我被带到 JEG 管理这个项目。我觉得这里值得一提的是(为了平息许多谣言),这笔交易根本不涉及任何金钱。我的“报酬”是 JEG 的股票(在撰写本文时，我还没有卖出或交易这些股票，因为它们实际上一文不值)。起初，Ikonboard 社区有很大的抗议，他们担心最坏的情况——一家大公司已经吞并了我们，他们因下载 Ikonboard 而被起诉只是时间问题——或者它会被拆分并出售。我花了很长时间和许多个人承诺来强调这不会发生，Ikonboard 将保持自由。

Ikonboard 相对于其自由竞争对手的优势是什么？T3】

当 Ikonboard 首次发布时，它的最大优势是开发速度、快速支持以及推动产品前进的社区交互性。Ikonboard 3 深受当时社区需求的影响。我对它的开发非常开放，并保留了一个想法和代码片段的公共日志，供社区审查和讨论。Zeff Hemel (YaBB 创始人)从我关于“iDatabase”(一个数据库抽象层)的笔记中受到启发，并在他从事 YaBB 2 Perl(很久以前就停止了对该产品的开发)时将这些想法和结构融入到他自己的 DBA 中。我还在这些代码笔记中为用户可选择的皮肤和模板打下了基础，远远早于任何其他板有这样的特性。毫不奇怪，在第一款 Ikonboard alpha 发布后不久，其他主板也模仿了这个想法。我并不介意——能为现代公告板的“特征库”做出贡献本身就是一项成就。

你能告诉我们更多关于极光的信息吗？当你离开 Ikonboard 和 JEG 的时候，它的发展和完成情况如何？T3】

当我离开 JEG 和 Ikonboard 时，Aurora 已经完成了 35%。没有版主控制面板或管理控制面板，除了基本的关闭、打开和删除主题之外，几乎没有其他审核工具。Aurora 的数据库结构与 Ikonboard 的数据库结构完全相同，所以我们可用的演示版本仍然使用 Perl 管理中心来创建类别和论坛等。要让它运行，需要大量的桥接代码——它远不是一个 alpha 产品，更不用说一个完整的发布产品了。它的主要目标是显示开发进度，并被用作显示新 PHP 版本有多快的工具。

今年早些时候，你离开了 JEG，不再在 Ikonboard 工作。原因是什么？T3】

在这一点上值得一提的是，JEG 是一家在场外交易/纳斯达克交易的公司，受美国证券交易委员会监管；这意味着我不能把他们放在可能影响他们业务的负面光线下，但我会在法律允许的范围内尽量公开。

早在 2001 年第四季度末，我就对 JEG 领导的 Ikonboard 公司的发展方向逐渐失去了幻想。我觉得我完全被误导了，为了说服我签合同，某些细节被夸大或省略了。然而，我已经承诺完成 Ikonboard 3，我决定在决定做什么之前，我会看到这个项目。

在 2001 年末和 2002 年初，我受到了来自 JEG 的巨大压力，要发布 Ikonboard 3 的最终版本，并编写一个全新的 Ikonboard 托管版本，名为 MyIkonboard。我知道 Ikonboard 3 还没有准备好最终发布，但是压力依然存在。我经常阅读新闻稿，声明我从未给出的发布日期，并被告知“确保完成”。我仍然在全职工作，晚上在 Ikonboard 上工作，经常到早上 5 点才能完成——考虑到我必须在 9 点上班，所以没有多少时间睡觉。我多次被告知，JEG 付不起我任何工资，因此不能雇用我——所以我别无选择，只能继续两者兼而有之。

需求与日俱增，直到我感到精神疲惫——我试图全职工作并维护 Ikonboard 3，开发 MyIkonboard 并编写“Aurora”(ikon board PHP)。当时，JEG 的另一名“雇员”和股东查尔斯·华纳(Charles Warner)认为，他的处境完全相同。他被指派处理 Ikonboard 产生的所有业务，并照顾 JEG 网络——这不是一项容易的任务，而且非常耗时。最后一根稻草出现在 2 月份，当时 JEG 的首席执行官向我和 Charles 发出指令，通知我们他想对 Ikonboard 收费，这将使我违背对 Ikonboard 社区的承诺。查尔斯和我讨论了我们的选择，并决定我们宁愿重新开始，而不是继续与 JEG 合作。放弃两年的工作不是一个容易的决定，但我觉得这是正确的事情。

你是否被法律要求将 Aurora 的所有开发移交给 JEG，即使他们只收购了 Ikonboard？在 JEG 收购 Ikonboard 之前，奥罗拉有计划吗？T3】

离开 Ikonboard 的时候，理所当然的把 Ikonboard，MyIkonboard，Aurora 的文件都还了。所有这些文件都是在 JEG 的服务器上实时开发的——但为了表示友好，我把硬盘上的副本还了回去，尽管 JEG 对我未来开发的所有作品没有直接的法律权利。JEG 收购了 Ikonboard，但并不是我开发的所有代码，然而，当我以 Ikonboard PHP 的身份开发 Aurora 时，我觉得彻底的决裂符合我们所有人的最佳利益。

##### 现在——IBM 论坛

***你和谁一起创办了 Ibforums？为了重新开始，你从外界得到了什么支持？**T3】*

查尔斯·华纳和我离开了 JEG，开始了我们自己的合作。我们尽快建立了 Invision Power Services，这是我们所有服务的母公司。我们从社区获得的支持简直令人惊叹。我曾希望一些成员会跟随我们到 vision 板，但我从来没有想到约 85%的常规海报跟随我们。就在那一刻，真正的社区让我们团结起来，让我们都觉得自己是大家庭的一员。我们建立了一个临时的讨论板，几天之内我们就有近一千名成员注册并积极发帖。

对我来说，社区和产品一样重要——尤其是在我们所处的行业。如果我们在营销一个社区建设产品，那么我们有一个自己的强大活跃的社区是有意义的——ikon Board 和 Invision Board 一直很幸运地获得了多样化的追随者。当前的 Invision 董事会团队既多样化又有才华。Ken (Heartcall)是最初的“iTeam”的创始成员之一，他是一名表演音乐家，我们的支持协调员 Luke 也是一名才华横溢的音乐家，最近在一场包括 Michael Jackson 在内的 20，000 人面前进行了现场表演。

安德烈和奥斯卡(诺米内尔)是我在很长一段时间里见过的最具原创性的图形艺术家，斯图尔特·坎佩尔在将我们的客户从其他论坛软件转换到 Invision Board 方面做了出色的工作——通常一次转换 20 次。自称“MadDocktor”的人是我们的安全专家——经常修补 Windows 软件的漏洞，并就如何保护我们的系统向我们提供建议。这份名单还在继续！我们非常幸运拥有业内最好的团队之一。我们专门的支持团队(由 20 名成员组成)竭尽全力为我们的免费和优先支持请求提供一流的服务，98%的请求支持的请求和帖子都会在几个小时内得到答复。由于我们有一个遍布全球的大型团队，让我们所有人都在一个集中的办公室工作是不现实的。互联网是一种交流媒介，我们用它来保持联系，定期在我们的私人 IRC 频道上举行会议。所有团队成员都通过支持板和即时消息软件保持联系。

您更喜欢用 PHP 编写 Ibforums，而不是之前选择用 Perl 编写 Ikonboard。有什么原因吗？你认为 PHP 比 Perl 更优越吗？T3】

当我在 1999 年末开始接触 Web 编程时，Perl 是最常用的语言——它被积极开发并安装在几乎所有的 Web 服务器上。通过为其他公告板编写修改，我变得非常熟悉 Perl，所以当我开始 Ikonboard 时，继续使用 Perl 是很自然的。我应该注意到，当时几乎所有主要的公告栏都在使用 Perl。UBB，现在的 UBB。线程)等。都是用 Perl 编写的，因为 PHP 还没有证明自己，在它的版本 3 中，它很笨拙，缺少一些使 Perl 更容易使用的强大功能。

在 Ikonboard 3 的开发过程中，PHP 迅速普及。新的公告板，比如 vBulletin，是用 PHP 编写的，吸引了很多人的兴趣，因为 PHP 更快，对服务器的占用更少(当编译成 Apache 时，大多数 PHP 安装都是这样)。Perl 开始看起来有点过时，我意识到如果 Ikonboard 还想竞争，它必须过渡到 PHP。我很早就看到了这些迹象——Perl 受到了负面报道，开发似乎停滞不前，mod_perl 的设置和编码是如此痛苦，对于一般的网站管理员来说它不是一个很好的选择，尽管 Ikonboard 3 与 mod_perl 完全兼容，但它并不经常在 mod_perl 下运行。PHP 绝对是未来。

2002 年初，Ikonboard 3 发布后，我订购了几本 PHP 书籍，开始学习这门语言。PHP 在很多方面都像 Perl，很快就学会了，我开始学习全新的 Ikonboard PHP(代码名为 Aurora)。不幸的是，在我任职 Ikonboard 期间，Ikonboard PHP 从未问世。当我离开这个项目时，我归还了所有的 Aurora 文件和开发笔记。

所以当我开始开发 Invision Board 时，PHP 似乎是自然的选择。它宽容的天性和快速的代码开发让我在短短的几个月内构建了 Invision Board 的基本结构。我不必担心 mod_perl、fast_cgi 或任何这类无聊的东西，我可以只编写 PHP 代码而不用担心(太多！)关于跨平台的问题——这就像是一股新鲜空气。我早期的 PHP 代码是从“Perlish”的角度编写的，以至于我将自己的风格命名为“PHPERL”。在使用 PHP 大约 5 个月之后，我觉得我对它就像 12 个月后使用 Perl 一样得心应手。

当你创建 Ibforums 时，你是否因为与 JEG 的合同而面临任何法律问题？你使用相同的程序代码，当然转换成 PHP，来启动 Ibforums 吗？T3】

我和 JEG 签的合同非常宽松。没有竞业禁止条款，也没有对我未来开发的任何作品提出索赔。JEG 仅获得 Ikonboard 及其所有权利；他们没有也不可能获得我或我的编码风格。请原谅我们的类比，如果你买了一幅梵高的画，雇佣梵高为你再画几幅，然后他离开了，你没有任何法律依据强迫他改变风格，仅仅因为他为你工作，你买了他的一幅画。

我有自己的工作方式和编码风格，这很自然地被带到了 Invision Board 上。Invision Board 开发很快，因为这是我在 18 个月内创作的第三个公告板脚本(Ikonboard 1/2 和 Ikonboard 3 完全不同)。毫无疑问，Ikonboard 的一些结构和方法在 Invision Board 中，但那是因为这是我的风格和我编码的方式。Invision Board 已经推出好几个月了，我知道 JEG 已经检查了 Invision Board 代码是否侵犯了版权。因为我没有听到他们的任何消息，所以可以认为 JEG 对 Invision Board 没有意见。

***Ibforums 和 Ikonboard 有什么相似之处吗，比如编码结构或者编程指南？您计划在 Ibforums 中实现或者已经实现了哪些新特性？**T3】*

几乎所有的公告板在基本层面上都是一样的。大多数允许注册、登录、搜索、帮助、发帖、审核和管理——不同的只是实现方式。Ikonboard 是用 Perl 编写的，而 Invision Board 是用 PHP 编写的——仅仅因为这个原因，就不可能有任何直接的代码相似之处。PHP 是一种脚本语言，它有预设数量的工具来完成一项工作；这是否意味着所有的脚本都是一样的，因为它们使用这些工具？很明显，答案是否定的，比如 PHP 只有一种连接 MySQL 数据库的方式；这意味着所有用 PHP 编写的使用 MySQL 数据库的公告板都必须共享几乎完全相同的一行代码——你怎么能获得版权呢？

一个产品是其所有部分的总和，如果你开始将它分解成例程和代码结构，你会发现它和任何其他 PHP 脚本都是一样的——当编码者被限制使用脚本语言提供的有限工具时，它不会有很大的不同。当我离开时，Aurora 有点乱，它的最终功能集还没有确定。Invision Board 允许我重新开始，我在 Ikonboard 3 开发期间使用 MySQL 获得的经验意味着我这次可以编写一个更高效的程序。Ikonboard 的 MySQL 数据库结构非常混乱，因为 iDatabase(一个专有的数据库抽象层)产生了自我强加的限制。

虽然 Invision Board 可能是从 Ikonboard 播下的种子开始的，但它已经发展成为一个完全不同的产品，有自己的优点。Invision Board 的新功能都是基于可用性的。我觉得我们已经达到了功能饱和的程度，这让最终用户有点冷漠和困惑。我正试图开发一个易于使用的产品，它具有坚实、强大的管理和调节功能，同时为最终用户保留良好的可用性。

YaBB 的创造者 Zef Hemel 现在正在帮助 Ibforums。你能告诉我们你是如何注册 Zef 来帮助 Ibforums 社区的吗？T3】

当我在 Ikonboard 的时候，Zef 是编码论坛上一个非常活跃和有能力的张贴者——当需要的时候，他总是愿意帮忙并提供他广泛的建议。他的名字是由一名团队成员作为潜在的主持人提出的。经过简短的讨论后，大家决定，由于他在该论坛的辛勤工作，他完全应该得到一个正式职位。我并不关心泽夫的其他活动，也不关心他创办了我们当时的竞争对手之一 YaBB，因为我非常了解泽夫，相信他会一直保持客观。当我们离开 Ikonboard 创办 Invision Board 时，Zeff 紧随其后，他的努力工作仍在继续。他是这个队的重要资产。

在这一点上，我真的不能说未来会怎样。Invision Board v1.1 可能是我自己进行的最后一个开发周期，因为所有未来版本都将在一个团队中开发。我仍然会非常积极地参与电路板的编码工作，但这会让我腾出一些时间来做我们正在进行的其他项目，比如我们处于非常早期阶段的 CMS/Portal 混合插件。如果 Zef 想参与编码级别的工作，他肯定会在潜在候选人的短名单上。

你目前是否面临着来自其他论坛的挑战，尤其是现在 Ikonboard 是你的主要竞争对手？T3】

我不得不说，Ikonboard 是一个很难效仿的表演！但是我一直认为竞争是一件好事，因为它带来了技术的快速发展。当你认为一个论坛脚本只不过是一个数据库和一堆 HTML 表单时，在 Web 技术进步之前，你没有更多的事情可以做，所以我们所能做的就是基于现有的想法，使它们更加用户友好。Ikonboard、Invision Board、UBB、vBulletin、phpBB 和 YaBB SE 都在“功能池”中引入了一些巧妙的小创意，我们都可以从中汲取灵感。如果只有一个论坛脚本存在，功能库就不会像现在这样活跃。

论坛脚本环境中竞争如何推动技术发展的一个很好的例子是不起眼的私人消息通知弹出窗口。如果我没记错的话，vBulletin 是第一个引入这种想法的人，当收到新的私人消息时，会弹出一个小的 JavaScript 对话框来通知您，这使得当您有新的私人消息时非常容易注意到。phpBB 采用了这个想法，并在此基础上创建了一个弹出浏览器窗口中的 HTML 页面。我接受了这个想法，并通过向 HTML 页面添加更多功能来进一步发展它——其他人很可能会进一步开发它，继续这个循环。Invision Board 面临的最大挑战是，你可以免费下载它，但这并不意味着它没有收费产品好。我觉得给时间，Invision Board 会证明这一点。

Netmag.co.uk 最近推荐 Ibforums 为最好的免费论坛之一。这有助于从其他论坛获得更多的人气和转化吗？T3】

目前，你们还提供付费服务，包括付费支持、安装和版权移除附加服务。这些服务需求量大吗？您的付费支持与您通过 Ibforums 获得的普通支持有何不同？T3】

Invision Board 是免费的，因此为了确保 Invision Board 的未来，我们推出了这些付费选项和独立于母公司 Invision Power Services 的服务。我们的支持团队昼夜不停地处理免费和优先支持票证。我们承诺在 24 小时内回复所有免费票，在 12 小时内回复优先票——在几乎所有情况下，我们都远远超过这些时间。我们的优先帐户持有人还受益于直接电话支持和访问公共支持委员会内的私人支持论坛，以请求支持或只是获得迅速回答的问题。

母公司 Invision Power Services 还提供论坛之外的其他服务，如托管和 Web 开发。这些与论坛有什么联系吗？这些服务中的任何一个会增加你的其他服务的流量吗？你能告诉我们更多关于你的“托管 Ibforums”服务吗？T3】

Invision Board 只是 Invision Power Services 业务的一个部门。我们的主要服务包括网站设计、定制脚本和托管(从论坛托管、虚拟托管到专用托管)。我们的论坛托管包很受欢迎。与我们的一些竞争对手不同，我们的产品包是“真正的”托管:您可以获得自己的网络空间分配、自己的 vision 板、自己的数据库和 FTP 访问，而不仅仅是一个为多个安装提供动力的“定制”vision 板。你可以自由地修改代码，做几乎所有你期望在常规虚拟主机计划中做的事情。我们拥有的论坛托管服务器全部针对 Invision Board 进行了全面优化，为 Board 提供了最佳环境。

Invision Board 和 Invision Power Services 之间的关系非常密切。Invision Board 由 Invision Power Services 产生的收入维持，并且 Invision Power Services 能够接触到大量受众，因为 Invision Board 产生了流量。

有计划在未来将 Ibforums 转换成 Perl 或者用 Perl 编写吗？T3】

我们曾考虑将 Invision Board PHP 移植到 Perl 版本，但在对社区进行投票后，我们基本上决定不这么做。12 个月前，当 Web 主机还在使用 PHP 时，可能就需要它了，但是现在 PHP 似乎已经达到了 Perl 的水平，因为 99%的 Web 主机已经安装了它并且可以免费使用。事实上，我敢打赌，支持 PHP 和 MySQL 的主机比那些花费时间和精力为 MySQL 安装 Perl 接口(DBI 模块)的主机要多。我们确实有一个“Lite”的视觉板在工作中，但这是在缓慢的时间开发，它的发布将在很大程度上取决于市场的状态。当 Perl 6 发布时，我们将重新审视对 Perl 版本的需求。

***有没有做过什么基准测试来比较 Ibforums 和其他脚本？不久前，为了比较 Ikonboard 和其他脚本进行了基准测试——你能告诉我们一些细节吗？**T3】*

我尽量远离纯粹的基准测试。仅仅为了速度而设计程序是愚蠢的。一个好的设计应该把安全性放在第一位，可用性放在第二位，效率放在第三位——原始速度应该接近底部。让我们面对它，0.03 和 0.1 之间的差异是你眨眼的时间，我们真的不能区分这些措施。我用 Invision Board、phpBB2、YaBB SE 和 vBulletin 运行了一些个人基准测试，理论上 Invision Board 更快，但从“人”的角度来看，没有太大优势。

当然，我接触 Invision Board 的目的并不是让它成为该领域最快的主板。当我在 Ikonboard PHP (Aurora)上工作时，David 根据我们完成的内容(阅读、注册、发布等)对其他板进行了一些测试。测试结果显示，在他的机器上，Aurora 速度更快，效率更高。贾维斯娱乐集团认为宣传这些数字是很好的公关手段。但是 Aurora 的最终版本是否会超越它的竞争对手可能永远不会知道。

Invision Board 是一款不断发展的产品。我们从社区中获得了一些好的反馈和特色想法，如果这些建议具有足够广泛的吸引力并且可行，那么几乎所有这些建议都会被考虑纳入。

Invision Board 的未来版本将主要关注优化和效率。我们也正在完成更多的数据库驱动程序，包括 MS-SQL 和 pgSQL。我们有几个保守秘密的想法，试图将 Invision Board 带入不同的市场，但这完全属于“明年”的范畴。

我将永远和董事会联系在一起。版本 1.1 可能是我自己开发的最后一个版本，因为我还有一些其他项目，我想为 Invision Power 服务做准备。话虽如此，但 Invision Board 总是会受到重视，我仍然对其开发和代码结构有很大的发言权——我只是觉得它已经到了主要核心已经开发和稳定的阶段，近期的未来版本将主要是服务版本。
