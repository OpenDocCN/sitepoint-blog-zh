# Mozilla 将 Firefox Home 提交到苹果 iPhone 应用商店

> 原文:[https://www.sitepoint.com/mozilla-firefox-apple-iphone/](https://www.sitepoint.com/mozilla-firefox-apple-iphone/)

继 Opera 的 iPhone 发布成功之后，该轮到 Mozilla 协商苹果复杂的验收授权了。

苹果公司为他们神圣的设备制定了严格的应用程序开发规则。这里列举的太多了，但本质上，你必须使用苹果的工具，坚持使用已发布的 API，不覆盖默认应用程序，不提供语言解释器，不创建软件模拟器，并按照要求去做。你不能创建一个与之竞争的浏览器，但是 Opera 被接受了，因为:

1.  他们通过代理服务器路由流量，代理服务器将压缩的页面内容返回给设备——没有本地 JavaScript 引擎
2.  虽然它更快，但 Opera 不如 Safari 无缝，也更受限制
3.  Opera 遵守规则，[公布了他们浏览器的提交文件](https://www.sitepoint.com/opera-apple-iphone-app-store/)。苹果没有合理的理由拒绝这款浏览器，并且知道如果拒绝的话会引起轩然大波。

Mozilla 采取了不同的角度。根据他们的常见问题:

> 我们没有为 iPhone 推出 Firefox 浏览器的计划。由于操作系统环境和发行版的限制，我们无法为 iPhone 用户提供 Firefox。Firefox 也适用于 Maemo 和 Android 等其他移动平台。

相反，该组织创建了一个名为 [Firefox Home](http://www.mozilla.com/en-US/mobile/home/) 的应用程序，让你可以访问桌面浏览器的历史记录、书签和打开 iPhone 上的标签。实际上，它是 Firefox Sync(以前称为 Weave Sync)的单向版本，作为一个[插件](https://addons.mozilla.org/en-US/firefox/addon/10868/)提供，但最终将集成到浏览器中。

Firefox 主页浏览器使用原生 iPhone Safari 浏览器。不过，它还提供了“牛逼吧”*(我还是很讨厌那个名字！)*这样你只需几个按键就能访问常用网站。

从表面上看，苹果没有理由拒绝 Firefox Home。这款应用可以说服用户坚持使用 Firefox，而不是在家用电脑上尝试 Safari，但我怀疑苹果会担心这一点。也就是说，苹果拒绝应用的原因更加神秘。
