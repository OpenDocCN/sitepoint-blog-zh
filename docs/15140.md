# 一炮而红！通过流量分析衡量成功

> 原文:[https://www.sitepoint.com/success-traffic-analysis/](https://www.sitepoint.com/success-traffic-analysis/)

网站管理最重要的一个方面就是流量分析。如果你不知道你的访问者来自哪里，人数是多少，你就不能有效地推广你的网站，或者衡量任何当前推广努力的效果。

检查网站的统计数据应该是每天的活动，如果你还没有这样做，现在是时候开始了！

##### 交通行话

有一些混乱的不同术语用来描述网站流量。这些术语的误用经常会导致误解，所以知道正确的单词和概念很重要..您会发现最常见的术语包括:

***击中***

向您的服务器发出 HTTP 请求。“Hit”通常用来描述印象，这是不正确的。向您的服务器发出的请求不仅针对每个 HTML 文件，还针对每个图像、每个电影以及每个包含的 javascript 或 css 文件。如果您使用框架，那么一个实际的页面视图可能会导致多次点击，因为多个文件组成了一个页面。在每次请求时，服务器在其日志文件中记录另一个条目，因此当日志分析程序读取这些文件时，它们将报告总命中数。人们经常认为这是总浏览量，他们不必要地兴奋起来——不要陷入同样的陷阱。

***印象**T3】*

页面视图。当有人浏览你的 HTML 页面时，就会产生一种印象。如果您使用框架，您应该只计算您的主要内容页面上的印象，而不是那些用于菜单或标题框架的页面上的印象。另一种看待这个问题的方式是只计算显示广告的页面上的印象。

***独特的***

在 24 小时内被唯一的人访问的页面。唯一性通常是通过识别使用你的网站的每个访问者的 IP 地址来衡量的。然而，一些服务，特别是 AOL，通过代理服务器发送他们所有的成员，所以成千上万的人可以共享同一个 IP 地址。这通常意味着，如果您通过查看唯一 IP 地址的印象来记录唯一的数量，您的实际数量将略高于日志中报告的数量。衡量唯一性的一个更好的方法是由 IP 地址、浏览器或用户代理以及操作系统组成的复合唯一值。

***引荐者***

链接到您的网站的页面。这不一定是一个实际的页面:例如，它可以是搜索引擎的结果集。看看你的推荐人会告诉你谁链接到你的网站。

***用户代理***

这是指用于访问您的网站的软件。有时被称为“浏览器”或“客户端”，术语“用户代理”可以描述一个 PHP 脚本、一个类似 Internet Explorer 的浏览器或一个类似 GoogleBot 的搜索引擎蜘蛛。如果你能识别什么软件被用来访问你的网站，你就能知道用户是否滥用它，以及搜索引擎上次抓取你的页面是什么时候。

##### 计数器和跟踪器

在网络的早期，计数器相当流行。计数器是一个简单的脚本，它在文本文件或数据库中记录网站的访问者数量，然后在网站上以文本或图形方式显示总数。你仍然可以在一些业余网页上找到它们，但在大多数情况下，它们的用途已经消失了——主要是因为网站所有者想要关于他们流量的更复杂的信息，但也因为这些计数器已经被视为不专业。

现在大多数专业或商业网站都使用追踪软件。跟踪软件告诉你的不仅仅是访问者的数量——它可以按照日期、时间、浏览器、浏览的页面、推荐人和无数其他值来分解访问者的统计数据。追踪器之所以这样命名，是因为它们能或多或少地为你提供访问者浏览你网站的详细路径，所以它们不仅仅是计算你的流量，它们还能追踪你的网站。您可以从三种主要类型的跟踪软件中进行选择，让我们来看看您的选择。

##### 跟踪软件的三种风格

***1。远程跟踪服务***

最容易安装的跟踪类型，也是最受欢迎的，是远程跟踪。这些跟踪服务将所有的流量记录脚本和报告存放在它们自己的服务器上，你可以登录这些服务器来检查你的统计数据。录制本身是通过页面上的 javascript 完成的。

尽管它们易于使用，但由于各种原因，这种类型的服务是最差的。通常，这是不准确的:因为流量记录依赖于远程服务器的连接(一个很可能陷入困境的服务器)，许多访问者可能不会被记录，因为请求只是超时。此外，该服务对 javascript 的依赖意味着它无法记录没有启用 javascript 的用户的访问。这是一个大问题——搜索引擎蜘蛛不使用 javascript，所以分析你的流量的一个关键好处(知道你什么时候被蜘蛛抓取了)被这些服务忽略了。此外，远程追踪器通常要求你在网站上放置一个按钮或图形，以换取免费使用他们的服务，这对大多数网站所有者来说并不理想。

所以尽量避免使用这些服务，除非您没有能力或专业知识在自己的服务器上执行任何类型的跟踪脚本。

示例:

*   [Hitbox](http://www.hitbox.com/)
*   [测位仪](http://www.sitemeter.com/)
*   [极值-DM](http://www.extreme-dm.com/)

***2。记录程序***

这是我首选的流量分析方法。日志程序是安装在服务器上的脚本，然后生成日志文件(在平面文件或数据库中)和报告。比起日志分析系统(下面讨论)，我更喜欢这种类型的程序，因为日志程序为网站所有者提供了更多的控制——您可以决定记录什么，不记录什么，并且只跟踪那些您想要跟踪的页面。

这样做的缺点是您必须维护您的日志文件，如果您的站点很受欢迎，它们会变得相当大。在我的一个网站上(一个月记录超过一百万次展示)，日志文件每天增长大约 15mb，所以我通常每 3 天轮换一次。现在，如果你使用一个日志分析程序，你仍然会遇到大的日志文件，但是这些是你的服务器的日志文件，因此它们会自动为你旋转和维护。

这类程序的另一个附加功能是，你有时可以用它们来跟踪来自你网站的链接，这样你就可以准确地确定你在链接交换中发送了多少流量。

示例:

*   [http://www.xav.com/scripts/axs/](http://www.xav.com/scripts/axs/)(推荐)
*   [http://www.ekstreme.com/phplabs/phpcounter.php](http://www.ekstreme.com/phplabs/phpcounter.php)

***3。日志分析程序***

这些程序分析您的服务器日志，然后相应地创建流量报告。有些可能包括高级过滤器，允许您指定您想要报告的确切内容，但大多数只会报告日志文件中的所有内容—通常包括总点击量、显示次数和唯一性。当然，生成的报告的质量将取决于您实际使用的软件。

一些日志分析器是免费的，并预装在许多主机帐户上，而其他的可能要花很多钱。

示例:

*   [网络化器](http://www.webalizer.com/)
*   [网络趋势](http://www.webtrends.com/)

##### 每天做什么

一旦你设置好了追踪软件，你就可以开始使用它了，但是你实际上应该寻找什么呢？您应该每天检查以下各种事情:

***推荐人***

你应该每天检查的第一件事是你的推荐人。我从个人经验中知道，如果你有一个受欢迎的网站，你的推荐人可以数以千计，所以每天阅读这个列表可能是一件苦差事，但这是必须的！

当你查看你的推荐人时，要注意两件事:

*   访问者在搜索引擎的什么地方找到你的列表，以及
*   访问者在哪些网页上找到了指向您网站的链接。

具体来说，这将有助于你检查你是否保持你的搜索引擎位置，这也将有助于你识别链接到你的新网站。当我发现一个新网站链接到我的网站时，我会将他们的网站提交给谷歌，这样谷歌就可以抓取他们的网站，看到链接，并提高我的链接受欢迎程度。有些人会建议不要这样做，理由是将他人的网站提交给搜索引擎是不道德的，但我不同意。

过去，一些搜索引擎会禁止提交过多的网站，但是谷歌从来没有，现在也不会这样做。如果已经提交了页面，您的请求将被忽略。由于提交别人的页面不会有什么不好的结果，我不认为这种做法是不道德的，尤其是因为许多拥有这些页面的人可能不知道如何提交他们的网站。当然，在这个问题上你应该自己做决定。

***IP 地址和用户代理***

你需要检查的第二件事是你的访问者的 IP 地址和用户代理。这些信息会告诉你两件事:

1.  当搜索引擎抓取你的网站时。

3.  如果有人滥用你的网站。

第一点很重要，因为除非你知道你的站点何时被蜘蛛抓取，否则你不能有效地解决搜索引擎列表的问题(例如，如果它们过时了，或者根本不出现)。许多人会记得他们提交到搜索引擎的时间，但是如果你问他们什么时候被蜘蛛抓取，他们也不知道。知道搜索引擎什么时候爬行，什么时候更新，可以让你预测你的列表什么时候会改变。

第二点很重要，因为有很多人无事可做，他们有很多方法可以滥用网站。一种方法是编写一个脚本，从网站上抓取内容并显示在你自己的网站上。

例如，有些脚本从 CNN.com 这样的网站上抓取新闻标题。然后，网站所有者在他们自己的网站上显示标题，以及返回 CNN 的链接。虽然从技术上来说，复制他们的标题是错误的，但它很容易被大玩家原谅，因为网站所有者正在使用标题链接到他们(有效地将流量带回他们的网站)。

然而，编写一个从网站上窃取文章并自己显示出来的脚本也是一样容易的。如果你是这些不当行为的受害者，你通常可以通过你的日志来判断。通常会有大量来自其 IP 地址(应该解析为 Web 服务器)的请求，以及来自名为“PHP”、“Perl”或另一种脚本语言的用户代理的过多点击计数。有时人们会下载你的整个网站，然后在他们的服务器上重新发布，但是他们有时会忘记重新编码一些链接，导致从他们的网站到你的原始网站的点击。一位 SitePoint 论坛顾问最近通过密切监视他的推荐者发现了这一点。

关于下载整个网站的话题，也有网站破坏者。通常被亲切地命名为“离线浏览器”，就像一些特洛伊木马被命名为“远程管理工具”一样，这些程序可以用来下载您的整个站点，不仅窃取您的站点(设计、内容等。)但可能会使服务器崩溃或严重降低速度。根据您网站的大小，这些程序可以通过查看 IP 地址来检测-如果您看到来自一个地址的数百或数千个显示，很可能是这些程序之一。你也可以寻找他们的用户代理——一些比较流行的是 Wget、Teleport、HTTrack 和 Web Reaper。我应该提到，Wget 是一个在 unix 服务器上用来下载文件的有效程序，比如补丁或驱动程序。然而，除非你在你的网站上提供这样的下载，否则任何在你的网站上使用这个代理的人都可能是在偷窃。

另一种形式的网站滥用是从网站上获取电子邮件——如果你运营一个社区网站，这一点尤其重要，用户经常在那里张贴他们的电子邮件地址。就像网站破坏者一样，你可以通过他们的用户代理来识别邮件收割机。

滥用网站的最后一个方法是屏蔽网站的广告。有些人认为这是冲浪者的权利，然而，我觉得这是偷窃。网站管理员放置广告，期望用户将免费观看广告和他们观看的内容。如果访问者屏蔽了广告，那么从伦理上讲，我认为他们根本不应该访问这个网站。一些网站管理员会将使用广告拦截程序的人重定向到一个页面，要求他们为网站访问付费，这种方法反映了多少网站管理员的感受:你要么用钱包付费，要么用眼球付费。像前面提到的例子一样，这可以通过监视用户代理来检测。

一旦你确定了那些滥用你的网站的 IP 地址或用户代理，你可以禁止他们(使用。htaccess，如果您运行 Apache 的话)，但是对此的完整解释显然超出了本文的范围。

***其他统计信息***

除了上面提到的，你还可以从你的统计数据中收集到很多信息。当你试图推销广告或重新评估你的促销努力时，这些信息通常是有用的。

**人口统计**

你的服务器统计数据可以提供有限的人口统计信息，这些信息对设计你的网站和吸引广告商都有帮助。例如，通过研究操作系统或用户代理的统计数据，你可以知道你的访问者使用的是 PC 还是 Mac、Internet Explorer 还是 Netscape。一些软件还可以通过解析访问者的 IP 地址来提供地理统计数据。虽然这些统计数据不是最准确的(并不总是能够准确地识别用户的原籍国)，但这些信息在向潜在的广告商展示广告包时，甚至在您决定是否对您的网站进行区域性更改时(例如，添加第二种语言的内容)，仍然是有价值的。

**搜索引擎统计**

除了浏览你的推荐人以确保你在搜索引擎中的位置，你还可以偶尔做一个更详细的分析，比较你从各种搜索引擎中获得的流量。这可以帮助你确定是否有一个特定的引擎表现不佳。然后，你可以确定哪些推荐人是你需要努力的——以增加他们给你带来的流量(尽管你应该记住，感觉到的“较低的流量水平”可能是搜索引擎不如你跟踪的其他搜索引擎受欢迎的结果)。

**游客行为**

你偶尔也可以分析访问者的行为。例如，快速查看统计数据可以显示访问者进入和离开你的网站的页面，这反过来可以告诉你网站的哪些部分最受欢迎，哪些部分需要改进。

如果你的网站跨越多个主题，这种分析也可以帮助你确定你的用户最感兴趣的主题。例如，如果你评论 Mac 和 PC 硬件，并且大多数访问者阅读 Mac 评论，那么你可能会考虑更多地关注 Mac 部分，逐步淘汰 PC 信息(或者开发一个单独的网站)。但是这些信息不仅方便你参考，在与潜在的广告商打交道时也很有用。此外，如果你运行一个基于社区的网站，这些细节可以表明你可能有多少潜伏者，如果你运行一个基于文章的网站，统计数据可以表明哪些文章或作者是最受欢迎的。

**一段时间内的交通模式**

其他值得关注的好的统计数据是那些测量流量模式的数据。这些不仅可以表明你的网站获得最多流量的时间，还可以提供对你的受众的真实洞察——一个访问者在一段时间内使用的清晰图片可以暗示他们访问你的网站的原因。

例如，我注意到我的教育网站的流量与学年密切相关，所以在周末、假期和夏天，我的流量水平会下降。这些信息表明我的主要用户是学生，这让我可以相应地有针对性地投放广告。了解站点何时流量最大的另一个重要好处是，您可以在使用率最低的时候安排停机时间(用于升级和维护)。
