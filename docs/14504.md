# OS X 的终点站

> 原文：<https://www.sitepoint.com/the-terminal-in-os-x/>

终端实用程序是 Macintosh OS X 的一个重要特性，在 Panther 版本的操作系统中，它是 Bash shell 上的标准。Bash 是 Unix 变体(包括 Linux)中最常见和最流行的 shells 之一。

幸运的是，对于我们这些经常在操作系统之间迁移的人来说，当回到 Mac 时，Bash 的功能与其他任何地方都一样。

如果您使用 OS X 作为您的主工作站操作系统，您可以通过 shell 访问来访问本地和远程服务器，并从终端执行所有的系统管理和其他工作。然而，有一些 Mac 特有的区域可能会派上用场。

对于初学者来说,“终端”位于“实用工具”内的“应用程序”文件夹中。如果您使用终端在 OS X 系统上工作，有些目录结构会略有不同。例如，有几个目录实际上在/private 中有别名，比如/etc 和/var。要查看这些，您可以在终端中键入“man hier”进行概述。

此外，sudo(成为单个命令行操作的超级用户的能力)对于机器的 admin 用户是自动开启和启用的。实际上，要编辑一个特权文件，用户可以键入“sudo VI/private/etc/httpd/httpd . conf”，系统会提示用户输入管理员密码，用户会被临时提升为超级用户。

您还会发现，您在 Mac 上创建的文件都有相关的资源分支。关于资源分叉的解释很长，所以我不会在这里赘述——但是在维基百科上可以找到一个很好的概述。

通常情况下，这些都不是问题，即使从 Panther 开始跨平台，系统内置了对 Windows 友好附件的支持，微软的 Office 更有能力打包文档以实现互操作性。然而，从开发人员或管理员的角度来看，许多文件操作可能发生在终端中。Apple 已经包含了一些实用程序来适应资源分叉。

例如，可以使用 cp 和 mv 等传统的 shell 程序来处理文件和目录。但是，这些文件将在没有资源分支的情况下被复制或移动，如果以后被恢复和访问，可能会丢失数据或重要的文件信息。

同上，CpMac 和 MvMac 解决了这些技术问题。这三者的工作方式很像 cp 和 mv，因为它们将文件或目录复制或移动到新的位置。Ditto 有一个方便的选项，通过使用“ditto-rsrc source file destination file”来保留资源分支。CpMac 和 MvMac 在使用时会自动保留元数据和分叉。一个限制是 MvMac 不能跨文件系统和卷工作，只能作为主卷挂载。

出于备份的目的，ditto 或 CpMac 可以很好地工作，保留重要的资源数据，同时将文件和目录移动到远程文件系统或外部卷(如 cd-rw 或硬盘驱动器)上。
