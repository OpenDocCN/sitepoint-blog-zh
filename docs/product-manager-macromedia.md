# 采访–Bob Regan，Macromedia 辅助功能产品经理

> 原文：<https://www.sitepoint.com/product-manager-macromedia/>

Bob Regan 是 Macromedia 的辅助功能产品经理，他创作了一系列引人注目的 Web 开发、图形和动画产品，包括 Dreamweaver、Flash 和 Director。

我最近与他交谈，了解 Macromedia 在可访问性方面的进展。

##### 背景

***你是如何第一次亲身参与到无障碍环境中的？***

我在威斯康星大学读博士生的时候，负责教职员工的网页设计指导。1999 年底，该大学通过了一项关于网络无障碍的政策。那时，我不仅要自己学习无障碍，还要把它教给 UW 的老师们。

我参与这个问题越多，就越发现它与我作为一名教师和一名研究人员的兴趣相吻合。可访问性将技术与包容性问题结合在一起，我觉得这种方式非常有吸引力。我很快改变了我的研究领域，以配合我在无障碍方面所做的工作。

同时，我对可访问性采取了一种脚踏实地的方法。我没有讲授标准的技术方面，而是重写了我们为 Macromedia 使用的培训材料。和我一起工作的大多数教职员工都在发布或维护网站，但他们甚至不知道 HTML。W3C 指南的技术要求对于教师来说通常很难理解。所以我把这个概念整合到我们之前提供的非技术培训中。这使得可访问性成为我们许多教师更容易理解的话题。

***您在 Macromedia 参与辅助功能有多久了？***

快两年了。Macromedia 的代表参观了 UW，并了解了我们在辅助功能方面的工作。几周后我被录用了。

Macromedia 何时开始关注其产品的可访问性？

当我第一次加入 Macromedia 时，我了解到的一件令人惊奇的事情是，关于可访问性的想法是多么普遍。Dreamweaver 团队从 Dreamweaver 3 开始就一直在考虑可访问性。当时的产品经理是一位名叫苏珊·莫罗的女性，她写了一份宣言，从社会和经济角度概述了可访问性的重要性。

就在那之后，有一个集团在一次收购后被短暂闲置。为了在那里开始开发新产品之前填补空闲时间，产品经理要求他们为 Dreamweaver 构建一个插件，该插件类似于可访问性验证工具 Bobby。这是同类插件中的第一个，从那以后，每个主要的验证公司都为创作工具创作了自己的版本。

然后，我遇到了一些人，他们正在努力推进这个问题，没有任何宣传或讨论。HomeSite 产品经理参加了在国家无障碍媒体中心举行的关于无障碍的会议。从那时起，他决定 HomeSite 应该是一个可访问的产品。

直到美国联邦政府网站的 508 条款，Macromedia 才决定雇人来协调这个问题。我是在 508 条款生效前加入的。

##### 动机

***Macromedia 致力于提高可访问性的驱动力是什么？***

首先，可访问性是应该做的事情。这是一个由有社会意识和积极的个人组成的公司。可访问性产生了一种对技术及其潜力的独特热情，使我们的社会成为一个更加开放和公平的地方。

与此同时，如果没有强大的商业模式，我们无法证明我们在可访问性方面的工作是正确的。随着美国、英国和世界各地的可访问性政策越来越多，我们努力使我们的工具在市场上最容易访问。当人们开始理解可访问性的重要性时，我们希望他们首先想到 Macromedia 工具。

***作为 Macromedia 的辅助功能产品经理，您如何看待 Macromedia 与辅助功能社区的整体关系？***

无障碍社区和残疾人社区是有区别的。它们重叠，但不是同义词。易访问性社区是一群参与易访问性标准、开发辅助技术并确保其他技术与辅助技术和标准互操作的人。这些人包括 W3C 的网页可访问性倡议，英国的电子特使办公室，GW Micro 这样的工具制造商，以及像我一样在公司、大学和政府办公室工作的可访问性经理。

残疾人社区包括皇家国家盲人研究所、国家盲人联合会和皇家国家聋人研究所等团体。这些团体经常深入参与可访问性社区，但它们不是一回事。

毫无疑问，如果没有与可访问性和残疾人社区的紧密联系，Macromedia 不会像我们在可访问性领域一样成功。我们投入大量时间倾听合作伙伴的心声，并帮助构建可访问内容的示例。

***您与可访问性社区中的哪些群体进行了积极的对话？***

Macromedia 通过 W3C 的标准工作与可访问性社区以及当地政府和教育部门建立了牢固的关系。对我们来说，重要的是继续参与到这些工作中，将对标准的支持直接构建到我们的工具中，同时也在工作组中提出问题，帮助他们理解标准是如何构建到产品中的。我在 W3C 的创作工具可访问性指南工作组工作。我还经常与美国、英国、加拿大和澳大利亚政府部门负责执行标准的同事保持联系。

在残疾人社区中，我经常与美国国家盲人联合会、英国 RNIB、澳大利亚视觉协会、聋人电信协会和美国残疾人协会等团体联系。为了解决没有组织成大型伞式组织的残疾人的问题，我花了很多时间与我开始工作的威斯康星大学的研究人员在一起。

##### 当前产品–输出

***您如何提高产品的可及性？***

有三个衡量标准决定了我们工具的输出。首先，我们根据 Web 和可访问性标准来衡量工具的输出。其次，我们检查工具与辅助技术(如屏幕阅读器)的互操作性。第三，也是最重要的一点，我们寻求残疾客户的意见，以收集对我们工具的反馈。

***您主要关注哪些产品，您是如何实现这一目标的？***

我们试图将可访问性融入到每一件产品中。然而，我们因与 Dreamweaver、Flash 和 Contribute 的合作而闻名。

***到目前为止你取得了哪些进步？***

太好了，但是我们还没完呢！可访问性通常涉及一个复杂的构建过程，以支持操作系统中的可访问性 API，如 MSAA、辅助技术和标准。在一些情况下，该行业遇到了一些非常严峻的挑战。

我们致力于不断改进我们的工具，使其对残障人士来说更好、更易访问、更易使用。

您的产品是否有比 XHTML 和 CSS 等其他技术更容易获得的输出？

可访问性需要从广义上理解。对于盲人来说，最容易理解的内容形式是纯文本，可能是用 xhtml 和 css 标记的。然而，对于有认知障碍的人来说，最难理解的内容形式是纯文本。

为了使多种形式的内容真正可访问，它们需要是多模态的。想起一门数学课。想象一下，在没有图像的情况下，试图理解直径的概念以及与周长的关系。使用富媒体技术(如 Flash ),教师可以交互式地展示这一概念，这可能会使许多有或没有认知障碍的学生更容易理解这一概念。

对于失明的儿童来说，交互式 Flash 电影可能不是最容易理解的格式，但这并不意味着患有其他残疾的儿童不应该使用它。这只意味着它不应该是访问这些内容的唯一方式。

##### 当前产品–环境

***您目前的产品环境的可访问性如何？***

Dreamweaver 和 Contribute 都被设计为盲人可以轻松使用的工具。我们现在正与全国盲人联合会合作，为盲人开发者开发使用我们的工具学习 html 和网页设计的教程。

***您会为残障开发人员改善产品环境的可访问性吗？***

是的。无论何时何地。

##### 开发者社区

***您是如何在您的开发者社区中提高可访问性意识的？***

我们致力于将可访问性纳入我们所有的培训和文档材料中，我们在自己的会议中包括关于可访问性的会议，并且我们在各种讨论可访问性的其他会议中提供会议。

***到目前为止你有过哪些回应？***

非常非常积极。越来越多的设计师被要求将易访问性融入他们的规范中。我们能够以设计师熟悉的方式引入主题，并与他们现有的工作流程保持一致。

##### 新产品

***我相信你很快就会有一款叫做 Breeze 的新产品问世。那里的可达性如何？***

Breeze 是目前市场上同类产品中最容易使用的产品。我们致力于通过 Flash player 和键盘增加对屏幕阅读器的支持，确保残障人士也可以使用该解决方案，以便更轻松地控制电影。

***Macromedia Central 对 Flash 的辅助功能有影响吗？***

不是，但是 Flash 的可达性影响了 Central 的可达性。目前，中央是不可及的，但像我们所有的产品一样，这是一个优先事项。

无法访问 Central，因为它使用独立的 Flash player。目前，只有 ActiveX Flash 播放器是可访问的。

***无障碍会集成到所有新产品中吗？***

每个产品都包含一个可访问性计划。我们尝试考虑每个产品可能的用例，并尽可能地整合支持。显然，Fireworks 的计划不同于 Dreamweaver 的计划。然而，每个产品都有一个计划。

非常感谢 Bob 抽出时间完成这次采访。
