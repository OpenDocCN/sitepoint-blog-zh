# 对可及性的焦虑

> 原文：<https://www.sitepoint.com/the-angst-of-accessibility/>

在 web 开发的世界中，有许多事情是有争议的，有争议的，或者会激起热烈的反应。有些人很容易被遗留的 IE 支持激怒，有些人则被厂商对 T2 CSS T3 的扩展是否是一个好主意所激怒；对一些人来说，你只需要提到 <abbr title="HyperText Markup Language Version 5">HTML5</abbr> 就能听到愤怒的抱怨的长篇大论！

但是没有一个主题能引起和可及性一样的反应。

我见过开发人员对 101 个不同的主题的“如何”、“什么”和“何处”变得各种各样，但是只有在对可访问性的反应中，我见过对仅仅是建议的愤怒。

所以继续我之前的帖子，[](https://www.sitepoint.com/the-art-of-accessibility/)可及性的艺术，我想考虑一下——可及性到底是什么让一些人如此愤怒？为什么一个如此清晰地植根于改善用户体验的问题会招致任何反对，更不用说如此强烈的反对了？

让我们检查几种可能性。

## 我们都有工作要做

我不能否认我是一个工作狂，我有时间成为工作狂——我可以用额外的时间学习新技能，并通过个人项目和测试案例来实践我已经掌握的技能。但是我们的行业充满了没有这种自由的人，他们受时间和资源的限制太多，根本不会花时间在那些对他们的主要用户群没有直接好处的事情上。

我认为，提供良好的可访问性是我们职业精神的一个基本部分。这是我们工作的一个重要部分，如果你不能或者不愿意给予它所需要的关注，那么你就不配称自己为专业开发人员。但是你怎么对一个有家庭和孩子要抚养的人说这些呢？当人们只是想有一份工作并供养他们的家属时，像我这样的态度会让他们感到被冒犯，这有什么奇怪的吗？

尤其是当结果看起来纯粹是偶然发现的时候！良好的可访问性的主要结果是为残疾人提供更好的用户体验，他们使用访问技术进行浏览，或者有特定的交互或认知限制。但是有这样的人访问你的网站吗？可能是的，但说实话，你并不真的知道。那么，为什么要花费时间和精力去迎合一群可能根本不存在的用户呢？

我确实有一个答案，我们稍后会回到这个问题。现在，我们可以理解为，被提及的问题*是*让许多兼职开发人员懊恼的来源，并且最终会激起愤怒的回应。

但是这样的回应不仅仅来自这个群体，他们来自各种情况下的各种开发者。所以这不可能是全部。

## 愤怒滋生愤怒

毫无疑问，在某些情况下，愤怒的回应是由愤怒的提议引起的。那些在可访问性上投入最多，也最大声支持可访问性的开发人员，往往会因为其他人缺乏兴趣而感到沮丧。一旦你自己决定某事是重要的，对那些不同意你观点的人感到沮丧是可以理解的。

因此，我们可以看到，可访问性的强烈支持者会对根本不准备考虑它的开发人员感到愤怒；[我知道我有](http://www.brothercake.com/site/resources/reference/hope/)。但这并不能真正原谅它，因为愤怒会滋生愤怒，如果你对一个话题过于激动，你将不可避免地激起其他人的类似反应。人们会生你的气，因为你生他们的气。

然而，即使这个提议没有这些诱惑，愤怒的回应还是会出现；我之前关于这个话题的帖子没有任何敌意或傲慢，但它仍然激起了那种反应。显然，这也不是故事的全部。

不，我认为真正的答案要深刻得多——直击我们所认为的公平的核心。

## 公平是什么意思？

政客们臭名昭著，因为他们说的话听起来很好，却没有给出任何明确的含义——到底什么是公平和不公平？事实上，它们在不同的时间有不同的含义，因为有两种公平。

第一种公平是<dfn>平等待人</dfn>。彩票就是这种公平的一个例子——每个参加的人都有相同的中奖机会。

第二种公平是<dfn>不平等地对待人们以获得平等的结果</dfn>。大多数国家的所得税是这第二种公平的一个例子——随着你收入的增加，你支付的不仅仅是更大的*金额*的税，而是更大的*比例*的收入税。

迎合可访问性意味着在少数群体的需求上花费额外的时间和精力，这样做是因为该少数群体本来就处于非常不利的地位。尽管无障碍环境并不特别耗时，但它确实比受益者的比例需要更多的时间。

所以可及性是第二种公平的一个例子——不平等地对待人们以获得平等的结果——这在其他情况下被称为<dfn>积极的区别对待。</dfn>

## 积极的区别对待有多积极？

在我看来，一些人反对迎合可访问性的最重要原因是**这相当于歧视**，他们认为这不公平。

从广义上讲，我不赞成积极的区别对待，因为我不认为通过手段来达到目的是正当的。我认为在我们的社会中有许多不平等现象，这些不平等现象开始时是善意的，试图消除不平衡，最终却造成了同样不公平的反不平衡。

但是特别是网络，一般来说是 IT，是一个特例——因为 T2 技术是可访问性的最大希望。

它不像物理世界，在那里有好的，有形的原因，为什么有些东西永远无法接近。一个盲人永远无法手动驾驶汽车；坐在轮椅上的人将永远无法登上古老的石头大教堂的台阶。但是技术不像物理世界——技术可以采取任何形式；技术是我们的奴隶，我们可以让它做我们想做的事情。

对于任何孤立的人来说，比如因为残疾、地理位置、精神或身体疾病，或者需要照顾他人，互联网是一条生命线。这是我们的工作，我们的责任，保持生命线的强大，因为没有它，一些人将与世界完全没有联系。

这就是可访问性很重要的原因，也是为什么——在我看来——在做得更好上花费过多的时间和精力是必要和合理的。但我也理解为什么这对许多开发者来说如此困难或不切实际。

我认为易访问性倡导者能做的最好的事情是从他们的肥皂箱中走出来，停止如此清教徒式的说教，并接受许多开发者根本不能满足他们的期望。零工开发人员能做的最好的事情就是试着意识到这些问题，并专注于我们都能做的快速简单的事情来提高可访问性——为图像编写有意义的`alt`文本；合乎逻辑地使用标题，着眼于标题创造的整体结构；不要用 JavaScript 生成主要内容。

但最重要的是，每个人都可以做的是避免用更多的坏态度来回应坏态度——这只会增加空气中坏态度的总量，这对任何人都没有好处。
