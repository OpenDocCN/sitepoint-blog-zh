# 管理外包人才的正确和错误方式

> 原文：<https://www.sitepoint.com/the-right-and-wrong-way-to-manage-outsourced-talent/>

我和美国的两个开发团队一起工作。两者都利用离岸人才。这两家公司对离岸人才收取较低的费用，而对他们自己作为项目经理的时间收取较高的费用。他们的时间大约是每周 1500 美元，而离岸资源是每周 300 美元。

因此，作为一名顾客，我的动机是尽可能砍掉昂贵的人才。

在一种情况下，这很容易做到。这是因为项目经理除了把我对网站的评论传递给海外人才之外，没有做更多的事情。没有增值建议、架构开发等。如果有的话，直接沟通会更有效率和效果，所以我甚至可以说项目经理是在消耗我的价值，而不是增加它。事实上，我向一个我认识的离岸资源投标完成这个项目，他的出价是我完成这个项目目前结构的 1/10。

在第二种情况下，项目经理解决问题，提供业务和技术见解，撤换表现不佳的人，等等。他确实提供了价值。即使在他的情况下，最终项目应该自我维持，这样他就不再被需要，或者他的角色可以减少。但他知道这一点，并通过开发一个非技术人员可以管理的平台，作为他的模型的一部分，帮助我们达到这一点。而且，如果我们的业务起飞，我们将需要增加他的角色，以继续建立产品。这似乎是建立和管理团队的正确方式。

我认为对抗外包的关键是服务于这个项目管理角色。但你必须确保你不仅仅是双方(客户和离岸资源)之间的一个来回管道。你必须真正提供一些价值，否则你将被淘汰。
