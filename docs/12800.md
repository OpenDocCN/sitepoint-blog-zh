# 微软社区项目:谷歌剧本中的一页

> 原文:[https://www . site point . com/Microsoft-community-projects-a-page-from-Google-playbook/](https://www.sitepoint.com/microsoft-community-projects-a-page-from-googles-playbook/)

众所周知，谷歌鼓励工程师花 20%的时间做兼职项目。根据谷歌工作页面的说法，空闲时间是谷歌员工的士气助推器，确保他们有时间“做他们真正热爱的事情”。谷歌的 20%项目已经产生了一些惊人的成果，比如 Orkut、内容 Adsense，以及最著名的 [Gmail](http://www.google.com/press/pressrel/gmail.html) 。

微软的 [Office Labs](http://www.officelabs.com/) 借鉴了谷歌的剧本，允许任何在微软工作的人向实验室团队提交想法进行原型制作。“[Office Labs]是来自在微软任何地方工作的正式员工和实习生的想法的试验场，”研究部门的公司说，该公司为 web 应用程序、桌面应用程序和软件+服务混合产品制作产品原型。

昨天，微软推出了第一个社区项目，一个名为 [Speed Launch](http://www.officelabs.com/projects/speedlaunch/Pages/default.aspx) 的可下载应用程序和网页启动器，可用于 Windows Vista 和 Windows XP。

![](../Images/5883e645081e6330a2c6eec754f7c6bb.png "msft-speed-launch")

在这个早期阶段，软件本身并不十分引人注目——例如，在我看来，来自 Humanized 的 [Enso Launcher](http://www.humanized.com/enso/) 使它相形见绌——但这并不是它有趣的地方。《极速发射》之所以有趣，是因为它的幕后黑手是谁。据《西雅图邮报》记者的[托德·毕晓普称，发射器应用程序的想法源于马特·戴尔。在为公司开发软件的微软员工中，戴尔是独一无二的，因为他不是一名开发人员，而是一名专利律师。](http://seattlepi.nwsource.com/business/373547_msftoffice05.html)

与在微软 IT 部门工作的 David Craig 一起，两人在过去一年的业余时间开发了 Speed Launch，然后将它带到 Office Labs 团队进行进一步完善。在大公司的软件开发中也是独一无二的？Dyor 和 Craig 从来没有见过面——一个在华盛顿州的雷德蒙工作，而另一个在德克萨斯州工作。所有的开发都是远程完成的。

与谷歌不同，微软不会让大多数员工在上班时间从事兼职项目——这就是为什么 Dyor 和 Craig 在午餐和下班后的业余时间开发了 Speed Launch。但是由非工程师雇员通过 Office Labs 团队鼓励这样的社区项目是朝着这个方向迈出的一步。
