# 给初学者的方便的 VI 编辑器提示

> 原文：<https://www.sitepoint.com/handy-vi-editor-tips-for-the-beginner/>

当我在 20 世纪 90 年代中期第一次接触 Unix 命令行时，我不得不承认我被吓到了。我已经使用 DOS 有一段时间了，但仅仅是在日常操作的意义上。

突然，我面临着学习从 shell 提示符下管理整个系统的问题。特别是，当我开始学习使用 VI 编辑器的细节时，事情变得更容易处理了。

VI 从命令行启动，特权用户和非特权用户都可以使用。在 web 开发领域，使用基于屏幕的编辑器(如 VI)可以方便地编辑 httpd.conf 文件、修改防火墙规则、快速编辑配置文件，当然还可以处理 html 或脚本页面(如 Perl、PHP 等)。

有些人发现 VI 比可视化编辑器更有效。我倾向于两者都用，然而，VI 功能齐全，可以用于完整的编辑过程，从创建新文件到后来的编辑。对于需要大量编辑的高级文件，我仍然依靠 OS X 上的 BBEdit 文本编辑器。然而，在终端上点击一下进行本地或远程编辑是很方便的！

VI 有两种模式，命令和插入。命令模式允许命令输入在文件中移动和搜索文本。插入模式允许完整的编辑控制和输入键盘输入到文件中。我已经包括了一些我在编辑文件时最常用的命令。

让我们看看 httpd.conf 的例子。由于这是一个受限文件，如果您以非特权用户身份打开 httpd.conf，它将处于只读模式。因此，对于编辑，您必须拥有 root 权限(或者您为此目的设置的任何用户)。

要打开这个文件，发出‘VI/etc/httpd/conf/httpd . conf’就可以了。一旦在文件中，有一些工具可以使用。

在命令模式下，键入会输入命令。例如，使用/告诉 VI 期望搜索某个东西。因此，要找到文件中与 SSL 动态可加载模块相关的所有条目，我可以输入“/HAVE_SSL”并按回车键。

VI 转到该搜索词的第一个匹配项。要移动到下一个条目并继续浏览文件，只需按 N 键(代表下一个)。

要移动到特定的行号，我可以使用冒号字符并输入一个数字，比如:900——这将移动到文件的第 900 行。这在追踪引用文件中行号的错误时非常有用。

如果您需要删除字符串(单词)、行或段落，有一些方便的删除文本的命令。以下是一些常见的删除按键:

*   删除当前单词–> dw
*   删除前面的单词–> db
*   删除整行–> DD
*   删除段落–> d }

删除的一个好处是，你可以用它来剪切和粘贴文本。例如，如果您删除了一个段落，并希望将其从文件的第 10 行移到第 1500 行，您可以执行以下操作:

将光标放在段落前面，然后键入 d}。文本消失。然后键入:1500，这将使您转到文件中的第 1500 行。只需输入“p”就可以将删除的文本粘贴到新的位置。

如果有必要进行更改，比如在 httpd.conf 文件中添加一个新的虚拟服务器条目，则需要切换到插入模式。这可以通过按键盘上的 Insert 键或 I 键来完成。

现在任何键盘输入都被添加到文件中，所以要小心。您可以使用箭头键在文件中移动，在文件中键入输入，并像在可视文本编辑器中一样使用 delete 或 backspace 键。要在完成更改后退出插入模式，请按下 escape 键(esc ),然后返回命令模式。

如果希望保存更改并退出，请键入:wq 并按 enter 键。要退出而不保存更改(可能出现了错误)，请按:q！，这将覆盖在插入模式下所做的编辑。最后，要简单地退出而不做任何更改，请使用:q。
