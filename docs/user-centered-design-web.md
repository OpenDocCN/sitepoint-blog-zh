# 回顾——用户体验的要素:以用户为中心的网页设计

> 原文：<https://www.sitepoint.com/user-centered-design-web/>

视觉设计。可用性。信息架构。商业目标。功能规格。信息设计。交互设计。导航设计。

你是否在试图整理一大堆相互竞争(而且经常是相互冲突)的关于网页设计过程的最佳方法的想法时感到眩晕？你是否对那些开出宽泛和一般处方的作者感到沮丧，这些处方并不总是适用于你经常处理的特殊情况？你知道史蒂夫·克鲁格的《不要让我思考》的标题不适用于网页设计师吗？

如果你对以上任何一个问题的回答是肯定的，那么你将会欣赏杰西·詹姆斯·加勒特的《用户体验的要素:以用户为中心的网页设计》( New Riders Publishing and The American Institute of Graphic Arts，2003)作为对现有网页设计文献的一个有价值的贡献。

##### 作者的历史

杰西·詹姆斯·加勒特是旧金山用户体验咨询公司 Adaptive Path 的创始人之一。自 1995 年以来，他一直在互联网行业工作，被公认为是信息架构学科发展的主要贡献者。特别值得一提的是他的可视化词汇表，这是一个用于信息架构文档的开放式符号系统。

这本书建立在加勒特于 2000 年春天首次在他的网站([www.jjg.net](http://www.jjg.net))上发表的图表的基础上。这个图表直观地描述了不同的网页设计方法之间的关系，展示了它们是如何作为一个更大的整体，由用户体验的共同主线来约束的。在最终产品中，Garrett 为理解和处理以用户为中心的设计所涉及的问题提供了一个有凝聚力的总体框架。

##### 这本书的理念和内容

用户体验，定义为产品在现实世界中的行为和使用方式，对网站的成功至关重要。如果你的用户在使用你的网站时没有积极的体验，他们再次光临的可能性就会大大降低。认识到这个重要的事实，作者断言“用户体验的一切都应该是你有意识的决定的结果。”因此，

用户体验开发过程就是要确保用户对你的网站的体验没有任何方面是没有你有意识的、明确的意图的。这意味着要考虑用户可能采取的每一个行动的每一种可能性，并理解用户在整个过程中每一步的期望。”

虽然这最初看起来是一项艰巨的任务，但 Garrett 通过将网页设计过程分为五个“平面”来帮助我们思考影响用户体验的各种问题:

*   策略(网站目标和用户需求)
*   范围(功能规格和内容要求)
*   结构(交互设计和信息架构)
*   框架(界面设计、交互设计和信息设计)
*   表面(视觉设计)

这些层面既相互关联又相互依赖，我们在每个层面上的选择都受到我们先前决定的限制。Garrett 从更抽象的(战略和规划)开始，向具体的(视觉设计)发展，确定了在每个阶段影响用户体验的问题。每一个层面都有完整的一章，讨论中加入了基于作者经验的实用建议，特别关注团队角色和过程。

欣赏每个网页设计项目遇到的独特情况，加勒特更关心的是帮助我们学会问正确的问题，而不是假设提供“正确的答案”加勒特在整本书中保持了这一重点，并避免冒险进入技术细节(在这种情况下，这是一件好事)。对于我们这些渴望更多信息的人来说，每章的结尾都为进一步阅读提供了有价值的参考。

设计成功的网站需要在利用(通常有限的)可用资源的同时满足商业目标和用户需求。要实现这一点，需要参与项目的每个人都理解“大局”以及他们在大局中的位置。无论你是自由职业者还是网页设计团队的一员，你都会喜欢这本书为向客户和其他利益相关者解释规划和“大局”思维的重要性提供的有力武器。

##### 底线

这本书读起来是一种享受。加勒特有一种轻松，但中肯的写作风格，并在适当的地方辅以有用的类比和插图。这本书只有 189 页，可以在一个下午/晚上轻松读完。虽然材料可以很快消化，但它肯定会让你思考。我知道它帮助我形成了一个更加一致的观点，即网页设计的所有不同元素是如何结合在一起的，从策略到视觉设计，以及介于两者之间的一切。

虽然这本书没有太多的新观点，但它最大的价值在于它清晰地定义、组织和呈现了重要概念。因此，用户体验的要素对于新手和经验丰富的从业者都是有价值的。不仅仅是因为它的清晰性，我强烈推荐给所有参与网站设计的人。
