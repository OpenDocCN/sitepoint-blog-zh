# GUI 和开源

> 原文:[https://www.sitepoint.com/guis-and-open-source/](https://www.sitepoint.com/guis-and-open-source/)

关于跨 Linux 和开源应用程序分层图形用户界面的影响，开源圈子里经常有争论。当用户可能不知道或不理解如何从系统外壳或通过单独的配置文件配置底层应用程序时，问题围绕着鼓励 GUI 使用来配置守护程序(Windows 服务的 Linux 版本)和服务器。

那些刚接触开源的人通常认为这是 Unix 势利。然而，令人担忧的是，GUI 有时可能变得不可用，需要在系统外壳或文本编辑器中“隐藏起来”。由于缺乏对应用程序(如 Apache 或 Postfix(邮件服务器))的深入了解，这可能会导致系统灾难。

我自己的哲学落在中间。我鼓励使用设计良好的 GUI 来配置服务器，同时鼓励新管理员花时间学习系统 shell，学习如何在 Pico 或 Vi 中编辑文本文件，并阅读他们使用的每个应用程序的手册页。因此，GUI 配置的便利性和速度占优势，并且在紧急情况下，管理员能够进入系统外壳并谨慎地继续管理任何有问题的应用程序。

一个发展迅速的开源 GUI 是 Webmin，它能够全面控制 Linux 服务器，包括管理 Apache、DNS、FTP、数据库服务和邮件服务以及系统级管理。
