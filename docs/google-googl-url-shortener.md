# Goo.gl 网址缩写已发布

> 原文：<https://www.sitepoint.com/google-googl-url-shortener/>

网址缩短服务比网址缩短 [*](#lie) 还多。TinyURL 于 2002 年推出，但 Twitter 140 个字符的限制导致了几十个备选方案，包括 [bit.ly](https://bit.ly/) 、 [is.gd](https://is.gd/) 、[snipurl.com](https://snipurl.com/)、 [Tweak](https://my.dot.tk/tweak/) 和极其荒谬的[urlshorteningservicefortwitter.com](https://urlshorteningservicefortwitter.com/)

我们需要另一个吗？谷歌是这样认为的，并公开推出了 [**goo.gl**](https://goo.gl/) 。自 2009 年 12 月以来，这项服务已经与其他谷歌应用程序集成在一起，但现在它对所有人开放。

与其他选择相比，Goo.gl 有一个主要优势:它由一家不太可能在短期内停止交易的公司支持。正如 [tr.im](https://tr.im/) 用户所知，一个封闭的服务会导致链接失效——你所有的短网址都会消失在一股电子烟中。URL 缩短作为一种商业模式，需要巨大的资源，赚钱很少。

除了稳定性，谷歌还声称安全性和速度。垃圾邮件链接会被自动删除，正常运行时间非常好，你很少会遇到响应缓慢的情况。

统计迷也会喜欢这些报告。我为 SitePoint.com—[goo.gl/Qm9S](https://goo.gl/Qm9S)创建了这个链接。点击它，你会看到你的数据立即出现在 goo.gl/info/Qm9S 的。

![](img/7f6c0a480fb699b02e204ff7b0b5781e.png)

还不服气？自动化[二维码](https://en.wikipedia.org/wiki/QR_Code)生成怎么样？简单的加上**。qr** 到短网址查看图片，即【goo.gl/Qm9S.qr】的。智能手机用户无需输入网址，只需扫描这张图片，就可以转到 SitePoint 的主页。

无论你对谷歌的野心或 URL 缩短的概念有什么看法，都很难否认 goo.gl 的好处。竞争对手的服务肯定会受到影响，但这并不是说他们一开始就有一个可行的业务。

会切换到 [goo.gl](https://goo.gl/) 吗？你知道提供更好设施的服务吗？URL 缩短是否会用不必要的 HTTP 重定向淹没网络？

显然，那不可能是真的，但你明白了！
