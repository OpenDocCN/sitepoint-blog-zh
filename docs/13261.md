# 微软终于适应了

> 原文:[https://www.sitepoint.com/microsoft-is-finally-adapting/](https://www.sitepoint.com/microsoft-is-finally-adapting/)

首先，我想说我是微软开发工具的支持者。在某些情况下，有些人甚至会称我为微软粉丝。然而，我对微软技术的感觉远没有对它的缺点视而不见。我花了几个小时让我的 CSS 布局在 IE 中看起来正确，用头撞墙调试 cryptic。NET 错误消息，并接受了我的 Treo 需要每月至少重置一次的事实。

幸运的是，微软最近一直在做一些改变；不仅仅是他们的技术，还有他们对待技术的方式。IE6 的长寿证明了微软的动作有多慢，但在去年，他们开始谈论更加敏捷，摆脱 2 年的延迟和缓慢的发布时间表。在今年早些时候的 MIX 会议和本周早些时候举行的 reMIX 波士顿会议之间，一些新方法实际上已经开始出现。有些变化是微妙的，有些是更重要的，但每一个都表明微软对开发者的意见反应更快。

在这两次会议之间，微软改变的第一个大的方面是他们如何与开发者交流。在 Mix，大多数主题演讲本质上都只是营销宣传。其中一个基调甚至导致大多数观众在营销变得太多时起身离开。在 reMIX，有一些不错的积极变化。开幕主题演讲较少谈论技术，更多的是演示和例子。他们甚至让米格尔·迪·伊卡萨谈论 Silverlight 的 Linux 版本。他们用 Molly Holzschlag 和 Lou Carbone 的两个精彩主题演讲取代了 MIX 的第二个噩梦主题演讲。我离开主题演讲时感觉很受启发，不像是营销焦点小组的一员。

另一个变化是在技术方面。Silverlight 1.0 是 MIX 的一大亮点，但对许多人来说并不理想，因为它仍然严重依赖 javascript，并且没有用于构建表单的控件。在 reMIX 上，有很多关于 Silverlight 1.1 的讨论。NET 1.0 版的跟随者)以及关于组件一正在开发的新控件包。我很惊讶地看到，在两次会议之间的时间里，微软不仅听取了 MIX 的反馈，而且实际上根据这些反馈采取了一些重大行动。

看看未来的发展是否会继续以这样的速度前进，这将是一件有趣的事情。同时，我正在使用 Silverlight 1.1 和 Visual Studio 2008 的预发布版本。如果一切顺利，我将发布一些关于新 Silverlight 的教程和 Visual Studio 2008 的快速回顾/概述。
