# 苹果的 MobileMe 没有击中要害

> 原文：<https://www.sitepoint.com/apples-mobileme-misses-the-mark/>

成千上万的开发者聚集在旧金山参加苹果 2008 年全球开发者大会。WWDC 的焦点是新的 iPhone 3G 和 iTunes 应用程序商店，iTunes 应用程序商店将于 7 月推出，为 iPhone 和 iPod touch 设备带来(官方)本地应用程序，但网络开发者一直在关注苹果重新推出的 [MobileMe](http://www.apple.com/mobileme/) 。Mac 服务。

对于 Gmail、Google Calendar 和 Picasa Web(或任何来自 Yahoo！和微软)，MobileMe 是一个新的竞争者，它承诺将你的在线更改自动推送到微软 Outlook 等桌面应用程序和 iPhone 等移动设备中(反之亦然)。然而，与竞争对手不同的是， **MobileMe 没有显示出为其他网络应用提供开放 API**来整合服务的迹象。

MobileMe 与其竞争对手的另一大区别是 MobileMe 的年费为 99 美元。一方面，你可以使用免费服务，这些服务集成了越来越多的网络应用和支持网络的桌面应用；另一方面，你可以*为一项服务*付费，这项服务*只有*可以在支持的设备和桌面应用之间无缝集成。

作为一个持卡的 Mac 粉丝，我现在将继续使用我的非苹果网络应用。由于他们的开放 API，我可以使用像 [Plaxo](http://www.plaxo.com/) 这样的免费服务将它们同步到我的桌面软件和设备上。这种体验并不像 MobileMe 提供的那样无缝，但也很接近了，作为一名开发人员，知道我的数据只是一个记录良好的 API 调用，这很令人欣慰。
