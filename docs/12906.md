今天的文章是由我们自己的 [Alex Walker](https://www.sitepoint.com/blog/) 撰写的，他有一种不健康的痴迷，喜欢发现异常现象并详细探索它(我坐在 Alex 旁边，所以我知道他在这个教程中花了多少时间)。本文就是这一探索的结果。

不久前，Alex 发表了一篇关于当你探索重叠的 CSS 背景图片时的乐趣的博文。在那个阶段，他产生的效果是有趣和有用的，尽管不可否认在有限的情况下(我们将它应用于 sitepoint.com 首页的广告，以推广我们的艺术&JavaScript 书的科学)。虽然并不是在每种情况下都能立即实现，但它展示了一种在页面中嵌入隐藏功能的方法——为用户提供一个“复活节彩蛋”。

这些复活节彩蛋不会成就或破坏你的设计——事实上，大多数人可能不会注意到它——但它们是一个很好的接触和对细节的关注，可能会让一些用户感到惊讶。完全相同的效果甚至被在 [Clearleft](http://clearleft.com) 到[的人们用来为他们的 Silverback 可用性测试软件的登陆页面创建视差效果。](https://www.sitepoint.com/andy-budd-design-usability-css/)

亚历克斯思考的下一个合乎逻辑的步骤是将这项技术推向极限。结果是只有当用户调整窗口大小时，动画才会运行*。*哦，而且不使用任何 JavaScript。[查看示例](https://www.sitepoint.com/examples/scanline/)。

![](../Images/adec5e53d4e4d4fd877fa6f2c5f650ba.png)

当然，一个显而易见的问题是*你为什么要大费周章？为这种效果创建图形绝非易事。*
