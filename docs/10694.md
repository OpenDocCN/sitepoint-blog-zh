## 显示抄本

凯文:2010 年 8 月 20 日。脸书盒子、Twitter 按钮、Adobe 字体和 jQuery 图表。我是凯文·杨克，这里是 SitePoint 播客#75:可怕的过度杀戮。

这是本周 SitePoint 播客的一集。我们有太多的新闻等着我们，我想几周前我通过谈论我们的非技术节目来试探命运，有很多东西要谈，很多都是技术方面的，布拉德，你为什么不带领我们。

Brad: 当然，首先，我们实际上还有另一个条目，那就是 Google 已经宣布他们将结束 Google Wave 的开发，所以大家都向 Wave 挥手告别吧。

凯文:哦，再见波。

Brad: 是的，Wave 实际上是在 2009 年 6 月推出的，所以它实际上只有一年多的历史，我相信它是在今年 5 月份的测试版中出来的，所以它只是在谷歌退出之前的几个月才正式成为一款产品。

Kevin:Google Wave 也有一个 SitePoint 连接。SitePoint 的作者 Cameron Adams，我与他共同编写了简单的 JavaScript，他也为我们的其他几本书做出了贡献，他是 Google Wave 的前端设计师之一。我不知道你们是否经常使用 Wave，但在它的早期，大约每周他们都会关闭应用程序进行维护，你看到的维护网页上写着类似“冷静点，Wave 一会儿就回来，”这是一个人在海滩上的脚，云彩飘过。还有卡梅隆·亚当斯的脚，这就是 Google Wave 宕机时你看到的。(笑声)

**帕特里克:**琐事！非常好。

凯文:琐事，是的。

Patrick: 你知道，当我看到这个公告时，我有点惊讶，我想 Twitter 上对它的负面评价，至少对我关注的人来说是这样的，关于 Wave，“Wave，哦，好吧，摆脱掉了，反正我没有用 Wave，它有什么用处，”等等。我对此感到有点惊讶，因为，我承认，我自己并没有马上进入 Wave，但我确实发现了它的有用性或它的力量，这要感谢我的朋友韦恩·萨顿[@韦恩·萨顿](http://twitter.com/waynesutton)，他对此深信不疑，他很遗憾看到它的离去，因为他用它进行了很多协作。我和他一起参与了一个项目，现在仍然是，我们用它来交谈，分享信息和数据，电子表格等等，它非常有用，所以我肯定能看到它的用处，在某种程度上，我很遗憾看到它的离去。

是的，我们在我的一个项目中使用了很多，实际上是独家使用；我们写了一大堆东西，大概有六七个人在 Wave 中贡献和编辑东西，到现在为止，它真的很好，是一个非常安全的系统。

Brad: 有一点我从一开始就不太喜欢 Wave，那就是它的实时性，我想我的感觉是，如果我在写信息或写电子邮件，我会在发送之前尽可能地让邮件或信息尽可能完美，所以我不想让我要发送的人实际上看到我准备发送的信息，因为我会删除单词和检查拼写等类似的事情。我想在发送之前确保它是正确的。我的意思是，我当然可以看到这样有意义的例子，就像在聊天室里，如果你和一群人聊天，但我从来没有真正喜欢过这方面，所以我从来没有真正进入过。

凯文:我想至少在最初的演示中，我记得有一个功能，如果你想处理一些没有被实时传输的文本，你可以关闭实时更新，然后当你满意时，你可以重新打开实时更新。这是在设计过程中可能已经放弃的功能之一，我有点觉得他们在用户界面的复杂程度上来来回回。但是，是的，当然是实时通信媒介和文档创作环境之间的奇怪混合，我想这是人们对它如此兴奋的原因。

布拉德:它是否超前于时代？如果五年后发布，它会做得好吗，是不是太超前了？

凯文:我觉得谷歌需要——他们通常非常擅长低价销售，但最近他们有点过度销售了。就像你回想起 Gmail 和 Wave 的推出很大程度上是 Gmail 战略的重演，他们在测试版推出这个东西，你需要从已经被邀请的其他人那里获得邀请，所以这种封闭的测试环境，他们希望将服务的用户转化为他们的营销力量。但是你在 Gmail 上看到的是，这是一种几乎没有公布的东西，我第一次听说 Gmail 是通过使用它的人说，看看这个，我甚至会给你发一个邀请。谷歌，他们在他们的大型谷歌 IO 大会上揭开了 Google Wave 的面纱，这个长达一小时的视频令人印象深刻，然后他们告诉你的下一件事是，不，你还不能使用它。所以我不知道这是不是种下了帕特里克所说的消极的种子，我们现在看到的这种取消，“哦，是的，我知道它注定会失败！”

帕特里克:嗯，我认为它是一个利基工具，在某些方面，我会说它与谷歌文档有多么不同，对吧；因为如果你分享文档，谷歌文档也有一个实时的方面。我知道，因为我在那里处理一些文档，总是有，哦，xyz 也在编辑这个，好吧，显然如果你想控制文档或者它是给特定人的消息，那么你可能不想在那里这样做，但我可能会看到他们将 Wave 的一些功能滚动到文档中，可能是可选的，对，可能不一定是默认功能，但人们可以使用。谷歌博客上的[博客文章指出，Wave 创新背后的核心代码和一些协议，也就是他们的话，像拖放和逐字符实时打字已经在开源中存在，所以客户和合作伙伴可以玩这些，并继续在其上创新。所以我不知道我们是否会从其他人那里看到其他形式的 Wave，但他们确实说他们会提供一些工具来从 Wave 中获取我们的数据。](http://googleblog.blogspot.com/2010/08/update-on-google-wave.html "Official Google Blog: Update on Google Wave")

斯蒂芬:如果它能转世成为医生，我会非常高兴。比如，如果他们在 Docs 中实现了一些功能，我会非常高兴，比如文档管理的方式等等；如果他们那样做，我会很高兴。

Brad: 你知道我最感到遗憾的可能是那些开发人员，他们实际上使用了 Google Wave API，花费了成百上千个小时来开发这些集成了 Wave 的很酷的应用程序，但现在他们基本上毫无用处。我认为，如果这种趋势继续下去，因为我们看到了 Google Wave，我们看到了 Pownce 下降，如果这种趋势继续下去，开发者将会更加犹豫，直到他们知道服务已经建立，他们不只是在浪费时间，这反过来可能不会帮助新服务或应用程序的增长，所以我的意思是，如果这种趋势继续下去，它肯定会对 API 世界的开发者不利。

凯文:这个话题，在我们讨论它的时候已经是旧闻了，因为 Wave 的取消是在我们上一集新闻之后宣布的。我认为这对我们来说是一个里程碑，伙计们，来自波兰的播客听众 Chris Trynkiewicz 写道，他实际上写了他对我们将要谈论的一些事情的看法。我喜欢这个；这是我们听众的倡议。克里斯写道:“我的猜测是，当 Wave 于 2010 年 5 月 18 日公开发布时，营销史诗失败了，这仅仅是两个半月前。考虑到 Gmail 获得市场份额的时间，人们可以认为这个决定来得太早了；此外，甚至没有一个可靠的方法来连接 Wave 和 Gmail 或任何其他电子邮件帐户。他们到底在想两个月后会发生什么？”所以，让我们稍微讨论一下。那两个月，是 Google Wave 的验证时间吗？

斯蒂芬:我希望不会。我希望他们考虑得更多一点。我的意思是，就在最近，他们发布了一些功能，比如你可以邀请任何没有谷歌邮箱地址的人加入 Wave 文档，就像，我甚至不知道这是多久以前的事了，可能是六周前，现在他们正在取消它。好吧，我希望没有人把他们的东西发给那些完成后无法访问的人。所以我不知道，似乎他们真正证明这项技术的时间真的很短，也许这真的是那些实验室的事情之一，他们从一开始就说这可能不会存在，谁知道呢。

Kevin:Wave 的最初版本让我印象深刻的是，它是在他们的开发者大会上宣布的，实际上他们将 Wave 视为底层的管道，他们构建的用户界面对他们来说只是一种概念验证。他们希望开发人员加入进来，并开始在他们组装的这个平台上进行构建，我觉得两个半月前的公开发布可能是他们最后一次尝试，以获得他们认为这个东西应该得到的流行度，在封闭的测试期之后，他们有点走了，嗯，这不起作用；也许如果我们把它公之于众，反而会奏效，所以这是他们让它得到他们想要的受欢迎程度的最后一次尝试。我不知道，这很难说，就我个人而言，我不时地使用 Wave，我想我已经用它写了一个相当大的文档，当然，当我和其他人一起观看他们最初的演示时，我受到了启发，但它从未真正起飞，它从未真正实现他们想要的愿景。也许是因为他们的视野如此广阔，而且正在改变世界，我的意思是他们正在接受电子邮件和即时通讯。感觉这种革命甚至在今天的网络上也需要五年时间，也许他们只是推动得太快了，或者试图一次做太多事情。我很难过看到它的离去，我的一部分仍然希望，因为 Wave 技术是开源的，也许心中有更长时间线的神秘捐助者会出现，拯救这项技术，我们可能会看到 Wave 从灰烬中崛起。称之为一厢情愿的想法，但我认为谷歌有一些好主意。

脸书，我不知道他们是否有好主意，但他们本周将从应用程序 API 中移除一些框。我不知道我们的听众中是否有人写过脸书 API，但是有一段时间脸书是应用程序平台，我不知道，几年前苹果的应用程序商店似乎抢了它的风头，但是直到那时脸书才是 web 应用程序开发人员考虑将他们的技能转移到 next 的地方。你可以在脸书的生态系统中开发应用程序，而不是在你自己的网站上建立一个独立的 web 应用程序，你希望人们发现你可以在的生态系统中开发应用程序，每个安装了该应用程序的脸书用户都会在他们的个人资料页面上显示一个你的应用程序的方框，只要访问你朋友的个人资料页面，你就可以发现他们正在使用的应用程序，并希望你自己安装它们，这是在这个社交环境中推广应用程序的一种方式。嗯，这一切都将在本周改变，因为盒子正在消失。取决于你问的是谁，这要么是一件大事，要么是没有人真正关心盒子了。伙计们，你们最后一次看到脸书盒子是什么时候？

布莱德:我有一些。

帕特里克:我会说最近。

凯文:最近，是吗？

**帕特里克:**最近。我不能确定具体的日期和时间，但我想是最近，当我看某人的资料时。我想另一个问题是我最后一次真正安装一个盒子是什么时候，那是很久以前的事了；我都不记得上次在个人资料上打勾是什么时候了。

凯文:人们认为，目前个人资料页面在脸书开始变得有点无关紧要，人们通过他们的新闻订阅和他们全屏使用的应用程序生活在脸书。访问某人的个人资料页面，查看他们的应用程序，他们的盒子，他们想通过个人资料页面表达什么，这种想法在脸书已经成为过去。

Patrick: 不过，我认为这是脸书的一部分，访问并查看人们对他们的个人资料做了什么。我不知道，我知道我们对 MySpace 有这种厌恶，我们认为它是音乐播放和闪光的图形以及所有吸引眼球的东西，这让很多人感到讨厌，但脸书做了很好的工作，我想说的是通过他们不同的风格要求减轻了这种厌恶，我想我会把他们称为他们看起来适合网站的盒子，他们匹配相同的配色方案，你知道这一切都可以一起工作，看起来很好。现在我只是在想，你知道，如果脸书个人资料只是一些东西，比如说，你可以有一个你的朋友的盒子，你可以有你的关系状态和你的生日，等等，你可以有你的状态更新。那么每个配置文件可能看起来都一样，除非他们自己增加了一些更深层次的定制选项，因为这是你定制你的配置文件、添加框和移动东西的选项。现在，如果改变的话，那将是标签，你知道这不是真正的定制。

凯文:我觉得脸书总是试图避免定制，这是他们不情愿做的事情，因为他们想把自己与 MySpace 的丑陋时代区分开来，当时每个人都定制自己的页面，这对所有人来说都是免费的，没有 MySpace 的外观，MySpace 的外观完全是混乱的。

布莱德:我认为这可能是个好主意。MySpace 在每个人都在上面的短暂时间里很有趣，但后来你对了，你去的每个页面都响起了刺耳的音乐，所有的东西都在闪烁，你陷入了痉挛，我的意思是这很可笑(笑声)。

帕特里克:这是真的发生了还是只是一种比喻？

布拉德:我是凭经验说话的。不，这当然是比喻。但是当每个人都开始看脸书时，就好像哇，与我们在 MySpace 上看到的相比，这里看起来太干净了，所以我认为他们不允许这样的事实是脸书今天这么大的原因。

你的意思是你不喜欢闪烁的星星背景？我是说来吧。

布莱德:我只能应付这么多跳舞的婴儿。

Patrick: 如果你相信你的个人资料应该反映你是谁，那么闪亮的背景可能就是你是谁，以及你想让别人看到你是谁。然而，如果你像我一样，你看我的 MySpace 个人资料，因为我从来没有真正进入它，它只是非常默认的东西，一首歌和一点点信息，仅此而已。但是脸书，我不知道，统一的个性，是我刚刚编造的一个花哨的短语还是一个真实的东西？因为就是这种感觉。

凯文:我觉得，对于那些围绕某种交互模式打造体验和设计应用的开发者来说，他们的应用可以通过他们个人资料页面上的这个小盒子向用户的社交网络展示自己，而现在应用宣传自己的标准用户界面是一个完整的标签。我现在看到的是，我安装的许多脸书应用程序现在可以放在标签上，但它们仍然被设计成一个小盒子的大小，所以现在你的个人资料上有整个标签，当你点击它时，你只会在页面顶部看到这一点点内容。这就像是用户体验、脸书 API 的用户界面标准发生了根本性的变化，这对他们的应用生态系统有什么影响？这就是我们期望一个平台在网络上持续并保持稳定的时间吗？已经两年了，如果我现在考虑开发一个脸书应用程序，我是否必须理解脸书在两年内可能会完全不同，这是我的一个移动目标？

是的，你说得对，凯文，这将迫使脸书的应用程序开发人员真正重新思考他们的应用程序如何工作，以及他们如何推广自己，因为我的意思是任何应用程序的主要焦点之一是让人们安装它；你希望尽可能多的人安装它，然后像病毒一样传播，并与他们的朋友分享。现在它被隐藏在一个标签上，不太引人注目，很少有人会看到它，这些开发者会想出更有创意的方法来将这些内容放入你的新闻订阅源，以推广他们的应用程序，因为没有其他方法可以做到这一点。所以我预计会看到更多类似垃圾邮件的新闻推送之类的帖子。

Patrick: 嗯，很多博主在他们的博客上使用 TweetMeme Twitter 转发按钮，包括我自己。比方说，那些日子屈指可数，因为 Twitter 已经推出了他们自己的 Tweet 按钮，你今天就可以将它嵌入到你的博客中。
