# 开源专利共享

> 原文:[https://www.sitepoint.com/open-source-patent-commons/](https://www.sitepoint.com/open-source-patent-commons/)

[开源开发实验室](http://osdl.org/newsroom/press_releases/2005/2005_08_09_beaverton.html)的声明进一步加强了我在上一篇文章中对即将到来的新 GPL 版本 3 的观察。

这对于继续将经过验证的开发解决方案集成到开源项目中应该大有帮助。像 IBM、RedHat、、Sun 和 Novell 这样的大公司成为 OSDL 的一部分并没有什么坏处。它也接受了这样一个事实:服务驱动的公司而不是软件许可公司在赚钱。

当然——我肯定这些人可能会保留一些关键的专利，并在这些专利上赚很多钱——但他们开始把钱用在有嘴的地方。

Linux 和开源解决方案的成功(据我所知，这对开源用户来说并不新鲜)很大程度上取决于专业知识、对客户业务领域的理解以及服务/支持。

虽然建立在此基础上的早期商业模式可能在 90 年代失败了——但失败更多地是因为公司的贪婪，他们采用自由软件，并认为他们可以从谦逊的服务消费者身上赚到数十亿美元。基于开源的企业现在看到的是，他们对使用开源作为解决方案的热情反映在他们的演示和仔细工作中，并根据价值和后续情况进行销售。

尽量不要进入 Andrew 的地盘，这更多的是关于我们与客户、合作伙伴和供应商建立的关系，而不是 Linux 或开源的事实。我们的信念和信誉建立在证明我们的解决方案有效并提供效率和经济收益的基础上。这使得通常对许多客户来说仍然陌生的关于开源机会的对话和最终销售变得更加容易执行。
