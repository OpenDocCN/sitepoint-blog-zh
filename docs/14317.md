# W3C 记录了网络的架构

> 原文：<https://www.sitepoint.com/w3c-documents-the-architecture-of-the-web/>

在 10 月份的一次会议上，万维网联盟的技术架构小组完成并发布了一份名为“万维网架构”的最终工作文件，该文件位于 T2。

TAG 正在接受公众对最终草案的意见，直到 12 月初，届时它将作为第一版和正式版本获得最终批准。

该文件旨在将构成网络的建筑和结构元素收集和组织在一个中心文件中。W3C 的 TAG 成立于 2001 年，正是为了通过记录和推广基于标准的 web 架构原则来实现这一目的。

它的目标是业余爱好者和专业内容和架构开发人员，将基础和核心 web 标准带到最前沿。它也为开发人员将来可能采用的新的网络相关技术奠定了基础。

该小组的成员包括主席蒂姆·伯纳斯·李(W3C)和:

*   联席主席斯图尔特·威廉姆斯(惠普公司)
*   丹·康诺利(W3C)
*   保罗·科顿(微软)

*   克里斯·莉莉(W3C)

*   诺姆·沃尔什(太阳)

小组成员 Chris Lilley(同时也是 SVG 工作组主席)在工作之余与 SitePoint 讨论了标签的文档使命

问:标签对这个文档的观点是什么？

答:WebArch 写下并向其他人提供关于网络如何工作的信息。有些是以前写下来的，但分散在各处；有些解释得很差，很多根本没有真正写下来，而是形成了一种隐藏的知识，是那些已经在网络上工作多年的人的知识。正如你所说，这对业余爱好者来说是一个很高的门槛，实际上对专业人士来说也是一个障碍。

当然，写下“隐藏的知识”有风险——其他人可能会发现它是不真实的，或者不再真实，或者有冲突的实施经验。这就是为什么在公共讨论列表中开发 WebArch，[www-tag@w3.org](mailto:www-tag@w3.org)是至关重要的。这一发展在最后一次呼吁(事实上，是两次)中达到高潮，以真正确保我们写下的内容是真实的、有效的和相关的。WebArch 最近进入了建议推荐阶段，这是成为 W3C 推荐标准(Web 标准)的最后一个阶段。

问:这是否有助于转移 web 开发人员和内容作者对 XML 的关注，或者鼓励他们将更多的注意力放在 XML 上，因为他们可能还没有花时间在 XML 上。

答:WebArch 的一部分确实讨论了什么时候适合使用 XML，什么时候不适合，例如视频数据。它还提供了使用 XML 的良好实践指南。所以，是的，随着我们转向一个更加基于标准的网络，它将在这一领域发挥作用。
