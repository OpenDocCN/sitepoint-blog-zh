# Web 3.0 meshup 风格——你想配薯条吗？

> 原文:[https://www . site point . com/we B- 30-meshup-style-would-you-like-fries-with-that/](https://www.sitepoint.com/web-30-meshup-style-would-you-like-fries-with-that/)

一直有一个持续的合唱(它有没有停止过)关于语义网的这一套，语义网的那一套，[干巴巴的…一点也不性感](http://www.mathewingram.com/work/2008/02/28/the-semantic-webs-biggest-problem/)，[有根本缺陷](http://www.shirky.com/writings/ontology_overrated.html) …混合和重复。

那么，上一次有人熬夜阅读[RFC 2616-HTTP 1.1](ftp://ftp.isi.edu/in-notes/rfc2616.txt)规范，或者 [XHTML 1.1 Rec](https://www.w3.org/TR/2001/REC-xhtml11-20010531/) 或者钻研各种风格的 RSS ( [0.9](http://www.rssboard.org/rss-0-9-0) ， [0.91](http://www.rssboard.org/rss-0-9-1-netscape) ， [0.91-userland](http://backend.userland.com/rss091) ， [1.0](http://web.resource.org/rss/1.0/) ， [0.92](http://backend.userland.com/rss092) 是什么时候？？我猜你们中没有多少人举手。或者你觉得太尴尬了，没关系——如果你让我知道，我很乐意和你聊聊:)

语义网是，知道吗，基础设施！哇，这不是一个产品，它很可能永远不会被收缩包装，放在货架上布拉茨娃娃旁边(感谢上帝)。就像 HTTP，RSS，POP，IMAP 等无数协议和语言都不在架子上。酷的东西，性感的东西，都是建立在这个基础设施之上的。网络工具、网站和系统，我们已经成长为爱所有的生活和呼吸这一基本的基础设施——我们不在乎！我们喜欢 flickr 上的快速标记和评论，我们喜欢所有能够快速工作的谷歌应用程序。但是我真的关心它们是建立在什么之上的吗？嗯，作为一名技术主管，我会这么做，但这不会出售系统或心灵和头脑。

这正是 Web 3.0/语义网的意义所在。黑仔新工具给科技头们新玩具玩。这就是 Web 2.0 风格[混搭](http://en.wikipedia.org/wiki/Mashup_(web_application_hybrid))和 Web 3.0 风格[混搭](http://www.openlinksw.com/blog/~kidehen/?date=2007-03-22)的区别。我们可以停止浪费时间把东西塞在一起，把方钉塞进圆孔。相反，我们可以利用更智能的数据来承担更多的负担，让我们的开发者梦想下一个 flickr。但这一次更好，更多拥挤的网络魔术和新的连接和更多的链接！必须有更多的链接。

即使你从来不使用语义网工具，为什么要浪费时间说它们永远不会工作呢？我不用 [Ruby](http://www.rubyonrails.org/) 或者 [Python](http://www.python.org/) 或者 [PHP](http://www.php.net/) 。我希望我做到了，我正在学习一点 Python 和一点 PHP，因为它们是很好的工具。我个人是否将它们用于关键任务？不。我会吗？不，因为我已经把时间投入到其他技术上了。但我认为它们很棒，我希望一天中有更多的时间，这样我就可以学习它们，见鬼，如果一天变得更长，我会学到一切。

几乎任何技术都有它的位置和用处，好吧，等等……最初任天堂的那个机器人是什么？？？旋转，闪光，嗯？我从来没有过，所以也许我很嫉妒(现在也是)有过孩子的我。
T3】

不要误会我的意思，这东西很难，需要坚定的决心(这是正在努力)。但这只是早期，有点像回到 1993 年。你们中有多少代码编织者在 1995 年创建谷歌地图，在 1998 年向崇拜你们的粉丝推送 RSS 订阅，或者在 2001 年播客你们的蹩脚音乐？可能不多。我没有，我甚至不会演奏音乐，更不用说蹩脚的音乐了。

当 RSS 真正流行起来，谷歌地图被逆向工程的时候，那就是我们现在的开始 API 和 mashable Web 的诞生。当语义技术变得更容易使用，人们开始“查看资源”并对一个很酷的 web 工具进行逆向工程时，也会有类似的效果……我等不及了。
