# 采访 vBulletin 的 Kier Darby

> 原文:[https://www.sitepoint.com/interview-kier-darby-vbulletin/](https://www.sitepoint.com/interview-kier-darby-vbulletin/)

**随着 vBulletin 3.0 黄金版的即将发布，我很高兴最近有机会采访了 Jelsoft 公司的首席开发人员兼产品经理 Kier Darby，该公司是英国著名的论坛系统开发公司。**

***首先，Kier，你能告诉我们一些关于你自己以及你参与 vBulletin 开发团队的情况吗？是什么首先让你进入软件开发的，你的编程背景是什么？你是如何开始开发基于 Web 的应用程序的？*T3】**

嗯，我 27 岁，住在英国。我已经作为 vBulletin 开发团队的一员在 Jelsoft 工作了几年，并在 2002 年末成为首席开发人员和产品经理(接替 John Percival)。

我的编程背景可以追溯到我在大学的时候，当时我正在攻读计算机可视化和动画学位，其中 50%的内容是软件开发(C 和 C++)。当时，我真的很鄙视编程，因为我更喜欢艺术方面，我想使用现有的工具制作动画，而不是编写软件来完成。

大学毕业后，我和与我同时毕业的另外两个朋友成立了一家 CGI 动画公司。我在这家公司工作期间，我们为几个大客户制作动画和图像，包括 AMD、微软和国际足联。

在我整理各种网站的时候，我开始通过 Javascript 重新开始编程。我很快就对 Javascript 的局限性感到沮丧(那时，IE4 和 NS4 之间的大战正在激烈进行，使 Javascript 跨浏览器兼容比编写程序花的时间还要多)。

最终，我发现自己构建了一个需要动态的、数据库驱动的后端的网站。我交谈过的每个人都推荐 PHP 和 MySQL，所以我开始和这两个合作。在某个时候，人们决定该网站需要一个留言板，而 vBulletin 正好符合这个要求。那时 vBulletin 还在版本 1 中，我还想在系统中加入很多额外的功能，所以我开始为 vB 1 和即将发布的 vB 2 编写修改。在编写这些修改(在 vBulletin 社区中称为“hacks ”)几个月后，我被正式邀请加入开发团队。

***Jelsoft 运行了多久，是什么让你决定开始开发 vBulletin 这样的论坛系统？它是从几个人在一个小系统上工作开始的吗？你现在如何管理这么大的项目？*T3】**

vBulletin 自 2000 年初开始向公众提供。当约翰·帕西瓦尔运营的一个网站发现越来越难以维持其留言板时，它就应运而生了，当时该网站运营着 Infopop 的 UBB。论坛变得非常大，某些操作如搜索消息会使服务器瘫痪，需要几分钟才能完成。vBulletin 最早的(非公开的)版本本质上是为运行 PHP 和 MySQL 的 UBB 而设计的，而不是 Perl 和平面文件存储。

很多人对这个新系统感兴趣，因此决定改进代码，以产生一个独特的产品，并将这个代码作为 vBulletin 发布给公众。当时 vBulletin 完全由 John Percival 和 James Limm 编写和管理。从那以后，更多的开发人员被雇佣来开发 vBulletin 2。vBulletin 2 的使用量非常大，所以他们决定雇佣一个专门的支持团队。

vBulletin 的最新版本已经发展成为一个非常强大的系统，同样能够运行一个中等规模的论坛或一个巨大的、有数百万帖子的论坛。管理这种规模的产品本身就是一种挑战，个体开发人员和 Jelsoft 各个部门之间的不断沟通是防止产品的复杂性超越个体团队成员知识的唯一方法。我们在一个私人的 IRC 房间里定期开会，或者(对那些有技术的人来说)使用视频会议。

***您第一次接触 PHP 是什么时候，是什么原因让您选择了 PHP，而不是 ASP、JSP 甚至 Perl/CGI 等其他技术来开发 vBulletin？你这么喜欢 PHP 的原因是什么？*T3】**

PHP 是在 1999 年由一个大学朋友介绍给我的。至于我为什么这么喜欢它，原因很简单——没有任何其他语言能让你如此快速地开发，同时为你提供深度的功能和突破极限的能力。

其次，PHP 是免费的，人们安装起来相对简单。这意味着越来越多的服务器开始支持 PHP。不像 ASP，它要求你使用微软的软件库，也不像 Perl，它带有“chmod this file to x and that file to y etc…”这对于没有经验的用户来说是一个噩梦，PHP 代表了一个易于用户操作的系统，不需要比以用户已经习惯的方式上传 HTML 脚本更多的技能。

当你在发展时，你更喜欢怎样工作:长时间的工作被长时间的休息所打断，每天稳定的进步，还是其他？你能给那些觉得“一天 24 小时不够用”的 PHP 开发人员提供一些建议吗？T3】

我经历了断断续续。众所周知，在压力大的时候，我会连续 72 小时进行编程，但一般来说，我更喜欢保持有规律的“办公时间”，尽管这些时间经常在时钟上滑动，所以我最终会从中午开始工作，一直工作到凌晨。不过，总的来说，我发现白天工作时，我完成的工作最多。

…而且一天 24 小时*还不够*！

***vBulletin/Jelsoft 是全职工作吗？当你不编码或不管理项目时，你喜欢怎样度过你的时间？*T3】**

在过去的 18 个月左右，vBulletin 是我的全职工作。当我不工作时，通常是因为我睡着了，但在那些不在键盘前的罕见场合，我是一个热心的摄影师，并尽可能多地骑着我的山地车出去。

***那么，你有什么秘密武器——你最喜欢的开发工具？你最常用的操作系统是什么，为什么？*T3】**

我的“基本”软件库包括 Macromedia/Allaire Homesite，用于编写 HTML 和 PHP，还有一点 Textpad，用于其块选择模式和高级正则表达式搜索功能。

我已经尝试了很多 PHP 专用的 IDE 解决方案，比如 Zend Studio，但目前它们感觉还很不成熟，尽管一些功能非常好(函数/变量名完成等。)缺乏可定制性和灵活性让我不得不回到 Homesite。Homesite 内置了大量的功能，我可以通过用 Javascript 等编写“动作脚本”来添加我自己的功能。

我也不能没有 Adobe Photoshop 来创建和处理图像，Tortoise CVS 来进行源代码控制，以及 Microsoft OneNote 来处理所有那些在同时处理许多领域时不可避免的重要笔记和涂鸦。

就操作系统而言，我希望坐在前面工作的任何机器都运行 Windows 2000 或 XP，而我所有的服务器都运行各种版本的 Linux。我觉得，有了这个设置，你就拥有了两个世界的精华，虽然我不会考虑将 Windows 机器作为服务器直接连接到互联网，但我同样反对坐在 Linux 机器前运行我的各种桌面应用程序。也许这只是个人喜好的问题，但我觉得这是各自系统的优势所在。

vBulletin 可以被认为是专业人士的选择——它被用于许多大型网站，如 SitePoint、Lycos 和 Winamp.com。与目前可用的其他系统相比，它有什么优势和特性？为什么你认为 vBulletin 是论坛软件的最佳选择？T3】

vBulletin 从一开始就被编写为能够处理繁重的负载。特别是 vBulletin 3，针对大型主板进行了进一步优化。然而，这种优化并没有以期望运行较小规模操作的站点为代价。

vBulletin 的魅力之一是它的可伸缩性，它使管理员能够调整他或她的安装，以精确地满足特定站点的需求。除了论坛/线程/帖子/用户等基本组件之外。，几乎每个功能都可以调整，甚至完全关闭，以便为网站的访问者提供快速响应的体验。

vBulletin 管理控制面板是非常全面的能力，它给管理员控制自己的网站；从打开和关闭整个系统的强制开关，到无需离开控制面板就能改变用户界面中每个元素的外观。

***不像其他很多主要的论坛软件包(比如 phpBB 或者 Invision Board)，用户要为 vBulletin 付费。是什么让你决定对软件收费？为什么人们应该使用 vBulletin 而不是免费的替代品之一？*T3】**

虽然 phpBB 是开源软件，Invision Power Services 提供他们所谓的无限免费试用，但对支持服务收费，但在早期阶段就决定 vBulletin 将使用商业许可系统，向客户收取许可证费用，但提供优先支持服务作为 vBulletin 许可证持有者权利的一部分。

向客户收取 vBulletin 许可费意味着 Jelsoft 可以向开发团队和支持团队支付非常合理的费用。如果员工对产品或公司有一定程度的忠诚，并且这些员工已经为公司及其产品工作了很长一段时间，这将极大地增加产品或公司的稳定性。在没有长期工作人员或工作人员更替率高得多的情况下，很难达到专业知识和专业水平。这一优势直接影响到我们的客户。

当客户购买 vBulletin 许可证时，他们将自动获得优先支持服务(除了在[vBulletin 社区论坛](http://www.vbulletin.com/forum)提供的社区支持之外)。这意味着任何拥有 vBulletin 许可证的人都可以在任何时候联系我们的支持人员，而不必担心可能会花多少钱。我们还发现，经常在 vBulletin 社区论坛上发帖的客户倾向于使用优先支持系统来解决他们认为可能显而易见或可能让他们在同行面前显得愚蠢的问题。优先支持系统的可用性有助于缓解这一问题，在该系统中，您的问题是保密的，并被分配给一个人，他将尽最大努力提供帮助。

我们的受薪支持人员遍布世界各地，其中一些人现在全职工作，我们在每个主要时区都有全天候的服务，我们对支持系统中问题的平均响应和解决时间感到自豪。

撰写本文时，vBulletin 3.0 已接近黄金版。当你第一次从版本 1 开始时，你对这个项目有什么目标？你对这个系统有什么启发吗？T3】

虽然我没有亲自参与 vBulletin 1.x 的开发，但我相信我可以代表 John 说，该项目的目标是生产一个易于安装和使用的公告板系统，借助数据库后端的能力，在搜索公告板关键字或查找某个用户的所有帖子等所需的时间方面实现了巨大的飞跃。

在这一目标实现之后，vBulletin 开发的首要目标是响应客户对工具和过程的建议，使管理员的工作更加轻松，并为网站访问者提供一个更加丰富的体验和交互环境。vBulletin 3 延续了这一传统。

你最喜欢在 vBulletin 的哪个部分工作？T3】

vBulletin 吸引我的第一件事是模板系统，它仍然是我最喜欢使用的领域，使管理员可以更容易地为他们的访问者创建丰富的定制环境。我对 vB3 的新样式系统特别满意，它允许管理员创建无限嵌套的样式，每个样式都继承其父样式中的定制。

***你认为一个论坛是一个大网站必不可少的一部分吗？你认为拥有一个论坛能给网站所有者带来什么好处？*T3】**

在一个重视与公众和/或客户互动的网站上，像 vBulletin 这样的留言板系统对于公司或组织与访问他们网站的人保持联系肯定会有很大的好处。留言板有一个明显的优势，一个访问者提出的问题可以被任何人一遍又一遍地引用，减少了花费在无休止地重复那些似乎一遍又一遍出现的问题的答案上的时间。

然而，我不会说留言板是每个大网站必不可少的部分。处理不好，一个论坛可以是一件非常糟糕的事情。我想到的一个例子是一家专门生产计算机硬件的公司。工作人员开辟了一组论坛，名为“支持论坛”。不幸的是，该公司的相关人员很少回答人们在论坛上提出的问题。那家公司因为明显没有技术支持服务而在互联网社区中名声很差。

当人们访问一个有活跃和蓬勃发展的论坛的网站时，他们倾向于在那里逗留，会见志同道合的人，讨论他们感兴趣的事情。从商业角度来看，定向广告的潜力等。对特定人群的影响不可低估。

你对 vBulletin 的未来有什么计划？以及最初发布后会如何发展？你计划开发新的功能或扩展到新的产品领域吗？T3】

我不能透露任何具体的细节，但我们未来的计划包括几个新产品，所有这些都将使用为 vBulletin 3 开发的极其灵活的基础代码来绑定到 vBulletin。首批发布的产品中包括尚未命名的内容管理系统。除此之外，恐怕你只能等着瞧了！

SitePoint 感谢 Kier 抽出时间。
