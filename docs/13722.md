# PHP 来拯救！

> 原文:[https://www.sitepoint.com/php-to-the-rescue/](https://www.sitepoint.com/php-to-the-rescue/)

如果你在寻找一本有思想的周六读物，你不会因为[错误代码或异常而走得太远？为什么可靠的软件这么难？](http://damienkatz.net/2006/04/error_code_vs_e.html)由[达米恩·卡茨](http://damienkatz.net/)拍摄，仅仅为了视觉插曲就值得了。

事实上，它不是关于错误代码/异常，而是关于当事情确实出错时你该怎么做——如何“摆脱”你所处的困境？

在 PHP 爱好者面前，需要引用的相关文章…

> 在语言和工具中，我们需要的是能够轻松地隔离我们的更改，以备不时之需，这样就看不到不完整的更改(要么全部，要么什么都没有)。我们不能否认，科学技术可以在任何时候击中风扇。当事情出错时，我们需要让检测变得容易，一旦出错，让做正确的事情变得简单。
> 
> 信不信由你，在 PHP 中，我们已经有了它的雏形。是的，又老又笨又简单的 PHP。在 web 服务器上，PHP 脚本没有共享状态，所以 PHP 脚本的每个实例都在自己的逻辑内存空间中运行。这些脚本不保持持久状态，所以每个脚本都像雏菊一样新鲜地开始，幸福地不知道前一次执行时发生了什么。
> 
> PHP 中唯一的共享状态存在于数据库级(或文件级，但不要去那里)，如果你在一个事务中提交所有的更改，你就基本上解决了甲板构建问题。您的代码可能无法更好地成功完成更新，但失败是孤立的，导致失败的所有操作都被遗忘，它不会在应用程序中引起进一步的问题或不一致。

这是整个[再一次没有共享任何东西](http://www.oracle.com/technology/pub/articles/php_experts/rasmus_php.html),但重点是交易而不是扩展。
