# 不要被人踩在脚下，已经设定了一些界限

> 原文：<https://www.sitepoint.com/stop-getting-walked-on-and-set-some-boundaries-already/>

作为两个男孩的父亲，我一直对他们性格上的差异感到惊讶和困惑。我的大儿子倾向于随波逐流，但我的小儿子总是觉得有必要突破界限。总有一天，他会长大成为你的客户。

如果他来了，你打算怎么对付他？作为父母，我的工作是界定界限，让他们在界限内玩耍。作为一名网络顾问，你的工作并没有什么不同。

我是在建议你像对待孩子一样对待你的客户吗？在一定程度上，是的。孩子们对他们周围的世界没有完全的了解。当任其自由发展时，他们往往会做出愚蠢的选择。同样，当客户踏入 web 设计和开发的世界时，他们对这个世界是如何工作的有一个不完整的描述，并且可能有不切实际的期望。比尔·科斯比曾经说过，有 200 个活跃的两岁小孩，他可以征服世界。一个过于活跃的客户端就可以让您的系统不堪重负。是时候设定一些界限了。

## 不要假设你的客户会阅读合同

界限和期望的生死取决于一份书面合同。不要以为客户会看你的。

> 没看就签合同？说真的！你在法律上必须遵守我的条件。确保你理解他们。你信任我很好，但是当你做了协议不允许的事情时，我们仍然会有一个尴尬的谈话。
> 
> —《如何解雇客户》一书的作者米歇尔·高尔

你可以通过与潜在客户坐下来，如果可能的话面对面，在他签字之前讨论合同的每一点来避免“尴尬的谈话”。

这并不像听起来那么伤脑筋。让我们来看看这个问题:即使你的客户延迟发送内容给你，你也能得到报酬。你可以通过建立一个付款时间表来完成，而不是将付款附加到生产里程碑上。以下是如何与你的客户进行对话。

首先，讨论项目时间表，让客户知道预计的完成时间是 60 天(或者你同意的任何时间框架)。从这里，很容易过渡到为什么项目会落后于进度的原因:

潜在客户先生，我经常遇到的一个问题是，忙碌的客户最终会延迟完成他们的网站，因为他们花太长时间给我发送书面副本和图像。这通常不是故意的，但当这种情况发生时，我最终会等上几周甚至几个月才能拿到工资。

为了防止这种情况，我的合同规定，最后付款应在 60 天内。这意味着，如果你没有给我发送我需要的内容，并且你的网站因此没有完成，你仍然同意支付我。我解释清楚了吗？

网站设计的每一个方面都有可能被你的客户误解、误解或曲解。这里有一个客户的例子，他认为我没有讨论的东西包括在内:在他的办公室电脑上设置所有的域电子邮件帐户。他不知道如何，毕竟，我为他注册了域名。(好痛苦；我讨厌做那种事，尤其是当我被免费卡住的时候。)

但是有些人看到了问题，有些人看到了机会。从那时起，我问每个客户他们是否能在内部处理这件事。对于那些需要帮助的人，我分包给了我表弟的网络业务(略微提高了他的费用)。在那之后，如果他们需要更多的工作，他就把他们作为客户。

此外，当我开始问同样的关于文案写作的问题时，我发现客户会额外付费让我提供这项服务。

考虑一下我们在本系列中讨论的一些“滥用”行为:范围蔓延、无休止的设计修改或技术支持、客户故意拖延付款、事后修改客户站点。你可以设置什么参数来最小化或防止这些，同时仍然为客户提供足够的空间来完成他想要的事情？为每个客户起草一份保单，并写下你将对客户说的话。坐下来解释每一个要点从来都是一种积极的体验。我发现对方会欣赏这一点并完全理解。此外，你刚刚为他或她节省了 45 分钟阅读和翻译你的法律术语的时间。
