# Web 2.0 连通性

> 原文:[https://www.sitepoint.com/web-20-connectedness/](https://www.sitepoint.com/web-20-connectedness/)

以下内容转载自《科技时报》第 129 期。

假期让我有机会接触一些新产品和服务，这些产品和服务已经在我的“尝试”清单上有一段时间了。虽然我对他们都很兴奋，但我对他们的分离感到沮丧。

随着每个人都在谈论“Web 2.0”，似乎网络上的每个聪明人都在专注于构建下一个杀手级应用程序。小巧而专注似乎是这个月的流行趋势: [Gmail](http://mail.google.com/) 擅长处理电子邮件， [Flickr](http://www.flickr.com/) 钉照片，CalendarHub 和它的同类公司的日历几乎被打败了。但是，当我们接受这些不同的服务时，我们进一步细分了我们日常依赖的数据，使得我们在需要时更难一起使用它们。

如果你相信权威人士(包括你的)的话，开放性和标准遵从性是定义任何真正的 Web 2.0 稳定成员的两个特征。您放入 Web 应用程序的任何数据都应该可以以可移植的格式再次取出，并且应该可以通过记录良好的 API 供其他应用程序访问。

至少在野外，开放 API 的事情似乎正在发生。这些接口中的一些没有得到应有的良好记录，但是如果你挖掘得足够深入，它们通常就在那里。“网络混搭”文化，最常见的形式是基于谷歌地图的第三方应用。但是有多少商业级的应用程序和服务支持这种互操作性呢？所有的大玩家似乎都专注于建设*新的*服务，而不是寻找更好的方式来一起使用现有的服务。

这是我看到的 Web 开发人员即将面临的最大挑战。2006 年会是解决这个问题的一年吗？这里只是想到的几个领域，它们有着比我们今天看到的更大的联系空间。

## 您的数字身份

你在各种网站上有多少账号？如果你像我一样，你早就数不清了。尽管我尽了最大的努力，我仍然经常遇到这样的情况，我的用户资料中包含了一个已经过期四年的家庭住址的详细信息。对我来说，这已经到了我甚至不记得*我是否在一些网站上有账户的地步，更不用说记得登录细节了。*

这已经是一个老问题了，大公司以前也尝试过解决这个问题。微软的 Passport 成为许可费和平台锁定的受害者，这迫使该服务只能在微软网站上使用。

但是这个问题将会变得更加重要。随着你在在线服务上投入了越来越多的个人数据，你会希望能够在这些服务之间共享这些数据，而不与世界其他地方共享。例如，如果你和你的配偶可以查看彼此的日历，即使他们由两个不同的在线服务托管，而不必将日历数据公开到整个网络，那就太好了。

统一身份验证，以及允许服务在适当的时候访问彼此的私有数据的框架，是许多服务实际可用性的关键。这种需求将很快达到临界质量，我希望这将有助于推动一个正在等待的羽翼未丰的解决方案。

## 统一出版

本世纪初，许多人担心互联网会成为纯粹的商业圣地，但现在它已经发展成为一个个人表达的平台。博客正在进入主流互联网用户的手中，可以想象到的每个主题都有论坛，大量的社交网络应用为人们提供了新的表达方式，通过[照片](http://www.flickr.com/)、[音乐](http://www.last.fm/)、[链接](http://del.icio.us/)等等。

虽然这些个人在线出版的每一种形式都得到了主流的采用，但将这些不同形式的表达以一种有凝聚力的方式结合在一起的能力迄今仍停留在 Web 开发人员的巫术中。

如果有人读了我的一篇他喜欢的文章，他应该能够点击进入我的个人网站，看到我公开写的所有东西，这是我希望他能够看到的。我的个人博客条目、我的工作博客条目、我的 [last.fm](http://www.last.fm/) 日志条目、我在网络公共论坛上的帖子、我的 del.icio.us 书签、我的 Flickr 照片……所有这些都应该可以在一个“我发布的内容”视图中访问。

现在，如果你是一个像我一样的 Web 开发人员，你会发现这在今天通过 RSS 提要、Web 服务和博客软件扩展的组合是最有可能的。但是*每个人*都应该有一个这样的个人表达中心，我希望用不了多久就会有一个时髦的解决方案让非技术用户也能接触到这种东西。

## 共享数据(和元数据)

我已经提到过，到目前为止，在线服务不太擅长彼此共享数据。基础以快速增长的开放 API 星系的形式存在，但是新的应用程序的出现比将它们一起使用的解决方案要快得多。尽管如此，我相信数据共享的问题会慢慢得到解决。我对元数据不太确定。

这里，标记是一个很好的例子。较新的服务让您可以为数据项分配多个标签，而不是构建严格的类别层次结构或文件夹结构来存储数据。我到处都有标签。我在 del.icio.us 上标记书签，在 Gmail 上标记邮件。我在 Flickr 上标记我的照片。我在 last.fm 中标记我的音乐，但在任何情况下，这些标记都被锁定在各自的应用程序中。当书签和电子邮件与我的时事通讯有关时，我可能会在 del.icio.us 和 Gmail 上分别标记“techtimes ”,但我没有地方可以查看我用“techtimes”标记的所有内容。

许多成功的新 Web 应用程序通过用各种元数据装饰静态数据，使静态数据变得更加生动。但是总的来说，当您从应用程序中取出数据时，您会丢失元数据。随着我们生活中的元数据变得和它所描述的数据一样有价值，开放性和可移植性对我们的元数据和我们的数据同样有价值。

同样，在大多数情况下，这是一个有很多空闲时间的网络极客可以解决的问题，但不久之后，主流用户将需要一个干净的解决方案。

因此，在你寻找下一个杀手级应用之前，停下来想想现有杀手级应用的一些问题，或许你可以解决这些问题。我敢打赌，这些问题的解决方案本身就是杀手级应用。
