# CGI 安装疑难解答

> 原文：<https://www.sitepoint.com/cgi-installations/>

安装该脚本有问题吗？我收到的大约 80%的电子邮件都问了同一个问题:我如何安装这个脚本？它对我不起作用。我不能说在你上传脚本之前应该采取的以下步骤总是有效的，但是在大多数情况下，它们是有效的。

##### 第一步:你的文件是什么格式？

如果你在 UNIX 服务器上运行你的站点，而你在 Windows 上做你的脚本配置，你将不得不尝试把它转换成 UNIX 格式。DOS 是 Windows PC 系统的主要部分之一，它使用回车和换行符来结束文本文件行。另一方面，UNIX 只使用换行符。

基本上，如果一个 Perl 程序是 DOS 格式的，它就会崩溃，不能在 Unix 系统上运行。如果你在编译一个 C 程序，它根本不会正确编译。以 ASCII 格式上传将消除这个问题。

##### 步骤 2:您是否正确地设置了 Perl 的路径？

大多数服务器的 Perl 都位于以下位置:

#!/usr/bin/perl

或者

#!/usr/local/bin/perl

如果这些位置不正确，请尝试在 telnet 提示符下键入“which perl ”(如果可以访问的话)。否则，请联系您的系统管理员。如果你想知道#！在位置之前，这是“shebang 行”，或通知服务器文件需要 Perl 运行的行，以及 Perl 的位置。

##### 第三步:你解压文件正确吗？

一位去 TheScripts.com 的游客向我指出了这一点:

下载脚本后，如果它是以 TAR 模式压缩的，您必须正确地解压缩它。所有 Windows 用户都应该记住，WinZip 默认为选项/配置/杂项部分下的选项“Tar Smart CR/LF”。禁用这个。启用后，它将无法正确解压缩。

##### 步骤 4:编辑配置文件时要小心

如果脚本要求您编辑您的配置文件，那么您要非常小心。大多数遇到安装问题的人要么以二进制模式上传，要么弄乱了配置文件。

记住要严格遵循作者的指示，因为即使少了一个括号也会把整个脚本搞砸。插入你的变量或任何你需要编辑的地方，只在那里。格式也应该保持不变。

##### 第五步:正确设置文件权限

所有的 CGI 程序都要求您正确设置它们的权限，以便正常运行。因此，它们必须设置为权限“755”这是可执行文件的权限。通过 FTP 进入他们服务器的人可以很容易地使用他们的 FTP 程序来设置权限。通常，这只是右击任何有问题的文件。Telnet 用户必须远程登录到他们的服务器，进入 CGI 文件所在的目录，并键入提示“chmod 755 filename.pl”。

有些文件需要不同的权限。作者将向您说明那些权限必须是什么。
