# 开发人员、设计师、导演、制片人——为什么你的职称不重要

> 原文:[https://www.sitepoint.com/job-titles-do-not-matter/](https://www.sitepoint.com/job-titles-do-not-matter/)

【2008 年独立网页设计调查的统计数据已经发布。这项调查收集了近 30，000 名在网络行业工作的人的信息，在大多数情况下，结果并不令人惊讶。典型的被调查者是居住在西方国家，年龄在 19 到 44 岁之间，拥有大学学位的白人男性。他可能是独立承包商或雇员，每周工作 30 至 49 小时，年收入高达 8 万美元。三分之二的人也有个人网站或博客(虽然我希望他们问多久更新一次！)

接下来是一个棘手的问题:*“你的职位是什么？”* …

几乎 28%的人回答“开发者”，13%的人回答“网页设计师”，8%的人回答“设计师”，接下来是各种冠冕堂皇的角色，如网站管理员、创意/艺术总监、网页制作人、信息架构师、可用性顾问、营销人员、教育家和易访问性顾问。最后，我们来到“其他”——却发现它占了总数的 26%。看起来，职称的差异比完成调查的人更大。

过去这要容易得多:你在 It 部门工作，除此之外没人关心太多。网络上似乎出现了太多的职位名称，但很少有人认识或理解。什么是站长？一个网络制作人到底是做什么的？与信息架构师有什么不同？

许多聪明的公司意识到，员工通常更关心自己的头衔，而不是薪水。当你可以让鲍勃成为超文本技术系统公司的副总裁时，为什么要大幅加薪呢？他的孩子可能会挨饿，但他会成为所有朋友羡慕的对象。

那么头衔重要吗？也许他们曾经有过，但现在已经到了职位头衔对大多数客户、潜在雇主甚至同事来说毫无意义的阶段。网络工作需要各种各样的技能，所以也许我们应该放弃给员工贴标签？

那么…
你的职位是什么？它准确描述了你的角色吗？需要向客户解释吗？这对你真的很重要吗？
