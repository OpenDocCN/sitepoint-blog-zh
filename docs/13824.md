# 蓝图 PHP 应用？

> 原文:[https://www.sitepoint.com/blueprint-php-application/](https://www.sitepoint.com/blueprint-php-application/)

这里有一个问题…如果有人来问你“我如何在 PHP 中组织自己？”，你说呢？特别是，如果他们是一个有经验的其他语言的程序员，但仍然在学习 PHP。

暂时忘记[框架](http://del.icio.us/tag/php+framework):事实上*尽管有*框架，PHP 应用程序仍在被编写。我真正想问的问题是，你会把哪些*应用*作为如何**使用** PHP 的例子。

这些天，我发现自己在 MySQL 的 bug 跟踪应用程序 [Eventum](http://dev.mysql.com/downloads/other/eventum/) 上给人们指点迷津。我*个人*对 Eventum 的写作方式有一些“问题”,但这些真的只是我个人的口味。Eventum 对我来说代表了一种*安全的*方法来构建 PHP 应用程序——合理地组织和结构化，但仍然是一个工作的应用程序，处理真实/常见/复杂的用例，并且[明显地](http://bugs.mysql.com/)可扩展。用于构建 Eventum 的工具集及其相应的学习曲线也是可以接受的；除了 PHP(和内置的 API)，它基本上只是 [Smarty](http://smarty.php.net) ，这是有据可查的。

鉴于有如此多的应用程序是用 PHP 编写的，其中一些非常受欢迎，有没有什么应用程序是你乐意给人们提供灵感的？不管你的个人品味如何，你认为哪些应用程序是做 PHP 的好蓝图？(如果你有时间)为什么？
