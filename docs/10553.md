# W3C 和“HTML5 还没准备好”的反弹

> 原文:[https://www . site point . com/the-W3C-and-the-html 5-not-ready-backlash/](https://www.sitepoint.com/the-w3c-and-the-html5-isnt-ready-backlash/)

以下 InfoWorld 的文章昨天像野火一样传遍了整个网络— [W3C:暂缓在网站中部署 html 5](http://www.infoworld.com/d/developer-world/w3c-hold-html5-in-websites-041)。这个故事是由 W3C 官员 [Philippe Le Hégaret](https://www.w3.org/People/LeHegaret/) 的评论引发的。尽管我怀疑 Le Hégaret 的许多引语被选择性地使用或断章取义，但社区还是一片哗然。即便如此，让我们仔细阅读文章中的每一个陈述…

> 我们现在面临的问题是已经有很多人对 HTML5 感兴趣了。

同意。也有很多炒作和困惑。对大多数人来说，HTML5 意味着“网络的未来”。你可以引用一个 5 年前关于“Web2.0”或“Ajax”的引语，并将其与今天的“HTML5”互换。主流媒体既不了解也不关心技术现实，试图解释 CSS3 是没有意义的！= HTML5。

> 部署它还为时过早，因为我们正面临互操作性问题，包括设备上视频的差异。
> 
> 我认为它还没有准备好投入生产，尤其是 W3C 仍然会在 API 上做一些改变。真正的问题是我们能让[HTML5]跨浏览器工作吗？目前，情况并非如此。

报道援引 Le Hégaret 的话说，不同的浏览器是不一样的。这是新闻吗？自从网络出现以来，我们一直在与差异作斗争。

有 4 个主要的渲染引擎:Trident，Gecko，Webkit 和 Presto。每个特征与另一个特征完全匹配的可能性为零。浏览器的变化是网络生活中的现实——这不是 HTML5 的问题。

> HTML5 被视为游戏规则的改变者。公司现在可以在他们的应用程序或内部网中部署 HTML5，在内部网中可以控制渲染引擎，但在互操作性是一个问题的开放 web 上，情况就不同了。

互操作性一直是并将永远是一个问题。支持一种浏览器很容易，但最好的开发人员可以让网站在任何设备上工作，无论是 Firefox 4.0、屏幕阅读器、移动设备、Lynx 还是 IE1.0。这并不总是值得付出努力，但 HTML5 不会改变这一过程。许多网站使用 XHTML1.0 或 1.1，但 IE8 及以下版本都不支持。

当然，开发者要用常识。如果你正在开发一个 90%的用户都是 IE6 用户的系统，不要指望 HTML5 `video`标签会起作用。其他元素的问题会更少，比如`header`。

> HTML5 规范本身支持视频和画布 2D。但其他技术，如 CSS 和 MathML，与 HTML5 一起被认为是“开放网络平台”的一部分，即使它们没有被实际的规范所涵盖。HTML5 规范引用了 SVG。

这句话最能说明问题。在文章中间是一个很长的没有意义的陈述，但是它在表达什么观点呢？Le Hégaret 是说 CSS 和 MathML 可以使用还是不可以使用，因为它们与 HTML5 是分开的？还是记者只是从采访中随便摘录了一句话？

> 我们不会很快让 Flash 退役。所有网络客户端都支持 HTML5 还需要几年时间。IE6 至今仍在网络上使用，它已经有 10 年的历史了。
> 
> 然而，随着时间的推移，HTML5 将成为网站的标准，你会看到越来越少的网站使用 Flash。

啊，古老的说法*“html 5 是闪存的替代品”*。这是苹果开始的吗？如果没有，他们市场部一定爱死了！

我不确定为什么这篇文章需要提到 Flash，但是 HTML5 不是 Flash 的替代品！标签和新的 CSS3 特性可能会减少对 Flash 的依赖，但这并不意味着 Flash 不能或不应该被使用。在未来的许多年里，Flash 将提供最可靠的跨浏览器视频媒体和游戏平台。

> HTML5 也不支持数字版权管理。这意味着一些视频制作者不会在没有这种保护的情况下在 HTML5 中部署他们的视频。
> 
> 如果我们要开发一个开放的 DRM 解决方案，它会在两天内被黑客破解。然而，在某个时候 HTML5 中有 DRM 的可能性，但是目前还不在计划中。

这是真的吗？根据定义，开放标准是开放的。DRM 从来都不是 HTML 的特性。如果您想要控制媒体分发，请使用 Flash 或 Silverlight 等替代工具，就像您现在所做的那样。

> HTML5 目前也缺乏创作工具。

记事本呢？的确，很少有所见即所得的编辑器生成 HTML5，但也很少有人生成像样的 HTML4。最好的开发人员使用文本编辑器或 IDE。可能会有一些颜色编码的怪癖和验证困难，但是没有什么可以阻止您今天编写 HTML5 代码。

## 我们要去哪里？

Web2.0 和 Ajax 是模糊的术语，而 HTML5 指的是一组具有规范和版本号的真实技术。InfoWorld 的文章相信，在某个时候，HTML5 将“完成”并准备投入生产。事实并非如此。

浏览器不断发展，新功能最终会被他人复制、调整或拒绝。如果至少有两家厂商同意，一个单一的特性可能会成为 W3C 规范的一部分——不管是 HTML5、CSS3、SVG、MathML 还是其他什么。然而，W3C 推荐并不自动意味着该工具将出现在其他浏览器中。幸运的是，当我们需要时，总会有变通办法、退路或填补空白。

HTML5 不是目的地，而是旅程。我们不确定我们要去哪里，但至少所有的浏览器供应商都在同一辆公交车上。
