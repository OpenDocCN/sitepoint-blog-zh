# 如何举办一场成功的设计竞赛

> 原文：<https://www.sitepoint.com/how-to-run-a-successful-design-contest/>

来自 conceptualist.com 的萨哈尔·萨里德知道如何举办一场设计竞赛。

上周，他宣布了通过他在 SitePoint 举办的标志设计竞赛获得的 2500 美元奖金的获胜者(获胜者是约翰·维克)。

有趣的是，看看萨哈尔作为比赛举办者所遵循的过程，以确保他获得的奖励物有所值。例如:

*   他提供了一大笔现金奖励，保证社区里最好的设计师会引起注意。事实上，这是迄今为止授予单个设计竞赛的最大奖项。
*   他在最初的简报中清晰而详细。
*   他对每位设计师提交的作品都给予了及时而全面的反馈。事实上，在 745 个反馈帖子中，有 576 个是萨哈尔的——大赛的 212 条评论中也有很多——总计 313，872 个字符或大约 52，000 个单词！
*   他坚持要参与这个过程。这是许多设计竞赛的诽谤者引用的一个失败因素:客户沟通被排除在外，导致工作质量下降。通过要求每一位设计师在每一次提交的作品中加入他们的基本原理，他让他们参与到一场讨论中，讨论某个概念如何以及为什么是一个很好的契合点，并且可以与设计师一起帮助塑造和完善这个想法。
*   他在自己的网站上推广比赛，各种 [博客](http://www.conceptualist.com/2007/11/29/sitepoint-design-contest-is-out-of-this-world/) [帖子](http://www.conceptualist.com/2007/11/21/conceptualistcom-sitepoint-logo-contest-take-ii/)报道比赛的[状态](http://www.conceptualist.com/2007/12/07/and-the-winner-is/)。事实上，人们对结果非常感兴趣，以至于其他人开始制作关于它的视频。
*   他提供了 1250 美元的推荐奖金，以鼓励其他设计师注册并提交他们的概念。

当然，许多比赛举办者可能没有萨哈尔那样的预算，或者他们有吗？如果你把这个奖看作是一笔营销费用，那么萨哈尔用他的营销费用换来的网站的病毒式曝光率可能比他通过更传统的努力得到的要好得多！

对于未来的比赛举办者来说，这无疑是一个值得思考的方法。

顺便说一下，那些对 SitePoint 设计竞赛的发展感兴趣的人，包括关于不断增加的增强可用性、保护设计师等功能的信息，应该关注 [SitePoint 开发博客](https://www.sitepoint.com/blog/)。
