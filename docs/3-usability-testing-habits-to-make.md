# 3 要养成的可用性测试习惯

> 原文：<https://www.sitepoint.com/3-usability-testing-habits-to-make/>

对于整天与其他人一起运行测试的人来说，有时将镜头转向我们自己是很重要的。以下是自我提升的三个步骤:

## 扪心自问

关于你的测试参与者，你能告诉我些什么？你可能会知道他们的职业、性别、年龄组、说话方式和网络知识。如果测试是面对面的，你会观察他们的服装选择、发型、眼神、气味(可能是古龙水味道有点重？)，面部表情，姿势，肢体语言，甚至可能还有他们开的车。

这些信息非常有用，你应该密切关注所有这些信息。请注意，当你这么做的时候，你的高效大脑正在使用[试探法](http://en.wikipedia.org/wiki/Heuristics#Psychology)来创建关于人们的假设，这些假设可能是真的，也可能不是真的。你的假设可能会受到过去与相似的人相处的经历、你那天的心情、你有多累，或者你认识的有同样刺耳声调的人的影响。最重要的是，他们可以被你想看到的结果所影响。

批判性地思考你根据自己的偏见得出的结论，将有助于让事情走上正轨。这很重要，因为这些假设有助于人物角色的创建，而人物角色会影响网站的内容、设计、布局和文案。

## 知道什么时候保持沉默

我发现，当有人明显对界面感到沮丧，并向我寻求帮助和指导时，保持沉默尤其具有挑战性。看着别人挣扎是很痛苦的，但是那些时刻对你的工作来说是无价的。一旦你发现哪怕是一个细微的线索，你就已经扭曲了你的结果。抵制帮助你的参与者完成任务的诱惑，代之以几句肯定的话，并提醒他们这是一个正在接受测试的系统。

如果你是一个注重人际关系的人，你会遇到真正合得来的人。很容易让你们的关系变得融洽，让私人谈话的空间溜进测试中。我发现这里或那里的注释有助于在测试过程中保持这种联系，这是很有帮助的，但是更多的注释会破坏测试并占用宝贵的时间。

## 嗯，是个健谈的人

创造融洽的关系和建立一个专业但友好的环境是一门学问。心理学家已经发现了四种会削弱你说话能力的语言形式:

*   犹豫:像“嗯”和“你知道”这样的词语会降低你的自信
*   **免责声明:**“我并不是这个话题的真正权威，但是……”或者“我知道这听起来很奇怪，但是……”这贬低了你将要说的话
*   限定词:“有点儿”、“有点儿”或“我猜”，它们会减少自信，增加怀疑的成分
*   **附加问题**:“你最好在完成这些任务的同时大声说出你的想法，你不觉得吗？”或者“这是最好的方法，不是吗？”

当你明显感到舒适、自信、享受你的工作时，每个人都会感觉更舒服。另外，你最终会更加享受它！同时，确保使用日常用语，避免行业术语。你的参与者越能像平常一样自然地使用网络，你的结果就越好。

好处是，养成这些习惯不仅会帮助你进行更好的可用性测试，还会在商业的各个领域帮助你。意识到你的偏见，知道什么时候不说话，自信地交流永远不会过时！
