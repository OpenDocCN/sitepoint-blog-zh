# 我们 ASP.NET 极客没有失去任何东西

> 原文：<https://www.sitepoint.com/we-aspnet-geeks-have-lost-nothing/>

非常非常棒的[亚音速持久性框架](http://subsonicproject.com/)的创建者 Rob Conery 写道，他已经忘记了很多 HTML 和 javascript。他没说谎。我接受了他的小小挑战，惨败。然后，我再次意识到我是多么高兴，我已经很久没有与手工表单处理作斗争了，也没有实现它。虽然我可能因为成为 ASP.NET 迷而失去了一些东西，但同时我获得了许多非常非常强大的工具来制作复杂的交互式 web 应用程序，而不必太担心如何连接 HTML 表单。

以下是 Rob 挑战的更现实版本:*

创建一个表单来收集事件的注册信息。我们要求你抓住:

*   完整的联系信息(姓名、地址、电子邮件等)
*   完整的人口统计信息(行业、雇主规模、购买力、主题焦点)
*   会话注册选择
*   酒店预订信息。
*   用户不应该看到一个巨大的表单。它应该被分解成一口大小的块。
*   用户应该能够前进和后退，通过一系列的形式和编辑他们的心血来潮的信息。
*   在将信息保存到数据库并通过 web 服务保存到单独的 CRM 系统之前，向用户显示一个确认屏幕，显示他们的所有选择。

这个应用程序必须在明天早上之前准备好投入使用。

*这真的真的很现实。上周我写了两个这样的向导。

你会如何使用“经典”技术解决这个问题？你要花多长时间才能让所有手工处理的邮件数据变得万无一失？

就我个人而言，与 Rob 不同，我并不觉得与我的“更酷”的兄弟们脱节。我觉得我有一个更好的工具集来制作复杂的、交互式的 web 应用程序，而不是那些仍然手工处理 html 表单的人。做一个. NET 人是可以的。
