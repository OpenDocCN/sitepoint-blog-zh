# 史努克的点心

> 原文:[https://www.sitepoint.com/snack-on-snooks-smacss/](https://www.sitepoint.com/snack-on-snooks-smacss/)

CSS 大师乔纳森·斯努克是 SitePoint 和[播客](https://www.sitepoint.com/podcast-125-taming-the-stylesheet-with-jonathan-snook/)的常客。他的最新项目解决了一个很少开发人员考虑到为时已晚的主题:CSS 架构。如果你曾经努力管理一个有几十个布局和小部件的大型复杂网站，我建议你阅读[CSS](http://smacss.com/book/)的可扩展和模块化架构——或简称 SMACSS(读作“smacks”)。

它不是一个操作指南、框架、代码片段库，甚至也不是一套建议。Jonathan 记录了他在为大型项目构建 CSS 代码时的观察。这本在线书籍推荐了一些技巧，可以让你的代码更有条理，更容易维护。这是关于一个主题的独特方法，在其他 CSS 指南中很少讨论。

[第一部分](http://smacss.com/book/fourtypes)确定了 CSS 的四种主要类型:

1.  :主要的 HTML 元素，如主体、表单标签、链接等。
2.  **[布局](http://smacss.com/book/type-layout)** :页面如何分成各个部分，包括页眉、页脚、侧边栏等。
3.  **[模块](http://smacss.com/book/type-module)** :离散的、可重用的、模块化的组件，如导航栏、产品列表、联系人表单等。
4.  **[状态](http://smacss.com/book/type-state)** :模块或布局在特定状态下如何变化，如出错、隐藏、展开、活动、不活动等。

本书以这些概念为基础，讨论了[主题](http://smacss.com/book/layering)、[深度适用性](http://smacss.com/book/applicability)、[管理状态变更](http://smacss.com/book/state)和[代码格式化](http://smacss.com/book/formatting)。

如果你已经开发了一段时间的布局，你可能会认识到许多模式。你可能不同意乔纳森的一些建议。然而，这本书是令人放心的简洁，你将开始考虑自己的方法来可伸缩的 CSS。

推荐。
