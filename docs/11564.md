# 头脑风暴:突破障碍指南

> 原文：<https://www.sitepoint.com/brainstorming-guide/>

头脑风暴是开发新想法、制定行动计划和跳出框框思考的宝贵工具。但是，如果你倾向于成为一个结构化的思考者，让你的大脑进入必要的自由形式模式以有效地进行头脑风暴可能是一个挑战。

与一群人进行不受限制的头脑风暴对于那些能够让他们的大脑采取必要路径的人来说是非常高效的，但也有可能以一种更受控制的方式成功地独自进行头脑风暴。

这个关于个人头脑风暴的三部分系列将为成功的头脑风暴会议提供结构。

## 准备好

以下是你开始头脑风暴所需要的东西:

*   一张纸和一支笔，或者电脑上的一个空白文档
*   安静的环境
*   一段时间(20 分钟是一个很好的起点),很少分心
*   计时器或闹钟
*   清晰专注的头脑

## 找出一个问题

成功的头脑风暴最重要的部分之一是限制你的焦点。尽管这看起来对头脑风暴过程有反作用，但是有必要定义你试图解决的一个问题或挑战。

这不仅使自由思维的整个过程不那么令人生畏和令人不知所措，而且还让你以更有成效的方式将你的创造力集中在一个有形的问题上。

## 设定基本规则

这一步只是确定你的挑战的局限性。例如，你可能不想要一个阻止你立即采取行动的解决方案，任何需要不切实际投资的东西，或者一个可能被认为不道德的解决方案。

虽然你应该在开始前把这些规则写在纸上，但不要在头脑风暴过程中专注于它们。只要确定了它们，您就能够在流程的评估阶段消除它们。

## 选择一个流程

有很多方法可以让你自己进行头脑风暴。不要试图一次涵盖所有选项，而是为每次头脑风暴会议选择一两种方法(是的，你可能需要不止一次会议，以便开发足够的工作想法，让你进入下一步)。

这里有几个头脑风暴过程可以考虑:

*   脑残——听起来确实如此。你让你的思维起飞，写下你想到的一切，忽略格式、语法、标点符号等等。这是一个很好的开始方法，因为它可以帮助你理清思路。
*   **列表**–列表可以帮助你制定与你所选问题相关的个人任务。我写作时经常使用这种方法，在我开始填写细节之前先写一个大纲。根据你的主题，你可能想要创建多个列表，将你的问题从一般观点分解到更具体的焦点。
*   **挑战**–头脑风暴的挑战方法包括写下所有与你的主题相关的假设，然后挑战它们。为你的每个假设回答这样的问题:为什么过程必须这样做？你有什么选择？别人会怎么做？

这只是一个简短的列表，给你一个起点。在第二篇文章中，我将更详细地介绍另外两种头脑风暴方法——[cubing 和 webbing](https://www.sitepoint.com/cubing-webbing-methods-brainstorming/) 。

## 评估你的结果

大多数头脑风暴会议都会产生许多想法，这些想法支持、定义或者旨在解决你的焦点问题。但是你很可能会有很多不相关的、不适用的或者跑题的想法要思考。评估和清理阶段是成功的头脑风暴会议的基本组成部分。
