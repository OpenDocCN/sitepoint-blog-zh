# 你的时间去哪里了？

> 原文：<https://www.sitepoint.com/where-does-your-time-go/>

在过去的几年里，我注意到一个令人担忧的趋势——许多 web 开发人员不能有效地跟踪他们的时间。当然，有很多经验丰富的网页设计师和程序员非常擅长记录和跟踪他们的活动。但是惊人的百分比使用不完整或不准确的方法记录他们的时间，或者根本不记录他们的时间。

为什么这如此重要？答案很简单。如果你能有效地跟踪你的时间，你就能有效地衡量你的利润。毕竟，作为一个人，你最大的财富是你的时间，你已经决定利用这些时间来赚钱。如果你有一份工作，你会用你的时间来换取一笔预先确定的钱。然而，你自己创业的决定意味着你在一个小时内赚多少钱是没有限制的。为什么不测量这个“每小时利润”并确保它随着你的业务增长而增长呢？

即使你已经知道你的时薪，了解你在每个特定项目中获利多少也是非常有用的。通过仔细跟踪你的时间和费用，你可以确定扣除营销和运营费用以及你的时间后的实际利润。一旦你清楚地了解你的项目到底有多赚钱，你就会做出更好的决策。你甚至会发现更大的项目并不总是更好。

例如，以下面三个假设项目为例:

项目 1:项目总价值 5000 美元，总费用 600 美元，总投资时间(包括销售、行政、法律和实际工作！)50 个小时。

项目 2:项目总价值 8000 美元，总费用 900 美元，总时间你投入 90 小时。

项目 3:项目总价值 10，000 美元，总费用 1，900 美元，总时间 113 小时。

想象一下，你可以用永无止境的项目来填满你所有的可用时间——你希望它们与项目 1、2 或 3 相匹配吗？

答案很简单，因为你知道为每个项目服务需要多少小时，然后你就可以确定每个项目每小时的利润。通过计算项目总价值(毛)减去费用(净)，然后除以您投入的小时数，我们可以确定我们工作的实际小时费率。

结果是这样的:

项目 1:88 美元/小时

项目 2:78 美元/小时

项目 3:71 美元/小时

差别很大！如果你选择了项目类型 1 而不是项目类型 3，你会随着时间的推移获得更多的利润，因为你可以以更高的速度处理更多的项目，尽管每个项目的价值更低。

如果可以选择，我会接受两个像#1 这样的项目，而不是一个像#3 这样的项目。项目总价值是相同的(10，000 美元)，但利润将超过 700 美元。最好的部分是，你增加的利润将在更短的时间内获得！处理两个像#2 这样的项目需要 100 个小时，处理项目#3 需要 113 个小时。正如你所看到的，计算小时工资给了你更好的决策权，这是一笔好生意。

这只是精确时间跟踪价值的一个例子。在制定项目提案、确定工作日程的优先顺序、决定追求哪些客户，或者决定哪些工作外包、哪些内部完成时，了解你的小时利润也非常有用。

一旦你开始精确地记录时间，结果可能会让你大吃一惊！当使用小时利润比较法时，许多开发人员震惊地发现他们的大型项目实际上是利润最少的。其他开发人员将会了解到，他们对一个项目平均需要多长时间的内部估计有 50%-100%的偏差(这是很常见的)。

然而，几乎所有的开发人员都将受益于对哪些任务和项目使他们获得绝对最大利润的清晰理解——你甚至可能因此改变你的商业模式。
