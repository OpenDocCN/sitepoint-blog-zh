# 吸取的教训

> 原文：<https://www.sitepoint.com/lessons-learnt/>

以防有人有任何疑问，昨天的条目绝对是一个笑话。早在 90 年代末，大卫·西格尔确实是一个互联网传奇，他开创的技术将网络带进了一个视觉上吸引人的网站的商业时代，挑战了最初 HTML 规范的限制。这些技巧已经过时了，现代网络带来了新的基于标准的技术，允许我们创建好看的网站而不牺牲底层代码的质量。

也就是说，在阅读大卫的档案时，可以学到很多关于网络的知识。事实上，大卫的很多作品都很有前瞻性。看看这些引言:

> HTML 是一种标记语言，而不是布局语言。它并不意味着向观众展示一幅画。它并不意味着容易阅读。它应该是可访问的。审美不是考虑因素。但是有些人想要更清晰有效的交流，对我们来说 HTML 是非常原始的。我们不需要屏幕上的 PostScript，我们需要的是更具视觉风味的 HTML。在那之前，我们有变通办法。

[来源:单像素 GIF 绝招](http://www.dsiegel.com/tips/wonk5/single.html)

> 在我们进入下一代网站设计之前，表格标签必须是我们的朋友。我们越是学会控制表格，我们就越能打破旧 HTML 的束缚，让我们的页面看起来更好。如果我们必须将所有文本放入表格中——这并不是 HTML 的设计者们真正想要的——我们会的。

[资料来源:黑仔表格-概述](http://www.killersites.com/killerSites/tutorial/index.html)

CSS 是大卫所说的 HTML 视觉风格的体现。事实上，他甚至促进了它的发展:

> 这一天，我将在巴黎，准备在 8 天的旅行后返回。对于那些关注我的网络职业生涯的人，我被邀请参加 W3 联盟关于样式表的会议，并就设计师在网络上想要什么发表我的意见。

来源:大卫杂志，1995 年 11 月 9 日

随着 97%的浏览器对 CSS 的出色支持(至少根据我们的统计)，David 对下一代网站设计的预言已经实现。遗憾的是，他创建的变通方法仍在如此多的网站上使用。
