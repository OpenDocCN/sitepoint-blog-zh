# 项目管理已经过时了吗？

> 原文：<https://www.sitepoint.com/has-project-management-gone-out-the-window/>

我正在和两个非常熟练的开发团队一起工作。都做优秀开发。都来自世界一流的教育机构，都有世界 500 强和新兴创业经验。

但是两者都没有提供我所期望的项目管理类型，包括:

*   有日期的项目计划
*   成功满足截止日期
*   经常与我交流进展、风险、问题(实际上，这仅适用于两者之一)
*   准确估计所需资源的能力
*   按时发布的版本
*   按时提供状态报告
*   询问我如何了解项目状态和更新

我觉得这很烦人，也很沮丧。我知道在敏捷开发、螺旋开发和其他形式的“边走边建桥”开发的趋势下，准确地预测项目时间表和问题是很困难的。

然而，沟通是客户关系管理的一个关键部分。许多熟练的 IT 专业人员需要在这方面提高。否则，客户将不会像以前那样忠诚于你。

在项目领导中，你必须管理 4 件事:

1.  客户期望(确保客户对项目将会做什么没有不切实际的梦想)。
2.  过程(经典项目管理 101)。
3.  结果(确保客户得到他们期望的结果，不管你是否完成了所有你认为你受雇要完成的任务)。
4.  关系(与客户建立长期关系，超越单一项目)。

我见过很少的 IT 专业人士能处理所有这四个问题。
