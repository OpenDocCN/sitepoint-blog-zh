# 卫星导航——离线网络技术的黑仔应用？

> 原文：<https://www.sitepoint.com/web-sat-nav-offline-technology/>

在昨天的[帖子中，我讨论了开发离线网络功能的问题，以及令人失望的应用程序数量。我总结道，离线功能似乎是一种等待应用的技术。](https://www.sitepoint.com/killer-offline-web-applications/)

然而，有一个杀手级应用几乎肯定会在未来几年内改变市场:基于网络的卫星导航系统。卫星导航已经存在了一段时间，但它们可能很昂贵，需要专用设备，并且不总是容易更新。也有手机软件，比如谷歌和诺基亚的软件，但是功能有限。

然而，一个纯粹的基于网络的卫星导航变得越来越可行，并提供了几个好处。

更好的位置搜索
如果你不知道目的地的街道名称或邮政编码，卫星导航几乎没有用处。基于网络的解决方案可以让你通过公司名称、地区、业务类型或任何其他使用自然语言的标准来定位目的地。

**即时更新**
新路线、道路工程、事故、交通拥堵等细节。可以实时更新并转发给 web 应用程序。一些 sat navs 提供这种功能，但许多需要订阅费，信息质量参差不齐。

数据混搭
基于网络的卫星导航可以从网络上的任何地方获取信息。一旦计算出你的路线，卫星导航就可以提供路边摄像头的实时视频、餐馆或酒店报价、感兴趣的地点等。

**协同路由**
每一个用户的旅程细节都可以被整理分析。司机可以访问一个巨大的数据库，里面有推荐路线、平均时间、有趣的旅程等等。司机可以选择最安静、最受欢迎、风景最好的地方，而不是选择最短或最快的路线。

替代交通工具
一个基于网络的系统可以从各种交通系统中获取信息——不仅仅是道路。例如，数据可能表明某条路线会因为假日交通而特别繁忙，推荐火车服务并立即订票。

支持现有硬件
许多智能手机已经提供了良好的网络浏览器和 GPS 功能。除了专用设备，你没有理由不使用上网本、平板电脑或类似的硬件。

**多种输入法**
专用 sat navs 的硬件和软件都有限制，但智能手机或 PC 可以支持其他输入输出设备。例如，网络摄像头或麦克风可以用来提供手势或语音控制。

谷歌地图、街景和地球等系统已经提供了免费的地图数据，这些数据可以被基于网络的卫星导航系统使用。这些系统也可以被付费播放音频或文本广告的公司商业化。)

然而，目前的问题是互联网连接。Wi-fi 并非普遍可用，当进入隧道或建筑区时，移动互联网连接可能会很慢或失败。显而易见的解决方案是离线 web 技术。一旦你上路了，应用程序可以缓存全程细节或下载半径 10 英里内的重要信息。

诺基亚和谷歌都已经为移动操作系统提供了免费导航系统的测试版。这一消息导致 TomTom 的股价在最近几周暴跌，尽管 TomTom 没有理由不利用这些技术。

你认为使用离线技术的基于浏览器的免费卫星导航会出现吗？会很快发生吗？
