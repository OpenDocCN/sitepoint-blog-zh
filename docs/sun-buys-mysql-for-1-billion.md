# Sun 以 10 亿美元收购 MySQL

> 原文：<https://www.sitepoint.com/sun-buys-mysql-for-1-billion/>

*转载自《科技时报》第 182 期。*

本周，当 Sun Microsystems 宣布将以 10 亿美元收购 MySQL AB 时，开源爱好者们感到震惊，MySQL AB 是开发流行的 MySQL 数据库软件的公司。

两个阵营的观点都是积极的，但是这笔交易对网络开发者意味着什么呢？它可能对其他开源项目产生什么影响？

要回答这些问题，我们需要考虑许多因素，包括 Sun 的开源技术历史、它的竞争对手以及其他已经商业化的开源技术的成功。

虽然也有一些批评者，但不可否认的是，Sun 在参与开源世界方面有着相当坚实的历史。该公司声称自己是开源社区的最大贡献者，这一说法基于其 2005 年发布的 Solaris 源代码和 2006 年发布的 Java 编程语言，很难反驳。

但还有其他证据表明 Sun 理解保持软件开源和免费的价值:像和这样的项目有健康的社区和有前途的路线图。事实上，如果您研究一下 Sun 目前向企业界提供的产品，就会发现该产品组合中唯一缺少的关键组件(Oracle、IBM 和 Microsoft 等竞争对手已经提供的组件)是数据库。MySQL 在全球有超过 1000 万的安装量(包括谷歌和脸书这样的高流量网站),是一个天然的选择。

web 开发人员可能担心的一个问题是许可条款会改变——如果 Sun 决定开始对 MySQL 的每个安装收取许可费，该怎么办？

在我看来，这种情况不会发生，因为 Sun 意识到它买的不是数据库，而是一个社区。员工、开发人员和客户的社区。

与此相关的问题是，随着 Sun 将精力集中在与 Solaris 平台的兼容性上，跨平台 MySQL 版本的开发是否会停滞不前。以下是 MySQL 社区关系副总裁 [Kaj Arn 对此事的看法](http://blogs.mysql.com/kaj/2008/01/16/sun-acquires-mysql/):

> “我不认为(在 Solaris 上开发 MySQL)会以其他流行的操作系统(Linux、Windows、Mac OS/X、其他 Unixes 等)为代价。)或开发环境(PHP、Ruby on Rails、Perl、Python、ODBC、C++、C#、VB 等。). **MySQL 伴随 LAMP 成长，核心没有 LAMP 的 MySQL 简直不可想象。Sun 最初感兴趣的是 LAMP 的 MySQL 部分。因此，我不认为 Sun 有一个平台迁移战略，而是继续成为. com 中不可或缺的一部分。”**

简而言之，Sun 了解开源软件的商业模式，当长期前景——在付费支持下推动 MySQL 进入企业领域——看起来如此光明时，它不会向一家公司投资 10 亿美元，只是为了短期利润而欺骗其用户。有许多其他开源数据库(例如 PostgreSQL)具有类似的特性和性能，如果客户不满意，他们会很快迁移到这些数据库。

至于孙所付出的数字呢？虽然有些人会认为，鉴于 MySQL 用户群的规模，这个数字很低，但在任何人眼里，10 亿美元都是一大笔钱。让我们来看看其他一些最近的开源收购[ [source](http://www.zdnetasia.com/news/software/0,39044164,62036683,00.htm) :

*   2.1 亿美元用于 SUSE Linux (Novell)
*   为 JBoss(红帽)投资 4 亿美元
*   XenSource(Citrix)5 亿美元
*   3.5 亿美元收购 Zimbra(雅虎！)

我想说 MySQL 团队现在会对自己感到非常满意。谁说开源软件没钱了？
