## 有哪些类型的虚拟主机？

有一件事使虚拟主机的选择变得复杂，那就是有太多的选择了！这里有一个快速纲要。

最实惠的托管计划是**共享托管**。它们的成本更低，因为你与他人共享一台服务器。您分摊了成本，但也分摊了存储和带宽。如果另一个网站的流量很大，你的网站可能会受到影响。

一个**专用服务器的**资源对你的网站完全可用。你得到所有的存储空间和带宽，你也可能负责管理计算机。

**云托管**是一个由服务器维护的集群，必要时可以扩展，为你的网站提供比专用服务器更多的资源。

在每种情况下，你都可以选择不同价位的不同方案。这些因提供商而异。

那么，哪种类型的主机最适合你的网站呢？回答以下问题，找出答案。

## 1.你的网站有多少页面？

你可能会惊讶于一个基本网站需要的空间是如此之小。你可能也会对一个网站的发展速度感到惊讶。因此，选择一个托管计划，提供足够的空间来存储你的网站，并在未来有足够的增长空间。

单个网页包含文本、格式和图像。这听起来很像一个简单的文字处理文档，从许多方面来看确实如此。为了直观显示您需要多少存储空间，将网页想象成一个 Word 文档，将网站想象成一个文档文件夹可能会有所帮助。

2016 年，一个网页的平均大小是 2.2 MB。这是 2014 年平均规模的两倍多，是 2010 年的两倍多。这种增长的主要原因是图像尺寸不断增加，以跟上我们计算机不断提高的屏幕分辨率。因此，在计算网站所需的空间时，要留出足够的增长空间。

让我们基于 2.2 MB 的平均值做一些估计。如果你的网站有 10 个页面，你至少需要 22 MB 来存储它们。一个有 100 页的网站需要 220 MB。1 GB 的空间平均可容纳 450 页——超过了许多网站的需求。

你认为你的网页需要多少空间？

鉴于一些最便宜的共享主机计划提供大约 10 GB 的存储空间，网页的大小不太可能是一个限制因素，尤其是对于一个全新的博客或小型企业网站。

但这只是考虑到网页。如果您允许访问者查看或下载额外的文件或媒体(视频、音频、高分辨率照片或文档),存储需求可能会迅速增加。我们将在下面的第 4 节中讨论这个问题。

## 2.每月有多少人会访问你的网站？

文件存储是你的网站需要的硬盘空间。带宽是每月从你的网站上被浏览或下载的数据量。

每次查看或访问网页时，都会增加您使用的带宽量。每个页面的文本和图像需要下载到每个访问者的计算机上，然后才能在他们的网络浏览器上显示。

你的带宽取决于你的网站每月有多少访问者，以及他们每次访问时访问了多少页面，换句话说，就是你每月的浏览量。希望这是将随着时间的推移而增长的东西，所以你需要选择一个托管计划，不仅要满足你目前的需求，而且要带你到未来。

网站流量很难提前预测，但我们会帮你做出一个估计。一旦你的网站建立并运行，你就可以用类似 [Google Analytics](http://analytics.google.com) 这样的工具来跟踪，但那是另一个教程的主题。

你预计会有多少游客？一个本地企业应该比一个全球性的公司更少的游客。一个新的博客应该比一个已经建立的博客有更少的流量。以下是一些大概的数字:

*   新博客通常每天会有 10 个左右的访客，他们可能会看完一篇博文就离开。这相当于一个月大约 300 次浏览量——大约 660 MB 的带宽。
*   随着博客的建立，流量可能会(也可能不会)增加到每天数百名访客。这相当于每月 10，000 次浏览量或 20 GB 的带宽。
*   一个当地的小企业每天可能会接待大约 30 名访客，也就是说，一个月将近 1000 名访客。如果每个访问者浏览三个页面，那么一个月就有 3000 个页面，或者超过 6 GB 的带宽。
*   一家拥有良好品牌的大型律师事务所将从谷歌获得大量流量，每天可能会有 3000 次访问，或每月 90000 次。
*   热门博客一个月能接待几十万访客。最大的每月接待[数千万游客](http://www.ebizmba.com/articles/blogs)。

回到你的网站。从这些大概的数字中我们可以得出什么结论？

*   受欢迎的托管公司表示，他们的共享托管计划可以应付每月数万浏览量和数百千兆字节的带宽。这对大多数博客和小型商业网站来说已经足够了。
*   随着你的博客越来越受欢迎，小心监控你的带宽。当您的流量接近每月 100，000 个访问者时，请计划迁移到专用服务器的策略。不要只监控月平均值，也要考虑高峰使用量。
*   拥有知名品牌的大公司可能会从谷歌获得大量流量。专用服务器会为他们提供更好的服务。
*   大型企业网站和流行的博客网络将需要一个指定的服务器，云托管或定制的托管解决方案。

## 3.你的网站将在什么平台上运行？

大多数网站使用像 WordPress、Drupal 或 Joomla 这样的内容管理系统。它们是 web 应用程序，使管理您的网站和添加内容更加容易。

像任何软件一样，这些应用程序有系统要求，您的 web 主机必须满足这些要求。其中包括:

*   足够的存储空间来安装软件。
*   运行编写软件的脚本语言的软件。
*   包含每个网页信息的数据库软件。
*   支持上述所有功能的操作系统。

WordPress 是这些应用中最受欢迎的，所以让我们以它为例。下面是它的主要[要求](https://wordpress.org/about/requirements/):

1.  PHP 版本 7 或更高版本。WordPress 是 PHP 脚本的集合，所以需要在你的 web 服务器上安装 PHP 软件来运行它。
2.  mySQL 版本 5.6 或更高版本。您的网页存储在数据库中，因此您需要 mySQL 或兼容软件来存储和访问这些数据。
3.  15-20 MB 的额外空间用来安装 WordPress 以及你的主题和插件。

因为 WordPress 如此受欢迎，你会期望大多数虚拟主机选项能够满足运行它的要求，你是对的。其他网络平台有更具体的需求。做好你的功课，确保你选择的主人可以满足他们。如果有疑问，请通过电子邮件联系该公司。

## 4.您将提供哪些额外的媒体服务？

当您提供的不仅仅是网页时，存储和带宽方面的考虑开始膨胀:

*   开发者可以向他们的用户提供软件下载。
*   公司可能为其员工托管大量的文档库和内容管理系统。
*   公司可以通过允许客户下载 PDF 文件来提供文档和用户手册。
*   电子学习网站可能会将多媒体整合到他们的课程中。
*   摄影师和平面设计师可能会存储大量高分辨率、可打印的照片和图像。
*   音乐制作人和播客可以访问成百上千的音频文件。
*   摄像师可以在线分享大量的视频项目和作品。

这些附加文件需要仔细考虑。通过检查文件在电脑硬盘上占用的空间来确定您需要的存储空间。并且通过估计每个月有多少文件被浏览或下载来决定你应该允许多少带宽。

以下是一些指导原则:

*   共享主机通常允许你存储千兆字节的空间，并每月下载数百千兆字节。
*   专用托管计划通常会增加到万亿字节。
*   考虑将这些文件托管在其他地方，以减少它们对您网站的影响。例如，你可以把你的视频文件放在 YouTube 上。当用户通过下载文件耗尽你的带宽时，你不想让你的网站慢下来。

## 5.你重视易用性还是全面控制？

像任何计算机一样，web 服务器有许多需要安装和维护的软件:

*   操作系统(如 Linux 或 Windows)，
*   虚拟主机软件(如 Apache)，
*   脚本语言(如 PHP 和 Perl)，
*   以及你所选择平台的软件(比如 WordPress 或 Joomla)。

如果你想完全控制你的网站，拥有所有这些选择可能是一件好事。但对我们大多数人来说，这是一个不受欢迎的负担。

一个**托管托管**计划是你的托管提供商将为你处理所有这些。他们承担技术负担，你从一个易于使用的控制面板配置你的网站。像 WordPress 这样的网络应用可能是预装的，或者只需点击一下就可以安装。

共享托管计划被管理。您正在与其他用户共享一台电脑，而不是自己管理它。许多专门的托管计划让你选择自己管理服务器或让它为你管理。

强烈考虑托管计划，即使管理自己的服务器的想法吸引你。最佳实践是，同一个人不应该同时尝试构建网站内容和管理基础设施。你不会像专注于这项任务的人那样做好任何一项工作。

## 6.其他考虑

在决定网站需要的托管类型时，这里有几个需要考虑的因素:

1.  你将来会增加更多的网站或网络应用吗？论坛、维基、票务支持系统、电子邮件系统？选择主机时，为未来的发展预留足够的存储空间、带宽和数据库。
2.  你的网站是关键任务吗？如果你的网站瘫痪导致你的业务停顿，投资优先考虑它的可靠性。以 99.99%的正常运行时间为目标，并确保有足够的带宽供大量访问者使用。
3.  选择和注册域名(你网站的网址，如*mybusiness.com*)从技术上来说不是虚拟主机的一部分，但是很多虚拟主机提供商可以帮你。这可能会简化您的网站设置，并减少您需要支付的公司数量。
