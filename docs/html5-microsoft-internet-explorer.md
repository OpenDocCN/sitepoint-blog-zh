# 微软会在 ie 浏览器中实现 HTML5 吗？

> 原文：<https://www.sitepoint.com/html5-microsoft-internet-explorer/>

微软对他们在 ie 浏览器中的 HTML5 计划保持着可疑的守口如瓶。谷歌、Mozilla、苹果和 Opera 都认为 HTML5 是未来的 T2，并急切地在他们的浏览器中实现功能。谷歌甚至利用 HTML5 技术开发了他们新的在线交流系统 Wave[。](https://www.sitepoint.com/google-wave-release-date/)

不幸的是，IE 是唯一一个不能提供基本 HTML5 支持的主流浏览器。Bruce Lawson 的文章，[没错，今天就可以用 HTML 5 了！](https://www.sitepoint.com/article/html-5-snapshot-2009/)提供了一个解决方案，但是它依赖于 JavaScript，所以并不是在所有情况下都可行。

然而，总部位于雷德蒙德的软件巨头似乎终于开始审视 HTML5 的含义了。以下消息是微软的阿德里安·贝特曼最近在[邮件列表](http://lists.w3.org/Archives/Public/public-html/2009Aug/0389.html)发布的:

> 作为我们未来工作计划的一部分，IE 团队正在审查 HTML5 规范的当前编辑草案，并收集我们的想法。我们希望分享我们的反馈，并在工作组中进行讨论。当我们收集笔记时，我会把它们贴出来，这样我们可以更快地重复我们的想法。在这个阶段，我们的问题多于答案，但我相信公开讨论这些问题是取得进展的最佳方式。

这篇文章揭示了一些对 HTML5 草案规范的有效问题和批评，尽管尚不清楚微软是否直接向 W3C/WHATWG 工作组提出了这些问题。

微软并不急于实现 HTML5，但也许这并不奇怪:它仍然是一个草案规范。没有 ie，HTML5 能成功吗？浏览器的市场份额可能正在减少，但如果 60%的用户无法使用 HTML5，开发者会如何考虑呢？

从历史上看，微软*和 T4*都遵循标准——甚至引领标准——但只是在对他们有利的时候。如果它是 Silverlight 等微软产品的竞争对手，他们会很方便地放弃功能。我怀疑 HTML5 会出现在 IE9 中，但怀疑微软是否会倾向于实现富媒体功能，如`canvas`、`audio`和`video`标签。我们仍在等待 SVG 支持！**

没有微软的全力支持，HTML5 能成功吗？如果 HTML5 没有在世界上最流行的浏览器中实现，它能被认为是一个标准吗？
