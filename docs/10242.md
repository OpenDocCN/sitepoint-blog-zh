**Paul:** 是的，Modernizr 基本上是一个功能检测库，它基本上对很多常见的东西都有最好的、最强大的检测功能，如果你想知道你当前使用的浏览器是否支持 CSS 转换或 border-radius 或 WebGL，Modernizr 已经收集了基本上检测这些东西的最好方法，然后给你一些不错的 CSS 钩子和 JavaScript 钩子，这样你就可以调整你的页面，使其对显示它的独特环境非常敏感。

Kevin: 我喜欢 Modernizr 的一点是，它将所有这些技巧归结为真正有用的东西。你得到什么；你把 Modernizr 放进一个页面，它运行，你把你的帽子挂在什么上面，它为你产生什么？

Paul: 它基本上给了你两个独立的 API。最有用的一个，也是人们通常抓住的第一个，是它在页面上的`html`元素上抛出了一堆类，所以现在`html`元素有了一个`border-radius`的类，但是对于灵活框模型，它可能有一个`no-webgl`和`no-flexbox`的类。因此，在你的样式中，你可以使用`box-shadow`，如果你使用的是`box-shadow`，你只需放入那些框阴影规则，但是如果你没有`box-shadow`，你实际上可以尝试做一个假的阴影，你可以将右边框设置为深灰色，底部边框设置为浅灰色，或者你可以引入背景图像来模拟它，这样你就可以控制什么时候没有支持，这就是你用 API 得到的结果。同时，它还公开了一个名为`modernizr`的全局 JavaScript 对象，因此如果支持 HTML5 视频，那么`modernizr.video`为真，如果支持 Ogg 视频编解码器，那么`modernizr.video.ogg`也为真。

保罗:对。所以我一直使用的方法是，如果旧浏览器没有支持的话，基本上“多填充”到旧浏览器中。所以“多填充物”这个词是雷米·夏普想出来的。
