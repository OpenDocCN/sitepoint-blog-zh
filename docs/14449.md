# SSL 和加密强度

> 原文:[https://www.sitepoint.com/ssl-and-encryption-strength/](https://www.sitepoint.com/ssl-and-encryption-strength/)

我最近有机会与 Thawte([www.thawte.com](https://www.thawte.com/))的安德鲁·肯纳德交谈，他回答了一些关于 ssl 和浏览器加密的问题，作为我最近专栏[用 SSL 保护 Apache 2](https://www.sitepoint.com/securing-apache-2-server-ssl/)的后续。

作为 Andrew 评论的序言，应该指出的是，随着购买使用最新浏览器的更新、低成本的机器，旧的 40 和 56 位加密浏览器的使用在国际上正在下降。然而，肯纳德让我们注意到这一点确实是有道理的。

我在文章中提出的一个假设引起了 Andrew 的注意，那就是我没有提到，即使是 128 位 SSL 证书的加密级别也会因访问安全服务器的浏览器而异。

“这意味着用户可以根据他们使用的浏览器版本以 40 位、56 位或 128 位连接，”他说。

大多数数字证书都是以这种方式运行的，即在浏览器和服务器之间提供受支持的加密连接。

“理解这一区别很重要，因为许多 ca 将它们的证书宣传为 128 位，而实际上它们将支持不同加密强度的会话(128 位是最强的加密级别)，”Kennard 补充道。

一些历史

美国过去的立法禁止出口 128 位加密技术，这导致了支持 40 位和 56 位加密的浏览器，肯纳德称之为“出口”浏览器。

1997 年，美国政府废除了对 128 位加密的禁令。然而，今天仍然有大量的出口版本浏览器在使用，主要是在国际上，但也在美国。

服务器门控密码术

根据肯纳德的说法，CA 的回应是开发服务器门控加密技术，将“导出”浏览器升级到 128 位加密。

肯纳德说:“只有少数几个认证机构提供这些证书，所以如果你需要 128 位加密增强功能，一定要找 SGC 技术公司。”。

专业行业需求

肯纳德认为，SGC 还可以满足法律或监管要求行业中的公司运行强加密的需求。

他补充说:「在这种情况下，使用 SGC 证书将是最佳选择(而非标准数码证书),因为 SGC 证书是确保符合 128 位加密要求的最积极尝试。」。
