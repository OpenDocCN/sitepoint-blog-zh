# Shadow Boxing:无图像、CSS3、光滑按钮

> 原文:[https://www . site point . com/shadow-boxing-image-free-css3-glossy-buttons/](https://www.sitepoint.com/shadow-boxing-image-free-css3-glossy-buttons/)

纽扣。作为网猴，我们用了很多。搜索按钮、购买按钮、发送按钮、关闭按钮。你一般会在我们网站最重要的路口找到它们，所以我们自然希望它们看起来很特别。传统上，这是 CSS 的工作——毫无疑问，现在仍然如此。使用图像的主要缺点是它们不能缩放或符合按钮的大小和形状。虽然对多个`background-image`和`background-size`的支持确实可以帮助我们，但至少可以说，对这些 CSS 属性的支持是不完整的。其他考虑因素包括更胖的文件、额外的服务器连接和工作流；你需要离开你的 CSS 编辑器来调整任何图像。最近，CSS3 渐变作为按钮的样式选项似乎变得越来越流行。他们也有很多让人喜欢的地方。它们很容易配置，不需要离开你的 CSS 编码器，它们毫不费力地拉伸和调整你的按钮区域。然而，这不全是阳光和棒棒糖。缺点是:

*   目前只有 Firefox、Chrome 和 Safari 支持任何类型的 CSS3 渐变。尽管 IE9 和 Opera 与 CSS3 一样好，但它们都倾向于用手指塞住耳朵，唱着“啦，啦，啦，..我听不到你……”说到 CSS 渐变。
*   渐变类型仅限于线性和径向。
*   渐变的 CSS 语法在少数支持它的浏览器之间非常不一致。

与其长篇大论地解释两者的区别，我想我应该只向您展示一个 CSS 代码片段，它在 Mozilla 和 WebKit 中呈现了相似的径向渐变:

```
-moz-radial-gradient(13% 24% 0deg,circle farthest-corner, #AB1A03, #FFFFFF, #C7C7C7 38%)-webkit-gradient(radial, 59 64, 324, 100 59, 15, from(#C7C7C7), to(#AB1A03), color-stop(.6,#FFFFFF))
```

不太直观。最近，我对 CSS 按钮采取了一种稍微不同的方法，我想我应该带您了解一下。让我们从一个基本的单色橙色按钮开始。我在这里建立了一个测试页面。它只是一个带有类和不起眼的悬停状态的链接。

### 步骤 1:给你的按钮一个标准的阴影

我们大多数人可能已经以某种形式使用过 CSS3 `box-shadow`属性。如果不是，基本语法相对容易理解。它是这样工作的:

|  | x 偏移量，以像素为单位 | 以像素为单位的 y 偏移 | 像素模糊 | 颜色(十六进制、RGB、RGBA 或命名颜色) |
| `box-shadow` | 2px | 1px | 3px | #662299 |

作为一名 Chrome 用户，我将在这些示例中使用 WebKit CSS，但我们将在稍后讨论对一系列浏览器的支持。`box-shadow`的一个有用的方面是你可以自由地在同一个物体上叠加尽可能多的阴影。您只需要用逗号分隔每个影子声明。首先，我将为我们的按钮添加两个基本阴影:

```
-webkit-box-shadow:    3px 3px 6px rgba(0, 0, 0, .20),    0px 0px 3px rgba(0, 0, 0, .40);
```

第一个阴影创建一个 20%不透明度的黑色阴影(0，0，0)，并将其向下偏移 3 个像素，向右偏移 3 个像素。还模糊了 6px 的边缘。我在这里使用 RGBA 颜色值，因为它允许我控制不透明度。这真的很酷，但是我发现标准的十六进制颜色更容易处理。我的第二个阴影没有偏移，所以只是在四周提供了一个微妙的暗辉。这一开始听起来可能很傻，但是我发现一种微妙的、黑暗的光芒给了一个物体一种重量感。![](../Images/96aeea86e4dedfcf8c398d8c0a1b08c3.png)

### 第二步:添加一个燃烧橙色的基础层

`box-shadow`的一个最简单的变化是`'inset'`值，它可以有效地反转你的阴影。您的阴影现在只在按钮区域内可见，而不是出现在按钮区域外。如果您还没有使用它，语法看起来像这样:

```
-webkit-box-shadow:inset -8px -8px 10px #fffafe;
```

现在，嵌入阴影的标准用法是经典的斜面效果:使底部边缘变暗，顶部边缘变亮，这样你就有了一个 3D 按钮。问题是，作为一个独立的效果，这实际上很少看起来有吸引力。取而代之的是，我们将建立起一层层的颜色，就像画家建立起一层层的颜料一样。

**note:**Building up Layers

从莱昂纳多到德加，画家们学会了通过一层又一层半透明的颜色来创造深沉、丰富、复杂的色调。我们正在尝试做一些类似的嵌入框阴影。

首先，我要用这段代码铺一个较暗的、焦橙色的阴影层:

```
inset 0px 25px 25px #930;
```

我给这个阴影层一个非常模糊的边缘(25px)，并把它从按钮的上边缘向下放置 25px。正如你在下面的图片中看到的，这在按钮的下半部分留下了一个柔和的，较浅的橙色部分。如果你观察真正的玻璃，它倾向于从上方捕捉光线，然后将光线聚焦到下半部分——有点像放大镜。我们在这里模仿这种效果。![Degas layers paint ](../Images/3b60746ff3374395053bdcf8434cbcc9.png)

### 第三步:添加高亮颜色

在现实世界中，光线往往来自我们的上方，所以我在按钮的上方 20px 放置了一个更亮的橙色阴影。我正在寻找一个硬边，地平线一样的反射，所以我要保持模糊到最低限度(2px):

```
inset 0px 20px 2px rgba(240, 150, 69, .5)
```

我想小心不要干扰文字的可读性太多，所以我设置这一层的不透明度为 50%。当心，这里的*是*潜在的陷阱。CSS 中描述的每个新的`box-shadow`都会自动插入到前一个下面的**中。这意味着我们需要在代码中深色之前插入高亮颜色，使其可见。令人恼火，但却是事实。结果应该是这样的:![Button step 3](../Images/f7b731198af07222aa2396a04fcc3c26.png)**

### 第四步:缓和反射

我们的按钮正在成形，但是高亮颜色有点太淡了。我更喜欢它朝着顶部边缘逐渐消散。实现这一点最简单的方法是在按钮的顶部边缘再覆盖一层模糊的深橙色。代码可能是这样的:

```
inset 0px 5px 12px #930
```

所以，现在我们有一个光滑的渲染按钮。![](../Images/7ae11a0ce79fcf2e380b82a1655878e7.png)

### 步骤 5:添加悬停状态

有时，我们可能希望按钮对悬停的光标做出反应。我将对按钮的悬停状态做两个调整。在现实世界中，当物体接近表面时，阴影会变得更清晰、更暗。我们将对我们的外部按钮阴影做同样的事情。我还将在鼠标悬停时将我们的水平线向下移动几个像素。悬停状态的 CSS 将如下所示:

```
a.bigorange:hover {  -webkit-box-shadow:    2px 2px 4px rgba(0, 0, 0, .2),    0px 0px 3px rgba(0, 0, 0, .1),    inset 0px 10px 12px #930,    inset 0px 20px 2px rgba(240, 150, 69, .7),    inset 0px 25px 25px #930;}
```

现在你知道了。一个纯 CSS，无图像，可调整大小，完全可配置，光滑的按钮。[查看本页底部完成的演示](https://www.sitepoint.com/examples/css3/shadowboxer.php)。

### 但是浏览器支持呢？

至此，我们有了一个可以在 Safari 和 Chrome 中完美运行的按钮。令人高兴的是，Firefox 中`box-shadow`的 CSS 语法实际上是相同的。我们只需复制我们的按钮代码，将 **`-webkit-box-shadow`** 替换为 **`-moz-box-shadow`** ，我们的按钮就可以使用 Firefox 了。 **Opera** 虽然 Opera 目前不支持渐变，但它对 W3C 标准`box-shadow`规范的支持近乎完美。制作你的按钮代码的另一个副本并简单地删除`'-webkit-'`前缀会让你的按钮在 Opera 中完美呈现。 **IE9** 和 Opera 一样，IE9 没有渐变支持，但似乎有完美的 W3C 认可的框影支持。如果你为 Opera 修复了它，它应该在 IE9 中工作。旧版 ie 浏览器没有支持`box-shadow`属性的版本，所以你必须在这些浏览器中使用一个平面的、工人阶级的按钮。当然，你也可以提供一个背景图片，只提供给年长的探险者，但这有点违背了初衷。但那是你的决定。

### 思想

这里有一点需要承认的是:我并不特别喜欢我今天展示的橙色按钮的例子；有点俗气，过了头。然而，我认为它背后的方法非常有用，希望这个演示可以帮助您对这种方法有所了解。不可否认，试图将复杂的渐变理解为一系列分层的阴影，起初可能会非常令人头痛。对我来说，纯粹的 CSS3 渐变支持目前在重要的 UI 元素中考虑得太零碎了。毫无疑问，很多时候，普通的背景图片会是你最安全的选择。无论如何，基于图像的按钮永远不会像纯 CSS 按钮那样灵活和容易重新配置，这是必须考虑的因素。我希望这给你的 UI 库增加了另一个武器。玩一玩，让我知道你怎么走。
