# 规划 iPhone 应用程序:从哪里开始

> 原文:[https://www . site point . com/planning-an-iphone-app-where-to-start/](https://www.sitepoint.com/planning-an-iphone-app-where-to-start/)

设想一个 iPhone 应用程序并把它变成现实可能是一个令人生畏的过程。这里有一些提示可以帮助你的应用程序想法有所行动。

## 提问

在你深入视觉之前，回答一些关于这个应用的问题是很重要的。

### 战略的

![](../Images/e7e6b5afefc4579245a6819c8c639335.png)

*   有哪些应用与你的想法相似？
*   为什么人们应该使用你的应用而不是其他应用？
*   为什么要做这个 app？这款应用会占用你的时间、精神能量，侵蚀社交关系，成为你最喜欢的话题，也是你最大的挫败感来源。你确定值得吗？

这是停下来反思的好时机。你能把后两个的答案卖给其他人吗？你们两个都相信吗？精彩！让我们继续:

*   你每周要花多少时间在这个应用上？别忘了考虑吃饭睡觉这些占用时间的小事。
*   应用程序必须在某个日期之前发布吗？如果是的话，你有时间吃饭睡觉还能发射吗？
*   这个 app 是为谁而存在的？他们知道他们需要或想要它会给他们什么吗？如果不是，你将如何讲述一个故事来证明为什么它是可取的，甚至是必要的？
*   你还会如何营销这个应用程序？

### 特殊的

*   有人可以使用你的应用完成哪些任务？
*   有哪些任务比其他任务更重要？这个层次结构将最终帮助你决定你的应用程序导航将会是什么样子。
*   你完成这些任务的终极功能列表是什么？在几次大型头脑风暴会议中把所有事情都写下来是很棒的。但是，如果你最终把清单分成两半，然后又分成两半(也许是第三次)，不要气馁。最好把你的核心能力做得非常好，而不是很多事情做得很差。
*   你将在什么时候开始测试应用程序？如果这是一个真正有趣的应用程序，那么你会在早期就有人要求成为测试者，并且你希望在他们询问时准备好答案。

## 获得灵感

[![Pttrns iPhone Screenshots](../Images/41904a245531f046315a8a972d1e642e.png)](http://www.pttrns.com)

Pttrns iPhone 截图

拯救理智从这个简单的任务开始:认识到创新不是凭空而来，而是一个进化过程。这意味着你无法拥有项目所需的所有知识、远见、创造力和远见。其他人的想法和投入对于我们完成最好的工作是必不可少的。

一个明显但有时被忽视的获得视角的策略是下载新的(希望是便宜的)应用程序来体验界面，寻找有趣的交互，并注意与你未来应用程序的相似之处。

下载应用程序是理想的，因为你可以完整地体验它们。退而求其次就是看截图。对于我购买的应用程序，我会不断地截屏，并将它们保存在我的系统中。对于别人的截图，我借鉴这两个成长中的站点:

## 开始画草图

在你忙于设计和开发之前先画草图。你如何做并不重要——我用过 paper、 [Balsamiq](http://www.balsamiq.com "Balsamiq prototyping") 、 [Omnigraffle](http://www.omnigroup.com/products/omnigraffle/ "OmniGraffle") (用 [Zurb](http://www.zurb.com/playground/iphone-stencils "Zurb OmniGraffle Stencils") 或 [Graffletopia](http://graffletopia.com/categories/iphone "Graffletopia") stencils)和 [Protoshare](http://www.protoshare.com/ "Protoshare prototyping tool") ，但是现在有几十种选择。在谷歌上搜索“iPhone 原型工具”会带你找到它们。

不管你用什么方法，只要找到你能玩的东西。如果你用的是 Mac，我建议你使用 LiveView，这样你就可以通过 wifi 直接从电脑上查看手机上的静态线框。当草图在你的手机上，在你的手里和在电脑屏幕上时，你对产品的感知会有很大的不同。

![Example of a task flow diagram](../Images/5715046b434324952dd52ba6aa58d696.png)

任务流程图示例

As I wireframe, I like to keep an updated task flow diagram handy so that I know where the screens I’m currently working on fit into the overall workflow. If you are working on a team project, then this will help keep everybody on the same page. It’s amazing how quickly miscommunications can sneak in.
