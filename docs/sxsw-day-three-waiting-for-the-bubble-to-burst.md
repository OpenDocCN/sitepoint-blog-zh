# SXSW 第三天——等待泡沫破灭？

> 原文：<https://www.sitepoint.com/sxsw-day-three-waiting-for-the-bubble-to-burst/>

所以你可能想知道我上面选择的标题是关于什么的。我昨晚才真正意识到，有人对我说，今年在 SXSW，感觉就像是网络繁荣时期。他这么说是因为，当时我们每个人都在喝那家店的第三或第四杯免费啤酒，周围都是做同样事情的人。这是[适应路径](http://www.adaptivepath.com/)派对，我们大约半小时前从隔壁[博主](http://blogger.com/)派对举行的地方搬到那里，但重要的是，那里他们提供的免费酒吧刚刚用完。就像所有免费的帽子和食物一样。是的，今年镇上的每个人似乎都想送人东西，但老实说，我真正想要的是休息一晚！[我在这里只是半开玩笑]

我第一次发现自己很难参加所有的会议，挤出一些时间在午餐和会议结束后坐下来吃顿饭，还参加许多不同的晚上聚会(正如你所知，那些提供免费东西/食物/饮料的聚会可能很有吸引力)。哦，每天结束后找时间写一些有意义的东西。这篇文章是在第四天的早上打出来的，而不是在第三天结束的时候，我喝了两杯咖啡，还用了一些牙签来保持眼睛睁开。我渴望晚上能睡五个小时以上！

除了派对，第三天怎么样？我参加了一个名为“Web 标准和搜索引擎:寻找共同点”的会议。理论上看起来很棒——两个我感兴趣的主题，一个从商业角度来看可能有很多东西可提供的主题，然而尽管演讲者很好，清晰且信息丰富，我还是有点失望。为什么？好吧，我可以用一句话来总结这一小时的会议:创建符合 web 标准的页面可能会提供一些优势(就语义丰富性而言)，但不要期望获得认可。本质上，如果您的站点验证为 XHTML strict 或者有 101 个验证错误，这没有什么区别，它仍然是关于内容的。出现的话题包括黑帽搜索引擎优化技术，谷歌轰炸的问题(当谈论这个问题的人来自雅虎时，这有点滑稽！)和明显的标准及其对搜索结果的影响。对我来说，这场演讲最精彩的部分其实在于观众。或者，更准确地说，是安迪·克拉克的一个问题，他说虽然他不相信自动可访问性检查服务(如[辛西娅所说的](http://cynthiasays.com)或 [WebXM](http://www.watchfire.com/products/webxm/default.aspx) )是网站可访问性的可靠证明，但他觉得通过这些服务之一应该会在一定程度上提高某个网站的排名。换句话说，这是一个可访问的网站，所以把它在列表中稍微靠前一点。这个简单的建议让安迪赢得了一片掌声。(蒂姆·梅尔)似乎没有意识到这种自动化工具的存在，这有点令人担忧。然而，他确实说他会“把它带回团队”并理解这个概念，并补充说，如果某样东西确实通过了这样的测试，它可能会有一个标志，然后可以用来过滤搜索(例如，搜索针织网站，但只搜索那些标记为可访问的网站)。第一次搜索之后还有一次搜索(搜索部分 1 & 2 ),但我决定去别处寻找一些我不知道的东西。

我今天的第二个话题是“你的博客有生意吗？’。不，你没有看错，不，这不是一个打印错误。这个小组由[菲比·埃斯皮里图](http://simplifierlab.com/)、[埃里克·赖斯](http://www.audioblog.com)、[肖恩·因曼](http://haveamint.com/)、[杰弗里·泽尔德曼](http://www.zeldman.com/)和[DL·拜伦](http://www.texturadesign.com/)组成，他们都在说同样的话——他们本质上是“偶然的企业家”。他们没有创业和思考“对，我们需要有一个博客”，他们有博客(或其变体)，因为他们参与了一个预先存在的社区(以及随着他们的商业想法而成长的社区)，他们能够把想法放在那里，并获得关于这些发展中的企业的极好的反馈。肖恩·因曼的故事就是一个很好的例子，他讲述了自己是如何从 shortstats 开始的，short stats 是他开发并免费提供的一种网络统计解决方案服务，但它成为了 Mint 的基础，Mint 是一种付费服务，现在拥有大量粉丝。他解释说，他写 Mint 主要是为了自己，因为没有一个统计服务做他想做的事情，并认为他不可能是唯一一个想要某些功能和行为的人(他让它看起来如此好的事实不会损害事情的商业方面)。博客的出现给了他进一步的方向，有效地指引了发展的方向并提供了灵感。菲比·埃斯皮里图提醒观众，在几年前，要获得像博客现在这样的反馈，可能需要营销人员(或团队)和焦点小组来获得任何值得使用或作为决策基础的东西，但有了博客，你可以很快进行有用的对话。我不得不说，我坐在观众席上，想知道有多少人在看这部电影时有自己的想法或正在萌芽的商业计划等待实现。我是说每个人都有一两个好主意，对吧？

随着官方 WaSP ( [Web 标准项目](http://webstandards.org/))的事务接管，下午的会议对我来说并不是自由的。首先，有一个关于 WaSP 任务组(或[WaSP-WTF)的会议？](http://2006.sxsw.com/interactive/programming/panels/?action=show&id=IAP060028))，各个 WaSP 任务组的负责人借此机会告诉 SXSW 与会者各个小组的工作以及他们在未来几个月和几年的目标。许多人不知道 Web 标准项目有这些更小的团体，他们有特定的目标(Dreameaver，accessibility，education 等),也包括那些不是全职黄蜂的人。第二次会议是[公开 WaSP 会议](http://2006.sxsw.com/interactive/programming/panels/?action=show&id=IAP060084)，这是对去年某个人(咳！斯科博。咳咳！)当我们在会议室外面的走廊上举行年度聚会时，他碰巧走过，因为他不能参加谈话而感到很不高兴。没有任何企图让任何人感到被冷落，这只是一个私人会议…除了它是在公共场所举行。所以，今年它被开放了，公众能够发现所有的黄蜂是谁，以及更多关于这个群体的历史。杰弗里·泽尔德曼在那里为新的 [WaSP 网站](http://webstandards.org/)设计剪彩(不幸的是，当时服务器出现了问题)，然后所有在场的 WaSP 向观众做了自我介绍，并回答了问题。实际上，我没有任何会议记录，因为我决定放下笔记本电脑，加入更多的讨论！因此，在这一点上，我将留给你一个承诺，我会尽快找到这两个事件的另一篇文章，并添加到下面的评论中。

还有一天，下一次 SXSW 互动就到此为止，我们可以在接下来的一年里将学到的所有技巧和窍门付诸实践。
