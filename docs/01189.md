## 技巧 1:使用功能组件

### 卡姆·杰克逊

如果您不需要组件的内部状态或生命周期方法，请使用功能组件。功能组件是一个纯 JavaScript 函数，它接受`props`作为其参数，并返回一个 React 元素。

**功能组件的好处:**

*   较少代码
*   更容易理解
*   该组件是无状态的，因此可以避免意外地将状态存储在不应该存储的组件上
*   该组件更容易测试
*   没有任何约束
*   更容易看出何时何地提取较小的组件

### 科里屋

有几种方法可以做到这一点。

2.  在渲染中使用箭头功能
3.  在构造函数中绑定
4.  在类属性中使用箭头函数

## 技巧 4:在`setState`中使用函数，而不是对象

### 索菲亚·舒梅克

根据 React 文档，React 不保证立即应用状态更改。

因此，在调用`setState()`之后立即读取`this.state`是一个潜在的陷阱，因为`this.state`可能不是您所想的那样。

因此，与其用一个对象更新状态，不如用一个函数来更新它，以避免这个潜在的陷阱。
