# 如何设置短信和语音菜单

> 原文:[https://www . site point . com/alternative-mobile-interfaces-how-to-set-up-SMS-and-voice-menus/](https://www.sitepoint.com/alternative-mobile-interfaces-how-to-set-up-sms-and-voice-menus/)

既然移动设备的数量超过了个人电脑，我们就必须考虑这些设备给我们带来的所有功能。有很多关于“移动网站”和“小屏幕设备”网站的讨论。无论你如何称呼这种无处不在、永远在线、永远连接的新型微型计算设备，我们都不应该忽视它们的一些最大资产。

纵观媒体史，当我们阅读一个新的范式时，我们倾向于回顾以前的范式，以获得灵感和指导。当电视出现时，它模仿了收音机。人们围坐在一张桌子旁讨论话题。广播和电视的唯一区别是你也能看到人们。随着时间的推移，人们意识到电视本身就是一种新的媒介。当网络首次成为可能时，我们也看到了类似的尝试。服务提供商提供娱乐“频道”。网络试图像电视一样。我们很快意识到网络是不同的，有更多的可能性。

我们现在正从大型台式机和笔记本电脑过渡到小型“智能”手机和“哑”互联网连接对象。所以，像所有的媒体转换一样，我们看着以前的，并试图应用相同的思维过程。通过手机获取信息与通过电脑终端获取信息有很大不同。这有各种各样的原因，从屏幕大小和分辨率，到处理能力，用户输入(T9，软/硬键盘，触摸，语音，视觉等。)，到情境特征。你是在上下班途中使用手机，还是在家里的沙发上使用手机，或者两者兼而有之？

当我们缓解当前网络体验的问题，以解释人们发现自己访问这些信息的许多新范式时，我们往往会忽略电话和其他移动设备相对于现有计算机体验的一些关键优势。

这篇文章不是关于如何做一个“移动网站”。使用媒体查询、针对特定设备、针对不同的浏览器和设备类型运行多个前端模板，甚至拥有单独的站点，这样更好吗？这些都忽略了一个更明显的问题，即网络首先是否是正确的媒介。

这个博客是关于移动建筑的。电话首先是设计来打电话的。手机还有一种奇特的能力，可以处理被称为 SMS 的异步推送消息。作为设计师和开发人员，我们倾向于直接进入我们最熟悉的领域 Web，但是花一些时间停下来思考一下替代方案可能对您的客户更有价值。

### 短信服务

短信是与客户互动的好方法。无论“社交媒体专家”如何阐述社交图的优点，无聊的旧邮件列表仍然会产生令人难以置信的流量、销售额和投资回报。手机短信也可以填补这个空白。它能够定期或交互式地向客户发送信息，是一种以直接和个性化的方式传达信息的工具。许多服务允许客户向一个短代码，一个 5 位数的号码，发送一条带有一些关键字的消息，并立即得到自动回复。这在请求旅行信息、场馆活动或其他数据时非常有用。你只需发送一个关键词，可能是一个公交站名，就能知道下一辆公交车还有多长时间到达。也许你可以发一个场地名称，看看那天晚上谁在比赛，或者发一个邮政编码，然后得到未来 5 天的天气预报。

由于这些消息是异步的，所以发送时的顺序并不重要。每一个都有时间标记，并且当网络能够传送它时将被接受。对于电视投票，你可以拨打一个号码，但线路可能会占线。有了短信，你发送信息，它会在稍后的日期被发送，并带有时间戳。这很重要，因为它可以处理大的尖峰。在紧急情况下，如海啸或地震，短信广播可以拯救生命。相反，像 [Ushahidi](http://www.ushahidi.com/) 这样的服务可以接受短信并追踪信息。

SMS 通信是实现带外通信的一种很好的方式。如果您忘记了您的密码，而不是通过电子邮件发送给您，可以发送一个临时代码。这允许除了用户名和密码之外的多一层身份验证。

有很多服务可以让你收发短信进行处理。我将在下一节讨论一个名为 [Tropo](http://tropo.com/) 的服务示例。

### 语音菜单

你有没有打电话给你的银行或其他公司，得到那个可怕的语音菜单系统？你知道，那个会问你“请按 1 找客服代表，请按 2 查询你的账户，请按 3 找丢失的卡…”他们真的很痛苦。它们冷漠、单调、乏味，通常对你的实际需求没有反应。让我们不要责怪实现问题破坏了语音菜单的整个概念。如果使用得当，它们可以为您的客户提供一种通过他们熟悉的媒介快速获取信息的绝佳方式。仅仅因为现在手机的销量超过了个人电脑，并不意味着每部手机都有 3G 网络上最新版本的 WebKit！很多人仍然每天使用非智能手机。这些人中有多少是你遗漏的潜在客户？

建立一个语音菜单系统似乎要花一大笔钱，只有企业级的公司才有能力做。如果那是你所想的，那就大错特错了。今天，像 Tropo 和 Twilio 这样的公司可以让你只用几行你最喜欢的编程语言就能创建复杂得惊人的电话应用程序。

为了了解这有多简单，现在让我们创建几个快速的应用程序。

#### 第四步

问个问题！使用语音，您可以接受菜单命令，并通过电话与客户互动。必须说，我们都习惯了非常糟糕的经历。如果你打算采用语音菜单，你真的应该做些功课。理解为什么取款机在菜单中选择深度而不是广度。多少选择才算多？有办法回去吗？这并不容易，但如果做得好，它会是一种宝贵的资源。

使用 Tropo，如果你想问一个问题，你只需添加`ask()`功能。它有两个参数，一个系统将读取的字符串和一个属性数组。在本例中，我将询问一个简单的晚餐菜单选项。你想要牛肉、鸡肉还是蔬菜？该数组有一个名为“choices”的属性，它是一个以逗号分隔的潜在答案列表。我还设置了超时和一些额外的参数来处理错误的选择、超时问题和正确的答案。这些函数中的每一个都被进一步定义来完成下一步。在这种情况下，顾客做出选择后，它简单地回复一个`say()`关于这个选择有多棒。

[![Tropo Step 4](../Images/48b3415d8646a1d922ec983769ca1780.png)](https://www.sitepoint.com/wp-content/uploads/2011/05/tropo-step-4.png)

Tropo 步骤 4

```
<?php
    say("Hello World");
    ask("What do you like better, beef, chicken or vegetables?", array(
        choices =>"beef, chicken, vegetables",
        timeout=>5.0,
        onTimeout=>"onTimeout",
        onBadChoice=>"onBadChoice",
        bargein=>false,
        onChoice=>"onChoice"
    ));

    function onChoice($event) {
        say("That's crazy, me too!");
    }

    function onBadChoice($event) {
        say("Sorry, I did not understand that!");
    }

    function onTimeout($event) {
        say("I know it's a hard choice, just call me back when you've figured it out!");
    }
?>
```

至此，您应该对如何使用 Tropo 通过手机发出命令和接受输入有了较高的理解。现在，您可以继续构建更加复杂的语音菜单系统来处理特定于您的服务的请求。

Tropo 有大量关于他们的 [API](https://www.tropo.com/docs/scripting/element_summary.htm "Tropo Method Summary") 和[脚本](https://www.tropo.com/docs/scripting/ "Tropo Scripting Examples")的文档。您可以阅读更多内容，了解如何通过电话播放 MP3 以及发送和接收短信。

调试音频菜单可能非常困难。如果你犯了错误，电话就不会响。它有很大的改进空间，但如果你打电话，它说“无效电话号码”，不要太担心。那是手机说“语法错误”的方式。

### 结论

通过语音或短信的移动界面，您可以通过大多数人口袋里已经存在的设备，为博物馆、场馆、交通和城市增加一层可选信息。想象一下，一个语音系统，你可以打电话询问一个便利设施的列表，或者给出两个街角，数据库就会给出相应的信息。不是通过一些闪亮的应用程序，需要几个不同的构建平台或来自供应商的许可，也不是一些 HTML5 地理定位网页，只在 1%的智能手机上可用。这种系统可以很容易地部署在世界范围内任何有信号发射塔的城市、城镇或村庄。

面向移动的构建不仅仅意味着 Web 标准、Android 或 iOS。有一个完整的音频和短信通信的世界向你敞开。手机的这些功能令人难以置信地强大、成熟，并且存在于比网络浏览器更多的手机上。因此，下次你考虑通过手机向客户开放你的公司和服务时，不要简单地重新包装现有的媒体，而是要把这些设备的优势作为一个独特的机会。
