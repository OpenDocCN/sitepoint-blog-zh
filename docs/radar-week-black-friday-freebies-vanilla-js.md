## 本周流行语:香草 JavaScript

香草 JavaScript 并不是一个奇特的新框架的名字，事实上恰恰相反。这个术语用来表示 JavaScript，它不依赖于任何第三方库(如 jQuery ),并且将在浏览器中运行。

![Vanilla JS Logo](img/a761923cb5b673bb88874970327de51b.png)

jQuery 如此受欢迎的原因之一是它抽象出了许多跨浏览器的问题和不一致性。然而，自 2006 年首次出现以来，浏览器的发展突飞猛进。以至于我们现在可以用 12 行普通的 JavaScript 代码实现一个现代的 html 5 light box[，如果你停下来想一想，这是非常酷的。](http://demosthenes.info/blog/962/A-Modern-HTML5-Lightbox-in-12-Lines-of-JavaScript)

随着移动和低功耗设备的激增，性能变得非常重要。这就是为什么现在可能是去掉 jQuery 中间人的好时机。

如果你不确定哪些浏览器支持哪些 JavaScript 特性，SitePoint 的 HTML 和 CSS 编辑器 Louis Lazaris 已经为 JavaScript 和 DOM 兼容性表收集了[大量资源。](https://www.sitepoint.com/resources-javascript-dom-compatibility-tables/)

[101 是一个 JavaScript 实用程序库](https://github.com/tjmehta/101/)，它并不试图重新发明轮子，而是添加到普通 JS 中以避免不必要的重叠。

这里有两个方法演示了如何在普通 JS 中实现[原生数据绑定。](http://www.sellarafaeli.com/blog/native_javascript_data_binding)

为了混淆视听，事实上有一个名为 [Vanilla JS](http://vanilla-js.com/) 的库(剧透:这是一个模仿)。
