# 从 IE6 中获得乐趣和收益的 5 个步骤

> 原文：<https://www.sitepoint.com/5-steps-to-ie6-profit/>

也许*‘好玩’*有点夸张。*【利润】*可能也有些牵强，但你和你的客户可能会对任何涉及微软老款浏览器的财务协议感到满意。

没有人喜欢支持 IE6，但当你忽视它时，问题就开始了。很少有客户声明 IE6 是一项要求，但是一些 web 开发人员认为这是放弃支持的信号。很少有客户会提到 HTML、CSS、Flash 或者主机——你会忽略这些技术吗？

在项目开始时忽视 IE6 会将困难转移到项目结束时。你可以创建一个客户喜欢的令人惊叹的网站，但是几天后，他们打电话说他们的主要客户(或者他们阿姨的邻居的第三个堂兄弟)无法访问这些页面。他们使用的是一台 10 年前的 PC，安装的是 Windows 98 和 IE6，但是，尽管你提出了请求和合理的解释，客户还是坚持认为他们是一个重要的客户。

如果你非常幸运，客户会愿意为 IE6 支持支付额外费用。否则，额外的费用将落在你的肩上。无论哪种情况，项目的完成都没有让客户满意，你的专业精神受到质疑，你们的关系也处于危险之中。

这种情况可以通过分析需求、预先解释问题并教育您的客户来避免…

第一步:请求一个浏览器支持列表
这是一个长期的尝试，但是询问你的客户是否有任何特定的浏览器必须被支持。可能会出现空白，但你应该能够确定他们的核心客户统计数据，例如青少年、老年用户、企业、政府部门等。

**第二步:分析现有统计数据**
如果客户已经有了网站，获取他们的浏览器统计数据。如果没有数据，根据客户的顾客做出判断。向政府或大型组织销售产品或服务的公司可能比销售 iPhone 配件的商店拥有更突出的 IE6 用户群。

然而，不要放弃网上商店的任何浏览器——没有人喜欢失去客户，不管他们的浏览设备有多模糊。

**步骤 3:在报价中包含 IE6**
IE6 会导致额外 25%的开发时间吗？或许接近 50%？不管是什么数字，把它加到你的报价里。

如果你担心这会让你比竞争对手更贵，向客户解释这种情况。说明 IE6 用户占网站访问者的 1/20(或任何统计数据表明的)，但是，如果他们对这些用户有降级的体验感到高兴，你可以相应地降低价格。

从心理上来说，这听起来比在没有 IE6 支持的情况下提供报价，然后在项目“完成”后增加成本要好得多。

**第四步:使用合同**
我以前说过，现在再说一遍:*始终提供合同*。不管你的客户是密友还是家人，都没有任何借口。

合同应该提到哪些浏览器和版本将被测试，强调 IE6 和决策的成本影响。你是诚实的，客户不会经历意外的惊喜。

第五步:理解并测试 IE6
在一个蓝光和 PVR 的时代，学习使用 Betamax 录像机可能看起来像是在学习，但 IE6 并不是许多开发者所说的恐怖故事。渲染错误和遗漏都有很好的文档记录，并有解决方法。如果你从一开始就测试浏览器，你很快就会理解它的古怪之处，并且不需要在项目结束时重新编写代码。

有了经验，任何开发人员都可以创建一个兼容 IE6 的现代网站，而不会产生大量的额外费用。这额外的 25%成为可爱的利润！
