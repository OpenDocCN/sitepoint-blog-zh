# HTML 电子邮件的新标准

> 原文:[https://www.sitepoint.com/a-new-standard-for-html-email/](https://www.sitepoint.com/a-new-standard-for-html-email/)

*以下内容转载自[科技时报#174](https://www.sitepoint.com/the-second-coming-of-the-productivity-tool/) 。*

如果你是使用 Outlook 2007 的《科技时报》的众多幸运读者之一——不管是不是自愿的——那么很可能你已经注意到了这篇时事通讯和许多其他自 Outlook 2003 升级以来看起来不太对劲的文章。解决方案可以是 HTML 电子邮件的标准吗？

![](../Images/4ab62f10944e9cfc752baff7b04ec361.png)**[【HTML 电子邮件新标准】在 SitePoint 论坛](https://www.sitepoint.com/forums/showthread.php?p=3562538)**

《科技时报》的长期读者可能还记得我在《科技时报》第 156 期中的咆哮，关于微软选择在 Outlook 2007 中用基于微软 Word 的新引擎替换 Outlook 2003 中的 Internet Explorer 渲染引擎。是的，*那个*微软的 Word。我们说话的时候，Clippy 正在看你的邮件。

用于创建和发送高质量 HTML 电子邮件简讯的[活动监控](http://campaignmonitor.com/)服务的开发者 [Freshview](http://www.freshview.com/) 的优秀员工一直在努力让微软明白原因并扭转这一举动，坦率地说，这让电子邮件技术倒退了十年。

由于这些请求似乎继续被置若罔闻，Freshview 提出了一个新的策略:定义一个(相对)易于支持的 HTML 和 CSS 子集，作为支持 HTML 的电子邮件客户端可能努力支持的标准。从理论上讲，通过设定一个比我们期望的 web 浏览器对 HTML/CSS 的全面支持更容易实现的目标，我们也许能够从微软这样的供应商那里获得一些改善 HTML 电子邮件前景的兴趣。

这个计划在[一篇非常有想法的文章](http://www.campaignmonitor.com/blog/archives/2007/09/why_we_need_web_standards_supp_1.html)中做了概述，这篇文章发表在[的《基线标准初步提案》中](http://www.campaignmonitor.com/blog/archives/2007/09/help_us_form_a_baseline_for_st.html)。

你认为为 HTML 电子邮件创建一个新的标准将有助于改善电子邮件客户端标准支持和互操作性的糟糕状态吗？
