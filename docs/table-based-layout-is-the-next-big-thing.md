# 基于表格的布局是下一件大事

> 原文：<https://www.sitepoint.com/table-based-layout-is-the-next-big-thing/>

**最近发布的 [SitePoint CSS 参考文献](https://reference.sitepoint.com/css/)(及其印刷版，[终极 CSS 参考文献](https://www.sitepoint.com/store/the-ultimate-css-reference/))广泛涵盖了 CSS 的一些更为晦涩的领域。随着今年晚些时候 IE8 的发布，CSS 表将成为一个非常重要的领域。**

多年来，接受 CSS 布局的开明设计师不得不竭尽全力来制作复杂的设计，而使用过去基于 HTML 表格的布局技术来制作这些设计是微不足道的。

设计者使用 CSS 技术制作一个简单的三列布局所需要的时间是如此之长，以至于许多网页设计者干脆放弃了 CSS，转而使用 HTML 表格进行布局。《科技时报》第 142 期和第 143 期专门讨论了这个问题。

随着今年 IE8 的发布，这一切都将改变。IE8 将是最后一个加入 CSS 表格支持的主流浏览器，这将使设计者能够使用基于表格的布局技术，而不会滥用 HTML 表格标记。

为了演示这一点，我使用 CSS 表格制作了以下 3 列页面布局示例:

![](img/00769f0266f16a604b40795fa259fdab.png)
[在浏览器中查看示例](https://www.sitepoint.com/examples/3col.html)

这种布局并不独特；它的特别之处在于实现起来是多么容易。我没有使用绝对定位或者脆弱和违反直觉的浮动技术，而是使用相对容易理解的 CSS 表格来生成它。

布局中三列中的每一列都是具有唯一 ID 的`<div>`。从左到右分别是`"navigation"`、`"content"`和`"sidebar"`。这三个都包含在另一个 ID 为`"body"`的`<div>`中。

神奇之处在于使用 CSS 使这三个不起眼的`<div>`看起来像是表格行中的单元格。首先，我们将包含的`<div>`设置为显示为表格:

```
#body {
  border-collapse: collapse;
  display: table;
  table-layout: fixed;
}
```

然后，我们将三个`<div>`中的每一个设置为显示为一个表格单元格，其宽度是我们想要的:

```
#navigation, #content, #sidebar {
  display: table-cell;
}
#navigation {
  width: 200px;
}
#content {
  width: 66%;
}
#sidebar {
  width: 33%;
}
```

通过添加一些边框和背景色，以及一些填充来给它最后的润色，我们得到了一个完全可用的布局。

<ins>Firefox 中的一个[长期存在的 bug](https://bugzilla.mozilla.org/show_bug.cgi?id=148810) 使得有必要在`"body"` `<div>`中添加第二个包含`<div>`的内容，并将其设置为`display: table-row`。否则，布局通常会在最初加载时中断。</ins>

这段代码不仅非常简单，而且结果也比使用早期 CSS 布局技术实现的典型三列布局更加健壮。无论哪一列包含的内容最多，每一列都会扩展页面的整个高度，并且布局不会因为字体或窗口过大而分崩离析。这种布局可以正常工作，就像基于 HTML 表格的布局一样。

一旦 IE8 退出，我们可以开始证明离开 IE7 是正确的，我们可以开始使用这些简单而强大的技术，因为所有其他主流浏览器*已经支持 CSS 表格。突然，基于 CSS 的页面布局将成为几乎任何人*都能理解的东西*！*

当然，我们不能在 IE8 发布的那一天就抛弃当前的 CSS 布局技术，不是吗？毕竟，IE7 和它的前辈将会存在很多年。如果你想让你的网站在这些浏览器上看起来一样好，你仍然需要突破更复杂和脆弱的布局技术。

但是，如果你能证明让 IE7 和更早版本的用户体验稍微下滑一点点是合理的，你就能最大限度地减少网站中需要的扭曲的 CSS 数量。例如，如果您在 IE7 中查看上面的示例，您将看到一个简化的两列版本的布局，它避免了混合像素宽度列和百分比宽度列的需要，从而简化了使布局在 IE7 和更早版本中工作所需的代码。

![](img/d29953666602dce09f66b1961d3bba38.png)

CSS 表格最终会让 CSS 的页面布局变得和过去的 HTML 表格布局技术一样简单。如果不支持旧的浏览器，设计者将不再有任何借口滥用表格标记来快速轻松地完成布局工作。如上面的例子所示，通过一点额外的计划，您甚至应该能够在布局中利用 CSS 表格的同时，为旧浏览器的用户提供合理的体验。
