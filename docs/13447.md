# 可用性扼杀创造力！

> 原文：<https://www.sitepoint.com/usability-stifles-creativity/>

我经常阅读各种可用性和交互设计邮件列表中的评论，讨论用户测试的利与弊。

似乎反复出现的一个论点是用户测试对创作过程的影响。有经验的设计师和用户体验专家经常争论用户测试是否真的扼杀了设计过程，阻碍了设计中真正的创造力。

以导航为例……可用的网站导航被认为是如此重要，以至于在整个网站的开发过程中，有一个行业的角色专门为它服务:交互设计师、信息架构师、可用性专家、易访问性专家、Ajaxian 神一样的生物……你能想到的。别忘了 Flash 开发人员！

但是回到我关于用户测试和直接设计的观点……有没有可能产生创造性的、功能性的和可用的设计，而不涉及某种类型的“用户干预”?

我也这么认为但是我认为这取决于设计者、用户和网站类型的技巧。

看一看 Etsy，一个为手工制品卖家和买家提供服务的网站——基本上是一个在线工艺品市场(对于周日早上的约会来说，这是一个不错的极客目的地，不是吗？！).该网站已经存在一年半了，有一些非常酷和有趣的导航和浏览产品的方式(也称为“购物”)。

他们整合的两个基于闪光灯的导航系统是“[时间机器](http://blog.etsy.com/timemachine.html)和“[颜色](http://www.etsy.com/color.php)”。

当前的时间机器是第三个也是最近的版本，它允许用户按时间倒序浏览最近列出的项目。以这种方式导航非常有趣，让我想起了我们有时在历史遗迹上看到的基于 Flash 的日期线。“颜色”为用户提供了一个调色板，从中可以识别特定色调的产品。

我可以看到这两个导航系统的一些真正积极的方面，为许多用户提供了一个更有用的界面；尤其是那些喜欢视觉搜索而不是通过基于文本的系统进行搜索的人。

多种浏览和搜索功能的可用性无疑增强了用户体验，如果成功的话，将会在更长的时间内吸引用户。我认为 Etsy 在这一点上做得很好…但是听着，Flash 导航是由一个对“生成图形和紧急系统的逼真属性”感兴趣的[程序员](http://www.etsy.cimg/about_4jared_big.jpg)设计的！在我看来，所有的用户测试都是通过 Etsy 博客上的反馈进行的。

所以我想这次我得保持中立。看起来在这种情况下，设计是成功的，尽管用户测试很少。
