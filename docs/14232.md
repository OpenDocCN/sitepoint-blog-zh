# 访问所有区域

> 原文:[https://www.sitepoint.com/access-all-areas/](https://www.sitepoint.com/access-all-areas/)

CSS 革命的一个令人高兴的副作用是它给了 web 可访问性问题新的关注点。通常，最初出于其他原因(例如速度、适应性、哲学原因等)而被无表格布局吸引的设计者和开发者，随着他们的发展，越来越相信提供可访问内容的好处和重要性。

当然，确信可访问性的价值是一回事，但是开发将这些原则转化为经过充分测试的、可工作的应用程序的日常过程是另一回事。

通常，当一个重要的新浏览器/设备出现时，支持它并不比安装新的客户端并在其上测试你的工作更困难。最坏的情况是，你可能需要找一台旧的 Mac 来运行 Safari 和 IE5Mac(就像我们一样)，或者可能需要一张 Konquerer 的 Knoppix CD。即使你不是一个专门的歌剧女主角，你仍然可以很容易地在浏览器上测试你的页面，而不需要任何费用。

屏幕阅读器似乎是一个更加困难前景。

普遍认为控制了屏幕阅读器市场大部分份额的 Freedom Scientific 的 Jaws 就是一个很好的例子。

想象一下，你刚刚向你的老板解释了为什么拥有。太好了！接下来向他/她解释这将花费他们 895 美元。

清理完咖啡后，他咳嗽着，语无伦次地走过桌子，悄悄告诉我还会有 120 美元的“软件维护费”。

这甚至不是“专业”版本。

*但是..试用版呢？我们当然可以为了测试而改进？你的老板恳求道。“*确定*！”你满怀希望地回答。“*‘试用版’60 天限时演示只需 39.95 美元……但是不要担心，如果你全款购买的话，那 40 美元会被扣除。**

你的老板看起来不太自信，因为他疯狂地寻找一些沉重的东西扔给你。

显然做个 cad 更便宜。

那么，Jaws 是一个难以置信的复杂的软件吗？我不完全确定这个问题的答案，但我记得我的 Amiga 在大约 1984 年做了一个相当不错的文本到语音转换。显然，屏幕阅读器不仅仅是一种声音，但是又有多少呢？

也许自由科学实际上对更易接近的网络并不感兴趣。当开发人员继续生产不可访问的内容时，总是需要一个复杂的“辅助应用程序”来尝试理解这一切。或者，如果我们都建立了不用视觉就能容易使用的网站，也许更简单(也更便宜)的工具会进化，杀死 Jaw 的市场。

如果我们想变得琐碎，也许这本身就是更易访问的设计的一个小动机？

所以，假设你不能说服你的老板为大白鲨买单，你可以用什么方法让你的工作尽可能的无障碍？我们自己仍在寻找这个问题的更多答案，但这里有一个开始。

CSS 布局是一个很好的起点，但不是真正的答案。尽管结构良好、语义化的 CSS 布局有助于更易访问的设计，但事实已经多次证明，构建一个非常难访问的基于 CSS 的网站是完全可能的。

在[链接](http://links.sourceforge.net/)、 [Lynx](http://lynx.browser.org/) 甚至 [Delorie 的 Lynx 模拟器](http://www.delorie.com/web/lynxview.html)中测试你的设计是检查你的页面的有用方法。尽管文本浏览器和屏幕阅读器从根本上来说是不同的，但是文本浏览器可以让你清楚地看到你的底层结构，同时也让对图像或 JavaScript 的依赖变得明显。

当然，使用众多的在线可访问性评级工具来评估你的页面，比如 Webaim 的 Accessibility Valet、HiSoftware 的 AccMonitor Online、 [Watchfire 的 Bobby Service](http://bobby.watchfire.com/bobby/html/en/index.jsp) 和其他工具，对于针对问题领域完成项目来说是非常出色的。
