# 你需要虚拟专用服务器吗？

> 原文：<https://www.sitepoint.com/virtual-private-server/>

虚拟专用服务器(VPS)是主机操作系统对硬件的模拟，允许另一个操作系统在其上运行。这对 Web 开发人员来说意味着一个 Web 托管公司可以在一个物理或“主机”服务器上托管多个“虚拟”服务器。

每个虚拟服务器都是相互隔离的(或“私有”)，对硬件资源(RAM、CPU、磁盘空间、网络吞吐量)的访问由“主机”服务器管理。例如，一台服务器可能有 2GB RAM、100GB 磁盘空间和 2 个 2GHz 处理器，上面有 10 个 VPSes。根据主机上使用的 VPS 软件，每个 VPS 可以共享硬件资源，或者保证访问 200MB RAM、10GB 的磁盘空间限制和 400Mhz 的有效 CPU。现在，这有点过于简单了，但是它给了你一个潜在的 VPS 场景的想法。

大多数 VPS 计划都托管在 Linux 或 FreeBSD 上，但是它们也存在于 Windows 世界中。我个人的经验和偏好是*nix 方面的东西，根据我自己的研究，我发现 Windows VPS 计划的成本往往比*nix 计划的成本要高得多，但这并不是说它们不适合特定的情况。但是，出于本文的目的，我将重点关注 Linux VPS 计划，因为我对它们很熟悉。

##### VPS 托管适合我吗？

VPS 托管计划填补了入门级 Web 托管计划与托管计划之间的空白，前者提供特定服务(如 PHP/MySQL 或 Perl/CGI ),共享服务器上的磁盘空间有限，而后者由您拥有和管理硬件，只需从数据中心租用“机架空间”和互联网连接。如果你已经超出了共享托管计划的范围，但还没有准备好共同定位的财务承诺，VPS 托管计划可能是一个理想的中间地带。

另一个可以考虑的选择是专用(或托管)托管，这通常比协同定位更便宜，并且相对于其他托管选择来说价格更合理。虽然它可能比 VPS 计划更贵，但专用主机可能仍然是一个比你想象的更实惠的选择。顾名思义，这为您提供了一台专用服务器:本质上，您从托管公司“租用”服务器，而不是像租用托管服务器一样，为您自己的服务器租用 rackspace。

VPS 托管计划具有以下主要优势:

*   root 访问权限使您能够安装和配置您想要的任何程序。想用 PostgreSQL 而不是 MySQL 运行 PHP？继续:安装它。想试试 Zope/Plone 吗？请自便。
*   您可以通过 Apache 的虚拟主机托管无限数量的网站。
*   您可以托管其他服务，如邮件服务器、FTP 服务器或任何类型的服务器。
*   您可以使用服务器进行备份、文件存储或任何您需要的操作。

但是，VPS 托管计划有以下缺点:

*   因为您控制着服务器，所以您还负责所有的安装、维护、安全和更新。如果您没有管理 Linux 服务器的经验，并且不愿意边走边学，这可能是一个很大的缺点。
*   虽然 VPS 托管计划为您提供了自己的完整操作系统，但您仍然可以共享硬件资源(CPU、RAM 等)。)与同一主机服务器上的其他 VPSes。因此，如果性能是关键，或者您正在运行 RAM 密集型程序，那么您可能更适合协同定位。

##### 它是如何工作的？

许多虚拟化技术使虚拟专用服务器成为可能:有商业公司，如 [VMWare](http://www.vmware.com) 和微软自己的[虚拟 PC](http://www.microsoft.com/windows/virtualpc/default.mspx) ，开源产品，如[用户模式 Linux](http://www.usermodelinux.org) 、 [QEMU](http://fabrice.bellard.free.fr/qemu/) ，以及最近的 [XEN](http://www.xensource.com/) 。这是目前 Linux 世界的一个热门话题，尤其是 XEN，受到了大量的关注和支持。甚至有可能它会与即将发布的 Linux 版本捆绑在一起。这对 Web 开发者来说是一件好事，因为虚拟化和 VPS 在虚拟主机中的应用越广泛，它将以更低的成本提供更多的自由。

##### 追根溯源

那么 VPS 账户能给你带来什么呢？好吧，你能得到的是对你的服务器的根访问权。如上所述，如果您喜欢远程管理 Linux 服务器，这当然是一件好事。这意味着你有完全的控制权，可以安装任何你想要的程序，做任何你选择的事情。但是，它也将配置和实现的责任交给了您。如果你对此感到不舒服，请远离 VPSes。

这是否意味着您需要从头开始安装整个操作系统？大概不会。大多数 VPS 计划都有许多“磁盘映像”可供选择，默认配置中有不同的发行版(如 Debian 或 Fedora Core)。您只需指定您想要的磁盘映像，它将在几分钟内配置到您的 VPS 上。建立帐户后，您就可以通过 ssh 获得 root 访问权限，并可以登录和定制您的服务器、安装新软件、删除不需要的软件以及根据需要更新现有软件。

##### 选择虚拟主机托管公司

那么如何选择 VPS 托管公司呢？这里有一点要注意。

了解硬件资源的详细信息:

*   主机服务器的规格是什么？(RAM、CPU、网络吞吐量等。)
*   您所在的主机服务器上运行了多少 VPS 实例？这一点很重要，因为您可以有效地与其他 VPSes 共享主机服务器的资源。
*   资源争用是如何处理的？换句话说，如果您的主机服务器上的所有其他 VPSes 都在运行 CPU/RAM 密集型进程，会发生什么情况？您是否有专用的最低 RAM/CPU/网络使用量？如果有，多少？

了解您的虚拟化技术的细节:

*   底层操作系统是什么:Linux，FreeBSD，Windows？如果是 Linux，提供什么发行版？
*   他们使用用户模式 Linux、QEMU 还是 XEN？为什么选择这种特定的技术？
*   硬件升级是如何处理的？假设您从一个提供 64MB 专用 RAM 和 2GB 磁盘空间的基本计划开始。六个月后，你意识到你需要更多的内存和磁盘空间。额外成本是什么？分配额外资源的程序是什么？

了解你的虚拟主机公司的详细情况。当你选择任何类型的虚拟主机时，这些都是比较传统的问题:

*   提供哪种支持(电话、电子邮件、即时消息、IRC–24/7？)
*   这家公司开业多久了？
*   公司的客户留存率是多少？
*   在过去的 12/24 个月中，他们经历了多长时间的停机？
*   他们有什么备份/冗余措施？

价格从低端的每月 20 美元到高端系统的每月 100 多美元不等，但要注意:价格并不总是更多功能或更好服务的标志。如果你仍然不知道 VPS 托管计划是否适合你，那么从低端开始可能是值得的(可能是 64MB RAM 和 2GB 磁盘空间 VPS)。你总是可以从那里扩展——只是要确保选择一个托管公司，可以容纳你心目中的扩展。如果您需要更多的硬件资源，可以选择协同定位。

然而，如果你有一个小的预算，但大的想法，VPS 可能只是给你。
