# 网络方向 UX 综述:安迪·巴德和史蒂夫·陈西贝

> 原文:[https://www . site point . com/we B- directions-UX-wrap-up-Andy-Budd-and-Steve-baty/](https://www.sitepoint.com/web-directions-ux-wrap-up-andy-budd-and-steve-baty/)

我刚刚发布了上周我在 UX 网络方向网站上采访安迪·巴德的文字记录。它很长，但非常值得一读——我们涵盖了各种各样的主题，如网页设计的职业生涯，CSS 的未来，IE8，HTML 5，可用性测试在设计过程中的作用，CSS 框架，CSS 画廊网站等等！

[![](../Images/bb6cd352da35d11ba584874cf6b3b48b.png)](http://www.flickr.com/photos/14052355@N04/1430554330/) 仔细阅读我上周五做的笔记，以下是我从另一位演讲者那里记下的一些片段，我从他的演讲中获益良多——[史蒂夫·陈西贝](http://ux08.webdirections.org/speakers/#post-3)，他谈到了[分析用户研究数据](http://ux08.webdirections.org/program/#post-3)。

史蒂夫设法引入了一些看起来相当可怕和复杂的统计公式，而没有让他的观众昏昏欲睡或转身向出口跑去。对他所选择的领域充满热情，并且是一个有魅力的演讲者，这当然有所帮助。也许这只是因为，摘下眼镜后，他看起来像《T2》中的查理([大卫·克鲁霍尔特兹](http://www.imdb.com/name/nm0472710/))，这可能加强了他在我心目中的可信度。

我从史蒂夫的演讲中得到的启示是，用户研究数据是无用的，除非你用它做一些事情，并且在你收集它之前，“一些事情”需要被很好地定义。他主张

*   定义您将预先测量的精确度
*   考虑样本数据的均值、方差和标准差，以及
*   获取两组数据，这样您就可以比较它们，以确定数据中的偏差是由于您正在测试的设计，还是由于测试组中用户之间的差异

Steve 在解释来自 A/B 测试的数据、任务完成率、完成时间和页面浏览量数据时推荐了一些方法，他对可用性测试的高度科学的方法强化了“用户科学”这个术语(我们大多数人可能会因为采取了一种更能体现“用户艺术”的方法而感到内疚)。
