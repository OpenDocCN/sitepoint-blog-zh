# 采访 MySQL 的 Zack Urlocker

> 原文：<https://www.sitepoint.com/interview-zack-urlocker-mysql/>

Web 开发人员通常将 MySQL 与 PHP 结合使用。虽然 MySQL 在 PHP5 中默认是不启用的，但一个名为 MySQL 的全新类极大地增强了开发人员可用的功能。

这个新类带来了 MySQL 4.1，它现在处于生产发布状态。存储过程和数据库集群等新特性使得 MySQL 成为许多商业和企业用户的首选。

但是商业方面，MySQL AB 如何继续开发这样一个著名的应用程序，同时还能盈利呢？最近，我请 MySQL AB 的营销副总裁扎克·乌尔洛克(Zack Urlocker)和其他几位回答了这个问题。

***首先，能否简单介绍一下你在 MySQL 担任营销副总裁的角色？***

我的背景一直是为开发者推广技术和基础设施。许多年前，我在 Borland 帮助创建了一个名为 Delphi 的产品。它后来成为有史以来最成功的客户机/服务器开发工具之一。这有助于引导我的职业生涯最终承担更具战略性的更大任务。

在 MySQL，我的重点是在企业客户和整个开源社区中推广我们的开源技术。我帮助确保我们向人们传达我们的计划——并且我们继续扩大评估和使用 MySQL 的人数。

***MySQL 数据库服务器是 Web 开发社区中最知名的服务器之一。你能为我们解释一下这个产品吗？***

当前版本 MySQL 4.1 于 10 月底投入生产(或 GA 状态)。它包含了大量的新特性，如预准备语句、更好的国际支持、Windows 和 Linux 上的新 GUI 安装程序、更快的性能、更好的安全性等等。它还包括用于高可用性应用程序的最新 MySQL 集群技术。这是我们推荐人们安装的版本。

还有 MySQL 5.0，目前有预生产的 Alpha 版本。它包括一些重要的高级关系特性，包括存储过程、触发器和视图。这些特性经常在企业应用程序中使用，但是当然，它们对其他用户也有帮助。

人们并不总是意识到的一件事是，有一个 MySQL 版本可以直接嵌入到应用程序中，例如，如果您想要在软件应用程序或设备中有一个“内置”数据库。嵌入式服务器被称为“libmysqld ”,可以很容易地从 C 或 Java 应用程序中调用，并且对用户完全隐藏。它有 4.1 和 5.0 两个版本。

总的来说，MySQL 的免费开源版本有多重要？你能解释它持续发展的原因吗？

由于开源社区的巨大支持，MySQL 到目前为止一直很成功。全世界有数百万 MySQL 用户，他们提供了大量的输入、测试和资源，帮助 MySQL 成为今天的产品。我们是世界上最受欢迎的开源数据库。我们也是市场上增长最快的数据库，甚至超过了微软、甲骨文、IBM 和其他公司。

我们完全致力于 MySQL 的开源战略。我们所有的软件都可以在“双重许可”下获得，这意味着它既可以在 GPL 下获得，也可以在那些希望与我们建立商业关系的人的商业许可下获得。

MySQL 的创始人 Monty Widenius 和 David Axmark 在大约十年前开始了开源战略，今天，我们仍然一如既往地致力于这一方法。我们现在看到越来越多的公司采用开源 LAMP 软件栈(Linux、Apache、MySQL、PHP/Perl/Python)，所以这对于 MySQL 的创始人来说是非常有见地的。他们的方法为我们提供了很好的服务，现在我们看到许多其他供应商，如微软、甲骨文和 CA 也开始采用开源实践。

MySQL 越来越受欢迎，主要是因为它与 PHP4 的集成。默认情况下，PHP4 支持 MySQL，允许 web 开发人员使用开源 PHP 和 MySQL 的组合非常容易地创建数据库驱动的网站。这在过去和现在对 MySQL 的发展有什么影响？

由于许多 PHP 用户采用了 MySQL，MySQL 无疑越来越受欢迎。PHP 是组成 LAMP 堆栈的最重要的语言之一(还有 Perl 和 Python)。我们一直在寻找让 PHP 开发人员更容易将我们的数据库集成到他们的 Web 应用程序中的方法——我们将继续与 PHP 社区的人们密切合作。例如，在最近发布的 MySQL 4.1 中，我们发现了一些技术故障，但我们能够快速解决这些问题，并在 24 小时内在我们的网站上进行修复。

***在 PHP5 中，MySQL 默认是不启用的——需要专门编译。MySQL 在多大程度上做出了这个决定，其背后的原因是什么？***

虽然与世界各地的 PHP 社区一起工作很棒，但我们也认识到他们的时间表并不总是与我们的时间表相同，他们的需求有时与我们的不同。他们对我们满足 PHP 需求的能力有些担忧，我们对灵活性也有些担忧。所以我觉得在这一点上，MySQL 不捆绑 PHP 5 没什么大不了的。我们将继续与 PHP 团队紧密合作。例如，PHP 5 为使用 MySQL 4.1 的开发人员提供了一个新的 MySQL 扩展，名为 MySQLi。这个扩展包括一个面向对象的接口和一个传统的过程化接口，并为一些最新的 MySQL 特性提供支持，比如预准备语句。

***你觉得 MySQL/PHP 组合为什么这么受欢迎？你觉得这种流行会在 PHP5 分裂后继续吗？***

我认为 MySQL 和 PHP 的结合是发展最快的开发平台。我们看到成千上万的公司和个人一起采用这些技术。随着宽带的普及，人们很容易从 http://dev.mysql.com 下载我们最新的服务器和驱动程序。

我们正在为 2005 年 MySQL 用户大会征集论文(CFP ),我们看到了很多关于 PHP 和 LAMP 堆栈的好文章，所以我认为这将会是一个热门话题。我们 MySQL 有很多 PHP 大师，包括 Georg Richter、Jim Winstead 和其他许多人。

你支持专门为 MySQL 开发的任何主要应用程序的开发吗，例如 phpMyAdmin？

我们鼓励我们的员工了解最新的开源项目，并尽可能地贡献或提供指导和反馈。当然，他们也相当忙于 MySQL 5.0 的工作。

但是我们确实看到很多人使用 phpMyAdmin，我们自己的 MySQL 管理员，以及各种第三方商业工具，如 Quest Software 的 TOAD 或 Embarcadero 的 DB Artisan。我们意识到，支持 MySQL 的工具越多，我们的产品就越能被开源社区和企业 IT 部门所接受。我们也和其他开源项目进行了很多讨论，无论是 OpenOffice.org、Apache、JBoss 还是 Eclipse 等等。

***MySQL 从最初发布到即将推出的 MySQL 5.0，进步了多少？与其他流行的数据库系统相比，MySQL 5 有何不同？就记录/表的最大数量而言，它有什么限制？***

正如我说过的，MySQL 5.0 现在处于 Alpha 阶段，我们有很多人在做一些非常好的工作。它还没有为“黄金时间”做好准备，但我们希望在 2004 年第四季度晚些时候有一个可靠的测试版，并在 2005 年 Q1 会议结束时正式发布。

MySQL 5.0 带来的关键是 Oracle、SQL Server、DB2 等中的许多企业特性。其中最主要的是存储过程、触发器和视图。我们可能还会加入一些其他功能，但这是主要的关注点。

我认为与其他数据库相比，MySQL 5.0 的有趣之处在于它的成功之处:快速、可靠、易用。大多数数据库都遭受了我称之为“服务器膨胀”的困扰。他们每年都在增加新功能，不管它们是否有用，每个用户最终都要为增加的复杂性、错误、可靠性等付出代价。某种程度上，这并不奇怪。出于财务原因，传统供应商不得不每隔几年进行一次升级，因此他们添加了大量额外的功能，但这些功能只会增加复杂性。

我们采取了不同的方法，将数据库视为一种商品。是的，它应该有一定的能力，但它不需要有所有的花里胡哨。许多人只是想要一个精简、简单、快速的数据库。我们认为自己是数据库的本田；如果你需要法拉利，你应该去买甲骨文。但也要准备好支付法拉利的价格，并拥有一名法拉利机械师。

即使使用 MySQL 4.0，我们也有客户构建拥有大量数据的应用程序。其中一些是非常复杂的数据仓库应用程序、报告系统、交易系统等等。由于我们的快速性能和可靠性，我们正在帮助 Adobe、Sabre、NASA、美联社、Google、Evite/CitySearch、Lufthansa 和数百万其他组织。

***你们还有哪些其他产品，你们是如何营销的？将 MySQL 作为免费产品提供是否能让你更有效地推广这些服务？***

我们以“双重许可”的方式提供 MySQL。那些不愿意受 GPL 开源许可要求限制的公司可以与我们公司建立直接的商业关系。我们不仅在双重许可下提供 MySQL 服务器，我们还提供用于管理数据库的图形工具，例如 MySQL Administrator 和最近发布的查询浏览器，这是编辑数据、执行交互式查询和调试查询的好方法。

我们还提供支持、培训和咨询服务。我们在这一领域发展迅速，并确保我们能够提供 24×7 支持和有保证的响应时间。这就是有一家公司支持我们产品的好处之一:用户可以两全其美。这种方法帮助我们以更低的成本生产更好的软件。我们得到的 QA 比世界上任何其他数据库都多，我们产品的质量证明了这一点。Reasoning Inc .的一项研究表明，MySQL 的缺陷率是其他封闭源代码产品的 1/6。我认为这个数据说明了开源的力量！

***MySQL 数据库服务器和你的其他产品的开发都投入了哪些工作(比如 MySQL 管理员)？你如何管理开发、特性请求和 bug 修复？***

我们使用基于 30 天 scrum 开发周期的敏捷开发过程。这意味着开发人员已经清楚地分配了他们需要完成的任务，并且他们定期检入代码。我们试图“尽早构建，经常构建”,这样每个项目都会有一个好的版本，即使我们处于 alpha 阶段。我们使用 bitkeeper 源代码控制系统，它很适合我们，因为我们有这样一个分布式环境。我们在 19 个不同的国家有员工，所以我们需要一个解决方案，使合并来自许多不同来源的代码变得容易。

我们跟踪人们提交给 bugs.mysql.com 的所有错误。我们还有一个专门为客户提供支持的系统，叫做 eventum(也是开源的)。该系统使我们的支持人员能够轻松跟踪问题并跟进客户。我们在自己的工作日志系统中跟踪每一个想法、建议和功能。

你与 Web 开发社区(特别是 PHP 开发人员)的合作有多紧密，以确保你的团队所做的改变是他们想要的？

我们总是在收集信息，无论是在我们自己的用户大会上，还是在 Oscon 或 LinuxTag 等行业活动、其他活动或在线活动中。我们有相当多的人真的加入了开源社区，所以这很容易。

我们的许多开发人员和工程师在[http://forums.mysql.com](http://forums.mysql.com)关注我们的论坛。我们最近聘用了一位资深医生兼培训师 Arjen Lentz，让他担任我们的社区关系经理，因此他总是在寻求来自人们的意见。当然，alpha 和 beta 阶段的反馈对我们的成功也至关重要。

***如何从商业角度看待开源软件作为发布产品的手段？在你看来，它能提供一个可行的、可持续的商业模式吗？***

我们相信开源不仅与商业兼容，而且开源只有在有商业模式的情况下才是可持续的。当 IBM、HP、Oracle 和其他公司接受 Linux 时，它才真正起飞。类似地，我们看到更多的创业公司使用开源，更多的商业模式探索，无论是我们拥有的双许可方法，像 Novell 或 JBoss 这样的服务商业模式，像 Red Hat 这样的订阅业务，还是其他一些组合。

最后，采用开源的公司不想放弃他们习惯于从商业产品中获得的高质量支持。当然，他们想要更低的 TCO 和开源软件提供的那种不受束缚的自由，但是他们不一定想要自己支持每一个修复。因此，他们将期待商业实体提供支持，培训，咨询，赔偿等。几年前，“开源商业”听起来像一个矛盾修饰法——就像“政府情报”！但现在我认为这两者可以很好地融合在一起。

***MySQL 最初起步的时候，联合创始人迈克尔·“蒙蒂”·维德纽斯和大卫·阿克斯马克对产品有什么愿景？进公司的编队是后来来的吗？***

Monty 和 David 在数据仓库领域的各种咨询项目中合作，这就是对性能和可靠性的最初需求的来源。最终，Monty 决定在他的引擎之上建立一个 SQL 层，这就是 MySQL 的诞生。它一直是开源的，但也一直是一个商业实体。

在早期，我认为他们忙于实现功能和响应用户，很难让某人接受订单，但最终他们雇用了一些额外的人，他们可以专注于业务方面——从那以后一直在增长——但它一直处于双重许可下，一直是一项业务。

我们热爱开源模式，但它从来就不是慈善机构或邪教。我们只是认为这是做生意的更好方式。我们很高兴，这么多年过去了，Monty 和 David 仍然在公司非常活跃，编写代码并推广我们的开源商业模式。

你如何看待 MySQL(公司和数据库服务器)的短期和长期前景？有什么新进展可以透露？

我们真正感到兴奋的新事物之一是 MySQL Cluster。这是我们大约一年前从爱立信获得的一项技术，用于提供高可用性的“五个 9”数据库，提供 99.999%的正常运行时间。这是一项非常复杂的技术，我们没有让它保持专有和封闭源代码，而是让它完全开源。这对开源世界来说是一个巨大的礼物。MySQL Cluster 是 MySQL 4.1 的一部分，我们对此感到非常兴奋。我们鼓励想要高可用性关键业务应用程序的客户下载并试用。

我们也一直在寻找使我们的数据库更容易管理和监控的方法——让它“更聪明”地帮助没有时间成为 MySQL 专家的开发人员和 DBA。也许他们已经被其他任务压得喘不过气来，他们想知道他们做的事情是否正确。或者，如果他们正在使用的某个功能有更新，他们可能希望得到通知。这些是我们正在调查的一些事情。

***你还有什么要补充的吗？***

如果你还没有看过 MySQL 4.1，那就去看看吧。它有很多很棒的新功能，比如预准备语句，可以真正提高性能。如果你有建议或想法，请在我们的论坛上留言，地址是[http://forums.mysql.com](http://forums.mysql.com)。我希望我们能在 4 月份在加州圣克拉拉举行的用户大会上看到你们中的许多人，这将是与 O'Reilly 联合制作的。我认为会很有趣——我们在那里安排了一些很棒的实践课程。你可以在 http://www.mysqluc.com 的[了解更多关于我们会议的信息。感谢您对 MySQL 的关注和支持！](http://www.mysqluc.com)
