所以我在本周的《科技时报》上写了一些关于这个的文章[，并且](https://www.sitepoint.com/host-your-own-short-urls/) [Jeffrey Zeldman 推荐](http://www.zeldman.com/2009/08/10/shorten-this/)使用 WordPress 为他的网站写博客。所以他使用了一个插件，叫做 WordPress 的[短 URL 插件，看起来相对不错。它直接集成到 WordPress 中。它甚至提供了与这些第三方服务相同的统计数据，你只需插入它，它就会立即为你的每一篇文章制作一个简短的 URL。](http://wordpress.org/extend/plugins/short-url-plugin/)

但是还有一些新的 WordPress 插件和用于所有内容管理系统和博客平台的插件即将问世。WordPress 的另一个例子， [la petite url](http://extrafuture.com/projects/la-petite-url/) ，它支持一个令人兴奋的功能，叫做[短 url 自动发现](http://relshorturl.appspot.com/)，这意味着除了为每篇文章创建一个短 URL，它还在你的文档头放置一个 [`<link>`标签](https://reference.sitepoint.com/html/link)，理论上，Twitter 客户端可以嗅到这个标签。当他们需要一个特定 URL 的短 URL 时，他们将下载页面并寻找那个`<link>`标签，如果它存在，他们将使用它作为短 URL。

因此，这将有助于确保当其他人引用您的内容时，他们也使用您首选的、自托管的短 URL。如果 Twitter 也这样做，而不是在 Bit.ly 可用的时候使用它，那就太好了，但这一切都是善意的，目前还只是理论上的。这是一个提议的协议。有几个网站正在这样做，但我还没有看到任何 Twitter 发布工具可以处理短 URL 自动发现。

这个想法是，你去这个网站，你上传一个，甚至一大堆你在你的网站上使用的图像文件。每个文件的最大大小为 500 千字节。它将分析每一个图像文件，并使用一些非常聪明的技术从这些图像文件中挤出额外的空间。这是连 Photoshop 的网页保存功能都做不到的事情。这是他们在所有分析中比较的基准。他们说，通常情况下，你会丢失 35%或更多的文件大小，即使是从 Photoshop 中导出。

他们这样做的方法之一叫做肮脏透明。出于某种原因，当您创建一个带有透明区域的 PNG 文件时，这些区域只是被标记为透明，但仍然有图像数据，您只是看不到，因为它已被标记为透明。该图像数据仍然占据图像内的空间。因此，punypng 所做的是用压缩效果更好的纯色来替换所有图像数据。
