# 三个网络开发者如何失去一个六位数的项目

> 原文：<https://www.sitepoint.com/how-three-web-developers-lost-a-six-figure-project/>

作为主要合伙人之一，我参与了一项技术投资。我们最近的里程碑是找到一家 web 开发公司来帮助我们设计、构建和构建我们的应用程序。我们采访了一些 web 开发公司，向他们展示了我们的规范，并经历了一系列密集的问题/对话。在这个过程中，我们选择面试的公司都有良好的业绩记录，并得到了高度推荐。

你猜怎么着？没有人进入决赛！原因如下:

–它们都没有提供“完整的解决方案”在所有情况下，没有一个提供了我们完成工作所需的所有专业知识，包括 QA、可用性和我们需要的全部应用程序。他们中没有一个人费心将这些职能作为分包商包括在他们的估算中。因此，我们很难理解他们会提供什么，不会提供什么，也很难估计最终产品的全部成本。

–没有一个给我们灌输信任，我们需要确信他们会在以后进行修改和改进，并且如果我们不得不更换供应商，代码会被很好地记录。

–没有人站在战略合作伙伴的角度，能够与我们就战略问题进行头脑风暴。他们都带着一种唯利是图、任务(相对于结果)导向的感觉对待项目。

–几乎没有一家公司拥有灵活、迭代、敏捷的方法，这是一家新公司(尽管资金充足)所需的方法。

–许多公司的态度有点浮夸，好像他们提议给我们这份工作是在帮我们的忙。

–许多公司在处理会议时不专业——迟到、团队成员缺席等。

所以他们都没有得到工作。相反，我们将自己管理项目，雇佣个体分包商来完成工作。这将花费我们更多的时间和精力，但却为我们节省了时间和金钱。

从本质上说，因为这些公司的行为就像供应商，而不是我们可以信任的战略顾问，我们选择将他们全部剔除，并雇佣一个由我们自己管理的独立承包商团队。

对你们中的一些人来说，这里有一个教训…
