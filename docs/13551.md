# Web 开发的现状:PHP 开发人员最有可能转向 Rails

> 原文:[https://www . site point . com/PHP-developers-most-likely-switch-to-rails/](https://www.sitepoint.com/php-developers-most-likely-to-switch-to-rails/)

在 SitePoint 和 Ektron 最近发布的《2006/2007 年 Web 开发状况报告》中的所有图表中，最让我印象深刻的是显示计划开始使用 Ruby on Rails 的开发人员当前使用的平台的图表:
![A graph of developers who have indicated they will start using Rails. The graph shows that PHP is by far the platform most used by these developers](../Images/1a724f0d12aa20c5424c0c44b3f1eeee.png)

图表显示 PHP 开发人员可能开始使用 Rails，这并不是事实——这是有道理的，因为如果有人已经在使用另一个开源平台，他们更有可能尝试另一个开源平台——而是事实:PHP 开发人员开始使用 Rails 的可能性是*的三倍多*，甚至超过 Java 开发人员！

这让我很惊讶，因为 Rails 运动的目标是 Java 开发人员(注意:这纯粹是我自己的观察，基于 Rails 的口号“能够愉快地编写真实世界的应用程序，比大多数框架做 XML 仰卧起坐花费的代码更少”)。对于 Java 程序员来说，Rails 可能看起来是实现相同结果的更简单的方法。但是对于一个只习惯于一起黑小应用的 PHP 开发者来说呢？

所以问题来了:PHP 程序员知道他们在做什么吗？这并不是说所有的 PHP 开发人员都在以一种非结构化的方式工作(我们都知道现在有大量的 PHP 框架为 Rails 提供 MVC 分离和其他类似的好处)，但是我想知道有多少 PHP 开发人员实际上在使用它们。看看外面的一些代码，我的猜测不到一半。

另一方面，Java 程序员已经更加自律了，如果仅仅是因为至少存在一个用于构建 Java web 应用程序的标准。那么热衷于迁移到 Rails 的 PHP 开发人员是训练有素的开发人员吗，Rails 可能是他们的天然选择？或者他们是黑客，他们还没有在他们的 web 应用程序中采用任何结构，但是被[各种](http://www.campfirenow.com/) [漂亮的](http://basecamphq.com/) [设计的](http://www.tadalist.com/) [Rails](http://backpackit.com/) [应用程序](http://www.strongspace.com/)所迷惑，并且被 [Rails 的主要传播者](http://www.loudthinking.com/)的魅力所打动？

我猜是后者，如果你急于跳槽，我的建议是:做好迷失方向的准备。我目前正在编辑 [SitePoint 即将出版的 Ruby on Rails 图书](https://www.sitepoint.com/premium/books/jump-start-rails)，虽然在 Ruby-land 花了这么多时间绝对是一件乐事，但毫无疑问，用 Ruby 编写代码需要一种完全不同的心态。一旦你完成了转变，事情变得非常直观，写起来几乎是一种乐趣。但是如果你以前的面向对象编程经验仅仅是 PHP 提供的方插圆孔的方法，那么 Ruby 可能是一个相当好的选择。一个好的头旋——就像[坐过山车](https://www.sitepoint.com/ruby-on-rails/)发出的嗡嗡声——但还是头旋。
