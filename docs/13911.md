# 验证码不是一种安全措施

> 原文：<https://www.sitepoint.com/captcha-are-not-a-security-measure/>

互联网上不时会出现一篇文章或博客，讲述基于图像(甚至是基于声音)的验证码(CAPTCHA)对于网络表单的安全性有多棒。我将指出为什么使用验证码来保证安全是一个疯狂的想法，为什么用它来控制垃圾邮件也是有缺陷的。

![CAPTCHA image with twisted letters](img/b5bfb10a60ab00703a6189f476c363ae.png)

验证码的第一个主要问题是，它们通常被认为是一种安全措施，而事实上它们并不是。验证码是一种简单的(自动的)区分人类和计算机的方式。它不会也不能测试用户是否可信。如果你为了安全而使用验证码，你是在一个错误的假设上工作，即人类可以被信任，而计算机(机器人)不能。

如果你的安全策略是“如果是人，我们可以信任他们”，那么你就有很多问题。

令人欣慰的是，大多数人意识到验证码根本不是关于安全的，它们只是试图减少(但不是消除)自动表单提交的许多方法之一。

因此，验证码的第一个基本问题是，它经常被认为是一种安全措施，而事实上它不是。认为验证码是一种安全措施就相当于认为所有人类都是值得信任的。

验证码的第二个主要问题是它们相对容易被利用。虽然一个人每小时只能看到一定数量的图像，但有大量时间的许多人每小时可以看到成千上万的图像。如果说互联网教会了我们什么，那就是互联网上有很多人有大把的时间。因此，虽然验证码会减慢一个人的速度，但它不会减慢数百人的速度。

这就是为什么免费色情剥削能够奏效。这是一种从网站上抓取验证码图片并显示在攻击者网站上的想法。攻击者的网站上有一个表格，指示其访问者识别图像中的字母，以便获得免费的色情内容。攻击者所要做的就是把一大群想要免费色情内容的人逼到那个表单上，然后他就能得到一大堆验证码的答案，然后他就可以自动地把这些答案提交给原始网站，来完成他的肮脏工作。

当然，它不一定是色情的——这只是说明验证码漏洞的一种流行方式。任何时候一个人想要什么东西，或者甚至有点无聊，你都可以让他们填写表格。在您的收件箱中获得免费笑话，请填写此表格。如果你让很多人对着验证码工作，验证码就会失效。

验证码的第三个主要问题，也是一个与安全性无关的问题，但我还是要提一下，它本质上阻碍了可用性和可访问性。

验证码将视力好的人和有图形浏览器的人排除在外。验证码的好处是由于它们的可访问性障碍。如果你让验证码更容易获取，它会变得不那么有效。替代品呢？对于盲人用户，你可以读出声音文件中的数字。现在你依赖于用户安装了正确的声音播放器插件。声音也需要更长的时间来下载，更多的可能会出错。那些喜欢使用文本模式浏览器或者因为带宽很低而隐藏图像的视力正常的人怎么办？

验证码也会损害可用性。即使是视力很好的人通过验证码也需要时间和努力。这是一个麻烦和烦恼。

提高可访问性最简单的方法是删除验证码，因为验证码的目的是降低可访问性。

如果一个验证码如此糟糕，有什么替代方案？

如果你使用验证码作为对抗安全问题的一种尝试，你需要重新考虑你的方法。验证码被破解作为一种安全措施。不幸的是，我认为最好的选择是在应用程序的规划阶段就考虑安全问题，而不是在垃圾邮件或企图利用开始涌入之后。设计一个安全的应用程序比让一个不安全的应用程序变得安全要容易得多，而这其中的一部分就是决定在你的应用程序中不包含什么。

在使用验证码来减少垃圾邮件的情况下，它们可以被其他方法代替。以我的经验来看，很多网上评论或表格都是由那些有很多时间可以出售或推广的人发出的。同样，由于验证码的第二个大问题(见上文)，我们看到验证码对于有很多时间的人来说是无效的。

处理这个问题的一个可能的方法是使用贝叶斯垃圾邮件过滤器。贝叶斯过滤器对人类和机器人提交的垃圾邮件同样有效。
