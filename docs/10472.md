## 新的用户界面

**图一。新的 UI**

![The new UI](../Images/5fe1f3e297522d6e54a9bf53a3f55993.png)

你首先会发现这种新金属有多薄。没有工具栏，极简的按钮，超薄的菜单栏，合并的搜索和地址栏:一切都变得非常干净和简单。

事实上，如果我没有弄错的话，它是开箱即用的最薄的菜单栏。

**图二。菜单栏高度(火狐 4b6，Opera 10.62，Chrome 6.0.472.63，Safari 5.0.2 和 IE9 Beta)**

![Menu bar height (Firefox 4b6, Opera 10.62, Chrome 6.0.472.63, Safari 5.0.2 and IE9 Beta)](../Images/769f9a65ee1e69617648cc2ccc548674.png)

细的菜单栏是一回事，美观大方；但是，像双彩虹一样，“这是什么意思？”嗯，这是你第一次注意到 IE9 最重要的特点是你最常访问的网站 。IE9 的目的是让你的内容浏览体验不再碍事。

这是一个关键的转变，旨在使 IE9 更好地反映您的网络体验和使用模式。其他发展更进一步。

例如，选项卡。最后，IE9 提供了令人敬畏的标签体验。它们现在可以很容易地撕下和拆分，而 Windows Snap 可以让您轻松地将它们放置在屏幕上。只需将一个标签拖放到屏幕上的任何地方，就可以给它一个自己的窗口。现在你可以拖动一个窗口到屏幕的每一边，窗口捕捉会自动调整它们的大小，使它们并排放置。您还可以将窗口捕捉到第二台显示器或另一个 IE9 实例中。

我最喜欢的 IE9 标签行为之一是，如果你通过 **控制** 从一个页面创建一个新的标签——点击打开一个超链接，原始页面和新页面都变成彩色编码，提供了一种将相关网站放在一起的视觉方式。这实际上从 IE8 开始就有了。

我们旅程的下一站是通知栏。任何时候 IE9 想要通知你任何事件——例如，下载、弹出窗口尝试、安全警告、密码保存等等——它都会在超级谨慎的通知栏中这样做。

**图 7。通知栏**

![Notification Bar](../Images/a6099fed3d4765270ef664b5a3a9fca6.png)

在您回复通知之前，该栏会一直显示在当前页面的底部。你可以继续使用该网站，并在你准备好的时候回复。

作为一个额外的奖励，你会看到当你运行几个下载时，你可以打开下载管理器来查看你所有的下载。您可以随意暂停和继续选定的下载。

## 硬件加速

我们 IE9 旅程的下一站是速度。微软已经尽了很大努力来确保 Internet Explorer 是最快的浏览器。这是一个重大的壮举，它在确保您获得最佳浏览体验方面处于领先地位。很久以来，当你的图形处理器无所事事地坐在沙滩上放松，啜饮着莫吉托鸡尾酒时，你的浏览器已经被降级为仅由 CPU 处理。不再有了，我的朋友们。是的，没错:IE9 在 GPU 中渲染你的 DOM(以及相关元素)。

这转化为一个现实世界的机会来建立改变游戏规则的在线体验，正如在[beautyoftheweb.com](https://www.sitepoint.com/blog/)网站上展示的那样。

我把硬件加速的网络比作构建桌面应用。桌面应用通常很快，连接到 GPU，并提供原生体验。这意味着，在执行需要大量计算的任务时，您不必缩减思维或代码。

想象一下在操作系统的文件管理器中查看 1，000 个缩略图的简单列表。现在，当你双击一个图像时，它会立即打开，或者当你缩放缩略图大小时，所有 1000 个缩略图都会立即缩放，对吗？以前，由于性能限制，您无法在浏览器中实现这种本机体验。有了 IE9，DOM 的 GPU 加速和 JavaScript 计算的优化让这成为现实。

真正了解这一点的最佳方式是浏览 Internet Explorer 试驾网站上的演示。 [Flickr explorer 演示](https://www.sitepoint.com/blog/)是一个非常令人印象深刻的例子，它展示了 IE9 如何利用其改进的性能在网站上呈现图像。它从 Flickr 上传来一堆图片，一旦图片到达，你可以点击任何一张图片将其弹出，然后再次放大。

**图 9。Flickr 浏览器**

![Flickr Explorer](../Images/24d6f0340eb3742b76b84bba29e341ec.png)

这些种类的原生体验之所以可能，部分是因为 GPU 加速。

在 Internet Explorer 试驾网站上有更多这样的速度演示，以及 IE 浏览器新功能的其他演示。随意[下载 IE9](https://www.sitepoint.com/blog/) 过去玩一玩。既然你在那里，为什么不与其他浏览器比较一下性能呢:根据我的经验，IE9 的评级非常高。请注意，要运行 IE9，您需要安装 Windows 7。

如果我没有讨论 IE9 如何在硬件加速之外还实现了一个全面的性能堆栈，那将是我的失职。

## 性能:不仅仅是看上去那么简单

IE 团队在他们的博客上发布了大量关于实现高性能浏览器的广泛考虑。它们涵盖了从 [JavaScript 基准](https://www.sitepoint.com/blog/)、[公共基准范围](https://www.sitepoint.com/blog/)、[构建页面中的多个子系统](https://www.sitepoint.com/blog/)，以及[一个页面是如何真正组成的](https://www.sitepoint.com/blog/)。

我想强调性能方面的一些常见考虑因素。一个比较流行的基准测试是测量纯 JavaScript 的。

事实证明，IE9 测试版自己处理，最快的 JavaScript 引擎和 IE9 的 Chakra 引擎之间的差距不到 100 毫秒。

**note:** Fun fact

这还不到人类眨眼时间(300-400 毫秒)的四分之一。

根据我的经验，浏览器性能差的首要原因是插件和附加组件。通常，当浏览器加载时间过长或在页面上挂起时，您可以指向插件、附加组件或工具栏。我并不是说应该避免加载项，但是确实需要关注它们，尤其是如果您希望针对快速加载时间进行优化的话。

首先，如果一个插件明显降低了加载速度，IE9 将通过通知栏发出通知。这将礼貌地通知你，你可能想检查你的加载时间。

## 站点锁定

最后一站，也是我最喜欢的创新是…站点锁定。我敢肯定，你现在已经听腻了，但是我觉得有必要再说一遍:IE9 把你访问的网站放在最前面。站点锁定就是一个典型的例子。

站点锁定是指将一个站点拖到 Windows 7 任务栏上，并在您需要时随时使用它的能力，就像它是一个 Windows 应用程序一样。

就是这样。只需从 OneBox(那是为老学生准备的地址栏)中抓取 favicon，然后放到任务栏上，你就有了一个固定的网站。

**图 15。钉住的网站**

![A pinned site](../Images/7d3bd7983f49f644f3d8c0039d0915c5.png)

在这个特殊的例子中，我用迈克尔·科尔达希的[个人博客](http://delicategeniusblog.com/)来演示。这里发生了很多事。让我们探索一下…

首先，你会注意到大头针图标比你的标准分辨率`favicon.ico`要大，标准分辨率通常是 16x16px。为了让你的`favicon.ico`在被钉住时看起来很棒，你需要编辑它至少支持 16×16、32×32 和 48×48。这将确保它在扩展的任务栏上看起来很棒。还要注意后退和前进按钮已经自动从`favicon.ico`中选择了强调色。

**note:** Fun Fact

。ico 文件支持多种分辨率。当你使用。ico 文件，并希望将它们放大。

接下来，你会注意到，即使你的浏览器关闭了，图标仍然在你的任务栏上。这使得访问者可以从他们的操作系统直接进入你的网站，就像他们已经安装的其他应用程序一样。

**图 16。一个未打开的固定网站**

![An unopened pinned site](../Images/dff03491ffe29d78d8a6bf6106e924ab.png)

现在，我们来看看跳跃清单。绝大多数 Windows 用户喜欢他们的任务栏应用程序和附带的跳转列表。IE9 让你能够添加一个跳转列表到你的站点，以便直接和更深入的访问。再次查看 Michael 博客图标的扩展上下文菜单，您会注意到有到他的主页、联系我和照片页面的直接链接。实现这一点很容易。

只需将其添加到您网站的标题中:

```
<link rel="shortcut icon" href="/favicon.ico" /> <meta name="msapplication-task" content="name=Home;action-uri=http://delicategeniusblog.com;icon-uri=http://delicategeniusblog.com/favicon.ico;" /> <meta name="msapplication-task" content="name=Contact Me;action-uri=http://delicategeniusblog.com/?page_id=674;icon-uri=http://delicategeniusblog.com/favicon.ico;" /> <meta name="msapplication-task" content="name=Photography;action-uri=http://delicategeniusblog.com/?cat=12;icon-uri=http://delicategeniusblog.com/favicon.ico;" />        
```

这里的语法相当简单；每个`msapplication-task`条目指向你网站上的一个 URL。这对你的静态 URL 来说非常好。但是 IE9 更进了一步。您可以动态地创建跳转列表项目。这是通过调用一个简单的 JavaScript 方法来完成的，如下所示:

```
<script type="text/javascript">  function initApp() {    // check of both IE9 and if the site is loaded from the taskbar.    if (window.external.msIsSiteMode()) {    // add the “Recent Visits” category in the jumplist    window.external.msSiteModeCreateJumpList("Recent Visits");    // add a jumplist item of the current page – this could be any URL    window.external.msSiteModeAddJumpListItem(document.title, document.URL, 'http://delicategeniusblog.com/favicon.ico');    // This will refresh the jumplist to reflect your changes    window.external.msSiteModeShowJumplist();    }  }  window.onload = initApp;</script>    
```

简单吧？您可以随时添加项目；你甚至可以指定一个替代者。每个跳转列表项的 ico 文件。

接下来，如果你的网站有活动，你可以放大任务栏上的主图标。如果你的访问者已经最小化了你的站点，这是非常有用的。

看看 Twitter 的这个实现。

**图 17。使用大头针图标跟踪网站活动**

![Use a pinned icon to track site activity](../Images/eb46f642f9e918767c16acde09f2b12f.png)

它通过叠加一个图标来显示新推文的数量。这可以通过一个非常简单的方法调用来实现:

```
// Adds an overlay iconwindow.external.msSiteModeSetIconOverlay(iconUri, toolTip);// removes an overlay iconwindow.external.msSiteModeClearIconOverlay();
```

最后，任务栏上的固定图标可以被指示从任务栏触发站点上的特定事件:

```
// add an event handlerdocument.addEventListener('msthumbnailclick', onButtonClicked, false); // add the buttonsvar btnPlay = window.external.msSiteModeAddThumbBarButton(iconUri, toolTip); // refresh the taskbarwindow.external.msSiteModeShowThumbBar(); // call a javascript function when the button is pressed.function onButtonClicked(e) {   switch (e.buttonID) {     case btnPlay:       play();       break;   } } 
```

结果是在 Michael 的任务栏上的固定图标上右击将显示控制频道 9 站点上的音频文件的选项，而不必在浏览器中打开该站点。

**图 18。使用固定图标触发动作**

![Use a pinned icon to trigger an action](../Images/47a3ac7782e483596ed5aa97716aaa87.png)
