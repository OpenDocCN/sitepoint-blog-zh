## HTML5 的时候到了

![HTML 5 Logo](../Images/2bee81e8e14013f943b154a12e26efbf.png)

HTML5 最近势头越来越猛。本周[它获得了 W3C 的推荐地位](https://www.w3.org/TR/2014/PR-html5-20140916/)，也让[得到了苹果](http://www.sencha.com/blog/apple-shows-love-for-html5-with-ios-8)的喜爱，因为很明显，iOS 8 的新功能将进一步推动它的发展。

HTML5 引入了许多新的标签，帮助我们编写更多的语义标记。这里有一个正确使用新的<时间>元素的指南。为什么就此打住？您还可以[使用日期输入创建一个日期选择器](http://demosthenes.info/blog/923/Using-The-HTML5-Date-Input)，这在以前只能用 JavaScript 实现。

<picture>元素为[提供了一种声明性的方法来加载图像资源](http://www.html5rocks.com/en/tutorials/responsive/picture-element/)，而< progress >元素为[提供了一条全新的途径来为长期运行的任务](http://www.htmlgoodies.com/html5/javascript/adding-patterns-and-animations-to-html5-progress-bars.html#fbid=ZsYpaAuUjXS)提供视觉反馈。</picture>

不确定哪些浏览器支持哪些 HTML5 功能？查看[caniuse.com](http://caniuse.com/)的综合列表。你也可以参考这篇(有点老的)文章来阅读 W3C 的 HTML5 交付计划[。](https://www.sitepoint.com/w3c-html5-2014-plan/)

## 本周流行语:Web 组件

![Polymer Logo](../Images/06632d978783af8bfe71865296a74670.png)

Web 组件是 W3C 引入的一组规范，允许在网页和 web 应用程序中创建可重用的小部件(或组件)。尽管浏览器支持目前还不完善，但许多开发者相信 web 组件将从根本上改变我们构建 web 的方式。

渴望了解更多信息？很好！这里有一个让你入门的教程。

[Polymer](https://www.polymer-project.org/) 是一组聚合填充，使我们能够在不兼容的浏览器中使用 web 组件。这里是[由一个聚合物开发者写的 web 组件指南](http://css-tricks.com/modular-future-web-components/)。同样的家伙也已经[发布了 YOLOmer！](http://www.youtube.com/watch?v=INH_OW4lFSs)–用于聚合物项目的约曼发电机。

Web 组件让 Jeremy Keith 既兴奋又紧张。阅读[他的博客文章](https://adactio.com/journal/7431)找出原因。

TJ VanToll 是另一个在这个问题上直言不讳的开发者。最近他宣布 [web 组件还没有为生产](http://developer.telerik.com/featured/web-components-arent-ready-production-yet/)做好准备，然后回来兜售并宣布[事实上他们是](http://developer.telerik.com/featured/web-components-ready-production/)。两篇文章读起来都很有趣。
