# Mac 上的 Windows 和 Linux 上的 Office

> 原文：<https://www.sitepoint.com/windows-on-mac-and-office-on-linux/>

最近的一个开发项目让我在不同的操作系统平台之间来回切换(感谢 KVM 交换机)。这并不是什么大不了的事情，因为它只有一个键盘、鼠标和显示器，我只是喜欢在深入构建和测试时使用一台或最少数量的机器。

首先，随着我的麦金塔电脑性能的大幅提升，我运行的是微软最新的[虚拟个人电脑](http://www.microsoft.com/mac/products/virtualpc/virtualpc.aspx?pid=virtualpc)，版本 7，运行的是 Windows XP Pro SP2。这使我可以在运行中同时访问两个操作系统进行任何测试。

和许多其他人一样，我的抱怨是 VPC 在以前版本中的迟钝。对我来说，这个问题在最新的版本中已经解决了(尽管我继续看到评论说不是这样)。不过有一点需要注意——我运行的是 1.25 GHz、1 GB 内存、120 GB 硬盘空间的 G4，这很可能有助于 VPC 的性能。我无法想象在这样的系统上运行这个软件。也许这是在新版本上产生混乱宣传的基准。

更棒的是，我最近发现了 [CodeWeaver 的](http://www.codeweavers.com)交叉办公室。虽然这并不新鲜，但我之前从未花时间研究过。然而现在，沉浸在 QA 中，我可以在我的 Linux 工作站上完成密集的测试脚本，并随时更新 Word 和 Excel 电子表格，而不用因为交叉办公而跳机。

当然，使用开放式办公套件是有争议的——然而——它并不总是符合企业客户的规格，包括用于质量保证的 VBA 增强型文档和电子表格。因此，在 Linux 上运行 Microsoft Office 的能力是非常宝贵的。

4.0 版本支持一些关键的应用程序，最有趣的是开发人员的 Macromedia Dreamweaver 和 Flash(后者被一些人认为是 Linux 金蛋——向 Linux 上的原生 Flash 开发迈出了一步)。它还运行 Adobe Photoshop、微软 Office 和其他几个程序。CodeWeaver 的网站上有一个全面的数据库，显示已测试和未测试的产品兼容性。
