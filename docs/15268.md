# AOL 电子邮件:为 AOL 用户格式化 HTML 电子邮件

> 原文:[https://www.sitepoint.com/format-html-email-aol/](https://www.sitepoint.com/format-html-email-aol/)

美国在线让世界各地的许多开发者和网络营销者感到沮丧。当你考虑 AOL 时，你必须记住你不仅仅是在处理一个定制的网络浏览器和 AOL 的电子邮件——你还必须考虑 AOL 如何运行他们的网络。

作为一般规则，我们通常向 AOL 订户发送电子邮件简讯的文本版本。据 AOL 几个月前报道，只有 50%的 AOL 用户升级到了 6.0 标准，该标准包含了比以前版本更好的 HTML 电子邮件支持。随着 AOL 7.0 的发布，我们预计会有更多的用户阅读 HTML 版本的时事通讯、电子杂志和其他商业通讯。

AOL 的浏览器只是 Internet Explorer 的定制“皮肤”或“品牌”版本。AOL 代理服务器使 Web 浏览器的行为不同于通过标准拨号或局域网连接的行为。这些差异的两个主要原因涉及 AOL 处理图形和处理缓存的方式。

### 这些东西中的一个和其他的不一样

一个常见的抱怨是，图像在 AOL 的浏览器上看起来模糊不清，在许多情况下，它们会出现黑条、失真或贯穿图形的线条。

美国在线利用其网络上运行的任何图形压缩软件来加速向其成员交付网页和 HTML。它通过压缩图像和在网络上运行的各种多媒体应用程序来实现这一点。AOL 使用专有的 Johnson-Grace 图像格式. ART。所有 BMP、GIF 或 JPG 图像都自动转换成。AOL 默认的艺术格式。为了避免这种情况，AOL 成员必须取消选择他们的 Web 首选项下的“使用压缩图形”选项。

### 如果可以的话，请把我藏起来！

一旦 AOL 用户选择不使用 AOL 的压缩图形选项，他们仍然必须清空他们的缓存才能看到差别。在他们的 WWW 首选项中，他们必须将其历史设置为“0”页，然后清除记录。完成后，他们必须删除临时 Internet 文件夹中的文件。AOL 就如何实现这一点提供了以下说明:

> 在浏览器窗口处于活动状态时，按住键盘上的“Control”键，同时用鼠标单击浏览器重新加载图标，也将清除您的代理缓存，并显示来自网站的最新数据。

### 一些背景信息

通常，当在 AOL 环境中查看时，HTML 电子邮件和网页设计中使用的背景将显示为“平铺”。这是因为 Johnson-Grace 压缩软件使用的压缩算法。AOL 将缩小超过 640 像素的 JPG。你可以用 GIF 来代替大背景的 JPG，或者试着把你的 JPG 保存为“渐进式 3 遍”，这是当前版本的 Johnson-Grace 软件所不能识别的。

### AOL 有自己的格式

AOL 只是适度支持字符格式。要为 AOL 格式化你的电子邮件，并了解你的电子邮件对 AOL 收件人来说会是什么样子，你应该从删除每行末尾的所有硬返回开始。

#### 对齐我的段落

所有段落都应该左对齐，没有缩进。每个句子之间应该有两个空格，因为这提供了空白。每次你想表示一个硬返回，把

#### “不要引用我的话…”

引号和撇号不能很好地翻译成 AOL 电子邮件。当您输入这些类型的字符时，请确保使用 ASCII 低位字符。此外，假设您在将文本/HTML 输入邮件之前，正在使用 Microsoft Word 或其他 Office 产品来准备文本/HTML。在这种情况下，您应该知道 Office 的自动格式化功能。

如果您从 Microsoft Word 中剪切和粘贴，总有可能有些格式不能很好地移植到 HTML 中。这在自动格式化的情况下最常见，当 MS Word 将标准的击键转换成符号时，包括-、“”和许多其他符号。这些被称为 Windows 字符，Word 将它们表示为浏览器无法理解的数值。

这就是为什么最好是在文本模式下工作，或者将你的文档保存为 dos 文本文档，并在传输到 HTML 电子邮件之前丢失所有格式。

如果您不使用记事本或 TextPad 等纯文本编辑器来编写内容，AOL 可能会无意中在您的电子邮件中插入奇怪的字符。

### 那不是很特别吗

您需要将所有特殊字符，即通过转义序列(™  )生成的字符，视为纯文本。目前还没有一个已知的解决方法可以让你有效地在你的 AOL 邮件中插入特殊字符。
