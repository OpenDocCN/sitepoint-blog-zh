# 电子邮件设计最佳实践

> 原文:[https://www.sitepoint.com/email-design-best-practices/](https://www.sitepoint.com/email-design-best-practices/)

有两种类型的电子邮件:纯文本和 HTML 电子邮件。每个使用电子邮件的人都见过没有格式的纯文本电子邮件。你仍然可以给你的订阅者发送纯文本邮件，但是在这篇文章中，我们将讨论 HTML 格式的邮件。

HTML 电子邮件应以多部分/替代 MIME 格式发送；这包括一个 HTML 版本，以及为关闭了 HTML 或缺少 HTML 功能的电子邮件客户端提供的纯文本回退。这意味着在编写和设计 HTML 电子邮件时，您仍然需要一个纯文本版本。幸运的是，大多数流行的电子邮件服务提供商会从你的 HTML 内容中创建一个纯文本的替代品。

## 设计约束条件

在很多方面，设计 HMTL 电子邮件与设计网页非常相似。它们使用 HTML 和 CSS 编写，并使用浏览器呈现。

如果你设计网站的时间足够长，还记得浏览器大战，那只会糟糕十倍。不同的电子邮件客户端会以不同的方式呈现电子邮件，而且有很多电子邮件客户端。有桌面电子邮件客户端，如 Microsoft Outlook、Windows Mail 和 Apple Mail，web 服务，如 Gmail、Yahoo Mail 和 Microsoft Hotmail，还有移动电子邮件客户端。

**样式表**

一些最大的电子邮件客户端(*咳咳*，Outlook，*咳咳*)仍然使用古老的渲染引擎，并提供糟糕的 CSS 支持。因为他们有很大的市场份额，你必须设计到最小的公分母。一些基于网络的客户端，如 Gmail，在支持 CSS 功能方面也非常有限。

本质上，你应该像过去为 Internet Explorer 5.5 设计网站一样设计你的电子邮件简讯。你需要使用内嵌样式，并且应该避免使用 CSS3。

关于流行的电子邮件客户端和每个客户端支持的 CSS 的完整列表，请查看 [CampaignMonitor 的电子邮件客户端 CSS 支持指南](http://www.campaignmonitor.com/css/)。

**尺寸**

电子邮件窗口大小不一，但由于边栏和导航的原因，通常比浏览器窗口小。最好让你的邮件宽度保持在 500 像素到 600 像素之间，或者使用流畅的布局。

**保持简单**

避免在电子邮件简讯中嵌入视频、JavaScript 或 Flash。如果你想显示视频，创建一个视频播放器的图形(甚至是截图)，并简单地将它链接到嵌入在你的网站页面上的视频，或基于网络的电子邮件版本。

**行动号召**

你希望你的订户做什么？你希望他们访问你的网站或博客来阅读一篇文章吗？你想让他们购买产品吗？确保清楚地定义你邮件的目标，并在邮件中包含几个清晰的行动号召。不要以为你的访问者会知道该做什么；将每封电子邮件视为一个独立的登录页面。

## 测试

Litmus 是一项电子邮件测试网络服务，它将向你展示你的信息在所有流行的电子邮件客户端，包括桌面和网络客户端，是如何显示的。 [CampaignMonitor](http://www.campaignmonitor.com) 提供了 Litmus 集成，因此你可以在不离开软件的情况下测试你的邮件。 [MailChimp](http://www.mailchimp.com) 有一个类似的内置服务，将执行相同的功能。

你也可以在每一个主要的网络服务上设置测试邮件账户，并在几台电脑上安装不同的桌面客户端。显然，除非你有几台计算机和大量的时间，测试服务可能会更快更容易。

无论您使用哪种方法，一定要测试任何新的或有实质性变化的模板，以确保它们在流行的电子邮件客户端中显示良好。
