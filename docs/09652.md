# 为移动设备设计—打造更好的鼠标

> 原文:[https://www . site point . com/design-for-mobile-build-a-better-mouse/](https://www.sitepoint.com/design-for-mobile-build-a-better-mouse/)

许多，如果不是大多数，新一代的移动设备使用触摸作为他们的主要输入方式。虽然我们通常应用于界面设计的许多原则是相同的，但是在思维模式上需要一些改变。

我们的手指非常灵巧，但却没有老鼠那么精确。只要小心，我们可以用鼠标精确定位界面上的单个像素——但尝试用手指触摸单个像素。用于 iOS 的苹果人机界面指南规定了不小于 44×44px 的推荐点击目标；比我们单个像素大 2000 倍左右。这是否意味着依赖触摸作为输入法的界面是一种倒退？当然不是。触摸作为一种交互方式，就是要去除我们和我们的界面之间的那一层。我们在准确性上失去的，我们在对交互的更好理解中得到，因为这种体验现在是可触知的，因此更直观。

无法适应触摸模式的界面并不是因为输入法的任何内在缺陷，而是因为设计者在屏幕上塞了太多东西，或者把按钮做得太小，手指无法轻松点击。触摸作为一种输入方式所固有的问题加剧了这一点:试图触摸界面元素的行为本身会使我们看不到该元素。此外，我们的用户需要能够在各种令人分心的环境中理解和使用我们的界面。在试图将尽可能多的信息放入移动屏幕的较小物理空间和提供足够大的目标以供笨拙的手指触摸之间有一个微妙的平衡。

我们也不应该忘记没有触摸屏的用户。他们也是顾客，他们的输入法可能会非常不友好。旧的功能手机(和一些智能手机)使用四向导航(通常称为 D-pad)作为他们的主要输入方法，迫使用户滚动我们界面中的几个元素来找到他们想要的按钮。黑莓的浏览器使用微型轨迹球或滚轮来导航界面；因此，只要有可能，限制屏幕上的元素数量是值得的。

最重要的是，它意味着简单的接口。简单意味着容易理解，从而导致易于使用。

简单是一个特征，而复杂不一定是一个缺点，我们需要保持一些观点。尽管我们可能在应用程序的深度功能上投入了很多，但是用户的行为不太可能符合我们的兴趣。如果我们能说服用户访问我们的应用程序，用户很可能只花几秒或几分钟的时间。

*   他们想要什么？
*   他们期望什么？
*   我们想让他们做什么？

我们不能假设用户有时间(或者倾向于)弄清楚我们的应用程序是如何工作的。

### 1.1.盘旋我

为触摸设备设计需要我们思维的转变。我们已经习惯使用的许多技术不再起作用，或者至少不像我们希望的那样起作用。其中最明显的是 hover:

仅依赖于 mousemove、mouseover、mouseout 或 CSS 伪类:hover 的元素在 iPad 或 iPhone 等触摸屏设备上的行为可能并不总是如预期的那样。— [摘自苹果公司的《为 iPad 准备网络内容》指南](http://developer.apple.com/library/safari/#technotes/tn2010/tn2262/index.html)

悬停作为一种交互模式渗透到网络中。我们习惯于鼠标移动触发悬停页面的变化:链接的不同颜色或状态，显示下拉导航，显示可操作的项目，等等。作为设计师，我们已经欣然接受了悬停状态给我们带来的可能性。大多数基于触摸的操作系统会在幕后做一些工作，试图确保界面以非灾难性的方式处理悬停状态，但我们将不得不开始改变我们的习惯。例如，代替悬停状态，考虑:

*   使按钮和超链接变得明显
*   拥有不依赖于`:hover`的内容；例如，增加文本的对比度
*   避免没有清晰视觉提示的下拉菜单

我们可能会失去一点天赋，但我们会获得清晰的界面。

### 1.2。小屏幕

不可避免的是，为移动设备设计意味着为小屏幕设计。移动设备的显示屏在物理尺寸和分辨率方面都比台式机小得多。这是一个被拥抱的机会。

然而，在小屏幕上显示信息和界面之间找到合适的平衡是一个棘手的问题。过多的细节会使我们的界面变得混乱和令人困惑。太少，我们的用户将不得不努力找到他们需要的信息。这并不一定意味着减少内容；这意味着减少混乱。

换句话说，不要害怕创建一个信息丰富的界面。一个精心设计的界面可以容纳大量的信息并有效地进行交流。在交互背后隐藏信息可能是阻力最小的路径，但不一定是我们应该走的路。人们使用他们的移动设备来完成任务或寻找信息:他们想知道电影何时播放，下一班火车何时到达，或者最近的咖啡馆在哪里。他们不愿意花时间去探索一个稀疏而微妙平衡的界面。我们希望在不使用户不知所措的情况下，向他们提供尽可能多的信息。

### 1.3.认知负荷

简化界面实际上是为了减少我们给用户带来的认知负担。这本质上是 [费茨定律背后压倒一切的原则。](http://en.wikipedia.org/wiki/Fitts%27s_law)对于外行人来说，费茨定律是一个交互模型，它已经成为理解用户界面设计的基础。它指出，获取目标的时间——比如说，将鼠标移到按钮上——是到目标的距离和目标大小的函数。简单地说，项目越大，离光标越近，就越容易点击。

适应这一原则的一个经典例子是 Mac OS X 中的菜单栏。它是一个小目标，通常离你的光标通常所在的位置相当远；然而，这是通过将菜单放在屏幕顶部来平衡的，防止用户超过目标。位于边缘有效地赋予了菜单栏无限的高度，导致用户更少的错误，更快地到达他们的目标。然而，对于触摸屏来说，菲特定律必须以不同的方式应用:我们的用户并不受限于他们鼠标的位置，所以他们移动的起点只是他们手指或拇指的默认位置。该位置在设备及其方向上变化很大；例如，对于移动设备，您可能会使用一只手的食指或两只手的拇指。

这里有一个实际应用中的例子。《无限之刃》是一款非常受 iOS 欢迎的游戏。它可以在 iPhone 和 iPad 上使用，两种设备使用相同的界面。游戏在横向模式下使用设备进行，控件固定在屏幕底部(你的拇指所在的位置)，如图 2.1所示。在 iPhone 上,“施法”按钮位于屏幕中间，任何一个拇指都可以触摸到。然而，这一功能在较大尺寸的 iPad 上就没那么有效了。“施法”按钮仍然在屏幕中间，但不再在我们手的默认位置范围内。

## 为什么重要？

据估计，到 2013 年，智能手机和其他装有浏览器的手机的总数量约为 18.2 亿部，而个人电脑为 17.8 亿部。众所周知，很难找到手机浏览器使用的可靠统计数据，但不管来源如何，趋势是明确的。根据 StatCounter 的数据，移动端在整个网络浏览中的份额目前为 4.36%，虽然这个数字看起来很小，但请记住，在过去两年中，这一数字增长了 430%。而这仅仅是移动浏览的开始。我们花在手机和其他移动设备上的时间永远不会比现在少。不可避免的是，更强大的移动设备和无处不在的互联网接入将成为常态。这些设备的使用环境将会迅速改变。我们的潜在客户使用移动设备的可能性越来越高。我们忽视移动网络是危险的。
