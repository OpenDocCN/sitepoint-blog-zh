# 收入者面临的定价挑战

> 原文: [https://www。站点点。com/a-定价-挑战-针对那些年收入低于七万英镑的人/](https://www.sitepoint.com/a-pricing-challenge-for-those-earning-less-than-70000-per-year/)

上几篇关于定价的帖子很能说明问题。你们中的一些人对最有利可图的定价方式有清晰的想法(例如，基于价值)，而你们中的一些人似乎陷入了低小时费率的泥潭。

对于后一类的你，你愿意走出那个泥潭吗？下次你和一个潜在客户谈论工作时，试试这个方法。询问许多关于您的解决方案将提供的商业利益的好问题(例如，“您的这个项目的商业目标是什么？”"假设你的网站很成功，你希望能多创造多少收入？""拥有一个更专业的网站感觉如何？""你现在在网络上可能会错过哪些类型的客户？").

2。根据其他客户(或其他网页设计师的客户)的体验，讲述你的解决方案的价值。

3。然后给个报价。有时，我甚至会问潜在客户他们愿意支付多少来获得这一价值，然后告诉他/她，在他们的预算允许的情况下，我能为他们做什么(以及不能做什么)。或者，如果以上对你来说太可怕了，尤其是如果你每小时的收费低于 35 美元，那么这样做:

下次你给一个项目报价时，试着要求比你通常的价格高出 50-100%。看看会发生什么。最糟糕的情况是，你以更高的价格“锚定”潜在客户，让他们想知道你提供的东西有多好。你总是可以降下来，突然之间，你平常价格的 25-50%对他们来说就不算离谱了。

尝试这些方法之一。它们在一定程度上是零敲碎打的，并没有涵盖你获得高利率所需要做的一切。

但无论如何要试试。如果你不这样做，那么你就没有权利抱怨“定价过低的自尊”或抱怨无法为你的服务定价以获得你想要的价格。
