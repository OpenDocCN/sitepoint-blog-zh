# 文档:一个重要的商业工具

> 原文：<https://www.sitepoint.com/documentation-a-vital-business-tool/>

上周，Miles [告诉我们](https://www.sitepoint.com/when-the-boss-is-away/)他因病意外休假，以及他的公司没有他是如何经营的。这让我想到了自己的处境。作为一名小企业主和一家机构的互动总监，我每天或每周都要完成数百项只有我自己知道如何完成的任务。其中一些是普通的技术任务，而另一些则需要更高层次的思考过程。有些人可能认为这是工作保障，但我知道这是一种责任。在绝对最坏的情况下，你可能会死亡或残疾。更有可能的是，你会时不时地生病，你可能会喜欢时不时地休假，而不会让你的手机不停地响。当我意识到在我工作的地方没有人能完成的任务数量时，我决定开始尽可能多地记录程序。下面是我如何开始记录日常程序的概述。**密码**要记录的最重要的信息之一就是你的密码。我使用 [SplashID](http://www.splashdata.com/splashid/) ，一个密码管理应用程序，来存储我所有的用户名、密码和其他重要信息。该软件本身也有密码保护，数据是加密的，所以如果我的笔记本电脑或 iPhone 丢失或被盗，它会保持安全。这些文件可以导出并存储在安全的地方。有很多方法可以管理密码，但关键是不要成为唯一知道密码的人。确保您将密码存储在安全的地方，并且在紧急情况下同事可以访问它们。我已经在我们的 [Google Apps](http://www.google.com/a) 账户中使用 [Google Docs](http://docs.google.com) 记录了程序。我创建了一个名为“流程和程序”的文件夹，并与我们整个团队分享。在其中，我已经创建了大约十几个文档(到目前为止)，详细介绍了如何执行不同的任务。如果你真的想一想流程中的每一步，即使是我们认为理所当然的简单程序也会显得相当复杂。例如，我记录的流程之一是购买或更新 SSL 证书。一开始听起来比较简单，但是程序其实挺复杂的。**包括什么**在记录程序时，避免含糊的命令，而应包括逐步说明。使用项目列表，并添加截图，或者，如果你有时间，添加实际过程的视频。越详细越好。你需要足够的信息，这样任何人都可以领会指令并成功完成任务。**保持文件更新**一旦你有了大部分(如果不是全部)的程序文件，确保它们保持更新。密码经常改变，web 服务也在不断改变它们的用户界面，所以文档很快就会过时。如果不准确就没用了，所以一定要在注意到任何变化的时候更新你的文档。内心的平静我在记录这里的所有程序时仅仅触及皮毛，但我已经感觉好多了。知道一个同事可以在紧急情况下很容易地挺身而出，在很少或没有指导的情况下接管工作，让我对长时间不在办公室感觉好多了——无论是计划内的假期还是计划外的紧急情况。现在记录你的过程的一个积极的附带好处是，如果你雇佣一个人来执行这些任务(也许是为了腾出你的一些时间)，你会有一份详细的手册。他们可能仍然需要一些额外的培训，但你将有一个良好的开端！
