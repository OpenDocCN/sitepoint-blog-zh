# 测量系统安全性

> 原文：<https://www.sitepoint.com/measure-system-security/>

作为对[更改默认密码](https://www.sitepoint.com/blog/)的跟进，我对我管理的许多系统进行了全面的安全审计，作为每月例行管理的一部分。

正如许多读者所知，我试图在我的许多专栏和博客文章中注入安全意识，引用身份验证、入侵检测、垃圾邮件预防和其他因素/产品来保护您的开源系统。因此，我决定探索互联网安全中心的 Linux 基准测试工具。目前它只支持 Red Hat 平台(Enterprise Linux 和 Fedora 内核)。

安装非常简单，只需解开下载并切换到 root 用户即可。该工具是只读的，因此在执行脚本时不必担心。在转移到任何客户硬件之前，我在我自己的专用 web 服务器上运行该工具作为测试。这个特定的服务器运行 Fedora Core 3，使用 iptables 作为防火墙，只允许 ssh 和 sftp 用于远程访问。它还包含常见的 LAMP 平台守护进程、用于入侵检测和配置文件更改控制的 Tripwire for Linux，以及作为 MTA 的 QMail(运行 vpopmail、qmailadmin、tcpserver 和 spamassassin)。

这个盒子运行得很严密，尽可能不留下入侵者可以溜进去的缝隙。然而，CIS 透露了一个令人大开眼界的数字细节，用于调整系统，将其收紧到一个几乎完全硬化的水平(除了关闭 Apache、Qmail 和 MySQL)，超出了我在构建它时加快的调整。

特别是，我喜欢该工具注意到启动或运行操作系统不必要的服务，这些服务可以在下次重新启动时禁用(例如，用于硬件发现的 Kudzu 对于桌面场景很方便，但对于服务器不是必需的，如果不需要打印，则使用 CUPS 进行打印，等等)。).

理想情况下，在 Red Hat 调味盒的干净构建之后运行该基准测试是最佳的——在将其投入生产之前对其进行调整。因为我们并不是生活在一个完美的世界中，所以我会简单地将这个工具构建到我定期检查和重新检查服务器的过程中。我鼓励所有阅读这篇文章的人都这样做。

如果您运行的是 BSD、Solaris 或 Windows，请参见 [CIS 主页](http://www.cisecurity.org/index.html)，因为有针对多个平台的基准测试工具。
