# 博主入门

> 原文:[https://www.sitepoint.com/bloggers-primer/](https://www.sitepoint.com/bloggers-primer/)

对许多人来说，博客是生活的一部分。你可以保留自己的博客，或者选择通过日常阅读访问其他几个博客。你甚至可能会坐下来想，“哎呀，我想写博客，但我不认为我会做得很好。”这篇文章旨在给你一个很好的起点，让你可以成功地进入博客世界。

博客不是一种技术，一种技巧，或者一个很酷的把戏。博客是内容网站的一种形式；重要的是它所呈现的信息，以及这些内容满足读者需求的方式。

新兴的博客写手需要首先考虑他们要写博客的领域，以及博客适合哪些受众或读者。下一步要考虑的是确保你的博客可以很容易地被读者通过搜索找到。博客背后的技术，以及它所基于的平台，是等式的最后一部分。

考虑到这种以内容为中心的优先级顺序，本文将采用一种稍微非正统的方法。首先，我们将简要探讨博客的概念:它是什么，为什么人们这样做。然后我们将讨论读者群，以及如何利用你的博客内容来吸引和留住目标读者群。一旦完成了这些，我们将更仔细地看看吸引读者的关键手段之一:搜索。我将解释一些可以确保你的博客被搜索引擎抓取和索引的技术。最后，我们将讨论博客平台的问题。市场上有各种各样的博客软件，但是希望这个讨论能够帮助你选择一个适合你需要的工具。

##### 为什么写博客？

很简单，博客是“网络日志”，通常以个人化的方式记录相关事件、发生的事情和事件，就像个人日记记录个人日常生活中发生的事情一样。博客的关键方面是时间:博客倾向于频繁更新(可能每隔几天)，并且它们经常呈现与时间相关的内容。

除此之外，博客是一种开放的格式。在网络上，你会发现博客关注你能想到的每一个话题。一些博客——例如绿色小足球或 insta punt Glenn Reynolds——考虑政治问题。

住在巴格达的牧师刘易斯从不同的角度看这场战争。Johnny Gulag 和 Root Prompt，就像 [SitePoint 博客](https://www.sitepoint.com/blogs/)一样，给世界带来了 Web 开发的话题。其他博客只不过是个人的私人日记。基本上，如果你有一种兴趣可以支持频繁的“报道”，那么你可以考虑写博客！

无论主题是什么，某些普遍原则适用于如何最有效地实施和管理博客。现在让我们来考虑这些原则。

##### 读者群

首先，让我们看看博客更抽象和无形的一个方面:读者群。这是什么？我们如何得到它？读者从哪里来？这些问题很重要，因为是读者让博客保持活力。

“读者群”指的是你的博客的访问者:它的观众。在这一部分，我们将从两个角度考虑读者群。首先，我们会看到你的博客如何吸引读者；其次，我们将讨论如何留住这些读者，并鼓励他们将来再次访问你的博客。

***获取读者***

你有一个兴趣，并且，通过你的博客，你想和世界上所有有同样兴趣的人分享你的想法。但是，你怎样才能接触到这些感兴趣的人呢？什么会促使他们访问你的博客？博客写手手边有许多工具可以帮助他们建立并继续建立稳固的读者群。

*口碑*

大多数美国人、加拿大人、澳大利亚人和欧洲人现在都能上网。许多人在家使用网络，但那些不在家的人通常可以在工作场所、通过网吧等方式上网。这意味着你可以很容易地把你的新博客告诉你的朋友、家人和——对博客发展最重要的——其他博主。你会惊讶地发现，有多少人对你要说的东西感兴趣，并会不厌其烦地阅读你的博客…如果他们只知道你写的话。

*其他博客*

推广一个博客最有效的方法之一，根本不花钱，就是阅读其他博客。博客圈里到处都是和你有相似或相关话题的人。找到那些博客。阅读它们。评论他们的作品。链接到他们的故事。为他们做贡献，他们也会为你做贡献。

然而，为博客投稿和仅仅创建一条信息作为链接到你自己博客的工具之间有一个关键的区别。这可能是一个大意如下的帖子，“嘿，我在 www 上的博客上写了这个…去看看吧！”糟糕，糟糕，糟糕。这样的帖子被称为垃圾评论；他们不被认可，而且几乎总是被博主淘汰。

*引用通告和 Pingbacks*

博客中最容易被误解的概念之一可能是引用通告和 pingbacks。不要太专业，引用通告是一个博客通知另一个博客它已经引用了那个博客上的帖子的方法。许多博客条目会提供一个“引用通告 URL”，这是一个特殊格式的 URL，博主 A 可以使用它来通知博主 B，他或她的帖子已在博主 A 的站点上被引用。您不会使用此 URL 链接到另一个博客上的帖子，但是，根据您使用的博客平台，可能有一种方法可以在您的条目中包含引用通告 URL。

Pingbacks 与 trackbacks 非常相似，尽管它们不需要您的参与。不是所有的博客平台都支持 pingbacks。WordPress 支持它，并且 [Textpattern](http://www.textpattern.com/) 在我最后一次检查的时候正在做一些非常接近它的东西。Pingbacks 可以更容易地理解为一种技术，允许“自动发现”链接到您的博客，而无需使用 trackback。

*一致内容*

我通过自己的“blogroll”(一种友好或相关博客的列表)链接到其他博客时使用的规则之一是，这些博客必须至少每隔一周更新一次。这是最低限度。最好是我链接的博主每隔几天就会发内容；理想情况下，他们会每天发布。每个人都明白，人们在博客之外也有自己的生活，有时可能会休假或在短时间内失去写博客的动力。但是，在最成功的博客上，内容是每天或每隔一天发布的。这让事情保持新鲜。它让读者每天都想回来看看你今天要说什么。

当然，所有这些技巧都需要时间:它们不会在一夜之间创造成功。成功的博主是坚持不懈的。发展读者群需要时间。事实是，没有人会知道你的博客，直到他们以某种方式意识到它。当人们开始发现你的博客，给它做书签，并定期访问时，神奇的事情就来了。

上面的方法会让访客进门，这是重要的第一步。但是他们不会让读者回来。现在来讨论一下读者留存。

***留住读者***

有一件事，而且只有一件事，会让你的读者保持兴趣:内容。

想想你每天访问的一些网站。你为什么去拜访他们？也许你每天都去 CNN.com，因为你想知道当今世界发生了什么。也许你会访问雅虎 T2！梦幻体育因为你已经组建了自己的梦幻体育队。或者你可能会在 Fark 停下来，每天开怀大笑。

不管你怎么看，不管你访问什么网站，有一件事是真的:我们访问的网站为我们的时间和努力提供了回报。出于这个原因，定期提供好的内容以保持你的读者有兴趣回来是很重要的。

那么什么定义“有趣”的内容呢？什么会吸引你的读者？下面是几个答案。

*当前条目*

我所说的“当前条目”是指与当今问题相关的条目。如果你写一个科技博客，如果你写的是 Windows '98，你的结果可能会很差。现在很少有人使用或关心 Windows 98。他们关心 Windows 2000 和 XP。所以，也许你的时间和精力花在这些操作系统上会更好。如果你喜欢写政治，如果你谈论乔治·布什 1993 年的增税，你可能不会得到多少读者，尽管政治是一种有趣的动物，你可能会证明我是错的。

底线是让你的内容关注并与今天的生活相关。

*相关条目*

这与第一点相似，但有所不同。有两种博客:一种是为自己写博客的，另一种是为别人写博客的。第一种博主写作是为了给自己一个出口；第二种类型试图满足读者的需求。当第一种类型试图成为第二种类型而失败时，问题就来了。

建立读者群需要时间。如果这是你的抱负，那很好——但先走后跑。准备好付出努力，做一个优秀的三流博客写手。关注你的读者——解决他们想阅读的话题，让你的帖子与他们的经历相关——成功迟早会到来。

*正规英语*

这是我最讨厌的:喜欢在 camelcase 写文章的博客作者(“我去了商场，和 Jj 在一起”)和对英语掌握不太好的博客作者(“dat”不是一个单词)。也不是“霍拉”)。

博客是关于写作的。你的写作可能是你的读者了解你的唯一途径，所以不要让你的写作让你失望。如果你的读者对你的唯一了解是你上一篇博文是没有教养、没有计划、没有方向的胡言乱语，你的声誉可能会长期受损。

*做你自己*

Sun Microsystems 在它的[公司博客政策](http://www.tbray.org/ongoing/When/200x/2004/05/02/Policy)中很好地总结了这一点:

<q>另一种有趣的方式是暴露你的个性；几乎所有成功的博客写手都写自己，写家庭、电影、书籍或游戏；或者他们贴出图片。人们喜欢知道什么样的人在写他们正在读的东西。再次呼吁平衡；博客是一个公共场所，你应该尽量避免让你的读者或公司尴尬。</q>

换句话说，说自己也行！是鼓励！但是我不想知道你去购物中心的事，除非这件事对我产生了影响——不管是一次有趣的交流还是你可能遇到的能说明我生活中某些事情的事情。

从表面上看，写博客似乎是一件容易的事情。但是要想写好博客就稍微难一点(虽然不一定难)。写博客需要努力和专注。你在博客上付出的所有努力对外界来说都是透明的。你可能有网络上最好的设计，由世界知名的设计师设计，他提供了你能想到的所有养眼的东西，但是如果内容不相关，这个设计就不会有丝毫的不同。你的养眼产品可能会赢得一些点击率，但不会留住他们。

##### 为搜索而建

现在，让我们深入博客的技术层面。在这里，我们将考虑搜索技术、搜索引擎以及我喜欢称之为“智能代码”或“智能 HTML”的一些基本方面作为一个博客作者，至少以一种基本的方式理解这些概念是值得的。

大多数人看到和解释的视觉 Web 和计算机解释的语义 Web 之间有一个关键的区别。自从谷歌、MSNs 和雅虎。世界上所有的人都是电脑，对于博客作者来说，试着通过他们的眼睛看世界是个好主意。

顺便说一句，这种争论很容易蔓延到 Web 开发的另一个领域，这也是最近的热门话题:可访问性。在这里，我将尽量避免这个特殊的话题，而是关注博客如何受到糟糕的标记的影响，包括糟糕的链接和糟糕的帖子主题行。

***标注***

标记是我们在网站上使用的 HTML 和 CSS 我们网站背后的样式。大多数博客平台处理主题、样式表、模板等方面的大部分标记，这确实减轻了博客作者的标记负担。然而，你可以控制你的博客条目。你决定哪些单词将被斜体、粗体或(上帝禁止)闪烁。

搜索引擎，如谷歌和雅虎！从上到下，从左到右阅读你的站点，忽略他们不理解的东西。他们了解的一些最重要的特性是:

```
<h1> - <h6>
```

这些是标题标签。大多数博客平台都给你空间来输入主题，然后主题就变成了条目的标题。这是相当透明的，但有些人可能想把其他标题放在他们的条目中。这完全没问题，尽管应该考虑某些因素。搜索引擎将标题标签理解为标题(自然地)，因此非常重要。标题优先:`<h1>`是最重要的，通常预留给页眉和标题。`<h2>`不如`<h1>`重要，但比`<h3>`重要，以此类推。不管怎样，在 [WordPress](http://www.wordpress.org/) 中最常用的条目标题是一个`<h2>`；使用标题的条目应该使用`<h3>`标签和下面的。

这些标题的内容在搜索中被认为是非常重要的，所以应该用来有效吸引搜索引擎。为你的标题使用描述性的文字，你会得到更好的搜索结果。标题为“改天”的主题不太可能获得很多点击。然而，一个主题，如“微软又毁了一天”，将获得更多的点击(包括我的！).

```
<a>
```

锚定标签是超链接中的关键成分。现在，如果你像这样编码一个链接，`<a href="http://www.espn.com">ESPN.com</a>`，链接将显示为:[ESPN.com](http://www.espn.com)。但重要的是要记住，搜索引擎非常仔细地查看链接的文本，在这种情况下，“ESPN.com”。这个描述告诉搜索引擎——和你的用户！—链接页面是关于什么的，并作为搜索的关键字。明智地使用你的文本来获得最好的搜索结果，你会吸引更多的读者。此外，一定要经常链接到很多地方。你链接得越多，你就会收到越多的互惠链接(链接回你的网站)。

*标签语义*

获得符合标准的 HTML 的基本概念的一个好地方是 W3Schools.org 的 T2，或者，如果你是一个技术阅读爱好者，可以去官方的 T4 XHTML 1 过渡规范 T5。然而，其中一些网站并没有解释为什么某个标签比其他标签更重要。曾经，如果用户想用粗体显示字体，他们可以简单地使用`<b>`标签，意思是“粗体”在语义网上，这个标签被替换为`<strong>`。对于人类的眼睛来说，它们做的是相同的事情，但是在语义网上，区别更加明显。

区别在于展示和强调的概念。“粗体”的意思就是“加粗选定的字体”“强”是指“将这一点作为一个重点来“强烈”关注。”对于`<i>`和`<em>`标签也是如此。它们都显示为斜体，但是`<em>`标记表示一种特殊的强调。

搜索引擎发现了这些细微差别。如果你使用的博客软件努力保持符合标准，他们可能会为你提供工具，帮助你写符合标准的条目。例如，WordPress 提供了可用于此目的的快速标签。

*永久链接*

这个术语是博客作者发明的一个奇特的词，用来表示单个条目的绝对 URL。这是一个永久的链接，即使六个月过去了，这个故事已经从博客的首页消失了，仍然可以在这个链接上找到它。

一些搜索引擎不喜欢包含大量变量的 URL，例如，`<a href="post.php?post_author=administrator&post_id=474">Permalink</a>`这个链接可以被描述为一个非搜索引擎友好的 URL:许多搜索引擎看不到任何超过`?`的内容。

MoveableType 和 [WordPress](http://www.wordpress.com) 都提供了创建搜索引擎友好的文档和 URL 的方法。但是，即使你的软件没有提供一个简单的方法来做到这一点，只要这个站点是在一个非 Windows 服务器上运行一个 mod_rewrite-enabled Apache 服务器(老实说，它描述了大多数 Web 主机)，那么搜索引擎友好的 URL 是可以生成的。这并不总是容易的，但它可以，也确实应该完成。

创建一个索引良好的博客，并获得良好的 Google PageRank 和 Yahoo！位置。但是除此之外，你还需要努力在你的帖子中加入关键词——如果你愿意的话，也可以称之为“流行语”——以确保你会因为发表热门帖子而被搜索引擎认出来。就我而言，这是博客发展最关键的方面。多花一分钟确保你的所有链接都使用描述性文字，你的标题不含糊，也许，它们也不太具体(这会缩小你的搜索受众)！

##### 博客平台

没有软件的博客算什么？让我们来看看 blogger 可以使用的一些平台，看看在安装和维护任务、资金投入和易用性方面您可以期待什么。

[**MoveableType**](http://www.sixapart.com/movabletype/)

通常被称为 MT 的 MoveableType 是事实上的博客平台，在很多博客上使用了很多年。它开发得很好，拥有巨大的用户基础，可以从中提取插件(增强系统功能的小脚本)和模板。

由于产品许可的变更，MoveableType 最近已经失去了大量用户到其他博客平台。2.6.x 流利用了一些旧的技术，但是作为一个可以运行多个博客、有多个作者的软件是免费的。然而，随着该软件 3.x 版本的发布，许可限制让 MT 损失了相当多的用户。制造商仍然提供免费版本，但仅限于一个作者不超过三个博客。

该产品不提供官方支持。MT 根据博客的范围和大小，提供起价 69.95 美元的支持版本。从好的方面来说，他们加入了一个允许创建动态页面的功能，这个功能可以显著提高博客的速度。对于 2.6.x 版本和 3.x 的标准安装，pages 必须在每个条目之后经历一个“重建”过程；随着博客的增长和包含更多的条目，这可能需要很长时间。

MoveableType 可以安装在你自己的托管空间上，但是我现在就在这里警告你…有时候，安装它是非常困难的。因为 MT 是用 Perl 编写的，所以检查并确保每个必要的库在您的服务器上都是可用的，并且每个文件都有基于其安装位置的正确权限是至关重要的。最近，我安装了一个 MT 驱动的博客，花了 3 个小时完成这个任务。除非你有时间和耐心，否则这个软件可能不适合你。

[**打字板**](http://www.sixapart.com/typepad/)

TypePad 本质上是 MoveableType 的托管版本。一个新用户[以每月 4.95 美元的价格注册了一个账户](http://www.sixapart.com/typepad/pricing)，并且几乎立刻就能获得一个博客。功能包括移动博客、类型列表功能(能够保存您选择的个性化列表)、照片管理、Trackback 支持、有效的开箱即用 XHTML 等等。他们确实提供 30 天的免费试用，没有设置是必要的。

[**博主**T3】](http://www.blogger.com/start)

Blogger 是一个谷歌拥有的平台，对于初学者来说非常容易快速进入。它是免费的，新用户只需要几分钟就可以注册并运行系统。还有一个额外的好处是，用户可以托管他们自己的 Blogger 驱动的博客，或者允许 Blogger 为他们托管它。

我发现了 Blogger 的几个主要问题。使用网络分析和监控方法的公司网络，如 SurfControl，喜欢将大多数博客标记为成人网站。更令人沮丧的是，为了让读者发表评论(记住:评论是博客世界的货币)，他们必须在 blogger 上注册一个帐户，或者对“匿名”发表评论感到满意。我通常建议用户避开 Blogger。

[**按**键](http://www.wordpress.org/)

WordPress 正迅速成为最受欢迎的博客软件。WordPress 是免费的，支持无限的用户，也有多用户版本，可以用来支持不同管理员的多个博客，很像 MT 的多博客设置。

WordPress 通过其论坛、[法典](http://codex.wordpress.org/)和遍布互联网的大量用户博客提供了大量的支持。WordPress 的插件和主题系统非常简单，安装可以在用户自己的网络空间中完成，只需 2 分钟。

WordPress 的缺点是(在我看来)各种 WordPress 赞助的邮件列表、论坛、bug 追踪器、CVS、SVN……还有很多。但那是 WordPress 组织的问题；不是软件本身的问题。

[文字模式 T3](http://textpattern.com/)

Textpattern 是一个成熟的平台，在可扩展性方面有一个轻微的学习曲线。原则上，许多博客作者所依赖的一些关键功能遭到了该软件开发者的反对，尤其是引用通告。它也是一个免费软件应用程序，可以在博主自己的网络空间上使用。

##### 摘要

写博客是一门学问，但学习曲线并不陡峭。可以肯定的是，在你写博客的经历中，你会学到我在博客中没有学到的东西。

虽然丰富的平台使进入博客变得容易，但它不会使你的博客成功。在你开始下载软件之前，想想你的博客将关注什么，你为谁写，你将如何满足他们的需求，以及如何定期。

博客世界是一个不断发展的行业，每天都有更多的博客出现。我希望你能成为他们中的一员！
