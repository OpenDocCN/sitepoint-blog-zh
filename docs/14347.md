# 寻找一个初学者友好的服务器

> 原文:[https://www . site point . com/in-search-of-a-beginner-friendly-server/](https://www.sitepoint.com/in-search-of-a-beginner-friendly-server/)

对于大多数人来说，学习 Java Web 开发的第一站(除了书店之外)是建立一个支持 Java 的 Web 服务器。如果你像我一样是个小气鬼，显而易见的选择是阿帕奇软件基金会的免费 Java 网络服务器 [Tomcat](http://jakarta.apache.org/tomcat/) 。

今天下午，我在 SitePoint 上更新了我的[JSP Windows 快速入门指南](https://www.sitepoint.com/java/)文章，以便使用 Tomcat 5.0。现在我不想贬低 Tomcat 背后的天才开发人员的成就，但是我确信**使用 Tomcat** 的唯一优势是它是免费的，即使在托管商业网站时也是如此。

不要误解我的意思，Tomcat 有一个围绕它的蓬勃发展的社区，最新版本在可用性方面取得了很大的进步——我对 Windows 系统托盘中的新服务器监视器印象特别深刻——但是简单的**配置任务仍然非常令人生畏**，并且文档似乎仍然是为 Java Web 开发专家编写的。

如果 Tomcat 不是新 Java Web 开发人员道路上的第一站，这一切都不会太糟糕(毕竟，物有所值，对吧？).所以从现在开始，我的任务是寻找一个初学者友好的 Java Web 应用服务器。

有人向我推荐了 Caucho 树脂，它的开源繁殖吸引了我。还有猎户座，它有一个很酷的名字，但我不知道还有什么。当然还有一些大公司， [IBM WebSphere](http://www.ibm.com/software/websphere/) 、 [BEA WebLogic](http://www.bea.com/framework.jsp?CNT=index.htm&FP=/content/products/server/) 和[Borland Enterprise Server(BES)](http://www.borland.com/bes/)，坦白地说，这些公司让我有点害怕，因为**他们有完整的工具集，而**我并不一定想使用这些工具集。

所有这些服务器都至少有一个免费的试用版可以下载，尽管其中许多服务器在一段时间后就会过期(特别是 WebSphere、BEA WebLogic 和 BES)。像 Caucho Resin 和 Orion 这样的小公司提供免费的开发许可，甚至让你免费部署非商业应用程序，所以我首先会考虑它们。
