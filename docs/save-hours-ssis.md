# 如何利用 SSI 节省时间

> 原文：<https://www.sitepoint.com/save-hours-ssis/>

在构建时事通讯的档案时，我遇到了一个大问题。我如何每周更新 70 多个页面的导航栏？一个可能的答案是使用我电脑上的搜索和替换工具，每周将所有文件重新上传到服务器。这不是一个理想的解决方案。相反，我最终使用服务器端包含(简称 SSI)来解决我的困境。这个简短的教程将向您展示如何在您的站点上利用 SSI 来节省工作时间。

虽然您不需要确切地知道服务器端 Includes 是如何工作的，但最好对当有人访问您站点上使用 SSI 的页面时会发生什么有一个基本的了解。首先，服务器读取被调用的 HTML 页面，并在其中搜索 SSI 调用(稍后会有更多相关内容)。如果它发现对另一个页面的 SSI 调用，它将打开另一个文件并对其进行解析，从而使两个文件成为一个文件。然后，解析后的文件被发送到请求页面的访问者的浏览器。因为所有的解析都发生在文件被提供之前，所以当您查看页面的源代码时，没有 SSI 调用的痕迹。

在开始试验 SSI 之前，您需要确保托管您的站点的服务器支持它。几乎所有的专业虚拟主机公司都支持 SSI 和一些免费的，其中 Hypermart.net 也允许你在你的网站上使用 SSI。

您可以在服务器上以两种不同的方式实现 SSI。默认情况下，大多数服务器被配置为只搜索和解析以. shtml 扩展名结尾的文件中的 SSI 调用。在一些服务器上，也可以调整设置，使文件以。htm 或者。html 被解析。联系你的虚拟主机公司，看看这在你的服务器上是否可行。如果不是，你必须重命名你所有的网页，使它们以。shtml 并更改所有内部链接，以便它们引用。shtml 页面，而不是以. htm 或。html 扩展。

一旦你把所有的信息都整理好了，就该把 SSI 调用放到你的页面上了。您需要使用的代码是:

<q>T2！–# include virtual = "/includes/something . htm "-></q>

它的作用是获取 something.htm 文件中的所有内容，并将其转储到放置 SSI 调用的位置。这意味着 something.htm 将没有标准的背景颜色和 HTML 定义来告诉浏览器背景和链接颜色、页面标题等。请记住，something.htm 中的所有内容都将被转储到调用它的文件中——您不需要在 something.htm 文件中指定背景/链接/文本颜色、Meta 标签和 Title 标签，只要包含 SSI 调用的文件有它们就行。

就是这样！将 something.htm 和引用它的文件上传到您的服务器，并进行测试。下一次你需要做出影响多个文件的更改时，你所要做的就是编辑 something.htm 并将其上传到你的服务器。

##### 服务器端包括常见问题解答:

3.  **Will search engines index .shtml files?** Yes, none of the search engines have problems dealing with files that have the .shtml extension
4.  ****Will search engines see the content that is in something.htm?**

    是的，搜索引擎看到的页面和你看到的完全一样。服务器在将页面发送给任何人之前都会对其进行解析，包括搜索引擎蜘蛛。** 
5.  ******Do Server Side Includes slow down the delivery of pages?**

    是的，服务器端包含在一定程度上降低了页面的服务速度，尤其是当你的服务器过载的时候。然而，在大多数情况下，页面加载的半秒钟左右的差异几乎察觉不到，这种便利是值得的。****
