# 你会转到 IE8 吗？

> 原文:[https://www.sitepoint.com/would-you-switch-to-ie8/](https://www.sitepoint.com/would-you-switch-to-ie8/)

我认识的几乎所有网络开发人员都将 Firefox 作为他们的首选浏览器(除了少数声乐歌剧迷——你知道你是谁)。像 [Firebug](http://getfirebug.com/) 、 [Greasemonkey](https://addons.mozilla.org/en-US/firefox/addon/748) 和 [Web Developer Toolbar](https://addons.mozilla.org/en-US/firefox/addon/60) 这样的插件把一个好的浏览器变成一个不可或缺的开发工具。

但是仅仅为了*浏览*呢？鉴于微软最近在 CSS 支持、开发工具、安全和隐私方面取得的进展，你会考虑切换回 IE 吗？

今天微软发布了 Internet Explorer 8 Beta 2(从 Microsoft 下载)。虽然我玩它的时间不长，但早期迹象表明，微软正在兑现他们的承诺，到测试版发布时，[将完全支持 CSS2.1](http://blogs.msdn.com/ie/archive/2008/03/06/ie8-and-css-2-1-testing.aspx) ，而[将通过 Acid2 测试](http://blogs.msdn.com/ie/archive/2007/12/19/internet-explorer-8-and-acid2-a-milestone.aspx)(不包括非标准标记)。

然而，似乎谈论最多的一个新功能是 [InPrivate 浏览模式](http://www.microsoft.com/windows/internet-explorer/beta/features/browse-privately.aspx)，它允许用户在不被发现的情况下浏览网站。启用 InPrivate 浏览模式时，不会记录用户的浏览历史记录、Internet 临时文件、表单数据、cookies、用户名和密码。

当然，这个功能已经通过火狐插件的[提供了一段时间。然而有趣的是，微软选择将它作为核心功能(也许是尝试](http://www.linux.com/feature/121738)[进入新市场？](http://www.webmonkey.com/blog/Feds_Claim_Internet_is_One_Percent_Porn))。

IE8 的 InPrivateBlocking 模式也很有趣，它可以阻止网站显示来自第三方的内容，从而防止这些第三方网站跟踪用户的浏览行为。鉴于该功能屏蔽的一些广告理论上可能包括他们自己的广告平台提供的广告，IE8 可能会对微软在利润丰厚的在线广告蛋糕中的份额产生负面影响。

你认为 IE8 进展如何？基于测试版，你会考虑从你目前的浏览器切换到 IE8 吗？请在评论中告诉我们。
