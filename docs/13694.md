# WCAG 2.0 已经崩溃，请留下你的评论！

> 原文：<https://www.sitepoint.com/wcag-20-is-broken-leave-your-comments-now/>

昨晚 Gian Sampson-Wild 在墨尔本 Web 标准小组会议上关于 Web 内容可访问性指南 2.0 版本的演讲让我明白了这个文档是 Web 的多大倒退。如果当前的工作草案取代了 W3C 批准的推荐标准 [WCAG 1.0](https://www.w3.org/TR/WCAG10/) ，那么每一个网页设计者或开发者花费时间学习、推广或构建无障碍网站的大量努力将会付之东流。

由于各种政治、内讧和企业迎合，网络社区寄予希望的关键要素——比如有效性——已经从 WCAG 2.0 中简单地省略了；打着面向未来的幌子，这份文件与技术无关，而是依赖于不断被重新定义的行话和模糊术语；它鼓励使用基线(这意味着一个网站可以简单地宣布“对不起，这个网站只允许 Flash 用户访问”，并仍然通过 WCAG 检查点-对不起，成功标准)；它小心翼翼地绕过不可避免的可用性重叠(对其不利)；它忽略了有认知学习障碍的用户所遇到的可访问性问题的关键方面。

简而言之，这份文件旨在取代现有的指导方针，并以鼓励创建可访问网站的方式推动网络向前发展；在目前的形式下，它似乎试图做一切，但。乔·克拉克已经[彻底剖析了 WCAG 2.0](http://www.alistapart.com/articles/tohellwithwcag2/) ，他提议成立一个[独立工作组](http://wcagsamurai.org/)来尝试修复文件中的一些巨大缺陷，这不仅是可以理解的——这可能是无障碍网络的唯一希望。

这是 WCAG 工作组 5 年工作的高潮，所以 W3C 渴望看到它完成。最后一次征求意见的时间是 5 月 31 日(是的，只有 5 天**了**，所以如果你强烈认为网络上的信息应该让尽可能多的人访问，不管身体残疾与否，现在就去[发表评论](https://www.w3.org/WAI/WCAG20/comments/)，让 W3C 知道这还不够好。
