# 获得对开放源码软件的支持

> 原文:[https://www.sitepoint.com/support-open-source-software/](https://www.sitepoint.com/support-open-source-software/)

对于没有经验的人来说，获得自由软件的帮助似乎是一项艰巨的任务。然而，如果你遵循正确的步骤，你可以很容易地得到关于自由软件的任何问题的及时回答。幸运的是，它不需要鸡骨头，innane 诵经，甚至不需要向黑客大师献祭。

##### 弄清楚你需要知道什么

在任何事情上获得帮助的第一步总是弄清楚你需要什么帮助。这听起来可能很傻，但是不注意这个细节可能会让你沮丧和悲伤。如果你需要用一些软件来完成一个特定的任务，确切地弄清楚**你想要完成什么。如果您遇到软件问题，请记录错误信息，并准确记下问题出在哪里。你知道的越多，获得正确帮助的机会就越大。**

##### 第一站:文档

在你迈出了寻求帮助的第一步之后，是时候迈出第一步了:文档。事实上，如果你在没有先阅读文档的情况下使用其他方法**寻求帮助，你通常会被忽略或者被告知 [RTFM](http://www.tuxedo.org/~esr/jargon/html/entry/RTFM.html) 。如果你用的是 Linux 系统，输入‘man foo’试试这个软件的**手册页**，这里‘foo’是你需要帮助的程序的名字。如果不起作用，请键入“info foo”查看软件是否有 Info 文档。您可能还想看看文档是否存在于/usr/doc/foo/中。如果您运行的是 Windows，请阅读帮助文件或检查程序的安装目录中名为“doc”的子目录。**

如果在您的机器上找不到任何文档，请查看该程序的网站。谷歌搜索引擎的 Linux 专用搜索页面是寻找自由软件网站的一个很好的起点。你可能还想在 [Freshmeat](http://www.freshmeat.net) 上搜索软件，或者查看一下 [Linux 文档项目](http://www.linuxdoc.org)，它存储了数百个关于自由软件的操作指南、指南、书籍和文档集。

尽管自由软件的文档对新手和专家来说似乎晦涩难懂，但它通常提供了最好的信息。大多数自由软件文档都非常详尽，批评者谴责为“难以理解”诚然，自由软件文档中的例子很少，但这是有充分理由的。大多数这种类型的软件包含了太多的选项，对于作者来说，除了一个软件的最常见的用法之外，他们无法给出一步一步的指导。

那么你应该如何阅读自由软件文档呢？不要从头到尾，因为这样的任务会让最勤奋的读者感到疲劳和沮丧。相反，要像对待参考书一样对待文档。浏览目录，找到您需要的主题区域。然后在看起来有希望的页面上搜索关键词。如果您找不到答案，在尝试下一级帮助之前，请尝试文档的另一部分或上面列出的其他来源之一。

##### 第二站:邮件列表和新闻组

如果你找不到任何文档来回答你的问题，你现在应该通过网络提问。大多数软件项目都会有一个邮件列表。经常使用的软件项目有时会有新闻组。在提出问题之前，请在列表存档中搜索答案；当人们被一遍又一遍地问同样的问题时，他们会感到恼火。

向新闻组或邮件列表发帖时要有礼貌。尽可能简单明了地问你的问题，并尽可能包括更多的细节。外面有人可能知道答案，很快就会回复。

##### 第三站:网络接力聊天

有时，没有时间使用邮件列表。要获得更快的响应，请尝试互联网中继聊天(IRC)。我最喜欢的 IRC 客户端是 [BitchX](http://www.bitchx.org) 。它可以在 Windows 和 Linux 上运行，但文本模式界面会让一些人感到困惑。如果你想要一个图形客户端，试试 [mIRC](http://www.mirc.com) (Windows)或者 [X-Chat](http://www.xchat.org) (Linux)。如果不想下载客户端，可以使用 Linux.com 基于浏览器的 java irc 客户端访问 Openprojects IRC 网络。可以在 [Linux.com 的网站](http://www.linux.com/live/chat.phtml)找到。所有这些包括的文档应该足以让你开始使用 IRC。

当你通过 IRC 寻求帮助时，你应该先试试 irc.openprojects.net 的 Openprojects 网络。Linux 帮助可以在#linuxhelp 频道获得。#linpeople 频道更轻松，但你可能不会那么快得到答案。很多软件项目在 openprojects 上都有 irc 频道:PHP(# PHP)；perl(# perl)；C/c++(# C)；emacs (#emacs)，甚至开源网页设计(#oswd)。其他值得一试的好服务器是 irc.xchat.org 和 irc.gimp.org。

当你加入一个频道后，向每个人问好，然后什么都不做，等一会儿。打断别人的谈话是不礼貌的。我个人会等到当前对话结束后再问。然而，人们可能会同时进行三次以上的谈话，所以不要等太久才开口。当你说话时，不要只说“我需要帮助”或“foo 不起作用”试着在你的第一句话中陈述你的整个问题，或者至少给出足够的信息让人们感兴趣(如果是一个很长的问题)。对帮助你的人要有耐心，他们也会对你有耐心。你也应该避免将大块的文本粘贴到通道中。大多数频道运营商机器人会因此把你踢出频道。

##### 第四站:真实的人际互动

如果你在文档中找不到你的答案，并且不能通过标准的方法让人们帮助你，你还有两个选择。第一种是从您当地的 Linux 用户组(LUG)获得帮助。LUGs 由感兴趣的爱好者、专业人士和天才组成，他们聚在一起互相帮助，讨论 Linux，学习技术。你可以通过查看任何地方的 Linux 用户群或者 T2 的 Linux 柜台来找到你身边的人。许多大学和学院也有 lug，所以你可以打电话给你当地的机构。几乎所有的 lug 都有邮件列表，如果你在你的区域找到一个，首先在邮件列表上提出你的问题。在 lug 的人更有可能花很多时间来帮助你；他们组成了一个对初学者友好的社区。你甚至可以问他们除了 Linux 之外的其他操作系统，因为 lug 中的许多人也使用其他操作系统。如果他们不能通过电子邮件帮助你，他们甚至会在下一次会议上亲自帮助你。最精彩的部分？这不会花你一分钱。

##### 第四站半:商业支持

有时候，时间紧迫，金钱唾手可得。在这种情况下，你总能得到商业上的支持。像 [Redhat](http://www.redhat.com/apps/support/programs.html) 、 [VA Linux](http://www.valinux.com) 和 [Linuxcare](http://www.linuxcare.com/support/index.epl) 这样的公司，以及许多主要的 Linux 发行公司，为自由软件提供高质量的收费支持。

不管你的问题是什么，不管你的问题是什么，总会有答案的。按照确定问题、检查文档和在正确的时间、正确的地点提问的步骤，你会找到正确的答案。所以把鸡骨头放回抽屉里。事实证明，为自由软件寻求帮助并不是那么难。
