# Atlas 在跨浏览器支持方面令人失望

> 原文：<https://www.sitepoint.com/atlas-disappoints-in-cross-browser-support/>

(via [digg](http://digg.com/programming/Microsoft_Atlas_Rated_D-_in_Cross-Browser_Test) )在 [Leland Scott](http://www.musingsfrommars.org/) 的博客上开始了一场[有趣的讨论](http://www.musingsfrommars.org/2006/04/nothing-to-cheer-here-microsoft-atlas.html)，此前他在各种(非 IE)浏览器上测试了微软的 [Atlas](http://atlas.asp.net/) AJAX 框架。

Scott 测试的要点是 Atlas 代码不能在所有浏览器上运行。在 Firefox 中测试时，大多数 UI 控件都被选中，但在 Opera 和 Safari 中，大多数控件要么失败，要么不能按预期工作。

这显然令人失望，考虑到 Atlas 已经被大肆宣传，并考虑到已经有如此多的[其他](http://developer.yahoo.com/yui/) [伟大的](http://prototype.conio.net/) [开放的](http://script.aculo.us/) [源代码](http://dojotoolkit.org/) [工具包](http://openrico.org/)可以在所有现代浏览器中工作。显然，在急于推出 Atlas 的过程中，“跨浏览器”被 Atlas 开发者解释为“让它在 Firefox 中工作”，而“社区技术预览版”被用作其他人失败的借口。

不幸的是，考虑到明显的反微软偏见，很难认真对待斯科特的大量报道。尽管如此，还是截图说明了问题，随着 ASP.NET 的人气飙升和 Atlas 背后的营销活动，该工具包肯定会被广泛使用。因此，除非在 Atlas 1.0 发布之前投入巨大努力，让它不仅仅在 Firefox 和 IE 上运行，否则 Web 2.0 将变得支离破碎。

有人说浏览器大战了吗？
