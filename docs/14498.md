# 蓝龙的想法…

> 原文:[https://www.sitepoint.com/bluedragon-thoughts/](https://www.sitepoint.com/bluedragon-thoughts/)

所以蓝龙公司的人看到了我的文章“[为 ColdFusion](https://www.sitepoint.com/making-case-coldfusion/) 辩护”，并想带我去吃午饭，“感谢我对蓝龙公司的赞美”。

所以我告诉他们如何到达我的办公室，然后我们去了当地的三明治店吃了点午餐。午餐总体来说不错，对话也不错。我们当然谈到了蓝龙和 ColdFusion。

我对蓝龙公司(新亚特兰大)的印象变得更加“深刻”。Brian(销售)和 Charlie(首席技术官)都是很棒的人，他们有很多事情要告诉我。当然，一顿午餐你不可能学到所有的东西，所以我希望在接下来的几周/几个月里会有更多的对话。

以下是我所学到的总结:

*   新亚特兰大非常注重社区——社区和他们的客户推动开发列车，并决定增加和不增加什么功能
*   蓝龙服务器(免费)与 JX 的(付费)版本并没有太大的不同
*   他们非常愿意与公司合作——他们知道设计店没有时间整合 BleuDragon，所以他们会提供帮助！
*   他们非常愿意与开发人员合作——如果你与他们交谈，你会惊讶于他们会不顾一切地帮助和支持你。
*   他们喜欢谈论他们的产品，通常会带午餐——去联系他们，让他们参观你的设计店！
*   他们不认为自己是 Macromedia 的直接竞争对手，而是一种替代选择——我们都知道他们是竞争对手，但竞争是好事。

所以基于这顿午餐，我决定冒险在一台旧的 XP 机器上建立我自己的蓝龙开发环境。安装非常顺利。如果你要这样做，请记住，在 windows 系统上，你必须首先通过 ODBC 建立数据库连接，因为 JDBC 驱动程序只随付费服务器(JX BD 服务器)一起提供。

那么我有什么抱怨呢？实际上只有几个。[list = 1]
[*]最初我的网站没有配置。cfm，。cfml，。由屋宇署提供的 cfc 档案。如果有一个工具能自动为我做这件事，而不是由我手动去做，那就好了。
[*]BD 管理界面有点难读。我到处都支持 BD blue，但菜单系统和许多其他领域对比不佳
[*]我不太喜欢建立 ODBC 连接，但我完全理解这是一个免费产品，所以我必须做一些工作。
[/list]

暂时就这样了。希望未来的任何问题都是小问题，只是因为我的暴躁！

哦，对了，还有一件事，如果 BD 的人正在读这篇文章，谢谢你的 T 恤衫(或者我办公室的同事称之为“药丸”)，我们什么时候能收到免费的 JX 牌照？
