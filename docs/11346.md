## 全球统计

有几个在线服务提供一般的网站统计报告。StatCounter 是其中较为知名的一个:它分析来自全球 300 万个网站的数据，以收集有意义的统计数据。这些信息是有用的，但是在使用这些数据做出商业决策时应该小心。请记住:

没有一个通用的网站。

根据内容和目标受众的不同，所有网站都有不同的使用和人口统计数据。我们可以得出一些明显的结论:

*   Mozilla.org 的火狐用户数量可能高于正常水平。
*   Apple.com 的 Mac 和 Safari 使用率将高于平均水平。
*   Opera 可能会有比大多数国家更多的 Opera 用户。
*   谷歌 Chrome 扩展页面将由谷歌 Chrome 用户主导。
*   一个中文网站将会有更多的亚洲用户。

例如，您可能会花费大量的时间和精力来支持 IE6，因为它拥有 14%的全球市场份额。然而，如果你运行一个以技术为基础的网站，主要是北美用户，IE6 的使用率可能低于 5%。

## 良好的统计数据

从需要用户操作(表单发布)并在数据库或文件中记录数据的 web 应用程序中生成有意义的统计数据是可能的。

例如，像 Flickr 和 YouTube 这样的网站会知道有多少用户注册，以及这些人上传了多少照片/视频。然而，大多数访问者是可能不进行数据更改的查看者；他们的统计数据将会受到针对[客户端](https://www.sitepoint.com/website-statistics-reports-wrong-2/)和[服务器端](https://www.sitepoint.com/website-statistics-reports-wrong-1/)数据整理所讨论的相同缺点的影响。

## 网站统计有用吗？

网站统计报告已经发展。它们最初被 web 开发人员和管理员用来评估服务器负载和性能瓶颈。它们现在可能更漂亮、更容易理解，但这并不意味着它们是正确的。它们对于评估总体趋势肯定是有用的，但是所有的统计数据都是没有意义的，除非你明白数据是如何整理的，以及所使用的分析方法的注意事项。

例如，您可能发现您的网站 Opera 用户数量很少，并决定放弃在该浏览器中进行测试。但是如果你当前的站点不支持 Opera 呢？较低的数字可能是因为这些用户无法查看您的主页。

网站报告可能看起来令人印象深刻，但很容易误解潜在的数据。我建议从两个或更多不同的来源获得报告，并警惕你的营销决策仅仅基于漂亮的折线图或条形图！
