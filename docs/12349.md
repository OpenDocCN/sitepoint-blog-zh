# PHP 框架计划社会责任编码竞赛

> 原文:[https://www . site point . com/PHP-framework-plans-social-responsible-coding-contest/](https://www.sitepoint.com/php-framework-plans-socially-responsible-coding-contest/)

根据我的个人经验，编程竞赛是展示 web 应用程序框架的最佳方式之一。在 Ruby on Rails 框架的早期，它赞助了“Rails Day”，这是一个 48 小时的编码竞赛，真正推动了我所拥有的 Rails 社区站点的发展。那次比赛给这个框架带来了如此多的关注，以至于很多关注都转移到了比赛的赞助商身上。铁路日已经让位给了一年一度的“铁路隆隆”，我们[在十月](https://www.sitepoint.com/rails-rumble-245000-lines-of-code-in-48-hours/)报道过。

超过 500 人参加了今年的 Rails Rumble 比赛，比赛被许多大型博客和主流计算机出版物报道。因此，难怪 PHP web 应用程序框架 [Symfony](http://www.symfony-project.org/) 想要模仿这种竞赛模式来传播他们的项目。今天，Symfony 的社区经理 Kris Wallsmith [在该项目的博客](http://www.symfony-project.org/blog/2009/01/23/who-wants-a-free-web-application)上宣布，他们打算在几个月后举办类似的 48 小时编程比赛。然而，Symfony 版本的铁路隆隆声有一个转折。

这些编程竞赛的一大好处是，它们展示了开发人员使用框架从构思到部署的速度有多快。例如，当 Rails Rumble 完成时，有超过 100 个完整的实际应用程序可以用来将 Rails 框架提升为一个良好的 web 应用程序开发平台。

不幸的是，尽管一些团队继续开发他们的应用程序，但是绝大多数开发出来的应用程序都被搁置起来，从未真正使用过。作为一种学习和推广工具，像隆隆声这样的比赛是很棒的。但是它们也代表了浪费在项目上的大量工时，这些工时很可能不会真正帮助任何真正的用户。

Wallsmith 不希望 Symfony 的竞争出现这种情况。因此，在两天疯狂的比赛中，参与者不再只是创建随机的应用程序，而是为特定的原因创建应用程序。“我们在比赛过程中创造的应用程序将被赠送给世界各地不同的有社会意识的组织，”Wallsmith 说。“一旦乐趣结束，我们的工作将继续下去，并有望带来很多好处。”

Symfony 已经向赞助商(奖品和基础设施)和“有社会意识的组织”发出呼吁，他们需要一个交互式的网络应用程序。开发团队注册将很快开放，比赛计划在几个月后举行。
