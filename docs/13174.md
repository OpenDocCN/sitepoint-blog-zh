# 今年在 Rails 中尝试的 6 件事

> 原文:[https://www . site point . com/6-this-to-try-in-rails-this-this-year/](https://www.sitepoint.com/6-things-to-try-in-rails-this-year/)

似乎新年前几周的博客帖子(顺便说一句，新年快乐)遵循了“今年要做的 x 件事”的迷因，作为对新年决心的虚拟敬意。我从来都不是一个逆势而为的人，我已经准备了一个简短的清单，列出了你应该在 Ruby 和 Rails 中尝试的事情——我希望在接下来的几周内更详细地讨论每个主题。

1.  安装 Rails:这是针对那些还没有尝试过 Ruby on Rails 的人的。投入使用——试一试吧，这里有大量的资源，而且不管您使用什么平台，安装起来都相当容易
2.  升级到 Rails 2.0:我在以前的几篇文章中已经介绍了 Rails 2.0 的新特性，如果你按照步骤操作并修改了任何反对通知，升级并不困难。
3.  写一个插件:插件允许你重用常见的模式，而不必求助于可怕的剪切粘贴键。Rails 有一个内置的插件生成器，可以给你提供框架代码，你需要做的就是混合正确的模块——哦，然后写代码…
4.  尝试 RESTful 路线:RESTful Rails 已经存在一段时间了，但是网上的许多教程仍然是基于 CRUD 的，所以如果你是 Rails 初学者，你可能还没有尝试过。
5.  使用 Ruby 作为您的脚本语言:因为 Rails 出色地完成了繁重的工作，所以您很可能从来没有手动连接到数据库或读取目录的内容，或者用 Ruby 执行其他普通的任务。如果你必须编写脚本，而不是 PERL 或 Python，使用 Ruby——它会帮助你更好地理解语言的细微差别。
6.  用 Ruby 的方式重构你的代码:如果你来自其他类似 C 的语言(比如 PHP 或 Java ),你会习惯这种编码风格的习惯用法。尝试重构你的代码，使用块而不是 for 循环，或者使用内联 if 语句——在 Ruby 中，你可以在一行中压缩很多内容，看看你能做到什么程度。同样，这是学习语言的一个很好的练习。

来吧，尝试一下——新年是尝试新事物的好时机！
