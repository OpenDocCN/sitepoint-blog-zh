# IE7 的更多 CSS 修复

> 原文:[https://www.sitepoint.com/more-css-fixes-for-ie7/](https://www.sitepoint.com/more-css-fixes-for-ie7/)

微软已经宣布了 IE7 的几个修复程序，这些程序会让 CSS 专家们在过道里跳舞。

Internet Explorer 中的一个长期存在的错误创建了一个“根节点”,它位于文档树中 html 元素的上方，允许以“* html”开头的选择器仅用于该浏览器。相反，因为 IE 目前不支持子选择器(>)，所以以“html > body”开头的规则可以用于除了 IE 之外的所有浏览器*。*

在 IE7 中，**根节点选择器消失了**并且**子选择器被完全支持**，使得浏览器能够使用与当前其他浏览器相同的 CSS 规则。假设微软修复了它的大多数 CSS 错误(对此我持谨慎乐观的态度)，这两个修复将使设计者在充分利用 IE7 的同时，很容易保持与 IE6 和更早版本的向后兼容性。

一个更微妙的修正是**对多类选择器**的完全支持，这将影响更少的设计者。在一个 HTML 文档中，一个元素可以被分配多个类，通过空格分隔它们的名字(例如< div class= "重要提示">)。根据 CSS 标准，如果您想要匹配具有特定类组合的元素(例如，div 同时具有' important '和' note '类)，您可以在 CSS 选择器中用点分隔它们(例如，div.important.note)。IE6 目前在这样的选择器中忽略除最后一个类名之外的所有类名(例如，div.important.note 将匹配任何带有' note '类的 div)，可能将该规则应用于不应该应用它的元素。IE7 会解决这个问题。

所有这些修复只会影响 IE7 严格模式。当运行在古怪模式下时，浏览器将继续当前的 IE6 行为。但是还有一个在严格模式和古怪模式下都会影响浏览器的修复:**正确解析伪元素选择器**。IE6 不支持伪元素选择器，比如:first-letter，除非它们是 CSS 选择器中的最后一个元素，后面跟一个空白字符。如果伪元素后跟一个类或伪类(例如，div:first-letter:hover)，或者如果它后面没有空白(例如，div:first-letter { float:left；})，则忽略整个规则。通过纠正这些解析器错误，IE7 将确保伪元素规则更加一致地工作，而不需要太多的改动。
