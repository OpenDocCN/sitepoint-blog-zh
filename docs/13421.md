# DevConnections 第 1 天总结:微软会议上的微软日

> 原文:[https://www . site point . com/dev connections-day-1-wrap-Microsoft-day-at-a-Microsoft-conference/](https://www.sitepoint.com/devconnections-day-1-wrap-microsoft-day-at-a-microsoft-conference/)

[参见第 0 天保险](https://www.sitepoint.com/devconnections-spring-2007-day-0-scottgu-ajax-entity-frameworks/)。

DevConnections 年春季的第一天以所谓的“微软日”开始基本上，这意味着微软员工整天都在场，并以非常非正式的“微软不插电”会议结束。

我以 SharePoint 上的一个会话开始了这一天。但是 SharePoint 很无聊，即使乔·奥尔森不无聊，所以我就不说细节了。接下来，我点击了 [WPF/E](https://www.microsoft.com/silverlight/) 上[斯科特·格思里](http://weblogs.asp.net/scottgu)的扩展演示。对于外行人来说，WPF/E 是微软当时的 Flash 黑仔。它的潜力比我之前认为的要大得多。它具有 HTML 的一些最好的特性——例如基于文本，易于使用 DOM 进行机器生成和操作——同时具有 Flash 的所有功能。亮点包括:

*   表情交融看起来非常非常好。感觉有点像 Flash Studio 遇到了 Visual Studio，但是是以一种好的方式。因为它提供了非常有用的图形编辑工具和良好的编码环境。Scott 非常喜欢他们设计的 Visual Studio 和 Expression line 能够很好地协同工作。
*   多媒体支持包括 MP3，WMA 和 WMV。非常好的消息是，浏览器插件包括编解码器等，所以没有必要单独下载。此外，请记住，除兼容性之外，WMV 从任何其他技术角度都将 FLV 打得落花流水。
*   用于加载 WPF/E 对象并与之交互的 JavaScript 库非常非常巧妙。它有效地公开了一个可以操作的 DOM，包括动态对象创建——或者通过 AJAX 包含。
*   出于兼容性目的，渲染引擎是基于软件的。事实上，跨平台能力是一个主要的焦点—下至包括移动设备。会议中最好的笑话:Scott 注意到网络上的移动客户端比网络上的 Linux 客户端多。
*   。NET 可编程性支持还没有出现，但是请继续关注 MIX 07 上关于这方面的重大声明。

结束了我在 ASP.NET 的一天，我还参观了杰夫·金非常有趣的视觉工作室 Orcas 演示。我必须说我真的等不及它上市了——它实在是太酷了。亮点:

*   Orcas 不会绑定到任何特定版本的。网络堆栈。好吧，只要你是 2.0 或更好。
*   Orcas 中的 CSS 支持是非常非常真实的。它看起来有点像集成到 Visual Studio 设计器中的类固醇 TopStyle Pro。他们仍然没有一个工具集，可以智能地分配 CSS 属性和选择器，以及一个知识渊博的 CSS 爱好者。但是他们至少设置了这个工具，这样你就不需要和它战斗了，而且你可以在集成预览中立即看到结果。最精彩的部分可能是能够直观地突出显示给定的 CSS 选择器应用于哪些实体。
*   JavaScript 智能感知非常非常真实。我怀疑它的工作原理与某些人所需的管道相同。NET 3.5 好东西，比如 lambda 类。但是 JavaScript 是一等公民。包括工作调试和断点。
*   link 2 Sql——一个用于 Sql Server 数据库的伪 ORM 看起来很巧妙。或者至少它演示得很好。无论如何，很明显 MS 已经开始优先考虑 ORM，而不是 WHYSIWIG 数据映射。
*   杰夫展示了我现在称之为有史以来最酷的 ASP.NET 控件:ListView。它基本上是一个 GridView，没有 HTML 网格。这是我梦寐以求的服务器控件。他结合一个非常实用的 LinqDataSource 控件演示了该控件。

来自[ADO.NET 团队](http://blogs.msdn.com/adonet/)的 Brian Dawson 做了两个关于实体框架的演讲，这给人留下了非常非常深刻的印象，并且非常非常有雄心。如果脱得合适的话，会非常非常酷。微软的 ORM 方法很有趣。他们为低端场景创建了 LINK2SQL 自动生成，仅 Sql Server 选项。和实体框架，一个更 N 层的解决方案。他的演示亮点:

*   实体框架将关注点分开，这样它可以使用 3 个不同的映射文件有效地与应用层和数据层对话。这使得开发人员在不把数据库完整性导向的 DBA 赶下墙的同时，也不会对询问持久性无知感到高兴。
*   从数据端来看，他们正在与其他数据供应商合作，以确保他们的数据库有可用的提供商。
*   实体框架被设计用来处理继承、多态、基于接口的设计和所有其他在几周前的 MVP 峰会上给微软留下深刻印象的 OO 疯狂。
*   它使用大量的 XML 来到达它要去的地方。

在短暂参观了蓝色大房间后，我设法回到了会议中心参加微软不插电活动。这很有娱乐性——任何由“两个穿着钓鱼衫的胖子”主持的活动，以他们向观众投掷填充卡通图标开始，都不会太糟糕。首先，Scott Guthrie 做了一个简短的 IIS 7 演示。这是一个非常标准的 IIS 7 演示，但他确实抛出了两个重磅炸弹:

*   预计很快就会看到 IIS 7 的上线许可。
*   MySpace 现在运行在 IIS 7 上，显然很喜欢它。

明天再来看看，如果我继续避免阳光直射，我会不会真的让我的头爆炸。
