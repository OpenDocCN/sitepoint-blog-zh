# 普适软件加入数据库竞赛

> 原文：<https://www.sitepoint.com/pervasive-software-joins-database-race/>

普适软件成为最新加入这个迅速发展的数据库供应商领域的公司，他们希望通过商业解决方案的方式利用开源软件。他们加入了 Sybase、MySQL 和其他公司的行列，寻找更大的中型和高端企业市场。

[Pervasive](http://www.pervasive.com/) 在某些圈子里被认为是数据库基础设施管理方面经验丰富的老手，现在它正寻求利用 PostgreSQL 的良好声誉来推出一种新的面向服务的解决方案。

乍一看，他们似乎正在攻击 MySQL——我个人仍然认为这是行动的一部分，在 MySQL 备受期待的版本 5 测试版发布之前取得进展。PostgreSQL 当然喜欢触发器、存储过程和 MySQL 用户热切期待的其他特性。

然而，普及型公司的产品总监 Lance Obermeyer 对 MySQL 的跳跃式发展轻描淡写。

“如果一个用户设计了他或她的应用程序，使其能够在 MySQL 的约束下工作，那么他们应该留在那里。肯定会有一些超越 MySQL 的用户，我们欢迎这些用户，”Obermeyer 告诉 SitePoint。“但是，我们的主要目标是生产业务应用程序的用户，他们需要主流功能和坚如磐石的数据完整性。这些将是从专有数据库或全新的应用程序中进行的迁移，而这些应用程序在 MySQL 上根本无法运行。”

我确信读者可能会对 Obermeyer 对 MySQL 的大胆攻击有一些评论。然而，这一产品确实吸引了那些跟踪大型开发项目且预算稳定的网络专家，以及那些希望增强内部数据库驱动的应用程序的中小型公司。

Pervasive 目前没有公开跟踪其市场份额，这表明“很难估计准确的市场份额数字，因为 PostgreSQL 可以通过如此多的渠道获得，其中大多数都没有被跟踪。”

人们可以通过 [Sourceforge](http://sourceforge.net/projects/pgsql/) 以及该项目的[官方主页](http://www.postgresql.org/about/newsarchive)来跟上 PostgreSQL 在市场上的增长。

当被要求分享一下在 PostgreSQL 之上的定制化时，Obermeyer 展示了公司的战略。

“我们围绕 PostgreSQL 所做的努力有两个方面，一是提高可用性，以便除了早期采用者和技术爱好者之外的其他人也可以部署它，二是扩展性能和功能范围。”

他说，该公司的最初版本侧重于可用性，收集和集成多个组件，如管理控制台和 JDBC 这样的连接驱动程序，他们已经创建了一个简化的配置工具。

“我们还将它们打包在一个通用的安装程序中，简化了安装过程。这些早期的改进已经改善了用户的第一印象和成功的可能性。

该公司还提供一系列咨询服务，包括支持、迁移、集成以及定制开发。
