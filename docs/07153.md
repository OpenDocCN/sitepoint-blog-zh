# 学习编程语言的三个误区

> 原文:[https://www . site point . com/the-3-myths-of-learning-programming-languages/](https://www.sitepoint.com/the-3-myths-of-learning-programming-languages/)

你已经开始学习你的第一门编程语言了吗？你为什么要推迟呢？如果你认为这太难了，比如学习一门真正的口语，那你就错了。

事实上，你已经成为我喜欢称之为*大编程语言谬误*—**错误地认为编程语言类似于真实语言的受害者。**

如果你是这个谬论的受害者，你可能会认为编程语言…

*   是“计算机的语言”
*   是外来的，很难读懂，而且
*   需要多年的学习。

让我告诉你，所有这三个信念都是神话。

继续阅读，看我如何反驳这三个神话，并发现学习你的第一门编程语言比你想象的要容易。

## 误解 1:编程语言是“计算机的语言”

快速的谷歌搜索发现人们在说这样的话…

> [如何熟练掌握 Python？](http://stackoverflow.com/questions/1507041/how-do-i-get-fluent-in-python)

这些人显然认为学习像 Python 这样的语言意味着…

*   学习“对着电脑说话”，
*   学习“像计算机一样思考”，以及
*   变得“流利”于一种对他们来说不是“母语”的语言。

他们错了。

事实上，绝大多数时候，编程语言是专门为你我这样的人设计的。

**原因如下……**

在计算机科学的背景下，有*低级*编程语言和*高级*编程语言。

像汇编语言这样的低级语言实际上是直接与计算机对话的——执行一长串处理器操作，一次一个字节。

但是汇编语言只是当今世界上使用最多的[第 19 位](http://www.tiobe.com/index.php/content/paperinfo/tpci/index.html)编程语言。绝大多数现代程序员只用 Python 这样的高级语言编写。

下面是[柯林斯英语词典](http://www.collinsdictionary.com/dictionary/english/high-level-language)对‘高级语言’的定义:

> 一种类似自然语言或数学符号的计算机编程语言…

高级语言类似于自然语言，并使用数学和逻辑等概念，因为它们被设计为易于人类理解，而不是计算机理解。

学习这些高级语言中的一种并不意味着学习“对着电脑说话”。它们存在的全部原因是为了让我们不必如此。

那么，如果高级语言是为了让我们容易理解而设计的，那么它实际上是什么样子的呢？继续阅读查看示例…

## 误解 2:编程语言是外来的，很难读懂

如果你试图阅读一些用你不熟悉的语言写的文章，你可能会面临…

*   一个不同于你所熟悉的字母表，
*   不熟悉的语法规则和句法，
*   最令人沮丧的是，一组你从未见过的全新词汇。

但是如果你要阅读一些高级编程语言代码，你将会面对这些东西中的 *none* 。

**让我证明给你看……**

看看这段用 SQL 编程语言编写的代码:

```
INSERT INTO Table
VALUES ('1', 'SQL', 'Programming language')
```

为了这个例子的目的，让我们假设你的母语是英语，我们将把它与 SQL 进行比较。

首先，字母表肯定不是外来的。所有使用的字符都可以在标准的罗马字母键盘上找到。

其次，看语法规则和句法。虽然有一点偏差，“插入到表中”是熟悉和可读的。作为一个说英语的人，你不难理解这个代码的意思。

最后，看单词。每一个单词在英语中都已经存在。没有外来词，没有难读的发音，没有绞尽脑汁想出每个单词的意思。

诚然，编程语言确实发明了缩写——比如 *regex* (正则表达式)和 *varchar* (可变字符)，这只是其中的几个例子。

但归根结底，一门编程语言的全部“词汇”完全来自现实世界的语言——无论是英语还是其他任何语言。

这就是为什么编程语言*根本不是语言。事实上，你可以说编程语言更像方言而不是语言。*

现在学习你的第一门编程语言似乎不像学习一门真实世界的语言那么难了。正如您接下来将看到的，这也不需要太长时间…

## 误解 3:编程语言需要数年才能学会

要学习一门外语，你需要学习…

*   词汇(这样你就有词语来表达你的想法)，
*   语法(能够将它们串成句子)，以及
*   读、听、读、说的能力。

那太多了。

因此，对于一个普通的以英语为母语的人来说，学习一门新的外语需要从 23 周至 1.7 年甚至更长的时间也就不足为奇了。

学习一门编程语言可能要花同样长的时间，如果不是更长的话——*如果它们真的是为计算机设计的，而且是外国的，难以阅读。*

**但我们已经知道这不是真的……**

学习编程语言的命令和功能(相当于学习真实世界语言的词汇)变得容易得多，因为编程语言使用与现有语言相同的单词(几乎总是英语)。

假设你会说这种语言，那就省去了大量的学习时间。

同样，学习阅读这种语言也没什么大不了的。所有的单词对你来说都是熟悉的，正如我们所看到的，破译编程语言代码并不困难。

学习编程语言的语法(相当于学习现实世界语言的语法)并不那么容易——但请记住，高级编程语言是为了易于理解而设计的，因此也不难。

最后，学习如何听、发音和说话的挑战实际上是不存在的。我不知道你，但我从来没有听到有人用编程语言大声说出来。

如你所见，编程语言中要学的东西远不如现实世界中的语言多。要学的东西越少，意味着需要学习的时间越少。

事实上，网上有这样的故事，有人学会了一门编程语言*和*，在[仅仅几个月](http://www.codecademy.com/stories/)，甚至在 [12 周](http://www.jamesfend.com/learned-ruby-rails-12-weeks-launch-freelancify)就取得了成果。

几年？我不这么认为。

## 底线是…

编程语言易读、易学，而且是为像你这样的人而设计的。
