# 面向网络开发者的搜索引擎优化

> 原文：<https://www.sitepoint.com/optimisation-web-developers/>

如今，如果想提高搜索引擎的流量，大多数网站所有者都会选择搜索引擎优化公司。然而，很多时候，搜索引擎成功的机会早在那之前就被决定了。

搜索引擎兼容性正在成为标准网站规范的重要补充。网站推出后，网站所有者意识到他们的内容永远不会被当前形式的搜索引擎索引已经太晚了。通常，由 Web 开发人员或设计人员来采取主动，引导客户开发一个优化的网站(或者至少是可优化的！)对于搜索引擎来说。

让我们来看看影响当今网站优化的最常见的设计和开发考虑因素。

##### 导航和可访问性

实现搜索引擎可见性的第一步是确保搜索引擎蜘蛛能够访问网站的所有页面。如果你的网页对蜘蛛来说是不可访问的，再多的内容优化、网站受欢迎程度或提交给搜索引擎都不会提高它们的性能——它们只是不会被添加到索引中。这意味着您需要为以下导航样式提供替代方案，仅举几个例子:

*   闪光
*   基于表单的导航
*   java 小程序
*   一些 Javascript/DHTML 菜单

“为什么？”我听到你哭泣。不幸的是，搜索引擎蜘蛛并没有为你的普通网络浏览器配备所有可用的插件。这意味着替代导航是必需的，除非你想让搜索引擎相信你的网站包含一页内容。

克服 99%的可访问性问题的最佳建议是在你的网站上维护一个 HTML 文本链接网络，并包含一个纯文本的网站地图，以确保搜索引擎可以找到并索引你所有的个人页面。

##### 网站架构和内容交付技术

搜索引擎的另一个症结是用于传递内容的技术。假设解决了第一个考虑因素，并且搜索引擎可以找到站点提供的所有 URL，那么你使用的交付技术或 Web 应用程序会对搜索引擎是否能够读取该 URL 上的内容产生巨大的影响。

在这里，我们再次发现我们的老朋友:Java、Javascript、Flash 和图像是罪魁祸首。如果你的网站设计大量使用了这些技术，你的客户需要意识到搜索引擎将很难确定内容，这会导致搜索引擎渗透率很低。在这一阶段，你需要决定这些技术是否对网站的成功至关重要，或者 HTML 是否也能做好这项工作，以及潜在的搜索引擎排名的额外好处。

框架也可能给搜索引擎带来困难的挑战，因为它们需要不同于普通网页的解释。它们还会让用户很难标记和深入链接到你的网站(这会严重影响你未来的在线营销策略)。随着 PHP、ASP、Cold Fusion 以及其他脚本语言和 Web 应用程序的使用，动态 URL 也变得非常冗长。尽管搜索引擎对动态 URL 的支持在不断增加，但强烈建议将 URL 上存储的变量数量保持在一两个，就像从 URL 中删除会话 id 一样。

注意:SitePoint 上有几篇关于 URL 重写的文章，它们不仅提供了创建搜索引擎友好的 URL 的技巧，同时还解决了一些常见的安全问题。我个人最喜欢的是 Tamas Turcsanyi 的[mod _ rewrite:URL 重写初学者指南](https://www.sitepoint.com/blog/)。

##### 内容优化

现在，你的网页是可访问的，搜索引擎可以理解你的内容，你的客户的网站将与大多数竞争对手处于同一水平，搜索引擎成功的潜力已经大大增加。

但是，在你掸掉手上的灰尘，启动 Quake 服务器来祝贺自己出色完成工作之前，请考虑一下网站经理每天都要进行的内容优化，如果他们真的要在搜索引擎大战中给网站一些牙齿的话。

您可以在内容管理系统中使用的一个很好的功能是允许您的客户轻松访问每个页面的标题、标题、alt 和 Meta 标签。这将使内容经理能够在灵活性和搜索引擎友好性方面将网站提升到一个新的水平。

有了这个功能，内容管理器可以将流行的搜索词直接包含到搜索引擎用来评估页面相关性的一些最重要的 HTML 元素中。

##### 结论

通过考虑本文中提出的问题，

*   搜索引擎可以访问我所有的页面吗？
*   我的内容被搜索引擎理解了吗？
*   最终用户可以编辑重要的页面元素吗？

你可以相应地调整你的设计方案，并且放心，你已经再一次为你的客户提供了在至关重要的搜索引擎游戏中实现巨大渗透的每一个可能的机会。
