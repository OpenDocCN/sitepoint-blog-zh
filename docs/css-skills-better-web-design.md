# CSS 技能=更好的网页设计？

> 原文：<https://www.sitepoint.com/css-skills-better-web-design/>

凯文·杨克

**本周，在  上，我们讨论了 *[性感网页设计](https://www.sitepoint.com/premium/library)* 作者 Elliot Jay Stocks [对那些不知道自己的 CSS 的设计师们说了什么](http://twitter.com/elliotjaystocks/statuses/9227592793):**

> 老实说，我很震惊在 2010 年我仍然遇到那些不会自己设计代码的“网页设计师”。没有借口。

斯托克在一篇冗长的博客文章中概述了他的立场。在这里，他提到了学习为他的设计编码所必需的 HTML 和 CSS 对他来说是一次解放的经历。

> 未知让我退缩；我不喜欢事物的运作方式，这让我不敢创作出最好的作品。但是通过一点点技术知识[…]，我“得到”了我们工作的媒介。

作为一个代码迷，我发现这里很容易赞同 Stocks 的推理。我最近和一个不了解 CSS 布局机制的设计师一起工作，我发现很难向他解释他的设计必须考虑的一些重要约束。

毫无疑问，设计一个网站而不了解网站是如何构建的，就像设计一座建筑而不了解建筑材料如何应对重力、风和偶尔发生的地震一样？

但我想知道，无知也能解放吗？一个不懂建筑材料的建筑师可能会设计出一个*不可建造的*建筑。然而，最初的设计可能会有新的想法，这些想法对于一个思维受到重力、拉伸强度和静摩擦力束缚的人来说是永远不会想到的。

也许一些最令人兴奋的网页设计想法被锁在一个不懂代码的设计师的头脑里，等待一个有才华的开发人员的实际解释。
