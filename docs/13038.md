# Opera 和 Safari 通过 Acid3 测试

> 原文：<https://www.sitepoint.com/opera-and-safari-pass-acid3-test/>

![Opera Narrowly Beats Safari to the Finish Line](img/8dd17995c5aa64e0949ca657e85f170c.png)Opera 和 [WebKit](http://webkit.org/) (为[苹果的 Safari 浏览器](http://apple.com/safari/)提供支持)的开发团队在过去一周都宣布，他们的浏览器渲染引擎在由 Web 标准项目运行的 JavaScript 和 DOM 标准符合性的 [Acid3 测试](http://acid3.acidtests.org/)中取得了 100/100 的成绩。

Opera Software 的核心技术负责人 Lars Erik Bolstad 显然充满了自豪，他写道:

> 自从[Acid3]测试最近正式公布以来，我们的核心开发人员一直在努力修复错误并添加缺失的标准支持。今天我们第一次达到了 100%的合格率！还有一些遗留问题需要解决，但我们希望很快就能解决。

几天后，Opera 团队[发布了一个公共版本](http://labs.opera.com/news/2008/03/28/)来证明他们不只是说说而已。

WebKit 开发人员 Maciej Stachowiak 肯定对在帖子中被击败感到有点失望，当他在 Surfin' Safari 博客上写道:

> 凭借 r31342，WebKit 成为第一个在 Acid3 上实现 100/100 的公开可用渲染引擎。最后一个测试 test 79 是对 SVG 文本渲染的残酷考验。

他还发布了团队面临的一些更困难的 bug 的细节；其中一些听起来真的很棒。

虽然来自两个开发团队的代表承认他们仍然有一些工作要做，以解决一些测试的性能问题，但还是要祝贺他们认真对待 web 标准，以及他们致力于使每个 web 开发人员的工作更加容易。两个团队都做得很好，提供了迄今为止最符合标准的浏览器！
