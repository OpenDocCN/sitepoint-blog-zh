# 自助可访问性

> 原文:[https://www.sitepoint.com/yourself-accessibility/](https://www.sitepoint.com/yourself-accessibility/)

如果万维网要成为真正的全球媒体，那么就必须做出改变，让每个人都能访问它。如今，大多数电视节目都为失聪的观众配有字幕，建筑物对坐轮椅的人来说是无障碍的，书籍也被录制下来，这样人们就可以听了。现在是时候将这些相同的原则应用到网络上了！

##### 可访问性——谁在乎？

可访问性仅仅意味着提供灵活性以适应每个用户的需求和偏好。在互联网环境中，可访问性使得计算机技术和互联网资源对更多的人有用。我们需要互联网可访问性，因为它将网站的潜在受众扩大到数百万需要使用替代浏览技术的人。

但是，即使对于没有任何影响计算机使用的具体身体或精神特征的人来说，已经发现采用通用设计原则可以:

*   减少疲劳，
*   提高速度，
*   减少错误，以及
*   减少所有用户的学习时间。

在许多方面，通用设计解决了更大的可用性问题，并使每个人的事情变得更容易。

##### 遵循指导方针

2000 年 5 月 5 日，万维网联盟(W3C)的网页可访问性倡议发布了网页内容可访问性指南 1.0，简称为 WCAG。WCAG 已经在许多网站上实现，可以非常有效地用于开发一个可访问的网站。

被指定为高优先级项目的内容可访问性准则是:

1.  提供听觉和视觉内容的等效替代

3.  不要只依赖颜色

5.  正确使用标记和样式表

7.  阐明自然语言的使用

9.  创建优雅转换的表格

11.  确保采用新技术的页面能够优雅地转变

13.  确保用户控制对时间敏感的内容更改

15.  确保嵌入式用户界面的直接可访问性

17.  针对设备独立性的设计

19.  使用临时解决方案

21.  使用 W3C 技术和指南

23.  提供上下文和方向信息

25.  提供清晰的导航机制

27.  确保文档清晰简单

##### 自助可访问性

那么，如何确保您的网站实际上是可访问的呢？你已经阅读了指导方针，确保尽可能严格地遵循了它们，但是你如何检查它们呢？幸运的是，你可以做很多事情。

首先，你可以自己查看网站:

1.  关闭图形

3.  关闭声音

5.  通过查看灰度页面来检查对比度

7.  关闭样式表

9.  关闭脚本、小程序或其他编程对象

11.  使用浏览器允许的最大字体大小

13.  调整浏览器窗口的大小

15.  选择所有文本，并将其复制到文字处理器中，以确保它是有意义的

一旦你对你的网站在这些条件下尽可能的有用感到满意，为什么不试着使用一个自动化的辅助工具呢？有许多免费的、易于使用的工具可以帮助你确保你的网站符合可访问性指南。

***1。巴比**T3】*

[www.cast.org/bobby](http://www.cast.org/bobby)

Web 内容可访问性指南解释了如何使多媒体内容(如图像和视频)更容易被更多的受众访问，Bobby 检查了这些指南中描述的可访问性问题。

Bobby 只能为有限数量的可访问性问题提供自动检查。例如，它有助于定位丢失的文本替代视觉效果和检查不整洁的 HTML 代码。但是，要确保所有用户都能访问网站，还需要做更多的工作。

例如，Bobby 无法评估一个站点的布局一致性、导航的便利性、上下文和方向信息的提供，或者是否使用了清晰易懂的语言。该程序也不能检查一个网站如果没有图形或颜色，在不同的分辨率下，或使用不同的字体大小，或通过纯文本浏览器会是什么样子。

因此，Bobby 产生了一个非常大的报告，其中大部分是由网站作者进行手动检查的建议组成的(这实际上可能是不必要的)。

***2。**一-提示*

[http://aprompt.snow.utoronto.ca/](http://aprompt.snow.utoronto.ca/)

A-Prompt 是多伦多大学开发的独立工具。开发人员在他们自己的计算机上安装程序，并指定他们想要评估的文件。然后，该程序引导用户完成一系列向导式的问题和答案，完成后，文件被更新，以纳入开发人员根据软件的建议所做的任何更改。

尽管 A-Prompt 根据合理的原则提出建议，但是对文件的实际更改是由开发人员做出的，因此这个人仍然需要对可访问性的基本原则有一个坚实的理解。

***3。电梯在线**T3】*

[http://www.usablenet.com/](http://www.usablenet.com/)

LIFT (UsableNet，2000)是一种核心的语义分析技术，UsableNet 将它打包成几种形式，以满足各种开发人员和测试人员的需求。

它在 2000 年 4 月作为可用性分析器首次发布，但是在过去的一年里自然地发展到包括可访问性。基本上，LIFT 是一个软件 Web 分析引擎，可以以多种方式应用。

有一个基于 Web 的试用版(在 UsableNet 主页上)，它将从一个指定的 URL 开始评估五个页面(您可以对任意多个 URL 重复这个过程)。

LIFT 也可以作为 Macromedia Dreamweaver 产品购买，它为 Dreamweaver 菜单添加了选项，允许开发人员测试可访问性。

***4。用于 Macromedia Dreamweaver 和 Dreamweaver UltraDev 的 508/W3C 辅助功能套件**T3】*

这个免费的扩展是由 UsableNet 开发的，它为开发者提供了根据一组可定制的可访问性指南检查单个页面甚至整个网站的能力。

508 可访问性套件提供了一套全面的测试，类似于 LIFT Online。也许最重要的是，它利用了 Dreamweaver 的创作环境，将 Web 内容设计人员/开发人员指向页面上需要修改的特定位置。

***5。**浪潮*

[http://www.temple.edu/inst_disabilities/piat/wave/](http://www.temple.edu/inst_disabilities/piat/wave/)

Wave 是一种工具，可以帮助人们执行那些需要人类判断的任务(例如，“这种替代文本与这种图像的功能等同吗？”“这个阅读顺序有意义吗？”).

波:

*   显示图像的可选文本，并突出显示页面上的区域，以便与图像进行比较
*   提供带编号的箭头，以显示线性化的阅读顺序
*   显示为小程序提供的 HTML 等价物(如果有的话)
*   执行自动检查(检测缺失或可疑的替代文本)

但是，此时它的功能性还远远没有覆盖所有的可访问性检查点。

***6。HTML 和 CSS 验证器***

[http://validator.w3.org/](http://validator.w3.org/)

[http://jigsaw.w3.org/css-validator/](http://jigsaw.w3.org/css-validator/)

W3C 的 HTML 验证器和 CSS 验证器对高级用户特别有帮助。这些免费工具不检查可访问性问题；相反，他们检查 HTML 和 CSS 的正确使用。

这对了解 HTML 和 CSS 但不了解有助于提高可访问性的相关技术的设计人员和开发人员很有帮助。HTML 和 CSS 验证器可以发现不正确的编码，并指出可能的解决方案。

##### 易访问性

可访问性不再是一个美好的愿望——它对网络的发展至关重要。验证应该包括自动和人工检查的结合。

自动化工具寻找页面可访问性的明显问题，而手动检查方法通常解决内容的连续性和流动。但在所有情况下，目标都是让目标受众中尽可能多的用户能够访问信息。不要让规则妨碍所有访问者获得他们需要的信息，也不要让他们得到他们不需要的信息。
