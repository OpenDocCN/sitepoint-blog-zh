## 采访记录

凯文:2010 年 3 月 26 日。与破坏性用户打交道；脸书和推特社区；以及设计在社区网站中的作用。我是凯文·杨克，这是 SitePoint 播客#54:与德里克·波瓦泽克一起建立社区，第二部分。

本周，我们继续我们在[播客#52](https://www.sitepoint.com/podcast-52-building-communities-derek-powazek-part-1/ "SitePoint Podcast #52: Building Communities with Derek Powazek, Part 1") 中停止的话题，与德里克·波瓦泽克和帕特里克·奥基夫就网络社区的来龙去脉进行第二次也是最后一次对话。当我们离开你的时候，我们在讨论当你的社区发展到问题用户占用你大量时间和资源的时候该怎么办。

凯文:所以我意识到我们可以花一个小时来讨论如何应对这些挑战

德里克:和我*有*。

凯文:你能把它浓缩一下吗？你只需要做个艰难的决定，说这不是我们要做的。我需要对禁止你感到放心，即使这可能会对你造成永久的伤害？

德里克:嗯，我想我们需要说几件事。每个社区都需要一个服务条款和一套社区指南。在这些文件中，它应该说“我们保留以任何理由拒绝向任何人提供服务的权利。”除非你是政府，否则你可以这么说。你应该这么做。你应该弄清楚什么样的事情是不好的。然后，是的，如果你有一个对社区有害的人，你必须做出艰难的决定，说我真的很抱歉，但你在这里已经完成了，网络上有很多其他网站，也许你在其他地方会更适合。回到向我们自己和其他人承认每个社区在某种程度上都是排外的，这是社区定义的一部分。

凯文:德里克，2009 年末，我有幸在珀斯听了你关于这类事情的演讲…

**德里克:** [网页边缘](http://www.edgeoftheweb.org.au/ "Edge of the Web - Web Conference 4-6 November 2009 - Perth, Western Australia")！那太有趣了。

凯文:…我会确保[链接到展示笔记中的那些幻灯片](http://www.slideshare.net/fraying/wisdom-of-community-four-kinds-of-crazy-2438361 "Wisdom of Community: Four Kinds Of Crazy")…

德里克:哦，太好了。

凯文:…如果人们想更详细地了解你对这类事情的想法。

德里克:非常好。

凯文:但是说到大型社区，让我们来谈谈那里的巨头、Facebooks 和 Twitters。人们很容易把脸书当成一个社区，也很容易把 Twitter 当成一个社区，但是真的，就像你之前说的，这些真的是工具；它们是建造社区的原材料。

**德里克:**毫米。

凯文:然而，看起来脸书和推特确实有他们自己的社区，正如你之前提到的，我们总是听到他们试图改变这些服务。但是这些是少数发声者吗？我的意思是说，这些社区是幻觉吗？

德里克:哦，这让我想说“在哲学课上，你会意识到所有的感知都是幻觉”，但这可能会把对话引向不同的方向。

凯文:对。

德里克:我认为这两个例子很能说明问题。脸书和推特都是非常分散的社区，没有中央饮水池。所以这是一个中枢辐射系统，你在中心，你有卷须伸向人们，伸向某些人，每个人都有一个轮子，它们都是重叠的。所以这是一个社交网络，而不是一个中心社区。

凯文:在 Twitter 的例子中，没有水坑，但有消防水管。

德里克:是啊！在推特上，这就是他们的共同点，这些重叠的社交圈。但不同的是，脸书至少传统上要求双边选择加入，所以基本上我必须对你说是，你必须对我说是，然后我们就有了联系。因为推特只是一种方式。你只要确定一个你想追随的人。在这方面，它就像 Flickr 一样，是一个开销更低但距离更远的社交结构，所以它不是关于创建友谊链接，而是关于关注你似乎感兴趣的人。而脸书，传统上，无论如何，默认为私人，所以你真的看不到任何东西，除非你知道有人。而 Twitter 传统上默认为公开，所以它更多的是公开表演，而不是脸书。有趣的是它们都朝着对方移动。因此，脸书已经改变了他们的隐私默认设置，我认为这是一种完全间接的、偷偷摸摸的方式，并试图让他们越来越多的内容公开，这将是非常有趣的，因为人们意识到谷歌现在将把他们所说的一切都索引为公开，并将提供给他们的老板。这就是脸书受欢迎的全部原因，因为它远离了你的老板、父母和兄弟的视线 Twitter 越来越多地添加群组和列表等，我认为这正朝着脸书版的自我分组发展。看着他们以那种方式相互回应，真是令人着迷。

我更感兴趣的是，如果你——我认识一些在 Twitter 工作的人，所以我不代表他们说话，但是如果你问他们，他们是一个社区吗？他们会说不是。他们会说我们就像电子邮件；我们是一个分销系统。我认为这是不真实的。它们不是一个分配系统。它们没有内置在网络服务器中，也不是 Twitter 的全球网络，而是 Twitter.com 和他们的 API。所以就有了 Twitter 社区，只是每个个体的 Twitter 社区都不一样，和其他所有人的略有重叠。我认为它真正吸引人的地方是它是我能想到的第一个社区——虚拟社区，我能想到的——它大部分是通过 API 发生的，很少是在 Twitter.com 发生的，这是新的，不寻常的，我认为这为网络和其他事物指明了一定的方向。

凯文:嗯。我们采访了 Twitter 的 Alex Payne，他是 Twitter API 的负责人，他承认这是他们偶然遇到的事情，但他认为这是他们目前最大的优势之一。

德里克:当然。他们对 API 的早期和丰富的使用真的使得 Twitter 相关的整个生态系统来到了这个世界。这就是为什么他们超越了所有人，没有人能赶上他们。Pownce 是一个更好的网络体验，更复杂的工具，只是跟不上，因为 Twitter 一直拿着球，从不放手。这是一个可怕的体育比喻。我认为这是因为他们发布了一个 API，所以所有的动作都在那里。因为说到底，给我的朋友发送 140 个字符并不是一件困难的事情，所以 Twitter 必须通过总是率先采用任何有趣的新功能，来领先于在其他地方实现该功能的容易程度。那么 Twitter 有多少 iPhone 客户端呢？我不知道，但这比任何公司都需要的多，而脸书是相反的方式，它真的是关于 Facebook.com 的，对，他们有一些联系，但真的是关于这一个网站。所以从那方面来说，它更像是一个集中的社区，但是仍然没有一个集中的水坑，没有一个类似于 [Flickr 帮助论坛](http://www.flickr.com/help/forum/ "Flickr: The Help Forum")的平台，Flickr 的每个成员都可以访问，对吧。

凯文:如果有一个论坛被它的成功冲昏了头脑，那一定是 Flickr 帮助论坛，在那里找不到任何东西。

德里克:嗯，我得说我很高兴管理那个论坛不是我的工作，因为我向那些管理论坛的人致敬。

凯文:所以，说到 Flickr 帮助论坛，一个强大的社区可以是一种祝福，也可以是一种诅咒。你有没有劝过一个网站经理不要把精力投入到社区建设上？

德里克:是的。通常，当客户来找我时，他们想的是当每个人都喜欢他们的时候，他们的社区在一年后会有多棒。这将为他们的品牌、杂志或报纸等创造价值。我早期的工作是让他们停止思考这个社区会有多棒，因为*显然*它会很棒，让他们开始思考工具开发，因为这才是我们真正在谈论的。这就是为什么我在一开始说你不能创造一个社区，你只能创造一个游乐场，你可以创造一个结构。所以一旦你开始谈论结构，你就必须谈论规则是什么，人们实际上在这里做什么，为什么这个东西在这里？希望你已经有了一个现存的社区；通常你称他们为观众或消费者，现在他们彼此不说话，至少不是你培养的。如果你把麦克风给他们，他们会说什么？所以你可能想做一些用户测试。如果你做用户测试，给他们一个麦克风，他们说“我恨你！”，那么也许你不想把话筒给他们。

凯文:是的。

德里克:如果你是微软，如果你是一家保险公司，但愿不会如此。想象一下 AIG 的公共论坛，对吗？有些公司的声誉或商业行为受到了严重损害，简单地将社区工具加入其中不会让事情变得更好，反而可能让事情变得更糟。

凯文:所以如果你的商业计划没有让人们喜欢你的公司。

德里克:是的。每个人都喜欢谈论 TiVo，就像我一样，但是每个人都喜欢他们的 TiVo，所以这是很自然的事情。虽然我不得不脱帽致敬，但这里有一个很大的例外，我认为是壳牌石油公司创建了一个论坛，在那里我们想谈论全球可持续发展和环境问题，我们想听听你们的意见，我们是一个巨大的整体石油公司，我认为，总的来说，它属于那种令人讨厌的类别，但他们勇敢地把它放在那里，他们邀请人们来处理它，并进行反击，开始这场对话。当你这样做的时候，令人惊奇的事情会发生，那就是如果你把麦克风给讨厌你的人，他们可以对着麦克风喊一会儿，在他们对着麦克风喊完之后，他们会感觉好一些，他们也不会那么讨厌你了。这是一件非常勇敢、非常困难的事情。就像玩炸弹一样；你必须非常小心，但这是可行的，他们做到了。我认为这是壳牌公司的谈话或壳牌公司的关心；我得去查一下网址。*【德里克后来让我知道它叫[告诉壳牌](http://web.archive.org/web/*/shell.com/tellshell)，但现在已经不存在了。凯文。]*

你知道，当人们问我是否应该做这个在线社区的事情时，不管那是什么，我的意思是我不会试图吓走他们，但我倾向于关注这样做实际上需要什么，因为这不是真正关于 XYZ，你不可能一年赚五千或十万美元，或者你会有这么多的人，或者这些事情中的任何一件会是积极的事情。这几乎就像你必须做什么才能到达那里。现在是什么时候了，你有没有可以奉献给它的人？你有全职的人来做这件事吗？你对此有预算吗？你能负担得起保持这个在线运行吗？即使几个月、六个月或不管多长时间后，事情不一定会进展顺利，你知道，你有资源支持这件事吗？因为当你有一个像 TiVo 这样的公司，有一个强大的社区时，它可能看起来很浮华，显然它有很多吸引人的原因，但它需要大量的工作、时间、努力、奉献和购买才能真正实现。所以我总是浏览一系列的事情，好吧，你需要这个人，你需要这个，你需要这个承诺，这笔钱，这些资源，等等。我不想吓唬人们远离这个，因为我喜欢这个，但它比许多人意识到的要难得多。

德里克:是的。我认为从一开始，另一个关键的事情是，如果你在谈论创造一个以前没有的环境，那就是坐下来进行一次非常冷静的对话，讨论成功案例看起来像什么，失败案例看起来像什么，并将其与指标联系起来，比如如果我们在下个月创建了一千个帐户，其中十个实际上登录了不止一次，这是成功案例还是失败案例？这样，一个月后你可以看看它，然后说它是否有效，如果无效，你需要马上承认，要么改变它，要么放弃，调整你的——调整它，就像看看你是否可以做出改变，以满足这个成功的案例。很多时候，特别是对于较小的创业公司，我们太喜欢我们要做的任何事情的想法，以至于我们不会停下来好好想想如果没有人来玩或者如果只有几个人来玩，它仍然值得做吗？然后，如果你必须在它已经推出后编造这些数字，那么你总是会捏造它说，这个月我们有 100 个用户使用它，所以这就是成功，100 个用户。如果你能从一开始就尽可能清醒地做这件事，那就好得多了，这样你就有了一些切实的东西，你可以看着并说这是有效的。因为社区可能非常模糊，所以很难说什么是成功。

在某些情况下，如今社区几乎被认为是新网络存在的必要组成部分，但脱离社区元素可能是一个大胆的决定。我在想那些选择关闭博客评论的人。

德里克:我刚刚做了。

凯文:嗯，你为什么不谈谈你最近对 Powazek.com[的重新设计。](http://powazek.com/ "Derek Powazek - Hi, I'm Derek.")

德里克:当然可以。嗯，我应该说这不是我——我可能会在将来的某个时候打开评论。Powazek.com 是一个个人网站，不是我的商业网站，我有咨询客户，我不寻找新的客户，所以我在 Powazek.com 没有一个大的商业网站。对我来说，这真的是一个传统的主页，让我成为我自己，并链接到有趣的东西；我在写博客，我们就直说吧。在过去三年左右的时间里，让我感兴趣的是，过去发生在博客评论中的许多随意对话已经迁移到 Twitter 上，我非常高兴看到这种情况发生，因为 Twitter 真的为此进行了优化。一个人对你大喊大叫的程度有 140 个字符的限制，我认为这有时真的很有效。你可以回复 140 个字符，这有时候对我来说很好。所以当我开始编辑模板并加入评论的时候，我做了这个重新设计，我只是认为评论没有给我任何东西，事实上它们让我不想写。最近，他们让我不想在我自己的网站上写东西，这不是重点，所以我打算不写了，只用 Twitter 来跟进，看看效果如何。到目前为止，这是一把双刃剑，你知道，你会失去很多我们都知道的愚蠢的评论，但你也会失去真正真正好的评论，这些人会花时间真正消化你所说的话，并为帖子添加一些有价值的东西。我想我接下来真正想做的是——网络日志上的评论如此糟糕的原因之一是因为网络日志评论系统还没有进化到删除或不删除评论的地步。这真的很蠢。我们应该能够把规则放入评论系统，说你知道什么，除非你用 50 个词或更多你不能说，这将结束所有的“我同意”，或“你很愚蠢”的帖子，或“第一”，对不对。所以系统应该说这个博客的所有者已经决定，如果它少于五十个字，那么它就不值得，所以…

凯文:它们需要是不同的单词，而不是感叹号。

德里克:对，没错。

帕特里克:也不是空格。

德里克:是的。如果全部大写，评论表单应该将内容改为标题大小写，顺便说一下，Flickr 评论就是这么做的。它还应该说如果超过 1000 字就太长了，我不会让你发布的。这是应该的——计算机确实擅长做这些事情来塑造进入它们的输入，而这些输入不是内置在这些系统中的。所以接下来我想做的事情实际上是创建一个 WordPress 插件，由知道如何做这些事情的人来创建一些规则。评论系统也应该在一段时间不活动后关闭。我的 Vox 博客仍然收到垃圾邮件，我从 1997 年起就没发过了。不，不是这样的，2007 年，对不起。

凯文:我相信当你开始写博客时会经历这样一个周期…

德里克:哦，天哪。

凯文:…你有这种理想主义，你认为我说的一切都将是永恒的，我希望新的对话在多年后从我的旧想法的灰烬中产生。但是在第二年左右，通常是你停止在博客上发表文章的一年零六个月后，旧文章上的新评论太多了，你就把它们关掉了。

德里克:对。发生的情况是，互联网上任何无人注意的开放式麦克风都会被用来发送垃圾邮件。有一些自动化的垃圾邮件发送者非常精明，非常非常善于发现任何漏洞，在网上发布链接。他们这样做是为了游戏谷歌。这不是真的要丑化你的网站，而是要为第三方创造谷歌果汁。但是，如果他们让这些麦克风开着，到处乱放，无人看管，这将成为任何社区平台的问题。所以一般来说，允许这些地方存在对网络是不好的，更不用说一个已经过时的 WordPress 安装了，会有自动蠕虫爬来爬去在你的模板中粘贴链接，因为他们发现了一个漏洞。我认为我们作为一个创作者群体，必须非常、非常清楚这些问题，因为它们真的会损害网络的健康。

Patrick: 我认为关于适度和低质量工具的评论是一个很好的观点。当我沉迷于博客的时候，我有一个很棒的文档系统，我有我们删除的每一个帖子的文档，保存，副本，我们采取的行动，所有的一切。我可以在任何时候回顾它，看看我们几年前做了什么。去博客写作并允许评论，只是看看，嗯，没有账户，没有办法以任何方式审查这一点，这里没有文字审查，似乎没有办法隐藏这一评论，除了删除和允许，这里真的没有什么，令人失望。我想我当时在我使用的平台上寻找插件，想知道这里有什么我可以用来更好地缓和这种情况的吗？当然，我找到了一对夫妇，但它远远落后于我所习惯的。这让事情变得很困难，因为我确实想要——我确实有评论指南。我试着运用类似的策略来维持一个良好的氛围和我想要的评论类型。但是总的来说，平台有时会让事情变得困难，就像你说的，博客评论，他们如此落后真有趣。有一些插件；我实际上有一个插件可以在十四天后关闭评论。

德里克:我也是。我在 WordPress 上有[评论超时。](http://jamesmckay.net/code/comment-timeout/ "james mckay dot net » Comment Timeout")

Patrick: 是的，我的意思是这些东西是有用的，但是如果有某种更大、更好的套件，比如审核工具、记录所有信息并记录等等，那就更好了。如果你需要登录的话，甚至在 WordPress 上创建一个账户似乎也有点弱。我不知道。所以如果你写插件的话，我绝对支持。

德里克:(笑)是的。
