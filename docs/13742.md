# 使用 Fluff 和 Hot Air 构建您自己的 Web 2.0 应用程序

> 原文：<https://www.sitepoint.com/application-fluff-hot-air/>

注意:这篇文章的发表是为了制造一些愚人节的笑料。真正的作者是 SitePoint 的马特·马加因。

所以你有一个伟大的想法，你想变成下一个大事情？那么，跳上 Web 2.0 的潮流吧！web 应用程序背后的发展势头意味着，现在是开始收获辛勤工作的回报的最佳时机。

等等！辛苦？对努力工作不感兴趣？没有时间花在熬夜修复错误和添加杀手级功能上？那么这篇文章就送给你了。我们将向您展示如何从想法到资金充足的应用程序，而无需编写任何代码。

以为你所要做的就是给你的产品添加“测试版”就能成功吗？不是真的！在今天的环境下，有可能安排第一轮风险投资，让你的月度最佳网络应用被雅虎抢购一空！在你到达贝塔之前。

我们称之为致富。

##### 什么是致富？

致富是更快、更容易的方式，甚至不用写该死的东西就能从你的热门网络应用程序中赚钱。

致富就是做一个假象来吸引感兴趣的人(比如投资者)，然后跳过烦人的编码和测试工作。

发财少了。少开会，少交流，少文档，少特性，少代码，少 bug，少讨厌的客户，少实际编程。

致富没有给你的顾客带来任何好处，所有的东西都进了你的口袋。

***老办法***

在过去，网络企业家必须解决一个需要解决的独特问题，并通过开发一个可用的、可靠的、可扩展的应用程序来为自己开辟一片天地。许多优秀的 web 应用程序利用了丰富的交互性，但在世纪之交未能获得动力，因为:

*   对于一个客户端处理器跟不上的时代来说，这款应用过于偏重客户端，或者
*   开发团队成员未能定期与客户沟通，因为他们忙于修复 bug

然而，那是当时。今天，竞争环境与 5 年前不同了——[网络正在进入一个新的时代](http://www.adaptivepath.com/publications/essays/archives/000430.php)，而致富正是利用了这一点。

***建少***

其实根本不要建什么。要启动一个成功的 Web 2.0 应用程序，你只需要做好七件关键的事情，其中没有一件是应用程序本身。

##### 成功的 Web 2.0 应用程序的七个要素

为了让你的应用程序成功，你需要具备以下七个要素。

***1。热门创意:*** 虽然它不一定是一个你可以实际解决的问题(或者，事实上，甚至是一个需要解决的问题)，但你的创意确实需要听起来吸引人，并利用一个以上的当前技术术语。最起码，您必须毫无例外地包含 AJAX——即使您的应用程序不恰当地使用了该技术。记住:如果 AJAX 没有被提及，您的应用程序将会失败。还有很多其他技术也将有助于您的事业，包括 RSS、微格式、SOAP 和 Ruby on Rails。

幸运的是，有几个 [有用的](http://andrewwooldridge.com/myapps/webtwopointoh.html) [资源](http://odio.us/plan/)可能会给你一些启发。理想的情况是，试着将这个时髦词融入应用程序的名称中。这是获得额外关注(以及流量)的一个简单方法，尤其是如果该技术的拥护者[听说你甚至都没有使用它。](http://alex.dojotoolkit.org/?p=551)

***2。一个很酷的名字:*** 如果你的应用程序的名字不够简短和吸引人，它就不会流行。试着录下你自己朗读杰弗雷·乔叟的《坎特伯雷故事集》的段落，然后慢慢倒放，寻找灵感。如果你仍然难以决定一个名字，你可以从《星球大战》中的角色中选择。

***3。一个新潮的网站设计:*** 这是给定的。所有 Web 2.0 应用程序的前端都有圆角、渐变、大量空白和[突出的配色方案](http://miles.burke.id.au/blog/wp-content/files/web2palette.jpg)。

不具备任何设计技能？没关系:你可以[按照一步一步的指示](http://mentalized.net/journal/2005/10/10/building_your_very_own_web20_layout/)或[生成一个布局](http://www.web20generator.com/)，直到你获得第一轮融资。一旦应用程序被收购并重新命名，你的布局无论如何都会改变，所以在这个阶段纠结于细节是没有意义的。

使用现在出现的众多 JavaScript 库之一来为您的站点增添更多光彩是非常有帮助的——从 Prototype、DoJo、script.aculo.us、FACE、AjaxTK、Moo.fx 和 Yahoo！UI 库。这看起来像是很大的努力，只是为了分散注意力，这和`<blink>`标签没什么不同，但是相信我，这些库给你的网站带来真正的优势是值得的。完成后，别忘了[验证你的网站](http://web2.0validator.com/)！

***4。一个博客:*** 与你的用户保持沟通是必要的，这样可以分散他们对你无法交付最终产品的注意力。博客的非正式性质意味着你可以跑题，比如谈论你 3 个月大的可卡犬如何把你的手机埋在你后院的番茄丛下，然后立刻被原谅！这种个人化的接触将帮助你认同你的用户，并向他们展示一个没有个性(也没有特色)的应用程序人性化的一面。

如果你想在离题后让事情回到正轨，有很多资源可以帮助你保持流行词汇的流行。

更好的是，制作一个播客，包含类似的个人内容，对网络未来的假设，以及你的产品如何弥合今天的用户想要的和你的误导竞争对手提供的之间的差距。不过，在记录你的会话之前，一定要先温习一下那些流行词汇——它们可能相当拗口。

***5。联合:*** 任何称职的 web 开发人员都会告诉你，Web 正在快速地从 HTML 转向 XML。通过为您的应用程序提供 RSS 和 Atom 提要，您可以自动增加窗口，让您的客户相信您的 web 应用程序确实在做一些事情。

如果您想知道应该在这些提要中包含什么内容，没关系。某一天发送一张随机的 [flickr](http://flickr.com/) 照片，第二天发送一个 [del.icio.us](http://del.icio.us/) 链接，创造一种多种 API 以真正的 Web 2.0 风格聚合的假象。RSS 还没有达到这种技术的用户懒得退订不产生高质量内容的 feeds 的地步，所以这仍然是一个尚未开发的渠道来加强门面。另外，你还在测试阶段！

***6。社区焦点:*** 在营销文案中，将你的应用描述为“社交”软件。使用流行词大众分类法和标签云，并强调你的应用程序如何让用户能够控制他们的内容，以帮助赢得任何反对者。

***7。一个开放的 API:*** 为你的应用发布一个 API，让开发者对它感兴趣。在这个阶段，具体方法的实现并不重要；只需放置一系列返回值 42 的存根方法。别担心，他们是书呆子——他们会让[读道格拉斯·亚当斯](http://www.amazon.com/gp/product/0345391802/)，所以他们会理解这个笑话。他们不仅会原谅你，还会因为你微妙的幽默而更加喜欢你。如果有人抱怨，提醒他们你的应用还在测试阶段。

##### 高效促销的七个方面

有很多方法可以让你创造自己的炒作，但我们已经缩小了范围。以下是我们发现的最有用的七个。

***1。直接进入私人测试阶段:*** 只邀请受邀者访问应用程序，但不邀请任何人。没有什么比产品的独特性更能激发人们的欲望了。如果没有人拥有它，每个人都会想要它。简单！

***2。在论坛上谈论它:*** 在著名的论坛上发表一些帖子，张贴截图，大肆宣传这款应用程序，以及它如何成为你的头号必备生产力助推器。注册多个用户名，并在帖子中与自己讨论。包括那些“收到邀请”的人的不同观点。如果你没有时间做这件事，[花钱请人帮你做](https://www.sitepoint.com/building-a-forum-with-bribes/)。

***3。涂鸦:*** 模板艺术目前是大城市流行的一种涂鸦形式，出于某种原因，它被认为是有创意和有见地的，而不是碍眼的，因此作为一种公共艺术形式得到了认可。利用这一点！用你的应用程序的域名在当地火车站的墙上喷漆，在一个戴着防毒面具的男人的图片下面，度过你的夜晚。图像与您的应用程序无关也没关系——无论如何，您不是在编写东西！

***4。手动博客评论垃圾:*** 向知名博客发送指向你的网络应用的链接(提示:尝试将你的回复与你发送的文章进行智能回复)。当网站所有者看到你的域名不包含任何涉及青少年性行为或伟哥的内容时，他们很可能会关注你的链接，只是想看看到底有什么大惊小怪的。当他们到达你的网站，并意识到他们不能登录，因为它是在私人测试中，他们会开始在网上搜索更多关于你的应用程序的信息，并忘记他们是如何找到你的网站的。如果你的家庭和工作的 IP 地址由于这种做法而被封锁，就利用公共图书馆的电脑。

***5。提交文章:*** 写一篇关于你的应用所突破的新领域的文章，并提交给科技杂志，如 [SitePoint](https://www.sitepoint.com) 。这些网站的编辑忙于编辑印刷出版物和参加知名的技术会议，没有太多时间关注提交给他们在线存在的文章。通常，你可以将任何旧内容偷偷放入他们的首页，而他们甚至不会注意到！

***6。创建一个“Web 2.0 应用审查网站”:*** 从 [TechCrunch](http://www.techcrunch.com/) 中抓取内容(相应转述)，在网站上巧妙地审查你自己的应用。让它成为你审核的第三个或第四个 app，这样才不会引起太多关注。

***7。寻求博主的帮助:*** 请一位一线博主评论你的应用程序(链接到你自己的“评论”，这样你就能为你的网络应用程序和评论它的网站增加流量)。虽然他们不太可能在没有真正登录和试用你的应用程序的情况下这样做，但如果你给他们一些激励，他们可能会改变主意。给他们发送一些 Photoshopped 截图，显示标签云和正在使用的应用程序的图像，并将服务器宕机归咎于你的数据中心——这是他们刚才无法登录的完美借口。然后向他们承诺，当你把应用程序卖给雅虎时，你可以获得 20%的利润。根据需要谈判——每个人都有自己的价格。

##### 该组织

***开会有毒***

会议成为致富障碍的原因有很多，尤其是当会议涉及到你的风险投资者时。你看着他们眼睛的时间越多，他们想要看到实际结果的机会就越大。

相反，向乔治·克斯坦萨的生产力哲学学习。让你的投资者有充分的理由认为你已经竭尽全力了——给他们发大量的电子邮件，告诉他们你繁忙的旅行日程，包括会见那些有兴趣在他们的公司内部网部署你的应用的客户。用你自己的三个问题来回答他们的每一个问题。在你的电脑上自动执行任务，在凌晨 3 点向投资者发送一封电子邮件，报告你刚刚发现并修复的一个模糊的错误。不断向他们发送虚假新闻稿的链接和虚假引用，这些虚假新闻稿包含新闻网站上热情洋溢的评论，要求用户付费订阅才能查看文章。听起来很冒险？放心吧！当投资者意识到他们必须付费才能阅读整篇文章时，他们就不会费心了。毕竟，他们已经在你的“生意”上投入了大量的现金，却毫无结果。在看到回报之前，他们不准备再掏一分钱。相反，他们会把你提供的引用当作真理，并把他们本该花在研究你的“组织”上的时间花在祈祷在季度结束前从这种虚假的媒体报道中获得任何利润上。

***少雇，晚雇***

为了最大限度地降低成本，在雇佣任何员工之前都要犹豫。如果你一定要带人入伙，那就把工作外包给印度。肯定会有人以低于你每月宽带连接费用的价格完成这项任务。

***贵公司愿景中的敏捷***

事实上，甚至不需要有一个愿景。如果你正确地执行了你的推广策略，人们不会在意你的应用程序是做什么的——他们会被浮华的网站和尖端技术的使用弄得眼花缭乱，以至于忘记他们实际上从未使用过这个东西。

没有愿景会让你适应能力更强。如果你想把你的应用程序带到一个不同的方向，你会发现如果你没有把自己放在任何特定的道路上，那么这样做要容易得多。

***文档无***

在寻求首轮融资时，不要留下任何交易的书面记录。另外，一定要定期清理你的电子邮件和回收站(至少每隔几个小时)。如果事情搞砸了，你希望能够声称自己完全无知。

***(不)造出来他们就来了***

一旦一切都准备好了，你已经启动了你的推广策略，舔舔嘴唇，等待风投的邀请开始滚滚而来！此时，在一家声誉较好的瑞士银行开立账户可能是明智的。成功的企业家往往发现，在百慕大定居有利于减少他们在即将到来的现金注入中可能要支付的大量税款。

***冲洗并重复***

当然，你第一次致富的尝试可能会失败。要知道，尽管这些原则是合理的，但并不是每个企业家都拥有成功执行这些原则的商业头脑。在这种情况下，剩下要做的就是重新开始——寻找一个新的想法，重塑你的网络形象，如果有必要的话，重塑你自己的身份，以便远离不开心的投资者。

假设你已经遵循了这篇文章中关于从一开始就不要见他们的建议，这应该不是问题。

##### 结论

你也可以不用写一点代码就能快速到达 Web 2.0 亿。

在本文中，我们介绍了一种简单的策略，用于构建一个外观，以保护投资基金，避免未能构建成功的 web 应用程序，我们称之为致富。我们探讨了如何将高度成功的 Web 2.0 应用程序的七个要素落实到位，并将它们与高效推广的七个方面相结合，以增加应用程序的流量。

简而言之:建立它并卖掉它——实施是失败者的事！
