# 谷歌不再支持 GMail 和应用程序中的“旧”浏览器

> 原文:[https://www . site point . com/Google-drops-support-for-old-browsers/](https://www.sitepoint.com/google-drops-support-for-old-browsers/)

浏览器测试是一种痛苦。当用户吃着金枪鱼三明治，面朝北方时，没有什么比完成你的网络杰作却发现它在 OS Z 上的浏览器 X，版本 Y 中崩溃更糟糕的了。

谷歌遭受了同样的困扰，但在两个月内，他们打算解决这个问题。根据他们的官方博客:

> 谷歌应用程序将只支持现代浏览器。从 8 月 1 日开始，我们将滚动支持 Chrome、Firefox、Internet Explorer 和 Safari 的当前和之前的主要版本。每次发布新版本时，我们将开始支持更新，并停止支持第三老的版本。

**note:** What about…

[戏曲](http://www.opera.com/)？奇怪的是谷歌很少提到第五大最受欢迎的浏览器。它可能有相对较小的 [2%的市场份额](https://www.sitepoint.com/browser-market-june-2011-chrome-11-worlds-second-favorite-browser/)，但 2%是很多人，Opera 有最常用的移动浏览器。饶了他们吧，谷歌！我们很多人都在测试 Opera——你也应该这样。

谷歌的新政策是由开发团队决定的，他们希望利用现代 HTML5 技术，如文件拖放或桌面集成。旧的浏览器应该还能使用一段时间，但是某些应用程序功能会失效或者被禁用。最终，旧的浏览器可能会完全停止工作。

这项政策引发了许多有趣的问题。谷歌和 Mozilla 都实施了快速发布计划，Firefox 5 将在几周内发布。到那时，谷歌将放弃 Firefox 3 . x——一款仅在几个月前被取代的浏览器，仍然享有相当大的 14%的市场份额。

更大的新闻是 IE10。它可能会在 2011 年底前发布，一旦发布，谷歌将停止对 IE8 的支持。I E8 是世界上最常用的浏览器，也是唯一可以安装在 Windows XP 上的 IE 版本。根据 StatCounter 的数据， [XP 被将近 46%的网络用户使用](https://gs.statcounter.com/#os-ww-monthly-201105-201105-bar),许多公司都被这个老化的操作系统所困扰。如果他们使用谷歌应用程序，他们要么需要升级，要么迁移到另一个浏览器。无论哪种方式，都会导致员工再培训或操作系统购买成本，这可能会超过改用谷歌产品所节省的成本。

鉴于 Chrome 和 Firefox 的快速更新，谷歌对*【过时】*浏览器的定义可能是已经存在 3 个多月的版本。很少有开发者测试 Chrome、Firefox、Safari 和 Opera 的当前版本，但不管你喜不喜欢，IE 仍然是一个特例。

这是谷歌的勇敢之举。我钦佩他们的推理，但我对实施类似的政策感到紧张。

你会吗？
