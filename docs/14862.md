# 21 世纪网页设计要点

> 原文:[https://www.sitepoint.com/web-design-essentials/](https://www.sitepoint.com/web-design-essentials/)

 **<q>***“科技是如此有趣，但我们可能会被科技淹没。信息的迷雾可以驱逐知识。”
——丹尼尔·j·博斯廷***</q>

回想一下 20 世纪 90 年代末伟大的互联网建设。当任何人只要有几个小时的 HTML 经验，就会立即被雇佣去设计网页，这就不难理解网络的起源。谁能忘记结果呢？繁荣时期的网站通常是视觉盛宴和过度宣传的技术的网络空间食物大战。

如今，大多数网站设计都集中在现有网站的改进和升级上。随着经济持续低迷，更多的重点放在维持现有的，而不是开辟新的领域。因此，生产率、投资回报率和其他效率指标已经进入了网页设计的词汇。

这种对生产力的关注使得网站设计回归基本。考虑到最终用户，今天的网站的核心目的是快速、简单地提供信息或进行交易，没有冗长的动画介绍、夸张的视觉效果或多重超链接黑洞的干扰。

因此，让我们来看看优秀网站设计的基础，以及一些当前可用的工具和技术，来帮助你创建强大、可用的网站。我们还将看看一些在设计理论和浏览器技术前沿的概念，只是为了让你站在座位的边缘！

##### 好的网站从好的网页设计开始

在令人兴奋的建设时期，网页设计师直接进入页面设计，一次一个网页地构建整体概念和用户体验。只有在所有的页面都构建好之后，它们才被连接在一起——结果通常是一团糟。

在今天的世界里，大多数网页设计者在创建他们真正的网页之前都是从开发一个概念开始的。一些技术已经发展到可以帮助设计者开发一个概念，例如线框架和站点架构。在任何一种情况下，设计者都要定义一个起点，然后在页面、文档、外部链接等之间建立各种父/子关系。此外，可以定义一个单独的“链接框架”来指向文档或外部目的地。当然，今天，在任何人开始构建任何页面之前，已经定义了动态标记语言、数据库连接和 Web 服务支持等编程元素。

就像交响乐表演一样，音乐将每件乐器和谐地结合在一起，所以网站的概念定义将每个参与者(平面设计师、文案、程序员)在设计和谐中结合在一起。正如指挥记录时间和管理绩效一样，Web 项目经理使用这个概念定义来协调资源，并确保 Web 项目按计划完成其里程碑。

当前的 Web 创作软件程序(如 Adobe GoLive)为 Web 设计人员提供了信息架构、用户体验定义和其他设计工具。使用这种工具包，您可以预先构建更好的用户体验。无论你是为自己还是为图形艺术家、设计师和文字编辑团队创建这个路线图，一个经过深思熟虑的网站设计将确保你的交响乐不会发出任何刺耳的声音。

为了一瞥信息架构的未来，看看加州大学伯克利分校的用户界面研究小组的一个新应用， [DENIM](http://guir.berkeley.edu/projects/denim/) 。这个应用程序“画出”设计草图，然后帮助你将它们连接起来，然后将设计导出为 HTML。

##### 1%的汗水，99%的展示

具有讽刺意味的是，一个有潜力向数十亿人提供丰富信息的通信媒介是围绕着桌子设计的。表格被优化为在由高度和宽度定义的刚性框中显示表格数据——如果您正在阅读电子表格，这很好，但如果您试图获得交互式体验，这就太糟糕了。当然，我们试过一段时间的框架，但我们都知道他们去的方向。那么，设计师们还剩下什么呢？

级联样式表(CSS)似乎为将设计从表格的限制中解放出来提供了最大的希望。简单地说，CSS 使设计者能够将信息的风格与其结构分开。

排版元素(字体、大小等)、颜色和其他设计元素被划分到一组单独的 HTML 中，用于页面上的所有数据，这有助于避免图像地图等较胖的元素。CSS 可以跨多个页面重用，这增加了它的多功能性。CSS Level 2 (CSS2)是对已使用多年的原始 CSS 规范的升级，它提供了额外的控制和功能，如“@media”，允许设计者为特定媒体设置特定的样式。

![1164_webdesigncss](../Images/8a6cf22dab2b6bb3c9356305a685eead.png)

CSS1 过了一段时间才被广泛采用，因为存在(事实上继续存在)与该技术相关的浏览器不兼容问题。虽然许多问题已经得到解决，但在部署基于 CSS 的设计之前，完成全面的浏览器测试仍然是明智的。至于 CSS2，它仍然很新，可能会遇到一些部署问题，所以要小心行事。参见[HTML Utopia-使用 CSS 设计无表格，第 1 和第 2 部分](https://www.sitepoint.com/blog/)了解该主题的最终介绍。

另一个值得新兴网页设计师注意的概念是“液体表格”可伸缩的桌子给了设计师相当大的创作灵活性，消除了标准桌子设计的严格限制。液体表设计基本上是通过设置宽度和高度的百分比来工作的，而不是特定的像素度量。

但是要预先警告——流动设计是一项复杂的技术，需要设计师将用户的屏幕尺寸作为设计中的一个独立元素来考虑。导航栏和菜单等元素变得更加复杂，因为在设计中必须考虑一系列的位置问题。无论如何，实现良好的流畅页面为设计者提供了更高的灵活性。关于液体设计的介绍，请参阅尼克·威尔逊的文章[什么是液体设计？](https://www.sitepoint.com/blog/)。想要了解这个话题的入门知识，可以试试鲁迪·利姆贝克的文章[。](http://www.evolt.org/article/Liquid_Tables/20/2321/index.html)

##### 关于 XML 有一些东西…

从某种意义上说，XML 是一种类似 HTML 或 PHP 的标记语言。然而，因为标签语义和标签集是未定义的，而不是固定的，所以 XML 实际上是一种“元语言”,用于描述您 Web 设计者——定义的其他语言。要使用 XML，您需要编写一个文档类型定义(DTD ),它描述了一系列标签以及与这些标签一起使用的元素。

![1164_webdesignxml](../Images/cadc71ba7f9b3f18c61c1aa6b386442e.png)

XML 的灵活性不仅是它最大的优点，也是它最大的缺点。为了让计算机系统使用 XML 相互交流，它们需要共享一个标准(以 DTD 的形式)。这带来了许多协调问题，尤其是如果不同的企业为同一应用程序创建不同的标准。安全性也是一个问题，特别是对于政府网站设计者来说，这也是因为不同的系统需要共享 dtd 才能相互通信。

除了问题之外，围绕 XML 已经做了很多工作，尤其是有关键内容管理需求的企业。中小型市场出版商一直在积极寻求基于 XML 的工作流，这些工作流可以优雅地存储印刷和 Web 数据，并允许跨平台共享这些数据。另外，这些自行开发的系统包含业务规则和其他逻辑，有助于内容路由、截止日期管理和归档。

对于 Web 设计来说，可扩展样式表语言转换(XSLT)尤其重要。它允许 Web 设计人员定义 XML 文档和基于 XML 的数据的转换，为最终用户创建表示结构。XSLT 还可以帮助将 XML 转换成非 XML 格式，比如 HTML。它可以在 Web 服务器的后台工作，也可以在更新浏览器的客户端工作。

XSLT 仍然是一种非常复杂的语言，需要一些编程技巧。它也没有被很好地整合到视觉设计工具中。不幸的是，今天的浏览器并不完全支持直接的 XML 呈现，所以大多数基于 XML 的网站仍然需要将 XML 转换回 HTML 以获得最佳兼容性。然而，随着设计应用程序和浏览器变得越来越复杂，预计会看到对 XML 和 XSLT 的更多支持。

##### 只是浏览一下，谢谢

每个网页设计师都有他/她最喜欢的浏览器错误(首尾相连，他们可以绕地球八圈)，所以我不会再增加更多了。尽管如此，浏览器的数量仍在继续增长——Safari 和 T2 Opera 是最近出现的两个。

不过，从根本上来说，终端用户体验与过去几年相比没有变化。内容和数据显示在一个窗口中，对其他页面或数据的请求通过该窗口进行。浏览器本质上仍然是瘦客户机，要求另一端的 Web 服务器在向用户呈现结果之前完成大部分繁重的代码解析工作。

如果你真的想一窥未来，考虑一下发生在浏览器概念极端边缘的事件。在这里，你会发现一些应用程序打破了窗口的隐喻，使用可视化、蜘蛛网图像和其他非传统元素来引导用户找到信息或目的地。一些更有趣的概念包括[移动器](http://www.esono.com/boris/projects/ambulator/)、[数据云](http://datacloud2.v2.nl/)和[网络跟踪者](http://www.backspace.org/iod/iod4Winupdates.html)。

这些想法有很大的潜力，因为传统的 PC/显示器接口迅速老化，无线设备和无线连接的激增迫使设计社区处理各种新的演示平台。

##### 我需要做些什么？

当你着手网站设计时，问自己以下问题:

1.  我的用户可以通过两次点击获得他们想要的信息吗？
2.  我的视觉效果是分散注意力的还是有用的？
3.  有哪些技术是合适且不矫枉过正的？
4.  我的用户是否一直知道他们在网站中的位置？
5.  用户能否一键返回首页/首页？
6.  残障用户可以访问我的内容吗？

这个列表很简单，但许多网站继续违反这些基本规则——并在这个过程中挫败用户。请记住，你的竞争对手的网站只有一次点击的距离，所以消除上述障碍将有助于保持用户(和他们的美元)在你的网站。

##### 好的设计是好的生意

繁荣的年代分散了许多人对优秀设计技术的注意力，因为我们被一系列新技术和竞争热情引入歧途。

在当今竞争越来越激烈、越来越受关注的设计世界中，提供干净、清晰的信息和交互设计可以提高设计效率，改善客户端用户体验。此外，随着时间的推移，设计应用程序得到了改进，融入了许多复杂的功能，使设计师能够优化他们的创造力，同时保持编程方面的检查。此外，这些工具不再被指责编写“臃肿”的代码——这是过去困扰这些应用程序的问题。

在今天的浏览器中，仍然有许多新兴的技术，越来越多的表示平台，以及一些挥之不去的问题，但归根结底——好的设计就是好的业务。
