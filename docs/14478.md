# 经典案例研究:对逾期付款客户的有效但有风险的策略

> 原文：<https://www.sitepoint.com/classic-case-study-an-effective-but-risky-strategy-for-late-paying-clients/>

“我仍然没有收到一个月前工作的报酬。我有点犹豫要不要加快你项目的进度，因为我有几个项目正在进行中，我的优先事项已经转向付费项目了。”

因此，一位网络开发人员前几天给我发了电子邮件。顺便说一句，我们(我的商业伙伴和我在一家 web 开发公司)非常喜欢她:她速度快，专业，收费合理，工作质量高。

她的电子邮件是对我的回复，我让她告诉我们什么时候我们可以期待更多的项目成果，并问她是否可以在我们的项目上投入更多的时间。

她的回答让我们有点震惊。我们开始工作的时候就预付了她，后来还邮寄了两张支票。显然她没有收到这两张支票，这是没有意义的。此外，虽然我们知道她在等待付款，但她从未告诉我们，她正处于放慢项目进度的边缘。

上面的策略引起了我们的注意，因为我们真的想加快我们项目的进展，到目前为止，她是我们的主要资源(设计/可用性是这个项目的瓶颈)。

与此同时，这也让我们感到沮丧，因为这是一个惊喜，因为我们以为“支票已经寄出去了”

因此，我们立即打电话给她，想知道如何解决这个问题。当我们这么做时，我们得知她在我们不知情的情况下搬了办公室，所以她的支票很可能被转寄了。在我们谈话之后，她确实加快了她的工作，而且事实上超出了我们的预期。

我认为，总的来说，她的方法是一种很好的处理逾期付款客户的方法，但有一些重要的警告。以下是一些进一步的想法:

为什么有效:

它引起了我们的注意。我们赶紧打电话给她，解决了这个问题。

它利用了她对我们的影响力——我们完成工作的需要。

–在我们交谈后，她以增加固体产量作为回应，从而弥补了紧张局势，重建了业务关系中的任何裂痕。对她来说，这是一个关键、明智的举动，也是给所有阅读这篇文章的人的一个很好的教训:奖励你想看到的行为。

为什么会有风险，以及如何更好地处理它:

–我们需要完成这个项目，因此就如何快速淘汰她进行了一些讨论(毕竟，我们认为支票已经寄出)。她的搬迁可能会让她付出很多努力。

–她应该先发出警告，这样我们就可以解决问题，而不会把它推到边缘。毕竟，我们双方的诚实错误导致了这个问题。警告应该是:“我仍然没有收到付款，我很担心。如果我们不能在一周内解决这个问题，我将需要把我的时间集中在其他付费项目上。”

–我们都应该讨论付款条件，以了解对方的期望。我们预先给她钱让她开始工作，但不知道她更喜欢这样做生意。像大多数公司一样，我们的正常计费模式是净 30 天。对她来说，工作完成后的 2-4 周是个大问题。我们已经达成协议。

从两个角度来看，关于支付的有趣案例研究，你不同意吗？
