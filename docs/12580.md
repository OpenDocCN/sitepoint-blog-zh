# 使用大量的 div 标签真的有那么糟糕吗？

> 原文：<https://www.sitepoint.com/is-using-lots-of-div-tags-really-that-bad/>

我们最新的书——[，有争议的书名](https://www.sitepoint.com/books/csswrong1/)——已经引起了很多争论和更多的下意识反应，比谷仓舞会上的一桶青蛙还要多。我经常看到的一个评论是“用显示为表格的`div`替换`table`标签没什么不同；你还不如直接用桌子。”这个论点很有趣，因为听起来好像评论者在暗示不使用`table`标签的唯一原因是所涉及的标签数量。

我要大胆地说是的，即使你只是将所有的`table`、`tr`和`td`标签替换为像那些表格元素一样显示的`div`标签，这也比使用 HTML 表格进行布局要好。

没有人会因为过度使用结构标签而受到负面影响。使用 HTML 表格进行布局就不一样了。

使用嵌套表格进行布局的网页将难以打印，难以在移动设备上查看，并且对于屏幕阅读器的用户来说难以导航。一个屏幕阅读器应用程序经常会被嵌套的 HTML 表格绊倒，当它进入表格内容时会发出类似“table with 5 rows and 2 columns”的声音，当它离开时会发出“table end”的声音。使用嵌套的`div`元素的同一个网页可以被设计成整齐地堆叠在一起，以便在移动设备上打印和查看，而屏幕阅读器会忽略`div`元素。

使用 HTML 表格意味着每个单元格的内容与其他二维单元格的内容相关联:在同一行和同一列。另一方面,`div`意味着与其他`div`元素没有这种关系。简直就是内容脚手架，没有从它们的使用中衍生出任何意义。

好吧，盲目地用`div`元素替换所有与表格相关的标签可能不是解决问题的正确方法，但说它与使用 HTML 表格进行布局“一样糟糕”是错误的。 [HTML 4.01](https://www.w3.org/TR/html401/struct/global.html#edef-DIV) 将`div`定义为“通用语言/风格容器”，或者更具体地说:

> `DIV`和`SPAN`元素，与`id`和`class`属性一起，提供了一种通用的机制来为文档添加结构。这些元素将内容定义为内联(`SPAN`)或块级(`DIV`)，但没有对内容强加其他表示习惯用法。因此，作者可以结合样式表、`lang`属性等使用这些元素。，根据自己的需求和口味来定制 HTML。

所以我们可以用`div`元素来定义内容块；听起来使用嵌套的`div`元素来构建内容是完全有效的。只要你不使用`div`元素来代替更合适的元素，比如标题、段落、块引号、列表，当然还有表格数据的表格，那么你就做得很好。

在使用 HTML 表格元素创建布局时，您所采用的方法是在样式化网页之前，首先锁定 HTML 中的布局。这是错误的，而且违背了既定的最佳实践:将你的演示与内容分开。

这与 CSS 布局方法完全不同，即使使用大量的`div`元素。例如，我可以将代表网站品牌的所有内容包装在一个`div`标签中，将代表附加网站信息的所有内容包装在另一个`div`标签中。事实上，在 CSS 中，我可能会应用一种样式，将网站品牌放在网页的顶部，将附加的网站信息放在底部，但这并不重要。在布局阶段，我可能会这样做，或者我可能会使用 CSS 表格相关的显示值，使它们成为列或行。可以用多种方式设计标记的样式，因为这种方法不会将布局锁定在 HTML 标记中。
