Kevin: Hash bangs 是在 twitter.com 等网站上使用的新网址，也是在过去一周推出重大重新设计的整个 Gawker 媒体网络。这个想法是，这些网站不再使用指向实际网页的 URL，这些网站上的所有 URL 都指向主页，然后有一个哈希，你怎么称呼它，一个包含 URL 的整个剩余部分的片段标识符。因此，当浏览器获得其中一个 URL 时，它会加载您请求的网站的主页，然后该页面中的一段 JavaScript 运行，使用 Ajax 实际加载内容。

有很多关于使用这些类型的 URL 的网站如何真正破坏互联网的讨论，我不得不同意这种观点，特别是因为使用这种格式的 URL 似乎没有比更健壮的开发方法更好的好处。当 Gawker 使用这些类型的 URL 发布新网站的那一天，他们的一行 JavaScript 代码出现了一个错误，导致他们的整个网站网络瘫痪了几个小时，这个问题就凸显出来了。在发布当天的几个小时里，任何人访问他们网站上的任何页面，基本上都是一个空白页面，因为 JavaScript 坏了，所以我看到有人提出这样一个论点，如果你的开发方法导致一行 JavaScript 不仅关闭了你的主页，不仅关闭了你的整个网站，还关闭了你的整个网站网络，那么你进行 web 开发的方式可能有点问题。

已经有几个关于这个的好故事了，我会把我在蒂姆·布雷找到的最好的一个贴在这里的聊天室里，但是我期待着和我们的一位客人谈论 Hashbangs，你们有什么想法吗？
