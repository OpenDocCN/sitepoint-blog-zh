### 媒体查询

响应式设计的基础之一是**媒体查询**，它最终使网站可以被最大数量的用户访问和使用，包括那些不使用 Internet Explorer、Mozilla Firefox 和 Chrome 等流行浏览器最新版本的用户。

让我们从媒体提问开始；它们代表了一个网站设计者使他们的网站反应灵敏和移动友好的最重要的工具之一。

媒体查询由**两个主要部分**组成:

*   **@媒体屏幕** —媒体查询的第一部分是支持类型。您可能在编写自己的 CSS 样式时认识到这种语法，特别是如果您以前在为网站设计打印选项时使用过这种方法。
*   ( **min-width: 500px** ) —第二部分是查询本身。它包括要评估的函数(在这种情况下，是浏览器窗口的最小宽度)和相应的值(在这种情况下，值为 500px)，以测试规则是否已被应用。

考虑到响应式网页设计，人们倾向于只关注宽度属性，但是在设计各种屏幕尺寸时，还有很多其他属性需要考虑。还有许多与媒体查询相关联的有效属性。它们是:

1.  **宽度/高度** —通过这些属性，您可以设置显示区域的宽度和高度，包括任何滚动条。您可以使用前缀 min 和 max。
2.  **device-width/height** —这些属性设置呈现表面的宽度和高度，即设备整个屏幕的宽度和高度，而不仅仅是文档显示区域的宽度和高度。即使在这种情况下，您仍然可以使用最小和最大前缀。
3.  **方向** —设置垂直或水平方向。通过在 CSS 样式表中指定特定的规则，您将能够定义 web 页面的各种元素如何在设备上显示，在这种情况下，这取决于我们使用的设备的方向。然后我们可以指定方向的类型；横向和纵向，允许我们根据浏览器的当前方向更改您的设计布局。
4.  **纵横比** —这是一个属性，设置文档显示的宽度和高度之间的比率。即使在这种情况下，你仍然可以*使用最小和最大前缀。*
5.  **设备长宽比** —该属性监控设备的宽高比。允许使用前缀 min 和 max。
6.  **color** —这将某些 CSS 样式应用于所有颜色设备。前缀 min 和 max 是允许的，但是很少有黑白设备(除了便宜的 Kindles)仍然是流行的浏览选项
7.  **颜色索引** —描述设备支持的调色板中的颜色数量。它可以有最小和最大前缀。
8.  **单色** —该属性表示单色设备的每个像素的位数(灰度)。
9.  **分辨率—** 分辨率设置输出设备的分辨率(即像素密度)。分辨率的值可以用 DPI(每英寸点数)或 DPCM(每厘米点数)来表示。
10.  **扫描** —电视屏幕的有效属性，表示扫描类型，隔行扫描或逐行扫描。这些值可以是精确渐进的或交错的。
11.  **网格** —网格表示设备是位图类型还是“网格”等效类型。

### 断点

在简要概述了媒体查询的主要属性(通常未被充分利用)之后，让我们继续指出什么是最佳响应的**断点**以及如何为给定的项目确定它们。

就响应式 web 设计而言，断点是具有媒体查询声明的浏览器宽度，以便在浏览器处于声明的宽度范围内时更改布局。更具体地说，断点是从 0 开始的一行中的一个点，在这个点上，页面布局发生了变化(通过 CSS)。断点是根据 CSS 样式表中输入的媒体查询用数值和度量单位定义的。

一般来说，每个有响应的网站都至少有两个断点——一个针对平板电脑，一个针对移动设备。每个断点对应一个媒体查询。在下面的媒体查询中，我编写了仅当浏览器窗口的最小宽度等于特定值(可以是 320 像素、480 像素、768 像素等)时才生效的 CSS..).

以下是其中一个标准断点的示例:

```
@media only screen and (min-width : 320px) {

/* Styles */

}
```

但是，一个网页设计师需要创建多少个断点呢？这取决于网站的特点、布局和你决定采用的技术。为了把你的设计变成真正的互动和响应，我们可以这样回答前面的问题:你需要为每一个你有意选择支持的设备设置一个断点。

#### 根据设备的断点

如今，流行的做法倾向于根据以下流行设备类型的屏幕(或浏览器窗口)大小来设置断点:

*   智能手机(纵向或横向)
*   平板电脑(纵向或横向)
*   上网本
*   高分辨率或超高分辨率的台式电脑显示器

将该方案转换为像素，您可以看到智能手机和平板电脑类别所采用的“标准”尺寸主要对应于 iPhone 和 iPad 的屏幕尺寸:

*   320px —竖排模式的 iPhone
*   480 像素—横向放置的 iPhone
*   768px —纵向 iPad
*   1024px —横向放置的 iPad(以及上网本，因为这些设备的水平分辨率通常为 1024px)。

有人可能会说，并非所有的智能手机和平板电脑都是 iPhones 或 iPads，它们无疑是正确的。但是，在这种情况下，苹果设备的屏幕尺寸更多地被用作更大类别设备的参考点。此外，为每个设备创建一个断点是完全荒谬的。

一个好的做法是定义一组主断点，可能结合一些辅助断点，以使文档适合特定的设备。您还可以考虑创建一个自定义断点；这非常简单，但是需要熟悉媒体查询，所以我建议从基本的媒体查询和常见的断点开始练习。

测试动态布局非常重要，包括断点本身，以及在特定屏幕尺寸条件下执行的 CSS。如果你没有把这两个方面都做好，你的网站就会出现明显的布局和功能问题，这使得你的网站的响应能力不如你没有尝试过的任何手机友好的设计。彻底测试你的设计！
