# 设置绊网

> 原文：<https://www.sitepoint.com/setting-a-tripwire/>

入侵检测和变更管理通常与昂贵的企业系统相关联，超出了许多提供托管和开发服务的个人和小型企业的能力范围。

开源社区的另一个黄金内核已经通过他们的企业解决方案的免费 Linux 实现解决了这个问题。Tripwire 一直是许多人的最爱，因为它对文件系统有精细的变更管理控制。可以从 [Tripwire 的开源网站](http://www.tripwire.org)下载 rpm 或源代码。Tripwire 最好立即部署在新建的服务器上，但是，一旦安装在任何服务器上，您就为系统安全设置了一个基准。

该程序监视不应该改变的文件的关键属性，包括二进制签名、大小等等。此外，对于预计会更改的文件(日志文件、httpd.conf 等。)，Tripwire 配置策略允许设置监视不应更改的元素，如用户、组和权限。

可以为您的特定情况定制配置策略，并且可以为不同的目录(如/etc、/usr、/root 和/var)建立不同级别的监控，并对监控的目录或文件元素进行特殊的规则和调整。

在初始安装和配置策略时，保存您的文件系统基线的 Tripwire 数据库是通过手动运行创建的，然后程序准备监视您的文件系统。只有使用管理员密码和 Tripwire 生成的签名密钥密码，才能加密和更改策略和数据库。

一旦按照您的偏好运行，Tripwire 就可以放在 cron 中按计划运行，并根据扫描结果通过电子邮件发送报告。该程序使您能够在查看每封电子邮件后，根据您的命令授权、暂停和/或回滚扫描中发现的更改。

对于真正的偏执狂来说，Tripwire 报告可以直接从服务器发送到打印机，因为入侵者需要物理访问才能阻止报告打印，而电子邮件可以被拦截。

最后，Tripwire 可以安装在多台服务器上，用一个配置文件作为中心策略(每台主机都有自定义指令)。Tripwire 的 Linux 开源软件通过 Sourceforge 上的论坛得到支持。

此外，Tripwire 还提供了一个升级途径，可以升级到其商业网站上的商业 Tripwire for Servers for Linux 和 Unix(以及 Windows)服务器。
