# Firefox 开发小部件

> 原文:[https://www.sitepoint.com/firefox-dev-widgets/](https://www.sitepoint.com/firefox-dev-widgets/)

我非常喜欢使用 Firefox 的一个原因是不断有新的 groovy 扩展出现。花几分钟浏览扩展列表总能让我回想起小时候的那种“棒棒糖的感觉”——“嗯，..我要一个…和…一个…哦！那个！肯定是那些中的*！”。所以我想我应该写一份我目前最喜欢的一些 webdev 的简要介绍。*

首先，我不会在“ [Web Developer Toolbar](http://chrispederick.com/work/firefox/webdeveloper/) ”上写任何东西——我之前已经谈过了，对我来说，它就像你的钱包——不应该不带它就离开家。

链接检查器 0 . 1 . 2–凯文·A·弗雷塔斯

一件非常巧妙的作品。只需右键单击任何页面，向下滚动到“检查页面链接”,敬畏地看着，每个链接的背景一个接一个地变成绿色(链接正常)、黄色(被阻止)或鲑鱼色(坏链接)。浏览大页面需要一点时间，但是一眼就能发现页面中的坏链接的想法很棒。

![](../Images/edf4bd8f84b3ed8a9d8953f61a0850e5.png)

**2)[ColorZilla 0 . 8 . 2](http://www.iosart.com/firefox/colorzilla/)–Alex Sirota**

如此小，却如此强大。Colorzilla 作为一个可爱的基于 Firefox 的吸管工具发布时，它是一个赢家。从那以后，它像野草一样疯长。

单击滴管，查看光标悬停时的状态栏反馈——不仅仅是颜色信息，还有 DOM 路径信息以及上次单击的 X 和 Y 偏移。

右键单击并访问增量放大工具，颜色混合器和无数的自动复制和格式化选项。一个真正的杀手 ap IMHO。

![](../Images/36147c8b3ed56b68cbf26d42bddf0952.png)

**3) [查看格式化的信号源](https://addons.mozilla.org/extensions/moreinfo.php?id=697&application=firefox)–菲利克斯·里特**

这个是最近才添加的，但是如果你喜欢像 jEdit 和 TopStyle 这样的编辑器的代码折叠特性，你可能会喜欢这个扩展。它给你两个选择:

–查看源代码格式化模式(左图):为您提供格式良好的彩色编码源代码，每个元素旁边都有小的“减号”按钮，允许您在浏览代码时将代码部分“隐藏”起来。

–内嵌模式(右图):有点像火狐内置“视图选择源”的升级版。打开它会重新呈现当前页面，除了这一次在每个主要页面元素(div、h1、h2 等)上附加了小的“源”按钮和彩色边框。单击该按钮将仅显示用于呈现该元素的源。

从开发的角度来看，这个扩展可能更有助于帮助您理解其他人的代码，而不是原始的开发，但它确实显示了一些前景。

![](../Images/4ff7d9ae043ce07bdc49cdaf3f0d1ab7.png)

**4) [测量它 0 . 2 . 8](http://www.kevinfreitas.net/extensions/measureit/)——凯文·A·弗雷塔斯**

凯文·弗雷塔斯的另一个伟大的小扩展。这实际上是他的第一个扩展，仍然是一个低调，功能优雅的小杰作。

正如你可能从标题中猜到的，“测量它”测量。单击状态栏中的图标并拖动一个框。没有右键单击选项。没有偏好。只是精确的测量。

像 ColorZilla 一样，有很多独立的应用程序在这方面做得很好，但让工具如此容易获得的好处不能被低估。

![](../Images/053c7d85b8c4f0a62633d1620ff5e530.png)

所以，它们是最近引起我注意的四个扩展。

我错过了一块真正的宝石吗？
