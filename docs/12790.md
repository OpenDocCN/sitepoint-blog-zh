# 为什么封闭的平台会毁掉网络

> 原文:[https://www . site point . com/will-closed-platforms-ruin-the-web/](https://www.sitepoint.com/will-closed-platforms-ruin-the-web/)

在麻省理工学院*科技评论*杂志的 7/8 月刊中，作家兼哈佛大学教授 Jonathan Zittrain [提出，网络](http://www.technologyreview.com/read_article.aspx?ch=specialsections&sc=futurebiz&id=20943&a=)的未来将包括回归封闭系统。“网络的未来可能会成为过去:放弃开放的标准和服务，回归封闭的社区，”他说。根据 Zittrain 的说法，为了避免这种未来，开发者必须向网络未来平台的制造商施压，让他们“放弃在任何时候以任何理由扼杀任何应用的能力。”在这方面，像脸书、MySpace、谷歌、苹果和 Salesforce.com 这样受欢迎的新应用平台的制造商是否应该步微软的后尘？

在同一篇*技术评论*文章中，Salesforce.com 创始人兼首席执行官马克·贝尼奥夫说，网络的未来“将完全是关于开发者的授权”贝尼奥夫指出,“在云中创建和运行强大的商业应用程序的能力……将永远改变软件业的经济状况。”

事实上，web 的新平台正在引领应用程序开发和部署的新时代，促使一些人怀疑 web 是否会取代传统的计算机操作系统。然而，随着创建和部署应用程序的门槛大幅降低，开发人员最终可能会牺牲对其应用程序的真正控制。这对用户来说也可能是一件坏事。

正如 Zittrain [在五月份告诉《新闻周刊》](http://www.newsweek.com/id/135150)的那样，新平台本质上是有围墙的花园，平台创建者拥有最终控制权。他说，脸书、谷歌和苹果的 iPhone 等平台“保留权利，以防万一他们需要它来删除任何他们不喜欢的应用程序，并控制数据流”。

> 由于历史的偶然，我们最终拥有了一个全球网络，它几乎允许任何人在任何时间与任何人交流。设备可以在任何时候被他们重新编程，包括其他人写的代码，所以你不必成为一个书呆子来获得重新编程的好处。[但]这是历史的偶然。现在，我看到了一种远离这个框架的趋势——尽管感觉上并不像是远离。[举例来说，]iPhone 只能由史蒂夫·乔布斯或者很快，通过软件开发工具包，由他个人批准的程序员通过他的 iPhone 应用程序商店进行更改。或者在脸书平台上运行的异想天开的应用程序或者新的谷歌应用程序。这些都可以由他们的供应商以比尔·盖茨做梦也想不到的方式控制 Windows 应用程序。—乔纳森·齐特林

事实上，史蒂夫·乔布斯今天[证实了有争议的“kill switch”的存在，它将允许苹果公司终止任何它已经批准的 iPhone 应用程序。然而，他们控制哪些应用程序甚至通过了他们的审批程序，并可以安装在手机上(至少在没有解锁的手机上)，这一事实本身就说明了 Zittrain 的观点。](http://www.engadget.com/2008/08/11/jobs-60-million-iphone-apps-downloaded-confirms-kill-switch/)

例如，在 Windows、OS X 或 Linux 上，用户可以控制添加或删除哪些应用程序。但是有了基于网络的新型平台，平台所有者拥有完全的控制权，理论上可以随意删除任何应用程序。从许多方面来说，这是有道理的——在云计算环境中，恶意应用程序可能会对平台上运行的其他应用程序产生严重的负面影响，而如果用户在非基于 web 的平台上安装了一个糟糕的应用程序，这种影响可能不会发生。但这也造成了应用程序开发者和平台所有者之间的紧张关系，让用户感觉他们并不真正拥有他们正在使用的应用程序。

“在这个新环境中，*开发者受到平台所有者的支配*，虽然这不是世界末日，但我认为这是一个经营企业的风险更大的环境，”尼克·奥尼尔最近在[社交时报](http://www.socialtimes.com/2008/08/a-new-era-of-platforms-is-this-web-30/)的一篇帖子中写道。“虽然向云的转变正在进行并将继续向前发展，但企业和平台所有者之间的关系物流仍然需要解决。”

奥尼尔不像齐特林那么悲观，但他们都表达了一个非常合理的担忧:如果网络的未来是封闭平台，那最终对网络来说不是一件好事。Zittrain 指出，到目前为止，不仅仅是本文中提到的平台。网络连接设备的全新浪潮正在引入新的封闭平台:Xbox、Tivo 等。随着计算离开我们的计算机，我们越来越倾向于一种环境，在这种环境中，少数平台提供商是代理我们体验的最终看门人。
