# Linux 和苹果

> 原文：<https://www.sitepoint.com/linux-and-apple/>

上周，苹果在旧金山举行的全球开发者大会上推出了 Tiger，这提供了一些有力的事实，表明开源社区正在产生商业影响。

苹果公司向我简要介绍了定于 2005 年初发布的新版 OS X 的客户端和服务器版本，其中几个功能主要是由他们的开源参与推动的。众所周知，OS X 的整个苹果内核都可以通过开源软件 Darwin 获得，甚至还有一个英特尔 PC 端口。

最重要的可能是针对 Java、Linux/Unix 和 Windows 的 Rendezvous 的发布。这是一个用于网络的零配置工具，包括网络协议、设备和服务(如打印机和本地/远程服务器)的识别和配置，并且基于开源技术。

那些运行混合环境的人和使用 Java 的程序员将能够在他们的工具中构建相同的特性集。随着 web 应用程序变得越来越复杂并集成到客户的本地以及 Internet 和 extranet 网络中，Rendezvous 可能是一个强大的功能，其中多服务器发现或工作站配置是 web 应用程序设置和使用的一部分。

Tiger 还包括苹果目录服务(LDAP)、安全认证(Kerberos)、邮件服务器(Postfix)、网络服务器(Apache)和许多其他功能的改进版本，几乎都基于现有的开源软件。

然而，在一个独立但相关的记录中，当我与海军医学研究中心的实验室经理迈克尔·丘特交谈时，一些事情更让我震惊。Chute 先生领导着一个团队，负责开发能够识别环境中生物武器的制剂，并培育疫苗。

他对我说，在对他们的研究(部分基于网络)的 Linux 和 Apple 集群解决方案的评论中，他们选择 Apple 仅仅是因为易用性和在有限的 IT 环境中进行管理的能力。

这通常是网页设计师面临的挑战。我们是经常需要强大的硬件和软件组合的用户，这也是因为我们构建的 web 应用程序变得越来越复杂，但并不总是具有高级系统管理功能。

在界面设计方面，Linux 厂商可以向苹果学习，尤其是因为苹果的工程师确实是在 Unix 内核上构建的。

这里重要的是 Chute 和他的团队 1)专门研究了苹果和 Linux，两者都是开源领域的重要参与者，2)在等式中完全忽略了微软(由于许可问题，这在科学界并不令人震惊), 3)没有发现 Linux 本身的任何缺陷，只是管理界面不太合适。

对我来说，这是对开放源码的第二次有力支持，并表明最大的障碍将是简化强大的底层 Linux 引擎的客户机和服务器接口。
