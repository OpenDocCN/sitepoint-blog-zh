# 雅虎！:Web 3.0 完全是关于桌面 RIA

> 原文:[https://www . site point . com/Yahoo-we b-30-is-all-about-offline-RIAs/](https://www.sitepoint.com/yahoo-web-30-is-all-about-offline-rias/)

雅虎！开发者网络技术传道者 [Mattt Thompson](http://matttthompson.com/) 今天在 [YDN 博客](http://developer.yahoo.net/blog/archives/2008/07/gears_vs_browserplus.html)上发表了一篇长文，阐述了他对网络未来的展望以及 Web 3.0 可能会是什么样子。Thompson 说，Web 3.0 将是丰富的互联网应用程序桌面/离线访问的普及。

汤普森通过定义 Web 2.0，或者至少是使 Web 2.0 成为可能的技术，为 Web 3.0 提供了框架。根据 Thompson 的说法，不管你对 Web 2.0 定义的细节有什么看法，我们所认为的当代 Web 的大部分都是由 Ajax 实现的。

他写道:“浏览器和服务器之间的异步通信(类似 Ajax)提供了重新创建常规桌面应用程序所需的延迟。”“在这个充斥着流行词汇和强烈观点的雷区，不去争论细节，但是丰富的用户交互的出现改变了一切。它让人们重新思考网站应该是什么。这是一个更新、更闪亮的电子管系列。”

汤普森认为，其他新技术可能会对网络产生类似的实质性影响，“让 Web 2.0 不再是遥远的记忆。”

他现在领先的候选人是[Gears](http://gears.google.com/)——谷歌的开源离线同步 API 和[雅虎！BrowserPlus](http://browserplus.yahoo.com/) ，一个为网络开发者设计的类似的离线 API，采用了一种不同的、比 Google 的 Gears 更模块化的方法。

汤普森说，尽管许多博客作者和开发者将 Gears 和 BrowserPlus 描述为竞争对手，但他们并不一定如此。“考虑到各自的设计目标，将 Gears 和 BrowserPlus 视为竞争对手是没有意义的。当然，两者都有交叉的功能集，包括桌面集成和自动软件更新，但相似之处仅此而已，”他说。

汤普森说，Gears 是后端技术，而 BrowserPlus 有潜力用于前端任务。总之，他们揭示了他对 Web 3.0 的愿景:在一个互联网中，应用程序可以在在线和离线环境之间无缝移动，从 Web 移动到桌面，就像今天所有的 Ajax-bit 一样。

“如果你回想一下 Ajax 是如何改变人们与网站互动的方式的，它的大部分是完全透明的。例如，删除带有实时反馈的列表中的项目是如此普遍，以至于它成为了一种预期的行为，”汤普森写道，他说他可以想象在一个网络中，Gears 和 BrowserPlus 支持的这类事情变得“同样普遍，但对普通用户来说[仍然]不可见。”

如果 RIA 是网络的未来——甚至是计算的未来——那么离线/桌面访问是极其重要的。浏览器不是一个非常好的操作系统，因此能够在桌面上运行应用程序是向 Thompson 的 Web 3.0 和瘦客户机计算的未来愿景迈出的重要一步。
