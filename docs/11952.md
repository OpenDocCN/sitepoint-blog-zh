# Opera Unite 寻求网络去中心化

> 原文:[https://www . site point . com/opera-unite-seeks-to-declarative-the-web/](https://www.sitepoint.com/opera-unite-seeks-to-decentralize-the-web/)

Opera Software 刚刚发布了 Opera 10 的最新测试版，同时还有一个微型网站展示了新的(alpha) Opera Unite 技术，该技术旨在为基于浏览器的应用程序实现点对点网络。在一篇关于此次发布的详细博客文章中，Opera 软件产品分析师 Lawrence Eng 解释了 Opera 10 如何将任何计算机或设备变成服务器，使运行在该平台上的应用程序能够直接与另一个网络用户(可能也运行 Opera 10)通信。

有了 Opera Unite，开发者将创建应用程序——或 Opera Unite 服务——在最终用户的浏览器中运行，直接连接到其他计算机，并共享数据。Eng 概述的示例场景包括媒体共享、即时通信和“网络社交软件类”中的任何内容 Eng 公告中建议的最终目标包括恢复最终用户的电力，以使用他们自己的基础设施(个人电脑、住宅互联网连接等)相互通信。).

在一篇关于 Dev 的文章中。Opera ，Chris Mills 解释了如何使用最新的 Opera Unite build。米尔斯为 Opera Unite 提供了一个相当简洁的描述——“一种使用[Opera]内部的紧凑服务器来共享数据和服务的协作技术。”

如谷歌、雅虎！微软寻求创建集中的平台并获得关注，Opera Unite 的目标是引导网络远离由“比其他设备更强大的计算机”(服务器)运行的集中管理架构。Opera 认为，在不需要服务器，应用程序可以直接与对等系统通信的情况下，存在一系列新的机会。如果 Opera 成功，这将打击用户驱动的内容网站的核心，其中许多网站依赖用户与一群精选的朋友分享媒体来获取流量。有趣的是，Opera Unite 似乎没有试图挑战基本上集中的系统，如搜索引擎，也没有试图取代依靠向普通观众分享内容而繁荣的在线社区(YouTube，有人吗？).

Opera Unite 的快速运行是有希望的 Opera 10 beta 中的一个向导(可在工具菜单下获得)带用户完成创建帐户的过程，并有益地提供尝试配置 UPnP(看不到端口转发指令——该系统显然为尽可能少的配置进行了优化)。设置完成后，Opera Unite 侧边栏会出现，并提供一个默认服务列表。

例如，启用文件共享应用程序会将文件共享条目添加到公共 URL(通常为 http://(计算机名))上的可用服务列表中。(Opera 用户名). operaunite.com)。这个公共 URL 代表了用户的各种登录页面，Opera 认为这是从最终用户的机器上提供的，尽管公共 URL 的主机名解析为 Opera 服务器。此外，虽然任何浏览器都可以用来浏览可用内容，但实际访问内容(如下载共享文件)似乎需要 Opera Unite。

最初的版本感觉非常稳定，尽管 Opera 强调 Unite 技术仍然处于 alpha 阶段，并且毫无疑问会有复杂网络环境的挑战(NAT，无 UPnP 等)。)这种技术尚未经过实战检验。构建版本适用于 Windows、Linux 和 Mac OS X。
