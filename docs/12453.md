# 谷歌 Chrome 离开测试版

> 原文:[https://www.sitepoint.com/google-chrome-leaves-beta/](https://www.sitepoint.com/google-chrome-leaves-beta/)

在推出他们的浏览器仅仅三个月之后，谷歌就移除了 [Chrome](http://www.google.com/chrome/) 的 beta 标签。这可能是谷歌产品从名字中去掉测试版的最快速度——例如，Gmail，~~已经测试了近 3 年，~~已经测试了 4 年。“我们已经去掉了测试版的标签，因为我们的稳定性和性能目标已经达到，但我们的工作还远远没有完成，”谷歌产品管理副总裁桑德尔·皮帅和工程总监莱纳斯·厄普森今天在[博客中写道。](http://googleblog.blogspot.com/2008/12/google-chrome-beta.html)

虽然 Chrome 可能足够稳定，可以去掉测试版标签，但这款浏览器仍然远远不够完善，不足以成为 Firefox 和 Safari 等其他替代浏览器的真正竞争对手。Chrome 仍然缺乏对表单自动填充和 RSS 等基本功能的支持，也许更重要的是，它没有对 Linux 和 Mac 平台的官方支持。

Chrome 目前缺乏的另一个主要功能是扩展。不过，增加对浏览器插件的支持是当务之急，谷歌最近[发布了一份文档](https://www.sitepoint.com/once-extensions-arrive-on-chrome-firefox-should-worry/)，详细介绍了他们对 Chromium 项目的扩展计划(Chromium 是 Chrome 背后的开源项目)。我们本月早些时候预测，一旦 Chrome 增加了扩展支持，该浏览器将立即对早期用户更具吸引力，并将开始从 Firefox 吸引更多用户。

Chrome 在浏览器市场的份额仍然很小，但谷歌并不羞于在 Google.com 上放置下载链接，即使该产品仍处于测试阶段。有了这样的曝光率，加上 Chrome 现在宣布稳定，Chrome 可能会从目前约 1%的份额开始攀升(谷歌声称有 1000 万活跃用户——显然包括在南极洲)。一旦我们上面提到的一些特征出现，那就更是如此。

本周早些时候，我们注意到 Chrome 是谷歌三管齐下的网络操作系统战略的重要组成部分。与 Gears(离线数据存储)和 Native Client(网络应用的本地 CPU 资源)一起，Chrome 为谷歌提供了一个引人注目的网络应用交付平台。在这方面，Chrome 与其说是对 Firefox 和 Internet Explorer 的攻击，不如说是对 Windows 的攻击。谷歌正在推动一个计算的未来，在这个未来中，应用程序从云端交付，计算机操作系统并不重要。客户端运行的是 Windows、Mac、Linux 还是其他系统，这与谷歌的 Web OS 愿景无关。
