# 。NET 开发人员正在研究 Ruby on Rails？

经过近十年的努力。NET 开发，我觉得有必要尝试不同的领域。因为几个。我所重视的网民们很乐意做 Rails 的工作，我决定尝试一下，我真的很喜欢它。显然，这已经成为一些人的共同趋势:

“2005 年左右，我一直听到关于 Rails 的传言。我调查了一下，但最初没有得到它。我不懂 MVC(当时我是一个 WebForms 的家伙)，Ruby 和 C#完全不同。我无法想象它会这样起飞。快进 3 年后，看着越来越多的人开始使用它，我决定开始认真尝试一下。”–本

“对我来说，这是学习新事物的一种方式——探索新的社区和技术——并希望学到一些我可以带回去的东西。NET 开发(我就是这么做的)。我很快就被自己的速度所折服，我可以使用脚手架建立一个 rails 网站，并在基本的 CRUD 屏幕上运行。直到我学习了 rails，web 应用程序中的 MVC 才有意义。”–吴镇男

当你开始使用像 C#这样的静态类型语言时，你会发现自己写了越来越多的代码，这些代码除了支持类型系统之外，实际上没有任何用途。说这令人沮丧是一种委婉的说法。大量的“冗余”代码，如接口、转换、纯粹用于测试的不正当依赖注入”——Jak

“除了喧嚣之外，我尊重的那些人的意见告诉我，我需要看一看。我职业生涯的大部分时间都在创建智能客户端应用程序，对 web 框架知之甚少。我不得不提及的唯一一次网络体验是 WebForms，这真的不算数。因此，当我为一家当地慈善机构做一个小型志愿者网络项目时，我决定看看 rails。”–JB

“在我决定更彻底地了解 Ruby 和 Rails 之前，我已经听说了一段时间了。我浏览了 Ruby 语言主页上的“20 分钟学会 Ruby ”,非常喜欢。语言的简洁是真正让我兴奋的地方。”迈克尔

“在一个令人窒息的公司 IT 部门工作真的让我看到了一个‘不那么企业化’的方向。由于我迷恋软件的单元测试文化，Ruby 和 Rails 似乎是显而易见的发展方向。”–蒂姆

从 2004 年开始，我开始看到越来越多的 Ruby 作为首选语言出现在年度敏捷会议以及其他小型会议上。我会花时间和里面的人配对，真的很享受。”–科里

我当然能理解这些引语。尽管我在美国有很棒的朋友。不幸的是，这个社区对微软的项目方向没有太大的影响。另一方面，Rails 社区完全基于现实世界的需求来驱动栈的方向。

# 喜欢它什么？什么不能？

当我第一次接触 Rails 时，我对自己用最少的 Rails 或 Ruby 知识所能做的事情印象深刻。因为它严重依赖约定而不是配置，所以如果遵循约定，可以完成很多工作。对于一个既不了解语言也不了解栈的人来说，这种成就感是我非常喜欢的。我也喜欢测试工具(比如 RSpec 和 Cucumber ),我觉得有了它们，我在编写测试时会比我在。净空间。

这些是最初吸引我的东西，随着我获得更多的经验，我开始欣赏其他的东西。

在这一点上，我真的想不出我不喜欢它的任何地方。可能有一些事情我还不太明白，或者在使用另一个堆栈这么多年后，我觉得很奇怪，但这没什么关系。

当我问我的朋友们他们喜欢或不喜欢 Ruby 和 Rails 的什么时，我得到了许多重叠的答案，以及某些对他们来说很重要但对其他人来说不那么重要的事情:

## 红宝石

*“我真的爱* Ruby。它疯狂、灵活、简洁、美丽。”本*

“我喜欢红宝石的简洁。”迈克尔

## 宝石

Gems 不仅使应用程序组件化成为可能，而且如此简单，以至于每个人都这么做。通常，当你遇到问题时，有人已经解决了这个问题，并将其打包成一个宝石供你使用。”–本

“在我坐下来自己编写代码之前，我已经通过搜索有助于实现某个功能的宝石节省了很多时间。gems 的另一个重要部分是它们通常带有很好的文档和备忘单。因此，我可以安装 gem，并在短时间内让 gem 执行我需要的工作。如果缺少文档，我可以打开 gem，自己浏览代码。"–JB

## 工具

“我喜欢命令行驱动的东西。我打字很快，这符合我的工作流程。”–本

“从 Visual Studio tools 的以 GUI 为中心的开发环境切换到一个极其轻量级且主要以命令行为中心的环境，就像呼吸了一口新鲜空气。”–蒂姆

## 测试

“我喜欢测试就是你所做的——而不是可有可无的事后想法。”–Jak

“我特别喜欢 Ruby 和 Rails 社区中测试和 TDD 领域的创新。围绕不同的实践和技术有很多讨论。我真的没有在其他社区看到同样水平的讨论。”–科里

“我喜欢测试是如何融入 Ruby 和 Rails 社区的。”–蒂姆

## 简单

“很容易建立并运行一个网站，并迅速为网站创造真正的价值。更重要的是，它不会妨碍我。我没有时间或耐心去处理不断妨碍我的工具和技术。工具不应该决定我们如何工作，而应该以支持我们的方式工作。Rails 和它周围的社区就是这样做的。改变 rails 的行为方式和使用方式非常容易。这指向了我最喜欢的两件事:ruby 语言和 ruby 社区。是 ruby 语言和围绕它的社区真正赋予了 rails 强大的力量。–吴镇男

“我喜欢简单。我喜欢的是，配置之上的约定深深地根植于一切事物之中。我喜欢那些“管用”的东西。我喜欢它不妨碍我，让我编写具有商业价值的代码。”–Jak

“Rails 真的是一个不碍事的框架，它允许你以一种非常简单的方式完成工作。我喜欢 Rails 的“配置惯例”方面。它让我在构建应用程序时考虑得更少。”–蒂姆

## 不仅仅是彩虹和独角兽

有些人提到“Rails 不仅仅是彩虹和独角兽”。一个不断出现的问题是轨道内部和周围事物变化的速度。由于社区推动发展，事情往往进展得很快…

“有时候很难看出哪里出了问题，因为这里有很多传统的&魔术在进行。尤其是生活在边缘的时候。Gems 并不总是保持同步，Rails 在不断变化，所以如果你打算开发最新的&great，你必须有很大的耐心。”–本

ruby 和 rails 社区的变化速度是惊人的，社区中有一个非常大的“闪亮新玩具”综合症。这样做的好处是不断地推动技术向前发展，但是经常会导致几天的头痛，试图找出不同 gem 的不同版本之间的不兼容问题。像 Bundler 这样的工具旨在帮助解决这个问题，并且在很大程度上做到了。然而，它们也常常是问题的一部分。”–吴镇男

让其他人感到沮丧的是 rails 的升级速度。Rails 可以运行得很快，如果你有一个 Rails 2.0 应用程序，寻找基础和教程的帮助可能会很困难，因为互联网已经转移到 Rails 3 或 3.1。”–JB

*“我不喜欢的是社区移动的速度。尽管我自认为是一名优秀的开发人员，但有些东西还是需要一段时间才能掌握，有时我觉得一旦掌握了一些东西，在最新版本中就会有所改变。”迈克尔*

## Rails 的快速变化总是一件坏事吗？

*“虽然速度快，但是变化很大。当使用 rails 时，我经常会想，‘为什么我需要做 X，难道 rails 不应该为我做这些吗？’几个月后，我看到 rails 的下一个版本将包含这些特性。这是我给你的惊喜。升级经常会解决我没有的问题。引用里奇·格威斯的话，“不存在的问题的最佳非解决方案”–JB*

# 错过什么。网？什么不能？

我错过了什么吗？网？我认为 LINQ 可能是最大的事情。Ruby 中有几个 LINQ 式的特性，但这不是一回事。除此之外，我当然不想念 TFS..一点都不！！即使。NET 开发人员当然可以使用其他的源代码控制系统(有些人确实这样做了)，但大多数人只是坚持使用 TFS，因为“这是微软的方式”。

起初，我以为我会错过我为 Visual Studio 开发的可爱的小插件:CodeRush 和 ReSharper。我不得不说我并不怀念它。我在 RubyMine 中获得了我需要的大部分重构、项目导航工具和自动完成功能。

还有其他几件事我不会错过，但它会与这些人提到的内容重叠太多:

“C#仍然是一门很酷的语言，但是我不怀念它。NET 框架真的没有。”–本

“我喜欢 C#所以我怀念它。我想我应该多看看 RubyMine，但我确实想念 ReSharper。我只是需要我的重构美德融入 Vim。”–JB

“我想说的是，我真的没有想象中那么想念 intellisense 和 ReSharper。”–蒂姆

最好是一个好的 IDE 来进行 Ruby/Rails 开发，因为这似乎是最大的障碍。NET 人员来到 Ruby/Rails。缺少像 IDE 这样的 Visual Studio 似乎让一些人感到痛苦，而作为一个开发工具，Visual Studio 非常好。”迈克尔

“我绝对不怀念每天在 Visual Studio 或 Windows 中工作的日子。”–蒂姆

“我不会错过任何你必须知道的样板知识，即使你被认为是一个‘中级’开发人员。网这几天。如果你不知道 IoC、TDD、SOLID 和十几个其他原则和模式，你就没有跟上在. NET 中创建好的架构的步伐。这些模式和实践中的许多在 Ruby 和 Rails 中仍然是必要的，但是语言和框架倾向于将这些知识作为一等公民融入其中，而不是要求你事后再加入。”–吴镇男

“我不想念 TFS，不想念那些认为你应该传递数据集的建筑师，不想念那些被旧技术困住的企业，不想念那些被微软奴役的开发人员，不想念那些懒得去读一本书，不想走出自己的舒适区去学习新的、令人惊奇的东西的开发人员。”–本

“我不怀念微软技术和团队倾向于使用的一体化工具。像 TFS 和 Visual Studio 这样的工具可能有很大的商业吸引力，因为它们之间的紧密集成，但是日常工作的现实是一个不同的故事。这些工具导致了非常僵化的流程，这些工具决定了人们如何工作和互动。它们在一定程度上解决了更大的业务问题，但是通过阻止变更和阻碍团队处理不属于这些紧密集成的工具的标准指导方针的特殊情况和需求的能力，它们制造的问题比它们解决的问题多得多。”–吴镇男

“我不怀念大多数人的‘微软说……’心态。NET 开发人员。我受不了也不理解。这就好像人们害怕学习或质疑任何事情，这与我所学的一切背道而驰。人们需要看看他们目前正在做的事情之外，以了解那里还有什么，如果没有其他原因，只是为了了解他们为什么要做他们正在做的事情。”–吴镇男

*“我不怀念文化。当试图引入测试、开源工具和敏捷实践时，永无止境的对抗架构师和经理。我生活在因挫折而产生的极度绝望中。”–JB*

“我不怀念与大多数开发人员不跳出小框框思考的技术一起工作。”–蒂姆

“老实说，我真的没有想念. Net 的任何东西。”——科里

# 为什么要切换到 Ruby on Rails？

有没有这样的情况？涉足 Rails 的. NET 开发人员决定转投 Rails？是的，实际上相当多。我的情况就是这样。但这是怎么发生的呢？通常学习新事物的最好方法是尽可能多地接触它。在业余时间做了几个月的 Rails 项目后，我得到了全职工作的机会。到目前为止，我非常喜欢它。当然，这里那里都有障碍，但总的来说还是值得的。

我认为这一切都是为了尽可能地开心，同时还能得到报酬。我已经用 Clipper、than FoxPro、than .NET 玩了好几年了，现在是时候开始玩点别的了。我又一次从这些人那里听到了类似的故事:

“我发现我能够非常快速地构建应用程序，并轻松地托管它们(在 Heroku 上)。当工作中出现需要类似敏捷性的项目时，我推荐 Rails。”–本

“在一个快节奏、组织有序的社区里，有机会做一些新的、令人兴奋的事情，这正是我在 9 年多的时间里一直在寻找的。网络开发。我构建了几个小站点来玩 Rails 和学习。我玩得越多，我就越想玩，越想学，越想做一些有意义的事情。在 2010 年下半年，我得到了一个机会，不仅可以全职投入到 Rails 工作中，还可以以独立承包商/顾问的身份独立工作。”–吴镇男

*“…虽然我没有跳槽，但我刚刚在一家实力雄厚的数字代理公司获得了一个新职位。网店。我用 Rails 做我自己的东西，这是一种受欢迎的解脱。”–Jak*

*“我的 rails 时刻发生在第一天晚上。我坐下来学习 rails，几个小时后我创建了一个应用程序，并让它免费托管在 Heroku 上。我的思绪被吹走了。我不需要花时间去搞清楚数据迁移、测试框架、托管计划等等。我可以立即专注于我想专注的事情:我希望我的应用程序做什么。”–JB*

*“我一直在做。NET 开发了将近十年，我决定是时候学习一些新的东西了。最后一次飞跃是我和我的朋友想出了乐队的计划。我真的不想这么做。NET(甚至使用 ASP.NET MVC)主要是因为我做到了。他真的很想学点新东西。”–科比*

“一年半前，当我想到我的未来时，我意识到我想不出一个单一的。对我有吸引力的网络工作。我真的可以转行了，因为奥斯汀的就业市场现在好得离谱，所以那些没有 Rails 经验或经验很少的开发人员都有机会。”–蒂姆

*“2007 年，我真的受够了 C#的开发经验。我会花时间写 Ruby，然后再回到 C#，这就像是夜以继日的灵活性。自 2004 年以来，我一直在做相当繁重的测试驱动开发，感觉好像我一直在和 C#战斗。我厌倦了编译器和语言对我要做的事情，比如未实现的方法，大喊大叫。每当我有机会在 Ruby 中工作时，这种体验都要顺畅得多。所以，我决定离开那个世界。我当时正在学习 Ruby，所以我花了几个月的时间用 Rails 构建了几个应用程序。后来，有人找到我，让我在一家做 Rails 的初创公司工作。”–科里*

# 总结

一个. NET 开发人员在研究 Ruby 或 Rails 时有很多东西要学。借助 C#的动态特性，开发人员可以从纯动态语言(如 Ruby)中学到很多东西。随着 ASP.NET MVC 越来越受欢迎，可以从 Rails(它已经存在了很长时间，并且比 ASP.NET MVC 发展得更快)学到很多东西。除此之外，还有一些起源于 Ruby 或 Rails 世界的东西已经进入了。NET，如 CoffeeScript、Sass、Cucumber、Jasmine 等。

换句话说，你不需要离开。NET 来从学习一些 Ruby 和 Rails 中获益。但是取决于你问谁，你可能会听到这样的话:

*“为什么……人们试用 Rails，却再也没有回来，这是一种模式？”–本*

或者最后但同样重要的是:

*“随时给我报搬家的消息。网友对 Ruby 说:太少太晚了——又一次！”–斯科特贝尔维尔*

如果你想看的话，我已经把我从这些人那里得到的[未经编辑的完整回复](http://lassala.net/2011/08/24/what-people-who-switched-to-ruby-from-net-have-to-say)贴到了我的个人博客上。
