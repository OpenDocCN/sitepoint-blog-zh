# 审阅–微软项目标准 2002

> 原文:[https://www.sitepoint.com/project-standard-2002/](https://www.sitepoint.com/project-standard-2002/)

项目管理有助于提高小型公司(至少有几个员工)以及中型和大型企业的生产力。Microsoft Project 2002 是市场上用于项目管理的顶级产品之一，但它适合您吗？请继续阅读，寻找答案。

##### 规划项目

Microsoft Project Standard 2002 使您能够从内部角度复杂地规划项目。这是一个非常强大的程序(因为它应该是价格！)和许多工具，包括:

*   记录项目任务的能力，
*   将这些任务分配给特定的资源(人和组)
*   分配预期的任务完成日期

…仅举几个例子。

使用 Project，您还可以跟踪和创建有关项目成本、设备、花费的时间和许多其他项目相关变量的报告。

您可以创建图表并使用甘特图和 PERT 图。您甚至可以计划周期性任务。Project 的 Web 导出功能允许您轻松地将项目信息转换为 HTML 格式，以便通过 Web 上传到您自己的服务器。

##### 简化跟踪和通信

Project 最有用的功能之一是能够通过 Internet 简化和自动化您对项目资源的跟踪和交流工作。这使您可以轻松地向团队成员提供项目的最新信息。Microsoft 提供了两种关键方法来实现这一点，如用户手册中所述，如下所示:

<q>*"**Microsoft Project Server:**Microsoft Project Server 及其 Web 组件 Microsoft Project Web Access 是 Microsoft Project 的配套产品，允许您与工作组的其他成员和经理共享项目的详细信息。他们甚至可以为项目计划创建新任务，并将任务委托给其他人。作为项目经理，您可以接收任务更新和文本状态报告。*</q>

<q>*" **电子邮件:**如果您的团队协作需求很简单，您可以使用现有的 MAPI 兼容的 32 位电子邮件系统与团队成员交换基本的任务和状态信息。一旦建立了电子连接，您和您的工作组就可以使用由 Microsoft Project 生成的特殊电子邮件来分配任务、发送任务更新和提交文本状态报告。*</q>

因此，正如您所看到的，Microsoft Project 2002 具有出色的项目协作能力。

##### 学习曲线

这种规模的项目的“缺点”之一是它有一个很大的学习曲线。

这是一个庞大的应用程序，需要投入大量的时间来学习和使用它的全部潜力。在开始时，你可能会有一段艰难的时间，但你最终会学会基本的功能，你将能够利用它们。

随着您继续使用该程序，您会发现 Project 更复杂的部分，相信我，有很多这样的部分！您可能会发现购买许多可用的 Microsoft Project 2002 帮助书中的一本会有所帮助；他们可能会为你加快这个过程。

只是不要为你明天需要计划的项目购买这个程序，并期望能够使用这个软件提供的所有工具来计划那个项目。考虑到程序的规模和它的许多功能，这是不合理的。

##### 不是一个人的乐队

这个节目真的不适合一个人的乐队或非常小的企业。当然，它有你可以使用的功能…但是如果你是一个一两个人的企业，你可能会花更多的时间学习使用这个程序来计划项目，而不是你实际完成项目。

如果产品的价格还没有清楚地表明这一点，这是一款为(至少)有几名员工的企业设计的软件，这些企业可以从规划和跟踪他们的时间中受益，他们的员工需要知道他们具体要在你公司正在进行的项目中做什么。

如果你没有几个员工，你最好坐下来用微软的 Word 或者一支笔，列出你自己的项目任务、完成日期和负责人。我可能更有效率——也更便宜！

##### 简而言之…

归根结底，学习或使用 Project 根本不是一件愉快的事情，而且它的价格过高。毫无疑问，这是一个强大的程序，但是如果你是一个普通的商务人士，谁也说不准你什么时候会发现并能够使用这个产品的所有功能。

如果您真的需要项目管理软件，并且预算不是问题，那么 Microsoft Project 2002 可能是值得的。否则，很可能不是。

**评分:**3.5/5
