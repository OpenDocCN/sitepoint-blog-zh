# 这是怎么回事？

> 原文：<https://www.sitepoint.com/whats-going-on/>

最近一周，web 标准社区出现了一些非常有趣的发展，这是由 W3C 最近在圣何塞举办的关于 Web 应用程序和复合文档的研讨会引发的。该研讨会聚集了大约 50 名对网络作为平台的未来感兴趣的人，每个参与者在参加之前都必须提交一份“立场文件”。立场文件[都可以在网上获得](https://www.w3.org/2004/04/webapps-cdf-ws/papers/)，但是与这次讨论最相关的文件是 Opera 和 Mozilla 基金会提交的[联合文件](https://www.w3.org/2004/04/webapps-cdf-ws/papers/opera.html)。以下是开篇段落:

> 我们认为 Web 应用程序是一个重要的领域，现有技术还没有充分地服务于它。

> 在联合开发规范之前，单一供应商解决方案解决这一问题的威胁越来越大。为了与该领域的其他参与者竞争，具有联合开发规范的初始实现的用户代理最好在 2004 年底之前发布。

文档的其余部分继续描述他们提出的解决问题的方法，最有趣的方面是致力于保持一切向后兼容 IE 6，这肯定会在至少未来几年内保持主流地位。

这一切都很有意义。不幸的是，这个想法在研讨会上被彻底否决了。伊恩·希克森(该提案的合著者)的[事后分析](http://ln.hixie.ch/?start=1086387609&count=1)表明，问题在于出席研讨会的大量插件和服务器端技术供应商——这一比例反映了 W3C 成员的整体趋势。W3C 似乎对扩展现有 web 技术的能力并不感兴趣，而是更喜欢引入新的创新来实现未来的标准，比如 XHTML 2 和 CSS 3。

问题是 HTML 4 还活着，还在踢，急需改进。在未来的许多年里，Internet Explorer 将以现在的形式伴随着我们，无论从技术角度来看，其他浏览器有多么优越，它都不太可能失去太多优势。与此同时，XAML(微软新的基于 XML 的接口语言)的幽灵威胁着在几年内用特定平台的应用程序污染网络。不难从字里行间看出 XAML，也许还有闪存，就像上面引用的段落中提到的“单一供应商解决方案”一样。

这个泥潭的解决方案是由 Web 超文本应用技术工作组提出的。就在几天前宣布，该组织“旨在开发基于 HTML 和相关技术的规范，以简化可互操作的 Web 应用程序的部署，并打算将结果提交给一个标准组织”。该计划似乎是在一个开放的环境中开发 HTML 4 的新扩展，同时避免破坏与一直存在的 Internet Explorer 的向后兼容。这个小组不是头脑发热的网络书呆子的夜间飞行组织:目前的成员[包括 Mozilla、Opera 和 Safari 浏览器团队的主要代表，他们中的大多数人都有从事 W3C 规范工作的经验。他们正在运行一个开放的邮件列表，并且已经有了一个关于](http://whatwg.org/charter#member)[网络应用标记语言 1.0](http://whatwg.org/specs/web-apps/current-work/) 的初稿文档，这是他们的第一个建议规范。

很难低估这一切的重要性。如果你对现代网络创新的缓慢速度感到沮丧，你需要看看这些家伙。这是一个开放的过程，所以没有什么可以阻止你参与其中，并且得到了四分之三的主要浏览器制造商的支持(以及 Tantek 的认可
