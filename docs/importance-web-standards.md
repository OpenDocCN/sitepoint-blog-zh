# 网络标准的重要性

> 原文：<https://www.sitepoint.com/importance-web-standards/>

早在 1996 年初，网络不仅仅是一长串由<和>标签链接在一起的文本文件，而是时不时地显示一张图片来增添趣味。它已经离蒂姆·伯纳斯·李的设想有了很大的距离，但在实现其全部潜力之前，仍有许多事情要设想。不用说，在过去的那些日子里，并不需要我们今天所说的标准化。

浏览器对 HTML 的解释是如此宽容，以至于任何人都可以掌握基本知识并建立一个网站。极少数勇敢的人很少使用 JavaScript，只是为了执行非常基本的任务。DHTML 和 Flash 等技术无处可寻。Netscape Communicator 2.0 正处于全盛时期，几乎没有人信任微软的 Internet Explorer 浏览器。在这个早已被遗忘的过去，万维网是极客和天才的专属游乐场。

男孩…那些日子。

随着网络的发展，全球成千上万的开发者的愿景也在发展。W3C 开始提出建议，以回应一小部分领先的 Web 构建者发起的压力，他们只是希望在表达他们的作品时有更多的自由。然而，大多数真正的压力来自强大的浏览器公司，他们只是觉得没有人——包括实际建立网络的人——有权告诉他们应该或不应该怎么做。

这些公司自己承担起支持新的标记标签的责任，这些标签只能在他们特定的产品中使用。然而，这些标签中的一些是如此糟糕，以至于它们只是帮助扩大了浏览器之间不兼容性的鸿沟。这种情况一直持续到建立一个在所有浏览器的所有版本中都显示良好的网站几乎是不可能的。

HTML 4.0 和 CSS-1 是 W3C 提出的官方 Web 标准，试图结束这种混乱局面。有史以来第一次，开发人员可以通过将 HTML 文档的内容与表示布局分离来标准化代码。巧妙的变通方法和技巧虽然非常有效和聪明，但却导致了 HTML 的饱和。这种语言已经完全偏离了它最初的初衷——一种为文本文档增加结构的元语言。是时候转向更好、更有能力的工具了。

这种需求见证了有前途的 XML 和 DOM 的出现。HTML 4.0 发展成了 XHTML 1.0，DHTML 也从最初的不稳定发展到了高效可靠的状态。引入了新的浏览器版本，更全面地支持这些标准。因此，网络的未来又一次变得光明了…

当我们考虑新兴技术时，Web 标准的相关性是最明显的。在这个飞速发展的时代，网络需要指导才能发挥其全部潜力——而标准可以作为帮助实现这一潜力的完美指南。现在让我们来看看标准背后的人，考虑标准本身，并触及为什么您应该遵守它们的一些原因。

##### 谁是网络标准的幕后黑手？

每个人都听说过 Web 标准，但是很少有人真正去遵循它们。目前，五个标准已经正式实施，还有两个正在制定中。大部分来自 W3 财团。然而，有一个标准起源于另一个名为 ECMA 的组织。

W3C 开发了开放规范来增强 Web 相关产品的互操作性，努力最终实现 WWW 的标准化。特别工作组从积极参与网络开发的公司和组织那里获得了普遍的一致意见。这些被称为工作草案和建议的文档被提交给 W3C 执行成员和董事委员会，作为 W3C 的“正式”建议进行正式批准。这些建议就是 Web 标准的全部内容。它们是未来 Web 构建的基础。它们包括 HTML、XML、CSS 和 DOM。

ECMA 的主要作用是制定信息和通信技术领域的标准和技术报告。由 ECMA 提出的 Web 标准已经被全世界接受为脚本标准的基础。它被称为 ECMAScript，或 ECMA-262，是一个标准化的、更简洁的 JavaScript 版本。

##### 标准是什么？

*   **HTML(超文本标记语言)，W3C 推出的第一个标准，在 Web 上被广泛使用。到目前为止，它是设计网页最常用的工具，这就是为什么人们通常把 HTML 称为 WWW 的主干。由于 HTML 结构化元素(标签)，浏览器解释的文本文档可以根据指定的格式显示其内容。HTML 是这些年来网络变得如此流行的主要原因之一。**
***   ***【XML(可扩展标记语言)*** 就像是打了兴奋剂的 HTML。虽然它支持 HMTL 4.0 中包含的大多数固定元素集，但它允许编码人员根据预设的 DTD(文档类型定义)或个人定义的 DTD 自定义新的标记元素。XML 比 HMTL 灵活得多，并且完全面向未来。XML 是一种标准，通过它可以实现 Web 的全部潜力。*   ***【CSS(层叠样式表)*** 是一个网页开发者的梦想成真。它是一种机制，通过这种机制，只需向 HTML 或 XML 元素声明特定的样式，就可以将外观和位置的变化分配给它们。整个网站的整体外观可以用 CSS 来定义，让它在几秒钟内就可以被重塑。该标准由 W3C 提出，旨在创建一个更简单、更结构化的万维网。**

 ***   ***【DOM(文档对象模型)*** 允许脚本语言在网页上发挥其全部能力和交互能力。它是一个平台和语言中立的界面，允许程序和脚本动态地访问和更新 Web 文档的内容、结构和风格。它们可以被进一步处理，然后处理结果可以被浏览器合并回网页中。使用 DOM 作为标准将简化 DHTML 编码，并与即将到来的技术改进完全兼容。
*   ***ECMAScript(JavaScript 的标准化版本)*** 是基于网景的 JavaScript、微软的 Jscript 等多种技术的客户端、面向对象的脚本语言。ECMAScript 的主要用途是操作 Web 文档中由 DOM 指定的对象。通过它，可以操纵、移动元素，或者改变某些属性，允许 Web 开发人员实现动画文本和图形翻转等效果。目前的规范是第三版，名为 ECMA-262。

W3C 也在研究两个将在不久的将来作为标准引入的建议:它们是 XHTML 和 XSL。XHTML(可扩展超文本标记语言)是 HTML 4.0 到 XML 应用程序的重构。XSL(可扩展样式表语言)将是一种机制，通过这种机制，XML 文档将使用专门为 XML 设计的词汇表进行转换和显示。

这就是了，伙计们:这些是网络最终应该依赖的标准。其他人将不可避免地出现，有些人注定会进化，甚至可能会被完全抛弃……但是，尽管这一切都很好，但一个问题仍然存在:为什么有人应该关心呢？

##### 标准——何苦呢？

标准的使用自动地使你构建的每个页面真正地跨浏览器和跨平台。任何活跃在 Web 开发领域超过 15 分钟的人都知道，为不同的浏览器版本进行双重编码是多么耗时和复杂。标准可以帮助你避免这种乏味。

***1。实现更稳定的网络***

随着手机和手持设备等新硬件的出现，编写 100%兼容的 Web 应用程序只会变得更加困难，甚至根本不可能。遵循这些标准是实现更好、更稳定的 Web 的最有效的方法，在这样的 Web 中，我们今天只能梦想的应用程序可以变成明天的现实。

***2。减少开发和维护时间***

按照标准编码可以缩短开发和维护时间。因此，调试和故障排除变得更加容易，因为代码遵循一种模式。可访问性是 Web 标准背后的另一个非常重要的问题:它不仅意味着允许残疾人充分使用 Web，还意味着允许使用非常规浏览器的人访问任何网页。

***3。允许向后兼容和验证***

标准的编写是为了与旧版本的浏览器兼容。这就是 W3C 所说的“向后兼容性”。虽然这些浏览器可能不会像最初预期的那样显示所有内容，但它们仍然会理解 Web 文档的基本结构，并相应地显示它们。兼容代码也可以通过验证服务进行验证。这又一次使开发人员的工作变得更加容易，并且可以节省大量的生产时间。

***4。增加搜索引擎成功***

从页面排名的角度来看，遵守标准也保证了更好的结果和增加在搜索引擎中的可见性。由于符合标准的文档中存在结构化信息，因此它们的索引更加准确，因为它们可以通过机械和人工方法轻松访问和评估。兼容的代码也可以很容易地转换成其他格式，如数据库或 Word 文档。这允许对文档中的信息进行更多用途的使用。它还简化了向电视和 PDA 等新系统的迁移。

***5。现在优雅退化，将来***

另一个需要记住的重要因素是，大多数 Web 标准在设计时都考虑了向前和向后兼容性。这意味着为旧版本标准编码的网页在新版本的浏览器中仍然受支持，而为新版本标准编码的网页在旧版本的浏览器中会适当地降级以产生可接受的结果。

***6。**常识*

Web 标准提供了一套规则，每个开发人员都可以遵循、理解并熟悉这些规则。当一个开发者根据标准建立一个网站时，另一个开发者将能够很容易地从前者停止的地方重新开始，就好像他是最初设计它的人一样。

##### 包扎

自从 1994 年 W3C 成立以来，网络已经走过了漫长的道路。虽然该联盟积极致力于改善事物，但仍有许多工作要做，这取决于全世界的 Web 开发者，让更好的 Web 梦想成为现实。上面提到的原因是 Web 标准对每个人都很重要的一些最重要的原因。

谁知道有多少人会用手机上网，甚至是明年？是 10%的网民吗？20%?更均匀？谁知道呢？我们唯一可以肯定的是，一定比例的潜在客户可能会说特定的电子商务网站很糟糕，因为它们无法从特定的互联网输出设备或旧版本的浏览器访问。因此，对于那些不遵守标准的零售商来说，这相当于潜在在线销售额的一个百分比，将化为乌有。用户永远不会错。你将永远是被认为是糟糕的开发者。

现在应该很痛了，你不觉得吗？

一个有才华的 Web 开发人员将永远是一个有才华的 Web 开发人员，不管他是否需要修改他的代码。被公认为一个面向 Web 未来的天才标准应用专家，似乎比一个专门研究日益过时的技术的天才变通代码黑客更有吸引力。
