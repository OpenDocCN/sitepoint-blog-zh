# 高级网页设计:入门

> 原文：<https://www.sitepoint.com/advanced-web-design-primer/>

制作网页很容易。如果有一个好的软件包，任何初学者都可以在不到一天的时间内创建一个网站…有时甚至不到一个小时！给你多一点时间，你的普通计算机用户可能也能很好地理解 HTML，对网站的外观和设计给予更多的控制。

接下来会发生什么呢？任何一个初出茅庐的网络作者一旦将自己的网站设置为超越现代 HTML 的限制，她就会面临一大堆令人生畏的首字母缩写词(CSS、ASP、PHP 和 XML，仅举几个例子)和先进技术(如动态 HTML、JavaScript、Cold Fusion 和 MySQL)。人们怎么能理解这一切呢？

在本文中，我将尝试这样做。本周我将首先在高级 Web 技术的两个主要类别之间画一条线:客户端和服务器端技术。我会谈谈它们的不同之处，以及各自的优缺点。然后，我将浏览一下客户端技术，用简单的英语描述它们各自的功能，并提供几个链接，您可以在那里了解更多信息。最后，我将对每一种服务器端技术做同样的事情。

到本文结束时，您应该对它们是如何组合在一起的有了更好的了解。希望你能够根据任何给定的技术能为你做什么来决定下一步要学什么。

##### 服务员和顾客:客户和服务器

正如我上面提到的，先进的网页设计技术可以分为两大类:服务器端和客户端。要理解这两者之间的区别，需要对人们在互联网上浏览网页时发生的事情有一个基本的了解。

你以前做过上百次，如果不是上千次的话。您已经在 Web 浏览器的地址栏中键入了一个网址(URL ),它已经加载并显示了相应的网页。但是幕后到底发生了什么呢？从最简单的意义上来说，这个过程中涉及到两台电脑:你的电脑，你的浏览器在那里运行，以及互联网上某个地方的电脑，它提供有问题的网页。在这种安排中，你的计算机被称为**客户端**，提供网页的计算机被称为**服务器**。把服务者想象成一个忙碌的餐馆里的服务员，把客户想象成一个吵着要引起他注意的顾客。就像在现实世界中，一个服务器(或服务员)负责满足许多客户(或顾客)的请求。

在我们繁忙的餐厅里，服务员从顾客那里接受订单，然后给他们(希望)送来他们点的东西。这与 Web 上客户端和服务器计算机之间的情况惊人地相似。如您所知，客户端计算机运行一个允许它查看网页的 Web 浏览器。当提供了网址时，该软件通过互联网向运行在服务器计算机上的另一软件程序发送对该地址的**请求**。这个被称为**网络服务器**的程序通过发回与该地址对应的网页来响应请求。然后由浏览器来解释网页，将其转换成人类可读的格式，并显示在客户端计算机的屏幕上。

互联网上的任何网页的检索和显示都是按照我刚才概述的相同路线进行的；然而，事情并不总是那么简单。最近，网页设计的大部分进步都伴随着上述过程中额外步骤的成本。额外的步骤是在服务员把订单交给顾客之前还是之后，这是客户端和服务器端技术的区别。

在大多数餐馆，服务员不是负责准备食物的人；那是厨师的工作。服务员只是接受订单，并把它传递给厨师。厨师准备好订单，然后交给服务员给顾客。在某种程度上，厨师协助服务员的工作:给顾客他们要求的东西。同样，在服务器计算机上运行的网络服务器软件可以有“助手”,让它不仅仅是提供现成的饭菜——呃，网页。这些助手是用于高级网页设计的服务器端技术。

现在，当顾客最终从服务员那里拿到他点的菜时，对他来说合乎逻辑的事情就是吃，对吗？但有时候并不是那么简单。例如，考虑一下，如果顾客点了法式吐司。通常，他还会得到一小杯枫糖浆。如果顾客只是吃他的食物，在每吃几口法国吐司后喝一大口枫糖浆，他可能会看到一两个滑稽的表情。相反，顾客应该在吃之前将糖浆涂在吐司上——准备食物的一小部分任务留给了顾客。出于同样的原因，对于浏览器来说，显示一些网页比简单地将 HTML 转换成屏幕上的图片要复杂得多。有时，网页浏览器必须完成额外的任务才能显示网页。任何需要浏览器成为更积极的参与者来决定屏幕上显示什么的东西都是高级网页设计的客户端技术。

#### 客户端技术

现在让我们来看看当前可用的客户端技术。请记住，所有这些技术的一个共同点是，它们需要浏览器做一些事情，而不是阅读纯 HTML 来显示网页。

##### JavaScript(也称为 JScript、ECMAScript)

这是一种网络浏览器能够理解的编程语言。JavaScript 最初由 Netscape 开发，名为 LiveScript，是一种相对简单的编程语言，可以嵌入到网页中。像大多数设计在另一个程序中运行的简单编程语言一样，JavaScript 被称为“脚本语言”。

HTML 网页也可以包含 JavaScript 代码，一旦从网络服务器接收到网页，网络浏览器就运行该代码。JavaScript 让网页作者做一些事情，比如使页面元素响应用户动作(例如，当用户将鼠标光标移到图像上时改变图像)和用户计算机上的条件(例如，显示“晚上好！”如果电脑的系统时钟显示时间是下午 6 点之后午夜之前，则在网页上)。

JavaScript 可以做很多有趣的事情，但是由于安全和隐私的原因，它在很多方面受到限制。跨浏览器兼容性也是一个问题，因为一些浏览器中的 JavaScript 支持与其他浏览器中的 JavaScript 支持不同。由于这个原因，JavaScript 的一些只在 Internet Explorer 或 Netscape(但不是两者)上工作的特性很少被使用。

微软实际上称他们的 JavaScript 版本为“JScript ”,以解释 Netscape 和 Internet Explorer 语言之间的细微差别。这两种浏览器(以及其他浏览器，如 Opera)正在逐步使它们的实现与 ECMAScript 标准兼容，ECMAScript 标准是这些语言中独立于浏览器的定义。

JavaScript 和 Java 经常被混淆，因为它们实际上是两种不同的编程语言，适用于不同的目的。它们名字相似的原因通常是因为它们都在高级网页设计中使用；然而，语言本身实际上看起来非常相似。如果你曾经用 C、C++或 Java 编写过计算机程序，你根本不会觉得 JavaScript 很难学。即使你对 JavaScript 完全陌生，学习起来也并不困难。

##### VBScript

Internet Explorer 浏览器支持另一种在范围和用途上类似于 JavaScript 的客户端脚本语言，称为 VBScript。由于 VBScript 能做的事情很少是 JavaScript 不能做的，所以通常最好坚持使用 JavaScript，除非您有一些特殊的原因要使用 VBScript。

##### 级联样式表(CSS)

[![Get Your Copy of Kevin Yanks Book NOW!](img/b0efd5d43fb184b59453aa181c0a55de.png)](https://www.sitepoint.com/books/?bookid=kevinyank) 层叠样式表听起来比实际可怕得多。CSS 是一种语言，用来描述你希望网页的元素是什么样子的。在你用标签像`<B>`、`<I>`和`<FONT>`来做这件事之前，CSS 允许你使用 HTML 来定义网页上显示的信息的结构，然后告诉浏览器你希望这些信息如何呈现。

例如，不用在整个站点使用`<FONT FACE="arial">`来将`<P>`标签中的文本设置为 Arial 字体，您可以只使用裸露的`<P>`标签，并创建一个 CSS 文件来指示 Web 浏览器所有的`<P>`标签都应该以 Arial 字体显示。更改站点的外观变得更加容易，因为您可以在一个地方进行更改，并立即影响整个站点，并且您的 HTML 代码变得更加整洁。

同样，不同的浏览器在不同程度上支持 CSS，事实上，当盲人用软件阅读网页时，有许多 CSS 语言处理网页的声音，这些语言还没有得到任何浏览器的支持。尽管如此，已经有足够的支持来节省你输入`<font>`标签的时间，而且你可能会发现 CSS 节省你的时间比你学习它的时间还要多！

##### 动态 HTML

“动态 HTML”这个术语本身实际上并不是一种语言，它指的是在没有 Web 服务器帮助的情况下，综合使用现代 Web 浏览器的各种功能来改变 Web 页面元素并响应用户输入的实践。在大多数情况下，这意味着同时使用 HTML、JavaScript 和级联样式表。如果不是因为 Web 浏览器对这些语言的支持不同，动态 HTML 将会是一个非常强大的工具。虽然使用动态 HTML 技术可以产生许多令人兴奋的效果，但在大多数情况下，要在两种主流浏览器上都能工作需要大量的经验。

##### Java 语言(一种计算机语言，尤用于创建网站)

Java 是一种像 C++一样的全功能编程语言，但是更简单，结构更紧密。Java 程序不是直接运行在计算机的操作系统上，而是运行在“Java 虚拟机”上，Java 虚拟机本身就是运行在计算机操作系统上的程序。这意味着，理论上，任何拥有 Java 虚拟机的操作系统(现在几乎所有主流操作系统)都可以运行任何 Java 程序。因为 Java 虚拟机的生产是高度标准化的，所以各种平台之间的不兼容性是最小的。缺点是 Java 程序往往运行较慢，因为虚拟机必须转换 Java 程序指令并将其传递给运行它的操作系统。

这些和高级网页设计有什么关系？嗯，现代的网络浏览器通常嵌入了 Java 虚拟机。这允许网页设计者将小的 Java 应用程序(称为小应用程序)嵌入网页。虽然学习 Java 和编写 Java 小程序并不简单，但是 Java 小程序可以做常规程序能做的任何事情，除了它们可以在网页内的矩形区域中做。

Java 小程序的常见用途包括聊天程序和在线游戏。然而，出于安全原因，Java 小程序不能访问您电脑上的文件或其他潜在的敏感信息，也不能连接到发送它们的 Web 服务器以外的电脑。

#### 服务器端技术

用于高级 Web 设计的服务器端技术在很大程度上是用户希望 Web 站点作为不断更新的大型、动态、交互式和可定制的信息源的结果。作为一名网页设计师，当这些目标落到你身上时，你很快就会意识到你所信赖的 HTML 编辑器并不能胜任这项任务。

想想官方的[奥运会网站](http://www.Olympics.com/)，在撰写本文时，它正在提供赛事结果和澳大利亚悉尼夏季奥运会的其他报道。如果你生活在一个幸运的国家，可以获得奥运会的电视直播报道，并且已经查看了该网站，你可能会对页面上的结果如此及时而印象深刻，以至于它们经常以诸如“在中途，比赛领先者如下…”这样的标题为特色，这与你将在电视上看到的一样新鲜！

这怎么可能？IBM(负责[制作网站](http://www.olympic.ibm.com/)的公司)是否让记者在每场活动中用笔记本电脑编辑 HTML？当然不是！这不仅是一场管理噩梦，而且如果没有几个小时的滞后时间，就不可能产生在网站上可以找到的质量和可靠性的结果(想想当几个事件同时发生时更新奖牌榜页面的混乱情况吧！).

不，普通的 HTML 页面肯定不能胜任这种任务，我们上次看到的任何客户端技术也不行。Java 可能是个例外，它可以做软件程序能做的任何事情(但 IBM 也可以让人们下载一个完整的程序来查看奥运会结果)，客户端技术要么太不可靠(就浏览器支持而言)，要么不够强大。

正如我们提到的，服务器端技术是这里的解决方案。虽然基本的 Web 服务器软件只是简单地发送 HTML 文件来响应 Web 浏览器的请求，但服务器端技术扩展了 Web 服务器的能力，允许它通过运行程序、连接到数据库以及响应浏览器请求而做其他有趣的事情来动态地生成 HTML 网页。

服务器端技术的最大优势在于，它们不依赖于发出请求的 Web 浏览器的任何特殊功能。使用 JavaScript(一种客户端技术)创建的动态网页要求浏览器理解编写页面所用的 JavaScript 语言。然而，使用 PHP(一种服务器端技术)创建的动态网页可以在任何理解 HTML 的浏览器中查看，因为 PHP 代码在发送到 Web 浏览器之前在 Web 服务器上运行并转换成普通的 HTML。因此有了这个术语:服务器端。

因为只有 Web 服务器本身需要支持任何给定的用于构建站点的服务器端技术，所以在这方面有更多的选择。在本文中，我们将介绍其中最普遍的一些，以便让您对外面的情况有一个好的了解。

##### 通用网关接口(CGI)

通用网关接口(CGI)是一种标准，它允许 Web 服务器执行外部程序，并将其输出发送给请求它的 Web 浏览器。因此，一个支持 CGI 的 Web 服务器，当收到一个请求，比如说“medalstandings.exe”时，不会简单地将该文件发送给浏览器。相反，它会将文件识别为可执行程序并运行它。Web 服务器捕获输出(通常是 HTML 文档，但也可以是从 GIF 图像到 Adobe Acrobat 文档的任何内容),并响应请求将其发送到 Web 浏览器。

CGI 是创建动态 Web 应用程序的原始方法。你可以用 C/C++、Perl 或任何能在你的 Web 服务器计算机上运行的语言编写一个程序，并告诉 Web 服务器把它当作一个 CGI 程序。随着服务器端脚本语言的兴起，CGI 最近变得非常不受欢迎(我们一会儿就会看到)，因为使用 CGI，Web 服务器必须为每个请求启动一个外部程序。一个每小时有 10，000 次点击的网站将导致大量的 CGI 程序同时运行，给网络服务器计算机带来巨大的压力，并降低访问网站的速度。

##### 服务器端脚本语言

服务器端脚本语言，如 PHP、ASP 和 PerlScript，都旨在完成与 CGI 相同的角色(允许 Web 服务器运行专门的程序来响应浏览器请求),而没有为每个请求启动外部程序的负担。

通过安装一个插件，你可以“教”Web 服务器软件如何自己运行用 Perl 或 PHP 编写的程序(见下面对这些程序的描述),而不必要求操作系统将它们作为单独的程序运行。当请求包含这些语言之一的网页时，Web 服务器使用其内部插件运行页面中的代码，然后将结果发送到 Web 浏览器。

这里的区别很微妙，但非常重要。例如，如果一个 Web 服务器知道如何自己解释 Perl 代码，它就不必浪费时间和资源将代码作为一个单独的程序启动来生成动态网页。此外，Web 服务器可以做一些聪明的事情，比如意识到特定的动态生成的页面最多每 5 分钟改变一次，因此，如果它运行一些 Perl 代码来响应来自一个 Web 浏览器的请求，它将在手头保留一份结果 HTML 的副本 5 分钟，这样，如果在这段时间内收到另一个相同的请求，它就不必重新运行相同的 Perl 代码。

##### Perl(也称为:PerlScript)

Perl 是一种擅长处理文本的编程语言。因此，它非常适合动态网页的开发。这并不是说高级 Web 设计是 Perl 的唯一应用——例如，它大量用于基于 Unix 的系统上的管理任务的自动化。

Perl 程序(通常称为“脚本”)既可以作为 CGI 程序安装，也可以使用 Apache Web 服务器的 mod_perl 插件作为服务器端脚本安装(在这种情况下，它们有时被称为用“PerlScript”编写)。

个人主页，PHP: Hypertext Preprocessor，或简称 PHP，是一种比 Perl 灵活性稍差的语言，但更专用于动态网页的创建。这意味着在高级 Web 设计中，您可以使用 PHP 完成几乎所有 Perl 能做的事情，但是语法更简单(使 PHP 更容易学习)。

这种语言是作为服务器端脚本语言而设计的(而 Perl 最初并不是为在 Web 上使用而设计的)，它与其他常见语言(Perl、C/C++、Java、JavaScript)有相似之处，使现有的程序员可以轻松使用。

    ##### 服务器端 Java(servlet、JavaServer Pages 等。)

    我们已经在本文的客户端部分提到了 Java，它是一种用于创建“小程序”的语言，这些小程序使用嵌入在 Web 浏览器中的“Java 虚拟机”(JVM)在 Web 页面中运行。然而，Java 作为一种成熟的编程语言，几乎可以在任何地方运行，包括在 Web 服务器计算机上。在过去的几年里，用于 Web 服务器的插件 JVM 赋予了它们将 Java 程序作为服务器端脚本运行的能力。

    虽然这种技术在 Web 页面需要复杂和可靠功能的领域(例如，在电子商务应用程序中)发展迅速，但是服务器端的 Java 和客户端的 Java 一样难学(如果不是更难的话)。如果你是一个有经验的程序员，那么服务器端 Java 是一个非常强大的工具，可以增加你的技能；否则，您可能会希望避开 PHP 或 ASP 等更简单的解决方案。

    ##### 数据库服务器

    从根本上说，数据库是一个信息库。例如，回头看看奥运会网站，网站上出现的所有赛事结果、奖牌榜和运动员传记实际上都存储在某个数据库中。数据库不擅长以吸引人的格式(如网页)显示信息，但它们非常擅长跟踪大量相关信息。

    数据库服务器是一个程序，它让另一个程序检索存储在数据库中的信息。就像 Apache 和 OmniHTTPD 是服务于网页的 Web 服务器一样，MySQL 和 Oracle 等程序是服务于数据库的数据库服务器。

    “数据库驱动的网站”是所有实际信息都存储在数据库中的网站，网页本身是由服务器端脚本生成的，这些脚本连接到数据库服务器以获取信息并“插入”有吸引力的网页设计，然后发送到您的 Web 浏览器。这样的网站可以提供对大量信息的访问，而维护开销却少得惊人，因为如果您对存储在数据库中的信息进行了更改，那么网站上显示该信息的所有网页都会自动更新。

        如果到目前为止，你主要涉足 HTML，你可能会问自己，“客户端和服务器端哪个更好？”虽然这是一个很自然的问题，但事实是两者都不比对方更好。相反，它们各有优缺点，对于任何给定的应用程序，由您来决定哪一个是最佳选择。

        就影响用户屏幕显示内容的能力而言，JavaScript 和 DHTML 等客户端技术往往非常“灵活”。这些技术直接运行在网络浏览器内部，可以直接访问浏览器窗口和网页元素，并可以修改这些内容以响应用户的动作，如鼠标移动和点击。所有客户端技术共有的弱点是它们依赖于浏览器支持，而这种支持在不同的浏览器上很少是相同的。

        服务器端技术完全克服了这一限制，它完全在 Web 服务器上运行，因此只需要在那一台计算机上正常工作，就可以在世界上的每一个 Web 浏览器上产生相同的结果。然而，一旦页面被发送到 web 浏览器进行显示，服务器端技术就不能做任何事情来修改页面，直到 Web 浏览器请求另一个页面(通常在用户点击链接或提交表单时)。服务器端技术的设置也更加复杂，因为它们需要您修改 Web 服务器软件的配置。

        由于这些原因，大多数现代网页都是结合使用客户端和服务器端技术创建的。只要需要可靠性、跨浏览器兼容性和对数据库等资源的直接访问，就使用服务器端。当服务器端技术不够“灵活”而无法产生预期的结果时，通常可以添加一点客户端技术，以便在功能和对用户输入的响应性之间达到适当的平衡。
