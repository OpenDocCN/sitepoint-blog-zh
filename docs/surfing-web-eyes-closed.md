# 试过闭着眼睛上网吗？

> 原文：<https://www.sitepoint.com/surfing-web-eyes-closed/>

要在互联网上发布面向全球的信息，需要一种全球通用的语言，一种所有计算机都可以理解和解释的语言。万维网使用的出版语言是 HTML(超文本标记语言),使用这种语言，任何人都可以制作至少一个基本的网页。结果，越来越多的内容每天出现在万维网上，这些内容是由普通用户创建的，很少有人考虑其他人是否可以访问它们。

可访问性意味着提供灵活性以适应每个用户的需求和偏好。在互联网环境中，可访问性使得计算机技术和互联网资源对更多的人有用。我们需要互联网可访问性，因为它将网站的潜在受众扩大到数百万需要使用替代浏览技术的人。

浏览万维网是一种极大的视觉体验。按钮菜单，动画，图像都是用来设置网站的情绪，帮助我们理解作者想要达到的目的。所用的字体、文本的大小和配色方案都有助于引导我们浏览网页的复杂体验。

然而，如果你闭着眼睛尝试浏览，浏览体验的本质会有什么变化呢？这听起来可能是一个愚蠢的想法，但是坚持住，不要忽视它。事实上，去尝试一下，看看会发生什么。访问一个你以前没有去过的网站，闭上你的眼睛——现在不作弊了！好的，你现在回来了，感觉怎么样？“几乎不可能”是不是一点都不贴切？

在你第一次不用视力浏览网页后，你可能会决定不再尝试，然而，有一整部分人别无选择。许多互联网用户不能使用所有可用的资源。他们可能看不到、听不到、动不了，或者根本不能处理某些类型的信息；他们可能没有或不能使用键盘或鼠标；或者他们可能处于耳朵或手很忙的情况。

这些用户在他们的计算机上使用自适应技术来访问 WWW。例如，一个盲人可以使用语音输出系统大声朗读屏幕上显示的文本。对于盲人用户，或者使用电话或微型掌上电脑的人，如果他们变得功能性失明，听到大声朗读的材料比尝试在屏幕上观看要容易得多。使用这种技术和其他替代技术的人在使用 WWW 时会遇到困难，因为页面的内容、结构和表现方面明显缺乏分离。为这些用户设计的网站更有可能对整个互联网社区有益，因为这将产生设计更好的页面。

如果万维网要成为一个真正的全球媒体，那么就必须做出改变，让每个人都可以使用它。如今，大多数电视节目都有针对失聪观众的字幕，建筑物已经可以让轮椅使用者进入，书籍已经被录制下来，以便人们可以收听——现在是时候将这些相同的原则应用到网络上了！

如果这些还不足以成为网站无障碍的理由，那么现在有许多政府正在考虑可以应用于网络的无障碍政策。《美国残疾人法案》( ADA)要求承保实体在必要时提供适当的辅助器具和服务，以确保与残疾人的有效沟通，除非这样做会导致计划或服务的根本改变或造成不适当的负担。由于这一政策，去年全国盲人联合会对美国在线提起诉讼，认为 AOL 的软件不能与屏幕阅读器一起工作，与 ADA 直接矛盾。这场诉讼后来被撤销了，因为 AOL 已经同意提供其软件的未来版本。

在网络上有许多值得称赞的努力，使它更容易访问。微软公司有一个无障碍和残疾网站([www.microsoft.com/enable/](http://www.microsoft.com/enable/))，上面列出了与其产品兼容的辅助工具。奖励可访问网站的一个著名的验证服务是“Bobby”(【www.cast.org/bobby】T2)。应用特殊技术中心 Bobby 的制作者建议您将 Bobby 作为您网站的第一个测试，然后邀请残疾用户进行探索和评论。1997 年 4 月，万维网联合会在圣克拉拉万维网会议上发起了网络无障碍倡议。( [W3C](http://www.w3c.org/WAI/) )负责创建指导方针和教育行业。如果你愿意，你也可以访问加拿大公共服务委员会的网页无障碍自我评估测试([http://www.psc-cfp.gc.ca/eepmp-pmpee/access/welcome1.htm](http://www.psc-cfp.gc.ca/eepmp-pmpee/access/welcome1.htm))。

然而，这些只是让网络无障碍的第一步。目前可用的工具很少，现有的开发无障碍网站的指南仅限于一般的无障碍问题。

目前，我们中的许多人都可以提供帮助，无论我们是 web 开发人员、组织内的决策者还是互联网用户。我们可以确保我们的页面是可访问的，我们可以建议我们的组织考虑可访问性的问题，我们可以支持许多人正在努力工作，试图使网络成为一个对所有人都更好的地方。

正如 W3C 主席蒂姆·伯纳斯·李所说:“网络的力量在于它的普遍性。每个人，无论是否残疾，都能获得服务是一个重要方面"。
