# 社区驱动的文档

> 原文：<https://www.sitepoint.com/community-driven-documentation/>

开源语言最大的优势之一是文档中充满了用户贡献的评论、提示和代码片段。Java 社区本身是相当开放的，但是这个平台最终是由一个实体控制的——以及文档。

令人欣慰的是，我们已经从令人敬畏的用户补充资源中吸取了教训，比如 PHP 手册 和 Java 已经开始在这一领域推出自己的产品。

Lucas Chan 是 SitePoint 的开发人员之一，他发现 KickJava 很有用。它是标准 Java 平台库(J2SE、J2EE、J2ME、Java TV、Java Phone 和 Java Card)中方法、类和包的完整索引，包含用户贡献的注释、提示和更常见的代码片段。如果你是一个有经验的开发人员，只需要**看到一个运行中的类来掌握它是如何工作的，**这可能是你理想的参考。

最近， [Javalobby](http://www.javalobby.org/) (他们上周也更新了他们的主网站)推出了 JDocs.com[。在撰写本文时，这个库允许用户提交对其中包含的任何类的评论，并且整个内容是完全可搜索的。如果你登录，你甚至可以设置**你最喜欢的类、包和 API 的个性化列表**，这样你需要的信息总是一个点击就能得到。](http://www.jdocs.com/)

当第一次上线时，有很多关于的静态消息，Sun 不希望他们在网站中包含核心 Java 库。 Sun 担心夹杂在官方文档中的用户意见可能会让开发人员感到困惑，并损害平台的完整性。

令人欣慰的是，Javalobby 的智囊们通过将对这些软件包的评论放在官方文档下面的一个单独的框架中，成功地让 Sun 加入进来。这个世界一切都好。
