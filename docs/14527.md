# 与项目负责人海伦·博里一起调查火鸟

> 原文：<https://www.sitepoint.com/helen-borrie-project-lead/>

当 InterBase 源代码在 2000 年 7 月由当时的 Inprise Corp .(现在的 Borland Software Corporation)发布时，Firebird 诞生了。关于 in price/Borland 以及那些年发生的事情还有第二个完整的故事，这里就不多说了。然而，在 Sourceforge 的火鸟网站上可以找到一些有趣的历史。

我最近有机会与 Firebird 的项目负责人 Helen Borrie 交谈，她描述了这个新兴的开源数据库平台的目标、竞争优势等。

海伦现在正与出版社合作写一本书，这本书将全面介绍火鸟。还没有公布出版日期，但是，它将在今年八月在旧金山的 Linuxworld 上发布。

***SP:能提供火鸟的概况和目标吗？***

火鸟一直是开源的；它是一个成熟的商业数据库 InterBase 的分支。InterBase 是一个企业级数据库——Firebird 也是一样，但更好。您会在许多网站背后发现 Firebird，它为电信公司存储记录，并为销售点系统管理数据——几乎在任何地方都可以找到 Oracle 或 SQL Server。

项目目标是在商业应用中取代 Oracle 和 SQL Server。我们的大多数用户都在构建新系统，与商业数据库上开发的系统竞争。使用 Firebird，他们可以获得相同的功能，但是他们可以以比竞争对手低得多的价格销售他们的系统。今天运行普通业务的软件太贵了，太难用了。火鸟正在帮助改变这种状况。

SP:哪些 Linux 发行版已经针对 Firebird 进行了测试和/或认证？

基金会、IBPhoenix 和 Broadview 软件公司赞助了对 Red Hat/Fedora、Mandrake 和 SuSe 的测试。Debian 项目的成员使得 Firebird 在那里可用。任何具有最新 gcc 的 Linux 都是 Firebird 的理想选择。

火鸟是如何在 OS X 上工作的？

我们有一个在 OS X 上运行的版本，几乎从苹果发布这个系统的那天起就有了。我们缺少的是 Mac 风格的工具和有时间进行宣传的 Mac 支持者。Mac 端口没有技术问题；它没有得到太多关注，因为我们的大多数开发人员和基金会成员使用 Windows、Linux、各种 BSD 发行版和少量商业 Unix 系统。

***SP:火鸟给 Web 开发者提供了什么？***

Firebird 为严肃的 Web 应用程序开发人员提供了很多东西。如果你正在建造你计划出售的东西，我们的许可证是一个显著的优势。如果您的应用程序混合了更新、输入和检索，我们有一个在负载下运行的可靠的基于事务的引擎。Firebird 拥有全面的驱动支持——Java、JBoss、Tomcat、PHP、Python、ODBC、。NET、Borland 兼容组件、ADODB……只要你说得出，我们都会与之交流。再加上 Firebird 的小内存占用和最小的管理需求，对于严肃的应用程序构建者来说，这是一个很好的解决方案。

Firebird 将肌肉放在应用程序后面，并消除了许多与应用程序开发相关的问题。存储过程和触发器将数据处理操作从应用程序移到数据库中。在数据库中运行代码不仅减少了应用程序的大小和复杂性，而且比将所有数据传递给应用程序进行处理要快得多。

Firebird 使用记录版本控制将一个用户的工作与另一个用户的工作隔离开来，因此您永远不会因为更新阻止您访问数据而出现延迟。读取器不会阻塞写入器，因此您的交互式 Web 应用程序不会遇到基于锁的并发控制所固有的吞吐量问题。

火鸟的足迹很小。服务器不到 1.5 MB，客户机大约 350 Kb，它几乎可以在任何能够启动操作系统的机器上运行。

它有热备份，你可以从一个守护进程或等效运行，以及一个影子功能，保持一个准确的，在线的数据副本。

SP:许多开发人员一直在寻找 XML 支持，虽然大多数数据库系统可以导出或转储到 XML，但他们正在寻找原生 XML 存储。是留给开发人员使用脚本语言还是编程语言，或者这可能是 Firebird 的一个特性？你的 XML 特性是什么？

没什么特别的。我们有像存储过程和用户定义函数这样的特性，允许开发人员生成 XML 输出或解析 XML 作为输入。当然，如果需要的话，可以将完全扩展的 XML 存储在 blobs 中。您可以编写“blob filter”模块，查询引擎可以访问这些模块来将 XML 解析为其他格式。

***SP:您能描述一下可用的支持选项种类吗？***

Firebird Foundation 是一个非盈利组织，它集中了许多关于 Firebird 项目和软件社区的管理方面。它筹集资金，赞助人们从事特定的工作并组织发布。它没有商业活动。

支持列表上提供免费支持。大多数开发者和许多有经验的用户都遵循这些列表，所以那里的答案远高于免费建议的平均值。除了通用的 Firebird 支持列表之外，还有针对特定领域的单独列表:Java、.NET、Delphi、PHP、ODBC、Visual Basic 等等。

许多公司都提供商业支持。主要的英语语言公司是 IBPhoenix，其负责人包括 Borland 的 InterBase 的前总经理和 InterBase 的最初开发者之一。其他团体在许多国家提供类似的服务，例如巴西、德国、俄罗斯、法国、西班牙、意大利和日本。

***SP:你有市场渗透的想法吗？***

我们每周从 Source Forge 获得大约 2000 次下载，但是原始的下载统计数据并不是一个可靠的使用指标。我们的支持列表是一个更好的指标，因为它们代表了真实的使用情况。自去年以来，主要列表已经翻了一番，现在每天产生大约 60 条消息，大部分是在工作日。周末的流量已经下降，这表明更多的人在工作中使用 Firebird，而不是在业余时间尝试它。

火鸟在美国以外的市场尤其成功。巴西的葡萄牙语支持列表超过 3000 个。最活跃的火鸟新闻网站之一在俄罗斯，那里有我们一些最多产的开发者。我们在日本有一个活跃的用户群，他们计划在九月份召开一次会议。

Firebird 与它的竞争对手——无论是商业的还是开源的——有什么不同？

分化有很多种形式，不是吗？价格和总拥有成本是无可匹敌的。事实上，它运行在许多操作系统和最少的硬件上。此外，还有常见的竞争优势、功能和性能。

Firebird 附带了可以跨所有平台工作的命令行工具。对于那些需要 GUI 的人来说，有大量优秀的图形工具，商业的和免费的，适合各种工作。

开发的开放性对那些饱受商业 RDBMS 供应商发布的错误版本、巨额支持费用和误导性发布信息折磨的客户有着强烈的吸引力。Firebird 可以做昂贵的商业产品所做的一切，只需花一半的力气就能让它工作并保持工作:这是一种相当引人注目的差异化。

在同一个部署空间(例如 PostgreSQL)中区分 Firebird 和开源 RDBMS 要困难一些。基于历史、特性、包链接、应用程序工具等等，每一个都有它的追随者。聪明的新用户会用特定于需求的数据来尝试这两种方法。试用是免费的，不受限制。您可以安装和评估 Firebird 和 PostgreSQL 的所有特性，只要您需要:没有超时，没有负载限制，没有保密协议，没有麻烦。

你能提供一些关于火鸟未来的路线图信息吗？

官方路线图正在建设中，但我们有一个大致的工作方向。

首先，让我给你一点历史的观点。当 Borland 发布 InterBase 代码时，它是只读的，构建脚本无法工作。Firebird 1.0 的目标是构建代码，运行代码，并修复数百个最严重的已知错误。

Firebird 1.5 有一个更激进的目标:将代码从 C 转换成 C++。在这个过程中，一些特性悄然而至，但这些特性也不是 1.5 的目标。只有到了版本 2，我们才掌握了代码，并准备好接受主要的特性。对于一个开源项目来说，Firebird 对于变更和发布是非常保守的。我们为开发人员构建，并且知道人们将他们的公司和职业生涯押在我们生产的软件上。虽然我们大部分开发人员都是志愿者，但是火鸟不是爱好。

今天，有三个不同的项目正在为版本 2 做出贡献。一个是普通的 head 分支，它有新的 SQL 特性、性能增强和各种变化，允许 Firebird 支持比四年前项目开始时想象的大得多的数据库。另一个是 Yaffil，一个以前的 Firebird 分支，我们正在将其与主代码库重新整合。第三个是 Vulcan，对整个代码库进行了重大重组，恢复并现代化了原来的 InterBase 架构，该架构已经被 20 年的修补所掩盖。

SP:与 MySQL 4.0.20 维护版再次提出的许可问题相比，您的许可与 MySQL 有何不同？

我们的许可证本质上是 Mozilla Public License v.1.1，只是为了反映代码不是 Netscape，以及最初的开发人员是 in price/Borland(针对他们发布的代码)和开发新代码的个人。这些是我们仅有的许可，它们明确允许商业产品包含和发布 Firebird。没有人为 Firebird 付费，每个人都在相同的许可下使用它。

***SP:火鸟队的主力都是谁？***

核心开发者是非常重要的参与者，我们珍惜他们。随着时间的推移，团队会发生变化，新面孔会出现，有些人会暂时退居幕后。在目前的阵容中，核心团队听从 Dmitry Yemanov，他在不同的开发方法之间进行仲裁。阿诺·布林克曼和尼古拉·萨莫法托夫是天才的设计师和程序员。亚历克斯·佩什科夫专门负责安全问题。Claudio Valderrama 仔细检查输入的代码。Pavel Cisar 正在开发新的试验台。许多其他人协助代码开发、清理和测试开发。

第二个是 Jim Starkey 为 IBPhoenix 的一个客户进行的一次重大架构重组。Jim 设计并编写了 Firebird 所基于的原始 InterBase 代码。客户希望 Jim 的工作——目前标记为“火神项目”——是开源的，为更广泛的 Firebird 代码库做出贡献。

第三个是 Firebird 1.0 Windows 服务器的增强实现，称为“Yaffil”。Yaffil 的所有者去年 11 月将源代码交给了 Firebird 项目。它的主要开发者 Oleg Loa 加入了 Firebird，负责监督 Yaffil 增强与 Firebird 2 代码的集成。

驱动开发项目也是火鸟产品的主要分支。其中包括 Jay bird JDBC 2 型和 4 型 Java 驱动程序(团队负责人:Roman Rokytskyy)、Firebird.NET provider 和 ADODB 驱动程序(Carlos Guzman Alvarez)以及 Firebird ODBC/JDBC 驱动程序(原作者:Jim Starkey，团队负责人 Vladimir Tsvigun)。
