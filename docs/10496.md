# 你坚持严格的 BBC 风格的 CSS 编码标准吗？

> 原文：<https://www.sitepoint.com/bbc-css-coding-standards/>

BBC 网站是网络上最受欢迎的目的地之一。它的网页排名为 9，以高质量的报道和资源而闻名。

在内部，BBC 也有一些为网络开发者所知的最严格的指导方针。不仅仅是浏览器的问题——技术发展的每个方面都有一套公开的规则。

我一直在研究最近更新的 CSS 标准。它们都是最佳实践技巧，但它们比许多公司政策更进一步！

## 一般 CSS 原则

BBC 对内容使用 XHTML 1.0 strict，指导方针规定内容必须在不启用 CSS 或 JavaScript 的情况下可读。这一点很重要，但是很多开发人员在匆忙添加最新的 jQuery 小部件时经常会忽略这一点。

大多数浏览器(或扩展如 [Web Developer Toolbar](https://addons.mozilla.org/en-US/firefox/addon/60/) )允许你禁用 CSS 和脚本。如果内容不可读，你可以保证谷歌和屏幕阅读器也看不到它。

根据指南，所有的 CSS 都必须符合 W3C 发布的推荐标准。我怀疑这导致了一些令人头痛的问题，因为它似乎排除了供应商前缀的属性，如`-moz-border-radius`和`-webkit-box-shadow`。然而，BBC 网站很少使用它们。

！重要是被禁止的，因为它覆盖了用户样式。这有点苛刻，因为它对 IE6 修复很有用。也就是说，当我应该解决级联问题的根源时，我滥用它进行快速和肮脏的攻击是有罪的。

最后，如果你已经放弃了 IE6，想想仍在测试 IE5.5 的 BBC 编码员吧！CSS 在那个浏览器中尤其令人讨厌:它试图解析属性，但是失败得很惨。

## 实现 CSS

所有的 CSS 都在压缩的外部样式表中实现，尽管当特定页面需要规则时，HTML `head`中的 CSS 是允许的。

禁止内嵌样式。那很好。任何被发现使用它们的程序员都应该被吊销网络开发许可证！

有趣的是，外部 CSS 文件不能用`@import`加载，因为它会损害浏览器缓存。是吗？我怀疑在现代浏览器中是否仍然如此。

## 字体和颜色

必须将衬线字体、无衬线字体、草书字体或等宽字体的单一通用字体名称添加到所有`font-family`属性的末尾。同样，这是开发人员经常忘记的事情:不是每个人的电脑上都有 Arial 或 Helvetica。

我喜欢双重否定原则:

> 除了在打印样式表中，在所有浏览器(如 px 和 pt)中，不得以不可调整大小的单位来指定排版大小。

BBC 建议在任何一级浏览器中，当大小增加两级时，em、%或关键字值和文本必须保持可读。我打赌那是一场测试噩梦！

最后，开发人员必须定义页面背景颜色。我在电脑上浏览了我的一个默认设置为病态绿色的网站后，就把这条规则纹在了我的额头上。

## 开发商天堂还是地狱？

如果你已经在这个行业工作了一段时间，你可能已经吸收了 BBC 制定的许多指导方针。但是新开发人员如何应对呢？从积极的一面来看，期望被很好地记录下来，这有助于编码人员避免基本的可用性错误。然而，开发是相当困难的——大多数人都要努力应对为 24 种相互关联的技术定义的多种规则和条例。

你的公司执行严格的编码指南吗？它们是一成不变的还是相当灵活的？他们是理智的还是可笑的？它们是定期更新还是你仍在为网景 3.0 编码？指南对你的日常开发任务有帮助还是有阻碍？

我也有兴趣听听 BBC 的任何开发者的意见…你是严格遵守规则还是偷偷摸摸的！当你的经理不在的时候很重要？
