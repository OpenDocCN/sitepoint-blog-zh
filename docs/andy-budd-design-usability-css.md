# 安迪·巴德谈可用性、设计和 CSS 之死

> 原文：<https://www.sitepoint.com/andy-budd-design-usability-css/>

安迪·巴德作为主题发言人出席了本周在墨尔本举行的 UX T2 网络方向大会。SitePoint 的 Matthew Magain 花了一些时间和他谈论博客、设计、CSS 框架、网络职业、可用性测试、网络的未来等等。

感谢你花时间和我们交流，安迪！首先，我想说你的博客，连同德鲁·麦克利兰的 [All In The Head](http://allinthehead.com) ，是我多年前开始关注的第一批博客之一。你能告诉我你的博客是如何塑造你的网络职业生涯的吗？

哦哇，酷。很难说它在多大程度上产生了影响，因为显然已经发生了一大堆事情。

但我确实认为这很重要。我很早就开始在英国写关于网络标准的博客，当时我只知道另外两个人也在英国写同样的东西，他们是[杰瑞米·基思](http://adactio.com)和[理查德·鲁特](http://clagnut.com)，我后来和他们一起创办了我的公司 [Clearleft](http://clearleft.com) 。过了一段时间，我发现了一些其他的博客，像德鲁的博客，其他一些人也开始做类似的事情。

这是一个相当早期的阶段，所以我的其他同行是像道格·鲍曼、T2、戴夫·谢伊、T4、卡梅隆·摩尔和其他人。所以我想我和那些人混在了一起，这肯定会带来荣誉。里奇、杰里米和我是在英国做类似事情的人。

但我也认为这很大程度上是运气——在正确的时间出现在正确的地方。我对网络标准非常感兴趣，我开始写博客是为了帮助理清我自己的思考过程。当我发现一些事情时，我会把它贴在我的博客上，而不是藏在心里，因为我认为如果它对我有帮助，可能会有其他人帮助我。

我想和大多数博客作者一样，这都是出于相当无私的原因。至少在早期，我认为大多数人写博客是出于相当利他的原因。这些天你会看到很多人通过写博客来获得名誉和财富，坦率地说，我认为如果你想获得名誉或财富，你应该学习弹吉他或学习表演。我不认为你会因为写博客而出名。

但是，是的，我想我是通过我的博客变得相当出名的。在某个阶段，它获得了巨大的流量，我想在某种程度上，去会议上发言的能力就是来自于此。不直接，但是当我接近 SXSW，例如，看我是否能说话，我猜测我是一个已知的实体的事实帮助了那。虽然 SXSW 一直相当擅长破新天赋；他们总是很擅长让以前没说过话的人开口说话。

我想当我为我的书《掌握 CSS》写提案时，我有一些演讲经验和博客的事实向我的出版商证实了至少有某种市场。作为出版商，你应该知道，在没有明确市场的情况下，让一个完全不认识的人写一本书有点冒险。然而，至少阅读我博客的人更有可能购买 CSS Mastery。

所以我想这是一种互相依赖的关系。这些天我说得越多，我不得不承认，我的博客真的被忽视了。如果我一个月能写一两篇博客，我就很幸运了。我感觉很糟糕，因为我真的很想写东西，有很多东西我想写。

但是运行 Clearleft，组织 [dConstruct](http://dconstruct.org) ，做所有这些发言意味着我的可用时间相当紧张。我觉得很悲哀的是，我之前提到的许多同事——戴夫·谢伊、道格·鲍曼、丹·塞德霍尔姆——也发现自己处于类似的情况，在过去的 18 个月里，他们写博客的数量真的有点减少，因为我们都更加努力地工作，做了很多事情。

你认为这部分可能是关于 CSS 的一切都已经说过了吗？

哦，当然，是的。我已经有好几年没有在我的博客上谈论 CSS 了，我关注的更多的是可用性和用户体验设计。我的意思是，CSS 一直只是我感兴趣的一个狭窄的部分，但它是我当时正在学习的部分，所以自然是我写博客的部分。我认为你是对的——我认为 CSS 在一定程度上已经走到了尽头——至少像你的 Doug Bowmans 和 Dave Sheas 这样的人已经发现了我们可能会发现的所有有趣的东西。

所以现在写关于 CSS 的博客的人是刚刚了解它，并且发现它的人。这种东西总是需要的——总是有人在学习，所以总是需要向你的同龄人或处于类似情况的人学习。但对我来说，在某种程度上，CSS 的事情就像是一个已经完成的交易。

你之前提到过你的商业伙伴，杰里米和理查德，他们在网络上有自己的身份。在左派的保护伞下，自我的冲突是如何发生的？

实际上很好。我们聚在一起共同开发 Clearleft 的原因是因为我们对网络应该是什么样子有着相当相似的信念。我们都相信语义标记和网络标准…我们相信好的设计、可用性和用户体验。所以我们都有相似的重叠利益。然而，就个人而言，我们有自己热爱的事情。

所以里奇对好的排版和易访问性有着惊人的热情，他是一个非常棒的交互设计师……非常优秀。他完成了工作——他就像是 Clearleft 的骨干，真的。他是做所有工作的人，杰里米和我得到所有的荣誉！

我对用户体验设计非常感兴趣和热情，我也非常擅长商业方面的东西，而杰里米和里奇对此并不感兴趣。所以我喜欢与人交谈，我喜欢去见客户，解释我们做什么，并对网络提供的所有伟大的东西充满热情。

杰里米非常聪明。我是说，他真的很能干。他知道他的代码和 JavaScript，但他也知道他的经典参考和类似的东西。他上过艺术学院，所以他有那种背景。另外他是个很棒的演讲者。

所以它真的运作得很好。显然，我们对某些事情有自己不同的看法，但总的来说，从哲学角度来说，我们相处得很好。

那就不会有太多的冲突了？

天啊，不，一点也不。我的意思是，有一些小的“捕捉点”总能引发讨论，其中之一就是“固定与流动”的讨论。围绕这一点总是有不同的意见。但那是因为它是那种“你能让多少骆驼穿过针眼？”类型情况。没有正确的答案，只有观点。一般来说，这是那些反复出现的经典讨论。

SP:我们来谈谈可用性测试。为什么这么重要？唱反调，难道不是剥夺了成为一名优秀设计师的专业技能吗？

嗯。如果你相信“天才设计”的概念，即人们天生就能创造出令人惊叹的东西——而且确实有天才设计师，比如史蒂夫·乔布斯，那么你可能会说可用性可能会扼杀天才设计。

但是对于 99%的设计师来说，他们不是伽利略或者达芬奇式的天才设计师，设计是一个非常有用的设计工具。我认为可用性测试的一个问题是，大多数人对它的体验是由大机构在项目结束时做的总结性测试。他们在上面加上“测试”这个词，这让我们设计师感觉我们的工作正在被测试和评分，如果你没有通过测试，那么你就是糟糕的设计师。

这很糟糕，因为这让我们设计师觉得我们生活在这种“复选框”的心态中。我喜欢的那种可用性测试叫做形成性测试，它把可用性测试作为一种设计工具。你知道，我们不能整天坐在我们孤立的小盒子里盯着 Photoshop，孤立地创造体验，并相信我们创造的体验将是可用的和完美的。因为经验表明，一旦你把某样东西放在外面，人们总是会以你意想不到的方式使用设备。人们会以你意想不到的方式发现挫折。

我认为一个大问题是，客户和雇主觉得你雇了一个设计师，他们应该是天才，他们应该第一次就把事情做好。坦率地说，如果你雇佣像 Clearleft 或 Adaptive Path 这样的公司，第一次迭代将会非常好——这将比你雇佣不了解可用性和用户体验的人好得多。但是你从来没有第一次就做对过。优秀的设计师明白设计是关于过程的，可用性是教育设计的一个非常重要的工具。所以我确实认为可用性是一种设计工具。你坐下来，作为一个好的设计师，你看到客户或用户使用你设计的产品。当你看到他们在一个特定的情况下挣扎，或者他们不明白一些事情，你就有了这些“灵光一现”的时刻，它会通知你的设计决策。这样你就可以回去，不断重复，做得越来越好。你花在可用性测试上的时间越多，观察用户使用你的产品的时间越多，你就越能成为一名优秀的设计师。我确实认为，认为你是一个天才设计师而不必担心用户如何使用你的东西是一种傲慢。

所以我认为可用性是一种设计工具。但是，我认为设计就是大写 D 的设计——我认为设计意味着设计系统、架构、用户流、信息、视觉界面、模板……它不仅仅是事物的图形方面。你必须把这些都做好，所有这些都是元设计。

你网站上的简历提到了 Flash。你使用 Flash 的经历对你现在使用 HTML、CSS 和 JavaScript 进行设计有什么影响？

哦，是吗？哎呀！是的，非常令人惊讶的是，我曾经是一个相当铁杆的 Flash 迷。与其说我是一名设计师，不如说我是一名铁杆动作脚本程序员。早在 2000 年，我就在那里，用 ActionScript 4 开发面向对象的 Flash 游戏。但我开始对 Flash 世界感到有点失望，因为当时人们正在构建的体验不是很有用，也不是很容易获得，我发现了这个勇敢的 web 标准新世界，它对我来说意义重大。所以我慢慢开始离开 Flash，进入 web 标准的世界，我想这是一种奇怪的转变——我不知道我的标准同事中有谁这样做过。

但另一方面，奇怪的是，它开始回到原点。我现在根本不做任何 Flash 的东西，但是如果你看看 Flex，你看看 ActionScript …基本上 ActionScript 是 ECMAScript，这是一个标准。Flash 使用 MXML 作为声明性标记语言，所以我的朋友们，比如做 Flex 开发的 [Aral Balkan](http://flashant.org) ，正在用一种标记语言编写 Flex，使用 CSS，非常类似于 HTML。

因此，了解基于标准的方法如何被 Flash 和 Flex 世界采用是一件有趣的事情。

我提到它的原因是因为我想知道你是否错过了许多 Flash 设计者在基于标记的解决方案上的自由？

我认为设计就是约束。有些人喜欢空白的画布，但我一点也不喜欢。我认为那真的是艺术家的事，不是设计师的事。我认为设计是关于约束的——商业约束，技术约束。正是这些限制创造了创新。

如果没有这些限制，我们就不会有所有这些很酷的 CSS 效果，或者所有这些很酷的 JavaScript 库。所以不，我觉得没那么大问题。然而，我对用户体验设计越来越感兴趣，我确实看到了 HTML、CSS 和 JavaScript 的局限性。是的，我开始窥视 Flash 开发人员现在正在构建的一些体验，并认为拥有一些灵活性是很好的——但具有可访问性和标记优势，以及 HTML 的开放性。

所以我再也不会做这种事了——我已经忘记了闪光灯这件事。但这很有趣，是的。

前几天你向世界展示了银背鹦鹉 T2。请给我们介绍一下这个应用程序。

是啊，当然。我一直在进行可用性研讨会的小小世界之旅，部分是因为我认为可用性测试是前进的方向。Silverback 的整个概念是作为一种方式，让设计师和开发人员进行真正低成本的游击式可用性测试。

市面上有一大堆不同的软件解决方案，价格相当昂贵，大约在 1，500 美元左右，它们很好，但功能太多，很难使用。所以我们决定创建 Silverback，它主要捕捉屏幕活动，鼠标点击，用户输入的东西，音频和视频，作为一种非常便宜的低成本可用性测试的方式。

SP:这个应用的营销很有趣，不是吗？你建立了一个注册页面，你之前告诉我你不打算推广它，但在我看来这是一个完美的病毒式宣传。

我想是的。虽然这真的是偶然的营销。事情是这样的，我们买下了这个域名，(链接:silverbackapp.com)，它刚刚被解析为一个域名公司，我们认为人们可能会开始听说它——我们的朋友或其他人。所以我请我们的设计师保罗贴一张展示页，我们有了来自乔恩·希克斯的漂亮的大猩猩设计。保罗是一个很棒的设计师，他创造了这个他一直在玩的漂亮的小视差效果。

我下午去了伦敦，没想到会发生什么，我要去见一个客户。他只是在推特上发布了这条消息——他只是发布了“伙计们，我已经把这个页面放上去了。”突然间，我们被这些推特信息和电子邮件淹没了，“哇，太酷了，银背是什么？”

人们因为这个标志而变得兴奋，人们喜欢这个页面，人们对这个应用程序非常感兴趣和好奇。这只是一个页面，我们不想透露太多信息。上面只写着“银背”。即将推出。”这似乎引起了很大的反响，第一天我们就有大约 2000 人在我们的公告列表上注册。到周末，我们有 5000 人注册，我不知道现在有多少人注册了。

所以对于一个小公司来说，做一个小小的桌面应用程序，这是非常了不起的。但是现在我们的一个担心是，因为我们对这个应用有点神秘，并且有一些原因，我马上会谈到，人们一直在想象可能发生的最好的事情。我想，因为我们是 Clearleft，我们相当出名，我和 Jeremy 和 Rich 一起工作，我们有这么多聪明的人和我们一起工作，突然人们开始认为这是某种应用程序，将解决世界饥饿，你知道，将使世界成为一个和平的地方。

事实上，这是一个非常简单的应用程序，它源于我们意识到这些天我们手中已经有了一个便携式可用性实验室——无论是 iMac 还是 MacBook Pro 或其他什么。如果它有屏幕，有麦克风，有内置摄像头，那么你就不需要这些大的听力实验室。不需要架个三脚架对着屏幕。您可以在一台设备中完成所有工作。

过去确实有这些便携式可用性实验室——事实上，你仍然可以得到它们。它们是这些装有摄像设备的盒子等等。你不需要那个——你已经得到了一切。但这个想法非常简单——屏幕捕捉、音频捕捉和视频捕捉——我们担心，如果我们透露了太多信息，已经有一个屏幕播放应用程序或正在做类似事情的人可能会快速轻松地添加功能。所以，在我们开发出它之前，我们不想大声嚷嚷。

此外，我们以前从未真正开发过桌面应用程序，我们不知道这要花多长时间——这比我们想象的要长。这可能比我们理想的预期要多花几个月的时间。修复和调整 bug，调整 UI。另外，如果你开发一个应用程序，你最终会沉迷于它，我们也会沉迷于大多数客户的网站。但如果是你的孩子，你要确保 100%正确。因此，我们没有谈论它或看到它是什么的事实更加助长了这种猜测。这就是神秘感的一部分——每个人都开始谈论“哦，这是什么银背龙？”人们开始写博客，我在会议上遇到一些人，他们会说“你能告诉我吗？”

所以是的，这种神秘感确实让它变得很流行，所以它确实帮助了我们。我不会说这纯粹是偶然的，因为正如你今天看到的，我们确实喜欢将这些小东西构建到我们的应用程序中，而且我们确实有一个很好的朋友网络，所以如果我们在 twitter 上发布一些东西，人们会发现的。但它肯定不是一个有意识的事情来保持它的秘密，以便它将燃料的欲望…它就像那样发生了。

你对 CSS 框架有什么看法，它们如何适应你们为客户开发解决方案的方法？

我们有一个用于线框的内部框架——我们的原型。线框不是独立存在的东西，所以它们的语义或纯粹性并不重要。

这个库，或者说“框架”,是 HTML、CSS 和 JavaScript 的混合体，我们一次又一次地用它来构建线框。它利用了 jQuery，我们合作的一家名为 [New Bamboo](http://new-bamboo.co.uk) 的公司帮助我们开发了一个名为 polyPage 的 jQuery 插件，它允许在线框内改变多个状态，这真的很酷。

所以我认为这些库——坦率地说，我认为“框架”对于像[蓝图](http://code.google.com/p/blueprintcss/)这样的东西来说是一个有点浮夸的术语。本质上，它是三到四个 CSS 文件和一页 HTML。这不是一个框架——我认为框架是一个更大的东西。那里有某种吹喇叭的元素在进行。通过称之为框架，它听起来就像是 Ruby on Rails 一样重要。大家都知道 Ruby on Rails 很火，大家都知道 Django 很火——姑且称之为 CSS 框架，它会很大。

我认为你需要做的是开发你自己的框架。你需要开发一系列工具，一个为你的团队工作的元素库，为你建立的网站类型工作，为你的团队拥有的内部信仰系统结构和知识工作。

我认为像 Blueprint 这样的通用库将你束缚在某些解决方案中。当然，他们有一定的灵活性。您可以选择任意数量的列中的一个—我想是 16 或 24 —并且您可以跨列等等。但是它仍然局限于这个列的行为。你受限于水槽的大小和窗户的宽度，如果这不一定符合你的设计，你会怎么做？你会花几个小时来修改你的图书馆以适应你的设计吗？

大概不会。我想很多人所做的是让他们的设计适合库，你最终会得到这种模板化的网页设计方法，就像那些真正专注于特定 CMS 的人一样——他们所有的网站都是围绕该 CMS 的功能设计的，而不是创建适合该解决方案的用户体验。

具有讽刺意味的是，我曾经和这些图书馆背后的一些人交谈过。在现实世界中，他们最初使用的是一个库，但它不起作用，他们不得不停止使用它并从头开始重建。所以库并不总是最快捷的，因为你可能使用了错误的工具。

杰里米有一句话，我认为是一句非常明智的话。他说，“如果你有一把锤子，所有东西看起来都像钉子。”我认为这是预建工具的问题。如果您已经有了一个设计，您可以尝试将所有内容都放入库中。

实际上，我认为 Blueprint 有一些基本问题，作为一家公司，我们并不特别喜欢。其中一个事实是，它的核心不是很语义化。它有表示元素，例如，它有名为“span-1”、“span-2”、“span-3”的类…它基本上重新创建了一个基于表格的布局。我的感觉是，当您可以使用一个表格时，为什么要添加 200 行 CSS 代码来重新创建一个基于表格的布局呢？如果你不在乎语义标记，并且愿意使用污染 HTML 语义本质的库，为什么不使用表呢？因为它已经构建好了，所有的代码都已经存在了。为什么要尝试重新发明轮子？

现在，我个人认为用于布局的表格不好，但我也认为非语义标记不好。

然而，人们会争辩说它并不是真正的非语义的。我认为，很明显，当你的 HTML 结构中的类名叫做“span-1”、“span-2”、“left”、“right”之类的时候，这就有点落后了。

当 IE8 在今年晚些时候推出时，它将支持 CSS 表格，并将是最后一个支持 CSS 表格的大型浏览器。你认为在隧道的尽头有一盏灯，有一天 web 开发人员将不必担心在使用库或表格进行布局之间进行选择？还是总会有遗留系统需要适应？

随着 HTML 5 的诞生，这是 web 标准世界中的一个大问题。关于存在大量遗留系统的事实，存在着很大的争议。有许多系统是在两年、三年、四年、五年或更久以前构建的，而站点管理员却无法重建它们。还有 10 年前的 CMS 系统还在使用旧式的标记。

所以当考虑开发 HTML 5 规范时，工作组想把字体标签放回去。他们的观点是，“好吧，已经有很多人在使用它了，我们的浏览器需要理解如何处理字体标签，所以我们需要把它融入到规范中。”我把它放入向后兼容性问题的规范中的问题是，你正在使它的使用合法化，并且通过使它的使用合法化，你正在破坏内容和表示分离的整个想法，不管它在某些情况下可能多么不切实际，我认为这是一个值得提倡和追求的好范例。

但另一方面，我理解微软的观点和他们引入的 meta 标签。当你把你的浏览器从 IE7 升级到 IE8 时，如果一个网站崩溃了，人们不会责怪网络开发者，他们会责怪浏览器制造商——即使浏览器做了正确的工作，他们也是受影响的人。

不幸的是，当他们从 IE6 迁移到 IE 7 时，IE7 要好得多，也更符合标准，标准世界都认为这一迁移是惊人的。但是他们从销售人员和客户那里收到了很多负面的公关和负面的电话，因为那次更新实际上破坏了现有的网站。所以向后兼容性是一件大事。但是我确实认为恢复字体标签的想法是疯狂和多余的。

所以回到你的问题，我们总是会剩下多余的页面和多余的标记。这是浏览器制造商不得不面对的一个大问题。浏览器制造商可能不支持 IE4，操作系统制造商可能不支持 Windows 95，在很长一段时间内，我们将不得不处理非常陈旧的标记。在这些页面中有重要的信息——有 10 年前的 Tripod 页面和 GeoCities 页面，它们有重要的信息，但没有 web 开发人员或设计人员或其他人仍然在照看它们，但它们仍然是历史数据。说到数字保存，在某个阶段，浏览器将无法反向渲染。

但另一方面，网络标准的全部目的，部分是为了数字保存——如果每个人都以正确的方式创建网站，我们首先就不会有这个问题。一厢情愿的想法，也许。

老实说，我认为 CSS 在下一次修订后基本上是一种死技术。我认为这真的很可悲，因为我是一个标准倡导者，但这是我所相信的。我不认为我们会看到一个 CSS4。我不认为我们会看到 CSS3 中一半的模块被实现，甚至最终出现在规范中，因为 CSS3 已经开发了 8 年之久，而在这 8 年中，我们还没有一个完整的模块。我们有几个快完成了。

但我确实认为，在这个世界上，我家里有一台 PS3，可以实时进行 3D 纹理映射渲染，为什么我们不能在 2008 年获得一个可以在盒子上渲染圆角的浏览器。你知道，我们都应该有喷气背包！圆角框或多个背景图片不应该是火箭科学。

我认为问题的一部分是标准开发机制存在固有的问题，我认为浏览器供应商也存在固有的问题，我确实认为我们将会看到其他技术——可悲的是，像 Flash 和 Flex、XAML、XML 和 MXML 这样的专有技术以及非本地 Web 技术——取而代之，因为我们对现有技术的能力有限。我们看到所有这些 JavaScript 库出现的原因是因为它们补充了标准和浏览器应该做的事情。这是一种愚蠢的情况，我们必须游说 W3C 让他们批准这些标准，我们必须游说浏览器供应商停止在琐碎的小事上相互争论并实施它们，并且需要彻底改变这些东西的创建方式。

**SP:[网络标准奖](http://webstandardsawards.com)是你和一群人不久前建立的一个网站，定期展示以符合标准的方式结合功能和形式的好例子。你认为这个网站会重新展示用一堆新技术建立的网站吗？**

哇，你记性真好！是的，这是我和卡梅伦·亚当斯(当时我并不认识他)以及一个来自瑞典的名叫约翰·埃德伦德的人(我也从未见过他)的事情。这就像是 [CSS 禅宗花园](http://csszengarden.com)的对立面，太棒了。但是 Zen Garden 是一群人通过在一个网站上工作来展示他们的 CSS 技能。

这太棒了，它改变了很多人对 CSS 是一个四四方方的东西的印象，变成了一个美丽的东西。但是没有任何实际的、实时的、有效的网站展示。让你的老板去看看一个“黑客”网站当然很好，但他们想看看知名公司的例子。

所以我们给像 PGA 高尔夫巡回赛这样的网站和老板们知道的大公司颁奖。这个想法是，人们会让他们的老板去这些网站，告诉他们这种网络标准的东西不仅仅是一个小众的东西——这是很多人都在做的事情。我们这样做了大约一年半，我们的想法是每周发布一到两个奖项——银奖——然后在月底，我们会提名其中一个为当月最佳网站。

它工作了一段时间，但后来我们觉得我们达到了一个临界点，标准开始变得如此普遍，有很多非常好的网站，奖励好的设计不再是不寻常的，开始变得司空见惯。其次，做类似事情的网站越来越多。有时尚晚会，CSS 跳马，CSS 美容，CSS 狂热，CSS 什么的…我不再跟了。我的 RSS 阅读器里有一个大约 30 个画廊网站的列表，而且我总是看到新的网站！

坦率地说，当有三个 CSS 画廊网站时，我们在想“好吧，谁需要第四个？”我不知道这些网站是如何生存下来的，但显然它们似乎是这样。他们都分享相同的设计——它在一个网站上展示，然后扩散到所有其他网站，所以很少有新的东西被分享。

所以我想我们认为这个网站已经完成了它的工作，不像许多这些新网站，他们在那里做谷歌广告，为运营网站的人建立品牌，网络标准奖的存在是因为我们真的热衷于突出优秀的设计。一旦达成协议，就没有必要再继续下去了。

除了银背，你们的未来还有什么？

我们现在有很多事情要做。再过几周，银背企鹅就要上市了。我们仍在做一些最后的调整；我们仍在优化输出，因为目前它们有点大。我们现在正在设计网站。我们正在黑进支付系统，诸如此类的东西。因此，希望在两到三周的时间内，Silverback 应该会推出。到处都有一大堆演示拷贝——我在我们的研讨会上分发了一些。

但是我们不想卖出更多的票，因为我们想把它保持在一个可管理的水平，我们不想让它变成 SXSW。我是说 SXSW 很牛逼。但是我认为，除了演讲者之外，会议最重要的事情之一是与会者，并允许这些走廊对话。如果你有 1500-2000 人在一个房间里，没有空间四处走动，就很难与人见面。所以 800 人是个不错的选择。所以今年我们可能会带大约 800 人。

所以我们对即将到来的建筑设计感到非常兴奋。该网站目前正在开发中，有望很快推出。我们有一个很棒的演讲者阵容，我们每年都做不同的主题。今年的主题是“为社交网络设计”。

**SP:自称“用户体验设计师”的人，大学一般没学过这个话题。你能从实践中，从参加像这样的会议中，从在这一领域处于领先地位的人们那里学到一些东西吗？对于没有以用户为中心的设计经验的人来说，从哪里开始呢？**

现在进入这个行业的人的问题是，所有在这个行业里是资深人士的人都是在这个行业刚刚起步的时候开始的。因此，随着技术和行业运作方式的进步，我们也随之进步。所以我们有 8… 9… 10 年的经验。当我们开始的时候，我们是垃圾，但是网络是垃圾。所以我们可能是垃圾，网络也可能是垃圾，我们只是学会了。

现在，质量水平和专业水平如此之高，以至于当人们离开大学或学院时，没有这方面的经验，也不是专家，就像我们开始时不是那样，实际上这方面的空间很小。因为如果你现在设计蹩脚的小网站，人们不会感兴趣，因为专业水平已经提高了。所以我不认为在你如何从有实际的愿望跳到能够实际执行的过程中有一点鸿沟——一点问题。因为事情已经过去了。

例如，当我建立我的第一个网站时，它是一个可怕的基于框架的小网站，看起来很垃圾。但是其他的网站看起来也很垃圾，所以没关系。现在，网站看起来很好，所以当你做你的第一个网站时，相比之下可能会很糟糕。

我确实认为上大学会有所帮助。我想如果你上大学，你可能会想学一些非常具体的东西。如果你对用户体验感兴趣，那么参加人机交互课程、认知心理学课程或社会心理学课程将会大有裨益。

但是，一旦你这样做了，一旦你离开了，你需要做几年的在职培训，这真的很难。因为你需要同时学习 HTML 和 CSS，学习设计，做自由职业者。当你离开时，如果你有一个好的投资组合要展示，你需要去一家机构工作一段时间，这家机构会接纳你，培养你，让你在两三年内变得足够好。在一个小机构，你可以学习那些技能，然后你可以去一个更大的机构或做自己的事情。

我觉得做实习之类的事情也可以不错。我们刚刚开始在 Clearleft 实习，实际上让人们在一段时间内实习，就像大学课程之间的间隔年。来到一家网络公司，获得真正的经验，我认为这种经验对你的发展非常重要。一旦课程结束，如果你能说“嗯，我已经在 Clearleft 工作了三个月，或者其他什么的，这里有一些我参与的真实项目”，那么你很有可能被录用。

我认为这真的很难，我认为你需要做的是专业化。我认为多面手的时代已经一去不复返了，因为我认为，可悲的是，如果你是一个多面手，那就意味着你做的每件事都没有达到最高质量。因为现在有这么多专家，你必须和他们竞争。因此，开始真正擅长 web 标准，例如，如果你想在行业中找到工作，如果你真的擅长 HTML 和 CSS，那么你会找到工作，学习 HTML 和 CSS 并不需要太长时间。

但是学习更软的可用性技能——你可以学习启发式，你可以学习基本原则。但是，直到你真正坐下来，建立了大量的网站，并把它们搞砸了，你才能做好。这就是为什么据我所知，很少有公司像 Clearleft、Adaptive Path 或 Happy Cog 这样做我们所做的事情。因为你需要那种广博的知识。你需要已经做了 10 年，犯了很多错误。所以大多数网络公司要么非常通才，要么非常专业，只做用户测试，或者只做用户研究……或者他们什么都做——编程、构建:一点这个，一点那个。

因此，我们明确地把重点放在了用户体验方面，并把自己定位为“web 架构师”。所以我们做所有建筑师会做的事情，只为一个网站。
