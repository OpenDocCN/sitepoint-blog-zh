# 无障碍网页设计介绍

> 原文:[https://www.sitepoint.com/accessible-web-design/](https://www.sitepoint.com/accessible-web-design/)

网页设计中的可访问性解决了创建所有用户都可以访问的网站的问题，而不管他们的身体能力或他们使用互联网的方式。

在这篇文章中，我们将讨论可访问网站设计的概念，当前增加网站可访问性的计划，以及使你的网站更易访问的指导方针。

##### 为什么可访问性很重要

为什么不是所有网站都可以访问？你可能会问自己，为什么存在“可访问性”的问题，为什么所有的网站并没有被所有的用户访问。有许多原因解释了无障碍设计的必要性。有些涉及用户可能有的残疾，而另一些只是源于不同的人选择访问和使用互联网的方式。

***视障用户***

视力残疾的用户从色盲到全盲都有。这些用户可能难以理解没有文字描述的图像。没有文字描述，看不到图像的用户就无法知道它是什么，或者它代表什么。色盲用户也可能很难辨别设计元素(包括文本)的颜色与其周围的元素(包括背景或页面颜色)没有足够的区别。

视力受损的用户也可能在理解不是为通过非视觉浏览器(如屏幕阅读器)进行“查看”而构建的站点方面存在问题。屏幕阅读器是一种 Web 浏览器，它可以大声朗读网站，从而使有视觉障碍的用户可以访问这些网站。通常一个视觉上看起来不错的网站在通过屏幕阅读器收听时会变得一塌糊涂。

***听力障碍***

与视力受损者面临的问题类似，网络用户也面临着听力问题。有听力障碍的用户无法理解通过声音传递的信息。简单的解决方案是提供一个不使用声音的替代方案，比如文本描述或图像。

***身体残疾***

如果你不是身体残疾，很难想象如果你是，你的在线体验会受到什么影响。例如，你有没有尝试过不用鼠标来使用网站？除非你幸运地访问了这个网站，否则你可能会发现它非常难。身体残疾的用户通常不能使用鼠标。除非您在创建网站导航和输入法时考虑到这些用户的需求，否则身体残疾的用户可能会发现您的网站完全无法访问。

***认知和神经障碍***

网站可能很复杂，对于我们中最有能力的人来说，找到我们想要的信息可能很困难。使用过于复杂的设计、不一致的导航和令人分心的重复动画的网站对这种情况没有帮助。这些设计元素会给有认知和神经障碍的用户带来复杂的问题，并使他们完全无法访问某些网站。

##### 超越残疾

正如我们所见，如果您有残疾，使用互联网可能是一项艰巨的任务。通过观察和理解无障碍网页设计的指导原则，您可以创建一个服务于其目的的网站，并且该网站对所有用户都是可访问的，而不仅仅是那些没有残疾的用户。

但这并不止于此。无障碍网页设计对其他用户也有好处。如果您遵循指南使您的网站具有可访问性，这些群体将会受益:

*   手机、网络电视和信息亭的用户
*   低带宽用户
*   嘈杂环境中的用户
*   “屏幕眩光”用户
*   开车的用户
*   文化水平低的用户
*   第二语言访问
*   不同学习风格的用户

处理可访问性问题也有所改进:

*   页面传输和网站维护
*   内容的机器索引
*   内容搜索

***集市***

 **让你的网站具有可访问性还有另外一个原因(如果你还需要的话！).根据目前的数据，在大多数国家，残疾用户目前约占人口的 10%至 20%。增加这个数字来代表你的用户中属于上述类别的部分，你会看到你的网站有 30%的市场份额。如果让 30%的市场能够访问你的网站还不能让你相信无障碍网页设计是值得的，那么你最好现在就停止阅读。

许多国家人口的平均年龄也在增加。衰老会导致许多无障碍问题，包括视觉和听觉变化以及灵活性和记忆力下降。如果你的市场包括大量的老年访问者，那么你可以将这 30%增加到更大比例的用户，他们将从无障碍网页设计中获益。

***法律要求***

对于某些网站，解决可访问性可能是一项法律要求。这通常是政府网站的情况，但也会影响其他网站。有关不同国家要求的更多信息，请参见 W3C 页面[与网页可访问性相关的政策](https://www.w3.org/WAI/Policy/)。

##### 无障碍倡议

世界各地的一些团体正在努力提高对无障碍问题的认识，并为无障碍网站的作者提供帮助。其中包括万维网联盟(W3C)、美国政府和 CAST/Watchfire。让我们更详细地看看这些小组都做了些什么。

***网络无障碍倡议***

自 1999 年以来，万维网联盟(W3C)T1(创建网络标准的组织)一直致力于其“网络无障碍倡议”或 WAI。该倡议的正式使命是:

万维网联盟(W3C)致力于引导网络发挥其全部潜力，包括促进残疾人的高度可用性。WAI 与世界各地的组织合作，通过五个主要工作领域实现网络无障碍:技术、指导方针、工具、教育和外联以及研究和开发。

迄今为止，这一举措的成果是三套准则:

*   [网页内容无障碍指南 1.0](https://www.w3.org/TR/WAI-WEBCONTENT/)
*   [创作工具可访问性指南 1.0](https://www.w3.org/TR/WAI-AUTOOLS/)
*   [用户代理可访问性指南 1.0](https://www.w3.org/TR/WAI-USERAGENT/)

网站内容指南旨在向网站作者展示如何使他们的网站具有可访问性。创作工具指南适用于编写可用于创建网站的程序的人。用户代理指南是针对那些创建 Web 浏览器的人的。让我们更深入地看看内容可访问性指南。

**网页内容无障碍指南 1.0**

Web 内容可访问性指南或 WCAG 于 1999 年 5 月 5 日成为 W3C 推荐标准(成为 W3C 推荐标准是一个极其艰难和漫长的过程)。

该指南的目的是向网站创建者解释网络技术的无障碍使用。这是通过 14 条指导原则实现的，总共有 60 个检查点必须遵守，以确保网站是可访问的。

为了适应人们为使他们的网站可访问而愿意付出的不同程度的努力，检查点被分为 3 个不同的优先级。[规范](https://www.w3.org/TR/WAI-WEBCONTENT/)列出了如下三个优先级:

**【优先级 1】**

Web 内容开发者必须满足这个检查点。否则，一个或多个组将无法访问文档中的信息。满足这个检查点是某些组能够使用 Web 文档的基本要求。

**【优先级 2】**

Web 内容开发者应该满足这个检查点。否则，一个或多个组将很难访问文档中的信息。满足这一检查点将消除访问 Web 文档的重大障碍。

**【优先级 3】**

Web 内容开发者可以解决这个检查点。否则，一个或多个组会发现访问文档中的信息有些困难。满足这个检查点将改善对 Web 文档的访问。

对于那些花时间制作符合任何上述优先级的页面的人，有一组匹配的一致性级别:

*   **级别“A”**—满足所有优先级为 1 的检查点。
*   **级别“双 A”**—满足所有优先级 1 和 2 的检查点。
*   **三 A 级** —满足所有优先级为 1、2 和 3 的检查点。

符合三个级别之一的页面、站点或站点的一部分可能会显示一个徽标，链接到 W3C 对该声明的适当解释:

[![952_wcagA](../Images/94c8ae4bf8c14e1226392dd3a130a7df.png)](https://www.w3.org/WAI/WCAG1A-Conformance)–A 级

[![952_wcagAA](../Images/22eb7b3fcba5fd06b037378c08f11ba8.png)](https://www.w3.org/WAI/WCAG1AA-Conformance)–双 A 级

[![952_wcagAAA](../Images/b427edf550cbdaf6aea7654154942586.png)](https://www.w3.org/WAI/WCAG1AAA-Conformance)–三 A 级

关于一致性标志的全部细节，请参见 [W3C WCAG 一致性标志](https://www.w3.org/WAI/WCAG1-Conformance.html)。

或者，可以通过文本解释来指定一致性，例如:

<q>*本页面符合 W3C 的“网页内容无障碍指南 1.0”，可在【https://www.w3.org/TR/1999/WAI-WEBCONTENT-19990505】T2，双 a 级*T5 获取</q>

文本必须说明:

1.  指南的标题是:“网页内容无障碍指南 1.0”
2.  指导方针是 https://www.w3.org/TR/1999/WAI-WEBCONTENT-19990505 的 URI:
3.  符合性级别为“A”、“双 A”或“三 A”。
4.  声明涵盖的范围(例如，页面、站点或站点的定义部分)。

网站内容可访问性指南由[网站内容可访问性指南工作组(WCAG 工作组)](https://www.w3.org/WAI/GL/)制定。如果您希望生成符合 WCAG 1.0 的页面，下面的 W3C 页面会很有用:

*   [指导方针](https://www.w3.org/TR/WAI-WEBCONTENT/#Guidelines)
*   [检查点清单](https://www.w3.org/TR/WAI-WEBCONTENT/full-checklist.html)
*   [遵循指南的技巧](https://www.w3.org/TR/WAI-WEBCONTENT-TECHS/)

***美国政府——第 508 节***

美国政府已经认可了 W3C 网站内容可访问性指南，要求所有联邦网站和受联邦合同约束的网站都必须遵守该指南。更多信息可从[部分 508](http://www.section508.gov/) 获得。

***巴比**T3】*

虽然有可能阅读 Web 内容可访问性指南并构建您的网站以符合该指南，但如果有一个工具为您测试它，不是更容易吗？你猜对了，确实有:一个名为 Bobby 的产品是由一个名为 [CAST](http://www.cast.org/) 的非营利组织制作的，目的是测试网站以确保它们遵守指导方针。它也知道一致性级别，并且会告诉你你的站点处于哪个级别(如果有的话)。

2001 年 12 月，Bobby 也开始支持 Section 508，这使得它成为最完整的可访问性测试器。2002 年 7 月，Bobby 被一家名为 [Watchfire](http://www.watchfire.com/) 的公司收购，该公司现在负责该产品。

Bobby 有自己的一套徽标，其工作方式与 W3C 徽标非常相似:

[![952_bobbyA](../Images/90f7e48e80164b7e83740827811ecad1.png)](http://bobby.watchfire.com/)–A 级

[![952_bobbyAA](../Images/1d1740499bd2ab704ceeee9b4c284873.png)](http://bobby.watchfire.com/)–双 A 级

[![952_bobbyAAA](../Images/93ada2f45de17b78545fc2235915f91e.png)](http://bobby.watchfire.com/)–三 A 级

[![952_bobby508](../Images/725b3d9f46d40ce0f85e57ab4fb7bc1c.png)](http://bobby.watchfire.com/)–由 Bobby 批准的 508 条款合规

你可以用两种方式使用鲍比。如果你只是想在这里或那里测试几个页面，它可以作为一个免费的在线工具在[http://bobby.watchfire.com/](http://bobby.watchfire.com/)获得，其工作方式与 W3C HTML 验证器非常相似(如果你用过的话)。您输入想要检查的文档的 URL，然后会收到一份报告，详细说明您在哪些地方遵守了，哪些地方没有遵守。由于可访问性准则的性质，不可能通过一个程序检查所有的东西，所以你仍然必须自己检查一些东西——Bobby 告诉你检查什么。

如果你想检查许多网页，在线工具有其局限性(你每小时只能检查少量网页)。在这种情况下，你最好购买可下载的版本，这样你就可以查看尽可能多的页面。在撰写本文时(2002 年 11 月)，价格仅为 99 美元。

Bobby 的一个很好的特性是，对于每一个指导方针，都可以很容易地得到关于如何使你的文档符合性的建议，以及为什么它对可访问性很重要。
