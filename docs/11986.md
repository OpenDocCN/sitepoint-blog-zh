# WebDU 2009:如何成为一名摇滚明星开发者

> 原文:[https://www . site point . com/webdu-2009-how-to-a-rock star-developer/](https://www.sitepoint.com/webdu-2009-how-to-be-a-rockstar-developer/)

Kay Smoljak 在最近由 SitePoint 赞助的 WebDU 会议上报道。

对于第二天的主题演讲，WebDU 组织者计划了一些稍微不同的东西:来自三家公司的演示，它们的竞争产品每天都影响着大多数网络开发者:谷歌、雅虎和微软。令许多观众失望的是，这些巨头的代表们讲述了他们为使 web 开发人员的生活变得更轻松所做的贡献，而不是一场笼中竞赛。

首先，谷歌的 Alan Noble 告诉我们，现在是成为一名网络开发人员的最佳时机。他列举了谷歌的三个重点领域来支持这一说法:开源工具集、强大的开发平台和货币化。艾伦随后概述了谷歌认为重要的创新领域，包括 Android、媒体数字化，当然，还有当下的流行语:云计算。

接下来，来自雅虎的 Neil Wilkinson，或“Wilf ”,谈到了该公司的开放战略和由此发展而来的开发工具:雅虎用户界面组件；GithubYSlowBOSS(自建搜索服务)；Flickr，这个星球上使用最多的照片 API 和 YQL 或 Yahoo 查询语言，将它们结合在一起的粘合剂。

微软的代表选择谈论 Silverlight 和 Surface 的界面增强，而不是当前的开发工具，其潜在主题是用户界面的速度不应该等于底层互联网连接的速度。

虽然他们的个人陈述很有趣，但这些公司都没有说什么特别惊天动地或开创性的话。他们确实展示了他们方法的不同——谷歌有一个简单但显然是排练过的演示，雅虎几乎完全依赖于以开发人员为中心的代码样本，微软则有漂亮的幻灯片和演示。到最后，许多代表变得焦躁不安，所以它被关闭到下一个会议。

![lisa](../Images/c12a7860e6caa2a4f6a9928a81d0abb5.png)拥挤的房间意味着只有 Lisa Herrod 的*站着的地方才能在你的工作流程中找到可用性。*随着用户体验作为一门学科的发展——足以保证 WebDU 的整个流——肯定有很多人对 Lisa 的主题感兴趣，这个主题适用于各种情况下的开发人员。

Gert Franz 关于替代 CFML 发动机 Railo 的报告也很受欢迎。Gert 介绍了 Railo 和 Adobe ColdFusion 之间的差异，指出虽然编程差异很小，但在安全性和易管理性方面有很多增强，这些增强使 Railo 成为共享托管环境的可行替代方案。

特别令人耳目一新的是 Gert 对 Railo 目标的陈述:成为头号 CFML 引擎——但不是通过转化现有的社区成员，而是通过将 CFML 扩展到新的市场，特别是那些专门寻找开源解决方案的市场。

很容易，当天最好的演讲是 Toby Tremanyne 的会议*step It Up*——针对我们其他人的企业发展，尽管这绝对不是我所期望的！Toby 概述了他所认为的企业或高级“摇滚明星”开发人员的素质和要求，然后概述了个人可以做些什么来从代码猴子上升到团队领导、受欢迎的自由职业者、企业主或非常受欢迎的会议演讲者。

虽然 Toby 的信息很有价值，毫无疑问，房间里的许多人都争先恐后地做笔记，但正是这种神奇的触摸引发了这场演示:阐明一个观点的闪光，以及说明企业开发人员必须时刻准备着的可怕的纸牌魔术。

WebDU 2009 的最后一场会议是演讲者圆桌会议——现在是 WebDU 的一个传统。一些演讲者坐在台上，试图回答观众的问题——杰夫拿着麦克风跑来跑去，像杰瑞·斯普林格一样。问题的范围从一般的——代表的公司在环境可持续性方面做了什么——到具体产品的营销，如雅虎的开发者产品和 Adobe 的 ColdFusion。

![panel](../Images/7cf8f0b335da7ee69bdc3110f9bee11e.png)

这是一个伟大的会议的结束——突出的是一个良好的场地和优秀的餐饮；包括国际和本地专家的优秀演讲者阵容；和有趣的社交活动，包括真棒 WebDU 卡交易游戏。如果我不得不吹毛求疵的话，wi-fi 的情况相当黯淡，五个独立的流确实让代表们很难看到他们感兴趣的每个发言者/主题。但总的来说，WebDU 2009 提供了一个出色的会议体验和一个伟大的社会氛围，我期待着守护进程团队在 2010 年的商店。
