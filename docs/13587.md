# 高阶 Perl(和 Javascript/Ruby)

> 原文:[https://www . site point . com/higher-order-perl-and-JavaScript-ruby/](https://www.sitepoint.com/higher-order-perl-and-javascript-ruby/)

目前在由[马克·杰森·多明纳斯](http://newbabe.pobox.com/~mjd/blog/prog/)编写的[高阶 Perl](http://hop.perl.plover.com/) 的副本中摆弄着大约 15 个书签。只是一个小小的回顾——更详细的回顾请点击这里的。

从内封面…

> 作为一名程序员，你的书架上可能堆满了没有改变你编程方式或思考编程的书籍。
> 
> 这本书你需要一个完全不同的书架。

不知道这是轻描淡写还是夸大其词。无论如何——这是一本优秀的书；最重要的是作为步入[函数式编程](http://en.wikipedia.org/wiki/Functional_programming)的中间地带，但是 Mark(我假设)的数学背景使得它与 web 框架和 CRUD 有了非常令人耳目一新的突破，有了一些真正有趣的问题领域。马克[在这里](http://www.theperlreview.com/Interviews/mjd-hop-20050407.html)解释了他的“隐藏议程”。

然而，这不全是数学(事实上，主要不是)。例如，关于解析的章节是我所见过的关于这个主题的最容易理解和最有用的章节之一(也许除了[游戏脚本大师](http://www.amazon.com/gp/product/1931841578/)，奇怪的是——[乔姆斯基语法](http://en.wikipedia.org/wiki/Chomsky_hierarchy)甚至没有被提及；)

否则，有一堆技术和想法几乎适用于任何地方，如果你习惯于从类和对象的角度思考，可能会让你大吃一惊，比如在解析配置文件时使用[散列表来调度调用](http://en.wikipedia.org/wiki/Virtual_table)。

后面章节中的一些例子变得相当沉重(因此有了书签),所以要注意——你需要一些不受干扰的时间来充分利用它。

当然，先决条件是熟悉 Perl，在这里这本书不会带走很多囚犯。但是好消息是，如果你不知道 Perl 书中例子的代码已经被翻译了两次——第一次是由[肖恩·m·伯克](http://interglacial.com/~sburke/)翻译成 [Javascript](http://interglacial.com/hoj/hoj.html) (比如我[剽窃了他的](https://www.sitepoint.com/us-ascii-transliterations-of-unicode-text/))第二次是由[詹姆士·格雷](http://blog.grayproductions.net/)翻译成 [Ruby](http://blog.grayproductions.net/articles/category/higher-order-ruby/) 。有人不怕麻烦，这应该会让你知道它有多好。你仍然需要这本书来解释，但是学习一些 Perl 也不错。

无论如何——感谢 Mark 作为作者“坚持到底”——已经很久没有这么喜欢一本关于编程的书了。
