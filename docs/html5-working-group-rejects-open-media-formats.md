# HTML5 工作组拒绝开放媒体格式

> 原文：<https://www.sitepoint.com/html5-working-group-rejects-open-media-formats/>

HTML5 规范看起来准备给这种语言引入新的音频和视频功能，但是 HTML5 工作组已经卷入了一场关于浏览器应该支持哪些编解码器的辩论。

诺基亚和苹果已经成功地从当前的草案中删除了 [Ogg Vorbis](http://en.wikipedia.org/wiki/Vorbis/) 和 [Theora](http://en.wikipedia.org/wiki/Theora/) ，理由是专利的不确定性(理解为:不愿支持一个没有包含 DRM 条款的标准)。

Manuel Amador 详细解释了为什么他认为[省略这些开放格式的决定是一场“令人愤慨的灾难”](http://rudd-o.com/archives/2007/12/11/removal-of-ogg-vorbis-and-theora-from-html5-an-outrageous-disaster/):

> (鉴于)Ogg 技术的无专利实现已经广为流传，苹果和诺基亚没有理由说他们不能真诚地实现之前制定的 HTML5。

WHATWG 的伊恩·希克森回应道[为什么 Ogg Vorbis 或 Theora 没有意义](http://lists.whatwg.org/pipermail/whatwg-whatwg.org/2007-December/013154.html)(主要是因为诺基亚和苹果可能担心因使用这种格式而被起诉)。

不管真正的动机是什么，总的来说，这似乎是 HTML 和数字媒体的倒退。
