# 为什么页面重量仍然很重要

> 原文:[https://www.sitepoint.com/minimizing-page-weight-matters/](https://www.sitepoint.com/minimizing-page-weight-matters/)

黑暗而神秘的互联网霸主在网络出现之初颁布了一条戒律。我们都遵循这条规则:你不应该创建超过 50kB 大小的网页。

在那个落后的时代，这是可以理解的。大多数互联网用户拥有拨号调制解调器，宽带是一个无法实现的梦想。很少有开发者敢超过 100kB，即使快速连接已经成为现实。

如今，我们很少担心页面权重。我们可能会使用 CSS 精灵和压缩的 JavaScript 库，但只有当它们实用的时候*(事实上，当我们用 57 个 Flash 广告填满我们的页面时，它们真的会有所不同吗？).*

但是我们应该关心吗？我最近有机会在没有笔记本电脑或可靠 wifi 覆盖的情况下工作。幸运的是，我有可靠的诺基亚和 E/GPRS 来帮助我保持联系。虽然不是 3G，但我并不介意多等几秒钟来加载页面。

男孩，是我错了。虽然电子邮件仍然可用，但访问大多数网站是痛苦的。许多页面的大小接近 1MB，过了一两分钟才出现。我甚至开始理解为什么[苹果禁止 Flash](https://www.sitepoint.com/adobe-apple-flash-criticism/)——任何可以精简页面的东西都会提高浏览速度。或许他们甚至考虑过屏蔽 CSS 和 JavaScript。

在西方，大约 10%的互联网用户仍然使用拨号服务。这个数字不太可能迅速下降；许多人是轻度用户，或者不愿意为快速连接付费。还有一部分用户因为住在偏远地区而无法接入宽带。

几乎没有必要担心这 10%，因为它不太可能由在线购物或使用网络应用程序的用户组成。然而，在接下来的几年中，社区中有两个部分将会经历指数级增长:

1.  亚洲和非洲的互联网用户。在电信基础设施得到加强之前，拨号上网和低速宽带仍将普遍存在。
2.  那些使用移动设备访问互联网的人。连接正在改善，但 3G 通常比宽带速度慢。此外，大多数移动用户要么按字节付费，要么使用没有 3G 的设备。

互联网人口统计正在发生变化。虽然 PC/宽带仍然占主导地位，但真正的增长领域是处理能力有限、连接速度较慢的小型设备。在美国，移动数据现在已经超过了语音流量，谷歌预测桌面计算的日子已经屈指可数了。

创建一个低带宽或移动版本的网站是可能的。这有时可能不切实际或者花费更多，但是没有理由不从你的页面上删除多余的脂肪。降低页面权重可以让你的应用程序运行更快，并节省托管成本。

你怎么想呢?开发者还应该关心页面权重吗？我们应该努力节省每一个字节吗？还是说快速连接无处不在，几乎没有必要去关心它？
