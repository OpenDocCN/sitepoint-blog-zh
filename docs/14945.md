# 我花 99 美元在 2 个月内建立了一个电子商务网站！

> 原文:[https://www.sitepoint.com/ecommerce-site-2-months-99/](https://www.sitepoint.com/ecommerce-site-2-months-99/)

过去五年来，我的梦想是推出一个互联网门户，为增值经销商 var 汇集最新的信息技术新闻和一些最好的文章和资源。我一直在为许多其他网站提供内容。现在我觉得是时候开始投资自己了。

##### 规划和准备

虽然我是一个经验丰富的互联网研究员，但我对设计、启动和推广电子商务网站知之甚少。然而，我知道我反复研究了哪些网站，所以我做的第一件事就是分析它们是如何让我回来的。好的内容是主要的动机，但还有什么呢？

##### 一点研究

我收藏并打印了许多网站的页面，这些页面展示了我认为“最好的”导航、颜色、布局、主题类别、描述这些类别的术语等。我还注意到了附加功能的使用，比如论坛、投票、时事通讯和收费内容。随着我增加了对以下内容的需求，我的任务列表急剧增长:

*   关于我们
*   广告
*   联系我们
*   捐助
*   链接到我们
*   许可
*   压
*   隐私
*   员工
*   使用条款
*   网站和时事通讯注册
*   自动回复信件

…等等。

我还记录了一些我觉得讨厌的特征，比如不能点击图标返回主页。虽然这似乎是一个小细节，但它只是我觉得我们的网站将为我们的访问者提供一个全面和用户友好的界面之前必须解决的数百个问题之一:一些互联网上最精通技术的冲浪者。

经过几天在我最喜欢的地方网上冲浪和检查我放弃的网站后，我列出了一系列我喜欢的特征:

**站点名称**

如果名字很容易记住，我可以直接输入，而不是浏览我的收藏夹。最好的名字是简单的和描述性的主题，如 internet.com，TechWeb.com，ITtoolbox.com，等等。可爱也起了作用，就像 AngryCoder.com 的例子一样。在玩了几个星期的名字组合和我的同义词查找器后，我决定用 DoctorVAR.com。

**清晰对焦**

如果网站清楚地陈述了它的目的，并且兑现了承诺，那么我马上就知道这个网站对我来说是否是一个有价值的资源。即使这不是我当时寻找的主题，我也会把它收藏起来，以备将来的项目使用。在从一个主题到另一个主题的网站中搜索通常是浪费时间。所以我把我们的目标声明放在我们主页的顶部。

**轻松导航**

在早期，航海技术通常很粗糙。通常，我会通过网站地图快速找到我需要的资源:白皮书、新闻稿等。我欣赏清晰，组织良好的导航和网站搜索功能。让我在发送任何内容之前浏览多个菜单级别和页面的网站让我抓狂。因此，我尽可能避免多层。

**免费内容**

如果网站让我注册以获取更多内容，我通常会像点击鼠标一样迅速离开！有这么多伟大的网站不需要如此投入的时间和精力——我为什么要为那些需要的网站费心呢？我决定在我的网站上，所有内容都是免费的。

**收藏链接**

我真的很感激其他来源的链接，因为它们使我的研究工作更容易。然而，如果我点击了两三个断开的链接，我开始对网站的更新程度失去信心。另外，断开的链接浪费我的时间。所以我做了一个记录，定期在我的网站上使用链接检查器，以保持内容的新鲜。

##### 其他人在说什么

一旦我确定了我在一个网站上寻找的特征，我就更进一步，阅读了许多关于其他人寻找什么的文章。我惊讶地发现，这么多人关心他们的隐私。调查显示，人们:

1.  想要一份清晰的隐私声明
2.  想知道如果他们注册论坛或时事通讯，他们的信息会发生什么变化
3.  希望能够联系到一个真实公司的真实人员，有真实的邮政地址，而不是邮政信箱号码

因此，我花了几周时间起草了我自己的关于我们、隐私和服务条款的声明——用简单的术语而不是法律术语——让我们的访客放心。

我还发现，设计最好的网站是有粘性的。你让访问者回来的次数越多——他们在你的网站上停留的时间越长——你就能越快地增加流量。因此，我们扩展了我们的网站计划，通过论坛、投票、贡献内容、免费分类广告和工作银行，让人们参与并成为社区的一部分。我还重新设计了我的电子快讯，使语气更加个人化，并让订户来网站“阅读全文”，或填写民意调查。

为了赚钱，我需要一个横幅广告管理器、日志分析器和购物车。我计划出售电子书、共享软件和其他物品来创收。最终我们可能会对优质内容收费，但目前是免费的。要发展流量，我觉得我付出的一定要比我计划得到的多很多很多。

##### 规划带来回报

我的开发者斯科特·克鲁格是奥马哈市哈德逊大道科技公司的老板，他说设计一个新网站的首要任务是制定一个全面的计划。

“在写一行代码之前，网站所有者和开发者应该对项目的范围有一个清晰的概念，”Scott 说。“因为 Linda 清楚地知道她想要的所有组件和功能，所以我能够推荐一个全面的解决方案来解决她的规格中提出的所有问题。这节省了大量时间和费用，也让我的工作不那么令人沮丧了。”

自学网站设计是乏味的，但对于一个好的结果是必不可少的。即使我非常精通网络，如果我没有广泛阅读网站设计，我也会忽略我的网站应该提供的许多功能。此外，我也不能问技术问题，也不能和我的开发人员进行有效的交流。

##### 发展

在记录了网站规范之后，我开始在网上寻找开发者。我知道我是在用啤酒钱包买香槟，但除非万不得已，否则我不想妥协。

很快，我联系的一个人给我发来了一系列试探性的问题:

*   你需要开发什么样的网站？
*   你是怎么选择 php 的？
*   需要管理界面吗？
*   你需要管理横幅广告吗？
*   实施后您的支持需求是什么？

我觉得我在考试。但是他询问的质量给了我信心，这个人想清楚地了解项目的范围，以及我管理网站的专业水平。

##### 开发商说…

很快，我安排了一次与 Scott Kroeger 的会面，他是奥马哈 NE 的 Hudson Avenue Technologies 的所有者，讨论了在有限的预算下启动这样一个复杂网站的挑战。在我和他一起查看了我的站点地图和规范之后，他推荐了一个经过验证和支持的开源内容管理系统(CMS): PostNuke。

我对集成受支持的公共领域软件感到兴奋，这些软件可以由众多的提供商维护，不管 Scott 和我出于什么原因分开。另外，不需要软件调试。

许多开发人员马上开始编码。然而，Scott 认为，首先，开发者应该调查哪些开源软件是可用的，并确定是否有可能将它们拼凑在一起以提供一个解决方案。“因为我的背景是集成，我对寻找开源软件、弄清楚代码如何工作，然后使用我的技术技能和编码使模块协同工作更感兴趣，”Scott 说。“这样我就不用花很多时间从头开始编程和调试代码了。”

但是，在阅读了 PostNuke 文档后，我开始担心，因为它不支持多页面的独特块/页面配置。由于我有超过 20 个独特的副主题或类别，我不知道 PostNuke 将如何工作。另外，我不喜欢文章的布局限制。我更喜欢设计 html 页面的灵活性。所以我把这些问题扔给了斯科特。

“通过检查其他 PostNuke 网站安装和阅读论坛讨论，我很快发现多个 PostNuke 安装可以解决页面布局问题，并提供对子网站块的完全控制，”Scott 说。“因此，我面临的挑战是，通过修改每个子站点查看的数据库表，找出如何让所有 28 个安装相互交流。”

“琳达想用 html 块来处理大量的内容，”斯科特说。“然而，PostNuke 只搜索主要模块，不搜索 html 页面。所以我集成了 Content Express——一个 PostNuke 模块，它为网站提供了一个非常友好的管理界面，用于添加 html 页面和控制网站导航，以及 html 页面的搜索引擎。不幸的是，Content Express 不是为多站点配置而构建的，所以我还必须弄清楚它在做什么，以便知道如何为多站点安装集成它。”

##### 两周成功！

不到两周，我就开始设计页面和上传数据。为了完善这个网站，Scott 集成了免费的 PostNuke 模块来提供电子杂志、论坛、工作银行和横幅/广告管理。我唯一需要购买的软件是一个 59 美元的分类广告模块和购物车，外加一个 30 美元的主题。其余的模块是免费的。

因为我曾经自学过网站设计，并且对网站规范的定义非常清楚，所以斯科特从一开始就清楚地知道自己应该做什么。这使得他的工作变得容易多了，并且，结合他的开源集成策略，节省了大量的时间和金钱。两个月后，我梦寐以求的网站建成了——在我的预算之内，而且没有牺牲一个功能或需求。
