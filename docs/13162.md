# 最佳网页设计漫画

> 原文:[https://www.sitepoint.com/the-best-web-design-comics/](https://www.sitepoint.com/the-best-web-design-comics/)

许多网页设计师也是不为人知的漫画迷。

今年早些时候，安迪·克拉克在 Web Directions South 发表了关于在漫画艺术中寻找网页设计灵感的演讲。

我注意到很多其他的极客不仅热爱漫画，而且也创作自己的漫画。事实上，当一个有创造力的网页设计师或开发者开始画漫画时，会产生非常有趣的结果。不仅仅是午餐时间的消遣，关于我们职业的连环漫画可以让我们对我们如何以及为什么做我们所做的事情有更深刻的认识。另外，让自己跟上我们这个行业正在发生的事情包括**不仅知道人们在谈论什么，而且知道他们在笑什么。**

看到在谷歌上搜索“网页设计漫画”或“科技漫画”不是没有任何结果，就是一堆垃圾，我想我应该贴上我最喜欢的关于网页设计的漫画。我不想假装这是一个详尽的列表——我希望你能帮我，在评论中列出你自己最喜欢的极客漫画。

首先，也是我最喜欢的，是**科波斯基**(【http://kopozky.net】T2)。这是我所知道的唯一一部严格意义上的网页设计漫画。这是一幅精美的画，运用了波兰和技巧，人物都是很棒的，真实的人物，你可以立即认同。哦，他们也很有趣。
[![kopozky.png](../Images/9cce31a38a32fdc87d70cb3b3c79622f.png)](http://kopozky.net/)

接下来是**确定/取消**(【http://ok-cancel.com/】T2)。这些人多年来一直在粗制滥造关于界面设计的连环画，我一直惊讶于他们是如何在创作新素材时保持自律的。再次，伟大的图纸和非常专业的完成。
[![OK-Cancel](../Images/0b6fe6c1a911604e8b56084a64b4c331.png)](http://ok-cancel.com/)

最后还有**Bug Bash**(【http://bugbash.net/】T2)。虽然更多的是关于软件开发而不是网页设计本身，但这些连环画评论了网页设计者面临的许多与团队、技术和管理相关的问题。
[![bug-bash.png](../Images/ce696b34f7c3ae0e799892949a000f5e.png)](http://bugbash.net/)

他们是我挑选的。我考虑过包括像[熊皮毯](http://www.bearskinrug.co.uk/)、[极客之道](http://taoofgeek.com/)、[科技之乐](http://www.geekculture.com/joyoftech/)、[豁开虚空](http://gapingvoid.com/)和原始极客漫画[呆伯特](http://www.unitedmedia.com/comics/dilbert/)之类的，但如果包括任何仅仅与科技相关的漫画，我们会转向一个更广泛的类型。

不管怎样，我肯定还有一些我错过的人。你最喜欢什么？
