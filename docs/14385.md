# W3C 的 Jigsaw 网络服务器

> 原文：<https://www.sitepoint.com/w3cs-jigsaw-web-server/>

许多 web 开发人员不知道的是 W3——Jigsaw 的一个实验性的、健壮的、不断发展的替代 web 服务器。

该项目始于 1996 年，随着今年 2.2.4 版本的发布，该项目一直保持活跃并不断扩展。Jigsaw 是一个基于 Java 的、平台中立的 web 服务器，能够在任何支持 Java 的系统上运行。

Jigsaw 的主要任务是展示新的协议特性，根据 W3(即 HTTP/1.1)。因此，这不一定是下一个阿帕奇。然而，其架构背后的哲学应该会吸引任何网站管理员。这有点像重新开始，因为对网站、内容和典型网站的其他方面的方法与我们习惯的 Apache、Sun 或 Microsoft web 服务器完全不同。

忠实于 W3 的可用性哲学，一个叫做 JigAdmin 的好工具可用于管理和配置任务——也是 Java 驱动的。

最有趣的是，它支持 SSL，可以在 chroot 监狱中运行，并且非常具有可扩展性，为程序员提供了教程来为应用程序添加功能。

对于那些一想到 Java web 服务器就抱怨的人来说——W3 对它进行了基准测试，发现它的性能和 CERN 服务器一样好。重要的是要记住，这并不一定是为了生产。
