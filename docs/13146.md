# WDN08:但是我没有中间名！

> 原文:[https://www . site point . com/wdn 08-but-I-don-have-a-middle-name/](https://www.sitepoint.com/wdn08-but-i-dont-have-a-middle-name/)

1 月 30 日，300 多名开发人员和设计人员来到加拿大温哥华的凯悦酒店，寻找 Web Directions North。

会议以 Jeffrey Zeldman 的主题演讲开始，Jeffrey Zeldman 是 Web 标准项目的创始人之一。Jeffrey 的发言涵盖了该项目的 10 年历史、面临的障碍和迄今取得的成就。他还概述了向客户销售 Web 标准，这是许多听众经常面临的任务。他以“ **SEO 变得容易”结束了推销。SEO make honest**，”指的是基于 CSS 的网站使用的干净代码。

他简要地提到了正在 IE8 中实现的版本定位，这是一个颇有争议的话题，我们最近在博客中谈到了这个话题，并且,[和其他许多人也谈到了这个话题。我相信参加会议的 Internet Explorer 团队的开发人员会回答很多问题(也会听到很多反对意见)。](http://adactio.com/journal/1403/)

第二位是乔希·威廉姆斯，他讲述了自己的个人故事，讲述了自己如何通过创造可销售的产品和为自己而不是客户工作，从一名设计师转变为一名企业家。当他问观众，有多少人曾为一个注定要失败的客户做过项目，尽管他们尽了最大努力，房间里超过一半的人举手了——这是一个明显的迹象，表明许多开发人员和设计师从他们在该行业的长期接触和经验中获得了根深蒂固的知识和直觉。

Jonathan Snook 是加拿大人，也是“[CSS 的艺术&科学](https://www.sitepoint.com/books/cssdesign1/)”一书的合著者，他向大家概述了所有主要的 AJAX 框架，其中大部分都是他亲自使用过的。

一天中最受欢迎的会议之一是 Jared Spool 题为“什么使设计看起来直观？”的会议，人们从其他房间拖椅子进来。通过许多幽默的例子(包括 Avis 用星号(*)标记可选的表单字段，从而难倒了那些认为“中间名”是必需的用户)。Jared 指出，当当前知识(用户已经知道的)和目标知识(用户完成任务需要知道的)之间的知识差距很小时，直觉设计就会发生。

他展示了 GTalk、Yahoo Messenger、AOL Messenger、Skype 和其他消息客户端的例子，以及它们如何处理设置各自的客户端所需的知识差距。策略包括要求用户立即知道防火墙和端口设置(不好)，自动检测功能(好)，帮助他们跨越知识鸿沟的上下文帮助(好)，以及 GTalk 中的帮助链接，将用户带到与手头任务无关的通用帮助页面(不好)。他指出，像向导这样的工具，如在 Microsoft Word 中管理邮件合并功能的工具，是帮助人们完成复杂任务的理想方式，否则这些任务需要跨越知识鸿沟。

总的来说，第一天非常成功，有可靠的信息和非常好的组织。
