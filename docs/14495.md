# Mambo 服务器 CMS

> 原文:[https://www.sitepoint.com/mambo-server-cms/](https://www.sitepoint.com/mambo-server-cms/)

我以前曾[谈过](https://www.sitepoint.com/blog/)开放源代码中的内容管理系统。然而，这些年来，让我如此兴奋的系统并不多——我已经评估了几十个，包括付费的和开源的。

最近，我开始尝试使用 [Mambo 服务器](http://www.mamboserver.com/menu/Features/)，首先增加我兴趣的是它在 OS X 上使用 Safari 的完美执行。虽然我日常生活中的许多事情都围绕着开源，但我的主要系统是运行 OS X 的 MAC 电脑，我的许多同行的系统也是如此。

该功能集几乎与 Macromedia 最新发布的 Contribute(版本 3)相匹配，但它是基于浏览器的，Contribute 可以通过多种协议进行连接。Mambo 的功能集包括以下重要功能:

*   非 HTML 文件的文档管理
*   多媒体和图像的媒体管理
*   控制项目何时上线和过期的内容计划
*   一个分层的用户系统，允许用户拥有不同级别的权限
*   页面的可选内容分级

虽然该网站暗示他们的目标是小型网站运营商，但有些功能也应该会吸引一些较大的网站。横幅广告管理简单明了，包括使用搜索引擎友好 url 的工具，以及 LDAP 集成和扩展用户配置文件等附加功能。

Mambo 似乎有一个强大的开发人员、支持者和用户团队——这对那些希望用这种 CMS 驱动商业网站的人来说很重要。我最喜欢 Mambo 的一点是它避免了成为另一个门户解决方案。安装后，用户可以直接编辑或创建自己的模板，并快速获得内容。

我很想听听那些尝试过 Mambo 或者在制作中使用它的人的反馈。
