CSS 框架是 web 设计和开发中最常用的工具之一。如果你已经涉足这个行业，在某个时候你可能已经听说过、见过或使用过 CSS 框架或库。许多开发人员没有意识到的是，由于开源运动，这些框架蓬勃发展。

像 Bootstrap 和 ZURB's Foundation 这样的框架提供了一个快速原型开发的平台，通过提供所有的公共构建模块来启动和运行您的站点。Bootstrap 和 Foundation 只是两个受欢迎的例子，还有成百上千个伟大的框架，旨在使您的工作更容易，并加快您的开发工作。

CSS 框架是使开源作为一个整体如此伟大的理想和动机的伟大代表。框架把设计者、开发者和爱好者聚集在一起，为每个人的集体利益而工作。

## 开源框架能给我们提供什么？

开源哲学的一个基本原则是思想和信息的自由交流，以改善整个社区。CSS 框架体现了这种精神。从开发人员的角度来看，框架和 CSS / JS 库让您从一个坚实的起点构建项目，利用社区的技能、经验和贡献。

无论您是否有兴趣加入一个社区来伸出援手，或者您是一名正在考虑开源方法是否适合您的项目的开发人员，都有大量的理由说明您为什么应该考虑拥抱开源以及它所能提供的一切。你可以在下面找到一些。

### 帮助塑造你感兴趣的社区

可以说，参与开源项目的一个最好的理由是，你可以积极地参与到产品进化的过程中。

与封闭源代码/专有项目不同，作为开发人员，您可以在改进和指导项目中扮演积极的角色。你不仅仅是一个偶然的观察者。

GitHub 上的[基金会项目就是一个很好的例子。该团队在 GitHub 中有他们的主要交互点，并使用它来跟踪 bug、功能请求和一般反馈。开发人员可以搜索整个基础框架，并提出更改，概述错误，通常使其整体变得更好。](https://github.com/zurb/foundation-sites)

这些项目的核心通常有一个开发人员或开发人员团队，他们管理他们引入项目核心的提交。大多数时候，管理这些项目的开发人员(包括 Foundation)很乐意让您提交变更——只要提议的变更是有用的，并且符合项目的目标。如果是这样的话，他们会被收养。这很好，因为如果你正在使用一个框架或库，而它没有关键的特性 X，你可以直接向开发者提供这个特性，他们可以把它引入他们框架的下一个版本。

您的里程数可能会有所不同，但对于我参与的大多数项目来说，bug 请求、特性插件和反馈都是项目所有者非常重视的，因为这表明他们的创造正在被积极地使用，并且对社区有价值。这是一个对双方都有利的伟大制度。不管你的贡献是大是小，你的每一点努力都是为了让这个项目整体上对每个人都更好。

### 成为社区精神的一部分

这听起来可能很主观，但是你从涉及开源的社区和活动中获得的氛围和情感确实是不同的，这是一种感觉，你是更大事物的一部分，一起工作来构建更棒的东西。

虽然像传统会议这样的活动都很好，但最能激励我们的是涉及社区的活动，在那里你可以得到真实而具体的投入。

凭借其开发者大会和有趣的互动活动，Mozilla 是开源开发中社区精神的一个很好的例子。同样可以说，你从参与一个类似于 Foundation 或 Bootstrap 的框架中获得了什么。在一个 CSS 框架上一起工作可以让你和其他人联系起来，增强你的个人和人际交往能力。你可以在更大的问题上作为一个团队集体工作，或者只是提供反馈和测试。

### 精炼您的开发技术

为开源框架做贡献的另一个好处是，你可以利用这个机会来完善你的开发技能。即使你是一个经验丰富的开发人员，通过阅读其他人的代码，你总能学到一些东西。

此外，参加社区给你一个展示你的知识和理解的机会，有点像你在学校时的展示和讲述。我知道，当我为开源框架做贡献时，我总是一遍又一遍地检查我的代码，以确保它是我能做出的最好的代码。作为一名开发人员，这种类型的自我评估对你非常有益。

### 帮助他人(和你自己)

加入开源项目的好处在于，你可以帮助社区中的其他开发者。你的贡献和其他人的贡献一起改善了整个项目。

在前端框架的情况下，您可以根据自己的意愿贡献尽可能多的内容。从错误报告、特性请求甚至提供反馈都是有价值的贡献。

同样，基金会是一个很好的例子。它的开发人员积极改进和完善他们的 CSS 框架。在过去的几年中，他们发布了几个主要版本(现在是版本 6)，每次来自社区的反馈都会带来更好的产品。如果您是 Foundation 的新手，请查看下面列出的文章，了解关于该框架最新版本的更多细节。

### 利用现有解决方案

这是显而易见的。使用现有的框架而不是从头开始构建更快。事实上，它让你不再想基本的事情，让你专注于工作的复杂性。

当考虑利用现有解决方案的优势时，ZURB 基金会的网格组件浮现在脑海中。您可以通过应用行、列和设置的组合来使用网格，以构建简单和复杂的布局。

这里的好处是网格可以正常工作，您不需要担心它会崩溃或者自己计算出响应逻辑，您可以利用其他开发人员的集体经验来帮助促进您的开发。

## 开源，Foundation 背后的团队怎么看？

![Foundation Open Source Frontend Framework](img/b4139c5656decef8f2b552c51fbbcd68.png)

虽然我向你推销开源的神奇之处和它能为你提供的一切都是好的，但实际上更有用的是听取已经接受开源的企业的意见。

CSS 基础框架的开发者 ZURB 开源了他们的整个项目。我最近与基金会首席开发人员凯文·鲍尔交谈，了解他们对开源及其业务的总体感觉。以下是他的回应:

### 让 Foundation 成为一个免费的开源项目的决策过程是怎样的？

> ZURB 的使命是改变人们设计互联产品和服务的方式，目标是覆盖 5000 万人。Foundation 的第一个版本来自我们为设计客户创建的内部编码风格指南。开源似乎是显而易见的，这样才能达到我们的目标。

### 作为一个企业，开源对你们来说有多重要？

> 我们认为开源是促进用户教育和创新的最有效的方式之一。
> 
> 除了教育人们和帮助他们更好地设计，开源还有许多其他的影响。首先，这是认识新朋友的好方法。当用户下载并使用我们的一些开源软件时，我们喜欢听到他们的反馈，他们对源代码的修改以及他们可能有的任何建议。
> 
> 我们最近评估了一些开源项目，注意到了三件大事:
> 
> *   对开源工具有巨大的需求，我们希望通过我们的[操场](http://zurb.com/playground)片段、基础和未来的创造性实验，尽我们所能帮助满足这种需求。
> *   在开源社区中，质量的标准非常高。这部分是因为，一旦一个产品发布，就有关于哪些可以改进的即时反馈
> *   维护开源软件需要大量的投入，因为人们信任一个新工具需要一定程度的投入。我们已经承诺通过贡献漂亮的源代码，让人们可以将这些代码整合到自己的设计中，让网络变得更加美好。这是我们想要兑现的承诺。"

### 让 foundation 开源对 Foundation 整体有提升吗？你对基金会的整体进展有什么看法？

> 有一句非洲谚语——“如果你想走得快，就一个人走。想走远，就一起走”。拥有庞大的人员群体肯定会产生更多的开销，或者导致一些事情进展缓慢，但基金会在 5 年内取得了比我们单独行动时更大的进步。

### 你对开源基金会或你的其他项目有什么问题或顾虑吗？

> 鉴于我们思考问题的方式，问题总是围绕着我们如何推动对话和设计世界向前发展。显然，作为一家企业，我们需要赚钱，我们确实有自己的项目，但从我们的角度来看，当我们创新并引领对话时，我们的业务就会蓬勃发展，我们所做的绝大多数工作都可以作为开源做得更好。

### 社区参与基金会有多重要？

> 从历史上看，我们的参与程度相对较高，在过去的一两年里，我们的参与程度确实有所提高，变得更受社区的推动。在 ZURB 之外，我们有一个伟大的核心贡献团队，最近几个版本大约 80%是社区代码。
> 
> 我们相信我们在愿景和方向方面可以提供很多，但鉴于我们的使命，社区和参与越强，我们就越有可能真正改变人们设计互联产品和服务的方式。

## 总结一下，关于开源的考虑

正如你所看到的，拥抱开源，尤其是在 CSS 框架和库方面，是非常有益的。ZURB 已经表明，当一个企业接受开源理念并使其成为企业的关键部分时，他们可以利用社区的集体力量为每个人造福。
