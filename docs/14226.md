# 构建您自己的 rpm

> 原文:[https://www.sitepoint.com/build-your-own-rpms/](https://www.sitepoint.com/build-your-own-rpms/)

我一直在寻找最简单的途径，将文件包、自定义 web 应用程序和其他数据/软件发送给客户。如果我自己不处理安装和实现，我尽量不要给他们发送一组复杂的部署说明。

因此，在多年使用和滥用 rpm 格式来满足自己的需求之后，我终于开始将它作为一种分发方法。首先，我花了大量的时间探索[rpm.org 如何](http://www.rpm.org/RPM-HOWTO/)，男子档案和试验规格文件。

spec 文件是构建 rpm 文件的核心，它控制着数据输出的目的地。虽然并不是我打包的所有东西都必须像 Apache 或 MySQL 那样“安装”,但我组装的解决方案与任何 web 应用程序一样，都有各种目标二进制文件、配置文件等等。

最终的场景是向客户发送一个 rpm，并让他们的技术人员在查看了一个自述文件后知道只需“rpm -i myfile.rpm ”,该文件详细说明了包含的内容以及文件的最终位置。

当一些开发人员不想从事安装和设置工作时，他们可能会发现这很方便。当远程客户端不能提供对接收您的解决方案的系统的防火墙后访问时，这样做还有一个明显的优势。
