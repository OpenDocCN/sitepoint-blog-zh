# 当事情变糟时

> 原文:[https://www.sitepoint.com/when-things-go-sour/](https://www.sitepoint.com/when-things-go-sour/)

在你的职业生涯中，不可避免地，一个项目会变得糟糕，你需要从一个项目中过渡出来。发生这种情况时，你可以轻松或艰难地倒下。下面是一些来自正在解决这个问题的人的建议:

1.别往心里去。不要自以为是。如果你这样做，你会损害你的声誉，制造更多你不需要的压力。例如，我认识一个开发人员，他实际上要求正式道歉作为过渡协议的一部分，这在当时对他来说可能是有意义的，但也让他看起来有点不成熟和幼稚。接受现实，专业一点，努力达成和解。

2.得到你应得的。去找合同。如果合同规定客户欠你钱，那就去拿。确保在任何终止协议中有一个条款，关于你如何获得报酬，你必须做什么，以及你何时获得报酬。如果可以的话，提前得到报酬。

3.获得一份正式的书面成交协议。一切都应该写下来。

4.提供保修。一个开发人员在一种情况下这样做了，它在建立信任和巩固关系方面发挥了作用。保证书规定，开发人员将修复任何现有的或新发现的错误或问题。

5.您的客户拥有任何代码，除非您另外指定。专业一点。

6.签订一份相互赔偿条款，这样你就有法律保障了。

7.如果你有勇气这样做，要求一封推荐信作为最终协议的一部分。#4 中的开发人员这样做了，我们也跟着做了，只是为了解决问题并继续前进。世事难料。
