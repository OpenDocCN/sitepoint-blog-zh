# 仅针对 Windows/IE 用户的飓风救援

> 原文：<https://www.sitepoint.com/hurricane-relief-for-windowsie-users-only/>

(Via [Ars Technica](http://arstechnica.com/news.ars/post/20050906-5278.html) )虽然美国[对公共资助网站的可访问性有严格的规定](http://www.section508.gov/)，但这些规定显然与负责新奥尔良当前灾害管理工作的联邦紧急事务管理局(FEMA)无关。它的[个人协助注册网站](https://disasteraid.fema.gov/IAC/displayPage.do?forward=home&)不仅需要 JavaScript 才能运行，而且它也只能在 Windows 浏览器的 Internet Explorer 中运行。

当试图在 Firefox 或其他基于 Mozilla 的浏览器中注册时，用户必须通过 CAPTCHA 测试，然后收到以下消息:

> 为了使用此网站，您必须启用 JavaScript 和 Internet Explorer 版本 6。从微软下载或致电 1-800-621-FEMA (3362)注册。

在默认伪装成 Internet Explorer 的 Opera 8 for Windows 上，验证码测试甚至不起作用。

联邦应急管理局声称，该系统最初是为内部使用而开发的，并正在开发一个替代系统，将适应所有流行的浏览器。然而，这只是表明，开发人员应该尽可能地采用最佳实践，即使是在为有限的部署构建应用程序时。

与此同时，麦金塔用户将不得不忍受电话热线，救援人员将不得不继续在现场安装昂贵且耗时的 Windows XP SP2 系统，以适应一个使用浏览器专用 JavaScript 的网站，而只需很少或不需要脚本的无障碍设计就可以完成同样的工作。应该有人给这个网站涂脂抹粉，让大家明白这一点。
