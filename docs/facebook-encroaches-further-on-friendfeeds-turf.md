# 脸书进一步侵占了 FriendFeed 的地盘

> 原文：<https://www.sitepoint.com/facebook-encroaches-further-on-friendfeeds-turf/>

当脸书[在 4 月和 5 月开始提供来自外部服务](http://www.readwriteweb.com/archives/facebooks_lifestream_adds_more_services.php)的生活流，并在上个月推出他们新设计的个人资料和新闻页面时，一些人称之为“[交友化](http://www.techcrunch.com/2008/07/21/the-friendfeedization-of-facebook/)或者，说得更直白一点，脸书正在缓慢但稳步地将 FriendFeed 带入主流……他们只是在没有 FriendFeed 的情况下这么做。

如今，脸书在 FriendFeed 的地盘上迈出了更大的步伐，在该网站的新闻订阅中加入了更好的过滤器。

当 FriendFeed 在去年三月突然出现时，它促使一些人想知道它是解决了一个问题还是仅仅突出了一个问题。FriendFeed 很擅长将我们和我们朋友的网上活动的所有不同部分整合在一起，但它并没有帮助我们更好地处理所有这些信息——事实上，通过围绕我们的网上信息流增加一个新的对话层，它实际上可能会导致信息过载。处理信息过载不仅仅是组织数据，还包括缩减数据大小。

正如弗雷德·奥利维拉去年三月写的那样，“Friendfeed *需要*过滤。如果我已经打开了 Twitteriffic，那么从我的 *friendfeed* 中过滤掉所有 Twitter 信息是有意义的。如果我有心情查看朋友的照片，我可能只想在 friendfeed 上看到照片。如果我在寻找我的朋友们在 Last.fm 上听了什么，我可能只想看那个。”

脸书网站上更好的过滤器也是用户一直在呼吁的。今天的改进是朝着那个方向迈出的一小步。新的过滤选项将允许新闻源按应用程序过滤——只查看与特定应用程序相关的新闻故事——以及按朋友群过滤。新配置文件设计的新闻源已经可以通过“头条新闻”、“状态更新”、“照片”和“发布的项目”进行过滤

额外的过滤器让用户能够更好地控制他们可以接触到多少数据以及这些数据是如何到达他们手中的。二月份， [Sean Ammirati 向](http://www.readwriteweb.com/archives/facebook_opens_news_feed_but_not_enough.php)询问了脸书过滤器的另外三个数据点，这也是我们希望看到的(尤其是第二点):

1.  查看我朋友的所有可能显示在我的订阅源中的活动
2.  对提要应用我自己的过滤器
3.  了解为什么一个特定项目被发送到我的订阅源

也就是说，通过支持额外的外部服务和更好的过滤器来继续改善新闻馈送，脸书正在继续他们成为主流一切的道路。从图片分享，到信息传递，到书签，以及生活流，脸书开始看起来像是为其庞大的主流观众提供所有最酷的 Web 2.0 服务而建立的。

我们预计脸书将在未来几个月继续增加过滤和功能增强。
