# Apple Xserve——简介

> 原文：<https://www.sitepoint.com/apple-xserve-introduction/>

苹果 Xserve 于 2002 年推出，在服务器市场上并没有大张旗鼓。整整两年后，这个平台的发展势头甚至超过了狂热的苹果观察人士的预期。

在过去的一年里，苹果的服务器销量持续两位数增长。虽然这些数字(每季度超过 11，000 台)只是惠普或戴尔吹嘘的销售数字的一小部分，但它们确实反映了资产负债表上的利润，因为 Xserve 吸引了苹果在学术、政府和科学领域的传统利基之外的走廊的兴趣。

在该公司成功进入高性能计算和 500 强超级计算机名单所引起的轰动中，苹果工程师也悄悄地为网络专业人员建立了一个严肃的网络应用和网站托管的英特尔替代品。然而，这是在没有考虑英特尔服务器使用份额的情况下完成的。Xserve 及其底层 OS X 架构的特征之一在于它与其他操作系统的集成能力。

##### 一段历史

虽然苹果在 1997 年首次进军服务器市场在运行 AIX 4 的网络服务器方面是一个惊人的失败，但 2000 年 OS X 的推出以及后来 IBM 推出的 G5 PowerPC 处理器为现在采用它提供了强有力的论据。

苹果公司的代表并不以服务器端公司闻名，但他们证实，该公司非常认真地努力渗透企业市场，并在产品开发期间与数百家公司进行了交谈。这种方法产生了一个功能强大的服务器，为 Web 托管提供了许多便利。

##### 往里看

将 Xserve 放入 Web 托管网络而几乎没有中断的能力可能是使用它的第一个也是最明显的好处。然而，在我们探索底层软件之前，看一下硬件规格就可以看出 Xserve 与戴尔、惠普和 IBM 等竞争对手之间的激烈竞争。

最近的 IDC 研究表明，超过一半的 Linux 服务器部署是基于双处理器的，因为正在承载更重的应用程序负载。考虑到许多 Web 主机运行多个域，提供带有论坛、店面等的数据库驱动的站点，下面的配置反映了这一趋势。

##### 软件神话被揭穿

苹果平台缺乏软件的论点不再有吸引力。随着它通过 OS X 的 BSD UNIX 基础进入开源领域，Web 开发人员熟知的大量应用程序现在可以在苹果硬件上使用。OS X 的 Panther 服务器与 Linux 服务器并行运行，因此大多数(如果不是全部)需要 LAMP (Linux、Apache、MySQL 和 Perl/PHP)环境的流行应用程序都在 Xserve 上按需运行。

硬件上交付的默认包包括:

*   Apache 1.3.x 和 2.x
*   赛勒斯(IMAP 和 POP)
*   文件传送协议
*   JBoss 应用服务器(用于企业 Java Beans)
*   邮差邮件列表软件
*   关系型数据库
*   OpenSSH
*   OpenSSL
*   Perl 和 PHP
*   后缀 MTA (SMTP)
*   网络邮件的松鼠邮件
*   Apache 为 Java 服务器页面和 Servlets 开发的 Tomcat
*   针对 CGI、Perl、Python、Ruby 和 shell 特定脚本的传统脚本环境

在这个列表中最突出的是几个难以配置的应用程序，一旦系统加电，它们就可以运行，潜在地节省了开发人员在编译和配置方面的时间。

在这个新的网络时代，流媒体和媒体服务几乎与数据库开发一样重要，QuickTime 流服务器的加入应该会引起一些人的注意。这为开发人员提供了处理 MPEG-4、MP3、AAC 和 RTP/RTSP 格式的能力。虽然一些人可能会对 QuickTime 的简单提及感到不满，但该应用程序现在是下一代视频编码(H.263)的核心标准，并正在迅速成为向移动设备提供音频/视频服务的主要编解码器工具，这是许多交互式 Web 开发人员正在努力的方向。

##### 在 Xserve 上控制 OS X Server

Panther Server (10.3)有一组内置的系统管理员友好的控制面板，允许配置系统的所有元素。其中包括一个相当全面的 Apache 配置应用程序，它能够为用户和组设置受密码保护的领域。

包含的 SSL 设置工具也应该受到大众的欢迎，因为它们为网站配置安全套接字层提供了可视化的提示。

对于有经验的管理员来说，深入研究单个应用程序的配置非常容易，他们可以使用终端(使用 Bash shell)，也可以使用 Webmin 等第三方工具，Webmin 可以在 OS X 上很好地运行，并提供对 Apache、MySQL、邮件服务器、防火墙等的根级控制。

对于更大的部署，苹果已经创建了一个防弹集成到开放 LDAP 和微软的活动目录。这包括精细的用户管理，可以很好地用于复杂的内部网或外部网站点。

##### 更上一层楼

对于那些有大型项目和大量存储需求(以及充足的预算)的人，苹果公司已经将 Xserve 制成可扩展的，以允许连接到 Xserve RAID，现在每个系统可以提供高达 5.6 的存储。虽然价格很高——从 6000 美元到 15000 美元不等——但它们继续低于其他平台上类似解决方案的价格。

##### 进一步探索 Xserve

随着平台的任何潜在转变，需要完成广泛的研究，以使决策者能够选择一个合理的解决方案。Apple 竭尽全力为 Xserve 可以实现的各种解决方案提供功能文档。通过花一些时间在苹果网站上，你可以回答大部分的问题，如果不是全部的话，关于这个硬件/软件组合在网络上开发和生产使用的能力。

还提供了一个特别好的包含软件功能的 [PDF 概述。](http://images.apple.com/server/pdfs/Web_Technologies_Admin.pdf)

##### 最后一点

今年早些时候，我有机会对 G5 Xserve 和 Xserve RAID 进行了一些实际评估，因为我来自 Linux 背景，所以相似性是显而易见的。从基于英特尔架构的 Linux 迁移到基于 Xserve 的 OS X 的过程相当简单。一个经过验证的媒体服务器和继续双运行 Linux(通过 Yellow Dog Linux，一个 Fedora fork)以及 OS X 的能力使得这个解决方案更有吸引力。

从那以后，我开始毫无困难地将几个生产站点迁移到 Xserve 平台上。我也必须承认，特别是在过去，我曾与邮差搏斗过，现在有了一个简单明了的经历，我松了一口气。
