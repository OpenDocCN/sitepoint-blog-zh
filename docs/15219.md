# 采访-凯恩·巴特利特

> 原文：<https://www.sitepoint.com/interview-kynn-bartlett/>

凯恩·巴特利特身兼数职。他共同创立了一家成功的网页设计公司；他是作家、教育家和公共演说家；他也是一个非常机智的人。不过，最重要的是，Kynn 因其在互联网无障碍方面的专业知识而闻名。

他最近与 SitePoint 谈论了可用性和易访问性，并论证了设计在支持这两个目标中的角色的案例*…*

你好，Kynn，感谢你接受这次采访。我想先问一下你的根。是什么让你决定从事新媒体的工作？具体来说，你是如何参与到可访问性这个话题中的？

感谢给我这个机会；我很高兴也很荣幸有机会阐明未来几年对 Web 开发者来说很重要的一个问题。

从 1986 年开始，我已经使用互联网很多年了。令我尴尬的是，我最近看到了我最早的新闻组帖子。当网站开发出来的时候，我正在一个特效工作室做夜班初级系统管理员。我的工作实际上包括放入备份磁带，键入 shell 命令，两小时后取出，重复这个过程，直到一天的 FX 工作完成。这给了我足够的时间在凌晨 5 点探索新技术，我在 1994 年成为节奏和色调工作室的正式网站管理员。

1995 年，我离开 R&H，为一家奥兰治县的网络开发公司工作，这家公司已经不存在了；几个月后，我告诉我的妻子利兹，“如果这些小丑能做到，任何人都可以。”她回答说“为什么我们不能呢？”那年 9 月，我们创办了田园诗山互联网，利兹至今仍在运营它；她有两名员工，他们设计和托管了一百多个网站。

在学习专业网站开发的过程中，HTML 作家协会的邮件列表对我非常有用，1996 年 6 月，我被邀请加入管理委员会。我一点也不知道我会做出一个超过 5 年的承诺，这将使 HWG 的成员从 6000 人增加到 125000 人，或者我最终会担任不止一次而是三次总统。

我努力推动的一项活动——并成功付诸行动——是公会加入了[万维网联盟(W3C)](https://www.w3.org/) ,成为第一个也是唯一一个加入 W3C 的工作 Web 开发人员协会。这是一个重要的角色，因为所有其他 W3C 成员，塑造网络标准的未来和网络的发展，代表商业或学术利益。前线 Web 开发人员的声音没有被听到，而 HWG 在 W3C 的成员身份为他们提供了一个发表意见的渠道。

作为 W3C 成员，HTML 作者协会可以指定志愿者为工作组贡献他们的时间和专业知识。当有机会参与到 Web Accessibility Initiative 中时，我承诺不仅要利用 HWG 的会员资格来推进 W3C 的行业协会事业，还要教育我们自己的会员关于 Web 开发的可访问方法的好处。

这一承诺导致了无障碍网络创作资源和教育(AWARE)中心的建立，以及我的网络无障碍课程。这两个网站都继续得到国际网站管理员协会的支持，该协会在 2001 年夏天合并后接管了 HWG 的运营。今年四月是第五个年度 HWG(现在的 HWG/IWA)网络无障碍月，为期一个月的焦点是网络开发者对无障碍意识的需求。

在无障碍领域，什么对你影响最大？

归根结底，我认为这是一个简单的公平和正义的概念，在这个时代看起来很天真。这是应该做的事情。

现在，一旦我说了这些，我就要定义当我们谈论网页可访问性时我到底指的是什么。最终，它不是一组清单、该做什么和不该做什么，也不是标签上的神秘规则；它更接近于一个过程和一种方法，但实际上它是一种心态。这种心态认为，“网络是一种适应性很强的媒介，允许每个人以最适合自己需求的方式获取信息。”按照我对网页可访问性的定义，它是关于包容和拥抱多样性的。这是关于关心你的观众——不管他们是谁，他们能做什么或不能做什么——能够使用你的网站。

网页可访问性实际上是一个人权问题。随着日常生活越来越多的方面——从购物到新闻到投票——转移到网上，确保我们不会无意中将那些最能从这场持续的革命和演变中受益的人排除在外至关重要。一个不能开车去商店的用户可以从网上杂货服务中获益良多；一个没有能力阅读印刷报纸的用户会发现在线版本的巨大价值。这些是互联网时代的真正好处，对于那些没有严重残疾的人来说，这些好处是理所当然的。

如果你有一种心态，将这些用户作为你的用户组的一个有效部分，你将设计一个他们可以使用的网站——你不能帮助它。确保广泛受众的可访问性成为第二天性。然而，如果你不包括这些用户——要么是因为你没有考虑过这个问题(无知),要么是因为你根本不在乎(恶意)——很可能你会构建一个对你的用户来说是严重障碍的网站。

你能强调一下你在工作中经常遇到的主要限制吗？

Web 上最明显的可访问性问题是图像缺少 alt 属性，我已经把它定义为“Web 可访问性的典型代表”。如果没有文字替代图形，屏幕阅读器软件的用户将只被告知页面上出现了一幅图像，而不知道那幅图像代表了什么。它是一个简单的插图，一个图表，一个导航菜单，或者只是另一个该死的猫图片？没有替代文本，就无从知晓。解决方法很简单——为每张图片赋予一个与图片功能相同的文本。

下一个最常见的可访问性错误与使用 JavaScript、Java、Flash、PDF、多媒体演示以及辅助技术(AT)不支持的其他语言和格式有关。辅助技术是一个广泛的范畴，包括硬件和软件(如屏幕阅读器)，允许有特殊需要的计算机用户运行程序，提供输入或输出功能，或两者兼而有之。大多数 ATs 无法处理复杂的 Web 编程效果和文件格式，因为它们在设计时没有考虑到可访问性。解决这些问题需要建立访问相同功能的替代方法，例如并行于 Java applet 的服务器端脚本或 PDF 表单的 HTML 等价物。

与网页可访问性相关的颜色使用经常被误解；颜色的使用本身并不存在可及性障碍。然而，使用颜色作为重要内容的唯一指示会给那些不能像你我一样感知颜色的用户带来很多问题。例如，我们假设火车时刻表显示所有标为绿色的时间代表上午，所有标为红色的时间代表下午。患有红绿色盲的人可能无法分辨，也不知道什么时候到达车站。要解决这个问题，你不需要关闭颜色，你只需要在日程表上的每个时间加上“上午”或“下午”。

你会注意到，在每一种情况下，我都没有说要拿走什么，而是说要增加什么。网页可访问性，如果做得好，就是通过扩展你的网站来扩大你的受众，而不是因为某个东西“不好”而删除它。不要破坏 Flash 演示文稿，只需提供带有 HTML 链接的重要内容。不要删除图形——添加替代文本，这样每个人都可以使用该网站。

事实上，图形本身可以增加许多残疾用户的可访问性，这是你以前可能没有听说过的。可访问性不仅仅是关于盲人用户；许多用户有认知障碍，这使得他们很难阅读大块的文本。通过插图和有意义的图标，你可以使你的网站对那些阅读或理解有困难的用户更容易。清晰、简单的文字和定义或词汇表的链接也可以提高理解能力，这也有利于英语不是他们主要语言的年轻用户或国际用户。

SP:人们经常交换易访问性和可用性这两个术语，就好像它们是同一个东西一样，你会说些什么来解释这两者的区别呢？

易访问性和可用性是非常相关的技术，但是很难说一个停止，另一个开始。两者都包括对用户需求的关注和改善用户体验的愿望；有些人会说易访问性是可用性的一个子集，而其他人会说正好相反。这两者都很重要，是交付 Web 服务的关键部分，让我们的受众受益。

我的看法是这样的——可访问性是关于某人是否可以访问内容，可用性是关于这样做有多容易、愉快和有效。例如，让我们考虑一个大学图书馆。四楼有一本书，内森想看，但他坐在轮椅上，而且没有电梯。他看不懂这本书的内容。

所以让我们假设他去找图书管理员并和他们谈了这件事。图书管理员接着说，“哦，好的，我去给你拿来。”她跑上楼梯，抓起那本书，拿给内森。内森现在可以访问内容了；这本书在他手里，他能看懂。然而，让我们来检查一下可用性——图书管理员必须有空，而不是忙于其他人，并且他已经花了时间索要这本书并等待它被拿下来。更重要的是，他不能浏览同一书架上的其他书籍，所以他不能很容易地找到同一主题的其他书籍。可用性比如果安装了电梯要低得多，但谁也不能否认他能接触到这本书。

作为 Web 开发人员，我们必须考虑这两个问题，对我们的残障和非残障用户都一样。他们能够访问这些内容吗？如果可以，他们这样做容易吗？

你将如何承担让设计师相信可访问性很重要的艰巨任务，非凡的设计和高水平的可访问性都有一席之地？

我不认为这是一项艰巨的任务。我认为这是一个简单明了的提议，建议每个人——不仅仅是身体完美的人——都应该能够使用这一 21 世纪的神奇工具。除了最冷酷的人之外，几乎任何人都会同意，如果你能做出能被更多人而不是更少人使用的东西，你就应该去做。

这里的问题是错误的信息被发送到了某个地方。给人的印象是无障碍网页设计意味着无聊的网页设计。这部分是网页可访问性支持者的错，他们倾向于将任何视觉上有趣的东西贴上“毫无价值的花里胡哨”的标签，强调“内容为王”,对吸引人的演示表示出不言而喻的不屑。就个人而言，我喜欢有吸引力和做得好的视觉网站；如前所述，有认知障碍的用户可以从一个可靠的、插图丰富的网站中获益匪浅。

现在，从我自己的个人网站看，你不会知道我喜欢有吸引力的网站！并非所有人都是艺术家，在这一点上，我也属于“不”的阵营。然而，比我的插图糟糕的网站更糟糕的是 Jakob Nielsen 傲视群雄的纯文本网站。虽然 Jakob 经常提出很好的观点，但他对纯文本网站的坚持导致了一个没有插图或像样的图形设计的网站。这种态度基本上是对图形艺术家所拥有的大量知识的拒绝，这些知识是关于我们这些有幸能够看见的人是如何处理视觉信息的。Jakob 和其他人——包括我自己——不应该否认这一点，而应该接受它，将其作为我们理解如何制作以受众为中心的网站的一部分。

动态视觉设计与可访问的内容界面并不矛盾。这两者是互补的学科，易访问性工程师可以从图形艺术家那里学到很多东西，反之亦然。

在你看来，为什么可访问性经常被认为是事后才想到的？

因为有太多其他的事情要担心。在 Web Accessibility Initiative 或其他可访问性组织的安全庇护下，人们很容易忘记世界上还有其他重要的事情，但事实是，Web 开发人员现在比以往任何时候都更关心他们的事情。今天的 Web 开发人员必须是图形艺术家、程序员、用户界面设计师、数据库管理员、营销专家、系统管理员、可用性专家、标记语言开发人员、动画师和系统集成商的某种组合，他们的身份各不相同。从仅仅制作在 Lynx 和 Mosaic 上看起来不错的 HTML 页面，我们已经走了很长一段路。

由于其他问题占用了他或她的时间，如果典型的 Web 开发人员意识到了这些问题，那么她没有将 Web 可访问性放在她考虑的首要位置，这有什么好奇怪的呢？大多数时候，Web 可访问性并不是一个考虑因素，因为 Web 开发人员根本就是无知的——我指的是以最中立的方式。他们只是不知道，尽管有些人可能会告诉你，不知道并不可耻。这并不像网络开发人员天生就知道如何制作易访问的网站，也不被认为是大多数学习项目的重要组成部分，从课程到书籍。

但是让我们假设我们假设的 Web 开发人员已经考虑了这些问题，并决定学习更多。他们看哪里？好吧，网页可访问性倡议是一个良好的开端，也是网页可访问性知识的权威来源。所以他们去了那里，读了一些书，他们发现了什么？[网页内容可访问性指南 1.0](https://www.w3.org/TR/WCAG/) ，这是一个很长的清单，你可以在一个完整的网站上使用，以查看它是否可能是可访问的。他们可以使用 CAST 的 [Bobby 程序来检查网站是否符合 WCAG 推荐标准。](http://www.cast.org/bobby/)

通过将 Web 可访问性简化为带有清单和评估工具的 QA 过程，W3C 实际上认可了 Web 可访问性的概念。当网站可访问性的领导机构发出同样的信息时，我不能责怪没有可访问性专门知识的网站设计者得到错误的印象。

要吸取的真正教训是，网页可访问性是一个过程，它来自于一种思维模式——这种思维模式认为，你不仅需要考虑如何使用你的网站，还要考虑谁会使用它。

谁是可访问性的主要参与者？你经常访问哪些网站？

网络无障碍世界的中心是 W3C 的[网络无障碍倡议](https://www.w3.org/WAI/)。国际 Web 规范联盟的这一分支致力于为 Web 开发人员、浏览器程序员和创作工具创建者创建指南和建议，以提高残障人士的访问能力。

对任何为美国联邦政府工作的人来说，特别重要的是由准入委员会运营的第 508 条款网站。Section 508 是一项联邦政策，它规定美国政府网站(和其他计算机系统)必须对残疾人(包括机构员工和公众)具有可访问性。针对网站的第 508 条规定是对 W3C 网站内容可访问性指南的部分改编。

两个很好的信息网站是[网络无障碍(WebAIM)](http://www.webaim.org/) 和[国际互联网残疾人资源中心(ICDRI)](http://www.icdri.org/) 网站。WebAIM 有大量的教程、演示和研究文章，而 ICDRI 有大量关于残疾用户使用 Web 的政策链接和信息。

威斯康辛州的 Trace 中心、T2 大学的 ATRC 和 T4 的应用特殊技术中心也都是很棒的地方。更多链接可以在[感知中心的网站](http://aware.hwg.org/)上找到。

SP:你能给那些想参与、倡导和教育网页可访问性的网页开发者什么建议？

学习、创造和分享。这就是神奇的公式。

通过访问上面列出的网站、参加网页可访问性课程或加入网页可访问性邮件列表，了解更多关于网页可访问性的信息。好的列表由 HTML 作者协会、WebAIM、Trace 和 W3C 管理。

将这些原则运用到你自己的网站上，将你激动人心的视觉设计与网页可访问性原则相结合，让你将这些设计扩展到所有的受众。

通过告诉其他 Web 开发人员、您的同事和您的客户为什么每个人都可以访问 21 世纪的 Web 是如此重要，来与他们分享。这不是关于教条的 HTML 验证或关于简化网站——这是关于像你或我这样的真实的人，他们像我们一样想使用网络。

我非常推荐去认识一些有残疾的用户。加入有特殊需求的用户的网络或电子邮件群，或者在你自己的社区中认识一些人。告诉他们什么容易使用，什么很难。获得对你自己和他人网站的评论。不要仅仅依靠一些 W3C 文档或书籍来告诉你如何创建可访问的网站。可访问性是关于人的，我们永远不应该忽视这一点。

我饶有兴趣地读到你正在写一本关于 CSS 的书。你能解释一下这是怎么发生的吗？你对有抱负的作家有什么建议吗？

我已经做了几年的技术编辑了，并且很享受这份工作。我也有机会为各种书籍写了一些章节，其中大部分你可能没有读过，也可能永远不会去读。

在最近一次科技编辑之后，SAMS 和 Studio B 几乎在同一时间找到了我，让我提交一份关于《24 小时自学 CSS》一书的提案，Studio B 是一家专门从事计算机书籍的文学机构。虽然不是每个人都喜欢有经纪人，但我发现 B 工作室的人在财务谈判中很有帮助，这对我来说一直是一个不舒服的障碍。SAMS 和 B 工作室的人都很好，对我的第一本书很有耐心。

在我的第一本书还没写完的时候，就给有抱负的作家提建议，我觉得有点奇怪，但无论如何我会试试。先要擅长写作。接下来要善于研究。最重要的是，遵守纪律，坚持写作。

最后，互联网可访问性的下一步是什么？你能预测 2 年后或 5 年后的情况吗？XML 会改变未来事物的面貌吗？

互联网的可访问性仍然是一个非常年轻的学科，在研究领域的发展过程中，它经历了许多成长的烦恼。在接下来的几年里，我们将会看到更多的其他学科，从图形设计到可用性，融入到易访问性知识的核心体中。我们对某些残疾类型的理解，如患有阅读障碍或光敏性癫痫症的用户，将随着我们对可访问性的理解而扩展，这不仅仅是“盲人的替代文本”

互联网在美国和国际社会各个方面的发展只会增加所有人的无障碍需求。随着越来越多的老一代人上网，网站开发人员将需要了解人们随着年龄增长所经历的与年龄相关的变化，包括生理和心理上的变化。你会为 1986 年出生的用户和 1926 年出生的用户设计同样的网站吗？

互联网服务的国际增长将导致更广泛的人群，他们的许多需求将通过使残疾用户能够访问的相同技术来满足。在一个为广大受众设计的网站上，连接速度慢、访问费用低或英语水平低都是需要解决的问题。

XML 已经开始改变 Web 可访问性的前景，从 SVG 和 SMIL 到 XSLT 和 XForms，我们将继续看到 XML 在可访问性中发挥关键作用。在 W3C 开发“语义网”的过程中，虽然倾向于盲目教条主义，但也有可能产生一些特别巧妙的创新。

这些年来，我一直在关注一项技术的发展(并对其有所帮助),那就是适应性网站的理念，它可以根据用户的需求进行自我调整。这可以通过多种方式实现，从简单地在数据库驱动的站点上提供不同的模板选择，到使用 CC/PP 驱动的 XSLT 来动态创建独特的用户界面的纯 XML 解决方案。这种适应性让我们 Web 开发人员能够鱼和熊掌兼得——为视觉用户设计一个最佳的外观，同时还能确保所有残疾用户获得一个为他们的使用而优化的网站。

十年前网络几乎不存在。网页可访问性的概念在 5 年前几乎不存在。随着网络的不断变化，接下来的几年将是至关重要的，这不仅是为了目前有残疾的人，也是为了我们这些没有残疾的人。随着年龄的增长，我们能够使用我们建立的这种新的信息媒介吗？我当然希望如此。为了每个人的利益，让我们把它建设好。
