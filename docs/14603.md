# 风险管理和开源

> 原文:[https://www.sitepoint.com/risk-management-and-open-source/](https://www.sitepoint.com/risk-management-and-open-source/)

开源开发的一个更棘手的部分通常是在商业闭源解决方案中利用开源代码和应用程序。开发人员的目标通常是通过使用经过验证的开源组件或已经过公众审查的应用程序来缩短应用程序的开发生命周期。

这对于内部专用解决方案(如专用企业内部网或外部网)来说可能不是问题。如果这种解决方案被转移到合作伙伴、客户或公众的分销或销售中，事情会变得棘手。如果完整的解决方案不打算作为开放源码发布，那就更棘手了。

澄清:我曾经参与过一个混合源代码项目，当我注意到一个包含开源但作为封闭源代码进行商业销售的解决方案时，我认为该解决方案将遵守所包含的那些组件的开源许可证，并为这些组件提供应用程序的源代码，同时不公开任何旨在保留所有权的代码。

知识产权(IP)管理、风险管理和代码托管已经以某种形式存在了一段时间。这些领域处于不同的成熟水平，并经常随着法院系统设定判例进行解释而发展。

然而，法律系统是昂贵的，对于寻求降低风险的公司来说，软件制造商已经提出了一个潜在的解决方案。

[黑鸭软件](http://www.blackducksoftware.com/)周一宣布为软件开发者发布一款新的风险管理工具。protexIP/Development 是一个软件工具，它与开发环境(包括 cvs 风格的存储库)相集成，并包含一个开源许可证知识库，可以检查项目参数和代码以找到冲突实例。

然而，这并不是一个便宜的工具，对于任何用开源组件开发和分发商业 web 应用程序的人来说，这都是值得回顾的。费用是每年每个座位 2500 美元。

此外，该公司还托管 protexIP/Registry，这为开发人员提供了向 Black Duck 安全提交项目简介的机会，一旦确认没有知识产权或其他许可证冲突，就会被列入反映其合规性的注册表中。这有助于客户关系以及 web 应用程序的销售或部署中包含保险的情况。这项服务的成本是每个项目 1000 美元。

虽然我并不支持这些产品，但它是第一个将商业软件和开源软件结合在一起的产品，并显示了这两者的结合在应用程序开发中变得多么普遍。
