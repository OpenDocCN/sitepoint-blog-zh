# 丹·切尔，不插电。

> 原文：<https://www.sitepoint.com/dan-cheail-unplugged/>

考虑到丹曾经运营过 RubySource，而我已经在几周前的一次 RORO 会议上见过这个名字，所以在这个周末能够更好地了解他是一件好事。此外，我觉得*不插电*的标题是合理的，因为他整个周末都在弹木吉他。他很友好，因为他很高，比一般人高，所以他很乐意回答我们一些问题！

告诉我们一些关于你自己的事情，丹。

我六岁的时候开始用 BASIC 编程，从那以后，我逐渐发展到 basic C、Pascal、Visual Basic，直到大约 95 年我发现了 web，并认为“对，这是我的”。开始构建静态网站，1999 年开始学习 PHP，大约五年前发现了 Ruby。我大半辈子都在写代码，这似乎是我擅长的事情，所以我决定以此为职业。

更具体地说，你从事 Ruby/Ruby on Rails 项目有多久了？

大约四到五年后，我记得检查了 Rails 的第一个测试版——0.8 或类似的版本——认为它非常酷，然后开始尝试用 PHP 实现它的酷部分。然后我意识到直接使用 Rails 可能会更好。于是我坐下来，学了 Rails，从此一去不返。

你之前给出了一些见解，但你还深入研究过哪些其他语言？

PHP，我做了五年的 PHP 开发人员，这可能是我当时接触到的最好的东西，我用它做了很多。在此之前，HTML 和 CSS，我一直在使用 CSS，因为它首次在 IE3 中可用。我记得当 Netscape 1.2 推出背景图片时，我对它的酷感到震惊。那段时间左右，我上学的时候也在做一些 Visual Basic。在那之前，我自学了 C 语言和 BASIC 语言，在学校学了一点 Pascal，所以什么都学了一点。

你最喜欢 Ruby/Ruby on Rails 的哪一点？迫使你想要在其中发展的东西。

我真的只是喜欢 Ruby 语法。Ruby 语法和它的 [Duck Typing](https://www.sitepoint.com/making-ruby-quack-why-we-love-duck-typing/) 方面。这是一种非常描述性的语言。这真的是一件令人兴奋的奇怪事情，这是我经常说的事情，但 Ruby 让我用大脑思考的方式表达自己，我不必重新思考事情来编码 Ruby。想法往往直接从我的大脑通过键盘，它的工作。我思考问题的方式，我解决问题的方式，Ruby 让我自然地去做，而不必回头去想语法或人工构造。这真的让我很感动。考虑到我一天花了九个多小时在上面工作，我需要享受它。与我多年来不得不使用的许多其他语言相比，它使编写代码变得愉快和容易得多。红宝石是一个奇妙的东西。

我想为有抱负的 Ruby 开发人员记录的一件事是，当你学习 Ruby 时，有没有什么大的感悟可以帮助其他人学习？

有几件事。第一个是理解 Blocks 和 Lambdas，以及匿名函数的概念和其中包含的元编程，事实上使用 Ruby，您可以在其他语言中无法调用的地方调用代码……确切地了解这在 Ruby 中是如何工作的令人大开眼界。一个具体的例子是理解“has_many”和“owners _ to”调用实际上是如何在幕后工作的。意识到它们不仅仅是魔术，它实际上是在类内调用方法，在类本身上，而不是在实例上。从概念上来说，理解这一点是一个巨大的飞跃。

除了鸭子打字方面，我可以向任何东西发送消息，我不需要担心它是什么，我只需要担心它是否会响应消息。以及它实际提供的灵活性。它允许我编写一些更智能的代码，我喜欢在我的工作中充分利用这些代码。
