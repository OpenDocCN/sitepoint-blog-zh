# 英国 PHP 大会–2006 年 2 月 10 日

> 原文：<https://www.sitepoint.com/php-conference-uk-10th-feb-2006/>

二月在伦敦？查看英国 2006 年 PHP 大会，该会议在[滑铁卢站和南岸大学](http://www.phpconference.co.uk/info/maps.html)旁边的 [Keyworth 中心](http://www.phpconference.co.uk/2006/info/venue.html)举行。

这个组织要感谢[马库斯](http://www.lastcraft.com/about.php)和 [PHP 伦敦用户组](http://www.phplondon.org/wiki/PHPLondonConference)的成员。价格(与大多数会议不同)是你实际上可以负担得起的——如果你提前注册，50.00 英镑(约 90 美元)——你需要这样做，因为会场的空间有限。

这是一个为期一天的活动，演讲者是[吴镇男·雷森斯](http://www.derickrethans.nl/)、[帕维尔·科夫斯基](http://www.phpconference.co.uk/2006/speakers/pawel_kozlowski.html)、[克里斯多夫
昆兹](http://www.christopher-kunz.de/)、[马特·赞斯特拉](http://www.phpconference.co.uk/2006/speakers/matt_zandstra.html)([PHP 5 对象、模式和实践](http://www.phpkitchen.com/index.php?/archives/711-PHP-5-Objects,-Patterns,-and-Practice.html)的作者)以及，呃..事实上是我。

**需要帮助…**

这是一个内幕消息——到目前为止，我一直在给马库斯施加压力，因为他没能按时发表演讲摘要(对不起，马库斯)。延迟的部分原因是第二次成为父亲(是个男孩)。问题的另一部分是，我仍然拿不定主意该做什么演讲，需要一些帮助来决定。

我想做的演讲应该有一个类似“用 PHP 解析”的标题；一些基础理论，在 PHP 中有效的不同解析策略，以及对 Dokuwiki 解析器的参考。但是有一个问题:作为一个主题，它可能是“有价值的”,但是它在最好的有趣和最坏的乏味之间摇摆不定。至少我是这么看的。

Marcus 告诉我的另一个话题(至少在 PHP 伦敦成员中)是关于 AJAX 和 JPSpan T1 的。现在，虽然这是一个更热门的话题，但我说的任何关于它的东西都很有可能倾向于“AJAX sux”，这不是很 [Web 2.0](http://www.oreillynet.com/pub/a/oreilly/tim/news/2005/09/30/what-is-web-20.html) / [超级爱好者](http://www.artima.com/forums/flat.jsp?forum=106&thread=141312)，是吗？当然，它会变得非常有趣，这取决于你的观点…

那么，哪一个？需要尽快做出决定，因为马库斯需要我尽快提供一份总结。
