# OSCON 2007:整体程序员

> 原文:[https://www . site point . com/oscon-2007-the-intelligent-programmer/](https://www.sitepoint.com/oscon-2007-the-holistic-programmer/)

Adam Keys 是一名软件开发人员和作家。他的博客在 therealadam.com。

一个人在自己的车库里把所有的硬件和软件组装在一起的日子已经一去不复返了——使用计算机越来越意味着在硬件和最终用户之间的某一层做一些非常专业的事情。

亚当提出了一个很好的观点，即尽管有这种专业化，但与计算机打交道的人应该至少对他们自己以外的层有一点熟悉——例如，一个在数据库工作的人学习印刷术，一个网页设计师学习编译器。

许多开发人员可能没有兴趣(或纪律)花时间学习与他们无关的主题，但是 Adam 描述的这样做的几个好处看起来很有吸引力:

*   能够与你的同事交流，更好地理解你的工作与他们的工作是如何相互配合的。
*   获得一个新的视角，让你在自己的领域找到意想不到的或创造性的解决问题的方法。

Adam 在演讲的最后讨论了刚刚提到的两个示例层——CSS/布局/排版和编译器。这可能不是每个开发人员都非常有兴趣了解的东西，但是对于像我这样一个从数据库层一直到面向最终用户的设计的人来说，这确实是非常有趣的东西。
