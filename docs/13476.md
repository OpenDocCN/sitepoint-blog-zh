# 让我们推测一下 2007 年

> 原文:[https://www.sitepoint.com/lets-speculate-for-2007/](https://www.sitepoint.com/lets-speculate-for-2007/)

在节日活动和鸡尾酒会之间，一些人会对未来的世界做出全面的预测，然后写下来。下面是我的思考。

。特定于网络的预测，排名不分先后:

*   WPF/E 将开始在富互联网应用领域与 Flash/Flex 一较高下。然而，Flash/Flex 仍将引领市场。在被微软打败之前，Adobe 很可能会认真地让 Flex 的后端成为一个更可行的开发环境。
*   Office 2007 & Vista 将推动桌面应用程序的重大变革。主要的应用程序——尤其是界面层——将开始用托管代码编写。用户界面设计将开始向更新、更友好的方式转变。图标在里面，长长的下拉菜单在外面。
*   带有 IIS7 的 Longhorn Server 将使 Windows 成为一个更可行的共享主机选项。
*   NHibernate 1.2 出来好评如潮。并立即被 MS 支持的 DLINQ 生吞活剥。
*   除了 NHibernate 之外，MS 将会吃掉另一个主要的开源项目，推出一个免费但受到支持和祝福的替代方案。

对 2007 年 web 应用世界的一些其他预测:

*   Ruby on Rails 将会经历一连串的成长之痛，但最终会找到它的航程，并开始被考虑用于重大的企业项目。
*   AJAX 将失去其迷人的基座，世界将意识到它只是盒子里的另一个工具。
*   视频仍将是网站上最热门/最有特色的项目。有人会想办法让 flash 视频好看。
*   随着新的基本安全问题的出现和竞争对手的成熟，PHP 将继续失宠。当一些人意识到根本重写是拯救语言的唯一方法时，fork 有很大的潜力。
*   使用 SQL 注入或跨站点脚本，而不是底层堆栈中的缓冲区溢出，可以攻击一个主要的 web 应用程序。
*   基于 Windows Mobile 的设备将开始接管企业移动电子邮件领域，原因是灵活的 Exchange 集成以及不需要从 Research in Motion 购买黑莓企业服务器。
*   夏令时的变化给嵌入式系统带来的问题要比上个世纪大肆宣传的 2000 年问题多得多。

新年快乐
