# 一个英国字谜

> 原文:[https://www.sitepoint.com/a-googlish-puzzle/](https://www.sitepoint.com/a-googlish-puzzle/)

可能已经到了这样一个地步，每个人都听厌了谷歌的面试过程，但最近听说了一个很好的问题，有人被要求解决——其中一个问题是“你能像程序员一样思考吗，T2？”类型问题。事情是这样的…

你有两个鸡蛋。这些是特殊的鸡蛋——它们能承受比普通鸡蛋更多的惩罚。但问题是他们能承受多少惩罚？

使用一座 100 层高的建筑，只有两个鸡蛋，在鸡蛋打破之前，你如何找出你可以从建筑的最高楼层扔下鸡蛋？

可能是 1 楼，但也可能是 99 楼——你不知道，但为了测试，你需要尝试从不同的楼层扔鸡蛋，看看会发生什么。

从技术上来说，没有“正确”的答案，尽管人们期望的是一种无论结果如何都需要最少测试次数的方法——一种无论结果如何都能提供一致性能的高效搜索算法。如果你正在努力寻找灵感，试试这里的——没有直接的帮助，但可能会激发一些横向的思考。

从某种程度上来说，如果不是谷歌在问，这样的问题现在有点不公平。随便挑一个数字，我猜想今天 95%的编程都与解决这类问题无关——它们都隐藏在 API(或数据库)后面，在 99%以上的时间里提供“足够好”的性能。如果你对“完成了吗？”的回答让大多数雇主没什么印象是“没有，但我最新的搜索算法正在踢”。

无论如何，在[等待哨声](http://fifaworldcup.yahoo.com/06/en/w/match/template.html?id=1&day=09&month=06&year=2006)的时候，一些不太*太*沉重的事情需要思考。
