# 设计一个优秀的客户端登录系统

> 原文:[https://www.sitepoint.com/great-client-login-system/](https://www.sitepoint.com/great-client-login-system/)

自由网站设计师、小型网站设计公司和大型开发企业应该考虑为他们的客户使用客户端登录页面。

客户端登录可以让您的客户端在一个方便的地方访问所有可用的功能，最大限度地减少客户端和您自己的麻烦。它允许您对机密信息保密，同时允许您的企业和客户在线协作。它可以大大增加你在客户眼中的可信度，同时减少沟通失误的机会，使客户管理过程更加顺利。

让我们看看您应该考虑在您的客户端登录系统中包含的功能。我们还将解决您可能遇到的一些关键问题，并探索如何克服它们。

##### 开始之前

您的客户端登录页面并不总是需要一个复杂的系统，但是有一些因素需要在您开始之前仔细考虑。从长远来看，现在注意这些细节可以为你节省很多时间。

***储存***

您需要存储某些客户端信息以便访问它，这样用户就可以轻松地更改设置，而不必每次在网站上做一些事情时都输入他们的姓名或电子邮件地址。在你开始开发之前，选择你喜欢的存储方法，并坚持下去！不要在 MySQL 中存储一半的信息，其余的使用平面文本文件。选一个，用一个。以下是一些储物方法的想法:

*   关系型数据库
*   Microsoft Access
*   平面文本文件

MySQL 非常出色，因为它安全且易于访问。这也是 PHP 程序员的首选方法。Microsoft Access 与 ASP 配合得非常好，对于更有组织的存储非常出色。平面文本文件一点也不安全，对于你不想让黑客看到的信息来说，这不是一个好主意。

您从上面的建议中选择的方法是一个简单的决定—您需要做的就是选择您的服务器最支持的选项。然而，你必须小心决定储存什么。您可能希望存储您拥有的客户的所有详细信息，但是如果黑客进入，他们将可以访问一些潜在的敏感和机密信息。因此，将您存储的信息量限制到最低限度，例如:

*   **客户网站名称** —其网站的全称
*   **客户的网站地址** —他们网站的 URL
*   **客户的全名** —这样你就可以在每一页亲自问候他们
*   **客户 ID 号(如果适用)** —用于支持请求和通信
*   **顾客已经购买的所有服务** —以及他们已经付费的服务

不要在你的网络空间、数据库或任何潜在黑客可以找到的地方存储纯文本密码。如果你必须存储它们，加密它们。看看 [MySQL 密码函数](http://www.mysql.com/doc/en/Miscellaneous_functions.html#IDX1330)，以及 [PHP 函数 MD5](http://www.php.net/manual/en/function.md5.php) 和 [crypt](http://www.php.net/manual/en/function.md5.php) 帮助保护密码。

 *确保客户信息页面的安全是需要记住的一个重要部分。它不仅能保护黑客，还能让页面知道谁在使用这个页面。正如我之前所说的，您可以为用户访问的每个页面赋予个人风格(例如，“欢迎，Mickiewicz 先生”)，当用户想要更改他们的设置或信息时，他们可以这样做，而不必每次都输入他们的用户名和密码。

有许多方法可以验证您的用户，我不会说得太详细，但这里有一些建议:

*   **.htaccess**
    You know those login boxes that pop up in your browser sometimes? This can be done with minimal fuss using .htaccess authentication. For more information on this, see Kevin Yank’s [Apache HTTP Authentication in PHP](http://webmasterbase.com/article/280).

    然而，这涉及到一定的安全风险——用户名和密码都是以明文形式发送到服务器的，很容易被“劫持”。

*   **PHP/MySQL 认证**
    这是我推荐你用于客户端登录页面的方法。它非常灵活，可定制，不像那些丑陋的。htaccess 弹出框。参见凯文·杨克的文章《用 PHP 会话和 MySQL 管理用户》,了解更多细节。

##### 去画板那儿！

现在，在纸上画出一个计划。概述你想要的链接、颜色和文本，以帮助你设计你的页面。你的计划不需要简洁——你不需要写出你要包含的全文，也不需要概括地勾勒出每张图片；只是概述一下每个地方会有什么就可以了。

开始之前的最后一步是决定您将在客户端登录页面中包含的功能。我在这里整理了一些建议，供你仔细考虑。

***网站控制面板访问***

如果你在设计服务中提供网站托管，你应该提供一个链接让用户登录他们网站的在线控制面板，在那里他们可以设置和管理账户，电子邮件地址等。

***更新简介***

例如，如果用户更改了地址或电话号码，您可以提供一个表单来更新他们的客户资料。如果您决定不在数据库中存储客户的全部信息，那么您可以设置一个表单邮件程序，通过电子邮件将表单内容发送给您，以便进行手动更新。

***工作进展页面***

让你的客户看到你已经完成了他们的项目。这可能包括一个日记风格的脚本，你可以张贴网站的主要部分已经完成。网站的链接也可以包括在内，还有一个时间表，这样他们就知道网站什么时候应该完工。

***支持区域***

这一部分可能包括客户如何联系您的详细信息，如果他们有与他们的网站相关的疑问或问题。可以包含一个简单的邮件表单，让他们不用离开页面就可以给你发邮件——他们甚至不需要 Outlook 或 Eudora 这样的邮件客户端！你可能也有客户支持论坛——如果是这样，从这里链接到他们。

***内容管理***

如果你已经为你的用户提供了一个内容管理系统，那么包含一个直接到管理面板的链接会很方便，这样他们只需点击一个按钮就可以添加新的内容。

***Webmail 访问***

你可以安装一个简单的网络邮件系统，这样你的客户就可以通过网络查看他们的电子邮件。Hotscripts 有一个很好的免费和授权脚本列表。

***域名到期和计费数据***

让您的客户知道他们的下一次付款何时到期，或者他们何时需要更新他们的托管帐户。

***服务器状态页面***

如果你为客户提供自己的主机，你可以包括一个服务器状态页面，用户可以在那里查看和报告停机时间。

***注销链接***

这听起来是显而易见的，但是你不能忘记这一点，尤其是如果用户在共享的计算机上。同样值得考虑的是“更改密码”页面。

当然，您可以在您的客户端登录系统中包含许多其他的可能性。请记住，由于大多数网站设计公司的服务随着客户需求的不同而不同，因此可能很难提供某些领域的链接。例如，如果你所做的只是为客户设计一个徽标，那么链接到托管控制面板是没有意义的:控制面板对他们来说是不存在的，他们会得到一个 404 错误。

解决这个问题的一种方法是建立一个单独的带有“是/否问题”的数据库表或页面。例如，字段名称可以包括“has_hosting”、“has_domain”、“has_cms”，并且每个用户的值可以根据他们实际拥有的内容而有所不同。然后，您只需要在页面上显示与特定用户相关的链接。

##### 获取编程！

说够了，你甚至还没有开始制作你的客户端登录页面呢！在考虑了我们已经讨论过的所有特性之后，你就可以开始你的编程、设计以及任何你需要做的事情了。然而，有些事情你还是需要记住的——我知道，太多了，但是你想让它看起来很好，很专业，并且正常工作，不是吗？

##### 可以光顾

光顾你的客户是可以的。大多数人雇佣专业人员来制作他们的网站，因为他们不知道怎么做。因此，他们不太可能成为网络专业人士。你可能需要向你的客户灌输“外行术语”和基本信息，这样他们才能准确理解你的意思。

例如:不要“点击这里进入你的 MySQL 驱动的基于 Web 的 PHP 内容管理系统管理面板”，而是考虑“点击这里进入你的网站的内容管理控制面板”。

##### 一致性是关键

你所有的网页需要有相同的布局，颜色，字体等。以减少用户的困惑。许多主机控制面板使用框架，因为这确保了一致的布局，所以你也可以考虑它。

事实上，只有少数用户能够看到你的登录系统，这并不是让它看起来无聊和匆忙的借口。努力让你的设计更有吸引力，更实用，你会发现你的客户会希望你回来。

我在本文中讨论的因素应该让您了解在制作客户端登录页面时应该考虑的功能。当然，由于所有的网站设计公司几乎在每个方面都是不同的，所以肯定会有其他的因素，你会想把它们加入到我们在这里讨论的基本系统中。我希望这激发了您创建新的客户端登录页面。祝你好运！
