# 过去和未来的活动:Adobe Max 北美和 CFCAMP 澳大利亚

> 原文:[https://www . sitepoint . com/events-past-and-future-adobe-max-north-America-and-cf camp-Australia/](https://www.sitepoint.com/events-past-and-future-adobe-max-north-america-and-cfcamp-australia/)

大多数 ColdFusion 程序员都听说过本·福塔(Ben Forta)这个自安拉时代以来的 CF“大师”。作为权威的“ColdFusion Web Application Programming Kit”书籍的至少六个版本的作者，Ben 在许多 CF 商店的办公室中被亲切地称为 CFWACK 或“圣经”,他作为 Adobe 的高级技术布道者在世界各地旅行，讲述和撰写有关 cold fusion 的内容。刚从 [Adobe MAX Europe](http://www.adobemax2007.com/) 会议(在西班牙举行)回来，他接下来将前往 Adobe Max Japan，然后在 11 月下旬前往澳大利亚参加一系列关于 ColdFusion 8、Flex 和 AIR 的活动。名为 CFCAMP 的澳大利亚活动将部分是正式介绍，部分是无会议，barcamp 风格，所以如果你在那里，他们应该值得一去——[注册现已开放](http://cfcamp.eventbrite.com/)。

在欧洲之前，马克斯于 10 月底在芝加哥开球。凯·柯尼希，新西兰的开发者，[博客作者，](http://www.bloginblack.de/) [Ventego 创意公司的培训师兼总监，他也在现场，并亲切地回答了我关于这次活动的一些问题。](http://www.ventego-creative.co.nz/)

凯:参加 Adobe MAX 2007 一定很令人兴奋。前几年你去过 MAX 吗？如果去过，在 Adobe 旗下有什么不同吗？

**Kai:** 是的，我参加的第一个大型(当时)Macromedia 会议是 Macromedia DevCon 2002。从那时起，MAX 会议就成了我必去的地方。

在过去的几年里(尤其是在 2006 年)，一些人避免参加 MAX 会议，因为他们觉得这是一个太多的企业，纯粹由市场驱动的事件。在某种程度上，我不得不同意这一点，Macromedia 和 Adobe 显然正在努力展示他们的技术，并说服人们采用 Adobe 平台的更多产品。但我建议不要高估这种影响——到目前为止，大多数演讲者都有很高的积极性，并像你我一样热爱技术，而且有大量的会议和学习内容可供选择。

凯:根据网站上的信息，有 200 多节课被分成了五个部分。Adobe 显然有很多产品要介绍 ColdFusion 的表现如何？很难决定去哪一个疗程吗？

**Kai:** 我先回答第二个问题:是的，很难打电话决定参加什么:)幸运的是，我一直和另一个人一起参加，所以 [Diane](http://www.humanfactors.co.nz/) 和我能够涵盖一整套不同的会议/技术。

当谈到 ColdFusion 时，您会惊讶地发现有多少与 CF 相关的会议。此外，我们还将重点讨论 CFC、定制标签、打印等特定的 CF 特性。还有很多会议展示了如何使用 ColdFusion 作为集成平台，例如，作为基于 Ajax 和/或 Flex 的富客户端的后端，将 CF 与 PDF 工作流引擎 LiveCycle 挂钩等。

就与会者而言，大多数与会者在注册过程中将自己描述为 ColdFusion 开发人员。你会遇到那些一直以这样或那样的方式与 CF 一起工作的人。

凯:对你来说最重要的是什么？

**Kai:** 嗯，真的很难说。关于展示的新产品和材料，我的亮点是先睹为快环节的一部分。Adobe 内部的计算机科学家之一 Scott Petersen 展示了一个工具，可以将 C 和 C++代码库转换为 ActionScript，以便在基本上任何 AS 3 编码中使用它们(通过一个特定的工具包将它们与 SWF 挂钩)。他展示了一个在 Flash 播放器中运行 Doom 的例子，在 Doom 中使用基于 Flash 的覆盖消息，即当玩家拿起武器时。大多数人不知道如果这种类型的集成公开可用，Flash 平台会突然变得多么强大，他也没有得到他应该得到的起立鼓掌和欢呼。

我的另一个非技术亮点是 MAX 大会的网络方面。在过去的几年里，我参加了很多社区活动，到处参加会议，一路上你“结识”了很多不常见面的朋友——仅仅因为他们住在不同的大陆或世界的另一端。马克斯总是一个很好的机会，可以叙叙旧，喝几杯啤酒，看“奇怪的”美国体育赛事(抱歉，美国人——但我在德国长大(足球)，现在住在新西兰(板球、橄榄球)，对我来说很难获得所有的兴奋:-)

哦，是的，一大堆东西。除了 C/C++-2-AS 转换器之外，Adobe 在将 Flash 平台和 Acrobat Reader 结合在一起时表现出了巨大的集成努力。他们演示的一个例子是在 Adobe Reader 中执行一个 Flex 应用程序，另外使用 Connect collaboration 技术，这实际上允许您在文档中工作。我们还看到了 ColdFusion 和 AIR 使用实验库进行的非常好的透明集成(有望在某个时候成为 CF 8 更新程序或 CF 9 的一部分)。

此外，Adobe 向托管服务的转变相当惊人。收购 Buzzword guys 已经足够棒了，但 Adobe 还致力于托管 Photoshop Express 解决方案(基于 Flex ),并对 Connect 平台的 VoIP 层进行了巨大改进。

凯:我们总是听说会议上的聚会——有什么好故事给我们吗？

**凯:**当事人？嗯——不知道你在说什么。显然，我不会在这里进入个人细节，但官方会议派对是某种复古未来主义的极客聚会，有来自 80 年代、90 年代和这十年我们喜欢的所有东西——谈论街机游戏和其他东西。所以，可能有一些“非工作”事件:)

凯:最后，我们不断听到谣言说 ColdFusion 已经死了。作为 ColdFusion 开发者，你担心吗？

凯:是啊，没错。这似乎是永远不会消失的谣言。我很清楚，自从 CF 4 或 cf5 以来，很多人已经不再关注技术的发展，甚至不太清楚 CF 8 能够提供什么。我自己业务的很大一部分都是基于 ColdFusion 的，我对此非常满意——这并不意味着我们不研究其他技术，例如 Ruby on Rails 非常有趣，基于 Java 的脚本语言 Groovy 也非常有趣。

在 MAX，我与蒂姆·邦特尔(蒂姆刚刚成为 Flex Builder 产品经理)和克里斯汀·施菲尔德(他将接替蒂姆在 CF 团队中担任产品营销经理的职位)就 ColdFusion 在 Adobe 内部和外部的情况进行了一次非常有趣的交谈，我非常有信心我们将看到 ColdFusion 的更多主要版本。特别是当 CF 与其他 Adobe 平台进行更深入的集成时，我预计在未来几年内会发生一系列非常酷的事情。

当人们选择反对 CF 时，一个总是出现的争论是价格和它不是开源的事实。正如你在最近的一篇文章中正确提到的——为预算非常少的项目获得 CF 托管，或者考虑使用替代的 CFML 引擎——Railo、IgniteFusion、BlueDragon 等等。您可能无法获得与 Adobe 的 ColdFusion 服务器完全相同的功能，但它可能非常适合您使用简单和标准的 web 应用程序。

说明一点:不，我一点都不担心 CF 会死。不过我注意到，ColdFusion 在澳大利亚和新西兰的销售数字(根据我的经验，在欧洲会非常相似)低于他们应有的水平，而 CF 8 正在打破美国的所有销售记录。这是 Adobe 必须尽快解决的问题，因为我觉得他们在过去几年中没有很好地照顾这些市场，但就我对我们澳大利亚人来说，我已经注意到事情开始有所进展，我希望在未来看到更多的改进。

凯:很高兴听到这个消息！谢谢凯，希望明年我也能来。
