# 澳大利亚政府支持 WCAG 2.0

> 原文：<https://www.sitepoint.com/australian-government-wcag-2-accessibility/>

澳大利亚政府已经宣布了他们遵守网页内容无障碍指南 2.0 的计划。到 2012 年，所有政府网站都必须达到 A 级标准，到 2015 年达到双 A 级标准。新标准取代了 WCAG 1.0，后者是在 2000 年作为机构的强制性要求引入的。

财政和放松管制部长林赛·坦纳说:

> 这是使民主对所有澳大利亚人更加开放、更易获得和更负责任的重要一步。
> 
> 澳大利亚政府正在逐步实施新的在线技术，并希望与更多的人在线联系。这些新标准将提高患有各种残疾的人抓住这些机会并通过互联网与政府接触的能力。

负责残疾人事务的政务次官 Bill Shorten 补充道:

> 残疾人仍然面临太多阻碍他们参与工作、教育和其他领域的障碍。这一举措将有助于确保残疾人不会被互联网的快速发展抛在后面。

这是一个雄心勃勃的项目。尽管许多政府都有无障碍立法，但很少有人理解其含义或贯彻执行。

## what 怎么了？

Web 内容无障碍指南(WCAG) 2.0 是由[W3C Web Accessibility Initiative](https://www.w3.org/WAI/)制作的文档。

这是一篇令人生畏的文章，尤其是如果你习惯于遵循 WCAG 1.0 版的指导方针。原始文档的大部分是专门针对 HTML 标记的，例如，不要使用表格进行布局，确保不用样式表就可以阅读内容，避免弹出窗口等。

WCAG 2.0 基于更广泛的指导原则。它不一定与技术有关，而是如何使用技术。有四个基本原则:

*   信息和 UI 组件必须是**可感知的**——它们不可能对用户的所有感官都是不可见的。
*   UI 组件和导航必须**可操作**。
*   信息和 UI 组件必须**可理解**。
*   内容必须**健壮**，这样它就可以被各种各样的用户代理解释，包括辅助技术。

每项原则包含 12 条指导方针，每条指导方针包含一系列与 A、AA 或 AAA 合规性相关的成功标准。

幸运的是，有许多支持文档和工具可以让您更容易地遵循这些指导原则:

*   [**如何满足 WCAG 2.0**](https://www.w3.org/WAI/WCAG20/quickref/) —描述如何满足 HTML、CSS 和其他技术的 A、AA 或 AAA 一致性的可定制文档。
*   [**了解 WCAG 2.0**](https://www.w3.org/TR/UNDERSTANDING-WCAG20/)
*   [**WCAG 技术 2.0**](https://www.w3.org/TR/WCAG20-TECHS/)
*   [**WCAG 2.0 与 WCAG 1.0 有何不同**](https://www.w3.org/WAI/WCAG20/from10/diff.php)
*   [**对比 WCAG 1.0 检查站与 WCAG 2.0**](https://www.w3.org/WAI/WCAG20/from10/comparison/)
*   [**如何将你的网站从 WCAG 1.0 升级到 WCAG 2.0**](https://www.w3.org/WAI/WCAG20/from10/websites.html)

看起来好像澳大利亚的开发者有一些睡前读物。但是来自其他国家的开发者不应该太自满——如果你们的政府还没有采用 WCAG 2.0 的话，他们几乎肯定会采用！
