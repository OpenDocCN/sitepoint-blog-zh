# Twitter 告诉第三方开发者放弃他们的应用

> 原文:[https://www . site point . com/Twitter-tells-developers-to-scrap-client-apps/](https://www.sitepoint.com/twitter-tells-developers-to-scrap-client-apps/)

Twitter 的增长是惊人的。每天发送 1.4 亿条推文，该系统继续享有指数级的采用率。这在很大程度上得益于利用公共 API 的第三方 Twitter 客户端。

但不会太久了。Twitter 要求开发者停止创建客户端应用。

根据 Ryan Sarver 在 3 月 11 日发布的官方 Twitter API 公告:

> 我们的用户研究表明，消费者仍然对第三方 Twitter 客户端显示推文并让用户与核心 Twitter 功能互动的不同方式感到困惑。例如，网站或客户端显示推文的方式不符合我们的设计准则，或者当服务将自己的动词放在推文上而不是 Twitter 上使用的动词时，人们会感到困惑。类似地，许多第三方消费者客户端使用他们自己版本的建议用户、趋势和其他数据流，使我们网络中的用户更加困惑。用户应该能够以同样的方式查看、转发和回复@nytimes 的推文。

为了创造“一致的用户体验”，萨弗说:

1.  Twitter 将在电话、电脑和其他设备上提供主要的主流消费者客户端体验。
2.  推文和推文动作必须以一致的方式呈现，以便人们无论在哪里都有相同的推文体验。

他继续说道:

> 开发者告诉我们，他们希望从我们这里得到更多关于在 Twitter 上开发的最佳机会的指导。更具体地说，开发者问我们，他们是否应该构建模仿或复制主流 Twitter 消费者客户端体验的客户端应用。答案是否定的
> 
> 如果您是现有的客户端应用程序开发人员，您可以继续为您的用户群提供服务，但我们将对您保持高标准，以确保您不侵犯用户隐私，确保您提供一致的用户体验，并严格遵守我们服务条款的所有方面。我们已经与 Twitter 生态系统中的主要客户端应用程序持续讨论了这些需求，并将继续确保保持高标准。

这个帖子引起了愤怒:*为什么 Twitter 背弃了培育它成长的社区？*

这引发了几个问题。

**1。为什么要提供 API？**
Twitter 的公共 API 允许开发者以公司从未设想过的方式使用网络。随着操纵和呈现数据的新方法的出现，不一致性肯定会出现。这是一件好事。

如果 Twitter 担心网络的发展，他们可以限制或取消 API。他们仍然可以。

**2。用户是否迷茫？**
有很多好的 Twitter 客户端。坏的也一样多。选择是好事。

用户负责安装客户端应用程序。如果他们对界面、术语或方法感到困惑，他们不必使用它——尤其是当有许多免费的替代品时。

还要记住，Twitter 是一项允许用户发布 140 个字符消息的服务。这并不复杂。

**3。Twitter 怕什么？**
萨佛州:

> 根据我们的数据，90%的活跃 Twitter 用户每月都会使用官方 Twitter 应用。

如果很少有人使用第三方客户端应用，Twitter 为什么会担心？

诚然，一些应用程序会违反 Twitter 的隐私政策和服务条款，但 Twitter 声称他们每周都会撤销数百个 API 令牌。这仍然会发生。

**4。他们的统计数据正确吗？**
单看点击量，Twitter.com 永远比第三方应用有更高的使用率:

*   新用户必须在 Twitter 网站注册。
*   许多第三方应用程序通过 Twitter.com 处理用户登录。
*   大约三分之二的新用户从未使用该服务超过几次。巨大的流量是由那些永远不会使用另一个客户端的人产生的。
*   不经常使用 Twitter 的用户更有可能坚持使用 Twitter 自己的客户端。

然而，Twitter 应该关注频繁用户——那些发布最多推文的用户。我怀疑大多数人都在使用替代软件。我不认为自己是一个推特迷，但我很少使用 twitterholic 它比竞争对手更慢，更笨重，功能也更少。

如果关键用户不能使用他们喜欢的应用程序，Twitter 还能生存多久？

**5。潜在的问题是什么？**
这纯粹是猜测，但我猜 Twitter 的声明更多是为了钱，而不是“一致的用户体验”。通过使用他们自己的应用程序，Twitter 可以控制你看到的内容——包括赞助推文、横幅广告和其他促销活动。当你使用第三方应用程序时，这不一定是可能的。

自 2006 年成立以来，该公司已经筹集了数亿美元。风险资本是有成本的，许多投资者希望在 5 年内看到投资回报。Twitter 还没有盈利。它继续根据其潜力而非“真实”估值进行交易。

Twitter 对第三方应用的抱怨似乎很虚伪——它创造了那个市场。该公司应该专注于创造人们真正想要使用的奇妙的客户端应用程序，而不是咄咄逼人地咬食喂养它的手。他们有资金和资源击败竞争对手。

现有的开发者会对这个声明感到失望。考虑自己申请的公司会重新评估自己的未来。Twitter 正在扰乱其热情的社区。这是一个冒险的策略。

你是 Twitter 客户端开发者吗？你是一个喜欢另类应用的重度 Twitter 用户吗？如果没有它，你会继续使用这项服务吗？
