# 一些最受欢迎的开源工具

> 原文：<https://www.sitepoint.com/some-favorite-open-source-utilities/>

在 Mac 的 Tao 和他的开源观察列表中，我受到了 Rui 的启发，思考一些我不能没有的开源工具。

有些被认为是老派，有些是新的，有些只是惊人的生产力件。

首先，我发现我们共享一个应用程序， [Newspipe](http://newspipe.sourceforge.net/) 。它是一个基于 python 的新闻阅读器，处理 RSS 订阅，并将标题、故事摘要和链接发送到您选择的电子邮件地址。我在 OS X 的工作站上用 cron 在后台运行我的博客，简单地回顾一下最新的新闻和我的电子邮件。不需要图形用户界面 RSS 阅读器。它有非常灵活的配置选项，如果你依赖于 RSS 提要，并且不需要打开另一个应用程序，它非常值得一看。它可以在 Linux、Mac 和装有 Python 2.3 的 Windows 上运行。

Webmin 是目前为止我最喜欢的系统管理应用。虽然我在服务器上花了大量时间使用命令行——但是有几个重复的任务使用 Webmin 要快得多。另外，对于多服务器环境，我可以将 webmin 安装在一起，并广播配置更改、更新等。(防火墙内部！).

与大多数控制面板不同，该工具面向系统管理员，因为它可以在根级别访问文件系统和底层敏感配置文件。2004 年，我在 Webmin 上写了一篇深入的文章。

默认情况下， [Vi](http://www.vim.org/) 在这个列表中是因为易于使用(在习惯了在终端编辑器中使用工具之后！)以及在编辑文件和接触配置文件时的快速性。虽然我知道有 Emacs 的粉丝，但我不能没有 Vi。

Tripwire 作为服务器上的优质安全资产胜出——并对 Linux 管理员免费开放。简而言之，Tripwire 会对您的文件系统进行快照，包括您在设置过程中指定的文件和目录。然后，它通过向管理员发送电子邮件并要求管理员通过命令行进行后续批准来保护这些文件，以使更改生效。如果你正在对一台机器进行实质性的修改，安全网系统可以通过额外的认证被覆盖(除非你喜欢阅读大量的电子邮件，否则这很有帮助)。

具体到 Mac OS X，我要表扬一下 Adium T1 的开发者，这是一个多协议即时通讯解决方案，允许我通过 AOL、MSN 和 Yahoo 用户名(它也支持其他用户名)在线。虽然我主要使用 AOL 的即时通讯帐户，但我的一些客户在其他网络上——这些人现在也可以通过即时通讯以及电子邮件和电话联系我。
