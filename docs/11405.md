## 采访记录

凯文:2009 年 12 月 20 日。布拉德·威廉姆斯采访了插件和主题的作者，这些插件和主题将 WordPress 变成了一个社交网络发电站。这是 SitePoint 播客#41:安迪·皮特林的好友新闻。

布拉德:欢迎大家来到 SitePoint 播客的另一集。我是你们的主持人布拉德·威廉姆斯，本周，我邀请到了 [BuddyPress](http://buddypress.org/) 的创始人安迪·皮特林先生。

欢迎来到这个节目，安迪。

安迪:嘿，布拉德，非常感谢你邀请我。

布莱德:当然。所以你为什么不向每个不熟悉 BuddyPress 的人解释一下它到底是什么，它到底是做什么的。

安迪:是的，BuddyPress 的想法是安装一个新的或现有的 WordPress，或者确切地说是 WordPress MU，而不是把焦点直接放在博客上，而是把焦点更多地放在用户身上。因此，你保留了现有的博客功能，但你添加了扩展的个人资料、私人信息、朋友、群组、活动流等功能。所以它试图允许 WordPress 安装上的用户与同一个安装上的其他用户进行社交互动。

布拉德:太好了，是的。BuddyPress 作为一个产品还相当年轻。我的意思是，我知道它已经存在了几年了，但是我想在开源世界里，这是一个比较年轻的项目。

这个项目到底是什么时候开始的？

Andy: 嗯，我是说，我实际上是在 2007 年中期开始有了这个想法，它始于 WordPress MU 的一个客户项目，他们想要建立一个社交网络，但我当时建立了一些插件，这就产生了这个想法，我想让我们做一些开源的东西，让其他人都可以使用。

这个想法就是从那时产生的。我在 2007 年中期开始开发，它在一些早期版本的一些功能构建中获得了一点动力。那时我只是一个自由职业者，所以我没有太多的时间去做它，它也就半途而废了。所以幸运的是，Automattic 选择了我，并雇用我继续全职工作，然后开发真的开始加速了。我想，我们在 2008 年 12 月有了第一个测试版。

开发第一个测试版花了大约 10 个月的时间，然后第二个测试版是在 2009 年 2 月，实际上第一个 1.0 版本是在 2009 年 4 月。实际上，1.0 的最终版本现在才出现，我想，大约 10 个月，9 个月——所以不会太久，甚至还不到一年。

Brad: 你提到了 BuddyPress 的一些特性。让我们更深入一点，更详细地解释一下它们到底是如何工作的；第一个是扩展配置文件。就像你说的，BuddyPress 实际上把焦点放在了用户身上，它本质上为 WordPress 的用户创建了一个社交网络档案，对吗？

安迪:对，没错。BuddyPress 是八个不同特征的组合，其中之一是个人资料。实际上，这基本上允许用户或网站管理员进入并创建用户可以填写的特定个人资料字段。

例如，如果您添加一个关注足球的社交网络，您可能会为用户创建字段。他们可以填写他们的球队名称，他们最喜欢的球是什么，或者他们使用什么品牌的足球鞋。因此，它允许用户登录并填写这些个人资料字段，然后创建一个个人资料，显示给网站上的其他用户。

它还增加了一些其他功能，如头像；他们可以上传自己的头像并显示在网站上。

**Brad:** 这些字段的灵活性如何；我们说的只是文本框还是有更多的内容？我们真的可以做下拉选择菜单和复选框之类的东西吗？

安迪:他们有几个选择。我认为有九到十种不同类型的字段可以放在那里——你有单选按钮字段、日期选择器、文本区域、普通文本框——有一个完整的范围。因此，根据您希望人们填写的信息或您提出的问题类型，您可以选择适当的字段类型。

Brad: 我想这就引出了一些问题，当你在网站上注册并创建你的自定义字段时，是整个网站范围内的还是特定于你所在的 WordPress MU 博客的？

安迪:不，我的意思是，BuddyPress 是一个站点范围的插件，所以当你在 WordPress MU 安装上激活 BuddyPress 时，它会在所有博客上全局激活，你使用你安装的根博客来设置一切。用户在 BuddyPress 安装上填写一个概要文件，该概要文件数据可以在任何博客上访问。因此，您可以在任何博客或用户所属的任何博客上显示个人资料小部件；所以全球都有。

Brad: 所以它几乎把所有的个人博客连接起来，组成了一个大社区。

安迪:没错，是的，它的目标是把所有这些东西粘在一起，让作者之间有更多的互动。

布拉德:太棒了。另一个很棒的功能是私人消息，这在 WordPress 上根本不存在，所以它完全是 WordPress 的一个功能。现在，这就像我们期待的脸书那样，我们可以给其他用户发消息吗？

安迪:嗯，算是吧。这是一种你可以私下给对方发信息的能力，它是以电子邮件的方式完成的，所以你有一个收件箱，一个发件箱，然后你可以撰写，它会在你的任何朋友联系上自动完成。因此，如果你开始输入朋友的名字，它会自动完成，你可以添加更多的用户。它的功能有点像电子邮件，它的功能与你在脸书看到的脸书邮件非常相似。我认为这很好，人们可以私下参与整个网站的对话。

**Brad:** 太好了，你提到了下一个功能，那就是朋友，所以你可以和网络中的其他用户成为朋友。

**Andy:** 没错，所以可以做朋友人脉。你可以用它来联系其他优惠。你可以将朋友添加到你的朋友列表中，然后你可以跟踪这些朋友的活动，在网站上跟踪他们，跟踪他们发布了什么博客帖子，诸如此类的事情。

所以，是的，你请求一个朋友/友谊，然后这个人接受了，友谊就建立了。

Brad: 你必须成为朋友才能给其他用户发消息，还是你可以给你网络中的任何人发消息？

安迪:不，你可以给任何人发信息，但自动完成功能只对朋友的名字有效，所以你可以开始输入朋友的名字，它会根据他们的真实姓名完成。您也可以在文本框中输入用户名，然后向系统中的任何人发送消息。

布莱德:太棒了。下一个特征是群体，我认为这是很多人(包括我)真正喜欢的特征。你能稍微解释一下团队以及他们是如何在 BuddyPress 中工作的吗？

安迪:是的，当然。我真的觉得群组是 BuddyPress 最强大的功能之一，它真的允许你对内容进行分组，并围绕特定的主题开始对话。因此，任何成员或网站成员都可以进入并创建一个群，您可以提供某些字段，如群名称、新闻和群描述，然后每个群都有一个活动流，您可以发布到该活动流。它目前有一条电线，你可以在那里张贴小消息。它也有一个论坛，你可以专门为该组创建论坛主题。然后，用户可以加入这个小组，你可以围绕特定的主题进行对话。

现在在 BuddyPress 中，有一个非常酷的组扩展 API，插件开发者可以进来为组创建新的特性。所以，真的很厉害。我认为这是 BuddyPress 目前最强大的功能之一，并且一直在改进。

布拉德:是的，我认为这可能是 BuddyPress 最大的吸引力之一。现在，就隐私选项而言，有没有办法建立，比如说，私人团体？

安迪:是的，有三个选项可以设置。有一个公共群组，它出现在整个网站上，在该群组中发布的任何内容，人们都可以公开看到，任何人都可以加入该群组。

还有私人群组，任何人都可以申请加入，但必须得到群组管理员或群组创建者的同意，群组内的所有活动实际上都是隐藏的，直到你成为成员。

最后，还有隐藏组，你在网站的任何地方都看不到该组。它不会显示在搜索结果中。你实际上必须被群创建者特别邀请才能加入群，甚至看到它的存在。

三个级别的隐私直接内置到组中。

布莱德:这是默认的。就像你说的……这个团体真的……它的运作很像有成员的脸书团体。

Andy: 这些隐私级别非常相似。

**Brad:** 我注意到在一个组中，你可以有组管理员，我假设是创建该组的用户，然后你也可以有模块。两个用户级别到底有什么区别？

Andy: 一个群管理员，他们有删除群的能力，也有提升群内成员的能力。他们还可以修改其他一些版主无权访问的细节。

如果有一个群组论坛，版主真的是在那里管理论坛的，我相信，他们也可以把用户从一个群组中踢出去，或者禁止用户回到这个群组。因此，他们有一些低级别的访问权限，但诸如删除等权限都仅限于组的管理员。

Brad: 你提到了论坛，那我们接下来就来谈谈吧。BuddyPress 有论坛直接集成在里面？

安迪:是的，它实际上使用了另一个名为 [bbPress](http://bbpress.org/) 的项目，这个项目和 WordPress 是一个思路。所以它使用 bbPress 来支持 BuddyPress 中的实际论坛功能，过去能够集成 bbPress 和 BuddyPress 有点疯狂，但在上一个版本中，引入了一些新功能，您只需单击一个按钮，它就会自动为您设置论坛功能，所以做起来非常简单。它不仅仅是一种全球级的论坛，也不仅仅是与网站的任何社交和互动功能完全分离的论坛，它实际上完全融入了群体。

当你创建一个群组时，你可以选择同时创建一个论坛，然后在该论坛上发布的帖子和主题会附加到该群组上。你可以创建一个关于特定主题的论坛，就像我们刚刚说的，给群组的版主和管理员实际的访问权限来删除主题，粘性和封闭主题。所以都是组内设置的。

**Brad:** 是的，然后是你提到的整合过程，我知道这是很多人要求的一个重要功能，甚至我自己也要求了…我第一次接触 BuddyPress 是在今年 1 月，在那之前，整合过程已经开始了，那是…你还记得有多少步骤吗？我想说大概有 12 到 15 步。

安迪:是的，我想是 15 岁。这有点像你必须…它必须是一个雨天，你必须坐在你的厨房里让它工作，这就像非常具体的场景。就像魔法一样。

布莱德:是的。我让它工作了，但是花了一些时间——你知道我不得不把我的头撞在桌子上一点点，我确实让它工作了，但是是的，一键集成是天赐之物。我知道这个社区真的…

安迪:我认为这对我很重要。你不想开始使用新软件，必须经历所有这些步骤。这只是一个有趣的正确方式来集成一个单独的 bbPress 作为一个单独的项目，并找到正确的方式来简单地集成它…这只是在第一个版本中没有发生，但实际上现在，这是一个点击，人们对此非常高兴。

**Brad:** 是的，所以 bbPress 是一个独立的项目，所以我想这就引出了一个问题，那就是……你是必须下载真正的 bbPress 软件，然后在 BuddyPress 中把它推上来，还是在你下载它的时候会附带 BuddyPress？

Andy: 它实际上是通过 Subversion 作为 BuddyPress 的外部链接。所以，实际上当你下载打包版本和压缩文件时，它会发出请求 bbPress。

布莱德:所以它会抓取最新版本。

安迪:它的确很抢眼……它是 bbPress 的最新标签版本。

我们采用最新的标签，我们知道这将是兼容的，主干的变化不会打破东西。所以，是自动拉进来的。除了进入管理页面，然后点击安装按钮，你真的不需要做什么。

**布拉德:**我就喜欢这样；只要按一下按钮，一切都正常了。

安迪:这是最好的方法。

布拉德:这是整合事物的唯一方法。让我们来谈谈博客，因为 WordPress MU 的主要特点是它允许在 WordPress 的一个安装中有多个博客，BuddyPress 位于 MU 之上，所以显然你仍然有 WordPress MU 的博客功能。那么这是如何融入 BuddyPress 的呢？

Andy: BuddyPress 尽量不去玩弄博客是如何整合的，博客是如何运作的。它有点不去管它，只是试图围绕那个博客来增强。所以如果你已经安装了 MU，它不会影响人们写博客的方式，也不会影响他们进入 WordPress 仪表盘写新文章和组织文章的方式。BuddyPress 实际上在你的安装顶部添加了一个导航栏，它出现在你的所有博客和 BuddyPress 的所有页面上，实际上让用户能够选择“我的博客”菜单，找到他们想要发布的博客，他们可以选择发布新的帖子或更改主题。因此，它允许用户进入，无论他们在哪个页面，都可以轻松地写一篇新文章。它还将跟踪他们写的任何博客帖子，他们在网站上留下的任何评论，无论它在哪个博客上，它还让他们能够轻松地创建一个新的博客，它还为此添加了一个新页面。

事实上，BuddyPress 中关于博客的大部分功能都是追踪功能。因此，他们会跟踪新的博客，跟踪帖子和评论，将这些整合到活动流中，并在你的个人资料中显示你是哪个博客的一部分，以及你在哪个博客上留下了哪些评论。它并不真正影响博客的任何发布或任何实际界面。它只是通过跟踪和添加一些新功能来增强性能。

Brad:BuddyPress 做的一件事是，很多人认为 WordPress MU 是现成的，但事实并非如此，它将所有博客内容聚合到一个中心位置，对吗？

安迪:是的，没错。所以真的只是跟踪。它监控新帖子何时发布，它们发布到哪个博客，新博客何时创建，以及整个网络上的任何评论，无论它们在哪个博客上；它将把所有这些主要或基本上带回活动流的一个中心位置。因此，网站上的任何人都可以查看活动流，看到发布的帖子，看到评论。所以它把所有的东西聚集在一起。

Brad: 说到活动流…你为什么不解释一下，因为我知道活动流在 BuddyPress 的最近几个版本中有所发展。活动流到底是如何工作的，它们做什么，它们有什么样的特征？

Andy: 在当前版本中，活动流实际上是将整个网站的所有活动聚合在一起。因此，它将跟踪博客帖子，它将跟踪个人资料更新，它将跟踪群创建，它将跟踪群创建的帖子，它将跟踪到群线的帖子，它将基本上跟踪整个网站的任何活动。还有一个过滤器，您可以过滤，只需告诉我最新的博客帖子，或者只告诉我任何已创建的群组或新的友谊连接，以及即将推出的新版本，这就更进一步了，我们为什么不聚合活动流，但也允许人们发布活动流的更新。所以你可以进去，你可以发布一个新的更新，说你在做什么，或者进入一个群组，发布一个针对该群组和该群组主题的新更新。

所以这是一个双向的过程。是的，它聚合了所有的东西，但是你也可以直接把东西发布到活动流中。实际上，我们在即将推出的 1.2 版本中引入了活动流注释。因此，人们实际上可以进来对你的活动流中的事情进行评论，而在此之前，你只是监视它，并说好吧，这正在发生，但没有真正的方式与之互动。所以它变得越来越互动，我认为这是它应该走的路，并且在最新的开发版本中被证明对人们来说更有粘性。他们想回来评论一些事情，他们想和其他人交流。所以我认为这很有用。

Brad: 有没有计划在这些状态更新中添加类似脸书的“赞”链接？

安迪:我看不出有何不可。我当然希望看到它成为一个插件。我认为这将是一个伟大的插件。事实上，这已经被请求了很多次，人们想说我喜欢在不同类型的内容上使用它，所以我可以肯定地看到它作为一个插件出现，我们将看到它去哪里以及它是如何被接收的。

布莱德:那就这样吧。如果你想参与 BuddyPress，有一个很棒的插件想法。这实际上提出了一个很好的观点。BuddyPress 确实允许插件，就像 WordPress 或者 WordPress MU 一样，它允许 BuddyPress 特定的插件。是这样吗？

安迪:没错。BuddyPress 是基于 WordPress 插件 API 的，所以它使用了所有相同的技术，比如过滤器和动作，基本上你的插件和 WordPress 插件是一样的。你只是检查 BuddyPress 是否处于活动状态，并使用 BuddyPress 提供的函数。所以如果你真的知道如何构建一个 WordPress 插件，你在某种程度上知道如何构建一个 BuddyPress 插件，因为基本原理非常相似。

BuddyPress 为 WordPress 编写的好处在于，你可以在现有 API 的基础上运行。所以我真的不需要考虑人们将如何扩展 BuddyPress，以及他们将如何做；做这件事的方法在 WordPress 中已经有了，只是用 BuddyPress 扩展一下而已。

布莱德:太棒了。BuddyPress 添加了我们刚刚经历的大量功能。我可以告诉你，你可能花了很多个漫长的夜晚埋头苦读这段代码。

Andy: 大约有八种不同的组件功能，但好的是你可以关闭这些功能。如果您只想创建一个网站，或者只是添加个人资料，或者创建一个只有个人资料和博客的网站，您可以关闭所有其他功能。所有不同的功能都有开关，你可以挑选你想要的。

**Brad:** 我认为这很棒，它可以帮助人们轻松进入 BuddyPress，测试 BuddyPress 的一个功能，看看他们的用户是否喜欢它，如果他们得到了良好的反馈，他们可能会打开 buddy press 其余插件的开关。

安迪:是的，没错。这不是全有或全无，你可以让它成为一个渐进的过程。

Brad: 我们来谈谈我正在查看的 BuddyPress 即将推出的功能。你实际上有一个[公开路线图](http://buddypress.org/about/roadmap/)的功能，将在下一个版本中出现，它实际上非常详细；我印象深刻，有时甚至比我想象的 WordPress 路线图还要深刻。所以我很好奇，路线图是由你决定的吗？是社区的努力还是路线图的来源？

安迪:嗯，我想大概是六七个月前，1.0 版本刚刚发布。我坐下来，思考在接下来的三到四个版本中会有什么很棒的特性，我想我列出了大约 30 或 40 个，我认为这是相当广泛的，我所做的是我去了 IRC，进入 BuddyPress 聊天室，与那里的一些插件开发者和人们讨论，他们通常出去谈论 BuddyPress，他们对这个列表的想法以及他们认为什么是重要的。我们只是对它进行了一番梳理，然后将其缩减到 20-25 个左右，我们认为这些功能对 BuddyPress 在接下来的几个版本中会非常有用。
