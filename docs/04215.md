## StatCounter vs NetMarketShare

上个月，rkellogg 评论道:

> 我一直对 StatCounter 和 NetMarketShare 浏览器统计数据的巨大差异感到惊讶。

StatCounter 和 [NetMarketShare](http://www.netmarketshare.com/) 是两家最受欢迎的浏览器使用分析公司，但他们的数据相差很大:

| 浏览器使用情况 2015 年 3 月 | 静态计数器 | 网络市场份额 |
| 铬 | 49.07% | 24.99% |
| 微软公司出品的 web 浏览器 | 17.88% | 56.54% |
| 火狐浏览器 | 16.81% | 11.89% |
| 旅行队 | 10.93% | 5.00% |
| 歌剧 | 1.62% | 1.15% |

*哪个是正确的？*

答案:*都不是*。准确收集浏览器统计数据是不可能的。这些结果是基于不稳定的假设。改变单个变量或数据收集方法会扭曲数据。

那是逃避责任！哪个更准确？

答案是:*这取决于你在寻找什么*。这些公司使用不同的数据收集和分析技术…

| 静态计数器 | 网络市场份额 |
| 记录来自 300 万个网站的数据 | 分析 4 万个网站的数据 |
| 每月分析 175 亿次页面浏览 | 每月分析 1.6 亿次独立访问 |
| 除了移除机器人活动和预渲染之外，不做任何调整或加权 | 根据 CIA 互联网流量数据应用国家权重。例如，如果巴西占全球流量的 4%,但仅占 NetMarketShare 数据的 2%,则结果会翻倍 |
| 是独立的，没有商业伙伴 | 是一家商业公司，合作伙伴为数据付费 |
| 报告基于页面视图 | 报告基于每日独立访客 |

关键的区别是 **StatCounter 记录的是访问量**，而 **NetMarketShare 记录的是用户**。考虑以下场景:

1.  你用 Chrome 访问了 1000 个页面，然后用 Internet Explorer 访问了一个页面。StatCounter 记录了 Chrome 的 1000 张“投票”和 IE 的一张。NetMarketShare 为两者都赢得了一票。
2.  你整天用火狐浏览 500 个网页。您的奶奶使用 Safari 访问了一个网站。StatCounter 记录了 500 张火狐的投票和一张 Safari 的投票。NetMarketShare 为两者都赢得了一票。
3.  你白天用 Chrome，IE，Firefox，Safari，Opera。你的使用在 StatCounter 上是加权的，例如，你使用 Opera 的次数是 Chrome 的两倍。NetMarketShare 将对每个浏览器进行同等排名。

很多人是轻度网络用户，他们通过默认浏览器(可能是 IE)访问。一小部分是超级用户，他们每天用自己喜欢的应用程序(可能是 Chrome)访问数千个页面。NetMarketShare 记录人；两种类型的用户都受到同等对待。StatCounter 记录使用模式；重度网络用户比其他人更容易影响搜索结果。因此:

*   根据 StatCounter 的数据，如果你在一个随机的网站上选择一个随机的**页面访问**,有 49.07%的可能性是来自 Chrome。
*   根据 NetMarketShare 的数据，如果你从一个随机的国家随机挑选一个互联网用户，他们有 24.99%的可能成为 Chrome 的固定用户。

尽管数字看起来不一致，但两者都是“正确的”。

*那么你为什么要用 StatCounter 来做浏览器趋势文章呢？*

StatCounter 有一个大的样本集，更简单的方法，不操纵数字和报告**浏览活动**。这对 web 开发者来说是一个重要的因素。

举个简单的例子:假设您的应用程序有两个用户——一个用 BrowserX 连续访问，另一个用 BrowserY 每月访问一次。您的开发和测试任务应该优先考虑 BrowserX。随着流量的增加，情况变得更加复杂，但页面访问量仍将是比独立用户或会话更好的浏览器使用指标。

这些有什么关系吗？

这些报告有助于发现 Chrome 的迅速崛起、IE 的衰落以及智能手机和平板电脑的流行等趋势。这些信息可以帮助您规划未来的开发任务。然而，你自己的网站统计数据比全球平均数据更能反映实际使用情况。
