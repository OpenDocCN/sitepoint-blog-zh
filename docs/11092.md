## 采访记录

帕特里克:2010 年 4 月 6 日。我们采访了一本针对开发者的 WordPress 新书的作者。这是 SitePoint 播客#56:布拉德·威廉姆斯、大卫·达姆斯特拉和哈尔·斯特恩的专业 WordPress。

大家好，欢迎来到另一期的 SitePoint 播客。我是 Patrick O'Keefe，和我在一起的是我通常的搭档主持人 Stephan Segraves，我们稍微翻一下剧本，采访我们通常的搭档主持人布拉德·威廉姆斯，还有 David Damstra 和 Hal Stern。他们一起合著了新书《专业 WordPress》，该书来自 Wiley 的一个部门 Wrox Press。

Professional WordPress 是程序员对程序员系列的一部分，因此它主要面向 WordPress 开发者，以及那些真正想进入代码或软件的人。根据 Wiley 网站的说法，这本书深入研究了 WordPress 系统的内部，超越了基础知识，深入系统的核心，概述了 WordPress 的功能方面以及插件和主题开发。如果你想了解更多，你可以在你最喜欢的书店找到这本书，也可以在这本书的网站上找到:professional-wp.com。事不宜迟，言归正传。

很高兴有布拉德加入我和凯文·杨克作为 SitePoint 播客主持人，他们也是已出版的作者。斯蒂芬，你知道你需要抓紧时间。

斯蒂芬:我正在写一部三部曲的剧本。

帕特里克:嗯，这就对了。让我们做一个快速的生物检查。在 30 秒或更短的时间内，你是谁，你做什么，你为什么写这本书。我想我们会从布拉德开始，因为没有人听这个节目真正了解他。

布莱德:当然可以。我是布拉德·威廉姆斯。我是 Webdevstudios.com[的首席执行官和创始人，我们主要使用开源软件来构建网站和不同的应用程序。所以很明显，WordPress 是我们选择的工具之一。](http://webdevstudios.com/ "Website Development and Design | Social Network Applications | Search Engine Optimization | Open Source Solutions - WebDevStudios, LLC")

帕特里克:好吧，大卫？

大卫:我是大卫·达姆斯特拉。我是 [CU*Answers](http://www.cuanswers.com/ "A Credit Union Owned CUSO : CU*Answers") 的网络服务经理，这是一家信用合作社服务机构。我的团队为信用社开发网站，我们使用 WordPress 作为 CMS。这对我们来说是一个很棒的工具。

帕特里克:酷。哈尔。

哈尔:我是哈尔·斯特恩。我大约六年前开始写博客，当时我在 Sun Microsystems 工作，当我想把一些内容带入一个更私人和个人的空间时，偶然发现了 WordPress，并决定我需要开始弄清楚它是如何从内到外工作的。在开源世界里，拆开泰迪熊和收音机的日子让我有些不知所措。

Patrick: 你们三个是如何合作写这本书的？

**大卫:**马特在 [Photomatt](http://ma.tt/ "Matt Mullenweg — aka Photo Matt — on WordPress, Web, Jazz, Life, and Photography") 上发了一个帖子，说威利和 Wrox 正在寻找一些 WordPress 作者来开始一个新的系列，我想至少我和布拉德，我相信，单独回应了，Hal…然后就走到了一起。

Brad: 是的，那篇文章实际上是——我实际上找到了它，因为我刚刚写了一篇关于它的博文。大约一年前的今天，也就是去年的 2 月 10 日，我们差不多走了一整年，从我们差不多都看到那篇关于 Wiley 希望作者实际上，你知道，发行并推广这本书的帖子开始。所以这很令人兴奋。

**David:** 我曾经和 Wrox 标签公司 Wiley 的执行编辑 Carol Long 一起工作过，我以前和她一起工作过，她和我一直在讨论一本书的想法，同样，独立地将不同的线索结合在一起也很好。

帕特里克:那么你们三个在哪里？我知道布拉德在新泽西。你在哪里，大卫？

大卫:我在密歇根州西部的大急流城。

**帕特里克:**还有哈尔？

哈尔:我在新泽西州的北部。所以即使我和布拉德在同一个州，我们也不是，因为我们在不同的区号。

布莱德:去海边玩。

帕特里克:酷。所以实际上是 Wrox——所以你们来到了 Wrox；我猜他们没来找你？在大卫的案例中，你已经和威利谈过了，他们一起工作，但是他们提出了一个要求，你们去找他们，对吗？

布莱德:是的。实际上，WordPress 的创始人之一是马特·莫楞威格。他们已经联系了 Matt 来获得一些曝光率，以吸引更多的作者。他们在寻找更多与 WordPress 相关的作者。所以很明显他有很大的影响力。嗯，我和大卫都看到了他的帖子，以及我所听到的许多其他人的帖子，我们只是发了电子邮件，表达了兴趣，就这样了。

帕特里克:太酷了。所以威利，当然，是一个巨大的，巨大的出版商，毫不奇怪，他们已经推出了一批 WordPress 书籍。就像阅读下面的列表一样，你有:当然是专业的 WordPress，Janet Majure 于 2010 年 3 月出版的[自学 WordPress](http://www.wiley.com/WileyCDA/WileyTitle/productCd-047057092X.html "Wiley::Teach Yourself Visually WordPress") ，前 SitePoint 论坛工作人员 Aaron Brazell 于 2 月出版的 [WordPress 圣经](http://www.wiley.com/WileyCDA/WileyTitle/productCd-0470568135.html "Wiley::WordPress Bible")，Thord Daniel Hedengren 于 2010 年 2 月出版的[粉碎 WordPress: Beyond the Blog](http://www.wiley.com/WileyCDA/WileyTitle/productCd-0470684151.html "Wiley::Smashing WordPress: Beyond the Blog") ，Thord Daniel Hedengren 于 2010 年 2 月出版的 [WordPress 24 小时培训师](http://www.wiley.com/WileyCDA/WileyTitle/productCd-0470554584.html "Wiley::WordPress 24-Hour Trainer: Watch, Read, and Learn How to Create and Customize WordPress Sites")，Lisa Sabin 出版的[第三版](http://www.wiley.com/WileyCDA/WileyTitle/productCd-0470592745.html "Wiley::WordPress For Dummies, 3rd Edition")你知道，我想问题是你觉得威利对你的书的定位和其他书相比如何？

哈尔:嗯，这是怎么回事。我会告诉你我对它的看法，因为我认为这个项目产生的部分原因是我们希望真正地对待这些材料，而不是根据嘿，你想开一个博客，这是你如何创建一个新的帖子，这是为什么你应该使用标签和类别，但要真正地考虑教人们如何构建代码和足够的 WordPress 内部机制，这样你就可以理解如果你想通过主题扩展它，你想通过插件扩展它。即使你只是想微调一下，你也知道所有的部分是如何组合在一起的。这是我的目标，最初在和 Carol Long 谈论这本书时，我认为我们三个人的合作非常好，因为我们在 WordPress 开发专业知识方面带来了三种非常不同的观点。显然，这本书的另一部分是如何构建的；显然不是一本初学者的书，试图瞄准受过良好教育的用户，或者我们希望成为受过良好教育的用户的人，或者在很多情况下，真正追求开发者读者，而不是那些，再次，他们决定要开始一个博客，所以他们要去 WordPress 上买书的人。

帕特里克:大卫，你对此有什么看法？

大卫:是的，我同意哈尔的观点。这本书不像一本食谱。这不是一个我如何做一个网站，我如何做这个，我如何做那个的问题，而是关于 WordPress 背后的理论的更深入的问题。我们不可能跟上一本关于 WordPress 不断发展的食谱书。我的意思是，甚至 3.0 现在随时都会出来，嗯，可能一个月后，但更侧重于理论和如何做事，这样你就有了使用 WordPress 的基础，而不是让你通过当前版本的 WordPress。

斯蒂芬:我是斯蒂芬。那么，你们参与了核心运动吗？你们三个都对 WordPress 的核心有贡献吗？

Brad: 我已经为 WordPress 的核心和一些姊妹项目如 MU 和 BuddyPress 贡献了一些代码和补丁。我不认为自己是一个巨大的核心贡献者，但有时我会遇到一个 bug，我肯定会尽我所能帮助修复项目本身的 bug。

斯蒂芬:大卫？

大卫:是的，我当然更像是一个潜伏者，我监视着它，看着它，但是现在我实际上还没有做出任何回报。

Stephan: 你们做插件吗？我知道布拉德有一些插件。哈尔或者大卫，你们都做插件吗？

**David:** 我们为特定的信用合作社网站提供插件，这些插件与信用合作社核心处理环境相集成。所以除了我们的客户对谁都没用。

实际上，我更多是从一个对数据库和数据管理非常感兴趣的人的角度出发，试图弄清楚所有的内容是如何存储的，各种数据项之间的关系是什么，以及最终，WordPress 将如何把它转化成在页面上显示的东西。因此，如果大卫是一个潜伏者，我想我是元潜伏者，但同样，这也是为什么当你有一群合作作者时，我扮演一些编辑角色，也只是试图确保一切都联系在一起，并真正尊重大卫和布拉德作为更深层次的开发者，他们都比我有更好的艺术感觉。

斯蒂芬:嗯，听起来你的三连胜真的很不错。那很好。

Brad: 我想这就是他们组建团队的目的，他们不想让三个不同的插件开发者写一本书，他们想真正引入一个对 WordPress 有很多不同经验的团队。就像你说的，Stephan，很明显，我在插件和编程方面有很多经验，而 Hal 和 David 也有他们自己的长处，而我并不擅长。所以我们真的很好地融合在一起，这真的帮助我们在写作过程中在某些方面互相依靠。

**Patrick:** 我将把这个问题抛给 Brad，但是 David 提到 3.0 即将到来，这让我想到我知道我在写自己的书，我的书不是一本软件书，谢天谢地，我决定避免这样做，因为软件一直在更新，你做了什么样的决定？你是如何防止过于版本化，从而使这本书不会比它需要的时间更早过时的？

Brad: 这是一个很好的问题，特别是对于开源软件，我的意思是，就像你说的，它正在快速发展。我的意思是 WordPress 3.0 将在不到一个月的时间内发布。我认为 5 月 1 日是目标日期。很明显，当这本书出版时，我们不可能知道已经出版的确切版本或者正在开发的版本。专业 WordPress 是基于 WordPress 2.9 的。

好的一面是，当我们写这本书的时候，我们确保了我们不仅谈论了一些可能特定于那个版本的主题，而且作为一个整体，它更多的是使用适当的方法，以挂钩到 WordPress，使用适当的功能，使用不同的 API，这些都是不太可能很快改变的事情。

帕特里克:希望如此。

布莱德:希望你是对的。有些事情会改变，但是 WordPress 的一个优点是它总是向后兼容。所以如果你今天写点什么，一年后会有用吗？它会公然工作。这是正确的做法吗？这很难说，因为它确实发展很快，但我们肯定会确保在我们写的所有东西上都使用最新的实践，所以我们知道我们的书会尽可能长时间保持新鲜。

帕特里克:做这本书的一部分，我想我会把这本书扔给大卫，但做这本书的一部分，我不知道你的合同读起来怎么样，但假设它卖出了合理的数量，你会做第二版吗？

大卫:我想你应该问问我们的家人，他们是否希望我们这么做。

**布拉德:**或者允许。

大卫:我们还没有和出版商直接讨论过这个问题，但我知道我们每个人都已经列出了一个清单，如果有机会，我们会为第二版更新什么。

Patrick: 现在，我们要把那部分删掉，这样人们就可以出去买了。我不知道，如果我是你的出版商…不，我只是在开玩笑，但不，这很有道理。我认为这是很自然的事情，对我来说有趣的是有多少 WordPress 的书已经出版和正在出版。我想这是一个让出版商跃跃欲试的热门领域，我想发生的事情是，第一本书一定做得非常好，当一本书在某个流派中对一个出版商非常好时，似乎其他出版商也会跃跃欲试。你认为有没有——我不知道这是不是一个问题，因为这对 WordPress 来说是一件好事，但是你认为有很多 WordPress 的书是不是有点太多了？

**Brad:** 我喜欢认为人们在得到一本书时会做研究，所以他们会看看评论，看看社区或该领域的其他人对这些书的评论，以了解哪本书适合他们，因为你是对的，这些书有一些重叠，而且有很多，但我认为就 Wiley 和 Wrox 而言， 我知道他们的目标是为 WordPress 的每个主要版本出一本新书，每本新书有三到六个月的周期。 显然，有很多书出版了，但它们仍然是相关的。

Patrick: 那么主要版本是 WordPress 1、WordPress 2、WordPress 3 还是？

Brad: 没有，所以目前的版本是 2.9.2，所以主要版本是 2.9。所以我们不是在谈论…

**帕特里克:**好吧，那么 2.9，3.0，3.1 等等。？

布拉德:对。

我还认为写书的方法之一是你想写一些足够传达的东西，我不想说批判性思维，因为这听起来像是你要在哈佛学到的东西。如果他们想教一门 WordPress 课程，他们当然非常欢迎把我们的书作为教科书，但是如果你想传达这是你如何处理代码，这是你如何处理一系列问题，正如 Brad 之前提到的，教人们如何去使用其他可用的资源，通过这样做，我认为这本书的整体结构在几个版本中保持不变。还有，我觉得不管人们在哪里相交，对他们来说都是一个很好的起点。同样，他们可以从 2.9.2 中获得帮助。这是循环的结构，这是模板文件的选择。太好了，我现在可以在 3.0 或之后的下一个主要版本中使用它了。

**Brad:** 稍微扩展一下，你知道 Hal 是完全正确的，所以很明显，我的意思是任何使用过 WordPress 的人都知道有很多不同的功能，可以做你想用 WordPress 做的任何事情。有一个功能可以拉回你的类别，拉回你的标签，拉回页面，链接，无论你想要什么。通常会有一个函数来完成这项工作。我们谈论的是成千上万的函数，所以显然不可能知道所有这些，但我们实际上打开了 WordPress 的一些核心文件，向你展示了与一篇文章相关的所有函数，与一个类别相关的所有函数，这是你阅读代码的方式，你可以查看内联文档。所以它真的训练你如何，而不是依赖，好的，这就是你如何做这个或那个或这个。你真的可以深入了解自己，看看这些功能，并真正开始了解它们是如何工作的，以及如何使用它们。

Patrick: 所以这是 Wrox 的程序员对程序员系列的一部分，对吗？

布莱德:是的。

Patrick: 考虑到这一点，观众并不是你想进入 WordPress 的典型人群。面向傻瓜的 WordPress 似乎更适合这一点。这本书更像是给那些对 PHP 编程有所期待或者有一定水平的人写的，我想——目标读者是谁？你对这项工作有什么期望？我想我会把那个扔给大卫。

大卫:是的，这肯定是给那些想扩展 WordPress 或真正的超级用户的，懂 PHP 的，懂主题 CSS 的，能浏览一些代码并得到让 WordPress 做他们想做的功能的。

**哈尔:**我总是喜欢举例回答。我和我的一个朋友一起工作，他为寻找工作问题、招聘人员、职位发布、有趣的新闻条目的人建立了一个网站，并希望能够按字母顺序对职位进行排序，因为这是人们倾向于认为查找招聘人员或寻找公司名称的方式，他找到我说，我知道我可以做到这一点，但我如何做到这一点？你知道，我知道我必须做一些事情，我该怎么做？这就像不完全是世界上最伟大的 PHP 程序员，或者肯定不是将对核心做出贡献的人，而是基本上想知道我要去哪里的人，我知道这是几行更改，我知道我只需要知道要更改哪些参数，我要去哪里，他是我的目标受众。我只是需要更多像他一样的人，但那是我的目标观众。

帕特里克:所以这里有一个问题。你们有没有，因为我知道这是我自己，因为我为一本关于 phpBB 的书写了前言，并在那种 phpBB 领域进行了连接，我已经看到这些书出版了，你们有没有听到很多读者说为什么要写一本关于这方面的书，你可以去在线手册或者你可以调出 WordPress 文档等等？你碰到那个人了吗？

**Brad:** 我想大部分人都已经一头扎进了 WordPress，因为当你真正开始学习如何定制 WordPress 的时候，你去的第一个地方是[codex](http://codex.wordpress.org/ "Main Page « WordPress Codex")，codex 本质上是 WordPress 在 WordPress.org 的文档，它是所有不同的文档，功能，它们如何工作，任何一头扎进 codex 的人都会很快意识到那里有很多信息，很多。我的意思是它已经生长了谁知道多少年了。你知道这个维基里面有成千上万个不同的页面。这很难理解，而且如果你刚开始使用 WordPress 并想知道如何扩展它，内容的组织方式也不容易学习。所以大多数人通常已经知道这一点，他们正在寻找一些更好的东西，无论是网站上的教程还是他们可以实际持有并用作参考的书籍。他们在寻找一些更专注、更有条理的东西，这正是我们这本书的切入点。

哈尔:同样，这并不意味着要取代法典；它真的应该是——在某一点上，我把它称为你的法典旅行指南，它告诉你想如何接近它，有趣的东西在哪里，你应该如何去解释你会在那里找到什么，这样，阅读这本书并且也有兴趣使用源代码作为参考的人将能够做到这两者。我认为很少有人能够拿起一本参考书，通读一遍，并从中学到一些东西。再说一次，写书的部分目的是帮助那种层次的组织。

帕特里克:我喜欢哈尔。他就像一个——退后，布拉德；这并不是要取代法典。我们不想激怒任何人。
