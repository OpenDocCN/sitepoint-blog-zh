## 谈话要点

> 值得一提的是，*保持小的 API*…很多人会根据边缘情况向您提出选项、不同选项和功能等要求。然后迎合每一个人不会有任何好处。如果你迎合了每一个边缘情况，如果你为了成为一个正在这样做的酷家伙的利益而添加了每一个选项，不。因为将要发生的是，现在你有了这个针对不同边缘情况的不同创可贴的庞大臃肿的库，并且你已经远离了最初的愿景，那只是针对 90%用例的这一小群坚如磐石的 API。请不要让这种情况发生在你的项目上！

* * *

> 看到我身后的电视了吗？有一个树莓馅饼挂在上面。我有开源的负罪感。我对开源有很深很深的负罪感，所以我想做一个更好的维护者。所以我决定让它整天出现在我面前，每一期都是这样。我这里有覆盆子酱。

* * *

> 不要只是拉下回购…那里有成千上万的人依赖它。这不是任何特定事物的细微差别；我只是说说而已！

Ken:

我要说的是，我在沃尔玛做的很多 React 工作非常有趣，因为它就像一个新事物，但它也是大规模的——非常大的规模；所以这很有趣。

其他的事情有流行的开源项目。我写了 [Slick Carousel](http://kenwheeler.github.io/slick/) ，实际上写它并不容易，但也不是超级难。这是事后处理，这很有趣。尝试和维护这些东西非常有趣——这适用于所有开源软件。我有很多库，但是我没有维护好它们。看到一些东西变得流行的过程，然后是不同的项目，一些项目有某些问题，而另一些没有，这是非常有趣的。像 Slick Carousel 这样的东西就像一个低垂的水果:谁不需要旋转木马呢？

几乎每一个你为了钱为别人建立的网站，他们都需要某种滑块。但是你也有其他的东西，比如库和类似的东西。有点……我不是要说“独家”，是受众更小；这是一个更集中的观众，所以你有不同类型的问题。有人会对你说，“我对 JavaScript 不太感兴趣。这是怎么做到的？”更像是，“我*是*对 JavaScript 超级狂热。我们需要优化这个”或类似的东西。

在开源领域处理所有这些事情非常有趣。我最近做的绝对最有趣的事情之一是[反应原生](https://facebook.github.io/react-native/)，这太疯狂了。你在用 JavaScript 编写真正的本地应用程序，这对我来说简直是疯了。

David:

对 React Native 略有耳闻，很好奇:你在 React Native 尝试开发时，面临着怎样的挑战？

Ken [7:30]:

使用它有几个不同的好处。首先，你可以跨平台，所以你可以一次写完，一般来说，你可以在 Android 和 iOS 上获得相同的体验——从一个代码库。如果你真的很棒，如果它们足够相似的话，你甚至可以与那个应用程序共享一些来自 web 的 Redux。接下来，你会得到一些很酷的东西，比如你可以跳过 Play 代码，这样你就可以绕过 App Store 提交过程(实际上最近已经有所改善)。

我在规模上看到的一件事非常酷，如果你有一群熟悉 JavaScript 或 React 的工程师，突然你的整个 JavaScript 团队都可以在你的移动应用上工作。再说一遍，一般来说。

我在沃尔玛做的是 React Native，这非常重要。在本土，有很多事情是非常复杂的。因此，JavaScript 方面的任何人都不能随意更改它。

React Native 不能替代某些东西。只是对某些事情没有意义。本质上，如果你有一个本地应用程序，并且你找到了一个可以使用 WebView 的地方(这是一个混合应用程序，对吗？).如果你有一个地方可以使用 web 视图，比如 Cordova 之类的东西，但你想让它感觉更自然，React Native 是一个很好的选择。

Tim:

是什么让 React Native 比 PhoneGap/Cordova 这种情况更有本土感？

Ken:

它确实是本地的。实际情况是，它到底是如何工作的——也许不完全是，但我个人在使用它并详细阅读它之后理解它是如何工作的——你有 JavaScript，它在一个线程上被解释，然后你有本机代码，它接受解释的 JavaScript 并以此构建本机视图。您实际上是在告诉您的本机程序如何使用 JavaScript 进行布局和操作——但不是在浏览器或 WebView 中。您不是在浏览器环境中运行 JavaScript。你没有渲染成 HTML。从来没有人使用过 HTML。你所做的是指定类型，像`view`和`text`之类的，这些代表了(如果你在 iOS 中)UI 视图、UI 文本、任何这些控件。更酷的是，这可能是我见过的最好的桥接系统，所以你的开发者体验是惊人的。

几乎任何你可以在本机上想象的东西都可以很容易地过渡到 JavaScript。我遇到过需要使用地理编码的情况。所以你有一个字符串，你想从这个字符串中得到一个 lat-long:*在 iOS* 中。

因此，您可以将它连接到您的 JavaScript 应用程序。您可以通过回调对地理编码器或已在本机模块类中设置的地理编码器类进行异步调用；它是桥接的。你可以给它打个电话，它会给你发回经纬度坐标。

这只是你可以做的事情的一个例子。你不仅可以像那样桥接模块，在那里你可以写一个 Objective-C 类，在那里你可以做任何事情:你还可以公开和呈现视图。任何你能想象到的视图——比如说任何控件——你都可以把它连接起来，桥接起来，然后用 props(本质上是 HTML 属性)在一个标签中表示出来，然后把它们连接起来控制它。

David:

那真是太好了。我很好奇你在沃尔玛面临的挑战是否更多地与处理它的本地方面有关，或者与处理你试图完成的事情的规模有关。

Ken:

老实说，大部分是平衡。他们在土著那边做了一些相当不平凡的事情。有很多优秀的移动工程师，他们有一些非常酷的基础知识。

对我来说，作为一个 JavaScript 玩家，我希望尽可能地玩得好。*【笑】*所以很多都是平衡。我想尽可能多地使用原生代码，同时在 JavaScript 方面保持灵活性。

规模——当然，这很困难，但很多不仅仅是用户规模(很多实际上是服务而不是 UI)。我们没有得到任何用户界面的并发性或类似的东西；那更多的是服务问题。但是我想规模将会是*开发者*规模、*组织*规模，这是我们必须要减轻的。

因此，在沃尔玛内部，我们有一个名为[电极](https://github.com/walmartlabs/electrode-react-ssr-optimization)的平台，在这里我们有大量的通用元素和通用组件，每个人都可以从中选择，并作为一个整体进行维护。

因此，我们有一个集中的组件和 Redux helpers 之类的东西的集合，而不是每个单独的应用去做他们自己的事情，这些东西是有组织地维护的。这个想法是，如果你有三个不同的团队在开发三个不同的应用程序，他们有相同的文本框组件，而不是做他们自己的事情，他们会去使用那个文本框组件。然后，如果其中一个团队发现文本框组件不像他们需要的那样灵活，他们就改进它，让它变得更好。这一过程不断延续，直到你拥有坚如磐石、真正优秀的组件。因为这些组件被添加、被请求、被彻底测试、被更好地审查，所以当你使用它们时，你可以确信你将拥有比你自己去做更稳定的东西。

Tim [13:04]:

那很酷。这几乎就像沃尔玛有自己的内部开源环境来跟踪所有这些。你能从沃尔玛如何做类似开源的工作中学到什么，并将其转化为实际的、更广泛的开源社区吗？

Ken:

绝对的。我们有一个 [ESLint](http://eslint.org/) 配置。我为[强大的](https://formidable.com/)工作；通过他们，我为沃尔玛工作。我们使用[沃尔玛的 lint configs](https://github.com/walmartlabs/eslint-config-defaults) ，所有的流程都在那里进行。我们还有一个叫做 [Builder](https://formidable.com/open-source/builder/) 的工具，我们用它来管理几个回购。如果你有类似沃尔玛的情况，你有大量的回购，我们使用 Builder 在所有这些方面基本上有相同的构建流程，并能够在所有这些方面进行更新。

因此，如果您有一个 Webpack 配置—您有十个 repos 然后您使用 Webpack 更改了其中一个配置，您必须进入每个配置并更新其中每个配置中的 Webpack 配置，以获得新的构建方式。因此，我们有类似的工具，我们有这个共享的构建系统和测试之类的东西——我们可以在一个地方做一个更改，然后它会传播到 90 个 repos 或其他地方？

David:

哇哦。这是这类事情如何应用的一个很好的例子，因为当你处理开源时，你让所有这些不同的人同时做不同的事情，并且试图跟踪所有这些事情可能是疯狂的。你如何平衡你的开源工作和你在公司内部做的工作？

Ken:

嗯，这实际上很有趣，因为我工作的公司，强大，我们标榜自己是一个开源公司，做合同工作来保持正常运转。这不像我去做我的日常工作，然后做开源工作。我每天的大部分工作都是开源工作。我们已经正式批准了开放源码，我在这里的朝九晚五的时间里有时间来研究它。

当我写 Slick Carousel 时，情况并非如此。我在一家很棒的公司工作，但是是在一个开源驱动的地方。我会在晚上做我的开源，然后一整天回来做客户工作——很多时候，通过在晚上做我的开源，或者让我工作得更快、更快、更好的工具，我的客户工作做得更好。

现在，在强大，开源被鼓励。我们有时间来研究它，而且我们有大量的开源软件。就我个人的开源来说，我个人的开源目前是苦不堪言的。原因只有一个，那就是父亲的身份。我回到家，我的妻子会穿上单身汉或类似的衣服(谁想看这个？)，所以我在我的开源上闲逛和工作，敲击键盘。如果她早上有重要的一天，她会在 10 点睡觉，我会玩电脑玩到凌晨 1 点。

现在不是这样的了。现在，我工作的时间，我要么让我爸爸上，我完全筋疲力尽，和/或喝酒。没有我希望的那么多时间来做我的个人开源，但是我让它工作。

我试着多做一点，而以前，无论我对什么感兴趣，我都会去做。现在我想说，“好吧，好吧，这需要发生在这里，这需要发生在这里。”我在 GitHub 上得到了很多这些新的热门赞词——“反应”,或者不管他们怎么称呼它们。我说，“啊，好吧，你得搞定那个。”
