# 最头疼的问题:平衡语义和 SEO

> 原文:[https://www . site point . com/heading-feathers-balancing-semantics-and-SEO/](https://www.sitepoint.com/heading-headaches-balancing-semantics-and-seo/)

我们 SitePoint 一直在对网站进行一些搜索引擎优化。当我们咨询这方面的专家时，我们收到的一些建议违背了我们之前遵循的“好 HTML”规则。事实证明，对许多人来说，规则已经改变了。

我们努力解决的核心问题是**标题结构**——我们如何在网站上使用`<h1>`到`<h6>`标签。上次我写这个的时候，我建议**永远不要跳过标题级别**。例如，`<h1>`后跟`<h3>`跳过了`<h2>`标题级别，这是不好的。这仍然是正确的——你应该避免标题结构中有空白。

改变的地方在于，像谷歌这样的搜索引擎似乎会奖励那些使用一个**标签作为页面上独特内容**标题的页面。这就产生了一个问题:如果你只能有一个`<h1>`，并且它出现在像你的站点导航这样的元素之后，那么你用什么标签来标记必须出现在`<h1>`之前的标题呢？

在自己纠结这个问题之后，SitePoint [的朋友 Roger Johansson](http://www.456bereastreet.com/) 最近发表了[他自己的发现](http://www.456bereastreet.com/archive/200911/headings_and_document_structure_conclusions/)。简而言之，在像 WAI-ARIA 这样的标准使我们能够标记像不带标题的导航这样的页面元素之前，我们必须使用以`<h2>`开头的标题结构(而不是像导航菜单、页眉、页脚和侧边栏这样的“样板”内容的`<h1>)`)。

所以现在，这是好的:

```
<h2>Site Navigation</h2>
  …
<h1>Content Title</h1>
  …
<h2>Related Links</h2> 
```

新制定的 WCAG 2.0 可访问性标准甚至在标题部分对这种方法进行了祝福，这个例子就是这样使用标题的。

这些事情总是在变化，我们可以期待浏览器对 HTML5 和 WAI-ARIA 等标准的支持在未来几年再次改变围绕标题标签的规则。现在，我将放心地使用一个单独的`<h1>`作为我的内容标题，并让`<h2>`落在它们可能落的地方。
