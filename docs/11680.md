# 欧盟将调查甲骨文收购 MySQL

> 原文:[https://www.sitepoint.com/eu-investigates-oracle-sun-mysql/](https://www.sitepoint.com/eu-investigates-oracle-sun-mysql/)

就在你以为欧洲一切都很平静的时候，欧盟委员会反击了。既然微软已经被迫屈服，他们已经将目光转向甲骨文，并将审视该公司对 MySQL 的意图。

委员会监管者已经表达了“严重的担忧”，该交易可能会导致数据库软件价格上涨。甲骨文靠数据库产品发家，是世界第二大企业软件公司。欧盟担心甲骨文可能会损害 MySQL——世界上最受欢迎的开源数据库系统——的开发和销售。

根据欧盟竞争事务专员 Neelie Kroes 的说法:

> 当世界领先的专有数据库公司提议接管世界领先的开源数据库时，委员会必须非常仔细地审查对欧洲竞争的影响。

2009 年 4 月，在与 IBM 的谈判破裂后，甲骨文出人意料地以 74 亿美元收购了太阳微系统公司，随后收购了 MySQL。这笔交易让甲骨文完全控制了 Sun 的硬件和备份解决方案、Java 编程语言、VirtualBox 虚拟机软件以及开源办公套件 OpenOffice。该公司打算将自己转变成一个全面的系统供应商，与 IBM 和 HP 直接竞争。

欧盟的调查将使太阳公司的收购推迟至少 90 天。我们需要再等一会儿才能知道 MySQL 的命运。

Oracle 会继续支持 MySQL 开发吗？这笔交易标志着数据库末日的开始吗？欧盟委员会干涉甲骨文的事务是正确的吗？
