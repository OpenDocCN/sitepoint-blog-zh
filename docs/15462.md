# 颜色和网页

> 原文：<https://www.sitepoint.com/color-web-pages/>

**颜色。它就在我们周围。放眼望去，我们的世界充满了色彩。颜色能吸引我们的注意力，有时能让我们产生某种情绪，吸引我们，排斥我们，烦扰我们，诱惑我们。网页也是如此。**

图形、彩色文本和背景、图片等。是一些让互联网如此美妙的事情，但这些同样的事情也是互联网在特定情况下最糟糕的方面。你如何在你的网站上有效地使用颜色？

您可以使用彩色背景或图案瓷砖或边框来为您的页面增添一点趣味。毕竟，谁愿意整天看着平淡无奇的黑白电影呢？挺无聊的吧？然而，在这个领域，理论上的好并不总是转化为实践上的好。

永远，永远，永远记住:你的背景瓷砖或你的酷图形或颜色组合不应该是你的网站的焦点。(不言而喻，互联网企业尤其如此。)相反，这些页面元素应该增强信息和/或产品或服务的描述/呈现。换句话说，你希望你的访问者离开时心里想，“哇！那个人真的知道他们的东西！我知道我在哪里买下一个[空白]。”你不希望游客离开时心里想着，“哇！多酷的图形啊！嘿…那个网站到底是关于什么的？”

在我看来，使用颜色最常见的错误是背景/文本颜色选择不当。如果你使用了糟糕的颜色组合，使你的文本不可读，你还不如把开发你的网站的钱拿出来，在街道中间点燃一堆篝火。从本质上来说，文本应该是可读的，这似乎是不言自明的，对吗？对于我们中的许多人来说，掌握这个非常基本的概念并不困难…直到我们发现整洁的字体颜色标签或钻研 CSS——然后我们会变得有点太可爱。

无数的页面证明可读性在许多设计师的优先考虑列表中并不重要。这听起来很无聊，但是白底黑字是最容易阅读的。您可以使用保守的边框背景或者通过使用表格和 TD 标记的 BGCOLOR 属性来添加颜色。只是不要太过火。

我特别提醒不要在彩色背景上使用彩色文字；我很少看到设计师成功完成这件作品。不幸的是，太多的设计师认为亮蓝色和亮红色是很好的颜色组合——错了！！明智地选择你的文本/背景颜色；如果你不这样做，你所有的努力都白费了。

我不认为明智地使用彩色文本或彩色背景上的白色文本有什么特别的错误；例如，在导航栏中使用这种设置是相当常见的。我要提醒的是，在你网站的大部分内容上，不要在彩色背景上使用彩色文字。不要忽视这可能对你的访问者对你网站的看法产生的负面影响。

最后，我认为在商业网站上保守一点总是一个好主意。如果你正在设计一个个人主页，用霓虹绿/粉红的配色方案让自己惊艳吧。但是，不要为了一个狂野或“酷”的颜色组合而冒失去生意的风险。你会花几个小时为你的小册子构思一个伟大的信息，然后告诉打印机柜台后面的人，“随便挑一种颜色”？不，当然不是；你意识到媒介至少和信息一样重要(在很多情况下甚至更重要)。在设计你的网站时，试着从这些方面考虑，我相信你会有更大的成功机会！
