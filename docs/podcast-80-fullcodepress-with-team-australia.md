## 采访记录

凯文:2010 年 9 月 24 日。澳大利亚团队的三名成员讨论如何在 24 小时内建立一个成功的网站。我是凯文·杨克，这是 SitePoint 播客#80:澳大利亚队的完整新闻。

欢迎来到 SitePoint 播客的特别一集。今年，为了召集澳大利亚 [FullCodePress](http://www.fullcodepress.com/ "FullCodePress") 团队的三名成员，我跑遍了 SitePoint 总部甚至更远的地方。这是今年的冠军队，他们的对手是新西兰队和海外全明星队。我相信我们会听到这些，但是在我们开始之前，我想请你们围着桌子做一下自我介绍；让我们从詹姆斯开始。

詹姆斯:谢谢凯文。是的，我叫詹姆斯·曼斯菲尔德，我是 [99designs](http://99designs.com/ "Logo Design, Web Design, Design Contests | 99designs.com") 的设计师，我在澳大利亚团队的角色是用户体验倡导者，主要是为用户着想，所以做交互设计，关注网站的可用性，我想还有信息架构，这就是我的角色。

凯文:酷。播客的老听众会记得两周前来自 T2 的詹姆斯；他和 Matt Magain 谈论用户体验设计。播客听众的另一个熟悉的面孔是亚当·希林；亚当曾经在 99designs 工作，这些天他在[八月创意](http://www.august.com.au/ "August - Online Marketing, Brand Strategy, Online Technologies & Email Marketing")，对吗亚当？

亚当:没错，是的，我是 August 的用户体验设计师，在 FullCodePress，我担任 HTML 和 CSS“marker-upper”的角色，这是一个技术术语；我想这是我去演那个角色的原因之一，(笑声)，是的。

凯文:拉克伦·唐纳德在 99designs 做我的工作，他是那里的首席技术官。

**拉克伦:**是的，你好，我是团队的网页开发员，就我而言，这是一份轻松的工作；我只需要把 WordPress 打造成一个形状。

**Kevin:** (笑)那么这些角色，让我们从这个开始，这些角色是硬编码的吗，FullCodePress 是否要求你需要一个开发人员，你需要一个 marker-upper，你需要一个用户体验倡导者，或者只是出现的任何人做他们能做的任何事情？

**Adam:** 对我来说，角色似乎是预先指定的，但我认为那天发生的一件很棒的事情是有很多角色分享，而且特别有益的是，我们中有很多人以前曾一起工作过。所以，是的，很多责任都相应地分配了出去。

**詹姆斯:**是的，肯定有固定的角色，所以 FullCodePress 说我们每个团队有六个职位，我想有六到七个，每个人都申请其中一个职位，我相信你也可以申请多个职位，所以有六个角色。但是，就像 Adam 说的，在那种 24 小时工作的环境中，很多角色重叠，很多人都在帮忙，比如我去帮忙做一些 HTML 和 CSS，但我是这方面的用户体验倡导者，所以在那种环境中，你真的需要分担任务，是的，这在一天中都会发生。

凯文:那么，拉克伦，你能记住多少规则？有你们三个，还有团队的其他三个成员；是这样吗？

拉克伦:是的，还有三个角色。因此，有一个内容编辑器，有一个设计师，也有一个项目经理。

凯文:酷。你们有 24 小时从零开始建立一个完整的网站，对吗？

拉克伦:是的。

詹姆斯:正确。

凯文:所以你们飞到了悉尼，今年是在那里吗？

詹姆斯:不，实际上今年是在惠灵顿。

凯文:惠灵顿。

**詹姆斯:**新西兰是一个可爱的地方，一个好地方，那里有很棒的咖啡，嗯，那是一个不错的城镇。天气很冷，有些早晨很冷，有些早晨有风，但这确实是一次很好的经历，是一个值得参观的好地方。

凯文:你记得做的第一件事是什么？

**亚当:**在惠灵顿？

凯文:嗯，是的，作为一个团队坐下来，你面前有这个大工程，你从哪里开始呢？

我记得当我们第一次到达惠灵顿时，我很喜欢亚当的行李(笑声)。他有一个漂亮的红色包，我认为它有爱心；上面有爱心吗，亚当？

亚当:嗯，(笑声)经常被误认为是爱情之心，是的，但不是。

凯文:这是一些平面设计的东西，对吗？

亚当:嗯，我也想这么想，但是不行(笑声)。关于惠灵顿，我记得的第一件事是在惠灵顿登陆；你有点像是在飞越海洋，然后前方有一个简易机场，但直到最后一刻你才能看到它，然后突然你着陆了，然后你仍然沿着这条轨迹快速前进，你看到简易机场的尽头是另一边的海洋。

是的，我记得我看着水面对你说，亚当，“是的，我希望很快就有跑道。”(笑声)

亚当:是的。就开始而言，我们很幸运，我们都住的酒店有一个会议室，我忘了是谁，我想是你詹姆斯或拉克伦；不知何故，我们设法在活动前一整天都在那个房间里。

詹姆斯:是啊，我们在那里相当幸运。

凯文:所以在你发现你将要建造什么之前，你有一个惊喜来制定战略。

**亚当:**正是。

拉克伦:是啊。我的意思是，对于我和亚当来说，一个很大的技术问题，就像前一天，WordPress 3 已经发布了，所以我们作为一个团队，我们开了大约五六次会，我们都决定 WordPress 是我们要走的路。然后 WordPress 3 出来了，亚当和我必须在几个小时前决定我们是否要在一个新的平台上下注，所以是的，我想我们那天花了很多时间玩所有新的儿童主题的东西，只是感受一下 WordPress 3。

詹姆斯:是的，我记得前一天在酒店的会议室里，你播放了介绍 WordPress 所有新功能的视频，我们坐在那里嘀咕着我们是否应该使用它。

拉克伦:是的。

凯文:所以你们决定加入 WordPress 你对你将要建立的网站类型有多少想法，比如你是基于什么决定使用 WordPress 的？

**拉克伦:**所以，我们要做的事情之一是过去的网站，我想是过去几年的，所以我们看了一下这些网站，对我们要做的网站和内容有了一些了解。

凯文:对于那些不知道 FullCodePress 的人来说，你总是在为一个慈善机构，一个非盈利机构建立一个网站。

**拉克伦:**对，你来自的地方总是有慈善机构，所以我们有澳大利亚的慈善机构，新西兰有新西兰的慈善机构；除了美国，是的，他们也有新西兰的慈善机构。

亚当:我认为在项目经理凯莉·利金斯和詹姆斯之间，我们已经基本上建立了一个，我不知道你是否称之为对开本，大概有 30 多个竞争对手网站，或者，嗯，其他慈善网站。因此，我们对慈善机构正在做的事情的类型以及他们是如何做的有了很好的了解，并且我们就如何应对这些挑战进行了几次讨论，如果它们在活动前的客户简报会上出现的话。

凯文:所以技术人员在开发 WordPress 詹姆斯，作为一名设计师，这是你做的主要工作吗？研究慈善网站的市场，找出哪些有用，哪些没用？

詹姆斯:绝对是。是的，我们作为一个团队在准备阶段聚在了一起。我们开了很多 Skype 会议，讨论想法，制定计划和策略。是的，我们都去研究其他非营利网站，我想 Smashing Magazine 上有一篇很棒的文章，我认为它是非营利网站和慈善网站的最佳实践，我们看了那篇文章，阅读了那里的许多建议和想法，还打印了许多网站。我记得和 Adam 和 Kylie 一起在飞机上，我想有一堆大约 30 页，30 个网站，浏览了我们喜欢和不喜欢的内容，以及我们认为在某些情况下合适的内容，所以有相当多的计划和准备，并思考如果有某些网站和某些慈善网站，我们会做些什么。

凯文:所以我想尽可能快地达到实际的 24 小时，但是在那次会议之后，你们基本上只是上床睡觉，尽可能多的睡觉，这是计划吗？

**亚当:**(笑)不，嗯，那天晚上我们出去了(笑)。

**拉克伦:**我们的策略是，我们讨论了前一天晚上试图把美国人灌醉的策略，我们中的一个人会带他们出去喝酒，但结果并不完全是这样。

詹姆斯:他们睡得真早，我觉得他们根本就没出来；我认为只有澳大利亚人和新西兰人。也许是时差的原因，他们从美国一路飞来，但是，是的，他们在前一天晚上缺席了。我想我们去了惠灵顿一家很棒的酒吧，叫 Hooch，如果有人的话，那是一家不错的鸡尾酒吧。

凯文:注意到了。所以美国队是今年 FullCodePress 的新人，他们是一个令人生畏的对手吗？

亚当:是的。

拉克伦:是的，我得说他们有一些大人物，比如我的意思是他们的开发者是 WordPress 的开发者之一，所以很明显这说服了 WordPress 的人发布第 3 版…

凯文:哦，哇。

拉克伦:…对比赛来说有点早。可能是我们被告知，啊，你知道。

凯文:(笑)不用说，他们用的是 WordPress 3。

拉克伦:是啊。

凯文:酷。好吧，所以你出去，你有一个很好的派对之夜，到达酒店，什么，明亮和第二天早上？

詹姆斯:我不知道你们是怎么想的，但我前一天晚上睡得很差，这对我参加活动没有帮助。我想我只是在脑子里想比赛会如何进行，我们会做什么，是的，我肯定没有睡个好觉。

Adam: 嗯，我记得我大概凌晨 1 点或 2 点还没睡，在做一些基于 em 的布局。

詹姆斯:哦，哇。

**拉克伦:**是的，没错，我当时正在尝试为 Flash 视频制作隐藏式字幕，这是我们认为我们将获得的视频之一，只是基于一种直觉，所以我们想知道如何实现这种权利的可访问性。

Kevin:Adam，基于 em 的布局可以随着字体大小的变化而缩放，对吗？

**亚当:**正是。

凯文:哇。是啊，有点棘手。我知道没有多少 CSS 框架做这种缩放。我认为 YUI 的 CSS 布局框架是我所知道的唯一一个开箱即用的框架。但你是从零开始编码的？

是的，我们很大程度上是从——实际上，不，等等，我们是基于一个子主题或者我们在 WordPress 的一个默认主题上创建了我们自己的子主题。

凯文:好的，在我们进入代码之前，让我们谈谈你在活动现场的布置。我记得我看过你们当时在 YouTube 上发布的一些视频；看起来你对桌子的布置很有想法。有人能告诉我怎么做吗？

**James:** 当然，每个人都有一个非常不同的安排，我们决定都面向同一方向坐着，面向墙壁，我们有许多索引卡和便利贴，并计划使用屠夫的文件来框出想法和分享想法，但我们非常想尽可能多地与彼此分享，而不是去我们自己的区域和单独做我们自己的事情，所以我们的桌子布局反映了这一点。我们坐在一起，面向同一个方向，看着那面墙，那面墙将会是我们的任务清单，也是我们想要达到的目标。所以，是的，这是故意的。

**Adam:** 我们的座位安排也是经过深思熟虑的，我们坐在一天中最有可能合作的人旁边。所以我记得是拉克伦(开发人员)，我自己(HTML，CSS)，然后是辛亚飞，他是我们的平面设计师，然后是詹姆斯(UX)，罗宾负责我们的版权工作，他也花了很多时间和客户在一起，我们会去找他，然后是项目经理凯莉。当我们面对着墙排成一行时，我们的桌子几乎成了一个 V 字形，这样我们就可以清楚地看到彼此，同时还可以交谈。

凯文:酷。那么我们来谈谈客户。我猜你和客户的会面是第一件发生的事，是在早上 9 点还是类似的时间？

詹姆斯:所以那是上午 11 点，11 点开球；他们在上午 11 点为我们每个人介绍了三个客户

**Kevin:** 你们都在现场，还是有团队代表与他们会面？

**James:** 我们都在那里，所以活动中的每个人，包括志愿者，都在大楼的门厅里，他们在那种情况下向每个人宣布，然后我们都从那里进入我们的房间，进行基本的工作。在此之前，我们已经讨论过这个问题，我们将花尽可能多的时间作为一个小组来讨论和了解他们的业务和业务需求。我相信我们比其他任何人，任何其他团队都花了更多的时间，坐下来非常详细地讨论他们在寻找什么，我们能做什么和不能做什么，并相互分享想法。

凯文:那里气氛很紧张吗？我可以想象一些团队成员会说“让我们开始吧！”

**詹姆斯:**没感觉到。我觉得每个人都很喜欢讨论，谈论想法，提出他们认为如何工作的想法。我们还采取了坐下来勾画想法的方法，所以我们花了大概两个小时与客户谈论他们的需求和品牌，了解他们现有的网站做得怎么样，做得好不好。他们有一个企业视频，我们也观看了这个视频，以便更好地了解他们的沟通风格，他们向谁发出呼吁，他们有什么内容。在所有这些讨论之后，我们都用纸笔和记号笔进行了头脑风暴，我们都勾画出了我们认为主页应该如何工作的想法，我认为这是一个非常重要的练习，所以我们都很清楚，我们都觉得我们的想法被听到和讨论了，这是一件好事。然后我们就像，好吧，我们已经进行了三四个小时，我们真的需要分手并继续下去，但是没有人——感觉没有人真的想离开并开始做他们自己的事情。

凯文:那么，当你最终分手并开始工作时，你觉得自己工作非常独立吗？

**亚当:**我不要；我觉得我们合作得非常紧密，就像拉克伦和我一样，我们开始谈论 WordPress 的设置，Lox 值得称赞的是，在最初的两三个小时里，我们做了大量的工作，建立了整个框架，启动并运行了主题，等等。所以，是的，我与设计师辛亚飞正在进行对话，詹姆斯继续与客户合作，找出他们如何应对即将到来的挑战；我发现这是相当合作的，凯莉四处闲逛，检查人们，确保我们在正轨上，每个人都对我们选择的道路感到高兴。

**James:** 我们一开始也计划每隔几个小时进行一次常规的交流，作为一个团队坐下来休息五分钟，远离我们的电脑，远离我们正在做的事情，我们知道进行这些交流很重要，我们一整天都在这样做。

**Kevin:** 那么，拉克伦作为开发者，提供了什么？基本上，我不知道，是不是有一个共享的主机环境，有一个空的目录，这就是你要处理的事情？

**拉克伦:**是的，所以基本上我们得到了自己的共享主机，上面有一个 LAMP 堆栈，MySQL，Apache PHP，从那里，是的，我们基本上只能使用自己的设备。

**Kevin:** 当时的环境中有没有什么意外，有没有什么是你不得不花时间去解决的，而你并没有预料到的？

拉克伦:嗯，实际上这是一场灾难。在比赛进行到一半时，我们有将近一个半小时的停机时间，因为服务器发生了一些问题，它完全离线了。所以，是的，谢天谢地，亚当和我决定使用本地开发虚拟机。

Kevin: 所以你只是部署到那个共享主机上，而不是在上面开发？

**拉克伦:**是的，所以唯一真正的问题是内容是在共享主机上编辑的，我们有点——有点想让它尽可能成为一项观众运动，我们在 GitHub 上有所有的代码，我们在网站顶部有一个小栏，可以让你随时了解我们推动了什么变化以及发生了什么。所以，是的，确实有这个方面，内容是在共享主机上编辑的，但我们实际上是在本地机器上进行开发。

凯文:好的。那么，Adam，我想听听你们每个人的意见，除了停机时间之外，你们认为在这个项目的特定领域面临的最大挑战是什么？还是例行公事；和你预期的完全一样吗？

亚当:是的，我做的事情并没有太多惊喜。我的意思是，如果有一两个挑战的话，那就是有很多代码要编写，我很幸运，Lox 和 James 在比赛快结束时在那里帮忙。我猜另一个卷曲的事情是计算尺寸排序的飞行，并确保布局将工作。

凯文:所以你之前提到过；这是一个深思熟虑的策略吗？就像是你从一开始就计划做的额外的事情，你认为这会让你在竞争中领先一步？

**Adam:** 在比赛前的一些聊天中，我们已经考虑过这种事情，这种“占上风”的想法，但直到我们见到客户后，我才决定我们会一直这样做。所以，我们的客户是[狮子听力犬](http://hearingdogs.asn.au/ "Hearing Dogs | We're all ears for the hearing impaired")，我们与有听力问题的人打交道，他们基本上有这些狗来帮助他们警惕危险或电话铃响之类的事情。在与客户的交谈中，他们发现一些有听力问题的人也有潜在的视力问题，因为他们可能是老年人，他们可能头部受过伤，所以他们可能有视力问题。

凯文:听起来，在不一定现代的浏览器上放大字体非常重要。

亚当:对，没错。

从用户体验的角度来看，你还有什么惊喜吗？

詹姆斯:我不认为有什么意外。我想补充一下 Adam 所说的，我们知道可达性也是我们在活动结束时被评估的关键因素之一，所以这就是为什么我认为 Adam，听着，我认为根据我对我们的理解，我认为我们在三个可达性指标中得分最高， 所以我认为亚当和拉克伦在这方面付出的努力确实帮助我们越过了终点线并赢得了比赛，所以我认为这无疑增加了他们的工作量，但这是非常值得的。

亚当:是的，我只是想说，考虑到它与 WCAG 2.0 AA 兼容，我对这个网站的表现非常满意。

詹姆斯:我认为在 24 小时内完成这项工作是一项非常艰巨的任务，因为我们有大量的时间和压力，而且所有团队都在努力实现设计工作的质量，这对你们来说是一项艰巨的挑战，所以，对你们来说，这也是一项巨大的挑战。

凯文:总体来说，拉克伦，你觉得你在这个项目中学到了很多关于 WordPress 的东西吗？我的意思是你以前有没有用 WordPress 建立过这样的端到端的东西？
