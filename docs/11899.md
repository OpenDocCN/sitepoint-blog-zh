# 宣布火狐 3.5 流行插件的更新

> 原文:[https://www . site point . com/announcing-updates-to-our-popular-add-ons-for-Firefox-35/](https://www.sitepoint.com/announcing-updates-to-our-popular-add-ons-for-firefox-35/)

CodeBurner 的新版本已经发布。这个版本不依赖于 Firebug——它是一个独立的扩展，并且在有限的时间内与一本免费的书捆绑在一起！[立即拿起你的。](https://www.sitepoint.com/premium/library)T3】

![The Refreshed Firefox Logo](../Images/2700e98f0b09f5dde5acf280e2ebbfaa.png)

随着最近 Firefox 3.5 的发布，我们非常高兴地宣布我们有用的开发工具的更新发布: [Dust-Me 选择器](https://www.sitepoint.com/dustmeselectors/)和用于 Firebug 的[code burner](https://www.sitepoint.com/browser-devtools-secrets-start-up-network-and-performance/)。

Dust-Me Selectors 是一个独立的扩展，它可以找到未使用的 <abbr title="Cascading Style Sheets">CSS</abbr> 选择器，对于更新旧网站特别有用，允许你丢弃所有那些多余的规则，而不用担心在某个地方可能有什么东西在使用它们。

顾名思义，CodeBurner for Firebug 是流行的开发工具 [Firebug](http://getfirebug.com/) 的扩展，并为 <abbr title="HyperText Markup Language">HTML</abbr> 和 <abbr title="Cascading Style Sheets">CSS</abbr> 添加了可搜索的参考资料。

我们一直在幕后努力更新这两个扩展，等待 Firefox 3.5 的最终版本——现在它已经发布了，它们就在这里！

Dust-Me 选择器特别受益于这次浏览器更新——由于 Firefox 的 JavaScript 引擎的改进，它现在的执行速度明显更快，典型的“查找…”操作比 Firefox 3.0 快 50%左右！

但这还不是全部！这两个新版本还带来了大量改进和新功能:

#### Dust-Me 选择器 2.2 版的新功能:

*   新的选项卡式首选项对话框更加整洁，更加用户友好
*   增加了一个新的蜘蛛偏好“不要蜘蛛外部网站”，这将蜘蛛限制在与站点地图相同域的页面上
*   修正了使“寻找…”和“蜘蛛…”键盘触发更可靠的跨平台
*   全面改进错误处理
*   删除了两个不必要的首选项，以简化爬行逻辑:“忽略 www 差异”和“忽略 URL 中的片段标识符”，这两个选项现在都一直处于启用状态

#### Firebug 1.2 的 CodeBurner 中的新功能:

*   改进的搜索算法给出了更好的结果
*   特殊搜索现在包括“对该元素有效的其他属性”以及实际为其定义的属性
*   CSS 属性的代码示例现在包括直接链接到[在线实例](https://reference.sitepoint.com/css/demos)
*   最初的搜索词现在显示在搜索结果中
*   稍微修改的许可证澄清了“使用”和“再分发”之间的区别

这两个更新都已经上传到了 Mozilla 附加组件网站，但是由于所有提交的更新都是手工审核的，所以可能需要一天左右的时间才能从那里或者通过自动更新渠道获得。因此，如果您等不了那么久，您可以直接从 SitePoint 获取它们:

*   [**Dust-Me 选择器 v2.2**](https://www.sitepoint.com/dustmeselectors/)
*   [**CodeBurner for Firebug v 1.2**]( https://www.sitepoint.com/browser-devtools-secrets-start-up-network-and-performance/)

一如既往，我们欢迎您对这两个扩展的评论和反馈——如果您对希望看到的功能有任何想法，请不要犹豫，让我们知道！
