# 采访——詹姆斯·阿特金森，phpBB 的创始人

> 原文：<https://www.sitepoint.com/james-atkinson-founder-phpbb/>

今天， [phpBB](http://www.phpbb.com/) 是互联网上最受欢迎的论坛之一，为数千个网站的社区提供动力。与大多数可用的脚本不同，phpBB 是开源的，在 GPL 下免费提供。它是一个成功的开源项目的很好的例子，即使不比它的商业对手更好，也一样好。

虽然 phpBB 拥有令人印象深刻的特性和出色的性能，但最大的优势之一是它的有用社区。该产品已经建立了一个很大的追随者，这保证了经验丰富的帮助只是一个快速搜索。

已经是一个伟大的脚本，新版本 2.2 即将发布。PhpBB 2.2 应该进一步改进管理功能，同时添加流行的新特性，使产品更加强大。随着版本 2.2 的发布，phpBB 开发团队将开始围绕产品的核心特性开发门户。这个门户项目可能会帮助 phpBB 在网站管理员社区中获得更多的欢迎。

最近，我有幸采访了 phpBB 项目的创始人兼项目经理詹姆斯·阿特金森(theFinn)。

【phpBB 是怎么起步的，为什么起步？phpBB 团队是怎么走到一起的？你有没有想过这个产品会像现在这样受欢迎？

我在 2000 年夏天创建了 phpBB，目标是为我妻子的网站提供一个论坛。当时，我所知道的唯一真正的论坛软件是 UBB 和 Phorum。对于一个穷大学生来说，UBB 太贵了，而且我一点也不喜欢 Phorum 的风格。我想要一个好的、开源的、平面风格的留言板，所以我着手克隆 UBB 的外观，并在 GPL 下免费提供。

在我在 DevShed 上发布了一条消息，请求人们测试我的新论坛之后，这个团队走到了一起。此后不久，我在 Sourceforge.net 上开展了一个项目，人们开始提交代码，并询问他们是否可以加入这个团队。从那以后就像滚雪球一样。

关于产品的受欢迎程度，当我开始这个项目时，我并没有真正的愿望。过去几年的增长速度真的让我震惊。

##### 开源的优势

**phpBB 是为数不多的开源公告板产品。你为什么决定让它开源？**

我不能说我曾经考虑过不让它开源。我和所有 phpBB 团队成员都相信自由软件运动的理想。

你认为 phpBB 开源后比其他商业论坛脚本有哪些优势？

我们最大的优势是开源给我们提供了比付费脚本更大的社区。加入 phpBB 社区是没有价格限制的——任何想在自己的网站上发布公告板的网站管理员都可以这样做。随着他们获得 phpBB 的经验，他们经常觉得他们想要回报一些东西，所以他们加入我们的社区，帮助软件的其他用户。这是开源运动最好的事情之一——人们互相帮助。

对于刚刚开始开源项目的开发者，你有什么建议吗？

最重要的是倾听你的用户。他们推动你的产品前进，并帮助你支持它。其次，你要坚持自己的理想。与其他“免费”论坛脚本相比，phpBB 如此成功的原因之一是因为我们的项目非常稳定。我们从来没有换过名字，我们从来没有换过领导，我们从来没有换过执照。我们坚持我们最初创建 phpBB 的理想，这种方法让我们的社区认为 phpBB 是强大而稳定的。

phpBB 的成功很大程度上要归功于其庞大的追随者和乐于助人的社区。看起来你不仅知道如何制作伟大的脚本，而且可能对网站管理员发布新论坛有建议…

我认为很多人在创建在线社区时犯的错误是他们没有从小处着手。我见过许多有 10 或 12 个论坛的版块，它们之间可能有 20 个帖子。这是不可能的。从 2 到 3 个论坛开始，让讨论真正开始，然后根据需要扩展。

此外，你需要一个挂钩——能够吸引人们进入论坛的东西，并成为讨论的起点。SitePoint 就是一个很好的例子。他们有一个包含大量内容的大型网站，吸引人们进入他们的论坛。人们讨论并最终将这些内容发展成更有用的资源。

##### 发展…详细

你能描述一下 phpBB 团队的成员是如何交流的吗？您如何决定是否要添加新功能？谁负责项目的哪些方面？

我们通过 phpBB.com 的论坛交流。“幕后”有员工论坛，我们在那里计划网站和软件会发生什么。

新功能的想法直接来自用户。在 Sourceforge.net，我们维护一个特性请求列表，开发团队(由 psoTFX 领导)决定如何以及何时实现所请求的特性。我们的开发周期是相当封闭的，因为我们不接受来自开发团队以外的人的代码；然而，我们总是乐意接受来自我们社区的功能请求和想法。

性能是论坛管理员最头疼的问题之一。phpBB 如何应对这方面的激烈竞争？你如何平衡新特性的实现和 phpBB 的整体性能？

开发团队总是考虑任何新特性实现对 phpBB 性能的总体影响。如果一个特性是一个巨大的资源猪，它要么从头开始重新设计(就像我们的搜索和许可系统已经做了几次)，要么被放弃。我们不认为软件的性能会因为一些华而不实的功能而受到影响。phpBB 2.0 的设计考虑了稳定性、安全性和性能。phpBB 的后续开发延续了这一重点 v2.2 的发布带来了巨大的性能提升和众多特性的增加。这些性能改进对于大型论坛尤其有价值。我们从拥有超大主板的用户身上学到了很多。

安全性也是一个重要因素，phpBB 看起来是一个非常安全的脚本。你计划做什么来确保 phpBB 在未来仍然安全？

安全性是 phpBB 早期版本(v1.x)的一个大问题。因此，我们非常积极地确保软件不会出现安全漏洞。在开发的每一步，任何大型特性的实现都要从安全的角度来考虑。我们的开发团队密切关注进入代码的一切，并确保没有办法破坏 SQL 查询来注入跨站点脚本问题。总的来说，我们在 v2.0 中遇到的最大问题是 JavaScript 被偷偷放入了适当的位置。然而，在 2.2 中，我们在实现一种更集中的方式来验证输入并确保它没有任何 XSS 式的问题方面取得了很大的进步。

保持 phpBB 的安全是我们所有开发的核心目标。

PhpBB 主要与 MySQL 一起使用，但它有一个用于其他数据库的数据库抽象类。你有在其他数据库上成功运行的例子吗？

我很想说，“哦，是的，X.com 使用 phpBB 和 Postgres，它运行得很好”，但我们听到的大多是 MySQL。似乎很少有人用软件使用其他数据库。对于大多数人来说，MySQL 显然是最受欢迎的数据库引擎。

然而，这不会阻止我们继续增加对其他数据库的支持。

##### phpBB 版内部

2.0 完全重写了 phpBB，看起来 2.2 也是如此。主要变化有哪些？

这不是完全重写。我们从零开始 2.0。2.2 重写了权限系统、审核功能、添加了用户控制面板，并在管理方面做了大量的添加。

phpBB 2.2 的主要新特性是什么？

全新的功能是增加了一个“用户控制面板”。phpBB 2.2 中处理用户详细信息的更模块化的方式，以及订阅的主题和论坛以及“朋友或敌人”列表是其他的补充。

我们还包括一个全新的版主控制面板，允许对论坛和主题进行更精细的控制，主题合并，以及向版主报告主题的能力。

帖子中的文件附件改编自非常受欢迎的文件附件模式，由 Acyd Burn 提供，他现在在我们的开发团队中。

大量的新东西已经被添加到管理部分:在线模板编辑，一个全新的 ban 控制系统，服务器负载/性能限制功能，允许 Google 和其他蜘蛛有效地抓取论坛的功能，以及更多需要永远在这里列出的功能！

我们还有很多其他功能，我想人们会喜欢的。

随着 2.2 版本的发布，将会开发一个门户脚本。你为什么决定做那件事？

这个项目是我们团队决定要做的。BartVB 是 phpBB 团队的长期成员，他是这个项目的先锋，这主要是他的想法。老实说，我很少参与这方面的事情，所以我不能很好地回答关于它的问题。

与市场上已有的其他产品相比，phpBB 门户的主要优势是什么？

主要的优势是它与 phpBB 的接口。想要快速建立一个基于论坛的网站的 Web 开发人员将能够做到这一点。目标之一是使产品高度模块化，因此它应该具有很强的可扩展性。

phpBB 的未来目标是什么？

phpBB 的未来目标一如既往:尽我们所能做最好的论坛软件！我们将遵循用户希望我们采取的方向。他们推动项目向前发展！

**当你回首往事时，有什么让你后悔的事吗？**

在 1.0 版本中拼错了“category”……我一直受到其他团队成员的批评。

我真的说不出有什么后悔的。这是一个伟大的旅程，我期待着未来十年左右运行这个项目。
