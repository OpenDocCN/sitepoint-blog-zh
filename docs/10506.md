# jQuery 走向移动

> 原文:[https://www.sitepoint.com/jquery-goes-mobile/](https://www.sitepoint.com/jquery-goes-mobile/)

一周前发布了 jQuery Mobile 库的早期版本。鉴于 jQuery 库在传统桌面 web 开发中的流行，我希望大多数 web 开发人员会对这个版本感到非常兴奋。但是 jQuery Mobile 到底是什么，在什么情况下你会考虑使用它？

## 名称又能代表什么呢

首先，jQuery Mobile 这个名字有点误导:它不是核心 jQuery 库的移动版本，而是一组可以轻松插入页面的接口组件。它位于您在常规页面中使用的相同 jQuery 库之上。实际上，它更像是 jQuery UI 库的移动版本。

## 好吧，那里面是什么？

jQuery Mobile 本质上是一个处理页面的框架。这些“页面”可能是整个 HTML 文件、外部页面，甚至是单个 HTML 文件中的各个部分。然后，该库使用一些时髦的过渡效果，通过 Ajax 将它们连接起来。在 pages 中，jQuery Mobile 为您提供了一系列风格一致的界面组件:工具栏、按钮、各种可以想象的列表和表单控件。查看[文档和演示](http://jquerymobile.com/demos/1.0a1/)页面，了解包含的所有内容的完整分类。

## 一路渐进增强，宝贝

jQuery Mobile 对该框架采用了一种有趣而新颖的方法:使用它所需的所有工作都将发生在您的标记中。无需编写一行 JavaScript 代码，就可以激活和激活大多数界面组件。这是通过 [HTML5 数据属性](http://html5doctor.com/html5-custom-data-attributes/)实现的。例如，要使用滑块组件，您只需在标记中加入以下内容:

```
<div data-role="fieldcontain">  <label for="slider">Input slider:</label>  <input type="range" name="slider" id="slider" value="0" min="0" max="100"  /></div>
```

这些`data-`属性为 jQuery Mobile 提供了创建如下滑块所需的线索:

![](../Images/93be56b8fcc44890c91f59518a70af8e.png) ** jQuery Mobile 的滑块控件**

jQuery Mobile 对它所做的一切都采用了类似的渐进式增强方法。这个想法是，不识别 HTML5 doctype 或`data-`属性的浏览器将只接收一个普通的 web 页面，这对于那些用户来说是可以接受的体验。总之，理论上是这样的。

## 我们告诉过你这是 alpha 版本，对吧？

现在这个框架仍然有很多问题。我无法在我的桌面浏览器(Firefox 4)上制作一个简单的外部链接，如果我在点击之前滚动页面，这些链接在我的 Android 2.1 设备上根本无法工作。鉴于该库的主要任务之一是在它不完全支持的任何平台上提供可接受但降级的体验，这些在所谓的 A 级浏览器上的破坏有点令人担忧。此外，我在一位同事的 iPhone 上运行了一些测试，性能远非理想:滚动比在普通网页上粗糙得多，过渡和动画有点起伏，一些链接无法可靠地工作。

## 走向本土化？

虽然 jQuery Mobile 确实是一项令人印象深刻的工作，但我不禁觉得它的适用性会相当狭窄。大多数移动网络应用程序在没有它的情况下会表现得更好，因为它们的大部分功能都依赖于浏览器，如果 iOS 上的应用程序实际上是本地的，它们会表现得更好，看起来也更本地。根据设计决策，比如在每个页面的顶部默认包含一个后退按钮，我猜测这个项目背后的主要意图是为最终的本地部署开发应用程序。如果它是为网络应用程序设计的，那么依靠浏览器(或设备)的后退按钮肯定会更好，对吗？如果你想使用 PhoneGap 这样的框架为 iOS 开发具有本土感觉的应用程序，并且可以移植到其他平台，那么一旦解决了这些问题，jQuery Mobile 可能会是一个有趣的解决方案。如果你不介意为非 iOS 用户提供一个本质上模仿默认 iOS 行为的界面。然而，如果你打算让你的应用程序在浏览器中运行，我觉得 jQuery Mobile 给了你很多不必要的惊人效果，可能会对你的一些用户体验产生负面影响。一个链接点击后会打开一个新页面，这没什么不好；这就是你的用户所期望的。添加一个滑动的(或弹出的，或淡出的)Ajax 负载——仅仅是为了“感觉自然”——似乎有点不对。用 JavaScripted 定制版本替换原生浏览器表单控件也是如此；大多数用户会更习惯于操作系统或浏览器提供的表单控件。为什么你的 web 应用程序需要一个 iOS 风格的开关切换，当你可以使用一个复选框来节省下载所有 JavaScript 的成本时？(当然，我确实相信在大多数 web 浏览器的当前版本中没有实现的自定义表单控件是有用的，比如上面的滑块。)但也许这只是我。如果你已经玩过 jQuery Mobile，你有什么想法？
