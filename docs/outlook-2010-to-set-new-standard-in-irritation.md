# 展望 2010 年，设定新的刺激标准

> 原文：<https://www.sitepoint.com/outlook-2010-to-set-new-standard-in-irritation/>

创建 HTML 电子邮件是一项艰巨的任务。

当你把所有相关的桌面客户端和普通的网络邮件提供商加起来的时候，你正在谈论 15-30 个你需要测试的平台。即使所有的邮件客户都是友好、现代和可预测的网民，这仍然是一项艰巨的任务。

他们不是。

几年前——令人欣喜的是——当[微软选择用 Word](https://www.sitepoint.com/microsoft-breaks-html-email-rendering-in-outlook/) 取代其 HTML 渲染引擎来削弱其旗舰邮件产品的渲染功能时，事情变得更加困难。

这大致相当于用一个沙楔代替你的割草机——你可以试一整天，但它就是不能割草。

虽然当时有人猜测这一决定是出于安全考虑，但微软已经明确表示，他们只是更感兴趣的是让 Outlook 2007 到 Outlook 2007 的体验更好(详情请阅读 [Campaign Monitor guys 的帖子](http://www.campaignmonitor.com/blog/post/2799/microsoft-to-ignore-web-standards-in-outlook-2010/))。

这必须停止。电子邮件是一个开放的标准，我们应该已经过了需要继续向微软争论的时候了。简而言之，Word 对 HTML 的可怕理解耗费了我们所有人的时间和金钱，同时也降低了每个人的体验。

在这个时候，微软似乎打算在 Outlook 2010 中使用 Word 渲染引擎——但还不算太晚。该应用仍处于早期测试阶段，微软正在征求反馈意见。

那我们该怎么办？

你能产生的最直接的影响就是去 http://fixoutlook.org/发推特让微软知道这并不酷。

微软对 fixoutlook.org 的倡议给出了[这个令人失望，坦白说相当蹩脚的第一个回应。](http://blogs.msdn.com/outlook/archive/2009/06/24/the-power-of-word-in-outlook.aspx)
