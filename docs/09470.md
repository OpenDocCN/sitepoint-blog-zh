# 面向网页设计师的云存储

> 原文：<https://www.sitepoint.com/cloud-storage-for-web-designers/>

云不再仅仅是程序员的专利。大多数云提供商都有简单、直观的界面来管理云产品和服务。作为一名网页设计师，了解如何使用这些服务可以让你的工作生活变得更好。

让我们回顾一下阻碍网页设计者深入云计算世界的一些常见误解:

## 误区:云存储只有在你需要巨大的可扩展性时才有用。

云存储的一大优点是它可以永远扩展。但这只是众多好处之一:

## 可靠性

当然，你的网络主机说它有夜间备份。但我保证当备份失败时他们有出路。大多数云存储服务在不同地理区域的多个数据中心之间提供 3 或 4 级冗余。大多数网络主机不能提供这一点。

## 表演

云存储通过两种方式提高网站性能:

*   如果流量激增，云提供商会自动复制常用资源，以便同时满足更多请求。
*   通过将您的图像、视频和其他下载内容从您的 web 服务器转移到云存储，您的 web 服务器现在需要处理的请求明显减少。这意味着在你需要升级之前，你的网络服务器可以处理更高的页面浏览量。

## 网站可移植性

想象一下，如果你曾经工作过的每个网站现在都将所有的图像、视频和其他更大的文件存储在云中。如果您需要将您的网站从一个主机迁移到另一个主机，或者将您的共享主机升级到专用主机，迁移是轻而易举的事情！我工作的大多数网站大约 90%以上是媒体文件存储，10%或更少是 html/样式/脚本文件存储。自从我开始在云中存储媒体后，移动、复制和备份我的工作变得更加容易和快捷。

## 误区:云存储只能通过 API 调用来访问。

完全错误。大多数流行的存储服务都提供了一个 web 界面来管理文件。更好的是，有大量的免费和商业应用程序可以让你从桌面上轻松管理云存储。根据我的经验，这些程序比 FTP 或 SSH 更快更容易使用。

我用来管理云存储文件的程序是 CloudBerry Explorer。它很好，因为它有一个免费的版本，它真的很容易使用。CloudBerry Explorer 的版本支持许多主要的云存储提供商，包括亚马逊 S3、谷歌文档、Nirvanix 和 Azure。它还支持 FTP 帐户。

CloudBerry 和其他类似的程序，工作起来很像你已经习惯的 FTP 软件。您可以创建多个帐户，并在任何位置之间拖放文件。您可以设置文件的权限；将它们组织到目录中，等等。

对于每个网页设计师来说，找到合适的工具是不同的。我建议查看最受欢迎的客户端(大多数都有免费版本或免费试用版)，并选择最适合您工作的客户端。最好的云存储客户端包括:CloudBerry、S3 浏览器、Cross FTP、3Hub、Panic、Transmit、S3Fox、Bucket Explorer 和 Gladient。

## 误区:云存储更贵。

当然，有一百万的网络主机提供无限的存储和带宽。但是，如果你曾经试图运行一个网站，实际上获得这些主机的流量-你知道你得到你所支付的。

云存储的成本非常有竞争力，大多数受欢迎的提供商都提供免费层，足以满足大多数小型网站或初创应用程序的需求。

但最重要的事实是，当你为你所使用的东西付费时(云存储中没有无限的计划)，你也在为比你从一个预算网络主机得到的更好的性能付费。

## 误解:云存储没有专用服务器快。

这部分是对的，但大部分是错的。如果您在最好的数据中心拥有最好的专用服务器，并连接到最好的互联网主干网，您将获得极快的性能…首先。

没有负载，你的服务器就火了。一切都会立即闪现在你的屏幕上。将这与从云存储中加载图像进行比较，您会注意到细微的差别。

但是一旦你的网站上线，流量开始增长，这种差别就不重要了。随着时间的推移，您的服务器将无法继续提供极快的速度。为了保持速度，您必须投资更多的服务器、集群和负载平衡，这反过来会降低性能。

如果您从云存储开始，您将从一开始就获得出色的性能(尽管不是绝对最佳的性能)。但最重要的是，这种性能永远不会随着网站和流量的增长而降低。最重要的是，您永远不必担心管理硬件来保持网站性能。

# 用亚马逊 S3 弄湿你的脚

这是您用云存储正确完成下一个项目的清单。我已经概述了开始使用亚马逊 S3 应该采取的步骤，但是这个过程与其他流行的云存储提供商非常相似:

## 1.创建亚马逊 AWS 帐户

如果你还没有亚马逊 AWS 账户，那就注册一个:http://aws.amazon.com。注册是免费的，他们的大多数产品和服务都有一个慷慨的免费层。你可以随心所欲地玩游戏，而不会产生任何费用。你确实需要申请一张信用卡，但是在你超过免费等级之前不会被收费。

## 2.启用 S3 服务

登录 AWS 管理控制台后，转到 S3 选项卡并启用服务。

## 3.安装并运行 CloudBerry Explorer(免费版)

安装 CloudBerry Explorer 后，找到程序顶部附近的“Source”下拉菜单。选择“新亚马逊 S3 账户”。您将需要访问密钥和秘密密钥才能继续(参见下一步)。

## 4.获取您的访问密钥和秘密密钥

*   在亚马逊 AWS 管理控制台；点击屏幕顶部的“帐户”。
*   找到并单击“安全凭证”链接。
*   滚动到“访问凭证”并单击秘密访问密钥下的“显示”。

## 5.创建您的第一个桶

在 S3，“桶”就像一个域名。事实上，当您创建一个 bucket 时，它会根据您提供的名称获得自己唯一的域名。要创建存储桶，请在您添加帐户的源下拉列表中单击“新建存储桶”按钮。它看起来像一个带蓝色/紫色的立方体。系统会提示您输入存储桶的名称。这个名称(像域名一样)在所有用户的所有亚马逊 S3 存储桶中必须是唯一的。

## 6.开放权限

默认情况下，您的 S3 存储桶是锁定的，只有您可以访问您在此添加的文件。对于您的网站，您希望任何人都能够访问这些文件，因此我们需要进行更改:

*   右键单击您的新桶。
*   选择“ACL 设置”。
*   找到标记为“所有用户”的行，并单击“读取”下的复选框。
*   单击复选框“应用于所有子文件夹和文件”。
*   保存您的更改。

## 7.上传您的第一个文件

首先，双击您的新桶将其打开。然后，使用拖放，您可以将文件直接从您的计算机上传到您的存储桶。为了便于测试，我建议上传一个图像文件。

## 8.查看您的文件

右键点击你刚上传的新文件，点击“网址”。这将打开一个对话框，显示您的文件的网址。将它复制并粘贴到浏览器中进行测试。哇，好多步骤啊！

不可否认的是，一开始启动和运行起来似乎需要很多步骤。但是如果你仔细想想，这些都是你设置一个 FTP 帐号和上传文件需要做的相同步骤。这些步骤中的大部分你只需要做一次:

*   注册一项服务
*   创建一个独特的位置
*   配置您的客户端软件
*   配置文件访问权限

云存储并不是每个项目的正确解决方案，尤其是当您只有很少的图像时。但对于大多数网站来说，云存储可以让管理任何项目的大量媒体存储变得更加容易。

你甚至可以在 CMS 和 WordPress 这样的博客平台上使用云存储。WordPress 有一些很棒的插件，可以很容易地存储你在亚马逊 S3 上传的图片。同样，这里最好的好处之一是可移植性。如果你需要移动或升级你的 WordPress 站点，这很容易。无论你把网站放在哪里，你上传的所有图片都会在你放的地方。

重要注意事项

请记住以下几点:

## 您客户的 AWS 帐户

如果你在做客户项目，让你的客户建立他们自己的亚马逊 AWS 账户。你可以使用自己选择的软件来管理上面的文件，但是账单(一旦超过免费层)将直接交给你的客户。不幸的是，很难跟踪云存储中特定文件或文件夹的流量和存储(尤其是流量)。您总是知道您的总带宽，但知道哪些文件或文件夹正在获得流量需要大量的工作和专业知识。有一些服务可以提供帮助，但最终还是很复杂。我建议让你的客户建立自己的亚马逊 AWS 账户。

## 继续了解云存储

每种云存储服务都提供了一套强大的功能——本文只是触及了您能做的事情的皮毛。一些功能对于一个普通网站来说有些过头了，但是许多功能对于网页设计者来说是有用的，即使是对于小而简单的项目。
