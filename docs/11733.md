# 慢点！接受紧急项目前要问的 5 个重要问题

> 原文：<https://www.sitepoint.com/questions-to-ask-before-taking-on-rush-projects/>

可以理解的是，所有客户都认为他们的项目很紧急。这是当今商业和社会的本质——我们希望现在就做，而且要做好。但是，在你承诺熬夜完成一个不切实际的最后期限之前，这里有五个问题可以问，这将有助于你清楚地知道你将要做什么。

## 为什么这么急？

很多时候，这个最初的问题会告诉你所有你需要知道的关于紧急事件背后的事情。两种可能性可能包括:

有些事情不尽如人意，影响了客户自己的顾客。

如果是这样的话，匆忙肯定是有道理的。更进一步，如果这是你最初为客户提供的工作的问题，那么你至少有责任检查一下。根据导致问题的原因和您与客户达成的协议，排除意外故障可能符合您的初始合同。

如果这项工作是由别人完成的，你可能会问为什么客户不回去找做这项工作的人。获取更多关于这背后的推理的信息也将帮助你确定如果你承担 rush 项目，你将可能扮演的角色。

这是一个新项目，客户很高兴看到它被执行。

虽然这很常见，实际上也是一个好的迹象，但它并不一定保证一个紧急的工作。尤其是在新项目启动的情况下，放慢速度，一步一步地进行项目可能是最有意义的。一定要给客户留下深刻的印象，花必要的时间合理地计划、组织和执行一个新项目的价值；他们可能会感谢你的建议。

## 范围是什么？

既然你知道了为什么会有这样的匆忙，花点时间去了解工作的细节，需要做什么，什么时候需要完成。一个好的起点是完成你的[项目申请表](https://www.sitepoint.com/vital-information-for-a-web-design-project-request-form/)或你开始与新客户合作时使用的任何其他工具。这取决于你是否清楚地理解了项目的每一个元素，所以试着问最重要的问题来获得你需要的信息。

## 客户是谁？

每当有人找我谈一个紧急项目，最大的决定因素之一就是我是否和客户有关系。在大多数情况下，如果工作涉及到我目前正在工作并且喜欢工作的客户，我愿意接受稍微有点挑战性的时间表。或者，如果客户是现有的重要客户或受人尊敬的同事推荐的，他们往往会更重视他们的请求。

## 客户能在自己的期限内完成吗？

我见过许多仓促的项目嘎然而止，因为客户不能满足他们自己的时间表。一旦你知道了范围，在开始工作之前，试着为你和客户制定一个行动项目和责任的清单。然后，确保客户承诺履行他的职责。

如果看起来他们很难给你提供你需要的信息，这可能不是一个好的紧急项目。如果发生这种情况，你可以和客户一起修改最后期限，这样对他们来说更现实，然后仍然为客户完成工作。

## 你能在要求的时间内完成这项工作吗？

我考虑的最后一个因素是，接受这个紧急项目是否会占用我目前正在做的其他工作的时间，是否会让我错过最后期限，是否会给我带来过度的压力或睡眠不足。当分析紧急工作时，不要忽视你自己的承诺和可用性。即使所有其他的因素都检查过了，也要确保你有时间承担这项工作，而不会面临任何其他的负面后果。
