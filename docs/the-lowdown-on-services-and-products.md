# 服务和产品的内幕

> 原文：<https://www.sitepoint.com/the-lowdown-on-services-and-products/>

以下内容转载自论坛报#424 网站。

你应该提供服务还是产品？

在讨论服务和基于网络的产品的优点时，经常会听到这样一句老话:“这山望着那山高。”。下面列出了这两种模型的一些优点和缺点。哪些适合你？

**产品—优势**

为您提供无限的收入潜力，因为时间限制很少(与销售服务不同)
为消费者提供更低的价格，因为成本可以分摊给更多的客户
允许您专注于构建一个主要项目，而不是几个小项目
提供不同收入模式的选择，这些模式随着时间的推移更容易修改
提供更好的剩余收入机会，持续的许可费
提供比小型 web 服务更容易的持续销售选项。

**产品—缺点**

需要更多的时间和金钱的前期投资，尤其是营销方面
可能会出现竞争对手创造类似产品并与你直接竞争的情况。
有必要进行市场调查，以避免最终客户群很少或没有。
面临接到比预期更多的支持查询的风险
由于缺乏即时现金流，可能会延迟长期增长

**服务—优势**

创收更快，因为向市场销售您的服务比销售全新产品更容易
为各种项目提供机会，保持您对每个新项目的兴趣
在满足市场需求方面提供更多功能，而不必重新构建成熟的应用程序

**服务—缺点**

降低您的收入能力，因为它纯粹基于您可以收取多少费用以及您需要多少员工
限制了短期增长；例如，雇用员工比让新服务器上线更耗时
比产品许可更容易受到市场繁荣的影响

还要记住，正如 [Dave 提到的](https://www.sitepoint.com/interview-with-dave-greiner-freshview/)，拥有两者的混合模式非常简单:销售你当前的服务，同时开发和提供产品。这类似于企业销售主机、库存图像和 CMS 许可的方式。
