# 使用 HTML 和 CSS 以正确的方式构建自己的网站

> 原文：<https://www.sitepoint.com/back-to-basics-with-build-your-own-web-site-the-right-way-using-html-css/>

随着谷歌似乎每周都会推出一款复杂的新网络应用,在 SitePoint 总部，人们很容易被每天冲击网络的酷酷的新东西所吸引。但是在过去的一周里，我有机会回归基础，回忆起创建你的第一个网页是什么感觉。

我的女朋友杰西卡正在完成一个研究生教学学位，她的任务之一是制作一个网站，描述计算机在课堂上的多种使用方式。在准备这项作业时，她的讲师简单演示了 Dreamweaver，并建议不懂这个程序的学生可以用 PowerPoint 设计他们的网站。)作为一系列交叉链接的幻灯片，并将它们导出为 HTML。

现在，杰斯正在学习教授戏剧和英语，所以她可以很容易地走上阻力最小的道路，但是当我们刚刚收到办公室的伊恩·劳埃德的《用 HTML & CSS 的正确方法建立自己的网站》时，我放了一本在她的桌子上，暗示她可能会觉得读起来很有趣。

过了不到一个星期，有一天下午她给我发了一条即时消息，里面有一个网址，问我:“你觉得怎么样？”做好了最坏的打算，我点击进入，惊讶地看到浏览器窗口底部弹出的绿色复选标记。站点*已通过验证！*

事实证明，Jess 非常喜欢这本书，她甚至在自己的网站上增加了一页来解释她是如何获得新的 HTML 和 CSS 技能的:

> 我读了这本书，这本书假设没有任何先验知识，并且很快就学会了如何使用有效的 HTML 和用 CSS 创建时髦的 web 布局，这意味着我可以使用一个样式表将我的样式应用到许多页面上——如果我想在所有页面上更改一些东西，这非常方便。
> 
> 结果呢？你正看着它。事实上，我创建的第一个页面，我的主页，是有效的 XHTML 1.0 Strict，这是非常非常少的第一次网页建设者可以声称的成就。事实上，我所有的网页都 100%通过了 HTML 和 CSS 验证，除了我的博客页面中的 Blogger 代码。但这是他们的问题，也是我为什么会理想地使用手工制作的博客技术的另一个原因，如果我和学生一起使用博客的话。
> 
> 我应该说，尽管我知道我正在推动博览会和纯粹吹牛之间的界限——这个网站只花了我五天时间。从打开书到写下结束语。这个事实甚至让我晕头转向。

我仍然习惯于我的女朋友宣扬 web 标准的福音，但我不能更自豪了——不仅因为 Jess 在她的第一个网站上做了如此专业的工作，还因为 Ian 出了一本书，可以激发一个没有 web 设计经验的人以正确的方式建立网站的热情。

现在把最新更新的 [<cite>HTML 乌托邦:使用 CSS 设计无表格，第二版</cite>](https://www.sitepoint.com/books/css2/) 放在她的桌子上…
