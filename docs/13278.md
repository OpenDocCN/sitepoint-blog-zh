# 电子邮件标准项目:缓和电子邮件战争

> 原文：<https://www.sitepoint.com/the-email-standards-project-moderating-the-email-wars/>

在我们今天发表的对大卫·格雷纳的采访中，我问他关于他和 [Freshview](http://freshview.com/) 团队即将成立的组织，该组织旨在建立电子邮件客户端标准支持的基线。

戴夫透露，这个组织的域名将是[email-standards.org](http://email-standards.org/)，这个组织的第一个网站将在下个月左右出现。

到目前为止，电子邮件标准项目(我假设它将被称为*，尽管缩写 ESP 已经代表了[一堆其他东西](http://en.wikipedia.org/wiki/Esp/))已经声明他们计划:

1.  建立 HTML 电子邮件发件人在电子邮件中需要支持的标准基线
2.  记录每个主要电子邮件客户端制造商为了支持 web 和相关标准而需要做出的重要更改(Dave 给了我一个付费服务的演示，他们将把该服务添加到他们的 Campaign Monitor 产品中以实现这一点)
3.  创建一个简单的 acid 测试，可以很容易地看出电子邮件客户端是否支持这个基线，很像 [WaSP](http://webstandards.org/) 的 [Acid2](http://www.webstandards.org/action/acid2/) 测试。

网络浏览器已经走过了漫长的道路，网络也因此变得更好；然而，电子邮件大战才刚刚开始。

*我认为对此进行推测会比再次采访戴夫*更有趣。*
