# @media 2008:从脚手架还是从零开始？

> 原文：<https://www.sitepoint.com/atmedia-2008-day-two/>

今天早上，我可能比许多与会者的眼睛都亮了一点，因为我错过了@media 派对，以便带我的女儿去看音乐剧《邪恶》(顺便说一句，这部音乐剧非常好)，所以我及时赶到了会场，以便获得前排座位，听 Nate Koechley 的主题演讲“专业的前端工程”。在过去的几年里，我觉得我们已经看到了前端开发人员作为一名专家的崛起，Nate 讨论了从事这一工作职能的人可以提高他们的技能，成为“不知疲倦的质量潮流引领者”的一些方法。内特还谈到了在雅虎工作。他们在那里发现了一些东西，比如对 CSS 使用@import 会造成性能下降，因为导入的样式表是最后加载的，而页面直到最后才加载。其效果就好像在结束 body 标记之前有一个链接的样式表一样。

我留在主跑道上听乔纳森·史努克的演讲“站在巨人的肩膀上”。他列举了一些我们可以使用现有代码构建自己的应用程序的方法——这成了那天的一个主题。他谈到了诸如 JavaScript 库、代码框架以及可用的 API 之类的东西，它们可以使创建应用程序变得更加容易。

在午餐前的最后一次会议上，我换到了另一个话题，听马克·帕切科谈论《卫报》网站的重新设计和《T4》节目的重新设计。Marc 解释了卫报如何拥有超过 500K 的 CSS，这很难维护。这是我两年前在@media 的一个专题讨论会上谈论过的一个话题，也是一个至今仍未解决的问题。拥有大量 CSS 的复杂网站很快变得非常难以维护。Guardian 做的一件事是在开发期间有许多单独的文件——分离出颜色、网格等，然后这些文件被组合成一个大文件，作为构建和部署过程的一部分。这意味着它们可以方便地使用单独的文件进行开发，但不会因为它们的存在而影响性能。他还提到了我们遇到的另一个问题，那就是用 Subversion 合并文件对于 CSS 来说效果并不好，你可以很容易地找到合并在一起的规则，而这些规则不应该有令人惊讶的结果。

Matt 谈到了 Dopplr，一个面向频繁旅行者的社交网络，强调了一些有趣的方式，他们正在使用其他 API 来增加他们已经拥有的数据。Dopplr 没有试图复制人们在其他社交网络上已经拥有的功能，因此也有一些功能，例如旅行者可以将他们的 Flickr 照片流与他们的旅行信息相链接，而不是构建他们自己的照片库应用程序。这种方法对于任何计划构建新的 web 应用程序的人来说都非常有帮助——决定你要做什么并专注于此，然后使用来自其他应用程序的 API 来使人们能够将他们的各种配置文件和数据链接在一起。这实际上是 Jonathan Snook 今天早些时候讨论的一些内容的延续。

午饭后是我自己的小组，其他小组成员希望我给我们一个热情洋溢的评论——或者也许我应该在这一点上评论一下观众。我们的主题是“交流最佳实践”，小组主席是保罗·博格，伴唱是默里·罗文、[帕特里克·h·劳克和我。我们在大型企业、学术环境中的内部开发和小型企业中积累了丰富的经验，因此我希望我们能够合理地概述我们可以沟通的方式，并在不同的工作方式中使用最佳实践。我认为我们都表现得很好，就像观众一样——那些我能透过灯光看到的人！](http://www.splintered.co.uk/)

座谈会后，我终于可以放松下来，享受这一天剩下的时光了。James Adam 和[Simon Williamson](http://simonwillison.net/)紧随其后，在名为“探索服务器端”的会议中，他们分别讨论了 [Ruby on Rails](http://www.rubyonrails.org/) 和 [Django](http://www.djangoproject.com/) 。James 向我们展示了一个 Rails 的快速示例——使用脚手架进行现场编码演示，然后 Simon 向我们展示了使用 Django 复制@media 2008 网站的简易性。虽然詹姆斯的现场编码确实很勇敢(我几乎不能在没有编码的情况下在观众面前说话)，但我觉得人们被脚手架产生的明显低劣的标记偏离了方向。在一个会议上，web 标准和不引人注目的脚本技术是议程的重点，避免使用支架方法可能会更好，因为它可能在生成页面的速度方面令人印象深刻。快速创建页面固然很好，但是如果这些页面还没有准备好投入生产，这真的有帮助吗？然而，这完全是另一篇博文的主题！我不是 Django 用户，也不是 Python 程序员，但我很喜欢 Simon 的演示，它让我觉得我想和 Django 玩一玩，看看用它建立和运行一个网站是多么容易。最后提出的一个问题是，为 Rails 或 Django 提供主机有多容易，主机是我的公司继续使用 PHP 的原因之一——我们希望能够为人们提供在主机出现问题时非常容易托管和移动的应用程序，然而，听起来好像在过去一年左右的时间里，有了更多的选择来托管 Rails 和 Django。当然，两者都有优秀的社区，所以如果你正在寻找主机，应该会有帮助。

这一天再一次以热点话题小组结束，重点是发展。小组成员是史蒂夫·福克纳、西蒙·威廉森、乔纳森·史努克、内特·凯切利，马特·比德尔夫担任主席。西蒙·威廉森(Simon Willison)再次为前端开发人员作为严肃的工程师进行了令人振奋的辩护，内特提到的一个小趣闻是，IE6 现在已经在雅虎上被 IE7 超越了！日志。

这是有趣的一天，除了担心我自己的小组，我真的很喜欢它，有趣的是，在这两天的过程中，在一个真正的网页设计会议上，一些演示是多么的技术化。我认为这表明了我们正在向前端开发本身成为一项更重要的技能的方向发展。对于英国的前端开发人员来说，好消息是，来自《卫报》和雅虎等大公司的几位发言者提到，他们正在招聘前端开发人员，因此需要具有扎实的 CSS、HTML 和 JavaScript 技能的人。我一直在寻找可以贴在这里的幻灯片，但由于许多演讲者在会议结束后要走一段距离才能回家，而其他人则在本周末参加了伦敦的 BarCamp。我会编辑这篇文章，在我找到它们时添加它们——如果你先找到它们，添加评论的链接。如果我在过去两天里讨论的任何会议听起来很有趣，这些发言都被记录下来，并将在不久的将来出现在 [@media 2008 网站](http://www.vivabit.com/atmedia2008/london/)上。
