# WD06: Jeremy Keith，“解释 Ajax”

> 原文：<https://www.sitepoint.com/wd06-jeremy-keith-explaining-ajax/>

凯文·杨克在澳大利亚悉尼举行的[网络指南南方](http://webdirections.org/)会议上报道。

正如所料，Jeremy Keith 今天在 Web Directions South 上对 AJAX 做了清晰有趣的介绍。我不会在这里重复这些材料，因为我们在 SitePoint 上已经有了大量的好的介绍材料。

当谈到“类似桌面”的 AJAX 应用程序，如基于 web 的即时消息应用程序 [Meebo](http://www.meebo.com/) 时，Keith 采取了一种有些相反的观点，认为 AJAX 有比简单地在浏览器范围内复制桌面应用程序更好的用途(并受到该环境的限制)。我倾向于同意。

Keith 还花了一些时间讨论了在网站中添加 AJAX 所带来的设计挑战。在他的清单上排在首位的是这样的问题，比如提供状态信息和关于每个 AJAX 应用程序核心的静默服务器通信的反馈。

Keith 的结束语是关于当前 AJAX 应用程序缺乏可访问性，以及这是如何阻碍 AJAX 成为现实世界 web 开发的一个普遍方面的最大挑战。

接下来的几周，这个会议将会在 [Web Directions South 博客](http://webdirections.org/)上进行播客。
