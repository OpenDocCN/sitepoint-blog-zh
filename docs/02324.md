Chris:

所以我们拭目以待。我们大概进行了一个月。大概还要好几个月才能投产。我对此感到兴奋。

对于那些稍微熟悉 CodePen 的人来说，我们在 CodePen 上有一个名为 Collab Mode 的 [PRO](https://codepen.io/pro/) 功能，有点像 Google Docs。当你打开文档，你把 URL 发送给其他人，他们也打开了文档，你会看到两个光标在那里，你们可以一起打字，看看对方在做什么。

这是 CodePen 的专业特性之一。有些人需要它。当你需要它的时候，它就在那里。你可以去专业的，并得到它，它的工作相当不错。但是想象一下在幕后发生了什么，我马上就会讲到。但是还有另一个叫做教授模式的功能，它可能更象征着这一点。

它类似于 Collab，但只是单向的。所以我在打字，我可以把这个网址发给——比如说——我所有的学生，如果我是一名教师的话。假设有 30 个人在这个网址上看着你:当我打字时，我的每一次击键都显示在他们的屏幕上，这不是视频，这就像我们实际的界面。

你可以看他们编码，但不仅仅是代码:就好像如果我打开一个菜单，菜单就会打开。如果我改变一个选择，这个选择也会改变；如果我输入内容，它们也会同步。所以他们真的在观察使用这个网站的感觉。那就是*状态*；一切都是国家的。

就像什么是开放的，什么是不开放的，事物的价值观是什么。我们就像，你知道我们应该怎么做吗？我们应该把它做得像一个大的 JSON——只是一个描述状态的 JSON 的 blob 这是我们几年前做的。

当事情发生变化时，我们会有一个小的发布/订阅模型，就像，好吧，这发生了变化，然后我会发布这发生了变化的事实，我会让所有这些客户端，这些学生，有一个开放的连接，你知道，一个 web 套接字连接，他们只是在等待事情被发布。

他们就像，有东西发布了，我订阅了那个东西:我要看到那个东西，在本地改变我的状态，然后让状态改变——比如打开菜单，或者给这个输入添加一个新字符。因此，我们编写了自己的小状态管理器、发布/订阅、web 套接字，将各种技术和 API 等结合在一起，以实现这一点。

它非常复杂，需要大量的工作，这就是为什么它是 CopePen 的一个专业功能。现在你看看 React 是如何工作的:React 和 Redux 是什么？它只是一个大的 JSON 状态对象。基本上是一样的，只是实际的框架是这样工作的。

当您在 Redux 或其他什么中改变状态时，它会通过 UI 相应地改变状态。但是我们不用用 jQuery 来做，不用编写我们自己的 Pub/Sub，也不用假装这一切——我对它如此感兴趣的原因是因为它已经是我们正在做的事情了。

只是我们必须手工缝合，现在我们能够切换到这个，我们就像，<q>哇，这感觉真的很自然。真的很好。</q>
