# Facebook 平台已经死了——好，让创新开始吧！

> 原文:[https://www . site point . com/the-Facebook-platform-is-dead-good-let-the-innovation-begin/](https://www.sitepoint.com/the-facebook-platform-is-dead-good-let-the-innovation-begin/)

在柏林的一次开发者大会上， [Lookery](https://www.lookery.com/) 的创始人 Scott Rafer，这是一家为社交网络应用发行商服务的广告网络，例如脸书的发行商，显然[宣布了脸书平台的死亡](https://www.allfacebook.com/2008/10/scott-rafer-the-facebook-platform-is-dead/)。根据 Rafer 的说法，自从 Facecbook 实现了他们的[重新设计的网页](https://www.sitepoint.com/will-facebooks-new-design-hurt-app-developers/)和画布页面以来，Lookery 网络上的出版商已经看到了近 50%的流量下降。

据 AllFacebook.com 的尼克·奥尼尔称，拉夫告诉观众，为了在新设计中生存下来，平台开发者应该跳槽，为脸书连接开发。Connect 是一种与第三方网站安全共享身份和社交图信息并将第三方数据带回脸书的方式。7 月，我们称脸书连接[信标做对了](https://www.facebook.com/sitepoint/)。

Rafer 认为 Connect 的机会对开发者来说是短期的，就像他认为的平台一样，开发者应该着眼于平台外的开发来创建应用程序。

Rafer 的建议并不坏——大多数应用程序开发者*应该*创建独立的属性，从外部与脸书挂钩，而不是把所有鸡蛋放在一个篮子里，可以这么说，把它们的存在置于脸书权力的摆布之下。如果他关于平台已死的断言是正确的，我认为这是一件好事。但我不认为它真的死了——它只是作为一个快速致富的方案而死了。在这个平台上仍然存在着巨大的机会来创建有意义的、有用的、大规模流行的应用程序。

许多无用的应用程序将开始消失，取而代之的是，脸书应用程序将成为外部应用程序的功能，平台和连接将在这方面发挥作用。对于许多应用程序来说，在脸书上有一席之地仍然很有意义，这就是机会所在。现在，一个新的应用程序要在脸书获得关注要困难得多，但对许多外部应用程序来说，整合脸书是一种增值，将有助于它们建立用户基础。

同样的演变可能会发生在其他热门应用平台上，比如 iPhone。随着 AppLoop 应用程序生成器(AppLoop Application Generator)的出现，降低了 iPhone 平台上创建应用程序的门槛，信噪比将会上升。想象一下，如果苹果批准每一个用 AppLoop Generator 制作的博客订阅应用，App Store 会是什么样子？呃。最终，随着噪音水平的上升，脸书将会发生同样的事情——平台将会“死亡”，而幸存下来的应用程序将(主要)是外部有用应用程序的扩展。在我看来，这对消费者来说是件好事。
