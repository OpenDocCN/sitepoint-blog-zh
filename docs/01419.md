你知道你需要一个快速的网站。你的访问者不想等待，谷歌奖励速度，你只是想创建最好的网站。

我们之前在文章[如何优化你的 WordPress 站点的性能](https://www.sitepoint.com/optimize-wordpress-sites-performance/)中讨论了一些加速你的站点的策略。我们在那篇文章中提到的一个关键策略是缓存。

在本文中，我们将解释这些选项，并帮助您做出决定。

## 缓存如何加快我的网站？

在你的网站上使用 WordPress 有很多好处。添加新帖子、调整网站外观和添加新功能都很容易。这绝对是我们要走的路，这也是为什么它是世界上最受欢迎的 CMS，支持着 7500 万个网站，或者超过 25%的网站。

但是所有这些便利都是有代价的。当有人访问您的网站时，您的网站有更多的工作要做，从而使它变慢。需要运行脚本，访问数据库，显示主题，运行插件。

缓存改变了这一切。

缓存是存储临时数据的地方。它把你的动态的、容易改变的网站，存储成静态的 HTML 文件，这样阅读起来更快。每次你的站点被修改，缓存需要被清除和重新生成，这通常是由 WordPress 插件触发的。

## 缓存的好处是什么？

使用 WordPress 缓存有三大好处:

1.  缓存提高了网站的速度和性能，尤其是页面加载时间。你的访客会花更少的时间等待，更多的时间阅读。
2.  缓存减少了 web 服务器的负载。您的数据库不需要经常被访问，并且使用的系统内存更少。这对于有限的共享主机计划来说有很大的不同。
3.  你会在搜索引擎上排名更好。在其他条件相同的情况下，谷歌通过在搜索结果中降低慢速网站的排名来惩罚它们。你的网站将变得更容易被发现，这可能会导致流量的增加。

那么，你应该使用 WordPress 缓存解决方案吗？对大多数人来说，答案是响亮的“是！”这是你能做的一件事，它能给你的网站带来巨大的性能提升，尤其是当你有高流量的时候。

什么时候不应该使用缓存？当你希望你的网站对每个访问者都有不同的显示时。一个例子是一个电子商务网站，其中的购物车将为每个用户不同的更新。

但这些都是特例。对于大多数 WordPress 网站来说，缓存是个好主意。

那么你从哪里得到一个呢？有两种选择:预包装，或自己卷。

## 选项 1:使用你的主机提供商的自定义缓存解决方案

一些主机提供商有他们自己的定制缓存解决方案，可能包括服务器端软件和定制的 WordPress 插件。这些都是精心调整，以最大限度地提高速度。

如果您的提供商提供这样的解决方案，请考虑使用它。这不仅会节省您的时间和精力，而且您不太可能在调整缓存以获得最大速度方面达到他们的结果。

一些例子:

SiteGround 的缓存解决方案是[超级缓存器](https://www.siteground.com/tutorials/supercacher/introduction)，基于 [NGINX](https://www.nginx.com/) (“引擎 X”)。它支持 HTTPS，据称可以让你的页面加载速度提高四倍:

> 超级跑车是 SiteGround 专为我们的客户开发的。它增加了一个网站可以处理的点击量，提高了网站的加载速度。超级缓存器包括 4 个不同的缓存选项，最大限度地优化您的网站。

这是一套四种不同的缓存解决方案(实际上，三种用于共享主机):

1.  静态缓存，默认情况下启用的静态资源的缓存解决方案。
2.  动态缓存，对 WordPress 等支持的动态网站有效。
3.  Memcached，它缓存数据库查询的结果。

为了在你的 WordPress 站点上获得最佳效果，请启用这三个选项。

[WP 引擎](https://wpengine.com/)调用他们的缓存解决方案 [Evercache](https://wpengine.com/our-infrastructure/) ，已经设置好，随时可以开箱即用。他们将它描述为他们的“秘方”,并声称加载页面的速度将提高 4-6 倍。他们建议你不要使用自己的缓存解决方案。事实上，他们明确阻止了许多缓存插件的使用。

> 因为 WP Engine 的 EverCache，不需要网站所有者或者开发者去管理缓存插件。WP Engine 会帮你解决这个问题。

[Varnish](https://varnish-cache.org/intro/index.html#intro) 是包括 Bluehost 和 DreamHost 在内的很多主机提供商使用的缓存解决方案，它可能是 WP Engine 的秘密酱的一部分。它很快，但是有一个主要的限制:它不支持 HTTPS。一些主机提供商将它与 NGINX 或其他软件一起运行，以避开这一限制。

你的主机提供商提供现成的缓存解决方案吗？在决定安装自己的缓存之前，先检查一下——它可以让您的工作轻松很多。事实上，选择主机提供商的一个很好的理由是，他们提供了一个快速且易于使用的缓存解决方案。

## 选项 2:使用 WordPress 缓存插件

如果你的主机提供商不提供缓存解决方案，或者你更喜欢 DIY 的方法，你可以安装一个 WordPress 缓存插件。额外的收获是，其中许多都有不仅仅是缓存的功能，使用我们在文章[如何优化你的 WordPress 站点的性能](https://www.sitepoint.com/optimize-wordpress-sites-performance/)中描述的技术。

你应该使用哪个插件？首先，它会给你一个显著的速度提升。其次，它将满足您对可配置性或易用性的偏好。第三，考虑成本。

今年早些时候完成的两个独立的基准测试(由 [Design Bombs](https://www.designbombs.com/top-wordpress-caching-plugins-compared/) 和 [Dev Shed](https://wpdevshed.com/best-wordpress-cache-plugin/) 进行)一致认为目前最快的三个 WordPress 缓存插件。他们对插件进行不同的排序，关于最好或最快的插件的争论是激烈且永无止境的。但是这三个表现很好，其中一个应该可以帮你搞定。

*   费用:单个站点 39 美元，3 个站点 99 美元，无限站点 199 美元(包括一年的更新和支持)
*   所有计划都有 30 天退款保证
*   根据一些基准测试，最快的缓存解决方案

> 不要浪费时间纠结复杂的插件设置。WP 火箭激活后发射——最小的配置，立竿见影的效果。

特点:

*   页面缓存:网页的快速加载
*   缓存预加载:通过搜索引擎改进索引
*   按需加载图像:图像仅在出现时才加载，从而缩短了初始页面加载时间
*   静态文件压缩:HTML、Javascript 和 CSS 文件被压缩，这样可以更快地下载
*   开发人员友好:旨在轻松定制。

### WordPress 的创造者 Automattic 的 WP 超级缓存

*   活跃安装数:100 多万
*   费用:免费
*   根据基准测试得出的“前 3 名”缓存解决方案
*   最快的免费缓存解决方案之一

> 这个插件从你的动态博客中生成静态 html 文件。在一个 html 文件生成后，你的网络服务器将服务于该文件，而不是处理相对繁重和昂贵的 WordPress PHP 脚本。

该插件以三种方式提供缓存文件，并且设置快速简单。建议大多数用户启用以下设置:

1.  PHP 缓存
2.  压缩页面
3.  不要为已知用户缓存页面
4.  缓存重建
5.  CDN 支持
6.  额外的主页检查。

这个插件不仅仅是缓存——它有一个类似 WP Rocket 的特性集。根据该网站，这些功能提供了以下好处:

*   减少页面加载时间，增加访问者在网站上的时间
*   改进的 web 服务器性能，以及承受高流量时段的能力

## 是时候做出选择了

哪个缓存选项最适合您？为您提供最佳平衡的产品:

1.  速度和功能
2.  可配置性与易用性
3.  成本。

这是我们的建议。

*   如果你的网络主机提供了一个缓存解决方案，强烈考虑使用它，特别是如果你重视易用性。我们首选的主机提供商 SiteGround 将一个优秀的缓存解决方案打包到他们负担得起的主机方案中，这将是一个很好的选择。
*   WP Rocket 快速且易于使用——你可以在 3-4 分钟内完成设置。如果你不介意 39 美元的价格，它可能非常适合你。
*   另一方面，如果你喜欢免费的用户友好的解决方案，选择 WP 超级缓存。它是由最了解 WordPress 的团队编写的，因为它的选项较少，所以更容易设置。
*   或者，如果您更喜欢更具可配置性的免费解决方案，请选择 W3 Total Cache。除了缓存之外，它还有一些功能可以让你不用安装额外的插件。
