# 完整的托管指南-第 1 部分

> 原文：<https://www.sitepoint.com/complete-guide-hosting-1/>

对于一个网站所有者来说，最重要的商业关系之一(无论网站是为了商业还是娱乐)应该是与虚拟主机公司的关系。

毕竟，你付钱给一个主机来照看你的网站——你应该期望他们好好照顾它。因此，显而易见，主机选择对任何管理或运营自己网站的人来说都很重要。

这个托管指南分为两部分。今天我们将:

*   看看为什么选择合适的主机很重要，
*   查看一份您可以根据自己的需求进行调整的托管评估清单，以及
*   考虑共享或专用主机的各个方面，以及选择服务器软件。

[然后，在第 2 部分](http://www.webmasterbase.com/article/732)，我们将详细探讨您在制定清单和评估不同主机时应该考虑的基本要素，包括:

*   磁盘空间
*   电子邮件帐户
*   FTP 访问
*   数据传送
*   带宽
*   多媒体支持

…以及更多。

但首先，让我们来看看为什么选择主机为您的网站是如此重要。

##### 挽回面子和金钱！

你选择的网站主机如何管理你网站的服务器将会影响到你作为网站所有者的形象。如果网站经常因为不可靠的服务器、糟糕的服务器配置或者服务器无法处理网站接收的流量而不可用，这会给你的用户留下你不知道自己在做什么的印象。对于业余爱好网站来说，这已经够糟糕了，但是如果你经营一家虚拟商店，或者有潜在雇主试图查看你的在线作品集，这就更值得关注了。

我们都愿意相信，一旦我们的网站建立并运行起来，任何人都可以随时查看它。但众所周知，事实并非如此。总会有这样的时候，你试图访问的网站由于某种原因无法访问。不幸的是，这就是互联网的不可预测性。

如果你的网站是一个商业网站，它可能会失去接受订单或享受品牌曝光的机会，而个人和爱好网站也可能会错过收入机会。如果访问者发现网站不可用或使用缓慢，即使是非营利网站也会名声不佳。

虽然潜在的收入损失是想要选择一个好的主机的最明显的原因，但你的决定通常会有其他影响，而不仅仅是运行时间。一台主机的问题经常会影响到与您的域相关的其他服务。电子邮件和 FTP 等服务也可能受到影响，所以你可能会发现，当你的网站关闭时，你也无法检查你的电子邮件。更糟糕的是，电子邮件可能会被退回给发件人，让你的访问者得出该网站已经不存在的结论…

##### 服务水平保证

虽然一些停机时间无疑是我们无法控制的，但是如果我们选择最适合站点需求的主机，我们可以降低风险。任何称职的主机都应该在任何计划停机发生之前通过电子邮件通知你，这将允许你在需要时制定替代计划。当然，没有办法预测任何计划外的停机时间，但许多主机将提供正常运行时间保证，以某种方式补偿您任何超出计划的停机时间。对自己的服务水平更有信心的东道主通常会提供比能力较弱的对手更好的协议条款。

认为你受到这些正常运行时间保证的保护是一回事，但当你认为你实际上有索赔权时又是另一回事了。这可能是一个想法，以保持打印和电子副本的服务条款和任何其他重要文件的一天，你购买一个新的托管计划，以防万一。

通常问题的发生仅仅是因为你不能 100%的时间自己监控网站。然而，有一些产品和服务可以进行各种测试来检查你的站点是否是活动的。但是，即使你能证明你的网站对你(或你的监控服务)不可用的时间超过了约定的可接受水平，你也应该检查一下附属细则:如果问题不是你的主机的错，他们可能不需要付钱！

##### 一分钱一分货！

需要记住的一点是，每个站点都有一个主机。我的完美主机可能是你的完美主机噩梦！非常适合您的一个站点的服务器可能会降低另一个站点的速度。而且没有合适的宿主，所以不要试图去访问和评估它们。

记住，为你的网站找到一个你能负担得起的最合适的主机是符合你的最大利益的，否则你很容易被认为是不可靠的，没有反应的和不关心的。

##### 建立清单

在你开始为你的网站寻找主机之前，你真的需要建立一个清单——一个你可以用来对任何你感兴趣的主机进行评级和评分的方法。当然，在你评估任何服务之前，你需要很好地了解你希望你未来的主机提供什么样的功能。这里有一个你正在寻找的每个特性的清单，以及一系列你想问的问题，非常方便。

从长远来看，在你寻找新的主机之前准备一份清单可以节省你的时间和金钱。它迫使你对网站的各种功能进行优先排序，当你预算有限，需要在功能和成本之间做出妥协时，这很方便。

本指南旨在帮助您准备一份功能清单，这些功能是根据您的特定网站的需求或预期的未来潜力量身定制的，无论您是否知道您想要的托管功能类型。

一旦你编制了清单，你就能更快地评估某个主机是否适合你的网站。由于许多主机争取网站所有者的关注(和辛苦赚来的钱！)，你可能会发现自己在研究过程中经常参考清单。但是这是双向的——考虑到大量的主机，用你遇到的第一个符合你标准的服务来托管你的站点是愚蠢的。要有眼光。

##### 给主机评级

一旦您整理好功能清单，就可以使用它来帮助您对每台主机进行评级。具体如何做将取决于每个特性对你的重要性。如果你已经将清单上的特征按重要性排序，那么你可以简单地给它们打分，1 分表示最不重要，2 分表示第二不重要，以此类推。然后，当你评估每台主机时，根据它们提供的清单上的每项功能给它们一分，并将它们相加得出最终得分。

以下是我个人认为主机应该具备的一些特性。

*   对 PHP 的支持(最不重要)
*   用户控制面板
*   站点统计
*   支持 ASP 和 Perl
*   价格低于每月 10 英镑(最重要)

当我拜访潜在的主机时，我会将可用的功能与我的清单进行比较。上面显示的示例评分系统首先支持最便宜的主机(成本对我来说很重要，因为我的是一个爱好网站)。如果我正在为一个商业网站寻求主机服务，那么成本显然不会那么重要，因为在这种情况下，服务本身的质量才是关键因素。但是不管你用什么标准来给每台主机打分，提供你想要的大部分功能的主机会得到更高的分数。

滑动量表可能是将你的清单转换成分数或评级的最简单的方法，但它绝不是唯一的方法。如果你有两个或更多同等重要的特征，那么你可以给它们相同的分数。无论您如何使用清单，它一定会使主机评估过程更容易。

记住，你得到你所付出的。

这是一个可悲但真实的生活事实，我们中的许多人有很少的托管预算。虽然有主机可以提供一个相当诱人的零花钱价格，不要指望这些主机提供类似的担保，但更昂贵的软件包。如果你的网站对你的生意或生活很重要，那就帮你自己一个忙，把你的托管看作是一项投资而不是一项支出。额外的现金有助于支付基础设施、备份和安全等费用，因此，如果出现任何问题，你将处于更有利的位置来做些什么。

清单是一个很好的方法来找出哪些主机最有可能提供你想要的那种主机，但是一旦你有了潜在主机的短名单，没有什么可以代替在新闻组和留言板上做额外的研究。最了解主人表现的人是顾客自己，不幸的是，他们有时很难找到。话虽如此，确定你应该避开的主机应该不会太难，因为大多数不开心的客户喜欢与志同道合的人分享他们的经历。

##### 1.共享还是专用托管？

有三种主要类型的服务器计划可以从托管公司获得，或者四种，如果你包括一些提供的特殊经销商包。您选择哪种服务器计划将取决于您当前和未来的站点需求，因此了解可用的计划类型以及每种计划的优缺点是一个好主意。

***共享或虚拟服务器***

这类计划会在与其他站点共享的服务器上为您分配有限的空间。这种主机的配额会因主机而异，但是典型的磁盘空间配额大约是 50Mb，这对一个站点来说足够了。这种虚拟主机通常是最便宜的选择(也许除了一些互联网服务提供商提供的免费网络空间)，所以你真的不应该指望在共享服务器上托管你的网站。

**受限语言**

在共享服务器上托管的一个主要问题是，可供您使用的功能和技术将会受到限制。由于安全和稳定性问题，ASP、PHP 和 Perl 并不总是在共享或虚拟计划中可用，即使提供了它们，您也可能没有您的站点正常运行所需的所有模块和组件。共享服务器上的 JSP 支持仍然很少，所以如果你使用它，你可能需要一个专用的服务器。

**其他站点的问题**

不要忘记，你的网站并不是服务器上的唯一一个，所以与你的网站托管在同一服务器上的其他网站的问题可能会导致你的速度问题，甚至使你的网站完全瘫痪。显然，服务器上的站点越多，主机对运行定制脚本越放松，你就越有可能遇到问题。

最好提前了解有多少其他站点可以共享一台服务器，以及如果超出磁盘空间或数据传输限制会受到的惩罚。很容易超过你的数据传输限制:你所需要做的就是让你的网站变得受欢迎(要么是有兴趣的用户，要么是一个不守规矩的网络机器人),你可能会发现自己要么是一个巨额账单，要么是一个兼职工作的网站。

如果是一个静态页面的低流量站点，在共享服务器上托管你的站点应该不是太大的问题，但是如果你预计你的站点会变得相当繁忙或复杂，你真的应该考虑某种类型的专用服务器。

***专用服务器***

设置专用服务器不像在共享服务器上租用空间那么简单，但是它可以为您的站点创造奇迹。首先，你可以自由使用服务器上的所有资源(因为服务器上不应该有你不知道的任何其他站点)。使用一台计算机的所有处理能力可以使一个站点有很大的不同，特别是当它使用数据库和其他服务器端技术时。

拥有一台专用服务器也给了你自由(在合理的范围内)去做你想做的事情。如果您的站点需要 JSP 支持，您可以自己安装(如果您选择带远程管理的服务器)，或者让您的主机为您安装。如果您构建自己的组件和模块，或者使用没有得到广泛支持的技术，这正是您需要的自由。

拥有专用服务器的另一个好处是，如果你愿意，你可以在上面存放多个站点。这可以帮助你保持低成本，但是你不应该忘记服务器上的站点越多，出错的地方就越多。

拥有自己的专用服务器可能会有很多工作要做，特别是如果你必须为所有托管在服务器上的域设置 DNS 条目、FTP 帐户和邮件设置。有些主机会为你做所有的辛苦工作，毫无疑问这也会体现在价格上。请注意隐性费用，如备份、重启机器和其他与专用主机相关的任务，这些可能是您最初没有想到的。

一个专用的服务器并不适合所有人，对于一个简单的静态网站来说，这可能是多余的，因为它只能吸引少量的访问者。如果你的网站的大部分内容使用服务器端技术和数据库，那么它肯定会从你从它自己的服务器获得的额外能力中受益。

***协同定位服务器***

这就像有一个专用的服务器。不同的是，你自己购买硬件和设置服务，然后租用数据中心的空间和互联网连接。

这个选项真的不适合初学者。至少有了专用服务器，主机会给你设置好服务器。有了同处一地，责任就在你身上。的确，他的选择在安装软件方面给了你最大的自由，但也是最复杂的。如果你不能去数据中心自己解决问题，你可以期待主机为你做的高额费用；备份和重启不太可能是免费或收费的。

将您的计算机放在同一个地方通常比简单地从您的主机上租用专用服务器要昂贵得多，而且也更耗时。这种托管选择最好留给更有技术头脑的人——对于大多数网站来说，这通常是不必要的。

***经销商托管***

这本身并不是主机的主要类型之一，但是它变得越来越流行了。本质上，它就像一个联盟计划:你以这样或那样的方式获得主机折扣，作为鼓励更多人使用该主机服务器的奖励。

您能够转售的客户通常位于虚拟(共享)服务器上。折扣的应用方式将取决于东道主；有些会向你收取经销商账户的预付费用，让你按照自己喜欢的方式出售，而其他人会为他们提供的每个标准计划提供折扣价。

你应该知道，现在有许多主机转售网络空间。虽然这可能不是问题，但您应该理解服务器的维护通常超出经销商的控制范围。最好检查一下你租赁空间的公司或个人是否是经销商，这样你就知道如果出了什么问题谁应该负责(你甚至可能希望直接使用他们转售的主机，只是为了获得更多的功能或空间)。

经销商账户也可以为那些拥有不止一个网站，但又买不起专用服务器的人提供一个方便的选择。然而，由于这些经销商帐户的性质，你应该记住，使用这些服务的更高级和更繁忙的网站不会像在更昂贵的主机选项上那样运行得那么快。

##### 2.服务器软件

操作系统和服务器的组合如此之多，以至于很难知道选择哪一种。两种最常见的服务器配置是运行 Internet Information Server (IIS)的 Windows 或运行 Apache Server 的*nix(基于 Unix)。如果您对服务器平台以及可以在这些平台上运行的服务器软件有更多的了解，您很快就会发现，并非所有的服务器都生来平等。

***Windows NT 和 Windows 2000***

自从 Windows 2000 取代 NT 以来，Windows NT 服务器变得越来越难找到，但是由于 2000 本质上是一个升级，这并不是什么大损失。大多数 Windows 服务器通常运行 IIS 4 (WinNT)或 IIS 5 (Win2000 ),但当然也有可能在 Windows 操作系统上运行其他服务器，如 Apache，即使它们的工作方式与在其他平台上不同。

使用 Windows 作为您的操作系统的一个主要问题是 sendmail，它对于一些 Perl 脚本的运行是必不可少的(尽管更好的脚本已经提供了解决这个问题的方法)。文件权限是可用的，尽管与一些人可能已经习惯的*nix 权限相比，它们是简单的(一个权限设置包括所有可能的用户)。

由于对文件权限的态度更加宽松，以及 Windows 通过扩展名关联文件类型的方式，您可能会发现在 Windows 计算机上安装脚本要容易得多，因为不需要忘记权限设置或路径。Windows 系统也提供密码保护，但是唯一的缺点是您需要访问实际的服务器来实现密码(除非您运行的是 Apache)。将敏感数据隐藏在主 Web 目录下总是一个好主意，而不是依靠密码保护目录免受窥探。

***【Unix(及 Unix 变种)***

与 Windows 不同，Unix 及其许多变体(Linux、FreeBSD 和 Sun)为用户提供了对服务器配置及其使用的软件的更多控制。通过使用 telnet 或 SSH 连接来连接到服务器，用户可以安排脚本自动运行(cron 作业)，编辑文件和目录属性和权限，甚至在脚本运行于远程服务器时进行调试！当然，为了能够有效地使用您的 telnet 访问，您至少应该了解 Unix 的基础知识；不幸的是，它比 Windows 要复杂一些，因为 Unix 操作系统倾向于使用基于文本的命令，而不是图形用户界面(GUI)。

因为基于 Unix 的系统使用的系统资源不如运行 Windows 的系统多，所以它往往是一个更稳定、更可靠的选择。给定相同的硬件规格，Unix 系统将更有效率，因为它不需要 Windows 系统仅仅用来更新和解释 GUI 的资源。如果您想要更多的服务器能力，Unix 是一个不错的选择:它不仅拥有更高效的资源管理，而且是开源的，也是最便宜的选择！

Unix 及其衍生产品通常是许多开发脚本的 Web 程序员选择的平台，其中许多脚本利用了内置的 Unix 程序 Sendmail。由于 Unix 文化的开源特性，有大量的免费脚本可供使用，其中许多是为在基于 Unix 的服务器上使用而优化的。

***互联网信息服务器(IIS)***

 **IIS 完全支持 ASP，而其他服务器在没有第三方插件的情况下不支持它。辣椒！软 ASP 对于那些想在*nix 服务器上使用 ASP 的人来说很受欢迎，但是微软版本的 ASP 和 Chili 之间有明显的区别！Soft 的版本，所以最好对 ASP 页面使用 Windows 服务器，以确保它们按预期工作。Perl 和 PHP 可以在 Windows 机器上愉快地工作，尽管一些更高级的功能(不经常使用)可能很奇怪或者完全不存在。

大多数好的 Windows 主机都会通过某种控制面板提供主机配置。更高级的控制面板将允许您编辑与您托管的域相关的设置，但大多数应该允许您更改文件的权限。更好的控制面板将让你定制你的各种错误页面，并为你的网站提供密码保护。

***阿帕奇***

Apache 是服务器软件的流行选择，因为它具有高度可配置的特性。因为它是一个开源服务器，所以发展很快，而且总是有很多志愿者来帮助查找和修复漏洞。

Apache 服务器最好的一点是，您可以使用。htaccess 文件。这包含了有关文件夹和其中的文件在被请求时将如何表现的信息，并允许您拥有许多自定义错误页面、密码保护，甚至是保护您的图像不在未经授权的网站上显示的方法！

Apache 的配置一开始可能会很混乱，但它值得追求，因为它非常灵活；花足够的时间在它身上，你就能让它做你想让它做的任何事情。如果您不想太熟悉 Apache 服务器的内部工作方式，那么您可以使用一个更加用户友好的控制面板来完成配置。这将使任务更容易理解，甚至可以让您有权更改服务器的某些部分的行为，否则您无权更改这些行为。

那么我应该选择哪种组合呢？

虽然在 Windows 服务器上安装 Apache 是可行的，但两种最常见的操作系统/服务器组合是*nix 上的 Apache 和 Windows 上的 IIS。IIS 目前只能在 Windows 系统上运行，所以在*nix 上运行 IIS 是不可行的。

*   如果您没有任何特定的服务器或脚本需求，那么您选择哪种组合并不重要；尽管从性能和正常运行时间的角度来看，选择*nix 服务器比选择 Windows 更物有所值。

*   如果您需要使用一种特定的技术，并且您知道它会受到服务器操作系统的限制，那么您应该已经清楚服务器软件的选择了。
*   ASP 开发人员最好使用运行 IIS 的 Windows 服务器，但是如果包含第三方插件，也可以使用 Unix 服务器。
*   PHP 和 Perl 开发人员可以使用任何支持他们技术的环境，但是值得注意的是 Apache 提供了提高这些语言效率的模块；所以也许它们更适合在运行 Apache 的计算机上使用。
*   如果任何 PHP 或 Perl 脚本需要 sendmail 程序，操作系统必须是*nix。

如果您刚刚起步，并且想知道哪个服务器是开发您的站点的正确选择，请根据您认为将来会用到的功能进行选择。尽早做出正确的选择会为你省去很多麻烦；将服务器端技术从一个平台切换到另一个平台并不像简单地上传文件并希望一切顺利那么简单！
