# 大约 2004 年 4 月的快照

> 原文：<https://www.sitepoint.com/flash-snapshot-circa-april-2004/>

嗯，自从 Flash 的最新版本及其增量升级和 Flash Player 的几个版本发布以来，已经有几个月了。我们现在在哪里？…

有无数的增强，增加和…咳咳！最新主要版本的组件 API 弃用，更不用说许多优秀的扩展和命令，使从 [XML 和 Web 服务中检索数据变得更加容易。](http://www.macromedia.com/cfusion/exchange/index.cfm#loc=en_us&view=sn106&viewName=Exchange%20Search%20Details&authorid=60639501&page=0&scrollPos=0&subcatid=0&snid=sn106&itemnumber=0&extid=1011923&catid=0)

总的来说，这是一个不错的发布，但是对于我们所熟悉和喜爱的应用程序的开发，我们已经走到了路的尽头了吗？Macromedia 是否已无计可施，只是计划对下一个版本进行调整和微小的增强，或者我们是否都将面临一波新功能的冲击。

我知道有些人似乎认为，随着这些年来的发展，Flash 已经有点迷失了方向；SWFs 试图通过一些组件模仿网页(尽管有一点点天赋),并且已经失去了与草根动画祖先的联系，但是我在这里感兴趣的是你对此事的想法。

你认为 Flash 需要重新审视它的根源，还是它正朝着正确的方向前进？你认为 Flash 播放器可以更重一点，包括更快的渲染或包括更多的功能吗？(标准正则表达式支持会很好)毕竟，朋友之间的几个 KB 算什么？

这是我的博客，我可以问我是否愿意:-)

我很想听听大家的意见
