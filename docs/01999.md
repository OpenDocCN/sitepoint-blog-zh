# Aurelia 的未来—路线图和即将推出的功能

SitePoint 的开源周到了！整整一周我们都在发布关于开源、自由软件和社区的文章，所以请继续查看 ***[OSW 标签](https://www.sitepoint.com/blog/)*** 获取最新更新。

本文由客座作者罗布·艾森伯格撰写。SitePoint 客座博文旨在为您带来 JavaScript 社区杰出作家和演讲者的精彩内容。

开源是我们与 Aurelia 合作的基础部分。这种模式提供的自由，以及随之而来的社区和协作，对我们的愿景至关重要。我们希望看到开放网络和开放平台继续发展。我们希望看到技术以一种能使全球社会繁荣的方式民主化。我们希望看到越来越多的人从消费心态转向创造和给予心态，我们认为这是开源精神的核心，对我们的行业和文明的未来至关重要。奥雷利亚是对这个梦想的一个小小的贡献，一个迄今为止令人兴奋的旅程和对未来充满希望的人。

就在一年前，我们推出了 Aurelia 的正式测试版。从那时起，我们成为第一个支持 v1 影子 DOM 的框架。我们还调整了我们的性能，以便我们能够在[主要第三方基准测试](http://www.stefankrause.net/wp/?p=316)中[击败 Angular 2 和 React](http://jdanyow.github.io/aurelia-dbmonster/) 。我们的社区以惊人的速度增长，我们的贡献者数量超过了 Angular 2 和 Vue。所有这些为我们今年 7 月发布的 1.0 版本奠定了基础，该版本继续在全球推广。这真是令人惊奇的一年！

但是我们不认为 1.0 是终点。当我们达到这个里程碑时，我们立即开始朝着我们的下一组目标努力，这是建立在稳定的 1.0 基础上的。明年，我们将在四个主要领域为 Aurelia 投入巨资:持续改进、工具、服务器端渲染和 Aurelia UX。

## 持续改进

奥雷利亚本身总有工作要做。随着我们与社区的合作以及我们自己开发更多的应用程序，我们将继续消除错误，启用新的用例，并提高我们所有模块的可用性。我们还将继续监控 web 标准，寻找利用新浏览器功能的方法，或者让我们的社区更简单地利用它们。一个有趣的例子是[这个原型绑定引擎增强](https://gist.run/?id=5f21e717c107011366221fe8cec50959),它利用一些新的 API 在 poplular DB Monster 性能基准测试中将 Aurelia 的性能提高了近一倍。(该 API 已经为 Chrome 和 Opera 提供，正在为 Edge 和 Firefox 开发)。

## 工具

在框架本身之上，我们正在努力构建一个稳固的生态系统。其中很大一部分是工具。这里有四个我们正在改进的工具，希望在接下来的几个月里能看到大的改进。

### 网络包

我们已经完成了在 Webpack 2 中使用 Aurelia 的初步工作。然而，我们还没完。该团队将继续投资于 Webpack，包括即将到来的对我们 Webpack 支持的重大更新。

### 硬币指示器 （coin-levelindicator 的缩写）命令行界面（Command Line Interface for batch scripting）

除了 Webpack，我们还在努力通过我们的 [Aurelia CLI](http://aurelia.io/hub.html#/doc/article/aurelia/framework/latest/the-aurelia-cli) 来简化开发。CLI 因其简化应用程序开发并让人们开始构建应用程序的能力而广受好评。我们已经为它计划了更多的特性，包括 Webpack 的集成、依赖安装的简化和对其他主机构建的支持，比如 Cordova 和 Electron。

### VS 代码插件

我们最近发布了 VS 代码的插件[，为 HTML 中的绑定命令增加了改进的语法高亮和基本智能感知。](http://blog.aurelia.io/2016/10/11/introducing-the-aurelia-vs-code-plugin/)

![The Aurelia VS Code Plugin](../Images/ba893806088baf6672e17c8bef84e884.png)

它还为 HTML 视图添加了新的颜色主题。我们已经为这个插件计划了很多，包括 HTML 中的视图模型智能感知和开发时模板验证。这让我想到…

### 模板验证

你的 transpiler，无论是 TypeScript 的 Babel，通常都能为你提供一大堆帮助，确保你的代码是正确的。然而，它不能为你的 HTML 提供任何智能。我们有一些新的工具，提供模板验证，以帮助确保您使用有效的语法和坚实的 HTML 实践。它目前在一个独立的库中[可用，作为您的 CI 或测试的一部分。我们正在努力继续改进验证，并将其集成到我们的标准项目设置和 VS 代码插件中。](https://github.com/MeirionHughes/aurelia-template-lint)

## 服务器端渲染

虽然大多数开发人员不需要为他们的 Aurelia 应用程序进行服务器端渲染，但它仍然是一个重要的功能，可以为我们的社区带来新的场景。作为测试版的一部分，我们开始将基础抽象放在适当的位置。现在，我们正朝着全面实施的方向努力。作为上面提到的模板验证工作的一部分，我们已经让 Aurelia 在 Node 下运行。我们目前正在努力将 Aurelia 与一个基于 Express 的应用程序集成在一起，以在服务器上呈现应用程序，将完整的 HTML 发送回客户端，无需客户端 JavaScript。这将支持在 SEO 关键应用中使用 Aurelia，并在与缓存策略智能结合时提高加载时性能。在这种服务器渲染的基本形式建立并运行之后，我们将扩展 Aurelia 框架的“拾取并继续”功能，以便从那时起服务器渲染的 UI 可以在浏览器中完全保真地运行。

## 奥雷利亚·UX

最近，[我们宣布我们已经开始致力于 Aurelia](http://blog.aurelia.io/2016/11/04/introducing-aurelia-ux/) 的开源姐妹框架。它叫奥雷利亚·UX。Aurelia 是一个前端框架，针对应用程序的软件设计，而 Aurelia UX 是一个用户体验框架，主要处理应用程序的 UI/UX 设计。那么，奥雷利亚·UX 会包含哪些类型的东西呢？

### 主机、平台和设计语言

主机是浏览器、电子和科尔多瓦等环境。这些主机使你能够在各种平台上运行，如 Window、Mac、Linux、Android 或 iOS。每个平台都有一个默认的设计语言，比如材料设计。我们正在将所有这些概念构建到奥雷利亚 UX 中，并作为一流的编程结构来支持它们。

### 样式、调色板和主题

当构建应用程序时，您希望您的样式被封装。能够使用支持主题化的调色板来构建它们也很不错。奥雷利亚·UX 的新样式系统支持在 CSS 中使用数据绑定，包括扩展 MVVM 来处理样式，为动态样式和布局带来了全新的力量。

### 组件和数据可视化

奥雷利亚·UX 将为现代应用程序中最常见的用户界面场景提供一套核心组件。这意味着按钮、输入控件、滑块、列表、卡片等等。随着时间的推移，我们将扩展这个库，并最终希望包括数据可视化以及复杂的控件，如数据网格。

![Aurelia UX](../Images/218c3d349eb89ec31e40a41821815563.png)

### 互动、运动和流动

建立一个吸引人的用户体验意味着你不仅要关注风格和组件，还要关注用户交互隐喻、信息流和应用程序内的移动。所有这些因素结合在一起，创造了一个伟大的体验。为了完成这幅画面，奥瑞利亚 UX 将在其组件的基础上进行构建，并添加这些更高层次的功能。

## 最后

与我们的社区一起建设 Aurelia 是一次奇妙的旅程。如你所见，这只是开始。我们比以往任何时候都更加努力地开发一些很酷的新技术。未来是光明的，我们迫不及待地想知道你会用它做什么。

我们希望听到您的反馈。你愿意和我们一起走这条路吗？
