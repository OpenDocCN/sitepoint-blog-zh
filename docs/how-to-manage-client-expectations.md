# 如何管理客户期望

> 原文：<https://www.sitepoint.com/how-to-manage-client-expectations/>

在进入自由职业者的世界之前，在项目之前、之中和之后管理客户的期望是我们很少有人欣赏的隐藏任务之一。有些伟大的客户会提供清晰的简报，并欣赏相关的工作。也有一些客户不知道他们想要什么，犹豫不决，低估了努力，低估了成本。

每当您听到以下陈述时，警钟就应该响起:

> 我只是需要一个像易贝/推特/脸书这样的网站。你来整理细节——我来检查一下，做完就付钱。
> 
> 我有一个好主意——它能赚几百万。你应该付钱给我来实现它！
> 
> 只需要几天，对吗？这对你的生意有好处。

[还有很多……](https://www.sitepoint.com/12-signs-of-problem-clients/)

几个简单的步骤可以帮助你管理客户的期望，并使你的努力最小化。

## 确定决策者

最好是在客户的组织内有一个单一的联系点。那个人将负责整理反馈并做出最终决定。询问谁是项目负责人，避免微观管理他们的员工！

## 评估你客户的能力水平

必要时，用通俗易懂的语言解释所有技术问题。避免开放式问题，例如“Widget X 应该如何工作？”你是专家，所以要授权给客户，提供简单的选择，并有明确的客户利益，例如

1.  如果 Widget X 可以做 A 和 B，用户将需要 2 分钟来完成目标 z，需要 3 天来实现，成本为 500 美元。然而，这个小部件需要每 12 个月维护一天，费用为 200 美元。
2.  如果 Widget X 可以做 A、B、C，用户需要 20 秒完成目标 z，实现需要 7 天，花费 1200 美元。不应产生额外的维护成本。

## 避免不必要的会议

不必要的会议会占用你一天的工作时间。你应该有一个固定的时间表和严格的议程，没有“任何其他业务”项目。如果可能的话，把会议安排在下午晚些时候——这有助于集中注意力，避免不必要的聊天。会议应该产生一系列明确的行动要点，如果没有其他人承担责任，你应该记录并分发这些要点。

如果你担心客户组织了太多无聊的会议，说明你将对所有额外的咨询工作按小时收费。

## 提供固定价格

如果你有一个很好的简介和规格，应该有可能以固定的成本完成该项目。如果有太多的未知数，按小时收费可能会更好，但要警惕这些情况——模糊的项目可能会有模糊的预算！

## 用合同！

如果你不做别的，永远，永远，永远提供一份合同。它可以很简短，但应包括:

*   所有交付物的简明概述。
*   一份时间表，列出您和客户的依赖关系。它应该包括关键决策，内容要求和付款日期。
*   免责条款:如果你或客户希望在某个特定阶段放弃项目，会发生什么。
*   所有费用的细目。

## 要求押金

总是向新客户或任何需要大量工作的人要求押金。总成本的 50%通常是可以接受的，尽管较大的项目可能有明确规定的付款时间表。

向客户解释未付款可能会影响他们的释放日期。

## 有效沟通

电子邮件很有用，但打电话通常更好。混淆的机会更少，会更快，而且会帮助你与客户建立关系。然而，注意不要用多个电话轰炸他们，也不要对他们的决定做假设。

## 负责

客户不太可能关心你的电脑崩溃、硬盘故障或儿童保育问题，但他们会关心时间表的延误。诚实，解释情况，风险，以及你正在做什么来解决问题。

网站开发者不应该提供额外的特性或功能，这会增加额外的费用。如果你以 300 美元的服务费离开了你的车，如果修车厂还以另外 200 美元升级了你的立体声系统，你会有点恼火，因为这是“一件很酷的事情”！要么与客户讨论增强功能，要么免费超额交付商定的功能。

## 知道什么时候该走

要明白，客户犹豫不决和讨价还价会占用你更多的时间。如果客户需要帮助定义他们的需求，你可以提供收费服务。
