# 管理通信过载的技巧

> 原文：<https://www.sitepoint.com/tips-for-managing-communication-overload/>

良好的沟通是与客户建立关系的最重要的部分之一，但是当客户需要的接触超过你愿意给予的接触时，该怎么办呢？每天接二连三的电子邮件、电话和会议请求会扼杀你的生产力，让你变得神志不清。但是不要马上结束这段关系；在放弃之前，尝试一下这些想法。

**使用电子邮件规则**

当我开始与新客户合作时，我很快就知道他们是否严重依赖电子邮件。我非常喜欢电子邮件，所以从与客户保持沟通的角度来说，这很好。但是如果客户发了很多信息，我的收件箱不可避免地会填满，我觉得我需要马上阅读所有的内容，我开始感到压力。更不用说在跟踪请求、信息和其他对项目至关重要的细节方面，这变成了一场噩梦。

我处理这个问题的一个方法是为倾向于发送大量邮件的客户设置电子邮件规则(对一些客户设置多个规则)。我通常会将来自客户端所有信息都放在一个文件夹中，在那里我可以看到未读邮件的数量，但各种邮件并不在我的收件箱中，它们会尖叫着让我打开它们。

**设置电子邮件的时间**

我的所有客户端邮件在到达时都会自动分类到相应的文件夹中。但我仍然需要阅读它们，记录信息，并采取行动或安排行动时间。根据客户、项目和时间表的不同，我每天会试着在 2-3 次会议中处理所有收到的信息，尤其是当一个问题上有很多来自一个客户的信息时。这对于生产效率来说是一个奇迹，因为它消除了阅读电子邮件时的走走停停，然后被一个可以等到以后再处理的问题拖住。

不过，我承认，我确实会先看一眼收到的所有东西，然后再把它们放到我脑子里的“以后”文件夹里。多亏了 Growl，我可以得到一个非侵入性的弹出窗口，在那里我可以得到消息的要点，并确保它不是我需要立即处理的紧急情况。

没有即时消息！

我不喜欢即时通讯。我让自己认为这可能是一个促进与客户沟通的好方法，然后在几个小时内，我就关闭了它。要么是客户滥用它——不停地发即时消息，即使我的状态显示我没空。或者他们滥用它——给我发很长很详细的信息，我需要立即采取行动或者复制到我的任务列表中。另外，我在非传统时间工作，有时会忘记登出。我真的不想在凌晨聊天，尤其是和那些想要项目状态更新的客户。

**要求提前调度**

与客户保持实时联系至关重要，无论是面对面还是通过电话。对我来说，这通常意味着电话会议，我试着和客户定期安排。我很少接临时电话，因为我要求(是的，要求)所有会议都要提前安排。即使只是提前一天，安排电话也能防止其他客户的工作被打断，让我有机会为会议做好准备，并设定开始时间和持续时间，这样就能有效利用时间。

另一种管理喜欢经常通过电话沟通的客户的方法是，每天或每周几次建立一个固定的时间段作为你的电话会议时间。你的客户可以在这段时间给你打电话，希望你有空。

**收取额外费用**

如果所有这些都失败了，你可以考虑通过提供电话/电子邮件支持计划作为你的附加服务，对过多的联系请求收取额外费用。正如另一个帖子的评论所说，这正是@realjustinlong 用来管理电话中断的方法。

你可以把这些建议都写进你的合同里，并在需要的时候向客户提及。最主要的一点是，设定你的界限，并坚持这些界限。然后，如果客户仍然不尊重你的时间，可能是时候离开了。
