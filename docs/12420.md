# Web 应用程序重置了游戏规则

> 原文：<https://www.sitepoint.com/web-applications-reset-the-playing-field/>

我通常不习惯发布未经证实的谣言，但我最近读到一些关于苹果本周可能发布基于云的软件即服务(SaaS)版本的 iWork T2 和 iMovie T4，这让我想到了 SaaS。也许这是一个显而易见的观点，但我只是刚刚才真正明白，所以我想我应该分享一下:SaaS 重新设定了游戏规则。

随着软件应用程序转移到云端，曾经占据主导地位的领导者将受到新的暴发户的挑战。微软 Word 将面临来自 Google Docs 和 Zoho Writer 的真正挑战。Quicken 和 Money 可能会受到 Mint 或 Cake Financial 等公司的挑战。Photoshop 将面临 Aviary、Pixlr、Splashup 和 Sumo Paint 等在线应用的竞争。iMovie 可能会发现自己正在追赶像 JayCut 这样的基于网络的应用程序。PowerPoint 将从 SlideRocket 和 280 张幻灯片中获得收益。

苹果是否真的在本周的 MacWorld 上把 iWork 或 iMovie 推进云端并不重要(尽管如果他们这样做了会很酷)，关键是，由于软件即服务越来越受欢迎，苹果和其他公司有一个惊人的机会，在他们以前从未竞争过的领域竞争。

例如，目前 iWork 是一个没有竞争力的桌面办公应用套件。尽管在某些情况下，iWork 在功能上与微软 Office 不相上下，但它仍然只是一款小众的 Mac 产品，因为已经有太多的人依赖 Office，兼容性问题通常会迫使人们为它买单。但是一个在线的、基于云的、跨平台的 iWork 版本会有所不同。没有公认的领先的文字处理、电子表格或演示的网络应用程序，所以任何应用程序开发者都有机会与当前桌面世界的重量级人物竞争。

SaaS 重置了比赛场地，并从讨论中抹去了既定的领导者——一旦软件完全迁移到云，在我们生活中如此根深蒂固的桌面版本就不再重要了——如果是这样，那么新的竞争对手就大有可为。这就是为什么最初由一个人开发的基于网络的应用程序 SlideRocket 最终可能会搞垮 PowerPoint。这就是为什么 37signals，一家只有少数员工的创业公司，可以用 Basecamp 挑战微软项目，也是为什么 Salesforce.com 能够挑战 SAP 和甲骨文。

这篇文章中提到的网络应用通常不会与桌面应用在功能上竞争。如果它们是桌面应用程序，它们可能没有竞争的机会，因为人们会因为功能不全或存在兼容性问题而不予考虑。但是网络应用有一些回旋余地。人们并不期望它们功能丰富，便利性和可访问性弥补了功能的不足。此外，网络应用程序的兼容性问题不太重要，因为你通常可以直接通过应用程序与合作者共享工作。

由于“软件即服务”模式使得一次性向所有客户推送更新变得更加容易，SaaS 应用程序到目前为止通常更加敏捷，能够比桌面应用程序更快地迭代和改进。在便利性、可访问性、协作性和敏捷性方面，SaaS 胜出——所以他们暂时没有考虑这些特性。

正如我所说的，这些可能是显而易见的观点，但它们只是最近才在我的脑海中变得清晰，所以我想我应该分享一下。如果你同意，不同意，或者有什么要补充的，请在下面的评论中回复。
