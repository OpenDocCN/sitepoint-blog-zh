# 信息架构——为什么第一页在最后

> 原文：<https://www.sitepoint.com/information-architecture-why-the-first-page-comes-last/>

主页、封面、登录页、站点根目录。不管你叫它什么，它通常是你的用户感知你的方式的核心，也是你的网站如何运作的核心。客户总是被它迷住，当你开始设计一个新的网站时，从在左上角敲打一个标志开始，并继续在它下面布置你的主页，这似乎是简单而直观的。

简单，是的。也许很直观，但不一定对实现你的设计目标有用

正如 Derek Powazek 在 ALA 上的新 IA 文章中指出的，它是你网站最深层的“原子元素”——文章、新闻、搜索结果和产品页面等内容单元——为你建立主页提供了最好的基础。事实上，我认为这些“深度页面”通常是最容易开始的，因为它们从一开始就有明确的目的和目标。

从理论上讲，当你的内容页面呈现出更加坚实的形式时，你所做的决定会自动反馈到类别和/或主页的工作方式中。在某些方面，他们设计自己。

如果你已经这样做了，这篇短文对你来说不会是火箭科学，但我得到的印象是，像这样简单的 IA 方法没有得到应有的理解。

很高兴看到这种想法展示给广大观众。文章不错，德里克。
