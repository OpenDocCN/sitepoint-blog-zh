# 纽约现场直播？不完全是…

> 原文:[https://www.sitepoint.com/live-from-new-york-not-quite/](https://www.sitepoint.com/live-from-new-york-not-quite/)

过去一周，我在纽约市参加了 Jupiter Media 的“搜索引擎策略”会议。尽管天气下雪，这项活动的出席人数还是非常多(好吧，座无虚席)。这是一场很棒的演出，只有一个小问题…互联网接入很糟糕。

我不介意在我的酒店房间里每天支付 10 美元的宽带连接，但当连接一直下降，通常比拨号线路慢，不能连续 10 分钟依赖时，这就成了一个问题。就连赞助商提供的免费“wi-fi 休息室”也遭遇了类似的问题。

因此，在一周努力下载电子邮件并保持业务运转后，回到德克萨斯州的家中，一切似乎都正常，这是一种解脱。

但是我欠我的读者一些消息，不是吗？以下是会议的一些亮点:

1)雅虎为他们的几个搜索产品宣布了一个网络服务 API([http://developer.yahoo.net/](http://developer.yahoo.net/))，这似乎为支持搜索的应用程序的开发者提供了几个优于谷歌 API 的强大优势。

2)信息检索和搜索引擎专家 E. Garcia 博士(在搜索引擎观察论坛上以 Orion 的身份发布帖子)参加了他的第一次 SES，由于旅行困难，他在第一天几乎就做了演示，并在接下来的 3 天里活跃在会议现场，启发每个人。

3)一场名为“索引峰会”的精彩会议，搜索引擎代表公开讨论了抓取和索引各种类型的网站和内容的挑战。有趣的是对谷歌链接的“rel=nofollow”属性的讨论，该属性旨在防止链接的蜘蛛搜索。搜索引擎支持这一点，但他们不准备承诺在未来不会跟踪这些链接。

4)由于涵盖搜索相关营销问题的新会议，SES 会议似乎正在吸引更广泛的受众。随着行业的成长，SES 也在成长。所有这些都应归功于丹尼·沙利文和帮助主持会议的团队。

我自己的会议(周一介绍搜索词研究，周四介绍高级搜索词研究工具)也非常“受欢迎”，这要归功于时间表的变化，将周四的最后一场会议提前了一点。过去，我们经常在会议的最后一天来到几乎空无一人的房间，但这次没有。

随着我们回到博客的正轨，并继续我们关于关键词策略的对话，本周将会有更多来自 SES 的片段。
