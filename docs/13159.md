# ColdFusion 一周:08 年 1 月 16-22 日:考试时间！

> 原文:[https://www . site point . com/the-week-in-cold fusion-16-2-Jan-08-exam-time/](https://www.sitepoint.com/the-week-in-coldfusion-16-2-jan-08-exam-time/)

那么博客圈这周有什么给辛勤工作的 ColdFusion 开发者的呢？

首先，开源项目一直很忙。关于即将发布的 FarCry CMS 第 5 版“FarCry Fortress”的更多信息已经发布，它现在将有多种部署选项，这意味着它可以更容易地在共享托管帐户上运行，这一直是许多开发者争论的焦点。Daemonite 博客的全部细节: [FarCry:共享主机变得容易](http://blog.daemon.com.au/index.cfm?objectid=9595141D-13D4-B1F1-48FA8150CD1D5789)。

在框架方面，Steve Nelson——fuse box 的原始开发者之一——写了一篇“一年之后”的文章，介绍他的简单的基于 CFC 的 MVC 框架。显然它没有太大的变化，当你考虑到它本来就是超级简单的时候，这就很有意义了！史蒂夫甚至用一段话解释了它是如何工作的(最终版本非常短):[一年后史蒂夫的 MVC 框架](http://www.webapper.net/index.cfm/2008/1/21/Steves-MVC-Framework-a-year-later)。

许多开发人员对 Ext JS Ajax 框架感兴趣，尤其是因为 Ext 1.0 是 CF8 中内置 Ajax 标签背后的核心。[贾斯汀·卡特已经开始为表单](http://www.madfellas.com/blog/index.cfm/2008/1/16/ColdExt--an-Ext-JS-tag-library)开发 Ext JS 标签库，并提供了一些截图。虽然他还没有发布任何代码，但 Justin 正在寻找关于如何继续这个项目的建议，所以如果你感兴趣，请过来看看他到目前为止得到了什么。

Adobe Devnet 已经更新了 Charlie Arehart 关于 CF8 监控的系列文章的第 4 部分:[多服务器监控、管理 API 监控等等](http://www.adobe.com/devnet/coldfusion/articles/monitoring_pt4.html)。该系列的前三部分都链接到了本文。当我们在服务器监控领域时， [FusionReactor 3 已经发布](http://www.fusion-reactor.com/fr/whatsnew.cfm)，有许多有趣的增强功能，包括 Flex 驱动的“企业仪表板”。

最后但同样重要的是，如果你对认证感兴趣，街上的消息是 ColdFusion 8 考试将在本月底发布。许多人对认证过程的有用性有强烈的看法，但如果你想充实你的简历，这肯定不会有什么坏处。我在考虑努力学习，你呢？
