# 结论——提供模型的公司到底发生了什么

> 原文：<https://www.sitepoint.com/conclusion-what-actually-happened-to-company-offering-mock-ups/>

感谢你在上一篇博客中发表的关于使用模型作为营销工具的精彩帖子。这个故事已经到了尾声，这篇文章将告诉你最后发生了什么与 eLance 投标人谁为我提供了一个模拟。

首先，基于上一篇博客中的一些帖子，澄清几点:

1.我向任何提出要求的设计师提供了非常详细的规范，包括完整的副本、建议的导航方案以及我想要传达的外观和感觉的细节(这些也应该在副本中显而易见)。我完全同意你们所有人的想法，不与客户进行详细的讨论，怎么会有人提供一个模型。

2.虽然这是一个非盈利性企业的简单网站(10 页和两个表格，可以通过电子邮件发给我进行手工处理),但在任何网络项目中，我作为消费者都会将更复杂的工作分成几个部分:文案、图形/网页设计、架构、基础设施、编码/开发。我发现没有哪家公司能在所有这些方面做得很好，并且倾向于分析它们——尤其是文案、图形和开发/基础设施/架构方面。因此，作为一个潜在客户，获得实体模型对我来说非常有吸引力，因为我可以隔离项目的这一方面。

3.你们中有几个人写了关于让客户参与对话的内容。这真的很重要。使用模拟 ups 的过程是建立信任、建立信誉和与潜在客户建立关系的许多方法之一。因此，模型不是在盲目投标的情况下最有效，而是在帮助你建立专业知识和职业精神的情况下。

第三点尤其被我的情况中最终发生的事情所证实:

eLance 上的公司提供了一个模型，做得很好，明显地将自己与其他 24 个投标人(现在达到 30 个)区分开来。然而，我仍然不完全满意他们的样机。这看起来有点老套和普通(正如你们中的一些人预测的那样)。

所以我邀请了一位我认识并喜欢的设计师(他是 Sitepoint 的忠实读者)，如果他想投标这个项目的话。我让他知道，我有一个来自另一家公司的模型，我喜欢他们模型的一些东西，但不喜欢其他东西。

他很快提供了自己的模型，并提供了一个出价，其中包括一些 eLance 公司不能提供的东西:免费托管、流量日志工具和无限制的修改。他的价格更高，但我们进行了电话交谈(相对于 eLance 上的私人董事会)来制定范围的细节。

所以我和他一起工作。

(我当初为什么不邀请他投标？因为我总是在寻找新的人来做工作，也因为我曾与他合作过盈利性项目，而不是非营利性项目。但我在对 eLance 投标人总体不满意后拜访了他。)

那些指出关系是关键的人，你们完全正确。

那些表示模型对你来说是一个很好的工具的人，你是完全正确的。

那些认为向客户提供模型是贬低自己和行业的人——这对我来说有点奇怪，但不管怎样。如果它能让你得到满意的客户，你为什么不去做呢？
