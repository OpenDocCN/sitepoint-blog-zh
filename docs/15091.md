# 采访——SitePoint.com 的凯文·杨克

> 原文:[https://www.sitepoint.com/kevin-yank-sitepoint-com/](https://www.sitepoint.com/kevin-yank-sitepoint-com/)

凯文·杨克集作家、编辑、程序员和艺术家于一身。他是一名加拿大的前帕特人，正在为澳大利亚墨尔本缺少 Twizzlers 而奋斗，他对自己在 SitePoint.com 的未来有着宏伟的计划

最近，我和凯文坐在一起，试图更多地了解这个人…

***SP:嘿凯文。感谢您抽出时间接受采访。我猜很多人在很多方面都视你为“神”，这对你来说并不奇怪。你写了畅销书籍“[用 PHP & MySQL](https://www.sitepoint.com/books) 构建你自己的数据库驱动网站”，这在 SitePoint 全球引起了涟漪。稍后我想深入了解这本书的一些细节，但首先，你能告诉我们你是如何参与 SitePoint 的吗？***

事实上，我之所以能出现在 SitePoint 是因为另一个网站管理员社区！1998 年，当我开始我的第一份“真正的”网页设计工作(一个加拿大铝业公司的内部网网站)时，HTML 编辑器的选择归结为香肠软件公司的全新(当时)HotDog Professional 5.0，或 Allaire 的 Homesite 4.0。

那时候，Homesite 相当简陋，HotDog 又亮又新，所以我把钱投在了那里。香肠软件公司有一个很棒的在线论坛系统来支持他们的产品(这个论坛已经发展成为 [一个非常活跃的社区](http://www.sizzlerforums.com/))，我带着一贯的热情加入其中。不到一年，我就成了论坛的版主，并开始写我的第一篇关于网络开发的文章。

当香肠软件公司的几名员工在马特·密茨凯维奇的帮助下离开公司，创办了一个名为 SitePoint 的新网站时，我在他们希望加入这家新公司的人员名单上。

这在商界是一句老生常谈的话，但找工作真的取决于你认识谁。参加像 SitePoint 社区这样的同行论坛(甚至发表一两篇文章)是获得关注的好方法。

SP:你有什么早期的经历可以证明是你被“灌输”到了 SitePoint 文化中吗？T3】

我记得，在我第二次拜访 SitePoint 办公室时，Jason 刚刚在办公室里安装了一个基于 Perl 的 MP3 自动点唱机。新鲜感一消失，点唱机就成了朱尔斯和贾斯之间较量的主题，他们都试图胜过对方，秘密地让服务器只播放自己最喜欢的歌曲！

SP:正如麦加之于穆斯林，耶路撒冷之于犹太人，澳大利亚墨尔本之于网络开发者。你最近去了南方的光辉城市朝圣，我想知道你适应得怎么样了？T3】

在新南威尔士州纽卡斯尔(悉尼附近)住了三年后，文化冲击并没有想象中那么严重。我很高兴地告诉大家，我已经进入了选择支持澳大利亚规则队的过程(我怀疑如果我选择西部牛头犬队，马克可能会给我加薪……)。

住在墨尔本最棒的一点是，每当你错过最后期限时，你可以说，“哦，我以为你指的是美国东部标准时间！”

SitePoint 的工作人员让你参加新人入会仪式了吗？T3】

嘿，我可不算新手！你不要给他们任何想法，现在…

SP:你吃过 Vegemite 或 Tim Tams 吗，如果吃过，能描述一下体验吗？T3】

蒂姆·塔姆是最棒的！我回家时，冰箱里有一盘新的耐嚼焦糖布丁在等我。嗯！但是，尽管它们很好，但它们不能代替[草莓夹心饼干。为什么在澳洲买不到红甘草？有贸易禁运什么的吗？！](http://www.twizzlers.com/)

至于[Vegemite](http://www.vegemite.com.au/)，我已经避开了那种东西。我是说来吧！澳大利亚人真的需要[另一种酵母饮料](http://members.tripod.com/~Greggles/mainpage.htm) ( **警告**:本页讨厌的音乐)？

##### 论技术

SP:我们看到你用 PHP 之外的语言，包括 ASP.NET 和 JSP，为 SitePoint 网络写了一些令人难以置信的多样化的文章。说到编程语言，你觉得自己的优势在哪里？T3】

我在大学最喜欢的科目之一是一门叫做“编程语言和范例”的课程。虽然课程使用 Scheme 和 Java 作为编程语言，但重点是所有编程语言的核心概念。

我的观点是，一般来说，你对编程语言理解得越好，学习新语言就越容易。我想说这是我真正的优势所在。

同样，当你在做项目时，你倾向于使用一种语言吗？如果是，为什么？T3】

对于像我这样的开发人员来说，编程语言就像工具一样。问一个开发者是否倾向于一种语言就像问一个木匠是否倾向于他的一种工具。

我想我目前最喜欢用 Java 工作，但是我尽量只根据语言对任务的适用性来选择任何给定项目的语言。

SP:一些 PHP 开发人员倾向于将 ASP.NET 与 PHP 进行比较，并对。NET 框架。你愿意就此发表意见吗？T3】

PHP 是一种服务器端脚本语言。ASP.NET 是一个面向对象的 Web 应用程序开发框架。就其本质而言，PHP 更适合中小型 Web 脚本任务，而 ASP.NET 更适合企业级 Web 应用程序。

也就是说，如果你了解并熟悉 PHP，并且它能够始终满足你的需求，那么你就不需要担心 ASP.NET。

***SP:你更喜欢。NET 还是 J2EE？为什么？**T3】*

我喜欢 J2EE 的平台独立性，因此它仍然是我企业 Web 开发的首选和推荐。的。NET Framework 对于开发人员来说更容易使用，并且更加适应当今一些常见的 Web 应用程序开发问题。如果在 [Mono 项目](http://www.go-mono.org/ )产生一些有用的东西之前，下一个版本的 J2EE 没有在这方面加大赌注，我的偏好可能会在不久的将来改变。

***SP:说说 Editize 吧。这是什么？你为什么写它？而我这个普通开发者为什么要用呢？**T3】*

哦，我明白了。不要脸的塞时间，是吗？好吧，如果你坚持… [Editize](http://www.editize.com) 最初被称为“章鱼编辑器”，是为了拯救我们当时的新编辑乔治娜，使其免于患腕管综合症。事实上，我只是为把我的文章以 Word 格式发送给她而感到内疚，因为我的文章中有大量的内嵌代码片段，她必须手动检查和标记。

第一个版本在办公室里大受欢迎，我们的设计师立即看到了它给我们的一些网页设计客户的网站带来的价值。“如果我们有这样的工具提供给我们所有的客户，”他们推理道，“我们在与其他设计公司竞标时就有竞争优势了！”所以我开始开发一个更通用的版本，产生 HTML 代码，而不是我们自己专有的标记语言；这个产品被称为 RapidEdit。

从我们第一个客户的反应来看，我们知道我们赢了。将它发布给开发人员社区是下一个合乎逻辑的步骤。现在你可以获得与其他网站设计公司相同的竞争优势，就像我们的设计师一样。只是不要在一个项目上和我们竞争，好吗？

##### 这本书

***SP:现在来说说“书”。“[使用 PHP & MySQL](https://www.sitepoint.com/books) 构建自己的数据库驱动网站”是一本非常受欢迎的知识书籍，它成功地启动了许多 PHP 开发人员的职业生涯。对此，我的第一个问题是…你意识到你惹了多大的麻烦了吗？***

为什么，你到底是什么意思…？实际上，在办公室里，我们喜欢开玩笑说这本书给了读者足够上吊的绳子！在本书中，我尽可能地坚持良好的编程实践，但读者往往会对本书让他们做的事情过于兴奋，以至于他们有时会跳过像“关系数据库设计”这样的重要章节。自从这本书出版以来，我在我的电子邮件收件箱里看到了一些非常难看的数据库布局！

***SP:你们有没有计划发布一个考虑到 PHP 中 register_globals 被禁用等变化的修订版？**T3】*

绝对的。这本书的修订版是我计划下个月重点关注的内容。

SP:你平均每天会收到多少关于这本书的问题？T3】

多亏了汤姆·鲁特，SitePoint 论坛的观众更熟悉他的名字 MMJ。他最近开始回复关于这本书的支持邮件并编辑。这些天我的个人收件箱里每天只收到一两封。

SP:如果有一个问题你一直在回答，你想给它一个一劳永逸的答案，它会是什么？T3】

不，我不想知道在家里卖伟哥一年能挣 50 万美元！谢谢关心！

SP:你也是 Wrox 出版社出版的一本 PHP 书籍的共同编辑。这是怎么做到的？你在哪个部门工作过吗？你是怎么得到那份工作的？T3】

实际上，我曾经担任过一些 Wrox 书籍的技术编辑，其中包括“初级 PHP4”和“专业 PHP4”。基本上，这本书的主编会给我发一些章节的草稿，这些草稿是由这本书的不同作者提交的。我有几天时间把这些提交的材料撕成碎片，以确保技术的清晰性、准确性和可疑的出身。

然后，我的评论会反馈给作者，作者会修改这一章，以修复我的评论中发现的问题，然后将这一章发送给总编辑，总编辑会清理拼写和语法等内容。

这份工作来自我最初的 PHP/MySQL 系列文章，后来成了我的书。Wrox 的一个人注意到了我在这个领域的专长，问我是否有兴趣做一些技术评审的工作。看到我的名字出现在一本“真正的”技术书籍的演职员表上是很有趣的，但后来我退出了 Wrox 的编辑工作，为其他优先事项腾出了时间。

典型的一天是从我早上 10:30 左右起床，穿着居家服查看邮件开始的。在家远程工作有它的好处！

但是现在我在 SitePoint 办公室，每天早上 7:30 开始上班。在典型的晨间活动之后，在狂风大作的霍德尔街上漫步 15 分钟，在上午 9 点过后不久，我就开始工作了。

我通常在办公室吃午饭，但我喜欢每周约一次和我的朋友阿奇(因香肠软件而出名)在市中心吃午饭。我还计划每周三上午在墨尔本成人教育中心(CAE)开始一门绘画课程。得去参加课外活动了！

在调整 Java 代码和回答面试问题的漫长一天后，我在下午 6 点左右回家。我的夜晚通常充满了电影(我正在家里组装一套可爱的 DVD)、陪伴或安静的阅读。

如上所述，我一直是一个艺术家。以前上课花无数个小时画漫画书，不用心。在过去的几年里，我已经让我在这方面的技能停滞不前，我期待着很快再次拿起它。

***SP:你看 3 年后的互联网在哪里？**T3】*

我认为桌面上的网络变化相对较小，除了继续向更高带宽的应用程序迁移，因为家庭互联网连接也在向同一方向移动。

真正的进步在于互联网将开始慢慢渗透到我们远离办公桌的日常生活中。连接互联网的手持设备和移动电话仍在努力成为屏幕非常小的便携式台式机以外的东西，但我认为在未来几年内，我们将看到它们开始成为真正有用、有用的设备。

我在考虑一个仪表板显示器，显示附近的可用停车位；一家没有收银台的商店，那里的货架上摆满了你可以订购并在当天送货上门的产品样品；和一个个人音乐播放器，它可以让你以合理的月订阅费在任何时候播放任何歌曲。

##### 未来

***SP:回到编程，编程需要帮助的时候去哪里？**T3】*

写书评是我工作的一部分，这让我有幸拥有了一个广泛的参考图书馆，所以这绝对有帮助。如果做不到这一点，我会像其他人一样浏览支持论坛、新闻组和其他在线资源。当我真的被困住了，花不起几块钱就能得到一个快速答案时，我发现 HotDispatch.com 是一个绝佳的求助地点。

SP:你认为 PHP 在未来 5 年要克服的主要障碍是什么？T3】

随着 PHP 4.0 的发布，PHP 越来越受欢迎，正进入一个大发展阶段。突然间，每个人和他的狗都成了 PHP 程序员，并且有了如何让 PHP 变得更好的想法。PHP 最大的挑战将是避免这种流行带来的爬行特征。

PHP 开发团队需要努力保持 PHP 的简单和易学，否则就有可能失去让 PHP 开始流行和有用的东西。

你会给那些试图为自己打造事业的年轻程序员什么建议？T3】

练习，练习，练习。许多初露头角的开发人员抱怨他们找不到工作，因为每个人都想要经验。就像一个艺术家不能指望他的第一幅画是一个利润丰厚的委托作品一样，一个程序员必须纯粹为了获得经验而建造几个项目，然后他或她才能期望被雇用。

在我的第一份付费编程工作(用 Java 为 [Gemplus](http://www.gemplus.com) 编写 LDAP 目录导入模块)之前，我写了一些供自己使用的小程序。我最早的 GUI 程序是命令行音乐(MOD)文件播放器的包装器。然后，我为一个电子邮件标语管理器编写了我的第一个平面文件数据库。我决定需要学习一点密码学，于是我写了一个个人日记程序，使用一个 Java Cryptography Extension (JCE)兼容库来保护日记条目的内容。

对于 Web 开发来说，就更容易了！总有一些非营利组织、朋友的朋友和其他熟人需要网站，但他们没钱聘请专业设计师/开发人员。如果其他方法都失败了，你可以建立一个你知道的最棒的个人网站。

游戏的名字是保持忙碌。用你的服务换自由软件、主机或任何你能换的东西，但是即使你不得不白干，记住这早期工作的真正价值是为你自己建立一个投资组合。接下来，你就可以带着两年的工作经验去参加面试了。

SP:最后，有好奇心的人必须知道…你如何看待丹麦乳制品市场与全球 PHP 用户数量增长的关系？T3】

成群结队的 PHP 编码的丹麦奶牛，不再需要生产牛奶来生存，每天都沉重地压在我的良心上。但这总比让他们转向网络犯罪要好！我说，让我们都喝大豆和快乐！
