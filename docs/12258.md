# 上游流动降价

> 原文：<https://www.sitepoint.com/flowing-markdown-upstream/>

Markdown 已经成为最近最流行的替代标记语言之一(不是 T2 最流行的)。其中一个原因是它已经被移植到了大多数平台上(或者是反过来？).最近，我尝试用它作为终端用户的输入格式，尽管结果好坏参半。那些精通技术的人，学得很快，结果比传统的所见即所得编辑器好得多，基于[内容可编辑](http://msdn.microsoft.com/en-us/library/ms537837(VS.85).aspx)。然而，不太懂技术的人开始期待 wysiwyg，编写标记语言的想法——即使像 markdown 一样优雅——对他们来说是陌生的。

受 javascript 的 [markdown 端口的启发，我有了一个想法，过去几周我一直在充实这个想法。这个想法是将降价输入和实时预览结合起来，结果是介于原始降价输入和所见即所得之间的一种中间状态；作为用户，你仍然需要写 markdown，但是反馈是直观和即时的。](http://attacklab.net/showdown/)

不幸的是，我没有任何候选人来试用它，但我现在已经把它放在 github 上，所以现在它就在那里。当然还有改进的空间，但基础已经具备。我甚至加入了一个扩展功能，因为这是我自己在 CMS 中使用 markdown 时用过的。

在[http://troelskn.googlepages.com/upflow.html](http://troelskn.googlepages.com/upflow.html)有个演示

你可以从[http://github.com/troelskn/upflow/tree](http://github.com/troelskn/upflow/tree)获取代码
