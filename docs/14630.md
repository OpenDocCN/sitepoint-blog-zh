# 桌面机会

> 原文:[https://www.sitepoint.com/desktop-opportunities/](https://www.sitepoint.com/desktop-opportunities/)

当安全问题像 Sasser 和那些在过去几年中臭名昭著的问题出现时，这又是一个与您的客户、同行和潜在客户谈论桌面安全的机会。这也是一个机会，在这种情况下，可以展示 Linux 和 OS X 桌面作为 Windows 的替代产品。

在我继续之前，我要补充一点，因为这是另一个有争议的问题。我的目标不是将 Windows 展示为一个糟糕的系统，并将开源作为唯一的出路。每当一个平台变得越来越受欢迎，安全问题也会随之而来。我们很多人都知道 Linux 是世界上最受攻击的系统之一(即受到攻击，而不是被破坏)。对于 Linux 和 OS X 台式机来说也是如此——病毒或至少是企图肯定会出现。

然而，基于 Unix 的系统在基础架构和大量开源商业工具方面比 Windows 更有优势，这些工具可以对服务器和桌面进行粒度控制以确保安全性。

此外，在我与客户和其他组织的许多互动中，每一站都有许多用户可以轻松地使用 Linux 或 OS X 桌面，只需适度的培训，过程中很少或没有中断。这是由于我所说的标准用户的三位一体:

*   他们用的是办公套间
*   他们使用电子邮件客户端
*   他们使用网络浏览器

随着 Linux 上桌面工具的激增，包括 Open Office(参见 [eWeek Labs real world review](http://www.eweek.com/article2/0,1759,1571626,00.asp) )、电子邮件的 Exchange 兼容客户端和成熟的浏览器技术，这些基线员工很容易成为培训和转换到非 Windows 桌面的目标，并立即开始改善外围设备安全问题。此外，由于微软对 OS X 的支持，以及 Office X 对 Exchange server 的支持，对于那些担心额外培训费用的用户来说，Mac 台式机可能是一个更好的主意。

更好的是，微软在最新发布的 Office for OS X 中将项目管理功能部署到 Entourage(支持 Exchange 的电子邮件客户端)中

解决第二层更复杂的用户是一个更大的挑战。在某些情况下，Windows 模拟器可能是一个选项，但是，它也重新打开了 Windows 操作系统上发现的安全问题。更好的办法是，在你计划任何类型的转换时，寻找能够为必要的应用程序提供支持的解决方案。

两个例子:

Mr Project ，一个基于 Gnome 的 Linux 项目管理工具，提供了 Microsoft Project 强大的计划功能，并且正在努力完成一个导入工具来接受 MS Project 文件。

微软继续在他们的主要应用程序中发布关于 XML 支持的模式信息。这使开发人员能够利用这些数据，并通过 Word 和 Visio 等应用程序进行导入/导出。 [Omni Graffle Pro](http://www.omnigroup.com/applications/omnigraffle/pro/) 以低得多的价格提供了与 of Visio 相同的强大绘图功能，并包括 Visio XML 支持，使文件能够在这两个软件包的用户之间来回传递。Linux KDE 桌面提供了 [Kivio](http://www.koffice.org/kivio/) 作为 Visio 风格的图表工具。

需要考虑的是流程要求——将应用程序分成 1)容易转换、2)中等难度转换和 3)几乎不可能转换的用户组。

在我工作过的一家公司，上面的第一组是接待员、客户服务人员和通信部门。由于 Microsoft Office 的支持、对基于 web 的 CRM 系统的持续访问以及对 Exchange 服务器的电子邮件访问，他们很容易转变为 OS X 用户。

第二组和第三组需要进行大量研究和考虑，以规划和设计满足用户需求的潜在桌面转换。随着更复杂的应用程序被移植到基于浏览器的访问，这可能会成为现实。
