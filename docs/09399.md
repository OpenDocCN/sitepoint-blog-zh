# 祝你 10 岁生日快乐！

> 原文：<https://www.sitepoint.com/ie6-is-10-today/>

我知道你们中很少有人想庆祝，但是今天是 IE6 十岁生日。我敢肯定，最初的开发人员在 2001 年 8 月 27 日发布最终版本时从未想到它会达到两位数。

十年对于任何软件来说都是一个惊人的成就——尤其是当你考虑到网络本身只有 20 岁的时候。即使在今天，仍有超过 1/30 的人在使用老化的浏览器。

尽管获得了众多奖项*、【有史以来最差的科技产品】、*、【地球上最不安全的软件】、*，IE6 在 2001 年仍然是一款出色的浏览器。它给了我们:*

*   改进了 CSS(并修复了 IE5 不正确的盒子模型)
*   DOM 级别 1
*   一个好的 XML API
*   时尚、快速的界面
*   一个有用的 IE 管理工具包，用于在组织内快速部署。

它杀死了网景 4.0 和 Mozilla 套件。有很多诉讼案件，但是，坦白地说，这些浏览器应该很快痛苦地死去。感谢你今天不必支持他们。

## 哪里出了问题？

微软赢得了浏览器大战，到 2001 年，超过 95%的网络用户使用 IE 浏览器。

很少有人认为 web 应用程序是桌面和微软支持的“智能客户端”的可行替代品；基于 Windows 的程序，具有良好的界面和互联网通信的优势。这是一个合乎逻辑的决定，只会加强 Windows 在操作系统市场的统治地位。微软宣布 IE6 将是他们最后的独立浏览器，开发团队被解散。

然后 Web2.0 出现了。Ajax 是支撑 Web2.0 的主要“技术”，具有讽刺意味的是，这只能归功于微软在 IE5.0 中引入的一个鲜为人知的 XMLHttpRequest 对象。

与此同时，Mozilla 终于推出了一款功能强大的浏览器，它摆脱了传统的网景代码，对 IE 浏览器构成了严重威胁。Firefox 遵循 web 标准，并提供允许开发人员创建复杂 web 应用程序的工具。

很快就发现 IE 落后了，阻碍了网络的发展。微软花了几年时间才意识到他们的错误，并发布了 IE7。它有更好的标准支持，但是破坏了 IE6 五年统治期间编写的许多 web 应用程序。许多公司发现他们无法升级。

## IE6 真的那么差吗？

抨击 IE6 很容易——微软自己动手。也就是说，一些 web 开发人员抗议得有点过分:斥责 IE6 远比“修复”代码容易。IE6 的大部分问题都已经被很好的理解了，并且有文档化的解决方法。假设你早测试，经常测试，在 IE6 诞生十年后仍然有可能支持它*(只要你忘记像素完美的徒劳尝试)*。

问题是:*该不该支持 IE6？*

对于大多数网站来说，在使用过时软件和有问题的 IT 技能的少数用户身上花费过多的开发时间是没有商业理由的。如果你在一家大公司或政府部门工作，情况可能会有所不同，但是，如果你不喜欢它，有一个简单的解决办法…

就我个人而言，IE6 给我带来了痛苦，但我已经足够老了，可以从一开始就使用它，并理解它的古怪行为。利润也部分抵消了开发障碍——我仍然通过修复 IE6 问题赚钱！几年来，它一直是我的默认浏览器，相比 IE7，我仍然更喜欢它。

所以，祝你十岁生日快乐。是时候永远退休了。

你会庆祝吗？
