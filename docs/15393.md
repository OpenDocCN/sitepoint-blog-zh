# 超越网景和 Internet Explorer

> 原文:[https://www.sitepoint.com/netscape-internet-explorer/](https://www.sitepoint.com/netscape-internet-explorer/)

当人们谈论互联网浏览器时，你马上会认为他们指的是 Internet Explorer 或 Netscape Navigator。然而，还有许多其他“另类”浏览器值得一看。在这篇文章中，我将回顾那些我认为值得安装的，作为两个大玩家的替代品。如果你不经常上网，也不热衷于在硬盘上再增加 12 MB 空间来安装一套你可能永远不会用到的浏览器，你可能要考虑另一种浏览器了。这些浏览器对开发者来说也非常有用，因为它们为网站提供了一个测试平台，以确保尽可能多的潜在访问者访问你的网站。

Opera 这必须是最流行的替代浏览器，并且与基于 Windows 的系统兼容。Opera 的出名之处在于它速度快，体积小，只占用 1.7 MB 的磁盘空间。此外，它还具有邮件和新闻组功能，支持 JavaScript 并能让你将自己喜欢的网站加入书签。Opera 是从零开始构建的，并不基于任何现有的浏览器代码，因此，它是根据用户反馈而设计和更新的。它唯一的缺点是没有任何本地 java 支持，而有些人可能会觉得这是必不可少的。然而，对于它的速度、它的小内存使用量以及它适合放在软盘上的事实来说，这是一个很小的代价。

马赛克 马赛克的成名在于它是第一个图形浏览器。它由国家超级计算应用中心开发，我们当中的开发者对它特别感兴趣。如果你对玩 Mosaic 浏览器感兴趣，你可以下载源代码，自己试一试。

**[Amaya](https://www.w3.org/Amaya/)**Amaya 不仅仅是一个浏览器，它还是一个强大的创作工具。W3C 使用它来演示和测试 Web 协议和数据格式的许多新发展。它是通用的和可扩展的，可以在 Unix 和 Windows 95/NT 平台上使用。使用 Amaya 你可以创建网页并上传到服务器上。作者可以从头开始创建文档，他们可以浏览 web 并找到他们需要的信息，将它复制并粘贴到他们的页面上，并创建到其他网站的链接。所有这些都以简单明了的方式完成，并且在一个一致的环境中执行操作。编辑和浏览功能无缝集成在一个工具中。对于一般的浏览者来说，它可能不是完美的浏览器，但是对于开发者来说，它是一个必不可少的工具，因为它严格遵循 HTML 标准，因此使它成为一个非常好的测试浏览器。

这款浏览器的美妙之处在于，它可以在任何装有 386SX 芯片或更好芯片的 IBM 个人电脑上运行在微软视窗系统下。这意味着，如果你有一个非常旧的系统，也许是你偶尔在旅途中使用的笔记本电脑，那么这是一个适合你的。Cello 与其他软件工具很好地集成在一起，你可以用它来准备 HTML 文档，提供 WWW 信息，从网上查看图形，听数字声音文件，以及许多其他事情

**[阿拉克尼](http://home.arachne.cz/)** 阿拉克尼 WWW 浏览器是一款全屏图形化的万维网浏览器，目前可用于 DOS 兼容的操作系统和 Linux/SVGAlib 平台。阿拉克尼支持 HTML/4.0 的子集，GIF，JPEG 和 PNG 图像，流行的互联网协议 http，ftp，smtp 和 pop3，等等。不是每个人都使用微软视窗作为他们的操作系统，像这样的浏览器给了这些用户上网冲浪的机会。像许多其他浏览器一样，阿拉克尼只需要 8g 内存，应该安装在 386 或更高的版本上。下载只有 529k，这意味着你可以很容易地从一张软盘安装。

**[HotJava](http://java.sun.com/products/hotjava/index.html)** 顾名思义，这款浏览器完全用 Java 编写，也就是说它会在任何支持它的平台上运行。尽管 Java 以速度慢著称，但这个问题在这个浏览器中并不存在。它和前面提到的其他浏览器一样快，并且不会消耗太多的计算能力。它最大的缺点是，你必须等待整个页面下载后才能查看任何内容，这可能是决定你是否要使用这个 Java 浏览器的唯一因素。

**[Lynx](http://lynx.browser.org/)** Lynx 是目前最小的浏览器之一。下载一个页面需要几秒钟，因为它是一个纯文本浏览器，所以它不支持图像(这是一个测试你的 ALT 标签是否相关的好方法！).Lynx 没有鼠标支持，所以你必须使用光标键，而且它不支持 HTML 的一些特性，比如表格和框架。Lynx 可用于大多数平台，是语音用户的完美浏览器。然而，纯文本方式可能会让一些人望而却步，但是如果你使用的是一台低规格的计算机，并且需要一种速度极快、处理能力小的浏览器，那么这可能是你理想的替代浏览器。

替代浏览器并没有成为过去，事实上许多潜在的访问者可能正在使用它们。然而，除非你能确保你的网站能被其他浏览器浏览，否则就不能保证你能让尽可能多的人看到你的网站。在应用更新的、更常用的浏览器所提供的功能和为其他浏览器用户提供支持之间找到一个折衷方案是很重要的。
