# IE7 用户代理字符串已发布

> 原文:[https://www.sitepoint.com/ie7-user-agent-string-published/](https://www.sitepoint.com/ie7-user-agent-string-published/)

IE7 黑客(微软黑客对真正的 IE7，即不是[狄恩·爱德华兹](http://dean.edwards.name/IE7/):)透露 [IE7 的用户代理字符串将是“Mozilla/4.0(兼容；MSIE 7.0bWindows NT 6.0)"](http://blogs.msdn.com/ie/archive/2005/04/27/412813.aspx) 。

你们中那些正在思考的人。谁在乎用户代理字符串？应该鼓励一下自己。我们，全世界的网络开发者，不应该关心他们。使用用户代理字符串嗅探浏览器是非常糟糕的，你不应该这样做；请改用功能嗅探，测试您想要使用的功能。不要测试用户代理字符串。我有点担心 IE 团队这么早就发布用户代理字符串的细节，因为这至少意味着他们希望人们开始测试它，这使他们意外地与我们应该做的事情脱节。这很令人惊讶，因为他们都是聪明的家伙(是的，他们是，停止刻薄的评论)，如果知道，比如说，[人们目前用来排斥 IE](http://centricle.com/ref/css/filters/) 的 CSS 黑客是否会继续工作会更有用。

不过，至少你现在可以开始检查你的服务器访问日志，看看 IE7 用户是否在浏览你的网站。
