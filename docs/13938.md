# Web 2.0 是关于…

> 原文:[https://www.sitepoint.com/web-20-is-about/](https://www.sitepoint.com/web-20-is-about/)

这是《科技时报》第 124 期的再版，给那些不关注该出版物的人。

Web 2.0 是关于开发社区认为的当今 Web 开发中新的好的想法和趋势。尽管象征 Web 2.0 的特定技术、公司和应用程序每天都在变化，但它们都有一些共同的核心主题。在我看来，正是这些主题定义了 Web 2.0。

我应该指出，我并不是自己设计了这个列表。在我之前，许多聪明的人已经解决了这个问题，这只是我对答案的看法。你可能想参考的其他权威是蒂姆·奥赖利的 <cite>[什么是 Web 2.0](http://www.oreillynet.com/pub/a/oreilly/tim/news/2005/09/30/what-is-web-20.html)</cite> 和杰夫·维恩的 <cite>[为 Web 2.0](http://www.veen.com/jeff/archives/000793.html)</cite> 设计。

## 作为应用程序的站点

当人们第一次听说 Web 2.0 时，他们通常是从这个角度来看待它的:具有丰富用户界面的网站，其行为更像桌面应用程序，而不是一组相互链接的页面。

第一个广泛使用的例子来自谷歌: [Gmail](http://mail.google.com/) 和[谷歌地图](http://maps.google.com/)。如果你还没有尝试过，你真的应该试试。

Macromedia 实际上在几年前就开始在 Rich Internet Applications (RIAs)的旗帜下将这种方法推广到 Web 设计中。在 Macromedia 看来，RIA 最好使用 Flash 作为前端来显示丰富的用户界面，这促使他们用一个接口组件库来改进 Flash MX，并发布 Flex，这是一个供桌面开发人员以熟悉的方式设计 RIA 的平台。

Macromedia 最近以新的名字[Flash Platform](http://www.macromedia.com/platform/)重新发起了这场运动，与作为谷歌流行应用核心的 [AJAX 驱动的 DOM Scripting](https://www.sitepoint.com/article/remote-scripting-ajax) 的替代方法竞争。

## 用户参与

Web 2.0 的很大一部分是将被动的访问者和读者转变为创造网站价值的积极参与者。如果你曾经沉迷于一个基于网络的论坛，那么你就会知道这有多么强大。Web 2.0 就是利用“社区的力量”并将其作为一个整体应用到网络中。

当然，我想到了博客。志趣相投的用户组成的社区，每个人都有自己的博客，将用户生成的内容放在前台和中心。但是博客仅仅是开始。维基百科和 Flickr 是目前用户生成内容的典型代表。

维基百科只是一个非常古老的想法的最近和流行的例子:维基是一个任何用户都可以编辑一切(或几乎一切)的网站。该理论认为，网站收集了所有用户关于某一特定主题的全部知识，产生的内容比其各个部分的总和更有价值，而且不会被任何一个人破坏。在百科全书的意义上，维基百科通过选择常识作为其主题而与众不同。

至于 Flickr，它不仅是一个展示完全由其用户制作的内容(即照片)的橱窗，而且 Flickr 还进一步允许网站的 T2 结构来自其用户。用户可以组建自己的特设小组来讨论相关的照片，并可以给他们的照片添加标签，这些标签是形成“大众分类法”的简单关键字，这是一种社区驱动的传统类别结构(一种分类法)。该网站只是为自己的用户提供精心设计的工具，以建立一个有价值的内容库。

更进一步说，*宁是一个为用户提供与 Flickr 相同的框架工具的网站，除了这些工具被一般化以允许任何类型的用户驱动的内容库涌现。*

大玩家也在跳上用户生成内容的潮流，谷歌控制着最容易使用的博客平台之一， [Blogger](http://www.blogger.com/) ，以及[雅虎新的用户生成内容焦点](http://www.readwriteweb.com/archives/002863.php)，其[播客](http://podcasts.yahoo.com/)和[博客搜索](http://www.ysearchblog.com/archives/000198.html)服务。

## 公开

许多传统媒体机构——特别是音乐和电影行业——正因为将业务建立在封闭模式上而受到很多批评:将内容锁定在专有格式和加密方案之后，这些格式和加密方案旨在限制用户可以对内容做什么。

Web 2.0 是关于相反的方法:通过开放获得成功。但是开放意味着很多事情，尤其是在网络环境中。

这可能意味着建立符合开放标准的网站，比如针对 [HTML](https://www.w3.org/MarkUp/) 、 [CSS](https://www.w3.org/Style/) 、[脚本](https://www.w3.org/DOM/)和[可访问性](https://www.w3.org/WAI/)的 [W3C](https://www.w3.org/) 建议。这并不一定会扼杀创新，因为微格式倡议正在构建新的(开放的)数据格式，以符合现有的标准，如 HTML。

这可能意味着让用户控制他们自己的数据，这样无论他们在你的网站上放了什么，他们都可以在接到通知后立即以一种标准格式再次发布，他们可以在其他应用程序和服务中使用这种格式。事实上，Flickr 将使用任何嵌入你的照片中的标签或说明[是一个好的开始。如果他们以同样的格式存储你输入网站的标签和说明，那就更好了，这样，如果你觉得有比 Flickr 更好的东西，你就可以下载你的照片，所有元数据都保持不变。](http://www.iptc.org/IPTC4XMP/)

这甚至意味着让用户控制你的数据，为组成你网站的服务发布 API，这样用户就可以在他们自己的网站上混合搭配这些服务。一个很好的例子是[的谷歌地图 API](http://www.google.com/apis/maps/) ，它允许任何人在自己的网站上显示一个全功能的动态地图。

围绕这些开放的 API，一种欣欣向荣的“mashups”文化正在兴起，这证明了在这种开放中存在着真正的机会。

Web 2.0 无疑意味着各种形式的开放。

当然，我在这里提到的一切都被认为是今年的 Web 2.0。到明年，“新思维”将会继续发展，“Web 2.0”一词的含义也将随之改变。

别走开——我们可以一起跟着它。
