# 如何告诉组织他们的网站无法访问

> 原文:[https://www . site point . com/how-to-tell-organizations-their-website-is-accessible/](https://www.sitepoint.com/how-to-tell-organizations-their-website-is-inaccessible/)

访问 SitePoint 的开发者是网络上最尽责的人。然而，你可能会惊讶地发现，有人在回避可访问性，回避标准，放弃渐进增强，低估可用性。恐怕是真的。对于每一个精心制作的可访问网站，都有数百个令人震惊的阻止访问网络社区的例子。我们都见过这样的网站:

*   中断键盘导航或强制使用鼠标
*   使用无法设置为更大尺寸的小而难以辨认的文本
*   实现没有 JavaScript 就会失败的表单或页面控件
*   有难以阅读的图案或颜色组合
*   使用分散注意力的动画，或者
*   闯入除组织使用的浏览器之外的任何浏览器。

很少有开发者考虑可访问性。他们的客户没有意识到其中的含义，利益似乎无关紧要(这只是关于盲人，对吗？)，很少有用户会注意到任何问题，并且它通常在优先级列表中较低或缺失。

幸运的是，W3C 的 Web Accessibility Initiative (WAI)发布了一份指导文件，供任何想就网站的可访问性问题联系某个组织的人使用。虽然它的目标是普通用户，但 [**就无法访问的网站联系组织**](https://www.w3.org/WAI/users/inaccessible) 提供的信息可能对那些想接触公司改善其劣质网站的机构和自由职业者有用。

该文件包含以下建议:

*   考虑你的方法和你想要达到的目标。
*   如何识别组织内的关键联系人？
*   描述可访问性问题。
*   在哪里可以找到辅助功能信息资源。
*   要求答复和打后续电话。
*   示例电子邮件。

我怀疑这份文件会改变许多人的态度，但它可能有助于提高网页可访问性的形象，并可以为组织提供一个进一步调查问题的理由。

你有没有联系过某个组织关于他们的网站无法访问？他们做了什么来解决这个问题吗？还是你的要求被忽略了？
