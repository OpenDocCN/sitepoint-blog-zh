### 案情摘要

使用 HTML、CSS 和 Javascript 这三种 web 标准制作一个按钮。只有。我们欢迎在你的 HTML 中增加数字 5，在你的 CSS 中增加数字 3。让你的同行们为你的纽扣的优雅风度而欣喜若狂，让他们惊叹于它的技术力量。

### 限制

*   零图像容差
*   强制动画
*   文件大小无情的最小化

所以你的目标是一个任何类型的按钮，它不使用任何图像来呈现，但是它必须以某种方式来动画化。所有这一切，同时保持您的文件总大小下降到一个无情的最小值。

### 自由

您可以自由地包含由 [Google Libraries API](http://code.google.com/apis/libraries/) 托管的任何开源 Javascript 库，或者它们的任意组合。但是，请注意，夹杂物**的重量将**计算在内。还要注意一个明显的例外，使用“SWFObject”自然会使提交无效。

“视觉奇观”和“技术奇迹”两个标准都将获得满分 200 分的奖励。然后，Javascript 包含的权重和我们对加载时间的度量将被输入到一个计算中，将初始分数 504(超时)减少到最终贡献。因此最高分为 904。

0。发明这个？绝对不是，大部分功劳归于 [10K 除了](http://10k.aneventapart.com/)豪门，还有他们反过来站在豪门肩膀上的，也就是 [5K](http://www.sylloge.com/5k/original.html) 暴民。还有，向[@马拉其](http://twitter.com/#!/malarkey)的[迪克](http://www.flickr.com/photos/andallthatmalarkey/4248650431/)脱帽致敬，从中获得了一点灵感。

**1。太自由了吗？**当然关于上面的问题。然而，我们不会限制您的 Javascript 包含，但我们会判断您选择的常识和敏感性，以及 504(超时)贡献。

**2。第三方？**不，不允许。提交按钮的所有内容都必须位于您的`root`文件夹中。除了自由图书馆没有其他的。因此，像 Flickr 和 Twitter 这样的第三方，以及脸书，都被排除在外。

**3。法律术语？**不完全是，但当然是有的。简单来说:我们保留在评判时行使自由裁量权的权利。提交作品风险自担。你的魅力被认可的风险是真实存在的。不会有任何信件往来。

**4。特殊工作？**绝对不会。如果你想参加挑战，请参加，并理解这不是任何法律意义上的比赛。如果有人确实想为你的工作支付报酬，在挑战范围内或根据一般优点，我们保证传递我们收到的任何询问。

**5。倍数？**允许一方多次录入。允许多方单独输入。所有都属于状态代码 300(多选)。多方成功提交将需要为剩余的多名与会者分摊费用。
