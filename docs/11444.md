# 谷歌的搜索淡入:有什么意义？

> 原文:[https://www.sitepoint.com/googles-search-fade-in-useless/](https://www.sitepoint.com/googles-search-fade-in-useless/)

谷歌很少抛头露面，他们一直在搜索主页上推广他们新的淡入效果。你可能看过。虽然你可能还没有——谷歌正在测试这个系统，并不是每个人都可以查看。

它是这样工作的:

1.  当你访问[Google.com](http://www.google.com/)时，只显示标志、搜索框、搜索和“我感觉很幸运”按钮。
2.  现在，您可以键入一个术语并按 return 键来执行搜索。
3.  但是，如果鼠标光标移动到浏览器视口中的任何位置，正常的链接和选项都会消失在视图中。

(注意 Google 使用了[渐进式增强](https://www.sitepoint.com/progressive-enhancement-1-html/)，所以没有 JavaScript 的用户会看到标准页面。)

据[谷歌博客](http://googleblog.blogspot.com/2009/12/now-you-see-it-now-you-dont.html)报道，该公司已经测试了 10 种淡入淡出的变体，公司员工*【非常喜欢】*。

对不起谷歌，但我不明白？淡入到底有什么好处呢？

向用户隐藏高级或很少使用的功能可能是一件好事。比如很多人已经学会喜欢微软 Office 里的 ribbon。然而，在这种情况下，将少数选项隐藏几毫秒有什么意义呢？我不敢相信许多用户被这些链接淹没了——大多数人忽略了它们。

如果淡入是针对新的电脑用户，我怀疑很少有人会注意到这种效果。根据我的经验，比起键盘，新手更倾向于过度使用鼠标。例如，许多人会切换到鼠标点击“确定”按钮，而不是点击回车键。一旦页面加载完毕，这些用户就会看到淡入效果。

我最初想知道这种淡入是否会帮助触摸屏或移动用户，但激活屏幕键盘也会显示链接？

另外，效果需要几行 JavaScript。对我们大多数人来说，这不是问题，但是额外的页面权重可能会大大增加谷歌的主机和带宽成本。这是一家为了节省几个宝贵的字节而不添加结束`body`和`html`标签的公司！

最终，我只能想到谷歌引入来自 [Bing](http://www.bing.com/) 的淡入…竞争的一个原因。必应更漂亮，提供的动画也更好，但谷歌的效果很难让逃兵起死回生。

不管是什么原因，很多人都在问如何禁用淡入，一些 [Greasemonkey 脚本](http://olivierfabre.com/fadein)已经出现。如果谷歌保持这种效果，我会感到惊讶。

如何看待谷歌的淡入效果？它有用吗？或者谷歌应该坚持基本的搜索而不使用那些噱头吗？
