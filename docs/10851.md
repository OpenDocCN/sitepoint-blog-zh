# 浏览器标签应该在地址栏的上面还是下面？

> 原文：<https://www.sitepoint.com/browser-tabs-above-below/>

Mozilla 决定在 Firefox 4.0 中默认地址栏上方会出现标签。标签定位的争论已经持续了很多年，但是这个组织已经确定在顶部是最合理的安排。

到目前为止，浏览器厂商的分布相当均匀。Opera 7 是第一批具有标签功能的浏览器之一——虽然后退、前进、刷新和主页图标在标签上方，但它们却位于地址栏上方。Opera 10 现在顶部有标签。

谷歌 Chrome 比大多数厂商走得更远，用标签栏取代了窗口标题。苹果在 Safari 4 beta 版中尝试了类似的安排，但放弃了这个想法，并在所有后续版本中回到地址栏下方的标签页。

尽管我怀疑 IE7/8 界面是由委员会设计的，但是微软把标签放在所有其他控件的下面。让我们希望他们在 IE9 中解决这个问题。

火狐的地址栏下面有标签，虽然有很多标签扩展，但是没有几个可以重新安排位置。

Mozilla 列举了他们改变观点四个主要原因:

1.  概念模型:地址栏和控件应用于当前选项卡。
2.  应用标签:和 Chrome 一样，Firefox 4.0 允许你将常用的小标签钉在标签栏上。这些 web 应用程序的地址栏和其他控件将被删除。
3.  基于标签的用户界面:Firefox 4.0 将在标签中显示窗口，如下载和书签管理器。让地址栏和其他控件可见是没有意义的。
4.  通知:一些错误和警告信息现在出现在地址栏下面。

我敢肯定 Chrome 和 Opera 也影响了他们的思维。

缺点是增加了鼠标到标签的距离。然而，选择位于屏幕边缘的窗口顶部的标签可能更容易。

Mozilla 已经征求反馈，并正在进行可用性测试。他们提供了一个包含视频和截图的页面，到目前为止，大多数人的反应是积极的。你还应该注意到，如果你更喜欢地址栏下面的标签或者不能适应新的布局，可以改变 Firefox 的配置。
