# 把握下一次面试的完整指南

> 原文：<https://www.sitepoint.com/complete-guide-nailing-next-interview/>

你申请了一份开发人员或设计师的工作，现在他们让你来面试。恭喜你！但是现在真正的工作开始了。你如何确保自己给人留下好印象，并得到这份工作？这里有一些帮助你搞定下一次面试的建议。

## 准备面试

### 了解公司

当你写申请信的时候，你可能已经仔细看过公司的网站了。现在是时候通过查看每一页来更深入地了解这家公司，并在其他地方寻找更多信息了。

试着看看公司是做什么的，他们看重什么。搜索公司名称，看看是否能找到更多关于公司历史的信息。记笔记，你找到的信息将构成你可能要问的任何问题的基础。

你可能会发现一些关于公司的负面信息。如果在面试中有人提到这一点，请随意询问更多。根据具体情况，可能是一些他们不愿谈论的敏感话题。所以不要过于用力。

研究你的面试官也是很好的。也许你们有共同的兴趣，可以在面试时提及。例如，如果你们都是电视剧《权力的游戏》的粉丝，你可以提到这个话题，让你们有一些与工作无关的话题可以聊。

### 跟上时代

确保你了解关于你当前技能的最新消息。如果你是一名 PHP 开发人员，你应该知道像 PSR、[作曲家](https://www.sitepoint.com/composer-cheatsheet/)、[贝哈特](https://www.sitepoint.com/8-must-have-php-quality-assurance-tools/)和[拉勒维尔](https://www.sitepoint.com/blog/)这样的热门话题。如果你对 HTML 和 CSS 更感兴趣，确保你对 [Sass](https://www.sitepoint.com/getting-started-with-sass/) 和 [LESS](https://www.sitepoint.com/a-comprehensive-introduction-to-less-mixins/) 了如指掌。

让自己跟上时代的最好方法之一就是在 Twitter 上关注那些在你的行业内有重大影响力的人。另一个好方法是注册 [SitePoint 简讯](https://www.sitepoint.com/newsletter/)或每日简讯[版本](https://www.sitepoint.com/versioning/)。

在面试中，使用技术术语是可以的。例如，你可以提到你参与了一个 PHP 项目，该项目已经过全面的单元测试并且符合 PSR 标准。

### 准备作品集

准备好投资组合应该是你的首要任务之一。没有什么比展示你的能力更好的了。我可以告诉世界上的每个人，我是最好的设计师，但没有任何证明，这不会有任何意义。

个人网站是最好的选择。不要说“我不擅长设计”而忽略了这个要求。现在，你可以在网上找到免费或便宜的主题，每个人都可以安装一个 WordPress 网站。请确保您可以展示您的设计，或者您可以通过链接到 Github 页面来展示您的开发技能。

如果你不能在网上展示任何东西，把你自己创作的东西发给他们，或者带上你的笔记本电脑。确保一切都准备好了，这样你就不会浪费时间在面试中打开一切。你是应用开发者吗？太好了！确保这些应用程序安装在你的手机或平板电脑上，并在演示中完美运行。

如果你准备好了你的作品集，你可以在谈论你的技能时指出这一点。如果公司重视单元测试代码，那么展示你如何单元测试你的代码。

问面试官你是否能给他们展示你创作的作品。通过采取主动，你表明你对自己创造的东西有信心。

### 选择合适的服装

你的穿着会有很大的不同。你能做的最好的事情就是查看该公司的团队页面，并尝试匹配你正在看到的风格。他们是比较正规的公司吗？那么你应该考虑穿西装。他们更像是一个非正式的公司吗？那么便装就不错了。

就算是随便，多花点心思还是好的。穿俗艳的衣服会分散面试者对你个人和技能的注意力。挑选一些没有分散注意力的图片或文字的中性服装是很好的。

### 准时

这一点怎么强调都不为过。在你去面试之前，确保你知道地址，你可以在哪里停车，以及你需要多长时间到达那里。与其让公司等着你，不如早点到，花 15 分钟消磨时间。他们的时间更宝贵。

## 你会被问到的问题

当你参加面试时，准备好被问很多问题，这样公司可以更好地了解你这个人。你需要准备。

[这里的](http://jobsearch.about.com/od/interviewquestionsanswers/a/interviewquest.htm)列出了一些非常常见的问题和你能给出的最佳答案。

一些显而易见的准备工作包括:

### 自我介绍

从你目前的情况开始:你的年龄，你目前的婚姻状况，你住在哪里。

然后告诉他们你生活中与工作相关的部分。从你的学校生涯开始，解释你学了什么。继续你做过的工作，解释你在一家特定公司的职能和职责。

在这期间，你可以告诉他们你在业余时间做过的任何事情:你完成的课程或者你参与的开源项目。

### 你为什么辞去上一份工作？

如果你离开是因为你在寻找新的挑战，那就要弄清楚那是什么。如果你因为经济原因或战略改变而被裁员，说出真相。最后，你不得不被踢出去。

如果你因为自己的所作所为被踢出局，你有两个选择。你可以公平竞争，说出真相。确保你总是能够解释为什么这种情况不会再发生，以及你从中学到了什么。或者，你可以说你不喜欢谈论它。但是在这种情况下，你可能不会得到面试官的任何分数。

总的来说，尽可能诚实总是最好的。面试官有可能会联系一些公司，最终可能会发现真相。

### 你的强项是什么？

努力做到独一无二。大多数人回答说，“我可以独立工作，也可以在团队中工作，”或者，“我擅长 Photoshop。”而是尽量说一些符合职位描述的话。如果他们在寻找真正擅长创建 API 的人，不要告诉他们你在 HTML 方面有多好。你应该看看你是否有与工作相匹配的优势，并强调这些优势。

### 你的弱点是什么？

这个问题迫使你关注自己性格中的消极方面。但是请注意，每一个消极的方面也可以有积极的一面，你也可以解释你是如何学会处理某个特定的弱点的。

如果你是一个没有耐心的人，那么告诉他们你已经知道不是每个人都能立即回复你就可以了。这种方式表明你从自己的弱点中学到了东西，并且你能处理好它。

试着看看你的简历，看看你是否能找到弱点。一定要准备好这些薄弱环节的问题的答案。

您可能会收到类似这样的问题:

*   你为什么没有读完大学？
*   你两年内换了六次工作，为什么？你每次都被解雇了吗？
*   所以你从视频编辑到 PHP 开发人员再到图形设计师。你想做什么？

试着对每一个答案都保持积极的态度。例如，永远不要责怪别人没有完成大学学业。保持简短，不要花五分钟讲述老师有多差的故事。

如果你因为个人生活中的问题而改变了职业，那也没什么。你不必细说，只要让他们知道你过得很艰难就行了。

或者，如果你走上了三条不同的职业道路，这表明你有能力做多件事，现在你终于找到了你真正的激情！

### 你在业余时间做什么？

虽然这个问题听起来很无辜，但它确实透露了很多关于你和你性格的信息。

如果你向他们展示即使在业余时间你也在发展自己，你就能抓住这一点。你阅读博客，尝试新工具等等。作为一名开发人员或设计师，你需要与行业保持同步。

当你谈论你的业余时间时，确保你有各种各样的话题可以聊。告诉他们你喜欢和朋友出去玩，你有时会玩电子游戏，你喜欢每周六去看望你的家人，这很好。不要拘泥于一个单一的主题，展现你的多样性。

你可以试着了解一下面试官在业余时间都做些什么。也许他们是你最喜欢的运动队的超级粉丝。这会让你和面试官有更好的联系，也可以让你谈论一个“非正式”的话题。

## 你可以问的问题

工作面试是一个双向过程，所以一定要写下你想问他们的任何问题。

您的问题可能包括:

*   公司内部的氛围如何？
*   我将和谁一起工作？
*   我的工作是什么？
*   我的工资呢？

查看这篇《福布斯》文章,了解你可以问的更普遍的问题。

但是，你也应该问一些与工作相关的问题，不仅是为了看看这份工作是否适合你，也是为了展示你的技能。

你可以问这样的问题:

*   你使用的是哪种 PHP 开发框架？
*   你使用哪些 Javascript 库？
*   你用的是哪种 CSS 前处理器？
*   你用哪些设计程序？

为了抓住面试的这一部分，总是用疑问词问开放性的问题，比如:谁，什么，哪里，为什么，哪个和如何。这些问题不能用简单的“是”或“不是”来回答，你可以把答案留给面试官。如果你问这样一个问题，“你是在用 Laravel 作为你的框架吗？”你在把面试推向那个话题，即使公司可能不会用 Laravel。

但是请记住，使用开放式问题，答案也不太容易预测。他们可能会突然提到一个你从未用过的工具，然后问你是否听说过它。如果你没有，可以说你使用过类似的工具，或者你学得很快。

## 面试的时候

### 握手

握手是你与面试官的第一次接触。确保你握手有力。通过有力的握手，你会给人一种自信的印象。确保和面试官有眼神交流。

不要马上说出你的名字，而是等几秒钟，因为那时你的名字会更容易被记住。你的两个大脑都忙于握手的任务，这可能意味着你的名字会被忘记。

一定不要忘记面试官的名字！在你谈话的时候说出面试官的名字，在你离开的时候用这个名字。这种方式表明你真的知道你在和谁说话。

### 身体语言

在面试中，你身体上表达自己的方式很重要。坐直，不要向后倾斜太多。此外，不要把你的胳膊放在对方身上，因为你给人的印象是你是一本合上的书。《福布斯》有一个极好的关于你在面试中应该避免的肢体语言的指南。

如果他们给你一些喝的，一定要接受，不管是咖啡还是水。你会说很多话。没有什么比不得不一直挠喉咙更糟糕的了。喝杯水可以让你继续清晰地说话，还可以分散你的注意力。

## 编码测试

如果你申请一份开发人员的工作，很有可能你会被要求做一个或多个编码测试。

每当你得到众多[编码测试](http://maxnoy.com/interviews.html)中的一个，试着想清楚。在大多数情况下，有一个小技巧。基于此，公司将对你的编码技能和解决问题的能力进行评分。在给出答案之前，确保你感到自信。

像[interview . io](http://interviewing.io/)这样的网站就是为了帮助你练习这种技能而设计的。

最受欢迎的测试之一是所谓的嘶嘶声测试。

> 写一个程序打印从 1 到 100 的数字。但是对于三的倍数打印“嘶嘶”而不是数字，对于五的倍数打印“嗡嗡”。对于同时是 3 和 5 的倍数的数字，打印“FizzBuzz”。

这里是用各种语言对这个难题的解决方案，这里是[对各种方法效率的比较](http://www.zoharbabin.com/which-fizzbuzz-solution-is-the-most-efficient/)。

## 面试后

### 显示

你应该做的第一件事是反思面试。你喜欢这家公司和这里的人吗？如果你有一种非常消极的感觉，让公司知道你不再感兴趣了。不是所有的工作都是你期望的或者你想要的。如果你不感兴趣也没关系，但是一定要给公司一个合理的理由，并感谢他们的时间。

### 发送电子邮件

发一封电子邮件说你喜欢这次面试，你觉得你能融入这家公司，这总是很好的。但是不要在你离开面试一小时后发这封邮件。等到晚上或者第二天早上再联系。

给公司提一些小建议，加分。在面试中，你可能会谈到他们刚刚创建的一个新网站。或者你可能谈到了他们正在开发的新工具。在你的邮件中，添加一些小的建议，比如设计改进或者他们应该看看的库。不要走得太远；保持最小化，只有当你感觉他们接受外部建议的时候。

### 保持联系

你应该确定一个公司再次与你联系的日期。也许他们告诉你他们会在周末联系你。如果他们在设定的时间内没有联系你，第二天不要拿起电话开始响。也许他们只是需要多一天的时间。在最初的截止日期后几天联系他们是最好的选择。

## 结论

这些是从我作为采访者和被采访者的真实生活经历中得到的一些提示。为面试做好准备是整个过程中最重要的部分。如果你开始得好，剩下的就会容易得多。

无论是作为面试官还是被面试者，你在面试中有过什么奇怪的时刻吗？你对这个过程有什么高招？我很乐意在下面的评论中听到你的意见。
