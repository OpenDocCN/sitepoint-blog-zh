# 微软的网络标准动机

> 原文:[https://www . site point . com/Microsoft-we b-standards-motivations/](https://www.sitepoint.com/microsofts-web-standards-motivations/)

尽管在 Internet Explorer 中获得对 web 标准的充分支持还有很长的路要走，但随着 Internet Explorer 7 的即将发布，微软*已经开始沿着这条路走下去。结果，像[网络标准项目(WaSP)](http://webstandards.org/) 这样的组织已经改变了他们游说反对微软的传统角色，并开始在这方面与*微软*合作。可以理解的是，开发商们持怀疑态度。*

在最近的一次采访中，WaSP 集团负责人[莫莉·霍尔茨施拉格](http://www.molly.com/)为微软的开发人员辩护，面对微软的商业现实，他们努力开发符合标准的浏览器:

> 如果你想恨微软，请不要认为开发人员本身就是我们的同事，他们正在进行最艰苦的战斗。

这次采访获得了一连串的评论，一些人表示担心 WaSP 最近对微软采取了温和的态度，另一些人则质疑微软到底有多坚持标准。

在他的博客上，克里斯·威尔逊(Internet Explorer 平台团队的小组项目经理)[表达了当他看到这些指控时的挫败感](http://blogs.msdn.com/cwilso/archive/2006/05/11/595536.aspx)。然而，除了发泄之外，Wilson 还花时间解释了在浏览器中有一个符合标准的*商业案例*是多么重要:

> 商业原因经得起时间的考验。纯粹利他的“道德”理由很难在股东面前得到辩护。我个人认为实现标准有一个商业案例，我认为我的工作之一就是在内部提出这个案例。

我必须说，听到——至少目前——微软已经成功地将标准支持的需求与公司的底线协调起来，这是令人欣慰的。不过，如果能听到那个商业案例是什么就好了，因为作为一名 web 开发人员，我希望有机会帮助支持它。我不希望看到微软再次搁置 ie 浏览器，仅仅是因为，例如，开发者没有像预测的那样接受标准的推动。

然而，就目前而言，威尔逊和微软内部的其他 web 标准支持者似乎对 Internet Explorer 的发展方向有很大的发言权，像我们这些开发者一样，他们中的许多人认为这是 Internet Explorer 的最后机会。然而，与我们其他人不同的是，威尔逊把自己的职业生涯押在了这个机会上:

> 是的，我有改变的力量。是的，我会继续改进 IE 的标准支持和合规性，让网络变得更好。这是我的工作、我的章程、我的愿景和我的激情。如果不是这样，我就辞职。IE 中基于标准的平台的开发再次被搁置的那一天，我会退出。我的管理层，包括比尔·盖茨在内，都说我们已经重新掌控了 IE，所以我有工作要做。
