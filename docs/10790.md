# iPhone 4 上移动 Safari 中清晰的图像

> 原文：<https://www.sitepoint.com/razor-sharp-images-in-mobile-safari-on-iphone-4/>

最新一代智能手机显示屏的像素密度比台式机显示屏高得多，比 iPhone 4 引以为豪的视网膜显示屏高出一倍多。这种差异现在如此之大，以至于移动浏览器被迫开始自动缩放内容，通常将 CSS 维度`1px`呈现为两个物理屏幕像素，以便将 web 内容保持在可读的大小。

只要你的网站的桌面渲染是优先的，并且桌面显示分辨率继续落后于智能手机，这种自动缩放将做得很好；然而，当客户越来越习惯于通过使用这些高分辨率显示器的每一个像素来实现锐利的细节时，相比之下，网站图形的缩放版本会显得越来越笨重。

![](img/dc49356285dbbda451b4c25bce03445b.png)

为了让你的网站在高分辨率显示器上看起来像在家里一样，你需要创建的图片的像素尺寸是桌面版图片的两倍；例如，一个 32×32 像素的图标在你的网站的桌面版本上必须被重绘为 64×64 像素才能在手机上以同样的尺寸显示(但是分辨率是原来的两倍)。您只需在 CSS 中将图像尺寸调整为`32px`乘`32px`，并吸收细节:

```
.icon {
  background-image: url(icon64x64.png);
  width: 32px;
  height: 32px;
}
```

但是等等:一个分辨率是两倍的图像将会是文件大小的四倍，桌面浏览器将不得不花费大量的精力来调整高分辨率图像的大小以显示一半的大小。尽管涉及到额外的工作，但最好准备好网站图像的低分辨率(比如 32×32 像素)和高分辨率(64×64 像素)版本，并使用 **CSS 媒体查询**将高分辨率图像仅发送到自动缩放浏览器:

```
.icon {
  background-image: url(icon32x32.png);
  width: 32px;
  height: 32px;
}

@media only screen and
    (-webkit-min-device-pixel-ratio: 2) {
  .icon {
    background-image: url(icon64x64.png);
  }
}
```

在`<img>`标签中加载图像的高分辨率版本更加棘手，因为您无法用 CSS 控制`src`属性。狡猾的开发人员想出了各种方法来实现它(例如，请求一个高分辨率的 CSS 图像设置一个 cookie，然后让服务器也发送高分辨率的内容图像)。

高分辨率图像可能看起来像是很多额外的工作，但是有一天你看到你的整个网站以两倍于你习惯的分辨率显示，这一切都是值得的。
