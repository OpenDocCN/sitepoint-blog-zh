# Joost 放弃了桌面

> 原文：<https://www.sitepoint.com/joost-giving-up-on-the-desktop/>

据 GigaOm 报道，基于 P2P 的电视服务 [Joost](http://www.joost.com/) 正计划放弃其桌面客户端，转而采用完全基于网络的模式。Om Malik 报道说，Joost 将很快宣布放弃其可下载的客户端软件，取而代之的是推出一个“嵌入浏览器的小插件，允许[用户]使用 P2P 技术获取文件。”

马利克说，原因是该公司的客户端已经失去了吸引力，因为网络世界越来越不接受下载和安装客户端软件的需求。

开发基于网络的 Joost 版本一直是该公司粉丝的愿望。大约一年前，独立 Flash 开发者 Paul Yanez [为许多离线视频播放器创建了基于 Flash 的版本](http://www.readwriteweb.com/archives/the_web_media_player_framework.php)，以及在线视频网站的光滑界面。他的 [Joost 模型](http://www.paulyanez.com/labs/joostbeta1)可能是在线 Joost 玩家的预演。

有迹象表明 Joost 已经计划这个转变有一段时间了。例如，在 5 月，Joost】发布了一个“插件开发人员”的招聘广告，该职位需要“包括 FLASH 在内的脚本应用经验”六月，Joost 首席执行官[迈克·沃尔皮承认](http://www.alleyinsider.com/2008/6/joost_ceo_we_ve_figured_it_out_this_time)Joost 有“一个不太正确的产品”

不过，对于 Joost 来说，这种在线转变可能太少了，也太迟了。优质内容已经可以在越来越多的重量级网站上获得，如免费的 Hulu 和收费的亚马逊新流媒体服务，而免费的用户生成内容仍由 YouTube 主导。

我们也不确定完全抛弃桌面是否是正确的选择。拥有一个基于网络的组件对于 Joost 来说可能是一个聪明且必要的举措(或者几乎所有的应用程序，尤其是那些严重依赖网络的应用程序)，但是我们也谈到了桌面对于富互联网应用程序的重要性。

“如果 RIA 是网络的未来，甚至是计算的未来，那么离线/桌面访问是至关重要的。浏览器不是一个非常好的操作系统，所以能够在桌面上运行应用程序是向 Web 3.0 和瘦客户端计算的未来迈出的重要过渡步骤，”我们在 7 月份写道。当然，谷歌本周发布的 Chrome 浏览器可能会改变这一点，该浏览器是为网络应用程序设计的操作系统。
