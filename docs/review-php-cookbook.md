# 回顾——PHP 食谱

> 原文：<https://www.sitepoint.com/review-php-cookbook/>

两年多前，我开始学习 PHP。第一年之后，我开始遇到一些真正有趣的问题，比如在 MySQL 数据库中搜索两个日期之间的记录、结果分页、使用 XML-RPC 在两个站点之间交换数据、信用卡信息的双向加密以及管理用户会话。这些大部分花了我大半年的时间才掌握。如果当时我只有 PHP 食谱，我可以利用这段时间来提高效率！

我最近需要更新我网站的系统。我不得不更换我的用户登录系统，我想增加一个新的加密级别。我以前从未使用过 Blowfish 加密，我想要一个双向方案，这样当我的用户忘记密码时，我可以恢复密码。我能够使用 PHP 食谱中的食谱来节省自己相当多的时间，这些食谱正好涵盖了我所需要的内容。

##### 谁应该读这本书？

PHP 食谱应该在每个 PHP 程序员的书架上。这是一个优秀的桌面参考，提供了常见问题的快速解决方案。

任何喜欢从头开始学习，直接进入代码示例的人都可以很好地掌握 PHP 指南。如果你读过奥赖利的“烹饪书”系列的其他书，你会对这种格式很熟悉:问题、解决方案、讨论。

##### 你会在这本书里发现什么

PHP 指南被分成易于参考的章节。其中的每一个都包含了给你实际解决问题的方法，作为一个 PHP 程序员，你每天都要面对这些问题。

食谱切中要点。他们描述问题；给你看解决方案；并讨论如何以及为什么这样做。以下是这本书的 21 个章节:

1.  用线串
2.  数字
3.  日期和时间
4.  数组
5.  变量
6.  功能
7.  类别和对象
8.  网络基础
9.  形式
10.  数据库访问
11.  网络自动化
12.  可扩展标记语言
13.  正则表达式
14.  加密和安全性
15.  制图法
16.  国际化和本地化
17.  互联网服务
18.  文件
19.  目录
20.  客户端 PHP
21.  梨树

如你所见，不缺话题！21 章中的每一章都被分成 8 到 27 节。这本书涵盖了大量的基础知识——例如，前几章向你介绍了字符串、数字、数组、变量和函数。后面的章节深入探讨了数据库访问、加密和安全、正则表达式和服务。在这一过程中，作者设法为你可能需要答案的几乎任何问题提供了实际例子。

要了解这本书的格式，你可以在网上查阅。奥赖利在他们的网站上发布了第八章。

##### 你找不到的

你在这本书里找不到可爱的轶事或诙谐的妙语。你不会发现充满理论的无尽章节，也不会发现对标准的福音式抨击。这本书没有浪费时间直接进入代码。

“Cookbook”系列已经设法保持事情简明扼要，并省略了作者的个人故事，PHP Cookbook 保持了这一哲学。

##### 好的…和坏的

PHP 烹饪书设法涵盖了我所希望的每一个主题。但是这不是一本新手可以期望从书中学习 PHP 语言的书。相反，它旨在帮助您掌握将极大地改进您的代码的提示和技巧。

这本书的某些部分没有达到我所期望的细节，而其他部分则冗长乏味。幸运的是，这些案例是例外——而不是普遍现象。像“烹饪书”系列中的其他书籍一样，你会发现从奥赖利出版的各种书籍中收集的食谱，所以你可能以前见过其中的一些。总而言之，好处大于坏处。

##### 群众演员！

我喜欢通过例子来学习，PHP 食谱中充满了这样的例子。它的唯一目的是提供您可以立即使用的代码，这将节省您宝贵的时间，并让您专注于您的其他项目。为了让事情变得更简单，奥赖利已经将书中所有食谱的代码通过奥赖利网站下载。你也会喜欢这本书非常详尽的索引，它提供了一种快速简单的方法来查找书中涉及的所有主题。

##### 结论

我不知道没有这本书我是怎么过的。我对 PHP 食谱非常满意，现在我发现自己每天都在使用它。也就是说，它并不适合所有人。如果你喜欢冗长的解释而不是代码示例，这本书可能不适合你。但是如果你是那些喜欢例子的人之一，这本书就是你一直在寻找的！
