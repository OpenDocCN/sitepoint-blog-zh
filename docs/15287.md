##### ASP.NET

虽然在撰写本文时仍处于 Beta 2 阶段，但微软的新 Web 开发框架已经到了特性稳定的阶段；也就是说，Beta 2 和 ASP.NET 最终版之间唯一的变化(以及。它所基于的. NET 框架)将是错误修复。出于这个原因，这个新平台上的书籍已经上架了。T3】

![Sams Teach Yourself ASP.NET in 21 Days](../Images/b37f5f8c303657e8ad8175c74fc35205.png)**[21 天自学 ASP.NET](https://amazon.com/exec/obidos/ASIN/0672321688/webmasterresou05/)【SAMS】**

作为辅导式书籍的大师，SAMS 并没有对这个头衔失望。这种材料通过一系列 21 个项目呈现的实用方式，每个项目都设计为在不到一天的时间内完成(有经验的开发人员应该能够在一两个小时的扎实阅读中完成每一章)，将吸引“动手型”类型的人，他们对尽可能给你提供更多信息的参考型书籍感到失望，希望你能够自己将它付诸实践。

尽管 ASP.NET 允许开发人员在任何。NET 框架支持的语言(VB。NET、C#和 JScript。NET)，这本书主要关注 VB.NET，许多现有的 ASP 开发人员会选择它，因为它与 VBScript 相似，VBScript 是 ASP 3.0 脚本最流行的语言。课程的第四天会简单介绍 c#；然而，寻找 c# ASP.NET 开发完整指南的读者可能会更喜欢 Wrox 出版社的新书《从 C #开始的 ASP.NET》，不幸的是在撰写本文时书店里还没有这本书。

现有的 ASP 开发者也会在*21 天自学 ASP.NET*中找到很多喜欢的东西。每一天都以一个标题为“那不是 ASP！”这涵盖了 ASP 以前的版本和那天所涉及的 ASP.NET 材料之间的差异。有经验的 ASP 用户可能希望在开始阅读各章之前，先阅读每章末尾的这一部分，以激起他们对 ASP.NET 提供的强大新功能的兴趣。

![Wrox Beginining ASP.NET using VB.NET](../Images/0e28e3d0270d591e15d805d7961704d4.png)**【ASP.NET】始于 VB.NET(Wrox 出版社)**

Wrox 出版社实际上提供了两本介绍 ASP.NET 的书；理论上，你的选择应该取决于哪个。NET 语言，你更喜欢使用的网页开发，VB.NET 或 C#。由于绝大多数 ASP.NET 开发人员至少在一开始会从 ASP(VBScript 是首选语言)迁移，从 ASP.NET 开始用 VB.NET 开始可能是更受欢迎的选择。有一些 Java(或者更好的是 JSP)经验的开发人员可能会更适应用 C# 开始 ASP.NET 的*，这在语法上更接近 Java。*

*从 VB.NET 开始 ASP.NET*涵盖的 ASP.NET 框架的核心部分比 SAMS 提供的要少，省略了诸如安全性和移动服务器控制的 WAP 开发等高级主题。随着这种对语言核心特性的额外关注，在技术方面有了更多的深度，尽管还不足以使这本书成为技术参考。然而，有技术头脑的 ASP.NET 入门读者可能更喜欢这本书提供的平衡。

这个标题和 SAMS 的主要区别是结构。*在 21 天内自学 ASP.NET*将材料组织成一整天的课程，以提供结构化的学习路径，*从 ASP.NET 开始与 VB.NET*更紧密地坚持基于特征的分解，其中每章涵盖 ASP.NET 框架的特定特征。因此，一旦完成了提供介绍性教程的任务，这本书作为案头参考将会有更大的用处。

不幸的是，对于 ASP.NET 开发者来说，本应是理想的介绍性文本，却被我发现的相当薄弱的介绍性章节所削弱。诸如生成表单的基本服务器控件之类的基本概念没有被足够详细地涵盖，并且倾向于受到低于标准的写作水平的影响。如果你乐于浏览这些章节，只是对材料有部分的理解，你应该能够通过检查后面章节中的例子来获得全部的理解；然而，这种薄弱的基础可能会让没有经验的读者感到沮丧。

![Wrox Professional ASP.NET](../Images/cc37e88d18f3ee9cc1fd6eafb4328dc8.png) ** [职业 ASP.NET](https://amazon.com/exec/obidos/ASIN/1861004885/webmasterresou05/)(Wrox 出版社)**

跟随 *Professional ASP 3.0* (见我的评论[这里](https://www.webmasterbase.com/article/419))的脚步，*Professional ASP.NET*为微软新的 Web 应用开发框架提供了最深入、最技术细节的参考。根据你是哪种开发人员，这可能意味着你今天会想出去买这本书，或者它可能会让你尽可能地远离它！

不熟悉服务器端编程的开发人员，或者对编程不太熟悉的开发人员应该去别处看看。这本书不是介绍 ASP.NET 基本知识的指南。虽然这本书的章节标题与上面 SAMS 的书非常相似，但这本书的作者采取了完全不同的态度。这本书没有让读者尽可能容易地学习基础知识，而是深入研究了 ASP.NET 涵盖的每个功能的技术细节，并将应用留给您来解决。

就像在专业 ASP 3.0 中一样，这本书的附录可能非常少。这样做的原因有两个:首先，这本书本身非常容易参考，所以提供附录可能被视为多余。其次，Wrox 刚刚宣布了一个新的标题，【ASP.NET 程序员参考*，这无疑将满足按字母顺序参考所有 ASP.NET 事物的需求。*

##### 服务器端编程语言（Professional Hypertext Preprocessor 的缩写）

![Build Your Own Database Driven Website using PHP & MySQL](../Images/22468e76d104fa3a7da2cc00683157ec.png) **[使用 PHP & MySQL ](https://www.sitepoint.com/books/?bookid=b8458a) (SitePoint)** 建立自己的数据库驱动网站

当然，我认为我自己的书是这方面最好的书！否则我就不会写了，不是吗？

不过，说真的，这本书确实提供了用 PHP 和 MySQL 构建数据库驱动网站的最佳指南。它假设没有预先的编程知识，并指导您完全理解 PHP 和 MySQL，在您从头开始构建数据库驱动的网站时“在运行中”学习这两种令人兴奋的技术。不相信我？在线免费查看[前四章](https://www.webmasterbase.com/article/228)，敬请期待第三方对 SitePoint.com 更公正的评论。

![Apress A Programmer's Introduction to PHP 4.0](../Images/9618a0b6b384508562554ef11d229627.png) ** [一个程序员的 PHP 4.0 入门](https://amazon.com/exec/obidos/ASIN/1893115852/webmasterresou05/) (Apress)**

这本书设计精美，非常容易阅读，是一个惊喜。老实说，我不认为有任何关于 PHP 语言的好的、最新的书籍。尽管它依赖于对编程基本概念(语句、变量、if 语句等)的熟悉。)，这本书很好地展示了 PHP 语言的基础知识。然而，非程序员可能会感到有点落后；如题所示，作者并没有浪费太多时间教你编程的基础。

本书中的主题是按照逻辑组织的，为您提供您需要的信息。例如，字符串操作函数与正则表达式包含在同一个章节中，并作为“正则表达式的替代物”出现，这是许多经验丰富的 PHP 开发人员都会赞同的描述。更高级的主题，如数据库交互、会话跟踪、XML 和安全性，都有足够的细节供有经验的开发人员学习基础知识并自己学习其余的内容，但是初学者可能很难在没有额外指导的情况下将这些诱人的内容组合在一起，以创建一个活生生的 Web 应用程序。

要知道，这本书并没有为 PHP 手册的方便函数参考提供替代品，也没有假装提供。这本书没有参考附录，作者从一开始就明确指出，这本书的作用是简化学习过程，而不是反刍在网上参考资料中很容易找到的内容。

![Professional PHP Programming](../Images/eadab9f6721b5142d262ad311652fd8c.png) ** [专业 PHP 编程](https://amazon.com/exec/obidos/ASIN/1861002963/webmasterresou05/) (Wrox 出版社)**

虽然从某些标准来看已经过时了，但对于刚开始接触 PHP 的非程序员来说，Wrox 的原始 PHP 参考仍然是我的选择。它缺少对 PHP4 特性的介绍，比如会话、heredoc 语法和 Perl 兼容的正则表达式，但是这些主题中的大部分可以在以后通过一两篇在线教程来学习。更重要的是，由于这是一本老书，如果你花一点时间货比三家，你很有可能在一本二手书上买到便宜货。

关于我对这个标题的完整评论，请看[这篇文章](https://www.webmasterbase.com/article/221)。

##### 服务器端 Java

![Wrox Professional JSP, 2nd Ed.](../Images/cf009278cc24d0fccc0f445c0c4e6883.png) ** [专业 JSP，第 2 版。](https://amazon.com/exec/obidos/ASIN/1861004958/webmasterresou05/) (Wrox 出版社)**

对于 JSP 方面的权威著作，Wrox 更新和大幅扩展的巨著 *Professional JSP，第 2 版不会错太多。*这是不少于 18 位作者共同努力的结果(他们的面部照片都出现在封面上，采用传统的 Wrox 风格)。除了核心 JSP 概念之外，我所能找到的其他任何一本书都没有提供如此完整的处理方法，包括使用定制标记扩展 JSP 的技术、良好的 JSP 编程实践、常见的 JSP 开发方法以及围绕 JSP 的主要技术。

然而，这种完整性伴随着巨大的代价，那些只是想尝试一下或者学习 JSP 基础知识的开发人员很容易被这本书的数量所淹没。一些读者可能还会对由这么多不同的人写的书总是带有的相当不带个人色彩的语气感到反感。

也就是说，很难对这些页面中发现的材料的深度提出异议。专业 JSP，第二版。正儿八经的 JSP 开发者梦想成真了。

![Wrox Beginning JSP Web Development](../Images/d3f93538ba0dee1c6574667c6b41ddcb.png) ** [开始 JSP 网页开发](https://amazon.com/exec/obidos/ASIN/1861002092/webmasterresou05/) (Wrox 出版社)**

虽然*专业 JSP，第 2 版。*涵盖了关于 JSP 的几乎所有知识以及更多内容，在那里可以找到的大量知识的展示根本不适合没有经验的开发人员。如果您计划将它作为 JSP 世界的入门，那么您最好是一名*非常有能力的 Java 程序员，能够很好地掌握 Web 应用程序编程概念，并且喜欢通过阅读技术上详细的参考资料来学习新技术。*

对于寻求开始使用 JSP 的典型 Web 开发人员来说，*开始 JSP Web 开发*是一个更好的选择。事实上，这是我对介绍性 JSP 文本的最佳推荐。这本书对 JSP 的工作提供了一个友好的介绍，没有遗漏细节，用一致的、面向任务的方法处理了该语言的所有核心方面，以及一些高级主题，如标记库、数据库交互和电子邮件。

作为 Wrox 的两个标题之间的差异的例子，我想引用每个标题中如何讨论用户会话。在*专业 JSP，第 2 版。关于什么是用户会话的知识几乎是预先假定的，并且书中没有一个章节涵盖如何利用 JSP 中的会话管理。在页面中启用会话并利用它们的方法在第 4 章(“基本 JSP”)中有所介绍，但是初学者在寻找会话使用的良好介绍时，甚至不会在目录中找到它们！*开始 JSP Web 开发*，同时，认为这是一个高级的话题，并且将第 11 章(“跟踪用户”)的全部内容用于讨论。文中详细解释了相关的概念，并通过几个实例说明了会话是 JSP 不可或缺的一部分，不应该单独考虑，而应该与该语言的其他功能一起使用。*

简而言之，*开始 JSP Web 开发*对于希望从头开始学习 JSP 的开发人员来说是一篇极好的介绍性文本，而有经验的 JSP 开发人员将会找到更大量的参考资料，即*专业 JSP，第二版。*不可或缺的案头参考。

![O'Reilly JavaServer Pages](../Images/fce5116882881c95fb7eb383c4067e0b.png) ** [JavaServer Pages](https://amazon.com/exec/obidos/ASIN/156592746X/webmasterresou05/) (奥赖利)**

与 Wrox 提供的大量内容相比(尽管对于刚开始 JSP Web 开发的*)O ' Reilly 的 *JavaServer Pages* 看起来确实很轻松。毫无疑问，这本书没有接近 Wrox 的*专业 JSP 第二版的范围。*，但是它所涵盖的内容 JavaServer Pages 的核心方面、它们与 Java Servlets 的关系，以及使用定制标记库扩展 JSP 的基础知识——都得到了清楚的解释，并提供了恰当的支持示例来完善每个主题。*

 *渴望开始使用 JSP 的开发人员，以及不希望被过多的补充技术材料或大量不必要的手把手的东西淹没的开发人员，将会在这本相对较薄的书中找到很多喜欢的东西。当然，它不会教你企业 JavaBeans (EJB)的基本知识，但是有关于这个主题的整本书。如果你是一个需要快速学习 JSP 的有经验的开发人员，O'Reilly 的 *JavaServer Pages* 将让你立刻编写 JSP 页面。

##### 冷聚变

![Sybex Mastering ColdFusion 5](../Images/f805d7f221b46373ae9881b6013d94a1.png) ** [掌握 ColdFusion 5 ](https://amazon.com/exec/obidos/ASIN/078212979X/webmasterresou05/) (Sybex)**

这本大部头的书(如果掉在地上很容易摔断脚趾)可能看起来是一本令人印象深刻的书，这可能会吓跑初学者。更懂技术的读者可能会认为这是对细节和报道深度的极大关注。最后，那些需要一份可以放在笔记本电脑包里随身携带的推荐信的人也可能会被它的庞大体积吓跑。然而，经过仔细观察，这三个假设都被证明是错误的。

在我看过的所有书籍中， *Mastering ColdFusion 5* 提供了最适合新手的演示和组织。本书提供了大量演示如何完成各种任务的手把手的分步指南，将为设计师和其他没有技术背景的读者提供 ColdFusion 的优秀教程介绍。丰富的截屏和使用更大的字体和页边空白使这本可能更薄的书显得更加充实；这是好事还是坏事，由你决定。

不幸的是，对于那些可能被这本书的重量所吸引的高级读者来说，它的大小掩盖了里面所包含的高级材料的相对缺乏。在本书的最后几章中，高级主题往往只是一带而过。

这本书的附录是吸引人的介绍，以便于参考和一般完整，但两列格式可能会惹恼一些读者。这本书的篇幅可能会妨碍它作为一个方便的参考，但封面光盘附带的可搜索 PDF 版本极大地增加了这本书的便携性。如果你是在旅途中从事 CF 开发的，那么在你的笔记本电脑上以电子格式提供这份完整的 ColdFusion 指南可能就是这本书的卖点。

![O'Reilly Programming ColdFusion](../Images/57421ee1d54103296dbc5e387ed242ce.png) ** [编程 ColdFusion](https://amazon.com/exec/obidos/ASIN/1565926986/webmasterresou05/) (奥莱利)**

O'Reilly 总是给我留下深刻的印象，它能够比竞争对手将一个主题的更完整的覆盖面打包到一个更小的包中。我想说，这是 ColdFusion 编程的主要优势。如果你是一个赶时间的专业人士，并且你需要通过最少的手握清单和带截图注释的分步指南来掌握 ColdFusion 的全面知识，那么 O'Reilly 的严肃方法肯定会很有吸引力。

本书的后半部分讨论了数量惊人的高级主题；然而，作者在这里的报道似乎是为了广度而不是深度。如果读者希望快速了解 ColdFusion 的所有功能，并且不介意在完全理解该语言的模糊功能之前进行一些实验并参考产品文档，这将会令他们感到满意。需要安全、WDDX 和外部对象集成等高级主题的完整指南的读者可能会想看看下面的 Macromedia 出版社提供的内容。

![Macromedia Press Macromedia ColdFusion 5 Web Application Construction Kit](../Images/39f811d8e90669e25f61faec9959f299.png)**[《Macromedia cold fusion 5 Web 应用构建工具包》，第 4 版。](https://amazon.com/exec/obidos/ASIN/0789725843/webmasterresou05/) (Macromedia 出版社)**

ColdFusion 的制作者(本书的主要作者之一是 Macromedia 的 ColdFusion 产品传播者 Ben Forta)提供了一个令人满意的平衡，通过省略 ColdFusion 的真正高级功能并将其放在单独的一卷中(见下文)，在覆盖深度和主题广度之间取得了令人满意的平衡。

如果有什么不同的话，这本书把太多的注意力放在了基础知识上。解释互联网如何工作，有哪些 Web 浏览器可用，以及 ColdFusion 等服务器端语言在大格局中的地位的章节可能会被更高级的读者视为悬而未决的事情(我认为当你拼错一个域名时会发生什么的半页截图有点夸张！).相对无用的附录(例如，只有一个可用标签列表，每个标签旁边都有一行描述)也非常令人失望。

如果你想完整地了解 ColdFusion 5 的基本知识，而没有高级功能或大量参考附录(毕竟，产品文档在这方面相当完整)，这本书绝对是一个不错的选择。事实上，它补充了 Macromedia 的 advanced title，*Advanced Macromedia cold fusion 5 Application Development*而没有重复，这对于需要这两卷集的完整性并且不介意价格冲击的读者来说也是一个很好的卖点(这本书是我所评论的书中最贵的，更不用说 Advanced volume 了，它将使 CF 参考库的最终价格翻一番！).*
