# 采访——糟糕网页的文森特·弗兰德斯

> 原文:[https://www.sitepoint.com/flanders-web-pages-suck/](https://www.sitepoint.com/flanders-web-pages-suck/)

**[网页点击](http://www.webpagesthatsuck.com/home.html)是互联网上最具争议的网站之一。它衍生出的两本书，恰如其分地命名为《】的网页》和《的网页之子》，概述了可用性缺陷、设计禁忌以及如何创建一个可用的网站。**

Vincent Flanders 创建了一些糟糕的网页来支持他的观点，即我们可以“通过观察糟糕的设计来学习好的设计”。但他是谁？他从哪里来？正如我们所知，他对网页设计做了什么？

上周，SitePoint 采访了这位自称“喜欢闪亮东西的营销黄鼠狼”并要求回答…

SitePoint: Vincent，首先，你能介绍一下你的背景吗？你是如何进入可用性领域的？

实际上，我来自讨论可用性的所有背景中最好的和最差的——我来自市场营销。“最好”的一面是——与设计界的误解相反——我对什么是网页设计和可用性的“最好”绝对没有宗教信仰。当谈到网站的设计和可用性时，我所关心的是“我必须做什么才能通过这个网站进行销售或传播信息？”在我的新书《糟糕的网页之子》中，你会看到我是可用性领域中唯一一个承认前沿设计也可以的人，除了“卖出去”之外，没有“真正的”规则。

为了向你表明我不属于艺术家或科学家的崇拜，我在书中讲述了一个牙医花了很多钱在 Flash 网站上的故事。牙医的 Flash 网站！当他给我看的时候，我立刻乐坏了，因为大多数牙医甚至不需要网站，更不用说基于 Flash 的网站了。无论如何，我告诉他，他被设计师骗了，因为网站加载时间太长，不是每个人都用 Flash，没有文本链接——通常的可用性热线。

然后，我问他这个网站给他带来了多少新业务，他告诉我“每周 1 万美元。”我毫不犹豫地回答:“你的设计师是凡人中的神。”诚然，他可能通过 HTML 网站赚了更多的钱——所以我为他想出了完美的解决方案，并在《糟糕的网页之子》中写道——暗示，暗示——但就像我说的，我并不迷恋艺术家或可用性。

现在，就营销的“最糟糕”方面而言，我就像其他营销黄鼠狼一样——我喜欢闪亮的东西。我喜欢闪电侠。我喜欢你在网上看到的所有多余的东西——神秘肉导航——大多数商业网站上不应该有的东西。我喜欢所有的附庸风雅的东西，但只有少数商业类型的网站，你应该使用它。

好吧，我来咬。你应该在哪里使用它们？

主要有两类。你必须使用它们的第一组网站是音乐、电影、乐队、网页设计公司和时尚网站——以及其他诸如此类的网站。任何你必须被认为很酷的网站——这是法律:你必须使用它们。

另一组是任何没有责任的网站。没有人会发现这个网站没有销售产品，也没有吸引人们进入商店。在市场营销中，我们称之为“品牌化”文森特·弗兰德斯对“品牌”的翻译是“我们会花你很多钱把你的名字放在公众面前，但我们会以这样的方式做，你不能证明我们搞砸了。”汽车公司是网络品牌中不负责任的一个很好的例子。除非你能在网上订购汽车，或者让访问者与你或你的经销商互动，否则你真的无法证明这个网站是否有效。

SitePoint:这样定义品牌是不是有点过分了？

当然了。品牌的确有效，但在互联网上并不特别好，正如互联网泡沫的余波所证明的那样。对于每一个雅虎！你有 10，000 个网站。那是我会想念的一家公司。我喜欢他们的组织者产品。

**SitePoint:回到你的背景…**

没错。我当时在一家软件公司负责数据库营销，有一天，我的老板走进我的办公室，指着我说:“我们要开一家 ISP，你是网站管理员。”于是我成了网站管理员——那是 1995 年。

然后在 1996 年，我教 HTML 课程，记得我有多讨厌无聊的老师，我向全班展示了其他人在他们的网站上犯的所有错误。我说“哦。十个动画形象。这真的很糟糕。”我的评论是班上每个人最喜欢的部分，因为它们不无聊，所以，从逻辑上讲，我决定建立一个网站…

**SitePoint:很烂的网页……**

是的，这是一个容易记忆的网址！

SitePoint:那名字呢？

嗯，我是一个营销黄鼠狼，我讨厌一切无聊的东西——就我个人而言，我认为无聊是唯一的罪过——所以我想出了一个吸引眼球的名字，而不是一个合适的名字。不要取笑雅各布(尼尔森)，因为雅各布真的是一个很酷的人，但我想雅各布会称之为“有错误的网页”如果我给它起了个“恰当”的名字，你今天就不会和我说话了。

SitePoint:那么，自从你第一次参与进来，网络(和设计)发生了怎样的变化？你在工作中经历过哪些关键的里程碑或转折点？

我最自豪的是，我是第一个说网页设计与艺术无关，而是与金钱有关的人。早在 1998 年，我为此受到了很多批评，但时间证明我是对的。我也很自豪我说过数据库知识很重要。

也许更好的办法是谈谈自 1998 年 4 月第一本书《糟糕的 网页》出版以来发生了什么变化。

好的，听起来很有逻辑。

第二大积极变化是带宽的增加。不幸的是，没有足够多的人可以访问大管道，所以它没有让设计师利用 500Kb 的页面加载像 50Kb 的页面。

但最大的积极变化是网站设计的整体改善，因为许多图形艺术家从印刷转向了网络。我们有很多有才华的人创建网站，所以网站设计的整体质量有所提高。

另一方面，第二个最大的负面变化是 CSS 没有成为设计网站的事实方式。如果你在 1998 年告诉我，CSS 的实现在 2002 年将会像现在这样不稳定，我会嘲笑你。这整个浏览器/CSS/什么的问题简直愚蠢得令人难以置信。

最大的负面变化是我们真的从网页设计的世界转移到了软件设计的世界。网络实际上是关于软件的——后端。在我看来，网页设计师正在变成内容错位(是的，“错位”)系统的模板制作者。服务器是新的上帝，大多数设计师从建筑师的身份被推下到室内油漆工。软件就是一切。

SitePoint:设计社区倾向于和可用性专家有一种“爱恨交加”的关系。你认为这是什么原因？为什么设计师似乎对可用性的概念和实现如此抵触？

一个好的设计师不喜欢可用性，因为一个好的设计师是艺术家，你成为艺术家不是为了让别人告诉你如何以有效的方式创造东西。如果一个人想要那样的职业，他会成为一名建筑师。不要忽视我——让我完成下一个想法:

可用性的问题是“他们是对的。”当我说“他们是对的”时，我的意思是可用性研究在很大程度上是基于研究的——“在这次测试中，64%的用户在两次点击后离开了网站，如果他们不能理解导航的话。”可用性测试带来了责任——如果有一个词是艺术家讨厌的，那就是“责任”艺术从不负责。莫奈有责任吗？梵高？

这就是为什么你可以在音乐、乐队和时尚网站上使用 Flash、神秘肉导航和任何其他面向不可用性的网页设计技术。对于大多数商业和非盈利网站，有一些方法来确定网站是否“工作”

有一个巨大的 mofo 银行想赢得设计奖并赚钱，所以他们建立了一个 Flash 网站。他们赢得了奖项，但没有人使用他们的网站——看，他们可以跟踪结果——所以他们转向了一个“普通”的网站。然后，银行开始赚钱。商业网站的目标是赚钱或传播信息，非营利组织仍然是“商业的”，因为他们试图向你推销一种信仰体系。“最好”的方法是确保你的网站能被尽可能多的人使用。

艺术家讨厌研究，因为艺术与研究无关。它是关于表达一个人的创造性情感。网络——当我说“网络”时，我指的是商业网络——是艺术的对立面。艺术、个人和实验网站不是商业的，没有边界。

设计师不喜欢可用性，因为让一个网站可用很无聊，设计师讨厌无聊——这是我由衷认同的事实。事实是，商业网页设计已经变得非常非常无聊。令人兴奋的东西都是实验性的——Flash、音乐、软件设计。

SitePoint:设计师最常忽略可用性的哪些方面？反过来说，他们在采用可用性和易访问性指南方面做得好在哪？

我不确定到底是谁该受到指责。是客户，项目经理，还是设计师？很难说。但我看到的最大错误与导航有关，我怀疑设计师与弄清楚导航方案没有太大关系，除非是一个人或两个人的商店。

设计师们仍然把他们的页面做得太大，他们在错误的地方使用 Flash，使用 splash 页面……这是设计师们永远都会犯的错误。

最大的错误是人们设计网站时没有考虑他们的受众。为什么？他们没有足够的信息。我在写《糟糕的网页之子》时发现的最有趣/最令人不安的事实之一是，使用网络电视的人——不管它叫什么——占一个网站付费用户的 2.4%。我说的不是访客。我说的是付费客户。我看了日志文件。这家公司每天收到 150 份来自网络电视用户的订单。他们的网页设计师必须确保他们的网站可供网络电视用户使用，否则他们每天会损失 10，000 美元。

我怀疑大多数设计师不知道他们的受众是谁，也不知道他们使用什么样的浏览器/系统来访问网站。虽然访客的统计数据很重要，但找出付费客户的统计数据更重要。

我不确定设计师是否已经“出类拔萃”，但至少他们关心这个问题，这是一个好现象。感谢互联网带来的负面影响；否则，我怀疑它会像往常一样设计——或者“不寻常”

SitePoint:你如何反驳可用性是一种将少数人的体验置于大多数人之上的哲学？设计师如何确保在满足那些访问网络内容有困难的用户的需求的同时，不会破坏那些完全没有困难的目标受众的用户体验？

实际上，这很容易反驳。这是个错误的问题。你混淆了可用性和易访问性。可用性与强加少数人的观点无关。可用性就是满足大多数人的需求。

SitePoint:那好吧。你如何反驳无障碍是一种将少数人的体验置于大多数人之上的哲学？

我不确定我能反驳它。我的问题是“如果疏远了这群人，你会损失多少钱？”我与迈克尔·威利斯(Michael Willis)共进午餐，他与我合著了第一本书，我责备他没有在自己的主页上使用 ALT=属性，这样更容易访问。他说，“盲人不会来我的网站——我是一名平面艺术家和网页设计师。”我不能和这样的逻辑争论。

从新书中删除的一个评论是我说的，“智能网站设计公司将“说服”他们的客户，他们需要为 1)网站的 Flash 版本 2)网站的高端图形版本 3)网站的低端文本/图形版本 4)网站的无障碍版本付费。四个不同的网站，五个的价格。<g>。</g>

看看政府会怎么做将会很有趣。他们是可访问性的最大支持者。看到他们说，“任何与我们做生意的人都必须有一个他们网站的无障碍版本”，我不会感到惊讶。这肯定会帮助设计公司赚更多的钱。

SitePoint:颜色对用户体验有多重要？你会如何着手开发一个网站的配色方案？网络安全调色板代表了颜色的界限吗？或者，随着支持图形的计算机的出现，我们现在可以安全地超越这个范围吗？

配色方案非常重要，但一个大问题是一个网站的配色方案可能在五十年前就已经确定了。IBM 被称为“蓝色巨人”，他们不得不在配色方案中使用“蓝色巨人”。在我有生之年，他们不会穿橙色/棕色的衣服。很多公司都是这样。配色方案已经确定了。

我对“我们是否已经超越了 256 色？“是的，”除非一家大公司的某人最近告诉我，“我们所有的采购代理都在配有廉价显卡的破旧电脑上工作。“再说一次，你必须了解你的市场。就像我在《狗娘养的》里一直说的，“看情况。“看看你的日志文件。没有人使用网络电视访问我的网站。我不考虑他们。我个人不喜欢色彩限制和屏幕限制。

SitePoint:富媒体(以及 DHTML、CSS、Javascript 和 Flash 等相关技术的使用)在塑造用户体验方面有什么样的作用？这些技术曾经被视为可用性禁忌而被回避。发生了什么改变了这一点？

我不确定我是否同意它们现在是可用性的肯定答案。似乎 12-16%的网络用户关闭了 Javascript2-4%的人不使用闪存；除非你参与一些非常花哨的编码，否则 CSS 是一个残骸，因为不兼容的浏览器情况——阅读:“网景 4”有 10-15%的市场份额，各种各样的黑客攻击是必要的；DuHTML——我喜欢这么称呼它——也有浏览器合规问题。再一次，这取决于你的观众。你会疏远多少人，你在乎吗？是的，这些技巧有它们的位置，但我是作为一个喜欢闪亮东西的营销人员来说的。

你说网页设计不是艺术，而是赚钱。你能详细说明一下吗？你觉得这适用于一般的设计，还是仅仅适用于网页设计的商业应用？

我只谈以销售产品或传播信息为目的的网站。为什么？因为写关于个人、艺术、实验场所的东西是没钱的。为什么？因为这本书只会包含这句话:“如果你正在创建一个个人的、艺术的或实验性的网站，做你想做的任何事情。上帝速度。”

你已经写了两本关于可用性的书。他们针对的是设计师还是网商？你能为我们描述一下他们的关键信息吗？

正如你可能已经收集到的，我喜欢最大化潜在的受众，所以每个人都有自己的东西。至少，设计公司可以买下这本书，把它贴在他们门厅的咖啡桌上，并附上一张纸条“我们设计的网站不会出现在这本书里”——当然，除非他们出现了<g>。</g>

关键的信息总是，“了解你的观众”和“知道他们对痛苦的容忍度。”我试着告诉你如何让你的网站对你的观众更有效。就像我说的，我不在乎一个网站是否可用。我只是关心你的网站让人们购买你卖的任何东西——一个产品或者一个信仰体系。可用性是一种工具。它似乎经常工作。

SitePoint:你认为未来几年网页设计和可用性的主要发展是什么？

一个词的答案是“肥皂”查一下。这是未来。我们正走向万维网博客。我们的目标是在没有任何 Lotus Notes 软件的情况下，将世界变成 Lotus Notes for Web。由于节省了成本，公司也将以真正透明的方式与其他公司共享数据。重要的是后端。

真正困扰我的是——我在之前的谈话中暗示过——网页设计变得几乎无关紧要了。现在都是关于编程的，这对我很好，因为我是一个作者，几乎所有的软件文档都很糟糕。我建了一个叫 documentation 的网站，很烂，但是我们都知道几乎所有的文档都很烂。尤其是开源和伪开源软件。如果你按照作者提供的说明去做，你一般是无法安装这个软件的。

我希望未来的软件将会有更多的可用性。想象一下，如果所有的博客软件都符合 W3C 标准会是什么样？只需点击“新主题”,你的整个网站就会变成 XHTML 兼容和可访问性。设计师将变得更像一个“网站概念化者”他/她会提出这个概念，速记池图形设计师——因为这是他们将要发生的事情——将创建初始模板和艺术，服务器的程序员将确保所有图形都针对每个观众优化，只针对他们正在使用的浏览器。

从此以后我们都会幸福地生活。
