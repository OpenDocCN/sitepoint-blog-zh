# 采访——PHP——GTK 的 Andrei Zmievski

> 原文:[https://www.sitepoint.com/php-gtks-andrei-zmievski/](https://www.sitepoint.com/php-gtks-andrei-zmievski/)

安德烈·兹米耶夫斯基是一个多才多艺的人。尽管他最为人所知的是他对 [PHP-GTK](http://gtk.php.net/) 和 [Smarty 模板引擎](http://www.phpinsider.com/php/code/Smarty/)项目的贡献，但他的兴趣范围远至历史和语言学。PHP 社区很幸运能让他关注这么久！

在这次独家采访中，Andrei 告诉我们是什么吸引他使用 PHP，以及他对未来有什么计划。

##### 开始

首先，你能简单介绍一下你的背景吗？最初是什么让你进入软件开发的？T3】

阿兹:我来自乌兹别克斯坦，在过去的 10 年里我一直生活在美国。我在 9 岁或 10 岁时开始编程，首先是在可编程计算器上，然后是在更大更好的机器上。我必须感谢我的父母认可并支持我在这方面的兴趣，给我买书、杂志，甚至电脑。

***SP:你第一次接触 PHP 是什么时候，是什么“让你上瘾”？**T3】*

AZ: PHP 第一次出现在我的视野是在 1997 年。在工作中，我们购买了 Apache StrongHold 包，它附带了 PHP 2 和一些文档。最初，这似乎只是一个简单的好奇，因为我们的系统都是用不同的、专有的第三方语言编写的。但是在 1998 年初，我们开始将我们所有的产品切换到 PHP，因为它被证明对具有 C 和 Perl 背景的人来说更熟悉，更灵活，有来自社区的很好的支持，但是最重要的是，它是免费的。就我个人而言，我喜欢它附带了一个相当好且易于使用的 API。这些原因至今仍是 PHP 的优势。

##### 在 PHP 上

***SP:你还在积极参与 PHP 项目吗？**T3】*

是的，尽管我最近休息了一段时间。三年后，我需要从邮件列表信息、bug 和其他类似的事情中稍微休息一下。我仍然没有订阅这些列表，而是通过网络界面来监控它们。对我来说，这是一个过渡期，但是我希望在夏天结束的时候回到我的位置上。

SP:你对 PHP 最近的变化有什么看法，比如 4.2.0 中的 register_globals off，以及 4.1.0 中新的预定义变量？T3】

阿兹:我认为这是朝着正确方向迈出的一步。这些变化同时使 PHP 更加一致和安全。

***SP:你觉得 PHP 已经成熟了吗？您认为未来还会有哪些重大的安全性增强？**T3】*

AZ:在这个领域，如何定义成熟？是用该语言编写的程序的数量吗？可供它使用的库的数量？它有多少文档和教程？如果是这样的话，那么 PHP 就轻松过关了。

就安全性而言，最重要的事情是彻底改革作为基本沙盒机制的安全模式。

***SP:PHP 在微软的光芒下站在哪里？网？**T3】*

阿兹:。NET 是一个相当有趣的技术，我特别喜欢它的类库。然而，我看不出它能给 PHP 带来什么特别的好处。举个例子。NET 还不能很好地处理松散类型的语言。然后就是 PHP 所依赖的所有那些扩展的问题。让 PHP 成为成熟的。网络语言需要投入大量的时间和精力，但潜在的回报却很少。

也就是说，已经有一个 dotnet 扩展可以让你实例化。NET 对象并在其上调用方法(目前只在 Win32 平台上)。

##### 自作聪明

***SP:关于编码逻辑与内容的 Smarty 和 general 分离，你觉得 PHP 有让 PHP 编码员和网页设计师都满意的解决方案吗？**T3】*

阿兹:是的。

史迪芬:你还想要什么，或者你打算用聪明来做什么？T3】

AZ:让它更模块化，更轻便，这样每个阶段只装载必要的部件。此外，我想返工的 API 和语言，以更加一致，更容易使用，但我们必须意识到向后兼容性。当然，文档可能需要改进，但是我觉得这个任务最好留给聪明的用户社区。

SP:PHP 的一个缺点是它包含了大量不同的编程方法，每种方法都有自己的开源项目支持(例如，Smarty 与 FastTemplate，请原谅我们的比较)。你觉得 [Pear 库](http://pear.php.net/)已经准备好为 PHP 提供一个标准的应用框架了吗？有没有你认为应该由 PHP 团队支持的替代项目？

AZ:首先，PHP 团队并不从事为项目或产品背书的业务。

其次，PEAR 在过去的几个月里取得了很大的进步，随着 PHP 4 . 3 . 0 的发布，最终可以透明地下载和安装任何你想要的库。但是 PEAR 不是一个应用程序框架，而是一个组件的集合，其中一些组件服务于相同的目的，可以帮助您快速开发应用程序。

第三，“大量不同的方法”似乎是开源开发的一个固有属性，而不一定是有害的。

SP:除了那些你参与过的，哪些用 PHP 编写的开源项目给你留下了最深刻的印象？T3】

AZ: [图库](http://sourceforge.net/projects/gallery)——在线相册软件。[Swwwing](http://uk.swwwing.com/)——丹麦一家公司制作的群件工具。[JPGraph](http://www.aditus.nu/jpgraph/)——一个图形绘图库。然后还有一个用 PHP 写的 [web 服务器](http://nanoweb.si.kz/) …

##### 安德烈和 PHP-GTK

你是如何成为 PHP-GTK 的首席开发者的？参与这个项目感觉如何，花了多长时间？T3】

阿兹:我想我是唯一一个足够疯狂去研究它的人。我在 2000 年 9 月开始编写初始代码，第一次发布是在 2001 年 3 月 1 日。从事这项工作非常有趣——弄清楚 PHP 是否能够处理与这样一个工具包的绑定，并在这个过程中发现和修复 PHP 中的一些缺陷。我坚信 PHP 是一个更好的项目，因为 PHP-GTK。

SP: PHP-GTK 允许开发者编写跨平台的窗口应用程序，让用户可以直接在他们的桌面上运行。PHP-GTK 能为有 PHP 建站经验的开发者做些什么？T3】

AZ:它给了他们另一种控制后台系统的方法，不受 HTML 和浏览器界面限制的影响。诸如拖放、网格、树形视图等功能。比浏览器中的同类产品功能强大得多，有利于提高工作效率。

***SP:PHP-GTK 如何与 Perl、C、TK 等平起平坐？？**T3】*

AZ:我收到了一些人的电子邮件，他们说他们真的很高兴发现了 PHP-GTK，因为现在他们可以使用他们最喜欢的语言在客户端构建解决方案，否则他们将不得不求助于 Java 或类似的东西。对我来说，这意味着 PHP-GTK 填补了一个急需的空白。

你如何描述 PHP-GTK 项目的现状？您希望添加什么？T3】

AZ:目前还没有做太多的开发，主要是错误修正。我正在等待我的时间表变得明朗一点，这样我就可以从 PHP-GTK 2 开始，它将支持 Gtk+ 2，并依赖于新版本的 Zend Engine，它修复了许多烦恼。

但是文档仍然是一项正在进行的工作。大约一半的类都有文档，这有点慢，可能是因为正确地编写文档需要理解 PHP、C 和 Gtk+。希望会有更多的人对帮助写文档感兴趣。

SP:你认为 PHP-GTK 和。NET 客户端框架？你能预见 PHP-GTK 将成为未来 PHP.NET 语言的客户端吗？T3】

阿兹:不。NET 已经包含了自己的 GUI 系统接口，称为 WinForms。在 Windows 上，WinForms 依赖于 GDI 和其他库，在 Linux 上它已经使用 Gtk+(在 [Mono 项目](http://www.go-mono.com/)中)。

SP:有成千上万有经验的 PHP 开发人员，有什么计划鼓励他们使用 PHP-GTK 吗？我们会很快看到 PHP-GTK 竞赛或编码竞赛(类似于 PHP 的[这次](http://codewalkers.com/php-contest.php))吗？

AZ:我希望更多的人了解 PHP-GTK 并开始使用它，但我只是一个人，我宁愿把我的可用时间奉献给从技术上改进 PHP-GTK。此外，我不是一个很好的市场学家。

开源项目有自我组织的社区，希望同样的事情也发生在这个项目上。如果你有任何关于如何鼓励开发者使用 PHP-GTK 的想法，让他们在 php-gtk-general 邮件列表中被听到。

编码比赛？当然可以！

SP:你还开发了 PHP 的 WDDX 扩展。这和 PHP-GTK 有共同的主题吗？有计划在 PHP-GTK 应用程序和支持 PHP 的网站之间建立一个标准接口吗？T3】

AZ:不，扩展并没有太多的共同点，但是我想 WDDX 可以作为客户端 PHP-GTK 程序和服务器端程序之间的数据交换协议。我不想把 PHP-GTK 绑定到任何特定的接口上。开发人员已经意识到需要这样的东西(不仅仅是 PHP-GTK)。 [SRM](https://web.archive.org/web/20090415230033/http://www.vl-srm.net:80/) 和 sockets 扩展可能部分是因为这个需求。

你自己用 PHP-GTK 做了什么？T3】

阿兹:你的意思是除了在会议上谈论它之外？哦，我做了一个小应用程序来管理我在 advogato.org 的日记。大概就是这样。

***SP:如果 SitePoint 的某个成员开发了一个“杀手级应用”，PHP-GTK 有能力把它放到每一个桌面上吗？**T3】*

阿兹:不是字面意思，这是缺点之一。目前，没有简单的方法来分发应用程序——对于 PHP-GTK 应用程序来说，没有什么比运行 setup.exe 或安装 RPM 包更简单的了。尽管我认为西蒙·惠勒在这方面做了一些有趣的工作。

除此之外，我认为 PHP-GTK 可以为你的“杀手级应用”提供一个很好的平台，尤其是我想到的下一个版本。

##### 安德烈的计划

***SP:你在 http://gravitonic.com/[的网站上](http://gravitonic.com/)引用约翰·巴勒斯:***

<q>***“我仍然觉得每一天都太短，无法容纳我想去想的所有想法，我想去走的所有路，我想读的所有书，我想去见的所有朋友。”***</q>

对于你参与的所有事情，有一个问题不得不问:你是如何找到时间的？T3】

阿兹:最近有些困难。通常我会试着为一周制定一个时间表，留出时间做运动、编程、阅读、娱乐、家务和其他杂事。但我天生是一个非常好奇的人，我经常会对几件事情感兴趣并同时着手去做，尽管有时我太“入迷”了，以至于几乎没有注意到时间的流逝。因此有了这句名言。

SP:你是新闻出版应用服务提供商 ispi 的研发总监。几年前，ASP 被誉为 ISP 的继承者。你认为这在多大程度上已经成为现实，你对 ASP 的未来有什么看法？T3】

阿兹:互联网服务提供商没有危险。很多人只是想要连通性。ASP 可能会从 ISP 那里抢走企业客户，但这是一个双赢的局面:企业通过 ASP 获得更好的服务，而 ISP 可以专注于他们最擅长的领域。

ASP 的未来？垂直市场和面向本地(城市或邻里级)社区。

总的来说，你认为互联网将会带来什么，无论是技术还是企业和个人如何使用它？T3】

阿兹:如果我知道答案，我会登录我的经纪账户做一些交易，如果你知道我的意思的话。但说真的，我只想引用艾伦·凯的一句话，“预测未来的最好方法就是创造未来。”

***SP:在 Zend 名人堂，引用你的话说:***

<q>***“mod _ perl 的能力给我留下了深刻的印象，我想让 PHP 跳过同样的环，这样就有可能用直接的 PHP 代码编写任何种类的 Apache 处理程序，”***</q>

我们有可能看到这种情况发生吗？这将允许开发人员用 PHP 做什么？T3】

AZ:我还没有听说有人在研究这个，但是我认为这将会是一个非常有趣的项目。尤其是随着 Apache 2 的到来。您基本上能够用 PHP 编写 Apache 模块:重写 URIs，基于 PHP 支持的任何一种模式执行认证，过滤 web 服务器的输入和输出，操纵头部，实现新的协议，等等。

你的未来会怎样？你现在在做什么？T3】

AZ:我正在学习更多的东西:C#、Scheme、编译器技术、语音和语言处理、投资和市场理论、西方文化史。我的阅读书目和我的胳膊一样长。迟早有一天，我还想回到学校攻读语言学硕士学位。赢得彩票。

我要感谢 Andrei 花时间回答 SitePoint 社区的问题！
