# 波士顿 Dx3 见！

> 原文:[https://www.sitepoint.com/see-you-at-dx3-in-boston/](https://www.sitepoint.com/see-you-at-dx3-in-boston/)

以下内容转载自《科技时报》第 159 期。

随着新年的到来，我已经开始制定 2007 年的旅行计划。我今年的第一个演讲约会是 5 月 15 日至 18 日在波士顿举行的 Dx3 会议。

[![See me speak at Dx3!](../Images/79e4c8cbd388827b7712c7662abdf6f7.png)](http://www.dx3conference.com/?ref=sitepoint) 此次大会将聚焦于各种设计方法、开发技术和部署平台，这些都是开发者和设计者可以利用的，他们开始思考 web 浏览器之外的丰富体验和可能性。

我将在大会上做两次演讲:

> **API 和混搭:将网络带到桌面**
> 
> 所有的大玩家都在争夺 API 的份额。谷歌、微软和雅虎！都想让你用他们的地图。亚马逊的 API 拥有你能买到的任何产品的独家新闻。令人印象深刻的基于网络的混搭和联盟程序比比皆是，但浏览器对能做多少事施加了限制。具有讽刺意味的是，桌面会成为 web APIs 发挥作用的舞台吗？如果是，哪种新的桌面平台(WPF？阿波罗？Java Web Start？)最有能力利用它们？

> **JavaScript 的正确方式**
> 
> JavaScript 是最强大的标准客户端语言，通常用于网页设计。因此，它给了你前所未有的把事情搞砸的空间。无论您是刚开始使用 JavaScript，还是经验丰富的老手，都在努力让您老化的脚本正常运行，学习这些简单、强大、不引人注目的脚本技术(摘自即将出版的 SitePoint 书籍)将会让您的生活变得更加轻松——并且您的代码也会看起来很漂亮。

如果你碰巧在波士顿地区或附近，[查看会议](http://www.dx3conference.com/?ref=sitepoint)并让我知道你是否打算参加！
