##### 我应该使用哪种格式？一个问题，两种解决方案

目前，网络上只支持两种绝对图像格式。它们是 Gif(图形交换格式)和 Jpg(联合图像专家组)。PNG 格式仍然被相当宽松地使用，并且还没有被大多数人使用(或者被除了较新的浏览器之外的任何浏览器支持)。这种新的图形格式有望在某一天取代通用的 Gif 和照片质量的 Jpg 格式。然而，我们不会在这里讨论 PNG 格式，因为目前它缺乏支持和实现。

***Gif 格式***

Gif 格式可能是当今网络上最广泛使用的图形格式。原因有很多，但最常见的一个是因为它能够很好地压缩数据，同时提供干净、清晰的图像以及最独立于平台的结果。其他质量包括透明度，交错和动画。

为什么 Gif 格式对创建网页图形最有用？简而言之，因为它允许你使用选择性调色板来创建图像。

当你创建一个 Gif 图像时，你正在生成一个在其 CLUT(颜色查找表)中最多使用 256 种颜色的图像。这个颜色的“列表”或索引包含在每个 Gif 图像的标题中，并构成了图像整体文件大小的大部分。

Gif 格式的一个很大的优点是你可以定制你想要用来产生你想要的结果的颜色的类型和数量。当您将图像保存为 gif 时，您可以选择仅用于组成图像的确切颜色，从而生成具有非常明确的调色板的图像，这将产生尽可能小的文件大小。这反过来又产生了一个干净的、快速加载的图形，它将更好地服务于它的预期目的。

以这个例子为例。假设您创建了一个简单的三色图形。如果您将此图像保存为 Gif 格式，您不会被迫使用索引调色板中的全部 256 种可用颜色，这也很好——这将生成比您需要的文件大得多的图形，而图像质量不会有太大损失。

相反，当您保存最终图像时，您可以从许多不同的调色板选项中进行选择，包括仅使用图像中包含的确切数量的颜色。

你在这里看到的是 Jpg 使用的压缩类型的产品。它对不包含渐变颜色混合(也称为“连续色调”)的图像有问题，这就是它产生的怪物。

这就是这两种图像格式的区别所在:

*   Gif 使用“无损”压缩方法。在压缩过程中，不会无意中从图像的调色板中删除任何数据。您可以 100%控制质量和文件大小。gif 总是可以被重新编辑而不会影响图像质量。
*   Jpg 使用一种“有损”压缩形式，从图像中删除它认为可以删除的数据。通常照片/图像中的高色调是它首先移除的，导致图像失去清晰度。一旦你对一张图片应用了 Jpg 压缩，重新编辑并再次压缩它将会进一步降低它的清晰度，从而每次都破坏它的可用性。

Gif 格式最适用于包含大面积平面颜色区域和文本的图形，很少或没有平滑的颜色混合，如渐变。然而，在某些情况下，情况并非如此——我们将在稍后讨论这些情况。

为了使用 Gif 格式创建一个微妙的颜色混合，你必须对你的图像应用抖动。这最终会增加图像包含的颜色数量，这将直接影响图像的文件大小，从而影响下载时间。但是，有时您可能没有其他选择，在这种情况下，稍微大一点的文件大小对于呈现难看的图形来说是一个很好的折衷。找到一个快乐的中间点将是你的挑战，也是你的决定。这是成为一名优秀网页设计师的一部分。

***Jpg 有什么用？**T3】*

另一方面，Jpg 喜欢优化图像，这些图像要么具有摄影质量，要么包含大量微妙的颜色渐变——这些被称为“连续色调”图像。事实上，这就是它最初被创造出来的目的。它做得很好，将图像压缩到原始文件大小的 100 倍。

gif 比不过那个！jpg 无法与 gif 竞争，因为它不能在保持文件大小最小的同时，创建干净、粗体、包含文本的平面彩色插图。

这是这两种最常用的网络图像格式的基本和主要区别。下面是一个例子，说明 Jpg 格式比 Gif 格式更好地处理了细微的刻度。这里我们有一个使用 60%压缩比保存为 Jpg 的图像。文件大小= 5.30kb。注意到清晰度的细微损失了吗？

##### 充分利用这两种格式

 **每当您开始创建特定类型的图形时，提前知道图像将被保存为哪种格式是有利的。在创建过程中，可以使用某些步骤来帮助您充分利用每种格式的特定压缩能力。

为了理解如何使用这两种图像格式中的每一种来获得良好的效果，我们将研究每种格式的一些关键特征。

***使用 Gif 格式***

你会发现，从长远来看，gif 比 jpg 有更多的功能，因此，它们需要更多的时间和经验来完全掌握。

当你开始创建 Gif 图像时，第一步是考虑你将使用什么样的调色板。对于网络图形，这里应该没有问题:你应该总是使用 216 网络安全调色板，不管你有什么程序。正如我们之前提到的，这种调色板的特定颜色集旨在允许图形在 PC 和 Mac 平台上同样良好地呈现，而没有抖动(没有随机异常)，并且是网页设计中使用的行业标准。然而目前的测试表明，这两种主要浏览器之间确实存在一些差异，以及它们如何呈现 Web 安全调色板。但是我们作为设计师目前对这个问题几乎没有任何控制。

你会注意到 256 和 216 调色板之间有 40 种颜色的差异。这 40 种额外的颜色是在使用 8 位系统调色板(256 种颜色)时两个平台都不常见的颜色，因此，它们被从 Web 调色板中删除。尽管 Web Safe 调色板确实有其自身的局限性，但在大多数情况下，它与 256 自适应调色板一样好用，甚至更好。

这是一个使用 216 网络调色板中的颜色创建和保存的图像，应用抖动来平滑渐变:文件大小= 2.95kb

![768_sample3-216.gif](../Images/589e086de5111e2d541d2f9ad001c4b7.png)

现在，这是使用 216 网络调色板中的颜色创建的同一张图像，但仅使用 256 自适应调色板中的 16 种颜色保存，并且应用了抖动:文件大小= 1.81kb

![768_sample3-256.gif](../Images/63eeea73f0d9574068d109344ac3d2cc.png)

你会注意到在上面的图像中，渐变区域有点模糊，或者有点像素化，文件大小比自适应调色板版本要大一些。这是因为在 216 色调色板中找不到使渐变平滑所需的中间色。

即使抖动被应用于尝试和平滑逐渐的颜色混合，它也只是部分有效，事实上是文件大小增加的原因。这将被认为是 216 网页调色板的一个弱点，如果你真的坚持使用这个调色板，在你创建网页图片之前，这是一个值得注意的因素。

我们刚刚在这里看到的是一个很好的例子，说明有时使用自适应调色板保存图像比使用 Web 调色板更有益。但只要有可能，就应该更多地把它作为最后的手段。不过，令人欣慰的是，似乎今天大多数网络用户至少使用 800×600 屏幕分辨率的 16 位系统调色板:这对设计师来说是一个优势！因此，自适应调色板正迅速变得越来越流行。

这里需要注意的重要一点是，因为我们使用了网页安全调色板中的颜色来创建我们的图像，即使我们选择使用自适应调色板来保存图像，颜色在两个平台和大多数浏览器中的呈现也应该是相当令人满意的。

***216 色网页调色板上的注释***

**你看到的并不总是你得到的**

216 Web 调色板在应用于使用渐变的图形时有局限性。主要原因是在图像中创建渐变，即使你只使用“网页安全色”,也会创建网页调色板中不包含的新中间色。

你的图形程序会在这里欺骗你。就它而言，给渐变添加额外的颜色，让你的图像在程序中看起来尽可能的好是没有问题的。但是当你只使用网页安全调色板中的颜色来保存图像时，你的程序为使渐变看起来漂亮和平滑而添加的所有中间颜色都将被删除。

然后你将会看到的是一张图片在访问者浏览器中真实呈现的样子。如果你想避免在保存图像时反复点击取消按钮，在开始创建图像之前，请记住这个事实。

**修复此网页调色板限制**

因为网站的每个访问者都有不同的系统设置、浏览器规格和上网习惯，所以网站美工应该密切关注他们的图形所占用的“屏幕空间”。除非一个网站是专门用来展示图形作品的，否则文本内容总是比伴随它的图片更重要。但从长远来看，这一事实可以成为设计师的优势。

**256 自适应调色板的脚注:挑剔**

如果你要创建一个只使用几种颜色的图像，那么选择一个 4 位[16 色]调色板通常可以达到这个目的。如果图像中没有渐变，除非图像包含曲线，否则不要使用抖动。但即使这样，在添加它之前，先检查它看起来没有抖动。抖动可以帮助平滑平面二维图形中曲线的边缘，但这样做会增加文件的大小。抖动产生的一个常见异常是纯色形状边缘的斑点。

这可以说明两件事:

1.  不应该使用抖动。
2.  你需要给图像的调色板添加更多的颜色。

这最好以增量的方式完成。将您的调色板从 16 色增加到 20 色、20 色到 24 色、24 色到 28 色等等..当边缘看起来足够干净时停止。

注意:某些图形程序在保存模式下不提供对最终图像调色板的这种控制。在这种情况下，尽你所能。如果你的程序是 Photoshop 插件兼容的，Ulead 的 Gif/Jpg SmartSaver 是一个不错的廉价投资。

这里有一个例子，告诉你什么时候应用抖动，什么时候不应用抖动，以及为什么你应该在保存最终图像之前尝试抖动。

左边是一个大小为 30×30 像素的渐变球图像。此图像是使用 4 位[16 色]自适应调色板保存的，没有应用抖动。大多数大小为 40 像素或更小并使用渐变的图像通常不需要抖动。相反，对于较大的图像，应该谨慎使用抖动，如下图所示。

![768_greenballbig.gif](../Images/885936459453480d178c0aa099f527f1.png)

这是同一个图像，但是大了 15 个像素，所以是 45×45 像素[640 字节]。它也是用 4 位[16 色]自适应调色板保存的，没有应用抖动。注意渐变是如何开始在从亮到暗的阴影之间显示条带的？这可以通过对图像应用抖动来解决，同时增加文件大小[708 字节]。或者，您也可以将图像调色板中的颜色数量从 16 种增加到 32 种，这也可以平滑渐变，但文件大小会稍微增加一些[1.65kb]。

**将更大或更复杂的图像保存为 gif 格式**

如果您创建的图像使用了大量不同的颜色，包括渐变，那么在保存时不要害怕使用自适应调色板。这种形式的压缩用来确定颜色的方法很容易理解。基本上，当选择自适应作为你的调色板选项时，你是在要求你的程序只使用图像中最好和最重要的颜色来保存你的图像。其余的将被删除。这并不总是意味着它会使用你的图像开始时的所有原始颜色，但它通常会在选择使用哪些颜色时做出正确的选择。

当你第一次选择使用自适应调色板时，使用所有 256 色的抖动。当你的程序选择了要使用的 256 种颜色后，(从原始图像的数千或数百万种颜色中)，如果你有图像的预览，看看它和原始图像有多接近。如果你仍然注意到任何污点或色带，那么这将是一个很好的迹象，Jpg 压缩可能是更好的办法。

要了解更多关于 Web 安全调色板和自适应调色板之间的差异，以及它们的行为方式，请参阅戳像素一节。

***使用 Jpg 格式***

当谈到使用 Jpg 格式的网页图形时，其实没什么可说的。Jpg 通常是包含复杂刻度的照片或图形的首选格式。本质上任何非说明性的东西。

关于在网上使用照片，需要记住的几件关键事情是，你应该尽量将照片的尺寸保持在合理的范围内。换句话说，如果它的唯一目的是吸引读者的眼球，或者在一段文字中展示主题的图形示例，那么将它缩小到几乎无法辨认的程度，增加 5 到 10 个像素，然后保存它。缩小照片后，由于抗锯齿，照片可能会变得有点模糊。应用锐化滤镜通常可以显示抗锯齿消除的细节。这种方法适用于大多数情况。

将照片保存到网上的另一个好处是，使用中等压缩比通常就足够了。有时这取决于照片的大小和内容。你自己的眼睛将是最好的法官。

***你应该知道的事情***

这里我们要做的第一件事是解释我们在“如何充分利用两种格式”一节中看到的“示例 3”图。

如果你还记得的话，当我们试图用网页安全调色板保存那张图片时，我们遇到了问题。主要的问题是，因为我们的图像包含梯度区域，216 网络调色板导致一些污点或像素化发生在这些地方，即使在抖动应用。这是因为渐变中的中间色在 Web 调色板中找不到。

为了帮助弥补这一限制，我创建了一个新版本的原始 216 调色板版本，以前没有转换好。这里是原来的，又是问题版本。在它的右边是我创建的新版本，做了一些修改，以解决 216 Web 调色板的限制。

以下是我所做的其他改变，以及原因:

1.  我进一步减少了像素的数量:

1.  not dithering the image when I saved it out, and
2.  通过创建更亮的渐变区域。

为了做到这一点，我简单地增加了光的颜色，并移动阴影区域靠近图像的底部。然后我把渐变中的黑色替换成网页安全调色板中最深的红色[51，0，0]。为什么？因为在 216 色调色板中红色的阴影比黑色的阴影要多——这使得图像更加网页安全兼容。在 216 调色板[0，0，51]中使用最深的蓝色也可以获得类似的结果。这一步还增加了更多的纯黄色网页安全色，并删除了一些原来使用的中间非网页安全色。

5.  我改变了单词“Example”的颜色，因为当我将阴影移近图像底部时，文本后面的阴影变暗了，并遮住了单词的底部。我用了暗红色，[比渐变阴影浅一个色度——102，0，0]，因为它与黄色的对比刚好清晰可辨。如果我使用任何包含蓝色文本的同样深的颜色，它会看起来更像是黑色对黄色，并且不会解决易读性问题。
6.  我完全去掉了最大的渐变区域，它在圆圈里。即使缩小了图像，这个区域对我来说还是太大了，如果不给图像添加更多的颜色，当然会增加文件的大小。我选择了一个地球色调的颜色，以补充浅黄色的数字，以及在梯度更大胆的黄色。这里的另一个选择是只使用黑色，但这往往会洗掉圆圈底部渐变为暗红色的地方。
7.  我做的最后一件事是在数字中使用相同的浅黄色，[216 调色板中最浅的黄色–FFFFCC]在图像的手柄部分创建一个硬塑料表面的错觉。我所做的就是用淡黄色创建一条 2 像素的直线，并把它放在手柄顶端的四分之一处。

这种“色彩技巧”对于创造这样的效果是很有价值的，并且在保持图像原始质量的同时将文件大小保持在最小；因为这条线是纯色的，而不是在第一条线之上的另一种渐变。然而，更重要的是，不需要对图像进行模糊处理，因此不会在图像中引入中间颜色，这些颜色会在保存图像时产生斑点或像素化伪像，并大大增加图像的文件大小。

这里有一些你可以做的事情来最大限度地利用 Gif 格式。

**1)使用配色方案**

创建快速加载页面并从 Web 调色板中获得最佳效果的最佳方法之一是将配色方案限制在 3 到 6 种颜色之间。当你在图片和页面元素中使用彩虹中的每一种颜色时，访问者的浏览器和操作系统将在你的图片显示的速度和质量方面发挥更大的作用。

使用一个特定的方案或主题(颜色和布局)也可以通过给你的页面一个特殊的外观和感觉来增强你的页面，这种外观和感觉贯穿于你的整个网站。如果你需要这种技术的演示，只要看看任何大公司的网站就可以了。这很常见，一旦你掌握了窍门，就会产生非常“专业”的结果。你会发现对色彩理论和艺术元素的良好理解在这方面是有益的。

2)尝试抖动和调色板尺寸

当您将更大或更复杂的图像保存为 gif 时，在应用抖动时密切关注文件大小，并调整图像的调色板以删除任何不必要的颜色。

一旦你纠正了任何条纹或斑点，你应该开始减少图像调色板中使用的颜色数量。从 256 色减少到 128 色开始。如果你的图像变化不大，但看起来还可以接受，那么再减少一半，变成 64 色。如果 64 不足以保持图像质量，将颜色增加 64 的一半[即增加 32]。继续这种加减颜色的方法，直到你得到一个使用最少颜色的图像，但同时保持可接受的质量水平。

我的经验表明，通常情况下，一个 16 色的自适应调色板应用抖动将产生令人满意的结果，大多数类型的图像适合 Gif 压缩。更复杂或更大的图像，使用多种渐变或微妙的混合，有时可能需要应用抖动的 32 色调色板。

**5)当好的 jpg 变坏时该怎么办**

你可能想知道将上面的 3D 按钮保存为 Jpg 而不是 g if 是否是一个好主意，因为 Jpg 压缩在具有微妙颜色混合的图像上效果更好。

嗯，这确实是一个可行的选择。然而，根据我的经验，我发现，对于某些类型的渐变效果，Gif 是正确的选择。

![768_gradbuttondiag.jpg](../Images/a6e20cd7a1f268c19c22a6ab2ae68d43.png)

下面是上面对角线渐变效果的 Jpg 版本。质量不仅比前一版本好一点，而且文件大小也小了很多——只有 99 字节(小于 1 kb！).所以在这种情况下，是的，Jpg 效果更好。为什么？因为在这种情况下，梯度效应在整个图像中是连续的。除了阴影之外，没有真正坚实、平坦的颜色区域，jpg 可以很好地处理这种类型的图像。

![768_sample3-216sm.jpg](../Images/ace09c1c4ccc2b2a9fc3336b28e5683a.png)

**6)记住:jpg 不是永远的**

另一方面，gif 是可回收的。

Jpg 使用称为“有损”的压缩格式。Gif 使用所谓的“无损”压缩(就像 Png 格式一样)，在这种压缩中，一旦保存了原始图像，就不会从原始图像调色板中删除任何数据。

有损压缩是指每次将图像或照片存储为 Jpg 时，压缩方法(或算法)都会从图像中删除某些数据。它移除的是它认为人眼不会错过的细节。大多数情况下，这是真的。这就是为什么 jpg 在用于照片和复杂、连续色调的艺术作品时看起来一样好。

然而，一旦您将图像保存为 Jpg 格式，您就不能再次在图形程序中打开它并期望获得与您第一次保存它时相同的质量结果。每次重新打开保存时，质量都会变得更差，直到最后，它不再可保存。因此，如果你打算重新编辑你保存为 Jpg 的任何东西，请确保你已经保存了原始照片或艺术作品的副本。这样，您将拥有最佳的图像质量。

我注意到，在有损压缩中，高光会在暗色调之前被首先移除。这可能会导致照片的清晰度或锐度下降，如果压缩过度，也会导致照片整体变暗。这种效应也是产生梯度模糊异常的原因。
