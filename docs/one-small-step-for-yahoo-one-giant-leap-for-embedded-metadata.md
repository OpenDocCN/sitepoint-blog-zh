# 雅虎的一小步，嵌入式元数据的一大步

> 原文：<https://www.sitepoint.com/one-small-step-for-yahoo-one-giant-leap-for-embedded-metadata/>

语义网对一小群人来说意味着很多不同的东西，而对一大群人来说几乎没有任何意义。

雅虎正在努力改变这种状况。众所周知，雅虎是一家思想非常超前的公司，他们有很多优秀的网络系统，并且是像 flickr 这样的网站上使用的微格式技术的主要早期采用者。嵌入的元数据是一座等待开发的金矿。

Firefox 3 将内置显示这些嵌入元数据的功能。由于 Firefox 3 还没有正式发布，Firefox 2 的用户可以开始用[操作符](https://www.kaply.com/weblog/operator)插件探索外面有什么。它为隐藏的元数据提供了一个可视化的指示器，并为您提供了用它做有趣事情的方法:

*   在谷歌地图上定位一个地址
*   将 web 联系人地址添加到您的本地通讯簿中
*   将即将举办的活动添加到您的日历中
*   [堆积更多](https://blog.mozilla.com/faaborg/2006/12/16/microformats-part-3-introducing-operator/) …

那么，这与语义网有什么关系，雅虎将如何推动语义网的发展？雅虎西班牙研究所的彼得·米卡刚刚发布了 T2 微搜索，这是一个搜索引擎:

> …不是隐藏元数据，而是将它放在最前面，从而向用户显示对于任何给定的查询有多少元数据。

因为这种隐藏的元数据是结构化数据，所以您可以用它做一些非常有用的事情，比如在可视时间线上覆盖页面，或者自动将页面和条目映射到地图上。

![Yahoo microsearch example web interface](img/338f3ce3d7512a9bcf730e8f84b8f121.png)

Microsearch 目前使用以下元数据格式提取有用的内容:

*   微格式(hCard、hCalendar、hReview)
*   链接 RDF——通过 HTML
*   RDFa 很快将成为 W3C 的推荐标准(我将会在博客中对此进行介绍)

这是重要的一步，一家主要的网络公司已经认识到语义标记的好处。很快，网站将有必要在页面中嵌入这些额外的元数据，这样任何人，甚至我的父母，都可以在运行时将它们混合在一起:)
