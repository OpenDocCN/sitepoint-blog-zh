# 调整到 QMail

> 原文:[https://www.sitepoint.com/adjusting-to-qmail/](https://www.sitepoint.com/adjusting-to-qmail/)

尽管任何参与网络管理、设计和开发的人都把大部分时间花在了网络应用、网站和网站管理上，但这绝不是责任的极限。

如果你托管或参与托管客户网站，命运将不得不与邮件服务器角力。直到去年，对我来说，摔跤几乎完全是与 Postfix 和 Sendmail 或公司邮件服务器，如 Domino 或 Exchange。

最近，我面临着管理 QMail 的问题——我对此毫无经验。后来我发现这是一个非常可靠而且有点容易使用的替代方法，我想分享一些我在这个过程中发现有用的网站。

QMail 的一个显著特点是它能快速处理邮件列表——这对 web 开发者和主机当然很重要，因为许多客户都需要或想要这个功能。

在我的书签中，第一个是 QMail 主页(http://cr.yp.to/qmail.html 的 T2 和 T3)。

也许更重要的是一个资源网站，里面充满了帮助和贡献的软件，让管理邮件服务器变得更容易一些——拉塞尔·尼尔森的 qmail 主页(【http://qmail.agarik.com/top.html】T2)。包括作者的文档和软件，以及商业支持的链接，贡献的脚本等等。

请仔细阅读这些脚本，因为有些非常强大，而且正如作者所指出的，非常尖锐，可能会咬人。

最后，我想建议使用[Webmin](http://www.webmin.com)(【http://www.webmin.com】T2)来管理 QMail。Webmin 包括一个健壮的邮件服务器模块，以及对 Postfix 和 Sendmail 的支持。这可能是通过 GUI 界面使用邮件服务器(或整个 Unix 服务器)的入门方法。
