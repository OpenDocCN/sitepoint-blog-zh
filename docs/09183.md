# 为什么谷歌镖会错过它的目标

> 原文:[https://www.sitepoint.com/google-dart-fail/](https://www.sitepoint.com/google-dart-fail/)

Dart 是 Google 的新编程语言，旨在创建结构化的 web 应用程序。你可以在服务器上运行它，但它也可以在客户端上运行。在浏览器中。

根据你阅读的内容，谷歌承认和否认 Dart 是 JavaScript 的替代品。然而，Chrome 很快就能使用这两种语言中的任何一种，如果可以选择的话，我相信谷歌更希望开发者使用他们自己控制的解决方案。

我最初对 Dart 持怀疑态度，但在获得更多信息之前保留判断。我现在可以明确地说，它几乎肯定会像 VBScript 在 Internet Explorer 中那样失败。

## Dart 的目标

Dart 是一个开源项目，其设计目标如下:

**1。为 web 创建一种结构化但灵活的编程语言。**
伟大。但是所有其他结构化和灵活的语言都有什么问题呢？网络最大的好处之一就是你可以使用任何你喜欢的服务器端语言:PHP、C#、VB、Perl、Java、Ruby、Python 等等。

总有改进的空间，但我们被选择宠坏了。Dart 并没有提供什么不同的东西，只是一种选择。

**2。让 Dart 对程序员来说感觉熟悉和自然，从而容易学习**
从语法上来说，Dart 与 Java、C++和 C#非常相似。那么谷歌为什么不干脆采用其中一种语言呢？那会使学习变得更加容易。

**3。使 Dart 适用于所有设备。**
谷歌声明他们*“对抗分散的移动平台”*。难道另一种语言不会把它进一步分割吗？

谷歌可能会为 Android 创建一个原生的 Dart 运行时。或许他们甚至会为 Windows 手机开发一个版本。苹果——最成功的智能手机厂商——怎么样？没门儿。

**4。提供使 Dart 在所有主流现代浏览器上快速运行的工具。**
微软、Mozilla、苹果或 Opera 会在浏览器中加入原生 Dart 客户端吗？不太可能。

谷歌可以为这些平台创建插件，但在插件拥有广泛的安装基础之前，web 开发人员不会编写 Dart 代码。不幸的是，在使用 Dart 开发出引人注目的应用程序之前，用户不会安装这个插件。第二十二条军规。

## JavaScript 编译

Google 实际上不需要 Dart 插件，因为他们已经创建了一个编译器，可以将 Dart 代码翻译成本地 JavaScript。

在你太激动之前，先来看看[编译的 Dart《Hello World》节目](https://gist.github.com/1277224)。九行 Dart 代码成功编译成… 17，259 行 JavaScript。

我相信情况会有所改善。即使在今天，它也可以通过谷歌的闭包编译器来运行，以使代码更加高效。但事实仍然是，由一个不太好的 JavaScript 开发人员编写的原生 JavaScript 将总是胜过编译的 Dart 代码。

即使您使用 Dart 进行开发，您也可能希望在某些时候使用 JavaScript 来提高效率。但是如果你已经可以编写高质量的 JavaScript，为什么还要用 Dart 开发呢？第二十二条军规。

## 给 JavaScript 一些爱

从 Google 的文档中可以清楚地看到，Dart 是针对不喜欢 JavaScript 的开发人员的。

尽管是世界上使用最多的编程语言，JavaScript 却是最容易被误解的。名字没什么帮助——它既不是 Java 也不是 script——但负面报道的最大原因来自专业程序员。

最初，JavaScript 看起来有点像 C++或 Java。了解这些语言的开发人员在手册中搜寻类语法，却发现它并不存在。他们认为 JavaScript 很糟糕，或者试图将经典的继承技术强加到他们的代码中。

我恳求你坚持下去。JavaScript 非常灵活，允许您以多种方式编写代码。一旦你理解了像原型继承这样的概念，JavaScript 就会赢得你的尊重。它可能并不完美，但是基于类的语言很快就会让人觉得笨拙。

不要指望它会在一夜之间发生。开发人员花了十多年时间才重新发现 JavaScript 的魅力。幸运的是，网络上有许多精彩的资源，JavaScript 被公认为一流的语言。

## 因为你无法与之抗争

Dart 的主要问题是 JavaScript 无处不在；从简陋的手机，到苹果 iPads，再到现代的桌面浏览器。微软甚至正在为 Windows 8 中的应用开发制定 HTML5 和 JavaScript 关键技术。

即使 Chrome 达到了 50%的市场份额，你会用一种在任何地方都受支持的语言开发，还是只在一半的设备上受支持？如果它在十年前发布，或者 JavaScript 非常糟糕，或者 Dart 是革命性的，也许它还有机会。没有一个是真的。

我很高兴谷歌继续创新，但 Dart 感觉像是一种倒退。你可能不喜欢 JavaScript，厌恶 HTML，鄙视 CSS——但是，作为一名 web 开发人员，你无法避免它们。
