# 完善工作

> 原文:[https://www.sitepoint.com/refining-the-work/](https://www.sitepoint.com/refining-the-work/)

看起来 Ajax(或远程脚本，或任何你想叫它的东西)正在转变，因为 JavaScript 正在转变同样的方向:将可能性细化为最佳实践。Cameron Adams 写了一篇关于远程脚本的可用交互性的文章，这是一个很好的例子；不仅仅是“哦！哦！远程脚本！”但是*如何*最好地使用这种技术来为你的客户提供更好的用户界面，这毕竟是练习的重点。现在有一些倡议正在酝酿中，希望将这一趋势带到一个更稳定的基础上；记录什么是最佳实践，您需要记住的事情，以及您可以做的其他简单的事情，我会在它们发生的时候提到它们。(不好意思调侃了。)

作为另一个例子，见 [OpenID](http://www.openid.net/) :他们已经为网站认证建立了一个分布式系统，所以你可以在许多网站上有一个身份，但*不需要*注册一个像 Passport 或 TypeKey 这样的中心服务。他们的重点之一是允许基于 JavaScript 的登录表单，这样就不需要刷新整个页面就可以用你的身份登录了。这种标准化的驱动力是突破主流的成功技术的标志之一；CSS 就是这样，它经历了“它很酷，但是人们不使用它”的阶段，然后是“哇！无限可能！”阶段，现在已经稳定下来，那里有好的文档(如果你试图学习一些东西，W3C 标准文档是*而不是*好的文档)和有用的网站，像[位置就是一切](http://www.positioniseverything.net/)和 [css-d Wiki](http://css-discuss.incutio.com/?page=FrontPage) 涵盖最佳实践等等。请关注 Atom，新的 RSS:它们仍然处于“构建复杂的 W3Cish 标准文档”阶段，但是很快就会出现“哇！无限可能！”随着人们开始将 Atom 不仅仅作为订阅新闻故事的一种方式。我敢肯定，这将涉及到大量的 JavaScript。

我可以继续列出有用和酷的新 JavaScript 东西，比如 [script.aculo.us](http://script.aculo.us/) 及其相关的[原型库](http://prototype.conio.net/)，或者 [TrimJunction](http://trimpath.com/project/wiki/TrimJunction) 。但是你应该回去编码，并向每个人展示下一个很酷的 JavaScript 可能是什么。我很想看看。
