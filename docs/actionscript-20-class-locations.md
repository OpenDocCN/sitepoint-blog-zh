# ActionScript 2.0 类位置

> 原文：<https://www.sitepoint.com/actionscript-20-class-locations/>

我最近在做一个项目，需要我将一大堆代码从 AS1.0 升级到 AS2.0，虽然这个任务很简单，但它涉及到几个类文件和数百个属性、方法和子类。

还需要更改一些函数的功能，因此我着手创建外部类文件，并彻底检查代码，将类文件放入 Flash MX 2004 '/Classes/文件夹中，并着手进行更改。

由于有许多变更，我需要不时地测试 SWF，但是类文件中的代码变更并没有传播到编译后的 SWF 中。非常奇怪，于是我重启了 Flash MX 2004，重新编译了 SWF，改动就出现了。

更奇怪的是，如果你删除了。作为 SWF 仍然编译的安装目录的“/Classes/”文件夹中的文件！我认为这里进行了大量的缓存，所以搜索了。作为我刚刚删除的文件，果然在另一个目录中有一个副本。我对这个模型做了一些简单的修改。作为文件，嘿，很快，我可以对类文件进行修改，而不必重启 Flash！

我记得在测试版中，文件的运行时位置对于每个用户都是不同的，所以我将这些使用外部类的简单技巧带给那些可能还没有涉足 AS2.0 的人。

[注意]这些提示是针对基于 Windows 的计算机的，但是您已经明白了。

[Drive] =安装驱动器
[Username] =您的登录帐户
[Language] = Flash MX 2004 本地化语言(对于国际英语，通常是“en”)

1.不要将您的类文件保存在 Flash MX 2004 安装目录中

[Drive]:程序文件 macromedia flash MX 2004 enfirst run classes

2.将类文件保存在 FLA 所在的文件夹中时要小心(以这种方式编辑类时，我得到了不同的效果)，有些人报告成功，而有些人报告失败

3.务必将您的类文件保存到本地设置文件夹中

[驱动器]:文档和设置[用户名]本地设置应用程序数据 macromedia flash MX 2004[语言]配置类

我对任何在 Mac OS 上开发 AS2.0 项目的人都很感兴趣，看看其他人是否遇到过类似的情况
