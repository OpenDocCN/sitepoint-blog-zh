# FullCodePress 采访:达伦·伍德，新西兰队

> 原文:[https://www . site point . com/full code press-Darren-wood-interview/](https://www.sitepoint.com/fullcodepress-darren-wood-interview/)

来自两个国家的两个团队要为两个慈善机构完成两个网站。24 小时后。观众注视着每一个动作。这就是 2009 年 5 月在 CeBIT 举行的 [FullCodePress](http://www.fullcodepress.com/) 背后的想法。

Darren Wood 是新西兰团队选择的前端 HTML/CSS 编码员，该团队完成了[获奖的彩虹青年网站](http://fcp-nz.com/)。他友好地与 SitePoint 分享他的经验。

你或你的团队在比赛前做了什么准备吗？

是的。多年来，我积累了相当多的代码片段，所以我把它们组织成一个可以快速访问的结构。我还下载了最新的 jQuery 和我可能需要的所有相关插件。这是一件好事——比赛期间，互联网有点不稳定。

在活动开始之前，Tim 和我已经决定使用 Ruby on Rails 作为后端技术。我以前从未使用过 Ruby，所以我做了一点研究，很高兴地发现 RoR 非常直观，使用起来很愉快。

你以前和其他团队成员合作过吗？

没有，但是一旦我们开始在彩虹青年网站上工作，就感觉好像我们已经一起工作了很多年。多么了不起的团队啊！

你选择了 browser CMS——一个相对不为人知的基于 Rails 的测试产品——作为内容管理系统？影响你决策的关键因素是什么？有没有造成什么问题或者限制？

我之前没有任何使用 Rails 的经验，所以 [Tim 的背景](https://www.sitepoint.com/fullcodepress-tim-connor-interview/)是选择 BrowserCMS 的主要因素。总的来说，CMS 没有什么大问题，尽管它是一个复杂的产品。

你用什么工具进行开发？IDE？源代码管理？测试套件？

我用了 TextMate，Photoshop 和 GIT。

最大的挑战是什么？

我主要关心的是如何保持连续 24 小时的清醒和警觉。事实证明这很容易。肾上腺素和咖啡因承担了大部分繁重的工作。最大的挑战是在早上 4 点到 7 点之间保持动力。

什么比你想象的要容易？

与团队和客户合作。把六个陌生人放在一个房间里(嗯——感觉更像一个鱼缸),让他们在 24 小时内建立一个功能齐全的网站，结果可能会很糟糕。我们的团队和客户 Clint 和 Daniel 都非常出色。*“a+++又要交易了！”*

**您是否需要放弃或简化某些功能，因为它们无法在 24 小时内实现？**

很遗憾，是的。客户有一个巨大的需求清单，所以我们的第一个主要工作是优先考虑。从前端代码的角度来看，我更喜欢使用更先进的技术——也许是 HTML5 和复杂的 jQuery good。

你认为自己在哪些领域击败了竞争对手？

很难说。澳大利亚团队制作了一个优秀的网站。我认为我们的优势是在凌晨选择激励性的音乐！

他们在哪些方面打败了你？

我认为澳大利亚团队花了更多的时间来界定他们的项目。我们试图包含尽可能多的客户需求，这可能会使我们的网站在某些方面不够完美。他们似乎也比我们压力小得多！

事后想想，你会做出什么不同的事情吗？

蒂姆和我一致认为，我们可能应该推出自己的内容管理系统。BrowserCMS 功能丰富，但有点太硬，无法在 24 小时内争论。我也希望花更多的时间来调整众多网站表单的布局。

你学到了什么可以用在日常工作中的东西吗？

是的——确保你每十二个小时左右睡一次觉！

您会再次参加 FullCodePress 活动吗？

绝对的！这是一次奇妙的经历，我希望有一天能再次体验。我们的团队很出色，我们都带着一群坚实的新伙伴离开了！

谢谢你，达伦。

白天，达伦·伍德是新西兰奥克兰 [Marker 工作室](http://www.markerstudio.com/)的界面设计师兼开发人员。到了晚上，达伦被称为网络极客和/或“微格式先生”。在任何时候，你都可以肯定他是可访问的、语义 HTML 的倡导者。
