# 回顾:Linux 服务器安全性

> 原文：<https://www.sitepoint.com/review-linux-server-security/>

继续我当前的安全主题，O'Reilly 出版了 Michael D. Bauer 的第二版 [Linux 服务器安全](http://www.oreilly.com/catalog/linuxss2/index.html)。这本书的目标读者是那些管理互联网连接系统的人，也就是众所周知的堡垒主机，这本书在 500 页中包含了强大的安全设计、理论和实际配置方案。

Bauer 因为在 Linux 杂志上发表的偏执企鹅专栏而声名狼藉，这最终激发了他写这本书的灵感。

突出的是 Bauer 在深入研究实际操作的具体细节之前花时间介绍了网络安全设计和风险管理。随着 web 专业人员的业务模式的改变，许多开发人员发现自己至少戴上了一顶兼职系统管理员的帽子。在进入命令外壳并在生产服务器上进行更改之前，了解底层网络设计的工作原理以及特定配置如何防止攻击至关重要。

Bauer 使用了一些流行的 Linux 发行版作为他的安全研究的基础，包括 Red Hat、Fedora、SUSE 和 Debian，尽管所涉及的大部分内容在大多数发行版和 BSD 上都可以很好地工作。对于那些熟悉几年前第一版的人来说，作者重新验证了大部分现有内容，并添加了以下内容:

*   用于认证服务的 LDAP
*   Bill Lubanovic 撰写了关于数据库安全的章节
*   使用 LDAP 和 Cyrus-Imapd 以及电子邮件加密作为电子邮件服务器安全性的一部分
*   非常需要覆盖 vsftpd 和 ProFTPD FTP 服务器

在介绍任何其他守护进程和服务之前，这本书正确地让管理员开始使用 iptables 并构建一个强大的防火墙。然后，Bauer 一层一层地深入研究如何保护 BIND 和 djbdns 域名服务器软件的安全，用 Sendmail 和 Postfix 控制邮件，以及管理 Apache 等等。

虽然许多 Linux 管理员可能不同意，但 Bauer 建议，加固将接触互联网的 Linux 服务器的第一步是确保没有安装 X Windows。如果一个服务器是“无头”运行的(没有监视器，因此可以远程管理)，它当然不需要安装 GNOME、KDE 等完整的 X 系统。，甚至可能不需要一个最小的，”他写道。

在我的例子中，除了一个本地开发 Linux 系统之外，我运行的所有系统都没有使用 X——这些系统运行得很好。通过 SSH 的命令行管理与 Webmin 等良好的根级 GUI 界面相结合，应该可以像通过 Gnome 或 KDE 访问一样简单地维护无头服务器。

也许这本书的一个漏洞是缺少对 QMail 的任何说明。毫无疑问，QMail 越来越受欢迎，至少与 Postfix 齐名。我不断遇到 Qmail 用户(包括我自己！)而且它会在电子邮件管理这一章中找到一个很好的归宿。

这本书以监控系统日志的重要技术以及使用我自己的最爱之一 Tripwire 来结束。后者是入侵检测技术的最后一章，也包括 Snort 的内容。

作为对读者的奖励，本书包含了两个完整的 iptables 启动脚本，一个基于本书涵盖的内容，用于位于互联网上的服务器，另一个用于多宿主主机，这对于内部和公共网络连接都存在的 DMZ 中的防火墙来说很好。
