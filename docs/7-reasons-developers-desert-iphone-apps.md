# 开发者放弃 iPhone 应用的 7 个原因

> 原文：<https://www.sitepoint.com/7-reasons-developers-desert-iphone-apps/>

iPhone 应用程序是去年的成功故事。开发者已经为苹果广受欢迎的手机制作了数以千计优秀的——也是可疑的——应用程序。一些公司以几美分的价格出售他们的软件，赚了一大笔钱。

然而，开发者越来越倾向于离开苹果的专有平台，回归网络技术。包括 Twitter、Techmeme 和 Nextstop 在内的许多公司都在开发能在 iPhone 上运行良好的移动网络应用，但在 Safari 浏览器中是基于 HTML5 的。例如， [jQuery iPhone reference](http://www.mrspeaker.net/2010/01/14/jquery-iphone-reference/) 使用 [jQTouch](http://www.jqtouch.com/) 来模拟 iPhone 界面并缓存 web 文件，因此它的安装和运行就像一个普通的应用程序。

这种方法有几个好处:

1.  没有必要使用 Mac 或购买 iPhone SDK。
2.  您可以使用任何服务器技术或语言——您不必被迫学习 Objective-C。
3.  现有的 web 应用程序无需进行根本性的重写就可以进行调整。
4.  不需要等待苹果的批准。苹果公司经常以神秘的理由拒绝 iPhone 应用程序。
5.  应用程序和更新可以立即发布，而不是等待 3 周的批准和进入苹果商店。
6.  最终的 web 应用程序是跨平台的，应该可以在 PC 或其他移动设备上运行。
7.  由于 iPhone 用户已经在使用浏览器，因此在启动网页时不会有太多不和谐的体验。

Mozilla 还声称，用于诺基亚、Windows 和 Android 手机的移动浏览器 Fennec 可能标志着苹果主导地位的终结。该组织声称拥有最快的移动 JavaScript 引擎*(他们不都是这样吗——尽管最近的[基准测试支持他们的断言](http://crave.cnet.co.uk/mobiles/0,39029453,49304091-8,00.htm) )* 。Mozilla 确信这将鼓励开发者为浏览器而不是多个移动平台编写代码。

Mozilla 移动副总裁 Jay Sullivan 表示:

> 随着开发者对质量保证、他们必须购买的手机数量以及他们的安全更新是否会通过 iPhone 的审批程序越来越失望，我认为他们会转向网络。
> 
> 任何懂 JavaScript 和 HTML 的人都可以开发出很棒的应用，而不必学习特定的移动平台。在过渡时期，应用程序会非常成功。随着时间的推移，网络会赢，因为它总是这样。

我怀疑 iPhone 应用注定要失败。许多人通过苹果商店成功地将应用程序货币化，微支付对于单干的公司来说很少是划算的。然而，如果你有其他收入来源，定制的移动应用可能不是最有效的解决方案。从长远来看，苹果对应用市场的垄断不太可能成功。
