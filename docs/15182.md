# 采访 GUI Bloopers 的杰夫·约翰逊

> 原文:[https://www.sitepoint.com/jeff-johnson-gui-bloopers/](https://www.sitepoint.com/jeff-johnson-gui-bloopers/)

杰夫·约翰逊说:“作为一名程序员和计算机用户，我强烈地感到计算机应该比以前更容易使用。

嗯……让电脑更容易使用，嗯？听起来是个好主意。但是杰夫成功实现了这个目标吗——这个他在研究生毕业后找工作时为自己设定的目标？SitePoint 决定问 Jeff 几个问题，以了解更多关于他到底完成了什么的信息。毕竟，到目前为止，他只主持过国际会议，为世界上一些领先的公司工作过，撰写过无数的文章和书籍，现在是他在 1996 年创立的产品可用性咨询公司的总裁。

SitePoint:你好，杰夫，谢谢你和我们交谈。首先，我想问你，你是如何开始关注可用性问题的。是什么让你决定进入这个领域？

在大学(耶鲁)和研究生院(斯坦福)，我学习了认知心理学和计算机科学——这两者的结合通常被称为认知科学。那时我的兴趣是设计人们如何推理和解决问题的计算机模型。

但是当我接近研究生毕业时，我意识到认知科学领域的工作机会是有限的。然而，在人机交互领域有很多机会。这是在 20 世纪 70 年代末，当时微型计算机(也称为个人计算机)行业刚刚起步。计算机和电信公司正在招聘具有我这种背景的人，以帮助他们的计算机系统更容易使用。

作为一名程序员和计算机用户，我强烈地感觉到计算机应该比现在更容易使用。这种感觉，加上大量的工作机会，吸引我进入用户界面领域。

1978 年，我去了硅谷一家名为 Cromemco 的小型微型计算机公司工作。它是在几年前由两个斯坦福电气工程研究生创建的。他们需要有人来设计易于使用的软件应用程序，并监督键盘等硬件组件的可用性。因为 Cromemco 的编程人员非常少，所以我必须在许多项目中既是设计师又是程序员。我为他们设计文字处理、统计和图形软件。

然后，在 20 世纪 80 年代中期，我离开了克罗梅科，到施乐公司工作。我没有在他们著名的帕洛阿尔托研究中心工作，而是在办公系统部门工作，该部门负责将 PARC 的技术转化为产品。我为施乐开创性但不成功的 [Star 工作站](http://www.geocities.com/SiliconValley/Office/7101/retrospect/)的继任者工作。和在 Cromemco 一样，我的角色是用户界面的设计者和实现者。

在“施乐大学”呆了几年后，我去了 US West(一家电话公司，现在叫 Qwest Communications)的研究岗位，然后去了惠普实验室。1992 年，惠普实验室解散人机交互研究小组后不久，我搬到了 Sun Microsystems 的一个名为“FirstPerson”的秘密子公司。

第一个人是 Sun 试图将其业务从计算机扩展到电子消费电器。FirstPerson 的创始人开发了一种叫做“Oak”(现在的 Java)的新语言。我是可用性方面的主要负责人:我帮助设计原型，并在真人身上测试。原型是像电视机顶盒、在线视频点播服务之类的东西。我们甚至不被允许与其他 Sun 员工谈论我们正在做的事情。在 FirstPerson 成为 JavaSoft 后不久，我离开去成立了自己的咨询公司 UI Wizards。

你认为用户对软件的了解比可用性更重要吗？随着时间的推移，这些知识是否增加了产品的可用性？

人类适应性很强:当他们被激励时，他们可以学到很多东西。问题是:他们什么时候被激励？我不认为每个软件产品都必须如此简单，以至于它的所有功能都可以立即使用。小提琴和汽车需要很大的努力才能学会使用。有些人学习它们；其他人在挣扎。但是学习它们的人喜欢它们能做的事情。

当人们在工作场所使用软件时，他们通常不会选择它。这是给他们使用的，作为他们工作的要求，因此他们有动力去学习使用它。如果需要一点时间去掌握，那大概还可以。但请记住:雇主购买软件是为了提高员工的工作效率。如果软件需要花很长时间来学习，非常容易出错，或者使用起来很乏味或痛苦，生产力就会受到影响。因此，雇主和雇员都有理由希望软件支持而不是妨碍真正的工作。

家用软件是一个不同的故事。在这里，几乎没有学习的动力。如果你不能很快想出办法，你会很快放弃它。网络用户更不能容忍难以使用的服务。当有几十个(或几百个)其他网站提供同样的东西而不那么麻烦的时候，为什么还要挣扎呢？我们只需点击“返回”并转到其他地方。

关于影响软件可用性的知识，研究表明，当人们说他们“使用”一个软件产品时，这实际上意味着他们只知道如何使用该软件总功能的一小部分。一旦大多数人学到足够多的东西来实现他们的目标，他们就会坚持下去，很少会费心去学更多的东西。例如，几乎没有人使用微软 Word 的样式表功能。大多数人让所有段落保持“正常”，直接设置字体、边距和其他格式。

**SP:你觉得设计师对可用性原则有抵触情绪吗？如果是这样的话，怎样做才能让设计师记住坚持可用性原则的重要性呢？**

大多数开发者认为可用性需要额外的时间。他们总是处于强烈的时间压力下，所以任何花费额外时间的事情都是不好的。事实上，从项目一开始就使用以用户为中心的设计可以节省时间。它帮助你找出用户真正需要的是什么，这样你就可以提供这些，而不是设计和构建无数没有人会使用的功能。

似乎永远没有时间一开始就把它做好，但是当顾客把它扔回到你面前时，总有时间重新做一遍。即使更多地关注可用性会延长上市时间，也可能会缩短盈利时间，因为一个更可用的产品具有:

1.  更快的市场接受度(即更高的初始销售额)，以及

3.  降低客户支持成本。

最后，当顾客真正喜欢一种产品时，他们是忠诚的。相反，当他们几乎不能忍受一种产品时，他们会立刻转向你的竞争对手。可用性培养客户忠诚度。

开发人员有时也会因为控制问题而抵制 UI 指导:他们认为 UI 设计人员夺走了他们的控制权。如果他们理性地而不是感性地思考，他们会意识到当团队成员贡献他们自己最好的技能时，产品会更好，他们的技能是在实现上，而不是 UI 设计上。令人高兴的是，一旦我让程序员经历了一个以用户为中心的设计过程(有时又踢又叫)，他们通常会“明白过来”，甚至会因为有人从他们的任务列表中去掉了一些艰苦的工作而感到宽慰。

你的书， [GUI 失误](http://www.gui-bloopers.com)，阐述了用户界面设计中常见的陷阱。你能告诉我设计师最常犯的错误是什么吗？他们如何避免犯错？

很难挑出最常见的错误，但最有可能区分的错误是“说话怪胎”。当错误消息、命令名和指令是用程序员的行话编写的，而不是用对用户有意义的术语编写时，就会出现这种错误。错误消息中的这个大错误的一个原因是未能将低级的软件到软件的通信转换成与用户试图做的事情相关的东西。这就是为什么我们会在错误消息中看到 Java 异常。

这一失误的主要原因是管理不善:将编写软件文本的工作分配给程序员显然不是最好的选择。程序员擅长写代码；他们通常不擅长写文字。谁是？技术作家。软件中的所有文本都应该在软件发布之前由技术人员编写，或者至少审查。

设计者和开发者在试图让一个程序“用户友好”的过程中，什么时候走得太远了？哪里可以找到平衡？

有很多方法可以超越 UI 设计。一个是花太多时间担心图形外观，以至于你没有足够重视用户界面更重要的部分:交互和任务流。这就是为什么软件开发团队需要交互设计师，而不仅仅是图形设计师。这也是为什么我建议通过首先开发一个概念模型来阐明软件*做什么*来开始 UI 设计，而不是直接跳到绘制屏幕和布局小部件。

另一种过度使用可用性的方法是，当你不打算对测试发现的问题做任何事情时，进行可用性测试。那只是浪费时间和金钱。我希望我能说没有人会犯这种错误，但这很常见。

最后，当用户界面设计者未能对他们在可用性测试或评审中发现的问题进行优先排序时，他们有时会走极端。有些问题比其他问题更重要。开发人员需要问题的优先列表，这样他们就可以专注于重要的问题，而将其他问题留待以后解决。话虽如此，一些低优先级的问题也很容易修复，例如，更改菜单命令的名称。

毫无疑问，增加一个网站的可用性可以带来更多的用户和重复流量。然而，网页设计师如何让他们的客户相信这是事实，并且可用性测试的投资会有回报？

有一本书叫做《证明可用性的成本》,作者是 Bias 和 Mayhew。这是前网络时代，但经济学的论点仍然适用于网站。

然而，有时候，唯一能真正传递信息的是“艰难学校”:把一个劣质的、令人讨厌的网站放在那里，让人们成群结队地离开。这通常会引起管理层的注意。

谁是可用性领域的大玩家，你经常访问哪些网站？

在我的咨询网站上，我列出了几本[方便的 UI 设计书籍](http://www.uiwizards.com/suggestedReading.html)——“大玩家”主要是写那些书的人。在我的领域，大多数人认为像道格·恩格尔巴特(1960 年代的 SRI)、斯图亚特·卡德(PARC 施乐公司)、吉姆·福利(佐治亚理工学院)和本·施耐德曼(马里兰大学)这样的研究者是这门学科智慧的源泉。当然，有一些从业者甚至在 UI 领域之外也很出名，比如雅各布·尼尔森、艾兰·库伯和唐·诺曼。

我经常访问的网站？嗯，有雅各布·尼尔森的网站。Dey Alexander 是澳大利亚的一名用户界面顾问，他的网站上有一个[每日网络博客](http://deyalexander.com/blooper/)。还有由 UI 顾问 Brian Hayes 运营的[用户界面耻辱厅](http://www.iarchitect.com/mshame.htm)。此外，Keith Instone 关于网络可用性的精彩资源[链接](http://www.usableweb.org)。还有由我的书的出版商运营的 GUI Bloopers 网站，最后还有文森特·弗兰德斯的网站。

像 Macromedia 的 Flash 这样的软件以创建无用的网站而闻名。这种情况会被克服还是会产生更大的问题？

一点点动画在一个网站里可以很好，但是很容易过了头。不幸的是，许多使用 Flash 的网站就是这么做的。此外，许多基于 Flash 的网站没有为没有 Flash 的访问者提供替代选择。

许多设计师认为人们会得到 Flash 只是为了看到他们精彩的内容，但他们大错特错了。大多数人不知道如何获得 Flash，即使他们知道，他们也认为(可能是正确的)试图安装它会变成一场耗时的噩梦，就像许多软件安装一样。所以他们不去打扰，错过了内容。谁输了？不是用户。

根据你丰富的专业知识，你会对任何想在可用性领域工作的人说些什么？

一般来说，进入这个领域的最好方法是通过可用性测试。进行可用性测试是一个很好的方法，可以看出什么有用，什么没用。这会给你重新设计软件或从头开始设计软件所需的经验。

人们也从其他相关领域进入这个领域:

*   **平面设计:**他们厌倦了画图标和金属按钮，想做一些关于交互的东西。
*   **技术写作:**他们意识到难以解释的软件很可能设计得很糟糕，所以他们开始参与设计。
*   编程:他们对构建没人喜欢的东西感到沮丧，想要做得更好。

鉴于目前的就业市场，我建议不要试图成为一名顾问——即使是有经验的顾问也很难找到足够的工作。而是找公司就业。

谢谢你的建议，杰夫。最后，你能告诉我们可用性的下一步是什么吗？你能预测两年后、五年后的情况吗？互联网会让可用性专家更加困难吗？

预测很容易，但很少是正确的。电视剧《杰森一家》的背景是 1999 年。我们的飞行汽车、喷气背包和管中午餐在哪里？考虑到这一点，以下是一些预测:

*   网络上的某些常见功能将会稳定下来(例如购物车)，并将被浏览器所取代。然后网站只需要指定他们有这个功能，浏览器就会处理它。
*   语音识别将变得更好，并更广泛地用于与计算机的交互。但人们不会把它们当成电脑，只是当成服务。
*   计算机将开始消失在特殊用途的信息设备中。
*   手机将承担更多的 PDA 功能。
*   随着越来越多的应用程序支持互联网，桌面软件和基于浏览器的网络服务之间的区别将会消失。越来越多的应用程序将通过互联网与数据库和其他应用程序进行通信，这对用户来说是不可见的。
*   垃圾邮件会不断增加，并威胁到互联网的崩溃。
