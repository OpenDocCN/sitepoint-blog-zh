# 2005 年编写 JavaScript 的十个好习惯

> 原文：<https://www.sitepoint.com/ten-good-practices-for-writing-javascript-in-2005/>

Bobby van der Sluis 在 2005 年为编写 Javascript 编写了十个好的实践指南。我怀疑我这里的大多数读者已经知道我们应该做这些事情:Bobby 谈到使用不引人注目的 Javascript 使您的页面可访问，编写易于其他开发人员应用的脚本，用对象检测来验证您的工作。这些都是明智的东西；阅读他的文章了解更多信息。关键的一点是，知道我们应该做这件事和实际去做是有区别的。我经常发现自己*很快*和*暂时*在标签上加入 onclick 属性，而不是从 JavaScript 正确地附加事件处理程序，我敢打赌，我并不孤单。严格地使用适当的技术，确实会使项目的初始构建变得更加复杂，更加费力。这是(你也知道这一点)当你在六个月后回到它，你必须通过 HTML 修改它来添加新的功能，你会想:我希望我第一次做得很好。

当然，除非你们都是编程的典范，只有我不是。任何证明事实如此的人都可以得到闪亮的英格兰银行英镑。

所以，读读 Bobby 的文章，找出任何你还不知道的技巧，然后(这是很重要的一点)一直应用这些知识。我们都将开始玩一个伟大的新游戏，在这个游戏中，我们可以使用 Ajax 技术来加速网站，浏览器制造商正在与我们合作，寻找我们想要的东西，web 服务就在那里等待集成。让它成为你今天的口头禅，你会以正确的方式做事。继续吧。对我来说。也为了其他人。
