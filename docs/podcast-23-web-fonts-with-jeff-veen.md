## 采访记录

Kevin:2009 年 8 月 14 日星期五的 SitePoint 播客第 23 集:“Jeff Veen 的网络字体”。

凯文:大家好，欢迎回到 SitePoint 播客——新闻、观点和网页开发者和设计者的新思维。我是主持人凯文·杨克，来自澳大利亚墨尔本的 SitePoint 总部，和我一起的还有我的联合主持人小组。

布莱德:网络工作室的布拉德·威廉姆斯。

iFroggy 网络的帕特里克·奥基夫。

斯蒂芬:来自德克萨斯州休斯顿的斯蒂芬·塞格拉夫斯。

凯文:今天是周五，这只能意味着一件事:SitePoint 播客的时间到了。这个星期又是我——或者，更确切地说，是我和我们的特别嘉宾——因为这个节目是对杰夫·维恩的采访，他是 [Typekit](http://typekit.com/) 背后的聪明人之一，这是一项新服务，杰夫有时将其描述为“网络字体的 iTunes 商店”。但与其由我来解释，不如让我们欢迎他自己:杰夫·维恩，欢迎来到 SitePoint 播客。

杰夫:嘿，非常感谢你，凯文。很高兴来到这里。

凯文:很高兴接到你的电话。所以，请为我们的听众，介绍一下你自己。

杰夫:当然，我很乐意。大家好，我叫杰夫·维恩，是[小批量公司](http://www.smallbatchinc.com/)的首席执行官，该公司目前正在开发一款名为 [Typekit](http://typekit.com/) 的产品，我想我们今天会有很多内容要谈。

在那之前，我花了一点时间在[谷歌](http://www.google.com/)，在那里我们围绕[谷歌分析](http://analytics.google.com/)做了一些设计工作，我相信你的很多听众可能都很熟悉。我们是通过收购一个叫做 Measure Map 的产品进入谷歌的，这个产品是我在一家叫做 [Adaptive Path](http://adaptivepath.com/) 的公司开发的，这是一家位于三藩市的咨询公司，专注于网页设计和用户体验。

简而言之，这就是我在过去几年里一直在做的事情，我期待着与你聊天。

凯文:好的。当有人在聚会上遇见你时，你是如何描述你所做的事情的？共同点是什么？

杰夫:你知道，这在很大程度上取决于派对。老实说，我告诉人们我在网上做东西，这是最基本的设计观点。嗯，我真的没有任何设计方面的训练；技术上来说，我算是掉进去了。当我在[连线杂志](http://www.wired.com/)工作的时候，我很早就接触到了网络，但我的职业生涯现在已经越来越倾向于从一种企业家的角度在网络上构建东西，但在过去的几年里，我已经开始为自己工作，并与许多出色的朋友和合作伙伴建立了公司，那时，他们问我是否可以再给我一杯饮料，然后他们就走开了。

Kevin: 也许这更多是因为你在 Adaptive Path 的背景，但看起来你是从可用性的角度来看待事情的，或者至少看起来是这样，每当 Jeff Veen 的一个新项目出来，似乎总是有比平均水平更多的注意力花在这个东西的可用性上。是网络的什么让你不断带着新的想法回来？是因为网站上的可用性不同于其他地方吗？

**杰夫:**在我的职业生涯中，我一直非常着迷的一件事是，在网上制造东西的人和消费东西的人之间的界限变得模糊了。就像电视甚至新闻业一样，当我在报社工作的时候，有很多这种小团体的人，受过高度训练，他们负责制作一切，当网络出现时，这就完全被抛到了九霄云外，对吗？所以我一直对社交媒体之类的东西非常感兴趣，一直追溯到博客，任何人都可以发表意见，任何人都可以发表文章，拥有观众和分享东西，我一直对此非常感兴趣，从这个角度来看，这要求我们在我们的设计工作中有很多同理心，我们真正关注最终用户，让他们参与进来，让他们成为这个过程的一部分，因为我们也是最终用户，他们也是开发者，他们也是内容提供商。所以那条模糊的线一直是我参与的所有项目中所有设计工作的驱动力。

凯文:太好了。你提到了你在谷歌的时光。作为一名前谷歌员工——Alex Walker，我们在 SitePoint 的设计师问我，你是否有任何关于 [Google Wave](http://wave.google.com/) 的观点可以分享。这是你可以谈论的事情吗？这是你个人的观点吗？你参与了吗？

Jeff: 事实上，没有。我没有参与其中，老实说，这是谷歌如此神秘的一个罕见例子，当我在网上看到演示时，其他人也看到了，就像是同时发生的一样。我知道悉尼办事处正在进行一个大项目，人们都搬到悉尼去工作，却没有人谈论它。我认为这很吸引人，就像现在一样，在谷歌之外，向内看，这是一个很好的例子，说明这种自下而上的倡议如何在谷歌一直奏效。总有这样一个神话，20%的时间和有才华的工程师可以去做他们想做的任何事情，并获得支持，我认为谷歌是一个很好的例子来实现这一点。有一个久经考验的工程师团队有一个好主意，他们得到了内部的支持，并付诸实施，他们的工作非常出色。

凯文:从可用性的角度来看，亚历克斯现在想知道的是，将所有这些简单得多的通信系统编织成一个大的解决方案是否可行，甚至是否可取？将所有这些整合到一个可能更复杂、更难使用的东西中，这样的东西有价值吗？

杰夫:这是一个完全合理的担忧，对吗？我认为，如果我把钱投在任何谷歌的解决方案上，他们会以一种开放和拥抱开发者社区的方式这样做，我认为这非常类似于 [Twitter](http://twitter.com/) 所采取的立场——老实说，我每天整天都在使用 Twitter，我几乎从不访问他们的网站。有客户，有我的手机，没错…我完全期待 Google Wave 成为第一个实现也许是一种与人联系和沟通的新方式，就像 Twitter 一样。我们不知道将会有什么样的规则，我们也不知道人们最终是否会做所有这些事情。就像 20 年前的电子邮件一样，它脱离了专有解决方案，变成了你可以随心所欲消费的东西。所以这是我的猜测，我的赌注。

凯文:十指交叉。

杰夫:对，没错。

凯文:几年前，你写了一本名为[《网页设计的艺术与科学》的书](http://www.veen.com/artsci/)，一位 SitePoint 论坛的用户，[亚历克斯·道森](https://www.sitepoint.com/forums/member.php?u=316859)想知道，如果你必须思考一下，自那本书出版以来，网页设计领域最重要的变化是什么？

杰夫:这是个很好的问题。那也是很久以前的事了；我想那本书实际上是在 1999 年出版的。所以发生了很多事情。如果你回想一下 2000 年，像标准空间网页设计这样的东西是一种边缘事物，像大公司并没有真正接受和支持它，它是通过像[泽尔德曼](http://www.zeldman.com/)和[埃里克迈耶](http://meyerweb.com/)这样的人的工作，坦率地说，数百名设计师真正专注于基于标准的设计，以及浏览器战争的重新出现，对吧，已经在某种程度上集中于这样一个事实，即我们现在可以认为这是理所当然的。就像这不是一场大仗，这是一个很好的设计实践，人们一直都在这么做。尤其是现在像 Firefox T7 和 Safari T9 这样的东西已经从主流玩家手中夺走了大量的市场份额。我认为这是最大的变化之一。

实际上，在那段时间里发生的另一个真正大的变化是新浏览器的采用曲线，我认为这是我们最近不太谈论的事情。我知道，当我开始我的网页设计生涯时，当一个新功能登陆浏览器时，我们都很兴奋地使用它，但我们知道这将是六个月，九个月，也许一年多之后，我们才能真正指望足够多的观众最终升级他们的浏览器，以便能够依赖这些新功能。看到的东西，例如用 [`@font-face`字体链接](https://reference.sitepoint.com/css/at-fontface)的地方，一个新的浏览器出来了，在几周之内，很大一部分观众已经升级了火狐或者升级了 Safari。在 Firefox 中，当你打开应用程序时，它会简单地问你一次，“嘿，你介意我更新一下吗？”它已经下载并更换了它，你有了新版本，这从根本上改变了设计师接受新技术的速度，坦率地说，浏览器供应商可以迭代并查看什么有效的速度。

**Kevin:** 其中一些与浏览器制造商更聪明地引入功能有关，在 CSS 属性名称上添加专有前缀的事情，不要太技术化，但这意味着新功能添加到浏览器的方式解放了开发人员，他们可以在不破坏兼容性的情况下尝试这些东西。

Jeff: 完全正确，然后设计师可以提供反馈，坦率地说，用他们的代码投票，对吗？你可以看到什么样的东西在工作。

在 [WebKit nightlies](http://nightly.webkit.org/) 中，我一直在密切关注我们制作这些令人惊叹的 CSS 动画的事情，它完全建立在核心操作系统技术之上，因此它具有很高的性能，这让我对我们可能在 2010 年的某个时候——如果我们达成一些共识，可能是 2010 年初——能够构建的东西感到非常兴奋。我只是认为这是一种绝对迷人的方式，真正展示了浏览器领域的许多创新，老实说，如果你在 10 年前问我，我会说，“不，我认为现在这基本上已经解决了，我认为 IE6 可能是我们看到的最后一个版本……”还记得所有这些东西吗？现在这些事情来得这么快，我再高兴不过了。

凯文:你提到了浏览器大战的重现，这是事实，但这确实是一场不同类型的“战争”第一次，有一种感觉，有人必须赢，我们都在为谁会赢而下注，但现在如果没有人赢，我们也很高兴，因为竞争越激烈，对我们越有利。

杰夫:完全正确。我完全同意。更像是我们有了浏览器，对，我们有——实际上这个过程现在正在工作，我们知道它是如何进行的，每个人都相处得很好…

**凯文:**你提到了`@font-face`，这确实是导致 Typekit 出现的新功能，对于那些可能不知道什么是 Typekit 的人来说？

**杰夫:** [Typekit](http://typekit.com/) 是我正在开发的一个应用程序，它介于所有对使用网页排版感兴趣的人之间。大约 18 个月前，Jeffrey Zeldman 的奇妙网络开发者网站[上有一个帖子，作者是](http://www.alistapart.com/) [Opera](http://opera.com/) 的首席技术官，他说[下一件大事将是网络字体](http://www.alistapart.com/articles/cssatten/)，他承诺 Opera 将在下一版本中支持它。紧接着，它开始出现在 Safari 中，然后 Firefox 宣布他们也将支持它。对于设计者来说，在他们的样式表中使用一个指向服务器上的字体的 URL 是一个非常简单的方法，然后他们的 HTML 就使用该字体进行渲染。因此，这非常类似于我们如何链接到一个图像，或者链接到一点 JavaScript。现在，我们可以链接到一个字体，当一个 HTML 页面在浏览器中呈现时，它会以该字体显示，而不必使用 Flash 或图像或其他任何东西。所以真的，整个排版可以在浏览器中开放。

现在，这是在 Internet Explorer 中一直支持的东西，我想是 Internet Explorer 4，但肯定是 Internet Explorer 5， 但它已经存在了一段时间，但 Internet Explorer 以一种非常专有的方式使用他们发明的字体格式，并获得了专利，在 Internet Explorer 中真正使用它的唯一方法是使用一种相对麻烦的工具，该工具应该将你的字体转换为他们的特殊格式，并将其链接到页面上，但它确实不适合设计师的工作方式，所以没有人真正使用它。 现在这些浏览器建议，“那么，我们为什么不使用 OpenType 或 TrueType 字体，这种字体在世界各地都存在，就像我们永远拥有它们一样，把它们放在你的服务器上，把它们连接起来，它就可以工作了，这已经引起了极大的兴奋。设计师们说，“这太棒了！这是我们长久以来想要的，太棒了。"

我们不得不放慢脚步，评估正在发生的事情，当然，是在字体设计和铸造方面，因为他们没有许可证，实际上允许你在服务器上粘贴字体，并在整个网络上分发。他们很谨慎，担心他们所有的知识产权会变得免费，广泛传播，无处不在。因此，他们试图找到一个有助于保护他们的知识产权的解决方案。

在过去的十年里，我们已经看到 DRM 解决方案显然是行不通的，对吗？几乎所有尝试过的解决方案，无论是 iTunes 还是微软的一些技术或其他什么，这些公司总是退缩。消费者就是不能容忍所有的晦涩难懂的规则和诸如此类的事情。所以这种东西是行不通的。

因此，Typekit 试图做的是在拼命将字体放到网页上的设计师和谨慎尝试进入这个新市场的代工厂之间找到一个中间地带，我们的解决方案是将字体托管在一个中央服务器上，采取一系列相关的轻量级保护措施，以便只有授权的网站， 只有拥有 Typekit 帐户的人才能真正使用这些字体，然后我们会给你一个链接，你可以将它放入你的页面，使字体工作，你可以继续像往常一样工作。 无论你使用 HTML 编辑工具还是手工编写代码，这个想法都是让设计者只创建他们的页面，但是字体对他们是可用的。

简而言之，这就是它的工作原理。

凯文:这个项目是什么吸引了你？

杰夫:当我在谷歌的时候，我一直在想接下来会发生什么。一年多前，我离开了谷歌，和我的老朋友兼商业伙伴布莱恩·梅森一起创办了这家名为 Small Batch 的新公司。他和我曾在 Adaptive Path 共事过。他经营着咨询公司 Adaptive Path 的业务，他正在寻找一个新的事情去做，所以他和我一起开始头脑风暴一些我们可以做的事情。我们开了一个会议，和 Twitter 一起做了一些工作，和他们一起做了一些咨询。我们和其他几位联合创始人聚在一起，他们也曾在谷歌和我们一起工作，并帮助建立了 Measure Map，他们是 Greg Veen 和 Ryan Carver。当我们在 Twitter 上做这项工作时，我们四个人聚在一起，问我们下一步可以做什么？我们的项目是什么？什么东西会真正吸引我们？

这是关于去年的假期，在那之前，有一些博客帖子。我想来自 [Clearleft](http://clearleft.com/) 的理查德·鲁特和来自澳大利亚、来自悉尼、 [Western Civ](http://westciv.com/) 和 [Web Directions](http://www.webdirections.org/) 会议的约翰·奥尔索普已经开始谈论，“嘿，既然浏览器支持字体，那么把字体带到网络上怎么样，也许会有一些新的商业模式？”

凯文:是的，那时候天气刚刚开始变热。

杰夫:是啊，那时候正在升温。我也——偶然地——去了澳大利亚，在悉尼度过了圣诞节假期，与约翰聚在一起，并与他共进午餐，在那里我们开始交谈，他开始告诉我这件事，好像这里有一个想法。我们可以做些事情，尝试将字体作为一种服务来提供，也许像 iTunes 是一种服务一样，也许字体可以成为字体的 iTunes。它完全引起了共鸣。这是一个观众——网络开发者观众，自从我 90 年代在[网猴](http://www.webmonkey.com/)写作以来，这是一个我一直与之联系并感到是其中一部分的观众，我在会议上见过他们，我觉得这是一个完美的观众，一套完美的技术，感觉有点像狂野的西部土地掠夺。就像，这种新技术出现了，没有人知道该做什么，人们对此非常兴奋，它是完美的。

所以我们回来后和约翰聊了更多，他是 Typekit 的顾问，所以我们一直和他聊这个。我们勾勒出了基本的架构，它是如何工作的，深入了解浏览器如何支持一切，走出去与投资者交谈，并说我们认为这将是非常大的，它将改变网页设计师的设计方式，网站的外观，我们认为这是一个巨大的机会。我们得到了一些出色的投资者，包括来自推特的伊万·威廉姆斯、来自 T2 的卡特林娜·法克、来自 T4 的马特·莫楞威格以及其他一大群人。因此，我们让非常熟悉网络的人说:“是的，我们相信它，我们支持你。”就这样，所有的一切都走到了一起……然后幸运的是， [Jason Santa Maria](http://www.jasonsantamaria.com/) ，世界上最大的字体极客和非凡的设计师之一，刚刚从 [Happy Cog](http://www.happycog.com/) 出来，他和 [Jeffrey Zeldman](http://www.zeldman.com/) 在那里工作了多年，开始寻找自己的项目，直到我们也开始与他合作。所有的东西都聚集在一起，这就是 Typekit 的起源。

凯文:那么，你认为自己是一个典型的极客吗？

杰夫:我一直对字体设计很着迷，但我不太擅长。

凯文:我认为这描述了很多网页设计师。

杰夫:是的，我知道，我想是的，坦白地说，我认为我们从来没有必要这样。网络上的字体选择是如此有限，我们对如何制作我们所拥有的字体有了越来越多的控制。我一直喜欢——我的意思是，当我放眼世界时，我确实看到了字体，就像我看到了标志，我看到了书和一切，我就像“太棒了，我喜欢。”我从来没有机会真正深入了解他们并与他们一起工作。所以我想是的，我想像很多网页设计师一样，我很高兴我们现在有这样的机会，这样我就可以最终提炼出我必须与文字一起工作的渴望。
