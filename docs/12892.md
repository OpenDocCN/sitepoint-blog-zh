# 马克·博尔顿的自由设计秘密

> 原文：<https://www.sitepoint.com/mark-boulton-freelance-design/>

马克·博尔顿设计公司是一家位于英国加的夫的小型设计工作室，专门设计简单、漂亮的网页界面。安迪·科瓦利克采访了老板兼设计师马克·博尔顿，以了解这一切是如何发生的，并获得他关于经营一家成功设计企业的重要因素的智慧。

首先，你能告诉我们一些关于你自己和你在网页设计方面的背景吗？

大约两年前，我开始全职经营这家工作室。在那之前，我在英国广播公司新媒体部门担任高级设计师，之前我在伦敦的 Agency.com 担任艺术总监几年。

十多年前，我开始从事网页设计，当时我在英国曼彻斯特的一家小设计公司工作。我的根很传统——我有一个印刷设计学位，在澳大利亚悉尼一家名为 Spike 的公司找到我的第一份全职网页设计师工作之前，我做了几年印刷设计师。

基于网络的网格设计目前似乎吸引了很多注意力。你对网格的迷恋是从哪里开始的？

我在大学学的是版式设计。课程的一个模块是国际印刷设计师协会学生评估计划。任务是设计一本分子生物学百科全书。对于这种范围的任何参考书，基础网格系统必须仔细设计，以适应各种页面类型。我想这就是一切开始的地方。

这个项目最初看起来令人望而生畏，但是在经过深思熟虑的网格系统设计之后，这本书基本上自己设计好了。就在那时，我意识到网格系统是设计师工具箱中非常有价值的工具。

你一直对好的排版艺术感兴趣吗？

我对印刷术的兴趣逐渐源于对插图的兴趣。我从艺术学校开始，想成为一名商业插画师，绘画类的。当我在大学的时候，我开始尝试用字体绘画。因此，人们的兴趣从那里开始增长。

继网格和印刷术之后，你认为相对于一个更固定的印刷环境，为网络设计的挑战是什么？

这些限制是网络上的一个持续挑战。事实上，因为媒介是如此的易变，发展如此之快，我们许多优秀的印刷设计同行发现很难进行转变。我相信这就是为什么我们在网上看到这么多 Flash 网站的原因。设计师们并不完全理解这种媒介和它的可能性，所以他们试图用印刷的方式来填充。即使十年过去了，我们仍然看到这种情况。

SP:那么你觉得网页设计最令人兴奋的是什么？你和其他开发者一样对 CSS3 感到兴奋吗？

我发现挑战和限制令人兴奋。我真的很喜欢在一个项目中工作，这个项目可以让我展开我的印刷翅膀。CSS3 听起来非常有趣，尤其是一些排版和布局模块，但是 CSS3 的问题是你会很快陷入技术规范中。这有望成为 css11 的切入点。W3C 需要采纳 CSS3 设计者的语言，尤其是在布局方面。

我从你的网站上看到你也是表情引擎的粉丝。是什么让你选择了这款 CMS，而不是如今众多的其他 CMS？

我现在用 EE 已经四年了。当我第一次开始使用它的时候，多亏了 [Colly](http://www.colly.com/) ，我几乎受够了[movable type](http://www.movabletype.org/)——我不得不到处修改它，只是为了让它做我想做的事情。此外，表达式引擎的概念非常类似于我们当时在 BBC 使用的 CMS，所以对我来说学习曲线相当浅。

在网络上或其他地方，有没有其他你信赖的工具，甚至是你觉得有用的很酷的东西？

我虔诚地使用[背包](http://backpackit.com/)。它组织我的生活。从日历到待办事项，从购物清单到企业现金流，无所不包。没有它我会完全迷失。

你即将出版一本名为 [*的书:五个简单的步骤:为网络设计*](http://fivesimplesteps.markboulton.co.uk/) 。你能给我们概述一下你将在书中涉及的内容吗，以及你认为设计师在读过这本书后能学到什么？

那本书已经出版很久了。它是在将近 18 个月前宣布的，而且已经非常非常晚了！也就是说，这本书将于今年完成，最初将提供 PDF 下载，然后是限量版精装本。

该书的前提很简单。有很多人——设计师和开发者——会从一些简单介绍的图形设计理论中受益。这个理论可以应用到他们的日常工作中。这将是一本由五部分组成的实用书籍:研究和想法、色彩、排版、网格系统、形式和功能。

在建立自己的设计公司之前，你已经在这个行业工作了很多年，作为一名自由职业者，独自工作的动力是什么？

我在英国广播公司工作，业余时间也从事自由职业。在我离开的一年前，我实际上在做两份工作。结果一切都受到了影响——我的健康，我的工作，我的关系。最后，有一份工作不得不离开，那就是 BBC 的工作。

这是一个容易做出的转变吗？

当我回头看的时候，是的。但它令人难以置信地害怕，现在仍然如此。有些早晨醒来时，我仍然非常担心未来的几个月，如果没有预定的工作。

你最近雇用了你的第一名员工。作为企业主和设计师，你是如何发现招聘流程的？

在加的夫相当困难。因为这里的设计基础完全集中在广播或印刷上。优秀的网络设计师很难找到。

看看你的投资组合，你已经吸引了许多著名的客户。你认为哪三件事最有助于赢得高质量的客户？

1.  能够融入他们的工作实践和环境。对于那些需要感觉你是团队一员的客户来说，在 BBC 客户端工作被证明是非常有价值的。

3.  能见度。幸运的是，[我的博客](http://www.markboulton.co.uk/)相当受欢迎(虽然此刻正在积灰！).

5.  做得很好。本质上，归结起来就是这样！

在你参与的项目中，你最喜欢的项目是什么？与其他项目相比，它有什么特别之处？

那是一个困难的问题。总的来说，我喜欢我参与的所有项目。最有收获的可能是我们今年早些时候为一个主要的电视体育网络设计的网络应用程序。这是一个持续了几个月的项目，我们从产品一开始就参与其中——从一开始就参与其中。我们能够影响很多产品的方向，不仅从 UI 设计的角度，而且从功能的角度。是啊，这是一个很好的。

灾难故事呢？有没有一次你发现自己从一个失败的项目中吸取了惨痛的教训？

几乎每月一次。我不会说它们是灾难，但是认识到你的错误并采取行动是很重要的。有时我们不得不放弃已经失败的项目。大多数情况下，这是因为沟通的中断，或者对期望的误解。这就是为什么最初的咨询阶段在一个项目中总是如此重要。这时你就和客户建立了关系。

这总是远程工作的困难之一。语气很难在电子邮件中传达，或者通过 [Basecamp](http://www.basecamphq.com/) 传达。我们总是喜欢以现实生活中的会议或电话讨论开始。

你不在电脑前的时候喜欢做什么？

目前，花时间和我的新女儿在一起。除此之外，我还喜欢钓鱼，我觉得这很放松。每年一到两次，我和妻子会花几周时间在阿尔卑斯山滑雪。山有一种对灵魂非常有益的东西…

如果你不是一名网页设计师，你会是…

一名宇航员。不，说真的，我可能会设计别的东西。我想这是我的天性。

离开伦敦的快车道去威尔士生活怎么样？(免责声明:我也生活在阳光明媚的威尔士！)

那是我妻子的错。她在卡迪夫找到了一份工作，我们已经厌倦了在伦敦的生活。我在曼彻斯特的边缘长大，所以我拥有两个世界的精华——起伏的乡村和半小时车程内的城市。我们在卡迪夫也有同样的地方，但我们也有海岸。我一刻也不后悔。

最后，感谢您抽出时间接受 SitePoint 读者的提问！

没问题！谢谢你邀请我。
