# Java SE 6:名字里有什么？

> 原文：<https://www.sitepoint.com/java-se-6-whats-in-a-name/>

谈到 Java 的命名，Sun 最终做出了明智的决定。为了了解这是多么史无前例，让我们停下来上一堂快速的历史课。

1996 年，有了 Java 1.0。一年后，我们有了 Java 1.1。然后在 1998 年底，Java 1.2 出现了，但 Sun 的营销头脑姗姗来迟地决定它值得一个全新的版本号，所以在发布后被重新命名为 Java 1.2 版。

到 2000 年底，Java 已经分成三个部分:Java 2 标准版(J2SE)1.3 版，用于桌面应用开发；Java 2 企业版(J2EE)，将 J2SE 扩展到构建企业应用程序(包括网站)；Java 2 Micro Edition (J2ME)，一个用于构建移动和嵌入式设备应用程序的精简版 Java。

J2SE 1.4 于 2002 年推出，然后在 2004 年我们推出了 J2SE 1.5。Sun 的市场部又一次在最后时刻介入，给了它一个新的版本号:Java 2 Standard Edition 5.0，version 1.5。如果这不是真的，那将是难以置信的。

在本周的 JavaOne 大会上，Sun [宣布了](http://www.onjava.com/pub/a/onjava/2005/06/29/javaone2005.html)一个新的 Java 命名方案。这不仅是一次明智的选择，而且在它将应用的第一个版本发布之前就已经宣布了。

Java 的下一个桌面版本将被称为 Java 标准版 6(简称 Java SE 6)。同样，我们会有 Java 企业版(Java EE)和 Java 移动版(Java ME)。
