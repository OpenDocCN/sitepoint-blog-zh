## 谈话要点

> 我试着记住我的代码不是我自己。我建造的东西，我写的东西，那些是我人格的延伸，但它们不是我的人格。它们与我的身份无关。如果我错了，我可以看着它，如果我愿意与之抗争，如果我认为我可能是对的，我会争辩。如果我意识到，<q>是的，我错了，</q>那么只要我愿意从那个错误中吸取教训，下次我写文章或对文章进行更正时，我认为那个批评——即使它是一个非常邪恶的批评，就像他们对它很刻薄——如果他们提出的观点是有效的，我从中吸取了教训，它仍然是有用的反馈。

* * *

> 我在博客上写下了我的整个哲学，就叫它自焚吧。如果你想尝试什么，可以考虑一下。你可以计划。你可以做一切准备工作，但除非你尝试过，否则你还没准备好。在某一点上，你只需要点燃火柴，看看是否会成功。

* * *

> 我一周工作 90 个小时，这太疯狂了。我完全是在努力工作，当你死的时候就睡觉，就像荣誉徽章一样。我的机构在发展。我有很棒的客户。我在英特尔和 PlayStation 这样的公司工作，从职业角度来说，我再开心不过了，但从个人角度来说，我很痛苦。

* * *

> 我决定让一切都过去。我当时想，我出局了。如果这意味着我不用再这样生活下去，我会毁掉我的事业。我坐下来，写下对我来说重要的事情。我当时想，好吧，我想过正常的生活。我想有爱好。我希望能够和我的朋友一起出去玩，而不用担心我的电子邮件是否会关闭。

* * *

> 我觉得很酷，就是这样。我要把它烧掉。我的客户要走了。我的收入会大幅下降，但不管怎样。我会很开心的。当我这样做时，我变得更有效率。我的生意蒸蒸日上。我发现我的工作效率比以前高了很多，这让我大吃一惊。我当时想，发生了什么事？我以为这是我的末日。我想我辞职是为了赚最少的钱，这是我为了拥有美好生活而做出的牺牲。结果是我的职业生活和个人生活都变得更好了。

* * *

> 如果我们看看具体的策略，我发现，在很多情况下，当我们发现自己一周工作 60、65 个小时时，如果我们真的看看我们是如何度过这些时间的，大部分时间都被浪费了。

* * *

> 对于你试图同时做的每一项任务，你会损失 20%的时间。如果你在做一件事，你 100%的时间都会用在这件事上。如果你在做两项任务，40%分配给一项，40%分配给另一项，20%分配给了上下文切换。三，突然你会因为上下文切换而损失 60%。如果你坐在办公室里，试着回复邮件，试着和这边的人聊天，试着工作，你什么也做不了。

* * *

> 我称之为原子技术:……一切都需要面向行动、及时、可拥有(单一所有者)和可测量。

Jason:

哦，这是个好问题。我想在我职业生涯的这个点上，我大概是 12.0 版本，也许是 12.3。如果我们在做像 12.3.157 这样的语义版本，因为我大约每 12 秒发布一个补丁。我觉得我之所以会这么说——我的版本号会这么高——是因为我的职业生涯是一场从一个追求到另一个追求的冒险，而它们通常没有那么大的关联。

我是作为一名音乐家起步的，这让我开始设计，因为我们没有钱，所以有人必须制作我们的海报……这落到了我的头上，因为——我不知道——当我还是个孩子的时候，我经常画加菲猫或什么的。当我开始设计海报时，这扩展到了其他商品的设计，然后像 MySpace 这样的事情发生了，所以我开始学习一点 CSS。然后 MySpace 就不酷了，所以我需要学习怎么做网站。在 21 世纪初，我建立了我的第一个糟糕的基于表格的网站，我发现我喜欢它，所以我就这样做了。

然后我试图去上大学，并从大学辍学(这对我来说并不奏效)。所以我回来了，在一家印刷店做平面设计师。我在一家金科公司做产品经理。我做过各种不同的工作。从那时起，我在一家网络开发自由职业代理处结束了自己的职业生涯。我在兼职。我在 Kinko's 有一份全职工作，但我真的不想呆在那里，所以我开始兼职做自由职业项目——起初只是前端，后来我意识到我需要后端，所以我开始做后端。然后我写了一本关于 PHP 的书，然后我写了一本关于 jQuery 的书，所以我有了一个非常奇怪的轨迹，我从前到后，从前到后。

现在我又回到了前端。我的正式头衔是 IBM 的高级前端开发人员。是的，这是一次彻底的改造，也是决定我想去哪里的过程，中间有很多小的调整。

David:

有趣的是，在我听来，你像我们的许多客人一样，是自学成才的。你没去拿工程学位。

Jason:

我没有，没有。我真的没有想到我会喜欢电脑。我小时候想画眼线穿紧身牛仔裤弹吉他。电脑与摇滚明星的整体形象格格不入。是的，这绝对是一种迂回的方式，更多的是出于需要而不是兴趣。利息随后而来。

David [4:10]:

我猜你是从 PHP 开始的。

Jason:

嗯，我从 HTML 开始，它是 HTML，CSS，然后当我意识到我不想为我所有的客户做所有的编辑时，那就是我跳入 PHP 的时候。它开始螺旋式上升，我在 PHP 中学到的东西会让我想学一些其他的东西，例如 JavaScript，然后这又让我想学一些其他的东西。然后我进入了数据库和所有这些，数据库架构。然后，我想实现更多的自动化，所以我进入了 DevOps，然后随着 Grunt 和 Gulp 等工具的出现，构建管道革命发生了，所以我开始沉迷于这些东西。

我甚至没有谈到这一点，但我的另一个副业是我做过程优化的咨询，让人们建立管道或教他们如何做前端工具，自动化过程和强制代码风格之类的东西。我有很多兴趣。我觉得他们大多倾向于那种我们如何能做得更好，更有效？

Tim:

在写这本关于 PHP 的书的过程中，你有没有想过，我可以现在就回去设计，没有人会注意到？我想这是写一本关于 PHP 的书和所有深层的后端类型的语言和设计之间的巨大不同的联系，这是一个非常不同的规模。你是如何完成这一转变的？

Jason:

回答你第一个关于想回到设计的问题，我从来没有。对我来说，从哲学的角度来看，设计和开发有很多重叠之处。无论是设计还是开发，你都在试图找出适合问题的正确形式。这可能是在设计方面，你试图找到一种方式来塑造颜色和信息的呈现，以便访问该网站或看到该设计的人获得你希望他们获得的信息，并对他们从该网站获得他们想要的东西感到自信。感觉质量很高，他们有信心，如果他们要购买或询问更多信息，或者你试图让他们做的任何事情，他们都可以做到，而不用担心会被敲竹杠或不完整，或者更糟糕的是，他们不会感到沮丧，也不会无法找到他们最初想要的东西。

随着发展，这是同样的事情。从后端的意义上来说，你试图从用户那里获取信息，并利用这些信息做一些事情，这样无论从另一端传回来什么，或者我们存储在后端的什么，都将成为对链条中的任何一个环节都非常有用的东西，无论是销售部门、客户服务团队还是客户本身。他们得到的是完成工作所需的东西。对我来说这一切都像是设计，但我不知道。有时候，我把它抽象了很多。

David:

这让我想到的是这些天来设计思维作为设计哲学的哲学趋势。

Jason:

是的，这在 IBM 是一件大事，我刚加入的时候并不知道。但是我刚刚坐了整整一个关于它的研讨会，这真的很吸引人——观察一个问题并试图真正思考结果的想法。他们有一个很酷的地图，你把问题看做是真实的，然后你试着得到所有的步骤。因为我觉得很多设计的缺点之一就是你只看到它令人兴奋的部分。你看着购物车，或者你看着主页，但是如果你不彻底，你会错过链中的其他链接，比如你结账后的感谢页面，或者沿途的过渡页面，或者让你知道你在这个过程中处于哪个步骤的小弹出消息，所有这些事情加起来就是一个好的体验。

David:

我认为，当你开始考虑应用程序不一定有页面，而是有更多的状态时，这一点变得更加重要。

Jason [8:00]:

是啊是啊。我真的很喜欢。这是我现在最喜欢的关于网页设计的事情之一，那就是我们不再认为网页是信息的载体，相反，我们从状态的角度来考虑它。它为许多很酷的事情打开了一扇门，比如基于状态的 UI 测试，而不是必须旋转整个幻影 JS 服务器来做这类事情。你可以通过输入一个 JSON 对象来测试你的 UI，我喜欢这样。

Tim:

我认为你在一秒钟内从设计过渡到开发是如此容易，这让我大吃一惊。太棒了。干得好。我现在信服了。我没有问题要问你了。

*【笑声】*

David:

我有更多的问题，因为你提到的一件事是你在 PHP 上做后端工程工作，然后哦，顺便说一下，我写了一本关于这个主题的书。这不是每个开发人员都想做的事情。你是怎么得出这个结论的？

Jason:

我有一个哲学，就是先做，然后问是否可能。当我第一次接触 PHP 时——这是我开始用 PHP 编程的几个月内——我决定我要写 WordPress 杀手；我要建立一个新的 CMS，它会杀死 WordPress，我会出名。整个世界都将改变。这是典型的 19 岁狂妄自大。

但是我的方法不仅仅是写代码，[我写了一篇关于我写的代码的文章](https://css-tricks.com/php-for-beginners-building-your-first-simple-cms/)。我找到 CSS-Tricks 的克里斯·科伊尔说，<q>嘿，我能在你的网站上发表一篇文章吗？</q>他说，<q>你可以。</q>这是在他有很多客座博文之前，他说，<q>我不为文章付费，但如果你想，你可以。</q>我发表了，它就爆了。在我谈论它做得有多好之前，让我先说一下，这是一篇糟糕的文章。我写的代码很好，但是它充满了安全漏洞。它还没有做好生产准备。这绝对不是黑仔。这是一个概念的证明，你好世界 CMS。我写了这篇文章。克里斯把它放了出来，这很棒。我的公众职业生涯要归功于克里斯·科伊尔，我想我从来没有告诉过他，但是嘿，克里斯，如果你在听的话…

那篇文章然后就火了。它登上了 Digg 的首页，这是在 Digg 进行大规模重新设计之前，就像凯文·罗斯和那些家伙有了那个巨大的播客一样。所以它上了 Digg 的首页。它瘫痪了克里斯的网站。然后它又出现了，得到了大量的评论。然后它攻击了黑客新闻(或其中一个网站),然后那些家伙攻击了它。他们喜欢，<q>这是可怕的代码。你做了一件可怕的事情。你是个坏人。</q>

这真的很有趣，因为一半的人说，<q>哦，我的上帝，这太棒了，因为我能理解你在说什么，</q>还有一半的人说，<q>这不够专业。你没有保护好安全。你没有报道这个。这是糟糕的代码。</q>

所以我学到了一些东西，第一是在我试图告诉人们如何做他们的工作之前，我应该做更多的研究。第二，对初级的、深入的材料有巨大的需求。这本书就是这样诞生的。一位编辑给我发了邮件。她的名字是米歇尔·洛曼。她在一家出版社工作(我不确定她是否还在那里)。她只是说，<q>嘿，我读了你关于 CSS-Tricks 的文章，这对初学者来说很容易理解。要不要给初学者写一本关于 PHP 的书？</q>现在提醒你，我很不幸没有资格写这本书，但我当时并不知道，所以我写了。基本上，我从头到尾浏览了一遍 PHP 手册。我构建了一些示例应用程序。我尽我所能地学习，然后我基本上以一种更容易被那些不知道我是一个程序员的背景的人理解的方式重新表述了 PHP 文档。

这本书卖得相当好。当时我不知道如何编辑，所以在源代码中出现了一些错误。我们有这样的问题，但总的来说，这本书很受欢迎，人们喜欢它，这就是为什么他们要求我写…之后我又写了两本。

但是，是的，这是一个非常有机的经验。我决定和克里斯一起做一些事情，因为我和克里斯一起做了一些事情，我得到了出版社的一些人的注意，因为我得到了出版社一些人的注意，这让我……有一段时间，我在《粉碎》杂志上写了一些东西，直到某个时候，它崩溃了。然后我在 Envato 为 Tuts+帝国写了一段时间。我做了一些类似的事情。它们都是有机的东西。我遇到了一个认识某人的人，这让我获得了另一个作家或演说家或作家的职位。

Tim [12:38]:

你提到了一些我们在版本控制节目中经常谈到的东西，这些人试图告诉你，<q>你不适合做开发人员，</q>因为他们在你的代码中发现了一个缺陷。当你写一些吸引公众眼球的东西时，尤其是写一些像砸杂志和 CSS-Tricks 这样的东西时，你是如何处理这种恐吓的？

Jason:

有一次我为 CSS-Tricks 写文章，有几次我为 Smashing Mag 写文章，老实说，部分是因为我不知道得更好，部分是因为我愿意承认我不是世界上最聪明的人。无论我写的代码有多好，都会有一些我没有想到的问题。当我年轻的时候，我没有完全理解 XSS 或 SQL 注入的一些风险。当时，PHP 之类的东西使用一个名为 PDO 的数据库库，它非常擅长代表你处理 SQL 注入，但我当时并不知道这一点。它要么是新的，要么没有得到很好的宣传，要么就是我的研究不够彻底。所以我用的是老派的 MySQL 查询 PHP 函数，我想现在他们已经说过了，<q>永远不要这样做。太可怕了。</q>我不知道更好。所以我把它放在那里，有人指出来。

我试着记住我的代码不是我自己。我建造的东西，我写的东西，那些是我人格的延伸，但它们不是我的人格。它们与我的身份无关。如果我错了，我可以看着它，如果我愿意与之抗争，如果我认为我可能是对的，我会争辩。如果我意识到，<q>是的，我错了，</q>那么只要我愿意从那个错误中吸取教训，下次我写或者修改文章时，我会认为那个评论——即使那是一个非常邪恶的评论，比如他们很刻薄——如果他们提出的观点是有效的，我从中吸取了教训，它仍然是有用的反馈。

David:

如果你在另一边，如果你习惯性的过度劳累，我曾经掉进过死亡陷阱。实际上，这很有趣，大卫，因为我正相反。我是那种在做咨询工作时过度劳累的人，我不能对任何事情说不。我差点写了本书。我对写书一无所知。顺便说一句，我很感激我拒绝了那个请求！但话虽如此，我选择忙碌，只是试图承担一切，周末工作，不与朋友出去玩，就像杰森说的那样。

我开始意识到我做得最好，当我最放松的时候，当我休息的时候，当我不试图去学习下一件事是什么的时候，我把我最聪明的大脑带到桌子上，并且领先于其他人。事实上，当我是最后一个看到最新事物的人时，我在我所做的事情上是最好的。我刚刚用 React 编写了一个生产应用程序。我也不知道能不能这么说。承认这一点，这可能会让我失业，但老实说，这些行为帮助我冷静下来，同时在工作中表现得更好。

David [36:34]:

我认为我有一个优势，因为我开始做远程工作的时候，我还是一名全职员工，很久很久以前，远程工作甚至是人们正在考虑的事情——当时远程办公是你必须积极向你的雇主推销的东西，你必须为此进行谈判，你必须做出各种各样的让步。但我开始作为全职员工做远程工作。当时我在苹果电脑公司。我发现，因为我是远程工作，这一天永远不会结束。我早上一起床就开始工作了。我一直工作到睡觉，然后第二天一醒来就又开始工作。

其中一部分是试图向我的雇主证明，作为一名远程工作者，作为一名远程工作者，我实际上是在做有成效的工作。但部分原因是因为我不知道如何给自己设定界限。正是在这个角色中，我了解到当你控制自己的时间表时，设定界限的重要性。这让我很好地进入了我自己的咨询工作，因为现在，我知道不能让自己的一天延长到晚上，延长到晚上。我知道什么是我的生产时间，什么是我的非生产时间，以及如何区分它们。

Tim:

我认为重要的是要记住，我们有时都会在这方面失败，我们也不应该过于气馁。人类不会在家坐在电脑前九个小时。这是一个非常新的东西。我们中的许多人刚刚成熟。如果你真的发现远程办公或在工作中工作，如果你因为发现你没有完成足够的工作或你自己超负荷工作而感到沮丧，你肯定不是唯一的一个。因为不仅仅是<q>哦，你应该这么做</q>。它涉及到你作为一个人如何工作，对吗？它包括你自己的怪癖是什么，对你来说专注于一项特定的任务有多容易。我认为这是一个非常微妙和难以理解的事情。
