# 用 10 个步骤设计有效的导航

> 原文:[https://www.sitepoint.com/navigation-10-steps/](https://www.sitepoint.com/navigation-10-steps/)

起初，设计网站的导航可能是一项相当棘手的任务。你需要研究你将占的类别的数量，你的访问者最有可能点击的地方，最能满足用户的配色方案，以及许多其他方面。有这么多的变量，导航设计看起来就像是没有正确答案的情况。今天，我们将讨论设计导航的任务——希望能给你一个良好的开端，为你的用户创建一个最好的导航系统。

在开发我自己的网站时，一天晚上我坐下来，在纸上画了一些可能的设计。在我看来，这是规划网站的最佳方式——在完成这个阶段之前，甚至不要打开电脑！我遵循的流程是首先设计我的导航，然后围绕它开发网站，将导航作为主要关注点。

当然，有很多方法可以为你的站点创建导航系统，从简单的文本驱动的工具栏到最先进的 Flash 动画界面。你面临的最重要的决定是选择一个适合你的网站和用户的方案。

记住，这可能不是一件容易的事。四处征求意见，设计一些试用导航系统，让你的朋友、家人、同事或任何你认识的人告诉你他们最喜欢什么，以及他们认为你的哪些设计最有效。目标受众的可用性测试当然是理想的——而且在预算范围内也不是不可能实现的！更多信息请参见 SitePoint 的可用性部分。

##### 十大导航提示

使用这个大纲作为一种检查表，以确保您的导航系统满足这些基本要求。如果是的话，你就已经成功了！

***1。不要让你的用户猜测！**T3】*

有时有必要向你的访问者灌输一些信息。仔细决定你的导航应该放在哪里，让它突出来。那么你的用户就不会坐在那里漫无目的地点击鼠标，徒劳地寻找某种方向。

***2。**居高不下*

大报编辑将他们最重要的信息——最新的标题、重要内容等——放在“版面上方”，意思是放在报纸的上半部分。从同样的角度考虑你的页面，尽可能把重要的信息，包括导航，放在最上面。这不仅意味着导航会首先在用户的浏览器中加载，而且会更快地“点击”用户。然而，在这一点上，请阅读技巧 3 中的警告！

***3。横幅下方***

听说过雪盲症吗？嗯，在网络用户中有一种被称为“横幅失明”的情况。通常，用户自然会忽略你网站上任何横幅广告上面的内容——他们只是认为这是进一步广告的空间。尽管这可能很诱人，但是不要在你的横幅广告上放置任何导航元素。

***4。一致性是关键***

我怎么强调这一点都不为过。在你网站的每一页上，无论是你的论坛，你的链接页，还是其他任何地方，把你的导航放在同一个地方，用同样的风格，不要改变任何东西！这样，你的用户就知道在哪里可以找到它。

***5。**不要冒险*

看到你的网站鹤立鸡群总是好的，但是当涉及到导航的时候，请试着和其他人融合在一起。这样，普通的互联网用户会在某种程度上习惯你的导航方式——就像他们在其他网站的类似系统上体验和学习的那样——而不需要学习任何新的东西。使用您的导航系统将会很容易！

***6。**添加一个‘主页’按钮*

你的主页是网站上最重要的页面，所以要确保你的用户总是知道如何回到主页。“主页”按钮不仅可以让点击你网站链接的人回到他们开始的地方，还可以让登陆你网站子页面的人找到你的主页——这可能会给你的网站带来新的回头客。

***7。保持速度***

不要误以为你的用户拥有超快的互联网连接。他们没有。尽管事实上很大一部分互联网社区确实有宽带(或类似的)连接，但仍有大量的冲浪者坚持使用 56k 调制解调器，他们喜欢尽可能地看到快速加载的页面。优化你的图片，你的 HTML 和你的样式表，以确保一切尽可能快地加载。

***8。质量，而不是数量***

构建你的导航，让它把用户“分流”到越来越具体的信息中。互联网用户在每一层都有几个选择(和导航按钮)比几百个更开心。使用带有适当导航的子部分和子类别，使用户能够快速找到他们想要的特定内容。

***9。网景没有死！**T3】*

记住，人们使用其他的浏览器和分辨率。在发布之前，检查你的网站在所有可能的浏览器和分辨率下的导航。通过这种方式，您可以在其他人之前发现您喜欢的 JavaScript 代码中的不对齐和错误，避免您在将来遇到很多尴尬。

***10。省去不重要的东西***

像“联系我们”和“隐私政策”这样的链接最好不要出现在你网站的主导航中。这些最常见的地方是在页面的底部。

##### 最流行的导航方法

下面是创建站点导航方案最常用的方法。在你决定使用哪一个之前，确保你考虑到了每一个的所有方面。

***1。文本驱动工具栏***

我会说，这些是最不能有效地抓住用户注意力的。对于静态 HTML，除了大号字体和亮色之外，你没有什么办法让它比页面的其他元素更突出。尽管这类导航系统加载速度极快，而且可以很快改变，但它们看起来往往很乏味，在某些情况下，实际上会让你的用户对网站看得更远。

一种非常受欢迎并且不费吹灰之力就能实现的效果——只需一点 JavaScript——是翻转表格单元格效果。在某些情况下，这些远优于滚动图形，而且，它们只需要很少的时间就可以加载。

**优点:**加载快
T3】缺点:可以平淡

***2。图像工具栏***

虽然图像有很大的潜力做好，而且在 99.9%的情况下，他们经常被误用。糟糕的设计、大尺寸的图片和花哨的颜色会让你的用户立刻失去兴趣。考虑使用单个图像映射是值得的，这将大大减小图像的大小——服务器只需执行一个 HTTP 事务，而不是几个(如果您有几个小的按钮图像)。

确保你为你的图片指定了 ALT 属性，否则那些使用纯文本浏览器的人，那些关掉图片的人，或者那些等不及页面加载完毕的人，仍然可以毫无困难地浏览你的站点。

**优点:**可以看起来很棒
缺点:可以载入很慢

***3。闪光菜单***

Flash 菜单现在变得越来越流行，主要是因为越来越多的人可以快速上网。但是要考虑到一个事实，你的很多用户仍然会使用 56k 调制解调器，所以不要过度使用你的闪光灯。

事实上，有时 Flash 可能是一个大禁忌。如果加载时间超过 10 秒，就放弃它。在任何情况下都不包括预装器。人们不希望在开始浏览网站之前就收到“正在下载”的信息。

也要考虑那些电脑上没有安装 Flash 插件的用户。虽然很少，他们不希望必须下载一个全新的软件才能使用你的网站。

**优点:**可以看起来很惊艳很专业
**缺点:**不是每个人都有 Flash 插件，可以缓慢加载

当然，在设计你的网站时，还有其他的可能性需要考虑；这里描述的方法只是最流行的。

##### 其他考虑

***让你的导航适合你网站的主题***

例如，如果你经营一个食谱网站，考虑为每个部分制作与食物和/或烹饪相关的小图标。也许你可以包括一个小的卡通蛋糕，标题链接到“蛋糕烘焙”部分。

***让你的导航适合你网站的目标受众***

如果你的大部分用户是儿童，为什么不让你的导航看起来像是属于儿童网站的呢？例如，如果你有一个笑话区，你可以用一个小卡通笑脸来象征讲笑话。

##### 你应该不惜一切代价记住的 3 点！

如果你决定忽略我到目前为止所说的一切，请阅读这三点——我认为它们在你设计网站导航时是最重要的。

***1。测试测试测试！**T3】*

确保在尽可能多的浏览器上彻底测试你的站点导航。这些应包括:

*   Internet Explorer 5+
*   网景 6.x
*   网景 4.x
*   歌剧

此外，请确保在多种屏幕分辨率下进行测试，这些分辨率应该是:

*   800×600
*   1024×768

从头到尾检查一遍。确保任何 JavaScript 都能在所有平台上运行。如果没有，确保不产生 JavaScript 错误。记住，对你的导航进行“过度测试”是不可能的！

***2。**善待你的用户*

把你最重要的按钮放在导航栏的最前面。这可能是“主页”链接。此外，向用户详细说明每个链接。没有文字描述的可爱小图标是远远不够的。你的访问者应该能够在点击链接之前准确地知道每一部分的内容。

***3。安置好***

确保你的用户可以清楚地看到你的导航。不要用其他内容包围它——让它从你的其他内容中脱颖而出，你就不会出错。此外，请记住前面讨论的“折叠线以上”和“横幅失明”的注意事项。

记住，导航是每个网站的一部分，没有它，互联网就无法生存。考虑到这一切，我祝你在网上创造你的新杰作时好运！
