### [卢尔科/菲尼克斯](https://github.com/lulco/phoenix)【16★】

作为非常流行的 Phinx 的替代方案，Phoenix 是一种为任何应用程序设置简单且自动的数据库迁移的方法。你可能想知道“为什么”？为什么不呢？如果新的轮子比我们以前使用的更强、更结实、更轻，那么重新发明轮子并不一定是件坏事。

Phoenix 更容易使用，具有更简单的 API，并且支持模块，正如作者所解释的:

> 我们正在为多个客户端创建一个应用程序，所以我们需要模块。每个客户端安装了不同的模块。使用 Phinx(在旧版本中),我们必须运行所有的迁移，例如，这也创建了不必要的表。对于 Phoenix，每个模块都有自己的迁移文件夹，只有在安装了模块的情况下，该文件夹才会注册到迁移配置中。

### [海脊/船尾](https://github.com/zeeshanu/opus)【502∝)

Opus 是面向团队的开源知识库应用程序。现在，您的团队也可以拥有一个内部指南、描述、规则和文档集合，来说明您如何以及为什么要做您正在做的事情，从而使新团队成员的加入或解决已经解决的问题变得更加容易。

非常适合编码实践、常见客户问题的解决方案、指南、安装特定软件的 wiki 条目等等。它的开发有点笨拙，没有考虑到最佳实践(没有测试，控制器做所有的事情，方法名中的错别字)，但是正如那句老话所说:**谁发货，谁就赢。**此外，这些错误是*为项目做贡献的绝佳*机会！

### [头脑/引擎](https://github.com/Minds/engine) [50 ★]

如果乳齿象是“干净的 Twitter ”, Minds 则截然相反。这个看起来像 Google 的社交网络关注言论自由，允许任何人说任何话(在合理的范围内——没有儿童色情等)。)是完全开源的，而且([也许错了，](https://www.reddit.com/r/PHP/comments/65bl3t/the_repo_of_the_php_backend_api_for_mindscom_the/))对所有用户都拥有加密和完全隐私。

如果你想了解高流量社交网络的后端实际上是什么样子，请查看回购。不过，他们肯定需要代码质量部门的帮助，所以如果你渴望为下一件大事做出贡献，这可能就是它了，Twitter/Youtube 由于审查而迁移到更开放的网络。

### [maze chazer/无声](https://gitlab.com/MazeChaZer/soundless) [3 ★]

我们的第一个 Gitlab sourcehunted repo，Soundless 是一个 PHP 的功能模板引擎。这是一种完全不同的编写模板的方式，虽然它可能会让你团队中的前端人员和设计师更加困惑，但它有它的用处。就像回购协议本身所说的:

> 无声给你
> 
> *   自动转义
> *   简单的源代码格式，没有 HTML 语法/空白
> *   无缝且类型安全地集成到应用程序的其他部分
> *   组合、抽象、类型安全的所有 PHP 特性…
> 
> 可能的缺点
> 
> *   语法有点笨拙
> *   与 HTML 相关的工具(如 IDE 的自动补全/检查)无法工作
> *   性能应该还可以，但是还没有测试过

就我个人而言，这对我来说太复杂了——我从不在后端编写复杂的模板，因为如果需要的话，我所有的视图现在都是 VueJS。但是我可以看到人们在高度复杂的应用程序中使用它们，这些应用程序还不是单页的，并且可以想象它在像社交网络这样实际上没有 JavaScript 也能工作的环境中特别强大(这在今天将是一个小奇迹)。

### [翼龙/面板](https://github.com/Pterodactyl/Panel) [169 ★]

有点小众，翼龙面板是一个开源的自托管控制面板，面向用户、网络和游戏提供商。它可以让你控制游戏和服务器，如《我的世界》，反恐精英，TF，团队发言，咕哝，等等。如果你曾经想为任何一个目的托管你自己的服务器，这个面板是必须的。

该面板使用 Docker 来管理服务器并保持可伸缩性，显示每台服务器的 CPU / RAM 使用情况及其当前数量，允许不同的主题，甚至提供了一个 API 来轻松访问所有托管服务器。

这个社区是强大而健康的，在 Discord 和论坛上有官方的讨论渠道，深入的安装说明等等，但是它仍然可以使用更多的测试人员和玩家来获得反馈。此外，有相当多的问题需要处理。

### 穷人/phpmnd【31】

一个检测类和方法中可能不应该存在的硬编码数字的包。例如:

```
if (mb_strlen($password) > 7) {
              throw new InvalidArgumentException("password");
         } 
```

这里的 7 是一个神奇的数字，不应该在这里——密码长度不应该在一个类中任意定义，而应该通过配置或常量来定义。正如该项目所定义的:

> [a]幻数是一个数字文字，它没有被定义为一个在以后阶段可能会改变的常数，但因此很难更新。不加解释地直接在源代码中使用数字是一种糟糕的编程实践。在大多数情况下，这使得程序更难阅读、理解和维护。

这似乎是对任何测试套件的一个很好的补充。对于增加的功能，有一些 PRs 和问题，所以请深入研究并帮助改进这个开发良好且已经相当稳定的代码质量工具。
