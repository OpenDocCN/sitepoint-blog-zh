# SWF 和 SVG–你应该选择哪一个？

> 原文:[https://www.sitepoint.com/v-svg-which-should-choose/](https://www.sitepoint.com/v-svg-which-should-choose/)

过去，一个网站只需要好的内容就能成为前沿。现在，任何像样的网站都将使用丰富的多媒体来吸引和留住访问者。有两种主要的多媒体解决方案可供您使用:Macromedia 的 Flash 或结构化矢量图形格式，也称为 SVG。

这两种格式背后的技术并不新鲜。这是一种叫做矢量的图形格式。传统 Web 图形(比如那些烦人的动画 gif)和矢量图形的区别在于最终图形是如何创建的。GIF 和 JPEG 图形光栅图形—每个图像都是逐个像素创建的。另一方面，矢量图形使用数学来构建图形。程序绘制 A 点、B 点和 C 点，在创建的空间中填充颜色，瞧！你有一个三角形。通常，矢量图形的文件大小比传统的 GIF 和 JPEG 图形小得多，这使得 Flash 等矢量程序非常受欢迎-迄今为止，已有超过 3.8 亿人下载了 Flash 插件。

Flash 是互联网上占主导地位的矢量图形程序。每天都有数百万用户访问数千个由 Flash 驱动的网站。HTML 本质上是一种静态语言，而 Flash 则不同，它是动态的，是专门为吸引用户而设计的。如果这还不够，每个 Flash 电影通常都很小，通过慢速拨号互联网连接实现类似 CD-ROM 的体验。随着 Flash 广告开始取代恼人的闪烁动画 gif，Flash 的成功可以通过在线广告行业的演变来看。其结果是一个更丰富，更有吸引力和创造性的广告解决方案。在 weather.com、SciFi.com 和 News.com 等受欢迎的网站上可以看到 Flash 广告的例子。

然而，Flash 并不是市场上唯一的矢量图形格式。还有 SVG，结构化矢量图形格式:一种 XML 格式的、可脚本化的图形格式。SVG 没有分享 Flash 在网络上无处不在的存在。它的优势来自它的支持者和支持者:Adobe 和万维网联盟集团，或 W3C。Adobe 是一家大公司，垄断了插图和图形管理程序的市场。W3C 由一手创建了万维网的人蒂姆·伯纳斯·李领导，他建立了 HTML、层叠样式表和 XML 等标准。SVG 是矢量图形王冠的有力竞争者。使用 SVG 的站点包括 [DBx Geomatics。](http://www.dbxgeomatics.com/svg.asp?Language=EN)

但是哪个更好呢？SWF 还是 SVG？

##### 创作过程

标准在网络上来来去去的速度令人震惊。有人记得网络频道吗？在 97 年的 6 个月内来来去去。它没有持续多久，因为建立网络渠道所需的工具还没有被创造出来。用户必须手工编写频道代码，这意味着大多数用户无法使用这项技术。显然，要让一个新概念发挥作用，必须为用户提供工具，让他们能够轻松地使用该技术的全部功能。

Macromedia 在 Flash 中遵循了这一理念。首先，他们在许多产品中内置了 Flash。Flash 是唯一完全公开所有功能的产品，不仅包括图形和动画，还可以访问 Flash 中一个名为 ActionScript 的强大脚本引擎。Freehand 和 Fireworks 是对 Flash 的补充，这两个以 Web 为中心的图形程序可以以 Flash 格式导出图像。此外，Macromedia 的 Dreamweaver 还为网站制作自定义 Flash 文本和按钮。

这很好，但不是每个人都喜欢只使用一家公司的产品。为了让更多的公司支持 Flash 文件格式，Macromedia 提供了制作 Flash 电影所需的源代码。这一举措取得了巨大的成功。现在，所有主要的图形工具，包括 Adobe Illustrator 和 Corel Draw，都以 Flash 格式导出。

导出 Flash 电影的第三方软件也开始涌现。现在有 50 多种不同的程序创建支持 Flash 的文件。这包括创建动画文本、3D Flash 网站和增强动画的产品，如图 a 所示。

相比较而言，SVG 仍然是一个新的竞争者。SVG 基于 XML，这是一种可扩展的标记语言，这意味着它可以非常容易地编写脚本。以下代码是矩形的 SVG。

```
<?xml version="1.0" standalone="no"?>  

<!DOCTYPE svg PUBLIC "-//W3C//DTD SVG 1.0//EN"  

 "https://www.w3.org/TR/2001/REC-SVG-20010904/DTD/svg10.dtd">  

<svg width="500" height="500">  

 <rect x="111" y="78" width="187" height="160"  

 style="fill:rgb(192,192,255);stroke:rgb(0,0,128);stroke-width:1"/>  

</svg>
```

乍一看，这可能看起来很复杂，但是对于 XML 开发人员来说，这段代码非常简单明了。遵循 svg10.dtd 文件确定的一些规则，任何 svg 图形都可以仅使用简单的记事本来创建。

以 XML 为核心，SVG 的文件格式对程序员更加开放，这很棒。唯一的问题是大部分设计师都不是程序员。

也就是说，以 SVG 文件格式导出文件的程序正在出现。最受欢迎的是 Adobe 的 Illustrator 10。Adobe 在 SVG 领域下了很大的赌注(你可以访问他们的 SVG 中心[这里](http://www.adobe.com/svg))，他们的许多产品已经支持 SVG。除此之外，还有大约十几个其他图形工具可以导出到 SVG，如 Beatware 的 e-Picture 2.0、Corel Draw 10 和 ITEDO 的 IsoDraw 5.0。

##### 观众——圣杯

SVG 和 Flash 都是正在成长和成熟的产品。两者都可以扩展到小型计算机设备，如 Palm Pilots，并且足够强大，可以成为拥有数百万点击量的网站的一部分。但必须有一个底线，简单来说就是:Flash player 预装在大多数操作系统上，超过 97%的网络用户都可以访问它；Adobe 的 SVG viewer 的最新版本在 2001 年 11 月初才推出，其渗透率仍然有限。此外，查看 SVG 文件需要下载一个 2.3 Mb 的插件(因为目前还没有本地支持 SVG 的浏览器)。与 219 Kb 的 Flash 插件相比，这已经很大了。

尽管 SVG 是基于成功的 XML 格式，但它目前的用户基础非常有限。创建网站的唯一原因是让尽可能多的用户与你的内容互动——如果他们看不到你的内容，他们就无法与之互动。

虽然 SVG 确实展现了一些美好的前景，并且很可能是一项令人兴奋的技术，但当前对其渗透的限制意味着，在可预见的未来，Flash 仍将是最受欢迎和最可行的矢量程序。
