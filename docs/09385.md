# 你需要 VPS 的五个理由

> 原文:[https://www.sitepoint.com/five-reasons-you-need-a-vps/](https://www.sitepoint.com/five-reasons-you-need-a-vps/)

像许多技术人员一样，我运行自己的在线服务器。虽然我有点过时——我没有向提供商支付我自己的虚拟专用服务器(或 VPS ),而是在我的家庭互联网连接上运行 Xen 服务器。

虽然随着时间的推移进行了升级，但我的服务器已经使用了将近三年。它装有五个硬盘，还有一堆风扇来保持凉爽。噪音还不算太坏，但是在夏天它会显著地提高我公寓的温度。

在购买的时候，VPS 对于个人使用来说太贵了。此外，管理工具也不像现在这样。我不需要可伸缩性(多服务器),在许多方面，拥有 VPS(尤其是更便宜的选项)会比我自己的服务器提供的灵活性更低。

当然，那是过去的事了，事情已经变了。亚马逊现在提供了一个 613MB 内存的微型实例，价格为[一年免费](https://aws.amazon.com/free/)(仅针对新客户)，之后每小时收费 0.02 美元(约 14.60 美元/月)。如果你没有资格获得亚马逊的免费 VPS，RackSpace 的[门槛甚至更低，每小时 0.015 美元(约 10.95 美元/月)，尽管只有 256Mb 的内存。](http://www.rackspace.com/cloud/cloud_hosting_products/servers/pricing/)

提到的 VPS 选项的一个缺点是，在不显著增加这些实例的价格的情况下，不能容易地增加 RAM 的数量。RackSpace 选件的 256Mb 内存足以完成一些基本任务，但如果不小心的话，这是一个限制。为了客观地看待这个数字，现在许多新手机都有 1Gb 的内存。将 RackSpace VPS 实例的大小增加到使用 512 MB RAM，存储也增加一倍，价格也增加一倍。花点钱(262.80 美元)，你可以很容易地买到一台 1Gb 内存和更多存储空间的上网本——想必 CPU 能力也差不多。

话虽如此，亚马逊和 RackSpace 都继续以惊人的速度降价。用不了多久，VPS 解决方案将成为进入个人消费服务器的最便宜的壁垒。考虑到电费和我公寓的廉价非冗余非超级可靠的互联网连接，包括缓慢的上行链路，我的水晶球显示，在不久的将来，我很可能会转向 VPS 解决方案，或许你也是如此。

现在我们已经确定了运行自己的在线服务器有多便宜，问题是你想要一个做什么？在这里，我提出了几乎每个人都能欣赏的五条建议:

## 移动中的信息访问和协作

如果你像我一样，你可能会发现自己处于这种情况；你出门，意外地买了点东西，最后意识到你把购物清单忘在家里了。

你可以一开始就在手机上写下你的购物清单，但是让我们面对现实——这并不总是可行的。有时你的配偶或室友可能想更新列表。为了提高效率，你可能还想在真正的键盘上输入/更新你的列表，但没有同步你的手机。无论哪种情况，你都会发现自己运气不好。

然而，通过运行你自己的个人的、私人的维基，你可以在任何时候访问它——在商店的备用电脑上，在你的电话上，或者任何其他地方。你甚至可以在外出时划掉购物单，这样如果你的配偶碰巧同时外出，他就不会再买同样的东西了。

你可以使用谷歌文档或类似的工具，但是并不是每个人都愿意信任谷歌的文档。不是每个人都想拥有一个谷歌账户。如果你运行自己的维基服务器，你就能控制你的数据。

就这一点而言，同步浏览器书签和其他信息也是如此。例如，Firefox Sync 允许你将书签与你自己的服务器同步。

## 文件存储器

听说过[谷歌音乐](http://music.google.com/)？简而言之，这是一项将你的音乐收藏上传到谷歌服务器的服务，以便稍后将它传输回你的电脑和设备。更简单地说？专为您的个人音乐收藏准备的存储空间。

我能理解对这种服务的需求。我经常发现自己出去想听一张特定的专辑，却发现自己没带在身边。要是我能以某种方式远程访问我的音乐收藏就好了。

谷歌的服务目前只对美国居民开放(不包括我自己)，但这并不重要。如果运行个人服务器(VPS 或其他)，您已经可以授予自己此功能。

它也不一定只限于音乐——你可以用它来播放自己的视频(而不是依赖 YouTube)，运行自己的 [Gitorious](https://gitorious.org/gitorious/) 服务器来共享代码(或者托管一个 [SparkleShare](http://sparkleshare.org/) 文件夹来模仿 Dropbox 功能)，甚至提供对你的照片和电子书收藏的即时访问。同样，有许多软件即服务(SaaS)解决方案可以做同样的事情，并取得不同程度的成功，但运行您自己的 VPS 可以让您控制您的数据，而不是其他人，并为您提供最大的灵活性。

## 通信和社交网络

我要坦白一件事。我没有 MySpace 或脸书的账户。如果我想写博客，我会在我自己的服务器上写——按照我的方式，不用担心被禁止、屏蔽或被删除。除非我的个人博客突然变得比预期的更成功，否则我的家庭互联网连接或 VPS 实例足以在一个基本的 WordPress 安装上存放我的内容。

当我不可避免地开始使用 VPS 时，随着互联网连接更加可靠，我也会用它来存放我的电子邮件——可能会用 [Dovecot](http://www.dovecot.org/) 来访问 IMAP，用 [SquirrelMail](http://www.squirrelmail.org/) 来访问 web 或者可能会研究一下 [ScalableOGo](http://www.sogo.nu/) ,如果需要日历和地址簿同步/共享的话。

没有理由止步于博客和电子邮件；如果你厌倦了 Twitter Fail Whale，你可以运行自己的 [XMPP](http://xmpp.org/xmpp-software/servers/) 服务器来发送即时消息，运行自己的 [StatusNet](http://status.net/) 服务器来发布微博，或者托管自己的闪烁风格的照片/图片收藏。可能性是无限的。

## 游戏托管

当我有一点空闲时间时(这并不经常)，我喜欢玩地板杀人游戏。这是一个基于团队的 FPS 僵尸射击游戏，类似于《死亡左 4 》,尽管它作为 UT2004 mod 已经存在了很长时间。

这个游戏支持专用服务器，没有什么比连接到别人托管的服务器上，然后遇到连接问题更糟糕的了。不幸的是，这种情况并不罕见，而且非常令人沮丧——这通常是由人们在不可靠的家庭互联网连接上托管自己的服务器造成的。

虽然我喜欢托管专用服务器的能力，但如果管理不当，这可能会带来令人沮丧的体验。也有过找不到近的服务器的时候，就自己托管一个服务器吧。这样做的一个不幸的影响是很多人加入，当我完成几个游戏想离开的时候，我不得不把每个人都踢出去——这让我感觉很糟糕。

如果我能让服务器一直运行，直到它不再被使用，然后让它终止，那就更好了。同时，我不想让我的游戏电脑在我不用的时候还在运行。它很吵，而且用了很多电。

我有个想法——如果我用一个虚拟主机来托管我的游戏会怎么样？如果我已经在我的低流量博客上使用了 VPS，那么让它继续运行也没关系。VPS 的唯一缺点是，我会有一些延迟(而如果在本地托管，就不存在延迟)，尽管理想情况下，我会在地理位置附近运行 VPS 以最小化延迟。

由于我在墨尔本，这可能会排除 AWS——新加坡是亚马逊最近的地区，但新的地区一直在增加，AWS 不是唯一可用的 VPS 提供商。然而，我想象运行游戏服务器托管 VPS 将很快成为许多人的实际选择，如果人们还没有在世界其他地方经常这样做的话。

## FreeNX 桌面

这里有一个大胆的想法——通过 [FreeNX](http://freenx.berlios.de/) 在 VPS 上运行整个桌面怎么样？我已经有了一个专门用于这个目的的 Xen DomU 实例，我在需要的时候启动它，而且几乎从我有 Xen 服务器的时候就有了。这听起来可能很疯狂，但它非常有用！

我发现的最好的用途之一是网站和网络浏览器测试。有时，我们办公室的人会抱怨某个服务器不工作。是我们的本地基础设施，比如办公室代理服务器还是防火墙？是我们的 ISP 吗？通过不同的 ISP 远程连接到我们办公室外运行的 FreeNX 实例并查看一下，这非常方便。

另一个实际用途可能是为那些在上网本上运行 Windows(或在 MacBook Air 上运行 OS X)的人，但希望在不运行本地虚拟机的情况下即时访问 GNU/Linux 桌面。FreeNX 符合要求。在你不拥有的没有安装你想要的软件的计算机上工作时，它也很方便——只需将 NX 客户端安装程序与你的密钥一起保存在 USB key 上，你可以插入并运行它。通过这种方式，您可以随时更新桌面上的所有软件、定制和设置，而不会在您所坐的机器上留下任何痕迹。

有时候，您可能希望长时间运行图形应用程序，拥有一个可以在分离模式下运行的 FreeNX 会话非常有用。例如，我妻子不喜欢命令行，但喜欢在 BitTorrent 上下载东西。她通过 FreeNX 连接并运行一个图形化的 BitTorrent 客户端要容易得多，而不是通过 CLI 进入我们服务器上的 Xen DomU。当她第二天早上醒来时，她用她的图形 NX 客户端从她的桌面登录，通常可以看到下载已经完成。

当然，如果稍后需要在本地复制文件，最后一个使用示例在本地硬件上运行 VM 实例比在 AWS 或 RackSpace 云托管中运行效果更好，但是在下载的文件大小相当大但没有很多种子/对等点可供下载的情况下，它仍然很有用。

## 敬未来

我在这里列出的建议是实用和有趣想法的结合，可能会吸引任何对技术、隐私或数据控制感兴趣的人。你不需要有一份需要处理大量数据来发现 VPS 产品有趣或有吸引力的网站开发人员或数学家的工作，你只需要一点想象力。

从长远来看，我个人喜欢互联网回归本源的想法——再次去中心化。我喜欢这样一种观点，即技术现在可以更容易地用于增强个人能力——而不仅仅是大型企业。像 [Diaspora](https://joindiaspora.com/) 、[比特币](http://www.bitcoin.org/)、 [StatusNet](http://status.net/) 、 [Tor](https://www.torproject.org/) 和 [BitTorrent](http://www.bittorrent.com/) 这样的技术是实现这一目标的一大进步，但是每个人都需要有自己的服务器来进行自己的计算，这样这个想法才能完全成功。

我不认为像我目前在公寓里运行的大型物理服务器是未来的发展方向。Freedombox Foundation 可能会在为大众简化运行个人服务器软件的过程中发挥作用，但是很多人不想或者不能运行 24×7 的专用硬件。因此，我预测 VPS 将是很多人的未来，甚至可能是你的未来。
