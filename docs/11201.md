## 确保页面准备就绪

在我们能够与页面上的 HTML 元素交互之前，这些元素需要已经被加载:我们只能在它们已经存在的时候改变它们。在 JavaScript 的早期，唯一可靠的方法是在运行任何脚本之前，等待整个页面(包括图像)加载完毕。

幸运的是，jQuery 有一个非常酷的内置事件，可以尽快执行我们的魔法。因此，我们的页面和应用程序对最终用户来说似乎加载速度更快:

**例 1。`chapter_02/01_document_ready/script.js`**

```
*$(document).ready(function() {*  alert('Welcome to StarTrackr! Now no longer under police …');*});*
```

这里重要的部分(粗体突出显示)说，“当我们的文档准备好了，运行我们的函数。”这是您可能会看到的最常见的 jQuery 片段之一。做这样一个简单的警告测试通常是一个好主意，以确保您已经正确地包含了 jQuery 库——并且没有任何有趣的事情发生。

**important:** You’ll Be Seeing $(document).ready() a Lot!

几乎你在 jQuery 中做的所有事情都需要在文档准备好 之后 *完成——所以我们会经常用到这个动作。从现在起，它将被称为文档就绪事件。除非另有说明，本书后面的每个例子都需要从文档就绪事件内部运行。不过你应该只需要在每页上声明一次。*

文档就绪习语是如此普遍，事实上，它有一个快捷版本:

```
$(function() { alert('Ready to do your bidding!'); });
```

如果你想用捷径，请便！扩展版本可以说是自文档化代码的一个更好的例子；一眼就能看出到底发生了什么要容易得多——尤其是当它隐藏在另一个开发人员的 JavaScript 页面中的时候！

粗略地看一下，文档就绪事件看起来与我们在 jQuery 剖析类中遇到的结构相去甚远，但是仔细观察，我们可以看到所有必需的部分都考虑到了:选择器是`document`；动作是`ready`；参数是运行一些代码的函数(我们的`alert`)。
