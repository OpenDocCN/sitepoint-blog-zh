# 为什么企业不升级 IE6，Chrome Frame 能有什么帮助

> 原文:[https://www.sitepoint.com/ie6-upgrades-google-chrome-frame/](https://www.sitepoint.com/ie6-upgrades-google-chrome-frame/)

一些开发者似乎认为没有升级 IE6 的公司懒惰，不关心安全，或者在技术上无能。虽然对一些公司来说这可能是真的，但并不总是这样。大企业的世界不可能总是按照我们希望的速度发展。

大公司和政府机构经常雇佣成千上万的员工。有些人的 IT 预算超出了想象，他们通常投资于需要 5 年、10 年或更长时间的长期项目。

在 20 世纪 90 年代末，政府和商业界开始意识到 web 作为应用平台的好处。大规模的网络项目被启动，详细的规范被制定出来，规定了哪些浏览器应该被支持。到了 2001 年，只有一个选择:IE6。网景公司被打败了，微软拥有令人眼红的市场份额。

主要的网络应用程序是为唯一可用的浏览器开发的。网络标准是新的，看起来不相关:IE6 是标准。网络技术也在发展:许多应用程序是 CGI 代码、经典 ASP、PHP3、ActiveX 或 Perl 的大杂烩。大多数程序员已经从桌面开发转移到一个*陌生的*客户机-服务器环境，像代码分离和[渐进增强](https://www.sitepoint.com/progressive-enhancement-graceful-degradation-basics/)这样的技术闻所未闻，更不用说实践了。

斥责当时的决策者和开发者很容易，但你这样做是事后诸葛亮。如果没有 Firefox，IE6 可能仍然是今天唯一可用的网络浏览器(这也是微软的意图)。

## 遗产

现在使用的关键任务系统只能在 IE6 中工作。一些应用程序可能是在过去几年中交付的，但尚未获得投资回报。更新软件以便用户可以选择不同的浏览器，要么在商业上不合理，要么可能需要很多年才能实现(尤其是当你考虑到令人讨厌的、非标准的嵌入式 HTML 时)。

因此，大公司和政府机构必须确保用户保留 IE6。甚至开发人员的个人电脑也被锁定，所以新的内部网络应用程序继续使用 IE6 作为基础。

尽管我们认识到了新浏览器的好处，但我们是从 web 开发的角度出发的。大企业不太可能被改进的 CSS 设施、HTML 画布、花哨的动画和更好的 YouTube 支持所说服。开发人员经常声称他们放弃 IE6 可以节省 20%的时间，但与更新主要商业应用程序和向所有用户推出浏览器升级的成本相比，这可以忽略不计。安全性经常被认为是升级的好理由，但许多公司已经阻止了全面的互联网访问，或者拥有比任何操作系统或浏览器都更好的阻止攻击者的解决方案。

当然，在一台电脑上安装多个浏览器是可能的，但是很少有用户会意识到其中的差异。支持团队可能会因为员工使用错误的浏览器来访问应用程序而不堪重负。

## 为什么 Chrome 框架与众不同

谷歌的解决方案很聪明，因为它:

1.  允许公司保留其关键任务应用程序所需的 IE 版本
2.  不需要在用户的桌面上添加另一个浏览器，因此可以最大限度地减少培训和支持
3.  当 Chrome Frame 可用时，允许 web 开发人员利用最新的技术
4.  使用 Webkit 渲染引擎，所以一个已经可以在 Chrome 浏览器中运行的网站应该不需要什么重要的测试。

不能保证公司会采用 Chrome Frame，但这样做的障碍比其他解决方案低得多。如果它稳定、安全且易于推广，内部 web 开发人员可能会说服他们的 it 部门安装它。
