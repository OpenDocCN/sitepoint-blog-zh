# 教训:隐私很重要

> 原文:[https://www.sitepoint.com/lesson-privacy-matters/](https://www.sitepoint.com/lesson-privacy-matters/)

对用户来说，隐私很重要。这很重要*。这应该是显而易见的，除了很多时候对我们来说似乎是一个小问题，实际上对我们的用户来说可能是一个大得多的问题。*

 *轶事:今天在他们的博客上，商业网络应用程序制造商 37signals 分享了一些来自他们高层 CRM 应用程序的总使用数据。公司创始人 Jason Fried 发布了一张图表，显示了每个国家的应用程序整个数据库(超过 800 万)中有多少联系人。因为用户在输入联系人时可以在 country 字段中输入他们想要的任何内容，所以这些数据描绘了一幅有趣的画面，显示了人们更喜欢如何引用国家(美国与美国、美国与美国、英国与英国等)。).

这些数据有点意思，我想这就是 Fried 发布它的原因——他认为这很不错，想分享一下。但几乎同时，用户们也表示了对隐私的担忧。“我是 37signals 的粉丝，所以请不要误会。但我不得不问，”一位评论者写道。“我们是否同意授权他人查阅我们的数据？你只是查看地址，还是有其他信息 37 信号工作人员梳理。我希望高层建筑中的数据能有更多的安全感。”

Fried immediate 向评论者保证，这是从数据库中提取的国家汇总数据，根本没有附加任何个人身份信息。“没有人的个人数据被审查或暴露或审查，或以任何方式受到损害。这是匿名的自动汇总数据，”他说。

但这并没有消除所有人的担忧。“所以我走到你家，从窗户往里看。我列出了我看到的所有东西。对你街区的每个人重复。如果我只报告高清电视的数量，这样可以吗？”一位评论者写道。

“我知道这并不意味着什么，但出于某种原因，我就是觉得……很有趣。另一个人写道:“如果你只查看账户所有者信息供内部使用，我会很好，但除此之外……我不知道。”。

一位说自己不是 37signals 客户的评论者指出，在 TOS 或隐私政策中，该公司没有提到收集客户存储信息的汇总数据。“看起来他们保留了对客户本身进行统计的权利，但没有对他们拥有的数据进行统计。甚至关于顾客数据的共享也仅限于法律要求&改善服务，我能找到的任何地方都没有提到营销，”他评论道。他提到，Google Analytics 为数据的聚合共享提供了客户选择，并建议 37signals 效仿这一模式。

当然，从长远来看，这真的不是一个故事 Fried 没有损害客户隐私，也没有暴露任何个人身份数据。这是小题大做。尽管如此，还是有相当多的评论者在帖子发布后很快表达了对隐私的担忧，所以这是一个教训。隐私对用户来说是至关重要的，尤其是对使用您的应用程序来存储客户和顾客信息的商业用户。让任何人怀疑你没有把保护隐私放在第一位，这从来都不是一个好主意，即使具体的关注最终是微不足道的。*
