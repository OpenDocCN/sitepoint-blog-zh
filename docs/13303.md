# Mac vs PC 与网络的未来

以下内容转载自《科技时报》第 173 期。

你是 Mac 用户还是 PC 用户？

最近我尝试了一下苹果，我有一个理论，关于为什么人们在试用 Mac 时似乎会爱上它。信不信由你，这关系到 Internet Explorer 和 Web 本身的未来！

## Mac 与 PC:是什么让你的船漂浮？

好吧，我承认:我深深地爱上了苹果电脑。就像我之前所有的极客时尚受害者一样，我已经开始依赖我的 Mac 电脑，就好像它是我的一部分一样，与之前的其他电脑相比，我更喜欢它。

请注意，我是带着适度的怀疑态度进去的。我决心不要成为那种把他的 Mac 抱在胸前，高兴地喊着“我喜欢它！就是管用！”哦，不，Mac 必须向我证明它确实比无处不在的个人电脑更好。

见鬼，直到苹果宣布你可以在 Mac 电脑上运行 Windows，我才愿意尝试一下。毕竟，如果 Mac 的生活方式不适合我，我可以把它变成一个 Windows 盒子。

不，我不会被吸引眼球的糖果，微妙的动画用户界面，或有品位的工业设计。“各地的网页设计者和开发者都在改变，”我解释道。“如果仅仅是为了研究，我需要了解这个 Mac 的事情。”

那么，为什么六个月后，我变成了苹果用户呢？很简单，Mac OS X 的包袱比 Windows 少得多。

Mac OS X 的桌面版于 2001 年发布，代表着与以前的彻底决裂。尽管它的 Unix 根基更深，但实际上，Mac OS X 的最终用户功能都没有超过 6 年。

同时，就支持的程序和用户体验而言，当前一代的 Windows 向后兼容 Windows 95(1995 年发布)。因此，Windows 操作系统的基本结构已经有 12 年的历史了，是 Mac OS X 的两倍。

对微软来说，这是一个值得骄傲的点。任何公司都可以部署业务关键软件，并知道它将在未来几十年继续开发新的操作系统版本，这是 Windows 的一个重要卖点。

与此同时，苹果在打破向后兼容性方面有着悠久的历史，这样做会给其产品带来巨大的价值。2001 年，当 Mac OS X 的用户体验从零开始重新设计时，每个 Mac 用户都必须重新学习使用电脑。到 2005 年，当苹果停止支持 OS X 之前的软件时，Mac 用户必须更换他们所有的软件。

这些与过去的决裂在 Windows 世界是完全不可接受的，但它们让苹果保持了 Mac 的新鲜感，没有杂乱。

作为一名高级桌面用户和开发者，我会说**我使用的功能不超过 Windows Vista 的 10%。操作系统的其余部分就在那里，一个摇摇欲坠的组件集合，支持我永远不会再使用的古老技术，占据了空间，妨碍了我。**

对于像我一样的绝大多数桌面用户来说，今天使用 Windows 就像是在一艘老化的游轮上进行一次单人环球航行。拥有三个舞厅、一个奥林匹克游泳池和一个属于自己的屋顶练习场是很好的，但它们可能为你做的只是在你忙于避开冰山时收集灰尘。

如果你乐意至少每四年更新一次你的软件、硬件和计算技能，那么 Mac 平台就是你想要创造世界纪录的时尚而快速的游艇。六个月后，我会说**我使用了 Mac OS X 提供的大约 60%的功能。**对于一个普通的桌面操作系统来说，这已经很不错了。

## 网络的未来

那么，这些 Mac 和 PC 的东西与 web 开发有什么关系呢？嗯，微软和苹果都写网络浏览器和操作系统，事实证明他们开发浏览器的方式和开发操作系统的方式差不多。

在即将到来的悉尼 2007 年南方网络方向会议上，微软的克里斯·威尔逊将以“推动网络前进”为主题发表演讲我希望这次演讲在精神上类似于[今年 4 月发表的长篇声明](http://lists.w3.org/Archives/Public/public-html/2007Apr/0612.html)，他在声明中解释了像 Internet Explorer 这样的浏览器保持向后兼容性是多么重要，这样才不会像他感觉 IE7 那样“破坏网络”。

这与微软对 Windows 的态度非常一致:在现有的基础上构建，如果可以的话，永远不要破坏向后兼容性。毕竟，有数亿人依赖旧网站，微软有责任让这些旧网站继续运行。

苹果对 Safari 的态度是，如果你使用的是最新的浏览器，你所依赖的网站也可能是相对最新的。如果一个旧网站显示不太正确，那是意料之中的。这并不像网站中包含的信息对世界而言是丢失的。如果网站有价值，有人会更新它以符合网络标准，这将确保它的长期可访问性。并且总是有视图源。

转换到 Mac 的人(因此远离了 Internet Explorer)是少数不访问旧网站也能生存的人吗？我不这么认为。我认为当用户有选择的时候，他们会很乐意忍受向后兼容性的损失。

我觉得微软发布 IE7 作为一个强制的自动更新犯了一个错误。其目的是让用户尽快使用更安全的浏览器，但将一个新的渲染引擎捆绑在强制安全更新中是一个错误。当旧网站在新浏览器中无法运行时，用户会觉得不兼容强加在他们身上，而不是选择做出牺牲来换取一个闪亮、现代的浏览器。

如果微软的计划成功了，世界各地的网络开发者将会在一夜之间采用更符合标准的 IE，但我们看到的最多只是不冷不热的采用。即使在像 SitePoint 这样的网站上，你会期望有很多早期用户，IE6 仍然代表了我们大约一半的 Internet Explorer 访问者。

苹果也自动进行 Safari 安全更新，但保留了对渲染和标准合规性的改进，用户可以选择安装或不安装更新。正因为如此，Safari 用户似乎更有可能安装最新版本。

不管是好是坏，微软编写浏览器的“不惜一切代价向后兼容”的新方法正在影响由克里斯·威尔逊主持的 HTML 5 规范的发展。微软希望取缔那些被采用的标准，这些标准迫使浏览器放弃对他们以前的、不符合标准的行为的支持。
