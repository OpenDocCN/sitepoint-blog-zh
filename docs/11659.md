# 渐进增强和优雅退化:综述

> 原文:[https://www . site point . com/progressive-enhancement-graceful-degradation-basics/](https://www.sitepoint.com/progressive-enhancement-graceful-degradation-basics/)

在这篇文章中，我将讨论渐进增强和优雅降级——这两个概念众所周知，但可能有点误解。

## 网络正在进化

网络处于不断变化的状态，并以惊人的速度发展。技术和技巧起起落落。今天的 Safari 4 就是明天的 IE6。我们希望利用最新的进展，为我们的用户提供更好的体验。

遗憾的是，没有办法强制浏览器升级。公司和个人不会因为开发商的要求而改变他们的工作环境。许多人乐于在完全不了解底层技术或升级的正当理由的情况下访问网页。操作系统和浏览器制造商有一定的影响力，但网络开发者只能赞美新浏览器的好处。

许多开发者认为支持过时的技术没有什么意义。然而，这种态度不太可能说服那些想要尽可能多的受众的客户或者那些依赖旧浏览器的用户。这些用户可能没有系统权限、财务资源或升级的技术能力。一些用户可能正在使用无法升级的专业辅助技术。没有人可以强迫用户使用软件或升级:这样做违背了他们选择的民主权利。如果有人声称是为你好，强行在你的电脑上安装软件，你会有什么感受？

该网络设计用于任何地点的任何设备。关于最终用户，我们所知道的是，他们正在使用一个浏览应用程序，该应用程序使用互联网协议发送和接收数据。我们无法对他们的设置或技术能力做出假设。

这使得 web 成为一个充满敌意的开发环境。当用户使用 8 年前的浏览器访问我们开发的应用程序时，这些应用程序可能是脆弱的，该浏览器禁用了 JavaScript 并且没有插件。

## 无障碍的需求

可访问性是一个重要的问题，它被认为只是关于政治正确性、alt 标签或盲人用户的观念所蒙蔽。远不止这些。可访问性是关于支持所有用户，不管他们使用什么浏览技术。

一个可以在各种设备、屏幕阅读器、PDA、手机、[游戏控制台](https://www.sitepoint.com/nintendo-wii-opera-browser/)和连接速度上运行的网站将比一个不能运行的网站拥有更广泛的影响力。

## 功能退化

优雅的退化是一种解决方案。它是构建网站或应用程序的实践，以便在现代浏览器中提供良好的用户体验。然而，对于那些使用旧浏览器的人来说，它会优雅地降级。该系统可能不那么令人愉快或漂亮，但基本功能将在旧系统上工作。

一个简单的例子是使用 24 位透明的 png。这些图像可以毫无问题地在现代浏览器上显示。IE5.5 和 IE6 将显示图像，但透明效果将失败*(如果需要，可以使其工作)*。不支持 PNG 的旧浏览器会显示替代文本或空格。

采用适度降级的开发人员通常会指定他们的浏览器支持级别，例如 1 级浏览器(最佳体验)和 2 级浏览器(降级体验)。

## 渐进增强

渐进增强与优雅降级的概念相似，但方向相反。网站或应用程序将为大多数浏览器建立基本的用户体验。当浏览器支持时，将会添加更多的高级功能。

渐进式改进不需要我们选择支持的浏览器或恢复到基于表格的布局。我们选择一种技术水平；即浏览器必须支持 HTML 4.01 和标准页面请求/响应。

回到我们的图像例子，我们可能决定我们的应用程序应该在所有的图形浏览器中起作用。默认情况下，我们可以使用低质量的 GIF 图像，但是如果浏览器支持的话，可以用 24 位的 png 图像来代替。
