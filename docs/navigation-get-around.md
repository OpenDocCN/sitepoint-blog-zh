# 导航——方便出行

> 原文：<https://www.sitepoint.com/navigation-get-around/>

“几乎所有好的信息架构的基础都是设计良好的层次结构。”

愤怒。困惑。挫败感。

这些是一个迷失在网络空间的网络冲浪者的情感。在一个制作拙劣的网站的阴暗、未知的黑暗中。我希望不是你的人！

虽然内容在网络上是王道可能是真的，但你的网站需要一个有序的结构——和许多路标——来帮助访问者绕开它。找到他们想找的任何东西。去他们想去的任何地方。快点。有效地。毫无痛苦。

##### 你的网站需要一个好的导航系统

术语“导航”在网络上用来描述当访问者在网站的不同部分移动时，他/她被引导的方式。在这篇文章中，我将讨论一个好的网站导航系统的要素。

你网站的目的是什么？

在大多数情况下，这将是为了满足特定人群的需求。信息资源期望让用户找到相关的内容。娱乐网站旨在给访问者一个愉快的体验。零售网站希望出售购物者需要或渴望的产品。

你的网站最重要的方面之一——当然除了内容——是你如何引导访问者找到他/她想要的东西。做到这一点的最好方法是…

**设身处地为客户着想**

你可以为你的网站规划和设计最佳导航系统的唯一方法就是用一个典型的访问者的思维方式去思考。

试着进入访问者的思维。

她/他如何找到你的网站？她/他将如何进入？然后她/他将采取什么步骤来找到有用的内容？她/他找到后会怎么样？

试图确定这些问题的答案将有助于你规划网站导航系统的蓝图。

##### 愉快的用户体验的价值

随着网络上新内容的爆炸式增长，吸引和打动网络冲浪者不再容易。你的网站必须给访问者提供一个非常好的体验，如果他/她记得它，把它加入书签，以后再回到它，告诉一个朋友(或许多朋友)关于它。

疏远首次访问者的一个有效方法是让你的网站很难被浏览。

如果你强迫人们点击多个链接，在你的网站上徒劳地转来转去，试图找到有用的内容，你可以肯定他们不会再回来。网上有太多更好的选择。

通过构建一个具有直观、优雅和简单的导航系统的网站，向访问者显示他们所在的位置，并解释如何到达网站的其他部分，您可以增强他们在您的网站上的体验。那时，也只有那时，他们才会回来。带上他们的朋友！

**确保流程顺畅**

大多数对信息或乐趣的搜索都遵循一个逻辑顺序。每个站点的顺序可能不同。如果你理解你的访问者可能遵循的模式，并将其应用到你的导航系统中，他们就能顺利地浏览你的网站。几乎不费吹灰之力，他们就被引向了他们所寻找的内容。

如何为你的网站设计一个优秀的导航方案？

如果你站在客户的立场上，想想在这样的网站上你最喜欢的东西，这并不困难。以下是一些需要回答的基本问题:

*   我在哪里？——对某人在现场的当前位置的帮助
*   我下一步该去哪里？–整个网站的路线图或目录
*   我如何到达那里？–直观或描述性的导航系统
*   我还在这个网站上吗？–网站不同部分的一致外观和感觉

##### 我在哪里？

还记得你上一次去国家公园的假日旅行，或者你在附近的新购物中心的疯狂购物吗？你在那里做的第一件事是什么？

我敢打赌，你在地图上搜索熟悉的*你在这里*标志。

一旦你找到了它，并在太空中找到了自己的方向，整个经历就会变得快乐而激动人心。你不再害怕迷路。

类似地，在一个拥有多个页面和大量内容的大型或复杂网站中，浏览者很容易感到迷失。如果她/他通过网站主页之外的途径进入，尤其如此。困惑中，她/他问了这个问题:“我在哪里？”

一个优秀的导航系统会回答这个问题——在每页的*上。*

通过提及页面的标题，或者突出主题，或者使用不同颜色或设计的按钮，你可以让你的访问者找出他/她所在的页面。

**提示和技巧**

在 HTML TITLE 标签中，使用描述性文本来解释页面内容。

*   确保你网站的名称和网址(甚至公司名称和其他相关细节)出现在你网站的每一页上。这甚至对打印出你的网页并离线阅读的人也有帮助。
*   Somewhere in your site’s content, try and fit in a TEXT based navigation aid that tells visitors where they are and possibly where they came from and where they can expect to go next. For instance, you could have a line that says:

    `Home > Articles > Diamonds > THE KOH-I-NOOR`

    最后一个(KOH-I-NOOR)描述了读者当前所在的页面。它之所以突出，是因为它全是大写字母，而不是超链接。其他单词告诉访问者通向这个页面的路径。通过使它们成为超链接，访问者可以很容易地点击它们来访问你网站的其他部分。

    这种横向和纵向导航的便利是网络的独特优势，一个优秀的导航系统将最大限度地利用这一优势。

*   或者，你可以用一组小图片来更形象地表达这一点。我最喜欢的一个是红色圆圈中的一组白色箭头，表示当前页面之前和之后的页面，黑色圆圈中的白色方块表示读者当前所在的页面。
*   另一种让你的访问者找到自己方位的方法是使用按钮技术——为网站导航栏中的背景或文本使用不同的颜色或阴影，以表明他/她所在的网站部分。

##### 我下一步去哪里？

如果你花时间仔细考虑了你的网站布局，你的网站内容将会顺畅地流动。为了达到这个目的，网站上相关或相似类型的内容必须以一种直观和合理的方式组合在一起。然后，访问者可以从更广泛的类别深入到更具体的内容。

例如，如果您的网站涉及电子出版，您可能会有一个名为“电子邮件出版”的顶级类别和几个“电子邮件通讯”、“电子邮件程序”、“电子书”等子类别。在“电子书”下，你可能会有更多的类别，比如“电子书软件”、“电子书教程”、“电子书零售商”等等。

有两个一般级别的站点组织-全球和本地。每一种都有独特的导航方法。全球导航关注的是帮助你的网站的访问者在广泛的类别中识别和移动。另一方面，本地导航深入到网站的一个主要部分，帮助引导访问者到一个更具体的位置。

**提示和技巧**

*   在你网站的每一页上，试着包含一个到你网站下一个逻辑部分的链接。这可以通过以下方式实现:
    1.  使用“下一页”、“阅读更多”或“点击此处继续”等文字，甚至是更刺激、更吸引人的文字来吸引读者继续阅读
    2.  使用微小的图片或符号，例如“> > >”代表前进，而“< <
    3.  Hyperlinking a part of the content of the page itself near the end. For instance, in this sentence:

        "因此，光之石是一颗有着神秘历史的迷人钻石."

        你可以把“神秘的过去”这几个字变成一个可以点击的超链接，链接到文章的下一部分。

*   为了帮助读者了解整个网站的布局，尽量在每个页面上提供一个主要区域或部分的列表。这样，如果一个读者想探索你的网站的一个完全不同的部分，他/她不必点击很多次来追溯他/她以前走过的路。
*   在具有多个兴趣区域的复杂或大型网站中，您可能需要为每个部分创建多个导航工具，以优化最终用户体验。

##### 我如何到达那里？

所以，现在你的网站访问者已经找到了他/她的方位，并决定了他/她下一步要探索网站的哪些部分。下一个要回答的问题是:“我必须做些什么才能到达那里？”显而易见的答案是:“点击链接。”

但是链接在哪里，会导致什么，并不总是显而易见的！

一个好的导航系统将确保链接很容易被识别。大多数 Web 浏览器的默认颜色编码显示蓝色的文本链接，通常带有下划线。如果出于某种原因，你需要改变这个链接的颜色，确保有其他方法来识别它是一个链接。

你的导航系统也可以通过明确链接指向哪里来提升用户体验。这可以通过使文本链接尽可能具有描述性并避免歧义和行话来实现，或者通过使用诸如当鼠标光标放在链接上时显示的 Javascript 弹出标题之类的工具来实现。

如果你需要在你的网站上使用不同的导航工具，或者如果很多用户可能很难理解你的导航过程，那么创建一个“帮助”页面来详细描述浏览你的网站的过程是值得的。

**提示和技巧**

*   最好保留用户 web 浏览器的默认颜色选择。这些颜色——蓝色代表未访问的链接，紫色代表已访问的链接——是由主要浏览器公司经过广泛研究和可用性测试后选择的。干预它们表明你对网站访问者缺乏关心。如果你觉得必须改变它们，确保你改变了整个网站。
*   使用图片作为超链接会让用户很难识别出是链接。试着通过添加文本指针让它变得更简单，比如:“点击下面的图标”
*   一项对新网民的调查表明，除非他们做了什么，否则他们有时不会注意到超链接。通过使用简单的 Javascript 或 CSS 技术，您可以将链接制作成动画，这样当用户将鼠标光标移到链接上时，链接就会发生变化。颜色、字体或大小的变化，或者超链接文本下的线条，使访问者更容易知道点击什么。
*   Javascript is a neat way to have a link description pop-up when the mouse cursor it placed over it. But here’s another simple trick that works with Microsoft Internet Explorer (MSIE) 5.0 and higher browsers (it doesn’t affect the link in Netscape).

    在链接的 HTML 代码中，使用一个`"title"`标签来添加描述链接的文本块。格式是:

    `<a href="link.htm" title="This is the link descriptor"> LINK </a>`

    在 MSIE 5.0 及更高版本的浏览器中查看页面时，鼠标光标放在链接文本上就会弹出链接描述！你自己试试。

*   如果你的网站吸引了许多仍在学习如何使用互联网的新手，或者如果你使用了一个复杂的导航系统，很难直观地理解(理想情况下，你不应该这样做！)创建一个帮助页面来解释您的导航系统可能是值得的。在这个页面上，详细说明你使用的所有导航元素，并向用户解释他们如何利用系统来浏览你的网站的各个部分。然后，最重要的是，从你网站的所有页面链接到这个帮助页面！

##### 我还在这里吗？

网络冲浪者喜欢网站的一致性。如果一个访问者点击了一个链接，并被带到了一个看起来与前一个完全不同的页面，她/他会感到困惑，并问“我还在同一个网站吗？还是我被传送到了一个不同的地方？”

一个优秀的导航系统需要在整个网站上保持一致的外观和感觉。这种恒定性强调了链接页面是同一网站的一部分这一事实。

##### 构建您的导航设备

网站设计的挑战之一是将你的导航系统整合到网站形象和品牌的整体架构中。有许多方法可以创建你的导航工具。

**纯文本:**文字用于描述链接

**仅图片:**一个小图形解释了该链接

**带图像的文本:**文字和图形都用作链接

**下拉菜单:**点击它会弹出一个选项列表供选择

**动态生成页面:**使用脚本从数据库中提取页面

通过明智地使用这些元素中的一个或多个，可以为任何网站设计一个有效的综合导航系统。

建造航标的一些一般原则

***文字链接***

*   使用明确的链接标题
*   尽量保持链接的长度一致
*   避免在链接标题中使用行话或俚语
*   确保链接文本作为链接清晰可辨

***图片链接***

*   确保图像尺寸较小，以最大限度地减少页面下载时间
*   图形必须直观或清晰地描述链接
*   将文本和图像链接结合起来可以提高清晰度
*   总是使用 ALT 标签来帮助关闭图片的浏览者

***下拉菜单***

*   有助于节省网页空间
*   需要一些编程专业知识来设置
*   使用旧浏览器的冲浪者可能无法访问

##### 导航工具的网页设计元素

随着你的网站的成长和发展，新的部分会随着时间的推移而增加。将它们包含在导航栏中意味着替换多个文件中的数据。

减少这一问题的两种方法是:

1.  在你的网站上使用“框架”,一个框架只包含导航栏

3.  对导航元素使用“服务器端包含”

使用这两种技术中的任何一种，您都可以只对一个文件进行更改，并使这些更改反映到站点的所有部分。

**不要只是交付:过度交付**

所以你的网站涵盖了所有这些要点。你有一个极好的导航系统。恭喜你。

但是现在还不是放松的时候。出于各种原因，用户可能不会对你的导航工具感到高兴。所以专注于为你的用户提供额外的价值。创建 SNS 辅助导航系统的首字母缩写。

顾名思义，社交网络包括其他导航工具，如:

*   搜索引擎
*   目录
*   站点地图索引

***搜索引擎***

每个人都听说过 HotBot 和 AltaVista 这样的巨型搜索引擎，它们列出了网络上所有的网站。也有可能有一个搜索引擎只搜索你网站上的内容。访问者可以使用搜索引擎来尝试并找到他/她使用导航工具不容易访问资料。

***目录***

通过列出你的网站的所有主要部分，目录使得访问者可以更容易地直接深入到一个特定的主题或类别。

***站点地图索引***

作为目录列表的变体，站点地图索引是通过深入站点上不同形式的内容而创建的。然后列出前 30 到 60 个感兴趣的领域，并提供相应内容的链接。

##### 测试你的导航系统

永远不要假设你的导航系统是完美的。虽然对你来说它看起来很直观和清晰，但它可能仍然会让网站访问者完全糊涂。

测试一下。

测试的广度取决于你的站点的大小和范围。如果你经营一个小型的业余爱好网站，找几个朋友和家人来帮忙测试就足够了。对于较大的专业站点，试点项目测试是一项严肃的任务。

最好是在试用用户浏览你的网站时，实际观察他们。看看他们对你网站布局的反应。他们容易识别助航设备吗？他们知道如何在现场走动吗？给他们一个目标——比方说，找到一个特定的页面或文章。然后看看他们能多快多容易地到达那里。

记笔记。寻求反馈。根据建议改进你的导航设备。调整，测试，再调整，再测试。

直到最后，你的网站是一种乐趣！
