# 关于 PHP 的用法

> 原文:[https://www.sitepoint.com/about-php-usage/](https://www.sitepoint.com/about-php-usage/)

*谁*在用 PHP 这个问题被问了很多次。我们都知道 PHP 用户包括每个人，从[布兰妮](http://www.britneyspears.com/index.php)到英国工党的大对话。事实上，加拿大梦想实验室维护着一个由 PHP 驱动的网站目录，这些网站按行业分类。

对我来说，现在更有趣的是查看在哪里使用了 PHP 它被部署在哪里(互联网对内部网/外部网)，人们用 PHP 解决了什么问题，当建立一个在线画廊对一个群件工具时，什么设计决策起作用，当比较一个在线杂志对一个公司内部网时，需求如何影响 CMS 设计，等等。等等。？

过去曾经有过关于 PHP 使用的调查，比如 Zend 的这个调查和 Interakt 的[的另一个调查，但是从我的角度来看，这两个调查都没有探究这类问题。](http://www.interakt.ro/index_art_11.html)

有两件事困扰着我，这也是促使我产生这种想法的原因。首先，我有一种预感，整个 PHP“用户群”没有被关注或被倾听，大部分被“主流”PHP 用户(那些在小型在线企业上建立个人网站的人)所吞噬。第二个问题是 PHP 的功能似乎没有被充分利用，最著名的是 [COM](http://www.php.net/com) 扩展，在某些情况下，它可能也等同于具有商业价值的软件(比如有人愿意为它付费)。

在互联网上，很少有关于如何在高流量网站上使用 PHP 的深入讨论。当然，这是一个小的用户群，但它也是高姿态。到目前为止，我所看到的最好的讨论是乔治·施洛斯纳格的《高级 PHP 编程》(这是一篇很好的文章，我将在不久的将来发表一篇更长的博客)，但是除此之外，很难清楚地了解在运营一个高流量网站时，什么是明智的技术决策。这种体验是“无处不在”的——Sitepoint 本身就是一个例子，按流量排名在[1500 强](http://www.alexa.com/data/details/?url=www.sitepoint.com)网站中。这里的问题似乎是让开发者分享他们的经验。

另一个(猜测)很大程度上被忽视的大用户群是那些在公司内部网/外部网上使用 PHP 的人(包括我，所以我有自己的看法)。我大胆猜测，在世界上排名前 1000 的公司中，80%以上在某个地方使用 PHP(尽管管理层可能也没有意识到这一事实)。

这个群体面临着一系列完全不同的问题——性能通常不太重要，流量相对较低。能够将 PHP 与一系列“后台”系统以及桌面应用程序集成是这个群体面临的最大问题，与 Windows 的集成是头号问题。

实现一个标准的、数据驱动的 CMS 通常是不可接受的，例如，用户要求能够在 Intranet 上发布 Word 文档，同时又希望避免维护同一文档的多个版本(一个在 web 服务器上，另一个在工作组服务器上)。用户可能还要求能够使用与他们的 NT 工作站相同的用户名/密码登录。

在这个“细分市场”中，PHP 的应用范围很广，例如系统管理工具、在线报告(从公司数据库中提取的数据)、发布公司文档、广播内部新闻、作为 CRM 系统的前端、在线群件工具、以 PHP 为“中间人”的系统互操作(例如 XML-RPC / SOAP)等等。与此同时，寻找用 PHP 编写的开源工具来帮助整合现有的企业系统可能是一件棘手的事情——你会在像 [Hotscripts](http://www.hotscripts.com/PHP/index.html) 这样的网站上找到很少(如果有的话)这类应用的例子。

特别是，COM 在主流 PHP 圈子里很少受到关注，但在内部网/外部网上却大有可为。不幸的是，如果你正在寻找一种方法来帮助用户发布 Word 文档，wvWare 可能是一个更好的地方，或者要求用户保存为 HTML，然后用 PHP 去掉你需要的内容。PHP < > Windows(实际上是运行在 Windows 上的应用程序)集成似乎没有得到很好的规划，这可能有很多原因，从 PHP 根本上说是一个糟糕的选择，到缺乏具有 PHP 和通用 Windows APIs 经验的开发人员。与此同时，我感到对能够将 Outlook 电子邮件放到网上的工具有巨大的(甚至是商业的)需求(这是可行的:当然有了 [Python](http://www.boddie.org.uk/python/COM.html) 和 PHP 的 COM 支持，与 Python 的支持并没有太大的不同，而且由于 [Wez](http://pecl.php.net/user/wez) ，PHP5 变得越来越好)。

总之——开始东拉西扯，犯了开始讨论却没有得出结论的罪。有人知道 PHP 其他较少被提及的用法吗？
