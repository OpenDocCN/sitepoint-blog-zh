# 亚马逊的底线:基础设施提供商与竞争对手

> 原文：<https://www.sitepoint.com/amazons-fine-line-infrastructure-provider-vs-competitor/>

今天，亚马逊取消了其灵活支付系统——一种类似 PayPal 的针对商家的服务——的测试(至少在美国)。他们还推出了一些“快速启动”，这是一组简化的 API，使商家更容易通过亚马逊的支付系统接受订单，作为吸引新账户的一种方式，亚马逊提供前三个月高达 50 万美元的免费支付处理。

对于消费者来说，用亚马逊支付是显而易见的。因为亚马逊本质上是电子商务的同义词，而且因为大多数消费者都有过良好的体验，所以更容易信任他们来处理你需要或想要打交道的较小、不太知名的商家的支付处理。与 PayPal 和 Google 相比，Amazon 这个名字给购买过程带来了一种安全感。

但对于商家来说，选择亚马逊的决定并不是板上钉钉的。亚马逊提供了可能是网络上最完整的端到端电子商务解决方案——他们真正提供了他们的整个基础设施。你可以[向亚马逊支付](https://payments.amazon.com/)，让他们[处理履行](http://www.amazonservices.com/content/fulfillment-by-amazon.htm)，接入他们[的后端基础设施](http://aws.amazon.com/)，[销售](http://www.amazon.com/gp/seller-account/mm-summary-page.html)或[宣传](http://www.amazon.com/Advertising/)你的产品，甚至让亚马逊[出版你的书、DVD 或 CD](https://www.createspace.com/) 。这听起来很棒，从很多方面来说确实如此。亚马逊的基础设施服务意味着创建电子商务业务的门槛非常低，许多与在线销售相关的头痛问题都会为你解决。

但这是有代价的:亚马逊也可能是你的主要竞争对手之一。正如我们在六个月前提到的，许多大型零售商对与亚马逊合作持谨慎态度，因为该零售商是竞争对手，让他们获得销售数据可能不符合他们的最佳利益。对于较小的电子商务企业来说，这可能不是什么大问题，但肯定仍是值得考虑的事情。

也许在加入像亚马逊这样的完整的端到端平台之前，更大的考虑是，如此依赖一家公司会让你的业务面临巨大风险。例如，如果亚马逊为你的公司处理支付、履行和广告，那么他们任何产品的变化都可能以你可能不喜欢但几乎无法控制的方式改变你的业务。

另一方面，将来自多个提供商的服务混合在一起可能不那么容易、不那么痛苦、不那么漂亮或不那么划算，但这确实意味着，如果失去了一个支持者，恢复会容易得多。亚马逊的端到端电子商务平台降低了创建电子商务业务的门槛，这很好，但企业所有者需要在部署的简单性和速度与安全性和灵活性之间做出选择。
