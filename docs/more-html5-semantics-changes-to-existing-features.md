# 更多 HTML5 语义:对现有功能的更改

> 原文：<https://www.sitepoint.com/more-html5-semantics-changes-to-existing-features/>

![htmlcss2thumb](img/d1b531ecd770df998ea32dbc2cf0f23c.png)

以下是我们的书的摘录，由 Alexis Goldstein、Louis Lazaris 和 Estelle Weyl 撰写的《真实世界的 CSS3，第二版。世界各地的商店都有出售，或者你可以在这里买到电子书。

虽然新元素和 API 是 HTML5 的主要焦点，但 web 标记的最新迭代也带来了对现有元素的更改。在很大程度上，所做的任何更改都考虑到了向后兼容性，以确保现有内容的标记仍然可用。

我们已经考虑了一些变化(例如，doctype 声明、字符编码和内容类型)。让我们看看 HTML5 规范中引入的其他重大变化。

### “已弃用”一词已弃用

在 HTML 和 XHTML 的早期版本中，不再推荐使用的元素(因此从规范中删除了)被认为是“不推荐使用的”在 HTML5 中，不再有不推荐使用的元素；现在使用的术语是“过时”

过时元素分为两个基本类别:“符合”过时特征和“不符合”过时特征。符合特性将在验证器中提供警告，但浏览器仍然支持。所以你可以使用它们，但是最好避免使用它们。

另一方面，不一致的特性被认为是完全过时的，不应该被使用。它们会在验证器中产生错误。

W3C 对这些特性有一个[描述，并附有例子。](https://www.w3.org/html/wg/drafts/html/master/obsolete.html)

### 阻止链接内的元素

尽管大多数浏览器在过去很好地处理了这种情况，但将块级元素(如`div`)放在`a`元素中实际上是无效的。相反，要生成有效的 HTML，你必须使用多个`a`元素，并使组看起来像一个块。

在 HTML5 中，你现在可以在一个`a`元素中包装几乎任何东西，而不必担心验证错误。唯一不能用`a`元素包装的块内容是其他交互元素，比如表单元素、按钮和其他`a`元素。

### 粗体文本

在 HTML5 中，粗体文本的语义定义方式发生了一些变化。在大多数浏览器中，有两种方法可以使文本加粗:使用`b`元素，或者`strong`元素。

尽管`b`元素从未被弃用，但在 HTML5 之前，人们不鼓励使用`strong`元素。先前的`b`元素是一种“让文本以粗体显示”的方式因为 HTML 应该是关于内容的意义，而把表示留给 CSS，这是不令人满意的。

[根据规范](http://dev.w3.org/html5/spec-preview/the-b-element.html)，在 HTML5 中，`b`元素已经被重新定义为表示一段文本“出于实用目的而引起注意，但没有传达任何额外的重要性，也没有替代声音或语气的含义。”给出的例子有文档摘要中的关键词、评论中的产品名称、交互式文本驱动软件中的可操作词或文章标题。

与此同时，`strong`元素仍然传达着或多或少相同的意思。在 HTML5 中，它代表“其内容非常重要、严肃或紧急”有趣的是，HTML5 规范允许嵌套`strong`元素。因此，如果一个完整的句子包含一个重要的警告，但是某些单词甚至更重要，那么这个句子可以被包装在一个`strong`元素中，每个重要的单词可以被包装在它自己嵌套的`strong`中。

### 斜体文本

随着对`b`和`strong`元素的修改，HTML5 中定义`i`元素的方式也发生了变化。

以前，`i`元素只是用来呈现斜体文本。和`b`一样，这个定义也不能令人满意。在 HTML5 中，这个定义已经更新为“一段不同声音或语气的文本，或者以某种方式偏离正常的散文，以表明不同的文本质量。”因此，文本的外观与语义无关，尽管它很可能仍然是斜体——这取决于您。

使用`i`标签可以抵消内容的一个例子可能是另一种语言的惯用短语，例如*reduction ad absurdum*，这是一个拉丁短语，意思是“减少到荒谬的程度”其他的例子可能是小说中代表梦序列的文本，或者是期刊文章中某个物种的学名。

元素没有改变，但是它的定义已经被扩展以阐明它的用途。它仍然指的是强调的文本，就像口语中的情况一样。例如，以下两个短语具有完全相同的措辞，但由于`em`的不同用法，它们的含义发生了变化:

```
<p>Harry’s Grill is the best <em>burger</em> joint in town.</p>
<p>Harry’s Grill <em>is</em> the best burger joint in town.</p>
```

在第一句中，因为强调了“汉堡”这个词，所以句子的意思集中在所讨论的“联合”的类型上。在第二句话中，重点放在“是”这个词上，从而将句子的焦点转移到“哈里烧烤店是否真的是镇上所有汉堡店中最好的”这个问题上。

`i`和`em`都不能用来标注出版物名称；相反，你应该使用`cite`。

在这里讨论的所有四个元素(`b`、`i`、`em`和`strong`)中，唯一赋予其内容上下文重要性的是`strong`元素。

### 大文本和小文本

`big`元素以前用于表示以大字体显示的文本。`big`元素现在是不符合要求的过时特征，不应使用。然而，`small`元素仍然有效，但是有了不同的含义。

以前，`small`是用来描述“小字体的文本”在 HTML5 中，它表示“小字等旁注”使用`small`的例子包括页脚文本、小字以及条款和条件中的信息。small 元素应该只用于短文本。所以你不会用`small`来标记整个“使用条款”页面的主体。

尽管定义中已经删除了`small`的表示含义，但是`small`标签中的文本很可能仍然会以比文档其余部分更小的字体出现。

例如，HTML5 先驱报的页脚包含版权声明。因为这本质上是法律细则，所以它非常适合于`small`元素:

```
&copy; SitePoint Pty. Ltd.
```

### 治眼睛痛的药

元素最初在 HTML5 中被重新定义，伴随着一些争议。在 HTML4 中，`cite`元素代表“对其他来源的引用或参考”在这个定义的范围内，规范允许一个人的名字被标上`cite`(例如，在引用某人的话时)。

HTML5 规范的早期版本禁止使用`cite`作为人名，这似乎违背了向后兼容的原则。现在，该规范又回到了一个与最初更相似的定义，将`cite`定义为“一个创造性作品的引用”。它必须包括作品的标题或作者的名字(人、人或组织)或 URL 参考，或缩写形式的参考。

下面是一个取自规范的示例:

```
<p>In the words of <cite>Charles Bukowski</cite> -
<q>An intellectual says a simple thing in a hard way. An artist says a hard thing in a simple way.</q></p>
```

### 描述(非定义)列表

现有的`dl`(定义列表)元素，连同其关联的`dt`(术语)和`dd`(描述)子元素，已经在 HTML5 规范中被重新定义。以前，除了术语和定义，规范还允许`dl`元素标记对话，但是规范现在禁止这样做。

在 HTML5 中，这些列表不再被称为“定义列表”；它们现在是听起来更通用的“描述列表”或“关联列表”它们应该用于标记任何种类的名称-值对，包括术语和定义、元数据主题和值，以及问题和答案。

下面是一个使用 CSS 术语及其定义的示例:

```
<dl>
  <dt>Selector:</dt>
  <dd>The element(s) targeted.</dd>
  <dt>Property:</dd>
  <dd>The feature used to add styling to the targeted element, defined before a colon.</dd>
  <dt>Value:</dd>
  <dd>The value given to the specified property, declared after the colon.</dd>
</dl>
```
