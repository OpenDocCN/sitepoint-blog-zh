# 启发式评估-一步一步的指导文章

> 原文:[https://www.sitepoint.com/heuristic-evaluation-guide/](https://www.sitepoint.com/heuristic-evaluation-guide/)

评估和测试是网站开发过程中的一个重要部分。可用性测试通过一组执行特定任务的用户来收集关于网站可用性的数据。

##### 什么是启发式评估？

启发式评估(最初由 Nielsen 和 Molich 在 1990 年提出)是一种快速、廉价和简单的用户界面评估方法。

这个过程要求一小组测试人员(或“评估人员”)检查界面，并判断它是否符合公认的可用性原则(“启发式”)。目标是识别任何可用性问题，以便它们可以作为迭代设计过程的一部分来解决。

启发式评估在 Web 开发界很流行，因为它在金钱、时间或专业知识方面需要很少的资源。因此，任何开发人员都可以享受可用性测试的好处——不仅仅是那些花费数千美元进行专业评估的人。启发式评估的特点是:

*   使用纸质模型或屏幕截图的小型测试场景，可以很容易地从一个测试场景转换到下一个测试场景
*   不需要心理学家的非正式评估基础
*   高成功率——因此只需要少量的测试人员
*   一些关键的指导方针

##### 我如何在我的网站上使用启发式评估？

***1。计划你的评价***

你将如何测试你的界面？启发式评估通常采用三种主要方法之一:

**1。制定一套任务，并要求你的评估者执行这些任务。**

确定并测试对网站成功至关重要的任务——你会希望所有的访问者都能完成这些任务——以及任何可能给网站访问者带来困难的因素。

**2。向评估者提供系统的目标，并允许他们开发自己的任务。**

一个示例目标可能是“用户应该能够找出产品 x 的价格。”然后，评估者可以将这个目标分解成适当的任务，并依次测试每个任务。

**3。请评估者评估你的对话内容。**

让评估者多次浏览界面，检查和评估那些有助于与网站访问者对话的网站元素的有效性。

选择哪种方法将取决于你自己、你的可用时间以及你的评估者。例如，如果你在评估小孩子，最合适的方法是制定一套任务并要求他们执行。孩子们会发现这比尝试开发他们自己的任务，或者没有任何明显目标的评估你的网站元素更容易实现。

***2。选择你的评价者***

你使用的评估者越多，你暴露的可用性问题就越多。然而，关于该主题的研究表明，在大约五个评估者时，效益/成本比率下降。那么这些评价者应该是谁呢？

*   **有经验的人**——如果你能找到 5 个软件工效学专家，并且是软件应用领域的专家，一个精心设计的评估程序通常能发现 81%-90%的界面可用性问题
*   **没有经验的人**——如果你手边没有 5 个免费专家，也不用担心。一个没有软件人机工程学知识的学生会发现 22%到 29%的可用性问题。

众所周知，如果由 3 到 5 个有经验的人来执行，启发式评估可以发现超过 90%的可用性问题…但是记住，有一个评估者总比没有好！

***3。复习启发式***

一旦你决定了你将采取的方法，并且你已经选择了你的评估者，你将需要向这些人简要介绍你希望他们评估你的网站的十种启发法。

注意:他们应该注意的例子可以在本文末尾的有用网站部分找到。

**1。系统状态的可见性**

网站访问者需要知道的最重要的两件事可能是:

*   “我在哪里？”和
*   “接下来我能去哪里？”。

所以你的界面必须让用户了解正在发生的事情。为了测试这一点，你的评估者应该在每次用户互动后的合理时间内寻找适当的反馈。

例如，一旦用户点击你的订单上的“提交”按钮，几秒钟内他们就会要求反馈，告诉他们他们的订单已经收到。这种反馈可能以单独的页面或弹出窗口的形式出现，其中还包含一个“返回网站”链接，指示用户下一步可以去哪里。

**2。系统和现实世界的匹配**

系统应该说用户的语言，使用用户熟悉的单词、短语和概念，而不是面向系统的术语。即使你可能会使用被认为是标准的行话来描述你的网站所关注的主题，也要考虑对你所使用的词汇进行进一步的简化或解释。

您的评估者应该确保您遵循了现实世界的惯例，并且您的信息以自然和逻辑的顺序出现。应用在网络上的一个真实世界的概念是购物车。在许多网站上，你点击一次来选择一件商品(相当于在真实的商店里从货架上拿起它)，再次点击“添加到篮子”(或放入你的手推车)，然后第三次确认你的购买意图(或移动到收银台)。

**3。用户控制和自由**

网站访问者经常会错误地选择系统功能，并且需要一个清晰的“紧急出口”来离开不想要的页面，而不需要进行长时间的对话。虽然你的网站确实需要秩序，但为了迎合更有经验的用户的需求，可能需要更大程度的用户控制。

你的评估者应该确保你的网站满足新手和有经验的用户的控制要求。控件元素的一个例子可能是出现在每个页面上的“主页”按钮。这是一种让用户感觉控制系统的简单方法——他们知道他们可以在过程的任何阶段“回家”(或选择退出)。

**4。一致性和标准**

用户不应该怀疑不同的词语、情境或动作是否意味着同一件事。最好遵循用户习惯的统一和/或平台惯例。

如果用户想返回主页面，那么就把你的链接标上“主页”或者“首页”，而不是一些晦涩难懂的引用。

**5。错误预防**

比好的错误消息更好的是从一开始就防止问题发生的精心设计。

避免错误的最好方法是进行测试，更多的测试，甚至更多的测试。但是，如果确实发生了错误，尽量用自然语言而不是代码提供用户友好的消息。

**6。识别而不是回忆**

确保对象、动作和选项高度可见。你的站点访问者不需要记住他们与你的站点对话的不同部分之间的信息。无论用户何时需要，系统的使用说明都应该是可见的——或者至少是容易检索的。这增加了你的访问者能够认出他们在哪里的机会，这样他们就不必从主页原路返回。

例如，如果你创建一个有很多子菜单的网站，那么使用一个能让用户随时知道他们在哪个部分的系统。您可以通过留下痕迹或者应用区分不同部分的配色方案来做到这一点。

**7。灵活性和易用性**

新手用户可能看不到的加速器通常可以加速专家用户的交互，并允许系统迎合两种类型的访问者。例如，您可以允许用户定制频繁的操作。

以亚马逊为例。他们保存顾客购买时提供的个人信息。然后，每当顾客再次购买时，他们只需点击一下鼠标就可以检索到他们的信息。通过这种方式，亚马逊为客户提供了一种避免每次在商店购买产品时填写大量表格的方法。

**8。美学和极简设计**

页面上的无关信息会分散注意力，降低速度。让很少需要的信息可以通过链接访问，这样细节是可用的，但不要干扰更相关的内容。

如果您的“联系我们”页面包含一个表格，以及您的所有物理联系信息，如地址、电话号码等。，没有必要附上一张详细说明如何到达你的住所的地图。相反，这可以在一个链接但独立的页面上提供-不是每个填写联系表单的人都希望每次访问该页面时都能看到地图。

**9。帮助用户识别、诊断和恢复错误**

尽管你尽了最大努力去防止，错误还是会发生。你的错误信息应该用简单的语言表达，不要使用代码或行话。他们应该详细说明问题，并建设性地提出解决方案。

例如，如果表单填写不正确，您的错误消息应该提醒您的访问者注意这一点，确定哪些字段需要重新填写，并且可能在用户消除错误消息后返回填写表单时突出显示这些字段。

10。帮助和文档

理想情况下，每个在线系统都可以在没有文档的情况下使用，但是，为了安全起见，可能有必要提供帮助和文档来满足所有用户的需求。

您的评估者应检查以确保帮助和其他文档:

*   很容易搜索
*   关注用户的任务
*   列出了用户实现目标所需的具体步骤
*   不算太大

通常，文档会完全集成到网站中。应该有从主帮助部分到特定子部分的链接，反之亦然。帮助甚至可以完全集成到每个页面中，这样用户就不会觉得帮助太遥远了。

***4。进行评估**T3*

使用以下任一方法进行评估:

*   **个人评估**–每个评估者单独审查界面，并向您报告问题。个人评估很容易在互联网上进行。与小组评估相比，它会发现更多的问题，但需要更多的时间来完成。
*   **小组评估**–评估者作为一个小组审查界面，而你记录问题。评估者不必就某个问题达成一致，但他们发现的每个问题都应该记录下来。小组评估比个人评估需要更多的计划，因为所有的评估者都需要集合起来，但是，评估只需要进行一次，因为所有的评估者可以同时完成他们的任务。

最基本的评估形式是从您的网站中随机选择一个页面，看看您的评估者是否能够:

*   告诉他们在哪里
*   告诉他们下一步可以/应该做什么

***5。分析你的结果***

一旦您的评估人员:

*   努力完成你设定的任务或目标，
*   根据十种启发法评估每一种方法，并
*   提供了他们的反馈，

你需要汇集所有的信息。删除任何重复的问题，合并类似的问题。剩下的将是一系列问题或评论，你可以解决它们来提高你的网站的可用性。

##### 记住黄金法则！

可用性的黄金法则是:

<q>*没有所谓的“用户错误”！*T3】</q>

确保你澄清了你的评估者发现的每一个问题——提出问题，以便你理解他们遇到的困难的具体性质。记住:

*   不要和你的评估者争论，
*   不要试图“解释”他们发现的问题

如果评估者发现你的网站的某个方面令人困惑，那么很有可能你的网站访问者也有同样的问题。
