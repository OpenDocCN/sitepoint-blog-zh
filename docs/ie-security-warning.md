# 又一天，又一个 IE 安全警告

> 原文：<https://www.sitepoint.com/ie-security-warning/>

在[怀疑中国对谷歌](https://www.sitepoint.com/google-threatens-to-quit-china/)的网络攻击促使一些[政府发布 IE 警告](https://www.sitepoint.com/german-government-stop-using-ie/)后不到两个月，微软的浏览器又遭受了一次安全攻击。根据[微软的报告](http://www.microsoft.com/technet/security/advisory/981374.mspx):

> 该漏洞的主要影响是远程代码执行。
> 
> 该漏洞是由于 Internet Explorer 中使用了无效的指针引用造成的。在某些情况下，删除对象后，可能会访问无效指针。在巧尽心思构建的攻击中，在试图访问被释放的对象时，可能会导致 Internet Explorer 允许远程执行代码。
> 
> 目前，我们知道有人试图利用此漏洞进行有针对性的攻击。

该漏洞攻击的目标是 IE6 和 IE7。Windows 2000 上的 IE8 和 IE5 尚未受到影响，但安全公司 Sophos 已经将威胁级别定为[“严重”](http://www.sophos.com/support/knowledgebase/article/110399.html)。

浏览器漏洞对供应商来说从来都不是好消息，但这一发现的时机对微软来说尤其成问题。在推出欧盟浏览器选择屏幕之后，该公司已经开始了一场大规模的 IE 宣传活动来抵制用户迁移。

从积极的一面来看，该漏洞可能会说服一些用户升级到 IE8。尽管很少有用户知道他们运行的是哪个版本——他们可以完全放弃浏览器。然而，更有可能的是，大多数 IE 用户会继续冲浪，完全忘记任何问题！

如果你依赖 IE6 或 ie7，你总是可以禁用脚本。又来了。
