# Scrum 工件:产品增量

以下是摘自我们的书《Scrum:新手到忍者，作者是 M. David Green。世界各地的商店都有出售，或者你可以在这里买到电子书。

scrum 的最后一个工件是实际的 **产品增量** ，因为它存在于 sprint 的末尾，sprint 中所有符合 done 定义的故事都被合并。在每个 sprint 结束时，完成的功能应该被添加到 sprint 演示的产品中。在这一点上，产品本身就是 scrum 过程的一个工件。

![ch5-08](../Images/d2af39d7eefe45eb86257fc2b03a2c22.png)

scrum 过程的一个目标是以这样一种方式开发特性，即产品在每个 sprint 结束时都处于完成状态，因此它可以被发布，向客户展示以获得反馈，或者被用作测试工具。虽然对于一个组织来说，按照 scrum 的时间表发布产品并不是强制性的，但是这个目标允许产品的状态成为 scrum 鼓励的开发、测试、评估和创新的迭代过程的一部分。

产品增量的所有权应该属于大多数组织中的发布工程师，并且应该完全为产品所有者所用。对于 web 和移动项目，产品增量通常是产品的一个现场演示分支，维护在一个安全但可访问的位置。

对于进行持续集成的团队来说，实时站点可能总是反映团队所做的最新工作。在这些情况下，公共网站或应用程序将成为产品增量。
