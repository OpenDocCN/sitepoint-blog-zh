# MSN 偏向 IIS 服务器？

> 原文：<https://www.sitepoint.com/msn-biased-to-favor-iis-servers/>

Ivor Hewitt 做了一些研究，似乎表明 MSN 搜索引擎中排名靠前的页面有更高的比例是由微软自己的 IIS 服务器平台托管的。这部作品很有趣，足以让[斜线点缀](http://slashdot.org/article.pl?sid=05/04/26/1228227&from=rss)，但现在下结论还为时过早。

几个月前，我和艾弗讨论过他的结果，在我接受阴谋论之前，有许多变量需要排除。一个非常大的问题是，这种倾斜是否是由于其他搜索引擎对 ASP.NET 和 IIS 服务器的偏见。

*   您可能知道，ASP.NET 应用程序喜欢在网页中插入一个表单字段(视图状态)。这是一个巨大的字符串(通常大约 15 千字节)的绝对乱码文本。这对 SEO 的影响还不清楚，但如果它确实有所影响，你就会明白为什么它会在搜索引擎的搜索结果中把运行在 IIS 服务器上的 ASP.NET 应用程序推下来，而这些搜索引擎并没有优先考虑(像 MSN 那样)解决它。
*   与 Apache 相比，在 IIS 服务器上运行的动态站点的百分比可能更高。运行 IIS 服务器的一个重要原因是运行在其上的 ASP 应用程序。从我自己的经验来看，我工作过的大多数 ASP/IIS 站点都有重复的内容问题，我们花费大量的时间和金钱来清理那些我们可以用 PHP/Apache 站点在几分钟内修复的东西。
*   拥有更高比例的动态站点(除非你使用 ASP，否则为什么要运行 IIS？)，如果 MSN 的引擎更擅长处理动态网站，或者处理来自 IIS 服务器的特定问题，你会发现这是一种偏见。这完全取决于他们如何排列页面。:D
*   实际上，如果 MSN 在处理动态网站方面“更差”,你可以看到一种偏见，因为他们可能会计算所有重复的内容，并给网站提供链接等。
*   MSN 总体上对动态网站有偏见吗？由于他们的技术，MSN 可能会给模板化页面更多的机会，因为他们将页面分成更小的部分进行分析。
*   如果你相信有更多的大型网站运行在 IIS 上(我不知道这是不是真的)，那么即使稍微偏向大型网站也会导致 SERPs 的一些变化。
*   也许，只是也许，更多的 IIS 网站是 MSN 的 BCentral Submit-It 服务的一部分，这就是真正的偏见所在，M$授予他们自己的目录的某种“权威奖金”。
*   如果 M$真的擅长过滤门户域名、链接农场和垃圾网站，这可能会对有这么多域名的低成本 Apache 托管环境产生偏见。毕竟，如果你花钱购买运行服务器的许可证，你更有可能实际上是在运行一个网站。
*   无论如何，Apache 的市场份额数字可能被夸大了，因为它们是基于主机和域的数量。我很想看看 Alexa Top 1000，看看市场份额细分是什么。那会更能说明问题…有人这样做过吗？

在任何情况下，MSN 的排名和检索算法都不太可能是由网站运营商选择的网络服务器平台决定的。
