# 脸书的新开放图形 API

> 原文：<https://www.sitepoint.com/facebooks-new-open-graph-api/>

上周在 f8 开发者大会上，脸书[宣布了对其 API](http://developers.facebook.com/blog/post/377) 的一些重大更新。你可能已经听说过这一点，但基本上归结为这一点:现在互联网上的每一页都是脸书的粉丝页。好吧，也许我有点夸张，但它的核心是真实的；任何页面现在都可以有一个“喜欢”按钮，这将使它的行为像一个脸书的粉丝页面。当访问者点击那个按钮时，你的页面将会出现在脸书的活动流中，而不需要他们做任何进一步的动作。这种巫术是如何完成的？

您的页面将在其`head`中有几个`meta`标签，这些标签使用脸书的新开放图协议来描述它。元数据的必需位是`title`、`type`、`url`、`site_name`和`image`，还有一些额外的可选位。多亏了这些，Like 按钮才知道如何在用户的脸书新闻订阅中显示你的网站，对它进行分类，并链接回它。脸书的文档网站对[开放图协议有很好很清晰的描述。](http://developers.facebook.com/docs/opengraph)

与普通的脸书粉丝页面一样，如果你将“喜欢”按钮链接到你的脸书账户，你就可以向所有喜欢你页面的人发送消息。

这项新功能附带了一个新的图形 API，供开发人员读写脸书数据，以及一些新的“社交插件”，供您的网站与脸书互动。

这些变化使得开发者可以非常容易地使他们的页面更具共享性，而不必投入资源创建一个单独的脸书粉丝页面，然后需要更新。当网站包含成百上千的页面，希望脸书用户能够分享时，这一点就更加重要了。imdb.com 是脸书发展博客上的一个例子，每个电影和演员页面都有一个赞按钮。

另一方面，一些人可能会认为这些进展是脸书希望其用户将整个互联网视为脸书的一部分。作为一名网页设计师或开发者，你如何看待这种变化？你会急着用脸书的开放图启用你网站的所有页面吗？或者这仅仅是脸书控制更多用户网络生活的又一个游戏？
