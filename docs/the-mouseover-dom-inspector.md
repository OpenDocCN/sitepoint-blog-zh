# 鼠标经过 DOM 检查器

> 原文：<https://www.sitepoint.com/the-mouseover-dom-inspector/>

史蒂夫·奇普曼在[杀手办公室](http://www.slayeroffice.com/)发布了[2.0 版本的鼠标悬停 DOM Inspector](http://slayeroffice.com/tools/modi/v2.0/modi_help.html) 书签。他称之为 MODI，“允许你简单地在文档周围移动鼠标就可以查看和操作网页的 DOM”。web 开发人员的工具，面向 web 开发人员。有了这样的工具，我们离一个完全在浏览器中运行的类似 Dreamweaver 的编辑器不远了。应该有人做那件事；不是浏览器内的富文本编辑器，而是一个合适的编辑套件。你唯一不能做的事情就是将图片或文本拖放到浏览器窗口中，我认为(尽管这可能是最好的方法)没有它我们也能生活。建立这样一个编辑器，世界就会幸福，我想；不要使用 XUL 或浏览器助手对象，没有“安装”程序，没有扩展，只有 DOM。这将是可能的，而且只要每个人都能使用它，这将是非常酷的。它甚至可以避免工具产生的粗制滥造的 HTML。
