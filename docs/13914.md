# IE7 中不会修复什么

> 原文:[https://www.sitepoint.com/what-wont-be-fixed-in-ie7/](https://www.sitepoint.com/what-wont-be-fixed-in-ie7/)

莫莉·霍尔茨施拉格今天来自[她对周二](http://www.molly.com/2005/11/02/microsoft-wasp-task-force-update-upcoming-products-xaml-acid2-sxsw-and-ie7-revealed/)[网络标准项目](http://www.webstandards.org/)的(WaSP) [微软任务组](http://webstandards.org/press/releases/archive/2005/07/05/)会议的报道。

讨论的话题包括微软即将推出的代号为“石英”的[网页设计工具](http://www.microsoft.com/products/expression/en/web_designer/)、 [XAML](http://windowssdk.msdn.microsoft.com/library/en-us/wcp_conceptual/html/0ff5f36e-dd84-44d1-aa3e-5bb4f147b169.asp) 、Internet Explorer 对 [Acid2 测试的方法](https://www.sitepoint.com/safari-wins-the-acid2-race/)，以及在 3 月份的 [SXSW](http://sxsw.com/) 大会上为 WaSP 制定的计划。

然而，一个被埋没在最后的金块是 IE 浏览器首席项目经理克里斯·威尔逊列出的可能不会在 IE7 中修复的事情:

> 但是，有些东西根本就不会有。生成的内容？“不会成功的”威尔逊告诉我们。有一个溢出问题可能不会被修复，图像对象很可能不会在 IE7 中被修复。
> 
> 然而，威尔逊仍然保持乐观和达观，他在结束我们的谈话时说:“当我们开始 IE7 时，我知道它对我们来说将是一个具有挑战性的版本，我们不会达到人们希望我们达到的程度。”

生成的内容——或者更准确地说，对 CSS2 [`content`属性](https://www.w3.org/TR/CSS21/generate.html#propdef-content)的支持——并不奇怪；大多数其他浏览器对此的支持也是有限的。但是令人遗憾的是，像不能对图像使用`<object>`标签和`overflow`属性的一个错误这样的长期痛点将不会得到解决，这可能会影响 CSS 页面布局。

也就是说，微软*和*似乎仍然忠于其宣称的目标，即使用 IE7 来纠正最影响 Web 开发人员完成工作能力的错误和问题，并将更多“为标准而标准”的修复留给 IE8。

现在公开测试版怎么样了？
