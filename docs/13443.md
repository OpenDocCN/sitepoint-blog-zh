# 处理禁用 JavaScript 的浏览器

> 原文：<https://www.sitepoint.com/handling-javascript-disabled-browsers/>

以下内容转载自《科技时报》第 159 期。

在之前的中，我提到过在你的网站中添加 JavaScript 时，你应该考虑三种主要人群——没有 JavaScript 能力的用户、没有鼠标的用户和屏幕阅读器的用户——并且支持这三种人群变得越来越困难。在本帖中，我们将关注第一类人，并发现用正确的方法来适应他们是多么容易。

在我最近合写的一本介绍 JavaScript 的书中，第一个大的例子是手风琴控件。这个用户界面元素折叠一系列内容块，以便只显示它们的标题，然后允许用户通过单击标题一次展开一个内容块。

![](img/c811db9272449ff3ea956a03181fa244.png)

这是一个很好的例子，说明了正确的方法如何适应没有 JavaScript 支持(或者禁用了 JavaScript)的用户，而不需要额外的工作。

编写页面的 CSS 代码来折叠 accordion 控件的块，然后编写 JavaScript 代码在适当的时候展开这些块，这很有诱惑力。但是禁用 JavaScript，用户只能看到折叠的块，无法阅读它们！

在过去，传统智慧提倡一种叫做**优雅降级**的方法，这种方法包括为用户或浏览器添加额外的“回退”代码，这些代码不能处理页面中更好的部分。

例如，解决 accordion 问题的优雅方法可能是将 accordion 内容的扩展版本放在`<noscript>...</noscript>`标记之间，这样没有启用 JavaScript 的浏览器就可以显示扩展的内容。这是额外的工作，也是让开发人员对可访问性感到不快的事情。

今天，我们明白这是错误的做法。

解决可访问性问题的一个更好的方法是通过**渐进增强**，在这里你开始构建一些在最简单、最易访问的环境中工作的东西(在这种情况下，一个禁用了 JavaScript 的浏览器)，然后用一些特性来渐进增强它，这些特性将在支持它们的浏览器中改善用户体验，或者在不支持它们的浏览器中无声地失败。

这对我们的手风琴意味着，我们应该编写页面的 CSS 代码，以便手风琴的内容是完全可见的。

![](img/ad217984fcd76b1821e4a2b1d601e05b.png)

JavaScript 代码将在加载后立即折叠手风琴，但在禁用 JavaScript 的浏览器中，手风琴将保持展开状态，阅读内容也不会有困难。而且不需要编写任何额外的代码或花费任何额外的时间来实现这一点——这只是一个方法的问题。

下一次，我们将看看如何确保这个 accordion 控件可以被那些不能使用鼠标的人或者只是喜欢用键盘导航的人使用。这将被证明更具挑战性，但在大多数项目中仍然非常实用。

如果你想阅读更多关于优雅退化和渐进增强的内容，我强烈推荐 SitePoint 的常客 Tommy Olsson 在 Accessites.org 发表的关于这个主题的文章。
