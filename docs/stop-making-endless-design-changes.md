# 停止无休止的设计变更

> 原文：<https://www.sitepoint.com/stop-making-endless-design-changes/>

上周，我谈到了[范围蔓延](https://www.sitepoint.com/stop-the-slippery-slope-of-scope-creep/ "Stop the Slippery Slope of Scope Creep")的滑坡，它可能始于客户在最后一刻对您已经批准的设计进行变更。

当客户在生产阶段对你的设计有第二个想法时，他们可能从一开始就没有完全满意过。或者可能是某个神秘的第三方建议了不同的布局、配色方案或最初没有讨论过的功能。

限制变更数量的诀窍是将最初的设计尽可能贴近客户的愿景。这意味着你需要问问题。很多问题。

我总是问的第一个问题是，“你心中有没有特别的设计或布局？”如果客户对他们希望网站是什么样子有预先的想法，我想事先知道。不管他们有没有，我问了他们喜欢的其他网站的例子，以及他们为什么喜欢这些网站。如果他们有一个现有的网站，我问他们喜欢和不喜欢它。

我问的下一件事是关于他们竞争对手的网站——他们喜欢哪些，不喜欢哪些，为什么。

一个有用的技巧是让客户列出一些形容词来描述他们想要的新网站的外观和感觉，比如“公司”、“保守”、“艺术”或“优雅”。然后我列出了大约 100 个不同的单词，并让客户圈出最能描述他们公司的单词。

我听到很多设计师抱怨说，当他们没有想到客户的想法时，他们“不是一个读心者”。但你不必是千里眼；只问正确的问题。

只有一次，我创造了一个与客户想要的完全不同的设计。她期待“艺术”，我给了她“公司”但这是我的错；我的设计概要问题不够彻底。

现在是时候向客户展示你的模型了。假设你确定了设计，任何改变都应该是微小的。如果客户希望他的 logo“向左 3 个像素”，这没问题，但是你会允许多少次呢？

如果你读了我的 **27.5 咨询式销售的必问问题**，那么你就知道确定所有决策者并提出正确的问题是很重要的。好了，现在你要重复这个过程来“完成”网站的设计。就像你们就一起做生意达成一致一样，现在是时候来来回回了，直到你们就设计达成一致。

你认为这个项目的“销售”部分已经结束了。

不管你是否允许两个或二十个设计变更，确保你的客户在这个阶段之前就知道你的限制。不管你选择什么数字，设定一个限制并不意味着你必须严格执行。我们都希望我们的客户满意；设定限制可以让你约束那些似乎永远不会被约束的人。
