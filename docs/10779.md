凯文:2010 年 7 月 23 日。在 WordPress 的世界里，谷歌可能会监控你的鼠标，并搜索合理的流量统计。这是 SitePoint 播客#71:旋转的互联网。

欢迎来到 SitePoint 播客！本周一场奇怪的非技术性表演。我通常是贡献所有代码 y 技术主题的人，伙计们，然后我看着他们，我说我们不可能在播客中涵盖这些；没人想听我在播客上朗读代码。但是这个星期什么都没有，全是搜索的东西，有分析的东西，还有布拉德，有一个法律问题。

布拉德:正如他们所说，WordPress 世界正在酝酿一场风暴，这场风暴实际上发生在 WordPress 平台和一个非常受欢迎的主题“论文”之间。归根结底，WordPress 本身是在 GPL 许可下发布的，特别是版本 2，它本质上规定了任何 WordPress 的衍生作品也必须在 GPL 下发布，这就是 GPL 许可的本质。问题是论文没有 GPL 许可；它实际上是在一个不同的软件许可下，一个专有许可，所以 WordPress，更确切地说是 WordPress 的创始人之一，最初的开发者之一，马特·莫楞威格，实际上已经开始攻击论文的开发者克里斯·皮尔森，本质上是试图让他转换他的许可以符合 GPL。

凯文:他一点也不接受。

布莱德:完全没有。这一切基本上始于 Twitter 上的一场唇枪舌剑，就像这些天他们通常所做的那样。

凯文:这一期的销售主题，是我不时听到的话题。所以这不是一个新问题，这个问题已经争论了几年了，对吗？

**Brad:** 是的，需要澄清的是，我刚刚看了很多博客帖子，有很多都是关于这个话题的，这并不是专门关于销售主题或者从主题中赚钱或者从 WordPress 中赚钱，这完全是关于许可的。所以你当然可以拥有一个 GPL 兼容的主题或插件，并出售它，这完全是你在 GPL 下的权利。

凯文:你能？

布莱德:对，绝对是。它唯一声明的是，如果你出售它，你必须提供源代码，原始源代码，诸如此类。

凯文:啊。

Brad: 所以如果我购买了你的主题，我可以随意修改它，而不用担心破坏任何软件许可，如果我愿意，我还可以重新发布它。所以这绝对是一个棘手的话题，因为围绕 GPL 的诉讼案件并不多，虽然有一些，但没有一个确定的答案。所以它真正归结为 WordPress 或任何其他平台中的一个主题，这被认为是 WordPress 平台本身的衍生作品吗？这才是真正的论点。

凯文:我读到的两个观点是，马特·莫楞威格认为主题是一个将 WordPress 作为一个库的软件。因此，如果你在你的 WordPress 博客上安装了论文，然后有人来了，想要查看一篇特定的博客文章，论文就开始说，哦，WordPress，我需要这篇文章的文本，WordPress，我需要这个，WordPress，我需要那个。每次它向 WordPress 请求某些东西时，它都在使用 WordPress 的一部分功能，如果你用一段 GPL 代码做这种事情，它会被很好地接受，即使不一定像你指出的那样在法庭上测试，但它被很好地接受和理解，如果你这样做，就像你说的，你必须发布你构建的软件的源代码。克里斯·皮尔森的反对观点似乎是主题不是使用 WordPress 的软件；更确切地说，WordPress 就像一个运行主题的运行时。就像如果你买了一个唱机，你把你的黑胶唱片放在上面，它就开始旋转，那张唱片不是建立在唱机上面的，唱机是播放唱片的工具，因此如果唱机是在 GPL 下构建的，你就不必发布你的黑胶唱片设计。我知道我在这里夸张了这个比喻，但这有点——这就是他的提议。看起来群众的力量并没有站在他这边。我知道这些年来我已经为一两个开源项目做出了贡献，GPL 代码，嗯，如果你正在从事一个 GPL 项目，GPL 是非常重要的。当你不在 GPL 代码上工作时，它似乎被看得更严重了。我参与了一个项目，它是在比 GPL 更宽松的许可下发布的，每次我贡献东西的时候，你都必须签署一份电子弃权声明，声明，是的，我贡献的代码没有一个是 GPL 许可的。因为如果是的话，他们称之为病毒许可，因为如果你在你的项目中使用任何 GPL 许可的东西，你的整个项目都必须是 GPL 许可的。

**Brad:** 这也是在 Twitter 和 Mixergy 的 Andrew Warner 大爆发后，实际上有 [Chris Pearson 和马特·莫楞威格在 live](http://mixergy.com/chris-pearson-matt-mullenweg/ "Would WordPress Sue The Maker Of Thesis, A Leading WordPress Theme? – with Chris Pearson and Matt Mullenweg") 上几乎是来来回回的战斗，WordPress 的所有核心开发人员和其他一些开发人员实际上是在论文代码中潜水，他们发现 WordPress 的完整代码几乎是直接从 WordPress 中一行一行提取出来的，然后在论文中使用。

凯文:哎哟。是啊。

Brad: 所以这显然是对 GPL 的侵犯。现在如果没有这些代码，你知道，如果这些代码不存在，皮尔森说他正在做的下一个版本中也不会有这些代码，但是即使这些代码不存在，如果他的主题不是 GPL 的一部分，这是否仍然被认为是违反 GPL 的？这是，我的意思是这是一个非常有趣的话题，尤其是在法庭上，因为有很多非常流行的开源平台，比如 Drupal 和 Joomla，它们都是在 GPL 下获得许可的。所以这真的可以为 GPL 的主题部分开创先例；有一个插件或模块被认为是属于 GPL 的。因此，这肯定是我们将密切关注的一个问题。

凯文:我认为，作为一个天真的软件开发人员，这是很容易犯的错误。通常，你知道，当你开始构建你的第一个 WordPress 主题时，我怀疑软件许可不是你首先考虑的事情。这是 Mullenweg 所说的，如果你想为一个博客平台做贡献，而 GPL 的核心信念，GPL 的文化，是你不认同的，那么你应该去 WordPress 以外的地方看看。我有点同意这一点。有些开源项目属于 GPL 世界，有些不属于 GPL 世界，在这两者之间有一堵非常坚固的墙。如果你想参与像 WordPress 这样的 GPL 软件领域，如果你想成为一名开发者，你必须尊重成千上万的其他开发者，他们付出了时间来构建一个像 WordPress 这样复杂的软件产品，所有人都明白他们的代码被贡献给了受 GPL 保护的东西，所以没有人可以从他们的工作中受益，如果不支付报酬，也就是说，如果不发布他们的进一步工作来进一步造福大众。这是 GPL 的理论，如果每个人都同意这一点，那么每个人都在为更大的利益而努力，但对于像克里斯·皮尔森这样的人来说，建立论文并说，不，我在这里画了线，我的工作是我的，我不在乎我站在谁的肩膀上，这就是我认为情绪问题所在，以及为什么这么多人对此感到不安。

我管理 phpBBhacks.com 已经十年了，所以我见过的开源政治比我这辈子想看到的要多得多。

凯文:(笑)

帕特里克:他们有时会变得非常非常情绪化，非常政治化，非常令人讨厌。我的意思是，你认为开源你认为鲜花和玫瑰之类的东西，但这并不总是那么回事。我不一定同意 WordPress 发生的所有事情以及他们处理这些事情的方式，我认为有些事情是有害的；他们最近一直在镇压。我真正开始谈论它并思考它是在五月的罗利世界语言营，仅仅是因为世界语言营周围发生的不同的事情，他们实际上说如果你运行一个非 GPL 的主题，你不能在世界时期的世界语言营演讲。这是一件有趣的事情。我觉得你可以走得太远；当你是一个开源项目时，你必须小心你如何施加控制。我认为这是一个很多人可以团结起来的主题，但是有很多灰色地带的主题，在那里，在文字上，精神上应用 GPL 可能不是最好的事情，因为当你如此开放，你如此自由地可用，当你最终施加控制时，你有一群人习惯于做他们想做的任何事情，现在他们认为你在控制他们。所以你必须非常小心。在这个特殊的问题上，我认为——我不认为皮尔逊是正确的，这也是我们在 phpBBhacks.com 一直做的，phpBB 是 GPL，是一个非常受欢迎的 GPL 项目，我们一直把这种风格的代码视为 GPL，图形和 CSS 不一定，图形当然不是默认的，因为图形显然是独立工作的，它们不占用任何 phpBB 代码， 所以我们一直认为这些至少是有版权的元素，而主题代码本身通常使用某种形式的 phpBB 代码，它必须发挥作用，所以这一直是 GPL。 我们的数据库总是超越这个标准，并且总是征求作者的同意，并且认为这是非常神圣的。但代码 GPL，CSS 和图形不是默认我会说，所以这就是我期望在这里发生的事情。我认为这是一件有趣的事情，因为他们已经谈论这个问题很多年了，就像你说的，我在罗利世界营和某人交谈，我说在一天结束的时候，我厌倦了谈论，就像我们要永远谈论这个吗？你们要去街上打一架吗？你会去法院提起诉讼吗？尽管去做吧，因为你已经谈论这个问题很多年了，到头来你只是在浪费很多人的时间。所以要么我们去做，要么我们不做，我们只是继续前进。所以我不知道这里的最终结果是什么，但我希望看到一个最终结果。

凯文:是的。

沿着同样的思路，马特·莫楞威格实际上与软件自由法律中心进行了交谈，以澄清 WordPress 主题是否属于 GPL 的范畴。[他们的结论](http://wordpress.org/news/2009/07/themes-are-gpl-too/ "WordPress › News » Themes are GPL, too")实际上是 PHP，就像你说的 Patrick，利用 WordPress 版权代码的 PHP 文件受 GPL 保护；然而，图像、CSS、JavaScript 不是，所以你实际上可以在 WordPress 主题上有一个分割许可，并且仍然完全在 GPL 之内，仍然不必担心任何类型的诉讼。所以，凯文，你是对的，如果你进入了 WordPress 主题，我的意思是，这是大多数人可能从来没有想过的事情，直到为时已晚，也许他们面临诉讼。所以你可以整天整夜地阅读这些东西，但我们会看到是否会发生什么，如果 WordPress 和 Mullenweg 站出来，实际上把它带到法庭上，这看起来像是下一步，因为在 Mixergy 采访上肯定没有任何形式的解决方案。

Patrick: 你知道软件自由法律中心，我不一定认识他们，但是从他们的名字我可以猜测他们可能对开源项目有一点偏见，也许，也许。

凯文:也许吧。

Patrick: 所以每个人都有自己的观点，就像我说的，这些观点已经听了两三年了；我是说有些事情迟早会有个结果。让我们开始起诉对方，或者做些别的事情，而不是恐吓对方，利用我们成千上万人的社区来攻击对方，让我们做些别的事情。

布莱德:起诉或者滚蛋。

Patrick: 基本上，是的，但我想说的另一件事是论文的一部分，我会说我有论文许可证，我不使用它，我不会使用它，我只是不想在这个时候使用它，但我已经购买了它，我已经看过它，论文的部分吸引力在于它的后端功能，便于定制。所以这些都和 WordPress 管理区有关系，所以它们当然最有可能是 GPL。我的意思是，这是这个主题的魅力所在。这不是一个真正的图像重主题。是的，CSS 是它的一部分，不管怎样，它是强大的，但我的意思是，论文的主要部分使它对人们如此重要，是编码，是管理领域的那些功能。我的意思是，是的，这绝对是一个强大的东西。这就是人们购买它的原因——而不是为了图像。

布拉德:这是框架。

凯文:是的，论文是这些主题中的一个，它给你打基础，一个你期望自己剥皮的基础，对吗，布拉德？

Brad: 是的，没错，这是一个主题框架，就像 Patrick 说的那样，我的意思是，即使你可以将 CSS 和 JavaScript 放在一个单独的许可证中，从技术上讲，Thesis 背后可能有 5%的代码库实际上属于这个框架。我的意思是它的大部分是 PHP 代码，你知道，它是一个框架，它有大量的后端选项，你可以很容易地点击来改变你的网站。但它不是非常图形重，它不像你买一个风格化的，设计好的主题，你买一个框架，允许你以你想要的方式开发主题。我认为这可能是克里斯·皮尔森有点担心的原因，因为他不能真的做分割许可证，这不会真的按照他想要的方式让他受益。

凯文:这个裁决，引用/未引用的裁决，来自软件自由法律中心，可以追溯到去年 7 月，所以这是认真进行了多久。真正的问题是，马特·莫楞威格是要跳起来还是让它滑下去，感觉他已经让它滑了至少六个月，或者至少一年了。问题是，WordPress 的所有者 Automattic 起诉 WordPress 开发社区的一个成员，一个杰出的成员，会对生态系统产生什么影响。但是帕特里克，我倾向于同意你的观点，争论和不确定性所造成的伤害对我来说也是一样糟糕。因此，也许自私地说，我真的希望他们继续进行诉讼；我可以这么说，因为我不用付律师费。

帕特里克:对。你在社区中有一个人是可见的，他无视许可证，让我们说，让我们离开任何指控；我们只能说他肯定违反了许可，他在向 WordPress 吐口水，不管怎样。那么，这值得起诉并为此花费金钱和时间吗？我的意思是，如果我们只是说，好吧，你知道吗，他正在这样做，我们已经暴露了这一点，人们知道这一点，那些强烈感觉不喜欢他的人，很好，我们知道了，他已经被吓到了。现在，有没有，我猜有没有一些理由继续进行诉讼？这是一个如此坚定的原则，好吧，我们被冒犯了，我们不被尊重，我们要花 20 万美元打官司，或者不管是什么，这是对我们资源和时间的一个很好的利用。不要只是说好吧，他已经做到了，让我们继续前进，让我们继续做出好的东西。我的意思是，我可以看到任何一种说法，但我只是说，WordPress 开发者和在 WordPress 本身的创建和维护中起作用的人在这方面花费了大量的时间和资源，所以我不知道这是否是——我不知道当我们谈论一个开源、免费可用的软件时，这种事情是否是对资源的最佳利用。通常这类事情是留给人们真正赚钱的时候，他们的钱受到影响，然后人们花钱来保护这些钱。不要误会我的意思，这里有很多赚钱的人，他们都参与了这场争论，但我不知道这是最好的利用资源来推进这一步，尽管可能有人希望这样做。
