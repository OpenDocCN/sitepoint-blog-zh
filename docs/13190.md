# 克拉克呼吁解散 CSS 工作组

> 原文:[https://www . site point . com/Clarke-calls-for-CSS-working-group-to-dissolved/](https://www.sitepoint.com/clarke-calls-for-css-working-group-to-be-disbanded/)

**最近宣布了 [CSS Eleven](http://csseleven.com/) 倡议，为 W3C 的 CSS 工作组提供设计师反馈和意见，安迪·克拉克通过[呼吁该工作组完全解散](http://www.stuffandnonsense.co.uk/malarkey/more/css_unworking_group/)并在没有浏览器供应商控制的情况下重建，对[Opera-微软反垄断行动](https://www.sitepoint.com/operas-antitrust-complaint-microsoft-must-support-standards)做出了回应。**

他认为这是必要的，不仅因为他怀疑 Opera 和微软的代表在陷入法律纠纷时能否在 CSS3 上合作，还因为他觉得是时候由我们这些最终将在日常工作中使用它们的人来引领 web 标准的未来，而不是那些希望通过制作浏览器赚钱的人。

Clarke 对 Opera 法律行为的控诉得到了许多网页设计社区的响应。CSS 专家 Eric Meyer 认为 Opera 的举动是一个糟糕的时机，正值微软展示 IE7 和即将到来的 IE8 的前景:

> 这是在错误的时间采取的错误行动，向微软发出了关于参与开发和支持开放标准的重要性的错误信号，我只能希望它悄无声息地结束。

但是似乎很少有人同意克拉克重组 CSS 工作组的提议。许多人认为该组织仍有生命力，而其他人则呼吁全盘放弃 W3C 过程。

所有这一切中理性的声音似乎来自 Dojo Toolkit 的亚历克斯·罗素。在他的文章 <cite>[中，W3C 无法拯救我们](http://alex.dojotoolkit.org/?p=642)</cite> ，他解释说，真正阻碍网络发展的是我们对网络标准的狂热追求，以及对网络标准能够决定网络浏览器应该增加哪些新功能的期望。

> 简而言之， *[泽尔德曼](http://www.zeldman.com/)正在伤害你*，只有你能让它停下来。无论是 CSS WG 还是 HTML 5 WG，甚至任何 W3C 工作组都无法定义未来。一旦未来成为过去，他们只能圆掉锋利的边缘，这就是我们对他们的所有期望。尽管他们告诉我们(和他们自己)他们可以，尽管他们真的很想这么做，W3C 也救不了我们。

科技时报的长期读者不会对我同意亚历克斯的观点感到惊讶。他的观点和我在 2006 年 4 月的 *[Tech Times #137](https://www.sitepoint.com/the-second-coming-of-the-productivity-tool/)* 中所说的差不多:

> 在我看来，从零开始开发新的标准不应该是 W3C 的工作，W3C 也不应该负责支持单个浏览器的新特性。这些是创新者和早期采用者的工作，他们推动可能性的边界，产生早期实现，为未来的标准铺平道路。

亚历克斯没有提到的一个症结是软件专利。如果浏览器为他们开发的每一个创新特性申请专利，这些特性将不会免费提供给 W3C 标准化以供其他浏览器采用。但也许这是一个比我们目前面临的问题更小的问题。

无论如何，W3C 需要停止展望未来；直到他们这样做，我们其余的人将停留在过去。W3C 非常有能力编写可靠的规范来描述今天的浏览器做什么。他们应该坚持这一点(这是一个大工程！)，并让世人知道在网页浏览器中加入非标准的功能并不是犯罪。

未来不是由一个工作组的共识建立的；它是由有远见的人不断尝试和犯错误建立起来的。
