# 参见目标会议总结

> 原文:[https://www.sitepoint.com/cfobjective-conference-wrap-up/](https://www.sitepoint.com/cfobjective-conference-wrap-up/)

由 Jared Rypka-Hauer 举办的“ColdFusion MX 程序员企业工程大会”的第三期活动 Objective() 2008 已经结束了一年。比较 Objective()在会议中的独特之处在于它几乎只关注高级主题。它似乎也在社区中产生了巨大的反响，并吸引了来自世界各地的与会者来到明尼阿波利斯。

看起来大多数代表已经回家并更新了他们的博客，所以我想是时候看看他们在这三天里都学到了什么…

Brian Rinaldi 为 Fusion Authority 撰写了一篇全面的总结文章。他指出，随着 CF 社区的不断成熟，开发人员正在寻找更高级的工具来解决问题，即 Java，这在许多会议主题中都很明显。他还指出，Flex 是一个热门话题，有太多的会议，开发人员可能会花整个会议的时间只关注 Flex。Brian 还详细回顾了几个环节:[在敌对店铺卖职业发展 Terrence Ryan](http://www.remotesynthesis.com/blog/index.cfm/2008/5/3/cfObjective-Professional-Development-Hostile-Shop-Terrence-Ryan)；[利用 Brian Kotek 的代码生成](http://www.remotesynthesis.com/blog/index.cfm/2008/5/2/cfObjective-Leveraging-Code-Generation-Brian-Kotek)；[Laura argu ello 的 Mate Flex 框架](http://www.remotesynthesis.com/blog/index.cfm/2008/5/2/cfObjective-Mate-Flex-Framework-Laura-Arguello)；Mark Mandel 的[传输缓存机制](http://www.remotesynthesis.com/blog/index.cfm/2008/5/3/cfObjective-Transfer-Caching-Mechanisms-Mark-Mandel)；和[Flex:Maxim Porges](http://www.remotesynthesis.com/blog/index.cfm/2008/5/3/cfObjective-Flex-No-Frameworks-Maxim-Porges)不需要框架。

马克·曼德尔写下了他的亮点，其中包括关于模型胶水、ColdSpring 的研讨会，以及特伦斯·瑞安关于在敌对商店销售职业发展的会议。在那里，Mark 介绍了 Transfer ORM(现在是 1.0 版本)的两个会议，反响很好，请关注不久的将来在 SitePoint 上对 Mark 的采访。他指出，ColdFusion 9 的信息被披露出来并没有什么特别令人惊讶的，但仍然是受欢迎的。哦，马克是任天堂 Wii 的幸运赢家！

Brian Meloche 发布了三份每日总结(第一天、第二天和第三天)以及一份总体评估，给会议打了“A”分。Brian 详细介绍了他参加的每场会议，并特别提到了 Open BlueDragon initiative(在会议期间发布了测试版),还介绍了 BF9“一丘之貉”会议上讨论的内容。他感到失望的是，Adobe 没有更多强调对专用 ColdFusion IDE 的需求。

会后发表了很多材料！更多亮点包括:

*   Brian Kotek 从会议中得出一个印象，ColdFusion 正处于十字路口。他对这门语言未来发展方向的想法很有趣。
*   [Terrence Ryan 以易于阅读的要点](http://www.numtopia.com/terry/blog/archives/2008/05/cfobjective_2008_postscript.cfm)发布了他的会议后记，并创建了一个 cf.Objective() 2009 待办事项列表。
*   彼得·贝尔总结了一大堆演示文稿，提到了目标()“震撼人心”。
*   Adam Lehman 发布了一个建议，即 [cf.Objective()可能会扩展到欧洲和/或澳大利亚](http://www.adrocknaphobia.com/post.cfm/cfobjective-in-australia-and-europe)。
*   向与会者分发了最新一期《融合权威季度更新》的免费副本。
*   安迪·鲍威尔用一个时髦的幻灯片发布了他的 cf.Objective() Flickr 流。

*   [彼得·J·法雷尔的 Mach-II 1.6 的新功能](http://blog.maestropublishing.com/index.cfm?mode=entry&entry=BB10C862-1372-3F66-7083755CF910D405)和使用 ANT 的

*   Andrew Powell 的 ColdFusion 和 Java 企业 MVC

几乎每个帖子都提到的一个因素是，是人让 cf.Objective()如此伟大。史蒂夫·布莱恩特抑制住了做一些抛头露面的冲动，而[本·纳德尔把这种经历比作在巨人中间行走](http://www.bennadel.com/index.cfm?dax=blog:1216.view)。这无疑是一个反复出现的主题——正如许多与会者希望回来参加 cf.Objective() 2009 一样！
