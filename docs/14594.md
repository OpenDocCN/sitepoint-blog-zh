# 关键区别:实践与项目领导

以下是我的回应:

首先，我希望 Sitepoint 的所有读者将他们的业务视为与客户的专业实践，而不是一系列项目或“演出”拥有一个专业的实践意味着当你与客户一起工作时，你有一种特定的态度，类似于律师、医生或会计师。你专注于服务、持续改进、卓越和长期关系。客户对他们赞不绝口(一般来说，大多数人都抱怨律师，但他们热爱自己的律师)，并在一生中对他们保持忠诚。他们认为他们是各自领域的专家，随着时间的推移，他们会将机密信息托付给他们。

相比之下，专注于项目和工作的 IT 专业人员是雇佣军和供应商，被迫进行价格竞争，几乎总是对项目进行投标。他们把自己放在第一位，顾客放在第二位。他们关注的是任务而不是结果，只是完成工作，然后继续前进。

我更愿意成为一名与客户打交道的专业人士。你呢？

同时，对于 Roly 提出的关于预算和没有空间与客户和潜在客户“畅谈”的问题:

无论你的实践多么专业，你仍然需要考虑预算和范围。在网页设计/开发和任何职业中，管理项目是成为专业人士的关键要素。事实上，在美国，最好的医生是那些找到优化质量和成本的方法的人，他们倾向于依赖严格的临床协议和指南。

每个潜在客户/客户都有固定的预算。因此，你的部分实践是帮助客户定义范围。直到你告诉他们，他们才意识到每个项目都需要平衡三件事:时间、资源和质量。最重要的是提前与你的潜在客户反复讨论，找出这三个元素中的哪一个是固定的，然后调整另外一两个元素，以达到一个令人满意的项目定义。与客户进行这样的对话并不总是容易的，与他们一起达成满意的解决方案需要时间。

在这个框架内，你可以说这样的话，“我知道你的预算是固定的，但如果你愿意多付一点，看看我们能为你做些什么……”这样，你就能准确地找到客户的真正界限在哪里。

然后，正如 Roly 指出的，你必须设法非常仔细地确定范围，以保持盈利。

因此，专业实践和严格按照预算管理并不排斥。他们一起去。

谢谢你的好问题！

附注:关于专业实践中的人员配备，各种模式都适用:承包商、兼职员工、全职员工、只有在有工作时才领工资的员工……一项实践的与众不同之处在于，所有者承诺制定标准，并采用一种方法/流程来确保高质量的工作、可靠的结果、客户的顺利流程以及相互有价值的关系。
