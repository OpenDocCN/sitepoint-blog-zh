# SXSW–第二天，依然强劲

> 原文：<https://www.sitepoint.com/sxsw-day-two-still-going-strong/>

SXSW 最大的问题之一是知道什么该参加，什么不该参加。在任何给定的时间内，您最多可以参加六个不同的会议。这意味着每天你不得不错过五次训练，四次以上。当你付了注册费后，错过这么多潜在的好机会似乎很遗憾，但事实就是如此。今天早上我不得不从以下两者中做出选择:

*   web 应用程序中的热点是什么
*   尊重年长的博客作者
*   重要的创业决策
*   如何/为什么播客活动
*   数字融合业务
*   设计和社会责任

你会选择哪个？今年，我试着不根据谁来做判断，而是只根据标题。许多会议听起来像我在以前的 SXSWs 上参加的一些会议，或者它们根本不适合我(就像上面提到的启动会议)，但也许在那个会议上的建议对所有网站都有好处？这很棘手，没有什么比从一个平淡无奇的会议中走出来，听到其他人对他们去过的会议赞不绝口更令人恼火的了——你押错了赌注！

第一节课，我选择了“[设计与社会责任](http://2006.sxsw.com/interactive/programming/panels/?action=show&id=IAP060015)”。对于内容可能是什么，我没有先入为主的想法，但发现贯穿着一个强烈的可访问性主题时，我并不太惊讶。使基于 Flash 的站点或应用程序可访问的问题突然出现了，这是不可避免的，并且包括了一个如何正确做的很好的例子。小组成员之一，[西娅·伊顿](http://www.knowbility.org/conference/?content=theaEaton)演示了 Flash 的典型用法——这是一种针对儿童的拖放式拼图游戏，也可以使用按键和音频反馈来工作，这样盲人用户可以使用不同的技术来完成拼图。显然，盲人用户和有视力的用户的体验是不同的，但却是相似的，这表明只要稍加思考，看似困难的任务对两者都是可行的。在附带的幻灯片上引用了短语“但是 flash 和 web 是可视化工具”和“我们将不得不牺牲交互性和趣味性”，这是我们都习惯听到的典型评论，看到这些被如此成功地揭穿令人耳目一新。小组成员之一， [Gordon Montgomery](http://www.gmeta.com/) ，做了一个我所见过的最快速的演讲，一张张幻灯片的引用和想法与评论完全同步地出现在屏幕上。这并不是说它很仓促，只是我很惊讶他是如何保持完整的(我们以前在做演示时都失去了位置，当你跳到下一张幻灯片的内容时，你必须赶上自己，对吗？).关于 Flickr ，他的一条评论对我来说是正确的。现在，我爱 Flickr，爱它的工作方式，爱它的社区方面和一切，但正如他所说的“我妈妈仍然不能使用它”。这是一个及时的提醒，即使是对你我来说很容易使用的东西，也可能会错过大部分潜在用户的目标。有人喜欢 Flickr lite 吗？

然后我去了一个名为“我们和他们:博客对话生存指南”的小组。我第一个承认，我并没有真的在我的[个人博客](http://lloydi.com/blog/)上投入任何努力，但我对公司博客的影响感兴趣(我仍然有些犹豫不决，或者至少像这样的东西可能对我工作的[公司](http://www.nationwide.co.uk/)有多大用处，我在那里提出了这个想法)。事实证明，关于企业方面并没有太多可怕的东西，而是归结为一些简单的常识性想法。在这些信息中，文明是最重要的(其中一条称之为“旅馆”规则，指的是帕特里克·斯威兹电影中的一个场景，他告诉门口的人，如果有人叫你白痴，就直接对他们友好——这并不总是一件容易的事情！).会议中的其他引用包括“不要点燃超过你能扑灭的火”，“坚持你自己”和古老的“不要喂巨魔”。

观众中有人讲述了一个很棒的故事，一群诗人在一个论坛被关闭时，突然失去了他们的虚拟聚会场所，因此为他们的诗歌找到了另一个家——在这个女人的留言板上。她说，有一群诗人被释放，他们认为她的留言板是为他们准备的，她必须赶走他们，讨厌的域名抢注者！观众中的另一个家伙告诉我们一个身份盗窃事件，有人用他的名字和网址在其他人的博客上发表评论，留下非常不恰当的评论，完全违反了公司的道德规范(他是一名作者，书中的内容说要采取与这个幽灵海报所采用的策略完全相反的做法)。这肯定不是我以前会考虑的可能性，只是表明写博客不仅仅是决定评论是否适度。

下午，我决定参加“ [Web 2.1:让 Web 2.0 变得可访问](http://2006.sxsw.com/interactive/programming/panels/?action=show&id=IAP060016)”会议。这是在最大的房间里举行的，出席人数很多——显然这是一个人们强烈关注的话题。这是我迄今为止做笔记最多的一次会议，我拼命想跟上小组成员的进度。会议的引言来自[肖恩·亨利](http://2006.sxsw.com/interactive/programming/panels/?action=bio&id=104282) (W3C，[网络无障碍倡议](https://www.w3.org/WAI/))，他在会议开始时解释了什么是 [WCAG 1.0](https://www.w3.org/TR/WAI-WEBCONTENT/) 、 [WCAG 2.0](https://www.w3.org/TR/WCAG20/) 以及其他各种不太为人所知的 W3C 文档。她在引用之前说:“不要断章取义”。继续引用这句话:“不要读 WCAG 2.0”。在这一点上有一段时间的怀疑，但她继续解释说，人们应该阅读的是“T10 理解 WCAG 2.0”文件，因为 WCAG 2.0 指南本身非常难以理解，主要是因为他们声明的目标是使他们成为技术不可知者。因此，长期过度复杂和非特定于 web 的术语(Shawn 给出了几个典型的冗长的例子作为参考)。

Shawn 传达的关键信息是，尽管有了新的指导方针，可访问性的基本问题仍然是相同的，即图像的`alt`属性，使用正确的链接短语等等。主要的区别在于新的指导方针是可测试的，或者应该是可测试的。[德里克·费瑟斯通](http://2006.sxsw.com/interactive/programming/panels/?action=bio&id=81092)提醒观众使用现成的 [JavaScript 和/或 AJAX 库](http://www.google.com/search?q=javascript+libraries+frameworks)的危险，所有这些似乎都以其当前格式存在可访问性问题。如果你想确保一个干净的、可访问的最终结果，你真的必须“自己动手”(或者把库改造成某种形状)。最后，Shawn 说我们需要做的是“让可访问性变得酷”。祝你好运！不过，说真的，她说的是我们仍然没有足够多的外观漂亮、完全可访问且功能丰富的网站的例子。这仍然是少数。

所以，今晚是一年一度的 [Web Awards](http://2006.sxsw.com/interactive/web_awards/) 决赛，正如肖恩在台上所说，有多少类别的获胜者将是完全无障碍的网站？我想我一只手就能数出来。但是我们明天会看到更多
