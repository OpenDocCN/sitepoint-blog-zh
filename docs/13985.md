# 成熟的开发和管理工具

> 原文:[https://www . site point . com/maturing-development-and-management-tools/](https://www.sitepoint.com/maturing-development-and-management-tools/)

在过去的几年里，越来越多的开源主流应用引发了许多关于商业模式的严肃讨论。特别是，在坚持开源理念的同时，什么将有助于创建成功的公司和真正的金钱。

在过去的五年中，反对严肃对待开源的争论已经基本平息，因为应用程序已经在世界上一些最大的环境(科学、零售、政府等)中的生产中得到证明..).

Open Sourcery 有几条线索探索了商业软件和开放源码的交集，到目前为止，大多数专业人员都认为可以通过支持和在这个十字路口工作来获利。

这也是一个突出商业产品的空间，这些商业产品是为开源应用或栈(像 Apache/PHP/MySQL 这样的应用捆绑包)而构建的。来自 Computerworld 的一篇[及时的文章(与 eWeek 一起是一个常见的来源，经常描述开放源代码的增长和高性能功能，尤其是数据库)显示了该领域中最明显的数据库的进一步渗透和接受——包括 MySQL 和 PostgreSQL。](http://www.computerworld.com/databasetopics/data/software/story/0,10801,104278,00.html)

然而，似乎是巧合的是，两种用于这些流行的数据库解决方案的新管理工具也出现了。AwareSoft 的 AwareIM 试图弥合开源数据库之间的巨大鸿沟(包括 IBM 最近推出的开源 DB Derby)。

该产品提供了一个丰富的 GUI 开发环境，试图隐藏几乎所有的编程和所有的数据库设计元素。代替这些构建 web 应用程序的传统步骤，该公司提供了一个屏幕向导来定义业务对象、属性和定义业务流程如何工作的规则。在后台，AwareIM 研究出表格结构、执行已定义逻辑的代码，并设计出一个友好的界面，可以进一步定制。

也许 AwareIM 最有趣的方面是除了开发人员之外，精明的商业用户也可以使用它。有三种情况:

*   开发人员将使用它来为中小型公司节省开发 web 应用程序的时间
*   开发人员将使用它，然后将其部署到更大的企业解决方案中，并在维护方面培训成熟的业务用户
*   玩弄开源并习惯于 GUI 开发的公司(想想 Visual Studio 和 MS SQL Server——aware im 实际上支持后者)将使用它来探索开源

Navicat 于去年年底向开源读者[推出，它也围绕 MySQL 的成功建立了自己的业务，并提升了其管理需求。今年夏天最新发布的 Navicat 2005 反映了产品的日益成熟。](https://www.sitepoint.com/taming-mysql-administration/)

Navicat 能够在 Linux、Mac OS X 和 Windows 上运行，它听取了用户的意见，推出了几个新功能。特别是，作为 Navicat 的用户，我喜欢新的 SSH 隧道功能，以及适用于我的 OS X 平台的视图生成器、SQL 控制台和新的同步工具。

虽然 Linux 和 Mac 用户的管理客户端非常丰富，但其 Windows 客户端迄今为止提供了最高级的功能，包括作为迁移的一部分导入 Access 和 Excel 文件，以及可视化报告生成器。

Navicat 网站上有一个涵盖所有平台的[完整特性矩阵](http://www.navicat.com/feature.html)。
