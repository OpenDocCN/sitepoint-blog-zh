“汉堡图标”。我们大多数人都很熟悉它，可能已经看过无数次了，甚至在我们的移动设备上点击过不止一次。

你可能知道它的另一个名字。“*三线菜单*”、“*抽屉*”、“*菜单图标*”、“*侧扫*”、“*三条纹*或通用的“*导航图标*”只是与该符号相关的几个名称。

你甚至可能知道它是道教宇宙学中的“天/天”三元图。

不管你给这个常用的图标取什么名字，它肯定是许多争论的主题。最近，Annarita Tranfici 从 UX 的角度看待这个问题。今天我想从纯平面设计的角度来看一下。

我不想只关注设计和包含是违反直觉的还是直觉的，我想知道我们是否能找到更好的东西。不仅如此，还要弄清楚设计的改变是否有效，或者是否应该检查和开发一个完全不同的替代方案。

## 制作“汉堡”图标

在讨论汉堡包图标的相对优点之前，了解它的起源可能是有用的。对许多人来说，这似乎是一夜之间突然出现的感觉。今天，访问移动网站时，很难不看到右上角那个熟悉的三行谜。

研究通常会提供一系列起源故事。

有人说苹果早在 2007 年就开始了(iPhone)。其他人指向谷歌 Chrome，而其他人仍然会发誓它始于脸书和 Path 等应用程序。

到目前为止，我几乎没有看到任何确凿的证据来支持这些说法，尽管我能找到的第一次提到“汉堡图标”这个词是在 2011 年底的一次 Chrome 讨论中。但是，虽然人们不应该仅仅因为缺乏文档就太快放弃这些东西，但我们可以说图标的图形形式并不是来自上述任何一家公司。

汉堡包图标图形似乎比我们想象的要古老得多。据任何人所知，早在 1981 年，我们第一次看到这个图标是在施乐之星。看看这个。

![Xerox Star](../Images/82b667283395ddcb19b7b9126d469d0d.png)

没错，我们有一个 30 多岁的偶像。

虽然看起来我们确实应该归功于施乐公司是第一家使用这个图标的公司，但我们必须把这归功于界面系统的设计者 T2·诺姆·考克斯。

尽管有些人认为这个图标充当了菜单图标，但应该指出的是，当 Cox 创建这个符号时，它是作为上下文菜单选项的“容器”。

他的目标是不仅要创造一个实用的、令人难忘的“路标”，还要让设计模仿一个菜单列表。

![xerox-star2](../Images/07c11377e3a710627243aba86ac52a9e.png)

这一发现的所有荣誉归于杰夫·奥尔德伊。多亏了他的询问和[这封电子邮件](https://www.evernote.com/shard/s207/sh/022f2237-4b4f-4096-87f2-053acd228c2d/ede2672bc3f39a1b0232f84e01ca0a83)的交流，三线图标背后的秘密才得以解开。

谁首先把它称为“汉堡”图标的问题现在仍然是个谜，但是如果你有一个理论，不要犹豫，分享一下。

## 大辩论

[![Two hamburger icons?](../Images/676042a4db774b0dc74df36eeaed6bea.png)](https://www.flickr.com/photos/mzmo/3314577855/)

图片:KM 历险记& G-Morris

既然我们现在知道了是谁设计了 navicon，那么是时候来讨论一下这个图标在设计界的现状了。早些时候我注意到这个图标已经成为最近讨论和辩论的话题，这是真的，的确如此。

我承认我不是一个关心时事的人，但在过去的一周里，我无意中发现了许多似乎反对“汉堡包”图标的文章。另一个承认，我甚至不知道人们是这样称呼它的，我实际上点击了这些文章，因为我想知道它是什么。不管怎样，很明显有不同的团体在游说支持和反对这个偶像。

人们在说什么？

总之，很多。你会发现一些文章和论坛说“汉堡包”图标是一个问题，因为它对用户隐藏了内容。据说这是非 UX 友好的，因为它可能会让用户忘记执行他们通常会将内容放在他们面前而不是隐藏的操作。

另一个问题是，如果你不熟悉手机，你甚至不会意识到这三行是一个菜单。这有点类似于“年轻一代不会明白软盘图标意味着保存”的论点。这里唯一的不同是，人们把它换了过来，说“老一代人不会认出汉堡包图标是什么”。

![A platter of hamburgers](../Images/89ba8c2452e3ac553dbfd1cd9174dab4.png)

开发者、设计师和 UX 爱好者提出了许多其他的论点，包括“三次点击”规则，以及与其说是图标本身，不如说是它的技术。

## 可能的视觉解决方案

撇开技术和规则不谈，至少现在，让我们谈谈可能的视觉解决方案。改变设计会有帮助吗？事实是，我们真的不知道，除非我们尝试。这里有一些已经提出或已经实施的想法。

### 移动优先方法

虽然从改变图标本身的意义上来说，这并不是一个确切的视觉解决方案，但作者 Paddi MacDonnell 在文章[的后半部分建议如何解决汉堡包图标问题](http://www.webdesignerdepot.com/2014/06/how-to-solve-the-hamburger-icon-problem/)，移动开发者应该将他们的网站和应用程序简化到最简单的形式。这种策略坚持认为，通过减少功能，你就减少了选项的数量，因此不再需要“汉堡包”图标

虽然这似乎是一个很好的田园诗般的方式，但我不太确定用户和开发人员会不会对仅仅为了显示通常隐藏的内容和选项而减少内容和功能感到兴奋。

请记住，移动设备的伟大之处在于，我们可以在旅途中从我们的设备上访问相同的信息，就像我们坐在电脑前一样。

### 文本方法

[![Ratchet vine template](../Images/0a5d447421b5bc636b4e932c0245c2d6.png)T2】](https://github.com/jakiestfu/Ratchet-Vine)

WordPress、网页开发和设计专家兼教育家 Morten Rand-Hendriksen 建议，我们不要走图标路线，而是用文本代替“汉堡包”。在这种情况下，图标将被替换为单词“菜单”。他并没有对“汉堡”图标过分苛刻，但是他在他的文章中提出了一些有趣的观点。他还建议了一些其他方法来对抗你对“汉堡包”的渴望。

对于一些人来说，将图像切换为文本的想法似乎是一个奇怪的选择，特别是如果你试图简化的话，但对于不熟悉它的人来说，它确实消除了三行的混淆。

### 网格方法

[![iPhone UI template offering the grid and the hamburger.](../Images/2df7b8425c311fd142d6ac9fd9f59eaa.png)](https://dribbble.com/shots/666574-iphone-ui-PSD-freebie)

维吉尔·帕纳的 iPhone 用户界面

到目前为止，我只见过谷歌在生产中使用它，而且这只是在你使用台式机或笔记本电脑的情况下。如果您在使用移动设备时访问任何谷歌附属页面，您仍会看到标准的三线图标。然而，台式机或笔记本电脑用户看到的是由九个小方块组成的网格。

有些人可能会认为这仍然隐藏了菜单，但我们不要忘记这个挑战是关于找到一个改进的视觉解决方案。我还没有看到任何关于使用网格代替三条线的讨论，所以这可能是一个可行的替代方案。

### 按钮方法

![Buttonized hamburger](../Images/56bf0f32a3f42d5890e8f7ec3133e24b.png)

按钮方法对标准导航设计进行了调整。图标呈现按钮化状态，以加强交互性。您通常会看到导航图标覆盖在标题或类似内容上。

因为设计不是平面的，它暗示了更多的交互性。使用按钮方法，您可以在图标周围创建一个边框，或者将它设置在块内部。一些网站已经实现了这一点。

Exis web 用这种方法进行了一次测试，虽然测试仍然表明汉堡包图标不是一个理想的解决方案，但“按钮方法”比其他两个选项被点击的次数更多。

你可以在他们的手机菜单 AB 测试文章中查看这个测试。
