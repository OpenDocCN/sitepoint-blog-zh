# 驱逐浏览器地址栏——用户界面走得太远了？

> 原文：<https://www.sitepoint.com/banishing-browser-address-bars/>

谷歌希望 Chrome 成为一个干净的、不受干扰的浏览体验。他们可能会采取最激进的措施。随着地址栏的移除，界面极简主义将达到它的顶点。

疯狂？

也许吧。但是 Mozilla 也在考虑同样的 UI 举措。

这个想法从技术用户那里得到了压倒性的负面反应。然而，在你准备演讲之前，要知道这只是一个可能永远不会公开的提议。如果真的发生了，这几乎肯定是一个选项，而且“紧凑视图”可能只允许在应用程序选项卡上使用。启用后，用户可能需要双击选项卡才能查看 URL。

那么，为什么谷歌认为 30 像素的增益如此重要呢？它将在一些平板电脑和上网本屏幕上提供额外的 5%的空间，但还有更深层次的原因…

我使用地址栏。你可能也在用它。但是很多用户没有。非技术用户很少理解 URLs 当你看到他们在谷歌的搜索框中输入*www.whatever.com*时，这是显而易见的。那么，为什么要保留一个很少有人使用的功能呢？

我们也应该考虑网络使用是如何变化的。我们知道浏览器是一个独立的应用程序，但随着操作系统供应商尝试一种更加集成的方法，它很可能会发展。图标、应用程序标签和固定站点只是一个开始。在线和离线的区别已经变得模糊，几年之内，用户将不会知道或关心应用程序驻留在哪里。

网络营销也有明显的转变。虽然公司仍然在广告媒体上推广他们的网址，但许多公司现在会发布更容易记住的谷歌或脸书搜索关键词。

最后，还有商业激励。没有酒吧，用户必须求助于搜索引擎；在到达目的地之前，他们总会看到一页搜索结果和付费广告。

但是弊端呢？如果看不到地址栏，输入一个网址就比较费劲了。如果用户真的不想要工具栏，它通常可以被隐藏，或者他们可以切换到全屏模式(大多数浏览器中的 F11)。

Web 开发人员也依赖于 URL——尤其是在测试 web 应用程序或 REST 服务时。取消酒吧会让我们的生活更加艰难。

最后，没有地址栏，就更难确保你在正确的网站上或检查安全设置。那些参与网络钓鱼诈骗的人将会热切期待 UI 的改变。

这个想法让我很不舒服。用户可能不明白网址，但删除栏不会帮助他们学习。我肯定许多汽车司机不理解液压，但这不是卸下刹车的理由*(好吧——糟糕的比喻，但没有 URL 的网络并非没有危险)。*

我完全支持 UI 简化，但这似乎有点过了。如果发生这种情况，谷歌应该给他们的浏览器重新命名:“Chrome-less”会更贴切。

你怎么想呢?地址栏该不该去？这是一种选择吗？风险太大了吗？
