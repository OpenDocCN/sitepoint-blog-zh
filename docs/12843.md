# Glam Media 推出应用平台

> 原文：<https://www.sitepoint.com/glam-media-launches-app-platform/>

![](img/7c985f6b9c6ad7a3a0620cbd06bd1829.png "glamplatform-logo")博客网 [Glam Media](http://www.glammedia.com/) 今天宣布推出其[应用平台](http://developer.glam.com/)。Glam 称该平台代号为“Atako”，是他们“技术热情的新词”，该平台使用谷歌小工具和 OpenSocial 标准(谷歌的一大胜利，因为脸书的平台也是开源的)，以及其他新兴的网络标准，如 OpenID，将允许应用程序开发人员接触 Glam 的 7700 万每月独立用户。

Glam 的博客网络包含超过 600 家不同规模的出版商，应用开发者将把目标锁定在拥有应用的出版商，而不是用户。这里的游戏名称是为博客出版商创造价值，然后通过向用户展示的广告将应用程序货币化(毕竟，Glam 是一个广告网络，计划在应用程序上销售广告，并与开发者分享收入)。这有点非正统，但肯定可行。Glam 庞大的每月独立用户数量与许多顶级社交网络不相上下，让出版商控制每个博客上使用的应用程序，将几乎消除脸书等社交网络开发平台面临的应用程序过载和垃圾邮件问题。

“对于出版商来说，Glam Apps 提供了一种添加新特性、功能和内容的方式；生成更多页面浏览量；建立更忠诚的观众；并获得更多收入，”Glam 在一份新闻稿中说。该公司还概述了他们的出版商想要的应用类型；像投票应用程序、照片库、幻灯片、电子商务小工具和品牌视频播放器这样的东西名列前茅。

Glam 将在应用商店中提供应用程序，允许应用程序开发者向出版商收取使用其应用程序的费用——尽管潜在客户群约有 600 人，其中大多数是小型出版商，付费应用程序的增长空间并不大。我们预计，大多数应用程序都将获得广告支持，只有少数应用程序会绑定外部订阅服务。

Glam 的应用平台是博客网络的一种独特方式，但该公司似乎有更宏伟的目标。开发平台的 [FAQ 页面](http://wiki.developer.glam.com/wiki/FrequentlyAskedQuestions)暗示了创建一个全网络魅力社交网络的可能性。“Glam 平台是社交网络吗？”问常见问题。“这次不行。Glam 平台的初始版本只支持匿名用户。Glam 应用程序知道发布者的资料，但不知道访问者的资料。我们计划在未来的版本中启用访问者档案和共享功能。”

这里的一个潜在障碍是，因为 Glam 的博客网络是通过收购独立博客或与独立博客合作而逐渐组装起来的，网络中的属性使用许多不同的博客平台。有的用 WordPress，有的用 Blogger，有的用 MoveableType 等。Glam 计划如何让应用程序在如此多的不同平台上运行还不清楚。(FAQ 提到了这个问题，但没有具体说明公司计划如何解决这个问题。)

该平台预计将于 2008 年第四季度推出，Joyent 将向应用开发者提供一年的免费托管服务。
