# 微软在 Sourceforge 上

> 原文：<https://www.sitepoint.com/microsoft-on-sourceforge/>

四月初，微软悄悄地在 Sourceforge 上发布了它的第一个开源项目 WIX[。WIX，根据项目页面，“Windows Installer XML (WiX)是一个从 XML 源代码构建 Windows 安装包的工具集。该工具集支持命令行环境，开发人员可以将该环境集成到他们的构建过程中，以构建 MSI 和 MSM 安装包。](http://sourceforge.net/projects/wix/)

虽然这不是一个巨大的源代码发布，但这是一个有趣的步骤，因为微软可能开始探索开源社区。项目负责人的博客上有一些有趣的博客条目，[在这里找到](http://blogs.msdn.com/robmen/)。该项目是在未经修改的[公共许可证](http://www.opensource.org/licenses/cpl.php)下发布的。

对于使用。Net 平台，因为微软自己的内部团队，包括 msn.com 分部，正在使用这个工具集来构建 web 应用程序的安装包。

更有趣的是这种对开源的尝试，尤其是 Sourceforge 的使用，它被认为是主要的开源开发门户。也许是与经验丰富的开源开发者社区合作的新体验，尤其是那些对看到更多为 Windows 构建或移植到 Windows 的开源解决方案感兴趣的人(反之亦然！想想吧。参见 [Mono 项目](http://www.go-mono.com/)。

该小组对贡献代码的控制受到了一些批评，但是，我曾经参与过面向商业消费的开源项目，对源代码控制有非常严格的条件，因此对项目实现修改后的代码所给予的关注应该不会令人震惊。
