# 如何同时运行 Apache 和 IIS:第 1 部分

> 原文：<https://www.sitepoint.com/run-apache-iis-same-time-1/>

无论你是自由职业者还是雇员，通常都需要使用各种平台和应用程序来完成工作。如果你是一个 PHP 开发人员，你可能会涉及到微软 SQL Server 和 Windows 主机。如果你是. NET 开发人员，你可能会被要求编写一个基于 Apache 的 URL 重写系统。不要害怕:拥抱工具并从经验中学习！

当然，这并不总是那么容易。最好使用与您的托管平台类似的开发环境，因此您通常需要 Apache HTTP Server 或 Microsoft Internet Information Server/Services(IIS)。但是如果你两者都需要呢？

## 您需要两台服务器吗？

从开发的角度来看，IIS 和 Apache 做的是同样的事情。如果你能对你的网络主机使用类似的设置，那当然很好，但这并不是绝对必要的。

如果您的要求相对适中，我建议只运行 IIS。它会很高兴地运行 ASP.NET，经典的 ASP 和 PHP 在同一个安装。

阿帕奇不(直接)支持 ASP.NET。你可以使用 Mono，开源的等价物，我怀疑一些聪明的黑客已经找到了在 Apache 上运行微软 ASP.NET 的方法。但这值得努力吗？您几乎肯定会遇到问题，尤其是在使用 Visual Studio 时。

如果您只想在 Windows PC 上维护一台服务器，IIS 可能是最佳解决方案。

## 在开发农场

如果您有两台或更多的机器，那么直接在一台 PC 上安装 Apache，在另一台 PC 上安装 IIS。可以配置共享文件夹，以便在远程服务器上部署和运行文件。

如果您在团队中工作，这是一个很好的解决方案，并且允许实施备份设施。但是，对一个人来说就大材小用了，肯定会增加你的电费。

## 有价值的虚拟机

VM 软件允许你在你的电脑里创建一个软件模拟的电脑。我是 VirtualBox 的忠实粉丝，尽管微软虚拟个人电脑和 VMware 各有千秋。

虚拟机的优势在于，您可以设置任意数量的服务器配置，并切换到最合适的配置。开发人员仍然可以访问共享文件夹，备份系统将像在真实服务器上一样工作。

但是，虚拟机增加了额外的工作量，必须进行配置和维护。在单台 PC 上，无论何时开始编码，您都需要启动虚拟机。这可能只需要一两分钟，但很快就会变得令人厌倦。最后，虚拟机只是数据块，很容易备份，但也很脆弱。操作系统崩溃、损坏或文件删除可能会损坏您的虚拟机并破坏您的工作。
